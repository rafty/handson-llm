Gmail

Gmail adalah klien email yang dikembangkan oleh Google di mana Anda dapat mengirim pesan email dengan lampiran file. Pesan Gmail dapat diurutkan dan disimpan di dalam kotak masuk email Anda menggunakan folder dan label. Anda dapat menggunakanAmazon Kendra untuk mengindeks pesan e-mail dan lampiran pesan. Anda juga dapat mengonfigurasiAmazon Kendra untuk menyertakan atau mengecualikan pesan e-mail tertentu, lampiran pesan, dan label untuk pengindeksan.

Anda dapat terhubungAmazon Kendra ke sumber data Gmail Anda menggunakan Amazon Kendrakonsol dan TemplateConfigurationAPI.

Untuk mengatasi masalah konektor sumber dataAmazon Kendra Gmail Anda, lihatMengatasi masalah sumber data.

Prasyarat

Sebelum dapat digunakanAmazon Kendra untuk mengindeks sumber data Gmail Anda, buat perubahan ini di Gmail danAWS akun Anda.

Di Gmail, pastikan Anda memiliki:

  • Membuat akun admin Google Cloud Platform dan telah membuat proyek Google Cloud.

  • Mengaktifkan Gmail API dan Admin SDK API di akun admin Anda.

  • Membuat akun layanan dan mengunduh kunci pribadi JSON untuk Gmail Anda. Untuk informasi tentang cara membuat dan mengakses kunci pribadi Anda, lihat dokumentasi Google Cloud tentang cara Membuat kunci akun layanan dan kredensi akun Layanan.

  • Menyalin email akun admin Anda, email akun layanan Anda, dan kunci pribadi Anda untuk digunakan untuk otentikasi.

  • Menambahkan cakupan Oauth berikut (menggunakan peran admin) untuk pengguna Anda dan direktori bersama yang ingin Anda indeks:

    • https://www.googleapis.com/auth/admin.directory.user.readonly

    • https://www.googleapis.com/auth/gmail.readonly

  • Memeriksa setiap dokumen unik di Gmail dan di seluruh sumber data lain yang akan Anda gunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensyal autentikasi Gmail AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat menghubungkan sumber data Gmail keAmazon Kendra. Jika Anda menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia yang ada, dan ID indeks.

Petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber data Gmail Anda, Anda harus memberikan detail kredensyal Gmail Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum mengkonfigurasi Gmail untukAmazon Kendra, lihatPrasyarat.

Console

Untuk terhubungAmazon Kendra ke Gmail

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di halaman Tambahkan sumber data, pilih Konektor Gmail, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan detail sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan nilai deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Pada halaman Tetapkan akses dan keamanan, masukkan informasi berikut:

    1. Di Otentikasi untuk AWSSecrets Managerrahasia —Pilih rahasia yang ada atau buatSecrets Manager rahasia baru untuk menyimpan kredensyal otentikasi Gmail Anda. Jika Anda memilih untuk membuat rahasia baru jendelaAWSSecrets Manager rahasia terbuka.

      1. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama Rahasia —Sebuah nama untuk rahasia Anda.

        2. Email klien —Email klien yang Anda salin dari akun layanan Google Anda.

        3. Email akun admin —Email akun admin yang ingin Anda gunakan.

        4. Kunci pribadi —Kunci pribadi yang Anda salin dari akun layanan Google Anda.

        5. Pilih Save (Simpan).

    2. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    3. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensyal repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    4. Pilih Selanjutnya.

  7. Pada halaman Konfigurasi pengaturan sinkronisasi, masukkan informasi berikut:

    1. Dalam lingkup Sinkronisasi, untuk Jenis entitas —Pilih Lampiran pesan untuk menyinkronkan lampiran pesan. Pesan akan disinkronkan secara default.

    2. (Opsional) Untuk konfigurasi tambahan, masukkan informasi berikut:

      1. Rentang tanggal —Masukkan rentang tanggal untuk menentukan tanggal mulai dan akhir email yang akan dirayapi.

      2. Domain email —Sertakan atau kecualikan email berdasarkan domain.

      3. Kata kunci dalam subjek —Sertakan atau kecualikan email berdasarkan kata kunci dalam subjeknya.

        catatan

        Anda juga dapat memilih untuk menyertakan dokumen apa pun yang cocok dengan semua kata kunci subjek yang telah Anda masukkan.

      4. Label —Tambahkan pola ekspresi reguler untuk menyertakan atau mengecualikan label tertentu. Anda dapat menambahkan hingga 100 pola.

      5. Lampiran —Tambahkan pola ekspresi reguler untuk menyertakan atau mengecualikan lampiran tertentu. Anda dapat menambahkan hingga 100 pola.

    3. Untuk mode Sinkronisasi, pilih cara memperbarui indeks saat konten sumber data berubah. Saat Anda menyinkronkan sumber dataAmazon Kendra untuk pertama kalinya, semua konten disinkronkan secara default.

      • Sinkronisasi penuh —Sinkronkan semua konten terlepas dari status sinkronisasi sebelumnya.

      • Sinkronisasi konten baru, dimodifikasi, atau dihapus —Hanya sinkronkan konten baru, dimodifikasi, dan dihapus.

        penting

        Karena tidak ada API untuk memperbarui pesan Gmail yang dihapus secara permanen, sinkronisasi konten Baru, dimodifikasi, atau dihapus:

        • Tidak akan menghapus pesan yang dihapus secara permanen dari Gmail dariAmazon Kendra indeks Anda

        • Tidak akan menyinkronkan perubahan pada label email Gmail

        Untuk menyinkronkan perubahan label sumber data Gmail dan pesan email yang dihapus secara permanen keAmazon Kendra indeks, Anda harus menjalankan crawl penuh secara berkala.

    4. Di Sinkronisasi jadwal jalankan, untuk Frekuensi —Seberapa seringAmazon Kendra akan disinkronkan dengan sumber data Anda.

    5. Pilih Selanjutnya.

  8. Pada halaman Set field pemetaan, masukkan informasi berikut:

    1. Untuk lampiran Pesan dan Pesan —Pilih dari bidang sumber data defaultAmazon Kendra yang dihasilkan yang ingin Anda petakan ke indeks Anda.

      catatan

      Amazon KendraKonektor sumber data Gmail tidak mendukung pembuatan kolom indeks kustom karena keterbatasan API.

    2. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

TemplateConfiguration API

Untuk terhubungAmazon Kendra ke Gmail

Anda harus menentukan yang berikut menggunakan TemplateConfigurationAPI:

  • Sumber data —Anda harus menentukan sumber data sebagaiGMAIL.

  • Skema sumber data —Sertakan JSON yang berisi skema sumber data. Untuk melihat skema template, lihat Skema sumber data.

  • KetikTEMPLATE —Specify sebagai Type saat Anda meneleponCreateDataSource.

  • Rahasia Amazon Resource Name (ARN) —Menyediakan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensyal otentikasi untuk akun Gmail Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "adminAccountEmailId": "${adminAccountEmailId}",
    "clientEmailId": "${clientEmailId}",
    "privateKey": "${privateKey}"
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk konektor Gmail danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber data Gmail.

Anda juga dapat menambahkan fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration saat Anda meneleponCreateDataSource. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

  • Mode sinkronisasi —Tentukan apakahAmazon Kendra harus memperbarui indeks Anda dengan menyinkronkan semua dokumen atau hanya dokumen baru, dimodifikasi, dan dihapus. Anda dapat memilih antara:

    • FORCED_FULL_CRAWLuntuk merayapi ulang semua konten dan mengganti konten yang ada setiap kali sumber data Anda disinkronkan dengan indeks

    • FULL_CRAWLmerangkak secara bertahap hanya konten baru, dimodifikasi, dan dihapus setiap kali sumber data Anda disinkronkan dengan indeks Anda

    penting

    Karena tidak ada API untuk memperbarui pesan Gmail yang dihapus secara permanen, sinkronisasi kontenFULL_CRAWL/Baru, dimodifikasi, atau dihapus:

    • Tidak akan menghapus pesan yang dihapus secara permanen dari Gmail dariAmazon Kendra indeks Anda

    • Tidak akan menyinkronkan perubahan pada label email Gmail

    Untuk menyinkronkan perubahan label sumber data Gmail dan pesan email yang dihapus secara permanen keAmazon Kendra indeks, Anda harus menjalankan crawl penuh secara berkala.

  • Filter inklusi dan pengecualian —Anda dapat menentukan apakah akan menyertakan lampiran. Anda juga dapat menetapkan pola ekspresi reguler untuk menyertakan atau mengecualikan pesan dan lampiran tertentu.

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak akan diindeks meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data Gmail Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

    catatan

    Amazon KendraKonektor sumber data Gmail tidak mendukung pembuatan kolom indeks kustom karena keterbatasan API.

Catatan

  • Karena tidak ada API untuk memperbarui pesan Gmail yang dihapus secara permanen, sinkronisasi kontenFULL_CRAWL/Baru, dimodifikasi, atau dihapus:

    • Tidak akan menghapus pesan yang dihapus secara permanen dari Gmail dariAmazon Kendra indeks Anda

    • Tidak akan menyinkronkan perubahan pada label email Gmail

    Untuk menyinkronkan perubahan label sumber data Gmail dan pesan email yang dihapus secara permanen keAmazon Kendra indeks, Anda harus menjalankan crawl penuh secara berkala.

  • Amazon KendraKonektor sumber data Gmail tidak mendukung pembuatan kolom indeks kustom karena keterbatasan API.
