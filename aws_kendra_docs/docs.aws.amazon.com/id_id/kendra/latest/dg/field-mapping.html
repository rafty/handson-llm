Memetakan bidang sumber data

Anda dapat memetakan bidang dokumen atau konten dari sumber data Anda ke bidang dalam indeks Anda. Misalnya, jika memiliki bidang dalam sumber data bernama “dept” yang berisi informasi departemen untuk dokumen, Anda dapat memetakannya ke bidang indeks bernama “Department”. Dengan begitu, Anda dapat menggunakan bidang saat menanyakan dokumen.

Anda juga dapat memetakan bidang standar atau cadangan seperti_created_at. Jika sumber data Anda memiliki bidang yang disebut “creation_date”, Anda dapat memetakan ini ke bidangAmazon Kendra cadangan setara yang disebut_created_at. Anda dapat memetakan bidang untuk sebagian besar sumber data.

Jika menyimpan dokumen dalam bucket S3, atau sumber data S3, Anda dapat memberikan atribut kustom langsung menggunakan file metadata. Untuk informasi selengkapnya, lihat Membuat bidang dokumen kustom.

Memetakan bidang sumber data ke bidang indeks memerlukan tiga langkah:

  1. Buat indeks. Untuk informasi selengkapnya, lihat Membuat indeks.

  2. Perbarui indeks untuk menambahkan bidang kustom.

  3. Buat sumber data yang memetakan bidang sumber data ke bidang indeks.

Untuk memperbarui indeks guna menambahkan bidang kustom, gunakan konsol tersebut atauUpdateIndex API. Anda dapat menambahkan total 500 bidang kustom ke indeks Anda.

Di konsol tersebut, Anda dapat memilih untuk memetakan bidang sumber data ke salah satu nama bidang terpesan. Atau, Anda dapat memilih untuk membuat bidang indeks baru yang memetakan ke lapangan. Untuk sumber data basis data, jika nama kolom basis data cocok dengan nama bidang terpesan, bidang dan kolom akan otomatis dipetakan.

Dengan UpdateIndexAPI tersebut, tambahkan bidang kustom dan terpesan menggunakanDocumentMetadataConfigurationUpdates parameter.

Contoh JSON berikut menggunakanDocumentMetadataConfigurationUpdates untuk menambahkan bidang bernama “Department” ke indeks.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Saat membuat bidang, Anda memiliki pilihan untuk menetapkan bagaimana bidang digunakan dalam pencarian. Anda dapat memilih dari opsi berikut:

  • Dapat ditampilkan —Menentukan apakah bidang dikembalikan dalam respons kueri. Defaultnya adalah true.

  • Dapat dibuat faset —Menunjukkan bahwa bidang dapat digunakan untuk membuat faset. Defaultnya adalah false.

  • Dapat dicari —Menentukan apakah bidang digunakan dalam pencarian. Secara default adalah true untuk bidang string dan false untuk bidang nomor dan tanggal.

  • Dapat diurutkan — Menunjukkan bahwa bidang dapat digunakan untuk mengurutkan respons dari kueri. Hanya dapat diatur untuk bidang tanggal, angka, dan string. Tidak dapat diatur untuk bidang daftar string.

Contoh JSON berikut menggunakanDocumentMetadataConfigurationUpdates untuk menambahkan bidang bernama “Department” ke indeks dan menandainya sebagai dapat dibuat faset.

"DocumentMetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE",
       "Search": {
           "Facetable": true
       }
   }
]

Menggunakan bidang dokumen bawaan Amazon Kendra

Dengan UpdateIndex API, Anda dapat membuat bidang cadangan atau built-in menggunakanDocumentMetadataConfigurationUpdates dan menentukan nama bidang cadangan untuk memetakan ke atribut dokumen atau bidang dokumen yang setara. Anda juga dapat membuat bidang kustom dengan cara ini. Jika Anda menggunakan konektor sumber data, sebagian besar menyertakan pemetaan bidang yang memetakan bidang dokumen sumber data Anda ke bidangAmazon Kendra indeks. Ketika Anda membuat bidang, Anda dapat mengkonfigurasiSearch objek untuk mengatur bidang sebagai ditampilkan, facetable, dicari, dan diurutkan. Anda dapat mengkonfigurasiRelevance objek untuk mengatur relevansi bidang meningkatkan durasi, kesegaran, kepentingan, urutan peringkat, dan nilai-nilai kepentingan. Anda tidak dapat mengubah jenis bidang setelah Anda membuat bidang.

Amazon Kendramemiliki bidang dokumen cadangan atau built-in berikut yang dapat Anda gunakan:

  • _authors—Daftar satu atau beberapa penulis yang bertanggung jawab atas isi dokumen.

  • _category—Kategori yang menempatkan dokumen dalam grup tertentu.

  • _created_at— Tanggal dan waktu dalam format ISO 8601 di mana dokumen dibuat. Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012 pukul 12:30 PM (ditambah 10 detik) di Waktu Eropa Tengah.

  • _data_source_id—Pengenal sumber data yang mengandungi dokumen.

  • _document_body—Isi dokumen.

  • _document_id—Pengenal unik untuk dokumen.

  • _document_title—Judul dokumen.

  • _excerpt_page_number—Nomor halaman dalam file PDF tempat kutipan dokumen muncul. Jika indeks Anda dibuat sebelum 8 September 2020, Anda harus mengindeks ulang dokumen sebelum dapat menggunakan atribut ini.

  • _faq_id—Jika ini adalah pertanyaan dan jawaban FAQ, pengenal unik untuk mereka.

  • _file_type—Jenis file dokumen, seperti pdf atau doc.

  • _last_updated_at— Tanggal dan waktu dalam format ISO 8601 ketika dokumen terakhir diperbarui. Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012 pukul 12:30 PM (ditambah 10 detik) di Waktu Eropa Tengah.

  • _source_uri—URI tempat dokumen tersedia. Misalnya, URI dokumen di situs web perusahaan.

  • _version—Pengenal bagi versi spesifik dokumen.

  • _view_count— Jumlah berapa kali dokumen dilihat.

  • _language_code(String) —Kode untuk bahasa yang berlaku untuk dokumen. Ini default-nya ke bahasa Inggris jika Anda tidak menentukan bahasa. Untuk informasi selengkapnya tentang bahasa yang didukung, termasuk kodenya, lihat Menambahkan dokumen dalam bahasa selain bahasa Inggris.

Untuk bidang kustom, Anda membuat bidang ini menggunakanDocumentMetadataConfigurationUpdates denganUpdateIndex API, seperti yang Anda lakukan saat membuat bidang cadangan atau built-in. Anda harus mengatur tipe data yang sesuai untuk bidang kustom Anda. Anda tidak dapat mengubah jenis bidang setelah Anda membuat bidang.

Berikut ini adalah jenis yang dapat Anda tetapkan untuk bidang khusus:

  • Tanggal

  • Jumlah

  • String

  • Daftar string

Anda membuat bidang kustom menggunakan konsol tersebut atau menggunakan UpdateIndexAPI. Setelah Anda membuat bidang kustom, Anda memetakannya ke atribut dokumen, seperti yang Anda lakukan dengan bidang tersimpan. Jika Anda menambahkan dokumen ke indeks dengan BatchPutDocumentAPI, Anda memetakan atribut dengan API. Untuk dokumen yang diindeks dari sumberAmazon S3 data, Anda memetakan atribut menggunakan file metadata yang berisi struktur JSON yang menggambarkan atribut dokumen. Untuk dokumen yang diindeks dengan database atau sumber data yang memungkinkan pemetaan lapangan, Anda memetakan atribut dengan konsol atau konfigurasi sumber data.
