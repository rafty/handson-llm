menyindir

Quip adalah perangkat lunak produktivitas kolaboratif yang menawarkan kemampuan penulisan dokumen waktu nyata. Anda dapat menggunakanAmazon Kendra untuk mengindeks folder Quip, file, komentar file, ruang obrolan, dan lampiran.

Anda dapat terhubungAmazon Kendra ke sumber data Quip Anda menggunakan Amazon Kendrakonsol dan QuipConfigurationAPI.

Untuk mengatasi masalah konektor sumber dataAmazon Kendra Quip Anda, lihatMengatasi masalah sumber data.

Fitur yang didukung

Amazon KendraKonektor sumber data menyindir mendukung fitur-fitur berikut:

  • Pemetaan lapangan

  • Pemfilteran konteks pengguna

  • Filter inklusi

  • Virtual Private Cloud (VPC)

Prasyarat

Sebelum Anda dapat menggunakanAmazon Kendra untuk mengindeks sumber data Quip Anda, buat perubahan ini di Quip danAWS akun Anda.

Dalam Quip, pastikan Anda memiliki:

  • Akun Quip dengan izin administratif.

  • Kredensyal otentikasi Quip yang dibuat yang menyertakan token akses pribadi. Lihat dokumentasi Quip tentang otentikasi untuk informasi selengkapnya.

  • Menyalin domain situs Quip Anda. Misalnya, https://quip-company.quipdomain.com/browse di mana quipdomain adalah domain.

  • Memeriksa setiap dokumen unik di Quip dan di seluruh sumber data lain yang Anda rencanakan untuk digunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensi autentikasi Quip AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat menghubungkan sumber data Quip keAmazon Kendra. Jika Anda menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia yang ada, dan ID indeks.

petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber data Quip Anda, Anda harus memberikan rincian yang diperlukan dari sumber data Quip Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum dikonfigurasi Quip untukAmazon Kendra melihatPrasyarat.

Console

Untuk terhubungAmazon Kendra ke Quip

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di halaman Tambahkan sumber data, pilih Konektor Quip, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan nilai deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Pada halaman Tetapkan akses dan keamanan, masukkan informasi berikut:

    1. Nama domain menyindir —Masukkan Quip yang Anda salin dari akun Quip Anda.

    2. AWSSecrets Managersecret —Pilih rahasia yang ada atau buatSecrets Manager rahasia baru untuk menyimpan kredensi otentikasi Quip Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      1. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-Quip-' secara otomatis ditambahkan ke nama rahasia Anda.

        2. Quip token —Masukkan token akses pribadi Quip yang Anda buat di akun Quip Anda.

      2. Pilih Save (Simpan).

    3. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    4. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensi repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    5. Pilih Selanjutnya.

  7. Pada halaman Konfigurasi pengaturan sinkronisasi, masukkan informasi berikut:

    1. Tambahkan ID folder Quip untuk di-crawl —ID folder Quip yang ingin Anda jelajahi.

    2. Konfigurasi tambahan (jenis konten) —Masukkan jenis konten yang ingin Anda jelajahi.

    3. Pola regex —Pola ekspresi reguler untuk menyertakan atau mengecualikan file tertentu. Anda dapat menambahkan hingga 100 pola.

    4. Di Sinkronisasi jadwal jalankan, untuk Frekuensi —Pilih seberapa sering Amazon Kendra akan disinkronkan dengan sumber data Anda.

    5. Pilih Selanjutnya.

  8. Pada halaman Set field pemetaan, masukkan informasi berikut:

    1. Pilih dari bidang sumber data default yang dihasilkan yang ingin Anda petakan keAmazon Kendra indeks.

    2. Tambahkan bidang —Untuk menambahkan bidang sumber data kustom untuk membuat nama bidang indeks untuk memetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk terhubungAmazon Kendra ke Quip

Anda harus menentukan yang berikut menggunakan QuipConfigurationAPI:

  • Quip domain situs —Misalnya, https://quip-company.quipdomain.com/browse di mana quipdomain adalah domain.

  • Rahasia Amazon Resource Name (ARN) —Berikan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensi otentikasi untuk akun Quip Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "accessToken": "token"
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk konektor Quip danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber data Quip.

Anda juga dapat menambahkan fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration sebagai bagian dari konfigurasi sumber data. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

  • Filter inklusi dan pengecualian —Tentukan apakah akan menyertakan atau mengecualikan file tertentu.

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak akan diindeks, meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data Quip Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

  • Pemfilteran konteks pengguna -Amazon Kendra merayapi Access Control List (ACL) untuk sumber data Anda secara default. Informasi ACL digunakan untuk memfilter hasil pencarian berdasarkan pengguna atau akses grup mereka ke dokumen. Untuk informasi selengkapnya, lihat Pemfilteran konteks pengguna untuk sumber data Quip.

Pelajari selengkapnya

Untuk mempelajari lebih lanjut tentang integrasiAmazon Kendra dengan sumber data Quip Anda, lihat:

  • Cari pengetahuan dalam dokumen Quip dengan pencarian cerdas menggunakan konektor Quip untukAmazon Kendra
