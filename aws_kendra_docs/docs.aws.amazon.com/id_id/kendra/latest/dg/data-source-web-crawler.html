Amazon KendraPerayap Web

Anda dapat menggunakanAmazon Kendra Web Crawler untuk merayapi dan mengindeks halaman web.

Anda hanya dapat menjelajahi situs web dan situs web yang menghadap publik yang menggunakan protokol komunikasi aman Hypertext Transfer Protocol Secure (HTTPS). Jika Anda menerima kesalahan saat merayapi situs web, bisa jadi situs web tersebut diblokir agar tidak merangkak. Untuk merayapi situs web internal, Anda dapat mengatur proxy web. Proxy web harus menghadap publik.

Saat memilih situs web untuk diindeks, Anda harus mematuhi Kebijakan Penggunaan yang Diterima Amazon dan semua syarat Amazon lainnya. Ingat bahwa Anda hanya boleh menggunakanAmazon Kendra web Anda sendiri, atau halaman web Anda sendiri, atau halaman web yang memiliki autorisasi untuk Anda indeks. Untuk mempelajari cara menghentikanAmazon Kendra web Anda, silakan lihatMengkonfigurasirobots.txt file untukAmazon Kendra Web Crawler.

Anda dapat terhubungAmazon Kendra ke sumber data crawler web Anda menggunakan Amazon Kendrakonsol dan WebCrawlerConfigurationAPI.

Untuk mengatasi masalah konektor sumber data crawlerAmazon Kendra web Anda, lihatMengatasi masalah sumber data.

Prasyarat

Sebelum Anda dapat menggunakanAmazon Kendra untuk mengindeks wesbites Anda, periksa detail situs web danAWS akun Anda.

Untuk situs web Anda, pastikan Anda memiliki:

  • Menyalin URL seed atau peta situs web yang ingin Anda indeks.

  • Untuk situs web yang memerlukan autentikasi dasar: Menyalin nama host situs web dan nomor port.

  • Opsional: Menyalin nama host situs web dan nomor port jika Anda ingin menggunakan proksi web untuk tersambung ke situs web internal yang ingin dirayapi. Proxy web harus menghadap publik. Amazon Kendramendukung koneksi ke server proksi web yang didukung oleh autentikasi dasar atau Anda dapat terhubung tanpa autentikasi.

  • Memeriksa setiap dokumen halaman web yang ingin Anda indeks unik dan di seluruh sumber data lain yang Anda rencanakan untuk digunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensyal autentikasi crawler web AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat menghubungkan sumber data perayap web keAmazon Kendra. Jika Anda menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia yang ada, dan ID indeks.

Petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber data perayap web Anda, Anda harus memberikan detail yang diperlukan dari sumber data perayap web Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum dikonfigurasi web crawler untukAmazon Kendra melihatPrasyarat.

Console

Untuk menyambungAmazon Kendra ke web crawler

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di halaman Tambahkan sumber data, pilih konektor perayap web, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan detail sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi — Masukkan deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Pada halaman Tentukan akses dan keamanan, masukkan informasi berikut:

    1. Untuk Sumber, pilih antara URL Sumber dan Peta situs Sumber tergantung pada kasus penggunaan Anda dan masukkan nilai untuk masing-masing.

      catatan

      Anda dapat menambahkan hingga 10 URL sumber dan 3 peta situs.

      Jika Anda ingin merayapi peta situs, penting untuk memeriksa apakah URL dasar atau root adalah URL dasar yang sama yang digunakan untuk URL yang tercantum di halaman peta situs Anda. Misalnya, jika URL peta situs Anda adalah https://example.com/sitemap-page.html, URL yang tercantum di halaman peta situs ini juga harus menggunakan URL dasar "https://example.com/”.

    2. (Opsional) Untuk Web proxy - masukkan informasi berikut:

      1. Nama host —Nama host di mana proxy web diperlukan.

      2. Nomor port —Port yang digunakan oleh protokol transport URL host. Nomor port harus berupa nilai numerik antara 0 dan 65535.

      3. Untuk kredensyal proxy Web —Pilih rahasia yang ada atau buat rahasia baru untuk menyimpan kredensyal otentikasi perayap web Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      4. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-WebCrawler -' secara otomatis ditambahkan ke nama rahasia Anda.

        2. Untuk Nama pengguna dan Kata Sandi —Masukkan kredensyal otentikasi dasar ini untuk situs web Anda.

        3. Pilih Save (Simpan).

    3. (Opsional) Host dengan otentikasi —Pilih untuk menambahkan host tambahan dengan otentikasi.

    4. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensyal repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    5. Pilih Selanjutnya.

  7. Pada halaman Konfigurasikan pengaturan sinkronisasi, masukkan informasi berikut:

    1. Rentang perayapan —Pilih jenis halaman web yang ingin Anda jelajahi.

    2. Crawl depth —Pilih jumlah level dari URL seed yangAmazon Kendra seharusnya merangkak.

    3. Pengaturan crawl lanjutan dan Konfigurasi tambahan masukkan informasi berikut:

      1. Ukuran file maksimum —Halaman web maksimum atau ukuran lampiran untuk di-crawl. Minimal 0.000001 MB (1 byte). Maksimal 50 MB.

      2. Tautan maksimum per halaman —Jumlah maksimum tautan yang dirayapi per halaman. Tautan dirayapi sesuai urutan. Minimal 1 link/halaman. Maksimum 1000 link/halaman.

      3. Pembatasan maksimum — Jumlah maksimum URL per nama host per menit. Minimal 1 URL/nama host/menit. Maksimum 300 URL/nama host/menit.

      4. Pola regex —Tambahkan pola untuk menyertakan hanya URL tertentu, atau untuk mengecualikan URL, jenis file, atau file tertentu di repositori Anda. Anda dapat memiliki total gabungan 100 pola.

    4. Di Sinkronkan jadwal jalankan, untuk Frekuensi —Pilih seberapa seringAmazon Kendra akan disinkronkan dengan sumber data Anda.

    5. Pilih Selanjutnya.

  8. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk menyambungAmazon Kendra ke web crawler

Anda harus menentukan WebCrawlerConfigurationAPI berikut ini:

  • URL —Tentukan URL seed atau titik awal situs web atau URL peta situs web yang ingin SeedUrlConfigurationdirayapi SiteMapsConfiguration.

    Jika Anda ingin merayapi peta situs, penting untuk memeriksa apakah URL dasar atau root adalah URL dasar yang sama yang digunakan untuk URL yang tercantum di halaman peta situs Anda. Misalnya, jika URL peta situs Anda adalah https://example.com/sitemap-page.html, URL yang tercantum di halaman peta situs ini juga harus menggunakan URL dasar "https://example.com/”.

  • RahasiAmazon Resource Name (ARN) — Jika situs web memerlukan autentikasi dasar, nomor port, dan rahasia yang menyimpan kredensyal autentikasi dasar nama pengguna dan kata sandi Anda. Anda menyediakan ARN menggunakan AuthenticationConfigurationAPI. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "username": "user name",
    "password": "password"
}

    Anda dapat memberikan kredensyal proksi web menggunakan rahasia secaraAWSSecrets Manager rahasia. Gunakan ProxyConfigurationAPI untuk memberikan nama port situs web, dan secara opsional rahasia yang menyimpan kredensyal proksi web Anda.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Anda juga dapat menambahkan fitur opsional berikut:

  • Mode perayapan—Pilih untuk merayapi nama host situs web saja, atau nama host dengan subdomain, atau juga merayapi domain lain yang ditautkan halaman web.

  • 'kedalaman' atau jumlah tingkat dalam situs web dari tingkat seed hingga Misalnya, jika situs web memiliki 3 tingkat — tingkat indeks (tingkat seed dalam contoh ini), tingkat subbagian, dan tingkat subbagian — dan Anda hanya ingin merayapi informasi dari tingkat indeks ke tingkat bagian (tingkat 0 hingga 1), Anda dapat mengatur kedalaman ke 1.

  • Jumlah maksimum URL pada satu halaman web.

  • Ukuran maksimum halaman web dalam MB.

  • Jumlah maksimum URL yang dirayapi per host situs web per menit.

  • Informasi proksi web untuk menyambungkan dan merayapi situs web internal.

  • Informasi autentikasi untuk mengakses dan merayapi situs web yang memerlukan autentikasi pengguna.

  • Anda dapat mengekstrak tag meta HTML sebagai bidang menggunakan alat Pengayaan Dokumen Kustom. Untuk informasi selengkapnya, lihat Menyesuaikan metadata dokumen selama proses konsumsi. Untuk contoh penggalian tag meta HTML, lihat contoh CDE.

  • Filter penyertaan dan pengecualian —Menentukan apakah akan menyertakan atau mengecualikan URL tertentu.

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak diindeks, meskipun cocok dengan filter penyertaan.
