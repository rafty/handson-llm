Menambahkan pertanyaan yang sering diajukan sering diajukan langsung ke indeks

Anda dapat menambahkan pertanyaan yang Sering Diajukan (Pertanyaan yang Sering Diajukan) langsung ke indeks menggunakan konsol tersebut atauCreateFaq API. Menambahkan Pertanyaan yang Sering Diajukan ke indeks adalah operasi asinkron. Anda memasukkan data untuk Pertanyaan yang Sering Diajukan di file yang disimpan diAmazon Simple Storage Service bucket. Anda dapat menggunakan file CSV atau JSON sebagai input untuk Pertanyaan yang Sering Diajukan:

  • CSV dasar — File CSV yang setiap barisnya berisi pertanyaan, jawaban, dan URL opsional dengan informasi lebih lanjut tentang jawaban.

  • CSV kustom — File CSV yang berisi pertanyaan, jawaban, dan header yang menentukan atribut kustom yang dapat Anda gunakan untuk membuat faset, menampilkan, atau mengurutkan respons Pertanyaan yang Sering Diajukan. Anda juga dapat menentukan atribut kontrol akses untuk membatasi respons Pertanyaan yang Sering Diajukan untuk pengguna dan grup tertentu.

  • JSON — File JSON yang berisi pertanyaan dan jawaban. Secara opsional, file JSON dapat mencakup atribut kontrol akses dan kustom. Anda dapat menentukan atribut untuk facet, display, dan mengurutkan tanggapan FAQ. Atau, Anda dapat menggunakan atribut kontrol akses yang membatasi respons Pertanyaan yang Sering Diajukan untuk pengguna dan grup tertentu.

Misalnya, berikut ini adalah file CSV dasar yang memberikan jawaban atas pertanyaan tentang klinik gratis di Spokane, AS.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

Saat menggunakan file CSV kustom atau JSON untuk input, Anda dapat mendeklarasikan atribut kustom untuk pertanyaan yang Sering Diajukan. Misalnya, Anda dapat membuat atribut kustom yang menetapkan departemen untuk setiap Pertanyaan yang Sering Diajukan. Ketika Pertanyaan yang Sering Diajukan dikembalikan dalam respons, Anda dapat menggunakan departemen sebagai faset untuk mempersempit pencarian.

Atribut kustom harus dipetakan ke bidang indeks. Anda dapat menggunakan bidang bawaan atau menentukan bidang indeks kustom. Di konsol tersebut, Anda menggunakan halaman Definisi faset untuk membuat bidang indeks. Saat menggunakan API, Anda harus terlebih dahulu membuat bidang indeks menggunakanUpdateIndex API.

Jenis atribut dalam file Pertanyaan yang Sering Diajukan harus sesuai dengan jenis bidang indeks terkait. Misalnya,_authors bidang bawaan adalah bidangSTRING_LIST tipe. Jadi, Anda harus memberikan nilai untuk_authors bidang sebagai daftar string dalam file FAQ Anda. Anda dapat memeriksa jenis bidang indeks menggunakan halaman Definisi faset di konsol tersebut atau menggunakanDescribeIndex API.

Ketika membuat bidang indeks yang dipetakan ke atribut kustom, Anda dapat menandainya sebagai dapat ditampilkan, dapat dibuat faset, atau dapat diurutkan. Anda tidak dapat menjadikan atribut kustom dapat dicari.

Selain atribut kustom, Anda juga dapat menggunakan atribut bawaan berikut dalam file CSV kustom atau JSON:

  • _authors(Daftar String list) —Daftar penulis jawaban atas Pertanyaan yang Sering Diajukan.

  • _category(String) —Kategori yang mengelompokkan jawaban atas Pertanyaan yang Sering Diajukan dengan dokumen serupa lainnya.

  • _created_at(String yang dikodekan dengan ISO 8601) —Tanggal dan waktu Pertanyaan yang Sering Diajukan dibuat. Tanggal dan waktu harus diformat sebagai string yang dikodekan dengan ISO 8601.

    Anda juga dapat menyertakan zona waktu dalam format tanggal-waktu ISO 8601 jika diperlukan. Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012, pukul 12:30 PM (ditambah 10 detik) di zona waktu Eropa Tengah.

  • _last_updated_at(String yang dikodekan dengan ISO 8601) — Tanggal dan waktu Pertanyaan yang Sering Diajukan diperbarui. Tanggal dan waktu harus di-fromatted sebagai string yang dikodekan dengan ISO 8601.

    Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012, pukul 12:30 PM (ditambah 10 detik) di zona waktu Eropa Tengah.

  • _source_uri(String) —URL untuk dokumen dengan informasi lebih lanjut tentang jawaban Pertanyaan yang Sering Diajukan.

  • _version(String) — Versi Pertanyaan yang Sering Diajukan.

  • _view_count(Long) — Frekuensi Pertanyaan yang Sering Diajukan dilihat dalam hasil pencarian.

File CSV dasar

Gunakan file CSV dasar ketika Anda ingin menggunakan struktur sederhana untuk Pertanyaan yang Sering Diajukan. Dalam file CSV dasar, setiap baris memiliki dua atau tiga bidang: pertanyaan, jawaban, dan URI sumber opsional yang mengarah ke dokumen dengan informasi lebih lanjut.

Isi file harus mengikuti Format Umum RFC 4180 dan Jenis MIME untuk file Nilai yang Dipisahkan Koma (CSV).

Berikut ini adalah file Pertanyaan yang Sering Diajukan dalam format CSV dasar.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

File CSV khusus

Gunakan file CSV kustom saat Anda ingin menambahkan atribut kustom ke Pertanyaan yang Sering Diajukan. Untuk file CSV kustom, gunakan baris header di file CSV untuk menentukan atribut tambahan.

File CSV harus berisi dua atribut wajib berikut:

  • _question—Pertanyaan yang sering diajukan

  • _answer—Jawaban atas pertanyaan yang sering diajukan

File Anda dapat berisi atribut bawaan dan kustom. Berikut ini adalah contoh file CSV kustom.

_question,_answer,_last_updated_at,custom_string
How many free clinics are in Spokane WA?, 13, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services
How many free clinics are there in Mountain View Missouri?, 7, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services

Isi file atribut kustom harus mengikuti Format Umum RFC 4180 dan Jenis MIME untuk file Nilai yang Dipisahkan Koma (CSV).

Ada empat jenis atribut kustom:

  • Tanggal — Nilai tanggal dan waktu yang dikodekan dengan ISO 8601.

    Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012, pukul 12:30 PM (ditambah 10 detik) di zona waktu Eropa Tengah.

  • Panjang — Angka, seperti1234.

  • String — Nilai string. Jika string berisi koma, apit seluruh nilai dalam tanda kutip ganda (“) (misalnya,"custom attribute, and more").

  • String list — Daftar nilai string. Cantumkan nilai dalam daftar yang dipisahkan koma, yang diapit dalam tanda kutip (“) (misalnya,"item1, item2, item3"). Jika daftar berisi hanya satu entri, Anda dapat menghilangkan tanda kutip (misalnya,item1).

File CSV khusus dapat berisi bidang konteks pengguna. Anda dapat menggunakan bidang ini untuk membatasi akses ke Pertanyaan yang Sering Diajukan untuk pengguna dan grup tertentu. Untuk memfilter konteks pengguna, pengguna harus memberikan informasi pengguna dan grup dalam kueri. Jika tidak, semua Pertanyaan yang Sering Diajukan yang relevan dikembalikan. Untuk informasi selengkapnya, lihat Penyaringan pada konteks pengguna.

Ada empat filter konteks pengguna untuk Pertanyaan yang Sering Diajukan:

  • _acl_user_allow—Pengguna dalam daftar yang diizinkan dapat melihat Pertanyaan yang Sering Diajukan dalam respons kueri. Pertanyaan yang Sering Diajukan tidak dikembalikan ke pengguna lain.

  • _acl_user_deny—Pengguna dalam daftar yang ditolak tidak dapat melihat Pertanyaan yang Sering Diajukan dalam respons kueri. Pertanyaan yang Sering Diajukan dikembalikan ke semua pengguna lain apabila relevan dengan kueri.

  • _acl_group_allow—Pengguna yang merupakan anggota grup yang diizinkan dapat melihat Pertanyaan yang Sering Diajukan dalam respons kueri. Pertanyaan yang Sering Diajukan tidak dikembalikan ke pengguna yang merupakan anggota grup lain.

  • _acl_group_deny—Pengguna yang merupakan anggota grup yang ditolak tidak dapat melihat Pertanyaan yang Sering Diajukan dalam respons kueri. Pertanyaan yang Sering Diajukan dikembalikan ke grup lain apabila relevan dengan kueri.

Berikan nilai untuk daftar yang diizinkan dan ditolak dalam daftar yang dipisahkan koma, diapit dalam tanda kutip (misalnya,"user1,user2,user3"). Anda dapat menyertakan pengguna atau grup dalam daftar yang diizinkan maupun ditolak, tetapi tidak keduanya. Jika menyertakan pengguna atau grup dalam keduanya, Anda akan menerima kesalahan.

Berikut ini adalah contoh file CSV kustom dengan informasi konteks pengguna.

_question, _answer, _acl_user_allow, _acl_user_deny, _acl_group_allow, _acl_group_deny
How many free clinics are in Spokane WA?, 13, userID4565, "userID6201, userID7552", groupBasicRate, "groupBasicPlusRate,groupPremiumRate"

File JSON

Anda dapat menggunakan file JSON untuk memberikan pertanyaan, jawaban, dan atribut untuk indeks Anda. Anda dapat menambahkan salah satu atribut bawaan dan atribut kustom ke Pertanyaan yang Sering Diajukan. Ini adalah skema untuk file JSON.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": string,
            "Answer": string,
            "Attributes": {
                string: object
                additional attributes
            },
            "AccessControlList": [
               {
                   "Name": string,
                   "Type": enum( "GROUP" | "USER" ),
                   "Access": enum( "ALLOW" | "DENY" )
               },
               additional user context
            ]
        },
        additional FAQ documents
    ]
}

Contoh file JSON ini menunjukkan dua dokumen FAQ. Salah satunya memiliki pertanyaan dan jawaban yang diperlukan saja. Yang lain juga memiliki atribut tambahan dan informasi konteks pengguna.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": "How many free clinics are in Spokane WA?",
            "Answer": "13"
        },
        {
            "Question": "How many free clinics are there in Mountain View Missouri?",
            "Answer": "7",
            "Attributes": {
                "_source_uri": "https://www.freeclinics.com",
                "_category": "Charitable Clinics"
            },
            "AccessControlList": [
               {
                   "Name": "user@amazon.com",
                   "Type": "USER",
                   "Access": "ALLOW"
               },
               {
                   "Name": "Admin",
                   "Type": "GROUP",
                   "Access": "ALLOW"
               }
            ]
        }
    ]
}

Ada empat jenis atribut kustom untuk file JSON:

  • Date — Nilai string JSON dengan nilai tanggal dan waktu yang dikodekan dengan ISO 8601.

    Penting agar zona waktu disertakan dalam format tanggal-waktu ISO 8601. Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012, pukul 12:30 PM (ditambah 10 detik) di zona waktu Eropa Tengah.

  • Long — Nilai angka JSON, seperti1234.

  • String-Sebuah nilai string JSON (misalnya,"custom attribute").

  • String list — Array nilai string JSON (misalnya,["item1,item2,item3"]).

Selain atribut bawaan dan kustom, Anda dapat memberikan informasi konteks pengguna untuk Pertanyaan yang Sering Diajukan dalam file JSON. Anda dapat memberikan informasi konteks pengguna untuk membatasi akses ke konten Pertanyaan yang Sering Diajukan berdasarkan pengguna dan grup. Anda dapat menyertakan pengguna atau grup dalam daftar yang diizinkan maupun ditolak, tetapi tidak keduanya. Jika menyertakan pengguna atau grup dalam keduanya, Anda akan menerima kesalahan. Untuk memfilter konteks pengguna, Anda harus memberikan informasi pengguna dan grup dalam kueri. Jika tidak, semua Pertanyaan yang Sering Diajukan yang relevan dikembalikan. Untuk informasi selengkapnya, lihat Penyaringan pada konteks pengguna.

Contoh JSON ini menambahkan konteks pengguna ke Pertanyaan yang Sering Diajukan.

"AccessControlList": [
                {
                    "Name": "group or user name",
                    "Type": "GROUP | USER",
                    "Access": "ALLOW | DENY"
                },
                additional user context
            ]

Menggunakan file Pertanyaan yang Sering Diajukan

Setelah menyimpan file input Pertanyaan yang Sering Diajukan dalam bucket S3, gunakan konsol tersebut atauCreateFaq API untuk memasukkan pertanyaan dan jawaban ke dalam indeks Anda. Jika ingin memperbarui Pertanyaan yang Sering Diajukan, hapus Pertanyaan yang Sering Diajukan dan buat lagi. GunakanDeleteFaq API untuk menghapus Pertanyaan yang Sering Diajukan.

Anda harus menyediakanIAM peran yang memiliki akses ke bucket S3 yang berisi file sumber Anda. Tentukan peran di konsol tersebut, atau di parameter RoleArn. Berikut ini adalah contoh program yang menambahkan file Pertanyaan yang Sering Diajukan ke indeks.

Python
import boto3

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the IAM role ARN required to index documents in an S3 bucket
role_arn = "arn:aws:iam::${accountId}:role/${roleName}"

# Provide the S3 bucket path information to the FAQ file
faq_path = {
    "Bucket": "bucket-name",
    "Key": "FreeClinicsUSA.csv"
}

response = kendra.create_faq(
    S3Path =  faq_path,
    Name = "FreeClinicsUSA",
    IndexId = index_id,
    RoleArn = role_arn
)

print(response)
Java
package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.CreateFaqRequest;
import software.amazon.awssdk.services.kendra.model.CreateFaqResponse;
import software.amazon.awssdk.services.kendra.model.S3Path;

public class AddFaqExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String indexId = "yourIndexId";
        String roleArn = "your role for accessing S3 files";

        CreateFaqRequest createFaqRequest = CreateFaqRequest
            .builder()
            .indexId(indexId)
            .name("FreeClinicsUSA")
            .roleArn(roleArn)
            .s3Path(
                S3Path
                    .builder()
                    .bucket("an-aws-kendra-test-bucket")
                    .key("FreeClinicsUSA.csv")
                    .build())
            .build();

        CreateFaqResponse response = kendra.createFaq(createFaqRequest);

        System.out.println(String.format("The result of creating FAQ: %s", response));

    }
}

File FAQ dalam bahasa selain bahasa Inggris

Anda dapat mengindeks FAQ dalam bahasa yang didukung. Amazon KendraPertanyaan yang Sering Diajukan dalam bahasa Inggris secara default jika Anda tidak menentukan bahasa. Anda menentukan kode bahasa saat Anda memanggil CreateFaqoperasi atau Anda dapat menyertakan kode bahasa untuk FAQ dalam metadata FAQ sebagai bidang. Jika FAQ tidak memiliki kode bahasa dalam metadata yang ditentukan dalam bidang metadata, FAQ diindeks menggunakan kode bahasa yang ditentukan saat Anda memanggilCreateFAQ operasi. Untuk mengindeks dokumen FAQ dalam bahasa yang didukung di konsol, buka FAQ dan pilih Tambahkan FAQ. Anda memilih bahasa dari dropdown Bahasa.
