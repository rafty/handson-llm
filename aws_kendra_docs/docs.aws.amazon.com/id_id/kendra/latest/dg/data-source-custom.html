konektor sumber data kustom

Gunakan sumber data kustom ketika Anda memiliki repositori yangAmazon Kendra belum diberi konektor sumber data kustom ketika Anda memiliki repositori yang belum diberi konektor sumber data. Anda dapat menggunakannya untuk melihat metrik riwayat operasi yang sama yang disediakan sumberAmazon Kendra data, meskipun Anda tidak dapat menggunakanAmazon Kendra sumber data untuk menyinkronkan repositori. Gunakan ini untuk menciptakan pengalaman pemantauan sinkronisasi yang konsisten antara sumberAmazon Kendra data dan sumber data kustom. Khususnya, gunakan sumber data kustom untuk melihat metrik sinkronisasi bagi konektor sumber data yang Anda buat menggunakan BatchDeleteDocumentAPI BatchPutDocumentdan.

Untuk mengatasi masalah konektor sumber data kustom Amazon Kendra, lihatMengatasi masalah sumber data.

Ketika membuat sumber data kustom, Anda memiliki kontrol penuh atas proses pemilihan dokumen yang akan diindeks. Amazon Kendrahanya menyediakan informasi metrik yang dapat Anda gunakan untuk memantau tugas sinkronisasi sumber data. Anda harus membuat dan menjalankan crawler yang menentukan dokumen yang diindeks sumber data Anda.

Anda harus menentukan judul utama dokumen Anda menggunakan objek Dokumen, dan_source_uri untuk memilikiDocumentTitle danDocumentURI termasuk dalam responsQuery hasilnya. DocumentAttribute

Buat pengenal untuk sumber data kustom menggunakan konsol tersebut atau menggunakan CreateDataSourceAPI. Untuk menggunakan konsol tersebut, beri nama sumber data Anda, serta deskripsi dan tanda sumber daya, jika perlu. Setelah sumber data dibuat, ID sumber data ditampilkan. Salin ID ini untuk digunakan saat Anda menyinkronkan sumber data dengan indeks.

Anda juga dapat membuat sumber data kustom menggunakanCreateDataSource API. API ini mengembalikan ID untuk digunakan saat Anda menyinkronkan sumber data. Ketika Anda menggunakanCreateDataSource API untuk membuat sumber data kustom, Anda tidak dapat mengaturConfiguration,RoleArn atauSchedule parameter. Jika Anda mengatur parameter ini,Amazon Kendra mengembalikanValidationException pengecualian.

Untuk menggunakan sumber data kustom, buat aplikasi yang bertanggung jawab untuk memperbaruiAmazon Kendra indeks kustom. Aplikasi tersebut bergantung pada crawler yang Anda buat. Crawler membaca dokumen dalam repositori Anda dan menentukan mana yang harus dikirimAmazon Kendra. Aplikasi Anda harus melakukan langkah-langkah berikut:

  1. Merayapi repositori Anda dan membuat daftar dokumen di repositori Anda yang ditambahkan, diperbarui, atau dihapus.

  2. Memanggil StartDataSourceSyncJobAPI untuk menandakan bahwa tugas sinkronisasi dimulai. Sediakan ID sumber data untuk mengidentifikasi sumber data yang sedang disinkronkan. Amazon Kendramengembalikan ID eksekusi untuk mengidentifikasi tugas sinkronisasi tertentu.

  3. Panggil BatchDeleteDocumentAPI untuk menghapus dokumen dari indeks. Berikan ID sumber data dan ID eksekusi untuk mengidentifikasi sumber data yang sedang disinkronkan dan tugas yang terkait dengan pembaruan ini.

  4. Memanggil StopDataSourceSyncJobAPI untuk menandakan akhir dari tugas sinkronisasi. Setelah Anda memanggilStopDataSourceSyncJob API, ID eksekusi yang terkait tidak lagi berlaku.

  5. Memanggil ListDataSourceSyncJobsAPI dengan pengenal indeks dan sumber data untuk mencantumkan tugas sinkronisasi bagi sumber data dan melihat metrik untuk tugas sinkronisasi.

Setelah menyelesaikan tugas sinkronisasi, Anda dapat memulai tugas sinkronisasi baru. Mungkin perlu waktu beberapa lama sebelum semua dokumen yang dikirim ditambahkan ke indeks. GunakanListDataSourceSyncJobs API untuk melihat status tugas sinkronisasi. Jika Status yang dikembalikan untuk tugas sinkronisasi adalah SYNCING_INDEXING, beberapa dokumen masih diindeks. Anda dapat memulai tugas sinkronisasi baru ketika status tugas sebelumnya adalah FAILED, SUCCEEDED, atau SYNCING_INDEX.

Setelah memanggilStopDataSourceSyncJob API, Anda tidak dapat menggunakan pengenal tugas sinkronisasi dalam panggilan keBatchDeleteDocument APIBatchPutDocument atau. Jika Anda melakukannya, semua dokumen yang dikirim akan dikembalikan dalam pesanFailedDocuments respons dari API.

Atribut yang diperlukan

Ketika Anda mengirimkan dokumen untukAmazon Kendra menggunakanBatchPutDocument API, setiap dokumen memerlukan dua atribut untuk mengidentifikasi sumber data dan operasi sinkronisasi yang mencakupnya. Anda harus memberikan dua atribut berikut:

  • _data_source_idPengenal sumber data. Ini dikembalikan ketika Anda membuat sumber data dengan konsol tersebut atauCreateDataSource API.

  • _data_source_sync_job_execution_idPengenal operasi sinkronisasi. Ini dikembalikan ketika Anda memulai sinkronisasi indeks indeks denganStartDataSourceSyncJob API.

Berikut ini adalah JSON yang diperlukan untuk indeks dokumen menggunakan sumber data kustom.

{
    "Documents": [
        {
            "Attributes": [
                {
                    "Key": "_data_source_id",
                    "Value": {
                        "StringValue": "data source identifier"
                    }
                },
                {
                    "Key": "_data_source_sync_job_execution_id",
                    "Value": {
                        "StringValue": "sync job identifier"
                    }
                }
            ],
            "Blob": "document content",
            "ContentType": "content type",
            "Id": "document identifier",
            "Title": "document title"
        }
    ],
    "IndexId": "index identifier",
    "RoleArn": "IAM role ARN"
}

Ketika menghapus dokumen dari indeks menggunakanBatchDeleteDocument API, Anda harus menentukan dua bidang berikut ini dalamDataSourceSyncJobMetricTarget parameter:

  • DataSourceIdPengenal sumber data. Ini dikembalikan ketika Anda membuat sumber data dengan konsol tersebut atauCreateDataSource API.

  • DataSourceSyncJobIdPengenal operasi sinkronisasi. Ini dikembalikan ketika Anda memulai sinkronisasi indeks indeks denganStartDataSourceSyncJob API.

Berikut ini adalah JSON yang diperlukan untuk menghapus dokumen dari indeks indeks menggunakanBatchDeleteDocument API.

        "DataSourceId": "data source identifier",
        "DataSourceSyncJobId": "sync job identifier"
    },
    "DocumentIdList": [
        "document identifier"
    ],
    "IndexId": "index identifier"
}

Melihat metrik

Setelah tugas sinkronisasi selesai, Anda dapat menggunakan DataSourceSyncJobMetricsAPI untuk mendapatkan metrik yang terkait dengan tugas sinkronisasi. Gunakan ini untuk memantau sinkronisasi sumber data kustom Anda.

Jika Anda mengirimkan dokumen yang sama beberapa kali, baik sebagai bagian dariBatchPutDocument API,BatchDeleteDocument API, API, atau jika dokumen dikirimkan untuk penambahan dan penghapusan, dokumen hanya dihitung sekali dalam metrik.

  • DocumentsAdded— Jumlah dokumen yang dikirimkan menggunakanBatchPutDocument API yang terkait dengan tugas sinkronisasi ini yang ditambahkan ke indeks untuk kali pertama. Jika dokumen dikirimkan untuk penambahan lebih dari sekali dalam sinkronisasi, dokumen hanya dihitung satu kali dalam metrik.

  • DocumentsDeleted— Jumlah dokumen yang dikirimkan menggunakanBatchDeleteDocument API yang terkait dengan tugas sinkronisasi ini yang dihapus dari indeks. Jika dokumen dikirimkan untuk penghapusan lebih dari sekali dalam sinkronisasi, dokumen hanya dihitung satu kali dalam metrik.

  • DocumentsFailed—Jumlah dokumen yang terkait dengan tugas sinkronisasi ini yang gagal diindeks. Ini adalah dokumen yang diterima olehAmazon Kendra untuk pengindeksan tetapi tidak dapat diindeks atau dihapus. Jika dokumen tidak diterima olehAmazon Kendra, pengidentifikasi untuk dokumen dikembalikan dalam propertiFailedDocuments respons dariBatchPutDocument danBatchDeleteDocument API.

  • DocumentsModified— Jumlah dokumen yang dimodifikasi yang dikirimkan menggunakanBatchPutDocument API yang terkait dengan tugas sinkronisasi ini yang diubah dalamAmazon Kendra indeks.

Amazon KendrajugaAmazon CloudWatch mentransmisikan metrik saat mengindeks dokumen. Untuk informasi selengkapnya, lihat MemantauAmazon Kendra denganAmazon CloudWatch.

Amazon Kendratidak mengembalikanDocumentsScanned metrik untuk sumber data kustom. Ini juga memancarkanCloudWatch metrik yang tercantum dalam dokumen Metrik untuk sumberAmazon Kendra data.
