Penerapan Amazon Kendra

Ketika tiba saatnya untuk men-deployAmazon Kendra pencarian ke situs Anda, kami menyediakan kode sumber yang dapat Anda gunakan dengan React untuk memulai lebih awal pada aplikasi Anda. Kode sumber disediakan tanpa biaya di bawah lisensi MIT dimodifikasi. Anda dapat menggunakan seperti apa adanya atau mengubahnya untuk kebutuhan Anda sendiri. Aplikasi React yang disediakan adalah contoh untuk membantu Anda memulai. Ini bukan aplikasi siap produksi.

Untuk menerapkan aplikasi pencarian tanpa kode dan menghasilkan URL titik akhir ke halaman pencarian Anda dengan kontrol akses, lihat PembuatAmazon Kendra Pengalaman.

Ada dua contoh yang dapat Anda gunakan dengan React:

  • https://kendrasamples.s3.amazonaws.com/kendrasamples-react-app .zip —Contoh aplikasi React yang menyediakan data sampel dan halaman pencarian.

  • https://kendrasamples.s3.amazonaws.com/kendrasamples.zip —Pustaka yang dapat Anda tambahkan ke aplikasi React yang ada.

Contohnya dimodelkan sesuai halaman pencarianAmazon Kendra konsol. Mereka memiliki fitur yang sama untuk mencari dan menampilkan hasil pencarian. Anda dapat menggunakan seluruh contoh, atau Anda dapat memilih salah satu fitur untuk Anda gunakan sendiri.

Untuk melihat tiga komponen halaman pencarian diAmazon Kendra konsol, pilih ikon kode (</>) dari menu sebelah kanan. Arahkan kursor ke setiap bagian untuk melihat deskripsi singkat komponen dan untuk mendapatkan URL sumber komponen.

Gambaran Umum

Anda menambahkan kode contoh ke aplikasi React yang ada untuk mengaktifkan pencarian. Berikut adalah struktur dari file pencarian dan komponen:

  • Halaman pencarian utama—ini adalah halaman utama yang berisi semua komponen. Di sinilah Anda akan mengintegrasikan aplikasi Anda denganAmazon Kendra API.

  • Bilah pencarian — ini adalah komponen di mana pengguna memasukkan istilah pencarian dan yang memanggil fungsi pencarian.

  • Hasil — ini adalah komponen yang menampilkan hasil dariAmazon Kendra. Ini memiliki tiga komponen: jawaban yang disarankan, hasil FAQ, dan dokumen yang direkomendasikan.

  • Faset — ini adalah komponen yang menunjukkan faset dalam hasil pencarian dan memungkinkan Anda untuk memilih faset untuk membatasi pencarian.

  • Pemberian nomor halaman - ini adalah komponen yang memberi nomor halaman pada respon dariAmazon Kendra.

Prasyarat

  • Lingkungan pengembangan dikonfigurasi dengan pustaka yang benar.

  • Node.js dan npm diinstal.

Awas

Aplikasi React belum siap produksi. Ini adalah contoh penerapan aplikasi untukAmazon Kendra penelusuran.

  1. Pastikan Anda telah menyelesaikanPrasyarat, termasuk mengunduh dan menginstal Node.js dan npm.

  2. Unduh kendra-samples-react-app .zip dan unzip.

  3. Buka terminal Anda dan pergi keaws-kendra-sample-app/src/services/. Bukalocal-dev-credentials-template.json dan berikan kredensi Anda. Jangan tambahkan file ini ke repositori publik apa pun.

  4. Pergi keaws-kendra-sample-app/src dan instal dependensi. Jalankan npm install.

  5. Luncurkan versi demo aplikasi Anda di server lokal Anda. Jalankan npm start. Anda dapat menghentikan server lokal dengan memasukkan pada keyboard AndaCmd/Ctrl + C.

  6. Anda dapat mengubah port atau host (misalnya, alamat IP) dengan pergi kepackage.json dan memperbarui host dan port:"start": "HOST=[host] PORT=[port] react-scripts start". Jika Anda menggunakan Windows:"start": "set HOST=[host] && set PORT=[port] && react-scripts start".

  7. Jika Anda memiliki domain situs web terdaftar, Anda dapatpackage.json menentukannya setelah nama aplikasi. Sebagai contoh, "homepage": "https://mywebsite.com". Anda harus menjalankannpm install lagi untuk memperbarui dependensi baru, dan kemudian jalankannpm start.

  8. Untuk membangun aplikasi, jalankannpm build. Unggah konten direktori build ke penyedia hosting Anda.

Halaman pencarian utama

Halaman pencarian utama berisi semua contoh komponen pencarian. Ini termasuk komponen bar pencarian untuk output, komponen hasil untuk menampilkan respon dari Query API, dan komponen pemberian nomor halaman untuk respon.

Komponen pencarian

Komponen pencarian menyediakan kotak teks untuk memasukkan teks kueri. onSearchFungsi ini adalah hook yang memanggil fungsi utamaSearch.tsx untuk membuat panggilanAmazon Kendra Query API.

Komponen hasil

Komponen hasil menunjukkan respon dariQuery API. Hasilnya ditunjukkan di tiga area terpisah.

  • Jawaban yang disarankan — Ini adalah hasil teratas yang dikembalikan olehQuery API. Ini berisi hingga tiga jawaban yang disarankan. Dalam respon, mereka mempunyai jenis hasil ANSWER.

  • Jawaban FAQ — Ini adalah hasil pertanyaan yang sering diajukan yang dikembalikan oleh respon. FAQ ditambahkan ke indeks secara terpisah. Dalam respon, mereka mempunyai jenis hasil QUESTION_ANSWER. Untuk informasi selengkapnya, lihat Pertanyaan dan jawaban.

  • Dokumen yang direkomendasikan — Ini adalah dokumen tambahan yangAmazon Kendra dikembalikan dalam respon. Dalam respons dariQuery API, mereka memiliki tipeDOCUMENT.

Komponen hasil berbagi satu set komponen untuk fitur seperti menyoroti, judul, tautan, dan lainnya. Komponen bersama harus hadir agar komponen hasil bisa bekerja.

Komponen faset

Komponen faset mencantumkan faset yang tersedia di hasil pencarian. Setiap faset mengklasifikasikan respon di sepanjang dimensi tertentu, seperti penulis. Anda dapat memperbaiki pencarian ke faset tertentu dengan memilih salah satu dari daftar.

Setelah Anda memilih faset, komponen memanggilQuery dengan filter atribut yang membatasi pencarian ke dokumen yang cocok dengan faset.

Komponen paginasi

Komponen pemberian nomor halaman memungkinkan Anda untuk menampilkan hasil pencarian dariQuery API di beberapa halaman. Ini memanggilQuery API denganPageNumber parameterPageSize dan untuk mendapatkan halaman hasil tertentu.
