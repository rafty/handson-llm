Kendur

Slack adalah aplikasi komunikasi perusahaan yang memungkinkan pengguna mengirim pesan dan lampiran melalui berbagai saluran publik dan pribadi. Anda dapat menggunakanAmazon Kendra untuk mengindeks saluran publik dan pribadi Slack Anda, pesan bot dan arsip, file dan lampiran, pesan langsung dan grup. Anda juga dapat memilih konten tertentu untuk memfilter.

Anda dapat terhubungAmazon Kendra ke sumber data Slack Anda menggunakan Amazon Kendrakonsol dan SlackConfigurationAPI.

Untuk mengatasi masalah konektor sumber dataAmazon Kendra Slack Anda, lihatMengatasi masalah sumber data.

Prasyarat

Sebelum Anda dapat menggunakanAmazon Kendra untuk mengindeks sumber data Slack Anda, buat perubahan ini di Slack danAWS akun Anda.

Di Slack, pastikan Anda memiliki:

  • Membuat token OAuth Pengguna Slack Bot atau token Slack User OAuth. Anda dapat memilih salah satu token untuk terhubungAmazon Kendra ke sumber data Slack Anda. Lihat dokumentasi Slack pada token akses untuk informasi lebih lanjut.

    catatan

    Jika Anda menggunakan token bot sebagai bagian dari kredensyal Slack Anda, Anda tidak dapat mengindeks pesan langsung dan pesan grup dan Anda harus menambahkan token bot ke saluran yang ingin Anda indeks.

  • Mencatat ID tim ruang kerja Slack Anda dari URL halaman utama ruang kerja Slack Anda. Misalnya, https://app.slack.com/client/T0123456789/... dimana T0123456789 adalah ID tim.

  • Menambahkan izin [cakupan Oauth/baca] berikut:

    • saluran: sejarah

    • saluran: baca

    • kelompok:sejarah

    • kelompok-kelompok: baca

    • im:sejarah

    • mpim:sejarah

    • mpim: baca

    • tim: baca

    • users.profile: baca

    • pengguna: baca

    • emoji: baca

    • file: baca

    • usergroups:baca

  • Memeriksa setiap dokumen unik di Slack dan di seluruh sumber data lain yang Anda rencanakan untuk digunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensyal autentikasi Slack AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat menghubungkan sumber data Slack keAmazon Kendra. Jika menggunakan API, Anda harus menyediakan ARN untukIAM role danSecrets Manager secret yang ada, dan ID indeks

Instruksi

Untuk terhubungAmazon Kendra ke sumber data Slack Anda, Anda harus memberikan rincian yang diperlukan dari sumber data Slack Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum dikonfigurasi Slack untukAmazon Kendra melihatPrasyarat.

Console

Untuk terhubungAmazon Kendra ke Slack

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di halaman Tambahkan sumber data, pilih Konektor kendur, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan detail sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan nilai deskripsi untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Di halaman Tetapkan akses dan keamanan, masukkan informasi berikut:

    1. ID tim ruang kerja Slack —ID tim dari ruang kerja Slack Anda.

    2. AWSSecrets Managersecret —Pilih rahasia yang ada atau buatSecrets Manager rahasia baru untuk menyimpan kredensyal autentikasi Slack Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      1. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-Slack-' secara otomatis ditambahkan ke nama rahasia Anda.

        2. Untuk token Slack —Masukkan nilai kredensi otentikasi yang Anda buat di akun Slack Anda.

      2. Pilih Save (Simpan).

    3. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    4. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensyal repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    5. Pilih Selanjutnya.

  7. Di halaman Konfigurasikan pengaturan sinkronisasi, masukkan informasi berikut:

    1. Pilih jenis konten yang akan di-crawl —Entitas Slack atau jenis konten yang ingin Anda jelajahi.

    2. Ubah log —Pilih untuk memperbarui indeks Anda alih-alih menyinkronkan semua file Anda.

    3. Pola regex —Pola ekspresi reguler untuk menyertakan atau mengecualikan file tertentu Anda dapat menambahkan hingga 100 pola.

    4. Pilih tanggal mulai crawl —Tanggal berapaAmazon Kendra akan mulai merangkak data Anda.

    5. Di Sinkronisasi jadwal jalankan, untuk Frekuensi —Seberapa seringAmazon Kendra akan disinkronkan dengan sumber data Anda.

    6. Pilih Selanjutnya.

  8. Di halaman Set

    1. Untuk pemetaan bidang Slack —Pilih dari bidang sumber data defaultAmazon Kendra yang dihasilkan yang ingin Anda petakan ke indeks Anda.

    2. Tambahkan bidang —Untuk menambahkan bidang sumber data kustom untuk membuat nama bidang indeks untuk memetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk terhubungAmazon Kendra ke Slack

Anda harus menentukan yang berikut menggunakan SlackConfigurationAPI:

  • ID tim ruang kerja Slack —ID tim Slack yang Anda salin dari URL halaman utama Slack Anda.

  • Daftar entitas yang akan diindeks —ApakahAmazon Kendra harus mengindeks saluran publik dan pribadi Anda, serta grup serta pesan langsung Anda.

  • Tanggal perayapan —Tanggal untuk mulai merangkak data Anda dari tim ruang kerja Slack Anda. Tanggal harus mengikuti format ini: yyyy-mm-dd.

  • Secret Amazon Resource Name (ARN) —Berikan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensyal otentikasi untuk akun Slack Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "slackToken": "token"
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk konektor Slack danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber data Slack.

Anda juga dapat menambahkan fitur-fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration sebagai bagian dari konfigurasi sumber data. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

  • Log —ApakahAmazon Kendra harus menggunakan mekanisme log Perubahan Slack untuk menentukan apakah dokumen harus ditambahkan, diperbarui, atau dihapus dalam indeks.

    catatan

    Gunakan log Perubahan jika tidakAmazon Kendra ingin memindai semua dokumen. Jika log Perubahan mungkin butuh waktuAmazon Kendra lebih sedikit untuk memindai dokumen di sumber data Slack daripada untuk memproses log perubahan. Jika Anda menyinkronkan sumber data Slack dengan indeks untuk kali pertama, semua dokumen akan dipindai.

  • Filter inklusi dan pengecualian —Tentukan apakah akan menyertakan atau mengecualikan saluran publik dan pribadi tertentu, pesan grup dan pribadi, serta bot dan pesan yang diarsipkan. Jika Anda menggunakan token bot sebagai bagian dari kredensyal autentikasi Slack, Anda harus menambahkan token bot ke saluran yang ingin Anda indeks. Anda tidak dapat mengindeks pesan langsung dan pesan grup menggunakan token bot.

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak akan diindeks meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data Slack Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

  • Pemfilteran konteks pengguna -Amazon Kendra merayapi Access Control List (ACL) untuk sumber data Anda secara default. Informasi ACL digunakan untuk memfilter hasil pencarian berdasarkan pengguna atau akses grup ke dokumen. Untuk informasi selengkapnya, lihat Pemfilteran konteks pengguna untuk sumber data Slack.
