Amazon RDS/Aurora

Anda dapat mengindeks dokumen yang disimpan dalam basis data menggunakan sumber data basis data. Setelah memberikan informasi koneksi untuk basis data,Amazon Kendra menghubungkan dan mengindeks dokumen.

Amazon Kendramendukung basis data berikut:

  • Amazon Aurora MySQL

  • Amazon Aurora PostgreSQL

  • Amazon RDS untuk MySQL

  • Amazon RDS untuk PostgreSQL

catatan

Basis data Aurora nirkabel server tidak didukung.

Anda dapat terhubungAmazon Kendra ke sumber data database Anda menggunakan Amazon Kendrakonsol dan DatabaseConfigurationAPI.

Untuk mengatasi masalah konektor sumber dataAmazon Kendra database Anda, lihatMengatasi masalah sumber data.

Fitur yang didukung

Amazon Kendrakonektor sumber data basis data mendukung fitur-fitur berikut:

  • Pemetaan lapangan

  • Penyaringan konteks pengguna

  • Virtual private cloud (VPC)

Prasyarat

Sebelum Anda dapat menggunakanAmazon Kendra untuk mengindeks sumber data database Anda, buat perubahan ini di database danAWS akun Anda.

Dalam basis data, pastikan Anda memiliki:

  • Mencatat pasangan kata sandi pengguna Anda sebagai kredensi otentikasi dasar untuk sumber data Anda.

  • Menyalin nama host, nomor port, alamat host, dan nama tabel data yang berisi data basis data. Untuk PostgreSQL, tabel data harus berupa tabel publik.

    catatan

    Host dan port memberi tahuAmazon Kendra tempat untuk menemukan server basis data di internet. Nama basis data dan nama tabel memberi tahuAmazon Kendra tempat untuk menemukan data dokumen pada server basis data.

  • Menyalin nama kolom dalam tabel data yang berisi data dokumen, ID dokumen, satu sampai lima kolom untuk mendeteksi jika dokumen telah berubah, dan kolom tabel data opsional yang dipetakan ke bidang indeks kustom. Anda dapat memetakan salahAmazon Kendra satu nama bidang khusus ke kolom tabel.

  • Menyalin informasi jenis mesin database seperti apakah Anda menggunakanAmazon RDS untuk MySQL atau jenis lain.

  • Memeriksa setiap dokumen unik dalam database dan di seluruh sumber data lain yang Anda rencanakan untuk digunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensyal otentikasi database AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat menghubungkan sumber data database AndaAmazon Kendra. Jika Anda menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia yang ada, dan ID indeks.

petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber data database Anda, Anda harus memberikan rincian yang diperlukan dari sumber data database Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum dikonfigurasi database untukAmazon Kendra melihatPrasyarat.

Console

Untuk menyambungAmazon Kendra ke database

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di halaman Tambahkan sumber data, pilih konektor database, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan detail sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan nilai deskripsi opsional untuk sumber data.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Di halaman Tentukan akses dan keamanan, masukkan informasi berikut:

    1. Endpoint —Nama host DNS, alamat IPv4, atau alamat IPv6.

    2. Port —A nomor port.

    3. Database —Nama database.

    4. Nama tabel —Nama tabel.

    5. Untuk Jenis otentikasi, pilih antara Existing dan New untuk menyimpan kredensyal otentikasi database Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      1. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-database-' secara otomatis ditambahkan ke nama rahasia Anda.

        2. Untuk Nama pengguna dan Kata Sandi —Masukkan nilai kredensi otentikasi dari akun database Anda.

        3. Pilih Simpan otentikasi.

    6. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

      catatan

      Anda harus menggunakan subnet privat. Jika instans RDS berada di subnet publik dalam VPC, Anda dapat membuat subnet privat yang memiliki akses keluar ke gateway NAT di subnet publik. Subnet yang disediakan dalam konfigurasi VPC harus berada di US West (Oregon), US East (N. Virginia).

    7. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensyal repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    8. Pilih Selanjutnya.

  7. Di halaman Konfigurasi pengaturan sinkronisasi, masukkan informasi berikut:

    1. Pilih antara Aurora MySQL, MySQL, Aurora PostgreSQL, dan PostgreSQL berdasarkan kasus penggunaan Anda.

    2. Lampirkan pengidentifikasi SQL dengan tanda kutip ganda —Pilih untuk menyertakan pengidentifikasi SQL dalam tanda kutip ganda. Misalnya, “columnName”.

    3. ACL kolom dan Ubah mendeteksi kolom —Gunakan untuk mengkonfigurasi kolom yangAmazon Kendra menggunakan untuk deteksi perubahan dan daftar kontrol akses.

    4. Di Sinkronkan jadwal jalankan, untuk Frekuensi —Pilih seberapa seringAmazon Kendra akan disinkronkan dengan sumber data Anda.

    5. Pilih Selanjutnya.

  8. Di halaman Set field pemetaan, masukkan informasi berikut:

    1. Amazon Kendrapemetaan bidang default —Pilih dari bidang sumber data defaultAmazon Kendra yang dihasilkan yang ingin Anda petakan ke indeks Anda. Anda harus menambahkan nilai kolom Database untukdocument_id dandocument_body

    2. Pemetaan bidang kustom —Untuk menambahkan bidang sumber data kustom untuk membuat nama bidang indeks untuk memetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk menyambungAmazon Kendra ke database

Anda harus menentukan DatabaseConfigurationAPI berikut ini:

  • ColumnConfiguration—Informasi tentang di mana indeks harus mendapatkan informasi dokumen dari basis data. Untuk rincian lebih lanjut, lihat ColumnConfiguration. Anda harus menentukan bidang DocumentDataColumnName dan DocumentIdColumnName. Kolom yang dipetakan ke bidang DocumentIdColumnName harus berupa kolom integer. Contoh berikut menunjukkan konfigurasi kolom sederhana untuk sumber data basis data: 

    "ColumnConfiguration": {
    "ChangeDetectingColumns": [
        "LastUpdateDate",
        "LastUpdateTime"
    ],
    "DocumentDataColumnName": "TextColumn",
    "DocumentIdColumnName": "IdentifierColumn",
    "DocoumentTitleColumnName": "TitleColumn",
    "FieldMappings": [
        {
            "DataSourceFieldName": "AbstractColumn",
            "IndexFieldName": "Abstract"
        }
    ]
}

  • ConnectionConfiguration—Jenis mesin basis data yang menjalankan basis data. DatabaseHostBidang harus menjadi titik akhirAmazon Relational Database Service (Amazon RDS) contoh untuk database. Jangan gunakan titik akhir klaster.

  • DatabaseEngineType—Configuration informasi yang diperlukan untuk terhubung ke database. Untuk rincian lebih lanjut, lihat ConnectionConfiguration.

  • Rahasia Amazon Resource Name (ARN) —Menyediakan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensi otentikasi untuk akun database Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "username": "user name",
    "password": "password"
}

    Rahasia dapat berisi lebih banyak informasi. Contoh berikut menunjukkan konfigurasi basis data.

    
"DatabaseConfiguration": {
"ConnectionConfiguration": {
"DatabaseHost": "host.subdomain.domain.tld",
        "DatabaseName": "DocumentDatabase",
        "DatabasePort": 3306,
        "SecretArn": "arn:aws:secretmanager:region:account ID:secret/secret name",
        "TableName": "DocumentTable"
    }
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk konektor database danAmazon Kendra. Untuk informasi lebih lanjut, lihat peran IAM untuk sumber data basis data.

Anda juga dapat menambahkan fitur-fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration sebagai bagian dari konfigurasi sumber data. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

    catatan

    Anda hanya harus menggunakan subnet privat. Jika instans RDS berada di subnet publik dalam VPC, Anda dapat membuat subnet privat yang memiliki akses keluar ke gateway NAT di subnet publik. Subnet yang disediakan dalam konfigurasi VPC harus berada di US West (Oregon), US East (N. Virginia).

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data database Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

  • Pemfilteran konteks pengguna -Amazon Kendra merayapi Access Control List (ACL) untuk sumber data Anda secara default. Informasi ACL digunakan untuk memfilter hasil pencarian berdasarkan pengguna atau akses grup ke dokumen. Untuk informasi lebih lanjut, lihat Penyaringan konteks pengguna untuk sumber data basis data.
