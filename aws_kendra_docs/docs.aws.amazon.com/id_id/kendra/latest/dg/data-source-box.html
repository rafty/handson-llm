Kotak

Box adalah layanan penyimpanan cloud yang menawarkan kemampuan hosting file. Anda dapat menggunakanAmazon Kendra untuk mengindeks konten dalam konten Box Anda, termasuk komentar, tugas, dan tautan web.

Anda dapat terhubungAmazon Kendra ke sumber data Box Anda menggunakan Amazon Kendrakonsol dan BoxConfigurationAPI.

Untuk mengatasi masalah konektor sumber dataAmazon Kendra Box Anda, lihatMengatasi masalah sumber data.

Fitur yang didukung

Amazon KendraKonektor sumber data kotak mendukung fitur-fitur berikut:

  • Perubahan log

  • Pemetaan lapangan

  • Pemfilteran konteks pengguna

  • Filter inklusi/pengecualian

  • Virtual Private Cloud (VPC)

Prasyarat

Sebelum Anda dapat menggunakanAmazon Kendra untuk mengindeks sumber data Box Anda, buat perubahan ini di Box danAWS akun Anda.

Dalam kotak, pastikan Anda memiliki:

  • Akun Box Enterprise atau Box Enterprise Plus.

  • Membuat aplikasi kustom Box di Box Developer Console dan mengkonfigurasinya untuk menggunakan Server Authentication (dengan JWT). Lihat dokumentasi Box tentang membuat dokumentasi Custom App and Box untuk mengonfigurasi JWT Auth untuk lebih jelasnya.

  • Tetapkan Level Akses Aplikasi Anda ke App + Enterprise Access dan izinkan untuk Melakukan panggilan API menggunakan header as-user.

  • Menggunakan pengguna admin untuk menambahkan Lingkup Aplikasi berikut di aplikasi Box Anda:

    • Tulis semua file dan folder yang tersimpan dalam Kotak

    • Mengelola pengguna

    • Mengelola grup

    • Mengelola properti perusahaan

  • key pair Publik/Privat yang dihasilkan dan diunduh termasuk ID klien, rahasia klien, ID kunci publik, ID kunci privat, frasa kelulusan, dan ID perusahaan untuk digunakan sebagai kredensi otentikasi. Lihat Keypair publik dan pribadi untuk detail selengkapnya.

  • Menyalin ID perusahaan Box Anda baik dari pengaturan Box Developer Console atau dari aplikasi Box Anda. Misalnya, 801234567.

  • Memeriksa setiap dokumen unik di Box dan di seluruh sumber data lain yang Anda rencanakan untuk digunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensi otentikasi Box AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat menghubungkan sumber data Box Anda keAmazon Kendra. Jika Anda menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia yang ada, dan ID indeks.

Petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber data Box Anda, Anda harus memberikan rincian yang diperlukan dari sumber data Box Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum dikonfigurasi Box untukAmazon Kendra melihatPrasyarat.

Console

Untuk terhubungAmazon Kendra ke Box

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di Tambahkan sumber data halaman, pilih Konektor kotak, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan nilai deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Pada halaman Tetapkan akses dan keamanan, masukkan informasi berikut:

    1. ID perusahaan kotak —Masukkan ID Box Enterprise Anda.

    2. AWSSecrets Managersecret —Pilih rahasia yang ada atau buatSecrets Manager rahasia baru untuk menyimpan kredensi otentikasi Box Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-Box-' secara otomatis ditambahkan ke nama rahasia Anda.

      2. Untuk Client ID, Client Secret, Public Key ID, Private Key ID, dan Pass Phrase —Masukkan nilai dari Public/Private Key yang Anda buat di akun Box Anda dan download dari akun Box Anda.

      3. Pilih Save (Simpan).

    3. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    4. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensi repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    5. Pilih Selanjutnya.

  7. Pada halaman Konfigurasi pengaturan sinkronisasi, masukkan informasi berikut:

    1. Pilih entitas atau jenis konten —Entitas Kotak atau jenis konten yang ingin Anda jelajahi. Setiap komentar diindeks sebagai dokumen terpisah.

    2. Ubah log —Pilih untuk memperbarui indeks Anda alih-alih menyinkronkan semua file Anda.

    3. Pola regex —Pola ekspresi reguler untuk menyertakan atau mengecualikan file tertentu. Anda dapat menambahkan hingga 100 pola.

    4. Di Sinkronkan jadwal jalankan, untuk Frekuensi —Pilih seberapa seringAmazon Kendra akan disinkronkan dengan sumber data Anda.

    5. Pilih Selanjutnya.

  8. Pada halaman Set field pemetaan, masukkan informasi berikut:

    1. Untuk File dan folder, Komentar, Tugas, dan Tautan Web —Pilih dari bidang sumber data defaultAmazon Kendra yang dihasilkan yang ingin Anda petakan ke indeks Anda.

    2. Tambahkan bidang —Untuk menambahkan bidang sumber data kustom untuk membuat nama bidang indeks untuk memetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk terhubungAmazon Kendra ke Box

Anda harus menentukan yang berikut menggunakan BoxConfigurationAPI:

ID perusahaan kotak — Berikan ID Box Enterprise Anda. Anda dapat menemukan ID perusahaan di setelan Box Developer Console atau saat membuat aplikasi di Box.

  • Rahasia Amazon Resource Name (ARN) —Menyediakan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensi otentikasi untuk akun Box Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "clientID": "client-id",
    "clientSecret": "client-secret",
    "publicKeyID": "public-key-id",
    "privateKey": "private-key",
    "passphrase": "pass-phrase"
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk konektor Box danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber data Box.

Anda juga dapat menambahkan fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration sebagai bagian dari konfigurasi sumber data. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

  • Ubah log —ApakahAmazon Kendra harus menggunakan mekanisme log perubahan sumber data Box untuk menentukan apakah dokumen harus ditambahkan, diperbarui, atau dihapus dalam indeks.

    catatan

    Gunakan log perubahan jika tidakAmazon Kendra ingin memindai semua dokumen. Jika log perubahan Anda berukuran besar, mungkin butuh waktuAmazon Kendra lebih sedikit untuk memindai dokumen di sumber data Box daripada untuk memproses log perubahan. Jika Anda menyinkronkan sumber data Box dengan indeks untuk kali pertama, semua dokumen dipindai.

  • Filter inklusi dan pengecualian —Tentukan apakah akan menyertakan atau mengecualikan file Box, folder, komentar, tugas, dan tautan web tertentu.

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak diindeks meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data Box Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

  • Pemfilteran konteks pengguna -Amazon Kendra merayapi Access Control List (ACL) untuk sumber data Anda secara default. Informasi ACL digunakan untuk memfilter hasil pencarian berdasarkan pengguna atau grup akses ke dokumen. Untuk informasi selengkapnya, lihat Pemfilteran konteks pengguna untuk sumber data Box.
