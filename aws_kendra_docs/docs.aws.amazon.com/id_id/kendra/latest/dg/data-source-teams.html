Tim Microsoft

Microsoft Teams adalah alat kolaborasi perusahaan untuk pengiriman pesan, rapat, dan berbagi file. Jika Anda pengguna Microsoft Teams, Anda dapat menggunakannyaAmazon Kendra untuk mengindeks sumber data Microsoft Teams Anda.

Anda dapat menyambungAmazon Kendra ke sumber data Microsoft Teams menggunakan Amazon Kendrakonsol dan TemplateConfigurationAPI.

Untuk mengatasi masalah konektor sumber dataAmazon Kendra Microsoft Teams Anda, lihatMengatasi masalah sumber data.

Prasyarat

Sebelum dapat digunakanAmazon Kendra untuk mengindeks sumber data Microsoft Teams, buat perubahan ini diAWS akun dan Microsoft Teams Anda.

Di Microsoft Teams, pastikan Anda memiliki:

  • Membuat akun Microsoft Teams di Office 365.

  • Menyalin ID penyewa Microsoft 365 Anda. Anda dapat menemukan ID penyewa Anda di Properti Portal Direktori Aktif Azure Anda.

  • Mengkonfigurasi token kredensi OAuth 2.0 yang berisi ID klien dan rahasia klien. Lihat dokumentasi Azure tentang mengelola token akses untuk Teams untuk informasi selengkapnya.

  • Ditambahkan izin yang diperlukan Anda dapat memilih untuk menambahkan semua izin, atau membatasi cakupan dengan memilih lebih sedikit izin berdasarkan entitas mana yang ingin Anda jelajahi. Di bawah ini adalah tabel izin oleh entitas yang sesuai:

  • Memeriksa setiap dokumen bersifat unik di Microsoft Teams dan di seluruh sumber data lain yang akan Anda gunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensi otentikasi Microsoft Teams AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat Anda menghubungkan sumber data Microsoft Teams AndaAmazon Kendra. Jika menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia yang ada, dan ID indeks.

Petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber data Microsoft Teams, Anda harus memberikan detail yang diperlukan dari sumber data Microsoft Teams agarAmazon Kendra dapat mengakses data Anda. Jika Anda belum mengkonfigurasi Microsoft Teams untukAmazon Kendra melihatPrasyarat.

Console

Untuk menyambungAmazon Kendra ke Microsoft Teams

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di halaman Tambahkan sumber data, pilih Konektor Microsoft Teams, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan detail sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Di halaman Tetapkan akses dan keamanan, masukkan informasi berikut:

    1. Sumber —Masukkan ID penyewa Microsoft 365 Anda. Anda dapat menemukan ID penyewa Anda di Properti Portal Direktori Aktif Azure Anda.

    2. AWSSecrets Managerrahasia —Pilih rahasia yang ada atau buatSecrets Manager rahasia baru untuk menyimpan kredensi otentikasi Microsoft Teams Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      1. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-Microsoft Teams-' secara otomatis ditambahkan ke nama rahasia Anda.

        2. Untuk Client ID dan Client secret —Masukkan nilai kredensi otentikasi yang Anda buat dari akun Microsoft Teams Anda.

      2. Pilih Save (Simpan).

    3. Model pembayaran — Anda dapat memilih model lisensi dan pembayaran untuk akun Microsoft Teams Anda. Model pembayaran Model A dibatasi untuk model lisensi dan pembayaran yang memerlukan kepatuhan keamanan. Model pembayaran Model B cocok untuk model perizinan dan pembayaran yang tidak memerlukan kepatuhan keamanan.

    4. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    5. Perayap identitas - Saat crawler identitas diaktifkan,Amazon Kendra sinkronkan informasi identitas. Jika Anda memilih untuk menonaktifkan crawler identitas, Anda harus mengunggah informasi utama menggunakan PutPrincipalMappingAPI.

    6. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensi repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    7. Pilih Selanjutnya.

  7. Di halaman Konfigurasi pengaturan sinkronisasi, masukkan informasi berikut:

    1. Sinkronkan konten —Pilih konten untuk disinkronkan.

    2. Konfigurasi tambahan — Anda dapat menggunakan pengaturan ini secara opsional untuk mengindeks konten tertentu alih-alih menyinkronkan semua dokumen.

    3. Mode sinkronisasi —Anda dapat memilih bagaimana Anda ingin memperbarui indeks saat konten sumber data berubah.

      1. Jika Anda memilih sinkronisasi penuh,Amazon Kendra akan menyinkronkan semua konten di semua entitas, terlepas dari status sinkronisasi sebelumnya.

      2. Jika Anda memilih sinkronisasi konten baru atau yang dimodifikasi, hanyaAmazon Kendra akan menyinkronkan konten baru atau yang dimodifikasi.

      3. Jika Anda memilih sinkronisasi konten baru, diubah atau dihapus, hanyaAmazon Kendra akan menyinkronkan konten baru, diubah atau dihapus.

  8. Di halaman Set field mappings, masukkan informasi berikut:

    1. Bidang sumber data default —Pilih dari bidang sumber data defaultAmazon Kendra yang dihasilkan yang ingin Anda petakan ke indeks Anda.

    2. Tambahkan bidang —Untuk menambahkan bidang sumber data kustom untuk membuat nama bidang indeks untuk memetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk menyambungAmazon Kendra ke Microsoft Teams

Anda harus menentukan JSON skema sumber data menggunakan TemplateConfigurationAPI. Anda harus memberikan informasi berikut ini:

  • Sumber data —Anda harus menentukan sumber data sebagaiMSTEAMS.

  • ID Penyewa —Anda dapat menemukan ID penyewa Anda di Properti Portal Direktori Aktif Azure Anda.

  • KetikTEMPLATE —Specify sebagai Type saat Anda meneleponCreateDataSource.

  • Rahasia Amazon Resource Name (ARN) —Berikan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensi otentikasi untuk akun Microsoft Teams Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "clientId": "client ID",
    "clientSecret": "clientSecret",
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn kapan Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk konektor Microsoft Teams danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber data Microsoft Teams.

Anda juga dapat menambahkan fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration saat Anda meneleponCreateDataSource. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

  • Aktifkan crawler identitas —Tentukan apakah akan mengaktifkan crawler identitas. Jika crawler identitas dinonaktifkan, Anda harus mengunggah informasi utama menggunakan PutPrincipalMappingAPI. Informasi identitas perayapan pada pengguna dan grup dengan akses ke dokumen tertentu berguna untuk penyaringan konteks pengguna. Hasil pencarian difilter berdasarkan pengguna atau akses grup mereka ke dokumen. Untuk informasi lebih lanjut, lihat Memfilter konteks pengguna.

  • Filter inklusi dan pengecualian —Tentukan apakah akan menyertakan atau mengecualikan dokumen tertentu yang ditemukan di saluran Microsoft Teams.

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak diindeks, meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data Microsoft Teams Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

Untuk daftar kunci JSON penting lainnya untuk dikonfigurasi, lihatSkema template Microsoft Teams.
