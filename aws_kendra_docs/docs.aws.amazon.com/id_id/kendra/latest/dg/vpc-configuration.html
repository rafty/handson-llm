MengkonfigurasiAmazon Kendra untuk menggunakan VPC

Amazon Kendradapat terhubung ke konten AndaAmazon Virtual Private Cloud ke indeks yang disimpan dalam sumber data atau basis data yang berjalan di cloud privat Anda. Ketika membuat konektor sumber data atau konektor basis data, berikan grup keamanan dan pengenal subnet untuk subnet yang berisi sumber data atau basis data Anda. Amazon Kendramenggunakan informasi ini untuk membuat elastic network interface yang digunakannya untuk berkomunikasi secara aman dengan sumber data atau basis data Anda.

Jika sumber data atau basis data Anda tidak berjalan diAmazon VPC, Anda dapat menghubungkan sumber data atau basis data ke jaringan pribadi Virtual (VPN).Amazon VPC Anda mendapatkan VPC default saat membuat akun Amazon. Untuk informasi tentang mengatur VPN, lihat AWS VPNDokumentasi.

Untuk menggunakan VPC, Anda harus memberi tahuAmazon Kendra pengenal subnet tempat basis data berasal dan pengenal setiap grup keamanan yangAmazon Kendra harus digunakan untuk mengakses subnet. Sebagai contoh, jika Anda menggunakan port default untuk basis data MySQL, grup keamanan harusAmazon Kendra memungkinkan mengakses port 3306 pada host yang menjalankan basis data.

Hanya gunakan subnet privat dalam konfigurasi VPC sumber data atau basis data. Jika instans RDS berada di subnet publik dalam VPC, Anda tidak dapat menggunakan subnet tersebut secara langsung untuk menyinkronkan sumber data atau basis data. Sebagai gantinya, buat subnet privat yang memiliki akses keluar ke gateway NAT di subnet publik. Saat mengonfigurasikan VPC untuk sumber data atau basis data, tentukan subnet privat tersebut. Untuk sumber data basis data yang dikonfigurasikan dengan VPC, subnet harus di salah satu ID Availability Zone berikut:

  • US West (Oregon) —usw2-az1, usw2-az2, usw2-az2, usw2-az2, usw2-az3

  • US East (N. Virginia) —use1-az1, use1-az2, use1-az2, use1-az4

  • EU (Irlandia) —euw1-az1, uew1-az2, euw1-az2, euw1-az3

Pengenal untuk subnet dan grup keamanan dikonfigurasikan di panelAmazon VPC kontrol. Untuk melihat pengenal, bukaAmazon VPC konsol sebagai berikut:

Untuk melihat pengenal subnet

  1. Masuk ke AWS Management Console dan buka konsol Amazon VPC di https://console.aws.amazon.com/vpc/.

  2. Dari panel navigasi, pilih Subnet.

  3. Dari daftar subnet, pilih subnet yang berisi server basis data Anda.

  4. Dari tab deskripsi, buat catatan pengenal di bidang ID Subnet.

Untuk melihat pengenal grup keamanan

  1. Masuk ke AWS Management Console dan buka konsol Amazon VPC di https://console.aws.amazon.com/vpc/.

  2. Dari panel navigasi, pilih Kelompok Keamanan.

  3. Dari daftar grup keamanan, pilih grup yang ingin Anda lihat pengenalnya.

  4. Dari tab deskripsi, buat catatan pengenal di bidang ID Grup.

JikaAmazon Kendra harus merutekan koneksi antara dua atau lebih subnet, Anda dapat memberikan beberapa subnet. Sebagai contoh, jika subnet yang berisi server basis data Anda kehabisan alamat IP,Amazon Kendra dapat terhubung ke subnet dengan alamat IP bebas dan merutekan koneksi ke subnet pertama. Jika Anda mencantumkan lebih dari satu subnet, subnet harus dapat berkomunikasi satu sama lain. Setiap subnet harus dikaitkan dengan tabel perutean yang menyediakan akses internet keluar menggunakan perangkat penerjemah alamat jaringan (NAT).

Anda juga dapat memberikan beberapa grup keamanan. Dampak gabungan grup keamanan harus memungkinkanAmazon Kendra mengakses sumber data atau server basis data yang telah Anda tentukan dalam konfigurasi koneksi.

Langkah 1: Konfigurasikan VPC

Konfigurasikan VPC Anda sehingga Anda memiliki subnet privat dan grup keamananAmazon Kendra untuk mengakses basis data MySQL yang berjalan di subnet. Subnet yang disediakan dalam konfigurasi VPC harus di US West (Oregon), US East (N. Virginia), EU (Irlandia).

Untuk mengonfigurasikan VPC

  1. Masuk ke AWS Management Console dan buka konsol Amazon VPC di https://console.aws.amazon.com/vpc/.

  2. Dari panel navigasi, pilih Tabel perutean, lalu pilih Buat tabel perutean.

  3. Untuk bidang Tag nama, masukkan Private subnet route table. Di bidang VPC, pilih VPC Anda, lalu pilih Buat. Pilih Tutup untuk kembali ke daftar tabel perutean.

  4. Dari panel navigasi, pilih Gateway NAT, lalu pilih Buat Gateway NAT.

  5. Di bidang Subnet, pilih subnet yang merupakan subnet publik dan catat ID subnet.

  6. Jika Anda tidak memiliki Alamat IP elastis, pilih Buat EIP Baru, pilih Buat Gateway NAT, lalu pilih Tutup.

  7. Dari panel navigasi, pilih Tabel Rute.

  8. Dari daftar tabel perutean, pilih Tabel rute subnet privat yang Anda buat pada langkah 3. Dari Tindakan, pilih Edit Rute.

  9. Pilih Tambahkan rute. Tambahkan tujuan 0.0.0.0/0 untuk mengizinkan semua lalu lintas keluar ke internet. Untuk Target, pilih Gateway NAT, lalu pilih gateway yang Anda buat pada langkah 4. Pilih Simpan rute, kemudian pilih Tutup.

  10. Dari Tindakan, pilih Edit pengaitan subnet.

  11. Pilih subnet yang ingin dijadikan privat. Jangan pilih subnet dengan gateway NAT yang Anda catat sebelumnya.

Langkah 2: Konfigurasikan keamanan.

Selanjutnya, konfigurasikan grup keamanan untuk basis data Anda.

Untuk membuat grup keamanan

  1. Masuk ke AWS Management Console dan buka konsol Amazon VPC di https://console.aws.amazon.com/vpc/.

  2. Dari deskripsi VPC Anda, catat CIDR IPv4 yang ada.

  3. Dari panel navigasi, pilih Grup Keamanan lalu pilih Buat grup keamanan.

  4. Di Nama grup keamanan, masukkan DataSourceInboundSecurityGroup. Berikan deskripsi, lalu pilih VPC Anda dari daftar. Pilih Buat, lalu pilih Tutup.

  5. Pilih tab Masuk.

  6. Pilih Edit aturan, lalu pilih Tambah Aturan

  7. Untuk basis data, masukkan nomor port untuk Rentang Port. Misalnya, untuk MySQL itu3306, dan, untuk HTTPS, itu443. Untuk Sumber, ketikkan Perutean Inter-Domain Tanpa Kelas (CIDR) VPC Anda. Pilih Simpan aturan, kemudian pilih Tutup.

Grup keamanan memungkinkan siapa pun dalam VPC untuk terhubung ke basis data, dan memungkinkan koneksi keluar ke internet.

Langkah 3: Buat basis data

Buat basis data untuk menyimpan dokumen, atau menggunakan basis data. Lihat Menggunakan sumber data database untuk daftar database yangAmazon Kendra mendukung.

Untuk petunjuk tentang cara membuat basis data MySQL, lihat Mulai Menggunakan sumber data basis data basis data basis data MySQL (konsol)Amazon RDS.

Langkah 4: Buat konektor sumber data basis data

Setelah mengonfigurasikan VPC dan membuat basis data, Anda dapat membuat konektor sumber data untuk basis data. Lihat Menggunakan sumber data basis data.

Pastikan Anda mengonfigurasikan VPC, subnet privat yang Anda buat di VPC, dan grup keamanan yang Anda buat di VPC untuk basis data Anda.
