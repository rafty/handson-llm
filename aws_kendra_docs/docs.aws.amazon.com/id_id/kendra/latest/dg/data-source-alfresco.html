Alfresco

Alfresco OnPrem adalah layanan manajemen konten yang membantu pelanggan menyimpan dan mengelola konten mereka. Anda dapat menggunakannyaAmazon Kendra untuk mengindeks pustaka Dokumen Alfresco, Wiki, dan Blog Anda.

Anda dapat terhubungAmazon Kendra ke sumber data Alfresco menggunakan Amazon Kendrakonsol dan AlfrescoConfigurationAPI.

catatan

Konektor sumber data alfresco saat ini dalam mode pratinjau. Otentikasi dasar saat ini didukung. Jika Anda ingin menggunakan konektor Alfresco dalam produksi, hubungi Support. PaaS Alfresco saat ini tidak didukung.

Fitur yang didukung

Amazon KendraKonektor sumber data alfresco mendukung fitur-fitur berikut:

  • Pemetaan lapangan

  • Pemfilteran teks pengguna

  • Filter inklusi

  • Virtual private cloud (VPC)

Prasyarat

Sebelum dapat digunakanAmazon Kendra untuk mengindeks sumber data Alfresco, buat perubahan ini di Alfresco danAWS akun Anda.

Dalam Alfresco, pastikan Anda memiliki:

  • Mencatat kredensyal autentikasi Alfresco Anda, yang mencakup nama pengguna dan kata sandi admin.

  • Menambahkan pengguna keALFRESCO_ADMINISTRATORS grup.

  • Menyalin URL situs Alfresco dan ID situs Alfresco Anda.

  • Memeriksa setiap dokumen bersifat unik di Alfresco dan di seluruh sumber data lain yang akan Anda gunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensi otentikasi Alfresco AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat menghubungkan sumber data AlfrescoAmazon Kendra. Jika menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia yang ada, dan ID indeks.

Petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber data Alfresco, Anda harus memberikan rincian yang diperlukan dari sumber data Alfresco Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum mengkonfigurasi Alfresco untukAmazon Kendra lihatPrasyarat.

Console

Untuk terhubungAmazon Kendra ke Alfresco

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Pada halaman Tambahkan sumber data, pilih Konektor Alfresco, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan detail sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan nilai deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Di halaman Tentukan akses dan keamanan, masukkan informasi berikut:

    1. URL situs Alfresco —Masukkan URL situs Alfresco Anda. Misalnya, https://hostname:8080.

    2. Lokasi sertifikat SSL —MasukkanAmazon S3 jalur.

    3. ID Situs —ID Situs Alfresco Anda.

    4. AWSSecrets Managersecret —Pilih rahasia yang ada atau buatSecrets Manager rahasia baru untuk menyimpan kredensi otentikasi Alfresco Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      1. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-Alfresco-' secara otomatis ditambahkan ke nama rahasia Anda.

        2. Untuk Nama pengguna dan Kata Sandi —Masukkan nama pengguna dan kata sandi Alfresco Anda.

      2. Pilih Save (Simpan).

    5. Virtual Private Cloud (VPC) Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    6. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensi repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    7. Pilih Selanjutnya.

  7. Di halaman Konfigurasi pengaturan sinkronisasi, masukkan informasi berikut:

    1. Pilih konten yang akan di-crawl —Entitas Alfresco atau jenis konten yang ingin Anda jelajahi.

    2. Pola regex —Pola ekspresi reguler untuk menyertakan atau mengecualikan file tertentu. Anda dapat menambahkan hingga 100 pola.

    3. Di Sinkronkan jadwal jalankan, untuk Frekuensi —Pilih seberapa seringAmazon Kendra akan disinkronkan dengan sumber data Anda.

    4. Pilih Selanjutnya.

  8. Di halaman Set field pemetaan, masukkan informasi berikut:

    1. Untuk perpustakaan Dokumen, Wiki, dan Blog —Pilih dari bidang sumber data defaultAmazon Kendra yang dihasilkan yang ingin Anda petakan ke indeks Anda.

    2. Tambahkan bidang —Untuk menambahkan bidang sumber data kustom untuk membuat nama bidang indeks untuk memetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk terhubungAmazon Kendra ke Alfresco

Anda harus menentukan yang berikut menggunakan AlfrescoConfigurationAPI:

  • Secret Amazon Resource Name (ARN) —Berikan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensi otentikasi untuk akun Alfresco Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "username": "user name",
    "password": "password"
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan memanggil API publik yang diperlukan untuk konektor Alfresco danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber data Alfresco.

Anda juga dapat menambahkan fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration sebagai bagian dari konfigurasi sumber data. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

  • Filter inklusi dan pengecualian —Tentukan apakah akan menyertakan atau mengecualikan file tertentu.

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan pengecualian tidak diindeks meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data Alfresco Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

  • Pemfilteran konteks pengguna -Amazon Kendra merayapi Access Control List (ACL) untuk sumber data Anda secara default. Informasi ACL digunakan untuk memfilter hasil pencarian berdasarkan pengguna atau akses grup ke dokumen. Untuk informasi selengkapnya, lihat Pemfilteran konteks pengguna untuk sumber data Alfresco.
