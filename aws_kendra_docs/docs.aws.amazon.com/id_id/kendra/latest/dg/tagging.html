Tanda

Kelola indeks, sumber data, dan FAQ Anda dengan menetapkan tanda atau label. Anda dapat menggunakan tanda untuk mengategorikanAmazon Kendra sumber daya Anda dengan berbagai cara. Misalnya, berdasarkan tujuan, pemilik, atau aplikasi, atau kombinasi apa pun. Setiap tag terdiri atas sebuah kunci dan sebuah nilai, yang keduanya Anda tentukan.

Tanda membantu Anda untuk:

  • Mengidentifikasi dan mengorganisir sumber daya AWS Anda. Banyak layanan AWS yang mendukung penandaan, sehingga Anda dapat menetapkan tanda yang sama ke sumber daya dari layanan yang berbeda untuk menunjukkan bahwa sumber daya tersebut terkait. Misalnya, Anda dapat menandai indeks danAmazon Lex bot yang menggunakan indeks dengan tag yang sama.

  • Alokasikan biaya. Anda mengaktifkan tag ini pada AWS Billing and Cost Management dasbor. AWS menggunakan tag untuk mengategorikan biaya Anda lalu mengirimkan laporan alokasi biaya bulanan kepada Anda. Untuk informasi selengkapnya, lihat Alokasi Biaya dan Pemberian Tanda di TentangAWS Billing and Cost Management.

  • Kontrol akses ke sumber daya Anda. Anda dapat menggunakan tanda dalam kebijakanAWS Identity and Access Management (IAM) yang mengontrol akses keAmazon Kendra sumber daya. Anda dapat melampirkan kebijakan ini keIAM peran atau pengguna untuk mengaktifkan kontrol akses berbasis tanda. Untuk informasi selengkapnya, lihat Otorisasi berdasarkan tanda Amazon Kendra.

Anda dapat membuat dan mengelola tag menggunakanAWS Management Console, theAWS Command Line Interface (AWS CLI), atauAmazon Kendra API.

Penandaan sumber daya

Jika Anda menggunakanAmazon Kendra konsol, Anda dapat menandai sumber daya saat membuatnya atau menambahkannya nanti. Anda juga dapat menggunakan konsol untuk memperbarui atau menghapus tanda.

Jika Anda menggunakanAWS Command Line Interface (AWS CLI) atauAmazon Kendra API, gunakan operasi berikut untuk mengelola tanda untuk sumber daya Anda:

  • CreateDataSource—Terapkan tanda saat Anda membuat sumber data.

  • CreateFaq—Terapkan tanda saat Anda membuat FAQ.

  • CreateIndex—Terapkan tanda saat Anda membuat indeks.

  • ListTagsForResource—Lihat tanda yang terkait dengan sumber daya.

  • TagResource—Tambahkan dan modifikasi tanda untuk sumber daya.

  • UntagResource—Hapus tanda dari sumber daya.

Pembatasan tanda

Batasan berikut berlaku untuk tanda padaAmazon Kendra sumber daya:

  • Jumlah maksimum tanda — 50

  • Panjang kunci maksimum — 128 karakter

  • Panjang nilai maksimum — 256 karakter

  • Karakter yang valid untuk kunci dan nilai — a-z, A-Z, spasi, dan karakter berikut: _.:/= + - dan @

  • Kunci dan nilai peka huruf besar dan kecil

  • Jangan gunakan aws: sebagai prefiks untuk kunci; ini dicadangkan untuk penggunaan AWS
