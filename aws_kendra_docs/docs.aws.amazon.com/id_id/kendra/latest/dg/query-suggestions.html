Saran kueri

Amazon KendraSaran kueri dapat membantu pengguna Anda mengetikkan kueri penelusuran mereka lebih cepat dan memandu pencarian mereka.

Amazon Kendramenyarankan kueri yang relevan untuk pengguna Anda berdasarkan kueri populer di riwayat kueri atau log kueri. Amazon Kendramenggunakan semua pertanyaan yang dicari pengguna Anda dan belajar dari kueri ini untuk memberi saran kepada pengguna Anda. Amazon Kendramenyarankan pertanyaan populer kepada pengguna ketika mereka mulai mengetik kueri mereka. Amazon Kendramenyarankan kueri jika prefiks atau beberapa karakter pertama kueri cocok dengan apa yang mulai diketik pengguna sebagai kueri mereka.

Misalnya, pengguna mulai mengetik kueri 'acara yang akan datang'. Amazon Kendratelah belajar dari log kueri bahwa banyak pengguna telah mencari 'acara mendatang 2050' berkali-kali. Pengguna melihat 'acara mendatang 2050' muncul langsung di bawah bilah pencarian mereka, melengkapi kueri penelusuran secara otomatis. Pengguna memilih saran kueri ini dengan memilih hasil pencarian pertama, yaitu dokumen 'Acara mendatang: Apa yang terjadi di 2050'.

Anda dapat menentukan caraAmazon Kendra memilih kueri yang memenuhi syarat untuk disarankan kepada pengguna Anda. Misalnya, Anda dapat menentukan bahwa saran kueri harus telah dicari oleh setidaknya 10 pengguna unik (default adalah 3), telah dicari dalam 30 hari terakhir, dan tidak mengandung kata atau frasa apa pun dari Daftar blok Anda. Amazon Kendramemerlukan kueri untuk memiliki setidaknya satu hasil pencarian dan berisi setidaknya satu kata dengan lebih dari empat karakter.

Saran kueri tidak peka huruf besar/kecil. Amazon Kendramengonversi prefiks kueri dan kueri yang disarankan menjadi huruf kecil, mengabaikan semua tanda kutip tunggal dan ganda, dan mengganti beberapa karakter spasi putih dengan satu spasi. Amazon Kendracocok dengan semua karakter khusus lainnya apa adanya. Amazon Kendratidak menunjukkan saran apa pun jika pengguna mengetikkan kurang dari dua karakter atau lebih dari 60 karakter.

Anda dapat mengambil saran kueri yang relevan untuk pengguna Anda dengan menggunakan GetQuerySuggestionsAPI. Saran kueri diaktifkan secara default tanpa biaya tambahan. Anda dapat menonaktifkan saran kueri setiap saat dengan menggunakan UpdateQuerySuggestionsConfigAPI. Anda dapat menguji pengaturan sebelum menerapkan saran ke aplikasi penelusuran dengan dua cara:

  • Dengan menggunakanUpdateQuerySuggestionsConfig API.

  • Melalui konsol di Pengaturan Saran kueri.

Anda menggunakanGetQuerySuggestions API untuk mengintegrasikan saran kueri dengan aplikasi konsol agar pengguna mulai melihat saran.

Pengaturan saran kueri

Anda dapat mengonfigurasi pengaturan berikut menggunakanUpdateQuerySuggestionsConfig API:

  • Mode —Query saran yang baikENABLED atauLEARN_ONLY. Amazon Kendramengaktifkan saran kueri secara default. LEARN_ONLYmematikan saran permintaan. Jika dimatikan,Amazon Kendra terus mempelajari saran, tetapi tidak membuat saran kueri untuk pengguna.

  • Jendela waktu log kueri — Seberapa baru kueri Anda berada di jendela waktu log kueri Anda. Rentang waktu adalah jumlah hari dari hari ini hingga hari sebelumnya.

  • Kueri tanpa informasi pengguna —SetelTRUE ke untuk menyertakan semua kueri atau setelFALSE ke hanya menyertakan kueri dengan informasi pengguna. Anda dapat menggunakan ini jika aplikasi pencarian Anda menyertakan informasi pengguna, seperti ID pengguna, saat pengguna mengeluarkan kueri. Pengaturan ini secara default tidak menyaring kueri jika tidak ada informasi pengguna tertentu yang terkait dengan kueri. Namun, Anda dapat menggunakan pengaturan ini untuk hanya memberikan saran berdasarkan kueri yang menyertakan informasi pengguna.

  • Pengguna unik — Jumlah minimum pengguna unik yang harus mencari kueri agar kueri memenuhi syarat untuk disarankan kepada pengguna Anda. Jumlah ini adalah nilai integer.

  • Jumlah kueri —Frekuensi minimum kueri harus dicari agar kueri memenuhi syarat untuk disarankan kepada pengguna Anda. Jumlah ini adalah nilai integer.

Setelan ini memengaruhi cara kueri dipilih sebagai kueri populer untuk disarankan kepada pengguna Anda. Bagaimana Anda menyetel pengaturan Anda akan bergantung pada kebutuhan spesifik Anda, misalnya:

  • Jika pengguna Anda biasanya menelusuri rata-rata sebulan sekali, maka Anda dapat mengatur jumlah hari dalam jendela waktu log kueri menjadi 30, sehingga Anda menangkap sebagian besar kueri terbaru pengguna sebelum menjadi usang di jendela waktu.

  • Jika hanya sejumlah kecil dari kueri Anda yang menyertakan informasi pengguna, dan Anda tidak ingin menyarankan kueri berdasarkan ukuran sampel yang kecil, Anda dapat menyetel kueri untuk menyertakan semua pengguna.

  • Jika Anda menentukan kueri populer sebagai sedang dicari oleh setidaknya 10 pengguna unik dan mencari setidaknya 100 kali, maka Anda setel pengguna unik menjadi 10 dan jumlah permintaan menjadi 100.

Perubahan pada pengaturan mungkin tidak segera diterapkan. Anda dapat melacak perubahan pengaturan dengan menggunakan DescribeQuerySuggestionsConfigAPI. Waktu diberlakukannya pembaruan pengaturan bergantung pada pembaruan yang dilakukan dan jumlah kueri pencarian dalam indeks Anda.

CLI

Untuk mengambil saran kueri

aws kendra get-query-suggestions \
--index-id index-id \
--query-text "query-text" \
--max-suggestions-count 1 // If you want to limit the number of suggestions.

Untuk memperbarui saran kueri

Misalnya, untuk mengubah jendela waktu log kueri dan berapa kali kueri harus dicari:

aws kendra update-query-suggestions-config \
--index-id index-id \
--query-log-look-back-window-in-days 30 \
--minimum-query-count 100
catatan

Waktu agar pengaturan yang diperbarui berlaku bergantung pada pembaruan yang Anda buat dan jumlah kueri penelusuran di indeks Anda.

Untuk menghapus saran

aws  kendra clear-query-suggestions \
--index-id index-id \
catatan

Amazon Kendramempelajari saran baru berdasarkan kueri baru yang ditambahkan ke log kueri sejak Anda terakhir kali menghapus saran.

Python

Untuk mengambil saran kueri

import boto3
from botocore.exceptions import ClientError

kendra = boto3.client("kendra")

print("Get query suggestions.")

# Provide the index ID
index_id = "index-id"

# Provide the query text
query_text = "query"

try:
    query_suggestions_response = kendra.get_query_suggestions(
        IndexId = index_id,
        QueryText = query_text,
        MaxSuggestionsCount = 5
    )

    # Print out the suggestions you received
    if ("Suggestions" in query_suggestions_response.keys()) {
        for (suggestion: query_suggestions_response["Suggestions"]) {
            print(suggestion["Value"]["Text"]["Text"]);
        }
    }
   
except ClientError as e:
        print("%s" % e)

print("Program ends.")

Untuk memperbarui saran kueri,

Misalnya, untuk mengubah jendela waktu log kueri dan berapa kali kueri harus dicari:

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Updating query suggestions settings/configuration for an index.")

# Provide the index ID
index_id = "index-id"

# Configure the settings you want to update
query_log_look_back_window_in_days = 30
minimum_query_count = 100

try:
    kendra.update_query_suggestions_config(
        IndexId = index_id,
        QueryLogLookBackWindowInDays = query_log_look_back_window_in_days,
        MinimumQueryCount = minimum_query_count
    )

    print("Wait for Amazon Kendra to update the query suggestions.")

    while True:
        # Get query suggestions description of settings/configuration
        query_sugg_config_response = kendra.describe_query_suggestions_config(
            IndexId = index_id
        )
        
        # If status is not UPDATING, then quit
        status = query_sugg_config_response["Status"]
        print(" Updating query suggestions config. Status: " + status)
        if status != "UPDATING":
            break
        time.sleep(60)

except ClientError as e:
        print("%s" % e)

print("Program ends.")

Anda dapat memeriksa pengaturan Anda saat ini dengan menggunakanDescribeQuerySuggestionsConfig API. Selain itu, operasi ini menunjukkan informasi berikut tentang saran kueri Anda untuk indeks:

  • Mode —Query saran yang baikENABLED atauLEARN_ONLY. Amazon Kendramengaktifkan saran kueri secara default. LEARN_ONLYmematikan saran permintaan. Anda dapat mengubah mode dengan menggunakanUpdateQuerySuggestionsConfig API. Jika dimatikan,Amazon Kendra terus mempelajari saran, tetapi tidak membuat saran kueri untuk pengguna.

  • Status —Query saran yang baikACTIVE atauUPDATING.

  • Jumlah saran — Jumlah total kueri yang siap disarankan kepada pengguna Anda. Jika hitungannya jauh lebih rendah dari yang Anda harapkan, itu bisa jadi karenaAmazon Kendra membutuhkan lebih banyak kueri untuk dipelajari atau pengaturan saran kueri Anda saat ini terlalu ketat.

  • Waktu pembuatan terakhir saran — terakhir kali saran diperbarui. Amazon Kendrasecara otomatis memperbarui saran setiap 24 jam, atau ketika Anda mengubah pengaturan atau ketika Anda menerapkan daftar blok.

  • Waktu terakhir saran dikosongkan — terakhir kali saran dihapus.

Memblokir kueri tertentu dari saran

Sebuah daftar blokir berhentiAmazon Kendra untuk menyarankan kueri populer tertentu kepada pengguna Anda. Ini adalah daftar kata atau frasa yang ingin Anda kecualikan dari saran kueri. Amazon Kendramengecualikan kueri yang berisi kecocokan persis kata atau frasa dalam daftar blokir.

Anda dapat menggunakan daftar blokir untuk melindungi dari kata-kata atau frasa yang menyinggung yang biasanya muncul di log kueri Anda dan ituAmazon Kendra dapat memilih sebagai saran. Daftar blok juga dapatAmazon Kendra mencegah menyarankan kueri populer yang berisi informasi yang belum siap dirilis atau diumumkan secara publik. Misalnya, jika pengguna Anda biasanya menanyakan tentang rilis produk baru yang tidak ingin Anda sarankan karena Anda belum siap untuk segera dirilis, maka Anda dapat memblokir kueri yang berisi nama produk dan informasi produk dari saran.

Anda dapat membuat daftar blok untuk kueri dengan menggunakan CreateQuerySuggestionsBlockListAPI. Anda meletakkan setiap blok kata atau frasa pada baris terpisah dalam file teks. Kemudian Anda mengunggah file teks ke bucket S3 Anda dan memberikan path atau lokasi ke file diAmazon S3. Amazon Kendrasaat ini mendukung pembuatan hanya untuk satu daftar blok.

Anda dapat mengganti file teks kata yang diblokir dan frasa dalamAmazon S3 bucket Anda dan menggunakan UpdateQuerySuggestionsBlockListAPI untuk memperbarui daftar blokAmazon Kendra.

Anda dapat menggunakan DescribeQuerySuggestionsBlockListAPI untuk mendapatkan status daftar blokir dan informasi berguna lainnya, seperti kapan daftar blokir terakhir diperbarui, berapa banyak kata atau frasa yang ada dalam daftar blokir Anda saat ini, dan pesan galat yang bermanfaat saat membuat daftar blokir. Anda juga dapat menggunakan ListQuerySuggestionsBlockListsAPI untuk mendapatkan daftar ringkasan daftar blok untuk indeks.

Untuk menghapus daftar blok Anda, gunakan DeleteQuerySuggestionsBlockListAPI.

Pembaruan daftar blokir Anda mungkin tidak segera diberlakukan. Anda dapat melacak pembaruan dengan menggunakanDescribeQuerySuggestionsBlockList API.

Python

Untuk membuat daftar blok

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Create a query suggestions block list.")

# Provide a name for the block list
block_list_name = "block-list-name"
# Provide an optional description for the block list
block_list_description = "block-list-description"
# Provide the IAM role ARN required for query suggestions block lists
block_list_role_arn = "role-arn"

# Provide the index ID
index_id = "index-id"

s3_bucket_name = "bucket-name"
s3_key = "query-suggestions/block_list.txt"
source_s3_path = {
    'Bucket': s3_bucket_name,
    'Key': s3_key
}

try:
    block_list_response = kendra.create_query_suggestions_block_list(
        Description = block_list_description,
        Name = block_list_name,
        RoleArn = block_list_role_arn,
        IndexId = index_id,
        SourceS3Path = source_s3_path
    )

    print(block_list_response)

    block_list_id = block_list_response["Id"]

    print("Wait for Amazon Kendra to create the block list.")

    while True:
        # Get block list description
        block_list_description = kendra.describe_query_suggestions_block_list(
            Id = block_list_id,
            IndexId = index_id
        )
        # If status is not CREATING, then quit
        status = block_list_description["Status"]
        print("Creating block list. Status: " + status)
        if status != "CREATING":
            break
        time.sleep(60)
        
except ClientError as e:
        print("%s" % e)

print("Program ends.")

Saran yang jelas

Anda dapat menghapus saran kueri dengan menggunakan ClearQuerySuggestionsAPI. Menghapus saran hanya menghapus saran kueri yang ada, bukan kueri di log kueri. Setelah menghapus saranAmazon Kendra baru berdasarkan kueri baru yang ditambahkan ke log kueri sejak Anda menghapus saran.

Saran tidak tersedia

Jika Anda tidak melihat saran untuk kueri, mungkin karena salah satu alasan berikut:

  • Tidak ada cukup kueri dalam indeks AndaAmazon Kendra untuk dipelajari.

  • Pengaturan saran kueri Anda terlalu ketat, sehingga sebagian besar kueri disaring dari saran.

  • Anda baru saja menghapus saran, danAmazon Kendra masih membutuhkan waktu untuk mengumpulkan kueri baru guna mempelajari saran baru.

Anda dapat memeriksa pengaturan Anda saat ini dengan menggunakanDescribeQuerySuggestionsConfig API.
