BatchPutDocument

Menambahkan satu atau lebih dokumen ke indeks.

BatchPutDocumentAPI memungkinkan Anda menyerap dokumen langsung atau satu set dokumen yang disimpan dalam bucket Amazon S3. Gunakan API ini untuk menyerap teks dan teks tidak terstruktur ke dalam indeks, menambahkan atribut kustom ke dokumen, dan melampirkan daftar kontrol akses ke dokumen yang ditambahkan ke indeks.

Dokumen tersebut diindeks secara asinkron. Anda dapat melihat kemajuan batch menggunakanAWS CloudWatch. Pesan kesalahan apa pun yang terkait dengan pemrosesan batch akan dikirim keAWS CloudWatch log Anda.

Untuk contoh menelan dokumen inline menggunakan Python dan Java SDK, lihat Menambahkan file langsung ke indeks.

Parameter Permintaan

Untuk informasi tentang parameter yang umum untuk semua tindakan, lihat Parameter Umum.

Permintaan menerima data berikut dalam format JSON.

CustomDocumentEnrichmentConfiguration

Informasi konfigurasi untuk mengubah metadata dokumen dan konten selama proses pengambilan dokumen saat Anda menggunakanBatchPutDocument API.

Untuk informasi selengkapnya tentang cara membuat, memodifikasi, dan menghapus metadata dokumen, atau membuat perubahan konten lainnya saat Anda memasukkan dokumen ke Amazon Kendra, lihat Menyesuaikan metadata dokumen selama proses konsumsi.

Tipe: Objek CustomDocumentEnrichmentConfiguration

Wajib: Tidak

Documents

Satu atau lebih dokumen untuk ditambahkan ke indeks.

Dokumen memiliki batas ukuran file berikut.

  • Ukuran total 50 MB untuk file apa pun

  • 5 MB teks yang diekstrak untuk file apa pun

Untuk informasi lebih lanjut, lihat Kuota .

Tipe: Array objek Document

Anggota Array: Jumlah minimum 1 item. Jumlah maksimum 10 item.

Wajib: Ya

IndexId

Pengenal indeks untuk menambahkan dokumen. Anda harus membuat indeks terlebih dahulu menggunakanCreateIndex API.

Jenis: String

Batas Panjang: Panjang tetap 36.

Pola: [a-zA-Z0-9][a-zA-Z0-9-]*

Diperlukan: Ya

RoleArn

Amazon Resource Name (ARN) dari IAM role dengan izin untuk mengakses bucket S3 Anda. Untuk informasi lebih lanjut, lihat Peran akses IAM untuk Amazon Kendra.

Jenis: String

Batasan Panjang: Panjang minimum 0. Panjang maksimum 1284.

Pola: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Diperlukan: Tidak

Elemen Respons

Jika tindakan berhasil, layanan mengirimkan kembali respons HTTP 200.

Layanan mengembalikan data berikut dalam format JSON.

FailedDocuments

Daftar dokumen yang tidak ditambahkan ke indeks karena gagal melewati pemeriksaan validasi. Setiap dokumen berisi pesan kesalahan yang menunjukkan mengapa dokumen tidak dapat ditambahkan ke indeks.

Jika ada kesalahan saat menambahkan dokumen ke indeks, kesalahan dilaporkan dalamAWS CloudWatch log Anda. Untuk informasi selengkapnya, lihat Memantau Amazon Kendra dengan Amazon CloudWatch Logs

Tipe: Array objek BatchPutDocumentResponseFailedDocument

Kesalahan

Untuk informasi tentang kesalahan yang umum untuk semua tindakan, lihat Kesalahan Umum.

AccessDeniedException

Anda tidak memiliki akses yang memadai untuk melakukan tindakan ini. Pastikan Anda memiliki kebijakan izin dan akun pengguna yang diperlukan dan coba lagi.

Kode Status HTTP: 400

ConflictException

Konflik terjadi dengan permintaan. Harap perbaiki ketidakkonsistenan dengan sumber daya Anda dan coba lagi.

Kode Status HTTP: 400

InternalServerException

Masalah terjadi dengan server internal yang digunakan untuk layanan Amazon Kendra Anda. Tunggu beberapa menit dan coba lagi, atau hubungi Support untuk bantuan.

Kode Status HTTP: 500

ResourceNotFoundException

Sumber daya yang ingin Anda gunakan tidak ada. Silakan periksa Anda telah menyediakan sumber daya yang benar dan coba lagi.

Kode Status HTTP: 400

ServiceQuotaExceededException

Anda telah melampaui batas yang ditetapkan untuk layanan Amazon Kendra Anda. Silakan lihat Kuota untuk informasi lebih lanjut, atau hubungi Support untuk menanyakan tentang peningkatan batas.

Kode Status HTTP: 400

ThrottlingException

Permintaan ditolak karena throttling permintaan. Kurangi jumlah permintaan dan coba lagi.

Kode Status HTTP: 400

ValidationException

Input gagal untuk memenuhi batasan yang ditetapkan oleh layanan Amazon Kendra. Berikan input yang benar dan coba lagi.

Kode Status HTTP: 400
