Parameter Permintaan

Untuk informasi tentang parameter yang umum untuk semua tindakan, lihat Parameter Umum.

Permintaan menerima data berikut dalam format JSON.

CapacityUnits

Menetapkan jumlah penyimpanan dan unit kapasitas kueri yang harus digunakan oleh indeks. Anda dapat mengubah kapasitas indeks hingga 5 kali per hari, atau melakukan 5 panggilan API.

Jika menggunakan unit penyimpanan, Anda tidak dapat mengurangi kapasitas penyimpanan di bawah jumlah yang ditentukan untuk memenuhi kebutuhan penyimpanan untuk indeks Anda.

Tipe: Objek CapacityUnitsConfiguration

Wajib: Tidak

Description

Deskripsi baru untuk indeks.

Jenis: String

Batasan Panjang: Panjang minimum sebesar 0. Panjang maksimum sebesar 1000.

Pola: ^\P{C}*$

Wajib: Tidak

DocumentMetadataConfigurationUpdates

Konfigurasi metadata dokumen yang ingin Anda perbarui untuk indeks. Metadata dokumen adalah bidang atau atribut yang terkait dengan dokumen Anda. Misalnya, nama departemen perusahaan yang terkait dengan setiap dokumen.

Tipe: Array objek DocumentMetadataConfiguration

Anggota Array: Jumlah minimum 0 item. Jumlah maksimum 500 item.

Wajib: Tidak

Id

Pengenal indeks yang ingin Anda perbarui.

Jenis: String

Batas Panjang: Panjang tetap 36.

Pola: [a-zA-Z0-9][a-zA-Z0-9-]*

Diperlukan: Ya

Name

Nama indeks yang ingin Anda perbarui.

Jenis: String

Batasan Panjang: Panjang minimum 1. Panjang maksimum 1000.

Pola: [a-zA-Z0-9][a-zA-Z0-9_-]*

Wajib: Tidak

RoleArn

AWS Identity and Access Management(IAM) role yang memberikan izin pada Amazon Kendra untuk mengakses metrik dan Amazon CloudWatch Logs.

Jenis: String

Batasan Panjang: Panjang minimum 0. Panjang maksimum 1284.

Pola: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Wajib: Tidak

UserContextPolicy

Kebijakan konteks pengguna.

Jenis: String

Nilai yang Valid: ATTRIBUTE_FILTER | USER_TOKEN

Wajib: Tidak

UserGroupResolutionConfiguration

Memungkinkan mengambil tingkat akses grup dan pengguna dari sumberAWS IAM Identity Center (successor to AWS Single Sign-On) identitas. Untuk mengkonfigurasi ini, lihat UserGroupResolutionConfiguration.

Tipe: Objek UserGroupResolutionConfiguration

Wajib: Tidak

UserTokenConfigurations

Konfigurasi token pengguna.

Tipe: Array objek UserTokenConfiguration

Anggota Array: Jumlah maksimum 1 item.

Wajib: Tidak

Elemen Respons

Jika tindakan berhasil, layanan mengirimkan kembali respons HTTP 200 dengan isi HTTP kosong.

Kesalahan

Untuk informasi tentang kesalahan yang umum untuk semua tindakan, lihat Kesalahan Umum.

AccessDeniedException

Anda tidak memiliki akses yang memadai untuk melakukan tindakan ini. Pastikan Anda memiliki kebijakan izin dan akun pengguna yang diperlukan dan coba lagi.

Kode Status HTTP: 400

ConflictException

Konflik terjadi dengan permintaan. Harap perbaiki ketidakkonsistenan dengan sumber daya Anda dan coba lagi.

Kode Status HTTP: 400

InternalServerException

Masalah terjadi dengan server internal yang digunakan untuk layanan Amazon Kendra Anda. Tunggu beberapa menit dan coba lagi, atau hubungi Support untuk bantuan.

Kode Status HTTP: 500

ResourceNotFoundException

Sumber daya yang ingin Anda gunakan. Silakan periksa Anda telah menyediakan sumber daya yang benar dan coba lagi.

Kode Status HTTP: 400

ServiceQuotaExceededException

Anda telah melampaui batas yang ditetapkan untuk layanan Amazon Kendra Anda. Silakan lihat Kuota untuk informasi lebih lanjut, atau hubungi Support untuk menanyakan tentang peningkatan batas.

Kode Status HTTP: 400

ThrottlingException

Permintaan ditolak karena throttling permintaan. Kurangi jumlah permintaan dan coba lagi.

Kode Status HTTP: 400

ValidationException

Input gagal untuk memenuhi batasan yang ditentukan oleh layanan Amazon Kendra. Berikan input yang benar dan coba lagi.

Kode Status HTTP: 400
