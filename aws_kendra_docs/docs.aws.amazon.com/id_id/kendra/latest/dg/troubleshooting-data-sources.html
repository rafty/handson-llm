Mengatasi masalah sumber data

Bagian ini dapat membantu Anda memperbaiki masalah dengan sumber data Amazon Kendra.

Dokumen saya tidak diindeks

Ketika Anda menyinkronkan indeks Kendra Anda dengan sumber data, Anda mungkin mengalami masalah yang mencegah dokumen diindeks. Pengindeksan adalah proses dua langkah. Pertama, sumber data diperiksa untuk dokumen baru dan diperbarui untuk indeks, dan untuk menemukan dokumen untuk dihapus dari indeks. Kedua, di tingkat dokumen, setiap dokumen diakses dan diindeks.

Kesalahan dapat terjadi di salah satu langkah berikut. Kesalahan tingkat sumber data dilaporkan di konsol dalam bagian riwayat jalan sinkronisasi dari halaman detail sumber data. Status tugas sinkronisasi dapat Berhasil, Tidak lengkap, atau Gagal. Anda juga dapat melihat jumlah dokumen yang diindeks dan dihapus selama tugas berlangsung. Jika statusnya adalah Gagal, pesan akan ditampilkan dalam kolom Detail.

Kesalahan tingkat dokumen dilaporkan di Amazon CloudWatch Logs. Anda dapat melihat kesalahan menggunakan CloudWatch konsol.

Tugas sinkronisasi saya gagal

Tugas sinkronisasi biasanya gagal ketika ada kesalahan konfigurasi dalam indeks atau sumber data. Pesan kesalahan di kolom detail sumber data memberikan informasi tentang apa yang salah. Masalah yang sering terjadi adalah indeks atau sumber data tidak memiliki izin IAM yang tepat. Pesan kesalahan menjelaskan izin yang hilang. Berikut beberapa pesan kesalahan yang dapat Anda terima:

Failed to create log group for job. Please make sure that the IAM role provided has sufficient permissions.

Jika peran indeks Anda tidak memiliki izin untuk digunakan CloudWatch, sumber data tidak akan dapat membuat CloudWatch log. Jika mengalami kesalahan ini, Anda harus menambahkan CloudWatch izin ke peran indeks.

Failed to access S3 file prefix (bucket name) while trying to crawl your metadata files. Please make sure the IAM Role (role ARN) provided has sufficient permissions.

Ketika Anda menggunakan sumber data Amazon S3, Kendra harus memiliki izin untuk mengakses bucket yang berisi dokumen. Anda perlu menambahkan izin bagi Kendra untuk membaca bucket ke IAM role sumber data.

The provided IAM Role (role ARN) could not be assumed. Please make sure Amazon Kendra is a trusted entity that is allowed to assume the role.

Kendra membutuhkan izin untuk mengambil peran IAM indeks dan sumber data. Anda perlu menambahkan kebijakan kepercayaan ke peran dengan izin untuk tindakan sts:AssumeRole.

Untuk kebijakan IAM yang dibutuhkan Kendra untuk mengindeks sumber data, lihatIAMakses peran untukAmazon Kendra.

Tugas sinkronisasi saya tidak lengkap

Tugas umumnya tidak lengkap ketika mereka telah menyelesaikan proses tingkat sumber data, tetapi memiliki beberapa kesalahan selama proses tingkat dokumen. Ketika tugas tidak lengkap, beberapa dokumen mungkin masih telah berhasil diindeks. Untuk sumber data Amazon S3, tugas yang tidak lengkap biasanya disebabkan oleh:

  • Metadata yang tidak valid untuk satu atau lebih dokumen.

  • Ketika dokumen diserahkan untuk pengindeksan, tetapi setidaknya satu dokumen tidak diserahkan.

  • Ketika dokumen diserahkan untuk dihapus dari indeks tetapi setidaknya satu dokumen tidak diserahkan.

Untuk memecahkan masalah tugas sinkronisasi yang tidak lengkap, lihat terlebih dulu CloudWatch log.

  1. Dari kolom detail, pilih Lihat detail di CloudWatch.

  2. Meninjau pesan kesalahan untuk melihat apa yang menyebabkan dokumen gagal.

Tugas sinkronisasi saya berhasil tetapi tidak ada dokumen yang diindeks

Kadang-kadang, menjalankan tugas sinkronisasi indeks akan ditandai sebagai Berhasil, tetapi tidak ada dokumen baru atau diperbarui yang diindeks seperti yang Anda harapkan. Alasan yang mungkin termasuk:

  • Periksa CloudWatch DocumentsSubmittedForIndexingFailed metrik untuk melihat apakah ada dokumen yang gagal disinkronkan. Periksa CloudWatch log Anda untuk detailnya.

  • Untuk sumber data Amazon S3, Anda mungkin telah memberikan Kendra nama bucket atau prefiks yang salah. Pastikan bahwa bucket yang Kendra gunakan adalah salah satu yang berisi dokumen untuk diindeks.

  • Ketika mengindeks ulang dokumen yang gagal untuk diindeks dalam tugas sebelumnya, Kendra tidak akan mengindeks kecuali Anda telah mengubah dokumen atau file metadata yang terkait.

Saya mengalami masalah format file saat menyinkronkan sumber data saya

Jika Anda mengalami masalah format file saat menambahkan file ke sumber data atau menyinkronkan sumber data, pastikan jenis dokumen Anda didukung Kendra. Untuk daftar jenis dokumen yang didukung olehAmazon Kendra lihat Jenis dokumen.

Jika Anda menggunakanBatchPutDocument API dengan file teks biasa, tentukanPLAIN_TEXT sebagai jenis konten.

Berapa banyak waktu yang dibutuhkan untuk menyinkronkan sumber data?

Jika tidak ada pembaruan pada dokumen, waktu sinkronisasi untuk indeks Amazon Kendra meningkat dalam proporsi linier dengan jumlah dokumen. Misalnya, 1.000 dokumen tanpa pembaruan akan memakan waktu sekitar lima menit untuk disinkronkan dan 2.000 dokumen tanpa pembaruan akan memakan waktu sekitar 10 menit. Jika ada pembaruan pada dokumen, maka waktu sinkronisasi akan meningkat berdasarkan jumlah dokumen yang diperbarui.

Berapa biaya untuk menyinkronkan sumber data?

Saat Anda menyinkronkan indeks, dibutuhkan dua menit untuk melakukan pemanasan dan mengaktifkan Amazon EC2 untuk membuat koneksi yang diperlukan. Anda tidak dikenakan biaya selama proses ini. Pengukur penggunaan Anda dimulai hanya setelah pekerjaan sinkronisasi dimulai. Untuk informasi lebih lanjut tentang harga Kendra, lihat harga Amazon Kendra.

Saya mendapatkan kesalahan otorisasi Amazon EC2

Jika kesalahan operasi tidak sah Amazon EC2 terjadi selama sinkronisasi untuk sumber data virtual private cloud (VPC), kemungkinan peran VPC IAM Anda tidak memiliki izin yang diperlukan. Harap periksa apakah peran IAM yang Anda gunakan untuk sumber data Anda memiliki izin terlampir. Untuk informasi selengkapnya, lihat Virtual Private Cloud (VPC)IAM role.

Saya tidak dapat menggunakan tautan indeks pencarian untuk membuka objek Amazon S3 saya

Indeks Kendra Anda hanya dapat mengakses file yang diberikan oleh sumber data Amazon S3 untuk diakses. Misalnya, Kendra tidak dapat memodifikasi izin Amazon S3 yang menentukan apakah objek dimaksudkan untuk publik atau dienkripsi. Kendra juga tidak memiliki izin default untuk membuat atau mengembalikan tautan yang ditandatangani untuk objek Amazon S3. Jika Anda ingin mengaktifkan penautan bertanda tangan untuk objek Amazon S3 dalam indeks Kendra, Anda memiliki dua opsi:

  • Anda dapat menggunakan tanda tangani hasil kueri indeks dengan objek uri sumber sebelum mengembalikan hasilnya ke halaman pencarian. Untuk step-by-step panduan proses ini, lihat Berbagi objek menggunakan URL yang telah ditandatangani sebelumnya.

  • Anda dapat mengganti uri sumber metadata objek Amazon S3 dan membuat layanan Anda tersedia melalui jaringan pengiriman CloudFront konten (CDN) yang terhubung ke bucket Amazon S3. Atau, Anda dapat menggunakan titik akhir proxy API Gateway yang mengembalikan URL yang telah ditandatangani sebelumnya dan mengalihkannya.

Saya mendapatkan pesan kesalahan AccessDenied Ketika Menggunakan SSL Certificate File

Jika Anda mendapatkan kesalahan akses ditolak saat menggunakan sertifikat SSL dengan sumber data Anda, pastikan peran IAM Anda memiliki izin untuk mengakses file sertifikat SSL di lokasi yang ditentukan. Jika sertifikat dienkripsi denganAWS KMS kunci, peran IAM Anda juga harus memiliki izin untuk mendekripsi menggunakanAWS KMS kunci. Untuk informasi selengkapnya, lihat Otentikasi dan kontrol akses untukAWS KMS.

Saya mendapatkan kesalahan otorisasi saat menggunakan sumber SharePoint data

Jika Anda mendapatkan kesalahan otorisasi saat menyinkronkan indeks dengan sumber SharePoint data, konfirmasikan bahwa Anda memiliki peran Admin Situs yang ditetapkan untuk Anda SharePoint.

Indeks saya tidak merayapi dokumen dari sumber data Confluence saya

Jika indeks Kendra Anda tidak merayapi dokumen dari sumber data Confluence Anda selama proses sinkronisasi, konfirmasikan bahwa Anda adalah bagian dari Grup Administrator di Pertemuan.
