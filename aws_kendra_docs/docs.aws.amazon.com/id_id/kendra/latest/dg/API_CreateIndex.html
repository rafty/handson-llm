CreateIndex

Membuat indeks Amazon Kendra. Pembuatan indeks merupakan API asinkron. Untuk menentukan apakah pembuatan indeks telah selesai, periksa bidang Status yang dikembalikan dari panggilan ke DescribeIndex. Bidang Status diatur ke ACTIVE saat indeks siap digunakan.

Setelah indeks aktif, Anda dapat mengindeks dokumen menggunakanBatchPutDocument API atau salah satu sumber data yang didukung.

Untuk contoh membuat indeks dan sumber data menggunakan Python SDK, lihat Memulai dengan Python SDK. Untuk contoh pembuatan indeks dan sumber data menggunakan Java SDK, lihat Memulai dengan Java SDK.

Parameter Permintaan

Untuk informasi tentang parameter yang umum untuk semua tindakan, lihat Parameter Umum.

Permintaan menerima data berikut dalam format JSON.

ClientToken

Token yang Anda berikan untuk mengidentifikasi permintaan untuk membuat indeks. Beberapa panggilan keCreateIndex API dengan token klien yang sama akan membuat satu indeks saja.

Jenis: String

Panjang Batasan: Panjang minimum 1. Panjang maksimum 100.

Wajib: Tidak

Description

Deskripsi untuk indeks.

Jenis: String

Batasan Panjang: Panjang minimum sebesar 0. Panjang maksimum sebesar 1000.

Pola: ^\P{C}*$

Wajib: Tidak

Edition

Edisi Amazon Kendra yang akan digunakan untuk indeks. Pilih DEVELOPER_EDITION untuk indeks yang dimaksudkan untuk pengembangan, pengujian, atau bukti konsep. GunakanENTERPRISE_EDITION untuk produksi. Setelah diatur, edisi untuk indeks tidak dapat diubah.

Parameter Edition bersifat opsional. Jika Anda tidak menyediakan nilai, nilai defaultnya adalahENTERPRISE_EDITION.

Untuk informasi lebih lanjut tentang batas kuota untuk edisi Enterprise dan Developer, lihat Kuota.

Jenis: String

Nilai yang Valid: DEVELOPER_EDITION | ENTERPRISE_EDITION

Wajib: Tidak

Name

Nama untuk indeks.

Jenis: String

Batasan Panjang: Panjang minimum 1. Panjang maksimum 1000.

Pola: [a-zA-Z0-9][a-zA-Z0-9_-]*

Diperlukan: Ya

RoleArn

Amazon Resource Name (ARN) dari IAM role dengan izin untuk mengakses CloudWatch log dan metrik Amazon Anda. Untuk informasi lebih lanjut, lihat Peran akses IAM untuk Amazon Kendra.

Jenis: String

Batasan Panjang: Panjang minimum 0. Panjang maksimum 1284.

Pola: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Diperlukan: Ya

ServerSideEncryptionConfiguration

Pengenal kunci terkelolaAWS KMS pelanggan (CMK) yang digunakan untuk mengenkripsi data yang diindeks oleh Amazon Kendra. Amazon Kendra tidak mendukung CMK asimetris.

Tipe: Objek ServerSideEncryptionConfiguration

Wajib: Tidak

Tags

Daftar pasangan nilai-kunci yang mengidentifikasi atau mengkategorikan indeks. Anda juga dapat menggunakan tanda untuk membantu mengontrol akses ke indeks. Kunci dan nilai tanda dapat terdiri dari huruf Unicode, angka, white space, dan salah satu simbol berikut: _ . : / = + - @.

Tipe: Array objek Tag

Anggota Array: Jumlah minimum 0 item. Jumlah maksimum 200 item.

Wajib: Tidak

UserContextPolicy

Kebijakan konteks pengguna.

ATTRIBUTE_FILTER

Semua konten yang diindeks dapat dicari dan dapat ditampilkan untuk semua pengguna. Jika Anda ingin memfilter hasil pencarian pada konteks pengguna, Anda dapat menggunakan filter atribut_user_id dan_group_ids atau Anda dapat memberikan informasi pengguna dan grup diUserContext.

USER_TOKEN

Mengaktifkan kontrol akses pengguna berbasis token untuk memfilter hasil pencarian pada konteks pengguna. Semua dokumen tanpa kontrol akses dan semua dokumen yang dapat diakses pengguna dapat dicari dan ditampilkan.

Jenis: String

Nilai yang Valid: ATTRIBUTE_FILTER | USER_TOKEN

Wajib: Tidak

UserGroupResolutionConfiguration

Mendapat pengguna dan kelompok dari sumberAWS IAM Identity Center (successor to AWS Single Sign-On) identitas. Untuk mengkonfigurasi ini, lihat UserGroupResolutionConfiguration.

Tipe: Objek UserGroupResolutionConfiguration

Wajib: Tidak

UserTokenConfigurations

Konfigurasi token pengguna.

Tipe: Array objek UserTokenConfiguration

Anggota Array: Jumlah maksimum 1 item.

Wajib: Tidak

Elemen Respons

Jika tindakan berhasil, layanan mengirimkan kembali respons HTTP 200.

Layanan mengembalikan data berikut dalam format JSON.

Id

Pengenal indeks. Gunakan pengenal ini ketika Anda membuat kueri indeks, mengatur sumber data, atau mengindeks dokumen.

Jenis: String

Batas Panjang: Panjang tetap 36.

Pola: [a-zA-Z0-9][a-zA-Z0-9-]*

Kesalahan

Untuk informasi tentang kesalahan yang umum untuk semua tindakan, lihat Kesalahan Umum.

AccessDeniedException

Anda tidak memiliki akses yang memadai untuk melakukan tindakan ini. Pastikan Anda memiliki kebijakan izin dan akun pengguna yang diperlukan dan coba lagi.

Kode Status HTTP: 400

ConflictException

Konflik terjadi dengan permintaan. Harap perbaiki ketidakkonsistenan dengan sumber daya Anda dan coba lagi.

Kode Status HTTP: 400

InternalServerException

Masalah terjadi dengan server internal yang digunakan untuk layanan Amazon Kendra Anda. Tunggu beberapa menit dan coba lagi, atau hubungi Support untuk bantuan.

Kode Status HTTP: 500

ResourceAlreadyExistException

Sumber daya yang ingin Anda gunakan sudah ada. Silakan periksa Anda telah menyediakan sumber daya yang benar dan coba lagi.

Kode Status HTTP: 400

ServiceQuotaExceededException

Anda telah melampaui batas yang ditetapkan untuk layanan Amazon Kendra Anda. Silakan lihat Kuota untuk informasi lebih lanjut, atau hubungi Support untuk menanyakan tentang peningkatan batas.

Kode Status HTTP: 400

ThrottlingException

Permintaan ditolak karena throttling permintaan. Kurangi jumlah permintaan dan coba lagi.

Kode Status HTTP: 400

ValidationException

Input gagal untuk memenuhi batasan yang ditetapkan oleh layanan Amazon Kendra. Berikan masukan yang benar dan coba lagi.

Kode Status HTTP: 400
