Indeks

Indeks menyimpan isi dokumen Anda dan disusun dengan cara untuk membuat dokumen dapat dicari. Cara Anda menambahkan dokumen ke indeks tergantung pada bagaimana Anda menyimpan dokumen Anda.

  • Jika Anda menyimpan dokumen dalam beberapa jenis repositori, sepertiAmazon S3 bucket atau SharePoint situs Microsoft, Anda menggunakan konektor sumber data untuk mengindeks dokumen Anda dari repositori Anda.

  • Jika Anda tidak menyimpan dokumen Anda dalam repositori, Anda menggunakan BatchPutDocumentAPI untuk langsung mengindeks dokumen Anda.

  • Untuk pertanyaan dan jawaban FAQ, yang harus disimpan dalam bucketAmazon Kendra (Amazon S3), Anda mengunggahnya dari bucket

Anda dapat membuat indeks denganAmazon Kendra konsol,AWS CLI, atauAWS SDK. Untuk informasi tentang jenis dokumen yang dapat diindeks, lihat Jenis dokumen SSM.

Menggunakan bidang dokumen bawaan Amazon Kendra

Dengan UpdateIndex API, Anda dapat membuat bidang cadangan atau built-in menggunakanDocumentMetadataConfigurationUpdates dan menentukan nama bidang yang dipesan untuk dipetakan ke atribut dokumen atau bidang dokumen yang setara. Anda juga dapat membuat bidang kustom dengan cara ini. Jika Anda menggunakan konektor sumber data, sebagian besar menyertakan pemetaan bidang yang memetakan bidang dokumen sumber data Anda ke bidangAmazon Kendra indeks. Ketika Anda membuat bidang, Anda dapat mengkonfigurasiSearch objek untuk mengatur bidang sebagai ditampilkan, facetable, dicari, dan diurutkan. Anda dapat mengkonfigurasiRelevance objek untuk mengatur relevansi bidang meningkatkan durasi, kesegaran, kepentingan, urutan peringkat, dan nilai-nilai kepentingan. Anda tidak dapat mengubah jenis bidang setelah Anda membuat bidang.

Amazon Kendramemiliki bidang dokumen cadangan atau built-in berikut yang dapat Anda gunakan:

  • _authors—Daftar satu atau beberapa penulis yang bertanggung jawab atas isi dokumen.

  • _category—Kategori yang menempatkan dokumen dalam grup tertentu.

  • _created_at—Tanggal dan waktu dalam format ISO 8601 di mana dokumen dibuat. Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012 pukul 12:30 PM (ditambah 10 detik) di Waktu Eropa Tengah.

  • _data_source_id—Pengenal sumber data yang berisi dokumen.

  • _document_body—Isi dokumen.

  • _document_id—Pengenal unik untuk dokumen.

  • _document_title—Judul dokumen.

  • _excerpt_page_number—Nomor halaman dalam file PDF tempat kutipan dokumen muncul. Jika indeks Anda dibuat sebelum 8 September 2020, Anda harus mengindeks ulang dokumen sebelum dapat menggunakan atribut ini.

  • _faq_id—Jika ini adalah pertanyaan dan jawaban FAQ, pengenal unik untuk mereka.

  • _file_type—Jenis file dokumen, seperti pdf atau doc.

  • _last_updated_at—Tanggal dan waktu dalam format ISO 8601 di mana dokumen terakhir diperbarui. Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012 pukul 12:30 PM (ditambah 10 detik) di Waktu Eropa Tengah.

  • _source_uri—URI tempat dokumen tersedia. Misalnya, URI dokumen di situs web perusahaan.

  • _version—Pengenal bagi versi tertentu dokumen.

  • _view_count—Jumlah berapa kali dokumen dilihat.

  • _language_code(String) —Kode untuk bahasa yang berlaku untuk dokumen. Default bahasa Inggris jika Anda tidak menentukan bahasa. Untuk informasi selengkapnya tentang bahasa yang didukung, termasuk kodenya, lihat Menambahkan dokumen dalam bahasa selain bahasa Inggris.

Untuk bidang kustom, Anda membuat bidang ini menggunakanDocumentMetadataConfigurationUpdates denganUpdateIndex API, seperti yang Anda lakukan saat membuat bidang cadangan atau built-in. Anda harus mengatur tipe data yang sesuai untuk bidang kustom Anda. Anda tidak dapat mengubah jenis bidang setelah Anda membuat bidang.

Berikut ini adalah jenis yang dapat Anda tetapkan untuk bidang khusus:

  • Tanggal

  • Jumlah

  • String

  • Daftar string

Anda membuat bidang kustom menggunakan konsol atau menggunakan UpdateIndexAPI. Setelah Anda membuat bidang kustom, Anda memetakannya ke atribut dokumen, seperti yang Anda lakukan dengan bidang tersimpan. Jika Anda menambahkan dokumen ke indeks dengan BatchPutDocumentAPI, Anda memetakan atribut dengan API. Untuk dokumen yang diindeks dari sumberAmazon S3 data, Anda memetakan atribut menggunakan file metadata yang berisi struktur JSON yang menggambarkan atribut dokumen. Untuk dokumen yang diindeks dengan database atau sumber data yang memungkinkan pemetaan lapangan, Anda memetakan atribut dengan konsol atau konfigurasi sumber data.
