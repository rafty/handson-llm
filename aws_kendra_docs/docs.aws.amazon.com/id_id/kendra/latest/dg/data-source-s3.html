Amazon S3

Amazon S3adalah layanan penyimpanan objek yang menyimpan data sebagai objek dalam bucket. Anda dapat menggunakanAmazon Kendra untuk mengindeks repositoriAmazon S3 bucket dokumen Anda.

Awas

Amazon Kendratidak menggunakan kebijakan bucket yang memberikan izin kepadaAmazon Kendra prinsipal untuk berinteraksi dengan bucket S3. Sebaliknya, ia menggunakanIAM peran. Pastikan ituAmazon Kendra tidak disertakan sebagai anggota tepercaya dalam kebijakan bucket Anda untuk menghindari masalah keamanan data yang secara tidak sengaja memberikan izin kepada prinsipal arbitrer. Namun, Anda dapat menambahkan kebijakan bucket untuk menggunakanAmazon S3 bucket di berbagai akun. Untuk informasi selengkapnya, lihat Kebijakan yang akan digunakanAmazon S3 di seluruh akun (dalam tabIAM peran S3, di bawah IAMperan untuk sumber data). Untuk informasi tentangIAM peran sumber data S3, lihat IAMperan.

Untuk terhubung keAmazon S3, Anda menentukan koneksi dan informasi lainnya di konsol, dengan menggunakan S3DataSourceConfiguration atau dengan menggunakan TemplateConfiguration. Jika Anda menggunakan TemplateConfiguration objek, Anda dapat menggunakan VPC untuk terhubung ke sumber data Anda. Anda menentukan VpcConfiguration kapan Anda menelepon CreateDataSource. Untuk informasi selengkapnya, lihat Mengkonfigurasi VPC.

catatan

S3DataSourceConfigurationAPI tidak mendukung konfigurasi VPC. HanyaTemplateConfiguration API dan konsol yang mendukung konfigurasi VPC.

Untuk mengatasi masalah konektor sumber dataAmazon Kendra S3 Anda, lihatMengatasi masalah sumber data.

Prasyarat

Sebelum Anda dapat menggunakanAmazon Kendra untuk mengindeks sumber data S3 Anda, buat perubahan ini di S3 danAWS akun Anda.

Dalam S3, pastikan Anda memiliki:

  • Menyalin nama namaAmazon S3 bucket Anda.

    catatan

    Bucket harus berada di wilayah yang sama denganAmazon Kendra indeks Anda dan indeks Anda harus memiliki izin untuk mengakses bucket yang berisi dokumen Anda.

  • Memeriksa setiap dokumen unik di S3 dan di seluruh sumber data lain yang Anda rencanakan untuk digunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

Jika Anda tidak memilikiIAM peran yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran baru saat menghubungkan sumber data S3 Anda keAmazon Kendra. Jika menggunakan API, Anda harus menyediakan ARNIAM peran yang sudah ada dan ID indeks.

Instruksi koneksi

Untuk terhubungAmazon Kendra ke sumber data S3 Anda, Anda harus memberikan rincian yang diperlukan dari sumber data S3 Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum mengkonfigurasi S3 untukAmazon Kendra lihatPrasyarat.

Console

Untuk terhubungAmazon Kendra keAmazon S3

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di halaman Tambahkan sumber data, pilih Konektor S3, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan detail sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan nilai deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Pada halaman Tentukan akses dan keamanan, masukkan informasi opsional berikut:

    1. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensi repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    2. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    3. Pilih Selanjutnya.

  7. Di halaman Konfigurasi pengaturan sinkronisasi, masukkan informasi berikut:

    1. Dalam lingkup Sinkronisasi, untuk Lokasi sumber data —Jalur keAmazon S3 bucket tempat data Anda disimpan. Pilih Browse S3 untuk memilih bucket Anda.

    2. (Opsional) File metadata awalan lokasi folder —Jalur ke folder tempat metadata Anda disimpan. Pilih Browse S3 untuk menemukan folder metadata Anda.

    3. (Opsional) Lokasi file konfigurasi daftar kontrol akses —Jalur ke lokasi file yang berisi struktur JSON yang menentukan pengaturan akses untuk file yang disimpan di sumber data S3 Anda. Pilih Browse S3 untuk menemukan file ACL Anda.

    4. (Opsional) Pilih kunci dekripsi —Pilih untuk menggunakan kunci dekripsi. Anda dapat memilih untuk menggunakanAWS KMS kunci yang sudah ada atau membuat kunci baru.

    5. (Opsional) Dalam konfigurasi tambahan, untuk pola Regex —Tambahkan pola ekspresi reguler untuk menyertakan atau mengecualikan dokumen dari indeks Anda. Semua jalur relatif terhadap lokasi sumber data S3 ember. Anda dapat menambahkan hingga 100 pola.

    6. Dalam mode Sinkronisasi, pilih antara Mode Sinkronisasi penuh dan Sinkronisasi konten baru, dimodifikasi, atau dihapus untuk menentukan bagaimana indeks Anda diperbarui saat konten sumber data Anda berubah. Saat Anda menyinkronkan sumber data dengan Amazon Kendra untuk pertama kalinya, semua konten disinkronkan secara default.

    7. Di Sinkronkan jadwal jalankan, untuk Frekuensi —Pilih seberapa seringAmazon Kendra akan disinkronkan dengan sumber data Anda.

    8. Pilih Selanjutnya.

  8. Pada halaman Set field pemetaan, masukkan informasi opsional berikut:

    1. Pemetaan bidang S3 —Pilih dari bidang sumber data defaultAmazon Kendra yang dihasilkan yang ingin Anda petakan ke indeks Anda.

    2. Tambahkan bidang —Pilih untuk menambahkan bidang sumber data khusus untuk membuat nama bidang indeks untuk dipetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

TemplateConfiguration API

Untuk terhubungAmazon Kendra keAmazon S3

Anda harus menentukan JSON skema sumber data menggunakan TemplateConfigurationAPI. Anda harus memberikan informasi berikut ini:

  • BucketName—Nama bucket yang berisi dokumen.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk konektor S3 danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber data S3.

Anda juga dapat menambahkan fitur opsional berikut:

  • Filter inklusi dan pengecualian —Tentukan apakah akan menyertakan atau mengecualikan nama file tertentu, jenis file, jalur file, dan menggunakan pola glob (pola yang dapat memperluas pola wildcard ke dalam daftar nama jalur yang cocok dengan pola yang diberikan). Contoh pola gumpalan meliputi:

    • /myapp/config/*—Semua file di dalam direktori konfigurasi

    • /**/*.png—Semua file.png di semua direktori

    • /**/*.{png,ico,md}—Semua file.png, .ico atau .md di semua direktori

    • /myapp/src/**/*.ts—Semua file.ts di dalam direktori src (dan semua subdirektorinya)

    • **/!(*.module).ts—Semua file.ts tetapi tidak .module.ts

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak akan diindeks meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data S3 Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration saat Anda meneleponCreateDataSource. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

  • Konfigurasi metadata dokumen —Tambahkan file metadata dokumen yang berisi informasi seperti informasi kontrol akses dokumen, sumber URI, penulis dokumen, dan atribut kustom. Setiap file metadata berisi metadata tentang satu dokumen.

Untuk daftar kunci JSON penting lainnya untuk dikonfigurasi, lihatAmazon S3skema Template.
