Apakah Amazon Kendra itu?

Amazon Kendraadalah layanan pencarian yang sangat akurat dan cerdas yang memungkinkan pengguna Anda untuk mencari data yang tidak terstruktur dan terstruktur menggunakan pemrosesan bahasa alami dan algoritma pencarian lanjutan. Amazon Kendra mengembalikan jawaban spesifik atas pertanyaan, menghadirkan sebuah pengalaman interaksi yang mendekati seorang ahli. Hal ini dapat diskalakan dan mampu memenuhi tuntutan performa, terintegrasi erat denganAWS layanan lainnya seperti Amazon S3dan Amazon Lex, serta menawarkan keamanan sekelas korporasi.

Untuk informasi tentangAmazon Kendra dan operasiAmazon Kendra Intelligent Ranking API, lihat dokumentasi referensi API.

Amazon Kendrapengguna dapat menanyakan jenis pertanyaan, atau kueri berikut:

  • Pertanyaan factoid —Sederhana siapa, apa, kapan, atau di mana pertanyaan, seperti Di mana pusat layanan terdekat ke Seattle? Pertanyaan factoid memiliki jawaban berbasis fakta yang dapat dikembalikan dalam bentuk satu kata atau frasa. Jawabannya diambil dari FAQ atau dari dokumen Anda yang diindeks.

  • Pertanyaan deskriptif —Pertanyaan di mana jawabannya bisa berupa kalimat, kutipan, atau seluruh dokumen. Misalnya, Bagaimana cara menghubungkan Echo Plus saya ke jaringan saya? Atau, Bagaimana cara mendapatkan manfaat pajak untuk keluarga berpenghasilan rendah? .

  • Pencarian kata kunci dan bahasa alami —Pertanyaan yang tidak hanya menyertakan kata kunci untuk dicari tetapi juga pertanyaan percakapan yang rumit. Ini termasuk pertanyaan di mana makna yang dimaksudkan tidak selalu jelas. Misalnya, alamat keynote. Karena 'alamat' seringkali memiliki beberapa arti,Amazon Kendra dapat menyimpulkan makna yang dimaksudkan pengguna di balik kueri pencarian untuk mengembalikan informasi yang relevan. Amazon Kendramenggunakan model deep learning untuk menangani jenis kueri.

Manfaat dariAmazon Kendra

Amazon Kendramemiliki manfaat sebagai berikut:

  • Akurasi —Tidak seperti layanan pencarian tradisional yang menggunakan pencarian kata kunci di mana hasil didasarkan pada pencocokan kata kunci dasar dan peringkat,Amazon Kendra upaya untuk memahami konteks pertanyaan. Amazon Kendramencari di seluruh data Anda dan melampaui penelusuran tradisional untuk mengembalikan kata, kutipan, atau dokumen yang paling relevan untuk kueri Anda. Amazon Kendramenggunakan machine learning untuk meningkatkan hasil pencarian dari waktu ke waktu.

  • Simplicity —Amazon Kendra menyediakan konsol dan API untuk mengelola dokumen yang ingin Anda cari. Anda dapat menggunakan API pencarian sederhana untuk mengintegrasikanAmazon Kendra ke dalam aplikasi klien Anda, seperti situs web atau aplikasi seluler.

  • Konektivitas —Amazon Kendra dapat terhubung ke repositori data pihak ketiga atau sumber data seperti Microsoft SharePoint. Anda dapat dengan mudah mengindeks dan mencari dokumen Anda menggunakan sumber data Anda.

  • Kontrol akses pengguna —Amazon Kendra memberikan pencarian korporasi yang sangat aman untuk aplikasi pencarian Anda. Hasil pencarian Anda mencerminkan model keamanan organisasi Anda. Hasil pencarian dapat disaring berdasarkan pengguna atau akses grup mereka ke dokumen. Pelanggan bertanggung jawab untuk mengautentikasi dan mengotorisasi akses pengguna.

Amazon KendraEdisi Developer

EdisiAmazon Kendra Developer menyediakan semua fitur denganAmazon Kendra biaya lebih rendah. Sudah termasuk tingkat gratis yang menyediakan 750 jam penggunaan. Edisi Developer sangat ideal untuk mengeksplorasi bagaimanaAmazon Kendra mengindeks dokumen Anda, untuk mencoba fitur, dan untuk mengembangkan aplikasi yang menggunakanAmazon Kendra.

Edisi Developer menyediakan berikut hal-hal berikut ini:

  • Hingga 5 indeks dengan masing-masing 5 sumber data.

  • 10,000 dokumen atau 3 GB teks yang diekstrak.

  • Sekitar 4.000 kueri per hari atau 0,05 kueri per detik.

  • Berjalan di 1 availability zone (AZ) —lihat Availability Zone (pusat data diAWS wilayah)

Anda tidak harus menggunakan edisi developer untuk aplikasi produksi. Edisi Developer tidak memberikan jaminan latensi atau ketersediaan.

Amazon KendraEdisi Enterprise Edition

GunakanAmazon Kendra Enterprise ketika Anda ingin mengindeks seluruh pustaka dokumen perusahaan Anda atau ketika aplikasi Anda siap untuk digunakan dalam lingkungan produksi.

Edisi Enterprise menyediakan hal-hal berikut ini:

  • Hingga 5 indeks dengan masing-masing hingga 50 sumber data.

  • 100.000 dokumen atau 30 GB teks yang diekstrak.

  • Sekitar 8.000 kueri per hari atau 0,1 kueri per detik.

  • Berjalan di 3 availability zone (AZ) —lihat Availability Zone (pusat data diAWS wilayah)

Anda dapat meningkatkan kuota ini menggunakan Konsol Service Quotas.

Harga untuk Amazon Kendra

Anda bisa memulai secara gratis dengan EdisiAmazon Kendra Developer yang menyediakan penggunaan hingga 750 jam untuk 30 hari pertama. Setelah uji coba berakhir, Anda akan ditagih untuk semuaAmazon Kendra indeks yang ditetapkan, meskipun mereka kosong dan tidak ada kueri yang dieksekusi. Setelah uji coba berakhir, ada biaya tambahan untuk memindai dan menyinkronkan dokumen menggunakan sumberAmazon Kendra data.

Apakah Anda pengguna Amazon Kendra baru?

Jika Anda baru pertama kali menggunakanAmazon Kendra, kami merekomendasikan agar Anda membaca bagian-bagian berikut secara berurutan:

  1. Bagaimana Amazon Kendra berhasil—MemperkenalkanAmazon Kendra komponen dan menjelaskan bagaimana Anda menggunakannya untuk membuat solusi pencarian.

  2. Mulai—Menjelaskan cara mengatur akun Anda dan menguji APIAmazon Kendra pencarian.

  3. Membuat indeks—Menjelaskan cara menggunakanAmazon Kendra untuk membuat indeks pencarian dan menambahkan sumber data untuk menyinkronkan dokumen Anda.

  4. Menambahkan dokumen langsung ke indeks—Menjelaskan cara menambahkan dokumen langsung keAmazon Kendra indeks.

  5. Indeks pencarian—Menjelaskan cara menggunakan APIAmazon Kendra pencarian untuk mencari indeks.

  6. Penerapan Amazon Kendra—Menyediakan contoh aplikasi yang dapat Anda gunakan untuk men-deployAmazon Kendra ke situs web Anda.
