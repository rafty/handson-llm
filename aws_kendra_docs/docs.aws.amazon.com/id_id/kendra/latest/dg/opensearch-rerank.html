Amazon KendraPeringkat Cerdas untuk dikelola sendiri OpenSearch

Anda dapat memanfaatkanAmazon Kendra kemampuan pencarian semantik untuk meningkatkan hasil pencarian dari OpenSearch, layanan pencarian open source yang dikelola sendiri berdasarkan Lisensi Apache 2.0. PluginAmazon Kendra Intelligent Ranking secara semantik kembali peringkat OpenSearch hasil menggunakanAmazon Kendra. Hal ini dilakukan dengan memahami arti dan konteks permintaan pencarian menggunakan bidang tertentu, seperti badan dokumen atau judul, dari hasil OpenSearch pencarian default.

Ambil, misalnya, kueri ini: “keynote address utama”. Karena 'alamat' memiliki beberapa arti,Amazon Kendra dapat menyimpulkan arti di balik kueri untuk mengembalikan informasi yang relevan sesuai dengan makna yang dimaksudkan. Dalam hal ini, ini adalah alamat utama konferensi. Layanan pencarian yang lebih sederhana mungkin tidak memperhitungkan maksud dan mungkin dapat mengembalikan hasil untuk alamat jalan di Main Street, misalnya.

Plugin Peringkat Cerdas untuk OpenSearch tersedia untuk OpenSearch (dikelola sendiri) versi 2.4.0 dan yang lebih baru. Anda dapat menginstal plugin menggunakan skrip Bash mulai cepat untuk membangun gambar Docker baru OpenSearch dengan plugin Peringkat Cerdas disertakan. LihatMenyiapkan plugin pencarian cerdas —ini adalah contoh pengaturan untuk membuat Anda bangun dan berjalan dengan cepat.

Cara kerja plugin pencarian cerdas

Proses keseluruhan plugin Intelligent Ranking untuk OpenSearch (dikelola sendiri) adalah sebagai berikut:

  1. OpenSearch Pengguna mengeluarkan kueri, dan OpenSearch memberikan respons kueri atau daftar dokumen yang relevan dengan kueri.

  2. Plugin Intelligent Ranking mengambil respons kueri dan mengekstrak informasi dari dokumen.

  3. Plugin Intelligent Ranking melakukan panggilan ke API ResoreAmazon Kendra Intelligent Ranking.

  4. RescoreAPI mengambil informasi yang diekstrak dari dokumen dan secara semantik memberi peringkat ulang hasil pencarian.

  5. RescoreAPI mengirimkan hasil pencarian peringkat ulang kembali ke plugin. Plugin mengatur ulang hasil pencarian dalam respons OpenSearch pencarian untuk mencerminkan peringkat semantik baru.

Plugin Intelligent Ranking memberi peringkat ulang hasil menggunakan bidang “body” dan “title”. Bidang plugin ini dapat dipetakan ke bidang dalam OpenSearch indeks Anda yang paling sesuai dengan definisi badan dokumen dan judul. Misalnya, jika indeks Anda berisi chapter buku dengan bidang seperti “chapter_heading” dan “chapter_contents”, Anda dapat memetakan yang pertama ke “title” dan yang terakhir ke “body” untuk mendapatkan hasil terbaik.

Menyiapkan OpenSearch (dikelola sendiri) dengan plugin Intelligent Ranking (pengaturan cepat)

Jika Anda sudah menggunakan gambar Dockeropensearch:2.4.0, Anda dapat menggunakan Dockerfile ini untuk membangun gambar baru OpenSearch 2.4.0 dengan plugin Intelligent Ranking. Anda menyertakan wadah untuk gambar baru dalam file docker-compose.yml-atau file opensearch.ymlmu. Anda juga menyertakan ID rencana eksekusi rescore yang dihasilkan dari membuat rencana eksekusi rescore, bersama dengan informasi wilayah dan titik akhir Anda—lihat langkah 2 untuk membuat rencana eksekusi rescore.

Jika sebelumnya Anda telah mengunduh versi imageopensearch Docker yang lebih lama dari 2.4.0, Anda harus menggunakan gambar Dockeropensearch:2.4.0 atau yang lebih baru dan membuat gambar baru dengan plugin Intelligent Ranking yang disertakan.

  1. Unduh dan instal Docker Desktop untuk sistem operasi Anda. Docker Desktop termasuk Docker Compose dan Docker Engine. Dianjurkan agar Anda memeriksa apakah komputer Anda memenuhi persyaratan sistem yang disebutkan dalam rincian instalasi Docker.

    Anda juga dapat meningkatkan persyaratan penggunaan memori Anda dalam pengaturan Desktop Docker Anda. Anda bertanggung jawab atas persyaratan penggunaan Docker di luar batas penggunaan yang tersedia secara bebas untuk layanan Docker. Lihat langganan Docker.

    Periksa status Docker Desktop adalah “berjalan”.

  2. Menyediakan PeringkatAmazon Kendra Cerdas dan persyaratan kapasitas Anda. Setelah Anda menyediakan PeringkatAmazon Kendra Cerdas, Anda akan dikenakan biaya per jam berdasarkan unit kapasitas yang Anda tetapkan. Lihat informasi tingkat dan harga gratis.

    Anda menggunakan CreateRescoreExecutionPlanAPI untuk menyediakanRescore API. Jika Anda tidak memerlukan unit kapasitas lebih dari satu unit default, jangan tambahkan lebih banyak unit dan berikan hanya nama untuk rencana eksekusi rescore Anda. Anda juga dapat memperbarui persyaratan kapasitas Anda dengan menggunakan UpdateRescoreExecutionPlanAPI. Untuk informasi selengkapnya, lihat secara semantik memberi peringkat hasil layanan penelusuran.

    Secara opsional, Anda dapat pergi ke langkah 3 untuk membuat rencana eksekusi rescore default ketika Anda menjalankan skrip Bash mulai cepat.

    Catatan untuk langkah 4 ID rencana eksekusi rescore yang disertakan dalam respons.

    CLI
    aws kendra-ranking create-rescore-execution-plan \
  --name MyRescoreExecutionPlan \ 
  --capacity-units '{"RescoreCapacityUnits":<integer number of additional capacity units>}'
 
Response:
 
{
    "Id": "<rescore execution plan ID>",
    "Arn": "arn:aws:kendra-ranking:<region>:<account-id>:rescore-execution-plan/<rescore-execution-plan-id>"
}
    Python
    import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra_ranking = boto3.client("kendra-ranking")

print("Create a rescore execution plan.")

# Provide a name for the rescore execution plan
name = "MyRescoreExecutionPlan"
# Set your required additional capacity units
# Don't set capacity units if you don't require more than 1 unit given by default
capacity_units = 1

try:
    rescore_execution_plan_response = kendra_ranking.create_rescore_execution_plan(
        Name = name,
        CapacityUnits = {"RescoreCapacityUnits":capacity_units}
    )

    pprint.pprint(rescore_execution_plan_response)

    rescore_execution_plan_id = rescore_execution_plan_response["Id"]

    print("Wait for Amazon Kendra to create the rescore execution plan.")

    while True:
        # Get the details of the rescore execution plan, such as the status
        rescore_execution_plan_description = kendra_ranking.describe_rescore_execution_plan(
            Id = rescore_execution_plan_id
        )
        # When status is not CREATING quit.
        status = rescore_execution_plan_description["Status"]
        print(" Creating rescore execution plan. Status: "+status)
        time.sleep(60)
        if status != "CREATING":
            break

except ClientError as e:
        print("%s" % e)

print("Program ends.")

    Saat menjalankan perintah ini, Anda memberikan ID rencana eksekusi rescore yang Anda catat di langkah 2 saat Anda menyediakan PeringkatAmazon Kendra Cerdas, bersama dengan informasi wilayah Anda. Secara opsional, Anda dapat menyediakan PeringkatAmazon Kendra Cerdas dengan menggunakan--create-execution-plan opsi. Ini menciptakan rencana eksekusi rescore dengan nama default dan kapasitas default. JikaAWS kredensyal tidak dapat diperoleh baik dari variabel lingkungan, profil default, atauAmazon EC2 instance, maka berikan kredensyal Anda menggunakan--profile opsi tersebut.

    Untuk tidak kehilangan indeks Anda ketika kontainer sementara default dihapus, Anda dapat memiliki indeks Anda tetap di seluruh eksekusi dengan memberikan nama volume data menggunakan--volume-name opsi. Jika sebelumnya Anda membuat indeks, Anda dapat menentukan volume di file docker-compose.yml atau opensearch.yml-mu.

    Skrip ini menggunakan gambar Docker untuk OpenSearch dan OpenSearch Dasbor menggunakan versi yang Anda pilih pada GitHub repositori skrip. Ini mengunduh file zip untuk plugin Intelligent Ranking, dan menghasilkanDockerfile untuk membangun citra Docker baru OpenSearch yang mencakup plugin. Ini juga membuat file docker-compose.ymlyang mencakup wadah untuk OpenSearch plugin dan OpenSearch Dasbor Peringkat Cerdas. Skrip menambahkan ID rencana eksekusi rescore Anda, informasi wilayah, dan titik akhir (menggunakan wilayah) ke file docker-compose.yml. Script kemudian berjalandocker-compose up untuk memulai wadah untuk OpenSearch dengan Intelligent Ranking disertakan dan OpenSearch Dasbor. Untuk menghentikan wadah tanpa mengeluarkannya, jalankandocker-compose stop. Untuk menghapus wadah, jalankandocker-compose down. Untuk membiarkan volume Anda utuh, jangan laridocker-compose down -v.

Berinteraksi dengan plugin pencarian cerdas

Setelah Anda mengatur OpenSearch (dikelola sendiri) dengan plugin Intelligent Ranking, Anda dapat berinteraksi dengan plugin menggunakan perintah curl atau pustaka OpenSearch klien. Kredensyal default untuk mengakses OpenSearch dengan plugin Intelligent Ranking adalah nama pengguna 'admin' dan kata sandi 'admin'.

Untuk menerapkan pengaturan plugin Intelligent Ranking ke OpenSearch indeks:

                "title_field": "title_field_name_here",
                "body_field": "body_field_name_here"
              }
          }
        }
      }
    }
  }
}

Membandingkan OpenSearch hasil denganAmazon Kendra hasil

Anda dapat membandingkanAmazon Kendra hasil peringkat side-by-side OpenSearch (dikelola sendiri) dengan hasil peringkat ulang. OpenSearch Dasbor versi 2.4.0 dan yang lebih baru menawarkan side-by-side hasil sehingga Anda dapat membandingkan bagaimana OpenSearch peringkat dokumen dengan bagaimanaAmazon Kendra atau plugin peringkat dokumen untuk permintaan pencarian.

Sebelum Anda dapat membandingkan hasil OpenSearch peringkat dengan hasilAmazon Kendra peringkat ulang, pastikan OpenSearch Dasbor Anda didukung oleh OpenSearch server dengan plugin Peringkat Cerdas. Anda dapat mengatur ini menggunakan Docker dan cepat mulai skrip Bash. Lihat Menyiapkan plugin pencarian cerdas.

Berikut ini menguraikan cara membandingkan OpenSearch danAmazon Kendra mencari hasil di OpenSearch Dasbor. Untuk informasi selengkapnya, lihat Dokumentasi OpenSearch .

Membandingkan hasil pencarian di OpenSearch Dasbor

  1. Buka http://localhost:5601 dan masuk ke OpenSearch Dasbor. Kredensyal default adalah nama pengguna 'admin' dan kata sandi 'admin'.

  2. Pilih Relevansi Pencarian dari OpenSearch plugin di menu navigasi.

  3. Masukkan teks pencarian ke bilah pencarian.

  4. Pilih indeks Anda untuk Query 1 dan masukkan kueri di DSL OpenSearch Query. Anda dapat menggunakan%SearchText% variabel untuk merujuk ke teks pencarian yang Anda masukkan di bilah pencarian. Untuk contoh kueri ini, lihat OpenSearch Dokumentasi. Hasil yang dikembalikan untuk kueri ini adalah OpenSearch hasil tanpa menggunakan plugin Intelligent Ranking.

  5. Pilih indeks yang sama untuk Query 2 dan masukkan query yang sama di OpenSearch Query DSL. Selain itu, sertakan ekstensi dengankendra_intelligent_ranking dan tentukan wajibbody_field untuk peringkat. Anda juga dapat menentukan bidang judul, tetapi bidang tubuh adalah wajib. Untuk contoh kueri ini, lihat OpenSearch Dokumentasi. Hasil yang dikembalikan untuk kueri ini adalah hasilAmazon Kendra peringkat ulang menggunakan plugin Intelligent Ranking. Plugin peringkat hingga 25 hasil.

  6. Pilih Cari untuk mengembalikan dan membandingkan hasil.
