GitHub

GitHub adalah layanan hosting berbasis web untuk pengembangan perangkat lunak yang menyediakan penyimpanan kode dan layanan manajemen dengan kontrol versi. Anda dapat menggunakannyaAmazon Kendra untuk mengindeks file repositori GitHub GitHub Enterprise Cloud (SaaS) dan Enterprise Server (On Prem), permintaan masalah dan tarik, mengeluarkan dan menarik komentar, serta mengeluarkan dan menarik lampiran komentar permintaan. Anda juga dapat memilih untuk menyertakan atau mengecualikan file tertentu.

Anda dapat terhubungAmazon Kendra ke sumber GitHub data Anda menggunakan Amazon Kendrakonsol dan GitHubConfigurationAPI.

Untuk mengatasi masalah konektor sumberAmazon Kendra GitHub data Anda, lihatMengatasi masalah sumber data.

Fitur yang didukung

Amazon Kendra GitHub konektor sumber data mendukung fitur berikut:

  • Perubahan

  • Pemetaan lapangan

  • Pemfilteran konteks

  • Filter inklusi

Prasyarat

Sebelum Anda dapat menggunakanAmazon Kendra untuk mengindeks sumber GitHub data Anda, buat perubahan ini di akun Anda GitHub danAWS akun.

Dalam GitHub, pastikan Anda memiliki:

  • Membuat GitHub pengguna dengan izin administratif untuk GitHub organisasi.

  • Membuat token akses pribadi untuk kredensi otentikasi. Lihat GitHub dokumentasi tentang membuat token akses pribadi.

  • Direkomendasikan: Membuat token OAuth untuk kredensi otentikasi. Gunakan token OAuth untuk batas throttle API yang lebih baik dan kinerja konektor. Lihat GitHub dokumentasi tentang otorisasi OAuth.

  • Opsional: Menginstal sertifikat SSL.

  • Mencatat URL GitHub host untuk jenis GitHub layanan yang Anda gunakan. Misalnya, URL host untuk GitHub cloud bisa berupa https://api.github.com dan URL host untuk GitHub server bisa berupa https://on-prem-host-url/api/v3/.

  • Mencatat nama GitHub organisasi untuk respositori Anda dari GitHub pengaturan Anda.

  • Ditambahkan izin berikut:

    Untuk GitHub Enterprise Cloud (SaaS)

    • repo: status

    • public_repo

    • repo: mengundang

    • baca: org

    • pengguna: email

    • baca: pengguna

    Untuk Server GitHub Perusahaan (Di Prem)

    • repo: status

    • public_repo

    • repo: mengundang

    • baca: org

    • pengguna: email

    • baca: pengguna

    • site_admin

  • Memeriksa setiap dokumen unik di dalam GitHub dan di seluruh sumber data lain yang Anda rencanakan untuk digunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensyal GitHub autentikasi AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru saat menghubungkan sumber GitHub dataAmazon Kendra. Jika Anda menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia, dan ID indeks.

Petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber GitHub data Anda, Anda harus memberikan rincian yang diperlukan dari sumber GitHub data Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum dikonfigurasi GitHub untukAmazon Kendra lihatPrasyarat.

Console

Untuk terhubungAmazon Kendra ke GitHub

  1. Masuk ke KonsolAWS Manajemen dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi yang ada di sebelah kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di Tambahkan sumber data halaman, pilih GitHub konektor, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Di halaman Tetapkan akses dan keamanan, masukkan informasi berikut:

    1. GitHubsumber —Pilih antara GitHub Enterprise Cloud dan GitHubEnterprise Server.

    2. GitHub URL host —Masukkan nama GitHub host Anda.

    3. GitHub nama organisasi —Masukkan nama GitHub organisasi Anda. Anda dapat menemukan informasi organisasi di GitHub akun.

    4. AWSSecrets Managersecret —Pilih rahasia yang ada atau buatSecrets Manager rahasia baru untuk menyimpan kredensyal GitHub autentikasi Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      1. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-GitHub -' secara otomatis ditambahkan ke nama rahasia Anda.

        2. Untuk GitHubtoken —Masukkan nilai kredensi otentikasi yang Anda buat di GitHub akun Anda.

      2. Pilih Save (Simpan).

    5. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    6. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensyal repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    7. Pilih Selanjutnya.

  7. Di halaman Konfigurasi pengaturan, masukkan informasi berikut:

    1. Pilih repositori untuk di-crawl GitHub —Entitas atau jenis konten yang ingin Anda jelajahi.

    2. Ubah log —Pilih untuk memperbarui indeks Anda alih-alih menyinkronkan semua file Anda.

    3. Jenis konten -Pilih untuk jenis file yang ingin Anda sertakan.

    4. Pola regex —Pola ekspresi reguler untuk menyertakan atau mengecualikan file tertentu. Anda dapat menambahkan hingga 100 pola.

    5. Di Sinkronisasi jadwal jalankan, untuk Frekuensi —Seberapa sering Amazon Kendra akan disinkronkan dengan sumber data Anda.

    6. Pilih Selanjutnya.

  8. Di halaman Set field, masukkan informasi berikut:

    1. Untuk Repository, Respository Commit, Issue Document, Issue Comment, Issue Attachment, Pull Request Comment, Pull Request Attachment —Select from theAmazon Kendra generated default data source fields you want to map to your index.

    2. Tambahkan bidang —Untuk menambahkan bidang sumber data kustom untuk membuat nama bidang indeks untuk memetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk terhubungAmazon Kendra ke GitHub

Anda harus menentukan yang berikut menggunakan objek GitHubConfigurationAPI:

  • Jenis sumber data —Tentukan tipe sumber data sebagai salah satuSAAS atauON_PREMISE.

  • Secret Amazon Resource Name (ARN) —Berikan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensyal autentikasi untuk GitHub akun Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "personalToken": "token"
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensi dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk GitHub konektor danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber GitHub data.

Anda juga dapat menambahkan fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration sebagai bagian dari konfigurasi sumber data. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

    catatan

    Jika Anda menggunakan GitHub server, Anda harus menggunakanAmazon VPC untuk terhubung ke GitHub server Anda.

  • Perubahan log —ApakahAmazon Kendra harus menggunakan mekanisme log perubahan sumber GitHub data untuk menentukan apakah dokumen harus ditambahkan, diperbarui, atau dihapus dalam indeks.

    catatan

    Gunakan log perubahan jika tidakAmazon Kendra ingin memindai semua dokumen. Jika log perubahan Anda berukuran besar, mungkin butuh waktuAmazon Kendra lebih sedikit untuk memindai dokumen di sumber GitHub data daripada untuk memproses log perubahan. Jika Anda menyinkronkan sumber GitHub data dengan indeks untuk kali, semua dokumen akan dipindai.

  • Filter inklusi dan pengecualian —Tentukan apakah akan menyertakan atau mengecualikan file tertentu.

    catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak diindeks, meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber GitHub data Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

  • Pemfilteran konteks pengguna -Amazon Kendra merayapi Access Control List (ACL) untuk sumber data Anda secara default. Informasi ACL digunakan untuk memfilter hasil pencarian berdasarkan pengguna atau akses grup ke dokumen. Untuk informasi selengkapnya, lihat Pemfilteran konteks pengguna untuk sumber GitHub data.
