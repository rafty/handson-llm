Zendesk

Zendesk adalah sistem manajemen hubungan pelanggan yang membantu bisnis mengotomatisasi dan meningkatkan interaksi dukungan pelanggan. Anda dapat menggunakanAmazon Kendra untuk mengindeks tiket dukungan Zendesk Anda, komentar tiket, lampiran tiket, artikel pusat bantuan, komentar artikel, lampiran komentar artikel, topik komunitas panduan, posting komunitas, dan komentar posting komunitas.

Anda dapat memfilter berdasarkan nama organisasi jika Anda ingin mengindeks tiket yang hanya ada dalam organisasi tertentu. Anda juga dapat memilih untuk mengatur tanggal perayapan kapan Anda ingin mulai merangkak data dari Zendesk.

Anda dapat terhubungAmazon Kendra ke sumber data Zendesk Anda menggunakan Amazon Kendrakonsol dan TemplateConfigurationAPI.

Untuk mengatasi masalah konektor sumber dataAmazon Kendra Zendesk Anda, lihatMengatasi masalah sumber data.

Prasyarat

Sebelum Anda dapat menggunakanAmazon Kendra untuk mengindeks sumber data Zendesk Anda, buat perubahan ini di Zendesk danAWS akun Anda.

Di Zendesk, pastikan Anda memiliki:

  • Membuat akun administratif Zendesk Suite (Professional/Enterprise).

  • Mencatat URL host Zendesk Anda. Misalnya, https://{sub-domain (https://{host/)}.zendesk.com/.

  • Menghasilkan token kredenisasi OAuth 2.0 yang berisi ID klien, rahasia klien, nama pengguna, dan kata sandi. Lihat dokumentasi Zendesk tentang pembuatan token OAuth 2.0 untuk informasi lebih lanjut.

  • Menambahkan cakupan OAuth 2.0 berikut:

    • baca

  • Opsional: Menginstal sertifikat SSLAmazon Kendra untuk memungkinkan tersambung.

  • Memeriksa setiap dokumen unik di Zendesk dan di seluruh sumber data lain yang Anda rencanakan untuk digunakan untuk indeks yang sama. Setiap sumber data yang ingin Anda gunakan untuk indeks tidak boleh berisi dokumen yang sama di seluruh sumber data. ID dokumen bersifat global untuk indeks dan harus unik per indeks.

DiAWS akun Anda, pastikan Anda memiliki:

  • MembuatAmazon Kendra indeks dan, jika menggunakan API, mencatat ID indeks.

  • MembuatIAM peran untuk sumber data Anda dan, jika menggunakan API, mencatat ARNIAM peran tersebut.

  • Menyimpan kredensi otentikasi Zendesk AndaAWSSecrets Manager secara rahasia dan, jika menggunakan API, mencatat ARN rahasianya.

    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

Jika Anda tidak memilikiIAM peran atau rahasia yang ada, Anda dapat menggunakan konsol untuk membuatIAM peran danSecrets Manager rahasia baru ketika Anda menghubungkan sumber data Zendesk Anda keAmazon Kendra. Jika Anda menggunakan API, Anda harus menyediakan ARNIAM peran danSecrets Manager rahasia yang ada, dan ID indeks.

Petunjuk koneksi

Untuk terhubungAmazon Kendra ke sumber data Zendesk Anda, Anda harus memberikan rincian yang diperlukan dari sumber data Zendesk Anda sehinggaAmazon Kendra dapat mengakses data Anda. Jika Anda belum dikonfigurasi Zendesk untukAmazon Kendra melihatPrasyarat.

Console

Untuk terhubungAmazon Kendra ke Zendesk

  1. Masuk keAWS Management Console dan buka Amazon Kendrakonsol.

  2. Dari panel navigasi kiri, pilih Indeks dan kemudian pilih indeks yang ingin Anda gunakan dari daftar indeks.

    catatan

    Anda dapat memilih untuk mengkonfigurasi atau mengedit pengaturan kontrol akses Pengguna Anda di bawah Pengaturan indeks.

  3. Di halaman Memulai, pilih Tambahkan sumber data.

  4. Di halaman Tambahkan sumber data, pilih Konektor Zendesk, lalu pilih Tambahkan sumber data.

  5. Di halaman Tentukan sumber data, masukkan informasi berikut:

    1. Di Nama dan deskripsi, untuk Nama sumber data —Masukkan nama untuk sumber data Anda. Anda dapat menyertakan tanda hubung tetapi tidak spasi.

    2. (Opsional) Deskripsi —Masukkan deskripsi opsional untuk sumber data Anda.

    3. Dalam Bahasa, untuk Bahasa default —Bahasa untuk memfilter dokumen Anda untuk indeks. Kecuali Anda menentukan sebaliknya, bahasa default ke bahasa Inggris. Bahasa yang ditentukan dalam metadata menimpa bahasa yang dipilih.

    4. Di Tag, untuk Tambahkan tag baru —Tag untuk mencari dan memfilter sumber daya Anda atau melacakAWS biaya Anda.

    5. Pilih Selanjutnya.

  6. Pada halaman Tentukan akses dan keamanan, masukkan informasi berikut:

    1. URL Zendesk —Masukkan URL Zendesk Anda.

    2. AWSSecrets Managersecret —Pilih rahasia yang ada atau buatSecrets Manager rahasia baru untuk menyimpan kredensi otentikasi Zendesk Anda. Jika Anda memilih untuk membuat rahasia baru, jendelaAWSSecrets Manager rahasia akan terbuka.

      1. Masukkan informasi berikut di jendela BuatAWSSecrets Manager rahasia:

        1. Nama rahasia —Sebuah nama untuk rahasia Anda. Awalan 'AmazonKendra-Zendesk-' secara otomatis ditambahkan ke nama rahasia Anda.

        2. Untuk ID Klien, Rahasia klien, Nama pengguna, Kata Sandi —Masukkan nilai kredensi otentikasi yang Anda buat di akun Zendesk Anda.

      2. Pilih Save (Simpan).

    3. Virtual Private Cloud (VPC) —Anda dapat memilih untuk menggunakan VPC. Jika demikian, Anda harus menambahkan grup keamanan Subnet dan VPC.

    4. Peran IAM —Pilih peran IAM yang ada atau buat peran IAM baru untuk mengakses kredensi repositori dan konten indeks Anda.

      catatan

      Peran IAM yang digunakan untuk indeks tidak dapat digunakan untuk sumber data. Jika Anda tidak yakin apakah peran yang ada digunakan untuk indeks atau FAQ, pilih Buat peran baru untuk menghindari kesalahan.

    5. Pilih Selanjutnya.

  7. Pada halaman Konfigurasi pengaturan, masukkan informasi berikut:

    1. Pilih entitas atau jenis konten —Entitas Zendesk atau jenis konten yang ingin Anda jelajahi.

    2. Ubah log —Pilih untuk memperbarui indeks Anda alih-alih menyinkronkan semua file Anda.

    3. Nama organisasi —Masukkan nama organisasi Zendesk untuk menyaring sinkronisasi Anda.

    4. Sinkronkan tanggal mulai —Tanggal dari mana Anda ingin mengindeks konten Anda.

    5. Pola regex —Reguler ekspresi untuk menyertakan atau mengecualikan file tertentu. Anda dapat menambahkan hingga 100 pola.

    6. Di Sinkronisasi jalankan jadwal untuk Frekuensi —Pilih seberapa seringAmazon Kendra akan disinkronkan dengan sumber data Anda.

    7. Pilih Selanjutnya.

  8. Pada halaman Set field, masukkan informasi berikut:

    1. Untuk Tiket, Komentar tiket, Lampiran komentar tiket, Artikel, Komentar artikel, Lampiran komentar artikel, Topik komunitas, Posting komunitas, Komentar posting komunitas —Pilih dari bidang sumber data defaultAmazon Kendra yang dihasilkan yang ingin Anda petakan ke indeks Anda.

    2. Tambahkan bidang —Untuk menambahkan bidang sumber data kustom untuk membuat nama bidang indeks untuk memetakan ke dan tipe data bidang.

    3. Pilih Selanjutnya.

  9. Pada halaman Tinjau dan buat, periksa apakah informasi yang Anda masukkan sudah benar lalu pilih Tambahkan sumber data. Anda juga dapat memilih untuk mengedit informasi Anda dari halaman ini. Sumber data Anda akan muncul di halaman Sumber data setelah berhasil ditambahkan.

API

Untuk terhubungAmazon Kendra ke Zendesk

Anda harus menentukan JSON skema sumber data menggunakan TemplateConfigurationAPI. Anda harus memberikan informasi berikut ini:

  • Sumber data —Tentukan sumber data sebagaiZENDESK.

  • URL Host —Berikan URL host Zendesk Anda sebagai bagian dari konfigurasi koneksi atau detail endpoint repositori. Misalnya, https://yoursubdomain.zendesk.com.

  • KetikTEMPLATE —Specify sebagai Type saat Anda meneleponCreateDataSource.

  • Rahasia Amazon Resource Name (ARN) —Menyediakan Amazon Resource Name (ARN)Secrets Manager rahasia yang berisi kredensi otentikasi untuk akun Zendesk Anda. Rahasianya disimpan dalam struktur JSON dengan kunci berikut:

    {
    "hostUrl": "https://yoursubdomain.zendesk.com",
    "clientId": "kendra",
    "clientSecret": "Zendesk client secret",
    "userName": "Zendesk user name",
    "password": "Zendesk password"
}
    catatan

    Pastikan untuk secara teratur menyegarkan atau memutar kredensyal dan rahasia Anda. Berikan hanya tingkat akses yang diperlukan untuk keamanan Anda sendiri. Menggunakan kembali kredensi dan rahasia di seluruh sumber data, dan konektor versi 1.0 dan 2.0 (jika berlaku), tidak disarankan.

  • Peran IAM —TentukanRoleArn saat Anda meneleponCreateDataSource untuk memberikanIAM peran dengan izin untuk mengaksesSecrets Manager rahasia Anda dan untuk memanggil API publik yang diperlukan untuk konektor Zendesk danAmazon Kendra. Untuk informasi selengkapnya, lihat peran IAM untuk sumber data Zendesk.

Anda juga dapat menambahkan fitur-fitur opsional berikut:

  • Virtual Private Cloud (VPC) —TentukanVpcConfiguration saat Anda meneleponCreateDataSource. Lihat MengkonfigurasiAmazon Kendra untuk menggunakan VPC.

  • Ubah log —ApakahAmazon Kendra harus menggunakan mekanisme log perubahan sumber data Zendesk untuk menentukan apakah dokumen harus ditambahkan, diperbarui, atau dihapus dalam indeks.

    catatan

    Gunakan log perubahan jika tidakAmazon Kendra ingin memindai semua dokumen. Jika log perubahan Anda berukuran besar, mungkin butuh waktuAmazon Kendra lebih sedikit untuk memindai dokumen di sumber data Zendesk daripada untuk memproses log Jika Anda menyinkronkan sumber data Zendesk dengan indeks untuk kali pertama, semua dokumen akan dipindai.

  • Filter inklusi dan pengecualian —Tentukan apakah akan menyertakan atau mengecualikan:

    • Tiket Support, komentar tiket, dan/atau lampiran komentar tiket

    • Artikel pusat bantuan, lampiran artikel, dan komentar artikel

    • Membimbing topik komunitas, posting, atau posting komentar

  • catatan

    Sebagian besar sumber data menggunakan pola ekspresi reguler, yang merupakan pola inklusi atau pengecualian yang disebut sebagai filter. Jika Anda menentukan filter inklusi, hanya konten yang cocok dengan filter inklusi yang diindeks. Dokumen apa pun yang tidak cocok dengan filter inklusi tidak diindeks. Jika Anda menetapkan filter penyertaan dan pengecualian, dokumen yang cocok dengan filter pengecualian tidak akan diindeks, meskipun cocok dengan filter penyertaan.

  • Pemetaan lapangan —Pilih untuk memetakan bidang sumber data Zendesk Anda ke bidangAmazon Kendra indeks Anda. Untuk informasi lebih lanjut, lihat Memetakan bidang sumber data.

  • Pemfilteran konteks pengguna -Amazon Kendra merayapi Access Control List (ACL) untuk sumber data Anda secara default. Informasi ACL digunakan untuk memfilter hasil pencarian berdasarkan pengguna atau akses grup ke dokumen. Untuk informasi selengkapnya, lihat Pemfilteran konteks pengguna untuk sumber data Zendesk.

Untuk daftar kunci JSON penting lainnya untuk dikonfigurasi, lihatSkema Template Zendesk.
