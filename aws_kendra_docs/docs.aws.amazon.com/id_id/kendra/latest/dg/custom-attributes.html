Membuat bidang dokumen kustom

Anda dapat menerapkan bidang metadata kustom atau bidang metadata yang relevan dengan dokumen tertentu Anda. Misalnya, Anda dapat membuat bidang kustom atau atribut bernama “Departemen” dengan nilai “HR”, “Penjualan”, dan “Manufaktur”. Anda dapat menggunakan bidang tersebut atau atribut untuk membatasi respons (hasil pencarian) ke dokumen di departemen “HR”, misalnya.

Anda dapat membuat hingga 500 bidang kustom atau atribut.

Untuk sebagian besar sumber data, petakan bidang di sumber data eksternal ke bidang yang sesuai diAmazon Kendra. Untuk informasi selengkapnya, lihat Memetakan bidang sumber data. Untuk sumber data S3, Anda dapat menerapkan bidang atau atribut khusus menggunakan file metadata.

Sebelum menggunakan bidang kustom atau atribut, Anda harus terlebih dahulu membuat bidang dalam indeks. Gunakan konsol tersebut atau UpdateIndexAPI untuk membuat bidang indeks. Anda tidak dapat mengubah tipe data bidang setelah Anda membuat bidang.

Anda juga dapat menggunakanAmazon Kendra bidang bawaan atau cadangan.

DenganUpdateIndex API tersebut, tambahkan bidang kustom atau atribut menggunakanDocumentMetadataConfigurationUpdates parameter.

Contoh JSON berikut menggunakanDocumentMetadataConfigurationUpdates untuk menambahkan bidang bernama “Departemen” ke indeks.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Menggunakan bidang dokumen bawaan Amazon Kendra

Dengan UpdateIndex API, Anda dapat membuat bidang cadangan atau built-in menggunakanDocumentMetadataConfigurationUpdates dan menentukan nama bidang cadangan untuk memetakan ke atribut dokumen atau bidang dokumen yang setara. Anda juga dapat membuat bidang kustom dengan cara ini. Jika Anda menggunakan konektor sumber data, sebagian besar menyertakan pemetaan bidang yang memetakan bidang dokumen sumber data Anda ke bidangAmazon Kendra indeks. Ketika Anda membuat bidang, Anda dapat mengkonfigurasiSearch objek untuk mengatur bidang sebagai ditampilkan, facetable, dicari, dan diurutkan. Anda dapat mengkonfigurasiRelevance objek untuk mengatur relevansi bidang meningkatkan durasi, kesegaran, kepentingan, urutan peringkat, dan nilai-nilai kepentingan. Anda tidak dapat mengubah jenis bidang setelah Anda membuat bidang.

Amazon Kendramemiliki bidang dokumen cadangan atau built-in berikut yang dapat Anda gunakan:

  • _authors—Daftar satu atau beberapa penulis yang bertanggung jawab atas isi dokumen.

  • _category—Kategori yang menempatkan dokumen dalam grup tertentu.

  • _created_at—Tanggal dan waktu dalam format ISO 8601 ketika dokumen dibuat. Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012 pukul 12:30 PM (ditambah 10 detik) di Waktu Eropa Tengah.

  • _data_source_id—Pengenal sumber data yang berisi dokumen.

  • _document_body—Isi dokumen.

  • _document_id—Pengenal unik untuk dokumen.

  • _document_title—Judul dokumen.

  • _excerpt_page_number—Nomor halaman dalam file PDF tempat kutipan dokumen muncul. Jika indeks Anda dibuat sebelum 8 September 2020, Anda harus mengindeks ulang dokumen sebelum dapat menggunakan atribut ini.

  • _faq_id—Jika ini adalah pertanyaan dan jawaban FAQ, pengenal unik tersebut untuk mereka.

  • _file_type—Jenis file dokumen, seperti pdf atau doc.

  • _last_updated_at—Tanggal dan waktu dalam format ISO 8601 di mana dokumen terakhir diperbarui. Sebagai contoh, 2012-03-25T12:30:10+ 01:00 adalah format tanggal-waktu ISO 8601 untuk 25 Maret 2012 pukul 12:30 PM (ditambah 10 detik) di Waktu Eropa Tengah.

  • _source_uri—URI tempat dokumen tersedia. Misalnya, URI dokumen di situs web perusahaan.

  • _version—Pengenal untuk versi spesifik dokumen.

  • _view_count—Jumlah berapa kali dokumen dilihat.

  • _language_code(String) —Kode untuk bahasa yang berlaku untuk dokumen. Ini default ke bahasa Inggris jika Anda tidak menentukan bahasa. Untuk informasi selengkapnya tentang bahasa yang didukung, termasuk kodenya, lihat Menambahkan dokumen dalam bahasa selain bahasa Inggris.

Untuk bidang kustom, Anda membuat bidang ini menggunakanDocumentMetadataConfigurationUpdates denganUpdateIndex API, seperti yang Anda lakukan saat membuat bidang cadangan atau built-in. Anda harus mengatur tipe data yang sesuai untuk bidang kustom Anda. Anda tidak dapat mengubah jenis bidang setelah Anda membuat bidang.

Berikut ini adalah jenis yang dapat Anda tetapkan untuk bidang khusus:

  • Tanggal

  • Jumlah

  • String

  • Daftar string

Anda membuat bidang kustom menggunakan konsol tersebut atau menggunakan UpdateIndexAPI. Setelah Anda membuat bidang kustom, Anda memetakannya ke atribut dokumen, seperti yang Anda lakukan dengan bidang tersimpan. Jika Anda menambahkan dokumen ke indeks dengan BatchPutDocumentAPI, Anda memetakan atribut dengan API. Untuk dokumen yang diindeks dari sumberAmazon S3 data, Anda memetakan atribut menggunakan file metadata yang berisi struktur JSON yang menggambarkan atribut dokumen. Untuk dokumen yang diindeks dengan database atau sumber data yang memungkinkan pemetaan lapangan, Anda memetakan atribut dengan konsol atau konfigurasi sumber data.

Menambahkan atribut atau bidang khusus dengan BatchPutDocument API

Saat menggunakanBatchPutDocument API untuk menambahkan dokumen ke indeks, tentukan bidang kustom atau atribut sebagai bagian dariAttributes. Anda dapat menambahkan beberapa bidang atau atribut ketika memanggil API. Anda dapat membuat hingga 500 bidang kustom atau atribut. Contoh berikut ini adalah bidang kustom atau atribut yang menambahkan “Departemen” ke dokumen.

"Attributes": 
    {
        "Department": "HR",
        "_category": "Vacation policy"
    }

Menambahkan atribut atau bidang khusus ke sumberAmazon S3 data

Saat Anda menggunakan bucket S3 sebagai sumber data untuk indeks, tambahkan metadata ke dokumen dengan file metadata pendamping. Letakkan file JSON metadata dalam struktur direktori yang paralel dengan dokumen Anda. Untuk informasi selengkapnya, lihat Amazon S3metadata dokumen.

Anda menentukan bidang atau atribut khusus dalam strukturAttributes JSON. Anda dapat membuat hingga 500 bidang kustom atau atribut. Misalnya, contoh berikut menggunakanAttributes untuk menentukan tiga bidang kustom atau atribut dan satu bidang terpesan.

"Attributes": {
        "brand": "Amazon Basics",
        "price": 1595,
        "_category": "sports",
        "subcategories": ["outdoors", "electronics"]
    }
