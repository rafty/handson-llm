Berbagi objek dengan URL yang telah ditandatangani sebelumnya

Semua objek Amazon S3 secara default bersifat privat, hanya pemilik objek yang memiliki izin untuk mengaksesnya. Namun, pemilik objek dapat berbagi objek dengan orang lain dengan membuat URL yang telah ditandatangani sebelumnya. URL yang ditandatangani sebelumnya menggunakan kredensyal keamanan untuk memberikan izin terbatas waktu untuk mengunduh objek. URL dapat dimasukkan dalam browser atau digunakan oleh program untuk mengunduh objek. Kredensyal yang digunakan oleh URL yang telah ditandatangani sebelumnya adalah milik AWS pengguna yang membuat URL.

Untuk informasi umum tentang URL yang telah ditandatangani, lihat. Bekerja dengan URLs yang telah ditandatangani

Anda dapat membuat sebuah URL presigned untuk berbagi objek tanpa tulis kode apa pun dengan menggunakan konsol Amazon S3, AWS Explorer for Visual Studio (Windows), atau. AWS Toolkit for Visual Studio Code Anda juga dapat membuat URL yang telah ditandatangani sebelumnya secara terprogram dengan menggunakan AWS Command Line Interface (AWS CLI) atau SDK. AWS

Anda dapat menggunakan konsol Amazon S3 untuk membuat URL yang telah ditandatangani sebelumnya untuk berbagi objek dengan mengikuti langkah-langkah ini. Saat menggunakan konsol, waktu kedaluwarsa maksimum untuk URL yang ditandatangani sebelumnya adalah 12 jam dari waktu pembuatan.

Untuk menghasilkan URL yang telah ditandatangani menggunakan konsol Amazon S3

  1. Masuk ke AWS Management Console dan buka konsol Amazon S3 di https://console.aws.amazon.com/s3/.

  2. Di panel navigasi kiri, pilih Bucket.

  3. Di Bucket daftar, pilih nama bucket yang berisi objek yang Anda inginkan URL yang ditandatangani sebelumnya.

  4. Dalam daftar Objects, pilih objek yang ingin Anda buat URL yang telah ditandatangani sebelumnya.

  5. Pada menu Tindakan objek, pilih Bagikan dengan URL yang telah ditandatangani sebelumnya.

  6. Tentukan berapa lama Anda ingin URL yang telah ditandatangani sebelumnya menjadi valid.

  7. Pilih Buat URL yang telah ditandatangani sebelumnya.

  8. Ketika konfirmasi muncul, URL secara otomatis disalin ke clipboard Anda. Anda akan melihat tombol untuk menyalin URL yang telah ditandatangani sebelumnya jika Anda perlu menyalinnya lagi.

AWS CLIPerintah contoh berikut menampilkan pembuatan sebuah URL presigned untuk berbagi objek dari bucket Amazon S3. Saat Anda menggunakanAWS CLI, waktu kedaluwarsa maksimum untuk URL yang ditandatangani sebelumnya adalah 7 hari sejak saat pembuatan. Untuk menggunakan contoh ini, ganti user input placeholdersdengan informasi Anda sendiri.

aws s3 presign s3://arn:aws:s3:::DOC-EXAMPLE-BUCKET1/mydoc.txt --expires-in 604800

  1. Connect untuk AWS menggunakan langkah-langkah berikut, Memberikan AWS kredensyal di AWS Toolkit for Visual StudioPanduan Pengguna.

  2. Pilih file di tampilan Amazon S3 dan buka menu konteks (klik kanan).

  3. Pilih Buat URL yang telah ditandatangani sebelumnya, lalu atur tanggal dan waktu kedaluwarsa untuk URL.

  1. Connect ke AWS menggunakan langkah-langkah berikut, Menghubungkan ke AWS Toolkit for Visual Studio Code dalam Panduan AWS Toolkit for Visual Studio Code Pengguna.

  2. Pilih AWS logo di panel kiri dalam Visual Studio Code.

  3. Di bawah EXPLORER, pilih S3.

  4. Pilih bucket dan file dan buka menu konteks (klik kanan).

  5. Pilih Buat URL yang telah ditandatangani sebelumnya, lalu atur waktu kedaluwarsa (dalam menit).

  6. Tekan Enter, dan URL yang telah ditandatangani sebelumnya akan disalin ke clipboard Anda.
