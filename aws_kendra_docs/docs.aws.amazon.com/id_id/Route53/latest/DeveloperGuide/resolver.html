Apa Amazon Route 53 Resolver?

Amazon Route 53 Resolvermerespons kueri DNS secara rekursif dariAWS sumber daya untuk catatan publik, nama DNS khusus VPC Amazon, dan zona host pribadi Amazon Route 53, dan tersedia secara default di semua VPC.

catatan

Amazon Route 53 Resolversebelumnya disebut server DNS Amazon, tetapi diganti namanya saat aturan Resolver, dan titik akhir masuk dan keluar diperkenalkan. Untuk informasi selengkapnya, lihat server DNS Amazon di Panduan Pengguna Amazon Virtual Private Cloud.

VPC Amazon terhubung ke Resolver Route 53 di alamat IP VPC+2. Alamat VPC+2 ini terhubung ke Resolver Route 53 dalam Availability Zone.

Resolver Route 53 secara otomatis menjawab kueri DNS untuk:

  • Nama domain VPC lokal untuk instans EC2 (misalnya: ec2-44.compute-1.amazonaws.com).

  • Catatan di zona host pribadi (misalnya, acme.example.com).

  • Untuk nama domain publik, Route 53 Resolver melakukan pencarian rekursif terhadap server nama publik di internet.

Jika Anda memiliki beban kerja yang memanfaatkan VPC dan sumber daya lokal, Anda juga perlu menyelesaikan data DNS yang dihosting di tempat. Demikian pula, sumber daya lokal ini mungkin perlu menyelesaikan nama yang dihostingAWS. Melalui titik akhir Resolver dan aturan penerusan bersyarat, Anda dapat menyelesaikan kueri DNS antara sumber daya lokal dan VPC untuk membuat penyiapan cloud hibrid melalui VPN atau Direct Connect (DX). Secara khusus:

  • Titik akhir Resolver masuk memungkinkan kueri DNS ke VPC dari jaringan lokal atau VPC lainnya.

  • Titik akhir Resolver Outbound memungkinkan kueri DNS dari VPC ke jaringan lokal atau VPC lainnya.

  • Aturan Resolver memungkinkan Anda membuat satu aturan penerusan untuk setiap nama domain dan tentukan nama domain dengan kueri DNS yang ingin Anda teruskan dari VPC ke penyelesai DNS dan dari lokal ke VPC. Aturan diterapkan langsung ke VPC Anda dan dapat dibagikan di beberapa akun.

Diagram berikut menunjukkan resolusi DNS hibrida dengan titik akhir Resolver. Perhatikan bahwa diagram disederhanakan untuk menampilkan hanya satu Availability Zone.

Diagram ini menggambarkan langkah-langkah berikut:

Outbound (panah padat 1—5):

  1. Instans Amazon EC2 perlu menyelesaikan kueri DNS ke domain internal.example.com. Server DNS otoritatif berada di pusat data on-premise. Kueri DNS ini dikirim ke VPC+2 di VPC yang terhubung ke Route 53 Resolver.

  2. Aturan penerusan Resolver Route 53 dikonfigurasi untuk meneruskan kueri ke internal.example.com di pusat data lokal.

  3. Query diteruskan ke endpoint keluar.

  4. Titik akhir keluar meneruskan kueri ke DNS resolver lokal melalui koneksi pribadi antaraAWS dan pusat data. Koneksi dapat berupaAWS Direct Connect atauAWS Site-to-Site VPN, digambarkan sebagai gateway pribadi virtual.

  5. Resolver DNS lokal menyelesaikan kueri DNS untuk internal.example.com dan mengembalikan jawaban ke instans Amazon EC2 melalui jalur yang sama secara terbalik.

Masuk (panah putus-putus a—e):

  1. Klien di pusat data lokal perlu menyelesaikan kueri DNS keAWS sumber daya untuk domain dev.example.com. Ini mengirimkan kueri ke DNS resolver lokal.

  2. Resolver DNS lokal memiliki aturan penerusan yang mengarahkan kueri ke dev.example.com ke titik akhir masuk.

  3. Kueri tiba di titik akhir masuk melalui koneksi pribadi, sepertiAWS Direct Connect atauAWS Site-to-Site VPN, digambarkan sebagai gateway virtual.

  4. Endpoint masuk mengirimkan kueri ke Route 53 Resolver di VPC +2.

  5. Route 53 Resolver menyelesaikan kueri DNS untuk dev.example.com dan mengembalikan jawaban ke klien melalui jalur yang sama secara terbalik.
