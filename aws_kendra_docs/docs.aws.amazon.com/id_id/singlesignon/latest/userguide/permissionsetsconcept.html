Set izin

Kumpulan izin adalah template yang Anda buat dan pertahankan yang mendefinisikan kumpulan satu atau beberapa kebijakan IAM. Set izin menyederhanakan penetapan akses AWS akun untuk pengguna dan grup di organisasi Anda. Misalnya, Anda dapat membuat set izin Admin Database yang mencakup kebijakan untuk mengelola layanan AWS RDS, DynamoDB, dan Aurora, dan menggunakan set izin tunggal tersebut untuk memberikan akses ke daftar AWS akun target dalam AWSOrganisasi Anda untuk administrator database Anda.

IAM Identity Center memberikan akses ke pengguna atau grup dalam satu atau lebih Akun AWS dengan set izin. Saat Anda menetapkan set izin, IAM Identity Center membuat peran IAM yang dikontrol oleh IAM Identity Center yang sesuai di setiap akun, dan melampirkan kebijakan yang ditentukan dalam izin yang ditetapkan ke peran tersebut. IAM Identity Center mengelola peran, dan memungkinkan pengguna resmi yang telah Anda tetapkan untuk mengambil peran tersebut, dengan menggunakan IAM Identity Center User Portal atau CLI. Anda dapat menambahkan kebijakan AWS terkelola, kebijakan terkelola pelanggan, kebijakan sebaris, dan kebijakan AWS terkelola untuk fungsi pekerjaan ke set izin Anda. Anda juga dapat menetapkan kebijakan AWS terkelola atau kebijakan yang dikelola pelanggan sebagai batas izin.
