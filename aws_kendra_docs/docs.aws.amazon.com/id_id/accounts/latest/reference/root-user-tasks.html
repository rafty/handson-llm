Tugas yang memerlukan kredensial pengguna root

penting

Tindakan berikut AWS Identity and Access Management (IAM) akan mencapai akhir dukungan standar pada Juli 2023: aws-portal:ModifyAccount dan. aws-portal:ViewAccount Lihat Menggunakan tindakan berbutir halus untuk mengganti AWS Billing tindakan ini dengan tindakan berbutir halus sehingga Anda memiliki akses keAWS Billing, Manajemen AWS Biaya, dan konsol akun. AWS

Jika Anda membuat akun atau AWS akun AWS Organizations Manajemen sebelum 6 Maret 2023, tindakan berbutir halus akan berlaku mulai Juli 2023. Kami menyarankan Anda untuk menambahkan tindakan berbutir halus, tetapi tidak menghapus izin yang ada dengan atau awalan. aws-portal purchase-orders

Jika Anda membuat AWS akun atau akun AWS Organizations Manajemen pada atau setelah 6 Maret 2023, tindakan halus akan segera berlaku.

Kami menyarankan Anda mengonfigurasi pengguna administratif AWS IAM Identity Center (successor to AWS Single Sign-On) untuk melakukan tugas harian dan mengakses AWS sumber daya. Namun, Anda dapat melakukan tugas yang tercantum di bawah ini hanya saat masuk sebagai pengguna root akun.

Tugas

  • Ubah pengaturan akun Anda. Ini mencakup nama akun, alamat email, kata sandi pengguna root, dan access key pengguna root. Pengaturan akun lainnya, seperti informasi kontak, preferensi mata uang pembayaran, danWilayah AWS, tidak memerlukan kredensi pengguna root.

  • Memulihkan izin pengguna IAM. Jika satu-satunya administrator IAM secara tidak sengaja mencabut izinnya sendiri, Anda dapat masuk sebagai pengguna root untuk mengedit kebijakan dan memulihkan izin tersebut.

  • Aktifkan akses IAM ke konsol Manajemen Penagihan dan Biaya.

  • Lihat faktur pajak tertentu. Pengguna IAM dengan aws-portal: ViewBilling izin dapat melihat dan mengunduh faktur PPN dari AWS Eropa, tetapi bukan AWS Inc. atau Amazon Internet Services Private Limited (AISPL).

  • Tutup Anda Akun AWS.

  • Mendaftar sebagai penjual di Marketplace Instans Cadangan.

  • Konfigurasi bucket Amazon S3 untuk mengaktifkan MFA (autentikasi multi-faktor).

  • Mengedit atau menghapus kebijakan bucket Amazon Simple Storage Service (Amazon S3) yang menyertakan ID virtual private cloud (VPC) atau VPC endpoint ID yang tidak valid.

  • Mendaftar untuk AWS GovCloud (US).

  • Minta kunci akses pengguna root AWS GovCloud (US) akun dariAWS Support.
