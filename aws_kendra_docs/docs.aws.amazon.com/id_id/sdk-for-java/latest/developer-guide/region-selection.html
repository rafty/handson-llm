AWSseleksi wilayah

Daerah memungkinkan Anda untuk mengaksesAWSlayanan yang secara fisik berada di wilayah geografis tertentu. Hal ini dapat berguna baik untuk redundansi dan untuk menjaga data dan aplikasi Anda berjalan dekat dengan tempat Anda dan pengguna Anda akan mengaksesnya.

MasukAWS SDK for Java2.x, semua wilayah yang berbeda terkait kelas dari versi 1.x telah runtuh menjadi satu kelas Region. Anda dapat menggunakan kelas ini untuk semua tindakan terkait wilayah seperti mengambil metadata tentang suatu wilayah atau memeriksa apakah layanan tersedia di suatu wilayah.

Memilih wilayah

Anda dapat menentukan nama wilayah dan SDK akan secara otomatis memilih endpoint yang sesuai untuk Anda.

Untuk mengatur wilayah secara eksplisit, kami menyarankan Anda menggunakan konstanta didefinisikan dalamWilayahkelas. Ini adalah pencacahan semua wilayah yang tersedia untuk umum. Untuk membuat klien dengan wilayah dari kelas, gunakan kode berikut.

Ec2Client ec2 = Ec2Client.builder()
          .region(Region.US_WEST_2)
          .build();

Jika wilayah yang Anda coba gunakan bukan salah satu konstanta diRegionclass, Anda dapat membuat wilayah baru menggunakanofmetode. Fitur ini memungkinkan Anda mengakses Wilayah baru tanpa meningkatkan SDK.

Region newRegion = Region.of("us-east-42");
Ec2Client ec2 = Ec2Client.builder()
          .region(newRegion)
          .build();
catatan

Setelah Anda membangun klien dengan pembangun, itutidak berubahdan wilayahtidak dapat diubah. Jika Anda bekerja dengan beberapaWilayah AWSuntuk layanan yang sama, Anda harus membuat beberapa klien — satu per wilayah.

Memilih titik akhir tertentu

MASINGAWSklien dapat dikonfigurasi untuk menggunakantitik akhir spesifikdalam suatu wilayah dengan memanggilendpointOverridemetode.

Misalnya, untuk mengkonfigurasiAmazon EC2klien untuk menggunakan Wilayah Eropa (Irlandia), gunakan kode berikut.

Ec2Client ec2 = Ec2Client.builder()
               .region(Region.EU_WEST_1)
               .endpointOverride(URI.create("https://ec2.eu-west-1.amazonaws.com"))
               .build();

Secara otomatis menentukan Wilayah dari lingkungan

Saat berjalanAmazon EC2atauAWS Lambda, Anda mungkin ingin mengkonfigurasi klien untuk menggunakan wilayah yang sama dengan kode Anda berjalan. Ini memisahkan kode Anda dari lingkungan yang dijalankannya dan mempermudah penerapan aplikasi Anda ke beberapa wilayah untuk latensi atau redundansi yang lebih rendah.

Untuk menggunakan rantai penyedia kredensial/wilayah default untuk menentukan wilayah dari lingkungan, gunakan pembuat kliencreatemetode.

Ec2Client ec2 = Ec2Client.create();

Jika Anda tidak secara eksplisit menetapkan wilayah menggunakanregionMetode, SDK berkonsultasi rantai penyedia wilayah default untuk mencoba dan menentukan wilayah yang akan digunakan.

Rantai penyedia wilayah default

Berikut ini adalah proses pencarian wilayah:

  1. Setiap wilayah eksplisit yang ditetapkan dengan menggunakanregionpada pembangun itu sendiri diutamakan atas apa pun.

  2. ParameterAWS_REGIONvariabel lingkungan diperiksa. Jika sudah diatur, wilayah tersebut digunakan untuk mengkonfigurasi klien.

    catatan

    Variabel lingkungan ini diatur olehLambdakontainer.

  3. SDK memeriksaAWSfile konfigurasi bersama (biasanya terletak di~/.aws/config). Jikadaerahproperti hadir, SDK menggunakannya.

    • ParameterAWS_CONFIG_FILEvariabel lingkungan dapat digunakan untuk menyesuaikan lokasi file konfigurasi bersama.

    • ParameterAWS_PROFILEvariabel lingkungan atauaws.profileproperti sistem dapat digunakan untuk menyesuaikan profil yang beban SDK.

  4. SDK mencoba untuk menggunakanAmazon EC2layanan metadata instance untuk menentukan wilayah yang sedang berjalanAmazon EC2contoh.

  5. Jika SDK masih belum menemukan suatu wilayah pada saat ini, pembuatan klien gagal dengan pengecualian.

Saat mengembangkanAWSaplikasi, pendekatan umum adalah dengan menggunakanfile konfigurasi bersama(dijelaskan dalamPesanan pengambilan kredenal) untuk mengatur wilayah untuk pengembangan lokal, dan bergantung pada rantai penyedia wilayah default untuk menentukan wilayah ketika berjalan padaAWSinfrastruktur. Ini sangat menyederhanakan pembuatan klien dan membuat aplikasi Anda tetap portabel.

Memeriksa ketersediaan layanan di Wilayah

Untuk melihat apakah tertentuLayanan AWStersedia di suatu wilayah, menggunakanserviceMetadatadanregionmetode pada layanan yang ingin Anda periksa.

DynamoDbClient.serviceMetadata().regions().forEach(System.out::println);
