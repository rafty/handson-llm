Mengontrol akses ke titik akhir VPC menggunakan kebijakan titik akhir

Saat membuat titik akhir antarmuka atau titik akhir gateway, Anda juga dapat melampirkan kebijakan titik akhir. Kebijakan VPC endpoint adalah dokumen kebijakan JSON yang mengendalikanAWS prinsipal mana yang dapat menggunakan VPC endpoint.

Anda tidak dapat melampirkan lebih dari satu kebijakan ke titik akhir. Namun, Anda dapat memodifikasi kebijakan titik akhir kapan saja.

Kebijakan titik akhir tidak menimpa atau menggantikan kebijakan berbasis identitas atau kebijakan berbasis sumber daya. Misalnya, jika Anda menggunakan titik akhir antarmuka untuk terhubung ke Amazon S3, Anda juga dapat menggunakan kebijakan bucket Amazon S3 untuk mengendalikan akses dari VPC endpoint tertentu atau VPC tertentu.

Kebijakan VPC endpoint

Anda melampirkan kebijakan endpoint VPC ke titik akhir. Jika Anda tidak menetapkan kebijakan titik akhir ketika membuat titik akhir, kami melampirkan kebijakan berikut, yang memungkinkan akses penuh ke layanan.

{
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": "*",
            "Action": "*",
            "Resource": "*"
        }
    ]
}
Pertimbangan-pertimbangan

  • Kebijakan Anda harus berisi elemen Principal.

  • Ukuran kebijakan titik akhir tidak boleh melebihi 20.480 karakter (termasuk spasi putih).

  • Tidak semua layanan endpoint mendukung kebijakan endpoint. Jika layanan tidak mendukung kebijakan titik akhir, titik akhir memungkinkan akses penuh ke layanan. Untuk informasi selengkapnya, lihat Melihat dukungan kebijakan titik akhir.

Prinsipal untuk titik akhir gateway

Dengan titik akhir gateway, Anda harus menggunakan kunciaws:PrincipalArn kondisi untuk memberikan akses ke prinsipal.

Jika Anda menentukan pokok dalam salah satu format berikut, akses diberikan kepadaPengguna root akun AWS satu-satunya, tidak semua pengguna dan peran untuk akun tersebut.

"AWS": "account_id"
"AWS": "arn:aws:iam::account_id:root"

Jika Anda menetapkan Amazon Resource Name (ARN) untuk prinsipal, ARN diubah menjadi ID prinsipal unik ketika kebijakan disimpan.

Misalnya kebijakan IAM untuk titik akhir gateway, lihat hal berikut:

  • Titik akhir untuk Amazon S3

  • Titik akhir untuk DynamoDB

Memperbarui kebijakan VPC endpoint

Gunakan prosedur berikut untuk memperbarui kebijakan titik akhir. Setelah memperbarui kebijakan titik akhir, memerlukan waktu beberapa menit sampai perubahan diterapkan.

Untuk memperbarui kebijakan VPC endpoint

  1. Buka konsol Amazon VPC di https://console.aws.amazon.com/vpc/.

  2. Di panel navigasi, pilih Titik akhir.

  3. Pilih VPC endpoint.

  4. Pilih Tindakan, Kelola kebijakan.

  5. Pilih Akses Penuh untuk memungkinkan akses penuh ke layanan, atau pilih Kustom dan lampirkan kebijakan khusus.

  6. Pilih Simpan.
