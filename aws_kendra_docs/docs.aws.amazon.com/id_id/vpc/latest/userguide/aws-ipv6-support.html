AWSlayanan yang mendukung IPv6

Komputer dan perangkat pintar menggunakan alamat IP untuk berkomunikasi satu sama lain melalui internet dan jaringan lainnya. Sebagai internet terus tumbuh, begitu juga kebutuhan untuk alamat IP. Format yang paling umum untuk alamat IP adalah IPv4. Format baru untuk alamat IP adalah IPv6, yang menyediakan ruang alamat yang lebih besar daripada IPv4.

Layanan AWSdukungan untuk IPv6 mencakup konfigurasi tumpukan-ganda (IPv4 dan IPv6) atau IPv6 saja. Misalnya, virtual private cloud (VPC) adalah bagian yang diisolasi secara logis diAWS Cloud mana Anda dapat meluncurkanAWS sumber daya. Dalam VPC, Anda dapat membuat subnet yang hanya IPv4, dual stack, atau IPv6 saja.

Layanan AWSmendukung akses melalui titik akhir publik. BeberapaLayanan AWS juga mendukung akses menggunakan endpoint pribadi yang didukung olehAWS PrivateLink. Layanan AWSdapat mendukung IPv6 melalui titik akhir pribadi mereka bahkan jika mereka tidak mendukung IPv6 melalui titik akhir publik mereka. Endpoint yang mendukung IPv6 dapat merespons kueri DNS dengan catatan AAAA.

Dukungan IPv6 tambahan

Hitung

  • Amazon EC2 mendukung peluncuran instans berdasarkan Sistem Nitro ke dalam subnet khusus IPv6.

  • Amazon EC2 menyediakan titik akhir IPv6 untuk Instance Metadata Service (IMDS) dan Amazon Time Sync.

Jaringan dan Pengiriman Konten

  • Amazon VPC mendukung pembuatan subnet khusus IPv6.

  • Amazon VPC membantuAWS sumber daya IPv6 berkomunikasi dengan sumber daya IPv4 dengan mendukung DNS64 pada subnet dan NAT64 di gateway NAT Anda.

Keamanan, Identitas, dan Kepatuhan

  • AWS Identity and Access Management(IAM) mendukung alamat IPv6 di kebijakan IAM.

  • Amazon Macie mendukung alamat IPv6 di informasi pengenal pribadi (PII).

Pengelolaan dan Tata Kelola

  • AWS CloudTrailcatatan termasuk sumber informasi IPv6.

  • AWS CLIv2 mendukung pengunduhan melalui koneksi IPv6 untuk klien IPv6 saja.
