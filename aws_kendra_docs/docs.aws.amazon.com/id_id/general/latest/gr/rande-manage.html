Mengelola Wilayah AWS

Sebuah Wilayah AWSkumpulanAWS sumber daya di area geografis. Setiap Wilayah AWS terisolasi dan independen dari Wilayah lain. Wilayah memberikan toleransi kesalahan, stabilitas, dan ketahanan, dan juga dapat mengurangi latensi. Anda dapat menyediakan sumber daya di beberapa Wilayah sehingga aplikasi Anda tetap tersedia dan tidak terpengaruh oleh pemadaman Regional.

Sumber daya yang Anda buat di satu wilayah tidak ada di wilayah lain mana pun, kecuali Anda secara eksplisit menggunakan fitur replikasi yang ditawarkan oleh layanan AWS. Misalnya, Amazon S3 dan Amazon EC2 mendukung replikasi lintas Wilayah. Beberapa layanan, seperti AWS Identity and Access Management (IAM), tidak memiliki sumber daya Regional.

Anda dapat menggunakan ketentuan kebijakan untuk mengontrol akses keLayanan AWS dalam fileWilayah AWS.

Akun Anda menentukan Wilayah yang tersedia untuk Anda.

  • SebuahAkun AWS menyediakan beberapa Wilayah sehingga Anda dapat meluncurkanAWS sumber daya di lokasi yang memenuhi persyaratan Anda. Misalnya, Anda mungkin ingin meluncurkan instans di Eropa agar lebih dekat dengan pelanggan Eropa Anda atau untuk memenuhi persyaratan hukum.

  • Sebuah akunAWSGovCloud (US-West) menyediakan akses ke WilayahAWSGovCloud (US-West) dan WilayahAWSGovCloud (US-East). Untuk informasi selengkapnya, lihat AWS GovCloud (US).

  • Akun Amazon AWS (China) hanya menyediakan akses ke Wilayah Beijing dan Ningxia. Untuk informasi lebih lanjut, lihat Amazon Web Services di China.

Aktifkan Wilayah

Sebelum Anda dapat menggunakan salah satu Wilayah berikut, Anda harus mengaktifkan Wilayah untuk akun Anda.

Saat mengaktifkan Wilayah, AWS lakukan tindakan untuk mempersiapkan akun Anda di Wilayah tersebut, seperti mendistribusikan sumber daya IAM ke Wilayah. Proses ini memerlukan waktu beberapa menit untuk sebagian besar akun, tetapi dapat memakan waktu beberapa jam. Anda tidak dapat menggunakan Wilayah sampai proses ini selesai.

Persyaratan

Untuk mengaktifkan Wilayah, Anda harus memiliki izin yang sesuai. Untuk melihat contoh kebijakan IAM, lihat Izinkan pengaktifan dan penonaktifanWilayah AWS di Panduan Pengguna IAM.

Untuk mengaktifkan Wilayah menggunakan konsol

  1. Masuk ke AWS Management Console.

  2. Di sudut kanan atas konsol, pilih Wilayah saat ini untuk memperluas daftar Wilayah, gulir ke bagian bawah daftar, lalu pilih Kelola Wilayah.

  3. Periksa kolom Action untuk Wilayah yang ingin Anda gunakan.

    • Jika kolom Status berisi Diaktifkan secara default, tidak diperlukan tindakan lebih lanjut. Wilayah diaktifkan secara otomatis untuk akun Anda.

    • Jika kolom Status berisi Dinonaktifkan, pilih Aktifkan. Tinjau teks informasional lalu pilih Aktifkan Wilayah. Anda harus menunggu sampai Wilayah siap digunakan.

Untuk mengaktifkan Region menggunakanAWS CLI

Gunakan perintah enable-region berikut.

aws account enable-region --region-name af-south-1

Menonaktifkan Wilayah

Jika Anda mengaktifkan Wilayah yang tidak diaktifkan secara otomatis, Anda dapat menonaktifkan Wilayah jika diperlukan. Setelah menonaktifkan Wilayah, sumber daya di Wilayah ini menjadi tidak tersedia, berdasarkan konsistensi. Namun, sumber daya tersebut tidak dihapus. Anda dapat mengaksesnya jika Anda mengaktifkan Wilayah ini lagi.

Persyaratan

  • Anda tidak dapat menonaktifkan Wilayah yang diaktifkan secara default. (Lihat tabel di bagian sebelumnya.)

  • Untuk menonaktifkan Wilayah, Anda harus memiliki izin untuk menonaktifkan Wilayah. Untuk melihat contoh kebijakan IAM, lihat Izinkan pengaktifan dan penonaktifan Wilayah AWS di Panduan Pengguna IAM.

  • Sebelum menonaktifkan Wilayah, sebaiknya hapus semua sumber daya dari Wilayah tersebut. Setelah menonaktifkan Wilayah, Anda tidak dapat lagi melihat atau mengelola sumber daya di Wilayah dari akun tersebut melalui konsolAWS Manajemen atauAWS API menggunakan prinsip IAM. Namun, sumber daya di Wilayah tersebut dapat terus dikenakan biaya. Untuk informasi selengkapnya, lihat Mengaktifkan dan menonaktifkan Wilayah di Panduan Pengguna AWS Billing and Cost Management.

Untuk nonaktifkan Wilayah menggunakan konsol

  1. Masuk ke AWS Management Console.

  2. Di sudut kanan atas konsol, pilih Wilayah saat ini untuk memperluas daftar Wilayah, gulir ke bagian bawah daftar, lalu pilih Kelola Wilayah.

  3. Di kolom Tindakan untuk Wilayah, pilih Nonaktifkan.

  4. Di kotak dialog, tinjau teks informasional, lalu pilih Nonaktifkan Wilayah.

Untuk nonaktifkan Wilayah menggunakanAWS CLI

Gunakan perintah disable-region berikut.

aws account disable-region --region-name af-south-1

Menjelaskan Wilayah Anda menggunakanAWS CLI

Gunakan perintah list-regions untuk menjelaskan semua Wilayah yang tersedia untuk akun Anda, termasuk statusnya.

aws account list-regions --max-results 50

Gunakan get-region-opt-statusperintah untuk menggambarkan status Wilayah tertentu.

aws account get-region-opt-status --region-name af-south-1

Untuk Wilayah yang diaktifkan secara otomatis untuk akun Anda, output dari perintah ini mencakup yang berikut:

"RegionOptStatus": "ENABLED_BY_DEFAULT"

Untuk Wilayah yang harus diaktifkan secara manual, outputnya mencakup hal berikut:

"RegionOptStatus": "DISABLED"

Setelah mengaktifkan Wilayah, outputnya mencakup hal berikut:

"RegionOptStatus": "ENABLED"
