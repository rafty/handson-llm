Menggunakan autentikasi multi-faktor (MFA) di AWS

Untuk keamanan yang meningkat, kami menyarankan Anda untuk mengonfigurasi autentikasi multi-faktor (MFA) untuk membantu melindungi sumber daya AWS Anda. Anda dapat mengaktifkan MFA untuk penggunaPengguna root akun AWS dan IAM. Saat Anda mengaktifkan MFA untuk pengguna akar, itu hanya memengaruhi kredensial pengguna akar. Pengguna IAM di akun memiliki identitas berbeda dengan kredensial mereka sendiri, dan setiap identitas memiliki konfigurasi MFA-nya sendiri. Anda dapat mendaftarkan hingga delapan perangkat MFA dari kombinasi jenis MFA yang saat ini didukung dengan pengguna AndaPengguna root akun AWS dan IAM. Untuk informasi lebih lanjut tentang jenis MFA yang didukung, lihatApa itu MFA?. Dengan beberapa perangkat MFA, hanya satu perangkat MFA yang diperlukan untuk masuk keAWS Management Console atau membuat sesi melaluiAWS CLI sebagai pengguna tersebut.

Apa itu MFA?

MFA menambahkan keamanan ekstra karena mengharuskan pengguna untuk memberikan autentikasi unik dari mekanisme MFA yang didukung AWS sebagai tambahan ke kredensial masuk reguler mereka ketika mereka mengakses situs web atau layanan AWS:

  • Kunci keamanan FIDO - Kunci keamanan perangkat keras bersertifikat FIDO disediakan oleh penyedia pihak ketiga. FIDO Alliance menyimpan daftar semua produk bersertifikat FIDO yang kompatibel dengan spesifikasi FIDO. Standar otentikasi FIDO didasarkan pada kriptografi kunci publik, yang memungkinkan otentikasi kuat dan tahan phishing yang lebih aman daripada kata sandi. Kunci keamanan FIDO mendukung beberapa akun root dan pengguna IAM menggunakan satu kunci keamanan. Untuk informasi lebih lanjut tentang pengaktifan kunci keamanan FIDO, lihatMengaktifkan kunci keamanan FIDO (konsol).

  • Sebuah aplikasi autentikasi virtual â€” Sebuah aplikasi autentikasi virtual yang berjalan di ponsel atau perangkat lain dan mengemulasi perangkat fisik. Aplikasi autentikator virtual mengimplementasikan algoritme kata sandi satu kali (TOTP) berbasis waktu dan mendukung beberapa token pada satu perangkat. Pengguna harus mengetik kode yang valid dari perangkat di halaman web kedua saat masuk. Setiap perangkat MFA virtual yang ditetapkan ke pengguna harus unik. Pengguna tidak dapat mengetik kode dari perangkat MFA virtual milik pengguna lain untuk mengautentikasi. Karena MFA virtual dapat berjalan di perangkat seluler yang tidak aman, MFA virtual mungkin tidak menyediakan tingkat kerahasiaan yang sama dengan kunci keamanan FIDO. Kami menyarankan agar Anda menggunakan perangkat MFA virtual saat menunggu untuk mendapatkan persetujuan pembelian perangkat keras atau saat menunggu kedatangan perangkat keras Anda. Untuk daftar beberapa aplikasi yang didukung yang dapat Anda gunakan sebagai perangkat MFA virtual, lihat Autentikasi Multi-Faktor. Untuk petunjuk tentang pengaturan perangkat MFA virtual dengan AWS, lihat Mengaktifkan perangkat Autentikasi Multi-Faktor (MFA) virtual (konsol).

  • Token TOTP perangkat keras - Sebuah perangkat keras yang menghasilkan kode numerik enam-waktu (TOTP). Pengguna harus mengetik kode yang valid dari perangkat di halaman web kedua saat masuk. Setiap perangkat MFA yang ditetapkan ke pengguna harus unik. Pengguna tidak dapat mengetik kode dari perangkat pengguna lain untuk diautentikasi. Untuk informasi tentang perangkat keras MFA yang didukung, lihat Autentikasi Multi-Faktor. Untuk petunjuk tentang pengaturan token TOTP perangkat keras denganAWS, lihatMengaktifkan token TOTP perangkat keras (konsol).

    MFA berbasis teks -AWS mengakhiri dukungan untuk mengaktifkan Multi-Factor Authentication (MFA). Kami merekomendasikan pelanggan IAM yang menggunakan MFA berbasis teks SMS untuk beralih ke salah satu metode alternatif berikut di: kunci keamanan FIDO, perangkat MFA virtual (berbasis perangkat lunak), atau MFA perangkat keras. Anda dapat mengidentifikasi pengguna di akun Anda dengan perangkat SMS MFA yang ditetapkan. Untuk melakukannya, kunjungi konsol IAM, pilih Pengguna dari panel navigasi, dan cari pengguna dengan SMS di kolom MFA di tabel.
