Gambaran umum manajemen identitasAWS: Pengguna

Anda dapat memberikan akses ke pengguna tertentu dan memberi mereka izin khusus untuk mengakses sumber daya di AndaAkun AWS.Akun AWS Anda dapat menggunakan IAM danAWS IAM Identity Center (successor to AWS Single Sign-On) untuk membuat pengguna baru atau federasi pengguna yang ada ke dalamAWS. Perbedaan utama antara keduanya adalah bahwa pengguna IAM diberikan kredensi jangka panjang keAWS sumber daya Anda sementara pengguna di IAM Identity Center memiliki kredensi sementara yang ditetapkan setiap kali pengguna masuk keAWS. Sebagai praktik terbaik, mewajibkan pengguna manusia untuk menggunakan federasi dengan penyedia identitas untuk mengaksesAWS menggunakan kredenial sementara alih-alih sebagai pengguna IAM. Penggunaan utama untuk pengguna IAM adalah memberikan beban kerja yang tidak dapat menggunakan peran IAM kemampuan untuk membuat permintaan terprogram keAWS layanan dengan menggunakan API atau CLI.

Hanya untuk akses pertama kali: Kredensial pengguna akar Anda

Saat membuat akunAkun AWS, Anda memulai dengan satu identitas masuk yang memiliki akses penuh ke semuaLayanan AWS sumber daya dalam akun tersebut. Identitas ini disebut pengguna root Akun AWS dan diakses dengan cara masuk menggunakan alamat email dan kata sandi yang Anda gunakan saat membuat akun. Kami sangat menyarankan agar Anda tidak menggunakan pengguna root untuk tugas sehari-hari. Lindungi kredensi pengguna root Anda dan gunakan kredensi pengguna root Anda dan gunakan untuk melakukan tugas yang hanya dapat dilakukan oleh pengguna root. Untuk daftar lengkap tugas yang mengharuskan Anda masuk sebagai pengguna root, lihat Tugas yang memerlukan kredensi pengguna root di PanduanAWS Account Management Referensi. Hanya kebijakan kontrol layanan (SCP) dalam organisasi yang dapat membatasi izin yang diberikan kepada pengguna root.

Pengguna IAM dan pengguna di IAM Identity Center

Pengguna IAM bukan akun terpisah; mereka adalah pengguna di dalam akun Anda. Setiap pengguna dapat memiliki kata sandi tersendiri untuk mengakses AWS Management Console. Anda juga dapat membuat access key individu untuk setiap pengguna sehingga pengguna dapat membuat permintaan terprogram untuk bekerja dengan sumber daya di akun Anda.

Pengguna IAM diberikan kredensi jangka panjang keAWS sumber daya Anda. Sebagai praktik terbaik, jangan membuat pengguna IAM dengan kredensi jangka panjang untuk pengguna manusia Anda. Sebagai gantinya, mintalah pengguna manusia Anda untuk menggunakan kredensi sementara saat mengaksesAWS.

catatan

Untuk skenario di mana Anda memerlukan pengguna IAM dengan akses terprogram dan kredensi jangka panjang, kami sarankan Anda memutar kunci akses. Untuk informasi selengkapnya, lihat Merotasi access key.

Sebaliknya,pengguna di AWS IAM Identity Center (penerus AWS Single Sign-On) diberikan kredensi jangka pendek untukAWS sumber daya Anda. Untuk manajemen akses terpusat, kami menyarankan Anda menggunakan AWS IAM Identity Center (successor to AWS Single Sign-On)(IAM Identity Center) untuk mengelola akses ke akun dan izin Anda dalam akun tersebut. IAM Identity Center secara otomatis dikonfigurasi dengan direktori Pusat Identitas sebagai sumber identitas default Anda di mana Anda dapat membuat pengguna dan grup, dan menetapkan tingkat akses mereka keAWS sumber daya Anda. Untuk informasi selengkapnya, lihat Apa itu AWS IAM Identity Center (successor to AWS Single Sign-On) dalam Panduan Pengguna AWS IAM Identity Center (successor to AWS Single Sign-On).

Menggabungkan pengguna yang ada

Jika pengguna di organisasi Anda sudah memiliki cara untuk diautentikasi, seperti dengan masuk ke jaringan perusahaan Anda, Anda tidak perlu membuat pengguna IAM terpisah untuk mereka. Sebagai gantinya, Anda dapat menggabungkan identitas pengguna tersebut ke dalamAWS menggunakan IAM atauAWS IAM Identity Center (successor to AWS Single Sign-On).

Diagram berikut menunjukkan bagaimana pengguna bisa mendapatkan kredensiAWS keamanan sementara untuk mengakses sumber daya di sumber daya di sumber daya AndaAkun AWS.

Federasi sangat berguna dalam kasus-kasus ini:

  • Pengguna Anda sudah ada di direktori perusahaan.

    Jika direktori perusahaan Anda kompatibel dengan Security Assertion Markup Language 2.0 (SAML 2.0), Anda dapat mengonfigurasi direktori perusahaan untuk memberikan akses masuk tunggal (SSO) ke AWS Management Console untuk pengguna Anda. Untuk informasi selengkapnya, lihat Skenario umum untuk kredensial sementara.

    Jika direktori perusahaan Anda tidak kompatibel dengan SAML 2.0, Anda dapat membuat aplikasi broker identitas untuk memberikan akses masuk tunggal (SSO) ke AWS Management Console untuk pengguna Anda. Untuk informasi selengkapnya, lihat Mengaktifkan akses broker identitas kustom ke konsol AWS.

    Jika direktori perusahaan Anda adalah Microsoft Active Directory, Anda dapat menggunakanAWS IAM Identity Center (successor to AWS Single Sign-On) untuk menghubungkan direktori yang dikelola sendiri di Active Directory atau direktori AWS Directory Serviceuntuk membangun kepercayaan antara direktori perusahaan Anda dan direktori perusahaan AndaAkun AWS.

    Jika Anda menggunakan penyedia identitas eksternal (IdP) seperti Okta atau Azure Active Directory untuk mengelola pengguna, Anda dapat menggunakanAWS IAM Identity Center (successor to AWS Single Sign-On) untuk membangun kepercayaan antara IdP Anda dan AndaAkun AWS. Untuk informasi selengkapnya, lihat Connect ke penyedia identitas eksternal di PanduanAWS IAM Identity Center (successor to AWS Single Sign-On) Pengguna.

  • Pengguna Anda sudah memiliki identitas Internet.

    Jika Anda membuat aplikasi seluler atau aplikasi berbasis web yang memungkinkan pengguna mengidentifikasi diri mereka melalui penyedia identitas Internet seperti Login with Amazon, Facebook, Google, atau penyedia identitas apa pun yang kompatibel dengan OpenID Connect (OIDC), aplikasi dapat menggunakan federasi untuk mengakses AWS. Untuk informasi selengkapnya, lihat Tentang federasi identitas web.

    Tip

    Untuk menggunakan federasi identitas dengan penyedia identitas Internet, kami sarankan Anda menggunakan Amazon Cognito.

Akses masuk tunggal untuk pengguna manusia, seperti pengguna tenaga kerja Anda, hinggaAWS sumber daya yang menggunakan IAM Identity Center

IAM Identity Center menyediakan tempat sentral yang menyatukan administrasi pengguna dan akses mereka keAkun AWS dan aplikasi cloud.

Anda dapat mengatur toko identitas dalam IAM Identity Center atau Anda dapat mengkonfigurasi federasi dengan penyedia identitas yang ada (IdP). Memberikan kredensi terbatas kepada pengguna manusia Anda keAWS sumber daya sesuai kebutuhan direkomendasikan sebagai praktik terbaik keamanan.

Pengguna memiliki pengalaman masuk yang lebih mudah dan Anda mempertahankan kontrol atas akses mereka ke sumber daya dari satu sistem. Pusat identitas IAM mendukung autentikasi multi-faktor (MFA) untuk keamanan akun tambahan.
