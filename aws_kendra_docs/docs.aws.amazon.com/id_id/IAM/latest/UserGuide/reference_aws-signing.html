Menandatangani permintaan API AWS

penting

Jika Anda menggunakanAWS SDK atau alat barisAWS perintah untuk mengirim permintaan APIAWS, alat ini menandatangani permintaan API untuk Anda. Anda hanya boleh menandatangani permintaanAWS API seperti yang dijelaskan dalam dokumentasi ini jika Anda tidak menggunakanAWS SDK atau alat barisAWS perintah untuk mengirim permintaanAWS API.

Saat Anda mengirim permintaan API keAWS, Anda harus menandatangani mereka sehinggaAWS dapat mengidentifikasi pengirim. Demi keamanan, sebagian besar permintaan ditandatangani menggunakan kredensiAWS keamanan Anda.

KetikaLayanan AWS menerima permintaan diautentikasi, itu menciptakan kembali tanda tangan menggunakan informasi otentikasi yang terkandung dalam permintaan. Jika tanda tangan cocok, layanan memproses permintaan. Jika tidak, ia menolak permintaan.

Signature Version 4 adalah saat protokolAWS penandatanganan. AWSjuga mendukung ekstensi, Signature Version 4A, yang mendukung tanda tangan untuk permintaan API Multi-region. Untuk informasi lebih lanjut, lihata-signing-examples proyek sigv4 di GitHub.

Kapan menandatangani permintaan

Saat Anda menulis kode khusus yang mengirim permintaan API keAWS, Anda harus menyertakan kode yang menandatangani permintaan. Anda mungkin menulis kode kustom karena:

  • Anda bekerja dengan bahasa pemrograman yang tidak ada SDK AWS.

  • Anda memerlukan kontrol penuh atas cara mengirim permintaanAWS.

Mengapa permintaan ditandatangani

Proses penandatanganan membantu mengamankan permintaan dengan cara berikut:

  • Merifikasi identitas peminta

    Permintaan harus dikirim oleh seseorang dengan access key yang valid.

  • Melindungi data saat transit

    Untuk mencegah gangguan pada permintaan saat sedang transit, beberapa elemen permintaan digunakan untuk menghitung hash (intisari) permintaan, dan nilai hash yang dihasilkan disertakan sebagai bagian dari permintaan. SaatLayanan AWS menerima permintaan, ia menggunakan informasi yang sama untuk menghitung hash dan mencocokkannya dengan nilai hash dalam permintaan Anda. Jika nilai tidak cocok, AWS menyangkal permintaan.

  • Melindungi terhadap potensi serangan replay

    Dalam kebanyakan kasus, permintaan harus mencapai AWS dalam waktu lima menit dari stempel waktu dalam permintaan. Jika tidak, AWSmenyangkal permintaan.
