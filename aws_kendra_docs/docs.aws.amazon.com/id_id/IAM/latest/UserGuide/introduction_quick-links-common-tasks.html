Tautan cepat ke tugas umum

Gunakan tautan berikut untuk mendapatkan bantuan tentang tugas umum yang terkait dengan IAM.

Masuk untuk jenis pengguna yang berbeda

Masuklah ke Konsol IAM dengan memilih Pengguna IAM dan masukkan akun alias atau ID Akun AWS Anda. Di halaman berikutnya, masukkan nama pengguna IAM dan kata sandi Anda.

Untuk masuk dengan pengguna IAM Identity Center, gunakan URL login yang dikirim ke alamat email Anda saat Anda membuat pengguna IAM Identity Center.

Untuk bantuan login menggunakan pengguna IAM Identity Center, lihat Masuk ke portalAWS akses di PanduanAWS Sign-In Pengguna.

Masuk ke pemilik AWS Management Consoleakun dengan memilih Pengguna root dan memasukkan alamatAkun AWS email Anda. Di laman berikutnya, masukkan kata sandi Anda.

Lihat Apa ituAWS Login di PanduanAWS Sign-In Pengguna untuk bantuan menentukan jenis pengguna dan halaman login Anda.

Kelola kata sandi untuk pengguna

Anda memerlukan kata sandi untuk mengakses AWS Management Console, termasuk akses ke informasi penagihan.

Untuk Pengguna root akun AWS Anda, lihat Mengkata sandi penggunaAkun AWS akar.

Untuk pengguna IAM, lihat Mengelola kata sandi untuk pengguna IAM.

Kelola izin untuk pengguna

Anda menggunakan kebijakan untuk memberikan izin kepada pengguna IAM diAkun AWS akun Pengguna IAM tidak memiliki izin saat dibuat, jadi Anda harus menambahkan izin agar mereka dapat menggunakanAWS sumber daya.

Untuk menyediakan akses, tambahkan izin ke pengguna, grup, atau peran Anda:

  • Pengguna dan grup diAWS IAM Identity Center (successor to AWS Single Sign-On):

    Buat set izin. Ikuti petunjuk di Buat set izin di PanduanAWS IAM Identity Center (successor to AWS Single Sign-On) Pengguna.

  • Pengguna yang dikelola dalam IAM melalui penyedia identitas:

    Membuat peran untuk federasi identitas Ikuti petunjuk dalam Membuat peran untuk penyedia identitas pihak ketiga (federasi) di Panduan Pengguna IAM.

  • Pengguna IAM:

    • Buat peran yang dapat diasumsikan pengguna Anda. Ikuti petunjuk dalam Membuat peran untuk pengguna IAM di Panduan Pengguna IAM.

    • (Tidak disarankan) Lampirkan kebijakan langsung ke pengguna atau tambahkan pengguna ke grup pengguna. Ikuti petunjuk dalam Menambahkan izin ke pengguna (konsol) di Panduan Pengguna IAM.

Untuk informasi selengkapnya, lihat Mengelola kebijakan IAM.

Tambahkan multi-factor authentication (MFA)

Untuk menambahkan perangkat MFA virtual, lihat salah satu hal berikut:

  • Aktifkan perangkat MFA virtual untuk konsol AndaPengguna root akun AWS

  • Aktifkan perangkat MFA virtual untuk pengguna IAM (konsol)

Untuk menambahkan kunci keamanan FIDO, lihat salah satu hal berikut:

  • Aktifkan kunci keamanan FIDO untuk penggunaAkun AWS akar (konsol)

  • Aktifkan kunci keamanan FIDO untuk pengguna IAM lainnya (konsol)

Untuk menambahkan perangkat keras MFA, lihat salah satu hal berikut:

  • Aktifkan TOTP perangkat keras untuk penggunaAkun AWS akar (konsol).

  • Aktifkan TOTP perangkat keras untuk pengguna IAM lain (konsol)

Dapatkan access key

Anda perlu access key jika ingin membuat permintaan AWS menggunakan SDK AWS, Alat Baris Perintah AWS, atau operasi API.

Sebelum Anda mulai membuat kunci akses, kami sangat menyarankan agar Anda membaca praktik terbaik keamanan untuk kunci akses. Untuk informasi selengkapnya, lihat Praktik terbaik untuk mengelola kunciAWS akses di Referensi Umum AWS.

Untuk AndaAkun AWS, lihat Akses terprogram di Referensi Umum AWS.

Untuk pengguna IAM, lihat Mengelola kunci akses untuk pengguna IAM.

Tag sumber daya IAM

Anda dapat menandai sumber daya IAM berikut:

  • Pengguna IAM

  • IAM role

  • Kebijakan yang dikelola pelanggan

  • Penyedia Identitas

  • Sertifikat server

  • Perangkat MFA virtual

Untuk mempelajari tentang tag di IAM, lihat Menandai sumber daya IAM.

Untuk mempelajari tentang penggunaan tanda untuk mengontrol akses ke sumber daya AWS, lihat Pengontrolan akses ke sumber daya AWS dengan menggunakan tanda.
