Fitur keamanan di luar IAM

Anda menggunakan IAM untuk mengontrol akses ke tugas yang dilakukanAWS Management Console menggunakan—Alat BarisAWS Perintah, atau operasi API layanan menggunakan AWSSDK. Beberapa produk AWS juga memiliki cara lain untuk mengamankan sumber dayanya. Daftar berikut ini memberikan beberapa contoh, meskipun tidak lengkap.

Amazon EC2

Di Amazon Elastic Compute Cloud, Anda masuk ke suatu instans dengan key pair (untuk instans Linux) atau menggunakan nama pengguna dan kata sandi (untuk instans Microsoft Windows).

Untuk informasi selengkapnya, lihat dokumentasi berikut:

  • Memulai dengan Instans Linux Amazon EC2 di Panduan Pengguna Amazon EC2 untuk Instans Linux

  • Memulai dengan Instans Windows Amazon EC2 di Panduan Pengguna Amazon EC2 untuk Instans Windows

Amazon RDS

Di Amazon Relational Database Service, Anda masuk ke mesin basis data dengan nama pengguna dan kata sandi yang terikat dengan basis data tersebut.

Untuk informasi selengkapnya, lihat Memulai dengan Amazon RDS di Panduan Pengguna Amazon RDS.

Amazon EC2 dan Amazon RDS

Di Amazon EC2 dan Amazon RDS, Anda menggunakan grup keamanan untuk mengontrol lalu lintas ke suatu instans atau basis data.

Untuk informasi selengkapnya, lihat dokumentasi berikut:

  • Grup Keamanan Amazon EC2 untuk Instans Linux di Panduan Pengguna Amazon EC2 untuk Instans Linux

  • Grup Keamanan Amazon EC2 untuk Instans Windows di Panduan Pengguna Amazon EC2 untuk Instans Windows

  • Grup Keamanan Amazon RDS di Panduan Pengguna Amazon RDS

WorkSpaces

Di Amazon WorkSpaces, pengguna masuk ke desktop dengan nama pengguna dan kata sandi.

Untuk informasi selengkapnya, lihat Memulai WorkSpaces di Panduan WorkSpaces Administrasi Amazon.

Amazon WorkDocs

Di Amazon WorkDocs, pengguna mendapatkan akses ke dokumen bersama dengan masuk menggunakan nama pengguna dan kata sandi.

Untuk informasi selengkapnya, lihat Memulai dengan Amazon WorkDocs di Panduan WorkDocs Administrasi Amazon.

Metode kontrol akses ini bukan bagian dari IAM. IAM memungkinkan Anda mengontrol bagaimanaAWS produk ini dikelola—membuat atau menghentikan instans Amazon EC2, menyiapkan WorkSpaces desktop baru, dan sebagainya. Yaitu, IAM membantu Anda mengontrol tugas yang dilakukan dengan membuat permintaan ke Amazon Web Services, dan membantu Anda mengontrol akses keAWS Management Console. Namun, IAM tidak membantu Anda mengelola keamanan untuk tugas seperti masuk ke sistem operasi (Amazon EC2), basis data (Amazon RDS), desktop (Amazon WorkSpaces), atau situs kolaborasi (Amazon WorkDocs).

Ketika Anda bekerja dengan produk AWS tertentu, pastikan Anda membaca dokumentasi untuk mempelajari opsi keamanan untuk semua sumber daya yang menjadi milik produk tersebut.
