AWS 리전 관리

하나의 AWS 리전계정은AWS 리소스를 지리적 영역에 모아 놓은 것입니다. 각 AWS 리전은 격리되어 있으며 다른 리전에 독립적입니다. 리전에서는 내결함성, 안정성 및 복원성을 지원하고 지연 시간을 줄일 수도 있습니다. 여러 리전에서 리소스를 프로비저닝하여 애플리케이션이 가용 상태를 유지하며 리전 중단의 영향을 받지 않도록 할 수 있습니다.

한 리전에서 생성한 리소스는 AWS 서비스에서 제공하는 복제 기능을 명시적으로 사용하지 않는 한 다른 리전에 존재하지 않습니다. 예를 들어, Amazon S3와 Amazon EC2 교차 리전 복제를 지원합니다. AWS Identity and Access Management(IAM) 과 같은 일부 서비스에는 지역 리소스가 없습니다.

정책 조건을 사용하여AWS 서비스 에 대한 액세스를 제어할 수AWS 리전 있습니다.

계정을 통해 자신이 사용할 수 있는 리전을 결정합니다.

  • 하나의 계정은 여러 개의 리전을AWS 계정 제공하므로 사용자는 자신의 요구 사항에 맞는 위치에서AWS 리소스를 시작할 수 있습니다. 예를 들어 유럽의 고객들과 좀더 가까운 곳에 위치하거나 또는 법적 요구사항을 준수하기 위해 유럽에 소재한 위치에서 인스턴스를 실행할 필요가 있을 수 있습니다.

  • AWSGovCloud(미국 서부) 계정은AWSGovCloud (미국 서부) 리전 및AWSGovCloud (미국 동부) 리전에 액세스할 수 있습니다. 자세한 정보는 AWS GovCloud (US)을 참조하세요.

  • Amazon AWS(중국) 계정은 오직 베이징 및 닝샤 리전에 대한 액세스 권한을 제공합니다. 자세한 내용은 중국 Amazon Web Services를 참조하세요.

리소스

  • 리전 이름 및 코드 목록은 이 표를 참조하십시오.

  • 사용할 수 있는 지역과 향후 출시될 지역의 지도를 보려면 지역 및 가용 영역을 참조하십시오.

  • 각 지역 (엔드포인트 제외) 에서 지원되는AWS 서비스 목록은 AWS지역 서비스 목록을 참조하십시오.

리전 활성화

다음 리전 중 하나를 사용하려면 먼저 해당 리전을 계정에 활성화해야 합니다.

이름 코드
아프리카(케이프타운) af-south-1
아시아 태평양(홍콩) ap-east-1
아시아 태평양(하이데라바드) ap-south-2
아시아 태평양(자카르타) ap-southeast-3
아시아 태평양(멜버른) ap-southeast-4
유럽(밀라노) eu-south-1
유럽(스페인) eu-south-2
유럽(취리히) eu-central-2
중동(바레인) me-south-1
중동(UAE) me-central-1

리전을 활성화하면 AWS에서 해당 리전의 계정을 준비하는 작업(예: IAM 리소스를 해당 리전으로 배포)을 수행합니다. 이 프로세스는 대부분의 계정에서 몇 분이 걸리지만 몇 시간이 걸릴 수도 있습니다. 이 프로세스가 완료되기 전까지는 지역을 사용할 수 없습니다.

요구 사항

리전을 활성화하려면 적절한 권한이 있어야 합니다. 예제 IAM 정책을 보려면 IAM 사용 설명서의 활성화 및 비활성화AWS 리전 허용을 참조하십시오.

콘솔을 이용하여 리전 활성화하려면

  1. AWS Management Console에 로그인합니다.

  2. 콘솔의 오른쪽 상단에서 현재 지역을 선택하여 지역 목록을 확장하고 목록 하단으로 스크롤한 다음 지역 관리를 선택합니다.

  3. 사용하려는 지역의 작업 열을 확인합니다.

    • 상태 열에 기본적으로 활성화됨이 포함된 경우 추가 조치가 필요하지 않습니다. 리전은 계정에 자동으로 활성화되었습니다.

    • 상태 열에 비활성화됨이 포함된 경우 활성화를 선택합니다. 정보 텍스트를 검토한 다음 지역 활성화를 선택합니다. 지역을 사용할 준비가 될 때까지 기다려야 합니다.

를 사용하여 지역을 활성화하려면AWS CLI

다음 영역 활성화 명령을 사용하십시오.

aws account enable-region --region-name af-south-1

리전 사용 중지

자동으로 활성화되지 않는 지역을 활성화한 경우 필요한 경우 지역을 비활성화할 수 있습니다. 지역을 비활성화하면 최종 일관성에 따라 이 지역의 리소스를 사용할 수 없게 됩니다. 하지만 리소스는 삭제되지 않습니다. 이 지역을 다시 활성화하면 액세스할 수 있습니다.

요구 사항

  • 기본적으로 활성화된 지역은 비활성화할 수 없습니다. (이전 섹션의 표 참조)

  • 지역을 비활성화하려면 지역을 비활성화할 수 있는 권한이 있어야 합니다. 예제 IAM 정책을 보려면 IAM 사용 설명서의 AWS지역 활성화 및 비활성화 허용을 참조하십시오.

  • 지역을 비활성화하기 전에 해당 지역에서 모든 리소스를 제거하는 것이 좋습니다. 리전을 비활성화한 후에는 IAM 보안 주체를 사용하는AWS 관리 콘솔 또는AWS API를 통해 해당 계정에서 더 이상 리전의 리소스를 보거나 관리할 수 없습니다. 하지만 해당 리전의 리소스에서는 계속 요금이 발생합니다. 자세한 내용은 AWS Billing and Cost Management사용자 안내서의 영역 활성화 및 비활성화를 참조하십시오.

콘솔을 이용하여 리전 사용 중지

  1. AWS Management Console에 로그인합니다.

  2. 콘솔의 오른쪽 상단에서 현재 지역을 선택하여 지역 목록을 확장하고 목록 하단으로 스크롤한 다음 지역 관리를 선택합니다.

  3. 해당 지역의 작업 열에서 비활성화를 선택합니다.

  4. 대화 상자에서 정보 텍스트를 검토한 다음 [영역 사용 안 함] 을 선택합니다.

를 사용하여 지역을 비활성화하려면AWS CLI

다음 비활성화 영역 명령을 사용하십시오.

aws account disable-region --region-name af-south-1

를 이용하여 리전 설명AWS CLI

list-region 명령을 사용하여 상태를 포함하여 계정에서 사용할 수 있는 모든 지역을 설명하세요.

aws account list-regions --max-results 50

get-region-opt-status명령을 사용하여 특정 지역의 상태를 설명합니다.

aws account get-region-opt-status --region-name af-south-1

계정에 자동으로 활성화되는 지역의 경우 이러한 명령의 출력에는 다음이 포함됩니다.

"RegionOptStatus": "ENABLED_BY_DEFAULT"

수동으로 활성화해야 하는 지역의 경우 출력에는 다음이 포함됩니다.

"RegionOptStatus": "DISABLED"

지역을 활성화하면 출력에 다음이 포함됩니다.

"RegionOptStatus": "ENABLED"
