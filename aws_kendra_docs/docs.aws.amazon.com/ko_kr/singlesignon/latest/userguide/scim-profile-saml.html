SCIdentity 프로파일 및 SAML 2.0 구현

SCIM과 SAML은 모두 IAM 아이덴티티 센터 구성 시 중요한 고려 사항입니다.

SAML 2.0 구현

IAM Identity Center는 SAML (Security Assertion Language) 2.0을 통한 아이덴티티 페더레이션을 지원합니다. 이를 통해 IAM Identity Center는 외부 ID 제공자의 ID를 인증할 수 있습니다 (). IdPs SAML 2.0은 SAML 어설션을 안전하게 교환하는 데 사용되는 개방형 표준입니다. SAML 2.0은 SAML 기관 (ID 공급자 또는 IdP라고 함) 과 SAML 소비자 (서비스 공급자 또는 SP라고 함) 간에 사용자에 대한 정보를 전달합니다. IAM Identity Center 서비스는 이 정보를 사용하여 페더레이션된 싱글 사인온을 제공합니다. Single Sign-On을 통해 사용자는 기존 ID 제공자 자격 증명을 기반으로 AWS 계정 및 구성된 애플리케이션에 액세스할 수 있습니다.

IAM ID 센터는 SAML IdP 기능을 IAM ID 센터 스토어 또는 외부 ID AWS Managed Microsoft AD 공급자에 추가합니다. 그런 다음 사용자는 Microsoft 365, Concur AWS Management Console 및 Salesforce와 같은 타사 애플리케이션을 포함하여 SAML을 지원하는 서비스에 싱글 사인온할 수 있습니다.

하지만 SAML 프로토콜은 IdP를 쿼리하여 사용자 및 그룹에 대해 알아보는 방법을 제공하지 않습니다. 따라서 IAM Identity Center에 사용자 및 그룹을 프로비저닝하여 IAM Identity Center에서 해당 사용자와 그룹을 인식하도록 해야 합니다.

SCIdentity 프로파일

IAM ID 센터는 도메인 간 ID 관리 시스템 (SCIM) v2.0 표준을 지원합니다. SCIM은 IAM ID 센터 ID를 IdP의 ID와 동기화합니다. 여기에는 IdP와 IAM Identity Center 간의 사용자 프로비저닝, 업데이트 및 프로비저닝 해제가 포함됩니다.

SCIM 구현 방법에 대한 자세한 내용은 단원을 참조하세요자동 프로비저닝. IAM 아이덴티티 센터의 SCIM 구현에 대한 자세한 내용은 IAM 아이덴티티 센터 SCIM 구현 개발자 가이드를 참조하십시오.
