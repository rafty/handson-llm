중요

다음AWS Identity and Access Management (IAM) 조치는 2023년 7월에 표준 지원이aws-portal:ViewAccount 종료됩니다.aws-portal:ModifyAccount 이러한 AWS Billing작업을 세분화된 작업으로 바꾸면AWS Billing,AWS 비용 관리 및AWS 계정 콘솔에 액세스할 수 있습니다.

2023년 3월 6일 이전에AWS 계정 또는AWS Organizations 관리 계정을 생성한 경우 세분화된 작업이 2023년 7월부터 적용됩니다. 세분화된 작업을 추가하는 것이 좋지만, 접두사가 aws-portal 또는purchase-orders인 기존 권한은 제거하지 않는 것이 좋습니다.

2023년 3월 6일 당일 또는 이후에 계정 또는AWS Organizations 관리 계정을AWS 생성한 경우 세분화된 작업이 즉시 적용됩니다.

AWS원래는 올 뉴 (AWS 리전All New) 를 활성화했으며, 따라서 사용자는 모든 리전에서 리소스를 생성할 수 있었습니다. 이제 리전을AWS 추가할 때 새 리전이 기본적으로 비활성화됩니다. 사용자가 새 리전에서 리소스를 생성할 수 있게 하려면 해당 리전을 활성화해야 합니다.

중요

AWS에서는 전역 엔드포인트 대신 리전별AWS Security Token Service (AWS STS) 엔드포인트를 사용하여 지연 시간을 줄일 것을 권장합니다. 리전별AWS STS 엔드포인트의 세션 토큰은 모든AWS 리전에서 유효합니다. 리전별AWS STS 엔드포인트를 사용하는 경우 변경할 필요가 없습니다.

하지만 글로벌AWS STS 엔드포인트 (https://sts.amazonaws.com) 의 세션 토큰은 사용자가 활성화하거나 기본적으로 활성화되어 있는 경우에만 유효합니다.AWS 리전 계정에 대해 새 리전을 활성화하려는 경우 리전별AWS STS 엔드포인트의 세션 토큰을 사용하거나 글로벌AWS STS 엔드포인트를 활성화하여 모든 경우에 유효한 세션 토큰을 발행할 수AWS 리전 있습니다. 모든 리전에서 유효한 세션 토큰이 더 큽니다. 세션 토큰을 저장하는 경우 이러한 큰 토큰은 시스템에 영향을 미칠 수 있습니다.

AWS STS엔드포인트가AWS 리전과 함께 작동하는 방식에 대한 자세한 내용은 AWS STSAWS리전에서의 관리를 참조하십시오.

활성화 및 비활성화에 대한 고려 사항AWS 리전

  • IAM 권한으로 리전에 대한 액세스를 제어할 수 있음

    AWS Identity and Access Management(IAM) 에는 리전 활성화, 비활성화, 가져오기 및 나열을 허용할 사용자를 제어할 수 있는 네 가지 권한이 있습니다. 자세한 내용은 AWS Billing and Cost Management사용 설명서의 Billing and Cost Management 작업 정책을 참조하십시오.

  • 리전 활성화는 무료

    리전 활성화에는 요금이 부과되지 않습니다. 새 리전에서 생성한 리소스에 대해서만 청구됩니다.

  • 리전을 비활성화하면 해당 리전의 리소스에 대한 액세스도 비활성화됨

    Amazon Elastic Compute Cloud (Amazon EC2) 인스턴스 같은AWS 리소스가 아직 있는 리전을 비활성화하면 해당 리전의 리소스에 대한 액세스 권한이 없게 됩니다. 예를 들어, 비활성화된AWS Management Console 리전에 있는 EC2 인스턴스의 구성을 보거나 변경할 때는 해당 또는 다른 프로그래밍 방법을 사용할 수 없습니다.

  • 지역을 비활성화해도 활성 리소스에 대한 요금은 계속 부과됩니다.

    AWS리소스가 아직 있는 리전을 비활성화하면 해당 리소스 (있는 경우) 에 대한 요금은 표준 요율로 계속 부과됩니다. 예를 들어 Amazon EC2 인스턴스가 있는 리전을 비활성화하여 인스턴스에 액세스할 수 없게 되었더라도 그러한 인스턴스의 요금은 그대로 지불해야 합니다.

  • 리전의 비활성 상태가 항상 즉시 표시되는 것은 아님

    지역을 비활성화한 후 서비스 및 콘솔이 일시적으로 표시될 수 있습니다. 지역을 비활성화하면 적용되려면 몇 분에서 몇 시간이 걸릴 수 있습니다.

  • 원래 리전은 기본적으로 활성화됩니다.

    2019년 3월 20일 이전에 도입된 지역은 모두 기본적으로 활성화되며 비활성화할 수 없습니다. 자세한 내용은AWS 리전 의 관리를 참조하십시오 AWS 일반 참조.

  • 리전 활성화에 대해 몇 분에서 몇 시간 소요

    조직 규모와 같은 여러 요인에 따라 리전 활성화에 최대 몇 시간이 걸릴 수 있습니다.

  • 조직에서는 주어진 시간에AWS Organizations 전체에서 20개의 지역 선택 요청을 열 수 있습니다.

    관리 계정에는 해당 조직이 완료될 때까지 언제든지 20개의 미해결 요청이 있을 수 있습니다. 한 번의 요청은 한 계정에서 특정 지역을 활성화하거나 비활성화하는 것과 같습니다.

  • 한 계정당 한 번에 6개의 지역 선택 요청을 진행 중일 수 있습니다.

    한 번의 요청은 한 계정에서 특정 지역을 활성화하거나 비활성화하는 것과 같습니다.

  • 아마존 EventBridge 통합

    고객은 에서 지역 선택 상태 업데이트 알림을 구독할 수 EventBridge 있습니다. 각 상태 변경에 대한 EventBridge 알림이 생성되어 고객이 워크플로를 자동화할 수 있습니다.

  • 표현형 영역-OPT 상태

    옵트인 영역을 활성화/비활성화하는 비동기식 특성으로 인해 region-opt 요청에는 다음과 같은 네 가지 상태가 발생할 수 있습니다.

    • ENABLING

    • DISABLING

    • ENABLED

    • DISABLED

    또는DISABLING 상태일 때는 옵트인 또는 옵트아웃을 취소할ENABLING 수 없습니다. 그렇지 않으면ConflictException a가 던져집니다. 완료된 (활성화/비활성화) region-opt 요청은 주요 기본AWS 서비스의 프로비저닝에 따라 달라집니다. 이 상태에도 불구하고 일부AWS 서비스를 즉시 사용할 수 없을 수도ENABLED 있습니다.

  • 완벽한 통합AWS Organizations

    관리 계정은 해당AWS 조직의 모든 구성원 계정에 대해 region-opt를 수정하거나 읽을 수 있습니다. 회원 계정은 지역 상태도 읽고 쓸 수 있습니다.

다음 예시에서는 조직 내 지정된 구성원 계정에 대한 지역을 활성화합니다. 사용하는 자격 증명은 조직의 관리 계정이나 계정 관리의 위임된 관리자 계정에서 가져온 것이어야 합니다.

aws account enable-region --region-name af-south-1

작업은 비동기식입니다. 다음 명령을 사용하면 요청의 최신 상태를 볼 수 있습니다.
