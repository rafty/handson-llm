AWS액세스 포털에 로그인

IAM 아이덴티티 센터의 사용자는 의 AWS Organizations 회원입니다. IAM Identity Center의 사용자는 특정 로그인 URL을 사용하여 AWS 액세스 포털에 로그인하여 여러 AWS 계정 비즈니스 애플리케이션에 액세스할 수 있습니다. 특정 로그인 URL에 대한 자세한 내용은 섹션을 참조하세요AWS액세스 포털.

IAM Identity Center AWS 계정 사용자로 에 로그인하기 전에 다음과 같은 필수 정보를 수집합니다.

  • 기업 사용자 이름

  • 기업 비밀번호

  • 특정 로그인 URL

참고

로그인한 후 AWS 액세스 포털 세션은 8시간 동안 유효합니다. 8시간 후에 다시 로그인해야 합니다.

AWS액세스 포털에 로그인하려면

  1. 이메일을 통해 제공된 로그인 URL (예https://your_subdomain.awsapps.com/start: 를 브라우저 창에 붙여넣습니다. 그런 다음 Enter 키를 누릅니다.

  2. 회사 자격 증명 (예: 사용자 이름 및 암호) 를 사용하여 로그인합니다.

    참고

    관리자가 이메일 일회용 암호 (OTP) 를 보냈는데 처음 로그인하는 경우 해당 암호를 입력하십시오. 로그인한 후에는 future 로그인할 때 사용할 새 암호를 만들어야 합니다.

  3. 인증 코드를 입력하라는 메시지가 표시되면 이메일에서 확인 코드를 확인하세요. 그런 다음 코드를 복사하여 로그인 페이지에 붙여넣습니다.

    참고

    인증 코드는 일반적으로 이메일을 통해 전송되지만 배송 방법은 다를 수 있습니다. 이메일로 인증 코드를 받지 못했다면 관리자에게 확인 코드에 대한 세부 정보를 확인하세요.

  4. IAM ID 센터에서 사용자에 대해 MFA를 활성화한 경우 이를 사용하여 인증합니다.

  5. 인증 후에는 포털에 나타나는 모든 AWS 계정 및 애플리케이션에 액세스할 수 있습니다.

    1. 로그인하려면 AWS 계정 타일을 AWS Management Console 선택한 다음 관리할 개별 계정을 선택합니다.

      사용자에게 부여된 권한은 관리 콘솔을 열거나 명령줄 또는 프로그래밍 방식으로 액세스하기 위한 자격 증명을 얻을 수 있는 링크와 함께 표시됩니다.

      관리 콘솔 링크를 선택합니다.

    2. 애플리케이션에 로그인하려면 액세스하려는 애플리케이션의 아이콘을 선택합니다.

IAM Identity Center에서 사용자로 로그인하면 세션이라고 하는 정해진 기간 동안 리소스에 액세스할 수 있는 자격 증명이 제공됩니다. 기본적으로 사용자는 한 시간 동안 에 로그인할 수 있습니다. AWS 계정 관리자가 세션에 설정할 수 있는 최대값은 12시간입니다. 세션이 종료되면 다시 로그인할 수 있습니다.
