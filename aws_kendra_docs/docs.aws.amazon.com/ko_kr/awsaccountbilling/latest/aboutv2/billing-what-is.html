AWS 결제란 무엇인가요?

결제 사용 설명서입니다.

AWS 빌링 콘솔에는 AWS 청구서를 결제하고 AWS 비용 및 사용량을 보고하는 기능이 있습니다. 또한 AWS Organizations에 속해 있는 경우 AWS 빌링 콘솔을 사용하여 통합 결제를 관리할 수 있습니다. Amazon Web Services는 AWS 계정을 등록할 때 입력한 신용 카드에 자동으로 비용을 청구합니다. 언제든지 AWS에서 청구할 다른 신용 카드를 지정하는 등 신용 카드 정보를 보거나 업데이트할 수 있습니다. 이 작업은 결제 콘솔의 결제 방법(Payment Methods) 페이지에서 할 수 있습니다. 사용 가능한 빌링 기능에 대한 자세한 내용은 AWS 빌링 콘솔의 기능 섹션을 참조하세요.

AWS 청구서에 대한 정보를 찾는 방법

AWS 청구서에 대해 궁금한 점이 있거나 청구 내역에 대해 이의를 제기하려면 즉시 AWS Support에 연락하여 문의 사항을 해결하세요. 도움을 받는 방법은 AWS 빌링과 관련하여 지원 받기 섹션을 참조하세요. 청구서 페이지 내용을 이해하려면 월별 요금 보기 섹션을 참조하세요.

AWS 빌링 콘솔과 AWS 비용 관리 콘솔 기능의 주요 차이점

결제 콘솔은 AWS Cost Management 콘솔과 긴밀하게 작동합니다. 빌링 콘솔은 진행 중인 결제와 AWS 계정에 등록된 결제 수단을 관리하기 위한 기능을 포함합니다. AWS 비용 관리 콘솔 기능은 미래 비용을 최적화하는 데 유용합니다. 이 두 가지 도구를 모두 사용하면 현재 비용과 지속적인 비용을 전반적으로 관리할 수 있습니다. 비용 최적화를 위한 AWS 리소스에 대한 자세한 내용은 AWS Cost Management User Guide를 참조하세요.

AWS 빌링 기능에 대한 요금 및 마케팅 자료

개별 기능 가격 및 마케팅 자료에 대한 자세한 내용은 AWS 사용한 클라우드 재무 관리를 참조하세요.

AWS 빌링 콘솔의 기능

AWS 빌링 콘솔의 기능을 사용하면 다음 태스크를 수행할 수 있습니다.

  • 계정 관리: AWS Management Console 및 빌링 콘솔을 사용하여 계정 설정을 관리합니다. 여기에는 기본 통화 지정, 대체 연락처 편집, AWS 리전 추가 또는 제거, 세금 정보 업데이트, AWS 계정 해지가 포함됩니다. close your account에는 AWS 계정을 해지하기 전에 고려해야 할 사항이 나와 있습니다. 예를 들어 사용하지 않는 서비스에 대해 요금이 부과되지 않도록, 계정을 해지하기 전에 리소스를 종료해야 합니다.

  • 청구서 보기: AWS 빌링 콘솔을 사용하여 언제든지 과거 청구서 세부 정보나 이번 달의 예상 요금을 볼 수 있습니다. 이 단원에서는 인보이스를 보고, 요금의 PDF 사본을 다운로드하고, 인보이스를 수신하도록 월별 이메일을 설정하는 방법에 대해 간략히 설명합니다. 또한 AWS Cost and Usage Reports와 같이 다른 리소스를 사용하는 방법을 다룹니다.

  • 결제 관리: 결제 통화를 설정하면 예상 청구액을 확인해 원하는 통화로 AWS 인보이스를 결제할 수 있습니다. AWS는 청구액이 확정된 후 원하는 통화로 청구서를 변환합니다. 그때까지는 콘솔에 표시된 원하는 통화 금액은 모두 USD로 예상된 금액입니다. AWS는 환율을 보장합니다. 이는 환불 시 원래 거래와 같은 환율을 사용하기 위함입니다.

    참고

    • AWS Marketplace 인보이스에는 이 서비스를 이용할 수 없습니다. 이러한 인보이스는 미국 달러로 처리됩니다.

    • 이 서비스는 기본 결제 방법이 Visa 또는 MasterCard일 경우에만 사용할 수 있습니다.

    • 환율은 매일 변경됩니다. 인보이스에 적용된 환율은 인보이스가 작성된 시간을 기준으로 한 현재 환율입니다. 결제 콘솔에서 현재 환율을 확인할 수 있습니다.

    • 통화 변환은 Amazon Services LLC에서 제공합니다.

  • AWS 구매 주문 관리: 한 곳에서 여러 구매 주문을 관리하여 셀프 서비스 방식으로 AWS 구매 주문을 관리합니다. 이를 통해 간접비를 줄이고 전반적인 조달 결제 프로세스의 정확성과 효율성을 높일 수 있습니다. 빌링 콘솔을 사용하여 구매 주문을 관리하고 구매 주문을 인보이스에 반영하는 방식을 구성하세요. 이 섹션에서는 콘솔에서 구매 주문과 관련된 알림 추가, 편집, 세부 정보 보기 및 설정을 수행하는 방법에 대해 알아봅니다.

  • AWS Cost Categories: 비용과 사용량을 의미 있는 범주에 매핑하여 AWS Cost Categories로 AWS 비용을 관리하세요. 이 단원에서는 지원되는 차원, 작업, 규칙 유형 및 상태에 대해 콘솔에서 사용되는 용어를 정의합니다. 또한 Cost Categories 내에서 요금을 생성, 편집, 삭제 및 분할하는 방법에 대한 자세한 내용을 제공합니다.

  • 결제 프로필: 결제 프로필을 사용하여 자동 결제에 여러 가지 결제 방법을 할당할 수 있습니다. 둘 이상의 AWS 서비스 제공업체(‘등록 판매자’)로부터 인보이스를 받는 경우 결제 프로필을 사용하여 각 제공업체별로 고유한 결제 방법을 할당할 수 있습니다. 서비스 제공업체의 결제 프로필을 생성하고 나면 결제 프로필을 통해 AWS 청구서가 자동으로 결제됩니다. 이 섹션에서는 Billing 콘솔을 사용하여 사용자 지정 결제 프로필을 설정하는 방법을 알아봅니다.

  • AWS Organizations에 대한 결제 통합: AWS Organizations에서 여러 AWS 계정에 대한 결제를 결합할 수 있도록 통합 결제 기능을 사용하세요. 이 장에서는 통합 결제 프로세스, Amazon Web Services India Private Limited 계정의 차이점 및 할인에 대한 세부 정보를 설명합니다.

관련 서비스

AWS Billing Conductor

AWS Billing Conductor는 재무 소유자별로 계정을 그룹화하고, 결제 파라미터를 구성하고, 그룹별 AWS Cost and Usage Report를 생성하는 사용자 지정 결제 서비스입니다. AWS Billing Conductor는 AWS 솔루션 공급업체 및 기업 고객의 쇼백 및 차지백 워크플로를 지원할 수 있는 완전관리형 서비스입니다.

IAM

결제 서비스와 AWS Cost Management 서비스는 AWS Identity and Access Management(IAM)와 긴밀히 통합되어 있습니다. IAM을 결제와 함께 사용하여 계정의 다른 사용자가 업무 수행에 필요한 수준의 액세스 권한을 갖게 할 수 있습니다.

또한 IAM을 사용하여 결제 정보뿐만 아니라 모든 AWS 리소스에 대한 액세스를 제어할 수 있습니다. AWS 계정 구조의 설정 과정을 너무 많이 진행하기 전에, IAM의 주요 개념과 모범 사례를 숙지하는 것이 중요합니다.

AWS Organizations(통합 결제)

AWS 제품 및 서비스는 소규모 신생 기업에서 대기업에 이르기까지 모든 규모의 회사가 사용할 수 있습니다. 회사 규모가 크거나 성장할 가능성이 있는 경우 통합 결제를 사용하여 회사의 특정 구조를 반영하는 여러 AWS 계정을 설정할 수 있습니다. 예를 들어 회사 전체의 계정 하나와 개별 직원의 계정을 만들거나, 회사 전체용 계정 하나를 만들고 각 직원은 IAM 사용자로 이용할 수 있습니다. 회사 전체의 계정, 회사 내 부서별 또는 팀별 계정, 그리고 각 직원의 계정을 만들 수 있습니다.

여러 계정을 생성하는 경우 AWS Organizations의 통합 결제 기능을 사용하여 관리 계정 아래 모든 멤버 계정을 통합할 수 있습니다. 이렇게 하면, 모든 멤버 계정에 대해 단일 인보이스를 받을 수 있습니다. 자세한 내용은 AWS Organizations의 통합 결제 섹션을 참조하세요.
