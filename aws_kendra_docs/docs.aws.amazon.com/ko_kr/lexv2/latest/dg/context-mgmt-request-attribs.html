요청 속성 설정

요청 속성은 요청별 정보를 포함하며 현재 요청에만 적용됩니다. 클라이언트 애플리케이션은 이 정보를 Amazon Lex로 전송합니다. 전체 세션에서 유지할 필요가 없는 정보를 전달하려면 요청 속성을 사용합니다. 요청 속성을 직접 생성할 수도 있고 사전 정의된 속성을 사용할 수도 있습니다. 요청 속성을 보내려면 요청의x-amz-lex-request-attributes 헤더 RecognizeUtterance또는requestAttributes 필드를 사용하십시오. RecognizeText 요청 속성은 세션 속성처럼 요청 전체에서 유지되지 않기 때문에RecognizeUtterance 반환되거나RecognizeText 응답되지 않습니다.

참고

여러 요청에 걸쳐 유지되는 정보를 보내려면 세션 속성을 사용하십시오.

사용자 정의 요청 속성 설정

사용자 정의 요청 속성은 각 요청에서 봇에 보내는 데이터입니다. 요청amz-lex-request-attributes 헤더나RecognizeUtterance 요청requestAttributes 필드에 정보를 보냅니다.RecognizeText

Amazon Lex로 요청 속성을 전송하려면 속성 string-to-string 맵을 생성해야 합니다. 다음은 요청 속성을 매핑하는 방법을 보여줍니다.

{
   "attributeName": "attributeValue",
   "attributeName": "attributeValue"
}

PostText작업을 수행하려면 다음과 같이requestAttributes 필드를 사용하여 요청 본문에 맵을 삽입합니다.

"requestAttributes": {
   "attributeName": "attributeValue",
   "attributeName": "attributeValue"
}

PostContent작업을 위해 맵을 base64로 인코딩한 다음x-amz-lex-request-attributes 헤더로 보냅니다.

요청 속성으로 이진수 또는 구조화된 데이터를 보내는 경우 먼저 데이터를 단순 문자열로 변환해야 합니다. 자세한 정보는 복잡한 속성 설정을 참조하세요.
