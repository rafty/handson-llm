작동 방식

Amazon Lex V2를 사용하면 텍스트 또는 음성 인터페이스를 사용하여 사용자와의 대화를 위한 애플리케이션을 구축할 수 있습니다. 다음은 Amazon Lex V2를 사용하기 위한 일반적인 단계입니다.

  1. 봇을 만들고 하나 이상의 언어를 추가하세요. 봇이 사용자의 목표를 이해하고, 사용자와 대화를 통해 정보를 끌어내고, 사용자의 의도를 충족하도록 구성하세요.

  2. 봇을 테스트합니다. Amazon Lex V2 콘솔에서 제공하는 테스트 윈도우 클라이언트를 사용할 수 있습니다.

  3. 버전을 게시하고 별칭을 만듭니다.

  4. 봇을 배포합니다. 자체 애플리케이션 또는 Facebook Messenger 또는 Slack과 같은 메시징 플랫폼에 봇을 배포할 수 있습니다.

시작하기 전에 Amazon Lex V2 V2 핵심 개념 및 용어를 익힙니다.

  • 봇 — 봇은 피자 주문, 호텔 예약, 꽃 주문 등과 같은 자동화된 작업을 수행합니다. Amazon Lex V2 봇은 자동 음성 인식 (ASR) 및 자연어 이해 (NLU) 기능으로 구동됩니다.

    Amazon Lex V2 봇은 텍스트 또는 음성으로 제공된 사용자 입력을 이해하고 자연어와 대화할 수 있습니다.

  • 언어 — Amazon Lex V2 봇은 하나 이상의 언어로 대화할 수 있습니다. 각 언어는 다른 언어와 독립적이므로 Amazon Lex V2에서 모국어 단어 및 구문을 사용하여 사용자와 대화하도록 구성할 수 있습니다. 자세한 정보는 Amazon Lex V2에서 지원하는 언어 및 로케일을 참조하세요.

  • 인텐트 — 인텐트는 사용자가 수행하고자 하는 작업을 나타냅니다. 하나 이상의 관련 의도를 지원하도록 봇을 생성합니다. 예를 들어, 피자와 음료를 주문하는 인텐트를 생성할 수 있습니다. 각 의도에 대해 다음 필수 정보를 제공합니다.

    • 인텐트 이름 — 인텐트를 설명하는 이름입니다. 예: OrderPizza.

    • 샘플 발언 — 사용자가 의도를 전달하는 방법 예를 들어, 사용자가 “피자를 주문할 수 있나요?” 또는 “피자를 주문하고 싶어요”라고 말할 수 있습니다.

    • 인텐트를 이행하는 방법 — 사용자가 필요한 정보를 제공한 후 인텐트를 이행하려는 방법. 인텐트를 충족하려면 Lambda 함수를 생성하는 것이 좋습니다.

      Amazon Lex V2가 필요한 처리를 위해 클라이언트 애플리케이션에 정보를 다시 반환하도록 인텐트를 선택적으로 구성할 수 있습니다.

    Amazon Lex V2는 사용자 지정 인텐트 외에도 봇을 빠르게 설정할 수 있는 내장 인텐트를 제공합니다. 자세한 정보는 내장 인텐트 및 슬롯 유형을 참조하세요.

    Amazon Lex는 항상 각 봇에 대한 폴백 인텐트를 포함합니다. 폴백 인텐트는 Amazon Lex가 사용자의 의도를 추론할 수 없을 때마다 사용됩니다. 자세한 정보는 아마존. FallbackIntent 을 참조하세요.

  • 슬롯 — 인텐트에는 0개 이상의 슬롯 또는 파라미터가 필요할 수 있습니다. 의도 구성의 일부로 슬롯을 추가합니다. Amazon Lex V2는 런타임 시 사용자에게 특정 슬롯 값을 입력하라는 메시지를 표시합니다. Amazon Lex V2가 인텐트를 충족하려면 사용자가 필요한 모든 슬롯에 대한 값을 입력해야 합니다.

    예를 들어OrderPizza 인텐트에는 크기, 크러스트 유형 및 피자 수와 같은 슬롯이 필요합니다. 각 슬롯에 대해 슬롯 유형과 Amazon Lex V2가 클라이언트로 보내는 하나 이상의 프롬프트를 제공하여 사용자로부터 값을 도출합니다. 사용자는 “큰 피자를 주세요” 또는 “작은 피자를 먹자”와 같은 추가 단어가 포함된 슬롯 값을 회신할 수 있습니다. Amazon Lex V2는 여전히 슬롯 가치를 이해하고 있습니다.

  • 슬롯 유형 — 각 슬롯에는 유형이 있습니다. 고유한 슬롯 유형을 생성할 수 있으며 기본 제공 슬롯 유형을 사용할 수 있습니다. 예를 들어, OrderPizza 의도에 대한 다음과 같은 슬롯 유형을 만들고 사용할 수 있습니다.

    • Size – 열거 값은 Small, Medium 및 Large입니다.

    • Crust – 열거 값은 Thick 및 Thin입니다.

    Amazon Lex V2에서는 기본 제공 슬롯 유형도 제공합니다. 예를 들어 AMAZON.Number는 주문한 피자의 개수로 사용할 수 있는 내장 슬롯 유형입니다. 자세한 정보는 내장 인텐트 및 슬롯 유형을 참조하세요.

  • 버전 — 버전은 번호가 매겨진 작업의 스냅샷으로, 개발, 베타 배포, 프로덕션 등 워크플로의 여러 부분에서 사용할 수 있도록 게시할 수 있습니다. 버전을 생성하면 버전 제작 당시의 봇을 그대로 사용할 수 있습니다. 버전을 만든 후에는 애플리케이션에서 계속 작업하는 동안 버전이 동일하게 유지됩니다.

  • 별칭 — 별칭은 봇의 특정 버전을 가리키는 포인터입니다. 별칭을 사용하면 클라이언트 애플리케이션에서 사용 중인 버전을 업데이트할 수 있습니다. 예를 들어, 별칭이 봇의 버전 1을 가리킬 수 있습니다. 봇을 업데이트할 준비가 되면 버전 2를 게시하고 별칭이 새 버전을 가리키도록 변경합니다. 애플리케이션은 특정 버전 대신 별칭을 사용하므로 모든 클라이언트는 업데이트할 필요 없이 새 기능을 사용할 수 있습니다.
