支援 IPv6 的 AWS 服務

電腦和智慧型裝置會使用 IP 地址以在網際網路和其他網路上彼此進行通訊。隨著網際網路不斷增長，IP 地址的需求也持續增加。IP 地址最常見的格式為 IPv4。IP 地址的新格式為 IPv6，可提供比 IPv4 更大的地址空間。

IPv6 的 AWS 服務 支援包括支援雙堆疊組態 (IPv4 和 IPv6) 或僅限 IPv6 組態。例如，虛擬私有雲端 (VPC) 是您可以啟動 AWS 資源的 AWS 雲端 邏輯隔離區段。在 VPC 中，您可以建立僅限 IPv4、雙堆疊或僅限 IPv6 的子網路。

AWS 服務 支援透過公有端點進行存取。有些 AWS 服務 也支援透過由 AWS PrivateLink 提供的私有端點進行存取。即使無法透過公有端點支援 IPv6，AWS 服務 仍可透過其私有端點支援 IPv6。支援 IPv6 的端點可以使用 AAAA 記錄回應 DNS 查詢。

其他 IPv6 支援

運算

  • Amazon EC2 支援根據 Nitro System 將執行個體啟動至僅限 IPv6 子網路中。

  • Amazon EC2 可為 Instance Metadata Service (IMDS) 和 Amazon Time Sync Service 提供 IPv6 端點。

聯網與內容交付

  • Amazon VPC 支援建立僅限 IPv6 子網路。

  • Amazon VPC 可支援您子網路上的 DNS64 和 NAT 閘道上的 NAT64，藉此協助 IPv6 AWS 資源與 IPv4 資源進行通訊。

安全性、身分與合規

  • AWS Identity and Access Management (IAM) 支援可在 IAM 政策中支援 IPv6 地址。

  • Amazon Macie 可在個人身分識別資訊 (PII) 中支援 IPv6 地址。

管理與管控

  • AWS CloudTrail 記錄包含來源 IPv6 資訊。

  • AWS CLI v2 支援透過僅限 IPv6 用戶端的 IPv6 連線進行下載。
