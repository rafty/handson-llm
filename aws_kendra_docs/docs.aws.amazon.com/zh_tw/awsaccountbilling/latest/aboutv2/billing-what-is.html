AWS Billing 是什麼？

歡迎使用 Billing 使用者指南。

AWS 帳單主控台包含支付 AWS 帳單並報告 AWS 費用和用量的功能。您也可以使用 AWS 帳單主控台來管理合併帳單 (如果您是 AWS Organizations 的一員)。當您註冊 AWS 帳戶時，Amazon Web Services 會自動向您提供的信用卡收費。您可以隨時檢視或更新信用卡資訊，包含指定其他信用卡來讓 AWS 收費。您可以在帳單主控台的 Payment Methods (付款方式) 頁面執行此操作。如需可用帳單功能的詳細資訊，請參閱 AWS 帳單主控台的功能。

如何找到有關 AWS 帳單的解答

如果您對 AWS 帳單有任何疑問或想針對費用提出申訴，請立即聯絡 AWS Support 以解決您的問題。如需如何取得協助的選項，請參閱 取得 AWS 帳單的協助。若要瞭解您的帳單頁面內容，請參閱 檢視您的每月費用。

AWS 帳單主控台和 AWS 成本管理主控台功能的主要差異

帳單主控台與 AWS 成本管理主控台密切配合。帳單主控台包含可用來管理進行中付款和註冊至您 AWS 帳戶的付款方式的功能。AWS 成本管理主控台的功能對最佳化您的未來成本非常有幫助。您可以同時使用兩者，全方位管理您目前的成本和持續的成本。如需 AWS 資源的相關資訊來最佳化您的成本，請參閱 AWS Cost Management 使用者指南。

AWS 帳單功能的定價和行銷材料

如需個別功能定價和行銷材料的詳細資訊，請參閱使用 AWS 的雲端財務管理。

AWS 帳單主控台的功能

使用 AWS 帳單主控台的功能，您可以執行下列任務：

  • 管理帳戶：使用 AWS Management Console 和帳單主控台來管理帳戶設定。這包括指定預設貨幣、編輯替代聯絡人、新增或移除 AWS 區域、更新稅務資訊以及關閉 AWS 帳戶 帳戶。關閉 AWS 帳戶 帳戶前的 close your account 重點注意事項。例如，應在關閉賬戶之前終止資源，這樣您就無需為未使用的服務付費。

  • 檢視帳單：您可以隨時使用 AWS 帳單主控台，來檢視過去的帳單詳細資訊或當月的預估費用。本節概述如何檢視帳單、下載費用的 PDF 複本、設定每月電子郵件以接收發票。此外還介紹了如何使用其他資源，如AWS 成本與用量報告。

  • 管理付款：透過設定支付貨幣，您可檢視預估帳單並用慣用貨幣支付 AWS 發票。在最終確定帳單後，AWS 才會將帳單轉換為慣用貨幣。接著，在主控台中顯示的慣用貨幣金額總計是以 USD 來預估。AWS 可保證您的匯率。這樣以來，退款會使用與您原始交易相同的匯率。

    注意

    • AWS Marketplace 發票不符合此服務的條件，並以美元處理。

    • 此服務只適用於您的預設付款方法為 Visa 或 MasterCard。

    • 匯率每天變動。發票所用的匯率是建立發票時的最新匯率。您可以在 Billing 主控台查看目前匯率。

    • 貨幣轉換由 Amazon Services LLC 提供。

  • AWS採購訂單管理：在一處即可處理多個採購訂單，以自助方式管理您的 AWS 採購訂單。這有助於降低您的間接費用，並提高從採購到付款整個程序的準確性和效率。使用帳單主控台來管理採購訂單，並設定採購訂單反映發票的方式。在本節中，您將瞭解如何在主控台中新增、編輯、檢視詳細資訊，以及設定與採購訂單相關的通知。

  • AWS 成本類別：可以將您的成本和用量映射到有意義的類別，以便使用 AWS 成本類別來管理您的 AWS 成本。本節定義主控台中針對支援的維度、操作、規則類型和狀態所使用的術語。本節也提供有關如何在 Cost Categories 中建立、編輯、刪除及分割費用的更多資訊。

  • 付款設定檔：您可以使用付款設定檔來為您的自動付款功能指派多種付款方式。如果收到多位 AWS 服務供應商 (「名義賣家」) 的發票，您可以使用付款設定檔為每位服務供應商指派唯一的付款方式。為服務供應商建立付款設定檔後，付款設定檔會自動支付您的 AWS 帳單。在本節中，瞭解如何使用帳單主控台來設定自訂付款設定檔。

  • AWS Organizations 的合併帳單：使用 AWS Organizations 的合併帳單功能，來合併多個 AWS 帳戶的帳單。本章概述了合併帳單程序、Amazon Web Services India Private Limited 帳戶的差異以及折扣詳細資訊。

相關服務

AWS Billing Conductor

AWS Billing Conductor 是一種自訂帳單服務，可依財務所有者對您的帳戶進行分組，配置帳單參數，並生成每個群組的 AWS 成本與用量報告。AWSBilling Conductor 是一項完全託管的服務，可支援 AWS 解決方案供應商和企業客戶的費用報表和拆帳。

IAM

帳單服務和 AWS 成本管理服務與 AWS Identity and Access Management (IAM) 緊密整合。您可以搭配使用 IAM 與 Billing，以確保在您帳戶中工作的其他人擁有完成工作所需的必要存取權。

您也可使用 IAM 來控制對所有 AWS 資源的存取，而不僅限於您的帳單資訊。請務必先熟悉 IAM 的主要概念和最佳實務，然後再設定 AWS 帳戶的結構。

AWS Organizations(合併帳單)

您可以使用 AWS 產品和服務來適應各種規模的公司，包括小型新創公司到大型企業。如果您的公司是大型公司或很可能繼續成長，您可能需要設定多個 AWS 帳戶以反映公司的特定架構。例如，您可擁有整個公司適用的一個帳戶以及每個員工的帳戶，或擁有整個公司適用的一個帳戶 (內含每個員工的 IAM 使用者)。您可擁有整個公司的帳戶、公司內各個部門或團隊的帳戶，以及每位員工的帳戶。

如果您建立多個帳戶，您可以使用 AWS Organizations 的合併帳單功能，在管理帳戶下合併所有成員帳戶。透過這種方式，您可以接收包含所有成員帳戶的單一帳單。如需更多詳細資訊，請參閱 AWS Organizations 的合併帳單。
