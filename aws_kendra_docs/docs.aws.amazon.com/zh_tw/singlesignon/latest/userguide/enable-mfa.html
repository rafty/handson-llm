Multi-Factor Authentication

啟用多重驗證 (MFA) 時，使用者必須使用自己的使用者名稱和密碼登入AWS存取入口網站。這是第一個因素，他們知道的東西。使用者也必須使用代碼或安全金鑰登入。這是第二個因素，他們擁有的東西或他們是什麼。第二個因素可能是從他們的流動裝置產生的驗證碼，或者透過點擊連接到其電腦的保安編碼。綜合而言，除非成功完成有效的 MFA 挑戰，否則這些因素可防止未經授權存取您的AWS資源，從而提供更高的安全性。每個使用者最多可註冊八個 MFA 裝置。

重要

我們強烈建議的安全性最佳實務是，啟用多重驗證。MFA 提供了一種簡單而安全的方法，在用戶名和密碼的默認身份驗證機制之上添加額外的保護層。
