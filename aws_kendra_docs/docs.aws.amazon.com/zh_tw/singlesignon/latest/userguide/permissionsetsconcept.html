許可集

權限集是您建立和維護的範本，可定義一或多個 IAM 政策的集合。權限集可簡化組織中使用者和群組的AWS帳戶存取指派作業。例如，您可以建立資料庫管理員權限集，其中包括管理 AWS RDS、DynamoDB 和 Aurora 服務的原則，並使用該單一權限集為資料庫管理員授與AWS組織內目標AWS帳戶清單的存取權。

IAM 身分中心AWS 帳戶透過權限集，將存取權指派給一或多個使用者或群組。指派許可集時，IAM Identity Center 會在每個帳戶中建立對應的 IAM Identity Center 控制的 IAM 角色，並將許可集中指定的政策連接到這些角色。IAM Identity Center 使用者入口網站或 AWS CLI 來管理角色，並允許您定義的授權使用者擔任該角色。 修改許可集時，IAM Identity Center 會確保有據此更新對應的 IAM 政策和角色。

您可以將AWS受管政策、客戶管理的政策、內嵌政策和工作職能的AWS受管理政策新增至您的權限集。您也可以指派受AWS管政策或客戶受管政策做為許可界限。

若要建立權限集，請參閱建立和管理權限集。
