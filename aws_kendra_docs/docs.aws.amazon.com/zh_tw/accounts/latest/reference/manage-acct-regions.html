指定AWS 區域您的帳戶可以使用

重要

下列AWS Identity and Access Management (IAM) 動作將於 2023 年 7 月終止標準支援：aws-portal:ModifyAccount和aws-portal:ViewAccount。請參閱使用精細AWS Billing動作用精細動作取代這些動作AWS Billing，以便您可以存取AWS成本管理和AWS帳戶主控台。

如果您AWS於 2023 年 3 月 6 日之前建立帳戶或AWS Organizations管理帳戶，則精細動作將在 2023 年 7 月開始生效。建議您新增精細動作，但不要移除具有 aws-portal 或 purchase-orders 字首的現有許可。

如果您AWS於 2023 年 3 月 6 日或之後建立帳戶或AWS Organizations管理帳戶，則精細動作將立即生效。

AWS原其預設啟用所有新AWS 區域區域，以便您的使用者在任何區域建立資源。現在新AWS增區域時，新區域則會在其預設為停用。若您想允許您的使用者在新區域建立資源，您需要啟用區域。

重要

AWS建議您使用地區AWS Security Token Service (AWS STS) 端點而非全域端點，以減少延遲。區域AWS STS端點的工作階段字符在所有AWS區域都有效。如果您使用區域AWS STS端點，您不需要進行任何變更。

不過，來自全域AWS STS端點 (https://sts.amazonaws.com) 的工作階段權杖只AWS 區域有在您啟用時才有效，或預設為啟用。如果您想要為帳戶啟用新的區域，您可以從區域端點使用工作階段字符，也可以從區域AWS STS端AWS STS點發行工作階段字符，這些字符在所有端點都有效AWS 區域。在所有區域都有效的工作階段字符較大。如果您存儲會話令牌，則這些較大的令牌可能會影響您的系統。

如需AWS STS端點如何使用區AWS域的詳細資訊，請參閱AWS STS在AWS區域中管理。

啟用和停用的考量AWS 區域

  • 您可使用 IAM 許可控制區域的存取權。

    AWS Identity and Access Management(IAM) 包括四種許可，可讓您控制使用者可以啟用、停用、取得和列出區域區域。如需詳細資訊，請參閱AWS Billing and Cost Management使用者指南中的 B illing and Cost Management 動作政策。

  • 啟用區域是免費的

    起用區域無須付費。您只需支付在新區域區域中建立的資源費用。

  • 停用區域會同時停用區域內的資源存取

    如果您停用的區域中仍有AWS資源 (例如 Amazon Elastic Compute Compute Compute Compute Compute Compute Compute Compute Compute Compute Compute Compute Compute Compute Compute 例如，您無法使用AWS Management Console或任何程式設計方法來檢視或變更停用區域中任何 EC2 執行個體的組態。

  • 如果停用「區域」，有效資源的費用會繼續計費

    如果您停用的區域中仍有AWS資源，則會以標準費率繼續以標準費率繼續累算資源費用。例如，若您停用的區域中含有 Amazon EC2 執行個體，即使您已無法存取該等執行個體，仍須為執行個體付費。

  • 停用區域可能不會立即生效

    停用某個區域後，服務和主控台可能會暫時顯示。停用區域可能需要幾分鐘到幾小時不等。

  • 原區域則會依預設啟用

    2019 年 3 月 20 日之前推出的區域均預設為啟用狀態，且無法停用。如需詳細資訊，請參閱AWS 區域中的管理AWS 一般參考。

  • 在某些情況下啟用區域可能需要幾分鐘到幾小時不等

    根據多種因素 (例如組織規模) 而定，啟用區域最多可能需要數小時。

  • 組 Organizations 在指定時間內可以在AWS組織中開啟 20 個區域選擇請求

    管理帳戶可以在任何時間點有 20 個未結請求，等待其組織完成。一個請求等於對一個帳戶啟用或停用一個特定區域。

  • 單個帳戶可以在任何給定時間進行 6 個區域選擇請求

    一個請求等於對一個帳戶啟用或停用一個特定區域。

  • 亞馬遜 EventBridge 整合

    客戶可以在中訂閱區域選擇狀態更新通知 EventBridge。系統會針對每個狀態變更建立 EventBridge 通知，讓客戶自動化工作流程。

  • 表現區域選擇狀態

    由於啟用/停用選擇加入區域的非同步性質，區域選擇要求有四種可能的狀態：

    • ENABLING

    • DISABLING

    • ENABLED

    • DISABLED

    當選擇加入或選擇退出處於或DISABLING狀態時，您無法取消該選擇加入ENABLING或退出。否則，ConflictException將拋出。完成 (啟用/停用) 區域選擇要求取決於主要基礎AWS服務的佈建。儘管狀態為，但可能有些AWS服務無法立即使用ENABLED。

  • 全面整合AWS Organizations

    管理帳戶可以修改或讀取區域選擇該AWS組織的任何成員帳戶。成員帳戶也可以讀取/寫入其區域狀態。

啟用或停用獨立帳戶的區域

若要更新您可存取AWS 帳戶的區域，請執行下列程序中的步驟。下面的AWS Management Console程序始終僅適用於獨立上下文。您可以使AWS Management Console用僅檢視或更新您用來呼叫作業之帳戶中的可用區域。

AWS Management Console
若要啟用或停用獨立版的區域AWS 帳戶
最低許可

若要執行下列程序中的步驟，IAM 使用者或角色必須具有以下許可：

  • aws-portal:ViewAccount（需要查看帳戶詳細信息頁面）

  • account:ListRegions(需要檢視清單以AWS 區域及目前是否已啟用或停用)。

  • account:EnableRegion

  • account:DisableRegion

  1. 以具有最低權限的 IAM 使用者或角色身分登入。AWS Management ConsoleAWS 帳戶根使用者

  2. 在視窗右上方選擇您的帳戶名稱，然後選擇「帳戶」。

  3. 在 [帳戶設定] 頁面上，向下捲動至區段AWS 區域。

    注意

    系統可能提示您核准存取此資訊。 AWS傳送要求至與帳戶相關聯的電子郵件地址，以及主要聯絡人電話號碼。選擇要求中的連結，在瀏覽器中開啟該連結，然後核准存取權。

  4. 在「動作」欄中AWS 區域具有選項的每個項目旁邊，根據您是否希望帳戶中的使用者能夠在該區域中建立和存取資源，選擇「啟用」或「停用」。

  5. 如果出現提示，請確認您的選擇。

  6. 完成所有變更後，請選擇 [更新]。

AWS CLI & SDKs

您可以使用下列AWS CLI命令或其AWS SDK 等效作業來啟用、停用、讀取和列出區域選擇狀態：

  • EnableRegion

  • DisableRegion

  • GetRegionOptStatus

  • ListRegions

最低許可

若要執行下列步驟，您必須具有對應至該操作的許可：

  • account:EnableRegion

  • account:DisableRegion

  • account:GetRegionOptStatus

  • account:ListRegions

如果您使用這些個別權限，則可以授予某些使用者僅讀取區域選擇資訊的能力，並授予其他使用者讀取和寫入的能力。

下列範例會為組織中指定的成員帳戶啟用區域。使用的認證必須來自組織的管理帳戶，或來自帳戶管理的委派管理員帳戶。

請注意，您也可以使用相同的指令停用區域，然後enable-region用取代disable-region。

aws account enable-region --region-name af-south-1

此命令如果成功就不會產生輸出。

該操作是異步的。以下命令將允許您查看請求的最新狀態。

aws account get-region-opt-status --region-name af-south-1
  {
    "RegionName": "af-south-1",
    "RegionOptStatus": "ENABLING"
  }

啟用或停用組織中的區域

若要為成員帳戶更新啟用的區域AWS Organizations，請執行下列程序中的步驟。

注意

您必須先執行下列作業，才能從組織中的管理帳戶或委派管理員帳戶執行這些作業：

  • 啟用組織中的所有功能，以管理成員帳戶的設定。這允許管理員控制成員帳戶。其預設在建立組織時，會設定此功能。如果您的組織設定為僅合併帳單，而您想要啟用所有功能，請參閱啟用組織中的所有功能。

  • 啟用「AWS帳戶管理」服務的受信任存取權。若要進行設定，請參閱啟用受信任的存取AWS帳戶管理。

注意

AWS Organizations受管理的政策AWSOrganizationsReadOnlyAccess或AWSOrganizationsFullAccess已更新，以提供存取AWS帳戶管理 API 的權限，以便您可以從AWS Organizations主控台存取帳戶資料。若要檢視更新的受管理策略，請參閱 Organ izationsAWS 受管理策略的更新。

AWS Management Console
若要在組織中啟用或停用區域

  1. 使用組織的管理帳戶認證登入AWS Organizations主控台。

  2. 在AWS 帳戶頁面上，選取您要更新的帳戶。

  3. 選擇 [帳戶設定] 索引標籤。

  4. 在區域下，選取您要啟用或停用的區域。

  5. 選擇 [動作]，然後選擇 [啟用] 或 [停用] 選項。

  6. 如果您選擇 [啟用] 選項，請檢閱顯示的文字，然後選擇 [啟用區域]。

  7. 如果您選擇 [停用] 選項，請檢閱顯示的文字，輸入 [停用] 以確認，然後選擇 [停用區域]。
