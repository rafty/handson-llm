簽署 AWS API 請求

重要

如果您使用 AWS SDK 或 AWS 命令列工具向 AWS 傳送 API 請求，則這些工具會為您簽署 API 請求。如果您不使用 AWS SDK 或 AWS 命令列工具傳送 AWS API 請求，則必須依照本文件所述簽署 AWS API 請求。

當您傳送 API 請求至 AWS 時，必須簽署這些請求，AWS 才能識別寄件者。為了安全起見，使用您的 AWS 安全憑證簽署大多數請求。

當 AWS 服務 收到經過驗證的請求時，其會使用請求中包含的驗證資訊重新建立簽章。如果簽章相符，則服務會處理請求。否則，它會拒絕請求。

Signature Version 4 是 AWS 簽署協定。AWS 也支援擴展，即 Signature Version 4A，其支援多區域 API 請求的簽章。如需詳細資訊，請參閱 GitHub 上的 sigv4a-signing-examples 專案。

簽署請求的時機

當您編寫向 AWS 傳送 API 請求的自訂程式碼時，必須包含簽署請求的程式碼。您可能會因為以下原因編寫自訂程式碼：

  • 您使用的程式設計語言沒有 AWS 軟體開發套件。

  • 您需要完全掌控請求傳送至 AWS 的狀況。

為什麼要簽署請求

簽署程序有助於以下列方式保護請求的安全：

  • 驗證請求者的身分

    必須由具有有效存取金鑰的人傳送請求。

  • 保護傳輸中的資料

    為了防止傳送中的請求遭到竄改，有些請求元素可用來計算請求的雜湊 (摘要)，而產生的雜湊值會包含在請求中。當 AWS 服務 收到請求時，其會使用相同的資訊來計算雜湊，並根據請求中的雜湊值進行比對。如果值不相符，AWS 便會拒絕請求。

  • 抵禦潛在重播攻擊的侵害

    在大部分情況下，請求必須在其時間戳記的五分鐘內到達 AWS。否則，AWS 會拒絕該請求。
