HookConfiguration

提供在中 AWS Lambda 叫用 Lambda 函數的組態資訊，以便在將文件擷取至 Amazon Kendra 時變更文件中繼資料和內容。PreExtractionHookConfiguration如果要在原版或原始文件上套用進階變更，您可以使用設定 Lambda 函數。如果要在 Amazon Kendra 結構化文件上套用進階變更，您必須使用 PostExtractionHookConfiguration. 設定 Lambda 函數。您只能叫用一個 Lambda 函數。不過，這個函數可以視需求叫用其他函數。

如需詳細資訊，請參閱在擷取過程中自訂文件中繼資料。

內容

InvocationCondition

用於應該叫用 Lambda 函數時機的條件。

例如，您可以指定一個條件，如果有空的日期時間值，則 Amazon Kendra 應該叫用插入目前日期時間的函數。

類型：DocumentAttributeCondition 物件

必要：否

LambdaArn

此為角色的 Amazon Resource Name (ARN)，該角色擁有在擷取過程中執行 Lambda 函數的許可。如需詳細資訊，請參閱 Amazon Kendra 的 IAM 角色。

類型：String

長度限制：長度下限為 1。長度上限為 2048。

模式：/arn:aws[a-zA-Z-]*:lambda:[a-z]+-[a-z]+-[0-9]:[0-9]{12}:function:[a-zA-Z0-9-_]+(\/[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12})?(:[a-zA-Z0-9-_]+)?/

必要：是

S3Bucket

在變更前後儲存原版、原始文件或結構化、已剖析的文件。如需詳細資訊，請參閱 Lambda 函數的資料合約。

類型：String

長度限制：長度下限為 3。長度上限為 63。

模式：[a-z0-9][\.\-a-z0-9]{1,61}[a-z0-9]

必要：是
