BatchPutDocument

將一或多個文件新增到索引。

BatchPutDocumentAPI 可讓您擷取 Amazon S3 儲存貯體內文件或一組存放在 Amazon S3 儲存貯體上的文件或一組文件。使用此 API 可將文字和非結構化文字內嵌到索引中、將自訂屬性新增至文件，以及將存取控制清單附加至新增至索引的文件。

文件會以非同步方式編製索引。您可以使用查看批次的進度AWS CloudWatch。任何與處理批次相關的錯誤訊息都會傳送至您的AWS CloudWatch 記錄。

如需使用 Python 和 Java SDK 擷取內嵌文件的範例，請參閱將檔案直接新增至索引。

請求參數

如需所有動作共用參數的相關資訊，請參閱一般參數。

請求接受採用 JSON 格式的下列資料。

CustomDocumentEnrichmentConfiguration

在您使用BatchPutDocument API 的文件擷取過程中，變更文件中繼資料和內容的組態資訊。

如需有關如何建立、修改和刪除文件中繼資料，或在將文件導入 Amazon Kendra 時進行其他內容變更的詳細資訊，請參閱擷取程序期間自訂文件中繼資料。

類型：CustomDocumentEnrichmentConfiguration 物件

必要：否

Documents

要新增至索引的一或多個文件。

文件具有下列檔案大小限制。

  • 任何檔案的總大小為 50 MB

  • 5 MB 為任何文件提取的文本

如需詳細資訊，請參閱 配額。

類型：Document 物件陣列

陣列成員：項目數下限為 1。項目數上限為 10。

必要：是

IndexId

要將文件新增到的索引識符。您需要首先使用CreateIndex API 創建索引。

類型：字串

長度限制：固定長度為 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必要：是

RoleArn

角色的 Amazon Resource Name (ARN)，具備可存取您的 S3 儲存貯體許可。如需詳細資訊，請參閱 Amazon Kendra 的 IAM 存取角色。

類型：字串

長度限制：長度下限為 0。長度上限為 1284。

模式：arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

必要：否

回應元素

如果動作成功，則服務傳回 HTTP 200 回應。

服務會傳回下列 JSON 格式的資料。

FailedDocuments

因為文件驗證檢查失敗而未新增至索引的文件清單。每個文件都包含錯誤訊息，指出無法將文件新增至索引的原因。

如果將文件新增至索引時發生錯誤，則會在AWS CloudWatch 記錄檔中報告錯誤。如需詳細資訊，請參閱使用 Amazon Kendra CloudWatch 日誌監控 Amazon Kendra

類型：BatchPutDocumentResponseFailedDocument 物件陣列

錯誤

如需所有動作常見的錯誤相關資訊，請參閱常見錯誤。

AccessDeniedException

您沒有足夠的存取權限，無法執行此動作。請確定您擁有必要的權限原則和使用者帳戶，然後再試一次。

HTTP 狀態碼：400

ConflictException

與請求發生衝突。請修正任何與資源不一致的問題，然後再試一次。

HTTP 狀態碼：400

InternalServerException

用於 Amazon Kendra 服務的內部伺服器發生問題。請稍候幾分鐘後再試一次，或聯絡 Sup port 尋求協助。

HTTP 狀態碼：500

ResourceNotFoundException

您要使用的資源不存在。請檢查您是否提供了正確的資源，然後再試一次。

HTTP 狀態碼：400

ServiceQuotaExceededException

您已超過 Amazon Kendra 服務的設定限制。如需詳細資訊，請參閱配額，或聯絡 Sup port 部門以詢問有關增加限制的問題。

HTTP 狀態碼：400

ThrottlingException

由於請求調節，因此請求遭到拒絕。請減少申請數目，然後再試一次。

HTTP 狀態碼：400

ValidationException

輸入不符合 Amazon Kendra 服務設定的限制。請提供正確的輸入，然後再試一次。

HTTP 狀態碼：400
