Microsoft Microsoft 團隊

微軟團隊是用於消息傳遞，會議和文件共享的企業協作工具。如果您是微軟團隊使用者，您可以使用索Amazon Kendra引您的微軟團隊資料來源。

您可以使用Amazon Kendra主控台和 TemplateConfigurationAPI 連線Amazon Kendra到您的微軟團隊資料來源。

如需疑難排解Amazon Kendra Microsoft Teams 資料來源連接器，請參閱對資料來源進行。

先決條件

在您可以用Amazon Kendra來索引您的 Microsoft 團隊資料來源之前，請先在您的 Microsoft 小組和AWS帳戶中進行這些變更。

在 Microsoft Teams 中，確定您具有：

  • 在辦公室 365 中創建了一個微軟團隊帳戶。

  • 複製您的微軟 365 租戶識別碼。您可以在 Azure 作用中目錄入口網站的內容中找到您的租用戶識別碼。

  • 配置 OAuth 2.0 憑證 Token，其中包含用戶端 ID 和用戶端密碼。如需詳細資訊，請參閱有關管理團隊存取權杖的 Azure 文件。

  • 添加了必要的權限。您可以選擇新增所有權限，也可以根據要編目的實體選取較少的權限來限制範圍。下面是對應實體的權限表：

  • 核取每個文件在 Microsoft Teams 中，以及您打算用於相同索引的其他資料來源中是唯一的。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，確定您具有：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引 ID。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

  • 將您的微軟團隊驗證憑據存儲在AWSSecrets Manager密鑰中，並且如果使用 API，則記錄密碼的 ARN。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

如果您沒有現有的IAM角色或密碼，則可以在將 Microsoft Teams 資料來源連線到時，使用主控台建立新IAM角色和Secrets Manager密碼Amazon Kendra。如果您使用 API，則必須提供現有IAM角色和Secrets Manager密碼的 ARN 以及索引 ID。

連接說明

若要連線Amazon Kendra到您的 Microsoft Teams 資料來源，您必須提供 Microsoft Teams 資料來源的必要詳細資料，Amazon Kendra以便能夠存取您的資料。如果您尚未配置微軟團隊查Amazon Kendra看先決條件。

Console

若要連線Amazon Kendra到微軟團隊

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 [Microsoft Teams 連接器]，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資訊頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，在新增標籤中，用來搜尋和篩選資源，或追蹤AWS成本。

    5. 選擇 下一步。

  6. 在定義存取和安全性頁面上，輸入以下資訊：

    1. 來源 — 輸入您的微軟 365 租用戶識別碼。您可以在 Azure 作用中目錄入口網站的內容中找到您的租用戶識別碼。

    2. AWSSecrets Manager密碼 — 選擇現有密碼或建立新Secrets Manager密碼來儲存您的 Microsoft Teams 驗證認證。如果您選擇建立新密碼，則會開啟一個AWSSecrets Manager密碼視窗。

      1. 在「建立AWSSecrets Manager密碼視窗」中輸入下列資訊：

        1. 秘密名稱 — 您秘密的名稱。前綴 'AmazonKendra-微軟團隊-' 會自動添加到您的密碼名稱中。

        2. 針對用戶端識別碼和用戶端密碼：輸入您從 Microsoft Teams 帳戶產生的驗證認證值。

      2. 選擇 儲存 。

    3. 付款模式 — 您可以為您的帳戶選擇授權和付款模式。Model A 付款模式僅限於需要符合安全性規範的授權和付款模式。B 型付款模式適用於不需要安全性合規的授權和付款模式。

    4. V@@ irtual Private Cloud (VPC) — 您可以選擇使用 VPC。如果是這樣，您必須新增子網路和 VPC 安全性群組。

    5. 身分爬行者程式 — 啟動身分爬行者程式時，會Amazon Kendra同步識別資訊。如果您選擇關閉身分識別爬行者程式，則必須使用 PutPrincipalMappingAPI 上傳主體資訊。

    6. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    7. 選擇 下一步。

  7. 在配置同步設定頁面上，輸入以下資訊：

    1. 同步內容 — 選取要同步的內容。

    2. 其他配置 — 您可以選擇性地使用這些設置來索引某些內容，而不是同步所有文件。

    3. 同步模式 — 您可以選擇在資料來源內容變更時更新索引的方式。

      1. 如果您選擇完全同步，Amazon Kendra將同步所有實體中的所有內容，無論先前的同步狀態為何。

      2. 如果您選擇新增或修改過的內容同步，只Amazon Kendra會同步新的或修改過的內容。

      3. 如果您選擇新增、修改或刪除的內容同步，只Amazon Kendra會同步新的、修改或刪除的內容。

  8. 在設定欄位映射頁面上，輸入以下資訊：

    1. 預設資料來源欄位 — 從要對映至索引的Amazon Kendra產生預設資料來源欄位中選取。

    2. 新增欄位 — 新增自訂資料來源欄位以建立要對映的索引欄位名稱和欄位資料類型。

    3. 選擇 下一步。

  9. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以從此頁面選擇編輯資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

API

若要連線Amazon Kendra到微軟團隊

您必須使用 TemplateConfigurationAPI 指定資料來源結構描述的 JSON。您必須提供下列資訊：

  • 資料來源 — 您必須將資料來源指定為MSTEAMS。

  • 租用戶識別碼 — 您可以在 Azure 作用中目錄入口網站的內容中找到您的租用戶識別碼。

  • 類型 — 在您呼叫時指定TEMPLATE為「類型」CreateDataSource。

  • 秘密亞馬遜資源名稱 (ARN) — 提供包含您微軟團隊帳戶身份驗證登入資料的Secrets Manager秘密的亞馬遜資源名稱 (ARN)。私密會儲存在具有下列索引鍵的 JSON 結構中：

    {
    "clientId": "client ID",
    "clientSecret": "clientSecret",
}
    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

  • IAM 角色 — 指定RoleArn何時呼叫CreateDataSource以提供具有存取Secrets Manager密碼的權限的IAM角色，以及呼叫 Microsoft Teams 連接器和所需的公用 API 的時間Amazon Kendra。如需詳細資訊，請參閱 Microsoft Teams 資料來源的 IAM 角色。

您也可以新增下列選用功能：

  • V@@ irtual Private Cloud (VPC) — 指定呼叫VpcConfiguration時CreateDataSource。請參閱 設Amazon Kendra定為使用 VPC。

  • 啟用識別爬行者程式 — 指定是否要啟動身分爬行者程式。如果身分識別爬行者程式已停用，您必須使用 PutPrincipalMappingAPI 上傳主體資訊。探索具有特定文件存取權的使用者和群組的識別資訊對於使用者前後關聯篩選非常有用。搜尋結果會根據使用者或使用者對文件的群組存取權進行篩選。如需詳細資訊，請參閱篩選使用者內容。

  • 包含和排除篩選器 — 指定要包含還是排除在 Microsoft Teams 頻道中找到的某些文件。

    注意

    大多數資料來源使用規則運算式模式，這些模式是稱為篩選器的包含或排除模式。如果您指定包含篩選器，則只有符合包含項目篩選器的內容會編製索引。不符合包含篩選器的任何文件都不會建立索引。如果您指定包含和排除篩選器，符合排除篩選器的文件將不會建立索引，即使它們符合包含篩選器也是如此。

  • 欄位對應 — 選擇將您的 Microsoft Teams 資料來源欄位對應至Amazon Kendra索引欄位。如需詳細資訊，請參閱映射資料來源欄位。

如需要設定的其他重要 JSON 金鑰清單，請參閱微軟團隊模板架構。
