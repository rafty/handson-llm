諷刺語

Quip 是一款協作生產力軟體，提供即時文件撰寫功能。您可以使用Amazon Kendra對 Quip 文件夾、文件、文件注解、聊天室和附件進行索引。

您可以使用Amazon Kendra控制台和 QuipConfigurationAPI 連接Amazon Kendra到 Quip 數據源。

如需Amazon Kendra Quip 資料來源連接器的疑難排解，請參閱對資料來源進行。

支援的功能

Amazon KendraQuip 資料來源連接條件支援下列功能：

  • 欄位映射

  • 使用者內容篩選

  • 包含/排除過濾器

  • Virtual Private Cloud (VPC)

先決條件

在您可以用Amazon Kendra來為 Quip 資料來源建立索引之前，請先在 Quip 和AWS帳戶中進行這些變更。

在 Quip 中，請確定您具有下列項目：

  • 具有管理權限的 Quip 帳戶。

  • 創建了包含個人訪問令牌的 Quip 身份驗證憑據。如需詳細資訊，請參閱有關驗證的 Quip 文件。

  • 複製您的 Quip 網站域。例如，https://quip-company.quipdomain.com/browse，其中金域是域名。

  • 核取的每個文件在 Quip 和您打算用於相同索引的其他資料來源中都是唯一的。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，確定您具有下列項目：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引識別碼。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

  • 將您的 Quip 身份驗證憑據存儲在AWSSecrets Manager密鑰中，如果使用 API，則記錄密鑰的 ARN。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

如果您沒有現有的IAM角色或密碼，則可以在將 Quip 資料來源連線到時使用主控台建立新IAM角色和Secrets Manager密碼Amazon Kendra。如果您使用 API，則必須提供現有IAM角色和Secrets Manager密碼的 ARN，以及索引 ID。

連接指示

要連接Amazon Kendra到 Quip 數據源，您必須提供 Quip 數據源的必要詳細信息，以便Amazon Kendra可以訪問您的數據。如果您尚未配置 Quip 以查Amazon Kendra看先決條件。

Console

若要連接Amazon Kendra到 Quip

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯您的用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 Quip 連接器，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資訊頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，在新增標籤中，標籤來搜尋和篩選資源，或追蹤AWS成本。

    5. 選擇 下一步。

  6. 在定義存取和安全性頁面，輸入以下資訊：

    1. Quip 網域名稱 — 輸入您從 Quip 帳戶複製的 Quip。

    2. AWSSecrets Manager秘密 — 選擇現有密碼或建立新Secrets Manager密碼來儲存您的 Quip 驗證認證。如果您選擇建立新密碼，則會開啟一個AWSSecrets Manager密碼視窗。

      1. 在「建立AWSSecrets Manager密碼視窗」中輸入下列資訊：

        1. 秘密名稱 — 您秘密的名稱。前綴 'AmazonKendra-Quip-' 會自動添加到您的密碼名稱中。

        2. Quip 權杖 — 輸入您在 Quip 帳戶中建立的 Quip 個人存取權杖。

      2. 選擇 儲存 。

    3. V@@ irtual Private Cloud (VPC) — 您可以選擇使用 VPC。如果是這樣，您必須新增子網路和 VPC 安全性群組。

    4. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    5. 選擇 下一步。

  7. 在配置同步設定頁面上，輸入以下資訊：

    1. 新增 Quip 資料夾 ID 以進行編目 — 您要編目的 Quip 資料夾 ID。

    2. 其他配置 (內容類型) — 輸入您要編目的內容類型。

    3. 正則表達式模式-包含或排除某些文件的正則表達式模式。您最多可新增 100 個圖樣。

    4. 在同步執行排程中，針對頻率 — 選擇 Amazon Kendra 與資料來源同步的頻率。

    5. 選擇 下一步。

  8. 在設定欄位映射頁面上，輸入以下資訊：

    1. 從產生的預設資料來源欄位中選取您要對映至Amazon Kendra索引的資料來源欄位。

    2. 新增欄位 — 新增自訂資料來源欄位，以建立要對映的索引欄位名稱和欄位資料類型。

    3. 選擇 下一步。

  9. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以從這個頁面來編輯資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

API

若要連接Amazon Kendra到 Quip

您必須使用 QuipConfigurationAPI 指定下列項目：

  • Quip 網站網域 — 例如，https://quip-company.quipdomain.com/browse，其中萬域是網域。

  • 秘密亞馬遜資源名稱 (ARN) — 提供包含 Quip 帳戶身份驗證登入資料的Secrets Manager密碼的亞馬遜資源名稱 (ARN)。私密會儲存在具有下列索引鍵的 JSON 結構中：

    {
    "accessToken": "token"
}
    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

  • IAM 角色 — 指定RoleArn何時呼叫CreateDataSource以提供具有存取Secrets Manager密碼的權限的IAM角色，以及呼叫 Quip 連接器和所需的公用 APIAmazon Kendra。如需詳細資訊，請參閱 Quip 資料來源的 IAM 角色。

您還可以新增下列選用功能：

  • V@@ irtual Private Cloud (VPC) — 指定VpcConfiguration為資料來源組態的一部分。請參閱設Amazon Kendra定以使用 VPC。

  • 包含和排除篩選器 — 指定要包含還是排除某些檔案。

    注意

    大多數資料來源使用規則運算式模式，這些模式是稱為篩選器的包含或排除模式。如果您指定包含篩選器，則只有符合包含項目篩選器的內容會編製索引。不符合包含篩選器的任何文件都不會建立索引。如果您指定包含和排除篩選器，符合排除篩選器的文件將不會建立索引，即使它們符合包含篩選器也是如此。

  • 欄位對映 — 選擇將 Quip 資料來源欄位對映至Amazon Kendra索引欄位。如需詳細資訊，請參閱映射資料來源欄位。

  • 使用者前後關聯篩選 — 依預設Amazon Kendra搜尋資料來源的存取控制清單 (ACL)。ACL 資訊可用來根據使用者或使用者對文件的群組存取權來篩選搜尋結果。如需詳細資訊，請參閱 Quip 資料來源的使用者內容篩選。
