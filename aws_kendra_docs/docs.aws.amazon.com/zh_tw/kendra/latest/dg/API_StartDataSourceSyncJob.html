StartDataSourceSyncJob

啟動資料來源連接器的同步化工作。如果同步任務已在進行中，Amazon Kendra 會傳回ResourceInUseException例外狀況。

請求語法

{
   "Id": "string",
   "IndexId": "string"
}

請求參數

如需有關所有動作共用之參數的資訊，請參閱一般參數。

請求接受採用 JSON 格式的下列資料。

Id

要同步之資料來源連接器的識別碼。

類型：字串

長度限制：長度下限為 1。長度上限為 100。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

必要：是

IndexId

與資料來源連接器搭配使用之索引的識別碼。

類型：字串

長度限制：固定長度為 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必要：是

回應語法

{
   "ExecutionId": "string"
}

回應元素

如果動作成功，則服務傳回 HTTP 200 回應。

服務會傳回下列 JSON 格式的資料。

ExecutionId

識別特定的同步化工作。

類型：字串

長度限制：長度下限為 1。長度上限為 2048。

錯誤

如需所有動作常見的錯誤相關資訊，請參閱常見錯誤。

AccessDeniedException

您沒有足夠的存取權限不符合此動作的限制。請確定您擁有必要的權限原則和使用者帳戶，然後再試一次。

HTTP 狀態碼：400

ConflictException

與請求發生衝突。請修正任何與資源不一致的問題，然後再試一次。

HTTP 狀態碼：400

InternalServerException

用於 Amazon Kendra 服務的內部伺服器發生問題。請稍候幾分鐘，然後再試一次，或聯絡 Sup port 尋求協助。

HTTP 狀態碼：500

ResourceInUseException

您要使用的資源目前正在使用中。請檢查您是否提供了正確的資源，然後再試一次。

HTTP 狀態碼：400

ResourceNotFoundException

您要使用的資源不存在。請檢查您是否提供了正確的資源，然後再試一次。

HTTP 狀態碼：400

ThrottlingException

由於請求調節，因此請求遭到拒絕。請減少申請數目，然後再試一次。

HTTP 狀態碼：400

ValidationException

輸入不符合 Amazon Kendra 服務規定的限制。請輸入不符合的輸入不符合的限制。

HTTP 狀態碼：400
