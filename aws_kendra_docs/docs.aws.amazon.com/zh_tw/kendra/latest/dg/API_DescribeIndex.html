DescribeIndex

取得現有 Amazon Kendra 索引的相關資訊。

請求語法

{
   "Id": "string"
}

請求參數

如需所有動作共用參數的相關資訊，請參閱一般參數。

請求接受採用 JSON 格式的下列資料。

Id

您要取得資訊的索引標識別碼。

類型：字串

長度限制：固定長度為 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必要：是

回應元素

如果動作成功，則服務傳回 HTTP 200 回應。

服務會傳回下列 JSON 格式的資料。

CapacityUnits

對於企業版索引，您可以選擇使用額外的容量來滿足應用程式的需求。這包含用於索引的容量單位。查詢或文件儲存容量為零表示索引使用預設容量。如需索引預設容量及調整此值的詳細資訊，請參閱調整容量。

類型：CapacityUnitsConfiguration 物件

CreatedAt

索引時的 Unix 時間戳記。

類型：Timestamp

Description

索引的描述。

類型：字串

長度限制：長度下限為 0。長度上限為 1000。

模式：^\P{C}*$

DocumentMetadataConfigurations

文件中繼資料或欄位的組態資訊。文件中繼資料是與文件相關聯的欄位或屬性。例如，與每份文件相關聯的公司部門名稱。

類型：DocumentMetadataConfiguration 物件陣列

陣列成員：項目數下限為 0。項目數上限為 500。

Edition

Amazon Kendra 版用於索引。您可以在建立索引時決定版本。

類型：字串

有效值： DEVELOPER_EDITION | ENTERPRISE_EDITION

ErrorMessage

當Status欄位值為時FAILED，ErrorMessage欄位會包含說明原因的訊息。

類型：字串

長度限制：長度下限為 1。長度上限為 2048。

模式：^\P{C}*$

Id

索引的識別碼。

類型：字串

長度限制：固定長度為 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

IndexStatistics

提供常見問題與答案的數目，以及編製索引的文字文件數目的相關資訊。

類型：IndexStatistics 物件

Name

索引的名稱。

類型：字串

長度限制：長度下限為 1。長度上限為 1000。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

RoleArn

為取得 Amazon Kendra 角色的 Amazon Resource Name (ARN)。

類型：字串

長度限制：長度下限為 0。長度上限為 1284。

模式：arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

ServerSideEncryptionConfiguration

用於加密資料的AWS KMS客戶主金鑰 (CMK) 的識別碼，用於加密資料。Amazon Kendra 不支援不對稱 CMK。

類型：ServerSideEncryptionConfiguration 物件

Status

索引的目前狀態。當值為ACTIVE取得索引已準備就緒可供使用。如果Status欄位值為FAILED，則ErrorMessage欄位會包含說明原因的訊息。

類型：字串

有效值： CREATING | ACTIVE | DELETING | FAILED | UPDATING | SYSTEM_UPDATING

UpdatedAt

上次更新索引時的 Unix。

類型：Timestamp

UserContextPolicy

Amazon Kendra 索引的使用者內容政策。

類型：字串

有效值： ATTRIBUTE_FILTER | USER_TOKEN

UserGroupResolutionConfiguration

您是否已啟用從AWS IAM Identity Center (successor to AWS Single Sign-On)身分識別來源擷取群組和使用者存取層級的組態。

類型：UserGroupResolutionConfiguration 物件

UserTokenConfigurations

Amazon Kendra 索引的使用者權杖組態。

類型：UserTokenConfiguration 物件陣列

陣列成員：最多 1 個項目。

錯誤

如需所有動作常見的錯誤相關資訊，請參閱常見錯誤。

AccessDeniedException

您沒有足夠的存取可執行此動作。請確定您擁有必要的權限原則和使用者帳戶，然後再試一次。

HTTP 狀態碼：400

InternalServerException

用於 Amazon Kendra 服務的內部伺服器發生問題。請稍候幾分鐘後再試一次，或聯絡 Sup port 尋求協助。

HTTP 狀態碼：500

ResourceNotFoundException

您要使用的資源不存在。請檢查您是否提供了正確的資源，然後再試一次。

HTTP 狀態碼：400

ThrottlingException

由於請求調節，因此請求遭到拒絕。請減少申請數目，然後重試。

HTTP 狀態碼：400

ValidationException

輸入不符合 Amazon Kendra 服務設定的限制。請提供正確的輸入，然後重試。

HTTP 狀態碼：400
