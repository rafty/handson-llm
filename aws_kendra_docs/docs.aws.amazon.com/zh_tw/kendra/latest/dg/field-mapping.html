對應資料來源欄位

您可以將資料來源中的文件或內容欄位對應至索引中的欄位。例如，如果您的資料來源中有一個名為「dept」的欄位，其中包含文件的部門資訊，則可以將其對應到名為「Deport」的索引欄位。這樣，您可以在查詢文檔時使用該字段。

您也可以對應標準或保留欄位，例如_created_at。如果您的資料來源有一個名為「creation_date」的欄位，您可以將其對應到名為的對等Amazon Kendra保留欄位_created_at。您可以對映大多數資料來源的欄位。

如果您將文件存放在 S3 儲存貯體或 S3 資料來源中，您可以使用中繼資料檔案直接提供自訂屬性。如需詳細資訊，請參閱建立自訂文件欄位。

將資料來源欄位對應至索引欄位需要三個步驟：

  1. 建立索引。如需詳細資訊，請參閱建立索引。

  2. 更新索引以新增自訂欄位。

  3. 建立將資料來源欄位對映至索引欄位的資料來源。

若要更新索引以新增自訂欄位，請使用主控台或UpdateIndex API。您總共可以在索引中新增 500 個自訂欄位。

在主控台中，您可以選擇將資料來源欄位對應至其中一個保留欄位名稱。或者，您可以選擇建立對應至欄位的新索引欄位。對於資料庫資料來源，如果資料庫欄的名稱與保留欄位的名稱相符，則會自動對應欄位和欄。

透過 UpdateIndexAPI，您可以使用DocumentMetadataConfigurationUpdates參數新增自訂欄位和保留欄位。

下面的 JSON 示例使DocumentMetadataConfigurationUpdates用一個名為「部門」的字段添加到索引。

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

在您建立欄位時，您可選擇設定欄位在搜尋中使用此欄位的方式。您可以選擇下列項目：

  • 可顯示 — 決定是否在查詢回應時傳回此欄位。預設值為 true。

  • Facet — 指出此欄位是否可用於建立多面。預設值為 false。

  • 可搜尋 — 決定是否在搜尋中使用此欄位。字串欄位的預設值是 true，以及數字和日期欄位的預設值是 false。

  • 可排序 — 指出此欄位是否可用於對查詢回應進行排序。只能為日期、數字和字串欄位設定。無法為字串清單欄位設定。

下面的 JSON 示例使DocumentMetadataConfigurationUpdates用一個名為「部門」的字段添加到索引，並將其標記為可面的。

"DocumentMetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE",
       "Search": {
           "Facetable": true
       }
   }
]

使用 Amazon Kendra 內置文檔字段

透過 UpdateIndex API，您可以使用DocumentMetadataConfigurationUpdates並指定保留欄位名稱來建立保留或內建欄位，以對應至對等的文件屬性或文件欄位。您也可以透過此方式建立自訂欄位。如果您使用資料來源連接器，大多數都會包含將資料來源文件欄位對映至Amazon Kendra索引欄位的欄位對映。當您建立欄位時，您可以設定Search物件，將欄位設定為可顯示、可圖面、可搜尋和可排序。您可以配置對Relevance象以設置字段的相關性提高持續時間，新鮮度，重要性，排名順序和重要性值。建立欄位後，就無法變更欄位類型。

Amazon Kendra您可以使用下列保留或內建的文件欄位：

  • _authors負責文件內容的一或多個作者清單。

  • _category將文件放置在特定群組中的類別。

  • _created_at以 ISO 8601 格式建立文件的日期和時間。例如，2012-03-25T12:30:10+01:00 是以 ISO 8601 日期時間格式表示的歐洲中部時間 2012 年 3 月 25 日中午 12:30 (加上 10 秒)。

  • _data_source_id包含文件之資料來源的識別碼。

  • _document_body文件的內容。

  • _document_id— 文件的唯一識別碼。

  • _document_title— 文件的標題。

  • _excerpt_page_number顯示文件摘錄的 PDF 檔案中的頁碼。如果您的索引是在 2020 年 9 月 8 日之前建立的，您必須重新建立文件索引，才能使用此屬性。

  • _faq_id如果這是 FAQ 問題和答案，則為他們提供一個唯一的標識符。

  • _file_type文件的檔案類型，例如 pdf 或 doc。

  • _last_updated_at上次更新文件的日期和時間，格式為 ISO 8601 格式。例如，2012-03-25T12:30:10+01:00 是以 ISO 8601 日期時間格式表示的歐洲中部時間 2012 年 3 月 25 日中午 12:30 (加上 10 秒)。

  • _source_uri可使用文件的 URI。例如，在公司網站上的文件 URI。

  • _version文件特定版本的識別元。

  • _view_count檢視文件的次數。

  • _language_code(字串)-套用至文件之語言的程式碼。如果您未指定語言，則預設為英文。如需支援語言 (包括其代碼) 的詳細資訊，請參閱以英文以外的語言新增文件。

對於自訂欄位，您可以使DocumentMetadataConfigurationUpdates用UpdateIndex API 建立這些欄位，就像建立保留欄位或內建欄位一樣。您必須為自訂欄位設定適當的資料類型。建立欄位後，就無法變更欄位類型。

以下是您可以為自訂欄位設定的類型：

  • 日期

  • Number

  • 字串

  • 字串清單

您可以使用主控台或使用 UpdateIndexAPI 建立自訂欄位。建立自訂欄位之後，您可以將其對應至文件屬性，就像使用保留欄位一樣。如果您使用 BatchPutDocumentAPI 將文件新增至索引，則會使用 API 對應屬性。對於從Amazon S3資料來源編製索引的文件，您可以使用包含描述文件屬性的 JSON 結構的中繼資料檔案來對應屬性。對於使用資料庫或允許欄位對應的資料來源編製索引的文件，您可以使用主控台或資料來源組態對應屬性。
