Amazon S3

Amazon S3是將資料當做物件存放在儲存貯體中的物件儲存服務。您可以使Amazon Kendra用索引文件的Amazon S3值區儲存庫。

警告

Amazon Kendra不使用將權限授予Amazon Kendra主體與 S3 儲存貯體互動的儲存貯體政策。而是使用IAM角色。請確定該成員Amazon Kendra未包含在儲存貯體政策中，以避免在意外授與任意主體權限時出現任何資料安全性問題。不過，您可以新增值區政策，以便跨不同帳戶使用Amazon S3儲存貯體。如需詳細資訊，請參閱Amazon S3跨帳戶使用的政策 (在 S3IAM 角色索引標籤的資料來源IAM角色下)。如需 S3 資料來源IAM角色的相關資訊，請參閱IAM角色。

若要連線到Amazon S3，您可以在主控台中指定連線和其他資訊，方法是使用 S3DataSourceConfiguration 或使用 TemplateConfiguration. 如果您使用 TemplateConfiguration 物件，則可以使用 VPC 連線至資料來源。您在打電話 VpcConfiguration 時指定 CreateDataSource。如需詳細資訊，請參閱設定 VPC。

注意

S3DataSourceConfigurationAPI 不支援設定 VPC。只有TemplateConfiguration API 和控制台支持 VPC 配置。

如需Amazon Kendra S3 資料來源連接器的疑難排解，請參閱對資料來源進行。

先決條件

在您可以用Amazon Kendra來為 S3 資料來源建立索引之前，請先在 S3 和AWS帳戶中進行這些變更。

在 S3 中，確定您具有：

  • 複製Amazon S3儲存貯體名稱。

    注意

    您的值區必須與Amazon Kendra索引位於相同的區域，且索引必須具有存取包含文件之值區的權限。

  • 已核取的每個文件在 S3 和您打算用於相同索引的其他資料來源中都是唯一的。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，請確定您具有：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引識別碼。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

如果您沒有現有IAM角色，則可以在將 S3 資料來源連接到時使用主控台建立新IAM角色Amazon Kendra。如果您使用 API，您必須提供現有IAM角色的 ARN 和索引 ID。

連接說明

若要連接Amazon Kendra到 S3 資料來源，您必須提供 S3 資料來源的必要詳細資訊，Amazon Kendra以便能夠存取您的資料。如果您尚未配置 S3 以查Amazon Kendra看先決條件。

Console

若要連線Amazon Kendra到Amazon S3

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯您的用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 [S3 連接器]，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資訊頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，用於新增標籤 — 用於搜尋和篩選資源或追蹤AWS成本。

    5. 選擇 下一步。

  6. 在「定義存取權與安全性」頁面上，輸入下列選用資訊：

    1. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    2. V@@ irtual Private Cloud (VPC) — 您可以選擇使用 VPC。如果是這樣，您必須新增子網路和 VPC 安全性群組。

    3. 選擇 下一步。

  7. 在設定同步設定頁面上，輸入以下資訊：

    1. 在同步範圍中，對於資料來源位置 — 儲存資料所在Amazon S3值區的路徑。選取瀏覽 S3 以選擇儲存貯體。

    2. (選擇性) 中繼資料檔案首碼資料夾位置-儲存中繼資料的資料夾路徑。選取瀏覽 S3 以尋找您的中繼資料資料夾。

    3. (選擇性) 存取控制清單組態檔案位置-包含 JSON 結構的檔案位置路徑，該 JSON 結構指定存放在 S3 資料來源中的檔案的存取設定。選取瀏覽 S3 以尋找您的 ACL 檔案。

    4. (選擇性) 選取解密金鑰 — 選取以使用解密金鑰。您可以選擇使用現有AWS KMS金鑰或建立新的金鑰。

    5. (選擇性) 在其他組態中，對於 Regex 模式 — 新增規則運算式模式以在索引中包含或排除文件。所有路徑均相對於資料來源位置 S3 儲存貯體。您最多可新增 100 個圖案。

    6. 在「同步」模式中，選擇「完整同步模式」和「新增」、「已修改或刪除的內容同步」，以決定當資料來源內容變更時，索引的更新方式。首次將資料來源與 Amazon Kendra 同步時，預設會同步所有內容。

    7. 在同步執行排程中，對於頻率 — 選擇與資料來源同步的頻率。Amazon Kendra

    8. 選擇 下一步。

  8. 在 [設定欄位對映] 頁面上，輸入下列選用資訊：

    1. S3 欄位對應 — 從Amazon Kendra產生的預設資料來源欄位中選取您要對映至索引的欄位。

    2. 新增欄位 — 選擇新增自訂資料來源欄位，以建立要對映的索引欄位名稱和欄位資料類型。

    3. 選擇 下一步。

  9. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以選擇從此頁面編輯資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

TemplateConfiguration API

若要連線Amazon Kendra到Amazon S3

您必須使用 TemplateConfigurationAPI 指定資料來源結構描述的 JSON。您必須提供下列資訊：

  • BucketName包含文件的儲存貯體名稱。

  • IAM 角色 — 指定RoleArn何時呼叫CreateDataSource以提供具有存取Secrets Manager密碼的權限的IAM角色，以及呼叫 S3 連接器和所需的公用 APIAmazon Kendra。如需詳細資訊，請參閱 S3 資料來源的 IAM 角色。

您也可以新增以下選用功能：

  • 包含和排除篩選器 — 指定是否要包含或排除特定檔案名稱、檔案類型、檔案路徑，以及使用 glob 模式 (可以將萬用字元模式擴充到符合指定模式的路徑名稱清單中的模式)。全球模式的例子包括：

    • /myapp/config/*配置目錄中的所有文件

    • /**/*.png— 所有目錄中的所有 .png 檔案

    • /**/*.{png,ico,md}-所有目錄中的所有 .png，.ico 或 .md 文件

    • /myapp/src/**/*.tssrc 目錄（及其所有子目錄）內的所有 .ts 文件

    • **/!(*.module).ts所有的 .ts 檔案，但不是 .module.ts

    注意

    大多數資料來源使用規則運算式模式，這些模式是稱為篩選器的包含或排除模式。如果您指定包含篩選器，則只有符合包含項目篩選器的內容會編製索引。不符合包含篩選器的任何文件都不會建立索引。如果您指定包含和排除篩選器，符合排除篩選器的文件將不會建立索引，即使它們符合包含篩選器也是如此。

  • 欄位對映 — 選擇將 S3 資料來源欄位對應至Amazon Kendra索引欄位。如需詳細資訊，請參閱映射資料來源欄位。

  • V@@ irtual Private Cloud (VPC) — 指定呼叫VpcConfiguration時間CreateDataSource。請參閱 設Amazon Kendra定為使用 VPC。

  • 文件中繼資料組態 — 新增包含文件存取控制資訊、來源 URI、文件作者和自訂屬性等資訊的文件中繼資訊。每個中繼資料檔案都包含單一文件的相關中繼資料。

如需要設定的其他重要 JSON 金鑰清單，請參閱Amazon S3模板架構。
