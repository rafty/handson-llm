DocumentAttributeTarget

將文件擷取至 Amazon Kendra 時，您想要變更的目標文件屬性或中繼資料欄位。

例如，您可以刪除與文件文件關聯的客戶識別號碼，其存在名為 'Customer_ID' 的文件中繼資料欄位中。將目標索引鍵設定為 'Customer_ID'，將刪除旗標設定為 TRUE。這將刪除欄位 'Customer_ID 中的所有客戶 ID 值。這將從每個文件的中繼資料中刪除個人識別資訊。

如果尚未將目標欄位建立為索引欄位，Amazon Kendra 則無法建立該欄位。建立索引欄位後，您就可以使用 DocumentAttributeTarget 建立文件中繼資料欄位。然後，Amazon Kendra 會將您新建立的中繼資料欄位映射到索引欄位。

您也可以將其與 DocumentAttributeCondition.

內容

TargetDocumentAttributeKey

目標文件屬性或中繼資料欄位的識別碼。

例如，'Department' 可以是目標屬性或中繼資料欄位的識別碼，其中包含與文件關聯的部門名稱。

類型：String

長度限制：長度下限為 1。長度上限為 200。

模式：[a-zA-Z0-9_][a-zA-Z0-9_-]*

必要：否

TargetDocumentAttributeValue

您想為目標屬性建立的目標值。

例如，'Finance' 可以是目標屬性索引鍵 'Department' 的目標值。

類型：DocumentAttributeValue 物件

必要：否

TargetDocumentAttributeValueDeletion

TRUE 表示刪除指定目標屬性索引鍵的現有目標值。您無法建立目標值並將其設定為 TRUE。若要建立目標值 (TargetDocumentAttributeValue)，請將此設定為 FALSE。

類型：布林值

必要：否
