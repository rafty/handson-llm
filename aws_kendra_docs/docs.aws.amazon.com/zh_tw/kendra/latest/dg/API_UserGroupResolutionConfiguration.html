UserGroupResolutionConfiguration

提供組態資訊，以從AWS IAM Identity Center (successor to AWS Single Sign-On)身分識別來源取得使用者和群組。這對於使用者前後關聯篩選非常有用，其中搜尋結果會根據使用者或其對文件的群組存取權進行篩選。您也可以使用 PutPrincipalMappingAPI 將使用者對應至其群組，以便在發出查詢時只需提供使用者 ID。

若要在主控台中設定 IAM 身分中心身分識別來源以搭配 Amazon Kendra 使用，請參閱開始使用 IAM 身分中心身分識別來源。您還必須授予必要的許可，才能將 IAM 身分中心與 Amazon Kendra 搭配使用。如需詳細資訊，請參閱 IAM Identity Center 的 IAM 角色。

Amazon Kendra 目前不支援將您的 IAM 身分中心識別來源的AWS組織成員帳戶UserGroupResolutionConfiguration搭配使用。您必須為組織建立線索引UserGroupResolutionConfiguration。

內容

UserGroupResolutionMode

您要用來取得使用者和群組的身分識別存放區提供者 (模式)。 AWS IAM Identity Center (successor to AWS Single Sign-On)是目前唯一可用的模式。您的使用者和群組必須存在於 IAM 身分中心身分識別來源中，才能使用此模式。

類型：字串

有效值： AWS_SSO | NONE

必要：是
