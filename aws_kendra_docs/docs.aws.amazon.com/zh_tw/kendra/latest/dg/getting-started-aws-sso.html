開始使用AWS IAM Identity Center (successor to AWS Single Sign-On)身分識別來源 (主控台)

AWS IAM Identity Center (successor to AWS Single Sign-On)身分識別來源包含有關您的使用者和群組的資訊。這對於設定使用者前後關聯篩選非常有用，其Amazon Kendra中會根據使用者或其群組對文件的存取權限來篩選不同使用者的搜尋結果。

若要建立 IAM 身分中心身分識別來源，您必須啟用 IAM 身分中心並在中建立組織AWS Organizations。當您首次啟用 IAM 身分中心並建立組織時，它會自動預設為身分識別中心目錄做為身分識別來源。您可以變更為使用中目錄 (Amazon 受管或自我管理) 或外部身分識別供應商做為您的身分來源。您必須遵循正確的指引 — 請參閱變更您的 IAM 身分中心身分來源。每個組織僅能有一個 Identity Center。

為了讓您的使用者和群組被指派不同層級的文件存取權限，當您將文件嵌入索引時，您需要在存取控制清單中包含使用者和群組。這可讓您的使用者和群組根據其存取層級來搜尋文件。Amazon Kendra當您發出查詢時，使用者 ID 必須與 IAM 身分中心中的使用者名稱完全相符。

您也必須授予所需的許可，以在上使用 IAM Identity CenterAmazon Kendra。如需詳細資訊，請參閱 IAMIAM Identity Center 的角色。

設定 IAM Identity Center Identity Center Identity Center 的來源

  1. 開啟 IAM 身分中心主控台。

  2. 選擇 [啟用 IAM 身分中心]，然後選擇 [建立AWS組織]。

    依預設會建立 Identity Center 目錄，並傳送電子郵件給您以驗證與組織相關聯的電子郵件地址。

  3. 若要將群組新增至您的AWS組織，請在功能窗格中選擇 [群組]。

  4. 在「群組」頁面上，選擇「建立群組」，然後在對話方塊中輸入群組名稱和說明。選擇 Create (建立)。

  5. 若要將使用者新增至 [Organizations]，請在導覽窗格中選擇 [使用者]。

  6. 在 Users (使用者) 頁面上，選擇 Add user (新增使用者)。在 User details (使用者詳細資訊) 下，指定所有必要欄位。在 Password (密碼) 中，選擇 Send an email to the user (傳送電子郵件給使用者)。選擇 Next (下一步)。

  7. 若要將使用者新增至群組，請選擇群組，然後選取群組。

  8. 在 [詳細資料] 頁面的 [群組成員] 下，選擇 [新增使用者]

  9. 在 [新增使用者至群組] 頁面上，選取要新增為群組成員的使用者。您可以選取多個使用者以新增至群組。

  10. 若要將您的使用者和群組清單與 IAM 身分中心同步，請將您的身分識別來源變更為 Active Directory 或外部身分識別提供者。

    Identity Center 目錄是預設的身分識別來源，如果您沒有由提供者管理自己的清單，則需要使用此來源手動新增使用者和群組。若要變更身分識別來源，您必須遵循正確的指引 — 請參閱變更 IAM 身分中心身分識別來源。

注意

如果使用 Active Directory 或外部身分識別提供者做為身分識別來源，當您指定跨網域身分識別管理 (SCIM) 通訊協定的系統時，必須將使用者的電子郵件地址對應至 IAM 身分中心使用者名稱。如需了解詳細資訊，請參閱啟用 IAM Identity Center 的 SCIM 指南。

設定 IAM 身分中心身分識別來源後，您可以在建立或編輯索引時在主控台啟用此身分識別來源。前往索引設定中的使用者存取控制，然後編輯設定，以允許從 IAM 身分中心擷取使用者群組資訊。您也可以使用UserGroupResolutionConfiguration物件啟用 IAM 身分中心。您可以提供UserGroupResolutionMode asAWS_SSO 並建立授與呼叫sso:ListDirectoryAssociations、、sso-directory:SearchUsers、權限的IAM角sso-directory:ListGroupsForUser色sso-directory:DescribeGroups。

警告

Amazon Kendra目前不支援UserGroupResolutionConfiguration與組AWS織成員帳戶搭配 IAM 身分中心身分識別來源使用。您必須在組織的管理帳戶內建立索引，才能為組織建立索引UserGroupResolutionConfiguration。

以下是如何使用者存取控制設定資料來源的概觀UserGroupResolutionConfiguration，以在使用者內容上篩選搜尋結果。這假設您已為索引建立索引和IAM角色。您可以建立索引，並使用 CreateIndexAPI 提供IAM角色。

使用UserGroupResolutionConfiguration和使用者內容篩選設定資料來源

  1. 建立可授與存取 IAM 身分中心身分識別來源權限的IAM角色。

  2. UserGroupResolutionConfiguration透過將模式設定為AWS_SSO並撥打電話更新索引UpdateIndex以使用 IAM 身分中心進行設定。

  3. 如果您想要使用權杖型使用者存取控制，以在使用者內容上篩選搜尋結果，請在呼叫USER_TOKEN時設UserContextPolicy為UpdateIndex。否則，請針對大多數資料來源連接器Amazon Kendra搜索每份文件的「存取控制清單」。您也可以在中提供使用者和群組資訊，根據 Query API 中的使用者內容篩選搜尋結果UserContext。您也可以使用將使用者對應至其群組，以PutPrincipalMapping便在發出查詢時只需提供使用者 ID。

  4. 建立授與存取資料來源權限的IAM角色。

  5. 設定您的資料來源。您必須提供連線至資料來源的必要連線資訊。

  6. 使用 CreateDataSourceAPI 建立資料來源。提供DataSourceConfiguration物件、索引的 ID、資料來源的IAM角色、資料來源類型，並為資料來源命名。您也可以更新資料來源。
