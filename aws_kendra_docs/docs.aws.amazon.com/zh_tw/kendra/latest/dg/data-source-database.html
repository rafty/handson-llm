Amazon RDS/Aurora

您可以使用資料庫資料來源索引儲存在資料庫中的文件。提供資料庫的連線資訊之後，請Amazon Kendra連接文件並建立索引。

Amazon Kendra支援下列資料庫：

  • Amazon Aurora MySQL

  • Amazon Aurora PostgreSQL

  • Amazon RDS適用於 MySQL 的

  • Amazon RDS (適用於 PostgreSQL)

注意

不支援無伺服器 Aurora 資料庫。

您可以使用Amazon Kendra主控台和 DatabaseConfigurationAPI 連線Amazon Kendra到資料庫資料來源。

若要疑難排解資料Amazon Kendra庫資料來源連接器，請參閱對資料來源進行。

支援的功能

Amazon Kendra資料庫資料來源連接器支援下列功能：

  • 欄位映射

  • 使用者內容篩選

  • Virtual Private Cloud (VPC) (虛擬私有雲端 (VPC))

先決條件

在您可以使用Amazon Kendra索引資料庫資料來源之前，請先在資料庫和AWS帳戶中進行這些變更。

在您的資料庫中，請確定您具有下列項目：

  • 將您的使用者密碼配對記錄為資料來源的基本驗證認證。

  • 複製包含資料庫資料之資料表的主機名稱、連接埠號碼、主機位址和名稱。對於 PostgreSQL，資料表必須是公用資料表。

    注意

    主機和端口告訴在Amazon Kendra哪裡可以找到互聯網上的數據庫服務器。數據庫名稱和表名稱告訴在Amazon Kendra哪裡可以找到數據庫服務器上的文檔數據。

  • 複製資料表中包含文件資料的欄名稱、文件 ID、一到五欄以偵測文件是否已變更，以及對應至自訂索引欄位的選擇性資料表格欄。您可以將任何Amazon Kendra保留的欄位名稱對應至資料表資料行。

  • 複製資料庫引擎類型資訊，例如您是否使Amazon RDS用 MySQL 或其他類型。

  • 核取每個文件在資料庫中，以及您打算用於相同索引的其他資料來源中都是唯一的。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，請確定您具有下列項目：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引識別碼。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

  • 將您的資料庫驗證認證儲存在AWSSecrets Manager密碼中，如果使用 API，則會記下密碼的 ARN。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

如果您沒有現有的IAM角色或密碼，則可以在將資料庫資料來源連線到時，使用主控台建立新IAM角色和Secrets Manager密碼Amazon Kendra。如果您使用 API，則必須提供現有IAM角色和Secrets Manager密碼的 ARN，以及索引 ID。

連接指示

若要連線Amazon Kendra到資料庫資料來源，您必須提供資料庫資料來源的必要詳細資訊，Amazon Kendra以便能夠存取您的資料。如果您尚未配置數據庫，請Amazon Kendra參閱先決條件。

Console

若要連線Amazon Kendra至資料庫

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯您的用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 [資料庫連接器]，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資訊頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，在新增標籤：用於搜尋及篩選資源或追蹤AWS成本的標籤。

    5. 選擇 下一步。

  6. 在定義存取及安全性頁面上，輸入以下資訊：

    1. 端點 — DNS 主機名稱、IPv4 位址或 IPv6 位址。

    2. 連接埠-連接埠號碼。

    3. 資料庫 — 資料庫名稱。

    4. 表格名稱 — 表格名稱。

    5. 在「驗證類型」中選擇「現有」和「新增」，以儲存您的資料庫驗證證明資料。如果您選擇建立新密碼，則會開啟一個AWSSecrets Manager密碼視窗。

      1. 在「建立AWSSecrets Manager密碼視窗」中輸入下列資訊：

        1. 秘密名稱 — 您秘密的名稱。前綴 'AmazonKendra-data-' 會自動添加到您的密碼名稱中。

        2. 對於使用者名稱和密碼 — 輸入資料庫帳戶中的驗證認證值。

        3. 選擇 [儲存驗證]。

    6. V@@ irtual Private Cloud (VPC)：您可以選擇使用 VPC。如果是這樣，您必須新增子網路和 VPC 安全性群組。

      注意

      您必須使用私有子網路。如果 RDS 執行個體位於 VPC 中的公有子網路中，您可以建立私有子網路，該子網路具有對公有子網路中 NAT 閘道的輸出存取權。VPC 組態中所提供的子網路必須在美國西部 (奧勒岡)、美國東部 (維吉尼亞北部) 及歐洲 (愛爾蘭)。

    7. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    8. 選擇 下一步。

  7. 在設定同步設定頁面上，輸入以下資訊：

    1. 根據您的使用案例，在 Aurora MySQL、MySQL、Aurora 和 Postgre SQL 之間進行選擇。

    2. 以雙引號括住 SQL 識別碼 — 選取此選項可用雙引號括住 SQL 識別碼。例如，「columnName」。

    3. ACL 欄和變更偵測欄 — 用於配置Amazon Kendra用於變更偵測和存取控制清單的欄。

    4. 在同步執行排程中，對於頻率 — 選擇與資料來源同步的頻率。Amazon Kendra

    5. 選擇 下一步。

  8. 在設定欄位映射頁面上，輸入以下資訊：

    1. Amazon Kendra預設欄位對映 — 從Amazon Kendra產生的預設資料來源欄位中選取您要對映至索引的欄位。您必須新增document_id和的 [資料庫] 欄值document_body

    2. 自訂欄位對映 — 新增自訂資料來源欄位以建立要對映的索引欄位名稱和欄位資料類型。

    3. 選擇 下一步。

  9. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以從此頁面中選擇編輯您的資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

API

若要連線Amazon Kendra至資料庫

您必須指定下列 DatabaseConfigurationAPI：

  • ColumnConfiguration有關索引應從資料庫何處取得文件資訊的資訊。如需詳細資訊，請參閱 ColumnConfiguration。您必須指定DocumentDataColumnName和DocumentIdColumnName欄位。對應至欄DocumentIdColumnName位的欄必須是整數資料行。以下範例顯示資料庫資料來源的簡單欄組態：

    "ColumnConfiguration": {
    "ChangeDetectingColumns": [
        "LastUpdateDate",
        "LastUpdateTime"
    ],
    "DocumentDataColumnName": "TextColumn",
    "DocumentIdColumnName": "IdentifierColumn",
    "DocoumentTitleColumnName": "TitleColumn",
    "FieldMappings": [
        {
            "DataSourceFieldName": "AbstractColumn",
            "IndexFieldName": "Abstract"
        }
    ]
}

  • ConnectionConfiguration執行資料庫的資料庫引擎類型。DatabaseHost欄位必須是資料庫的Amazon Relational Database Service (Amazon RDS) 執行個體端點。請勿使用叢集端點。

  • DatabaseEngineType連線至資料庫所需的組態資訊。如需詳細資訊，請參閱 ConnectionConfiguration。

  • 秘密 Amazon 資源名稱 (ARN) — 提供包含資料庫帳戶身份驗證登入資料的Secrets Manager密碼的 Amazon 資源名稱 (ARN)。私密會以下列索引鍵儲存在 JSON 結構中：

    {
    "username": "user name",
    "password": "password"
}

    密碼可以包含更多資訊。以下範例顯示一個資料庫組態。

    
"DatabaseConfiguration": {
"ConnectionConfiguration": {
"DatabaseHost": "host.subdomain.domain.tld",
        "DatabaseName": "DocumentDatabase",
        "DatabasePort": 3306,
        "SecretArn": "arn:aws:secretmanager:region:account ID:secret/secret name",
        "TableName": "DocumentTable"
    }
}
    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

  • IAM 角色 — 指定RoleArn何時呼叫CreateDataSource以提供具有存取Secrets Manager密碼的權限的IAM角色，以及呼叫資料庫連接器和所需的公用 APIAmazon Kendra。如需詳細資訊，請參閱資料庫資料來源的 IAM 角色。

您也可以新增以下選用功能：

  • V@@ irtual Private Cloud (VPC) — 指定VpcConfiguration為資料來源組態的一部分。請參閱設Amazon Kendra定以使用 VPC。

    注意

    您只能使用私有子網路。如果 RDS 執行個體位於 VPC 中的公有子網路中，您可以建立私有子網路，該子網路具有對公有子網路中 NAT 閘道的輸出存取權。VPC 組態中所提供的子網路必須在美國西部 (奧勒岡)、美國東部 (維吉尼亞北部) 及歐洲 (愛爾蘭)。

  • 欄位對映 — 選擇將資料庫資料來源欄位對映至Amazon Kendra索引欄位。如需詳細資訊，請參閱映射資料來源欄位。

  • 使用者前後關聯篩選 — 依預設Amazon Kendra搜尋資料來源的存取控制清單 (ACL)。ACL 資訊可用來根據使用者或使用者對文件的群組存取權來篩選搜尋結果。如需詳細資訊，請參閱資料庫資料來源的使用者內容篩選。
