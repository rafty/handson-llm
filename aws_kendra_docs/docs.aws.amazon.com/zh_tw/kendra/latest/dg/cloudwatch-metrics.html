使用亞馬遜監控亞馬遜肯德拉 CloudWatch

若要追蹤索引的運作狀態，請使用 Amazon CloudWatch。使用 CloudWatch，您可以取得索引文件同步處理的度量。您也可以設定 CloudWatch 警示，以便在一或多個量度超過您定義的臨界值時收到通知。例如，您可以監視提交要編製索引的文件數目，或無法編製索引的文件數目。

您必須擁有適當的 CloudWatch 許可才能使用監控 Amazon Kendra CloudWatch。如需詳細資訊，請參閱 Amazon CloudWatch使用者指南 CloudWatch中的 Amazon 身份驗證和存取控制。

檢視 Amazon Kendra 指標

使用 CloudWatch 主控台檢視 Amazon Kendra 指標。

若要檢視指標 (CloudWatch 主控台)

  1. 請登入AWS Management Console並開啟 CloudWatch 主控台，網址為 https://console.aws.amazon.com/cloudwatch/。

  2. 選擇量度，選擇所有度量，然後選擇 Kendra。

  3. 選擇維度、選擇指標名稱，再選擇 Add to graph (新增至圖形)。

  4. 選擇日期範圍的值。所選日期範圍的指標計數會顯示在圖形中。

建立警示

CloudWatch 警示會監控指定時間內的單一指標，並執行一個或多個動作：傳送通知到 Amazon SNS 頂級或 Auto Scaling 政策。動作或動作是以在您指定數個期間與指定閾值相關的指定閾值，以及相對於指定閾值的指標值。 CloudWatch 也可以在警示變更狀態時傳送 Amazon SNS 訊息給您。

CloudWatch 警示只有在狀態變更且持續您指定的一段時間後，才會叫用動作。

設定警示

  1. 請登入AWS Management Console並開啟 CloudWatch 主控台，網址為 https://console.aws.amazon.com/cloudwatch/。

  2. 選擇 Alarms (警示)，然後選擇 Create Alarm (建立警示)。

  3. 選擇 Kendra 量度，然後選擇量度。

  4. 對於 Time Range (時間範圍)，選擇要監控的時間範圍，然後選擇 Next (下一步)。

  5. 輸入 Name (名稱) 和 Description (描述)。

  6. 對於 Whenever (每當)，選擇 >= 並輸入最大值。

  7. 如果您希望 CloudWatch 在達到警示狀態時傳送電子郵件，請在動作區段中，選擇 State is AL ARM (狀態為警示)。對於「傳送通知至」，請選擇郵寄清單，或選擇「新增清單」並建立新的郵寄清單

  8. 在 Alarm Preview (警示預覽) 區段中預覽警示。如果警示符合您的要求，選擇 Create Alarm (建立警示)。

CloudWatch 索引同步工作的測量結果

下表說明資料來源同步任務的 Amazon Kendra 指標。

指標 描述
DocumentsCrawled

執行期間掃描或探索的同步化工作的文件數目。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSubmittedForIndexing

同步化工作提交至索引的文件數目。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSubmittedForIndexingFailed

編製索引失敗的文件數目。如需詳細資訊，請檢查同步化工作的 CloudWatch 記錄內容。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSubmittedForDeletion

同步處理工作要求從索引中移除的文件數目。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSubmittedForDeletionFailed

無法刪除的文件數目。如需詳細資訊，請檢查同步化工作的 CloudWatch 記錄內容。

維度：

  • IndexId

  • DataSourceId

單位：計數

Amazon Kendra 資料來源的指標

下表說明資料來源同步任務的 Amazon Kendra 指標。標有星號 (*) 的指標只能用於 Amazon S3 資料來源。

指標 描述
DocumentsSkippedNoChange *

已檢查且發現沒有變更的文件數量，因此不會提交以進行索引。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSkippedInvalidMetadata *

因為關聯的中繼資料檔案發生問題，所以略過的文件數目。如需詳細資訊，請檢查同步處理執行的 CloudWatch 記錄內容。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsCrawled

檢查的文件檔案數目。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSubmittedForDeletion

已從資料來源刪除並提交刪除的已檢查文件數目。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSubmittedForDeletionFailed

從資料來源刪除失敗的文件數目。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSubmittedForIndexing

檢查並提交以供索引使用的文件數目。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsSubmittedForIndexingFailed

無法編製索引的提交進行理想的文件數目。

維度：

  • IndexId

  • DataSourceId

單位：計數

指標 描述
DocumentsIndexed

已編製索引的文件數目。

維度：

  • IndexId

  • DataSourceId

單位：計數
DocumentsFailedToIndex

維度：

  • IndexId

  • DataSourceId

單位：計數
IndexQueryCount

每分鐘的索引查詢數目。

維度：

  • IndexId

單位：計數
