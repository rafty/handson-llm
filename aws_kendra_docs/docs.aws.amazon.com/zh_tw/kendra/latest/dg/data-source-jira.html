吉拉

Jira 是用於軟件開發，產品管理和錯誤跟踪的項目管理工具。您可以使Amazon Kendra用編制 Jira 專案、問題、注解、附件、工作日誌和狀態的索引。

Amazon Kendra目前僅支持吉拉雲。

您可以使用Amazon Kendra控制台或 JiraConfiguration API 連接Amazon Kendra到 Jira 數據源。如需各項功能支援的清單，請參閱支援的功能。

如需對Amazon Kendra Jira 資料來源連接器進行疑難排解，請參閱對資料來源進行。

支援的功能

Amazon KendraJira 資料來源連接器支援下列功能：

  • 變更日誌

  • 欄位映射

  • 使用者內容篩選

  • 包含/排除過濾器

  • Virtual Private Cloud (VPC) (虛擬私有雲端 (VPC))

先決條件

在您可以用Amazon Kendra來為 Jira 資料來源建立索引之前，請先在 Jira 和AWS帳戶中進行這些變更。

在 Jira，請確定您具有下列項目：

  • 創建了包含 Jira ID（用戶名或電子郵件）和 Jira 憑據（Jira API 令牌）的 Jira API 令牌身份驗證憑據。請參閱有關管理 API 權杖的使用說明文件。

  • 記下您的 Jira 帳戶設定中的 Jira 帳戶網址。例如，公司的 .atlassian.net。

  • 已核取每個文件在 Jira 以及您打算用於相同索引的其他資料來源中都是唯一的。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，請確定您具有下列項目：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引識別碼。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

  • 將您的 Jira 身份驗證憑據存儲在AWSSecrets Manager密鑰中，並且如果使用 API，則記錄密鑰的 ARN。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

如果您沒有現有的IAM角色或密碼，則可以在將 Jira 資料來源連線至時使用主控台建立新IAM角色和Secrets Manager密碼Amazon Kendra。如果您使用 API，則必須提供現有IAM角色和Secrets Manager密碼的 ARN 以及索引 ID。

連接說明

若要連線Amazon Kendra到 Jira 資料來源，您必須提供 Jira 資料來源的必要詳細資訊，Amazon Kendra以便能夠存取您的資料。如果您尚未配置 Jira 以查Amazon Kendra看先決條件。

Console

若要連接Amazon Kendra到吉拉

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯您的用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 [Jira 連接器]，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資訊頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，在新增標籤中 — 標籤來搜尋和篩選資源，或追蹤AWS成本。

    5. 選擇 下一步。

  6. 在定義存取和安全性頁面上，輸入以下資訊：

    1. Jira 帳戶網址 — 輸入您的 Jira 帳戶網址。

    2. AWSSecrets Manager秘密 — 選擇現有密碼或建立新Secrets Manager密碼來儲存您的 Jira 驗證認證。如果您選擇建立新密碼，則會開啟一個AWSSecrets Manager密碼視窗。

      1. 在「建立AWSSecrets Manager密碼視窗」中輸入下列資訊：

        1. 秘密名稱 — 您秘密的名稱。前綴 'AmazonKendra-Jira-' 會自動添加到您的密碼名稱中。

        2. 對於 Jira ID — 輸入 Jira 使用者名稱或電子郵件。

        3. 對於密碼/令牌 — 輸入您從 Jira 帳戶創建的 Jira API 令牌。

      2. 選擇 儲存 。

    3. V@@ irtual Private Cloud (VPC) (VPC) — 您可以選擇使用 VPC。如果是這樣，您必須新增子網路和 VPC 安全性群組。

    4. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    5. 選擇 下一步。

  7. 在配置同步設定頁面上，輸入以下資訊：

    1. 選取要建立索引的 Jira 專案 — 您要編目的 Jira 實體或內容類型。

    2. 狀態、其他元素和問題類型 — 選取內容以精簡索引的範圍。

    3. 變更記錄 — 選取此選項可更新索引，而不是同步所有檔案。

    4. 正則表達式模式-包含或排除某些文件的正則表達式模式。您最多可新增 100 個圖樣。

    5. 在同步執行排程中，針對頻率 — 選擇 Amazon Kendra 與資料來源同步的頻率。

    6. 選擇 下一步。

  8. 在 [設定欄位映射] 頁面上，輸入以下資訊：

    1. 針對「專案」、「問題」、「註解」、「附件」、「工作日誌」— 從Amazon Kendra產生的預設資料來源欄位中選取您要對映至索引的欄位。

    2. 新增欄位 — 新增自訂資料來源欄位，以建立要對映的索引欄位名稱和欄位資料類型。

    3. 選擇 下一步。

  9. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以從此頁面中選擇編輯資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

API

若要連接Amazon Kendra到吉拉

您必須使用 JiraConfigurationAPI 指定下列項目：

  • 資料來源網址 — 指定您的 Jira 帳戶 URL。例如，公司的 .atlassian.net。

  • 秘密亞馬遜資源名稱 (ARN) — 提供包含您 Jira 帳戶身份驗證登入資料的Secrets Manager密碼的亞馬遜資源名稱 (ARN)。密碼會儲存在具有下列索引鍵的 JSON 結構中：

    {
    "jiraId": "Jira user name or email",
    "jiraCredential": "Jira API token"
}
    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

  • IAM 角色 — 指定RoleArn何時呼叫CreateDataSource以提供具有存取Secrets Manager密碼的權限的IAM角色，以及呼叫 Jira 連接器和所需的公用 API 的時間Amazon Kendra。如需詳細資訊，請參閱 Jira 資料來源的 IAM 角色。

您還可以新增下列選用功能：

  • V@@ irtual Private Cloud (VPC) — 指定VpcConfiguration為資料來源組態的一部分。請參閱設Amazon Kendra定以使用 VPC。

  • 變更記錄 — 是否Amazon Kendra應該使用 Jira 資料來源變更記錄機制來決定是否必須在索引中新增、更新或刪除文件。

    注意

    如果您不想Amazon Kendra掃描所有文檔，請使用更改日誌。如果您的變更記錄很大，掃描 Jira 資料來源中的文件所需的時間可能會比處理變更記錄還要Amazon Kendra少。如果您是第一次將 Jira 資料來源與索引同步，則會掃描所有文件。

  • 包含和排除篩選器 — 您可以指定是否要包含或排除某些專案、問題、註解、附件、工作日誌和狀態。

    注意

    大多數資料來源使用規則運算式模式，這些模式是稱為篩選器的包含或排除模式。如果您指定包含篩選器，則只有符合包含項目篩選器的內容會編製索引。不符合包含篩選器的任何文件都不會建立索引。如果您指定包含和排除篩選器，符合排除篩選器的文件將不會建立索引，即使它們符合包含篩選器也是如此。

  • 欄位對映 — 選擇將您的 Jira 資料來源欄位對映至Amazon Kendra索引欄位。如需詳細資訊，請參閱映射資料來源欄位。

  • 使用者前後關聯篩選 — 依預設Amazon Kendra搜尋資料來源的存取控制清單 (ACL)。ACL 資訊可用來根據使用者或使用者對文件的群組存取權來篩選搜尋結果。如需詳細資訊，請參閱 Jira 資料來源的使用者內容篩選。
