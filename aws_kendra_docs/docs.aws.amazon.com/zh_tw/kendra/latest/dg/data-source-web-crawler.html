Amazon Kendra網絡爬蟲

您可以使用Amazon Kendra Web 爬蟲來檢索和索引網頁。

您只能網路爬取使用安全通訊協定「超文字安全傳輸通訊協定」(HTTPS) 的公用網站和網站。如果在網路爬取網站時收到錯誤訊息，可能是網站阻止網路爬取。要檢索內部網站，您可以設置 Web 代理。Web 代理必須面對公開。

選取要索引的網站時，您必須遵守 Amazon 可接受的使用政策以及 Amazon所有其他條款。請注意，您只能使用Amazon Kendra Web 爬蟲程式索引自己的網頁或您有權索引的網頁。要了解如何阻止Amazon Kendra Web 爬蟲索引您的網站，請參閱設定Amazon Kendra網頁爬行者程式的robots.txt檔案。

您可以使用Amazon Kendra主控台和 WebCrawlerConfigurationAPI 連線Amazon Kendra到 Web 爬蟲資料來源。

如需Amazon Kendra Web 爬行者程式資料來源連接器的疑難排解，請參閱對資料來源進行。

先決條件

在您可以使用索Amazon Kendra引您的 wesbites 之前，請檢查您的網站和AWS帳戶的詳細信息。

對於您的網站，請確定您具有下列項目：

  • 複製要索引的網站的種子或網站的網站的網站的網站的網站的網站

  • 對於需要基本驗證才能編目的網站：複製網站的主機名稱和連接埠號碼。

  • 選用性：如果您想要使用 Web Proxy 連線至您要檢索的內部網站，請複製網站的主機名稱和連接埠號碼。Web 代理必須面對公開。 Amazon Kendra支持連接到基本身份驗證支持的 Web 代理服務器，或者您可以在沒有身份驗證的情況下連接。

  • 檢查您要索引的每個網頁文檔是唯一的，並且在您打算用於相同索引的其他數據源中。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，確定您具有下列項目：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引 ID。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

  • 將您的 Web 爬蟲身份驗證憑據存儲在AWSSecrets Manager密鑰中，並且如果使用 API，則記錄密鑰的 ARN。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

如果您沒有現有的IAM角色或密碼，則可以在將 Web 爬行者程式資料來源連線到時，使用主控台建立新IAM角色和Secrets Manager密碼Amazon Kendra。如果您使用 API，則必須提供現有IAM角色和Secrets Manager密碼的 ARN 以及索引 ID。

連接說明

若要連線Amazon Kendra到 Web 爬行者程式資料來源，您必須提供 Web 爬行者程式資料來源的必要詳細資訊，Amazon Kendra以便能夠存取您的資料。如果您尚未設定網路爬蟲以供Amazon Kendra參閱先決條件。

Console

連線Amazon Kendra至網路爬行者程式

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯您的用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 Web 爬行者程式連接器，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資訊頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，對於新增標籤 — 標籤，以搜尋和篩選資源或追蹤AWS成本。

    5. 選擇 下一步。

  6. 在定義存取和安全性頁面上，輸入以下資訊：

    1. 對於「來源」，請根據您的使用案例選擇「來源 URL」和「來源網站地圖」，然後輸入每個網站地圖的值。

      注意

      您最多可新增 10 個來源 URL 和 3 個網站地圖。

      如果您想要檢索 Sitemap，請務必檢查基礎或根網址是否與 Sitemap 頁面上列出的網址所用的基本 URL 相同。例如，如果您的網站地圖網址為 https://example.com/sitemap-page.html，則此網站地圖頁面上列出的網址也應使用基本網址「https://example.com/」。

    2. (選擇性) 對於 Web 代理伺服器 — 輸入下列資訊：

      1. 主機名稱 — 需要 Web Proxy 的主機名稱。

      2. 連接埠號碼 — 主機 URL 傳輸通訊協定所使用的連接埠。連接埠號碼應該是介於 0 到 65535 之間的數值。

      3. 對於 Web Proxy 認證 — 選擇現有密碼或建立新密碼以儲存您的 Web 爬行者程式驗證認證。如果您選擇建立新密碼，則會開啟一個AWSSecrets Manager密碼視窗。

      4. 在「建立AWSSecrets Manager密碼視窗」中輸入下列資訊：

        1. 秘密名稱 — 您秘密的名稱。前綴 'AmazonKendraWebCrawler-' 會自動添加到您的密碼名稱中。

        2. 針對使用者名稱和密碼 — 輸入您網站的這些基本驗證憑證。

        3. 選擇 儲存 。

    3. (選擇性) 具有驗證的主機 — 選取此選項可新增具有驗證的其他主機。

    4. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    5. 選擇 下一步。

  7. 在配置同步設定頁面上，輸入以下資訊：

    1. 抓取範圍 — 選擇您要抓取的網頁類型。

    2. 編目深度 — 從Amazon Kendra應編目的種子 URL 中選取層級數。

    3. 進階編目設定和其他組態輸入下列資訊：

      1. 檔案大小上限 — 要編目的網頁或附件大小上限。最小 1 MB (1 個位元組)。最大值 50 MB。

      2. 每頁最大連結數 — 每頁編目的連結數目上限。連結會依出現順序進行網路爬取。最少 1 個連結/頁面。每頁最多 1000 個連結。

      3. 最大節流 — 每分鐘每個主機名稱網路爬取的 URL 數目上限。至少 1 個網址/主機名稱/分鐘。最多 300 個網址/主機名稱/分鐘。

      4. 正則表達式模式 — 添加模式以僅包含某些 URL，或排除存儲庫中的 URL，文件類型或特定文件。您總共可以有 100 個圖案。

    4. 在同步執行排程中，對於頻率 — 選擇與資料來源同步的頻率。Amazon Kendra

    5. 選擇 下一步。

  8. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以從此頁面上選擇編輯您的資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

連線Amazon Kendra至網路爬行者程式

您必須指定下列 WebCrawlerConfigurationAPI：

  • URL — 指定網站的種子或起點 URL，或要使用SeedUrlConfiguration和SiteMapsConfiguration網站網站的網站的網站的網站的網站的網站的網站的網站的網站的網站的網站

    如果您想要檢索 Sitemap，請務必檢查基礎或根網址是否與 Sitemap 頁面上列出的網址所用的基本 URL 相同。例如，如果您的網站地圖網址為 https://example.com/sitemap-page.html，則此網站地圖頁面上列出的網址也應使用基本網址「https://example.com/」。

  • 秘密 Amazon 資源名稱 (ARN) — 如果網站需要基本身份驗證，您需要提供主機名稱、連接埠號碼和密碼，以儲存使用者名稱和密碼的基本身份驗證登入資料。您可以使用 AuthenticationConfigurationAPI 提供 ARN。私密會儲存在具有下列索引鍵的 JSON 結構中：

    {
    "username": "user name",
    "password": "password"
}

    您可以使用秘密提供 Web 代理憑據。AWSSecrets Manager您可以使用 ProxyConfigurationAPI 提供網站主機名稱和連接埠號碼，以及選擇性地提供儲存 Web Proxy 憑證的密碼。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

您也可以新增以下選用功能：

  • 編目模式 — 選擇僅編目網站主機名稱，或是編目具有子網域的主機名稱，或者也可以編目網頁連結的其他網域。

  • 從種子層級到網站的「深度」或層級數目。例如，如果網站具有 3 個層級 — 索引層級 (本範例中的種子層級)、區段層級和子區段層級，且您只對網路爬取至區段層級的資訊感興趣 (層級 0 到 1)，則您可以將深度設定為 1。

  • 要網路爬取的單個網頁上的 URL 數目上限。

  • 要抓取的網頁大小上限 (以 MB 為單位)。

  • 每分鐘每個網站主機網路爬取的 URL 數目上限。

  • 用於連接和抓取內部網站的 Web Proxy 資訊。

  • 存取和網路爬取需要使用者驗證網站的驗證資訊。

  • 您可以使用「自訂文件強化」工具將 HTML 中繼標記擷取為欄位。如需詳細資訊，請參閱在擷取過程中自訂文件中繼資料。如需擷取 HTML 中繼標記的範例，請參閱 CDE 範例。

  • 包含和排除篩選器 — 指定要包含還是排除特定 URL。

    注意

    大多數資料來源使用規則運算式模式，這些模式是稱為篩選器的包含或排除模式。如果您指定包含篩選器，則只有符合包含項目篩選器的內容會編製索引。不符合包含篩選器的任何文件都不會建立索引。如果您指定包含和排除篩選器，符合排除篩選器的文件將不會建立索引，即使它們符合包含篩選器也是如此。
