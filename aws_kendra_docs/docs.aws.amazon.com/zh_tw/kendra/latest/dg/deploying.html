部署 Amazon Kendra

當需要將Amazon Kendra搜尋部署到您的網站時，我們會提供原始程式碼，讓您可以與 React 一起使用，讓您的應用程式獲得良好的起點。源代碼在修改後的 MIT 許可證下免費提供。您可以按原樣使用它，也可以根據自己的需求進行更改。提供的 React app 是可協助您開始使用的範例。它不是生產就緒應用程式。

若要部署不含程式碼的搜尋應用程式，並在具有存取控制的搜尋頁面產生端點 URL，請參閱Amazon Kendra體驗產生器。

有兩個你可以用 React 的例子：

  • https://kendrasamples.s3.amazonaws.com/kendrasamples-react-app .zip-一個示例反應應用程序，提供示例數據和搜索頁面。

  • https://kendrasamples.s3.amazonaws.com/kendrasamples.zip-您可以添加到現有的 React 應用程序中的庫。

這些範例會在Amazon Kendra主控台的搜尋頁面之後建立模型。它們具有搜索和顯示搜索結果相同的功能。您可以使用整個示例，也可以僅選擇其中一個功能供您自己使用。

若要在Amazon Kendra主控台中查看搜尋頁面的三個元件，請從右側功能表中選擇程式碼圖示 (</>)。將指標懸停在每個部分上，以查看組件的簡短描述並獲取組件的源代碼的 URL。

概要

您可以將示例代碼添加到現有的 React 應用程序以激活搜索。搜尋檔案和元件的結構如下：

  • 主要搜尋頁面 — 這是包含所有元件的主要頁面。這是您將應用程序與Amazon Kendra API 集成的地方。

  • 搜尋列-這是使用者輸入搜尋詞彙並呼叫搜尋功能的元件。

  • 結果-這是顯示結果的元件Amazon Kendra。它有三個組成部分：建議的答案，常見問題解答結果和推薦的文檔。

  • 面 (Facet)-這是在搜尋結果中顯示多面的元件，可讓您選擇多面來限制搜尋。

  • 分頁-這是用來分頁回應的元件Amazon Kendra。

先決條件

開始之前，您必須準備好以下事項：

  • 現有的 React Web 應用程式或範例應用程式。

  • 使用正確程式庫設定的開發環境。

  • Node.js 和故宮已安裝。

  • SDK for Java或AWS SDK for JavaScript。

有關所需庫和AWS SDK 的信息在 zip 文件中的讀我文件中。

設定範例

將Amazon Kendra搜索添加到 React 應用程序的完整過程在示例 zip 文件中包含的自述文件中。

開始使用 kendra-samples-react-app .zip
警告

React 應用程式尚未準備好生產環境。這是部署用於Amazon Kendra搜索的應用程序的示例。

  1. 確保您已完成先決條件，包括下載和安裝 Node.js 和 npm。

  2. 下載 kendra-samples-react-app .zip 並解壓縮。

  3. 打開您的終端機並轉到aws-kendra-sample-app/src/services/。打開local-dev-credentials-template.json並提供您的憑據。請勿將此檔案新增到任何公用存放庫。

  4. 轉到aws-kendra-sample-app/src並安裝依賴項。執行 npm install。

  5. 在本機伺服器啟動應用程式的演示版。執行 npm start。您可以通過在鍵盤上輸入來停止本地服務器Cmd/Ctrl + C。

  6. 您可以移至package.json並更新主機和連接埠:來變更連接埠或主機 (例如 IP 位址)"start": "HOST=[host] PORT=[port] react-scripts start"。如果您使用視窗:"start": "set HOST=[host] && set PORT=[port] && react-scripts start".

  7. 如果您有註冊的網站網域，則可以在應用程式名稱package.json後指定此網域。例如："homepage": "https://mywebsite.com"。您必須npm install再次執行以更新新的相依性，然後執行npm start。

  8. 若要建置應用程式，請執行npm build。將構建目錄的內容上傳到您的託管服務提供商。

主要搜尋頁面

主搜尋頁面包含所有範例搜尋元件。它包括用於輸出的搜索欄組件，用於顯示來自查詢 API 響應的結果組件以及用於分頁響應的分頁組件。

搜尋元件

搜尋元件提供用於輸入查詢文字的文字方塊。onSearch函數是呼叫中的主函數Search.tsx以進行Amazon Kendra查詢 API 呼叫的掛接。

結果元件

結果元件會顯示來自Query API 的回應。結果會顯示在三個獨立的區域中。

  • 建議的答案 — 這些是Query API 傳回的最常見結果。它包含最多三個建議的答案。在響應中，它們具有結果類型ANSWER。

  • 常見問題解答 — 這些是回應傳回的常見問題結果。常見問題解答會分別新增至索引。在響應中，它們具有類型QUESTION_ANSWER。如需詳細資訊，請參閱問題與解答。

  • 建議文件 — 這些是在回應中Amazon Kendra傳回的其他文件。在來自Query API 的響應中，它們具有類型DOCUMENT。

結果元件會共用一組元件，用於亮顯、標題、連結等功能。共用元件必須存在，結果元件才能運作。

小平面元件

多面元件會列出搜尋結果中可用的多面。每個 Facet 會沿著特定維度 (例如作者) 對回應進行分類。您可以從清單中選擇一個，將搜尋精簡為特定 Facet。

選取 Facet 之後，元件會Query使用屬性篩選器呼叫，該篩選器會將搜尋限制為符合 Facet 的文件。

分頁組件

分頁組件允許您在多個頁面中顯示來自Query API 的搜索結果。它會呼叫帶有PageSize和PageNumber參數的Query API，以取得特定的結果頁面。
