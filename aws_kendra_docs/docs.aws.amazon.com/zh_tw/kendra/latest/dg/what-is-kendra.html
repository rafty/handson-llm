什麼是 Amazon Kendra？

Amazon Kendra是一項高度精確且智慧的搜尋服務，讓您的使用者能夠使用自然語言處理和進階搜尋演算法來搜尋非結構化和結構化的資料。它返回問題的具體答案，為用戶提供接近與人類專家互動的體驗。它具有高度擴充能力，能夠滿足效能需求，與和等其他AWS服務緊密整合 Amazon Lex，Amazon S3並提供企業級安全性。

如需Amazon Kendra和Amazon Kendra智慧排名 API 作業的相關資訊，請參閱 API 參考文件。

Amazon Kendra用戶可以提出以下類型的問題或查詢：

  • Factoid 問題 — 簡單的是誰，什麼，何時或在哪裡問題，例如離西雅圖最近的服務中心在哪裡？ Factoid 問題具有基於事實的答案，可以以單個單詞或短語的形式返回。答案是從常見問題集或索引文件中擷取的。

  • 描述性問題 — 答案可以是句子、段落或整份文件的問題。例如，如何將我的 Echo Plus 連接到我的網絡？ 或者，如何為低收入家庭獲得稅收優惠？ 。

  • 關鍵字和自然語言搜尋：問題不僅包括要搜尋的關鍵字，還包括複雜的對話式問題。這包括預期意義並不總是清楚的問題。例如，關鍵註記地址。由於「地址」通常具有多種含義，因此Amazon Kendra可以在搜索查詢背後推斷用戶的意圖含義以返回相關信息。 Amazon Kendra使用深度學習模型來處理這種查詢。

Amazon Kendra 的優點

Amazon Kendra具有以優優優優優點：

  • 準確性：與使用關鍵字搜尋的傳統搜尋服務不同，其結果是以基本關鍵字比對和排名為基礎，這些搜尋服務會Amazon Kendra嘗試瞭解問題的內容。 Amazon Kendra搜索您的數據並超越傳統搜索，以返回與您的查詢最相關的單詞，片段或文檔。 Amazon Kendra使用機器學習來改善搜尋結果。

  • 簡易性 —Amazon Kendra 提供主控台和 API 來管理您要搜尋的文件。您可以使用簡單的搜尋 API 整合Amazon Kendra至用戶端應用程式，例如網站或行動應用程式。

  • 連線能力 —Amazon Kendra 可連線至協力廠商資料儲存庫或資料來源 (例如 Microsoft) SharePoint。您可以使用資料來源輕鬆索引和搜尋文件。

  • 使用者存取控制 — 為您的搜尋應用程式Amazon Kendra提供高度安全的企業搜尋。搜尋結果會反映組織的安全性模型。搜尋結果可依使用者或他們對文件的群組存取權限企業版。客戶須負責驗證及授權使用者存取權。

Amazon Kendra開發人員版

開Amazon Kendra發人員版以較低的成本提供Amazon Kendra的所有功能。它包括一個免費方案，提供 750 小時的使用時間。開發人員版非常適合探索如何索Amazon Kendra引您的文檔，試用功能，以及開發使用Amazon Kendra.

開發人員版提供以下內容：

  • 最多 5 個索引，每個索引最多 5 個資料來源。

  • 10,000 份文件或 3 GB 擷取的文字。

  • 每天大約 4,000 個查詢或每秒 0.05 個查詢。

  • 在 1 個可用區域 (AZ) 中執行 — 請參閱可用區域 (區域中的資料中AWS心)

您不應將開發人員版用於生產應用程序。開發人員版不提供任何延遲或可用性的保證。

Amazon Kendra僅限企業版

當您想要索引整個Amazon Kendra企業文件庫，或應用程式已準備好可在生產環境中使用時，請使用企業版。

企業版提供下列功能：

  • 最多 5 個索引，每個索引最多 50 個資料來源。

  • 10 萬份文件或 30 GB 擷取的文字。

  • 每天大約 8,000 個查詢或每秒可查詢 0.1 個查詢。

  • 在 3 個可用區域 (AZ) 中執行 — 請參閱可用區域 (區域中的資料中AWS心)

您可以使用 Service Quotas 主控台增加此配額。

Amazon Kendra 的定價

您可以使用開發Amazon Kendra人員版免費開始使用，該版本在前 30 天最多可使用 750 小時。試用期到期後，即使所有佈建的Amazon Kendra索引為空且未執行任何查詢，您仍需支付所有佈建索引的費用。試用期到期後，使用Amazon Kendra資料來源掃描和同步文件需支付額外費用。

如需的完整收費清單與定價，請參閱Amazon Kendra定價
