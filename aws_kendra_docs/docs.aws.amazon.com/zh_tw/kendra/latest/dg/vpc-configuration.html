設Amazon Kendra定為使用 VPC

Amazon Kendra可以連接Amazon Virtual Private Cloud到您的索引存儲在數據源或私有雲中運行的數據庫中的內容。當您建立資料來源連接器或資料庫連接器時，您會為包含資料來源或資料庫的子網路提供安全群組和子網路識別碼。 Amazon Kendra使用此資訊建立 elastic network interface，以便與資料來源或資料庫進行安全通訊。

如果您的資料來源或資料庫未在上執行Amazon VPC，您可以Amazon VPC使用虛擬私人網路 (VPN) 將資料來源或資料庫連線到您的資料來源或資料庫。當你創建你的亞馬遜帳戶時，你會得到一個默認的 VPC。如需設定 VPN 的資訊，請參閱AWS VPN文件。

若要使用 VPC，您必須告知Amazon Kendra資料庫所屬子網路的識別碼，以及Amazon Kendra必須用來存取子網路之任何安全群組的識別碼。例如，如果您使用 MySQL 資料庫的預設連接埠，安全性群組必須允許存Amazon Kendra取執行資料庫之主機上的連接埠 3306。

僅在資料來源或資料庫的 VPC 組態中使用私有子網路。如果 RDS 執行個體位於 VPC 的公有子網路中，則無法直接使用該子網路來同步資料來源或資料庫。請改為建立具有對公有子網路中 NAT 閘道具有傳出存取權的私有子網路。為資料來源或資料庫設定 VPC 時，請指定該私有子網路。對於使用 VPC 設定的資料庫資料來源，子網路必須位於下列其中一個可用區域 ID 中：

  • 美國西部 (奧勒岡) — usw2-az1 us2-az2、usw2-az3

  • 美國東部 (維吉尼亞北部) — use1-az2、use1-az4

  • 歐盟 (愛爾蘭) — euw1-az1、euw1-az3

子網路和安全群組的識別碼在Amazon VPC控制台中設定。要查看標識符，請按如下方式打開Amazon VPC控制台：

檢視子網路識別碼

  1. 登入 AWS Management Console，並在 https://console.aws.amazon.com/vpc/ 開啟 Amazon VPC 主控台。

  2. 從導覽窗格選擇子網路。

  3. 從子網路清單中，選擇包含資料庫伺服器的子網路。

  4. 在「描述」標籤中，記下「子網路 ID」欄位中的識別碼。

若要檢視安全群組識別碼

  1. 登入 AWS Management Console，並在 https://console.aws.amazon.com/vpc/ 開啟 Amazon VPC 主控台。

  2. 從導覽窗格中，選擇 Security Groups (安全群組)。

  3. 從安全群組清單選擇您要為其識別碼的群組。

  4. 在「描述」標籤中，記下「群組 ID」欄位中的識別碼。

如果Amazon Kendra必須在兩個或多個子網路之間路由連線，您可以提供多個子網路。例如，如果包含資料庫伺服器的子網路不在 IP 位址，則Amazon Kendra可以連線到具有可用 IP 位址的子網路，並將連線路由至第一個子網路。如果您列出多個子網路，則子網路必須能互相通訊。每個子網路都必須具有關聯的路由表，必須使用網路位轉換器 (NAT) 裝置，提供傳出網際網路存取。

您也可以提供多個安全群組。安全性群組的綜合效果必須Amazon Kendra允許存取您在連線組態中指定的資料來源或資料庫伺服器。

連線至 VPC 中的資料庫

下列範例會顯示如何連線至 VPC 中執行的 MySQL 資料庫。此範例假設您從預設 VPC 開始，且需要建立 MySQL 資料庫。如果您已經擁有 VPC，請確定其設定如圖所示。如果你有一個 MySQL 數據庫，你可以使用它，而不是創建一個新的。

主題
  • 步驟 1：設定 VPC
  • 步驟 2：設定安全性
  • 步驟 3：建立資料庫
  • 步驟 4：建立資料庫資料來源連接器

步驟 1：設定 VPC

設定您的 VPC，讓您擁有私有子網路和安全群組，Amazon Kendra以存取子網路中執行的 MySQL 資料庫。VPC 組態中提供的子網路必須位於美國西部 (奧勒岡)、美國東部 (維吉尼亞北部) 及歐洲 (愛爾蘭)。

設定 VPC

  1. 登入 AWS Management Console，並在 https://console.aws.amazon.com/vpc/ 開啟 Amazon VPC 主控台。

  2. 在導覽窗格中，選擇「路由表」，然後選擇「建立路由表」。

  3. 在「名稱」標籤欄位中輸入Private subnet route table。在 VPC 欄位中，選擇您的 VPC，然後選擇 [建立]。選擇關閉返回路由表清單。

  4. 在瀏覽窗格中，選擇「NAT 閘道」，然後選擇「建立 NAT 閘道」。

  5. 在 [子網路] 欄位中，選擇作為公用子網路的子網路，並記下子網路 ID。

  6. 如果您沒有彈性 IP 位址，請選擇 [建立新的 EIP]，選擇 [建立 NAT 閘道]，然後選擇 [關閉]。

  7. 從導覽窗格選擇路由表。

  8. 從路由表清單中，選擇您在步驟 3 中建立的私人子網路路由表格。從「動作」 中選擇 「編輯路由」。

  9. 選擇 Add route (新增路由)。新增目的地 0.0.0.0.0.0/0，允許所有傳出流量傳到網際網路。針對「目標」，選擇「NAT 閘道」，然後選擇您在步驟 4 中建立的閘道。選擇 Save routes (儲存路由)，然後選擇 Close (關閉)。

  10. 從動作中，選擇編輯子網路關聯。

  11. 選擇您要設為私人的子網路。請勿選擇包含您先前記下 NAT 閘道的子網路。

步驟 2：設定安全性

接下來，設定資料庫的安全群組。

建立安全群組

  1. 登入 AWS Management Console，並在 https://console.aws.amazon.com/vpc/ 開啟 Amazon VPC 主控台。

  2. 從虛擬私人 VPC 的說明中，記下 IPv4 CIDR。

  3. 從功能窗格中，選擇 [安全性群組]，然後選擇 [建立安全性群組]。

  4. 在安全性群組名稱中輸入DataSourceInboundSecurityGroup。提供描述，然後從清單選擇您的 VPC。選擇建立，然後選擇關閉。

  5. 選擇 Inbound (傳入) 標籤。

  6. 選擇 [編輯規則]，然後選擇 [新增規則]

  7. 對於資料庫，請輸入連接埠範圍的連接埠號碼。例如，對於 MySQL 來說3306，它是，對於 HTTPS 來說，它是443。對於來源，請輸入 VPC 的無類別域間路由 (CIDR)。選擇儲存規則，然後選擇「關閉」。

安全性群組允許 VPC 中的任何人連線到資料庫，並允許對網際網路的輸出連線。

步驟 3：建立資料庫

創建一個數據庫來保存您的文檔，或者您可以使用現有的數據庫。如需Amazon Kendra支援的資料庫清單，請參閱使用資料庫資料來源。

有關如何創建 MySQL 數據庫的說明，請參閱使用 MySQL 數據庫數據源（控制台）入門Amazon RDS。

步驟 4：建立資料庫資料來源連接器

設定 VPC 並建立資料庫之後，您可以為資料庫建立資料來源連接器。請參閱使用資料庫資料來源。

請確定您已設定 VPC、您在 VPC 中建立的私有子網路，以及您在 VPC 中為資料庫建立的安全性群組。

有關如何為 MySQL 數據庫創建數據源的說明，請參閱使用Amazon RDS.
