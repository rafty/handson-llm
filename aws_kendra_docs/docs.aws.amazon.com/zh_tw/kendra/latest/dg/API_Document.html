內容

AccessControlConfigurationId

您希望套用到文件的存取控制組態的識別碼。

類型：String

長度限制：長度下限為 1。長度限限限限限限限為限

模式：[a-zA-Z0-9-]+

必要：否

AccessControlList

有關主參與者 (使用者和/或群組) 及其應該存取哪些文件的資訊。這對於使用者前後關聯篩選非常有用，其中搜尋結果會根據使用者或他們對文件的群組存取權進行篩選。

類型：Principal 物件陣列

必要：否

Attributes

要套用至文件的自訂屬性。使用自訂屬性提供搜尋的其他資訊、提供用於精簡搜尋的 Facet，以及在查詢回應中提供其他資訊。

例如，'' 和DataSourceIdDataSourceSyncJobId '是自訂屬性，可提供有關在資料來源上執行之文件同步處理的資訊。請注意，'DataSourceSyncJobId' 可能是選用的自訂屬性，因為 Amazon Kendra 將使用執行中同步任務的 ID。

類型：DocumentAttribute 物件陣列

必要：否

Blob

文件內容。

傳遞至Blob參數的文件必須採用 basebasebase編碼。如果您使用AWS開發套件呼叫 Amazon Kendra API，您的程式碼可能不需要對文件檔案位元組進行編碼。如果您要使用 REST 直接呼叫 Amazon Kendra 端點，則必須在傳送之前對內容進行對內容進行 Base64 編碼。

類型：Base64 編碼的二進位資料物件

必要：否

ContentType

欄位中文件的檔案類Blob型。

類型：String

有效值： PDF | HTML | MS_WORD | PLAIN_TEXT | PPT | RTF | XML | XSLT | MS_EXCEL | CSV | JSON | MD

必要：否

HierarchicalAccessControlList

主參與者清單，用於定義文件使用者應具有存取權限的階層。

類型：HierarchicalPrincipal 物件陣列

陣列成員：項目數下限為 1。項目數上限為限為限為限為限

必要：否

Id

索引中文件的識別碼。

請注意，每個索引的每個文件 ID 必須是唯一的。您無法建立資料來源以使用其唯一 ID 為文件建立索引，然後使用BatchPutDocument API 來索引相同的文件，反之亦然。您可以刪除資料來源，然後使用BatchPutDocument API 索引相同的文件，反之亦然。

類型：String

長度限制：長度下限為 1。長度上限為 2048。

必要：是

S3Path

在 Amazon S3 儲存貯體中尋找特定檔案所需的資訊。

類型：S3Path 物件

必要：否

Title

文件標題。

類型：字串

必要：否
