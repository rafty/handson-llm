露天

Alfresco OnPrem 是一項內容管理服務，可幫助客戶存儲和管理其內容。您可以使Amazon Kendra用索引您的露天文件庫，維基和博客。

您可以使用Amazon Kendra控制台和 AlfrescoConfigurationAPI 連接Amazon Kendra到 Alfresco 數據源。

注意

露天資料來源連接器目前處於預覽模式。目前支援基本驗證。如果您想在生產環境中使用 Alfresco 連接器，請聯絡 Sup port 部門。目前不支援露天 PaaS。

有關Amazon Kendra Alfresco 資料來源連接器的疑難排解，請參閱對資料來源進行。

支援的功能

Amazon KendraAlfresco 資料來源連接器支援下列功能：

  • 欄位映射

  • 使用者內容篩選

  • 包含/排除過濾器

  • Virtual Private Cloud (VPC) (虛擬私有雲端 (VPC))

先決條件

在您可以用Amazon Kendra來為 Alfresco 資料來源建立索引之前，請先在 Alfresco 和AWS帳戶中進行這些變更。

在露天，請確定您具有：

  • 記下您的 Alfresco 身份驗證憑據，其中包括管理員用戶名和密碼。

  • 已將使用者新增至ALFRESCO_ADMINISTRATORS群組。

  • 複製了您的露天網站網址和露天網站 ID。

  • 檢查每個文檔在 Alfresco 和您打算用於同一索引的其他數據源中是唯一的。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，確定您具有：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引 ID。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

  • 將您的 Alfresco 身份驗證憑據存儲在AWSSecrets Manager秘密中，並且如果使用 API，則記錄密鑰的 ARN。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

如果您沒有現有的IAM角色或密碼，則可以在將 Alfresco 資料來源連線至時使用主控台建立新IAM角色和Secrets Manager密碼Amazon Kendra。如果您使用 API，則必須提供現有IAM角色和Secrets Manager密碼的 ARN 以及索引 ID。

連接說明

若要連線Amazon Kendra到 Alfresco 資料來源，您必須提供 Alfresco 資料來源的必要詳細資料，Amazon Kendra以便能夠存取您的資料。如果您尚未配置露天查Amazon Kendra看先決條件。

Console

連接Amazon Kendra到露天

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 [Alfresco 連接器]，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資訊頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，在新增標籤中 — 標籤來搜尋和篩選資源，或追蹤AWS成本。

    5. 選擇 下一步。

  6. 在定義存取和安全性頁面上，輸入以下資訊：

    1. 露天網站網址 — 輸入您的露天網站 URL。例如：https://hostname:8080。

    2. SSL 憑證位置 — 輸入Amazon S3路徑。

    3. 網站 ID — 您的露天網站 ID。

    4. AWSSecrets Manager秘密 — 選擇現有密碼或建立新Secrets Manager密碼來儲存您的 Alfresco 驗證憑證。如果您選擇建立新密碼，則會開啟一個AWSSecrets Manager密碼視窗。

      1. 在「建立AWSSecrets Manager密碼視窗」中輸入下列資訊：

        1. 秘密名稱 — 您秘密的名稱。前綴 'AmazonKendra-Alfreco-' 會自動添加到您的密碼名稱中。

        2. 對於使用者名稱和密碼 — 輸入您的 Alfresco 使用者名稱和密碼。

      2. 選擇 儲存 。

    5. V@@ irtual Private Cloud (VPC) (VPC) — 您可以選擇使用 VPC。如果是這樣，您必須新增子網路和 VPC 安全性群組。

    6. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    7. 選擇 下一步。

  7. 在配置同步設定頁面上，輸入以下資訊：

    1. 選取要編目的內容 — 您要編目的 Alfresco 實體或內容類型。

    2. 正則表達式模式-包含或排除某些文件的正則表達式模式。最多可以新增 100 個圖樣。

    3. 在同步執行排程中，對於頻率 — 選擇與資料來源同步的頻率。Amazon Kendra

    4. 選擇 下一步。

  8. 在設定欄位映射頁面上，輸入以下資訊：

    1. 對於「文件庫」、「Wiki」和「部落格」— 從Amazon Kendra產生的預設資料來源欄位中選取您要對應至索引的資料來源欄位。

    2. 新增欄位 — 新增自訂資料來源欄位，以建立要對映的索引欄位名稱和欄位資料類型。

    3. 選擇 下一步。

  9. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以從此頁面來編輯資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

API

連接Amazon Kendra到露天

您必須使用 AlfrescoConfigurationAPI 指定下列項目：

  • 秘密亞馬遜資源名稱 (ARN) — 提供包含您戶外帳戶身份驗證登入資料的機Secrets Manager密的亞馬遜資源名稱 (ARN)。密碼會儲存在具有下列索引鍵的 JSON 結構中：

    {
    "username": "user name",
    "password": "password"
}
    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

  • IAM 角色 — 指定RoleArn何時呼叫CreateDataSource以提供具有存取Secrets Manager密碼權限的IAM角色，以及呼叫 Alfresco 連接器和所需的公用 API 的時間Amazon Kendra。如需詳細資訊，請參閱 Alfresco 資料來源的 IAM 角色。

您也可以新增下列選用功能：

  • V@@ irtual Private Cloud (VPC) — 指定VpcConfiguration為資料來源組態的一部分。請參閱設Amazon Kendra定以使用 VPC。

  • 包含和排除篩選器 — 指定要包含還是排除某些檔案。

    注意

    大多數資料來源使用規則運算式模式，這些模式是稱為篩選器的包含或排除模式。如果您指定包含篩選器，則只有符合包含項目篩選器的內容會編製索引。不符合包含篩選器的任何文件都不會建立索引。如果您指定包含和排除篩選器，符合排除篩選器的文件將不會建立索引，即使它們符合包含篩選器也是如此。

  • 欄位對映 — 選擇將 Alfresco 資料來源欄位對映至Amazon Kendra索引欄位。如需詳細資訊，請參閱映射資料來源欄位。

  • 使用者前後關聯篩選 — 依預設Amazon Kendra搜尋資料來源的存取控制清單 (ACL)。ACL 資訊可用來根據使用者或使用者對文件的群組存取來篩選搜尋結果。如需詳細資訊，請參閱 Alfresco 資料來源的使用者內容篩選。
