Slack

Slack 是一款企業通信應用程序，可讓用戶通過各種公共和私有渠道發送消息和附件。您可以用Amazon Kendra來為 Slack 公開和私人通道、機器人和封存訊息、檔案和附件、直接和群組訊息建立索引。您也可以選擇要篩選的特定內容。

您可以使用Amazon Kendra主控台和 SlackConfigurationAPI 連線Amazon Kendra到 Slack 資料來源。

如需Amazon Kendra Slack 資料來源連接器的疑難排解，請參閱對資料來源進行。

支援的功能

Amazon KendraSlack 資料來源連接器支援下列功能：

  • 變更日誌

  • 欄位映射

  • 使用者內容篩選

  • 包含/排除過濾器

  • Virtual Private Cloud (VPC) (虛擬私有雲端 (VPC))

先決條件

在您可以用Amazon Kendra來為 Slack 資料來源建立索引之前，請先在 Slack 和AWS帳戶中進行這些變更。

在 Slack 中，請確定您具有：

  • 創建了一個鬆弛機器人用戶 OAuth 令牌或鬆弛用戶 OAuth 令牌。您可以選擇任一權杖來連線Amazon Kendra至 Slack 資料來源。有關更多信息，請參閱有關訪問令牌的 Slack 文檔。

    注意

    如果您將機器人令牌用作 Slack 憑據的一部分，則無法為直接消息和群組消息編制索引，並且必須將機器人令牌添加到要索引的渠道中。

  • 從 Slack 工作區主頁面網址記下您的 Slack 工作區團隊 ID。舉個例子, https://app.slack.com/client/T0123456789/... 其中 T0123456789 是群組識別碼。

  • 新增下列 [Oauth 範圍/讀取] 權限：

    • 頻道:歷史

    • 頻道:讀取

    • 群組:歷程

    • 群組:讀取

    • im: 歷程

    • MPIM: 歷程

    • MPM: 讀取

    • team:read

    • 使用者. 設定檔:讀取

    • 使用者:讀取

    • 表情符號:讀取

    • 檔案:讀取

    • 使用者群組:讀取

  • 已勾選的每個文件在 Slack 和您打算用於相同索引的其他資料來源中都是唯一的。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，請確定您具有：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引識別碼。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

  • 將您的 Slack 驗證憑證儲存在AWSSecrets Manager密碼中，如果使用 API，則會記錄密碼的 ARN。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

如果您沒有現有的IAM角色或密碼，則可以在將 Slack 資料來源連線至時使用主控台建立新IAM角色和Secrets Manager密碼Amazon Kendra。如果您使用 API，則必須提供現有IAM角色和Secrets Manager密碼的 ARN 以及索引 ID。

連接說明

若要連線Amazon Kendra至 Slack 資料來源，您必須提供 Slack 資料來源的必要詳細資訊，Amazon Kendra以便存取您的資料。如果您尚未配置 Slack 以查Amazon Kendra看先決條件。

Console

若要連線Amazon Kendra至鬆弛

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯您的用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 [Slack 連接器]，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資料頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，用於新增標籤 — 標籤來搜尋和篩選資源，或追蹤AWS成本。

    5. 選擇 下一步。

  6. 在定義存取和安全性頁面上，輸入以下資訊：

    1. 鬆弛工作區團隊 ID — Slack 工作區的團隊 ID。

    2. AWSSecrets Manager秘密：選擇現有密碼或建立新Secrets Manager密碼來儲存您的 Slack 驗證憑證。如果您選擇建立新密碼，則會開啟一個AWSSecrets Manager密碼視窗。

      1. 在「建立AWSSecrets Manager密碼視窗」中輸入下列資訊：

        1. 秘密名稱 — 您秘密的名稱。前綴 'AmazonKendra-Slack-' 會自動添加到您的密碼名稱中。

        2. 針對 Slack 權杖：輸入您在 Slack 帳戶中建立的驗證憑證值。

      2. 選擇 儲存 。

    3. V@@ irtual Private Cloud (VPC) (VPC) — 您可以選擇使用 VPC。如果是這樣，您必須新增子網路和 VPC 安全性群組。

    4. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    5. 選擇 下一步。

  7. 在配置同步設定頁面上，輸入以下資訊：

    1. 選取要編目的內容類型 — 您要編目的 Slack 實體或內容類型。

    2. 變更記錄 — 選取此選項可更新索引，而不是同步所有檔案。

    3. 正則表達式模式-包含或排除某些文件的正則表達式模式。您最多可新增 100 個圖樣。

    4. 選取編目開始日期 —Amazon Kendra 開始檢索資料的起始日期。

    5. 在同步執行排程中，對於頻率 — 與資料來源同步的頻率。Amazon Kendra

    6. 選擇 下一步。

  8. 在設定欄位映射頁面上，輸入以下資訊：

    1. 對於 Slack 欄位對映 — 從Amazon Kendra產生的預設資料來源欄位中選取您要對映至索引的資料來源欄位。

    2. 新增欄位 — 新增自訂資料來源欄位，以建立要對映的索引欄位名稱和欄位資料類型。

    3. 選擇 下一步。

  9. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以從這個頁面來編輯資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

API

若要連線Amazon Kendra至鬆弛

您必須使用 SlackConfigurationAPI 指定下列項目：

  • 鬆弛工作區團隊 ID — 您從 Slack 主頁面 URL 複製的 Slack 團隊 ID。

  • 要索引的實體清單 — 是否Amazon Kendra應為公開和私人通道、群組和直接訊息建立索引。

  • 編目日期 — 開始從 Slack 工作區團隊檢索資料的日期。日期必須遵循以下格式： yyyy-mm-dd。

  • 秘密亞馬遜資源名稱 (ARN) — 提供包含 Slack 帳戶身份驗證登入資料的Secrets Manager密碼的亞馬遜資源名稱 (ARN)。密碼會儲存在具有下列索引鍵的 JSON 結構中：

    {
    "slackToken": "token"
}
    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

  • IAM 角色 — 指定RoleArn何時呼叫CreateDataSource以提供具有存取Secrets Manager密碼的權限的IAM角色，以及呼叫 Slack 連接器和所需的公用 APIAmazon Kendra。如需詳細資訊，請參閱 Slack 資料來源的 IAM 角色。

您也可以新增下列選用功能：

  • V@@ irtual Private Cloud (VPC) — 指定VpcConfiguration為資料來源組態的一部分。請參閱設Amazon Kendra定以使用 VPC。

  • 變更記錄 — 是否Amazon Kendra應使用 Slack 資料來源變更記錄機制來決定是否必須在索引中新增、更新或刪除文件。

    注意

    如果您不想Amazon Kendra掃描所有文檔，請使用更改日誌。如果您的變更記錄檔很大，掃描 Slack 資料來源中的文件所需的時間可能會比處理變更記錄更Amazon Kendra少。如果您是第一次將 Slack 資料來源與索引同步，則會掃描所有文件。

  • 包含和排除篩選器：指定是否要包含或排除特定的公開和私人通道、群組和私人訊息，以及機器人和封存訊息。如果您使用機器人令牌作為 Slack 身份驗證憑據的一部分，則必須將機器人令牌添加到要索引的通道中。您無法使用機器人權杖建立直接訊息和群組訊息的索引。

    注意

    大多數資料來源使用規則運算式模式，這些模式是稱為篩選器的包含或排除模式。如果您指定包含篩選器，則只有符合包含項目篩選器的內容會編製索引。不符合包含篩選器的任何文件都不會建立索引。如果您指定包含和排除篩選器，符合排除篩選器的文件將不會建立索引，即使它們符合包含篩選器也是如此。

  • 欄位對映 — 選擇將 Slack 資料來源欄位對映至Amazon Kendra索引欄位。如需詳細資訊，請參閱映射資料來源欄位。

  • 使用者前後關聯篩選 — 依預設Amazon Kendra搜尋資料來源的存取控制清單 (ACL)。ACL 資訊可用來根據使用者或使用者對文件的群組存取權來篩選搜尋結果。如需詳細資訊，請參閱 Slack 資料來源的使用者內容篩選。
