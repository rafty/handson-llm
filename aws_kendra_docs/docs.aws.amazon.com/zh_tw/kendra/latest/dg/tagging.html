Tags (標籤)

透過指派標籤或標籤來管理索引、資料來源和常見問題集。您可以使用標籤以各種方式分類Amazon Kendra資源。例如，依據目的、擁有者或應用程式，或任意組合。每個標籤皆包含由您定義的索引鍵和值。

標籤可協助您：

  • 識別和組織您的 AWS 資源。許多AWS服務支援標記，因此您可以對不同服務中的資源指派相同的標籤，指派相同的標籤，以指派資源是相同的標籤。例如，您可以使用相同的標籤來標記索引和使用索引的Amazon Lex機器人。

  • 配置成本。您可以在AWS Billing and Cost Management管控面板上啟用標籤。 AWS使用標籤來分類您的成本，並交付每月成本分配報告給您。如需詳細資訊，請參閱關於 BAWS illing and Cost Management 中的成本分配和標記。

  • 控制對 資源的存取。您可以在AWS Identity and Access Management (IAM) 原則中使用控制Amazon Kendra資源存取的標籤。您可以將這些原則附加至IAM角色或使用者，以啟動以標籤為基礎的存取控制。如需詳細資訊，請參閱以 Amazon Kendra 標籤為基礎的授權。

您可以使用AWS Management Console、AWS Command Line Interface (AWS CLI) 或Amazon Kendra API 建立和管理標籤。

標記 資源

如果使用Amazon Kendra主控台，則可以在建立資源時標記資源，或稍後新增資源。您也可以使用主控台來更新或移除標籤。

如果您使用的是AWS Command Line Interface (AWS CLI) 或Amazon Kendra API，請使用下列操作來管理資源的標籤：

  • CreateDataSource在建立資料來源時套用標籤。

  • CreateFaq建立常見問題集時套用標籤。

  • CreateIndex建立索引時套用標籤。

  • ListTagsForResource— 檢視與資源相關聯的標籤。

  • TagResource新增和修改資源的標籤。

  • UntagResource從資源移除標籤。

標籤限制

以下為適用於Amazon Kendra資源的標籤：

  • 標籤數上限-50

  • 金鑰長度上限-128 個字元

  • 長度上限-256 個字元

  • 金鑰與值的有效字元 — _.:/= +-及 @

  • 金鑰和值會區分大小寫

  • 請不要使用 aws: 做為金鑰的字首；它已保留供 AWS 使用
