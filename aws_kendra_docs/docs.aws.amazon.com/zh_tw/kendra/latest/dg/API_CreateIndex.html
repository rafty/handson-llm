CreateIndex

建立 Amazon Kendra 索引。索引創建是一個異步 API。若要判斷索引建立是否已完成，請檢查呼叫傳回的Status欄位DescribeIndex。該Status字段設置為ACTIVE當索引準備使用時。

索引處於作用中心可以使用BatchPutDocument API 或使用其中一個支援的資料來源。

如需使用 Python SDK 建立索引和資料來源的範例，請參閱開始使用 Python SDK 一文。如需使用 Java SDK 建立索引和資料來源的範例，請參閱 Java SDK 入門。

請求參數

如需有關所有動作共用之參數的資訊，請參閱一般參數。

請求接受採用 JSON 格式的下列資料。

ClientToken

您提供用來識別建立索引之要求的權杖。對具有相同客戶端令牌的CreateIndex API 進行多次調用將僅創建一個索引。

類型：字串

長度限制：長度下限為 1。長度上限為 100。

必要：否

Description

索引的描述。

類型：字串

長度限制：長度下限為 0。長度上限為 1000。

模式：^\P{C}*$

必要：否

Edition

Amazon Kendra 版用於索引。選擇用DEVELOPER_EDITION於開發、測試或概念驗證的索引。用ENTERPRISE_EDITION於生產。一旦您設定索引的版本，就無法變更。

Edition 為選用參數。如果您不提供值，則預設為ENTERPRISE_EDITION.

如需企業版和開發人員版配額限制的詳細資訊，請參閱配額。

類型：字串

有效值： DEVELOPER_EDITION | ENTERPRISE_EDITION

必要：否

Name

索引的名稱。

類型：字串

長度限制：長度下限為 1。長度上限為 1000。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

必要：是

RoleArn

具有存取 Amazon Resource Name (ARN)。 CloudWatch 如需詳細資訊，請參閱 Amazon Kendra 的 Amazon Kendra。

類型：字串

長度限制：長度下限為 0。長度上限為 1284。

模式：arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

必要：是

ServerSideEncryptionConfiguration

AWS KMS客戶受管金鑰的識別碼，用於加密由 Amazon Kendra 編製索引的資料。Amazon Kendra 不支援不對稱 CMK。

類型：ServerSideEncryptionConfiguration 物件

必要：否

Tags

識別或分類索引鍵值組清單。您也可以使用標籤來協助控制對索引的存取。標籤索引鍵和值可以包含任意組成 Unicode 字母、空格和下列符號：_.:/= +-@。

類型：Tag 物件陣列

陣列成員：項目數下限為 0。項目數上限為 200。

必要：否

UserContextPolicy

使用者內容政策。

ATTRIBUTE_FILTER

所有已編製索引的內容可供所有使用者搜尋並顯示。如果您要在使用者內容上篩選搜尋結果，可以使用 _user_id 和 _group_ids 的屬性篩選條件，或者您可以在 UserContext 中提供使用者和群組資訊。

USER_TOKEN

啟用權杖型使用者存取控制，以在使用者內容上篩選搜尋結果。所有無存取控制的文件和使用者可存取的所有文件都可搜尋和顯示。

類型：字串

有效值： ATTRIBUTE_FILTER | USER_TOKEN

必要：否

UserGroupResolutionConfiguration

從AWS IAM Identity Center (successor to AWS Single Sign-On)身分識別來源取得使用者和群組。若要對此進行設定，請參閱UserGroupResolutionConfiguration。

類型：UserGroupResolutionConfiguration 物件

必要：否

UserTokenConfigurations

使用者權杖組態。

類型：UserTokenConfiguration 物件陣列

陣列成員：最多 1 個項目。

必要：否

回應語法

{
   "Id": "string"
}

回應元素

如果動作成功，則服務傳回 HTTP 200 回應。

服務會傳回下列 JSON 格式的資料。

Id

索引的識別碼。當您查詢索引、設定資料來源或為文件建立索引時，請使用此識別碼。

類型：字串

長度限制：固定長度為 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

錯誤

如需所有動作常見的錯誤相關資訊，請參閱常見錯誤。

AccessDeniedException

您沒有足夠的存取權限。請確定您擁有必要的權限原則和使用者帳戶，然後再試一次。

HTTP 狀態碼：400

ConflictException

與請求發生衝突。請修正任何與資源不一致的問題，然後再試一次。

HTTP 狀態碼：400

InternalServerException

用於 Amazon Kendra 服務的內部伺服器發生問題。請稍候幾分鐘，然後再試一次，或聯絡 Sup port 尋求協助。

HTTP 狀態碼：500

ResourceAlreadyExistException

您要使用的資源已存在。請檢查您是否提供了正確的資源，然後再試一次。

HTTP 狀態碼：400

ServiceQuotaExceededException

您已超過 Amazon Kendra 服務的設定限制。如需詳細資訊，請參閱配額，或聯絡 Sup port 部門以詢問有關增加限制的問題。

HTTP 狀態碼：400

ThrottlingException

由於請求調節，因此請求遭到拒絕。請減少要求數目，然後再試。

HTTP 狀態碼：400

ValidationException

輸入不符合 Amazon Kendra Kendra Kendra 的限制。請提供正確的輸入，然後再試。

HTTP 狀態碼：400
