CreateDataSource

建立您要與 Amazon Kendra 索引搭配使用的資料來源連接器。

您可以為資料來源指定名稱、資料來源連接器類型和描述。您也可以指定資料來源連接器的組態資訊。

CreateDataSource是一個同步操作。如果成功建立資料來源，作業會傳回 200。否則，引發異常。

如需使用 Python SDK 建立索引和資料來源的範例，請參閱開始使用 Python SDK 一文。如需使用 Java SDK 建立索引和資料來源的範例，請參閱 Java SDK 入門。

請求參數

如需有關所有動作共用之參數的資訊，請參閱一般參數。

請求接受採用 JSON 格式的下列資料。

ClientToken

您提供的字元，用來識別建立資料來源連接器之請求。對具有相同用戶端 Token 的CreateDataSource API 進行多次呼叫，只會建立一個資料來源連接器。

類型：字串

長度限制：長度下限為 1。長度上限為 100。

必要：否

Configuration

連線至資料來源儲存庫的組態資訊。

當 Type 參數設定為 CUSTOM 時，您無法指定 Configuration 參數。如果這樣做，您會收到ValidationException例外狀況。

所有其他資料來源都需要此 Configuration 參數。

類型：DataSourceConfiguration 物件

必要：否

CustomDocumentEnrichmentConfiguration

在文件擷取過程中，變更文件中繼資料和內容的組態資訊。

如需有關如何建立、修改和刪除文件中繼資料，或在將文件導入 Amazon Kendra 時進行其他內容變更的詳細資訊，請參閱擷取程序期間自訂文件中繼資料。

類型：CustomDocumentEnrichmentConfiguration 物件

必要：否

Description

資料來源連接器的描述。

類型：字串

長度限制：長度下限為 0。長度上限為 1000。

模式：^\P{C}*$

必要：否

IndexId

您要與資料來源連接器搭配使用之索引的識別碼。

類型：字串

長長限為 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必要：是

LanguageCode

一種語言的代碼。這可讓您在建立資料來源連接器時支援所有文件的語言。預設支援英文。如需支援語言 (包括其代碼) 的詳細資訊，請參閱以英文以外的語言新增文件。

類型：字串

長長限為 2。長限為 10。

模式：[a-zA-Z-]*

必要：否

Name

資料來源連接器的名稱。

類型：字串

長度限制：長度下限為 1。長度上限為 1000。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

必要：是

RoleArn

具有存取取取取得資料來源角色 Amazon Resource Name (ARN)。如需詳細資訊，請參閱 Amazon Kendra 的 IAM 存取角色。 。

當 Type 參數設定為 CUSTOM 時，您無法指定 RoleArn 參數。如果這樣做，您會收到ValidationException例外狀況。

所有其他資料來源都需要此 RoleArn 參數。

類型：字串

長度限制：長度下限為 0。長長長為 1284。

模式：arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

必要：否

Schedule

設定 Amazon Kendra 檢查資料來源儲存庫中的文件並更新頻率。如果您未設定排程，Amazon Kendra 將不會定期更新。您可以呼叫StartDataSourceSyncJob API 來更新索引。

指定cron-格式明細表字串或空字串，以指出索引會隨需更新。

當 Type 參數設定為 CUSTOM 時，您無法指定 Schedule 參數。如果這樣做，您會收到ValidationException例外狀況。

類型：字串

必要：否

Tags

識別或分類資源連接器的鍵值組清單。您也可以使用標籤來協助控制對資料來源連接器的存取。標籤索引鍵和值可包含任意組成 Unicode 字母、數字、空格和下列符號：_.:/= +-@。

類型：Tag 物件陣列

陣列成員：項目數下限為 0。項目數為 200。

必要：否

Type

資料來源儲存庫的類型。例如：SHAREPOINT。

類型：字串

有效值： S3 | SHAREPOINT | DATABASE | SALESFORCE | ONEDRIVE | SERVICENOW | CUSTOM | CONFLUENCE | GOOGLEDRIVE | WEBCRAWLER | WORKDOCS | FSX | SLACK | BOX | QUIP | JIRA | GITHUB | ALFRESCO | TEMPLATE

必要：是

VpcConfiguration

連接到資料來源的 Amazon Virtual Private Cloud 的組態資訊。如需詳細資訊，請參閱設定 VPC。

類型：DataSourceVpcConfiguration 物件

必要：否

回應語法

{
   "Id": "string"
}

回應元素

如果動作成功，則服務傳回 HTTP 200 回應。

服務會傳回下列 JSON 格式的資料。

Id

資料來源連接器的識別碼。

類型：字串

長度限制：長度下限為 1。長度上限為 100。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

錯誤

如需所有動作常見的錯誤相關資訊，請參閱常見錯誤。

AccessDeniedException

您沒有足夠的存取。請確定您擁有必要的權限原則和使用者帳戶，然後再試一次。

HTTP 狀態碼：400

ConflictException

與請求發生衝突。請修正任何與資源不一致的問題，然後再試一次。

HTTP 狀態碼：400

InternalServerException

用於 Amazon Kendra 服務的內部伺服器發生問題。請稍候幾分鐘，然後再試一次，或聯絡 Sup port 尋求協助。

HTTP 狀態碼：500

ResourceAlreadyExistException

您要使用的資源已存在。請檢查您是否提供了正確的資源，然後再試一次。

HTTP 狀態碼：400

ResourceNotFoundException

您要使用的資源不存在。請檢查您是否提供了正確的資源，然後再試一次。

HTTP 狀態碼：400

ServiceQuotaExceededException

您已超過 Amazon Kendra 服務的設定限制。如需詳細資訊，請參閱配額，或聯絡 Sup port 部門以詢問有關增加限制的問題。

HTTP 狀態碼：400

ThrottlingException

由於請求調節，因此請求遭到拒絕。請減少要求數目，然後再試一次。

HTTP 狀態碼：400

ValidationException

輸入不符合 Amazon Kendra 服務設定的限為止。請提供正確的輸入，然後再試一次。

HTTP 狀態碼：400
