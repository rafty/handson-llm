GitHub

GitHub 是一種基於 Web 的託管服務，用於軟件開發，提供代碼存儲和管理服務以及版本控制。您可以用Amazon Kendra來為 GitHub 企業雲端 (SaaS) 和 GitHub 企業伺服器 (在 Prem) 儲存庫檔案建立索引、發出和提取要求、發出和提取要求註解，以及發出和提取要求註解附件。您也可以選擇包含或排除特定檔案。

您可以使用Amazon Kendra主控台和 GitHubConfigurationAPI 連線Amazon Kendra到 GitHub 資料來源。

如需疑難排解資Amazon Kendra GitHub 料來源連接器，請參閱對資料來源進行。

支援的功能

Amazon Kendra GitHub 資料來源連接器支援下列功能：

  • 變更日誌

  • 欄位映射

  • 使用者內容篩選

  • 包含/排除過濾器

先決條件

在您可以用來Amazon Kendra為資 GitHub 料來源建立索引之前，請先在 GitHub 和AWS帳戶中進行這些變更。

在中 GitHub，確定您具有下列項目：

  • 建立具有 GitHub 組 GitHub 織管理權限的使用者。

  • 為身份驗證憑據創建了個人訪問令牌。請參GitHub 閱有關建立個人存取權杖的文件。

  • 建議：為驗證憑證建立 OAuth 權杖。使用 OAuth 令牌可獲得更好的 API 節流限制和連接器性能。請參GitHub 閱有關 OAuth 授權的文件。

  • 選用性：安裝 SSL 憑證。

  • 記下您使用之 GitHub服務類型的 GitHub 主機 URL。舉例來說， GitHub 雲端的主機網址可以是 https://api.github.com，而 GitHub 伺服器的主on-prem-host-url機網址可以是 https:

  • 從您的設定中記下您的資料庫的 GitHub GitHub 組織名稱。

  • 已新增下列權限：

    適用於 GitHub 企業雲端 (SaaS)

    • 回收:狀態

    • 公共回購

    • 回收:邀請

    • 讀取:組織

    • 用戶:電子郵件

    • 讀取:使用者

    適用於 GitHub 企業伺服器 (在內部部署)

    • 回收:狀態

    • 公共回購

    • 回收:邀請

    • 讀取:組織

    • 用戶:電子郵件

    • 讀取:使用者

    • 管理網站

  • 已勾選的每個文件在您打算用於相同索引的其他資料來源中 GitHub 和其他資料來源之間都是唯一的。您要用於索引的每個資料來源不得包含跨資料來源的相同文件。文件 ID 對索引來說是全域的，而且每個索引必須是唯一的。

在您的AWS帳戶中，確定您具有下列項目：

  • 建立Amazon Kendra索引，如果使用 API，則會記下索引 ID。

  • 為您的資料來源建立IAM角色，如果使用 API，則會記錄IAM角色的 ARN。

  • 將您的 GitHub 身份驗證憑據存儲在AWSSecrets Manager密鑰中，如果使用 API，則記錄密鑰的 ARN。

    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

如果您沒有現有的IAM角色或密碼，則可以在將 GitHub 資料來源連線到時使用主控台建立新IAM角色和Secrets Manager密碼Amazon Kendra。如果您使用 API，則必須提供現有IAM角色和Secrets Manager密碼的 ARN，以及索引 ID。

連接指示

若要連線Amazon Kendra到 GitHub 資料來源，您必須提供資料來源的必要詳細 GitHub 資訊，Amazon Kendra以便能夠存取您的資料。如果您尚未設定「Amazon Kendra請 GitHub 參閱先決條件」。

Console

若要連線Amazon Kendra到 GitHub

  1. 登入 AWS 管理主控台，並開啟 Amazon Kendra 主控台。

  2. 在左側導覽窗格中，選擇 [索引]，然後從索引清單中選擇要使用的索引。

    注意

    您可以選擇在索引設置下配置或編輯您的用戶訪問控制設置。

  3. 在 [開始使用] 頁面上，選擇 [新增資料來源]。

  4. 在 [新增資料來源] 頁面上，選擇 [GitHub 連接器]，然後選擇 [新增資料來源]。

  5. 在指定資料來源詳細資訊頁面上，輸入以下資訊：

    1. 在名稱和說明中，對於資料來源名稱 — 輸入資料來源的名稱。您可以包含連字號，但不能包含空格。

    2. (選擇性) 說明 — 輸入資料來源的選用說明。

    3. 在語言中，對於預設語言-用於篩選索引文件的語言。除非另有指定，否則語言預設為英文。中繼資料中指定的語言會覆寫所選語言

    4. 在標籤中，在新增標籤中 — 標籤來搜尋和篩選資源，或追蹤AWS成本。

    5. 選擇 下一步。

  6. 在定義存取和安全性頁面上，輸入以下資訊：

    1. GitHub來源 — 在GitHub 企業雲和GitHub企業服務器之間進行選擇。

    2. GitHub 主機 URL — 輸入您的 GitHub 主機名稱。

    3. GitHub 組織名稱 — 輸入您的 GitHub組織名稱。您可以在您的 GitHub 帳戶中找到您的組織資訊。

    4. AWSSecrets Manager祕密 — 選擇現有密碼或建立新Secrets Manager密碼來儲存您的 GitHub 驗證認證。如果您選擇建立新密碼，則會開啟一個AWSSecrets Manager密碼視窗。

      1. 在「建立AWSSecrets Manager密碼視窗」中輸入下列資訊：

        1. 秘密名稱 — 您秘密的名稱。前綴 'AmazonKendraGitHub-' 會自動添加到您的密碼名稱中。

        2. 對於 GitHubToken — 輸入您在 GitHub 帳戶中建立的驗證認證值。

      2. 選擇 儲存 。

    5. V@@ irtual Private Cloud (VPC) — 您可以選擇使用 VPC。如果是這樣，您必須新增子網路和 VPC 安全性群組。

    6. IAM 角色 — 選擇現有的 IAM 角色或建立新的 IAM 角色以存取儲存庫登入資料和索引內容。

      注意

      用於索引的 IAM 角色無法用於資料來源。如果您不確定索引或常見問題集是否使用現有角色，請選擇 [建立新角色] 以避免發生錯誤。

    7. 選擇 下一步。

  7. 在配置同步設定頁面上，輸入以下資訊：

    1. 選取要編目的存放庫 — 您要編目的 GitHub 實體或內容類型。

    2. 變更記錄 — 選取此選項可更新索引，而不是同步所有檔案。

    3. 內容類型 — 選取您要包含的檔案類型。

    4. 正則表達式模式-包含或排除某些文件的正則表達式模式。最多可新增 100 個圖樣。

    5. 在同步執行排程中，針對頻率 — Amazon Kendra 與您的資料來源同步的頻率。

    6. 選擇 下一步。

  8. 在設定欄位映射頁面上，輸入以下資訊：

    1. 針對儲存庫、存放庫確認、發行文件、發行註解、發佈附件、提取請求註解、提取請求文件、提取請求附件 — 從Amazon Kendra產生的預設資料來源欄位中選取您要對映至索引的預設資料來源欄位。

    2. 新增欄位 — 新增自訂資料來源欄位，以建立要對映的索引欄位名稱和欄位資料類型。

    3. 選擇 下一步。

  9. 在 [檢閱並建立] 頁面上，檢查您輸入的資訊是否正確，然後選取 [新增資料來源]。您也可以從此頁面中選擇編輯資訊。成功新增資料來源後，資料來源將顯示在 [資料來源] 頁面上。

API

若要連線Amazon Kendra到 GitHub

您必須使用 GitHubConfigurationAPI 物件指定下列項目：

  • 資料來源類型 — 將資料來源類型指定為SAAS或ON_PREMISE。

  • 秘密亞馬遜資源名稱 (ARN) — 提供包含您 GitHub 帳戶身份驗證登入資料的Secrets Manager密碼的亞馬遜資源名稱 (ARN)。私密會儲存在具有下列索引鍵的 JSON 結構中：

    {
    "personalToken": "token"
}
    注意

    請務必定期重新整理或旋轉您的憑證和密碼。僅為您自己的安全提供必要的訪問級別。不建議跨資料來源以及連接器 1.0 和 2.0 版 (如果適用) 重複使用認證和密碼。

  • IAM 角色 — 指定RoleArn何時呼叫CreateDataSource以提供具有存取Secrets Manager密碼的權限的IAM角色，以及呼叫 GitHub 連接器和所需的公用 APIAmazon Kendra。如需詳細資訊，請參閱資 GitHub 料來源的 IAM 角色。

您也可以新增下列選用功能：

  • V@@ irtual Private Cloud (VPC) — 指定VpcConfiguration為資料來源組態的一部分。請參閱設Amazon Kendra定以使用 VPC。

    注意

    如果您使用 GitHub 伺服器，則必須使用連線Amazon VPC到 GitHub伺服器。

  • 變更記錄 — 是否Amazon Kendra應該使用 GitHub 資料來源變更記錄機制來決定是否必須在索引中新增、更新或刪除文件。

    注意

    如果您不想Amazon Kendra掃描所有文檔，請使用更改日誌。如果您的變更記錄檔很大，掃描 GitHub 資料來源中的文件可能比處理變更記錄所花費的時間更Amazon Kendra少。如果您是第一次將 GitHub資料來源與索引同步，則會掃描所有文件。

  • 包含和排除篩選器 — 指定要包含還是排除某些檔案。

    注意

    大多數資料來源使用規則運算式模式，這些模式是稱為篩選器的包含或排除模式。如果您指定包含篩選器，則只有符合包含項目篩選器的內容會編製索引。不符合包含篩選器的任何文件都不會建立索引。如果您指定包含和排除篩選器，符合排除篩選器的文件將不會建立索引，即使它們符合包含篩選器也是如此。

  • 欄位對映 — 選擇將資 GitHub 料來源欄位對映至Amazon Kendra索引欄位。如需詳細資訊，請參閱映射資料來源欄位。

  • 使用者前後關聯篩選 — 依預設Amazon Kendra搜尋資料來源的存取控制清單 (ACL)。ACL 資訊可用來根據使用者或使用者對文件的群組存取權來篩選搜尋結果。有關詳情，請參閱資 GitHub 料來源的使用者內容篩選。
