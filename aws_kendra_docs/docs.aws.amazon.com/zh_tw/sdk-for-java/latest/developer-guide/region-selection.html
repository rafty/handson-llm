AWS選擇區域

區域允許您訪問AWS服務實際位於特定地理區域。這對於備援以及讓您的資料和應用程式在靠近您和您的使用者存取位置附近執行，都很有用。

InAWS SDK for Java2.x，版本 1.x 的所有不同區域相關類別都已收合到一個區域類別。您可以將此類別用於所有區域相關的動作，例如擷取區域的中繼資料或檢查區域中是否有可用服務。

選擇區域

您可以指定區域名稱，開發套件會自動為您選擇適當的端點。

若要明確設定區域，我們建議您使用 Region 類別中定義的常數。這是所有公開可用區域的列舉。若要使用類別中的區域來建立用戶端，請使用下列程式碼。

Ec2Client ec2 = Ec2Client.builder()
          .region(Region.US_WEST_2)
          .build();

如果您嘗試使用的區域不是 Region 類別中的其中一個常數，您可以使用 of 方法建立一個新的區域。此功能可讓您存取新的區域，無需升級開發套件。

Region newRegion = Region.of("us-east-42");
Ec2Client ec2 = Ec2Client.builder()
          .region(newRegion)
          .build();
注意

使用建置器建置用戶端後，這是不可變的，而且區域無法變更。如果您正在使用多個AWS 區域，您應該建立多個用户端-每個區域一個。

選擇特定端點

EASHAWS客户端可以配置為使用特定端點，方法是調用endpointOverride方法。

例如，要配置Amazon EC2用户端使用歐洲 (愛爾蘭) 區域，請使用下列程式碼。

Ec2Client ec2 = Ec2Client.builder()
               .region(Region.EU_WEST_1)
               .endpointOverride(URI.create("https://ec2.eu-west-1.amazonaws.com"))
               .build();

請參閱區域與端點如需所有區域及其相對應端點的最新清單AWS服務。

從環境中自動確定區域

在 Amazon EC2 或 AWS Lambda 上執行時，您可能會希望設定用戶端為使用您程式碼執行的相同區域。這會讓您的程式碼與其執行環境分離，也更容易將應用程式部署到多個區域，以降低延遲或提供備援。

若要使用預設的登入資料/區域供應者鏈結來從環境判斷區域，請使用用戶端建置器的 create 方法。

Ec2Client ec2 = Ec2Client.create();

如果您不使用 region 方法明確設定區域，開發套件會查閱預設區域供應者鏈結，並嘗試判斷要使用的區域。

預設區域供應者鏈結

以下是區域查詢程序：

  1. 在建置器上使用 region 設定的任何明確區域優先於其他項目。

  2. 檢查 AWS_REGION 環境變數。如果有設定，會使用該區域來設定用戶端。

    注意

    此環境變數由 Lambda 容器設定。

  3. 軟件開發工具包檢查AWS共用組態檔案 (通常位於~/.aws/config。如果 region 屬性存在，開發套件會予以使用。

    • AWS_CONFIG_FILE 環境變數可用於自訂共用組態檔的位置。

    • 所以此AWS_PROFILE環境變數或aws.profile系統屬性可用於自訂開發套件載入的設定檔。

  4. 開發套件會嘗試使用 Amazon EC2 執行個體中繼資料服務，判斷目前執行 Amazon EC2 執行個體的區域。

  5. 如果開發套件在這個時候仍找不到區域，用戶端建立會失敗，並出現例外狀況。

開發時AWS應用程式時，常見的方法是使用共用組態檔案（詳加了登入資料取回順序) 來設定適用於本機開發的區域，並依賴預設區域供應者鏈結以判斷在AWS基礎設施。這可大幅簡化用戶端建立並讓您的應用程式保持可攜式。

檢查區域的服務可用性

查看是否特定AWS 服務在某個地區可用，請使用serviceMetadata和region方法來提供您要檢查的服務。

DynamoDbClient.serviceMetadata().regions().forEach(System.out::println);

請參閱 Region 類別文件以取得您可以指定的區域，並使用服務的端點字首來查詢。
