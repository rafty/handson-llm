管理 AWS 區域

AWS 區域 是地理區域中 AWS 資源的集合。每個 AWS 區域 都是隔離的，且與其他區域各自獨立。區域提供容錯能力、穩定性和恢復能力，也可降低延遲。它們可讓您建立冗餘資源，這些資源會保持可用且不受區域運行中斷影響。

您在某個區域中建立的資源在任何其他區域中都不存在，除非您明確使用 AWS 服務提供的複寫功能。例如，Amazon S3 與 Amazon EC2 支援跨區域複寫。某些服務 (例如AWS Identity and Access Management (IAM) 沒有區域資源。

您可以使用政策條件來控制AWS區域中AWS服務的存取。

請求

若要選擇擇加入區域，您必須具有相應的許可。若要檢視 IAM 政策範例，請參閱 IAM 使用者指南中的允許啟用和停用AWS區域。

若要選擇擇加入區域

  1. 登入 AWS Management Console。

  2. 在主控台的右上角，選擇您的帳戶名稱或號碼，然後選擇 [帳戶]。

  3. 在「區AWS域」下的「區域」的「作業」欄中，選擇「啟用」。

    提示

    如果「狀態」欄顯示「預設為啟用」，則您不需要選擇加入「地區」。

  4. 在對話方塊中，檢閱資訊文字，然後選擇「啟用區域」。

  5. 等到區域備妥可供使用。

請求

  • 您無法選擇退出預設為開啟的區域。

  • 若要退出某個地區，您必須擁有選擇退出區域的權限。若要檢視 IAM 政策範例，請參閱 IAM 使用者指南中的允許啟用和停用AWS區域。

若要選擇退出某個地區

  1. 登入 AWS Management Console。

  2. 在主控台的右上角，選擇您的帳戶名稱或號碼，然後選擇 [帳戶]。

  3. 在「區AWS域」下的「作業」欄位中，選擇「停用」。

  4. 在對話方塊中，檢閱資訊文字，然後選擇「停用區域」。

使用 AWS CLI 描述區域

使用描述區域命令來描述您帳戶可用的地區，以查看您的選擇加入狀態。

aws ec2 describe-regions --all-regions

如果您不需要選擇加入「區域」，則輸出包括以下內容：

"OptInStatus": "opt-in-not-required"

如果您未選擇加入「地區」，則輸出包括以下內容：

"OptInStatus": "not-opted-in"

選擇加入「區域」後，輸出內容包括以下內容：

"OptInStatus": "opted-in"
