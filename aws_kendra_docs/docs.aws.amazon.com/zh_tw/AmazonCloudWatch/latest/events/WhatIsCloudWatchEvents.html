什麼是 Amazon CloudWatch Events？

注意

Amazon EventBridge 是管理事件的首選方式。CloudWatch Events 和 EventBridge 是相同的基礎服務和 API，但 EventBridge 提供了更多功能。您在 CloudWatch 或 EventBridge 中所做的變更將會出現在每個主控台中。如需詳細資訊，請參閱 Amazon EventBridge。

Amazon CloudWatch Events 會提供近乎即時的系統事件串流，說明 Amazon Web Services (AWS) 資源的變動情形。使用您可以快速設定的簡單規則，您可以比對事件並將它們路由到一或多個目標函數或串流。CloudWatch Events 在操作變更時會查覺到。CloudWatch Events 會回應這些操作變更並視需要進行修正動作，透過傳送訊息來回應環境、啟用功能、執行變更和擷取狀態資訊。

您也可使用 CloudWatch Events 排定自動化動作，讓動作使用 Cron 或 Rate 表達式在特定時間自動觸發。如需更多詳細資訊，請參閱 排程規則表達式。

您可以設定以下 AWS 服務做為 CloudWatch Events 的目標：

  • Amazon EC2 執行個體

  • AWS Lambda 函式

  • Amazon Kinesis Data Streams 中的串流

  • Amazon Kinesis Data Firehose 中的交付串流

  • Amazon CloudWatch Logs 中的記錄群組

  • Amazon ECS 任務

  • Systems Manager Run Command

  • Systems Manager Automation

  • AWS Batch 工作

  • Step Functions 狀態機器

  • CodePipeline 中的管道

  • CodeBuild 專案

  • Amazon Inspector 評定範本

  • Amazon SNS 主題

  • Amazon SQS 佇列

  • 內建目標：EC2 CreateSnapshot API call、EC2 RebootInstances API call、EC2 StopInstances API call 和 EC2 TerminateInstances API call。

  • 另一個 AWS 帳戶的預設事件匯流排

Concepts

開始使用 CloudWatch Events 之前，您應該了解以下概念：

  • 事件— 事件表示您的 AWS 環境中的變更。AWS 資源可以在其狀態變更時產生事件。例如，當 EC2 執行個體的狀態由待處理變更為執行中時，Amazon EC2 會產生事件，而當它啟動或終止執行個體時，Amazon EC2 Auto Scaling 會產生事件。當您發出 API 呼叫時，AWS CloudTrail 會發佈事件。​ 您可以產生自訂應用程式層級事件並將這些事件發佈到 CloudWatch Events。您也可以設定定期產生的排程事件。如需產生事件的服務清單，以及每個服務的範例事件，請參閱 受支援服務的 CloudWatch Events 事件範例。

  • 規則 – 規則會比對連入事件，並將這些事件路由到目標以進行處理。單一規則可路由到多個目標，所有的路由都可以同時處理。規則不會依特定順序進行處理。這可讓組織的不同部分尋找和處理與他們利害相關的事件。規則可透過僅傳送某個部分或以常數進行覆寫，以自訂傳送到目標的 JSON。

  • 目標 – 目標會處理事件。目標可以包含 Amazon EC2 執行個體、AWS Lambda 函數、Kinesis 串流、Amazon ECS 任務、Step Functions 狀態機器、Amazon SNS 主題、Amazon SQS 佇列和內建目標。目標會收到 JSON 格式的事件。

    規則的目標必須位於和規則相同的區域。

相關 AWS 服務

以下服務係與 CloudWatch Events 一起使用：

  • AWS CloudTrail 可讓您為帳戶監控對 CloudWatch Events API 所做的呼叫，包括由 AWS Management Console、AWS CLI 和其他服務的呼叫。當 CloudTrail 記錄處於開啟狀態，CloudWatch Events 會將日誌檔寫入到 S3 儲存貯體。每個日誌檔案包含一個或多個記錄，取決於執行了多少動作來滿足請求。如需更多詳細資訊，請參閱 使用 AWS CloudTrail 記錄 Amazon CloudWatch Events API 呼叫。

  • AWS CloudFormation 可讓您建置 AWS 資源的模型並進行設定。您建立一個描述所需之 AWS 資源的範本，而 AWS CloudFormation 負責為您佈建與設定那些資源。您可以使用 AWS CloudFormation 範本中的 CloudWatch Events 規則。如需詳細資訊，請參閱《AWS CloudFormation 使用者指南》中的 AWS::Events::Rule。

  • AWS Config 可讓您記錄 AWS 資源的組態變更。這包括資源彼此之間如何關聯，以及它們在過去的設定方式，因此您可以看到組態與關係如何隨著時間變更。您也可以建立 AWS Config 規則，以檢查您的資源是否符合組織的政策。如需詳細資訊，請參閱《AWS Config 開發人員指南》。

  • AWS Identity and Access Management (IAM) 可協助為您的使用者安全地控管對 AWS 資源的存取。使用 IAM 以控制誰可以使用 AWS 資源 (身份驗證)、他們可以使用哪些資源和使用的方式 (授權)。如需詳細資訊，請參閱 IAM 使用者指南。

  • Amazon Kinesis Data Streams 可快速和近乎持續的進行資料擷取和彙總。使用的資料類型包括 IT 基礎架構日誌資料、應用程式日誌、社交媒體、市場資料摘要和 web 點擊流資料。因為資料擷取和處理的回應時間是即時的，所以處理通常是輕量的。如需詳細資訊，請參閱《Amazon Kinesis Data Streams 開發人員指南》。

  • AWS Lambda 讓您建置可快速回應新資訊的應用程式。上傳您的應用程式碼做為 Lambda 函數，而 Lambda 會在高可用性運算基礎設施上執行您的程式碼。Lambda 執行所有運算資源的管理，包括伺服器和作業系統維護、容量佈建、自動調整規模、程式碼與安全性修補程式部署，以及程式碼監控和記錄。如需詳細資訊，請參閱《AWS Lambda 開發人員指南》。
