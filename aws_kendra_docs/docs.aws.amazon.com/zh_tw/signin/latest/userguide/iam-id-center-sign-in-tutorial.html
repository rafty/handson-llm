登入AWS存取入口網站

IAM 身分中心的使用者是其中的成員AWS Organizations。IAM 身分中心的使用者可以使用特定登入 URL 登入存取入口網站，AWS存取多個AWS 帳戶商務應用程式。如需特定登入 URL 的詳細資訊，請參閱AWS訪問門戶。

在您以使用者AWS 帳戶身分登入 IAM Identity Center 前，請收集以下必要資訊。

  • 公司使用者名稱

  • 企業密碼

  • 特定登入 URL

注意

登入後，您的AWS存取入口網站工作階段有效期為 8 小時。您需要在 8 小時後重新登錄。

若要登入AWS存取入口網站

  1. 在瀏覽器視窗中，貼上電子郵件提供給您的登入 URL，例如https://your_subdomain.awsapps.com/start。然後按 Enter 鍵。

  2. 使用您的公司憑證 (如使用者名稱和密碼) 登入。

    注意

    如果您的管理員向您發送了一次性密碼（OTP）的電子郵件，並且這是您第一次登錄，請輸入該密碼。登入後，您必須建立新密碼以供 future 登入使用。

  3. 如果系統要求您輸入驗證碼，請檢查您的電子郵件。然後將代碼複製並粘貼到登錄頁面中。

    注意

    驗證碼通常是透過電子郵件傳送，但傳送方式可能會有所不同。如果您的電子郵件中沒有收到驗證碼，請洽詢您的管理員以取得驗證碼的詳細資訊。

  4. 如果在 IAM 身分中為您的使用者啟用了 MFA，則您可以使用它進行驗證。

  5. 驗證後，您可以訪問入口網站中顯示的任何AWS 帳戶和應用程式。

    1. 若要登入，AWS Management Console請選取AWS 帳戶圖標，然後選取要管理的個別帳戶。

      系統會顯示使用者擁有的權限，以及開啟管理主控台或取得命令列或程式設計存取認證的連結。

      選取管理主控台連結。

    2. 若要登入應用程式，請選擇您要存取的應用程式圖示。

在 IAM Identity Center 中以使用者身分登入，可讓您在設定的持續時間 (稱為工作階段) 內存取資源的登入資源。根據預設，使用者可以登入AWS 帳戶一個小時。管理員可為工作階段設定的最大值為 12 小時。工作階段結束後，您可以再次登入。
