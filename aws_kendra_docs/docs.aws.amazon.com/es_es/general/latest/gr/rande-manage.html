Administración de Regiones de AWS

Una Región de AWSes una colección deAWS recursos de que se encuentran en un área geográfica. Cada Región de AWS es independiente y está aislada de las demás regiones. Las regiones proporcionar tolerancia a errores, estabilidad y resistencia, y también pueden reducir la latencia. Puede aprovisionar recursos en varias regiones de manera que sus aplicaciones sigan estando disponibles y no resulten afectadas por una interrupción regional.

Los recursos que crea en una región no existen en ninguna otra región salvo que utilice explícitamente una característica de replicación ofrecida por un servicio de AWS. Por ejemplo, Amazon S3 y Amazon EC2 admiten la replicación entre regiones. Algunos servicios, comoAWS Identity and Access Management (IAM), no cuentan con recursos regionales.

Puede utilizar las condiciones de la política para controlar el acceso aServicios de AWS en unRegión de AWS.

Su cuenta determina las regiones que están disponibles para usted.

  • UnaCuenta de AWS proporciona varias regiones de manera que se puedan lanzarAWS recursos en ubicaciones que cumplan sus requisitos. Por ejemplo, podría desear lanzar instancias en Europa, para estar más cerca de sus clientes europeos o para cumplir requisitos legales.

  • Una cuenta deAWSGovCloud (Oeste de EE. UU.) proporciona acceso a la regiónAWSGovCloud (Oeste de EE. UU.) y la regiónAWSGovCloud (Este de EE. UU.). Para obtener más información, consulte AWS GovCloud (US).

  • Una cuenta de Amazon AWS (China) proporciona acceso solo a las regiones de Pekín y Ningxia. Para obtener más información, consulte Amazon Web Services en China.

Al activar una región, AWS realiza acciones para preparar su cuenta en dicha región, como la distribución de sus recursos de IAM a la región. Este proceso tarda unos minutos para la mayoría de las cuentas, pero puede tardar varias horas. No puedes usar una región hasta que se complete este proceso.

Requisitos

Para habilitar una región, debe tener los permisos adecuados. Para ver un ejemplo de política de IAM, consulte Permitir la activación y la desactivaciónRegiones de AWS en la Guía del usuario de IAM.

Para habilitar una región utilizando la consola

  1. Inicie sesión en AWS Management Console.

  2. En la esquina superior derecha de la consola, selecciona la región actual para ampliar la lista de regiones, desplázate hasta la parte inferior de la lista y, a continuación, selecciona Administrar regiones.

  3. Marque la columna Acción de la región que desee utilizar.

    • Si la columna Estado contiene Habilitado de forma predeterminada, no es necesario realizar ninguna otra acción. La región se activó automáticamente para su cuenta.

    • Si la columna Estado contiene Desactivado, seleccione Habilitar. Revise el texto informativo y, a continuación, seleccione Habilitar región. Debe esperar hasta que la región esté lista para su uso.

Para habilitar una región mediante elAWS CLI

Utilice el siguiente comando enable-region.

aws account enable-region --region-name af-south-1

Desactivación de una región

Si habilitó una región que no se habilita automáticamente, puede deshabilitarla si es necesario. Después de deshabilitar una región, los recursos de esta región dejarán de estar disponibles, en función de la coherencia final. Sin embargo, los recursos no se eliminan. Puede acceder a ellos si vuelve a habilitar esta región.

Requisitos

  • No puede deshabilitar una región que esté habilitada de forma predeterminada. (Consulte la tabla de la sección anterior.)

  • Para deshabilitar una región, debe tener permiso para deshabilitar las regiones. Para ver un ejemplo de política de IAM, consulte Permitir habilitar y deshabilitarAWS regiones en la Guía del usuario de IAM.

  • Antes de deshabilitar una región, le recomendamos que elimine todos los recursos de esa región. Tras deshabilitar una región, ya no podrá ver ni administrar los recursos de la región desde esa cuenta a través de la consolaAWS de administración o lasAWS API mediante los principios de IAM. Sin embargo, los recursos de dicha región pueden seguir generando cargos. Para obtener más información, consulte Habilitar y deshabilitar regiones en la Guía delAWS Billing and Cost Management usuario.

Para desactivar una región utilizando la consola

  1. Inicie sesión en AWS Management Console.

  2. En la esquina superior derecha de la consola, selecciona la región actual para ampliar la lista de regiones, desplázate hasta la parte inferior de la lista y, a continuación, selecciona Administrar regiones.

  3. En la columna Acción de la región, seleccione Desactivar.

  4. En el cuadro de diálogo, revise el texto informativo y, a continuación, seleccione Deshabilitar región.

Para deshabilitar una región mediante elAWS CLI

Utilice el siguiente comando disable-region.

aws account disable-region --region-name af-south-1

Describir sus regiones utilizandoAWS CLI

Utilice el comando list-regions para describir todas las regiones disponibles para su cuenta, incluido su estado.

aws account list-regions --max-results 50

Utilice el get-region-opt-statuscomando para describir el estado de una región específica.

aws account get-region-opt-status --region-name af-south-1

En el caso de las regiones que se habilitan automáticamente en su cuenta, el resultado de estos comandos incluye lo siguiente:

"RegionOptStatus": "ENABLED_BY_DEFAULT"

En el caso de las regiones que debe habilitar manualmente, el resultado incluye lo siguiente:

"RegionOptStatus": "DISABLED"

Después de habilitar una región, el resultado incluye lo siguiente:

"RegionOptStatus": "ENABLED"
