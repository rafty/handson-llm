Cómo funciona

Amazon Lex V2 le permite crear aplicaciones mediante una interfaz de texto o voz para mantener una conversación con un usuario. A continuación se indican los pasos habituales para trabajar con Amazon Lex V2:

  1. Crea un bot y añade uno o más idiomas. Configure el bot para que comprenda el objetivo del usuario, entable una conversación con el usuario para obtener información y cumpla con la intención del usuario.

  2. Pruebe el bot. Puede utilizar el cliente de ventana de prueba que proporciona la consola Amazon Lex V2.

  3. Publique una versión y cree un alias.

  4. Implemente el bot. Puedes implementar el bot en tus propias aplicaciones o plataformas de mensajería, como Facebook Messenger o Slack.

Antes de empezar, familiarícese con los siguientes conceptos principales y la terminología de Amazon Lex V2:

  • Bot: un bot realiza tareas automatizadas, como pedir una pizza, reservar un hotel, pedir flores, etc. Un bot de Amazon Lex V2 cuenta con capacidades de reconocimiento automático de voz (ASR) y comprensión del lenguaje natural (NLU).

    Los bots de Amazon Lex V2 pueden entender las entradas del usuario proporcionadas con texto o voz y conversar en lenguaje natural.

  • Idioma: un bot de Amazon Lex V2 puede conversar en uno o más idiomas. Cada idioma es independiente de los demás. Puede configurar Amazon Lex V2 para que converse con un usuario mediante palabras y frases nativas. Para obtener más información, consulte Lenguajes y configuraciones regionales admitidos por Amazon Lex V2.

  • Intención: una intención representa una acción que el usuario desea llevar a cabo. Puede crear un bot que admita una o más intenciones relacionadas. Por ejemplo, podría crear un anuncio que pida pizzas y bebidas. Para cada intención, debe proporcionar la siguiente información obligatoria:

    • Nombre de intención: nombre descriptivo de la intención. Por ejemplo, OrderPizza.

    • Ejemplos de expresiones: cómo un usuario puede transmitir la intención. Por ejemplo, un usuario podría decir «¿Puedo pedir una pizza?» o «Quiero pedir una pizza».

    • Cómo cumplir la intención: cómo desea cumplir la intención después de que el usuario proporcione la información necesaria. Le recomendamos crear una función de Lambda para cumplir con la intención.

      Si lo desea, puede configurar la intención para que Amazon Lex V2 devuelva la información a la aplicación cliente para el cumplimiento necesario.

    Además de las intenciones personalizadas, Amazon Lex V2 incluye intenciones integradas para configurar rápidamente su bot. Para obtener más información, consulte Intenciones y tipos de ranuras incorporados.

    Amazon Lex siempre incluye una intención alternativa para cada bot. La intención alternativa se utiliza cuando Amazon Lex no puede deducir la intención del usuario. Para obtener más información, consulte AMAZONA. FallbackIntent.

  • Ranura: una intención puede requerir cero o más ranuras o parámetros. Puede añadir parámetros como parte de la configuración. En tiempo de ejecución, Amazon Lex V2 solicita al usuario valores de ranura específicos. El usuario debe proporcionar los valores de todos los espacios necesarios para que Amazon Lex V2 pueda cumplir con su intención.

    Por ejemplo, laOrderPizza intención requiere intervalos como el tamaño, el tipo de masa y la cantidad de pizzas. Para cada ranura, debe proporcionar el tipo de ranura y una o más solicitudes que Amazon Lex V2 envía al cliente para obtener valores del usuario. El usuario puede responder con un valor de espacio que contenga palabras adicionales, como «pizza grande, por favor» o «sigamos con la pequeña». Amazon Lex V2 sigue entendiendo el valor de la ranura.

  • Tipo de ranura: cada ranura tiene un tipo. Puede crear su propio tipo de ranura o utilizar tipos de ranura integrados. Por ejemplo, puede crear y utilizar los siguientes tipos de slot para la intención OrderPizza:

    • Tamaño: con los valores de enumeración Small, Medium y Large.

    • Masa: con los valores de enumeración Thick y Thin.

    Amazon Lex V2 también ofrece los tipos de ranuras integradas. Por ejemplo, AMAZON.Number es un tipo de slot integrado que puede utilizar con el número de pizzas encargadas. Para obtener más información, consulte Intenciones y tipos de ranuras incorporados.

  • Versión: una versión es una instantánea numerada de tu trabajo que puedes publicar para utilizarla en diferentes partes del flujo de trabajo, como el desarrollo, la implementación beta y la producción. Una vez que hayas creado una versión, puedes usar un bot tal y como existía cuando se creó la versión. Después de crear una versión, permanece igual mientras sigues trabajando en la aplicación.

  • Alias: un alias es un puntero a una versión específica de un bot. Con un alias, puede actualizar la versión que utilizan las aplicaciones de su cliente. Por ejemplo, puede apuntar un alias hacia la versión 1 de su bot. Cuando esté listo para actualizar el bot, puede publicar la versión 2 y cambiar el alias para que apunte a la nueva versión. Dado que sus aplicaciones utilizan el alias en lugar de una versión específica, todos los clientes obtienen las nuevas funcionalidades sin necesidad de actualizarse.

Para obtener una lista de lasAWS regiones en las que Amazon Lex V2 está disponible, consulte los puntos de enlace y las cuotas de Amazon Lex V2 en la Referencia general de Amazon Web Services.
