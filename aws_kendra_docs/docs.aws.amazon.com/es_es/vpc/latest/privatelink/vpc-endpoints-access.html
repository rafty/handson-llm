Uso de políticas de punto de conexión para controlar el acceso a puntos de conexión de VPC

Cuando crea un punto de conexión de interfaz o un punto de conexión de puerta de enlace, puede adjuntar una política de punto de conexión. La política de punto de conexión es un documento de política de JSON que controla qué entidades principales de AWS pueden utilizar el punto de conexión de VPC para acceder al servicio de punto de conexión.

No puede asociar más de una política a un punto de enlace. Sin embargo, puede modificar una política de punto de conexión en cualquier momento.

Una política de punto de conexión no anula ni reemplaza las políticas basadas en identidad o basadas en recursos. Por ejemplo, si utiliza un punto de conexión de interfaz para conectarse a Amazon S3, también puede utilizar las políticas de bucket de Amazon S3 para controlar el acceso a los buckets desde puntos de conexión específicos o VPC específicas.

Políticas de punto de conexión de VPC

Asocia una política de punto de conexión de VPC a un punto de conexión. Si no especifica una política de punto de conexión cuando crea un punto de conexión, adjuntaremos la siguiente política, que permite el acceso completo al servicio.

{
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": "*",
            "Action": "*",
            "Resource": "*"
        }
    ]
}
Consideraciones

  • La política debe contener un elemento principal.

  • El tamaño de una política de punto de enlace no puede tener más de 20 480 caracteres (incluidos espacios en blanco).

  • No todos los servicios de punto de conexión son compatibles con las políticas de punto de conexión. Si un servicio no es compatible con las políticas de puntos de enlace, el punto de enlace permite el acceso completo al servicio. Para obtener más información, consulte Ver la compatibilidad con las políticas de puntos de conexión.

Entidades principales para puntos de conexión de puerta de enlace

En los puntos de conexión de puerta de enlace, debe utilizar la clave de condición aws:PrincipalArn para conceder el acceso a una entidad principal.

Si especifica la entidad principal en uno de los siguientes formatos, se concede acceso solo a Usuario raíz de la cuenta de AWS y no a todos los usuarios y roles de la cuenta.

"AWS": "account_id"
"AWS": "arn:aws:iam::account_id:root"

Si especifica un nombre de recurso de Amazon (ARN) para la entidad principal, el ARN se transforma en un ID de entidad principal único cuando se guarda la política.

Actualización de una política de punto de conexión de VPC

Utilice el siguiente procedimiento para actualizar una política de punto de conexión. Después de la actualización de una política de punto de conexión, los cambios pueden tardar unos minutos en aplicarse.

Para actualizar una política de punto de conexión de VPC

  1. Abra la consola de Amazon VPC en https://console.aws.amazon.com/vpc/.

  2. En el panel de navegación, elija Puntos de conexión.

  3. Seleccione el punto de conexión de VPC.

  4. Elija Acciones, Administrar política.

  5. Elija Acceso completo para permitir el acceso completo al servicio, o bien, elija Personalizar y adjunte una política personalizada.

  6. Seleccione Guardar.
