Tareas que requieren credenciales de usuario raíz

importante

Las siguientes acciones AWS Identity and Access Management (IAM) llegarán al final del soporte estándar en julio de 2023: aws-portal:ModifyAccount yaws-portal:ViewAccount. Consulte la sección Uso de acciones detalladas para AWS Billing reemplazar estas acciones por acciones detalladas para tener acceso a las consolas de administración de AWS costos y AWS Billing cuentas. AWS

Si creó su o su AWS cuenta AWS Organizations de administración de antes del 6 de marzo de 2023, las acciones detalladas entrarán en vigor a partir de julio de 2023. Le recomendamos que agregue las acciones detalladas, pero no elimine los permisos existentes con el aws-portal o los prefijos de purchase-orders.

Si creó su o su AWS cuenta AWS Organizations de administración el 6 de marzo de 2023 o después, las acciones detalladas entran en vigor inmediatamente.

Le recomendamos que configure un usuario administrativo AWS IAM Identity Center (successor to AWS Single Sign-On) para que realice las tareas diarias y acceda a AWS los recursos. Sin embargo, las tareas que se enumeran a continuación únicamente se pueden realizar cuando se inicia sesión como usuario raíz de una cuenta.

Tareas

  • Cambie la configuración de la cuenta. Esto incluye el nombre de la cuenta, la dirección de correo electrónico, la contraseña del usuario raíz y las claves de acceso del usuario raíz. Otros ajustes de la cuenta, como la información de contacto, la preferencia de moneda de pago yRegiones de AWS, además, no requieren credenciales de usuario raíz.

  • Restaure los permisos de usuario de IAM. Si el único administrador de IAM revoca accidentalmente sus propios permisos, puede iniciar sesión como usuario raíz para editar las políticas y restaurar esos permisos.

  • Active el acceso de IAM a la consola de Billing and Cost Management.

  • Consulte ciertas facturas de impuestos. Un usuario de IAM con el portal aws-portal: con ViewBilling permiso puede ver y descargar facturas con IVA de AWS Europa, pero no de AWS Inc. ni de Amazon Internet Services Private Limited (AISPL).

  • Cierra tu Cuenta de AWS.

  • Se ha registrado como vendedor en el marketplace de instancias reservadas.

  • Configure un bucket de Amazon S3 para habilitar la autenticación multifactor.

  • Edite o elimine una política de bucket de Amazon Simple Storage Service (Amazon S3) que incluya un ID de nube privada virtual (VPC) o un ID de punto de enlace de VPC no válidos.

  • Regístrese en AWS GovCloud (US).

  • Solicite las claves de acceso del usuario raíz de la AWS GovCloud (US) cuenta desdeAWS Support.
