Steuern des Zugriffs auf VPC-Endpunkte mithilfe von Endpunktrichtlinien

Wenn Sie einen Schnittstellenendpunkt oder einen Gateway-Endpunkt erstellen, können Sie eine Endpunktrichtlinie anfügen. Die Endpunktrichtlinie ist ein JSON-Richtliniendokument, das steuert, welche AWS-Prinzipale den VPC-Endpunkt für den Zugriff auf den Endpunktservice verwenden können.

Sie können einem Endpunkt mehr als eine Richtlinie anfügen. Sie können jedoch jederzeit eine Endpunktrichtlinie ändern.

Eine Endpunktrichtlinie setzt keine identitätsbasierten Richtlinien oder ressourcenbasierten Richtlinien außer Kraft oder ersetzt sie. Beispiel: Wenn Sie einen Schnittstellenendpunkt verwenden, um eine Verbindung zu Amazon S3 herzustellen, können Sie auch Amazon-S3-Bucket-Richtlinien verwenden, um den Zugriff auf Buckets von bestimmten Endpunkten oder VPCs zu steuern.

VPC-Endpunktrichtlinien

Sie fügen einem Endpunkt eine VPC-Endpunktrichtlinie an. Wenn Sie beim Erstellen eines Endpunkts keine Endpunktrichtlinie angeben, wird ihm die folgende Richtlinie zugeordnet, die den vollen Zugriff auf den Service ermöglicht.

{
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": "*",
            "Action": "*",
            "Resource": "*"
        }
    ]
}
Überlegungen

  • Ihre Richtlinie muss ein Prinzipal-Element enthalten.

  • Die Größe einer Endpunktrichtlinie darf 20.480 Zeichen (einschließlich Leerzeichen) nicht überschreiten.

  • Nicht alle Endpunktservices unterstützen Endpunktrichtlinien. Wenn ein Service keine Endpunktrichtlinien unterstützt, ermöglicht der Endpunkt vollen Zugriff auf den Service. Weitere Informationen finden Sie unter Anzeigen der Unterstützung für Endpunkt-Richtlinien.

Prinzipale für Gateway-Endpunkte

Bei Gateway-Endpunkten müssen Sie den Bedingungsschlüssel aws:PrincipalArn verwenden, um einem Prinzipal den Zugriff zu gewähren.

Wenn Sie den Prinzipal in einem der folgenden Formate angeben, wird der Zugriff nur dem Root-Benutzer des AWS-Kontos gewährt, nicht aber allen Benutzern und Rollen für das Konto.

"AWS": "account_id"
"AWS": "arn:aws:iam::account_id:root"

Wenn Sie einen Amazon-Ressourcennamen (ARN) für den Prinzipal angeben, wird der ARN beim Speichern der Richtlinie in eine eindeutige Prinzipal-ID umgewandelt.

Beispiele für IAM-Richtlinien für Gateway-Endpunkte finden Sie in den folgenden Themen:

Aktualisieren einer VPC-Endpunktrichtlinie

Gehen Sie wie folgt vor, um eine Endpunktrichtlinie zu aktualisieren. Nachdem Sie eine Endpunktrichtlinie aktualisiert haben, kann es einige Minuten dauern, bis die Änderungen wirksam werden.

So aktualisieren Sie eine VPC-Endpunktrichtlinie

  1. Öffnen Sie die Amazon VPC-Konsole unter https://console.aws.amazon.com/vpc/.

  2. Wählen Sie im Navigationsbereich Endpunkte aus.

  3. Wählen Sie den VPC-Endpunkt.

  4. Wählen Sie Actions (Aktionen), Manage policy (Verwalten von Richtlinien).

  5. Wählen Sie Full Access (Voller Zugriff), um vollen Zugriff auf den Service zu gewähren, oder wählen Sie Custom (Benutzerdefiniert), und fügen Sie eine benutzerdefinierte Richtlinie hinzu.

  6. Wählen Sie Save (Speichern).
