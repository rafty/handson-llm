Aufgaben, die Stammbenutzer-Anmeldeinformationen erfordern

Wichtig

Für die folgenden AWS Identity and Access Management (IAM-) Maßnahmen läuft die Standardunterstützung im Juli 2023 aus: aws-portal:ModifyAccount undaws-portal:ViewAccount. Weitere Informationen finden Sie unter Verwenden von detaillierten AWS Billing Aktionen, um zu erfahren, wie Sie diese Aktionen durch detaillierte Aktionen ersetzenAWS Billing, damit Sie Zugriff auf die AWS Kostenmanagement- und -Kontokonsolen haben. AWS

Wenn Sie Ihr AWS Konto oder AWS Organizations -Verwaltungskonto vor dem 6. März 2023 erstellt haben, werden die detaillierten Aktionen ab Juli 2023 wirksam. Wir empfehlen Ihnen, die detaillierten Aktionen hinzuzufügen, aber Ihre vorhandenen Berechtigungen mit aws-portal- oder purchase-orders-Präfixen nicht zu entfernen.

Wenn Sie Ihr AWS Konto oder AWS Organizations -Verwaltungskonto am oder nach dem 6. März 2023 erstellt haben, werden die detaillierten Aktionen sofort wirksam.

Es wird empfohlen, einen Administratorbenutzer für die Ausführung der täglichen Aufgaben und den Zugriff auf AWS Ressourcen AWS IAM Identity Center (successor to AWS Single Sign-On) zu konfigurieren. Sie können die unten aufgeführten Aufgaben aber nur ausführen, wenn Sie als Stammbenutzer eines Kontos angemeldet sind.

Aufgaben

  • Ändern Ihrer Kontoeinstellungen. Dazu gehören der Kontoname, die E-Mail-Adresse, das Passwort des Stammbenutzers und die Zugriffsschlüssel des Stammbenutzers. Andere Kontoeinstellungen wie Kontaktinformationen, bevorzugte Zahlungswährung usw. erfordern keine Stammbenutzer-Anmeldeinformationen. AWS-Regionen

  • Stellen Sie IAM-Benutzerberechtigungen wieder her. Wenn der einzige IAM-Administrator versehentlich die eigenen Berechtigungen widerruft, können Sie sich als Stammbenutzer anmelden, um Richtlinien zu bearbeiten und diese Berechtigungen wiederherzustellen.

  • Aktivieren Sie IAM-Zugriff auf die Konsole für Fakturierung und Kostenmanagement.

  • Anzeigen bestimmter Steuerrechnungen. Ein IAM-Benutzer mit der ViewBillingaws-portal:aws-portal:Berechtigung kann MwSt.-Rechnungen von AWS Europe anzeigen und herunterladen, aber nicht von AWS Inc oder Amazon Internet Services Private (AISPL).

  • Schließen Sie IhrAWS-Konto.

  • Sie haben sich im Reserved Instance Marketplace als Verkäufer registriert.

  • Konfigurieren Sie einen Amazon S3 S3-Bucket, um MFA (Multi-Faktor-Authentifizierung) zu aktivieren.

  • Bearbeiten oder löschen Sie eine Amazon Simple Storage Service (Amazon S3) -Bucket-Richtlinie, die eine ungültige Virtual Private Cloud (VPC) -ID oder VPC-Endpunkt-ID enthält.

  • Registrieren Sie sich für AWS GovCloud (US).

  • Fordern Sie die Zugriffsschlüssel für den Stammbenutzer eines AWS GovCloud (US) -Kontos anAWS Support.
