Geben Sie an, wasAWS-Regionen Ihr Konto verwenden kann

Wichtig

Die folgenden MaßnahmenAWS Identity and Access Management (IAM) enden im Juli 2023 mit dem Ende des Standardsupports:aws-portal:ModifyAccount undaws-portal:ViewAccount. Weitere Informationen finden Sie unter Verwenden vonAWS Billing detaillierten Aktionen, um diese Aktionen durch detaillierte Aktionen zu ersetzenAWS Billing, damit Sie Zugriff aufAWS Kostenmanagement undAWS Kontenkonsolen haben.

Wenn Sie IhrAWS Konto oderAWS Organizations Verwaltungskonto vor dem 6. März 2023 erstellt haben, werden die detaillierten Aktionen ab Juli 2023 wirksam. Wir empfehlen Ihnen, die detaillierten Aktionen hinzuzufügen, aber Ihre vorhandenen Berechtigungen mit aws-portal- oder purchase-orders-Präfixen nicht zu entfernen.

Wenn Sie IhrAWS Konto oder IhrAWS Organizations Verwaltungskonto am oder nach dem 6. März 2023 erstellt haben, werden die detaillierten Aktionen sofort wirksam.

AWSursprünglichAWS-Regionen standardmäßig aktiviert, sodass Ihre Benutzer Ressourcen in jeder Region erstellen konnten. Wenn jetzt eine RegionAWS hinzufügt, wird die neue Region standardmäßig deaktiviert. Falls Ihre Benutzer Ressourcen in einer neuen Region erstellen möchten, aktivieren Sie die Region.

Wichtig

AWSempfiehlt die Verwendung von regionalenAWS Security Token Service (AWS STS) -Endpunkte statt des globalen Endpunkts zu verwenden, um die Latenz zu reduzieren. Sitzungstoken von regionalenAWS STS -Endpunkten sind in allenAWS Regionen gültig. Wenn Sie regionaleAWS STS -Endpunkte verwenden, müssen Sie keine Änderungen vornehmen.

Sitzungstoken vom globalenAWS STS Endpunkt (https://sts.amazonaws.com) sind jedoch nur gültigAWS-Regionen, wenn Sie sie aktivieren oder die standardmäßig aktiviert sind. Wenn Sie eine neue Region für Ihr Konto aktivieren möchten, können Sie entweder Sitzungstoken von regionalenAWS STS -Endpunkten verwenden oder den globalenAWS STS Endpunkt aktivieren, um Sitzungstoken auszugeben, die für alle gültig sindAWS-Regionen. Sitzungstoken, die in allen Regionen gültig sind, sind größer. Wenn Sie Sitzungstoken speichern, können sich diese größeren Token auf Ihre Systeme auswirken.

Weitere Informationen zur Zusammenarbeit vonAWS STS Endpunkten mitAWS Regionen finden Sie unter AWS STSIn einerAWS Region verwalten.

Überlegungen zur Aktivierung und DeaktivierungAWS-Regionen

  • Sie können IAM-Berechtigungen zur Kontrolle des Zugriffs auf Regionen verwenden.

    AWS Identity and Access Management(IAM) enthält vier Berechtigungen, mit denen Sie kontrollieren können, welche Benutzer Regionen aktivieren, deaktivieren, abrufen und auflisten können. Weitere Informationen finden Sie unter Aktionsrichtlinien für Billing and Cost Management im AWS Billing and Cost ManagementBenutzerhandbuch.

  • Das Aktivieren einer Region ist kostenlos.

    Für das Aktivieren einer Region wird keine Gebühr erhoben. Berechnet werden Ihnen nur Ressourcen, die Ressourcen, die Sie in der neuen Region erstellen.

  • Durch Deaktivieren einer Region wird der Zugriff auf Ressourcen in der Region deaktiviert.

    Wenn Sie eine Region deaktivieren, die nochAWS Ressourcen enthält, wie etwa Amazon Elastic Compute Cloud (Amazon EC2) Instances, verlieren Sie den Zugriff auf die Ressourcen in dieser Region. Beispielsweise können Sie dieAWS Management Console oder eine andere programmatische Methode nicht verwenden, um die Konfiguration von EC2-Instances in einer deaktivierten Region anzuzeigen oder zu ändern.

  • Wenn du eine Region deaktivierst, fallen weiterhin Gebühren für aktive Ressourcen an.

    Wenn Sie eine Region deaktivieren, die nochAWS -Ressourcen enthält, fallen Gebühren für diese Ressourcen weiterhin zum Standardsatz an. Beispiel: Falls Sie eine Region deaktivieren, die Amazon-EC2-Instances enthält, müssen Sie dennoch Gebühren für diese Instances bezahlen, auch wenn diese Instances nicht mehr aufgerufen werden können.

  • Das Deaktivieren einer Region ist nicht immer sofort sichtbar

    Dienste und Konsolen sind möglicherweise vorübergehend sichtbar, nachdem eine Region deaktiviert wurde. Es kann einige Minuten bis mehrere Stunden dauern, bis die Deaktivierung einer Region wirksam wird.

  • Originalregionen sind standardmäßig aktiviert

    Regionen, die vor dem 20. März 2019 eingeführt wurden, sind alle standardmäßig aktiviert und können nicht deaktiviert werden. Weitere Informationen finden Sie unter Verwalten von AWS-Regionen im Allgemeine AWS-Referenz.

  • Das Aktivieren einer Region kann einige Minuten bis mehrere Stunden in Anspruch nehmen.

    Abhängig von mehreren Faktoren, wie der Größe Ihrer Organisation, kann die Aktivierung einer Region bis zu mehreren Stunden in Anspruch nehmen.

  • Organizations können innerhalb einerAWS Organisation 20 Anfragen zur Einreichung von Opt-Optionen für Regionen gleichzeitig stellen.

    Das Verwaltungskonto kann zu jedem Zeitpunkt 20 offene Anfragen für die Organisation haben, deren Abschluss noch aussteht. Eine Anfrage entspricht entweder der Aktivierung oder Deaktivierung einer bestimmten Region für ein Konto.

  • Für ein einzelnes Konto können 6 Region-Opt-Anfragen gleichzeitig bearbeitet werden.

    Eine Anfrage entspricht entweder der Aktivierung oder Deaktivierung einer bestimmten Region für ein Konto.

  • EventBridge Amazon-Integration

    Kunden können Benachrichtigungen zur Statusaktualisierung von Region-Opt unter abonnieren EventBridge. Für jede Statusänderung wird eine EventBridge Benachrichtigung erstellt, sodass Kunden Arbeitsabläufe automatisieren können.

  • Ausdrucksstarker Region-Opt-Status

    Aufgrund des asynchronen Charakters der Aktivierung/Deaktivierung einer Opt-In-Region gibt es vier mögliche Zustände für eine Region-Opt-Anfrage:

    • ENABLING

    • DISABLING

    • ENABLED

    • DISABLED

    Sie können ein Opt-In oder Opt-Out nicht stornieren, wenn es sich imDISABLING Status entwederENABLING oder befindet. AndernfallsConflictException wird ein geworfen. Eine abgeschlossene (aktivierte/deaktivierte) Region-Opt-Anfrage hängt von der Bereitstellung der wichtigsten zugrunde liegendenAWS Dienste ab. Möglicherweise gibt es einigeAWS Dienste, die trotz des Status nicht sofort nutzbar sindENABLED.

  • Vollständige Integration mitAWS Organizations

    Ein Verwaltungskonto kann Region-Opt für jedes Mitgliedskonto dieserAWS Organisation ändern oder lesen. Ein Mitgliedskonto kann auch den Status seiner Region lesen/schreiben.

Eine Region für eigenständige Konten aktivieren oder deaktivieren

Führen SieAWS-Konto die Schritte in den folgenden Verfahren aus, um zu aktualisieren, auf welche Regionen Sie Zugriff haben. Das folgendeAWS Management Console Verfahren funktioniert immer nur im Standalone-Kontext. Sie können den verwendenAWS Management Console, um nur die verfügbaren Regionen in dem Konto anzuzeigen oder zu aktualisieren, mit dem Sie den Vorgang aufgerufen haben.

AWS Management Console
So aktivieren oder deaktivieren Sie eine Region für ein Standalone-GerätAWS-Konto
Mindestberechtigungen

Um die Schritte im folgenden Verfahren ausführen zu können, benötigt ein IAM-Benutzer oder eine IAM-Rolle die folgenden Berechtigungen:

  • aws-portal:ViewAccount(erforderlich, um die Seite mit den Kontodetails aufzurufen)

  • account:ListRegions(erforderlich, um die Liste der zu sehenAWS-Regionen und festzustellen, ob sie derzeit aktiviert oder deaktiviert sind).

  • account:EnableRegion

  • account:DisableRegion

  1. Melden Sie sich entweder AWS Management Consoleals der oder als IAM-BenutzerRoot-Benutzer des AWS-Kontos oder als IAM-Rolle an, die über die Mindestberechtigungen verfügt.

  2. Wählen Sie oben rechts im Fenster Ihren Kontonamen und wählen Sie dann Konto.

  3. Scrollen Sie auf der Seite mit den Kontoeinstellungen nach unten zum Abschnitt AWS-Regionen.

    Anmerkung

    Sie werden möglicherweise aufgefordert, Ihren Zugriff auf diese Informationen zu genehmigen. AWSsendet eine Anfrage an die mit dem Konto verknüpfte E-Mail-Adresse und an die primäre Kontakttelefonnummer. Wählen Sie den Link in der Anfrage, um sie in Ihrem Browser zu öffnen, und genehmigen Sie den Zugriff.

  4. Wählen Sie neben jederAWS-Region Option in der Spalte Aktion entweder Aktivieren oder Deaktivieren aus, je nachdem, ob die Benutzer in Ihrem Konto Ressourcen in dieser Region erstellen und darauf zugreifen können.

  5. Wenn Sie dazu aufgefordert, bestätigen Sie Ihre Auswahl.

  6. Nachdem Sie alle Änderungen vorgenommen haben, wählen Sie Aktualisieren.

  • account:ListRegions

Wenn Sie diese individuellen Berechtigungen verwenden, können Sie einigen Benutzern die Möglichkeit geben, nur Informationen zu den Regionen zu lesen, und anderen die Möglichkeit geben, sowohl zu lesen als auch zu schreiben.

Das folgende Beispiel aktiviert eine Region für das angegebene Mitgliedskonto in einer Organisation. Die verwendeten Anmeldeinformationen müssen entweder vom Verwaltungskonto der Organisation oder vom delegierten Administratorkonto der Kontoverwaltung stammen.

Beachten Sie, dass Sie eine Region auch mit demselben Befehl deaktivieren und dann durchenable-region ersetzen könnendisable-region.

aws account enable-region --region-name af-south-1

Dieser Befehl erzeugt keine Ausgabe, wenn er erfolgreich ist.

Eine Region in Ihrer Organisation aktivieren oder deaktivieren

Führen Sie die Schritte in den folgenden Verfahren ausAWS Organizations, um die aktivierten Regionen für Ihr Mitgliedskonto zu aktualisieren.

Anmerkung

Bevor Sie diese Vorgänge über das Verwaltungskonto oder ein delegiertes Administratorkonto in einer Organisation zur Verwendung mit Mitgliedskonten ausführen können, müssen Sie:

  • Aktivieren Sie alle Funktionen in Ihrer Organisation, um die Einstellungen Ihrer Mitgliedskonten zu verwalten. Dies ermöglicht die Administratorkontrolle über die Mitgliedskonten. Dies ist standardmäßig festgelegt, wenn Sie Ihre Organisation erstellen. Wenn Ihre Organisation die Option „Nur konsolidierte Fakturierung“ eingestellt hat und Sie alle Funktionen aktivieren möchten, finden Sie weitere Informationen unter Aktivieren aller Funktionen in Ihrer Organisation.

  • Aktivieren Sie den vertrauenswürdigen Zugriff für denAWS Kontoverwaltungsdienst. Informationen zur Einrichtung finden Sie unterAktivieren des vertrauenswürdigen ZugriffsAWSKontenverwaltung.

Anmerkung

DieAWS Organizations verwalteten RichtlinienAWSOrganizationsReadOnlyAccess oderAWSOrganizationsFullAccess wurden aktualisiert, sodass Sie Zugriff auf dieAWS Kontoverwaltungs-APIs erhalten, sodass Sie von derAWS Organizations Konsole aus auf Kontodaten zugreifen können. Die aktualisierten verwalteten Richtlinien finden Sie unter Aktualisierungen derAWS verwalteten Richtlinien von Organizations.

AWS Management Console
So aktivieren oder deaktivieren Sie eine Region in Ihrer Organisation

  1. Melden Sie sich bei derAWS Organizations Konsole mit den Anmeldeinformationen Ihres Unternehmens an.

  2. Wählen Sie auf der AWS-KontenSeite das Konto aus, das Sie aktualisieren möchten.

  3. Wählen Sie den Tab Kontoeinstellungen.

  4. Wählen Sie unter Regionen die Region aus, die Region aus, die Sie aktivieren oder deaktivieren möchten.

  5. Wählen Sie Aktionen und dann entweder die Option Aktivieren oder Deaktivieren.

  6. Wenn Sie die Option Aktivieren ausgewählt haben, überprüfen Sie den angezeigten Text und wählen Sie dann Region aktivieren.

  7. Wenn Sie die Option „Deaktivieren“ ausgewählt haben, überprüfen Sie den angezeigten Text, geben Sie zur Bestätigung „Disable“ ein, und wählen Sie dann „Bereich deaktivieren“.
