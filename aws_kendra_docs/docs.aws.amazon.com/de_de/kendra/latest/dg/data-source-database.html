Amazon RDS/Aurora

Sie können Dokumente, die in einer Datenbank gespeichert sind, mithilfe einer Datenbankdatenquelle indizieren. Nachdem Sie Verbindungsinformationen für die Datenbank angegeben haben, werden DokumenteAmazon Kendra verbunden und indexiert.

Amazon Kendraunterstützt die folgenden Datenbanken:

  • Amazon Aurora MySQL

  • Amazon Aurora PostgreSQL

  • Amazon RDS for MySQL

  • Amazon RDS for PostgreSQL

Anmerkung

Serverlose Aurora-Datenbanken werden nicht unterstützt.

Sie können mithilfe der Amazon KendraKonsole und der DatabaseConfigurationAPI eine VerbindungAmazon Kendra zu Ihrer Datenbankdatenquelle herstellen.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra Datenbank-Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Unterstützte Funktionen

Amazon KendraDer -Daten-Quellen-Connector unterstützt die folgenden Funktionen:

  • Feldzuordnungen

  • Filterung des Benutzerkontextes

  • Virtual Private Cloud (VPC)

Voraussetzungen

Bevor Sie Ihre DatenquelleAmazon Kendra zur Indexierung verwenden können, nehmen Sie diese Änderungen in Ihrer Datenbank und IhrenAWS Konten vor.

In Ihrer Datenbank müssen Sie über

  • Ihr Benutzerkennwortpaar wurde als grundlegende Authentifizierungsdaten für Ihre Datenquelle notiert.

  • Kopierte den Hostnamen, die Portnummer, die Hostadresse und den Namen der Datentabelle, die die Datenbankdaten enthält. Für PostgreSQL muss die Datentabelle eine öffentliche Tabelle sein.

    Anmerkung

    Der Host und der Port gebenAmazon Kendra an, wo der Datenbankserver im Internet zu finden ist. Der Datenbankname und der Tabellenname geben an,Amazon Kendra wo die Dokumentdaten auf dem Datenbankserver zu finden sind.

  • Kopierte die Namen der Spalten in der Datentabelle, die die Dokumentdaten, die Dokument-ID, ein bis fünf Spalten, um festzustellen, ob ein Dokument geändert wurde, und optionale Datentabellenspalten, die benutzerdefinierten Indexfeldern zugeordnet sind, enthalten. Sie können jeden derAmazon Kendra reservierten Feldnamen einer Tabellenspalte zuordnen.

  • Kopierte die Typinformationen der Datenbank-Engine, z. B. ob Sie sieAmazon RDS für MySQL oder einen anderen Typ verwenden.

  • Geprüft, dass jedes Dokument in der Datenbank und in anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

In IhremAWS Konto müssen Sie über

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Ihre Anmeldeinformationen für die Datenbankauthentifizierung wurdenAWSSecrets Manager geheim gespeichert und, falls Sie die API verwenden, den ARN des Geheims notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder kein Geheimnis haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn Sie Ihre Datenbankdatenquelle mit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Um eine VerbindungAmazon Kendra zu Ihrer Datenbankdatenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer Datenbankdatenquelle angeben, damit diese auf Ihre Daten zugreifenAmazon Kendra kann. Wenn Sie die Datenbank fürAmazon Kendra see noch nicht konfiguriert habenVoraussetzungen.

Console

So stellen Sie eine VerbindungAmazon Kendra zu einer Datenbank her

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich die Option Indizes und dann in der Liste der Indizes aus.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option Datenbankconnector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite -Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. In Tags für Neues Tag hinzufügen —Tags, um Ressourcen zu suchen und zu filtern oder IhreAWS -Kosten zu verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit die folgenden Informationen ein:

    1. Endpunkt — Ein DNS-Hostname, eine IPv4-Adresse oder eine IPv6-Adresse.

    2. Port — Eine Portnummer.

    3. Datenbank — Datenbankname.

    4. Tabellenname —Tabellenname.

    5. Wählen Sie unter Authentifizierungstyp zwischen Bestehend und Neu, um Ihre Datenbankauthentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-database-' wird automatisch zu Ihrem geheimen Namen hinzugefügt.

        2. Für Benutzername und Passwort — Geben Sie die Werte für die Authentifizierungsdaten aus Ihrem Datenbankkonto ein.

        3. Wählen Sie Authentifizierung speichern.

    6. Virtual Private Cloud (VPC) — Sie können eine VPC verwenden. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

      Anmerkung

      Sie müssen ein privates Subnetz verwenden. Wenn sich Ihre RDS-Instance in einem öffentlichen Subnetz in Ihrer VPC befindet, können Sie ein privates Subnetz erstellen, das ausgehenden Zugriff auf ein NAT-Gateway im öffentlichen Subnetz hat. Die in der VPC-Konfiguration bereitgestellten Subnetze müssen sich entweder in USA West (Oregon), USA Ost (Nord-Virginia) und EU (Irland) befinden.

    7. IAM-Rolle — Wählen Sie eine vorhandene IAM-Rolle aus, oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Daten und -Inhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    8. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite die folgenden Informationen ein:

    1. Wählen Sie je nach Anwendungsfall zwischen Aurora MySQL, MySQL, Aurora PostgreSQL und PostgreSQL.

    2. SQL-Bezeichner in doppelte Anführungszeichen setzen — Wählen Sie diese Option, um SQL-Bezeichner in doppelte Anführungszeichen zu setzen. Zum Beispiel „ColumnName“.

    3. ACL-Spalte und Spalten zur Erkennung von Änderungen — Hiermit konfigurieren Sie die Spalten, die für Änderungserkennungs- und ZugriffskontrolllistenAmazon Kendra verwendet werden.

    4. Wählen Sie im Synchronisierungslaufzeitplan für Häufigkeit aus, wie oft mit Ihrer Datenquelle synchronisiertAmazon Kendra werden soll.

    5. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen ein:

    1. Amazon KendraStandardfeldzuordnungen — Wählen Sie eines derAmazon Kendra generierten Standarddatenquellenfelder aus, die Sie Ihrem Index zuordnen möchten. Sie müssen die Datenbank-Spaltenwerte fürdocument_id und hinzufügendocument_body

    2. Benutzerdefinierte Feldzuordnungen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

So stellen Sie eine VerbindungAmazon Kendra zu einer Datenbank her

Sie müssen die folgende DatabaseConfigurationAPI angeben:

  • ColumnConfiguration—Informationen darüber, wo der Index die Dokumentinformationen aus der Datenbank abrufen soll. Weitere Details finden Sie unter ColumnConfiguration. Sie müssen dieDocumentIdColumnName FelderDocumentDataColumnName und angeben. Die demDocumentIdColumnName Feld zugeordnete Spalte muss eine Ganzzahlspalte sein. Das folgende Beispiel zeigt eine einfache Spaltenkonfiguration für eine Datenquelle: 

    "ColumnConfiguration": {
    "ChangeDetectingColumns": [
        "LastUpdateDate",
        "LastUpdateTime"
    ],
    "DocumentDataColumnName": "TextColumn",
    "DocumentIdColumnName": "IdentifierColumn",
    "DocoumentTitleColumnName": "TitleColumn",
    "FieldMappings": [
        {
            "DataSourceFieldName": "AbstractColumn",
            "IndexFieldName": "Abstract"
        }
    ]
}

  • ConnectionConfiguration— Der Typ des Datenbank-Engine, das die Datenbank ausführt. DasDatabaseHost Feld muss derAmazon Relational Database Service (Amazon RDS) Instanzendpunkt für die Datenbank sein. Verwenden Sie nicht den Cluster-Endpunkt.

  • DatabaseEngineType— Konfigurationsinformationen, die für die Verbindung mit einer Datenbank erforderlich sind. Weitere Details finden Sie unter ConnectionConfiguration.

  • Geheimer Amazon Resource Name (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr Datenbankkonto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "username": "user name",
    "password": "password"
}

    Das Geheimnis kann mehr Informationen enthalten. Das folgende Beispiel zeigt eine Datenbankkonfiguration.

    
"DatabaseConfiguration": {
"ConnectionConfiguration": {
"DatabaseHost": "host.subdomain.domain.tld",
        "DatabaseName": "DocumentDatabase",
        "DatabasePort": 3306,
        "SecretArn": "arn:aws:secretmanager:region:account ID:secret/secret name",
        "TableName": "DocumentTable"
    }
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle —Geben Sie an,RoleArn wann Sie aufrufenCreateDataSource, um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager geheimes Element und zum Aufrufen der erforderlichen öffentlichen APIs für den Datenbankconnector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für Datenquellen.

Sie können auch die folgenden Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie dies imVpcConfiguration Rahmen der Datenquellenkonfiguration an. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

    Anmerkung

    Sie dürfen nur ein privates Subnetz verwenden. Wenn sich Ihre RDS-Instance in einem öffentlichen Subnetz in Ihrer VPC befindet, können Sie ein privates Subnetz erstellen, das ausgehenden Zugriff auf ein NAT-Gateway im öffentlichen Subnetz hat. Die in der VPC-Konfiguration bereitgestellten Subnetze müssen sich entweder in USA West (Oregon), USA Ost (Nord-Virginia) und EU (Irland) befinden.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre Datenbank-Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

  • BenutzerkontextfilterungAmazon Kendra — durchsucht standardmäßig die Access Control List (ACL) für Ihre Datenquelle. Die ACL-Informationen werden verwendet, um Suchergebnisse auf der Grundlage des Zugriffs des Benutzers oder seiner Gruppe auf Dokumente zu filtern. Weitere Informationen finden Sie unter Benutzerkontextfilterung für Datenbankdatenquellen.
