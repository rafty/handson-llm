Erstellen benutzerdefinierter Dokumentfelder

Sie können benutzerdefinierte Attribute oder Metadatenfelder anwenden, die für Ihre speziellen Dokumente relevant sind. Sie können beispielsweise ein benutzerdefiniertes Feld oder Attribut namens „Abteilung“ mit den Werten „HR“, „Vertrieb“ und „Fertigung“ erstellen. Sie können diese Felder oder Attribute verwenden, um die Antwort (die Suchergebnisse) beispielsweise auf Dokumente in der Abteilung „HR“ zu beschränken.

Sie können bis zu 500 benutzerdefinierte Felder oder Attribute erstellen.

Bei den meisten Datenquellen ordnen Sie Felder in der externen Datenquelle den entsprechenden Feldern in zuAmazon Kendra. Weitere Informationen finden Sie unter Zugewiesene Datenquellen-Felder. Für S3-Datenquellen können Sie benutzerdefinierte Felder oder Attribute mithilfe von Metadatendateien anwenden.

Bevor Sie ein benutzerdefiniertes Feld oder Attribut verwenden können, müssen Sie das Feld zuerst im Index verwenden können. Verwenden Sie die Konsole oder die UpdateIndexAPI, um das Indexfeld zu erstellen. Sie können den Felddatentyp nicht mehr ändern, nachdem Sie das Feld erstellt haben.

Sie können auch die integrierten oder reservierten Felder verwendenAmazon Kendra.

Mit derUpdateIndex API fügen Sie mithilfe desDocumentMetadataConfigurationUpdates Parameters benutzerdefinierte Felder oder Attribute hinzu.

Das folgende JSON-Beispiel verwendetDocumentMetadataConfigurationUpdates, um dem Index ein Feld mit dem Namen „Department“ hinzuzufügen.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Verwenden der integrierten Dokumentfelder von Amazon Kendra

Mit der UpdateIndex API können Sie reservierte oder integrierte Felder erstellen, indem Sie den reservierten Feldnamen verwendenDocumentMetadataConfigurationUpdates und angeben, um ihn Ihrem entsprechenden Dokumentattribut oder Dokumentfeld zuzuordnen. Auf diese Weise können Sie auch benutzerdefinierte Felder erstellen. Wenn Sie einen Datenquellenconnector verwenden, enthalten die meisten Feldzuordnungen, die Ihre DatenquellendokumentfelderAmazon Kendra Indexfeldern zuordnen. Wenn Sie ein Feld erstellen, können Sie dasSearch Objekt so konfigurieren, dass das Feld als anzeigbar, facettierbar, durchsuchbar und sortierbar festgelegt wird. Sie können dasRelevance Objekt so konfigurieren, dass die Werte für die Dauer, Aktualität, Wichtigkeit, Rangfolge und Wichtigkeit des Felds zur Steigerung der Relevanz festgelegt werden. Sie können den Feldtyp nicht mehr ändern, nachdem Sie das Feld erstellt haben.

Amazon Kendraverfügt über die folgenden reservierten oder integrierten Dokumentfelder, die Sie verwenden können:

  • _authors— Eine Liste von einem oder mehreren Autoren, die für den Inhalt des Dokuments verantwortlich sind.

  • _category— Eine Kategorie, die ein Dokument in eine bestimmte Gruppe einordnet.

  • _created_at— Datum und Uhrzeit im ISO 8601-Format, zu denen das Dokument erstellt wurde. Beispiel: 2012-03-25T12:30:10+01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in mitteleuropäischer Zeit (CET).

  • _data_source_id— Der Bezeichner der Datenquelle, die das Dokument enthält.

  • _document_body— Der Inhalt des Dokuments.

  • _document_id— Eine eindeutige Kennung für das Dokument.

  • _document_title— Der Titel des Dokuments.

  • _excerpt_page_number— Die Seitenzahl in einer PDF-Datei, auf der der Dokumentauszug erscheint. Wenn Ihr Index vor dem 8. September 2020 erstellt wurde, müssen Sie Ihre Dokumente neu indizieren, bevor Sie dieses Attribut verwenden können.

  • _faq_id—Wenn dies eine häufig gestellte Frage und Antwort ist, eine eindeutige Kennung für sie.

  • _file_type— Der Dateityp des Dokuments, z. B. PDF oder Dokument.

  • _last_updated_at— Datum und Uhrzeit im ISO-8601-Format, zu der das Dokument zuletzt aktualisiert wurde. Beispiel: 2012-03-25T12:30:10+01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in mitteleuropäischer Zeit (CET).

  • _source_uri— Die URI, unter der das Dokument verfügbar ist. Zum Beispiel die URI des Dokuments auf einer Unternehmenswebsite.

  • _version— Ein Bezeichner für die spezifische Version eines Dokuments.

  • _view_count— Wie oft das Dokument angesehen wurde.

  • _language_code(String) — Der Code für eine Sprache, die für das Dokument gilt. Standardmäßig verwendet Englisch, wenn Sie keine Sprache angeben. Weitere Informationen zu unterstützten Sprachen, einschließlich ihrer Codes, finden Sie unter Hinzufügen von Dokumenten in anderen Sprachen als Englisch.

Für benutzerdefinierte Felder erstellen Sie diese FelderDocumentMetadataConfigurationUpdates mithilfe derUpdateIndex API, genau wie beim Erstellen eines reservierten oder integrierten Felds. Sie müssen den entsprechenden Datentyp für Ihr benutzerdefiniertes Feld festlegen. Sie können den Feldtyp nicht mehr ändern, nachdem Sie das Feld erstellt haben.

Die folgenden Typen können Sie für benutzerdefinierte Felder festlegen:

  • Datum

  • Zahl

  • Zeichenfolge

  • Zeichenfolgenliste

Sie erstellen ein benutzerdefiniertes Feld mithilfe der Konsole oder mithilfe der UpdateIndexAPI. Nachdem Sie ein benutzerdefiniertes Feld erstellt haben, ordnen Sie es einem Dokumentattribut zu, genau wie Sie es bei einem reservierten Feld tun. Wenn Sie ein Dokument mit BatchPutDocumentAPI zum Index hinzugefügt haben, ordnen Sie die Attribute der API zu. Für Dokumente, die aus einerAmazon S3 Datenquelle indexiert wurden, ordnen Sie die Attribute mithilfe einer Metadatendatei zu, die eine JSON-Struktur enthält, die die Dokumentattribute beschreibt. Für Dokumente, die mit einer Datenbank oder einer Datenquelle indexiert sind, die eine Feldzuordnung ermöglicht, ordnen Sie Attribute der Konsole oder der Datenquellenkonfiguration zu.

Hinzufügen von benutzerdefinierten Attributen oder Feldern mit der BatchPutDocument API

Wenn Sie dieBatchPutDocument API verwenden, um Ihrem Index ein Dokument hinzuzufügen, geben Sie benutzerdefinierte Felder oder Attribute als Teil von anAttributes. Sie können mehrere Felder oder Attribute hinzufügen, wenn Sie die API aufrufen. Sie können bis zu 500 benutzerdefinierte Felder oder Attribute erstellen. Das folgende Beispiel ist ein benutzerdefiniertes Feld oder Attribut, das einem Dokument „Abteilung“ hinzufügt.

"Attributes": 
    {
        "Department": "HR",
        "_category": "Vacation policy"
    }

Hinzufügen von benutzerdefinierten Attributen oder Feldern zu einerAmazon S3 Datenquelle

Wenn Sie einen S3-Bucket als Datenquelle für Ihren Index verwenden, fügen Sie den Dokumenten Metadaten mit zugehörigen Metadatendateien hinzu. Sie platzieren die JSON-Metadatendateien in einer Verzeichnisstruktur, die parallel zu Ihren Dokumenten verläuft. Weitere Informationen finden Sie unter Amazon S3Metadaten des Dokuments.

Sie geben benutzerdefinierte Felder oder Attribute in derAttributes JSON-Struktur an. Sie können bis zu 500 benutzerdefinierte Felder oder Attribute erstellen. Im folgenden Beispiel werden beispielsweise dreiAttributes benutzerdefinierte Felder oder Attribute und ein reserviertes Feld definiert.

"Attributes": {
        "brand": "Amazon Basics",
        "price": 1595,
        "_category": "sports",
        "subcategories": ["outdoors", "electronics"]
    }
