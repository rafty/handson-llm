Optimieren von Suchresultaten

Amazon KendraAbfragen führen zu Suchergebnissen, die nach ihrer Relevanz geordnet sind. Die durchsuchbaren Felder oder Attribute im Index tragen alle zu diesem Ranking bei.

Sie können die Auswirkung eines Felds oder Attributs auf die Suchrelevanz durch Relevanzoptimierung ändern. Die Optimierung der Suchrelevanz kann entweder manuell auf Indexebene erfolgen, wo Sie Optimierungskonfigurationen für Ihren Index festlegen, oder auf Abfrageebene, indem Sie die auf Indexebene festgelegten Konfigurationen überschreiben.

Wenn Sie die Optimieren von Relevanz haben, erhalten die Ergebnisse einen Reaktionsschub. Sie geben auch an, wie viel Boost das Dokument erhält, wenn es eine Übereinstimmung gibt. Die Relevanzoptimierung führt nichtAmazon Kendra dazu, dass ein Dokument in die Abfrageantwort aufgenommen wird. Sie ist nur einer der Faktoren,Amazon Kendra anhand derer die Relevanz eines Dokuments bestimmt wird.

Sie können bestimmte Felder oder Attribute in Ihrem Index hervorheben, um bestimmten Antworten mehr Bedeutung zuzuweisen. Zum Beispiel, wenn jemand nach „Wann ist re:Invent?“ sucht. Sie könnten die Relevanz der Aktualität von Dokumenten im Feld „_last_update_at“ erhöhen. Oder Sie könnten in einem Index von Forschungsberichten eine bestimmte Datenquelle im Feld „Quelle“ hervorheben.

Sie können Dokumente auch auf der Grundlage von Stimmen oder der Anzahl der Aufrufe aktualisieren, was in Foren und anderen Support-Wissensdatenbanken üblich ist. Sie können Boosts kombinieren, um beispielsweise Dokumente zu verbessern, die häufiger und aktueller sind.

Mithilfe desImportance Parameters legen Sie fest, wie viel Boost ein Dokument erhält. Je höher der WertImportance, desto mehr erhöht das Feld oder Attribut die Relevanz eines Dokuments. Wenn Sie Ihren Index optimieren oder auf Abfrageebene optimieren, erhöhen Sie den Wert desImportance Parameters in kleinen Schritten, bis Sie den gewünschten Effekt erzielen. Um festzustellen, ob Sie die Suchergebnisse verbessern, führen Sie die Suche durch und vergleichen Sie die Ergebnisse mit früheren Abfragen.

Sie können Datums-, Zahlen- oder Zeichenfolgenattribute angeben, um einen Index zu optimieren oder auf Abfrageebene zu optimieren. Sie können Felder oder Attribute, die diesen Typ haben,StringList nur auf Indexebene optimieren. Jedes Feld oder Attribut hat spezifische Kriterien dafür, wann es ein Ergebnis verbessert.

  • Datumsfelder oder Attribute —Es gibt drei spezifische Kriterien für DatumsfelderDuration,Freshness undRankOrder.

    • Durationlegt den Zeitraum fest, für den der Schub gilt. Wenn Sie den Zeitraum beispielsweise auf 86400 Sekunden (d. h. einen Tag) festlegen, beginnt der Boost nach einem Tag nachzulassen. Je höher die Bedeutung, desto schneller fällt der Schub ab.

    • Freshnessbestimmt, wie aktuell ein Dokument ist, wenn es auf ein Feld oder Attribut angewendet wird. Wenn Sie sich entwederFreshness auf das Feld für das Erstellungsdatum oder das Datum der letzten Aktualisierung bewerben, gilt ein kürzlich erstelltes oder zuletzt aktualisiertes Dokument als „aktueller“ als ein älteres Dokument. Beispiel: Wenn Dokument 1 am 14. November und Dokument 2 am 5. November erstellt wurde, ist Dokument 1 „frischer“ als Dokument 2. Und wenn Dokument 1 am 14. November und Dokument 2 zuletzt am 20. November aktualisiert wurde, ist Dokument 2 „frischer“ als Dokument 1. Je frischer das Dokument ist, desto mehr wird dieser Boost angewendet. Sie können nur einFreshness Feld in Ihrem Index haben.

    • RankOrderwendet den Schub entweder in auf- oder absteigender Reihenfolge an. Wenn Sie angebenASCENDING, haben spätere Daten Vorrang. Wenn Sie angebenDESCENDING, haben frühere Daten Vorrang.

  • Zahlenfelder oder Attribute — Für Zahlenfelder oder Attribute können Sie die Rangfolge angeben, die bei der Bestimmung der Relevanz des Felds oder Attributs verwendet werdenAmazon Kendra soll. Wenn Sie angebenASCENDING, haben höhere Zahlen Vorrang. Wenn Sie angebenDESCENDING, haben niedrigere Zahlen Vorrang.

  • Zeichenfolgenfelder oder Attribute — Für Zeichenfolgenfelder oder Attribute können Sie Kategorien eines Felds erstellen, um jeder Kategorie einen anderen Schub zu geben. Wenn Sie beispielsweise ein Feld oder Attribut mit dem Namen „Abteilung“ aufwerten, können Sie Dokumenten aus „HR“ einen anderen Boost geben als Dokumenten aus „Recht“. Sie können ein Feld oder Attribut des Typs boostenString. Sie könnenStringList Felder nur auf Indexebene boosten.

Relevanzoptimierung auf Indexebene

Sie optimieren die Relevanz eines Felds oder Attributs auf Indexebene, indem Sie entweder die Konsole verwenden, um die Optimierung in den Indexdetails festzulegen, oder die UpdateIndexAPI.

Im folgenden Beispiel wird das Feld „_last_updated_at“ alsFreshness Feld für ein Dokument festgelegt.

"DocumentMetadataConfigurationUpdates" : [
    {
        "Name": "_last_updated_at",
        "Type": "DATE_VALUE",
        "Relevance": {
            "Freshness": TRUE,
            "Importance": 2
        }
    }
]

Im folgenden Beispiel wird den verschiedenen Kategorien im Feld „Abteilung“ unterschiedliche Bedeutung beigemessen.

"DocumentMetadataConfigurationUpdates" : [
    {
        "Name": "department",
        "Type": "STRING_VALUE",
        "Relevance": {
            "Importance": 2,
            "ValueImportanceMap": {
                "HR": 3,
                "Legal": 1
            }
        }
    }
]

Relevanzoptimierung auf Abfrageebene

Mithilfe der Abfrage-API optimieren Sie die Relevanz eines Felds oder Attributs auf Abfrageebene.

Die Relevanzoptimierung auf Abfrageebene wird in der Konsole nicht unterstützt.

Die Optimierung auf Abfrageebene kann den Prozess der Testrelevanzoptimierung beschleunigen, da Sie die Optimierungskonfigurationen im Index nicht für jeden Test manuell aktualisieren müssen. Sie können die Relevanz eines Dokuments optimieren, indem Sie Optimierungskonfigurationen in der Abfrage übergeben. Dann können Sie die verschiedenen Ergebnisse sehen, die Sie aus verschiedenen Konfigurationen erhalten. Eine Konfiguration, die in der Abfrage übergeben wird, hat Vorrang vor der Konfiguration, die auf Indexebene festgelegt ist.

Das folgende Beispiel überschreibt die Wichtigkeit, die dem Feld „Abteilung“ und jeder Abteilungskategorie auf Indexebene zugewiesen wurde, wie im obigen Beispiel gezeigt. Wenn ein Benutzer seine Suchanfrage eingibt, hat das Feld „Abteilung“ ein angemessenes Maß an Wichtigkeit und die Rechtsabteilung hat eine höhere Bedeutung als die Personalabteilung.

"DocumentRelevanceOverrideConfigurations" : [
    {
        "Name": "department",
        "Type": "STRING_VALUE",
        "Relevance": {
            "Importance": 2,
            "ValueImportanceMap": {
                "HR": 2,
                "Legal": 8
            }
        }
    }
]
