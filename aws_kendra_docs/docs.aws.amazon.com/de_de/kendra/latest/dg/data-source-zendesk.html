Zendesk

Zendesk ist ein Kundenbeziehungsmanagementsystem, das Unternehmen dabei unterstützt, Interaktionen mit dem Kundensupport zu automatisieren und zu verbessern. Sie könnenAmazon Kendra damit Ihre Zendesk-Supporttickets, Ticketkommentare, Ticketanhänge, Help-Center-Artikel, Artikelkommentare, Anhänge zu Artikelkommentaren, Leitfaden-Community-Themen, Community-Posts und Kommentare zu Community-Posts indexieren.

Sie können nach dem Namen der Organisation filtern, wenn Sie Tickets indexieren möchten, die sich nur innerhalb einer bestimmten Organisation befinden. Sie können auch ein Crawldatum festlegen, an dem Sie mit dem Crawlen von Daten aus Zendesk beginnen möchten.

Sie können mithilfe der Amazon KendraKonsole und der TemplateConfigurationAPI eine VerbindungAmazon Kendra zu Ihrer Zendesk-Datenquelle herstellen.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra Zendesk-Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Unterstützte Funktionen

Amazon KendraDer Zendesk-Daten-Quell-Connector unterstützt die folgenden Funktionen:

  • Änderungsprotokoll

  • Feldkartierung

  • Filterung des Benutzerkontextes

  • Einschluss-/Ausschlussfilter

  • Virtual Private Cloud (VPC)

Voraussetzungen

Bevor Sie Ihre Zendesk-DatenquelleAmazon Kendra zur Indexierung verwenden können, nehmen Sie diese Änderungen in Ihrem Zendesk und IhrenAWS Konten vor.

Stellen Sie in Zendesk über Folgendes verfügen:

  • Hat ein Zendesk Suite-Administratorkonto (Professional/Enterprise) erstellt.

  • Ich habe Ihre Zendesk-Host-URL notiert. Zum Beispiel https://{sub-domain (https://{host/)}.zendesk.com/.

  • Generierte ein OAuth 2.0-Anmeldungstoken, das eine Client-ID, ein Client-Geheimnis, einen Benutzernamen und ein Passwort enthält. Weitere Informationen finden Sie in der Zendesk-Dokumentation zur Generierung von OAuth 2.0-Token.

  • Der folgende OAuth 2.0-Bereich wurde hinzugefügt:

    • read

  • Optional: Es wurde ein SSL-Zertifikat installiert,Amazon Kendra um eine Verbindung herzustellen.

  • Geprüft, dass jedes Dokument in Zendesk und in anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

Stellen Sie in IhremAWS Konto sicher, dass Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Ihre Zendesk-Authentifizierungsdaten wurdenAWSSecrets Manager geheim gespeichert und, falls Sie die API verwenden, den ARN des Geheims notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder kein Geheimnis haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn Sie Ihre Zendesk-Datenquelle damit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Anweisungen zum Anschluss

Um eine VerbindungAmazon Kendra zu Ihrer Zendesk-Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer Zendesk-Datenquelle angeben, damit diese auf Ihre Daten zugreifenAmazon Kendra kann. Falls Sie Zendesk noch nicht konfiguriert haben,Amazon Kendra sehen Sie sich das anVoraussetzungen.

Console

So stellen Sie eine VerbindungAmazon Kendra zu Zendesk her

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich Indizes und dann in der Liste der Indizes den Index aus, den Sie verwenden möchten.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option Zendesk-Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite -Details die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Fügen Sie unter Tags für Neues Tag hinzufügen —Tags ein, um Ressourcen zu suchen und zu filtern oder IhreAWS -Kosten zu verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. Zendesk-URL —Geben Sie Ihre Zendesk-URL ein.

    2. AWSSecrets Managersecret — Wählen Sie ein vorhandenes Geheimnis aus, oder erstellen Sie ein neuesSecrets Manager Geheimnis, um Ihre Zendesk-Authentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-Zendesk-' wird automatisch zu Ihrem geheimen Namen hinzugefügt.

        2. Geben Sie für Client-ID, Client-Geheimnis, Benutzername, Passwort die Werte der Authentifizierungsdaten ein, die Sie in Ihrem Zendesk-Konto erstellt haben.

      2. Wählen Sie Speichern.

    3. Virtual Private Cloud (VPC) — Sie können auswählen, ob Sie eine VPC verwenden möchten. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    4. IAM-Rolle —Wählen Sie eine vorhandene IAM-Rolle oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und -Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    5. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Wählen Sie Entitäten oder Inhaltstypen aus — Die Zendesk-Entitäten oder Inhaltstypen, die Sie crawlen möchten.

    2. Änderungsprotokoll — Wählen Sie diese Option aus, um Ihren Index zu aktualisieren, anstatt alle Ihre Dateien zu synchronisieren.

    3. Organisationsname — Geben Sie die Namen der Zendesk-Organisation ein, um Ihre Synchronisation zu filtern.

    4. Startdatum synchronisieren — Das Datum, ab dem Sie Ihre Inhalte indexieren möchten.

    5. Regex-Muster — Reguläre Ausdrucksmuster zum Ein- oder Ausschließen bestimmter Dateien. Sie können bis zu 100 Muster hinzufügen.

    6. Wählen Sie unter Synchronisierungslaufzeitplan für Frequenz aus, wie oftAmazon Kendra die Synchronisierung mit Ihrer Datenquelle erfolgen soll.

    7. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuweisungen festlegen die folgenden Informationen ein:

    1. Für Tickets, Ticketkommentar, Ticketkommentaranhang, Artikel, Artikelkommentaranhang, Community-Thema, Community-Beitrag, Community-Post-Kommentar — Wählen Sie eines derAmazon Kendra generierten Standard-Datenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können auch festlegen, dass Sie Ihre Informationen von dieser Seite aus bearbeiten möchten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

So stellen Sie eine VerbindungAmazon Kendra zu Zendesk her

Sie müssen einen JSON des Datenquellenschemas mithilfe der TemplateConfigurationAPI angeben. Sie müssen die folgenden Informationen angeben:

  • Datenquelle —Geben Sie die Datenquelle an alsZENDESK.

  • Host-URL — Geben Sie Ihre Zendesk-Host-URL als Teil der Verbindungskonfiguration oder der Repository-Endpunktdetails an. Zum Beispiel https://yoursubdomain.zendesk.com.

  • Typ —Geben SieTEMPLATE dies als Typ an, wenn Sie anrufenCreateDataSource.

  • Geheimer Amazon Resource Name (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr Zendesk-Konto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "hostUrl": "https://yoursubdomain.zendesk.com",
    "clientId": "kendra",
    "clientSecret": "Zendesk client secret",
    "userName": "Zendesk user name",
    "password": "Zendesk password"
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager geheimes Element und zum Aufrufen der erforderlichen öffentlichen APIs für den Zendesk-Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für Zendesk-Datenquellen.

Sie können auch die folgenden optionalen Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie anVpcConfiguration, wann Sie anrufenCreateDataSource. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

  • Änderungsprotokoll — Gibt an, ob der Change-Log-Mechanismus für die Zendesk-Datenquelle verwendet werdenAmazon Kendra soll, um zu bestimmen, ob ein Dokument dem Index hinzugefügt, aktualisiert oder gelöscht werden muss.

    Anmerkung

    Verwenden Sie das Änderungsprotokoll, wenn Sie nicht alle Dokumente scannen möchtenAmazon Kendra. Wenn Ihr Änderungsprotokoll groß ist, dauert das Scannen der Dokumente in der Zendesk-Datenquelle möglicherweiseAmazon Kendra weniger Zeit als das Verarbeiten des Änderungsprotokolls. Wenn Sie Ihre Zendesk-Datenquelle zum ersten Mal mit Ihrem Index synchronisieren, werden alle Dokumente gescannt.

  • Ein- und Ausschlussfilter — Geben Sie an, ob Folgendes ein- oder ausgeschlossen werden soll:

    • Supporttickets, Ticketkommentare und/oder Anhänge zu Ticketkommentaren

    • Hilfecenter-Artikel, Artikelanhänge und Artikelkommentare

    • Leite Community-Themen, Beiträge oder verfasse Kommentare

  • Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre Zendesk-Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

  • BenutzerkontextfilterungAmazon Kendra — durchsucht standardmäßig die Access Control List (ACL) für Ihre Datenquelle. Die ACL-Informationen werden verwendet, um Suchergebnisse auf der Grundlage des Zugriffs des Benutzers oder seiner Gruppe auf Dokumente zu filtern. Weitere Informationen finden Sie unter Benutzerkontextfilterung für Zendesk-Datenquellen.

Eine Liste weiterer wichtiger zu konfigurierender JSON-Schlüssel finden Sie unterZendesk-Vorlagenschema.
