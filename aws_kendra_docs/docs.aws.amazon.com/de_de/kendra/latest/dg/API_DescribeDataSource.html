DescribeDataSource

Ruft Informationen über einen Amazon Kendra Datenquellenkonnektor bereit.

Anforderungssyntax

{
   "Id": "string",
   "IndexId": "string"
}

Anfrageparameter

Informationen zu den Parametern, die allen Aktionen gemeinsam sind, finden Sie unter Allgemeine Parameter.

Die Anforderung akzeptiert die folgenden Daten im JSON-Format.

Id

Der Bezeichner Datenquellenkonnektor.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 100 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Erforderlich: Ja

IndexId

Der Bezeichner des Index, der für den Datenquellenkonnektor verwendet wird.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge von 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Erforderlich: Ja

Antwortelemente

Wenn die Aktion erfolgreich ist, sendet der Service eine HTTP 200-Antwort zurück.

Die folgenden Daten werden vom Service im JSON-Format zurückgegeben.

Configuration

Konfigurationsinformationen für den Datenquellenkonnektor. Dies zeigt, wie die Datenquelle konfiguriert ist. Die Konfigurationsoptionen für eine Datenquelle hängen vom Datenquellenanbieter ab.

Typ: DataSourceConfiguration Objekt

CreatedAt

Der Unix-Zeitstempel, als der Datenquellenconnector erstellt wurde.

Typ: Zeitstempel

CustomDocumentEnrichmentConfiguration

Konfigurationsinformationen zum Ändern der Metadaten und des Inhalts von Dokumenten während der Dokumenten-Erfassung.

Weitere Informationen zum Erstellen, Ändern und Löschen von Dokumentmetadaten oder zum Vornehmen anderer Inhaltsänderungen bei der Aufnahme von Dokumenten in Amazon Kendra finden Sie unter Anpassen von Dokumentmetadaten während des Erfassungsprozesses.

Typ: CustomDocumentEnrichmentConfiguration Objekt

Description

Die Beschreibung für den Datenquellenkonnektor.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Höchstlänge = 1 000 Zeichen.

Pattern: ^\P{C}*$

ErrorMessage

Wenn derStatus Feldwert istFAILED, enthält dasErrorMessage Feld eine Beschreibung des Fehlers, der zum Ausfall der Datenquelle geführt hat.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Maximale Länge beträgt 2048 Zeichen.

Pattern: ^\P{C}*$

Id

Der Bezeichner Datenquellenkonnektor.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 100 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

IndexId

Der Bezeichner des Index, der für den Datenquellenkonnektor verwendet wird.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge von 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

LanguageCode

Der Code für eine Sprache. Dies zeigt eine unterstützte Sprache für alle Dokumente in der Datenquelle. Englisch wird standardmäßig unterstützt. Weitere Informationen zu unterstützten Sprachen, einschließlich ihrer Codes, finden Sie unter Hinzufügen von Dokumenten in anderen Sprachen als Englisch.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 2. Maximale Länge von 10.

Pattern: [a-zA-Z-]*

Name

Der Name für den Datenquellenkonnektor.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 1000 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

RoleArn

Der Amazon-Ressourcenname (ARN) der Rolle mit der Berechtigung zum Zugriff auf die Datenquelle und die erforderlichen Ressourcen.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Höchstlänge = 1284 Zeichen.

Pattern: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Schedule

Der Zeitplan für Amazon Kendra zur Aktualisierung des Index.

Typ: Zeichenfolge

Status

Der aktuelle Status des Datenquellenkonnektor. Wenn der Status hat, kannACTIVE die Datenquelle verwendet werden. Wenn der Status lautetFAILED, enthält dasErrorMessage Feld den Grund für den Ausfall der Datenquelle.

Typ: Zeichenfolge

Zulässige Werte: CREATING | DELETING | FAILED | UPDATING | ACTIVE

Type

Der Typ der Datenquelle. Beispiel, SHAREPOINT.

Typ: Zeichenfolge

Zulässige Werte: S3 | SHAREPOINT | DATABASE | SALESFORCE | ONEDRIVE | SERVICENOW | CUSTOM | CONFLUENCE | GOOGLEDRIVE | WEBCRAWLER | WORKDOCS | FSX | SLACK | BOX | QUIP | JIRA | GITHUB | ALFRESCO | TEMPLATE

UpdatedAt

Der Unix-Zeitstempel der letzten Aktualisierung des Datenquellenkonnektor.

Typ: Zeitstempel

VpcConfiguration

Konfigurationsinformationen für eine Amazon Virtual Private Cloud zum Herstellen der Verbindung mit Ihrer Datenquelle. Weitere Informationen finden Sie unter Konfigurieren einer VPC.

Typ: DataSourceVpcConfiguration Objekt

Fehler

Hinweise zu den Fehlern, die allen Aktionen gemeinsam sind, finden Sie unterHäufige Fehler.

AccessDeniedException

Sie haben keinen ausreichenden Zugriff zum Ausführen dieser Aktion. Bitte stellen Sie sicher, dass Sie über die erforderlichen Berechtigungsrichtlinien und Benutzerkonten verfügen, und versuchen Sie es erneut.

HTTP Status Code: 400

InternalServerException

Ein Problem ist mit dem internen Server aufgetreten, der für Ihren Amazon Kendra-Service verwendet wird. Bitte warten Sie ein paar Minuten und versuchen Sie es erneut, oder wenden Sie sich an den Support, um Hilfe zu erhalten.

HTTP Status Code: 500

ResourceNotFoundException

Die Ressource, die Sie verwenden möchten, existiert nicht. Bitte überprüfen Sie, ob Sie die richtige Ressource angegeben haben, und versuchen Sie es erneut.

HTTP Status Code: 400

ThrottlingException

Die Anforderung wurde aufgrund der Drosselung von Anforderungen abgelehnt. Bitte reduzieren Sie die Anzahl der Anfragen und versuchen Sie es erneut.

HTTP Status Code: 400

ValidationException

Die Eingabe erfüllt nicht die vom Amazon Kendra Service festgelegten Einschränkungen. Bitte geben Sie die richtige Eingabe ein und versuchen Sie es erneut.

HTTP Status Code: 400
