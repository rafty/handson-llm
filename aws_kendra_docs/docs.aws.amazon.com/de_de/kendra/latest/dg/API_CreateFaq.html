CreateFaq

Erstellt einen Satz häufig gestellten Fragen (häufig gestellten Amazon S3.

Das Hinzufügen von häufig gestellten Fragen zu einem Index ist ein asynchroner Vorgang.

Ein Beispiel für das Hinzufügen einer FAQ zu einem Index mithilfe von Python- und Java-SDKs finden Sie unter Verwenden Ihrer FAQ-Datei.

Anforderungssyntax

{
   "ClientToken": "string",
   "Description": "string",
   "FileFormat": "string",
   "IndexId": "string",
   "LanguageCode": "string",
   "Name": "string",
   "RoleArn": "string",
   "S3Path": { 
      "Bucket": "string",
      "Key": "string"
   },
   "Tags": [ 
      { 
         "Key": "string",
         "Value": "string"
      }
   ]
}

Anfrageparameter

Informationen zu den Parametern, die allen Aktionen gemeinsam sind, finden Sie unter Allgemeine Parameter.

Die Anforderung akzeptiert die folgenden Daten im JSON-Format.

ClientToken

Ein Token, das Sie angeben, um die Anfrage zur Erstellung einer FAQ zu identifizieren. Bei mehrerenCreateFaqRequest API-Aufrufen mit demselben Client-Token wird nur eine häufig gestellte Frage erstellt.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 100 Zeichen.

Required: No

Description

Eine Beschreibung für häufig gestellte Fragen.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Höchstlänge = 1 000 Zeichen.

Pattern: ^\P{C}*$

Erforderlich: Nein

FileFormat

Das Format der häufig gestellten Fragen. Sie können zwischen einem grundlegenden CSV-Format, einem CSV-Format mit benutzerdefinierten Attributen in einem Header und einem JSON-Format mit benutzerdefinierten Attributen wählen.

Das Format muss mit dem Format der Datei übereinstimmen, die in dem S3-Bucket gespeichert ist, der imS3Path Parameter Parameter Parameter angegeben wird.

Weitere Informationen finden Sie unter Hinzufügen von Fragen und Antworten.

Typ: Zeichenfolge

Zulässige Werte: CSV | CSV_WITH_HEADER | JSON

Required: No

IndexId

Die Kennung des Indexes mit den häufig gestellten Fragen.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge von 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Erforderlich: Ja

LanguageCode

Der Code für eine Sprache. Auf diese Weise können Sie eine Sprache für das FAQ-Dokument unterstützen. Englisch wird standardmäßig unterstützt. Weitere Informationen zu unterstützten Sprachen, einschließlich ihrer Codes, finden Sie unter Hinzufügen von Dokumenten in anderen Sprachen als Englisch.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 2. Maximale Länge von 10.

Pattern: [a-zA-Z-]*

Erforderlich: Nein

Name

Ein Name für häufig gestellte Fragen.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 100 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Erforderlich: Ja

RoleArn

Der Amazon-Ressourcenname (ARN) einer IAM-Rolle mit der Berechtigung zum Zugriff auf den S3-Bucket, der die häufig gestellten Fragen enthält. Weitere Informationen finden Sie unter IAM-Zugriffsrollen für Amazon Kendra.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Maximale Länge von 1 084.

Pattern: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Erforderlich: Ja

S3Path

Der Pfad zu der häufig gestellten Fragen.

Typ: S3Path Objekt

Erforderlich: Ja

Tags

Eine Liste von Schlüssel-Wert-Paaren, die häufig gestellten Fragen. Sie können die Tags verwenden, um Ihre -Ressourcen zu identifizieren und zu verwalten und den Zugriff auf -Ressourcen.

Typ: Array von Tag-Objekten

Array-Mitglieder: Die Mindestanzahl beträgt 0 Elemente. Die maximale Anzahl beträgt 50 Elemente.

Required: No

Antwortsyntax

{
   "Id": "string"
}

Antwortelemente

Wenn die Aktion erfolgreich ist, sendet der Service eine HTTP 200-Antwort zurück.

Die folgenden Daten werden vom Service im JSON-Format zurückgegeben.

Id

Die Kennung der häufig gestellten Fragen.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 100 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Fehler

Hinweise zu den Fehlern, die allen Aktionen gemeinsam sind, finden Sie unterHäufige Fehler.

AccessDeniedException

Sie haben keinen ausreichenden Zugriff, um diese Aktion auszuführen. Bitte stellen Sie sicher, dass Sie über die erforderlichen Berechtigungsrichtlinien und Benutzerkonten verfügen, und versuchen Sie es erneut.

HTTP Status Code: 400

ConflictException

Bei der Anfrage ist ein Konflikt aufgetreten. Bitte korrigieren Sie alle Unstimmigkeiten mit Ihren Ressourcen und versuchen Sie es erneut.

HTTP Status Code: 400

InternalServerException

Ein Problem ist mit dem internen Server aufgetreten, der für Ihren Amazon Kendra-Service verwendet wird. Bitte warten Sie ein paar Minuten und versuchen Sie es erneut, oder wenden Sie sich an den Support, um Hilfe zu erhalten.

HTTP Status Code: 500

ResourceNotFoundException

Die Ressource, die Sie verwenden möchten, existiert nicht. Bitte überprüfen Sie, ob Sie die richtige Ressource angegeben haben, und versuchen Sie es erneut.

HTTP Status Code: 400

ServiceQuotaExceededException

Sie haben die festgelegten Grenzwerte für Ihren Amazon Kendra-Service überschritten. Weitere Informationen finden Sie unter Kontingente oder wenden Sie sich an den Support, um Informationen zu einer Erhöhung der Limits zu erhalten.

HTTP Status Code: 400

ThrottlingException

Die Anforderung wurde aufgrund der Drosselung von Anforderungen abgelehnt. Bitte reduzieren Sie die Anzahl der Anfragen und versuchen Sie es

HTTP Status Code: 400

ValidationException

Die Eingabe erfüllt nicht die vom Amazon Kendra-Service definierten Einschränkungen. Bitte geben Sie die richtige Eingabe und versuchen Sie es

HTTP Status Code: 400
