Abfrage-Vorschläge

Amazon KendraAbfragevorschläge können Ihren Benutzern helfen, ihre Suchanfragen schneller einzugeben und ihre Suche zu steuern.

Amazon Kendraschlägt Abfragen vor, die für Ihre Benutzer relevant sind, basierend auf beliebten Abfragen im Abfrageverlauf oder im Abfrageprotokoll. Amazon Kendraverwendet alle Abfragen, nach denen Ihre Benutzer suchen, und lernt aus diesen Abfragen, um Ihren Benutzern Vorschläge zu unterbreiten. Amazon Kendraschlägt Benutzern beliebte Abfragen vor, wenn sie mit der Eingabe ihrer Abfrage beginnen. Amazon Kendraschlägt eine Abfrage vor, wenn das Präfix oder die ersten Zeichen der Abfrage mit dem übereinstimmen, was der Benutzer als Abfrage eingibt.

Ein Benutzer beginnt beispielsweise, die Abfrage „bevorstehende Ereignisse“ einzugeben. Amazon Kendrahat aus dem Abfrageprotokoll erfahren, dass viele Benutzer oft nach 'bevorstehenden Ereignissen 2050' gesucht haben. Der Benutzer sieht, dass „bevorstehende Ereignisse 2050“ direkt unter seiner Suchleiste angezeigt werden, wodurch seine Suchabfrage automatisch vervollständigt wird. Der Benutzer wählt diesen Abfragevorschlag aus, indem er das erste Suchergebnis auswählt, bei dem es sich um das Dokument „Kommende Ereignisse: Was passiert im Jahr 2050“ handelt.

Sie können angeben, wie berechtigte AbfragenAmazon Kendra ausgewählt werden, um Ihren Benutzern vorzuschlagen. Sie können beispielsweise angeben, dass ein Abfragevorschlag von mindestens 10 eindeutigen Benutzern gesucht worden sein muss (Standard ist 3), innerhalb der letzten 30 Tage gesucht wurde und keine Wörter oder Ausdrücke aus Ihrer Blockliste enthält. Amazon Kendraerfordert, dass eine Abfrage mindestens ein Suchergebnis enthält und mindestens ein Wort mit mehr als vier Zeichen enthält.

Bei Abfragevorschlägen wird nicht zwischen Groß- und Kleinschreibung unterschieden. Amazon Kendrakonvertiert das Abfragepräfix und die vorgeschlagene Abfrage in Kleinbuchstaben, ignoriert alle einfachen und doppelten Anführungszeichen und ersetzt mehrere Leerzeichen durch ein einzelnes Leerzeichen. Amazon Kendraentspricht allen anderen Sonderzeichen unverändert. Amazon Kendrazeigt keine Vorschläge an, wenn ein Benutzer weniger als zwei Zeichen oder mehr als 60 Zeichen eingibt.

Mithilfe der GetQuerySuggestionsAPI können Sie Abfragevorschläge abrufen, die für Ihre Benutzer relevant sind. Abfrage-Vorschläge sind standardmäßig und ohne zusätzliche Kosten aktiviert. Sie können Abfrage-Vorschläge jederzeit mithilfe der UpdateQuerySuggestionsConfigAPI deaktivieren. Sie können Ihre Einstellungen auf zwei Arten testen, bevor Sie Vorschläge auf Ihre Suchanwendung anwenden:

  • Durch die Verwendung derUpdateQuerySuggestionsConfig API.

  • In der Konsole in den Einstellungen für Abfragevorschläge.

Sie verwenden dieGetQuerySuggestions API, um Abfragevorschläge in Ihre Konsolenanwendung zu integrieren, damit Ihre Benutzer die Vorschläge sehen können.

Einstellungen für Abfrage-Vorschläge

Sie können die folgenden Einstellungen mithilfe derUpdateQuerySuggestionsConfig API konfigurieren:

  • Modus —Abfragevorschläge lauten entwederENABLED oderLEARN_ONLY. Amazon Kendraaktiviert standardmäßig Abfragevorschläge. LEARN_ONLYdeaktiviert Abfragevorschläge. Wenn diese Option deaktiviert ist, lernt esAmazon Kendra weiterhin Vorschläge, macht Benutzern jedoch keine Abfragevorschläge.

  • Zeitfenster für das Abfrageprotokoll — Wie aktuell Ihre Abfragen im Zeitfenster Ihres Abfrageprotokolls sind. Das Zeitfenster ist ein ganzzahliger Wert für die Anzahl der Tage vom aktuellen Tag bis zu den vergangenen Tagen.

  • Abfragen ohne Benutzerinformationen — Auf festlegen,TRUE dass alle Abfragen einbezogen werden, oder so,FALSE dass nur Abfragen mit Benutzerinformationen berücksichtigt werden. Sie können dies verwenden, wenn Ihre Suchanwendung Benutzerinformationen enthält, z. B. die Benutzer-ID, wenn ein Benutzer eine Anfrage stellt. Diese Einstellung filtert standardmäßig keine Abfragen heraus, wenn den Abfragen keine spezifischen Benutzerinformationen zugeordnet sind. Sie können diese Einstellung jedoch verwenden, um nur Vorschläge zu machen, die auf Abfragen basieren, die Benutzerinformationen enthalten.

  • Einzelne Benutzer — Die Mindestanzahl eindeutiger Benutzer, die nach einer Abfrage suchen müssen, um Ihren Benutzern Vorschläge unterbreiten zu können. Diese Zahl ist ein Ganzzahlwert.

  • Anzahl der Abfragen — Die Mindestanzahl, mit der eine Abfrage gesucht werden muss, damit sie Ihren Benutzern vorgeschlagen werden kann. Diese Zahl ist ein Ganzzahlwert.

Diese Einstellungen wirken sich darauf aus, wie Abfragen als beliebte Abfragen ausgewählt werden, die Ihren Benutzern vorgeschlagen werden. Wie Sie Ihre Einstellungen anpassen, hängt von Ihren spezifischen Bedürfnissen ab, zum Beispiel:

  • Wenn Ihre Benutzer normalerweise im Durchschnitt einmal im Monat suchen, können Sie die Anzahl der Tage im Zeitfenster des Abfrageprotokolls auf 30 festlegen, sodass Sie die meisten der letzten Abfragen Ihrer Benutzer erfassen, bevor sie im Zeitfenster veraltet sind.

  • Wenn nur eine kleine Anzahl Ihrer Abfragen Benutzerinformationen enthält und Sie keine Abfragen vorschlagen möchten, die auf einer kleinen Stichprobengröße basieren, können Sie Abfragen so einrichten, dass sie alle Benutzer einbeziehen.

  • Wenn Sie beliebte Abfragen so definieren, dass sie von mindestens 10 eindeutigen Benutzern und mindestens 100 Mal gesucht werden, legen Sie die eindeutigen Benutzer auf 10 und die Anzahl der Abfragen auf 100 fest.

Ihre Änderungen an den Einstellungen werden möglicherweise nicht sofort wirksam. Sie können die Änderungen der Einstellungen mithilfe der DescribeQuerySuggestionsConfigAPI verfolgen. Die Zeit, bis Ihre aktualisierten Einstellungen wirksam werden, hängt von den von Ihnen vorgenommenen Aktualisierungen und der Anzahl der Suchanfragen in Ihrem Index ab.

Python

Um Abfragevorschläge abzurufen

import boto3
from botocore.exceptions import ClientError

kendra = boto3.client("kendra")

print("Get query suggestions.")

# Provide the index ID
index_id = "index-id"

# Provide the query text
query_text = "query"

try:
    query_suggestions_response = kendra.get_query_suggestions(
        IndexId = index_id,
        QueryText = query_text,
        MaxSuggestionsCount = 5
    )

    # Print out the suggestions you received
    if ("Suggestions" in query_suggestions_response.keys()) {
        for (suggestion: query_suggestions_response["Suggestions"]) {
            print(suggestion["Value"]["Text"]["Text"]);
        }
    }
   
except ClientError as e:
        print("%s" % e)

print("Program ends.")

Um Abfragevorschläge zu aktualisieren,

Um beispielsweise das Zeitfenster für das Abfrageprotokoll und die Mindestanzahl, mit der eine Abfrage gesucht werden muss, zu ändern, gehen Sie wie folgt vor:

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Updating query suggestions settings/configuration for an index.")

# Provide the index ID
index_id = "index-id"

# Configure the settings you want to update
query_log_look_back_window_in_days = 30
minimum_query_count = 100

try:
    kendra.update_query_suggestions_config(
        IndexId = index_id,
        QueryLogLookBackWindowInDays = query_log_look_back_window_in_days,
        MinimumQueryCount = minimum_query_count
    )

    print("Wait for Amazon Kendra to update the query suggestions.")

    while True:
        # Get query suggestions description of settings/configuration
        query_sugg_config_response = kendra.describe_query_suggestions_config(
            IndexId = index_id
        )
        
        # If status is not UPDATING, then quit
        status = query_sugg_config_response["Status"]
        print(" Updating query suggestions config. Status: " + status)
        if status != "UPDATING":
            break
        time.sleep(60)

except ClientError as e:
        print("%s" % e)

print("Program ends.")

Sie können Ihre aktuellen Einstellungen mithilfe derDescribeQuerySuggestionsConfig API überprüfen. Darüber hinaus zeigt dieser Vorgang die folgenden Informationen zu Ihren Abfragevorschlägen für einen Index an:

  • Modus —Abfragevorschläge lauten entwederENABLED oderLEARN_ONLY. Amazon Kendraaktiviert standardmäßig Abfragevorschläge. LEARN_ONLYdeaktiviert Abfragevorschläge. Sie können den Modus mithilfe derUpdateQuerySuggestionsConfig API ändern. Wenn diese Option deaktiviert ist, lernt esAmazon Kendra weiterhin Vorschläge, macht Benutzern jedoch keine Abfragevorschläge.

  • Status —Abfragevorschläge lauten entwederACTIVE oderUPDATING.

  • Anzahl der Vorschläge — Die Gesamtzahl der Abfragen, die bereit sind, Ihren Benutzern vorgeschlagen zu werden. Wenn die Anzahl viel niedriger ist als erwartet, kann dies daran liegen, dass mehr Abfragen erforderlichAmazon Kendra sind, um daraus zu lernen, oder Ihre aktuellen Einstellungen für Abfragevorschläge sind zu streng.

  • Letzte Erstellungszeit der Vorschläge — Der Zeitpunkt, zu dem die Vorschläge zuletzt aktualisiert wurden. Amazon Kendraaktualisiert Vorschläge automatisch alle 24 Stunden oder wenn Sie eine Einstellung ändern oder wenn Sie eine Blockliste anwenden.

  • Zeitpunkt der letzten Löschung der Vorschläge — Der Zeitpunkt, zu dem die Vorschläge zuletzt gelöscht wurden.

Blockieren Sie bestimmte Abfragen aus Vorschlägen

Eine BlocklisteAmazon Kendra verhindert, dass Ihren Benutzern bestimmte beliebte Abfragen vorgeschlagen werden. Es ist eine Liste von Wörtern oder Ausdrücken, die Sie von den Abfragevorschlägen ausschließen möchten. Amazon Kendraschließt Abfragen aus, die exakt mit den Wörtern oder Ausdrücken in der Blockliste übereinstimmen.

Sie können eine Sperrliste verwenden, um sich vor anstößigen Wörtern oder Ausdrücken zu schützen, die häufig in Ihrem Abfrageprotokoll vorkommen und die als Vorschläge ausgewählt werdenAmazon Kendra könnten. Eine Sperrliste kann auchAmazon Kendra verhindern, dass beliebte Abfragen vorgeschlagen werden, die Informationen enthalten, die noch nicht veröffentlicht oder angekündigt werden können. Wenn Ihre Benutzer beispielsweise häufig Fragen zu einer Veröffentlichung eines neuen Produkts stellen, das Sie nicht vorschlagen möchten, weil Sie noch nicht bereit sind, es bald zu veröffentlichen, können Sie Abfragen, die den Produktnamen und Produktinformationen enthalten, von Vorschlägen ausschließen.

Mithilfe der CreateQuerySuggestionsBlockListAPI können Sie eine Blockliste für Abfragen erstellen. Sie setzen jedes Blockwort oder jede Phrase in eine separate Zeile in einer Textdatei. Dann laden Sie die Textdatei in Ihren S3-Bucket hoch und geben den Pfad oder Speicherort der Datei anAmazon S3. Amazon Kendraunterstützt derzeit das Erstellen von nur einer Blockliste.

Sie können die Textdatei Ihrer blockierten Wörter und Ausdrücke in IhremAmazon S3 Bucket ersetzen und die UpdateQuerySuggestionsBlockListAPI verwenden, um die Blockliste zu aktualisierenAmazon Kendra.

Sie können die DescribeQuerySuggestionsBlockListAPI verwenden, um den Status Ihrer Blockliste und andere nützliche Informationen abzurufen, z. B. wann Ihre Blockliste zuletzt aktualisiert wurde, wie viele Wörter oder Ausdrücke sich in Ihrer aktuellen Blockliste befinden und um hilfreiche Fehlermeldungen beim Erstellen einer Blockliste zu erhalten. Sie können die ListQuerySuggestionsBlockListsAPI auch verwenden, um eine Liste von Blocklistenzusammenfassungen für einen Index abzurufen.

Verwenden Sie die DeleteQuerySuggestionsBlockListAPI, um Ihre Blockliste zu löschen.

Ihre Aktualisierungen der Blockliste werden möglicherweise nicht sofort wirksam. Sie können Updates mithilfe derDescribeQuerySuggestionsBlockList API verfolgen.

Python

Um eine Blockliste zu erstellen

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Create a query suggestions block list.")

# Provide a name for the block list
block_list_name = "block-list-name"
# Provide an optional description for the block list
block_list_description = "block-list-description"
# Provide the IAM role ARN required for query suggestions block lists
block_list_role_arn = "role-arn"

# Provide the index ID
index_id = "index-id"

s3_bucket_name = "bucket-name"
s3_key = "query-suggestions/block_list.txt"
source_s3_path = {
    'Bucket': s3_bucket_name,
    'Key': s3_key
}

try:
    block_list_response = kendra.create_query_suggestions_block_list(
        Description = block_list_description,
        Name = block_list_name,
        RoleArn = block_list_role_arn,
        IndexId = index_id,
        SourceS3Path = source_s3_path
    )

    print(block_list_response)

    block_list_id = block_list_response["Id"]

    print("Wait for Amazon Kendra to create the block list.")

    while True:
        # Get block list description
        block_list_description = kendra.describe_query_suggestions_block_list(
            Id = block_list_id,
            IndexId = index_id
        )
        # If status is not CREATING, then quit
        status = block_list_description["Status"]
        print("Creating block list. Status: " + status)
        if status != "CREATING":
            break
        time.sleep(60)
        
except ClientError as e:
        print("%s" % e)

print("Program ends.")

Klare Vorschläge

Sie können Abfragevorschläge mithilfe der ClearQuerySuggestionsAPI löschen. Beim Löschen von Vorschlägen werden nur vorhandene Abfragevorschläge gelöscht, nicht die Abfragen im Abfrageprotokoll. Nachdem Sie Vorschläge gelöscht haben,Amazon Kendra lernt es neue Vorschläge, die auf neuen Abfragen basieren, die dem Abfrageprotokoll hinzugefügt wurden, nachdem Sie Vorschläge gelöscht haben.

Keine Vorschläge verfügbar

Wenn Sie keine Vorschläge für eine Abfrage sehen, kann das einen der folgenden Gründe haben:

  • Ihr Index enthält nicht genügend Abfragen, aus denenAmazon Kendra Sie lernen könnten.

  • Ihre Einstellungen für Abfragevorschläge sind zu streng, sodass die meisten Abfragen aus den Vorschlägen herausgefiltert werden.

  • Sie haben kürzlich Vorschläge gelöscht und benötigenAmazon Kendra immer noch Zeit, bis sich neue Abfragen ansammeln, um neue Vorschläge zu lernen.

Sie können Ihre aktuellen Einstellungen mithilfe derDescribeQuerySuggestionsConfig API überprüfen.
