Box (Kasten)

Box ist ein Cloud-Speicherdienst, der Datei-Hosting-Funktionen bietet. Sie könnenAmazon Kendra es verwenden, um Inhalte in Ihrem Box-Inhalt zu indexieren, einschließlich Kommentare, Aufgaben und Weblinks.

Sie können überAmazon Kendra die Amazon KendraKonsole und die BoxConfigurationAPI eine Verbindung zu Ihrer Box-Datenquelle herstellen.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra Box-Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Unterstützte Funktionen

Amazon KendraDer Box -Datenquellenkonnektor unterstützt die folgenden Funktionen:

  • Änderungsprotokoll

  • Feldzuordnungen

  • Filterung des Benutzerkontextes

  • Einschluss-/Ausschlussfilter

  • Virtual Private Cloud (VPC)

Voraussetzungen

Bevor Sie Ihre Box-DatenquelleAmazon Kendra zur Indexierung verwenden können, nehmen Sie diese Änderungen an Ihrer Box und IhrenAWS Konten vor.

In Box müssen Sie über Folgendes verfügen:

  • Ein Box Enterprise- oder Box Enterprise Plus-Konto.

  • In der Box Developer Console wurde eine benutzerdefinierte Box-App erstellt und für die Verwendung der Serverauthentifizierung (mit JWT) konfiguriert. Weitere Informationen finden Sie in der Box-Dokumentation zum Erstellen einer benutzerdefinierten App und in der Box-Dokumentation zur Konfiguration von JWT Auth.

  • Stellen Sie Ihre App-Zugriffsebene auf App + Enterprise Access ein und erlauben Sie ihr, API-Aufrufe mithilfe des as-user-Headers zu tätigen.

  • Hat den Admin-Benutzer verwendet, um Ihrer Box-App die folgenden Anwendungsbereiche hinzuzufügen:

    • Schreiben Sie alle in einer Box gespeicherten Dateien und Ordner

    • Verwalten von Benutzern

    • Verwalten von Gruppen

    • Unternehmensimmobilien verwalten

  • Generiertes und heruntergeladenes öffentliches/privates key pair, das eine Client-ID, ein Client-Geheimnis, eine öffentliche Schlüssel-ID, eine private Schlüssel-ID, eine Passphrase und eine Unternehmens-ID zur Verwendung als Authentifizierungsdaten enthält. Weitere Informationen finden Sie unter Öffentliches und privates Schlüsselpaar.

  • Ihre Box Enterprise ID wurde entweder aus Ihren Box Developer Console-Einstellungen oder aus Ihrer Box-App kopiert. Zum Beispiel 801234567.

  • Hat aktiviert, dass jedes Dokument in Box und in anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

In IhremAWS Konto müssen Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Hat Ihre Box-Authentifizierungsdaten in einemAWSSecrets Manager Secret gespeichert und, falls Sie die API verwenden, den ARN des Geheims notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder kein Geheimnis haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn Sie Ihre Box-Datenquelle damit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Anweisungen zum Anschluss

Um eine VerbindungAmazon Kendra zu Ihrer Box-Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer Box-Datenquelle angeben, damit diese auf Ihre Daten zugreifenAmazon Kendra kann. Falls Sie Box noch nicht konfiguriert haben,Amazon Kendra sehen Sie sich das anVoraussetzungen.

Console

Um eine VerbindungAmazon Kendra zu Box herzustellen

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich Indizes und dann in der Liste der Indizes den Index aus, den Sie verwenden möchten.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option Box Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Daten die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags für Neues Tag hinzufügen —Tags können Sie Ressourcen suchen und filtern oder IhreAWS Kosten verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit die folgenden Informationen ein:

    1. Box Enterprise ID —Geben Sie Ihre Box Enterprise ID ein.

    2. AWSSecrets Managergeheim — Wählen Sie ein vorhandenes Geheimnis aus oder erstellen Sie ein neuesSecrets Manager Geheimnis, um Ihre Box-Authentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-Box' wird automatisch zu Ihrem geheimen Namen hinzugefügt.

      2. Geben Sie für Client-ID, Client Secret, Public Key ID, Private Key ID und Passphrase die Werte des öffentlichen/privaten Schlüssels ein, den Sie in Ihrem Box-Konto generiert und von Ihrem Box-Konto heruntergeladen haben.

      3. Wählen Sie Speichern.

    3. Virtual Private Cloud (VPC) — Sie können eine VPC verwenden. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    4. IAM-Rolle — Wählen Sie eine vorhandene IAM-Rolle oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    5. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Wählen Sie Entitäten oder Inhaltstypen aus — Die Box-Entitäten oder Inhaltstypen, die Sie crawlen möchten. Jeder Kommentar wird als separates Dokument indexiert.

    2. Änderungsprotokoll — Wählen Sie diese Option aus, um Ihren Index zu aktualisieren, anstatt alle Ihre Dateien zu synchronisieren.

    3. Regex-Muster — Reguläre Ausdrucksmuster zum Ein- oder Ausschließen bestimmter Dateien. Sie können bis zu 100 Muster hinzufügen.

    4. Wählen Sie im Synchronisierungslaufzeitplan für Häufigkeit aus, wie oft mit Ihrer Datenquelle synchronisiertAmazon Kendra werden soll.

    5. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen die folgenden Informationen ein:

    1. Für Dateien und Ordner, Kommentare, Aufgaben und Weblinks — Wählen Sie eines derAmazon Kendra generierten Standarddatenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

Um eine VerbindungAmazon Kendra zu Box herzustellen

Sie müssen mithilfe der BoxConfigurationAPI Folgendes angeben:

Box Enterprise ID —Geben Sie Ihre Box Enterprise ID ein. Sie finden die Unternehmens-ID in den Einstellungen der Box Developer Console oder wenn Sie eine App in Box erstellen.

  • Geheimer Amazon Resource Name (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr Box-Konto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "clientID": "client-id",
    "clientSecret": "client-secret",
    "publicKeyID": "public-key-id",
    "privateKey": "private-key",
    "passphrase": "pass-phrase"
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager geheimes Element und zum Aufrufen der erforderlichen öffentlichen APIs für den Box-Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für Box-Datenquellen.

Sie können auch die folgenden optionalen-Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie dies imVpcConfiguration Rahmen der Datenquellenkonfiguration an. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

  • Änderungsprotokoll — Gibt an, ob der Change-Log-Mechanismus für die Box-Datenquelle verwendet werdenAmazon Kendra soll, um zu bestimmen, ob ein Dokument im Index hinzugefügt, aktualisiert oder gelöscht werden muss.

    Anmerkung

    Verwenden Sie das Änderungsprotokoll, wenn Sie nicht alle Dokumente scannen möchtenAmazon Kendra. Wenn Ihr Änderungsprotokoll groß ist, dauert das Scannen der Dokumente in der Box-Datenquelle möglicherweiseAmazon Kendra weniger Zeit als das Verarbeiten des Änderungsprotokolls. Wenn Sie Ihre Box-Datenquelle zum ersten Mal mit Ihrem Index synchronisieren, werden alle Dokumente gescannt.

  • Ein- und Ausschlussfilter — Geben Sie an, ob bestimmte Box-Dateien, Ordner, Kommentare, Aufgaben und Weblinks ein- oder ausgeschlossen werden sollen.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre Box-Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

  • BenutzerkontextfilterungAmazon Kendra — durchsucht standardmäßig die Access Control List (ACL) für Ihre Datenquelle. Die ACL-Informationen werden verwendet, um Suchergebnisse auf der Grundlage des Zugriffs des Benutzers oder seiner Gruppe auf Dokumente zu filtern. Weitere Informationen finden Sie unter Benutzerkontextfilterung für Box-Datenquellen.
