BatchPutDocument

Fügt einem Index ein oder mehrere Dokumente hinzu

Mit derBatchPutDocument API können Sie Inline-Dokumente oder eine Reihe von Dokumenten, die in einem Amazon-S3-Bucket gespeichert waren. Verwenden Sie diese API, um Ihren Text und unstrukturierten Text in einen Index aufzunehmen, benutzerdefinierte Attribute zu den Dokumenten hinzuzufügen und eine Zugriffskontrollliste an die dem Index hinzugefügten Dokumente anzuhängen.

Die Dokumente werden asynchron indexiert. Sie können den Fortschritt des Batches mit verfolgenAWS CloudWatch. Alle Fehlermeldungen im Zusammenhang mit der Verarbeitung des Batches werden an IhrAWS CloudWatch Protokoll gesendet.

Ein Beispiel für die Aufnahme von Inline-Dokumenten mit Python- und Java-SDKs finden Sie unter Direktes Hinzufügen von Dateien zu einem Index.

Anfrageparameter

Informationen zu den Parametern, die allen Aktionen gemeinsam sind, finden Sie unter Allgemeine Parameter.

Die Anforderung akzeptiert die folgenden Daten im JSON-Format.

CustomDocumentEnrichmentConfiguration

Konfigurationsinformationen zum Ändern der Metadaten und des Inhalts von Dokumenten während derBatchPutDocument Dokumenten-Erfassung.

Weitere Informationen zum Erstellen, Ändern und Löschen von Dokumentmetadaten oder zum Vornehmen anderer Inhaltsänderungen bei der Aufnahme von Dokumenten in Amazon Kendra finden Sie unter Anpassen von Dokumentmetadaten während des Erfassungsprozesses.

Typ: CustomDocumentEnrichmentConfiguration Objekt

Required: No

Documents

Ein oder mehrere Dokumente, die dem Index hinzugefügt werden sollen.

Dokumente haben die folgenden Dateigrößenbeschränkungen.

  • 50 MB Gesamtgröße für jede Datei

  • 5 MB extrahierter Text für jede Datei

Weitere Informationen finden Sie unter Kontingente.

Typ: Array von Document-Objekten

Array-Mitglieder: Die Mindestanzahl beträgt 1 Element. Die maximale Anzahl beträgt 10 Elemente.

Erforderlich: Ja

IndexId

Die Kennung des Index, dem die Dokumente hinzugefügt werden sollen. Sie müssen den Index zuerst mithilfe derCreateIndex API erstellen.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge von 36Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Erforderlich: Ja

RoleArn

Der Amazon-Ressourcenname (ARN) einer IAM-Rolle mit der Berechtigung für den Zugriff auf Ihren S3-Bucket. Weitere Informationen finden Sie unter IAM-Zugriffsrollen für Amazon Kendra.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Maximale Länge beträgt 1284 Zeichen.

Pattern: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Erforderlich: Nein

Antwortelemente

Wenn die Aktion erfolgreich ist, sendet der Service eine HTTP 200-Antwort zurück.

Die folgenden Daten werden vom Service im JSON-Format zurückgegeben.

FailedDocuments

Eine Liste von Dokumenten, die nicht zum Index hinzugefügt wurden, weil das Dokument eine Validierungsprüfung nicht bestanden hat. Jedes Dokument enthält eine Fehlermeldung, die angibt, warum das Dokument nicht zum Index hinzugefügt werden konnte.

Wenn beim Hinzufügen eines Dokuments zu einem Index ein Fehler aufgetreten ist, wird der Fehler in IhremAWS CloudWatch Protokoll gemeldet. Weitere Informationen finden Sie unter Überwachung von Amazon Kendra mit CloudWatch Amazon-Protoken-Erfassung.

Typ: Array von BatchPutDocumentResponseFailedDocument-Objekten
