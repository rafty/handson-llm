DescribeIndex

Ruft Informationen zu einem vorhandenen Amazon-Kendra-Index ab.

Anforderungssyntax

{
   "Id": "string"
}

Anfrageparameter

Informationen zu den Parametern, die allen Aktionen gemeinsam sind, finden Sie unter Allgemeine Parameter.

Die Anforderung akzeptiert die folgenden Daten im JSON-Format.

Id

Die Kennung des Index, zu dem Sie Informationen erhalten möchten.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge beträgt 36 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Erforderlich: Ja

Antwortelemente

Wenn die Aktion erfolgreich ist, sendet der Service eine HTTP 200-Antwort zurück.

Die folgenden Daten werden vom Service im JSON-Format zurückgegeben.

CapacityUnits

Für Enterprise Edition-Indizes können Sie wählen, ob Sie zusätzliche Kapazität verwenden möchten, um die Anforderungen Ihrer Anwendung zu erfüllen. Dies enthält die für den Index verwendeten Kapazitätseinheiten. Eine Abfrage- oder Dokumentspeicherkapazität von Null gibt an, dass der Index die Standardkapazität verwendet. Weitere Informationen zur Standardkapazität für einen Index und deren Anpassung finden Sie unter Kapazität anpassen.

Typ: CapacityUnitsConfiguration Objekt

CreatedAt

Der Unix-Zeitstempel, zu dem der Index erstellt wurde.

Typ: Zeitstempel

Description

Die Beschreibung für den Index.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Höchstlänge = 1 000 Zeichen.

Pattern: ^\P{C}*$

DocumentMetadataConfigurations

Konfigurationsinformationen für Dokumentmetadaten oder Felder. Dokumentmetadaten sind Felder oder Attribute, die mit Ihren Dokumenten verknüpft sind. Zum Beispiel der Name der Unternehmensabteilung, der jedem Dokument zugeordnet ist.

Typ: Array von DocumentMetadataConfiguration-Objekten

Array-Mitglieder: Die Mindestanzahl beträgt 0 Elemente. Die maximale Anzahl beträgt 500 Elemente.

Edition

Die für den Index verwendete Amazon Kendra-Ausgabe. Sie entscheiden über die Edition, wenn Sie den Index erstellen.

Typ: Zeichenfolge

Zulässige Werte: DEVELOPER_EDITION | ENTERPRISE_EDITION

ErrorMessage

Wenn derStatus Feldwert istFAILED, enthält dasErrorMessage Feld eine Meldung, die erklärt, warum.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Maximale Länge beträgt 2048 Zeichen.

Pattern: ^\P{C}*$

Id

Die Kennung des Index.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge beträgt 36 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

IndexStatistics

Enthält Informationen über die Anzahl der häufig gestellten Fragen und Antworten sowie über die Anzahl der indexierten Textdokumente.

Typ: IndexStatistics Objekt

Name

Der Indexname.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 1000 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

RoleArn

Der Amazon-Ressourcenname (ARN) der IAM-Rolle, die Amazon Kendra berechtigt, in Ihre Amazon-CloudWatch-Protokolle zu schreiben.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Höchstlänge 1284 Zeichen.

Pattern: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

ServerSideEncryptionConfiguration

Die Kennung desAWS KMS KundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKundenKunden Amazon Kendra unterstützt keine asymmetrischen CMKs.

Typ: ServerSideEncryptionConfiguration Objekt

Status

Der aktuelle Status des Index. Wenn der Wert istACTIVE, ist der Index jetzt verfügbar. Wenn derStatus Feldwert lautetFAILED, enthält dasErrorMessage Feld eine Meldung, die erklärt, warum.

Typ: Zeichenfolge

Zulässige Werte: CREATING | ACTIVE | DELETING | FAILED | UPDATING | SYSTEM_UPDATING

UpdatedAt

Das Unix, zu dem der Index zuletzt aktualisiert wurde.

Typ: Zeitstempel

UserContextPolicy

Die Benutzerkontextrichtlinie für den Amazon Kendra-Index.

Typ: Zeichenfolge

Zulässige Werte: ATTRIBUTE_FILTER | USER_TOKEN

UserGroupResolutionConfiguration

Ob Sie die Konfiguration für das Abrufen von Zugriffsebenen von Gruppen und Benutzern aus einerAWS IAM Identity Center (successor to AWS Single Sign-On) Identitätsquelle aktiviert haben.

Typ: UserGroupResolutionConfiguration Objekt

UserTokenConfigurations

Die Benutzertokenkonfiguration für den Amazon Kendra-Index.

Typ: Array von UserTokenConfiguration-Objekten

Array-Mitglieder: Maximale Anzahl von 1 Element.

Fehler

Hinweise zu den Fehlern, die allen Aktionen gemeinsam sind, finden Sie unterHäufige Fehler.

AccessDeniedException

Sie haben keinen ausreichenden Zugriff, um diese Aktion auszuführen. Bitte stellen Sie sicher, dass Sie über die erforderlichen Berechtigungsrichtlinien und Benutzerkonten verfügen, und versuchen Sie es erneut.

HTTP Status Code: 400

InternalServerException

Es ist ein Problem mit dem internen Server aufgetreten, der für Ihren Amazon Kendra-Service verwendet wird. Bitte warten Sie ein paar Minuten und versuchen Sie es erneut, oder wenden Sie sich an den Support, um Hilfe zu erhalten.

HTTP Status Code: 500

ResourceNotFoundException

Die Ressource, die Sie verwenden möchten, existiert nicht. Bitte überprüfen Sie, ob Sie die richtige Ressource angegeben haben, und versuchen Sie es erneut.

HTTP Status Code: 400

ThrottlingException

Die Anforderung wurde aufgrund der Drosselung von Anforderungen abgelehnt. Reduzieren Sie die Anzahl der Anfragen und versuchen Sie es erneut.

HTTP Status Code: 400

ValidationException

Die Eingabe erfüllt nicht die von Amazon Kendra festgelegten Einschränkungen. Bitte geben Sie die richtige Eingabe ein und versuchen Sie es erneut.

HTTP Status Code: 400
