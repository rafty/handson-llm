Amazon Kendra mit Amazon überwachen CloudWatch

Verwenden Sie Amazon, um den Zustand Ihrer Indizes zu verfolgen CloudWatch. Mit CloudWatch können Sie Metriken für die Dokumentensynchronisierung für Ihren Index abrufen. Sie können auch CloudWatch Alarme einrichten, um benachrichtigt zu werden, wenn eine oder mehrere Metriken einen von Ihnen definierten Schwellenwert überschreiten. Sie können beispielsweise die Anzahl der Dokumente überwachen, die zur Indexierung eingereicht wurden, oder die Anzahl der Dokumente, die nicht indexiert werden konnten.

Sie müssen über die entsprechenden CloudWatch Berechtigungen verfügen, um Amazon Kendra zu überwachen CloudWatch. Weitere Informationen finden Sie unter Authentifizierung und Zugriffskontrolle für Amazon CloudWatch im CloudWatchAmazon-Benutzerhandbuch.

Anzeigen von Amazon-Kendra-Metriken

Sehen Sie sich die Amazon Kendra Kendra-Metriken mithilfe der CloudWatch Konsole an.

Um Metriken anzuzeigen (CloudWatch Konsole)

  1. Melden Sie sich bei der anAWS Management Console und öffnen Sie die CloudWatch Konsole unter https://console.aws.amazon.com/cloudwatch/.

  2. Wählen Sie Metriken, wählen Sie Alle Metriken und dann Kendra.

  3. Wählen Sie die Dimension, den Namen einer Metrik und schließlich Add to graph (Dem Diagramm hinzufügen) aus.

  4. Wählen Sie einen Wert für den Datumsbereich aus. Die Anzahl der Kennzahlen für den ausgewählten Zeitraum wird im Diagramm angezeigt.

Erstellen eines Alarms

Ein CloudWatch Alarm überwacht eine einzelne Metrik über einen angegebenen Zeitraum und führt eine oder mehrere Aktionen durch: Senden einer Benachrichtigung an eine Amazon Simple Notification Service (Amazon SNS) -Richtlinie oder eine Auto-Scaling-Richtlinie. Die Aktionen oder Aktionen basieren auf dem Wert der Metrik im Vergleich zu einem gegebenen Schwellenwert über eine Reihe von Ihnen angegebenen Zeiträumen. CloudWatch kann Ihnen auch eine Amazon SNS SNS-Nachricht senden, wenn sich der Status des Alarms ändert.

CloudWatch Alarme rufen nur dann Aktionen auf, wenn sich der Status ändert und für den von Ihnen angegebenen Zeitraum anhält.

So legen Sie einen Alarm fest

  1. Melden Sie sich bei der anAWS Management Console und öffnen Sie die CloudWatch Konsole unter https://console.aws.amazon.com/cloudwatch/.

  2. Wählen Sie die Option Alarme und anschließend Alarm erstellen aus.

  3. Wählen Sie Kendra-Metriken und dann eine Metrik aus.

  4. Wählen Sie für Time Range den zu überwachenden Zeitbereich und dann Next.

  5. Geben Sie Name (Name) und Description (Beschreibung) ein.

  6. Wählen Sie für Whenever den Wert >= und geben Sie einen Maximalwert ein.

  7. Wenn Sie eine E-Mail-Nachricht senden CloudWatch möchten, wenn der Alarmstatus erreicht ist, wählen Sie im Abschnitt Aktionen für Wann immer dieser Alarm ausgegeben wird: die Option Status lautet ALARM aus. Wählen Sie unter Benachrichtigung senden an eine Mailingliste aus oder wählen Sie Neue Liste und erstellen Sie eine neue Mailingliste

  8. Nutzen Sie die Alarmvorschau im Bereich Alarm Preview. Wenn Sie mit dem Alarm zufrieden sind, wählen Sie Create Alarm.

CloudWatch Metriken für Indexsynchronisierungsaufträge

Die folgende Tabelle beschreibt die Amazon-Kendra-Metriken für Datenquellensynchronisationsaufträge.

Metrik Beschreibung
DocumentsCrawled

Die Anzahl der Dokumente, die der Synchronisationsjob während der Ausführung gescannt oder entdeckt hat.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSubmittedForIndexing

Die Anzahl der Dokumente, die der Synchronisationsjob an den Index übermittelt hat.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSubmittedForIndexingFailed

Die Anzahl der Dokumente, bei denen die Indizierung fehlgeschlagen ist. Einzelheiten finden Sie im Inhalt des CloudWatch Protokolls für den Synchronisationsjob.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSubmittedForDeletion

Die Anzahl der Dokumente, deren Entfernung durch den Synchronisationsjob aus dem Index angefordert wurde.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSubmittedForDeletionFailed

Die Anzahl der Dokumente, die nicht gelöscht werden konnten. Einzelheiten finden Sie im Inhalt des CloudWatch Protokolls für den Synchronisationsjob.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl

Metriken für Amazon Kendra Kendra-Datenquellen

Die folgende Tabelle beschreibt die Amazon-Kendra-Metriken für Datenquellensynchronisationsaufträge. Mit einem Sternchen (*) markierte Metriken werden nur für Amazon-S3-Datenquellen verwendet.

Metrik Beschreibung
DocumentsSkippedNoChange *

Die Anzahl der geprüften Dokumente, bei denen festgestellt wurde, dass sie sich nicht geändert haben, sodass sie nicht zur Indexierung eingereicht wurden.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSkippedInvalidMetadata *

Die Anzahl der Dokumente, die übersprungen wurden, weil ein Problem mit der zugehörigen Metadatendatei aufgetreten ist. Einzelheiten finden Sie im Inhalt des CloudWatch Protokolls für den Synchronisationslauf.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsCrawled

Die Anzahl der untersuchten Dokumentdateien.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSubmittedForDeletion

Die Anzahl der untersuchten Dokumente, die aus der Datenquelle gelöscht und zur Löschung eingereicht wurden.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSubmittedForDeletionFailed

Die Anzahl der Dokumente, deren Löschung aus einer Datenquelle fehlgeschlagen ist.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSubmittedForIndexing

Die Anzahl der geprüften und zur Indexierung eingereichten Dokumente.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsSubmittedForIndexingFailed

Die Anzahl der zur Indizierung eingereichten Dokumente, die nicht indexiert werden konnten.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl

Metriken für indexierte Dokumente

Die folgende Tabelle beschreibt die Amazon-Kendra-Metriken für indizierte Dokumente. Für Dokumente, die mithilfe derBatchPutDocument Operation indexiert werden, wird nur dieIndexId Dimension unterstützt.

Metrik Beschreibung
DocumentsIndexed

Die Anzahl der indexierten Dokumente.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
DocumentsFailedToIndex

Die Anzahl der Dokumente, die nicht indexiert werden konnten. Einzelheiten finden Sie im Inhalt des CloudWatch Protokolls.

Dimensionen:

  • IndexId

  • DataSourceId

Einheit: Anzahl
IndexQueryCount

Die Anzahl der Indexabfragen pro Minute.

Dimensionen:

  • IndexId

Einheit: Anzahl
