Amazon WorkDocs

Amazon WorkDocsist ein sicherer Service für die Zusammenarbeit von Inhalten zum Erstellen, Bearbeiten, Speichern und Teilen von Inhalten. Sie können es verwendenAmazon Kendra, um IhreAmazon WorkDocs Datenquelle zu indexieren.

Sie können mithilfe der Amazon KendraKonsole und der WorkDocsConfigurationAPI eine VerbindungAmazon Kendra zu IhrerAmazon WorkDocs Datenquelle herstellen.

Amazon WorkDocsist in den Regionen Oregon, Nord-Virginia, Sydney, Singapur und Irland verfügbar.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra WorkDocs Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Voraussetzungen

Bevor Sie Ihre WorkDocs DatenquelleAmazon Kendra zum Indizieren verwenden können, nehmen Sie diese Änderungen in Ihren WorkDocs undAWS Konten vor.

In WorkDocs, stellen Sie sicher, dass Sie über Folgendes verfügen:

  • Habe dieAmazon WorkDocs Verzeichnis-ID (Organisations-ID) für IhrAmazon WorkDocs Repository notiert.

  • Hat geprüft, dass jedes Dokument in WorkDocs und zwischen anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

Stellen Sie in IhremAWS Konto sicher, dass Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

Wenn Sie keine vorhandeneIAM Rolle haben, können Sie die Konsole verwenden, um eine neueIAM Rolle zu erstellen, wenn Sie Ihre WorkDocs Datenquelle mit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und eine Index-ID angeben.

Verbindungsanweisungen

Um eine VerbindungAmazon Kendra zu Ihrer WorkDocs Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer WorkDocs Datenquelle angeben, damit auf Ihre Daten zugegriffen werdenAmazon Kendra kann. Wenn Sie es noch nicht konfiguriert WorkDocs haben,Amazon Kendra sehen SieVoraussetzungen.

Console

Um eine VerbindungAmazon Kendra herzustellenAmazon WorkDocs

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich Indizes und dann in der Liste der Indizes den Index aus, den Sie verwenden möchten.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option WorkDocs Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Datenquellendetails spezifizieren -Datenquellendetails die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags für Neues Tag hinzufügen —Tags können Sie Ressourcen suchen und filtern oder IhreAWS -Kosten verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. Spezifische Organisations-ID für IhreAmazon WorkDocs Website — Wählen Sie die ID derAmazon WorkDocs Website aus, die Sie indexieren möchten. Sie müssen bereits eine Site erstellt haben.

    2. IAM-Rolle —Wählen Sie eine vorhandene IAM-Rolle oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    3. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen konfigurieren-Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Dokumentkommentare durchforsten — DieAmazon WorkDocs Entitäten oder Inhaltstypen, die Sie crawlen möchten.

    2. Änderungsprotokolle verwenden — Wählen Sie diese Option aus, um Ihren Index zu aktualisieren, anstatt all Ihre Dateien zu synchronisieren.

    3. Regex-Muster — Reguläre Ausdrucksmuster zum Ein- oder Ausschließen bestimmter Dateien. Sie können bis zu 100 Muster hinzufügen.

    4. Wählen Sie unter Synchronisierungslaufzeitplan für Frequenz aus, wie oftAmazon Kendra die Synchronisierung mit Ihrer Datenquelle erfolgen soll.

    5. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen festlegen die folgenden Informationen ein:

    1. Standarddatenquellenfelder — Wählen Sie eines derAmazon Kendra generierten Standarddatenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

Um eine VerbindungAmazon Kendra herzustellenAmazon WorkDocs

Sie müssen mithilfe der WorkDocsConfigurationAPI Folgendes angeben:

  • Amazon WorkDocsVerzeichnis-ID —Geben Sie die Organisations-ID IhresAmazon WorkDocs Verzeichnisses an. Sie finden die Organisations-ID im AWS Directory Service, indem Sie zu Active Directory und dann zu Verzeichnissen gehen.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf das WorkDocs Verzeichnis und den Aufruf der erforderlichen öffentlichen APIs für den WorkDocs Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für WorkDocs Datenquellen.

Sie können auch die folgenden optionalen-Funktionen hinzufügen:

  • Änderungsprotokoll — Gibt an, ob der Änderungsprotokollmechanismus der WorkDocs Datenquelle verwendet werdenAmazon Kendra soll, um zu bestimmen, ob ein Dokument dem Index hinzugefügt, aktualisiert oder gelöscht werden muss.

    Anmerkung

    Verwenden Sie das Änderungsprotokoll, wenn Sie nicht alle Dokumente scannen möchtenAmazon Kendra. Wenn Ihr Änderungsprotokoll groß ist, dauert das Scannen der Dokumente in der WorkDocs Datenquelle möglicherweiseAmazon Kendra weniger Zeit als das Verarbeiten des Änderungsprotokolls. Wenn Sie Ihre WorkDocs Datenquelle zum ersten Mal mit Ihrem Index synchronisieren, werden alle Dokumente gescannt.

  • Ein- und Ausschlussfilter — Geben Sie an, ob bestimmte Dokumente und Dokumentkommentare ein- oder ausgeschlossen werden sollen. Jeder Kommentar wird als separates Dokument indexiert.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre WorkDocs Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

  • BenutzerkontextfilterungAmazon Kendra — durchsucht standardmäßig die Access Control List (ACL) für Ihre Datenquelle. Die ACL-Informationen werden verwendet, um Suchergebnisse auf der Grundlage des Zugriffs des Benutzers oder seiner Gruppe auf Dokumente zu filtern. Weitere Informationen finden Sie unter Benutzerkontextfilterung für WorkDocs Datenquellen.
