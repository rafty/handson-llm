Zugewiesene Datenquellen-Felder

Sie können Dokument- oder Inhaltsfelder aus Ihrer Datenquelle Feldern in Ihrem Index zuordnen. Wenn Sie beispielsweise ein Feld in Ihrer Datenquelle mit dem Namen „Abteilung“ haben, das Abteilungsinformationen für ein Dokument enthält, können Sie es einem Indexfeld namens „Abteilung“ zuordnen. Auf diese Weise können Sie das Feld verwenden, wenn Sie Dokumente abfragen.

Sie können auch Standardfelder oder reservierte Felder zuordnen, z._created_at B. Wenn Ihre Datenquelle ein Feld mit dem Namen „creation_date“ hat, können Sie dieses dem entsprechendenAmazon Kendra reservierten Feld namens zuordnen_created_at. Sie können Felder für die meisten Datenquellen zuordnen.

Sie können Feldzuordnungen für die folgenden Datenquellen erstellen:

  • Im Freien

  • Datenbank

  • Amazon FSx

  • Amazon WorkDocs

  • Box (Kasten)

  • Confluence

  • Dropbox

  • GitHub

  • Google Workspace Drives

  • Gmail

  • Jira

  • Microsoft Exchange

  • Microsoft OneDrive

  • Microsoft SharePoint

  • Microsoft-Teams

  • Microsoft Yammer

  • Witz

  • Salesforce

  • ServiceNow

  • Slack

  • Zendesk

Wenn Sie Ihre Dokumente in einem S3-Bucket oder einer S3-Datenquelle speichern, können Sie benutzerdefinierte Attribute direkt mithilfe von Metadatendateien angeben. Weitere Informationen finden Sie unter Erstellen benutzerdefinierter Dokumentfelder.

Das Zuordnen Ihrer Datenquellenfelder zu einem Indexfeld erfolgt in drei Schritten:

  1. Erstellen Sie einen Index. Weitere Informationen finden Sie unter Erstellen eines Index.

  2. Aktualisieren Sie den Index, um benutzerdefinierte Felder hinzuzufügen.

  3. Erstellen Sie eine Datenquelle, die Datenquellenfelder den Indexfeldern zuordnet.

Verwenden Sie die Konsole oder dieUpdateIndex API, um den Index zu aktualisieren, um benutzerdefinierte Felder hinzuzufügen. Sie können Ihrem Index insgesamt 500 benutzerdefinierte Felder hinzufügen.

In der Konsole können Sie wählen, ob Sie ein Datenquellenfeld einem der reservierten Feldnamen zuordnen möchten. Sie können sich auch dafür entscheiden, ein neues Indexfeld zu erstellen, das dem Feld zugeordnet ist. Wenn bei Datenbankdatenquellen der Name der Datenbankspalte mit dem Namen eines reservierten Felds übereinstimmt, werden das Feld und die Spalte automatisch zugeordnet.

Mit der UpdateIndexAPI fügen Sie mithilfe desDocumentMetadataConfigurationUpdates Parameters benutzerdefinierte und reservierte Felder hinzu.

Das folgende JSON-Beispiel verwendetDocumentMetadataConfigurationUpdates, um dem Index ein Feld mit dem Namen „Department“ hinzuzufügen.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Wenn Sie das Feld erstellen, haben Sie die Möglichkeit, festzulegen, wie das Feld bei Suchen verwendet wird. Sie können aus den folgenden Optionen auswählen:

  • Displayable —Bestimmt, ob das Feld in der Abfrageantwort zurückgegeben wird. Der Standardwert ist true.

  • Facettable —Gibt an, dass das Feld zum Erstellen von Facetten verwendet werden kann. Der Standardwert ist false.

  • Durchsuchbar —Bestimmt, ob das Feld bei der Suche verwendet wird. Die Standardeinstellung bei Zeichenfolgenfeldern ist true und bei Zahlen- und Datumsfeldern false.

  • Sortierbar —Gibt an, dass das Feld zum Sortieren der Antwort aus einer Abfrage verwendet werden kann. Kann nur für Datums-, Zahlen- und Zeichenfolgenfelder festgelegt werden. Kann nicht für Zeichenkettenlistenfelder festgelegt werden.

Das folgende JSON-Beispiel verwendetDocumentMetadataConfigurationUpdates, um dem Index ein Feld mit dem Namen „Department“ hinzuzufügen und es als Facetable zu markieren.

"DocumentMetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE",
       "Search": {
           "Facetable": true
       }
   }
]

Verwenden der integrierten Dokumentfelder von Amazon Kendra

Mit der UpdateIndex API können Sie reservierte oder integrierte Felder erstellen, indem Sie den reservierten Feldnamen verwendenDocumentMetadataConfigurationUpdates und angeben, um ihn Ihrem entsprechenden Dokumentattribut oder Dokumentfeld zuzuordnen. Auf diese Weise können Sie auch benutzerdefinierte Felder erstellen. Wenn Sie einen Datenquellenconnector verwenden, enthalten die meisten Feldzuordnungen, die Ihre DatenquellendokumentfelderAmazon Kendra Indexfeldern zuordnen. Wenn Sie ein Feld erstellen, können Sie dasSearch Objekt so konfigurieren, dass das Feld als anzeigbar, facettierbar, durchsuchbar und sortierbar festgelegt wird. Sie können dasRelevance Objekt so konfigurieren, dass es die Werte für die Dauer, Aktualität, Wichtigkeit, Rangfolge und Wichtigkeit des Felds festlegt, um die Relevanz zu erhöhen. Sie können den Feldtyp nicht mehr ändern, nachdem Sie das Feld erstellt haben.

Amazon Kendraverfügt über die folgenden reservierten oder integrierten Dokumentfelder, die Sie verwenden können:

  • _authors— Eine Liste von einem oder mehreren Autoren, die für den Inhalt des Dokuments verantwortlich sind.

  • _category— Eine Kategorie, die ein Dokument in eine bestimmte Gruppe einordnet.

  • _created_at— Datum und Uhrzeit im ISO 8601-Format, zu denen das Dokument erstellt wurde. Beispiel: 2012-03-25T12:30:10+01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in mitteleuropäischer Zeit (CET).

  • _data_source_id— Der Bezeichner der Datenquelle, die das Dokument enthält.

  • _document_body— Der Inhalt des Dokuments.

  • _document_id— Eindeutiger Bezeichner für das Dokument.

  • _document_title— Der Titel des Dokuments.

  • _excerpt_page_number— Die Seitenzahl in einer PDF-Datei, auf der der Dokumentauszug erscheint. Wenn Ihr Index vor dem 8. September 2020 erstellt wurde, müssen Sie Ihre Dokumente neu indizieren, bevor Sie dieses Attribut verwenden können.

  • _faq_id—Wenn es sich um eine häufig gestellte Frage und Antwort handelt, eine eindeutige Kennung für sie.

  • _file_type— Der Dateityp des Dokuments, z. B. PDF oder Dokument.

  • _last_updated_at— Datum und Uhrzeit im Format ISO 8601, zu der das Dokument zuletzt aktualisiert wurde. Beispiel: 2012-03-25T12:30:10+01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in mitteleuropäischer Zeit (CET).

  • _source_uri— Die URI, unter der das Dokument verfügbar ist. Zum Beispiel die URI des Dokuments auf einer Unternehmenswebsite.

  • _version— Ein Bezeichner für die spezifische Version eines Dokuments.

  • _view_count— Wie oft das Dokument angesehen wurde.

  • _language_code(String) — Der Code für eine Sprache, die für das Dokument gilt. Standardmäßig verwendet Englisch, wenn Sie keine Sprache angeben. Weitere Informationen zu unterstützten Sprachen, einschließlich ihrer Codes, finden Sie unter Hinzufügen von Dokumenten in anderen Sprachen als Englisch.

Für benutzerdefinierte Felder erstellen Sie diese FelderDocumentMetadataConfigurationUpdates mithilfe derUpdateIndex API, genau wie Sie es beim Erstellen eines reservierten oder integrierten Felds tun. Sie müssen den entsprechenden Datentyp für Ihr benutzerdefiniertes Feld festlegen. Sie können den Feldtyp nicht mehr ändern, nachdem Sie das Feld erstellt haben.

Die folgenden Typen können Sie für benutzerdefinierte Felder festlegen:

  • Datum

  • Zahl

  • Zeichenfolge

  • Zeichenfolgenliste

Sie erstellen ein benutzerdefiniertes Feld mithilfe der Konsole oder mithilfe der UpdateIndexAPI. Nachdem Sie ein benutzerdefiniertes Feld erstellt haben, ordnen Sie es einem Dokumentattribut zu, genau wie Sie es bei einem reservierten Feld tun. Wenn Sie ein Dokument mit BatchPutDocumentAPI zum Index hinzugefügt haben, ordnen Sie die Attribute der API zu. Für Dokumente, die aus einerAmazon S3 Datenquelle indexiert wurden, ordnen Sie die Attribute mithilfe einer Metadatendatei zu, die eine JSON-Struktur enthält, die die Dokumentattribute beschreibt. Für Dokumente, die mit einer Datenbank oder einer Datenquelle indexiert sind, die eine Feldzuordnung ermöglicht, ordnen Sie Attribute der Konsole oder der Datenquellenkonfiguration zu.
