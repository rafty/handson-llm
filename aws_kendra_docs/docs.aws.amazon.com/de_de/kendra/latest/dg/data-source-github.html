GitHub

GitHub ist ein webbasierter Hosting-Dienst für Softwareentwicklung, der Codespeicher- und Verwaltungsdienste mit Versionskontrolle bereitstellt. Sie könnenAmazon Kendra damit Ihre Repository-Dateien für GitHub Enterprise Cloud (SaaS) und GitHub Enterprise Server (On Prem), Issue- und Pull-Requests, Issue- und Pull-Request-Kommentare sowie Anhänge zu Issue- und Pull-Request-Kommentaren indexieren. Sie können auch bestimmte Dateien ein- oder ausschließen.

Sie können mithilfe der Amazon KendraKonsole und der GitHubConfigurationAPI eine VerbindungAmazon Kendra zu Ihrer GitHub Datenquelle herstellen.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra GitHub Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Unterstützte Funktionen

Amazon Kendra GitHub Der Datenquellenconnector unterstützt die folgenden Funktionen:

  • Änderungsprotokoll

  • Feldzuordnungen

  • Filterung des Benutzerkontextes

  • Einschluss-/Ausschlussfilter

Voraussetzungen

Bevor Sie Ihre GitHub DatenquelleAmazon Kendra zum Indizieren verwenden können, nehmen Sie diese Änderungen in Ihren GitHub undAWS Konten vor.

In GitHub, stellen Sie sicher, dass Sie:

  • Hat einen GitHub Benutzer mit Administratorrechten für die GitHub Organisation erstellt.

  • Es wurde ein persönliches Zugriffstoken für Authentifizierungsdaten erstellt. Weitere Informationen finden Sie in derGitHub Dokumentation zum Erstellen eines persönlichen Zugriffstokens.

  • Empfehlung: Es wurde ein OAuth-Token für Authentifizierungsdaten erstellt. Verwenden Sie das OAuth-Token für bessere API-Drossellimits und eine bessere Connector-Leistung. Weitere Informationen finden Sie in derGitHub Dokumentation zur OAuth-Autorisierung.

  • Optional: Es wurde ein SSL-Zertifikat installiert.

  • Habe die GitHub Host-URL für den Typ des GitHub Dienstes, den Sie verwenden, notiert. Beispielsweise könnte die Host-URL für die GitHub Cloud https://api.github.com und die Host-URL für den GitHub Server https://on-prem-host-url/api/v3/ lauten.

  • Den Namen der GitHub Organisation für Ihre Repositorys habe ich in Ihren GitHub Einstellungen notiert.

  • Folgende Berechtigungen wurden hinzugefügt:

    Für GitHub Enterprise Cloud (SaaS)

    • Repo:Status

    • public_repo

    • Repo: einladen

    • lesen:org

    • Benutzer:E-Mail

    • lesen:Benutzer

    Für GitHub Enterprise Server (On-Prem)

    • Repo:Status

    • public_repo

    • Repo: einladen

    • lesen:org

    • Benutzer:E-Mail

    • lesen:Benutzer

    • site_admin

  • Hat geprüft, dass jedes Dokument in GitHub und zwischen anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

Stellen Sie in IhremAWS Konto sicher, dass Sie über:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Ihre GitHub Authentifizierungsdaten wurdenAWSSecrets Manager geheim gespeichert und, falls Sie die API verwenden, den ARN des Geheimnisses notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder kein Geheimnis haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn Sie Ihre GitHub Datenquelle damit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Anweisungen zum Anschluss

Um eine VerbindungAmazon Kendra zu Ihrer GitHub Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer GitHub Datenquelle angeben, damit auf Ihre Daten zugegriffen werdenAmazon Kendra kann. Wenn Sie es noch nicht konfiguriert GitHub haben,Amazon Kendra sehen SieVoraussetzungen.

Console

Um eine VerbindungAmazon Kendra herzustellen GitHub

  1. Melden Sie sich bei derAWS Amazon Kendra-Managementkonsole an.

  2. Wählen Sie im linken Navigationsbereich Indizes aus der Liste der Indizes aus.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option GitHub Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Datenquellendetails die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags für Neues Tag hinzufügen —Tags können Sie Ressourcen suchen und filtern oder IhreAWS -Kosten verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. GitHubQuelle —Wählen Sie zwischen GitHub Enterprise Cloud und GitHubEnterprise Server.

    2. GitHub Host-URL —Geben Sie Ihren GitHub Hostnamen ein.

    3. GitHub Name der Organisation —Geben Sie den Namen Ihrer GitHub Organisation ein. Sie finden Ihre Organisationsinformationen in Ihrem GitHub Konto.

    4. AWSSecrets Managersecret —Wählen Sie ein vorhandenes Geheimnis aus, oder erstellen Sie ein neuesSecrets Manager Geheimnis, um Ihre GitHub Authentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-GitHub -' wird automatisch zu Ihrem geheimen Namen hinzugefügt.

        2. Für GitHubToken — Geben Sie die Werte der Authentifizierungsdaten ein, die Sie in GitHub Ihrem Konto erstellt haben.

      2. Wählen Sie Speichern.

    5. Virtual Private Cloud (VPC) — Sie können eine VPC verwenden. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    6. IAM-Rolle —Wählen Sie eine vorhandene IAM-Rolle aus IAM-Rolle aus, um auf Ihre Repository-Anmeldeinformationen und Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    7. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Wählen Sie die zu durchforstenden Repositorys aus — Die GitHub Entitäten oder Inhaltstypen, die Sie crawlen möchten.

    2. Änderungsprotokoll — Wählen Sie diese Option aus, um Ihren Index zu aktualisieren, anstatt alle Ihre Dateien zu synchronisieren.

    3. Inhaltstypen — Wählen Sie die Dateitypen aus, die Sie einbeziehen möchten.

    4. Regex-Muster — Reguläre Ausdrucksmuster zum Ein- oder Ausschließen bestimmter Dateien. Sie können bis zu 100 Muster hinzufügen.

    5. In Sync run schedule, für Frequenz — Wie oft Amazon Kendra mit Ihrer Datenquelle synchronisiert wird.

    6. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen festlegen die folgenden Informationen ein:

    1. Für Repository, Repository-Commit, Issue Document, Issue Comment, Issue Attachment, Pull Request Comment, Pull Request Document, Pull Request Attachment — Wählen Sie eines derAmazon Kendra generierten Standard-Datenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können auch Ihre Informationen von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

Um eine VerbindungAmazon Kendra herzustellen GitHub

Mit dem GitHubConfigurationAPI-Objekt müssen Sie Folgendes angeben:

  • Datenquellentyp —Geben Sie den Datenquellentyp entwederSAAS oder anON_PREMISE.

  • Geheimer Amazon-Ressourcenname (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr GitHub Konto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "personalToken": "token"
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufenCreateDataSource, um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager geheimes Element und zum Aufrufen der erforderlichen öffentlichen APIs für den GitHub Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für GitHub Datenquellen.

Sie können auch die folgenden optionalen-Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie dies imVpcConfiguration Rahmen der Datenquellenkonfiguration an. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

    Anmerkung

    Wenn Sie einen GitHub Server verwenden, müssen Sie einen verwenden,Amazon VPC um eine Verbindung zu Ihrem GitHub Server herzustellen.

  • Änderungsprotokoll — Gibt an, ob der Änderungsprotokollmechanismus der GitHub Datenquelle verwendet werdenAmazon Kendra soll, um zu bestimmen, ob ein Dokument dem Index hinzugefügt, aktualisiert oder gelöscht werden muss.

    Anmerkung

    Verwenden Sie das Änderungsprotokoll, wenn Sie nicht alle Dokumente scannen möchtenAmazon Kendra. Wenn Ihr Änderungsprotokoll groß ist, dauert das Scannen der Dokumente in der GitHub Datenquelle möglicherweiseAmazon Kendra weniger Zeit als das Verarbeiten des Änderungsprotokolls. Wenn Sie Ihre GitHub Datenquelle zum ersten Mal mit Ihrem Index synchronisieren, werden alle Dokumente gescannt.

  • Ein- und Ausschlussfilter — Geben Sie an, ob bestimmte Dateien ein- oder ausgeschlossen werden sollen.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre GitHub Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

  • BenutzerkontextfilterungAmazon Kendra — durchsucht standardmäßig die Access Control List (ACL) für Ihre Datenquelle. Die ACL-Informationen werden verwendet, um Suchergebnisse auf der Grundlage des Zugriffs des Benutzers oder seiner Gruppe auf Dokumente zu filtern. Weitere Informationen finden Sie unter Benutzerkontextfilterung für GitHub Datenquellen.
