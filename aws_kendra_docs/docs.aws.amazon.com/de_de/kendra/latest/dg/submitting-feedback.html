Feedback für inkrementelles Lernen einreichen

Amazon Kendranutzt inkrementelles Lernen, um die Suchergebnisse zu verbessern. Mithilfe des Feedbacks aus Abfragen verbessert das inkrementelle Lernen die Ranking-Algorithmen und optimiert die Suchergebnisse für eine höhere Genauigkeit.

Nehmen wir beispielsweise an, dass Ihre Benutzer enthalten das Wort „Leistungen im Gesundheitswesen“. Wenn Benutzer konsequent das zweite Ergebnis aus der Liste auswählen, wird dieses Ergebnis im Laufe der Zeit auf den ersten PlatzAmazon Kendra erhöht. Der Boost nimmt mit der Zeit ab. Wenn Benutzer also aufhören, ein Ergebnis auszuwählen, wird esAmazon Kendra irgendwann entfernt und stattdessen wird ein anderes, beliebteres Ergebnis angezeigt. Dies hilft dabei, Ergebnisse nach Relevanz, Alter und Inhalt zuAmazon Kendra priorisieren.

Inkrementelles Lernen ist für alle Indizes und für alle unterstützten Dokumenttypen aktiviert.

Amazon Kendrabeginnt zu lernen, sobald Sie Feedback geben. Es kann jedoch über 24 Stunden dauern, bis die Ergebnisse des Feedbacks sichtbar sind. Amazon Kendrabietet drei Methoden, mit denen Sie Feedback einreichen können: dieAWS Konsole, eine JavaScript Bibliothek, die Sie in Ihre Suchergebnisseite aufnehmen können, und eine API, die Sie verwenden können.

Amazon Kendraakzeptiert zwei Arten von Benutzerfeedback:

  • Klicks — Informationen darüber, welche Abfrageergebnisse der Benutzer ausgewählt hat. Das Feedback enthält die Ergebnis-ID und den Unix-Zeitstempel des Datums und der Uhrzeit der Auswahl des Suchergebnisses.

    Um Klick-Feedback zu senden, muss Ihre Anwendung Klickinformationen aus den Aktivitäten Ihrer Benutzer sammeln und diese Informationen dann an sendenAmazon Kendra. Sie können Klickinformationen mit der Konsole, der JavaScript Bibliothek und derAmazon Kendra API sammeln.

  • Relevanz — Informationen über die Relevanz eines Suchergebnisses, die der Benutzer in der Regel bereitstellt. Das Feedback enthält die Ergebnis-ID und einen Relevanzindikator (RELEVANToderNOT_RELEVANT). Der Benutzer bestimmt die Relevanzinformationen.

    Um Feedback zur Relevanz einzureichen, muss Ihre Anwendung über einen Feedback-Mechanismus verfügen, der es dem Benutzer ermöglicht, die entsprechende Relevanz für ein Abfrageergebnis auszuwählen und diese Informationen dann an zu sendenAmazon Kendra. Sie können Relevanzinformationen nur mit der Konsole und derAmazon Kendra API sammeln.

Feedback wird verwendet, solange der Index aktiv ist. Feedback wirkt sich nur auf den Index aus, an den es gesendet wurde. Es kann nicht indexübergreifend oder für verschiedene Konten verwendet werden.

Sie sollten zusätzlichen Benutzerkontext angeben, wenn Sie IhrenAmazon Kendra Index abfragen. Wenn Sie den Benutzerkontext angeben,Amazon Kendra kann er erkennen, ob das Feedback von einem einzelnen Benutzer oder von mehreren Benutzern stammt, und die Suchergebnisse entsprechend anpassen.

Wenn Sie den Benutzerkontext angeben, wird das Feedback für die Abfrage dem spezifischen Benutzer zugeordnet, der im Kontext angegeben wurde. Wenn Sie keinen Benutzerkontext angeben, können Sie eine Besucher-ID angeben, die zum Gruppieren und Aggregieren von Abfragen verwendet wird.

Wenn Sie keinen Benutzerkontext oder keine Besucher-ID angeben, ist das Feedback anonym und wird mit anderem anonymen Feedback aggregiert.

Der folgende Code zeigt, wie der Benutzerkontext als Token oder die Besucher-ID eingebunden wird.

response = kendra.query(
    QueryText = query,
    IndexId = index,
    UserToken = {
        Token = "token"
    })
    
    OR
    
    response = kendra.query(
    QueryText = query,
    IndexId = index,
    VisitorId = "visitor-id")

Für Webanwendungen können Sie Cookies, Standorte oder Browserbenutzer verwenden, um für jeden Benutzer eine Besucher-ID zu generieren.

Bei Kopfabfragen, der größten Anzahl von Abfragen, bietet die Bereitstellung von Click-Through-Feedback genügend Informationen, um die allgemeine Genauigkeit zu verbessern. Bei Anfragen, die selten sind, sollten Fachexperten relevante und nicht relevante Rückmeldungen einreichen, um die Genauigkeit dieser Anfragen zu verbessern.

Zusätzlich zur Konsole können Sie eine von zwei Methoden verwenden: eine JavaScript Bibliothek oder dieSubmitFeedback API. Sie sollten nur eine Methode verwenden, um Feedback zu sammeln. Um optimale Ergebnisse zu erzielen, sollten Sie innerhalb von 24 Stunden nach der Anfrage Feedback einreichen.
