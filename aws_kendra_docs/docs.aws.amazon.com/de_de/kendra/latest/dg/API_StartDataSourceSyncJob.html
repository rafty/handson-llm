StartDataSourceSyncJob

Startet einen Synchronisations-Service für einen Datenquellenkonnektor. Wenn bereits ein Synchronisierungsauftrag ausgeführt wird, gibt Amazon Kendra eineResourceInUseException Ausnahme zurück.

Anforderungssyntax

{
   "Id": "string",
   "IndexId": "string"
}

Anfrageparameter

Informationen zu den Parametern, die allen Aktionen gemeinsam sind, finden Sie unter Allgemeine Parameter.

Die Anforderung akzeptiert die folgenden Daten im JSON-Format.

Id

Der Bezeichner des zu synchronisierenden Datenquellenkonnektor.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 100 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Erforderlich: Ja

IndexId

Der Bezeichner des Index, der für den Datenquellenkonnektor definierten Index.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge von 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Erforderlich: Ja

Antwortsyntax

{
   "ExecutionId": "string"
}

Antwortelemente

Wenn die Aktion erfolgreich ist, sendet der Service eine HTTP 200-Antwort zurück.

Die folgenden Daten werden vom Service im JSON-Format zurückgegeben.

ExecutionId

Identifiziert einen bestimmten Synchronisationsjob.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Maximale Länge beträgt 2048 Zeichen.

Fehler

Hinweise zu den Fehlern, die allen Aktionen gemeinsam sind, finden Sie unterHäufige Fehler.

AccessDeniedException

Sie haben keinen ausreichenden Zugriff, um diese Aktion auszuführen. Bitte stellen Sie sicher, dass Sie über die erforderlichen Berechtigungsrichtlinien und Benutzerkonten verfügen, und versuchen Sie es erneut.

HTTP Status Code: 400

ConflictException

Bei der Anfrage ist ein Konflikt aufgetreten. Bitte korrigieren Sie alle Unstimmigkeiten mit Ihren Ressourcen und versuchen Sie es erneut.

HTTP Status Code: 400

InternalServerException

Es ist ein Problem mit dem internen Server aufgetreten, der für Ihren Amazon Kendra-Service verwendet wird. Bitte warten Sie ein paar Minuten und versuchen Sie es erneut, oder wenden Sie sich an den Support, um Hilfe zu erhalten.

HTTP Status Code: 500

ResourceInUseException

Die Ressource, die Sie verwenden möchten, wird derzeit verwendet. Bitte überprüfen Sie, ob Sie die richtige Ressource angegeben haben, und versuchen Sie es erneut.

HTTP Status Code: 400

ResourceNotFoundException

Die Ressource, die Sie verwenden möchten, existiert nicht. Bitte überprüfen Sie, ob Sie die richtige Ressource angegeben haben, und versuchen Sie es erneut.

HTTP Status Code: 400

ThrottlingException

Die Anforderung wurde aufgrund der Drosselung von Anforderungen abgelehnt. Bitte reduzieren Sie die Anzahl der Anfragen und versuchen Sie es erneut.

HTTP Status Code: 400

ValidationException

Die Eingabe erfüllt nicht die von dem -Service Amazon Kendra Einschränkungen. Bitte geben Sie die korrekte Eingabe ein und versuchen Sie es erneut.

HTTP Status Code: 400
