Amazon KendraIntelligentes Ranking für Selbstverwalter OpenSearch

Sie können die semantischen Suchfunktionen nutzenAmazon Kendra, um die Suchergebnisse von OpenSearch, dem selbstverwalteten Open-Source-Suchdienst, der auf der Apache 2.0-Lizenz basiert, zu verbessern. DasAmazon Kendra Intelligent Ranking-Plugin ordnet die Ergebnisse anhand OpenSearch von semantisch neu einAmazon Kendra. Dies geschieht, indem die Bedeutung und der Kontext einer Suchabfrage anhand bestimmter Felder, z. B. des Hauptteils oder des Titels des Dokuments, anhand der OpenSearch Standardsuchergebnisse verstanden werden.

Nehmen wir zum Beispiel diese Abfrage: „Haupt-Keynote-Adresse“. Da „Adresse“ mehrere Bedeutungen hat,Amazon Kendra kann ich die Bedeutung hinter der Anfrage ableiten, um relevante Informationen zurückzugeben, die der beabsichtigten Bedeutung entsprechen. In diesem Zusammenhang ist es eine Keynote-Rede auf der Konferenz. Ein einfacherer Suchdienst berücksichtigt die Absicht möglicherweise nicht und könnte möglicherweise Ergebnisse für eine Straßenadresse an der Main Street zurückgeben, beispielsweise.

Das Intelligent Ranking-Plugin für OpenSearch ist für die OpenSearch (selbstverwaltete) Version 2.4.0 und höher verfügbar. Sie können das Plugin mit einem Schnellstart-Bash-Skript installieren, um ein neues Docker-Image von zu erstellen, in OpenSearch dem das Intelligent Ranking-Plugin enthalten ist. Sehen SieEinrichten des intelligenten Suchfeld — dies ist ein Beispiel für ein Setup, mit dem Sie schnell loslegen können.

So funktioniert das intelligente Such-Plugin

Der Gesamtprozess des Intelligent Ranking-Plugins für OpenSearch (selbst verwaltet) sieht wie folgt aus:

  1. Ein OpenSearch Benutzer stellt eine Abfrage und OpenSearch stellt eine Abfrageantwort oder eine Liste von Dokumenten bereit, die für die Abfrage relevant sind.

  2. Das Intelligent Ranking-Plugin nimmt die Abfrageantwort und extrahiert Informationen aus den Dokumenten.

  3. Das Intelligent Ranking-Plugin ruft die Rescore-API vonAmazon Kendra Intelligent Ranking auf.

  4. DieRescore API nimmt die extrahierten Informationen aus den Dokumenten und ordnet die Suchergebnisse semantisch neu.

  5. DieRescore API sendet die neu eingestuften Suchergebnisse zurück an das Plugin. Das Plugin ordnet die Suchergebnisse in der OpenSearch Suchantwort neu an, um das neue semantische Ranking widerzuspiegeln.

Das Intelligent Ranking-Plugin ordnet die Ergebnisse anhand der Felder „body“ und „title“ neu ein. Diese Plugin-Felder können Feldern in Ihrem OpenSearch Index zugeordnet werden, die am besten zur Definition eines Textteils und Titels eines Dokuments passen. Wenn Ihr Index beispielsweise Kapitel eines Buches mit Feldern wie „Kapitelüberschrift“ und „Kapitelinhalt“ enthält, können Sie ersteres dem Titel und letzteres dem Text „Hauptteil“ zuordnen, um die besten Ergebnisse zu erzielen.

    import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra_ranking = boto3.client("kendra-ranking")

print("Create a rescore execution plan.")

# Provide a name for the rescore execution plan
name = "MyRescoreExecutionPlan"
# Set your required additional capacity units
# Don't set capacity units if you don't require more than 1 unit given by default
capacity_units = 1

try:
    rescore_execution_plan_response = kendra_ranking.create_rescore_execution_plan(
        Name = name,
        CapacityUnits = {"RescoreCapacityUnits":capacity_units}
    )

    pprint.pprint(rescore_execution_plan_response)

    rescore_execution_plan_id = rescore_execution_plan_response["Id"]

    print("Wait for Amazon Kendra to create the rescore execution plan.")

    while True:
        # Get the details of the rescore execution plan, such as the status
        rescore_execution_plan_description = kendra_ranking.describe_rescore_execution_plan(
            Id = rescore_execution_plan_id
        )
        # When status is not CREATING quit.
        status = rescore_execution_plan_description["Status"]
        print(" Creating rescore execution plan. Status: "+status)
        time.sleep(60)
        if status != "CREATING":
            break

except ClientError as e:
        print("%s" % e)

print("Program ends.")

Interaktion mit dem intelligenten Such-Plugin

Sobald Sie das Intelligent Ranking-Plugin eingerichtet OpenSearch (selbst verwaltet) haben, können Sie mit dem Plugin mithilfe von Curl-Befehlen oder OpenSearch Client-Bibliotheken interagieren. Die Standardanmeldeinformationen für den Zugriff OpenSearch mit dem Intelligent Ranking-Plugin sind der Benutzername „admin“ und das Passwort „admin“.

So wenden Sie die Intelligent Ranking-Plug-In-Einstellungen auf einen OpenSearch Index an:

OpenSearch Ergebnisse mitAmazon Kendra Ergebnissen vergleichen

Sie können die side-by-side OpenSearch (selbst verwalteten) Ranglisten mit den ErgebnissenAmazon Kendra der neu eingestuften Rangliste vergleichen. OpenSearch Dashboards Version 2.4.0 und höher bieten side-by-side Ergebnisse, sodass Sie die OpenSearch Rangfolge von Dokumenten mit der Reihenfolge vergleichen können, wieAmazon Kendra oder das Plugin Dokumente für eine Suchabfrage einstuft.

Bevor Sie OpenSearch Ranglistenergebnisse mit Ergebnissen mitAmazon Kendra neu eingestuften Ergebnissen vergleichen können, stellen Sie sicher, dass Ihre OpenSearch Dashboards von einem OpenSearch Server mit dem Intelligent Ranking-Plugin unterstützt werden. Sie können dies mit Docker und einem Schnellstart-Bash-Skript einrichten. Siehe Einrichten des intelligenten Suchfeld.

Im Folgenden wird beschrieben, wie Sie Ergebnisse in OpenSearch Dashboards vergleichen OpenSearch undAmazon Kendra suchen. Weitere Informationen finden Sie in der OpenSearch Dokumentation.

Suchergebnisse in OpenSearch Dashboards vergleichen

  1. Öffnen Sie http://localhost:5601 und melden Sie sich bei OpenSearch Dashboards an. Die Standardanmeldeinformationen sind der Benutzername 'admin' und das Passwort 'admin'.

  2. Wählen Sie im Navigationsmenü unter den OpenSearch Plugins die Option Suchrelevanz aus.

  3. Geben Sie im Suchfeld ein.

  4. Wählen Sie Ihren Index für Query 1 aus und geben Sie eine Query in die OpenSearch Query DSL ein. Sie können die%SearchText% Variable verwenden, um auf den Suchtext zu verweisen, den Sie in die Suchleiste eingegeben haben. Ein Beispiel für diese Abfrage finden Sie in der OpenSearch Dokumentation. Die für diese Abfrage zurückgegebenen Ergebnisse sind die OpenSearch Ergebnisse ohne Verwendung des Intelligent Ranking-Plug-ins.

  5. Wählen Sie den gleichen Index für Query 2 und geben Sie dieselbe Query in die OpenSearch Query DSL ein. Geben Sie außerdem die Erweiterung mit ankendra_intelligent_ranking und geben Sie an, für welche Rangfolgebody_field Sie erforderlich sind. Sie können auch das Titelfeld angeben, das Textfeld ist jedoch obligatorisch. Ein Beispiel für diese Abfrage finden Sie in der OpenSearch Dokumentation. Die für diese Abfrage zurückgegebenen Ergebnisse sind die Ergebnisse, die mithilfe des Intelligent Ranking-Plug-insAmazon Kendra neu eingestuft wurden. Das Plugin bewertet bis zu 25 Ergebnisse.

  6. Wählen Sie Suchen aus, um die Ergebnisse zurückzugeben und zu vergleichen.
