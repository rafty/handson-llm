Funktionsweise von Amazon Kendra

Amazon Kendrabietet Suchfunktionen für Ihre Anwendung. Es indexiert Ihre Dokumente direkt oder aus Ihrem Dokumentenspeicher eines Drittanbieters und stellt Ihren Benutzern auf intelligente Weise relevante Informationen zur Verfügung. Sie könnenAmazon Kendra damit einen aktualisierbaren Index von Dokumenten verschiedener Typen erstellen. Eine Liste der unterstützten DokumenttypenAmazon Kendra finden Sie unter Dokumenttypen.

Amazon Kendralässt sich in andere -Services integrieren. Beispielsweise können Sie Amazon LexChat-Bots mit derAmazon Kendra Suche versorgen, um nützliche Antworten auf die Fragen der Benutzer zu geben. Sie können einen Amazon Simple Storage ServiceBucket als Datenquelle verwenden, um eine VerbindungAmazon Kendra zu Ihren Dokumenten herzustellen und sie zu indexieren. Und Sie können Zugriffsrichtlinien oder Berechtigungen für Ressourcen einrichten, die Sie verwenden AWS Identity and Access Management.

Amazon Kendra umfasst die folgenden Komponenten:

  • Ein Index, der Ihre Dokumente enthält und sie durchsuchbar macht.

  • Eine Datenquelle, die Ihre Dokumente speichert undAmazon Kendra mit der eine Verbindung hergestellt wird. Sie können eine Datenquelle automatisch mit einemAmazon Kendra Index synchronisieren, sodass Ihr Index mit Ihrem Quell-Repository aktualisiert wird.

  • Eine API zum Hinzufügen von Dokumenten, die Dokumente direkt zu einem Index hinzufügt.

Sie können esAmazon Kendra über die -Konsole oder die -API verwenden. Sie können Indizes erstellen, aktualisieren und löschen. Durch das Löschen eines Indexes werden alle zugehörigen Datenquellenkonnektoren und alle Ihre Dokumentinformationen aus dem Index dauerhaft gelöschtAmazon Kendra.
