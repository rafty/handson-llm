Amazon S3

Amazon S3ist ein Objektspeicherdienst, der Daten als Objekte in Buckets speichert. Sie können es verwendenAmazon Kendra, um IhrAmazon S3 Bucket-Repository mit Dokumenten zu indexieren.

Warnung

Amazon Kendraverwendet keine Bucket-Richtlinie, die einemAmazon Kendra Principal Berechtigungen zur Interaktion mit einem S3-Bucket gewährt. Stattdessen verwendet esIAM Rollen. Stellen Sie sicher, dass diesAmazon Kendra nicht als vertrauenswürdiges Mitglied in Ihrer Bucket-Richtlinie aufgeführt ist, um Datensicherheitsprobleme zu vermeiden, die entstehen, wenn versehentlich beliebigen Prinzipalen Berechtigungen erteilt werden. Sie können jedoch eine Bucket-Richtlinie hinzufügen, um einenAmazon S3 Bucket für verschiedene Konten zu verwenden. Weitere Informationen finden Sie unter Richtlinien für dieAmazon S3 kontenübergreifende Verwendung (auf der RegisterkarteIAM S3-Rollen unter IAMRollen für Datenquellen). Informationen zuIAM Rollen für S3-Datenquellen finden Sie unter IAMRollen.

Um eine Verbindung herzustellenAmazon S3, geben Sie die Verbindung und andere Informationen in der Konsole an, indem Sie S3DataSourceConfiguration oder den verwenden TemplateConfiguration. Wenn Sie das TemplateConfiguration Objekt verwenden, können Sie eine VPC verwenden, um eine Verbindung zu Ihrer Datenquelle herzustellen. Sie geben an VpcConfiguration , wann Sie anrufen CreateDataSource. Weitere Informationen finden Sie unter Konfigurieren einer VPC.

Anmerkung

S3DataSourceConfigurationDie API unterstützt die Konfiguration einer VPC nicht. Nur dieTemplateConfiguration API und die Konsole unterstützen die VPC-Konfiguration.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra S3-Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Voraussetzungen

Bevor Sie Ihre S3-DatenquelleAmazon Kendra zur Indexierung verwenden können, nehmen Sie diese Änderungen in Ihrem S3 und IhrenAWS Konten vor.

Stellen Sie in S3 sicher, dass Sie über Folgendes verfügen:

  • Hat den Namen von IhremAmazon S3 Bucket-Namen kopiert.

    Anmerkung

    Ihr Bucket muss sich in derselben Region wie IhrAmazon Kendra Index befinden, und Ihr Index muss über die Berechtigung verfügen, auf den Bucket zuzugreifen, der Ihre Dokumente enthält.

  • Geprüft, dass jedes Dokument in S3 und in anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

Stellen Sie in IhremAWS Konto sicher, dass Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

Wenn Sie keine vorhandeneIAM Rolle haben, können Sie die Konsole verwenden, um eine neueIAM Rolle zu erstellen, wenn Sie Ihre S3-Datenquelle mit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und eine Index-ID angeben.

Verbindungsanweisungen

Um eine VerbindungAmazon Kendra zu Ihrer S3-Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer S3-Datenquelle angeben, damit diese auf Ihre Daten zugreifenAmazon Kendra kann. Wenn Sie S3 noch nicht konfiguriert haben,Amazon Kendra sehen Sie sich das anVoraussetzungen.

Console

Um eine VerbindungAmazon Kendra herzustellenAmazon S3

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich die Option Indizes und dann in der Liste der Indizes den Index aus, den Sie verwenden möchten.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option S3-Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Datenquellendetails die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags für Neues Tag hinzufügen — Tags können Sie Ressourcen suchen und filtern oder IhreAWS -Kosten verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite „Zugriff und Sicherheit definieren“ die folgenden optionalen Informationen ein:

    1. IAM-Rolle — Wählen Sie eine vorhandene IAM-Rolle aus, oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und den Index-Inhalt zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    2. Virtual Private Cloud (VPC) — Sie können sich für die Verwendung einer VPC entscheiden. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    3. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Im Bereich Synchronisieren für Datenquellenstandort — Der Pfad zu demAmazon S3 Bucket, in dem Ihre Daten gespeichert sind. Wählen Sie Browse S3 aus, um Ihren Bucket auszuwählen.

    2. (Optional) Speicherort des Präfixes für Metadatendateien — Der Pfad zu dem Ordner, in dem Ihre Metadaten gespeichert sind. Wählen Sie Browse S3 aus, um Ihren Metadatenordner zu finden.

    3. (Optional) Speicherort der Konfigurationsdatei für die Zugriffskontrollliste — Der Pfad zum Speicherort einer Datei, die eine JSON-Struktur enthält, die die Zugriffseinstellungen für die in Ihrer S3-Datenquelle gespeicherten Dateien angibt. Wählen Sie Browse S3 aus, um Ihre ACL-Datei zu finden.

    4. (Optional) Entschlüsselungsschlüssel wählen — Wählen Sie diese Option aus, um einen Entschlüsselungsschlüssel zu verwenden. Wählen Sie, ob ein vorhandenerAWS KMS Schlüssel verwendet werden soll, oder erstellen Sie einen neuen.

    5. (Optional) Unter Zusätzliche Konfiguration für Regex-Muster — Fügen Sie reguläre Ausdrucksmuster hinzu, um Dokumente in Ihren Index einzubeziehen oder daraus auszuschließen. Alle Pfade beziehen sich auf den S3-Bucket des Datenquellenstandorts. Sie können bis zu 100 Muster hinzufügen.

    6. Wählen Sie im Synchronisierungsmodus zwischen dem Vollsynchronisierungsmodus und der Synchronisierung neuer, geänderter oder gelöschter Inhalte, um zu bestimmen, wie Ihr Index aktualisiert wird, wenn sich Ihr Datenquelleninhalt ändert. Wenn Sie Ihre Datenquelle zum ersten Mal mit Amazon Kendra synchronisieren, werden alle Inhalte standardmäßig synchronisiert.

    7. Wählen Sie im Synchronisierungslaufzeitplan für Häufigkeit aus, wie oft mit Ihrer Datenquelle synchronisiertAmazon Kendra werden soll.

    8. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen festlegen die folgenden optionalen Informationen ein:

    1. S3-Feldzuordnung — Wählen Sie eines derAmazon Kendra generierten Standarddatenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Wählen Sie, ob Sie benutzerdefinierte Datenquellenfelder hinzufügen möchten, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

TemplateConfiguration API

Um eine VerbindungAmazon Kendra herzustellenAmazon S3

Sie müssen einen JSON des Datenquellenschemas mithilfe der TemplateConfigurationAPI angeben. Sie müssen die folgenden Informationen angeben:

  • BucketName— Der Name des Buckets, der die Dokumente enthält.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager geheimes Element und zum Aufrufen der erforderlichen öffentlichen APIs für den S3-Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für S3-Datenquellen.

Sie können auch die folgenden optionalen-Funktionen hinzufügen:

  • Ein- und Ausschlussfilter — Geben Sie an, ob bestimmte Dateinamen, Dateitypen und Dateipfade ein- oder ausgeschlossen werden sollen, und verwenden Sie Glob-Muster (Muster, die ein Platzhaltermuster zu einer Liste von Pfadnamen erweitern können, die dem angegebenen Muster entsprechen). Zu den Beispielen für Glob-Muster gehören:

    • /myapp/config/*— Alle Dateien im Konfigurationsverzeichnis

    • /**/*.png—Alle PNG-Dateien in allen Verzeichnissen

    • /**/*.{png,ico,md}—Alle .png-, .ico- oder .md-Dateien in allen Verzeichnissen

    • /myapp/src/**/*.ts—Alle .ts-Dateien im Verzeichnis src (und all seinen Unterverzeichnissen)

    • **/!(*.module).ts—Alle .ts-Dateien, aber nicht .module.ts

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie aus, ob Sie Ihre S3-Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuordnen möchten. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

  • Virtual Private Cloud (VPC) — Geben Sie Virtual Private Cloud (VPC) — Geben SieVpcConfiguration bei Ihrem Anruf anCreateDataSource. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

  • DokumenturI, den Quell-URI, den Quell-URI, den Quell-URI, den Quell-URI, den Quell-URI, den Quell-URI, den Quell-URI, den Quell-URI, den DokumenturI, den Dokument Jede Metadatendatei enthält Metadaten zu einem einzelnen Dokument.

Eine Liste weiterer wichtiger zu konfigurierender JSON-Schlüssel finden Sie unterAmazon S3Schablonenschema.
