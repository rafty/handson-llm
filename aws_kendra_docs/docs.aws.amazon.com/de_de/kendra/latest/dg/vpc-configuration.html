KonfigurationAmazon Kendra für die Verwendung einer VPC

Amazon Kendrakann eine Verbindung zu Ihnen herstellenAmazon Virtual Private Cloud, um Inhalte zu indexieren, die in Datenquellen oder Datenbanken gespeichert sind, die in Ihrer privaten Cloud laufen. Wenn Sie einen Datenquellenconnector oder einen Datenbankconnector erstellen, geben Sie Sicherheitsgruppen- und Subnetzkennungen für das Subnetz an, das Ihre Datenquelle oder Datenbank enthält. Amazon Kendraverwendet diese Informationen, um eine elastic network interface zu erstellen, die für die sichere Kommunikation mit Ihrer Datenquelle oder Datenbank verwendet wird.

Wenn Ihre Datenquelle oder Datenbank nicht auf einem läuftAmazon VPC, können Sie Ihre Datenquelle oder DatenbankAmazon VPC mithilfe eines Virtual Private Network (VPN) mit Ihnen verbinden. Sie erhalten eine Standard-VPC, wenn Sie Ihr Amazon-Konto erstellen. Informationen zum Einrichten eines VPN finden Sie in der AWS VPNDokumentation.

Um eine VPC zu verwenden, müssen SieAmazon Kendra den Bezeichner des Subnetzes, zu dem die Datenbank gehört, und die Identifikatoren aller Sicherheitsgruppen, die für den Zugriff auf das Subnetz verwendetAmazon Kendra werden müssen, angeben. Wenn Sie beispielsweise den Standardport für eine MySQL-Datenbank verwenden, müssen die Sicherheitsgruppen denAmazon Kendra Zugriff auf Port 3306 auf dem Host zulassen, auf dem die Datenbank ausgeführt wird.

Verwenden Sie in der VPC-Konfiguration Ihrer Datenquelle oder Datenbank nur private Subnetze. Wenn sich Ihre RDS-Instance in einem öffentlichen Subnetz in Ihrer VPC befindet, können Sie dieses Subnetz nicht direkt verwenden, um Ihre Datenquelle oder Datenbank zu synchronisieren. Erstellen Sie stattdessen ein privates Subnetz mit ausgehendem Zugriff auf ein NAT-Gateway im öffentlichen Subnetz. Wenn Sie die VPC für Ihre Datenquelle oder Datenbank konfigurieren, geben Sie dieses private Subnetz an. Für eine mit einer VPC konfigurierte Datenbankdatenquelle müssen sich die Subnetze in einer der folgenden Availability Zone-IDs befinden:

  • USA West (Oregon) — usw2-az1, usw2-az2, usw2-az3

  • USA Ost (Nord-Virginia) — use1-az2, use1-az4

  • EU (Irland) — euw1-az1, uew1-az2, euw1-az3

Die Kennungen für Subnetze und Sicherheitsgruppen werden imAmazon VPC Control Panel konfiguriert. Um die Identifikatoren zu sehen, öffnen Sie dieAmazon VPC Konsole wie folgt:

Um Subnetzkennungen anzuzeigen

  1. Melden Sie sich bei der AWS Management Console an und öffnen Sie die Amazon-VPC-Konsole unter https://console.aws.amazon.com/vpc/.

  2. Wählen Sie im Navigationsbereich Subnetze aus.

  3. Wählen Sie aus der Subnetzliste das Subnetz aus, das Ihren Datenbankserver enthält.

  4. Notieren Sie sich auf der Registerkarte Beschreibung die Kennung im Feld Subnetz-ID.

Um Sicherheitsgruppen-IDs anzuzeigen

  1. Melden Sie sich bei der AWS Management Console an und öffnen Sie die Amazon-VPC-Konsole unter https://console.aws.amazon.com/vpc/.

  2. Wählen Sie im Navigationsbereich Sicherheitsgruppen aus.

  3. Wählen Sie in der Liste der Sicherheitsgruppen die Gruppe aus, für die Sie die Kennung verwenden möchten.

  4. Notieren Sie sich auf der Registerkarte „Beschreibung“ die Kennung im Feld Gruppen-ID.

Wenn die Verbindung zwischen zwei oder mehr Subnetzen weitergeleitet werdenAmazon Kendra muss, können Sie mehrere Subnetze angeben. Wenn das Subnetz, das Ihren Datenbankserver enthält, beispielsweise keine IP-Adressen mehr hat,Amazon Kendra können Sie eine Verbindung zu einem Subnetz mit freien IP-Adressen herstellen und die Verbindung zum ersten Subnetz weiterleiten. Wenn Sie mehrere Subnetze auflisten, müssen die Subnetze miteinander kommunizieren können. Jedem Subnetz muss eine Routing-Tabelle zugeordnet werden, die über ein NAT-Gerät (Network Address Translator) den ausgehenden Internetzugang ermöglicht.

Sie können auch mehrere Sicherheitsgruppen angeben. Die kombinierte Wirkung der Sicherheitsgruppen muss denAmazon Kendra Zugriff auf die Datenquelle oder den Datenbankserver ermöglichen, die Sie in der Verbindungskonfiguration angegeben haben.

Schritt 1: Konfigurieren einer VPC

Konfigurieren Sie Ihre VPC so, dass Sie über ein privates Subnetz und eine Sicherheitsgruppe verfügen,Amazon Kendra um auf eine MySQL-Datenbank zuzugreifen, die im Subnetz läuft. Die in der VPC-Konfiguration bereitgestellten Subnetze müssen sich entweder in USA West (Oregon), USA Ost (Nord-Virginia) oder EU (Irland) befinden.

So konfigurieren Sie eine VPC

  1. Melden Sie sich bei der AWS Management Console an und öffnen Sie die Amazon-VPC-Konsole unter https://console.aws.amazon.com/vpc/.

  2. Wählen Sie im Navigationsbereich Routentabellen und anschließend Routentabelle erstellen aus.

  3. Geben Sie für das Feld Namensschild den Wert einPrivate subnet route table. Wählen Sie im Feld VPC Ihre VPC aus und klicken Sie dann auf Erstellen. Wählen Sie Close, um zur Liste der Routing-Tabellen zurückzukehren.

  4. Wählen Sie im Navigationsbereich NAT Gateways und dann Create NAT Gateway aus.

  5. Wählen Sie im Feld Subnetz das Subnetz aus, das das öffentliche Subnetz ist, und notieren Sie sich die Subnetz-ID.

  6. Wenn Sie keine Elastic IP-Adresse haben, wählen Sie Create New EIP, wählen Sie Create a NAT Gateway und wählen Sie dann Close.

  7. Wählen Sie im Navigationsbereich Routing-Tabellen aus.

  8. Wählen Sie aus der Liste der Routentabellen die Routentabelle für private Subnetze aus, die Sie in Schritt 3 erstellt haben. Wählen Sie unter Aktionen die Option Routen bearbeiten aus.

  9. Wählen Sie Add Route (Route hinzufügen) aus. Fügen Sie das Ziel 0.0.0.0/0 hinzu, um den gesamten ausgehenden Datenverkehr ins Internet zuzulassen. Wählen Sie für Target NAT Gateway und dann das Gateway aus, das Sie in Schritt 4 erstellt haben. Wählen Sie Save routes (Routen speichern) aus. Wählen Sie dann Close (Schließen) aus.

  10. Wählen Sie unter Aktionen die Option Subnetzzuordnungen bearbeiten aus.

  11. Wählen Sie die Subnetze aus, die Sie privat haben möchten. Wählen Sie nicht das Subnetz mit dem NAT-Gateway, das Sie zuvor notiert haben.

Schritt 2: Konfigurieren der Sicherheit

Als Nächstes konfigurieren Sie Sicherheitsgruppen für Ihre Datenbank.

So erstellen Sie Sicherheitsgruppen:

  1. Melden Sie sich bei der AWS Management Console an und öffnen Sie die Amazon-VPC-Konsole unter https://console.aws.amazon.com/vpc/.

  2. Notieren Sie sich in der Beschreibung Ihrer VPC den IPv4-CIDR.

  3. Wählen Sie im Navigationsbereich Sicherheitsgruppen und dann Sicherheitsgruppe erstellen aus.

  4. Geben Sie im Feld Sicherheitsgruppenname einDataSourceInboundSecurityGroup. Geben Sie eine Beschreibung ein und wählen Sie dann Ihre VPC aus der Liste aus. Wählen Sie Erstellen und dann Schließen.

  5. Wählen Sie die Registerkarte Inbound (Eingehend).

  6. Wählen Sie Regeln bearbeiten und dann Regel hinzufügen

  7. Geben Sie für eine Datenbank die Portnummer für den Portbereich ein. Zum Beispiel für MySQL ist3306 es und für HTTPS ist es443. Geben Sie als Quelle Classless Inter-Domain Routing (CIDR) Ihrer VPC ein. Wählen Sie Regeln speichern und dann Schließen.

Die Sicherheitsgruppe ermöglicht es jedem innerhalb der VPC, eine Verbindung zur Datenbank herzustellen, und sie erlaubt ausgehende Verbindungen zum Internet.
