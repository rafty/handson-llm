Document

Ein Dokument in einem Index.

Inhalt

AccessControlConfigurationId

Die ID, die Sie auf das Dokument anwenden möchten.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Maximale Länge beträgt 36 Zeichen.

Pattern: [a-zA-Z0-9-]+

Erforderlich: Nein

AccessControlList

Informationen zu den Auftraggebern (Benutzern und/oder Gruppen) und zu den Dokumenten, auf die sie Zugriff haben sollten. Dies ist nützlich für die Benutzerkontextfilterung, bei der Suchergebnisse auf der Grundlage des Zugriffs des Benutzers oder seiner Gruppe auf Dokumente gefiltert werden.

Typ: Array von Principal-Objekten

Required: No

Attributes

Benutzerdefinierte Attribute, die auf das Dokument angewendet werden sollen. Verwenden Sie die benutzerdefinierten Attribute, um zusätzliche Informationen für die Suche bereitzustellen, um Facetten für die Verfeinerung von Suchen bereitzustellen und um zusätzliche Informationen in der Abfrageantwort bereitzustellen.

Beispielsweise sind '' undDataSourceId 'DataSourceSyncJobId' benutzerdefinierte Attribute, die Informationen zur Synchronisation von Dokumenten bereitstellen, die auf einer Datenquelle ausgeführt werden. Beachten Sie, dassDataSourceSyncJobId '' ein optionales benutzerdefiniertes Attribut sein könnte, da Amazon Kendra die ID eines laufenden Synchronisierungsjobs verwendet.

Typ: Array von DocumentAttribute-Objekten

Required: No

Blob

Der Inhalt.

An denBlob Parameter. Ihr Code muss die Bytes der Dokumentdatei möglicherweise nicht codieren, wenn Sie einAWS SDK verwenden, um Amazon Kendra Kendra-APIs aufzurufen. Wenn Sie den Amazon Kendra Kendra-Endpunkt direkt mit REST aufrufen, müssen Sie den Inhalt vor dem Senden mit Base64 codieren.

Typ: Base64-kodiertes Binärdatenobjekt

Required: No

ContentType

Der Dateityp des Dokuments imBlob Feld.

Typ: Zeichenfolge

Zulässige Werte: PDF | HTML | MS_WORD | PLAIN_TEXT | PPT | RTF | XML | XSLT | MS_EXCEL | CSV | JSON | MD

Required: No

HierarchicalAccessControlList

Die Liste der Hauptlisten, die die Hierarchie definieren, auf die Benutzer Zugriff haben sollen.

Typ: Array von HierarchicalPrincipal-Objekten

Array-Mitglieder: Die Mindestanzahl beträgt 1 Element. Maximale Anzahl 30 Elemente.

Required: No

Id

Eine Kennung des Dokuments im Index.

Beachten Sie, dass jede Dokument-ID pro Index eindeutig sein muss. Sie können keine Datenquelle erstellen, um Ihre Dokumente mit ihren eindeutigen IDs zu indexieren, und dann dieBatchPutDocument API verwenden, um dieselben Dokumente zu indexieren, oder umgekehrt. Sie können eine Datenquelle löschen und dann dieBatchPutDocument API verwenden, um dieselben Dokumente zu indexieren, oder umgekehrt.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Maximale Länge beträgt 2048 Zeichen.

Erforderlich: Ja

S3Path

Erforderliche Informationen für die Suche einer bestimmten Datei in einem Amazon S3-Bucket.

Typ: S3Path Objekt

Required: No

Title

Der dem Dokumentinhalt.

Typ: Zeichenfolge

Erforderlich: Nein
