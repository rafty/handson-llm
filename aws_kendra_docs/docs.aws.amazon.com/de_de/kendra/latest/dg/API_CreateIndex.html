CreateIndex

Erstellt einen Amazon-Kendra-Index. Die Indexerstellung ist eine asynchrone API. Um festzustellen, ob die Indexerstellung abgeschlossen ist, überprüfen Sie dasStatus Feld, das bei einem Aufruf von zurückgegeben wurdeDescribeIndex. DasStatus Feld wird auf gesetztACTIVE, wenn der Index einsatzbereit ist.

Sobald der Index aktiv ist, können Sie Ihre Dokumente mithilfe derBatchPutDocument API oder mithilfe einer der unterstützten Datenquellen indizieren.

Ein Beispiel für die Erstellung eines Index und einer Datenquelle mit dem Python SDK finden Sie unter Erste Schritte mit dem Python-SDK. Ein Beispiel für die Erstellung eines Index und einer Datenquelle mithilfe des Java SDK finden Sie unter Erste Schritte mit dem Java SDK.

Anfrageparameter

Informationen zu den Parametern, die allen Aktionen gemeinsam sind, finden Sie unter Allgemeine Parameter.

Die Anforderung akzeptiert die folgenden Daten im JSON-Format.

ClientToken

Ein Token, das Sie angeben, um die Anfrage zum Erstellen eines Indexes zu identifizieren. Bei mehrerenCreateIndex API-Aufrufen mit demselben Client-Token wird nur ein Index erstellt.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 100 Zeichen.

Required: No

Description

Eine Beschreibung für den Index.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Höchstlänge = 1 000 Zeichen.

Pattern: ^\P{C}*$

Erforderlich: Nein

Edition

Die Amazon Kendra Edition, die für den Index verwendet werden soll. Wählen SieDEVELOPER_EDITION Indizes, die für die Entwicklung, das Testen oder den Machbarkeitsnachweis bestimmt sind. ENTERPRISE_EDITIONFür die Produktion verwenden. Sobald Sie die Edition für einen Index festgelegt haben, kann sie nicht mehr geändert werden.

Der Parameter Edition ist optional. Wenn Sie keinen Wert angeben, wird als Standard eingestelltENTERPRISE_EDITION.

Weitere Informationen zu Kontingentbeschränkungen für die Enterprise- und Developer-Editionen finden Sie unter Kontingente.

Typ: Zeichenfolge

Zulässige Werte: DEVELOPER_EDITION | ENTERPRISE_EDITION

Required: No

Name

Einen Namen für den Index.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 1000 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Erforderlich: Ja

RoleArn

Der Amazon-Ressourcenname (ARN) einer IAM-Rolle mit der Berechtigung zum Zugriff auf Ihre CloudWatch Amazon-Protokolle und -Metriken. Weitere Informationen finden Sie unter IAM-Zugriffsrollen für Amazon Kendra.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Maximale Länge beträgt 1284 Zeichen.

Pattern: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Erforderlich: Ja

ServerSideEncryptionConfiguration

Die Kennung desAWS KMS kundenverwalteten Schlüssels (CMK), der zur Verschlüsselung von Daten verwendet wird, die von Amazon Kendra indiziert wurden. Amazon Kendra unterstützt keine asymmetrischen CMKs.

Typ: ServerSideEncryptionConfiguration Objekt

Required: No

Tags

Eine Liste von Schlüssel-Wert-Paaren, die den Index identifizieren oder kategorisieren. Sie können auch Tags verwenden, um den Zugriff auf den Index zu steuern. Tag-Schlüssel und Werte können Unicode-Buchstaben, Ziffern, Leerzeichen sowie die folgenden Sonderzeichen: _.:/= + - @.

Typ: Array von Tag-Objekten

Array-Mitglieder: Die Mindestanzahl beträgt 0 Elemente. Die maximale Anzahl beträgt 50 Elemente.

Required: No

UserContextPolicy

Die Richtlinie für den Benutzerkontext.

ATTRIBUTE_FILTER

Alle indizierten Inhalte sind durchsuchbar und für alle Benutzer anzeigbar. Wenn Sie Suchergebnisse nach Benutzerkontext filtern möchten, können Sie die Attributfilter von _user_id und _group_ids verwenden oder Benutzer- und Gruppeninformationen in UserContext bereitstellen.

USER_TOKEN

Aktiviert die tokenbasierte Benutzerzugriffskontrolle, um die Suchergebnisse nach dem Benutzerkontext zu filtern. Alle Dokumente ohne Zugriffskontrolle und alle Dokumente, die dem Benutzer zugänglich sind, sind durchsuchbar und darstellbar.

Typ: Zeichenfolge

Zulässige Werte: ATTRIBUTE_FILTER | USER_TOKEN

Required: No

UserGroupResolutionConfiguration

Ruft Benutzer und Gruppen aus derAWS IAM Identity Center (successor to AWS Single Sign-On) Identitätsquelle ab. Informationen zur Konfiguration finden Sie unter UserGroupResolutionConfiguration.

Typ: UserGroupResolutionConfiguration Objekt

Required: No

UserTokenConfigurations

Die Benutzertokenkonfiguration.

Typ: Array von UserTokenConfiguration-Objekten

Array-Mitglieder: Maximale Anzahl von 1 Element.

Required: No

Antwortelemente

Wenn die Aktion erfolgreich ist, sendet der Service eine HTTP 200-Antwort zurück.

Die folgenden Daten werden vom Service im JSON-Format zurückgegeben.

Id

Die Kennung des Index. Verwenden Sie diesen Bezeichner, wenn Sie einen Index abfragen, eine Datenquelle einrichten oder ein Dokument indexieren.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge beträgt 36 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Fehler

Hinweise zu den Fehlern, die allen Aktionen gemeinsam sind, finden Sie unterHäufige Fehler.

AccessDeniedException

Sie haben keinen ausreichenden Zugriff, um diese Aktion auszuführen. Bitte stellen Sie sicher, dass Sie über die erforderlichen Berechtigungsrichtlinien und Benutzerkonten verfügen, und versuchen Sie es erneut.

HTTP Status Code: 400

ConflictException

Bei der Anfrage ist ein Konflikt aufgetreten. Bitte korrigieren Sie alle Unstimmigkeiten mit Ihren Ressourcen und versuchen Sie es erneut.

HTTP Status Code: 400

InternalServerException

Ein Problem ist mit dem internen Server aufgetreten, der für Ihren Amazon Kendra-Service verwendet wird. Bitte warten Sie ein paar Minuten und versuchen Sie es erneut, oder wenden Sie sich an den Support, um Hilfe zu erhalten.

HTTP Status Code: 500

ResourceAlreadyExistException

Die Ressource, die Sie verwenden möchten, ist bereits vorhanden. Bitte überprüfen Sie, ob Sie die richtige Ressource angegeben haben, und versuchen Sie es erneut.

HTTP Status Code: 400

ServiceQuotaExceededException

Sie haben die festgelegten Grenzwerte für Ihren Amazon Kendra-Service überschritten. Weitere Informationen finden Sie unter Kontingente oder wenden Sie sich an den Support, um Informationen zu einer Erhöhung der Limits zu erhalten.

HTTP Status Code: 400

ThrottlingException

Die Anforderung wurde aufgrund der Drosselung von Anforderungen abgelehnt. Bitte reduzieren Sie die Anzahl der Anfragen und versuchen Sie es erneut.

HTTP Status Code: 400

ValidationException

Die Eingabe erfüllt nicht die vom Amazon-Kendra--Service festgelegten Einschränkungen. Bitte geben Sie die richtige Eingabe ein und versuchen Sie es erneut.

HTTP Status Code: 400
