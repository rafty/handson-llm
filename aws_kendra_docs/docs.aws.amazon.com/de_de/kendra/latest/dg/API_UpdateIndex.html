Anfrageparameter

Informationen zu den Parametern, die allen Aktionen gemeinsam sind, finden Sie unter Allgemeine Parameter.

Die Anforderung akzeptiert die folgenden Daten im JSON-Format.

CapacityUnits

Legt die Anzahl zusätzlicher Dokumentspeicher- und Abfragekapazitätseinheiten fest, die vom Index verwendet werden sollen. Sie können die Kapazität des Index bis zu 5 Mal pro Tag ändern oder 5 API-Aufrufe tätigen.

Wenn Sie zusätzliche Speichereinheiten verwenden, können Sie die Speicherkapazität nicht unter den Wert reduzieren, der erforderlich ist, um den Speicherbedarf für Ihren Index zu decken.

Typ: CapacityUnitsConfiguration Objekt

Required: No

Description

Eine neue Beschreibung für den Index.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Höchstlänge = 1 000 Zeichen.

Pattern: ^\P{C}*$

Erforderlich: Nein

DocumentMetadataConfigurationUpdates

Die Konfiguration der Dokumentmetadaten, die Sie für den Index aktualisieren möchten. Dokumentmetadaten sind Felder oder Attribute, die mit Ihren Dokumenten verknüpft sind. Zum Beispiel der Name der Unternehmensabteilung, der jedem Dokument zugeordnet ist.

Typ: Array von DocumentMetadataConfiguration-Objekten

Array-Mitglieder: Die Mindestanzahl beträgt 0 Elemente. Die maximale Anzahl beträgt 500 Elemente.

Required: No

Id

Der Bezeichner des Index, der aktualisiert werden soll.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge von 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Erforderlich: Ja

Name

Der Name des Index, der aktualisiert werden soll.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 1. Höchstlänge = 1000 Zeichen.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Erforderlich: Nein

RoleArn

EineAWS Identity and Access Management (IAM) -Rolle, die Amazon Kendra Berechtigungen für den Zugriff auf CloudWatch Amazon-Protokolle und -Metriken gewährt.

Typ: Zeichenfolge

Längenbeschränkungen: Minimale Länge von 0. Maximale Länge von 1 084 Zeichen.

Pattern: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Erforderlich: Nein

UserContextPolicy

Die Richtlinie für den Benutzerkontext.

Typ: Zeichenfolge

Zulässige Werte: ATTRIBUTE_FILTER | USER_TOKEN

Required: No

UserGroupResolutionConfiguration

Ermöglicht das Abrufen von Zugriffsebenen von Gruppen und Benutzern aus einerAWS IAM Identity Center (successor to AWS Single Sign-On) Identitätsquelle. Informationen zur Konfiguration finden Sie unter UserGroupResolutionConfiguration.

Typ: UserGroupResolutionConfiguration Objekt

Required: No

UserTokenConfigurations

Die Benutzertokenkonfiguration.

Typ: Array von UserTokenConfiguration-Objekten

Array-Mitglieder: Maximale Anzahl von 1 Element.

Required: No

Antwortelemente

Wenn die Aktion erfolgreich ist, gibt der Dienst eine HTTP 200-Antwort mit leerem HTTP-Textinhalt zurück.

Fehler

Hinweise zu den Fehlern, die allen Aktionen gemeinsam sind, finden Sie unterHäufige Fehler.

AccessDeniedException

Sie haben keinen ausreichenden Zugriff, um diese Aktion auszuführen. Bitte stellen Sie sicher, dass Sie über die erforderlichen Berechtigungsrichtlinien und Benutzerkonten verfügen, und versuchen Sie es erneut.

HTTP Status Code: 400

ConflictException

Bei der Anfrage ist ein Konflikt aufgetreten. Bitte korrigieren Sie alle Unstimmigkeiten mit Ihren Ressourcen und versuchen Sie es erneut.

HTTP Status Code: 400

InternalServerException

Ein Problem ist mit dem internen Server aufgetreten, der für Ihren Amazon Kendra-Service verwendet wird. Bitte warten Sie ein paar Minuten und versuchen Sie es erneut, oder wenden Sie sich an den Support, um Hilfe zu erhalten.

HTTP Status Code: 500

ResourceNotFoundException

Die Ressource, die Sie verwenden möchten, existiert nicht. Bitte überprüfen Sie, ob Sie die richtige Ressource angegeben haben, und versuchen Sie es erneut.

HTTP Status Code: 400

ServiceQuotaExceededException

Sie haben die festgelegten Grenzwerte für Ihren Amazon Kendra-Service überschritten. Weitere Informationen finden Sie unter Kontingente oder wenden Sie sich an den Support, um Informationen zu einer Erhöhung der Limits zu erhalten.

HTTP Status Code: 400

ThrottlingException

Die Anforderung wurde aufgrund der Drosselung von Anforderungen abgelehnt. Bitte reduzieren Sie die Anzahl der Anfragen und versuchen Sie es erneut.

HTTP Status Code: 400

ValidationException

Die Eingabe erfüllt nicht die Einschränkungen des Amazon Kendra-Dienstes. Bitte geben Sie die korrekte Eingabe und versuchen Sie es erneut.

HTTP Status Code: 400
