Anpassen der Kapazität

Amazon Kendrastellt Ressourcen für Ihren Index in Kapazitätseinheiten bereit. Jede Kapazitätseinheit stellt zusätzliche Ressourcen für Ihren Index bereit. Es gibt separate Kapazitätseinheiten für die Dokumentenablage und für Abfragen. Sie können nur Kapazitätseinheiten zuAmazon Kendra Enterprise Edition-Indizes hinzufügen. Sie können einem Developer Edition-Index keine Kapazität hinzufügen.

Eine Speicherkapazitätseinheit für Dokumente bietet den folgenden zusätzlichen Speicherplatz für Ihren Index.

  • 100.000 Dokumente oder 30 GB Speicherplatz.

Eine Abfragekapazitätseinheit stellt die folgenden zusätzlichen Abfragen für Ihren Index bereit.

  • 0,1 Abfragen pro Sekunde oder ungefähr 8 000 Abfragen pro Tag.

Jeder Index verfügt über eine Basiskapazität, die einer Kapazitätseinheit entspricht (30 GB Speicher und 0,1 Abfragen pro Sekunde). Für jede zusätzliche Kapazitätseinheit fallen zusätzliche Kosten an. Weitere Details finden Sie unter Amazon Kendra-Preise.

Sie können Ihrem Speicher bis zu 100 zusätzliche Kapazitätseinheiten hinzufügen und Ressourcen für einen Index abfragen. Wenn Sie mehr Einheiten benötigen, wenden Sie sich an denAWS Support.

Sie können Kapazitätseinheiten bis zu 5 Mal pro Tag an Ihre Anforderungen anpassen. Sie können die Speicherkapazität für Dokumente nicht unter die Anzahl der in Ihrem Index gespeicherten Dokumente reduzieren. Wenn Sie beispielsweise 150.000 Dokumente speichern, können Sie die Speicherkapazität nicht unter eine zusätzliche Einheit reduzieren.

Sie können die Ressourcen, die ein Index verwendet, in der Konsole anzeigen, indem Sie den Namen des Index auswählen, um die Indexeinstellungen und andere Informationen zu öffnen, oder Sie können dieDescribeIndex API verwenden. Amazon Kendragibt auch Ausnahmen zurück, wenn Sie die Kapazität eines Indexes überschreiten. Sie erhalten einServiceQuotaExceededException, wenn die gesamte extrahierte Größe aller Dokumente den Grenzwert für einen Index überschreitet. Sie erhaltenInvalidRequest für jedes Dokument eine, wenn die Anzahl der Dokumente den Grenzwert für einen Index überschreitet. Sie erhalten eineThrottlingException, wenn die Anzahl an Abfragen pro Sekunde das Limit überschreitet. Weitere Informationen zu Limits finden Sie unter Kontingente fürAmazon Kendra.

Kapazität der Anzeige

Sehen Sie sich die Ressourcen an, die Ihr Index mit derAmazon Kendra Konsole verwendet, indem Sie den Namen Ihres Index auswählen, um auf die Details zuzugreifen. Die Konsole bietet auch Nutzungsdiagramme, anhand derer Sie ermitteln können, wie viel Speicher- und Abfragekapazität Ihr Index verwendet. Sie können diese Informationen verwenden, um zu planen, wann zusätzliche Kapazität hinzugefügt werden soll.

Verwenden Sie (Konsole), um den Dokumentspeicher anzuzeigen und abzufragen

  1. Melden Sie sich bei der anAWS Management Console und öffnen Sie dieAmazon Kendra Konsole unter https://console.aws.amazon.com/kendra/home.

  2. Suchen Sie in der Liste der Indizes den Index aus, auf den Sie zugreifen möchten.

  3. Scrollen Sie zum Abschnitt Einstellungen, um den aktuellen Gesamtspeicher für Dokumente und die Abfragekapazität anzuzeigen.

Verwenden Sie denCapacityUnits Parameter in derAmazon Kendra API, um die Kapazität mithilfe derDescribeIndex API anzuzeigen.

Hinzufügen und Entfernen von Kapazität

Wenn Sie zusätzliche Kapazität für Ihren Index benötigen, können Sie ihn mithilfe der Konsole oder derAmazon Kendra API hinzufügen.

So können Sie Speicher oder Abfragekapazität hinzufügen oder entfernen (Konsole)

  1. Melden Sie sich bei der anAWS Management Console und öffnen Sie dieAmazon Kendra Konsole unter https://console.aws.amazon.com/kendra/home.

  2. Suchen Sie in der Liste der Indizes den Index aus, auf den Sie zugreifen möchten.

  3. Wählen Sie Bearbeiten aus, oder wählen Sie in der Dropdownliste Aktionen die Option Bearbeiten aus.

  4. Wählen Sie Weiter, um zur Seite mit den Bereitstellungsdetails zu gelangen.

  5. Fügen Sie Dokumentspeicher hinzu oder entfernen Sie Kapazitätseinheiten und/oder fragen Sie Kapazitätseinheiten ab.

  6. Wählen Sie weiter Weiter, um zur Überprüfungsseite zu gelangen, und wählen Sie dann Aktualisieren aus, um Ihre Änderungen zu speichern.

Nachdem Sie die Kapazität Ihres Index aktualisiert haben, kann es einige Minuten dauern, bis die Änderungen wirksam werden.

Verwenden Sie denCapacityUnits Parameter in derAmazon Kendra API, um Kapazität mithilfe derUpdateIndex API hinzuzufügen oder zu entfernen.

Amazon KendraIntelligente Ranking-Kapazität

Eine Kapazitätseinheit stellt die folgenden zusätzlichen Rescore-Anforderungen pro Sekunde für einen Rescore-Ausführungsplan bereit. Ein Rescore-Ausführungsplan ist eine Ressource, die zur Bereitstellung der Rescore-API verwendet wird.

  • 0,01 Anforderungen pro Sekunde.

Jeder Rescore-Ausführungsplan verfügt über eine Basiskapazität, die einer Kapazitätseinheit entspricht (0,01 Anfragen pro Sekunde). Für jede zusätzliche Kapazitätseinheit fallen zusätzliche Kosten an. Weitere Details finden Sie unter Amazon Kendra-Preise.

Sie können bis zu 1000 zusätzliche Kapazitätseinheiten für einen Rescore-Ausführungsplan hinzufügen. Wenn Sie mehr Einheiten benötigen, wenden Sie sich an denAWS Support.

Abfragevorschläge Die Kapazität

Bei der Verwendung von Abfragevorschlägen gibt es eine Basisabfragekapazität von 2,5 GetQuerySuggestionsAufrufen pro Sekunde. DieGetQuerySuggestions Kapazität ist das Fünffache der bereitgestellten Abfragekapazität für einen Index oder die Basiskapazität von 2,5 Aufrufen pro Sekunde, je nachdem, welcher Wert höher ist. Zum Beispiel beträgt die Basiskapazität für einen Index 0,1 Abfragen pro Sekunde, und die GetQuerySuggestions-Kapazität hat eine Basis von 2,5 Anrufen pro Sekunde. Wenn Sie weitere 0,1 Abfragen pro Sekunde hinzufügen, um insgesamt 0,2 Abfragen pro Sekunde für einen Index zu erhalten, ist die GetQuerySuggestions-Kapazität 2,5 Aufrufe pro Sekunde (höher als fünf mal 0,2 Abfragen pro Sekunde).

Sucherlebnis Die Kapazität

Amazon Kendrabeginnt, zu drosselnQueryQuerySuggestions,SubmitFeedback für IhreAmazon Kendra Erfahrung bei 15 Anfragen pro Sekunde und 40 Anfragen pro Sekunde für das Bursting von Abfragen. Für einen Index mit mehr als 150 Abfragekapazitätseinheiten gelten diese Grenzwerte weiterhin.

Ihre Abfragekapazitätseinheiten für Ihren Index sind beispielsweise 150, sodass Ihre Search Experience-Anwendung 15 Anfragen pro Sekunde verarbeiten kann. Wenn Sie jedoch auf 200 Abfragekapazitätseinheiten skalieren würden, würde Ihre Search-Experience-App immer noch nur 15 Anfragen pro Sekunde verarbeiten. Wenn Sie Ihren Index auf 100 Abfragekapazitätseinheiten beschränken, würde Ihre Search-Experience-App nur 10 Anfragen pro Sekunde verarbeiten.

Adaptives Abfrage-Bursting

Amazon Kendrahat eine bereitgestellte Basiskapazität von 1 Abfragekapazitätseinheit. Sie können bis zu 8 000 Abfragen pro Tag mit einem Mindestdurchsatz von 0,1 Abfragen pro Sekunde (pro Abfragekapazitätseinheit) verwenden. Kumulierte Abfragen dauern bis zu 24 Stunden und können Datenverkehrsschübe bewältigen. Die zulässige Menge an Burst variiert, da sie von der Auslastung des Clusters zu einem bestimmten Zeitpunkt abhängt. Stellen Sie genügend Abfragekapazitätseinheiten bereit, um Ihre Spitzenlastniveaus zu bewältigen.

Das integrierte adaptive Query-Bursting istAmazon Kendra ein adaptiver Ansatz für den Umgang mit unerwarteten Datenverkehrsausbrüchen, die über den bereitgestellten Durchsatz hinausgehen. Adaptives Query Bursting ist in der Enterprise Edition von verfügbarAmazon Kendra.

Adaptives Query Bursting ist eine integrierte Funktion, mit der Sie ungenutzte Abfragekapazität verwenden können, um unerwarteten Datenverkehr zu verarbeiten. Amazon Kendraakkumuliert Ihre ungenutzten Abfragen mit der Rate der bereitgestellten Abfragen pro Sekunde, jede Sekunde, bis zur maximalen Anzahl von Abfragen, die Sie für IhrenAmazon Kendra Index bereitgestellt haben. Diese gesammelten Abfragen werden für unerwarteten Datenverkehr verwendet, der die zugewiesene Kapazität übersteigt. Die optimale Leistung von adaptivem Query Bursting kann variieren und hängt von verschiedenen Faktoren ab, wie z. B. der Gesamtindexgröße, der Abfragekomplexität, den angesammelten ungenutzten Abfragen und der Gesamtlast Ihres Index. Es wird empfohlen, eigene Belastungstests durchzuführen, um die Bursting-Kapazität genau zu messen.
