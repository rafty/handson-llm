Index

Ein Index enthält den Inhalt Ihrer Dokumente und ist so strukturiert, dass die Dokumente durchsuchbar sind. Wie Sie Dokumente zum Index hinzufügen, hängt davon ab, wie Sie Ihre Dokumente speichern.

  • Wenn Sie Ihre Dokumente in einem Repository speichern, z. B. in einemAmazon S3 Bucket oder einer SharePoint Microsoft-Site, verwenden Sie einen Datenquellenconnector, um Ihre Dokumente aus Ihrem Repository zu indexieren.

  • Wenn Sie Ihre Dokumente nicht in einem Repository speichern, verwenden Sie die BatchPutDocumentAPI, um Ihre Dokumente direkt zu indexieren.

  • Für FAQ-Fragen und -Antworten, die in einemAmazon Kendra (Amazon S3) Bucket gespeichert werden müssen, lädst du sie aus dem Bucket hoch

Sie können Indizes mit derAmazon Kendra KonsoleAWS CLI, dem oder einemAWS SDK erstellen. Informationen zu den Dokumenttypen, die indexiert werden können, finden Sie unterTypen von -Dokumenten.

Verwenden der integrierten Dokumentfelder von Amazon Kendra

Mit der UpdateIndex API können Sie reservierte oder integrierte Felder erstellen, indem Sie den reservierten Feldnamen verwendenDocumentMetadataConfigurationUpdates und angeben, um ihn Ihrem entsprechenden Dokumentattribut oder Dokumentfeld zuzuordnen. Sie können auch benutzerdefinierte Felder auf diese Weise erstellen. Wenn Sie einen Datenquellenconnector verwenden, enthalten die meisten Feldzuordnungen, die Ihre DatenquellendokumentfelderAmazon Kendra Indexfeldern zuordnen. Wenn Sie ein Feld erstellen, können Sie dasSearch Objekt so konfigurieren, dass das Feld als anzeigbar, facettierbar, durchsuchbar und sortierbar festgelegt wird. Sie können dasRelevance Objekt so konfigurieren, dass die Werte für die Dauer, Aktualität, Wichtigkeit, Rangfolge und Wichtigkeit des Felds zur Steigerung der Relevanz festgelegt werden. Sie können den Feldtyp nicht mehr ändern, nachdem Sie das Feld erstellt haben.

Amazon Kendraverfügt über die folgenden reservierten oder integrierten Dokumentfelder, die Sie verwenden können:

  • _authors— Eine Liste von einem oder mehreren Autoren, die für den Inhalt des Dokuments verantwortlich sind.

  • _category— Eine Kategorie, die ein Dokument in eine bestimmte Gruppe einordnet.

  • _created_at— Datum und Uhrzeit im ISO 8601-Format, zu denen das Dokument erstellt wurde. Beispiel: 2012-03-25T12:30:10+01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in mitteleuropäischer Zeit (CET).

  • _data_source_id— Bezeichner der Datenquelle, die das Dokument enthält.

  • _document_body— Der Inhalt des Dokuments.

  • _document_id—Eindeutiger Bezeichner für das Dokument.

  • _document_title— Der Titel des Dokuments.

  • _excerpt_page_number— Die Seitenzahl in einer PDF-Datei, auf der der Dokumentauszug erscheint. Wenn Ihr Index vor dem 8. September 2020 erstellt wurde, müssen Sie Ihre Dokumente neu indizieren, bevor Sie dieses Attribut verwenden können.

  • _faq_id—Wenn es sich um eine häufig gestellte Frage und Antwort handelt, eine eindeutige Kennung für sie.

  • _file_type— Der Dateityp des Dokuments, z. B. PDF oder Dokument.

  • _last_updated_at— Datum und Uhrzeit der letzten Aktualisierung des Dokuments im ISO-8601-Format. Beispiel: 2012-03-25T12:30:10+01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in mitteleuropäischer Zeit (CET).

  • _source_uri— Die URI, unter der das Dokument verfügbar ist. Zum Beispiel die URI des Dokuments auf einer Unternehmenswebsite.

  • _version— Ein Bezeichner für die spezifische Version eines Dokuments.

  • _view_count— Wie oft das Dokument angesehen wurde.

  • _language_code(String) — Der Code für eine Sprache, die für das Dokument gilt. Die Standardeinstellung ist Englisch, wenn Sie keine Sprache angeben. Weitere Informationen zu unterstützten Sprachen, einschließlich ihrer Codes, finden Sie unter Hinzufügen von Dokumenten in anderen Sprachen als Englisch.

Für benutzerdefinierte Felder erstellen Sie diese FelderDocumentMetadataConfigurationUpdates mithilfe derUpdateIndex API, genau wie beim Erstellen eines reservierten oder integrierten Felds. Sie müssen den entsprechenden Datentyp für Ihr benutzerdefiniertes Feld festlegen. Sie können den Feldtyp nicht mehr ändern, nachdem Sie das Feld erstellt haben.

Die folgenden Typen können Sie für benutzerdefinierte Felder festlegen:

  • Datum

  • Zahl

  • Zeichenfolge

  • Zeichenfolgenliste

Sie erstellen ein benutzerdefiniertes Feld mithilfe der Konsole oder mithilfe der UpdateIndexAPI. Nachdem Sie ein benutzerdefiniertes Feld erstellt haben, ordnen Sie es einem Dokumentattribut zu, genau wie Sie es bei einem reservierten Feld tun. Wenn Sie ein Dokument mit BatchPutDocumentAPI zum Index hinzugefügt haben, ordnen Sie die Attribute der API zu. Für Dokumente, die aus einerAmazon S3 Datenquelle indexiert wurden, ordnen Sie die Attribute mithilfe einer Metadatendatei zu, die eine JSON-Struktur enthält, die die Dokumentattribute beschreibt. Für Dokumente, die mit einer Datenbank oder einer Datenquelle indexiert sind, die eine Feldzuordnung ermöglicht, ordnen Sie Attribute der Konsole oder der Datenquellenkonfiguration zu.
