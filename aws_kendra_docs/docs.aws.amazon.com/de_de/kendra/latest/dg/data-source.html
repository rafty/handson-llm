Erstellen eines Datenquellen-Connectors

Sie können einen Datenquellen-Connector erstellen,Amazon Kendra um eine Verbindung mit Ihren Dokumenten herzustellen und sie zu indexieren. Amazon Kendrakann eine Verbindung zu Microsoft SharePoint, Google Drive und vielen anderen Anbietern herstellen. Wenn Sie einen Datenquellenconnector erstellen, geben SieAmazon Kendra die Konfigurationsinformationen an, die für die Verbindung mit Ihrem Quell-Repository erforderlich sind. Im Gegensatz zum direkten Hinzufügen von Dokumenten zu einem Index können Sie die Datenquelle regelmäßig scannen, um den Index zu aktualisieren.

Beispiel: Angenommen, Sie haben ein Repository mit Steuerdokumenten, die in einem S3-Bucket gespeichert sind. Von Zeit zu Zeit werden bestehende Dokumente geändert und neue Dokumente werden dem Repository hinzugefügt. Wenn Sie das RepositoryAmazon Kendra als Datenquelle hinzufügen, können Sie Ihren Index auf dem neuesten Stand halten, indem Sie regelmäßige Synchronisationen zwischen Ihrer Datenquelle und dem Index einrichten.

Sie können wählen, ob Sie einen Index manuell über die Konsole oder dieStartDataSourceSyncJob API aktualisieren möchten. Andernfalls richten Sie einen Zeitplan ein, um einen Index zu aktualisieren und ihn mit Ihrer Datenquelle zu synchronisieren.

Ein Index kann über mehr als eine Datenquelle verfügen. Jede Datenquelle kann ihren eigenen Aktualisierungsplan haben. Beispielsweise können Sie den Index Ihrer Arbeitsdokumente täglich oder sogar stündlich aktualisieren, während Sie Ihre archivierten Dokumente manuell aktualisieren, wenn sich das Archiv ändert.

Wenn Sie die Metadaten oder Attribute und den Inhalt Ihres Dokuments während des Dokumentaufnahmevorgangs ändern möchten, finden Sie unter Amazon KendraBenutzerdefinierte Dokumentanreicherung.

Beachten Sie, dass jede Dokument-ID pro Index eindeutig sein muss. Sie können keine Datenquelle erstellen, um Ihre Dokumente mit ihren eindeutigen IDs zu indexieren, und dann dieBatchPutDocument API verwenden, um dieselben Dokumente zu indexieren, oder umgekehrt. Sie können eine Datenquelle löschen und dann dieBatchPutDocument API verwenden, um dieselben Dokumente zu indexieren, oder umgekehrt.

Einen Aktualisierungszeitplan festlegen

Konfigurieren Sie Ihre Datenquelle so, dass sie regelmäßig mit der Konsole oder mithilfe desSchedule Parameters aktualisiert wird, wenn Sie eine Datenquelle erstellen oder aktualisieren. Der Inhalt des Parameters ist eine Zeichenfolge, die entweder eine Zeitplanzeichenfolgecron im -Format oder eine leere Zeichenfolge enthält, die angibt, dass der Index bei Bedarf aktualisiert wird. Informationen zum Format eines Cron-Ausdrucks finden Sie im Amazon CloudWatch EventsBenutzerhandbuch unter Ausdrücke für Regeln planen. Amazon Kendraunterstützt nur Cron-Ausdrücke. Es unterstützt keine Bewertungsausdrücke.

Festlegen einer Sprache

Sie können alle Ihre Dokumente in einer Datenquelle in einer unterstützten Sprache indexieren. Sie geben den Sprachcode für all Ihre Dokumente in Ihrer Datenquelle an, wenn Sie anrufen CreateDataSource. Wenn für ein Dokument kein Sprachcode in einem Metadatenfeld angegeben ist, wird das Dokument mit dem Sprachcode indexiert, der für alle Dokumente auf Datenquellenebene angegeben ist. Wenn Sie keine Sprache angeben, werden Dokumente in einer Datenquelle standardmäßig auf EnglischAmazon Kendra indexiert. Weitere Informationen zu unterstützten Sprachen, einschließlich ihrer Codes, finden Sie unter Hinzufügen von Dokumenten in anderen Sprachen als Englisch.

Mithilfe der Konsole indexieren Sie alle Ihre Dokumente in einer Datenquelle in einer unterstützten Sprache. Gehen Sie zu Datenquellen und bearbeiten Sie Ihre Datenquelle oder fügen Sie Datenquelle hinzu, wenn Sie eine neue Datenquelle hinzufügen. Wählen Sie auf der Seite „Datenquellendetails angeben“ eine Sprache aus der Dropdown-Liste Sprache aus. Wählen Sie Aktualisieren aus oder geben Sie weiterhin die Konfigurationsinformationen ein, um eine Verbindung mit Ihrer Datenquelle herzustellen.
