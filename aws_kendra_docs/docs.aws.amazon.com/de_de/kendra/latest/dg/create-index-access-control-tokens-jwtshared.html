Verwendung eines JSON-WebTokens (JW) mit einem gemeinsamen Schlüssel

Die folgenden Beispiele zeigen, wie Sie ein JSON-Web-Token (JWT) mit einem gemeinsamen geheimen Token für die Benutzerzugriffskontrolle verwenden, wenn Sie einen Index erstellen.

Console

  1. Wählen Sie Index erstellen, um mit der Erstellung eines neuen Index zu beginnen.

  2. Geben Sie auf der Seite Indexdetails angeben Ihrem Index einen Namen und eine Beschreibung.

  3. Wählen Sie als IAM-Rolle eine Rolle aus, oder wählen Sie Neue Rolle erstellen für aus und geben Sie einen Rollennamen an, um eine neue Rolle zu erstellen. Die IAM-Rolle wird das Präfix "AmazonKendra-“ haben.

  4. Belassen Sie alle anderen Felder auf ihren Standardwerten. Wählen Sie Next (Weiter).

  5. Wählen Sie auf der Seite Benutzerzugriffskontrolle konfigurieren unter Zugriffskontrolleinstellungen die Option Ja aus, um Token für die Zugriffskontrolle zu verwenden.

  6. Wählen Sie unter Tokenkonfiguration JWT mit Shared Secret als Tokentyp aus.

  7. Wählen Sie unter Parameter für das Signieren eines gemeinsamen Geheimnisses den Typ des geheimen Schlüssels aus. Sie können ein vorhandenesAWS Secrets Manager Shared Secret verwenden oder ein neues Shared Secret erstellen.

    Um ein neues Shared Secret zu erstellen, wählen Sie Neu und gehen Sie dann wie folgt vor:

    1. Geben Sie unter NeuerAWS Secrets Manager geheimer Name einen geheimen Namen an. Das PräfixAmazonKendra- wird hinzugefügt, wenn Sie den öffentlichen Schlüssel speichern.

    2. Geben Sie eine Schlüssel-ID an. Die Schlüssel-ID ist ein Hinweis darauf, welcher Schlüssel verwendet wurde, um die JSON-Websignatur des Tokens zu sichern.

    3. Wählen Sie den Signaturalgorithmus für das Token. Dies ist der kryptografische Algorithmus verwendet, um das ID-Token zu sichern. For more information on RSA, see RSA Cryptography.

    4. Geben Sie ein Shared Secret an, indem Sie ein Base64-URL-kodiertes Geheimnis eingeben. Sie können auch Geheimnis generieren wählen, um ein Geheimnis für Sie generieren zu lassen. Sie müssen sicherstellen, dass das Geheimnis ein Base64-URL-kodiertes Geheimnis ist.

    5. (Optional) Geben Sie an, wann das Shared Secret gültig ist. Sie können das Datum und die Uhrzeit angeben, ab dem ein Geheimnis gültig ist, gültig bis oder beides. Das Geheimnis wird in dem angegebenen Intervall gültig sein.

    6. Wählen Sie Geheimnis speichern aus, um das neue Geheimnis zu speichern.

  8. (Optional) Unter Erweiterte Konfiguration:

    1. Geben Sie einen Benutzernamen an, der bei der ACL-Prüfung verwendet werden soll.

    2. Geben Sie eine oder mehrere Gruppen an, die bei der ACL-Prüfung verwendet werden sollen.

    3. Geben Sie den Issuer an, der den Token-Emittenten validieren soll.

    4. Geben Sie die Antrags-ID (n) an. Sie müssen einen regulären Ausdruck angeben, der der Zielgruppe im JWT entspricht.

  9. Wählen Sie auf der Seite mit den Bereitstellungsdetails die Developer Edition aus.

  10. Wählen Sie Erstellen, um Ihren Index zu erstellen.

  11. Warten Sie, bis Ihr Index erstellt wurde. Amazon Kendrastellt die Hardware für Ihren Index bereit. Dieser Vorgang kann einige Zeit in Anspruch nehmen.

CLI

Sie können das JWT-Token mit einem gemeinsamen Geheimnis in verwendenAWS Secrets Manager. Der Parameter muss ein base64-URL-codiertes Secret sein. Sie benötigen denSecrets Manager ARN und IhreAmazon Kendra Rolle muss ZugriffGetSecretValue auf dieSecrets Manager Ressource haben. Wenn Sie dieSecrets Manager Ressource mit verschlüsselnAWS KMS, muss die Rolle auch Zugriff auf die Entschlüsselungsaktion haben.

Um einen Index mit derAWS CLI Verwendung einer JSON-Eingabedatei zu erstellen, erstellen Sie zunächst eine JSON-Datei mit Ihren gewünschten Parametern: 

{
    "Name": "user-context",
    "Edition": "ENTERPRISE_EDITION",
    "RoleArn": "arn:aws:iam::account-id:role:/my-role",
    "UserTokenConfigurations": [
        {
            "JwtTokenTypeConfiguration": {
                "KeyLocation": "SECRET_MANAGER",
                "Issuer": "optional: specify the issuer url",
                "ClaimRegex": "optional: regex to validate claims in the token",
                "UserNameAttributeField": "optional: user",
                "GroupAttributeField": "optional: group",
                "SecretManagerArn": "arn:aws:secretsmanager:us-west-2:account id:secret:/my-user-context-secret
            }
        }
    ],    
    "UserContextPolicy": "USER_TOKEN"
}

Sie können die standardmäßigen Benutzer- und Gruppenfeldnamen überschreiben. Der Standardwert fürUserNameAttributeField ist „user“. Der Standardwert fürGroupAttributeField ist „Gruppen“.

Rufen Sie als Nächstescreate-index mit der Eingabedatei auf. Wenn der Name Ihrer JSON-Datei beispielsweise lautetcreate-index-openid.json, können Sie Folgendes verwenden: 

aws kendra create-index --cli-input-json file://create-index-openid.json

Das Geheimnis muss das folgende Format haben inAWS Secrets Manager:

{
  "keys": [
    {
      "kid": "key_id",
      "alg": "HS256|HS384|HS512",
      "kty": "OCT", 
      "use": "sig", //this value can be sig only for now
      "k": "secret",
      "nbf":"ISO1806 date format"
      "exp":"ISO1806 date format"
    }
  ]
}

Weitere Informationen zu JWT finden Sie unter jwt.io.

Python

Sie können das JWT-Token mit einem gemeinsamen Geheimnis in verwendenAWS Secrets Manager. Der Parameter muss ein base64-URL-codiertes Secret sein. Sie benötigen denSecrets Manager ARN und IhreAmazon Kendra Rolle muss ZugriffGetSecretValue auf dieSecrets Manager Ressource haben. Wenn Sie dieSecrets Manager Ressource mit verschlüsselnAWS KMS, muss die Rolle auch Zugriff auf die Entschlüsselungsaktion haben.

response = kendra.create_index(
    Name='user-context',
    Edition='ENTERPRISE_EDITION',
    RoleArn='arn:aws:iam::account-id:role:/my-role',
    UserTokenConfigurations=[
        {
            "JwtTokenTypeConfiguration": {
                "KeyLocation": "URL",
                "Issuer": "optional: specify the issuer url",
                "ClaimRegex": "optional: regex to validate claims in the token",
                "UserNameAttributeField": "optional: user",
                "GroupAttributeField": "optional: group",
                "SecretManagerArn": "arn:aws:secretsmanager:us-west-2:account id:secret:/my-user-context-secret"
            }
        }
    ],
    UserContextPolicy='USER_TOKEN'
)
