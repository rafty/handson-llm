Fehlerbehebung bei Datenquellen

Dieser Abschnitt kann Ihnen helfen, Probleme mit Amazon Kendra Kendra-Datenquellen zu beheben.

Meine Dokumente wurden nicht indexiert

Wenn Sie Ihren Kendra-Index mit einer Datenquelle synchronisieren, können Probleme auftreten, die die Indexierung der Dokumente verhindern. Die Indexieren ist ein zweistufiger Prozess. Zunächst wird die Datenquelle auf neue und aktualisierte Dokumente zum Indexieren und auf Dokumente überprüft, die aus dem Index entfernt werden sollen. Zweitens wird auf Dokumentebene auf jedes Dokument zugegriffen und es indexiert.

In jedem dieser Schritte kann ein Fehler auftreten. Fehler auf Datenquellenebene werden in der Konsole im Abschnitt Synchronisierungsverlauf auf der Seite mit den Datenquellendetails gemeldet. Der Status des Synchronisationsauftrags kann erfolgreich, unvollständig oder Fehlgeschlagen lauten. Sie können auch die Anzahl der Dokumente sehen, die während des Jobs indexiert und gelöscht wurden. Wenn der Status Fehlgeschlagen ist, wird in der Spalte Details eine Meldung angezeigt.

Fehler auf Dokumentebene werden in Amazon CloudWatch Logs gemeldet. Sie können die Fehler mithilfe der CloudWatch Konsole sehen.

Mein Synchronisationsjob ist fehlgeschlagen

Ein Synchronisationsjob schlägt normalerweise fehl, wenn ein Konfigurationsfehler im Index oder in der Datenquelle vorliegt. Die Fehlermeldung in der Spalte Details der Datenquelle enthält Informationen darüber, was schief gelaufen ist. Das Problem besteht normalerweise darin, dass der Index oder die Datenquelle nicht über die richtigen IAM-Berechtigungen verfügt. Die Fehlermeldung beschreibt die fehlenden Berechtigungen. Hier sind ein paar der Fehlermeldungen, die Sie erhalten können:

Failed to create log group for job. Please make sure that the IAM role provided has sufficient permissions.

Wenn Ihre Indexrolle nicht zur Verwendung berechtigt ist CloudWatch, kann die Datenquelle kein CloudWatch Protokoll erstellen. Wenn dieser Fehler angezeigt wird, müssen Sie der Indexrolle CloudWatch Berechtigungen hinzufügen.

Failed to access S3 file prefix (bucket name) while trying to crawl your metadata files. Please make sure the IAM Role (role ARN) provided has sufficient permissions.

Wenn Sie eine Amazon S3 S3-Datenquelle verwenden, muss Kendra über die Berechtigung verfügen, auf den Bucket zuzugreifen, der die Dokumente enthält. Sie müssen der IAM-Rolle der Datenquelle die Berechtigung für Kendra hinzufügen, den Bucket zu lesen.

The provided IAM Role (role ARN) could not be assumed. Please make sure Amazon Kendra is a trusted entity that is allowed to assume the role.

Kendra benötigt die Erlaubnis, um die IAM-Rollen Index und Datenquelle zu übernehmen. Sie müssen den Rollen eine Vertrauensrichtlinie mit der Berechtigung für diests:AssumeRole Aktion hinzufügen.

Die IAM-Richtlinien, die Kendra benötigt, um eine Datenquelle zu indexieren, finden Sie unterIAMZugriffsrollen fürAmazon Kendra.

Mein Synchronisationsjob ist unvollständig

Aufträge sind im Allgemeinen unvollständig, wenn sie den Prozess auf Datenquellenebene abgeschlossen haben, während des Prozesses auf Dokumentebene jedoch einige Fehler auftreten. Wenn ein Job unvollständig ist, wurden einige Dokumente möglicherweise immer noch erfolgreich indexiert. Bei einer Amazon-S3-Datenquelle wird ein unvollständiger Job in der Regel durch:

  • Die Metadaten für ein oder mehrere Dokumente waren ungültig.

  • Wenn Dokumente zur Indexierung eingereicht werden, aber mindestens ein Dokument nicht eingereicht wurde.

  • Wenn Dokumente zum Löschen aus dem Index eingereicht werden, aber mindestens ein Dokument nicht eingereicht wurde.

Um einen unvollständigen Synchronisierungsjob zu beheben, schauen Sie sich zuerst Ihre CloudWatch Protokolle an.

  1. Wählen Sie in der Detailspalte die Option Details anzeigen in CloudWatch.

  2. Überprüfen Sie die Fehlermeldungen, um zu sehen, warum das Dokument fehlgeschlagen ist.

Mein Synchronisationsjob war erfolgreich, aber es gibt keine indizierten Dokumente

Gelegentlich wird eine Ausführung eines Indexsynchronisierungsauftrags als erfolgreich markiert, aber es wurden keine neuen oder aktualisierten Dokumente indexiert, wenn Sie sie erwarten. Mögliche Gründe sind:

  • Überprüfen Sie die CloudWatch DocumentsSubmittedForIndexingFailed Metrik, um festzustellen, ob Dokumente nicht synchronisiert werden konnten. Einzelheiten finden Sie in Ihren CloudWatch Protokollen.

  • Für eine Amazon S3 S3-Datenquelle haben Sie Kendra möglicherweise den falschen Bucket-Namen oder das falsche Präfix gegeben. Stellen Sie sicher, dass der Bucket, den Kendra verwendet, der die zu indexierenden Dokumente enthält.

  • Wenn Sie ein Dokument erneut indexieren, das in einem früheren Job nicht indexiert werden konnte, indexiert Kendra es nur, wenn Sie das Dokument oder die zugehörige Metadatendatei geändert haben.

Ich habe Probleme mit dem Dateiformat beim Synchronisieren meiner Datenquelle

Wenn Sie beim Hinzufügen von Dateien zu Ihrer Datenquelle oder beim Synchronisieren Ihrer Datenquelle auf Probleme mit dem Dateiformat stoßen, stellen Sie sicher, dass Ihre Dokumenttypen von Kendra unterstützt werden. Eine Liste der unterstützten DokumenttypenAmazon Kendra finden Sie unter Dokumenttypen.

Wenn Sie dieBatchPutDocument API mit einfachen Textdateien verwenden, geben Sie diesPLAIN_TEXT als Inhaltstyp an.

Wie viel Zeit dauert das Synchronisieren einer Datenquelle?

Wenn Dokumente nicht aktualisiert werden, erhöht sich die Synchronisierungszeit für einen Amazon Kendra Kendra-Index linear proportional zur Anzahl der Dokumente. Beispielsweise würde die Synchronisierung von 1.000 Dokumenten ohne Aktualisierungen etwa fünf Minuten dauern, und 2.000 Dokumente ohne Aktualisierungen etwa 10 Minuten. Wenn die Dokumente aktualisiert werden, erhöht sich die Synchronisierungszeit je nach Anzahl der aktualisierten Dokumente.

Wie hoch ist die Gebühr für die Synchronisierung einer Datenquelle?

Wenn Sie Ihren Index synchronisieren, dauert das Aufwärmen und Aktivieren von Amazon EC2 zwei Minuten, um die erforderlichen Verbindungen herzustellen. Während dieses Vorgangs werden Ihnen keine Gebühren berechnet. Ihre Nutzungsanzeige beginnt erst, nachdem der Synchronisierungsjob gestartet wurde. Weitere Informationen zur Preisgestaltung von Kendra finden Sie unter Amazon Kendra Kendra-Preise.

Ich erhalte einen Amazon EC2 EC2-Autorisierungsfehler

Wenn während einer Synchronisierung für eine Virtual Private Cloud (VPC) -Datenquelle ein unautorisierter Amazon EC2 EC2-Betriebsfehler auftritt, ist es wahrscheinlich, dass Ihrer VPC IAM-Rolle die erforderlichen Berechtigungen fehlen. Bitte überprüfen Sie, ob die IAM-Rolle, die Sie für Ihre Datenquelle verwenden, über die zugehörigen Berechtigungen verfügt. Weitere Informationen finden Sie unter IAMRolle Virtual Private Cloud (VPC).

Ich kann keine Suchindexlinks verwenden, um meine Amazon S3 S3-Objekte zu öffnen

Ihr Kendra-Index kann nur auf Dateien zugreifen, für die ihm eine Amazon S3 S3-Datenquelle Zugriffsberechtigungen gewährt. Kendra kann beispielsweise die Amazon S3 S3-Berechtigungen, die bestimmen, ob ein Objekt öffentlich oder verschlüsselt sein soll, nicht ändern. Kendra verfügt auch nicht über die Standardberechtigungen, um einen signierten Link für Amazon S3 S3-Objekte zu erstellen oder zurückzugeben. Wenn Sie signierte Verknüpfungen für Amazon S3 S3-Objekte in einem Kendra-Index aktivieren möchten, haben Sie zwei Möglichkeiten:

  • Sie können Ihre Indexabfrageergebnisse mit dem Quell-URI-Objekt signieren, bevor Sie das Ergebnis an die Suchseite zurückgeben. Eine step-by-step exemplarische Vorgehensweise für diesen Vorgang finden Sie unter Objekte mit vorsignierten URLs teilen.

  • Sie können die Quell-URI der Amazon S3 S3-Objektmetadatenquelle überschreiben und Ihren Service über ein CloudFront Content Delivery Network (CDN) verfügbar machen, das mit einem Amazon S3 S3-Bucket verbunden ist. Oder Sie können ein API Gateway Gateway-Proxy-Endpunkt verwenden, der ein vorgängen- und einen vorgängen URL zurückgibt.

Ich erhalte die Fehlermeldung „AccessDenied When Using SSL Certificate File“

Wenn Sie bei der Verwendung eines SSL-Zertifikats mit Ihrer Datenquelle die Fehlermeldung „Zugriff verweigert“ erhalten, stellen Sie sicher, dass Ihre IAM-Rolle über die Berechtigung verfügt, auf die SSL-Zertifikatsdatei am angegebenen Speicherort zuzugreifen. Wenn das Zertifikat mit einemAWS KMS Schlüssel verschlüsselt ist, sollte Ihre IAM-Rolle auch über die Berechtigung verfügen, mithilfe desAWS KMS Schlüssels zu entschlüsseln. Weitere Informationen finden Sie unter Authentifizierung und Zugriffskontrolle fürAWS KMS.

Ich erhalte einen Autorisierungsfehler, wenn ich eine SharePoint Datenquelle verwende

Wenn Sie beim Synchronisieren Ihres Index mit einer SharePoint Datenquelle einen Autorisierungsfehler erhalten, vergewissern Sie sich, dass Ihnen eine Site-Administratorrolle zugewiesen ist SharePoint.

Mein Index crawlt keine Dokumente aus meiner Confluence-Datenquelle

Wenn Ihr Kendra-Index während des Synchronisierungsprozesses keine Dokumente aus Ihrer Confluence-Datenquelle crawlt, vergewissern Sie sich, dass Sie Teil der Administratorgruppen in Confluence sind.
