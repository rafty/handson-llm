Amazon FSx

Amazon FSx ist ein vollständig verwaltetes, cloudbasiertes Dateiserversystem, das gemeinsam genutzte Speicherfunktionen bietet. Wenn Sie ein Amazon FSx-Benutzer sind, können SieAmazon Kendra damit Ihre Amazon FSx-Datenquelle indexieren.

Amazon Kendraunterstützt derzeit nurAmazon FSx Windows File Server.

Sie können mithilfe der Amazon KendraKonsole und der FsxConfigurationAPI eine VerbindungAmazon Kendra zu Ihrer Amazon FSx-Datenquelle herstellen.

Informationen zur FehlerbehebungAmazon Kendra bei Ihrem Amazon FSx-Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Unterstützte Funktionen

Amazon KendraDer Amazon FSx-Datenquellenkonnektor unterstützt die folgenden Funktionen:

  • Änderungsprotokoll

  • Feldzuordnungen

  • Filterung des Benutzerkontextes

  • Einschluss-/Ausschlussfilter

  • Virtual Private Cloud (VPC)

Voraussetzungen

Bevor Sie Ihre Amazon FSx-DatenquelleAmazon Kendra zur Indexierung verwenden können, nehmen Sie diese Änderungen in Ihrem Amazon FSx- undAWS Konto vor.

Stellen Sie in Amazon FSx über Folgendes verfügen:

  • Ein Amazon FSx-Konto mit Lese- und Mountingberechtigungen.

  • Ich habe IhreAmazon FSx Authentifizierungsdaten für ein Active Directory-Benutzerkonto notiert. Dazu gehören Ihr Active Directory-Benutzername und Ihr Domain Name System (DNS). Zum Beispiel user@corp.example.com.

  • IhreAmazon FSx Dateisystem-ID kopiert.

  • Hat eineAmazon VPC (AWSVPC) verwendet, an der SieAmazon FSx wohnen.

  • Geprüft, dass jedes Dokument in Amazon FSx und in anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

Stellen Sie in IhremAWS Konto sicher, dass Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Ihre Amazon FSx-Authentifizierungsdaten wurdenAWSSecrets Manager geheim gespeichert und, falls Sie die API verwenden, den ARN des Geheims notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder kein Geheimnis haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn Sie Ihre Amazon FSx-Datenquelle damit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Anweisungen zum Anschluss

Um eine VerbindungAmazon Kendra zu Ihrer Amazon FSx-Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer Amazon FSx-Datenquelle angeben, damit diese auf Ihre Daten zugreifenAmazon Kendra kann. Falls Sie Amazon FSx noch nicht konfiguriert haben,Amazon Kendra sehen Sie sich das anVoraussetzungen.

Console

So stellen Sie eine VerbindungAmazon Kendra zu Amazon FSx her

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich Indizes und dann in der Liste der Indizes den Index aus, den Sie verwenden möchten.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option Amazon FSx Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Daten -Quelldetails die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags unter Neues Tag hinzufügen — Tags können Sie Ressourcen suchen und filtern oder IhreAWS Kosten verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. Quelle — Wählen SieAmazon FSx für Windows-Dateiserver.

    2. Amazon FSxDateisystem-ID —Wählen Sie Ihre Dateisystem-ID oder erstellen Sie ein neues Verzeichnis.

    3. AWSSecrets Managersecret — Wählen Sie ein vorhandenes Geheimnis aus oder erstellen Sie ein neuesSecrets Manager Geheimnis, um Ihre Amazon FSx-Authentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-Amazon FSx-' wird automatisch zu Ihrem geheimen Namen hinzugefügt.

        2. Für den Benutzernamen —Geben Sie den Benutzernamen für dasAmazon FSx Active Directory-Konto ein.

        3. Für Passwort —Geben Sie das Passwort für dasAmazon FSx Active Directory-Konto ein.

      2. Wählen Sie Speichern.

    4. Virtual Private Cloud (VPC) — Sie müssen eine VPC mit IhrerAmazon FSx Datenquelle verwenden. Sie müssen Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    5. IAM-Rolle — Wählen Sie eine vorhandene IAM-Rolle oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    6. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Änderungsprotokoll — Wählen Sie diese Option aus, um Ihren Index zu aktualisieren, anstatt alle Ihre Dateien zu synchronisieren.

    2. Regex-Muster — Fügen Sie reguläre Ausdrucksmuster hinzu, um bestimmte Inhalte ein- oder auszuschließen. Sie können bis zu 100 Muster hinzufügen.

    3. Wählen Sie im Synchronisierungslaufzeitplan für Häufigkeit aus, wie oft die Synchronisierung mit Ihrer Datenquelle erfolgenAmazon Kendra soll.

    4. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen die folgenden Informationen ein:

    1. Amazon FSx Windows-Feldzuordnungen — Wählen Sie eines derAmazon Kendra generierten Standard-Datenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

So stellen Sie eine VerbindungAmazon Kendra zu Amazon FSx her

Sie müssen mithilfe der FsxConfigurationAPI Folgendes angeben:

  • Geheimer Amazon Resource Name (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr Amazon FSx-Konto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "username": "user@corp.example.com",
    "password": "password"
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager geheimes Element und zum Aufrufen der erforderlichen öffentlichen APIs für den Amazon FSx-Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für Amazon FSx-Datenquellen.

Sie können auch die folgenden optionalen-Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie dies imVpcConfiguration Rahmen der Datenquellenkonfiguration an. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

  • Änderungsprotokoll — Gibt an, ob der Amazon FSx-Mechanismus für Datenquellenänderungen verwendet werdenAmazon Kendra soll, um festzustellen, ob ein Dokument im Index hinzugefügt, aktualisiert oder gelöscht werden muss.

    Anmerkung

    Verwenden Sie das Änderungsprotokoll, wenn Sie nicht alle Dokumente scannen möchtenAmazon Kendra. Wenn Ihr Änderungsprotokoll groß ist, dauert das Scannen der Dokumente in der Amazon FSx-Datenquelle möglicherweiseAmazon Kendra weniger Zeit als das Verarbeiten des Änderungsprotokolls. Wenn Sie Ihre Amazon FSx-Datenquelle zum ersten Mal mit Ihrem Index synchronisieren, werden alle Dokumente gescannt.

  • Ein- und Ausschlussfilter — Sie können angeben, ob bestimmte Inhalte und Inhaltstypen ein- oder ausgeschlossen werden sollen.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre Amazon FSx-Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

Anmerkung

BenutzerkontextfilterungAmazon Kendra — durchsucht standardmäßig die Access Control List (ACL) für Ihre Datenquelle. Die ACL-Informationen werden verwendet, um die Ergebnisse eines Benutzers auf der Grundlage des Benutzer- oder Gruppenzugriffs auf Dokumente zu filtern.

Um die Benutzerkontextfilterung für einen Benutzer zu testen, müssen Sie den DNS-Domainnamen als Teil des Benutzernamens angeben, wenn Sie die Abfrage stellen. Sie benötigen Administratorrechte für die Active Directory-Domäne. Sie können auch die Benutzerkontextfilterung für einen Gruppennamen testen. Weitere Informationen finden Sie unter Benutzerkontextfilterung für Amazon FSx-Datenquellen.
