Bereitstellen von Amazon Kendra

Wenn es darum geht, dieAmazon Kendra Suche auf Ihrer Website bereitzustellen, stellen wir Quellcode zur Verfügung, den Sie mit React verwenden können, um sich einen Vorsprung in Ihrer Anwendung zu verschaffen. Der Quellcode wird unter einer modifizierten MIT-Lizenz kostenlos zur Verfügung gestellt. Sie können es so verwenden, wie es ist, oder es für Ihre eigenen Bedürfnisse ändern. Die mitgelieferte React-App ist ein Beispiel, das Ihnen bei den ersten Schritten hilft. Es ist keine produktionsreife App.

Informationen zum Bereitstellen einer Suchanwendung ohne Code und zum Generieren einer Endpunkt-URL zu Ihrer Suchseite mit Zugriffskontrolle finden Sie unter Amazon KendraExperience Builder.

Es gibt zwei Beispiele, die Sie mit React verwenden können:

  • https://kendrasamples.s3.amazonaws.com/kendrasamples-react-app .zip — Ein Beispiel für eine React-Anwendung, die Beispieldaten und eine Suchseite bereitstellt.

  • https://kendrasamples.s3.amazonaws.com/kendrasamples.zip —Eine Bibliothek, die Sie zu einer vorhandenen React-Anwendung hinzufügen können.

Die Beispiele sind der Suchseite derAmazon Kendra Konsole nachempfunden. Sie haben dieselben Funktionen für die Suche und Anzeige von Suchergebnissen. Sie können das gesamte Beispiel verwenden oder nur eine der Funktionen für Ihren eigenen Gebrauch auswählen.

Um die drei Komponenten der Suchseite in derAmazon Kendra Konsole zu sehen, wählen Sie im rechten Menü das Codesymbol (</>). Zeigen Sie mit dem Mauszeiger auf die einzelnen Abschnitte, um eine kurze Beschreibung der Komponente und die URL der Quelle der Komponente zu erhalten.

Übersicht

Sie fügen den Beispielcode zu einer vorhandenen React-Anwendung hinzu, um die Suche zu aktivieren. Die Suchdateien und Komponenten sind wie folgt strukturiert:

  • Hauptsuchseite — Dies ist die Hauptseite, die alle Komponenten enthält. Hier integrieren Sie Ihre Anwendung in dieAmazon Kendra API.

  • Suchleiste — Dies ist die Komponente, in die ein Benutzer einen Suchbegriff eingibt und die die Suchfunktion aufruft.

  • Ergebnisse — Dies ist die Komponente, die die Ergebnisse von anzeigtAmazon Kendra. Es besteht aus drei Komponenten: Vorgeschlagene Antworten, FAQ-Ergebnisse und empfohlene Dokumente.

  • Facetten — Dies ist die Komponente, die die Facetten in den Suchergebnissen anzeigt und es Ihnen ermöglicht, eine Facette auszuwählen, um die Suche einzuschränken.

  • Paginierung — Dies ist die Komponente, von der die Antwort paginiert wirdAmazon Kendra.

Voraussetzungen

Bevor Sie beginnen, muss Folgendes sichergestellt sein:

  • Eine vorhandene React-Webanwendung oder die Beispielanwendung.

  • Eine Entwicklungsumgebung, die mit den richtigen Bibliotheken konfiguriert ist.

  • Node.js und npm installiert.

  • DasSDK for Java oderAWS SDK for JavaScript.

Informationen zu den erforderlichen Bibliotheken undAWS SDKs finden Sie in der Readme-Datei in den Zip-Dateien.

Warnung

Die React-App ist nicht produktionsbereit. Es ist ein Beispiel für die Bereitstellung einer App für dieAmazon Kendra Suche.

  1. Stellen Sie sicher, dass Sie den abgeschlossen habenVoraussetzungen, einschließlich des Herunterladens und der Installation von Node.js und npm.

  2. Laden Sie kendra-samples-react-app .zip herunter und entpacken Sie es.

  3. Öffne dein Terminal und gehe zuaws-kendra-sample-app/src/services/. Öffnen Sielocal-dev-credentials-template.json und geben Sie Ihre Anmeldeinformationen ein. Fügen Sie diese Datei keinem öffentlichen Repository hinzu.

  4. Gehe zuaws-kendra-sample-app/src und installiere die Abhängigkeiten. Führen Sie npm install.

  5. Starten Sie eine Demoversion Ihrer App auf Ihrem lokalen Server. Führen Sie npm start. Sie können den lokalen Server stoppen, indem Sie ihn auf Ihrer Tastatur eingebenCmd/Ctrl + C.

  6. Sie können den Port oder Host (z. B. die IP-Adresse) ändern, indem Sie den Hostpackage.json und den Port aktualisieren:"start": "HOST=[host] PORT=[port] react-scripts start". Wenn Sie Windows verwenden:"start": "set HOST=[host] && set PORT=[port] && react-scripts start".

  7. Wenn Sie eine registrierte Website-Domain haben, können Sie diesepackage.json hinter Ihrem App-Namen angeben. Zum Beispiel "homepage": "https://mywebsite.com". Sie müssennpm install erneut ausführen, um neue Abhängigkeiten zu aktualisieren, und dann ausführennpm start.

  8. Führen Sie aus, um die App zu erstellennpm build. Laden Sie den Inhalt des Build-Verzeichnisses auf Ihren Hosting-Anbieter hoch.

Haupt-Suchseite

Die Hauptsuchseite enthält alle Beispiel-Suchkomponenten. Es enthält die Suchleistenkomponente für die Ausgabe, die Ergebniskomponenten zur Anzeige der Antwort von der Abfrage-API und eine Paginierungskomponente für das Durchblättern der Antwort.

Suchkomponente

Die Suchkomponente stellt ein Textfeld zur Eingabe von Abfragetext bereit. DieonSearch Funktion ist ein Hook, der die Hauptfunktion aufruftSearch.tsx, um denAmazon Kendra Abfrage-API-Aufruf durchzuführen.

Komponente „Ergebnisse“

Die Ergebniskomponente zeigt die Antwort derQuery API. Die Ergebnisse werden in drei separaten Bereichen dargestellt.

  • Vorgeschlagene Antworten — Dies sind die wichtigsten Ergebnisse, die von derQuery API zurückgegeben werden. Es enthält bis zu drei Antwortvorschläge. In der Antwort haben sie den ErgebnistypANSWER.

  • FAQ-Antworten — Dies sind die Ergebnisse der häufig gestellten Fragen, die in der Antwort zurückgegeben wurden. Häufig gestellte Fragen werden dem Index separat hinzugefügt. In der Antwort haben sie den TypQUESTION_ANSWER. Weitere Informationen finden Sie unter Fragen und Antworten.

  • Empfohlene Dokumente — Dies sind zusätzliche Dokumente, die in der AntwortAmazon Kendra zurückgegeben werden. In der Antwort derQuery API haben sie den TypDOCUMENT.

Die Ergebniskomponenten teilen sich eine Reihe von Komponenten für Funktionen wie Markierungen, Titel, Links und mehr. Die gemeinsam genutzten Komponenten müssen vorhanden sein, damit die Ergebniskomponenten funktionieren.

Facett-Komponente

Die Facettenkomponente listet die in den Suchergebnissen verfügbaren Facetten auf. Jede Facette klassifiziert die Antwort anhand einer bestimmten Dimension, z. B. nach dem Autor. Sie können die Suche auf eine bestimmte Facette einschränken, indem Sie eine Facette aus der Liste auswählen.

Nachdem Sie eine Facette ausgewählt haben, wird die KomponenteQuery mit einem Attributfilter aufgerufen, der die Suche auf Dokumente beschränkt, die der Facette entsprechen.

Komponente „Paginierung“

Mit der Paginierungskomponente können Sie die Suchergebnisse derQuery API auf mehreren Seiten anzeigen. Es ruft dieQuery API mit denPageNumber ParameternPageSize und auf, um eine bestimmte Ergebnisseite abzurufen.
