BatchGetDocumentStatus

Gibt den Indexierungsstatus für ein oder mehrere Dokumente zurück, die mit der BatchPutDocumentAPI übermittelt wurden.

Wenn Sie dieBatchPutDocument API verwenden, werden Dokumente asynchron indexiert. Sie können dieBatchGetDocumentStatus API verwenden, um den aktuellen Status einer Liste von Dokumenten abzurufen, sodass Sie feststellen können, ob sie erfolgreich indexiert wurden.

Sie können dieBatchGetDocumentStatus API auch verwenden, um den Status der BatchDeleteDocumentAPI zu überprüfen. Wenn ein Dokument aus dem Index gelöscht wird, kehrt Amazon KendraNOT_FOUND als Status zurück.

Anfrageparameter

Informationen zu den Parametern, die allen Aktionen gemeinsam sind, finden Sie unter Allgemeine Parameter.

Die Anforderung akzeptiert die folgenden Daten im JSON-Format.

DocumentInfoList

Eine Liste vonDocumentInfo Objekten, die die Dokumente identifizieren, für die der Status abgerufen werden soll. Sie identifizieren die Dokumente anhand ihrer Dokument-ID und optionaler Attribute.

Typ: Array von DocumentInfo-Objekten

Array-Mitglieder: Die Mindestanzahl beträgt 1 Element. Die maximale Anzahl beträgt 10 Elemente.

Erforderlich: Ja

IndexId

Die Kennung des Indexes, dem Dokumente hinzugefügt werden sollen. Die Index-ID wird von der CreateIndexAPI zurückgegeben.

Typ: Zeichenfolge

Längenbeschränkungen: Feste Länge von 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Erforderlich: Ja

Antwortelemente

Wenn die Aktion erfolgreich ist, sendet der Service eine HTTP 200-Antwort zurück.

Die folgenden Daten werden vom Service im JSON-Format zurückgegeben.

DocumentStatusList

Der Status von Dokumenten. Der Status gibt an, ob das Dokument auf die Indexierung wartet, gerade indexiert wird, ob die Indizierung abgeschlossen ist oder ob die Indizierung fehlgeschlagen ist. Wenn ein Dokument nicht indexiert werden konnte, gibt der Status den Grund dafür an.

Typ: Array von Status-Objekten

Errors

Eine Liste von Dokumenten, für die Amazon Kendra den Status nicht erhalten konnte. Die Liste enthält die ID des Dokuments und den Grund, warum der Status nicht gefunden werden konnte.

Typ: Array von BatchGetDocumentStatusResponseError-Objekten
