Amazon KendraWebcrawler

Sie könnenAmazon Kendra Web Crawler verwenden, um Webseiten zu crawlen und zu indexieren.

Sie können nur öffentlich zugängliche Websites und Websites, die das sichere Kommunikationsprotokoll Hypertext Transfer Protocol Secure (HTTPS) verwenden. Wenn Sie beim Crawling einer Website einen Fehler erhalten, kann es sein, dass die Website für das Crawling gesperrt ist. Um interne Websites zu crawlen, können Sie einen Web-Proxy einrichten. Der Webproxy muss öffentlich zugänglich sein.

Bei der Auswahl der zu indizierenden Websites müssen Sie die Amazon Acceptable Use Policy (Richtlinie zur zulässigen Nutzung) und alle anderen Amazon-Bedingungen einhalten. Denken Sie daran, dass Sie denAmazon Kendra Webcrawler nur verwenden dürfen. Zu indizieren. Informationen dazu, wie SieAmazon Kendra Web Crawler daran hindern können, Ihre Website (n) zu indexieren, finden Sie unterKonfiguration derrobots.txt Datei fürAmazon Kendra Web Crawler.

Sie können mithilfe der Amazon KendraKonsole und der WebCrawlerConfigurationAPI eine VerbindungAmazon Kendra zu Ihrer Webcrawler-Datenquelle herstellen.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra Web Crawler-Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Voraussetzungen

Bevor Sie Ihre WebsitesAmazon Kendra zum Indexieren verwenden können, überprüfen Sie die Details Ihrer Websites undAWS Konten.

Für Ihre Websites müssen Sie über Folgendes verfügen:

  • Die Seed- oder Sitemap-URLs der Websites, die Sie indizieren möchten.

  • Für Websites, für deren Crawling eine Standardauthentifizierung erforderlich ist: Der Hostname der Website und die Portnummer wurden kopiert.

  • Optional: Der Hostname der Website und die Portnummer wurden kopiert, wenn Sie einen Web-Proxy verwenden möchten, um eine Verbindung zu internen Websites herzustellen, die Sie crawlen möchten. Der Webproxy muss öffentlich zugänglich sein. Amazon Kendraunterstützt die Verbindung zu Web-Proxyservern, die durch eine Standardauthentifizierung unterstützt werden, oder Sie können eine Verbindung ohne Authentifizierung herstellen.

  • Hat geprüft, ob jedes Webseitendokument, das Sie indexieren möchten, einzigartig ist und dass es sich um andere Datenquellen handelt, die Sie für denselben Index verwenden möchten. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

In IhremAWS Konto müssen Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Hat Ihre Anmeldeinformationen für die Webcrawler-Authentifizierung in einemAWSSecrets Manager Secret gespeichert und, falls Sie die API verwenden, den ARN des Geheims notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder kein Geheimnis haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn Sie Ihre Webcrawler-Datenquelle damit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Verbindungsanweisungen

Um eine VerbindungAmazon Kendra zu Ihrer Webcrawler-Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer Webcrawler-Datenquelle angeben, damit diese auf Ihre Daten zugreifenAmazon Kendra kann. Falls Sie den Webcrawler noch nicht konfiguriert haben,Amazon Kendra sehen Sie sich das anVoraussetzungen.

Console

Um eine VerbindungAmazon Kendra zum Webcrawler herzustellen

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich die Option Indizierung. Wählen Sie dann in der Liste der Indizierung.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option Web Crawler Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Datenquellendetails die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. In Tags, für Neues Tag hinzufügen — können Sie Ihre Ressourcen suchen und filtern oder IhreAWS -Kosten verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. Wählen Sie für Source je nach Anwendungsfall zwischen Quell-URLs und Quell-Sitemaps und geben Sie die Werte für jede ein.

      Anmerkung

      Sie können bis zu 10 Quell-URLs und 3 Sitemaps hinzufügen.

      Wenn du eine Sitemap crawlen möchtest, ist es wichtig zu überprüfen, ob die Basis- oder Root-URL dieselbe Basis-URL ist, die für die auf deiner Sitemap-Seite aufgeführten URLs verwendet wird. Wenn Ihre Sitemap-URL beispielsweise https://example.com/sitemap-page.html lautet, sollten die auf dieser Sitemap-Seite aufgeführten URLs auch die Basis-URL "https://example.com/“ verwenden.

    2. (Optional) Geben Sie für den Web-Proxy die folgenden Informationen ein:

      1. Hostname — Der Hostname, für den der Web-Proxy erforderlich ist.

      2. Portnummer — Der vom Host-URL-Transportprotokoll verwendete Port. Die Portnummer sollte ein numerischer Wert zwischen 0 und 65535 sein.

      3. Für Web-Proxy-Anmeldeinformationen — Wählen Sie ein vorhandenes Geheimnis aus, oder erstellen Sie ein neues Geheimnis, um Ihre Anmeldeinformationen für die Webcrawler-Authentifizierung zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      4. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-WebCrawler -' wird automatisch zu Ihrem geheimen Namen hinzugefügt.

        2. Für Benutzername und Passwort — Geben Sie diese grundlegenden Authentifizierungsdaten für Ihre Websites ein.

        3. Wählen Sie Speichern.

    3. (Optional) Hosts mit Authentifizierung — Wählen Sie diese Option aus, um weitere Hosts mit Authentifizierung hinzuzufügen.

    4. IAM-Rolle — Wählen Sie eine vorhandene IAM-Rolle aus oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Repository-Anmeldeinformationen und Indexierung.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    5. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Crawlbereich — Wählen Sie die Art der Webseiten aus, die Sie crawlen möchten.

    2. Crawl-Tiefe — Wählen Sie die Anzahl der Ebenen aus der Seed-URL aus, die gecrawltAmazon Kendra werden sollen.

    3. Geben Sie unter Erweiterte Crawling-Einstellungen und Zusätzliche Konfiguration die folgenden Informationen ein:

      1. Maximale Dateigröße — Die maximale Größe der Webseite oder des Anhangs, die gecrawlt werden soll. Mindestens 0,000001 MB (1 Byte). Maximal 50 MB.

      2. Maximale Anzahl von Links pro Seite — Die maximale Anzahl von Links, die pro Seite gecrawlt wurden. Links werden in der Reihenfolge ihres Auftretens gecrawlt. Mindestens 1 Link/Seite. Maximal 1000 Links/Seite.

      3. Maximale Drosselung — Die maximale Anzahl an URLs, die pro Hostname pro Minute gecrawlt werden. Mindestens 1 URLs/Hostname/Minute. Maximal 300 URLs/Hostname/Minute.

      4. Regex-Muster — Fügen Sie ein Muster hinzu, um nur bestimmte URLs einzubeziehen oder um URLs, Dateitypen oder bestimmte Dateien in Ihrem Repository auszuschließen. Sie können insgesamt 100 Muster haben.

    4. Wählen Sie im Synchronisierungslaufzeitplan für Häufigkeit aus, wie oft mit Ihrer Datenquelle synchronisiertAmazon Kendra werden soll.

    5. Wählen Sie Next (Weiter).

  8. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

Um eine VerbindungAmazon Kendra zum Webcrawler herzustellen

Sie müssen die folgende WebCrawlerConfigurationAPI angeben:

  • URLs — Geben Sie die Seed- oder Startpunkt-URLs der Websites oder die Sitemap-URLs der Websites an, mit SeedUrlConfigurationdenen Sie crawlen möchten SiteMapsConfiguration.

    Wenn du eine Sitemap crawlen möchtest, ist es wichtig zu überprüfen, ob die Basis- oder Root-URL dieselbe Basis-URL ist, die für die auf deiner Sitemap-Seite aufgeführten URLs verwendet wird. Wenn Ihre Sitemap-URL beispielsweise https://example.com/sitemap-page.html lautet, sollten die auf dieser Sitemap-Seite aufgeführten URLs auch die Basis-URL "https://example.com/“ verwenden.

  • Geheimer Amazon Resource Name (ARN) — Wenn eine Website eine Standardauthentifizierung erfordert, geben Sie den Hostnamen, die Portnummer und ein Secret an, das Ihre grundlegenden Authentifizierungsdaten wie Ihren Benutzernamen und Ihr Passwort speichert. Sie stellen den ARN mithilfe der AuthenticationConfigurationAPI bereit. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "username": "user name",
    "password": "password"
}

    Sie können Web-Proxy-Anmeldeinformationen mithilfe eines geheimenAWSSecrets Manager Geheimnisses angeben. Sie verwenden die ProxyConfigurationAPI, um den Hostnamen und die Portnummer der Website sowie optional das Geheimnis anzugeben, in dem Ihre Webproxy-Anmeldeinformationen gespeichert sind.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Sie können auch die folgenden optionalen Funktionen hinzufügen:

  • Crawlmodus — Wählen Sie aus, ob nur die Hostnamen von Websites oder Hostnamen mit Subdomänen oder auch andere Domains gecrawlt werden sollen, auf die die Webseiten verlinken.

  • Die „Tiefe“ oder Anzahl der Ebenen in einer Website von der Seed Ebene bis zum Crawl. Wenn eine Website beispielsweise über 3 Ebenen verfügt — Indexebene (d. h. Seed in diesem Beispiel), Unterabschnittsebene und Unterabschnittsebene — und Sie nur daran interessiert sind, Informationen von der Indexebene zur Abschnittsebene (Ebenen 0 bis 1) zu crawlen, können Sie Ihre Tiefe auf 1 setzen.

  • Die maximale Anzahl an URLs auf einer einzelnen Webseite, die Sie crawlen möchten.

  • Die maximale Größe einer zu crawlen Webseite in MB.

  • Die maximale Anzahl an URLs, die pro Website-Host pro Minute gecrawlt werden.

  • Die Web-Proxy-Informationen zum Herstellen einer Verbindung zu internen Websites und zum Crawlen interner Websites.

  • Die Authentifizierungsinformationen für den Zugriff und das Crawling von Websites, die eine Benutzerauthentifizierung erfordern.

  • Mit dem Tool Custom Document Enrichment können Sie HTML-Metatags als Felder extrahieren. Weitere Informationen finden Sie unter Anpassen der Metadaten von Dokumenten während des Erfassungsprozesses. Ein Beispiel für das Extrahieren von HTML-Metatags finden Sie unter CDE-Beispiele.

  • Ein- und Ausschlussfilter — Geben Sie an, ob bestimmte URLs ein- oder ausgeschlossen werden sollen.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.
