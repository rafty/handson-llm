Microsoft-Teams

Microsoft Teams ist ein Tool für die Zusammenarbeit in Unternehmen für Messaging, Besprechungen und Filesharing. Wenn Sie ein Microsoft Teams-Benutzer sind, können SieAmazon Kendra damit Ihre Microsoft Teams-Datenquelle indexieren.

Sie können mithilfe der Amazon KendraKonsole und der TemplateConfigurationAPI eine VerbindungAmazon Kendra zu Ihrer Microsoft Teams-Datenquelle herstellen.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra Microsoft Teams-Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Voraussetzungen

Bevor Sie Ihre Microsoft Teams-DatenquelleAmazon Kendra zur Indexierung verwenden können, nehmen Sie diese Änderungen in Ihren Microsoft Teams undAWS Konten vor.

Stellen Sie in Microsoft Teams sicher, dass Sie über Folgendes verfügen:

  • Hat ein Microsoft Teams-Konto in Office 365 erstellt.

  • Ihre Microsoft 365-Mandanten-ID wurde kopiert. Sie finden Ihre Mandanten-ID in den Eigenschaften Ihres Azure Active Directory-Portals.

  • Konfigurierte ein OAuth 2.0-Berechtigungstoken, das eine Client-ID und einen geheimen Client-Schlüssel enthält. Weitere Informationen finden Sie in der Azure-Dokumentation zur Verwaltung von Zugriffstoken für Teams.

  • Die erforderlichen Berechtigungen wurden hinzugefügt. Sie können wählen, ob Sie alle Berechtigungen hinzufügen möchten, oder Sie können den Umfang einschränken, indem Sie weniger Berechtigungen auswählen, je nachdem, welche Entitäten gecrawlt werden sollen.   • Geprüft, dass jedes Dokument in Microsoft Teams und in anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

Stellen Sie in IhremAWS Konto sicher, dass Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Hat Ihre Microsoft Teams-Authentifizierungsdaten in einemAWSSecrets Manager Secret gespeichert und, falls Sie die API verwenden, den ARN des Geheims notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder kein Geheimnis haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn Sie Ihre Microsoft Teams-Datenquelle damit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Verbindungsanweisungen

Um eine VerbindungAmazon Kendra zu Ihrer Microsoft Teams-Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer Microsoft Teams-Datenquelle angeben, damit auf Ihre Daten zugreifenAmazon Kendra kann. Falls Sie Microsoft Teams noch nicht konfiguriert haben,Amazon Kendra sehen Sie sich das anVoraussetzungen.

Console

So stellen Sie eine VerbindungAmazon Kendra zu Microsoft Teams her

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich Indizes und dann in der Liste der Indizes den Index aus, den Sie verwenden möchten.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option Microsoft Teams Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Daten angeben die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags, for Neues Tag hinzufügen — Tags, um Ressourcen zu suchen und zu filtern oder IhreAWS -Kosten zu verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. Quelle — Geben Sie Ihre Microsoft 365-Mandanten-ID ein. Sie finden Ihre Mandanten-ID in den Eigenschaften Ihres Azure Active Directory-Portals.

    2. AWSSecrets Managersecret — Wählen Sie ein vorhandenes Geheimnis aus oder erstellen Sie ein neuesSecrets Manager Geheimnis, um Ihre Microsoft Teams-Authentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-Microsoft Teams-' wird automatisch zu Ihrem geheimen Namen hinzugefügt.

        2. Für Client-ID und Client-Schlüssel — Geben Sie die Werte der Authentifizierungsdaten ein, die Sie mit Ihrem Microsoft Teams-Konto generiert haben.

      2. Wählen Sie Speichern.

    3. Zahlungsmodell — Sie können ein Lizenz- und Zahlungsmodell für Ihr Microsoft Teams-Konto wählen. Die Zahlungsmodelle des Modells A sind auf Lizenz- und Zahlungsmodelle beschränkt, für die die Einhaltung der Sicherheitsvorschriften erforderlich ist. Die Zahlungsmodelle des Modells B eignen sich für Lizenz- und Zahlungsmodelle, für die keine Sicherheitsvorschriften erforderlich sind.

    4. Virtual Private Cloud (VPC) — Sie können wählen, ob Sie eine VPC verwenden möchten. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    5. Identity Crawler — Wenn der Identity Crawler aktiviert ist, werden IdentitätsinformationenAmazon Kendra synchronisiert. Wenn Sie den Identity Crawler ausschalten möchten, müssen Sie die Hauptinformationen mithilfe der PutPrincipalMappingAPI hochladen.

    6. IAM-Rolle —Wählen Sie eine vorhandene IAM-Rolle oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    7. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen konfigurieren die folgenden Informationen ein:

    1. Inhalt synchronisieren — Wählen Sie die zu synchronisierenden Inhalte aus.

    2. Zusätzliche Konfiguration — Sie können diese Einstellungen optional verwenden, um bestimmte Inhalte zu indexieren, anstatt alle Dokumente zu synchronisieren.

    3. Synchronisierungsmodus — Sie können wählen, wie Sie Ihren Index aktualisieren möchten, wenn sich der Inhalt Ihrer Datenquelle ändert.

      1. Wenn Sie die vollständige Synchronisierung wählen,Amazon Kendra werden alle Inhalte in allen Entitäten synchronisiert, unabhängig vom vorherigen Synchronisierungsstatus.

      2. Wenn Sie sich für die Synchronisierung neuer oder geänderter Inhalte entscheiden,Amazon Kendra werden nur neue oder geänderte Inhalte synchronisiert.

      3. Wenn Sie sich für die Synchronisierung neuer, geänderter oder gelöschter Inhalte entscheiden,Amazon Kendra werden nur neue, geänderte oder gelöschte Inhalte synchronisiert.

  8. Geben Sie auf der Seite Feldzuordnungen festlegen die folgenden Informationen ein:

    1. Standarddatenquellenfelder — Wählen Sie eines derAmazon Kendra generierten Standarddatenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

So stellen Sie eine VerbindungAmazon Kendra zu Microsoft Teams her

Sie müssen einen JSON des Datenquellenschemas mithilfe der TemplateConfigurationAPI angeben. Sie müssen die folgenden Informationen angeben:

  • Datenquelle —Sie müssen die Datenquelle angeben alsMSTEAMS.

  • Mandanten-ID — Sie finden Ihre Mandanten-ID in den Eigenschaften Ihres Azure Active Directory-Portals.

  • Typ —Geben SieTEMPLATE dies als Typ an, wenn Sie anrufenCreateDataSource.

  • Geheimer Amazon Resource Name (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr Microsoft Teams-Konto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "clientId": "client ID",
    "clientSecret": "clientSecret",
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager geheimes Element und zum Aufrufen der erforderlichen öffentlichen APIs für den Microsoft Teams-Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für Microsoft Teams-Datenquellen.

Sie können auch die folgenden optionalen-Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie anVpcConfiguration, wann Sie anrufenCreateDataSource. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

  • Identity Crawler aktivieren —Geben Sie an, ob der Identity Crawler aktiviert werden soll. Wenn Identity Crawler deaktiviert ist, müssen Sie die Hauptinformationen mithilfe der PutPrincipalMappingAPI hochladen. Das Crawlen von Identitätsinformationen von Benutzern und Gruppen mit Zugriff auf bestimmte Dokumente ist für die Filterung des Benutzerkontextes nützlich. Die Suchergebnisse werden nach dem Zugriff des Benutzers oder seiner Gruppe auf Dokumente gefiltert. Weitere Informationen finden Sie unter Filtern nach Benutzerkontext.

  • Ein- und Ausschlussfilter — Geben Sie an, ob bestimmte Dokumente aus Microsoft Teams-Channels ein- oder ausgeschlossen werden sollen.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre Microsoft Teams-Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

Eine Liste weiterer wichtiger zu konfigurierender JSON-Schlüssel finden Sie unterMicrosoft Teams-Vorlagenschema.
