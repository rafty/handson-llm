Direktes Hinzufügen von häufig gestellten Fragen (FAQs) zu einem Index

Mit der Konsole oder derCreateFaq API können Sie häufig gestellte Fragen (FAQs) direkt zu Ihrem Index hinzufügen. Hinzufügen von FAQs zu einem Index ist ein asynchroner Vorgang. Sie legen die Daten für die FAQ in eine Datei, die Sie in einemAmazon Simple Storage Service Bucket speichern. Sie können CSV- oder JSON-Dateien als Eingabe für Ihre FAQ verwenden:

  • Basic CSV — Eine CSV-Datei, in der jede Zeile eine Frage, eine Antwort und eine optionale URL mit weiteren Informationen zur Antwort enthält.

  • Benutzerdefiniertes CSV — Eine CSV-Datei, die Fragen, Antworten und einen Header enthält, der benutzerdefinierte Attribute definiert, die Sie verwenden können, um FAQ-Antworten zu facettieren, anzuzeigen oder zu sortieren. Sie können auch Zugriffskontrollattribute definieren, um die Antworten auf häufig gestellte Fragen auf bestimmte Benutzer und Gruppen zu beschränken.

  • JSON — Eine JSON-Datei, die Fragen und Antworten enthält. Optional kann sie auch benutzerdefinierte Attribute und Zugriffskontrollattribute enthalten. Sie können Attribute definieren, um FAQ-Antworten zu facettieren, anzuzeigen und zu sortieren. Oder Sie können Zugriffskontrollattribute verwenden, die die Antworten auf häufig gestellte Fragen auf bestimmte Benutzer und Gruppen beschränken.

Im Folgenden finden Sie beispielsweise eine einfache CSV-Datei, die Antworten auf Fragen zu kostenlosen Kliniken in Spokane, Washington, USA, und Mountain View, Missouri, USA, enthält.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

Wenn Sie eine benutzerdefinierte CSV- oder JSON-Datei für die Eingabe verwenden, können Sie benutzerdefinierte Attribute für Ihre FAQ-Fragen deklarieren. Sie können beispielsweise ein benutzerdefiniertes Attribut erstellen, das jeder FAQ-Frage eine Abteilung zuweist. Wenn die häufig gestellten Fragen in einer Antwort zurückgegeben werden, können Sie die Abteilung als Facette verwenden, um die Suche einzugrenzen.

Ein benutzerdefiniertes Attribut muss einem Indexfeld zugeordnet werden. Sie können ein integriertes Feld verwenden oder ein benutzerdefiniertes Indexfeld verwenden. In der Konsole verwenden Sie die Facet-Definitionsseite, um ein Indexfeld zu erstellen. Wenn Sie die API verwenden, müssen Sie zuerst ein Indexfeld mit derUpdateIndex API erstellen.

Der Attributtyp in der FAQ-Datei muss mit dem Typ des zugehörigen Indexfeldes übereinstimmen. Das integrierte_authors Feld ist beispielsweise einSTRING_LIST Typfeld. Sie müssen also Werte für das_authors Feld als Zeichenkettenliste in Ihrer FAQ-Datei angeben. Sie können den Typ der Indexfelder mithilfe der Facet-Definitionsseite in der Konsole oder mithilfe derDescribeIndex API überprüfen.

Wenn Sie ein Indexfeld erstellen, das einem benutzerdefinierten Attribut zugeordnet ist, können Sie es als anzeigbar, facetabell oder sortierbar markieren. Sie können ein benutzerdefiniertes Attribut nicht durchsuchbar machen.

Zusätzlich zu den benutzerdefinierten Attributen können Sie auch die folgenden integrierten Attribute in einer benutzerdefinierten CSV- oder JSON-Datei verwenden:

  • _authors(Stringliste) — Eine Liste der Autoren der Antworten auf die häufig gestellten Fragen.

  • _category(String) — Eine Kategorie, die die Antworten auf häufig gestellte Fragen mit anderen ähnlichen Dokumenten gruppiert.

  • _created_at(ISO 8601-codierte Zeichenfolge) — Datum und Uhrzeit der Erstellung der FAQ-Frage. Datum und Uhrzeit müssen als ISO 8601-codierte Zeichenfolge formatiert sein.

    Sie können die Zeitzone bei Bedarf auch im ISO-8601-Format angeben. Beispiel: 2012-03-25T 12:30:10 + 01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in der mitteleuropäischen Zeitzone (plus 10 Sekunden).

  • _last_updated_at(ISO 8601-codierte Zeichenfolge) — Datum und Uhrzeit der Aktualisierung der FAQ-Frage. Datum und Uhrzeit müssen als ISO 8601-codierte Zeichenfolge formatiert sein.

    Beispiel: 2012-03-25T 12:30:10 + 01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in der mitteleuropäischen Zeitzone (plus 10 Sekunden).

  • _source_uri(String) — Eine URL für ein Dokument mit weiteren Informationen zur FAQ-Antwort.

  • _version(String) — Die Version der FAQ-Frage.

  • _view_count(Lang) — Die Häufigkeit, mit der die FAQ-Frage in den Suchergebnissen angezeigt wurde.

Einfache CSV-Datei

Verwenden Sie eine einfache CSV-Datei, wenn Sie eine einfache Struktur für Ihre FAQs verwenden möchten. In einer einfachen CSV-Datei hat jede Zeile zwei oder drei Felder: eine Frage, eine Antwort und eine optionale Quell-URI, die auf ein Dokument mit weiteren Informationen verweist.

Der Inhalt der Datei muss dem RFC 4180 Common Format and MIME-Typ für CSV-Dateien (Comma Separated Values) entsprechen.

Im Folgenden finden Sie eine FAQ-Datei im grundlegenden CSV-Format.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

Benutzerdefinierte CS-Datei

Verwenden Sie eine benutzerdefinierte CSV-Datei, wenn Sie Ihren FAQ-Fragen benutzerdefinierte Attribute hinzufügen möchten. Für eine benutzerdefinierte CSV-Datei verwenden Sie eine Kopfzeile in Ihrer CSV-Datei, um die zusätzlichen Attribute zu definieren.

Die CS-Datei muss die folgenden zwei erforderlichen Attribute enthalten:

  • _question—Die häufig gestellte Frage

  • _answer—Die Antwort auf die häufig gestellte Frage

Ihre Datei kann integrierte und benutzerdefinierte Attribute enthalten. Im Folgenden sehen Sie ein Beispiel für eine benutzerdefinierte CS-Datei.

_question,_answer,_last_updated_at,custom_string
How many free clinics are in Spokane WA?, 13, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services
How many free clinics are there in Mountain View Missouri?, 7, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services

Der Inhalt der Datei mit benutzerdefinierten Attributen muss dem RFC 4180 Common Format and MIME-Typ für CSV-Dateien (Comma Separated Values) entsprechen.

Es gibt vier Arten von benutzerdefinierten Attributen:

  • Datum — ISO-8601-codierte Datum- und Uhrzeitwerte.

    Beispiel: 2012-03-25T 12:30:10 + 01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in der mitteleuropäischen Zeitzone (plus 10 Sekunden).

  • Lang — Zahlen, z.1234 B.

  • Zeichenfolge — Zeichenkettenwerte. Wenn Ihre Zeichenfolge Kommas enthält, schließen Sie den gesamten Wert in doppelte Anführungszeichen („) ein (z. B."custom attribute, and more").

  • Zeichenfolge — Eine Liste von Zeichenfolgewerten. Führen Sie die Werte in einer kommagetrennten Liste auf, die in Anführungszeichen („) enthalten ist (z. B."item1, item2, item3"). Wenn die Liste nur einen einzigen Eintrag enthält, können Sie die Anführungszeichen weglassen (z. B.item1).

Eine benutzerdefinierte CSV-Datei kann Benutzerkontextfelder enthalten. Sie können diese Felder verwenden, um den Zugriff auf die FAQ auf bestimmte Benutzer und Gruppen zu beschränken. Um nach dem Benutzerkontext zu filtern, muss der Benutzer Benutzer- und Gruppeninformationen in der Abfrage angeben. Andernfalls werden alle relevanten FAQs zurückgegeben. Weitere Informationen finden Sie unter Filterung nach Benutzerkontext.

Es gibt vier Benutzerkontextfilter für häufig gestellte Fragen:

  • _acl_user_allow— Benutzer in der Zulassungsliste können die häufig gestellten Fragen in der Abfrageantwort sehen. Die FAQ wird nicht an andere Benutzer zurückgegeben.

  • _acl_user_deny—Benutzer in der Ablehnungsliste können die häufig gestellten Fragen in der Abfrageantwort nicht sehen. Die häufig gestellten Fragen werden an alle anderen Benutzer zurückgegeben, wenn sie für die Anfrage relevant sind.

  • _acl_group_allow— Benutzer, die Mitglieder einer zugelassenen Gruppe sind, können die häufig gestellten Fragen in der Abfrageantwort sehen. Die häufig gestellten Fragen werden nicht an Benutzer zurückgegeben, die Mitglieder einer anderen Gruppe sind.

  • _acl_group_deny—Benutzer, die Mitglieder einer abgelehnten Gruppe sind, können die häufig gestellten Fragen in der Abfrageantwort nicht sehen. Die häufig gestellten Fragen werden an andere Gruppen zurückgegeben, wenn sie für die Anfrage relevant sind.

Geben Sie die Werte für die Zulassen- und Verweigerungslisten in kommagetrennten Listen an, die in Anführungszeichen stehen (z. B."user1,user2,user3"). Sie können einen Benutzer oder eine Gruppe entweder in eine Zulassungsliste oder eine Ablehnungsliste aufnehmen, aber nicht in beide. Wenn Sie einen Benutzer oder eine Gruppe in beide aufnehmen, erhalten Sie einen Fehler.

Im Folgenden sehen Sie ein Beispiel für eine benutzerdefinierte CS-Datei mit Benutzerkontext-Informationen.

_question, _answer, _acl_user_allow, _acl_user_deny, _acl_group_allow, _acl_group_deny
How many free clinics are in Spokane WA?, 13, userID4565, "userID6201, userID7552", groupBasicRate, "groupBasicPlusRate,groupPremiumRate"

JSON-Datei

Sie können eine JSON-Datei verwenden, um Fragen, Antworten und Attribute für Ihren Index bereitzustellen. Sie können jedes der integrierten Attribute und benutzerdefinierten Attribute zu den FAQ hinzufügen. Dies ist das Schema für die JSON-Datei.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": string,
            "Answer": string,
            "Attributes": {
                string: object
                additional attributes
            },
            "AccessControlList": [
               {
                   "Name": string,
                   "Type": enum( "GROUP" | "USER" ),
                   "Access": enum( "ALLOW" | "DENY" )
               },
               additional user context
            ]
        },
        additional FAQ documents
    ]
}

Diese JSON-Beispieldatei zeigt zwei FAQ-Dokumente. Einer von ihnen hat nur die erforderliche Frage und Antwort. Die andere hat auch zusätzliche Attribute und Benutzerkontextinformationen.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": "How many free clinics are in Spokane WA?",
            "Answer": "13"
        },
        {
            "Question": "How many free clinics are there in Mountain View Missouri?",
            "Answer": "7",
            "Attributes": {
                "_source_uri": "https://www.freeclinics.com",
                "_category": "Charitable Clinics"
            },
            "AccessControlList": [
               {
                   "Name": "user@amazon.com",
                   "Type": "USER",
                   "Access": "ALLOW"
               },
               {
                   "Name": "Admin",
                   "Type": "GROUP",
                   "Access": "ALLOW"
               }
            ]
        }
    ]
}

Es gibt vier Arten von benutzerdefinierten Attributen für JSON-Dateien:

  • Datum — Ein JSON-Zeichenkettenwert mit ISO 8601-kodierten Datums- und Uhrzeitwerten.

    Es ist wichtig, dass die Zeitzone im ISO-8601-Format enthalten ist. Beispiel: 2012-03-25T 12:30:10 + 01:00 ist das ISO-8601-Datums-/Uhrzeitformat für den 25. März 2012 um 12:30 Uhr (plus 10 Sekunden) in der mitteleuropäischen Zeitzone (plus 10 Sekunden).

  • Long — Ein JSON-Nummernwert, z.1234 B.

  • Zeichenfolge — Ein JSON-Zeichenfolgenwert (z. B."custom attribute").

  • Zeichenkettenliste — Ein JSON-Array mit Zeichenfolgenwerten (z. B.["item1,item2,item3"]).

Zusätzlich zu den integrierten und benutzerdefinierten Attributen können Sie Benutzerkontextinformationen für die häufig gestellten Fragen in einer JSON-Datei angeben. Sie können Benutzerkontextinformationen angeben, um den Zugriff auf die FAQ-Inhalte nach Benutzern und Gruppen zu beschränken. Sie können einen Benutzer oder eine Gruppe entweder in eine Zulassungsliste oder eine Ablehnungsliste aufnehmen, aber nicht in beide. Wenn Sie einen Benutzer oder eine Gruppe in beide aufnehmen, erhalten Sie einen Fehler. Um nach dem Benutzerkontext zu filtern, müssen Sie Benutzer- und Gruppeninformationen in der Abfrage angeben. Andernfalls werden alle relevanten FAQs zurückgegeben. Weitere Informationen finden Sie unter Filterung nach Benutzerkontext.

Dieses JSON-Beispiel fügt Benutzerkontext zu einer FAQ hinzu.

"AccessControlList": [
                {
                    "Name": "group or user name",
                    "Type": "GROUP | USER",
                    "Access": "ALLOW | DENY"
                },
                additional user context
            ]

Verwenden Sie Ihre FAQ-Datei

Nachdem Sie Ihre FAQ-Eingabedatei in einem S3-Bucket gespeichert haben, verwenden Sie die Konsole oder dieCreateFaq API, um die Fragen und Antworten in Ihren Index aufzunehmen. Wenn Sie eine FAQ aktualisieren möchten, löschen Sie die FAQ und erstellen Sie sie erneut. Sie verwenden dieDeleteFaq API, um eine FAQ zu löschen.

Sie müssen eineIAM Rolle angeben, die Zugriff auf den S3-Bucket hat, der Ihre Quelldateien enthält. Sie geben die Rolle in der Konsole oder imRoleArn Parameter an. Im Folgenden sehen Sie ein Beispiel für ein Programm, das eine FAQ-Datei zu einem Index hinzufügt.

Python
import boto3

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the IAM role ARN required to index documents in an S3 bucket
role_arn = "arn:aws:iam::${accountId}:role/${roleName}"

# Provide the S3 bucket path information to the FAQ file
faq_path = {
    "Bucket": "bucket-name",
    "Key": "FreeClinicsUSA.csv"
}

response = kendra.create_faq(
    S3Path =  faq_path,
    Name = "FreeClinicsUSA",
    IndexId = index_id,
    RoleArn = role_arn
)

print(response)
Java
package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.CreateFaqRequest;
import software.amazon.awssdk.services.kendra.model.CreateFaqResponse;
import software.amazon.awssdk.services.kendra.model.S3Path;

public class AddFaqExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String indexId = "yourIndexId";
        String roleArn = "your role for accessing S3 files";

        CreateFaqRequest createFaqRequest = CreateFaqRequest
            .builder()
            .indexId(indexId)
            .name("FreeClinicsUSA")
            .roleArn(roleArn)
            .s3Path(
                S3Path
                    .builder()
                    .bucket("an-aws-kendra-test-bucket")
                    .key("FreeClinicsUSA.csv")
                    .build())
            .build();

        CreateFaqResponse response = kendra.createFaq(createFaqRequest);

        System.out.println(String.format("The result of creating FAQ: %s", response));

    }
}

FAQ-Dateien in anderen Sprachen als Englisch

Sie können eine häufig gestellte Frage in einer unterstützten Sprache indexieren. Amazon Kendraindexiert häufig gestellte Fragen standardmäßig auf Englisch, wenn Sie keine Sprache angeben. Sie geben den Sprachcode an, wenn Sie den CreateFaqVorgang aufrufen, oder Sie können den Sprachcode für eine FAQ als Feld in die FAQ-Metadaten aufnehmen. Wenn eine FAQ in ihren Metadaten, die in einem Metadatenfeld angegeben sind, keinen Sprachcode enthält, wird die FAQ mit dem Sprachcode indexiert, der beim Aufrufen desCreateFAQ Vorgangs angegeben wurde. Um ein FAQ-Dokument in einer unterstützten Sprache in der Konsole zu indexieren, gehen Sie zu FAQs und wählen Sie FAQ hinzufügen aus. Sie wählen eine Sprache aus der Dropdown-Liste Sprache aus.
