Gmail

Gmail ist ein von Google entwickelter E-Mail-Client, mit dem Sie E-Mail-Nachrichten mit Dateianhängen senden können. Gmail-Nachrichten können mithilfe von Ordnern und Labels sortiert und in Ihrem E-Mail-Posteingang gespeichert werden. Sie können es verwendenAmazon Kendra, um Ihre E-Mail-Nachrichten und Nachrichtenanhänge zu indexieren. Sie können auch konfigurierenAmazon Kendra, dass bestimmte E-Mail-Nachrichten, Nachrichtenanhänge und Bezeichnungen für die Indizierung ein- oder ausgeschlossen werden.

Sie können mithilfe der Amazon KendraKonsole und der TemplateConfigurationAPI eine VerbindungAmazon Kendra zu Ihrer Gmail-Datenquelle herstellen.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra Gmail-Datenquellenconnector finden Sie unterFehlerbehebung bei Datenquellen.

Voraussetzungen

Bevor Sie Ihre Gmail-DatenquelleAmazon Kendra zur Indexierung verwenden können, nehmen Sie diese Änderungen in Ihrem Gmail und IhrenAWS Konten vor.

In Gmail müssen Sie über Folgendes verfügen:

  • Ich habe ein Google Cloud Platform-Administratorkonto erstellt und ein Google Cloud-Projekt erstellt.

  • Aktivierte Gmail API und Admin SDK API in Ihrem Admin-Konto.

  • Sie haben ein Dienstkonto erstellt und einen privaten JSON-Schlüssel für Gmail heruntergeladen. Informationen zur Erstellung und zum Zugriff auf Ihren privaten Schlüssel finden Sie in der Google Cloud-Dokumentation zum Erstellen eines Dienstkontoschlüssels und der Anmeldeinformationen für das Dienstkonto.

  • Die E-Mail-Adresse Ihres Administratorkontos, die E-Mail-Adresse Ihres Dienstkontos und Ihren privaten Schlüssel zur Authentifizierung wurden kopiert.

  • Die folgenden OAuth-Bereiche (mit einer Administratorrolle) für Ihren Benutzer und die geteilten Verzeichnisse, die Sie indexieren möchten, wurden hinzugefügt:

    • https://www.googleapis.com/auth/admin.directory.user.readonly

    • https://www.googleapis.com/auth/gmail.readonly

  • Hat geprüft, dass jedes Dokument in Gmail und in anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

In IhremAWS Konto müssen Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Hat Ihre Gmail-AuthentifizierungsdatenAWSSecrets Manager geheim gespeichert und, falls Sie die API verwenden, den ARN des Geheimnisses notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder keinen geheimen Schlüssel haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und einen neuenSecrets Manager Schlüssel zu erstellen, wenn Sie Ihre Gmail-Datenquelle damit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Anweisungen zum Anschluss

Um eine VerbindungAmazon Kendra zu Ihrer Gmail-Datenquelle herzustellen, müssen Sie Details zu Ihren Gmail-Anmeldedaten angeben, damit Sie auf Ihre Daten zugreifenAmazon Kendra können. Falls Sie Gmail noch nicht für konfiguriert habenAmazon Kendra, finden Sie weitere Informationen unterVoraussetzungen.

Console

So stellen Sie eine VerbindungAmazon Kendra zu Gmail her

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich die Option Indizes und dann in der Liste der Indizes den Index aus, den Sie verwenden möchten.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option Gmail-Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Daten quellendetails die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags gibt es die Option Neues Tag hinzufügen — Tags, um Ressourcen zu suchen und zu filtern oder IhreAWS -Kosten zu verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. Unter Authentifizierung für AWSSecrets Managergeheim — Wählen Sie einen vorhandenen geheimen Schlüssel aus oder erstellen Sie einen neuenSecrets Manager geheimen Schlüssel, um Ihre Gmail-Authentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis.

        2. Kunden-E-Mail — Die Kunden-E-Mail, die Sie aus Ihrem Google-Dienstkonto kopiert haben.

        3. E-Mail-Adresse des Administratorkontos — Die E-Mail-Adresse des Administratorkontos, die Sie verwenden möchten.

        4. Privater Schlüssel — Der private Schlüssel, den Sie aus Ihrem Google-Dienstkonto kopiert haben.

        5. Wählen Sie Speichern.

    2. Virtual Private Cloud (VPC) — Sie können wählen, ob Sie eine VPC verwenden möchten. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    3. IAM-Rolle —Wählen Sie eine vorhandene IAM-Rolle oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und -Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    4. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Wählen Sie im Bereich Synchronisieren für Entitätstypen die Option Nachrichtenanhänge aus, um Nachrichtenanhänge zu synchronisieren. Nachrichten werden standardmäßig synchronisiert.

    2. (Optional) Geben Sie für zusätzliche Konfiguration die folgenden Informationen ein:

      1. Datumsbereich —Geben Sie einen Datumsbereich ein, um das Start- und Enddatum der zu crawlenden E-Mails anzugeben.

      2. E-Mail-Domains — Schließt E-Mails ein, die auf Domains basieren, oder schließt sie aus.

      3. Stichwörter in Betreffs — Schließt E-Mails auf der Grundlage von Schlüsselwörtern in ihren Betreffs ein oder aus.

        Anmerkung

        Sie können sich auch dafür entscheiden, alle Dokumente einzubeziehen, die allen von Ihnen eingegebenen Stichwörtern entsprechen.

      4. Labels — Fügen Sie reguläre Ausdrucksmuster hinzu, um bestimmte Bezeichnungen ein- oder auszuschließen. Sie können bis zu 100 Muster hinzufügen.

      5. Anlagen — Fügen Sie reguläre Ausdrucksmuster hinzu, um bestimmte Anlagen ein- oder auszuschließen. Sie können bis zu 100 Muster hinzufügen.

    3. Wählen Sie für den Synchronisierungsmodus aus, wie Sie Ihren Index aktualisieren möchten, wenn sich der Inhalt Ihrer Datenquelle ändert. Wenn Sie Ihre DatenquelleAmazon Kendra zum ersten Mal mit synchronisieren, werden standardmäßig alle Inhalte synchronisiert.

      • Vollständige Synchronisierung —Synchronisiert den gesamten Inhalt unabhängig vom vorherigen Synchronisierungsstatus.

      • Synchronisieren neuer, geänderter oder gelöschter Inhalte — Synchronisieren Sie nur neue, geänderte und gelöschte Inhalte.

        Wichtig

        Da es keine API zum Aktualisieren dauerhaft gelöschter Gmail-Nachrichten gibt, ist eine neue, geänderte oder gelöschte Inhaltssynchronisierung wie folgt:

        • Entfernt Nachrichten, die dauerhaft aus Gmail gelöscht wurden, nicht aus IhremAmazon Kendra Index

        • Änderungen in den E-Mail-Labels von Gmail werden nicht synchronisiert

        Um Ihre Änderungen an der Bezeichnung Ihrer Gmail-Datenquelle und dauerhaft gelöschte E-Mail-Nachrichten mit IhremAmazon Kendra Index zu synchronisieren, müssen Sie in regelmäßigen Abständen vollständige Crawls durchführen.

    4. Geben Sie im Synchronisierungslaufzeitplan für Häufigkeit an, wie oftAmazon Kendra die Synchronisierung mit Ihrer Datenquelle erfolgt.

    5. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen festlegen die folgenden Informationen ein:

    1. Für Nachrichten und Nachrichtenanhänge — Wählen Sie eines derAmazon Kendra generierten Standarddatenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

      Anmerkung

      Amazon KendraDer Gmail-Datenquellenconnector unterstützt das Erstellen benutzerdefinierter Indexfelder aufgrund von API-Einschränkungen nicht.

    2. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

TemplateConfiguration API

So stellen Sie eine VerbindungAmazon Kendra zu Gmail her

Sie müssen mithilfe der TemplateConfigurationAPI Folgendes angeben:

  • Datenquelle —Sie müssen die Datenquelle angeben alsGMAIL.

  • Datenquellenschema — Schließt ein JSON ein, das das Datenquellenschema enthält. Informationen zum Anzeigen des Vorlagenschemas finden Sie unter Datenquellenschemas.

  • Typ —Geben SieTEMPLATE dies als Typ an, wenn Sie anrufenCreateDataSource.

  • Geheimer Amazon-Ressourcenname (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr Gmail-Konto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "adminAccountEmailId": "${adminAccountEmailId}",
    "clientEmailId": "${clientEmailId}",
    "privateKey": "${privateKey}"
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager geheimes Passwort und zum Aufrufen der erforderlichen öffentlichen APIs für den Gmail-Connector und zu gebenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für Gmail-Datenquellen.

Sie können auch die folgenden optionalen Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie anVpcConfiguration, wann Sie anrufenCreateDataSource. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

  • Synchronisierungsmodus — Geben Sie an, ob Ihr Index aktualisiert werdenAmazon Kendra soll, indem alle Dokumente oder nur neue, geänderte und gelöschte Dokumente synchronisiert werden. Sie können wählen zwischen:

    • FORCED_FULL_CRAWLum bei jeder Synchronisierung Ihrer Datenquelle mit Ihrem Index den gesamten Inhalt neu zu crawlen und vorhandene Inhalte zu ersetzen

    • FULL_CRAWLJedes Mal, wenn Ihre Datenquelle mit Ihrem Index synchronisiert wird, nur neue, geänderte und gelöschte Inhalte inkrementell crawlen

    Wichtig

    Da es keine API zum Aktualisieren dauerhaft gelöschter Gmail-Nachrichten gibt, gibt es eine Synchronisierung vonFULL_CRAWL/Neuer, geänderter oder gelöschter Inhalte:

    • Entfernt Nachrichten, die dauerhaft aus Gmail gelöscht wurden, nicht aus IhremAmazon Kendra Index

    • Änderungen in den E-Mail-Labels von Gmail werden nicht synchronisiert

    Um Ihre Änderungen an der Bezeichnung Ihrer Gmail-Datenquelle und dauerhaft gelöschte E-Mail-Nachrichten mit IhremAmazon Kendra Index zu synchronisieren, müssen Sie in regelmäßigen Abständen vollständige Crawls durchführen.

  • Ein- und Ausschlussfilter — Sie können angeben, ob Anlagen einbezogen werden sollen. Sie können auch reguläre Ausdrucksmuster angeben, um bestimmte Nachrichten und Anlagen ein- oder auszuschließen.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre Gmail-Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

    Anmerkung

    Amazon KendraDer Gmail-Datenquellenconnector unterstützt das Erstellen benutzerdefinierter Indexfelder aufgrund von API-Einschränkungen nicht.
