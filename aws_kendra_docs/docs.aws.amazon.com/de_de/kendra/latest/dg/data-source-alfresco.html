Im Freien

Alfresco OnPrem ist ein Content Management Service, der Kunden hilft, ihre Inhalte zu speichern und zu verwalten. Sie könnenAmazon Kendra es verwenden, um Ihre Alfresco-Dokumentbibliothek, Ihr Wiki und Ihren Blog zu indexieren.

Sie können überAmazon Kendra die Amazon KendraKonsole und die AlfrescoConfigurationAPI eine Verbindung zu Ihrer Alfresco-Datenquelle herstellen.

Anmerkung

Der Alfresco Data Source Connector befindet sich derzeit im Vorschaumodus. Die Standardauthentifizierung wird derzeit unterstützt. Wenn Sie den Alfresco Connector in der Produktion verwenden möchten, wenden Sie sich an den Support. Alfresco PaaS wird derzeit nicht unterstützt.

Informationen zur Fehlerbehebung bei IhremAmazon Kendra Alfresco Data Source Connector finden Sie unterFehlerbehebung bei Datenquellen.

Unterstützte Funktionen

Amazon Kendra Der Alfresco Data Source Connector unterstützt die folgenden Funktionen:

  • Feldzuordnungen

  • Filterung des Benutzerkontextes

  • Einschluss-/Ausschlussfilter

  • Virtual Private Cloud (VPC)

Voraussetzungen

Bevor Sie Ihre Alfresco-DatenquelleAmazon Kendra zur Indexierung verwenden können, nehmen Sie diese Änderungen in Ihrem Alfresco und IhrenAWS Konten vor.

In Alfresco müssen Sie über Folgendes verfügen:

  • Ich habe Ihre Alfresco-Authentifizierungsdaten notiert, die einen Admin-Benutzernamen und ein Passwort enthalten.

  • Ein Benutzer wurde zurALFRESCO_ADMINISTRATORS Gruppe hinzugefügt.

  • Ihre Alfresco-Site-URL und Ihre Alfresco-Site-ID wurden kopiert.

  • Geprüft, dass jedes Dokument in Alfresco und in anderen Datenquellen, die Sie für denselben Index verwenden möchten, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

In IhremAWS Konto müssen Sie über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Hat Ihre Alfresco-Authentifizierungsdaten in einemAWSSecrets Manager Secret gespeichert und, falls Sie die API verwenden, den ARN des Geheims notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn Sie keine bestehendeIAM Rolle oder kein Geheimnis haben, können Sie die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn Sie Ihre Alfresco-Datenquelle damit verbindenAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Verbindungsanweisungen

Um eine VerbindungAmazon Kendra zu Ihrer Alfresco-Datenquelle herzustellen, müssen Sie die erforderlichen Details Ihrer Alfresco-Datenquelle angeben, damit diese auf Ihre Daten zugreifenAmazon Kendra kann. Falls Sie Alfresco noch nicht fürAmazon Kendra see konfiguriert habenVoraussetzungen.

Console

Um eine VerbindungAmazon Kendra zu Alfresco herzustellen

  1. Melden Sie sich bei derAWS Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich Indizes und dann aus der Liste der Indizes den Index aus.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wählen Sie auf der Seite Datenquelle hinzufügen die Option Alfresco Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Datenquellendetails die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags für Neues Tag hinzufügen — Tags können Sie Ressourcen suchen und filtern oder IhreAWS -Kosten verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. Alfresco-Site-URL —Geben Sie Ihre Alfresco-Site-URL ein. Zum Beispiel https://hostname:8080.

    2. Standort des SSL-Zertifikats —Geben Sie denAmazon S3 Pfad ein.

    3. Site-ID — Ihre Alfresco Site-ID.

    4. AWSSecrets Managersecret —Wählen Sie ein vorhandenes Geheimnis oder erstellen Sie ein neuesSecrets Manager Geheimnis, um Ihre Alfresco-Authentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-Alfresco-' wird automatisch zu Ihrem geheimen Namen hinzugefügt.

        2. Für Benutzername und Passwort — Geben Sie Ihren Alfresco-Benutzernamen und Ihr Passwort ein.

      2. Wählen Sie Speichern.

    5. Virtual Private Cloud (VPC) — Sie können eine VPC verwenden. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    6. IAM-Rolle — Wählen Sie eine vorhandene IAM-Rolle aus oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und -Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    7. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Inhalt zum Crawlen auswählen — Die Alfresco-Entitäten oder Inhaltstypen, die Sie crawlen möchten.

    2. Regex-Muster — Reguläre Ausdrucksmuster zum Ein- oder Ausschließen bestimmter Dateien. Sie können bis zu 100 Muster hinzufügen.

    3. Wählen Sie im Synchronisierungslaufzeitplan für Häufigkeit aus, wie oft mit Ihrer Datenquelle synchronisiertAmazon Kendra werden soll.

    4. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen die folgenden Informationen ein:

    1. Für Dokumentbibliothek, Wiki und Blog — Wählen Sie eines derAmazon Kendra generierten Standard-Datenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können Ihre Informationen auch von dieser Seite aus bearbeiten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

Um eine VerbindungAmazon Kendra zu Alfresco herzustellen

Sie müssen mithilfe der AlfrescoConfigurationAPI Folgendes angeben:

  • Geheimer Amazon Resource Name (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr Alfresco-Konto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "username": "user name",
    "password": "password"
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle —Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager Secret und den Aufruf der erforderlichen öffentlichen APIs für den Alfresco-Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für Alfresco-Datenquellen.

Sie können auch die folgenden optionalen-Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie dies imVpcConfiguration Rahmen der Datenquellenkonfiguration an. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

  • Ein- und Ausschlussfilter — Geben Sie an, ob bestimmte Dateien ein- oder ausgeschlossen werden sollen.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wählen Sie diese Option, um Ihre Alfresco-Datenquellenfelder IhrenAmazon Kendra Indexfeldern zuzuordnen. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

  • BenutzerkontextfilterungAmazon Kendra — durchsucht standardmäßig die Access Control List (ACL) für Ihre Datenquelle. Die ACL-Informationen werden verwendet, um Suchergebnisse auf der Grundlage des Zugriffs des Benutzers oder seiner Gruppe auf Dokumente zu filtern. Weitere Informationen finden Sie unter Benutzerkontextfilterung für Alfresco-Datenquellen.
