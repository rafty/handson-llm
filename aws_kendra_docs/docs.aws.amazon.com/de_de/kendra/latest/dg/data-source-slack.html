Slack

Slack ist eine Kommunikations-App für Unternehmen, mit der Benutzer Nachrichten und Anhänge über verschiedene öffentliche und private Kanäle senden können. Du kannstAmazon Kendra es verwenden, um deine öffentlichen und privaten Slack-Channels, Bot- und Archivnachrichten, Dateien und Anhänge, Direkt- und Gruppennachrichten zu indexieren. Sie können auch bestimmte Inhalte zum Filtern auswählen.

Du kannst überAmazon Kendra die Amazon KendraKonsole und die SlackConfigurationAPI eine Verbindung zu deiner Slack-Datenquelle herstellen.

Informationen zur Fehlerbehebung bei deinemAmazon Kendra Slack-Datenquellen-Connector findest du unterFehlerbehebung bei Datenquellen.

Unterstützte Funktionen

Amazon KendraDer Slack-Daten-Quell-Connector unterstützt die folgenden Funktionen:

  • Änderungsprotokoll

  • Feldzuordnungen

  • Filterung des Benutzerkontextes

  • Einschluss-/Ausschlussfilter

  • Virtual Private Cloud (VPC)

Voraussetzungen

Bevor du deine Slack-DatenquelleAmazon Kendra zur Indexierung verwenden kannst, nimm diese Änderungen in deinem Slack und deinenAWS Accounts vor.

In Slack musst du über Folgendes verfügen:

  • Hat ein Slack-Bot-Benutzer-OAuth-Token oder ein Slack-Benutzer-OAuth-Token erstellt. Du kannst eines der beiden Token wählen, um eine VerbindungAmazon Kendra zu deiner Slack-Datenquelle herzustellen. Weitere Informationen finden Sie in der Slack-Dokumentation zu Zugriffstoken.

    Anmerkung

    Wenn du das Bot-Token als Teil deiner Slack-Anmeldeinformationen verwendest, kannst du Direktnachrichten und Gruppennachrichten nicht indexieren und du musst das Bot-Token dem Channel hinzufügen, den du indexieren möchtest.

  • Ich habe deine Slack-Workspace-Team-ID aus der URL deiner Slack-Workspace-Hauptseite notiert. Zum Beispiel https://app.slack.com/client/T0123456789/... wobei T0123456789 die Team-ID ist.

  • Die folgenden [Oauth scopes/ read] -Berechtigungen wurden hinzugefügt:

    • Kanäle:Geschichte

    • Kanäle:lesen

    • Gruppen:Geschichte

    • Gruppen:lesen

    • im:history

    • mpim: geschichte

    • mpim:lesen

    • team:lesen

    • user.profil:read

    • Benutzer:lesen

    • Emoji: lesen

    • Dateien:lesen

    • Benutzergruppen:lesen

  • Hat geprüft, dass jedes Dokument in Slack und in anderen Datenquellen, die du für denselben Index verwenden möchtest, einzigartig ist. Jede Datenquelle, die Sie für einen Index verwenden möchten, darf nicht dasselbe Dokument in allen Datenquellen enthalten. Dokument-IDs gelten für einen Index global und müssen pro Index eindeutig sein.

Stellen Sie in IhremAWS Konto über Folgendes verfügen:

  • Hat einenAmazon Kendra Index erstellt und, falls Sie die API verwenden, die Index-ID notiert.

  • Hat eineIAM Rolle für Ihre Datenquelle erstellt und, falls Sie die API verwenden, den ARN derIAM Rolle notiert.

  • Hat deine Slack-Authentifizierungsdaten in einemAWSSecrets Manager Secret gespeichert und, falls du die API verwendest, den ARN des Geheimnisses notiert.

    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

Wenn du keine bestehendeIAM Rolle oder kein Geheimnis hast, kannst du die Konsole verwenden, um eine neueIAM Rolle und ein neuesSecrets Manager Geheimnis zu erstellen, wenn du deine Slack-Datenquelle damit verbindestAmazon Kendra. Wenn Sie die API verwenden, müssen Sie den ARN einer vorhandenenIAM Rolle und einesSecrets Manager Secrets sowie eine Index-ID angeben.

Anweisungen zum Anschluss

Um eine VerbindungAmazon Kendra zu deiner Slack-Datenquelle herzustellen, musst du die notwendigen Details deiner Slack-Datenquelle angeben, damit diese auf deine Daten zugreifenAmazon Kendra kann. Falls du Slack noch nicht konfiguriert hast,Amazon Kendra sieheVoraussetzungen.

Console

Um eine VerbindungAmazon Kendra zu Slack herzustellen

  1. Melden Sie sich bei derAWS -Managementkonsole an und öffnen Sie die Amazon Kendra-Konsole.

  2. Wählen Sie im linken Navigationsbereich Indizes und dann in der Liste der Indizes den Index aus, den Sie verwenden möchten.

    Anmerkung

    Sie können Ihre Einstellungen für die Benutzerzugriffskontrolle unter Indexeinstellungen konfigurieren oder bearbeiten.

  3. Wählen Sie auf der Seite Erste Schritte die Option Datenquelle hinzufügen aus.

  4. Wähle auf der Seite Datenquelle hinzufügen die Option Slack Connector und dann Datenquelle hinzufügen aus.

  5. Geben Sie auf der Seite Datenquellendetails die folgenden Informationen ein:

    1. Geben Sie unter Name und Beschreibung für Datenquellenname einen Namen für Ihre Datenquelle ein. Sie können Bindestriche, aber keine Leerzeichen verwenden.

    2. (Optional) Beschreibung — Geben Sie eine optionale Beschreibung für Ihre Datenquelle ein.

    3. Unter Sprache für Standardsprache — Eine Sprache, um Ihre Dokumente für den Index zu filtern. Sofern Sie nichts anderes angeben, ist die Standardsprache Englisch. Die in den Metadaten angegebene Sprache hat Vorrang vor der ausgewählten Sprache.

    4. Unter Tags für Neues Tag hinzufügen —Tags können Sie Ressourcen suchen und filtern oder IhreAWS Kosten verfolgen.

    5. Wählen Sie Next (Weiter).

  6. Geben Sie auf der Seite Zugriff und Sicherheit definieren die folgenden Informationen ein:

    1. Team-ID des Slack-Workspace — Die Team-ID deines Slack-Workspace.

    2. AWSSecrets Managersecret — Wähle ein vorhandenes Geheimnis oder erstelle ein neuesSecrets Manager Geheimnis, um deine Slack-Authentifizierungsdaten zu speichern. Wenn Sie ein neues Geheimnis erstellen möchten, öffnet sich einAWSSecrets Manager geheimes Fenster.

      1. Geben Sie die folgenden Informationen in das Fenster Create anAWSSecrets Manager secret ein:

        1. Geheimer Name — Ein Name für Ihr Geheimnis. Das Präfix 'AmazonKendra-Slack-' wird automatisch zu deinem geheimen Namen hinzugefügt.

        2. Für Slack-Token — Gib die Werte der Authentifizierungsdaten ein, die du in deinem Slack-Konto erstellt hast.

      2. Wählen Sie Speichern.

    3. Virtual Private Cloud (VPC) — Sie können wählen, ob Sie eine VPC verwenden möchten. In diesem Fall müssen Sie Subnetze und VPC-Sicherheitsgruppen hinzufügen.

    4. IAM-Rolle —Wählen Sie eine vorhandene IAM-Rolle oder erstellen Sie eine neue IAM-Rolle, um auf Ihre Repository-Anmeldeinformationen und Indexinhalte zuzugreifen.

      Anmerkung

      IAM-Rollen, die für Indizes verwendet werden, können nicht für Datenquellen verwendet werden. Wenn Sie sich nicht sicher sind, ob eine vorhandene Rolle für einen Index oder häufig gestellte Fragen verwendet wird, wählen Sie Neue Rolle erstellen, um Fehler zu vermeiden.

    5. Wählen Sie Next (Weiter).

  7. Geben Sie auf der Seite Synchronisierungseinstellungen die folgenden Informationen ein:

    1. Wählen Sie die Art des zu crawlenden Inhalts aus — Die Slack-Entitäten oder Inhaltstypen, die Sie crawlen möchten.

    2. Änderungsprotokoll — Wählen Sie diese Option aus, um Ihren Index zu aktualisieren, anstatt alle Ihre Dateien zu synchronisieren.

    3. Regex-Muster — Reguläre Ausdrucksmuster zum Ein- oder Ausschließen bestimmter Dateien. Sie können bis zu 100 Muster hinzufügen.

    4. Startdatum des Crawls auswählen — Ab welchem DatumAmazon Kendra werden Ihre Daten gecrawlt?

    5. Geben Sie im Synchronisierungslaufzeitplan für Häufigkeit an, wie oftAmazon Kendra die Synchronisierung mit Ihrer Datenquelle erfolgt.

    6. Wählen Sie Next (Weiter).

  8. Geben Sie auf der Seite Feldzuordnungen die folgenden Informationen ein:

    1. Für Slack-Feldzuordnungen — Wählen Sie eines derAmazon Kendra generierten Standard-Datenquellenfelder aus, die Sie Ihrem Index zuordnen möchten.

    2. Feld hinzufügen — Um benutzerdefinierte Datenquellenfelder hinzuzufügen, um einen Indexfeldnamen für die Zuordnung und den Felddatentyp zu erstellen.

    3. Wählen Sie Next (Weiter).

  9. Überprüfen Sie auf der Seite Überprüfen und erstellen, ob die von Ihnen eingegebenen Informationen korrekt sind, und wählen Sie dann Datenquelle hinzufügen aus. Sie können auch festlegen, dass Sie Ihre Informationen von dieser Seite aus bearbeiten möchten. Ihre Datenquelle wird auf der Seite Datenquellen angezeigt, sobald sie erfolgreich hinzugefügt wurde.

API

Um eine VerbindungAmazon Kendra zu Slack herzustellen

Sie müssen mithilfe der SlackConfigurationAPI Folgendes angeben:

  • Team-ID des Slack-Workspace-Teams — Die Slack-Team-ID, die du von der URL deiner Slack-Hauptseite kopiert hast.

  • Liste der zu indexierenden Entitäten — Gibt an, ob Ihre öffentlichen und privaten Kanäle sowie Ihre Gruppen- und Direktnachrichten indexiert werdenAmazon Kendra sollen.

  • Crawling-Datum — Das Datum, an dem mit dem Crawlen deiner Daten aus deinem Slack Workspace-Team begonnen werden soll. Das Datum muss diesem Format folgen: yyyy-mm-dd.

  • Geheimer Amazon-Ressourcenname (ARN) — Geben Sie den Amazon-Ressourcennamen (ARN) einesSecrets Manager geheimen Codes an, der die Authentifizierungsdaten für Ihr Slack-Konto enthält. Das Geheimnis wird in einer JSON-Struktur mit den folgenden Schlüsseln gespeichert:

    {
    "slackToken": "token"
}
    Anmerkung

    Stellen Sie sicher, dass Sie Ihre Anmeldeinformationen und Ihr Secret regelmäßig aktualisieren oder wechseln. Geben Sie zu Ihrer eigenen Sicherheit nur die erforderliche Zugriffsebene an. Die Wiederverwendung von Anmeldeinformationen und Geheimnissen in allen Datenquellen und den Connectorversionen 1.0 und 2.0 (sofern zutreffend) wird nicht empfohlen.

  • IAM-Rolle — Geben Sie an,RoleArn wann Sie aufrufen,CreateDataSource um einerIAM Rolle Berechtigungen für den Zugriff auf IhrSecrets Manager Secret und zum Aufrufen der erforderlichen öffentlichen APIs für den Slack-Connector und zu gewährenAmazon Kendra. Weitere Informationen finden Sie unter IAM-Rollen für Slack-Datenquellen.

Sie können auch die folgenden optionalen-Funktionen hinzufügen:

  • Virtual Private Cloud (VPC) — Geben Sie dies imVpcConfiguration Rahmen der Datenquellenkonfiguration an. Siehe KonfigurationAmazon Kendra für die Verwendung einer VPC.

  • Änderungsprotokoll — Ob der Change-Log-Mechanismus für Datenquellen von Slack verwendet werdenAmazon Kendra soll, um zu bestimmen, ob ein Dokument im Index hinzugefügt, aktualisiert oder gelöscht werden muss.

    Anmerkung

    Verwenden Sie das Änderungsprotokoll, wenn Sie nicht alle Dokumente scannen möchtenAmazon Kendra. Wenn dein Änderungsprotokoll groß ist, dauert es möglicherweiseAmazon Kendra weniger Zeit, die Dokumente in der Slack-Datenquelle zu scannen, als das Änderungsprotokoll zu verarbeiten. Wenn du deine Slack-Datenquelle zum ersten Mal mit deinem Index synchronisierst, werden alle Dokumente gescannt.

  • Ein- und Ausschlussfilter — Geben Sie an, ob bestimmte öffentliche und private Kanäle, Gruppen- und Privatnachrichten sowie Bot- und archivierte Nachrichten ein- oder ausgeschlossen werden sollen. Wenn du ein Bot-Token als Teil deiner Slack-Authentifizierungsdaten verwendest, musst du das Bot-Token dem Channel hinzufügen, den du indexieren möchtest. Sie können Direktnachrichten und Gruppennachrichten nicht mit einem Bot-Token indexieren.

    Anmerkung

    Die meisten Datenquellen verwenden reguläre Ausdrucksmuster, bei denen es sich um Einschluss- oder Ausschlussmuster handelt, die als Filter bezeichnet werden. Wenn Sie einen Inklusionsfilter angeben, werden nur Inhalte indexiert, die dem Inklusionsfilter entsprechen. Jedes Dokument, das nicht dem Inklusionsfilter entspricht, wird nicht indexiert. Wenn Sie einen Inklusions- und Ausschlussfilter angeben, werden Dokumente, die dem Ausschlussfilter entsprechen, nicht indexiert, auch wenn sie dem Einschlussfilter entsprechen.

  • Feldzuordnungen — Wähle aus, ob du deine Slack-Datenquellenfelder deinenAmazon Kendra Indexfeldern zuordnen möchtest. Weitere Informationen finden Sie unter Zuweisen von Datenquellenfeldern.

  • BenutzerkontextfilterungAmazon Kendra — durchsucht standardmäßig die Access Control List (ACL) für Ihre Datenquelle. Die ACL-Informationen werden verwendet, um Suchergebnisse auf der Grundlage des Zugriffs des Benutzers oder seiner Gruppe auf Dokumente zu filtern. Weitere Informationen findest du unter Benutzerkontextfilterung für Slack-Datenquellen.
