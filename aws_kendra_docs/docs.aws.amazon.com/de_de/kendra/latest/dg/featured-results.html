Ausgewählte Suchergebnisse

Sie können bestimmte Dokumente in den Suchergebnissen anzeigen, wenn Ihre Benutzer bestimmte Abfragen stellen. Dies trägt dazu bei, die Ergebnisse für Ihre Benutzer sichtbarer und prominenter zu machen. Die ausgewählten Ergebnisse werden von der üblichen Ergebnisliste getrennt und oben auf der Suchseite angezeigt. Sie können damit experimentieren, verschiedene Dokumente für verschiedene Abfragen anzuzeigen, oder sicherstellen, dass bestimmte Dokumente die Sichtbarkeit erhalten, die sie verdienen.

Sie ordnen bestimmte Abfragen bestimmten Dokumenten zu, um sie in den Ergebnissen zu berücksichtigen. Wenn eine Abfrage eine exakte Übereinstimmung enthält, werden ein oder mehrere spezifische Dokumente in den Suchergebnissen angezeigt.

Sie können beispielsweise festlegen, dass, wenn Ihre Benutzer die Abfrage „Neue Produkte 2023“ stellen, die Dokumente mit den Titeln „Was ist neu“ und „Demnächst“ auswählen, die oben auf der Suchergebnisseite erscheinen sollen. Dies trägt dazu bei, dass diese Dokumente zu neuen Produkten die Sichtbarkeit erhalten, die sie verdienen.

Amazon Kendradupliziert keine Suchergebnisse, wenn bereits ein Ergebnis ausgewählt wurde, um es oben auf der Suchergebnisseite anzuzeigen. Ein hervorgehobenes Ergebnis wird nicht erneut als erstes Ergebnis eingestuft, wenn es bereits vor allen anderen Ergebnissen steht.

Um bestimmte Ergebnisse anzuzeigen, müssen Sie eine exakte Übereinstimmung mit einer Volltextabfrage angeben, nicht eine teilweise Übereinstimmung einer Abfrage, indem Sie ein in einer Abfrage enthaltenes Schlüsselwort oder eine Phrase verwenden. Wenn Sie beispielsweise nur die Abfrage 'Kendra' in einer Feature-Ergebnismenge angeben, werden Abfragen wie „Wie ordnet Kendra Ergebnisse semantisch ein?“ wird die vorgestellten Ergebnisse nicht rendern. Die ausgewählten Ergebnisse sind für spezifische Abfragen konzipiert und nicht für Abfragen, deren Umfang zu weit gefasst ist. Amazon Kendraverarbeitet Abfragen vom Typ Keywords auf natürliche Weise, um die nützlichsten Dokumente in den Suchergebnissen zu ordnen, und vermeidet so eine übermäßige Darstellung von Ergebnissen, die auf einfachen Schlüsselwörtern basieren.

Wenn es bestimmte Abfragen gibt, die Ihre Benutzer häufig verwenden, können Sie diese Abfragen für ausgewählte Ergebnisse angeben. Wenn Sie sich beispielsweise Ihre häufigsten Suchanfragen mit Amazon KendraAnalytics ansehen und diese spezifischen Abfragen finden, z. B. „Wie ordnet Kendra Ergebnisse semantisch ein?“ und 'Kendra Semantic Search' werden häufig verwendet, dann könnten diese Abfragen nützlich sein, um sie für das Dokument mit dem Titel „Amazon Kendrasearch 101“ zu spezifizieren.

Amazon Kendrabehandelt Abfragen nach ausgewählten Ergebnissen ohne Berücksichtigung von Groß- und Kleinschreibung. Amazon Kendrawandelt eine Abfrage in Kleinbuchstaben um und ersetzt nachfolgende Leerzeichen durch ein einzelnes Leerzeichen. Amazon Kendraentspricht allen anderen Zeichen, so wie sie sind, wenn Sie Ihre Abfragen für ausgewählte Ergebnisse angeben.

Sie erstellen eine Reihe von Feature-Ergebnissen, die Sie mithilfe der CreateFeaturedResultsSetAPI bestimmten Abfragen zuordnen. Wenn Sie die Konsole verwenden, wählen Sie Ihren Index und dann im Navigationsmenü die Option Ausgewählte Ergebnisse aus, um einen Satz ausgewählter Ergebnisse zu erstellen. Sie können bis zu 50 Gruppen von ausgewählten Ergebnissen pro Index, bis zu vier Dokumente, die pro Satz vorgestellt werden sollen, und bis zu 49 Abfragetexte pro ausgewähltem Ergebnissatz erstellen. Sie können eine Erhöhung dieser Limits beantragen, indem Sie sich an den Support wenden.

Sie können dasselbe Dokument für mehrere Gruppen von ausgewählten Ergebnissen auswählen. Sie dürfen jedoch nicht denselben Exact-Match-Abfragetext für mehrere Sätze verwenden. Die Abfragen, die Sie für ausgewählte Ergebnisse angeben, müssen pro Feature-Ergebnissatz für jeden Index eindeutig sein.

Sie können die Reihenfolge der Dokumente festlegen, wenn Sie bis zu vier ausgewählte Dokumente auswählen. Wenn Sie die API verwenden, entspricht die Reihenfolge, in der Sie die ausgewählten Dokumente auflisten, der Reihenfolge, in der sie in den ausgewählten Ergebnissen angezeigt werden. Wenn Sie die Konsole verwenden, können Sie die Reihenfolge der Dokumente einfach per Drag-and-Drop verschieben, wenn Sie Dokumente auswählen, die in den Ergebnissen angezeigt werden sollen.

Die Zugriffskontrolle, bei der bestimmte Benutzer und Gruppen Zugriff auf bestimmte Dokumente haben und andere nicht, wird bei der Konfiguration der ausgewählten Ergebnisse weiterhin berücksichtigt. Das gilt auch für die Filterung des Benutzerkontextes. Beispielsweise gehört Benutzer A zur Unternehmensgruppe „Praktikanten“, die nicht auf Dokumente über Unternehmensgeheimnisse zugreifen sollte. Wenn Benutzer A eine Abfrage eingibt, die ein unternehmensgeheimes Dokument enthält, sieht Benutzer A dieses Dokument nicht in seinen Ergebnissen. Das gilt auch für alle anderen Ergebnisse auf der Suchergebnisseite. Sie können Tags auch verwenden, um den Zugriff auf einen Feature-Ergebnissatz zu steuern. Dabei handelt es sich um eineAmazon Kendra Ressource, für die Sie den Zugriff kontrollieren.

Im Folgenden finden Sie ein Beispiel für die Erstellung einer Reihe von Feature-Ergebnissen mit den Abfragen „Neue Produkte 2023", „Neue Produkte verfügbar“, die den Dokumenten mit den Titeln „What's new“ (doc-id-1) und „Coming soon“ (doc-id-2) zugeordnet sind.

CLI
aws kendra create-featured-results-set \
 --featured-results-set-name 'New product docs to feature' \
 --description "Featuring What's new and Coming soon docs" \
 --index-id index-id \
 --query-texts 'new products 2023' 'new products available' \
 --featured-documents '{"Id":"doc-id-1", "Id":"doc-id-2"}'
Python
import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Create a featured results set.")

# Provide a name for the featured results set
featured_results_name = "New product docs to feature"
# Provide an optional decription for the featured results set
description = "Featuring What's new and Coming soon docs"
# Provide the index ID for the featured results set
index = index-id
# Provide a list of query texts for the featured results set
queries = ['new products 2023', 'new products available']
# Provide a list of document IDs for the featured results set
featured_doc_ids = ['doc-id-1', 'doc-id-2']

try:
    featured_results_set_response = kendra.create_featured_results_set(
        FeaturedResultsSetName = featured_results_name,
        Decription = description,
        Index = index,
        QueryTexts = queries,
        FeaturedDocuments = featured_doc_ids
    )

    pprint.pprint(featured_results_set_response)

    featured_results_set_id = featured_results_set_response["FeaturedResultsSetId"]

    while True:
        # Get the details of the featured results set, such as the status
        featured_results_set_description = kendra.describe_featured_results_set(
            Id = featured_results_set_id
        )
        status = featured_results_set_description["Status"]
        print(" Featured results set status: "+status)
            
except  ClientError as e:
        print("%s" % e)

print("Program ends.")
