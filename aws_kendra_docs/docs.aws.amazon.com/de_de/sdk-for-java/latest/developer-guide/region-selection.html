AWSAuswahl von Regionen

Regionen ermöglichen Ihnen den ZugriffAWSDienstleistungen, die sich physisch in einem bestimmten geografischen Gebiet befinden. Dies ist nicht nur für die Redundanz nützlich, sondern sorgt auch dafür, dass Ihre Daten und Anwendungen in der Nähe Ihres Standorts sowie des Standorts Ihrer Benutzer ausgeführt werden.

In :AWS SDK for Java2.x wurden alle unterschiedlichen regionalen Klassen ab Version 1.x in eine Region-Klasse zusammengefasst. Sie können diese Klasse für alle regionenbezogenen Aktionen verwenden, wie z. B. das Abrufen von Metadaten zu einer Region oder das Prüfen, ob ein Service in einer Region verfügbar ist.

Auswahl einer Region

Sie können den Namen einer Region angeben und das SDK wählt automatisch einen passenden Endpunkt für Sie aus.

Um explizit eine Region festzulegen, empfehlen wir, dass Sie die in der Klasse Region definierten Konstanten verwenden. Dabei handelt es sich um eine Aufzählung aller öffentlich verfügbaren Regionen. Mit dem folgenden Code können Sie einen Client mit einer Region aus der Klasse erstellen.

Ec2Client ec2 = Ec2Client.builder()
          .region(Region.US_WEST_2)
          .build();

Wenn die Region, die Sie verwenden möchten, keine der Konstanten in der Klasse Region ist, können Sie mit der of-Methode eine neue Region erstellen. Diese Funktion ermöglicht es Ihnen, auf neue Regionen zuzugreifen, ohne das SDK zu aktualisieren.

Region newRegion = Region.of("us-east-42");
Ec2Client ec2 = Ec2Client.builder()
          .region(newRegion)
          .build();
Anmerkung

Nachdem Sie einen Client mit dem Generator erstellt haben, ist er unveränderlich und die Region kann nicht mehr geändert werden. Wenn Sie mit mehreren arbeitenAWS-RegionenFür den gleichen Service sollten Sie mehrere Clients erstellen, und zwar einen pro Region.

Auswahl eines bestimmten Endpunkts

EACHAWSClient kann so konfiguriert werden, dass er einespezifischer Endpunktinnerhalb einer Region, indem Sie dieendpointOverride-Methode

Zum Beispiel, um denAmazon EC2Mit dem folgenden Code können Sie die Region Europa (Irland) verwenden.

Ec2Client ec2 = Ec2Client.builder()
               .region(Region.EU_WEST_1)
               .endpointOverride(URI.create("https://ec2.eu-west-1.amazonaws.com"))
               .build();

Siehe .-Regionen und -Endpunktefür die aktuelle Liste der Regionen und ihrer entsprechenden Endpunkte für alleAWS-Services.

Bestimmen Sie die Region automatisch aus der Umgebung

Bei der Ausführung auf Amazon EC2 oder AWS Lambda sollten Sie die Clients so konfigurieren, dass sie die gleiche Region verwenden, auf der auch Ihr Code ausgeführt wird. So wird der Code von der Umgebung abgekoppelt, in der er läuft, wodurch die Bereitstellung Ihrer Anwendung in mehreren Regionen einfacher wird. Dies wiederum sorgt für weniger Latenz und mehr Redundanz.

Rufen Sie die create-Methode des Client-Generators auf, um die Region aus der Umgebung mithilfe der Standard-Anbieterkette für Anmeldeinformationen/Regionen zu bestimmen.

Ec2Client ec2 = Ec2Client.create();

Wenn Sie nicht explizit eine Region mit der region-Methode festlegen, nutzt das SDK die Standard-Anbieterkette für Regionen und versucht, die zu nutzende Region zu ermitteln.

Standardregions-Anbieterkette

Folgende Regeln gelten für das Nachschlagen der Region:

  1. Etwaige explizite, mit region festgelegte Regionen direkt im Generator haben Vorrang vor allen anderen.

  2. Die Umgebungsvariable AWS_REGION wird geprüft. Wenn sie festgelegt ist, wird die zugehörige Region zur Konfiguration des Clients verwendet.

    Anmerkung

    Diese Umgebungsvariable wird vom Lambda-Container gesetzt.

  3. Das SDK überprüft dasAWSfreigegebene Konfigurationsdatei (normalerweise)~/.aws/config) enthalten. Ist die Eigenschaft region vorhanden, wird sie vom SDK verwendet.

    • Die Umgebungsvariable AWS_CONFIG_FILE kann verwendet werden, um den Speicherort der gemeinsam genutzten Konfigurationsdatei anzupassen.

    • DieAWS_PROFILEUmgebungsvariable oderaws.profileDie Systemeigenschaft kann zum Ermitteln des Profils genutzt werden, das vom SDK geladen wird.

  4. Das SDK versucht, den Metadaten-Service der Amazon EC2-Instance zu verwenden, um die Region der aktuell ausgeführten Amazon EC2-Instance zu ermitteln.

  5. Hat das SDK zu diesem Zeitpunkt immer noch keine Region gefunden, schlägt die Erstellung des Clients mit einer Ausnahme fehl.

Bei der EntwicklungAWSBei Anwendungen besteht ein gängiger Ansatz infreigegebene Konfigurationsdatei(beschrieben inAbruhenfolge für Anmeldeinformationen) um die Region für die lokale Entwicklung festzulegen und sich bei der Bestimmung der Region auf die standardmäßige Anbieterkette zu verlassenAWS-Infrastruktur Dies vereinfacht die Client-Erstellung stark und sorgt dafür, dass Ihre Anwendung portabel bleibt.

Auf Serviceverfügbarkeit in einer Region prüfen

Um zu sehen, ob ein bestimmtesAWS-Serviceist in einer Region verfügbar, benutzen Sie dieserviceMetadataundregion-Methode für den Service, den Sie überprüfen möchten.

DynamoDbClient.serviceMetadata().regions().forEach(System.out::println);
