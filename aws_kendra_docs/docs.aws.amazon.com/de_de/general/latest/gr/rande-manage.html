Verwalten von AWS-Regionen

Eine AWS-Region ist eine Sammlung von AWS-Ressourcen in einem geografischen Bereich. Jede AWS-Region ist isoliert und unabhängig von den anderen Regionen. Regionen bieten Fehlertoleranz, Stabilität und Ausfallsicherheit und können auch die Latenz verkürzen. Sie ermöglichen Ihnen das Erstellen redundanter Ressourcen, die verfügbar bleiben und von einem regionalen Ausfall nicht betroffen sind.

Die Ressourcen, die Sie in einer Region erstellen, existieren nicht in einer anderen Region, es sei denn, Sie verwenden explizit eine Replikationsfunktion, die von einem AWS-Service angeboten wird. Beispielsweise unterstützen Amazon S3 und Amazon EC2 die regionsübergreifende Replikation. Einige Services, z. B. AWS Identity and Access Management (IAM), verfügen über keine regionalen Ressourcen.

Sie können Richtlinienbedingungen verwenden, um den Zugriff auf AWS-Services in einer AWS-Region zu steuern.

Voraussetzungen

Anmelden für eine Region

  1. Melden Sie sich an der AWS Management Console an.

  2. Wählen Sie auf der oberen rechten Ecke der Konsole rechts oben Ihren Kontonamen oder Ihre Kontonummer und dann Konto aus.

  3. Wählen Sie unter AWSRegionen in der Spalte Aktion für die Region die Option Aktivieren aus.

    Tipp

    Wenn in der Statusspalte standardmäßig Aktiviert steht, müssen Sie sich nicht für die Region anmelden.

  4. Überprüfen Sie im Dialogfeld den Informationstext und wählen Sie dann Region aktivieren aus.

  5. Warten Sie, bis die Region zur Verwendung bereit ist.

Um sich von einer Region abzumelden

  1. Melden Sie sich an der AWS Management Console an.

  2. Wählen Sie auf der oberen rechten Ecke der Konsole rechts oben Ihren Kontonamen oder Ihre Kontonummer und dann Konto aus.

  3. Wählen Sie unter AWSRegionen in der Spalte Aktion für die Region die Option Deaktivieren aus.

  4. Überprüfen Sie im Dialogfeld den Informationstext und wählen Sie dann Region deaktivieren.

Beschreiben Ihrer Regionen mithilfe der AWS CLI

Verwenden Sie den Befehl describe-regions, um die für Ihr Konto verfügbaren Regionen zu beschreiben und Ihren Opt-In-Status einzusehen.

aws ec2 describe-regions --all-regions

Wenn Sie sich nicht für eine Region anmelden müssen, umfasst die Ausgabe Folgendes:

"OptInStatus": "opt-in-not-required"

Wenn Sie nicht für die Region angemeldet sind, umfasst die Ausgabe Folgendes:

"OptInStatus": "not-opted-in"

Nachdem Sie sich für eine Region angemeldet haben, umfasst die Ausgabe Folgendes:

"OptInStatus": "opted-in"
