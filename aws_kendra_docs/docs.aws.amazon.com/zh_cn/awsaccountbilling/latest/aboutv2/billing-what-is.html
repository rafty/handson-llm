什么是 AWS Billing?

欢迎阅读《Billing 用户指南》。

AWS Billing 控制台包含支付 AWS 账单与报告 AWS 费用及使用量的功能。如果您是 AWS Organizations 的成员，还可以使用 AWS Billing 控制台管理整合账单。Amazon Web Services 会自动从您在注册 AWS 账户时提供的信用卡中扣费。您可以随时查看或更新信用卡信息，包括更改用于 AWS 扣费的信用卡。您可以在账单控制台的 Payment Methods（付款方式）页面执行此操作。有关 Billing 可用功能的更多详细信息，请参阅 AWS Billing 控制台功能。

如何查找有关 AWS 账单的答案

如对 AWS 账单或收费情况有任何疑问，请立即联系 AWS Support 解决问题。有关如何获取帮助的选项，请参阅 使用 AWS Billing 获取帮助。要了解账单页面内容，请参阅 查看您的月度费用。

AWS Billing 控制台与 AWS Cost Management 控制台功能之间的主要区别

账单控制台可与 AWS 成本管理控制台紧密结合。Billing 控制台包含管理持续付款以及注册到 AWS 账户的付款方式所需的功能。AWS Cost Management 控制台功能有助于优化未来的成本。您可以同时使用这两个控制台以全面管理当前成本和持续成本。要了解有关可帮助优化成本的AWS资源的信息，请参阅《AWS Cost Management 用户指南》。

AWS Billing 控制台功能

利用 AWS Billing 控制台中的功能，您可以执行以下任务：

  • 管理账户：使用 AWS Management Console 和 Billing 控制台管理账户设置。这包括指定默认货币、编辑备用联系人、添加或删除 AWS 区域、更新税务信息以及关闭 AWS 账户。关闭 AWS 账户 之前的 close your account 重点注意事项。例如，在关闭账户之前终止资源，以免为未使用的服务付费。

  • 查看账单：您可以随时使用 AWS Billing 控制台查看历史账单详细信息或当月估计费用。这一部分概括介绍了如何查看账单、下载费用的 PDF 副本、设置月度电子邮件以接收发票。此外还介绍了如何利用其他资，例如 AWS 成本和使用情况报告。

  • 管理付款：您不仅可以查看估计账单，也可以设置付款货币，从而以首选货币来支付 AWS 发票。AWS 会在账单最终确定后将账单转换为首选货币。在此之前，控制台中显示的所有首选货币金额均为估计的美元金额。AWS 提供汇率保证，因此退款使用的汇率与原始交易使用的汇率相同。

    注意

    • AWS Marketplace 发票不适用于此服务，这些发票将以美元进行处理。

    • 此服务仅适用于默认付款方式为 Visa 或 MasterCard 的情况。

    • 汇率每天都会变化。您的发票使用的汇率是发票创建之时的汇率。您可以在 Billing 控制台中查看当前汇率。

    • 货币兑换服务由 Amazon Services LLC 提供。

  • AWS Purchase Order Management：通过一站式集中处理多个采购订单，以自助方式管理 AWS 采购订单。这有利于降低管理费用，并提高整个从采购到付款过程的准确性和效率。使用 Billing 控制台管理采购订单并配置其在发票上的反映方式。在这一部分中，了解如何在控制台中添加、编辑、查看详细信息以及设置采购订单的相关通知。

  • AWS Cost Categories：使用 AWS Cost Categories 将成本和使用量映射到有意义的类别中，从而管理 AWS 成本。这一部分定义了控制台中使用的有关支持的维度、运算、规则类型和状态术语。这一部分还提供了有关如何在 Cost Categories 中创建、编辑、删除和拆分费用的更多信息。

  • 付款配置文件：您可以使用付款配置文件为自动付款分配多种付款方式。如果您收到来自 AWS 服务提供商（“记录卖家”）的发票，则可以使用付款配置文件为每个发票分配唯一的付款方式。在为服务提供商创建支付配置文件后，您的支付配置文件将自动支付您的 AWS 账单。在这一部分，了解如何使用 Billing Console 设置自定义付款配置文件。

  • AWS Organizations 整合账单：使用 AWS Organizations 的整合账单功能来整合多个 AWS 账户的账单。本章概括介绍了整合账单流程、不同 Amazon Web Services India Private Limited 账户的差异以及折扣详情。

相关服务

AWS Billing Conductor

AWS Billing Conductor 是一项自定义计费服务，用于按财务拥有者对账户进行分组，配置计费参数，并按组生成 AWS 成本和使用情况报告。AWSBilling Conductor 是一项完全托管式服务，可以支持 AWS 解决方案提供商和企业客户的记账和退款。

IAM

账单服务和 AWS 成本管理服务与 AWS Identity and Access Management（IAM）紧密集成。您可以将 IAM 与 Billing 结合使用，以确保使用您的账户工作的其他人只拥有完成其工作所需的访问权限。

您也可以使用 IAM 来控制对所有 AWS 资源（而不仅仅是您的账单信息）的访问。在您着手设置 AWS 账户结构之前，首先熟悉 IAM 的主要概念和最佳实践十分重要。

AWS Organizations（整合账单）

您可以使用 AWS 产品和服务来满足从小型初创公司到大型企业的各种规模公司的需求。如果公司规模较大，或成长潜力很大，则可以根据公司的具体架构设置多个 AWS 账户。例如，您可以为整个公司设置一个账户，再为每个员工设置单独的员工账户，或者也可以为整个公司设置一个账户，并在其中为每个员工创建 IAM 用户。您可以为整个公司设置账户、为公司内的每个部门或小组设置账户，以及为每个员工设置账户。

如果您创建多个账户，则可以使用 AWS Organizations 的整合账单功能将所有成员账户合并到一个管理账户下，从而接收包含所有成员账户的统一账单。有关更多信息，请参阅 AWS Organizations 的整合账单。
