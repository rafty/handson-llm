关于启用和禁用的注意事项AWS 区域

  • 您可以使用 IAM 权限来控制对区域的访问

    AWS Identity and Access Management(IAM) 包括四种权限，可让您控制哪些用户可以启用、获取和列出区域。有关更多信息，请参阅AWS Billing and Cost Management用户指南中的B illing and Cost Management 操作政策。

  • 启用区域是免费的

    启用区域是免费的。您只需为在新在新新区域中创建的资源付费。

  • 禁用区域会禁用对区域中资源的访问

    如果将Amazon Elastic Compute Cloud (Amazon Elastic Compute Cloud (Amazon EC2) 实例，则无法访问该区域中资源的访问权限。AWS例如，您不能使用AWS Management Console或任何编程方法来查看或更改禁用区域中任何 EC2 实例的配置。

  • 如果您禁用区域，则继续对活跃资源收费

    如果禁用仍包含AWS资源的区域，则这些资源（如果有）的费用（如果有）将继续按标准费率的标准费率计算。例如，如果禁用包含 Amazon EC2 实例的区域，则即使实例不可访问，您仍然必须为这些实例支付费用。

  • 禁用区域的结果并不总是立即可见

    禁用区域后，服务和控制台可能会暂时可见。禁用区域可能需要几分钟到几个小时才能生效。

  • 默认启用启用启用启用启用启用启用启用启用启用启用启用

    2019 年 3 月 20 日之前引入的区域在默认情况下均处于启用状态，无法禁用。有关更多信息，请参阅AWS 区域中的管理AWS 一般参考。

  • 启用区域需要几分钟到几小时。

    启用一个区域最多可能需要几个小时，具体取决于多个因素，例如您的组织规模。

  • 在给定时间，Organizations 内可以在 20 个区域选择请求处于待处理AWS状态

    管理账户在任何时候都可能有 20 个待处理的组织申请。一个请求等于为一个账户启用或禁用一个特定区域。

  • 一个账户在任何给定时间可以有 6 个正在处理的 region-opt 请求

    一个请求等于为一个账户启用或禁用一个特定区域。

  • 亚马逊 EventBridge 集成

    客户可以在中订阅 region-opt 状态更新通知 EventBridge。将为每次状态更改创建 EventBridge 通知，允许客户自动执行工作流程。

  • 表达式 Region-Opt 状态

    由于启用/禁用选择加入区域的异步性质，区域选择请求有四种潜在状态：

    • ENABLING

    • DISABLING

    • ENABLED

    • DISABLED

    当选择加入或选择退出处于任一DISABLING状态时，您无法取消该订阅ENABLING或选择退出。否则，ConflictException将抛出 a。已完成（启用/禁用）region-opt 请求取决于关键基础AWS服务的配置。尽管状态为，但有些AWS服务可能无法立即使用ENABLED。
