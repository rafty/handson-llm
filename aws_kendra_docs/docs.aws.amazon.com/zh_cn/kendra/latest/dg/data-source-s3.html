Amazon S3

Amazon S3是一种对象存储服务，可将数据以对象形式存储在存储桶中。您可以使用Amazon Kendra索引存储Amazon S3桶中的文档。

警告

Amazon Kendra不使用向Amazon Kendra委托人授予与 S3 存储桶交互的权限的存储桶策略。相反，它使用IAM角色。确保Amazon Kendra不将其作为可信成员包含在您的存储段策略中，以避免在意外向任意主体授予权限时出现任何数据安全问题。但是，您可以添加存储桶策略以在不同的账户中使用Amazon S3存储桶。有关更多信息，请参阅Amazon S3跨账户使用的策略（在 S3IAM 角色选项卡中，在数据源的IAM角色下）。有关 S3 数据源IAM角色的信息，请参阅IAM角色。

要连接到Amazon S3，您可以使用 S3DataSourceConfiguration 或使用，在控制台中指定连接和其他信息TemplateConfiguration。如果您使用 TemplateConfiguration 对象，则可以使用 VPC 连接到您的数据源。您可以指定 VpcConfiguration 何时通话 CreateDataSource。有关更多信息，请参阅配置 VPC。

注意

S3DataSourceConfigurationAPI 不支持配置 VPC。只有TemplateConfiguration API 和控制台支持 VPC 配置。

要Amazon Kendra对 S3 数据源连接器进行故障排除，请参阅解决数据源问题。

先决条件

在使用Amazon Kendra索引 S3 数据源之前，请先在 S3 和AWS账户中进行这些更改。

在 S3 中，确保您具有：

  • 已复制存储Amazon S3桶名称的名称。

    注意

    您的存储段必须与您的Amazon Kendra索引位于同一区域，并且您的索引必须有权访问包含您的文档的存储桶。

  • 已检查每个文档在 S3 以及您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

如果您没有现有IAM角色，则可以在将 S3 数据源连接到时使用控制台创建新IAM角色Amazon Kendra。如果您使用 API，则必须提供现有IAM角色的 ARN 和索引 ID。

TemplateConfiguration API

要连接Amazon Kendra到Amazon S3

您必须使用 TemplateConfigurationAPI 指定数据源架构的 JSON。您必须提供以下信息：

  • BucketName—包含文档的桶的名称。

  • IAM 角色 — 指定您RoleArn何时调用，CreateDataSource以便为IAM角色提供访问您的Secrets Manager密钥和调用 S3 连接器所需的公共 API 的权限，以及Amazon Kendra。有关更多信息，请参阅 S3 数据源的 IAM 角色。

您还可以添加以下可选功能：

  • 包含和排除过滤器-指定是包括还是排除某些文件名、文件类型、文件路径，并使用全局模式（可以将通配符模式扩展为与给定模式匹配的路径名列表的模式）。全局模式的示例包括：

    • /myapp/config/*—配置目录中的所有文件

    • /**/*.png—所有目录中的所有.png 文件

    • /**/*.{png,ico,md}—所有目录中的所有.png、.ico 或.md 文件

    • /myapp/src/**/*.ts—src 目录（及其所有子目录）内的所有.ts 文件

    • **/!(*.module).ts—所有.ts 文件，但不是 .module.ts

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 S3 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • V@@ irtual Private Cloud (VPC)-指定您VpcConfiguration何时CreateDataSource 请参阅配置Amazon Kendra为使用 VPC。

  • 文档元数据配置-添加包含诸如文档访问控制信息、源 URI、文档作者和自定义属性等此类信息的文档元数据文件。每个元数据文件都包含有关单个文档的元数据。

有关要配置的其他重要 JSON 密钥的列表，请参阅Amazon S3模板架构。
