Microsoft Exchange

Microsoft Exchange 是一款用于消息、会议和文件共享的企业协作工具。如果你是微软 Exchange 用户，你可以用它Amazon Kendra来索引你的微软 Exchange 数据源。

你可以使用Amazon Kendra控制台和 TemplateConfigurationAPIAmazon Kendra 连接到你的微软 Exchange 数据源。

要对Amazon Kendra微软 Exchange 数据源连接器进行故障排除，请参见解决数据源问题。

支持的功能

  • 字段映射

  • 包含/排除过滤器

  • Virtual Private Cloud (VPC)

  • 同步所有文档/仅同步新的、修改过的或已删除的文档

先决条件

在使用Amazon Kendra微软 Exchange 数据源编制索引之前，请在微软 Exchange 和AWS账户中进行这些更改。

在微软 Exchange 中，请确保您具有：

  • 在 Office 365 中创建了微软交易所账户。

  • 复制了你的微软 365 租户 ID。你可以在 Azure Active Directory 门户的属性中找到你的租户 ID。有关更多详细信息，请参阅查找您的微软 365 租户 ID。

  • 配置了包含客户端 ID 和客户端密钥的 OAuth 2.0 凭证令牌。

  • 为连接器应用程序添加了以下权限：

    • 日历。阅读

    • 邮件。阅读

    • 邮件。 ReadBasic

    • 邮件。 ReadBasic.All

    • User.Read.All

    • 联系人。阅读

    • 目录。读取。全部

    • 联系人。阅读

    • 备注。阅读。全部

  • 已检查每个文档在 Microsoft Exchange 以及你计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将你的 Microsoft Exchange 身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，则记下密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或密钥，则可以在将 Microsoft Exchange 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接指令

要Amazon Kendra连接到你的微软 Exchange 数据源，你必须提供微软 Exchange 数据源的必要详细信息，这样Amazon Kendra才能访问你的数据。如果你尚未配置 Microsoft Exchange，Amazon Kendra请参阅先决条件。

Console

Amazon Kendra连接到微软 Exchange

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在 “入门” 页面上，选择 “添加数据源”。

  4. 在 “添加数据源” 页面上，选择 Microsoft Exchange 连接器，然后选择 “添加数据源”。

  5. 在指定数据来源详细信息页面输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）说明-为数据来源输入可选说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tags 中，为添加新标签 —Tags，用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在定义访问和安全页面上，输入以下信息：

    1. 来源 —输入你的微软 365 租户 ID。你可以在 Azure Active Directory 门户的属性中找到你的租户 ID。

    2. AWSSecrets Manager密钥 —选择现有密钥或创建新Secrets Manager密钥来存储您的 Microsoft Exchange 身份验证凭据。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      1. 在 “创建AWSSecrets Manager密钥” 窗口中输入以下信息：

        1. 秘密名称 — 您的秘密名称。前缀 “AmazonKendra-微软 Exchange

        2. 对于客户端 ID 和客户端密钥 — 输入你在 Microsoft Exchange 帐户中创建的身份验证凭据值。

      2. 选择保存。

    3. V@@ irtual Private Cloud (VPC) —您可以选择使用 VPC。如果是，则必须添加子网和 VPC 安全组。

    4. 身份搜寻器-选择激活Amazon Kendra身份搜寻器以同步身份信息。如果您选择关闭 Identity Crawler，则必须使用 PutPrincipalMapping API 上传主体信息。

    5. IAM 角色 —选择现有 IAM 角色或创建新的 IAM 角色来访问您的存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    6. 选择下一步。

  7. 在配置同步设置页面上，输入以下信息：

    1. 同步内容-选择要同步的内容。

    2. 其他配置-您可以选择使用这些选项来索引某些内容，而不是同步所有文档。

    3. 同步模式-您可以选择在数据源内容发生变化时更新索引的方式。

      1. 如果您选择完全同步，则无论之前的同步状态如何，都Amazon Kendra将同步所有实体中的所有内容。

      2. 如果您选择新的或修改过的内容同步，则只Amazon Kendra会同步新的或修改过的内容。

      3. 如果您选择新增、修改或删除的内容同步，则只Amazon Kendra会同步新的、修改过的或已删除的内容。

  8. 在 Set 字段映射页面上，输入以下信息：

    1. 默认数据源字段-从Amazon Kendra生成的默认数据源字段中选择要映射到索引的字段。

    2. 添加字段-添加自定义数据源字段以创建要映射到的索引字段名称和字段数据类型。

    3. 选择下一步。

  9. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面编辑信息。成功添加数据源后，该数据源将显示在数据源页面上。

API

Amazon Kendra连接到微软 Exchange

您必须使用 TemplateConfigurationAPI 指定数据源架构的 JSON。您必须提供以下信息：

  • 数据源-必须将数据源指定为MSEXCHANGE。

  • 租户 ID — 可以在 Azure Active Directory 门户的属性中找到你的租户 ID。

  • 类型 —在呼叫时指定TEMPLATE为类型CreateDataSource。

  • 秘密亚马逊资源名称 (ARN) — 提供包含微软 Exchange 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，具有以下键值：

    {
    "client Id": "client ID",
    "client Secret": "client Secret",
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 — 指定您RoleArn何时调用，CreateDataSource以便为IAM角色提供访问您的Secrets Manager密钥以及调用 Microsoft Exchange 连接器所需的公共 API 的权限，以及Amazon Kendra。有关更多信息，请参阅微软 Exchgle 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC) —指定您VpcConfiguration何时致电CreateDataSource。请参阅配置Amazon Kendra为使用 VPC。

  • 启用身份搜寻器-指定是否激活身份搜寻器。如果禁用身份搜寻器，则必须使用 PutPrincipalMappingAPI 上传主体信息。抓取有权访问某些文档的用户和群组的身份信息对于用户上下文筛选很有用。搜索结果根据用户或其群组对文档的访问权限进行筛选。有关更多信息，请参阅在用户上下文中筛选。

  • 注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将你的 Microsoft Exchange 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。
