查询建议

Amazon Kendra查询建议可以帮助您的用户更快地键入搜索查询并指导他们的搜索。

Amazon Kendra根据查询历史记录或查询日志中的热门查询建议与您的用户相关的查询。 Amazon Kendra使用您的用户搜索的所有查询并从这些查询中学习，向您的用户提出建议。 Amazon Kendra在用户开始键入查询时向他们推荐热门查询。 Amazon Kendra如果查询的前缀或前几个字符与用户开始键入的查询内容相匹配，则建议进行查询。

例如，用户开始键入 “即将发生的事件” 查询。 Amazon Kendra从查询日志中了解到，许多用户已经多次搜索 “2050 年即将举行的活动”。用户看到 “2050 年即将举行的活动” 直接出现在他们的搜索栏下方，自动完成他们的搜索查询。用户通过选择第一个搜索结果来选择此查询建议，即文档 “即将发生的事件：2050 年发生的事情”。

您可以指定如何Amazon Kendra选择符合条件的查询向用户建议。例如，您可以指定查询建议必须由至少 10 个唯一用户搜索过（默认为 3），在过去 30 天内搜索过，并且不包含您的屏蔽列表中的任何单词或短语。 Amazon Kendra要求查询至少有一个搜索结果并且包含至少一个超过四个字符的单词。

查询建议不区分大小写。 Amazon Kendra将查询前缀和建议查询转换为小写，忽略所有单引号和双引号，并将多个空格替换为单个空格字符。 Amazon Kendra按原样匹配所有其他特殊字符。 Amazon Kendra如果用户键入的字符少于两个或超过 60 个字符，则不显示任何建议。

您可以使用 GetQuerySuggestionsAPI 检索与您的用户相关的查询建议。查询建议在默认情况下处于激活状态，这不会产生额外的费用。您可以随时使用 UpdateQuerySuggestionsConfigAPI 禁用查询建议。在将建议应用到搜索应用程序之前，您可以通过两种方式测试设置：

  • 通过使用UpdateQuerySuggestionsConfig API。

  • 在控制台的 “查询建议设置” 中。

您可以使用GetQuerySuggestions API 将查询建议与控制台应用程序集成，以便用户开始查看建议。

查询建议设置

您可以使用UpdateQuerySuggestionsConfig API 配置以下设置：

  • 模式 —查询建议为ENABLED或LEARN_ONLY。 Amazon Kendra默认情况下激活查询建议。 LEARN_ONLY关闭查询建议。如果关闭，则会Amazon Kendra继续学习建议，但不会向用户提出查询建议。

  • 查询日志时间窗口-查询日志时间窗口中的查询最近有多长。时间窗是从当天到过去几天的天数的整数值。

  • 不包含用户信息的查询-设置TRUE为包括所有查询或设置FALSE为仅包括包含用户信息的查询。如果您的搜索应用程序在用户发出查询时包含用户信息（例如用户 ID），则可以使用此选项。默认情况下，如果没有与查询相关的特定用户信息，则此设置不会过滤掉查询。但是，您只能使用此设置根据包含用户信息的查询提出建议。

  • 唯一用户-必须搜索查询的唯一用户的最小数量，才有资格向您的用户推荐查询。这个数字是一个整数值。

  • 查询次数-必须搜索查询的最小次数才有资格向用户推荐查询。这个数字是一个整数值。

这些设置会影响如何选择查询作为向用户推荐的热门查询。如何调整设置将取决于您的特定需求，例如：

  • 如果您的用户通常平均每月搜索一次，则可以将查询日志时间窗口中的天数设置为 30，这样您就可以捕获大多数用户最近的查询，以免它们在时间窗口内过期。

  • 如果您的查询中只有一小部分包含用户信息，并且您不想根据较小的样本量提出查询建议，则可以将查询设置为包括所有用户。

  • 如果您将热门查询定义为由至少 10 个唯一用户搜索并搜索至少 100 次，则将唯一用户设置为 10，查询计数设置为 100。

您对设置所做的更改可能不会立即生效。您可以使用 DescribeQuerySuggestionsConfigAPI 跟踪设置更改。更新后的设置生效时间取决于您所做的更新和索引中搜索查询的数量。

Console
检查查询建议是否已激活并准备就绪

  1. 在左侧导航窗格的 “索引” 下，转到您的索引，然后在 “增强” 中选择 “查询建议”。

  2. 在查询建议页面上，转到查询建议设置并执行以下操作：

    1. 检查状态是否为 “已启用”。

    2. 检查准备提出建议的查询数是否超过 0。

编辑查询建议设置

  1. 在左侧导航窗格的 “索引” 下，转到您的索引，然后在 “增强” 中选择 “查询建议”。

  2. 在查询建议页面上，转到查询建议设置并选择编辑。

要清除建议

  1. 在左侧导航窗格的 “索引” 下，转到您的索引，然后在 “增强” 中选择 “查询建议”。

  2. 在 “查询建议” 页面上，转到 “查询建议设置”，然后在 “操作” 中选择 “清除建议”。

CLI

检索查询建议

aws kendra get-query-suggestions \
--index-id index-id \
--query-text "query-text" \
--max-suggestions-count 1 // If you want to limit the number of suggestions.

更新查询建议

例如，要更改查询日志时间窗口和必须搜索查询的最小次数：

aws kendra update-query-suggestions-config \
--index-id index-id \
--query-log-look-back-window-in-days 30 \
--minimum-query-count 100
注意

更新后的设置生效时间取决于您所做的更新和索引中搜索查询的数量。

清除建议

aws  kendra clear-query-suggestions \
--index-id index-id \
注意

Amazon Kendra根据自上次清除建议以来添加到查询日志中的新查询学习新建议。

Python

检索查询建议

import boto3
from botocore.exceptions import ClientError

kendra = boto3.client("kendra")

print("Get query suggestions.")

# Provide the index ID
index_id = "index-id"

# Provide the query text
query_text = "query"

try:
    query_suggestions_response = kendra.get_query_suggestions(
        IndexId = index_id,
        QueryText = query_text,
        MaxSuggestionsCount = 5
    )

    # Print out the suggestions you received
    if ("Suggestions" in query_suggestions_response.keys()) {
        for (suggestion: query_suggestions_response["Suggestions"]) {
            print(suggestion["Value"]["Text"]["Text"]);
        }
    }
   
except ClientError as e:
        print("%s" % e)

print("Program ends.")

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Updating query suggestions settings/configuration for an index.")

# Provide the index ID
index_id = "index-id"

# Configure the settings you want to update
query_log_look_back_window_in_days = 30
minimum_query_count = 100

try:
    kendra.update_query_suggestions_config(
        IndexId = index_id,
        QueryLogLookBackWindowInDays = query_log_look_back_window_in_days,
        MinimumQueryCount = minimum_query_count
    )

    print("Wait for Amazon Kendra to update the query suggestions.")

    while True:
        # Get query suggestions description of settings/configuration
        query_sugg_config_response = kendra.describe_query_suggestions_config(
            IndexId = index_id
        )
        
        # If status is not UPDATING, then quit
        status = query_sugg_config_response["Status"]
        print(" Updating query suggestions config. Status: " + status)
        if status != "UPDATING":
            break
        time.sleep(60)

except ClientError as e:
        print("%s" % e)

print("Program ends.")

清除建议

import boto3
from botocore.exceptions import ClientError

kendra = boto3.client("kendra")

print("Clearing out query suggestions for an index.")

# Provide the index ID
index_id = "index-id"

try:
    kendra.clear_query_suggestions(
        IndexId = index_id
    )

    # Confirm last cleared date-time and that there are no suggestions
    query_sugg_config_response = kendra.describe_query_suggestions_config(
        IndexId = index_id
    )
    print("Query Suggestions last cleared at: " + str(query_sugg_config_response["LastClearTime"]));
    print("Number of suggestions available to use after clearing: " + str(query_sugg_config_response["TotalSuggestionsCount"]));
        
except ClientError as e:
        print("%s" % e)

您可以使用DescribeQuerySuggestionsConfig API 检查您的当前设置。此外，此操作还显示有关您的索引查询建议的以下信息：

  • 模式 —查询建议为ENABLED或LEARN_ONLY。 Amazon Kendra默认情况下激活查询建议。 LEARN_ONLY关闭查询建议。您可以使用UpdateQuerySuggestionsConfig API 更改模式。如果关闭，则会Amazon Kendra继续学习建议，但不会向用户提出查询建议。

  • 状态 —查询建议为ACTIVE或UPDATING。

  • 建议数量-准备向用户推荐的查询总数。如果数量远低于您的预期，则可能是因为Amazon Kendra需要更多查询来学习，或者您当前的查询建议设置过于严格。

  • 上次生成时的建议-上次更新建议的时间。 Amazon Kendra每 24 小时自动更新一次建议，或者在您更改设置或应用屏蔽列表时自动更新建议。

  • 上次清除建议的时间-上次清除建议的时间。

屏蔽建议中的某些查询

屏蔽Amazon Kendra列表无法向您的用户推荐某些热门查询。它是您要从查询建议中排除的单词或短语的列表。 Amazon Kendra排除包含与阻止列表中单词或短语完全匹配的查询。

您可以使用屏蔽列表来防范查询日志中经常出现并Amazon Kendra可能被选作建议的攻击性词语或短语。屏蔽列表还可以Amazon Kendra防止推荐包含尚未准备好公开发布或公布的信息的热门查询。例如，如果您的用户经常询问新产品的发布情况，而您不想提出建议，因为您还没有准备好很快发布，那么您可以屏蔽包含建议中产品名称和产品信息的查询。

您可以使用 CreateQuerySuggestionsBlockListAPI 为查询创建阻止列表。将每个方块单词或短语放在文本文件中的单独一行中。然后，您将文本文件上传到 S3 存储桶，并在中提供该文件的路径或位置Amazon S3。 Amazon Kendra目前仅支持创建一个阻止列表。

您可以替换Amazon S3存储桶中被屏蔽的单词和短语的文本文件，并使用 UpdateQuerySuggestionsBlockListAPI 更新中的屏蔽列表Amazon Kendra。

您可以使用 DescribeQuerySuggestionsBlockListAPI 获取屏蔽列表的状态和其他有用信息，例如上次更新屏蔽列表的时间、当前的屏蔽列表中有多少字词或短语，以及创建屏蔽列表时的有用错误消息。您还可以使用 ListQuerySuggestionsBlockListsAPI 获取索引的黑名单摘要列表。

要删除您的屏蔽列表，请使用 DeleteQuerySuggestionsBlockListAPI。

您对屏蔽列表的更新可能不会立即生效。您可以使用DescribeQuerySuggestionsBlockList API 跟踪更新。

Python

创建黑名单

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Create a query suggestions block list.")

# Provide a name for the block list
block_list_name = "block-list-name"
# Provide an optional description for the block list
block_list_description = "block-list-description"
# Provide the IAM role ARN required for query suggestions block lists
block_list_role_arn = "role-arn"

# Provide the index ID
index_id = "index-id"

s3_bucket_name = "bucket-name"
s3_key = "query-suggestions/block_list.txt"
source_s3_path = {
    'Bucket': s3_bucket_name,
    'Key': s3_key
}

try:
    block_list_response = kendra.create_query_suggestions_block_list(
        Description = block_list_description,
        Name = block_list_name,
        RoleArn = block_list_role_arn,
        IndexId = index_id,
        SourceS3Path = source_s3_path
    )

    print(block_list_response)

    block_list_id = block_list_response["Id"]

    print("Wait for Amazon Kendra to create the block list.")

    while True:
        # Get block list description
        block_list_description = kendra.describe_query_suggestions_block_list(
            Id = block_list_id,
            IndexId = index_id
        )
        # If status is not CREATING, then quit
        status = block_list_description["Status"]
        print("Creating block list. Status: " + status)
        if status != "CREATING":
            break
        time.sleep(60)
        
except ClientError as e:
        print("%s" % e)

print("Program ends.")

明确建议

您可以使用 ClearQuerySuggestionsAPI 清除查询建议。清除建议仅删除现有的查询建议，不会删除查询日志中的查询。清除建议后，Amazon Kendra会根据从清除建议之时起添加到查询日志中的新查询来学习新的建议。

没有建议

如果您没有看到查询建议，出于以下原因之一：

  • 您的索引中没有足够的查询Amazon Kendra可供学习。

  • 您的查询建议设置过于严格，导致大多数查询被从建议中过滤掉。

  • 您最近清除了建议，Amazon Kendra但仍需要时间积累新的查询才能学习新的建议。

您可以使用DescribeQuerySuggestionsConfig API 检查您的当前设置。
