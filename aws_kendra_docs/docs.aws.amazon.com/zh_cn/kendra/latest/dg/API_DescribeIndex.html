DescribeIndex

获取有关现有的 Amazon Kendra 索引的信息。

请求语法

{
   "Id": "string"
}

请求参数

有关所有操作的通用参数的信息，请参阅通用参数。

请求接受采用 JSON 格式的以下数据。

Id

您要获取信息的索引的标识符。

类型：字符串

长度限制：固定长度为 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必需：是

响应元素

如果此操作成功，则该服务将会发送回 HTTP 200 响应。

服务以 JSON 格式返回的以下数据。

CapacityUnits

对于企业版索引，您可以选择使用额外容量来满足应用程序的需求。这包含用于索引的容量单位。查询或文档存储容量为零表示索引正在使用默认容量。有关索引默认容量及其调整的更多信息，请参阅调整容量。

类型：CapacityUnitsConfiguration 对象

CreatedAt

创建索引时的时间戳。

类型：时间戳

Description

索引的描述。

类型：字符串

长度限制：最小长度为 0。最大长度为 1000。

模式：^\P{C}*$

DocumentMetadataConfigurations

文档元数据或字段的配置信息。文档元数据是与您的文档相关的字段或属性。例如，与每个文档关联的公司部门名称。

类型：DocumentMetadataConfiguration 对象数组

数组成员：最少 0 项。最多 500 项。

Edition

用于索引的Amazon Kendra 版本。在创建索引时决定版本。

类型：字符串

有效值: DEVELOPER_EDITION | ENTERPRISE_EDITION

ErrorMessage

当Status字段值为时FAILED，该ErrorMessage字段包含一条解释原因的消息。

类型：字符串

长度限制：最小长度为 1。最大长度为 2048。

模式：^\P{C}*$

Id

索引的标识符。

类型：字符串

长度限制：固定长度为 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

IndexStatistics

提供有关常见问题解答问题和答案的数量以及索引的文本文档数量的信息。

类型：IndexStatistics 对象

Name

索引的名称。

类型：字符串

长度限制：最小长度为 1。长度上限为 1000。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

RoleArn

可供Amazon Kendra 代入以获取 Amazon Cloudwatch 日志的 IAM 角色的 Amazon 资源名称 (ARN)。

类型：字符串

长度约束：最小长度为 0。长度上限为 1284。

模式：arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

ServerSideEncryptionConfiguration

用于加密您的数据的AWS KMS客户主密钥（CMK）的标识符。Amazon Kendra 不支持非对称 CMK。

类型：ServerSideEncryptionConfiguration 对象

Status

索引的当前状态。当值为时ACTIVE，索引已准备就绪，可供使用。如果Status字段值为FAILED，则该ErrorMessage字段包含一条解释原因的消息。

类型：字符串

有效值: CREATING | ACTIVE | DELETING | FAILED | UPDATING | SYSTEM_UPDATING

UpdatedAt

上次更新索引时的 Unix。

类型：时间戳

UserContextPolicy

Amazon Kendra 指数的用户上下文政策。

类型：字符串

有效值: ATTRIBUTE_FILTER | USER_TOKEN

UserGroupResolutionConfiguration

您是否启用了从AWS IAM Identity Center (successor to AWS Single Sign-On)身份源获取群组和用户的访问级别的配置。

类型：UserGroupResolutionConfiguration 对象

UserTokenConfigurations

Amazon Kendra 索引的用户令牌配置。

类型：UserTokenConfiguration 对象数组

数组成员：最多 1 项。

错误

有关所有操作常见错误的信息，请参阅常见错误。

AccessDeniedException

您没有足够的权限执行此操作。请确保您拥有所需的权限策略和用户帐户，然后重试。

HTTP 状态代码：400

InternalServerException

用于您的Amazon Kendra 服务的内部服务器出现问题。请稍等几分钟，然后重试，或联系Su pport 部门寻求帮助。

HTTP 状态代码：500

ResourceNotFoundException

你要使用的资源不存在。请检查您提供的资源是否正确，然后重试。

HTTP 状态代码：400

ThrottlingException

由于请求限制而导致请求被拒绝。请减少请求数量，然后重试。

HTTP 状态代码：400

ValidationException

输入未能满足 Amazon Kendra 服务设置的限制。请提供正确的输入并重试。

HTTP 状态代码：400
