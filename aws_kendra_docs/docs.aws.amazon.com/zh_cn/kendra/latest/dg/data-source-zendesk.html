Zendesk

Zendesk是一个客户关系管理系统，可帮助企业自动化和增强客户支持互动。您可以使用Amazon Kendra索引 Zendesk 支持票证、票证评论、票证附件、帮助中心文章、文章评论、文章评论附件、指南社区主题、社区帖子和社区帖子评论。

如果您想为仅位于特定组织内的票证编制索引，则可以按组织名称进行筛选。您也可以选择为何时开始从 Zendesk 抓取数据设置抓取日期。

您可以使用Amazon Kendra控制台和 TemplateConfigurationAPI 连接Amazon Kendra到 Zendesk 数据源。

要对Amazon Kendra Zendesk 数据源连接器进行故障排除，请参阅解决数据源问题。

支持的功能

Amazon KendraZendesk 数据源连接器支持以下功能：

  • 更改日志

  • 字段映射

  • 用户上下文过滤

  • 包含/排除过滤器

  • Virtual Private Cloud (VPC)

先决条件

在使用Amazon Kendra索引 Zendesk 数据源之前，请先在 Zendesk 和AWS帐户中进行这些更改。

在 Zendesk 中，确保您具有：

  • 创建了 Zendesk Suite（专业版/企业版）管理账户。

  • 记下了你的 Zendesk 主机网址。例如，https://{sub-domain (https://{host/)}.zendesk.com/。

  • 生成了 OAuth 2.0 凭证令牌，其中包含客户端 ID、客户端密钥、用户名和密码。有关更多信息，请参阅 Zendesk 关于生成 OAuth 2.0 令牌的文档。

  • 添加了以下 OAuth 2.0 作用域：

    • read

  • 可选：已安装Amazon Kendra允许连接的 SSL 证书。

  • 已检查每个文档在 Zendesk 以及您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的 Zendesk 身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下该密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或机密，则可以在将 Zendesk 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接指令

要Amazon Kendra连接到您的 Zendesk 数据源，您必须提供 Zendesk 数据源的必要详细信息，Amazon Kendra以便访问您的数据。如果您尚未配置 Zendesk，请参Amazon Kendra阅先决条件。

Console

连接 ZenAmazon Kendra desk

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在入门页面上，选择添加数据源。

  4. 在 “添加数据源” 页面上，选择 Zendesk 连接器，然后选择 “添加数据源”。

  5. 在指定数据来源详细信息页面输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）说明-为数据来源输入可选的说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tags 中，Add ne w tag（添加标签），用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在 Def ine Security（定义访问和安全性）页面上，输入以下信息：

    1. Zendesk 网址 —输入你的 Zendesk 网址。

    2. AWSSecrets Manager密钥 —选择现有密钥或创建新Secrets Manager密钥来存储您的 Zendesk 身份验证凭据。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      1. 在 “创建AWSSecrets Manager密钥” 窗口中输入以下信息：

        1. 秘密名称 — 您的秘密名称。前缀 “AmazonKendra-Zendesk-” 会自动添加到您的机密名称中。

        2. 对于客户端 ID、客户端密钥、用户名、密码 — 输入您在 Zendesk 帐户中创建的身份验证凭证值。

      2. 选择保存。

    3. V@@ irtual Private Cloud (VPC) —您可以选择使用 VPC。如果是，则必须添加子网和 VPC 安全组。

    4. IAM 角色 —选择现有 IAM 角色或创建新的 IAM 角色，以访问存储库凭证和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    5. 选择下一步。

  7. 在配置同步设置页面输入以下信息：

    1. 选择实体或内容类型 — 要抓取的 Zendesk 实体或内容类型。

    2. 更改日志-选择更新索引，而不是同步所有文件。

    3. 组织名称 —输入 Zendesk 组织名称以筛选您的同步。

    4. 同步开始日期-您想要对内容进行索引的起始日期。

    5. 正则表达式@@ 模式 —包含或排除某些文件的正则表达式模式。您最多可以添加 100 个模式。

    6. 在 F requency 的同步运行时间表中-选择与数据源同步的频率。Amazon Kendra

    7. 选择下一步。

  8. 在 Set fild pription（设置字段映射）页面上，输入以下信息：

    1. 对于票证、票证评论、票证评论附件、文章、文章评论、文章评论附件、社区主题、社区帖子评论-从Amazon Kendra生成的默认数据源字段中选择要映射到索引的默认数据源字段。

    2. 添加字段-添加自定义数据源字段以创建要映射到的索引字段名称和字段数据类型。

    3. 选择下一步。

  9. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面编辑信息。成功添加数据源后，它将显示在数据源页面上。

API

连接 ZenAmazon Kendra desk

您必须使用 TemplateConfigurationAPI 指定数据源架构的 JSON。您必须提供以下信息：

  • 数据源-将数据源指定为ZENDESK。

  • 主机 URL —提供您的 Zendesk 主机 URL 作为连接配置或存储库端点详细信息的一部分。例如，https://yoursubdomain.zendesk.com。

  • 类型 —在呼叫时指定TEMPLATE为类型CreateDataSource。

  • 秘密亚马逊资源名称 (ARN)-提供包含您的 Zendesk 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，密钥以下密钥：

    {
    "hostUrl": "https://yoursubdomain.zendesk.com",
    "clientId": "kendra",
    "clientSecret": "Zendesk client secret",
    "userName": "Zendesk user name",
    "password": "Zendesk password"
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 —指定RoleArn何时调用CreateDataSource，为IAM角色提供访问您的Secrets Manager密钥以及调用 Zendesk 连接器所需的公共 API 的权限，以及Amazon Kendra. 有关更多信息，请参阅 Zendesk 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC) —指定VpcConfiguration何时进行CreateDataSource操作。请参阅配置Amazon Kendra为使用 VPC。

  • 更改日志-是否Amazon Kendra应使用 Zendesk 数据源更改日志机制来确定是否必须在索引中添加、更新或删除文档。

    注意

    如果您不想扫描所有文档Amazon Kendra，请使用更改日志。如果您的更改日志很大，则扫描 Zendesk 数据源中的文档所花费的时间可能Amazon Kendra少于处理更改日志所花费的时间。如果您是第一次将 Zendesk 数据源与索引同步，则会扫描所有文档。

  • 包含和排除过滤器-指定是包含还是排除：

    • Support 票证、票证评论和/或票证评论附件

    • 帮助中心文章、文章附件和文章评论

    • 指导社区话题、帖子或发表评论

  • 注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 Zendesk 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • 用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或其群组对文档的访问权限筛选搜索结果。有关更多信息，请参阅 Zendesk 数据源的用户上下文筛选。

有关要配置的其他重要 JSON 密钥的列表，请参阅Zendesk 模板架构。
