DocumentAttributeTarget

在提取文档到 Amazon Kendra 时，您想要更改的目标文档属性或元数据字段。

例如，您可以删除与文档关联的客户标识号，这些标识号存储在名为“Customer_ID”的文档元数据字段中。您要将目标键设置为“Customer_ID”，将删除标志设置为 TRUE。这将移除“Customer_ID”字段中的所有客户 ID 值。这样做会清理每个文档的元数据中的个人身份信息。

如果目标字段尚未创建并作为索引字段，则 Amazon Kendra 无法创建该字段。创建索引字段后，您可以使用 DocumentAttributeTarget 创建文档元数据字段。然后，Amazon Kendra 会将您新创建的元数据字段映射到您的索引字段。

您也可以将其与... 一起使用DocumentAttributeCondition。

目录

TargetDocumentAttributeKey

目标文档属性或元数据字段的标识符。

例如，“Department”可以是目标属性或元数据字段的标识符，其中包含与文档关联的部门名称。

类型：字符串

长度限制：最小长度为 1。最大长度为 200。

模式：[a-zA-Z0-9_][a-zA-Z0-9_-]*

必需：否

TargetDocumentAttributeValue

您想要为目标属性创建的目标值。

例如，“Finance”可以是“Department”目标属性键的目标值。

类型：DocumentAttributeValue 对象

必需：否

TargetDocumentAttributeValueDeletion

TRUE 会删除指定目标属性键的现有目标值。您无法创建目标值并将其设置为 TRUE。要创建目标值（TargetDocumentAttributeValue），将其设置为 FALSE。

类型：布尔值

必需：否
