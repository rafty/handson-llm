AWS IAM Identity Center (successor to AWS Single Sign-On)身份源包含有关您的用户和群组的信息。这对于设置用户上下文筛选很有用，即根据用户或其组对文档的访问权限Amazon Kendra筛选不同用户的搜索结果。

要创建 IAM 身份中心身份源，您必须激活 IAM 身份中心并在中创建组织AWS Organizations。当您激活 IAM Identity Center 并首次创建组织时，它会自动默认为 Identity Center 目录作为身份来源。您可以更改为 Active Directory（亚马逊管理或自我管理）或外部身份提供商作为您的身份来源。为此，您必须遵循正确的指导方针 — 请参阅更改 IAM 身份中心身份来源。每个组织只能有一个Identitter Center Center Cer

为了为您的用户和群组分配不同的文档访问级别，在将文档载入索引时，您需要将您的用户和群组包括在访问控制列表中。这允许您的用户和群组根据Amazon Kendra其访问级别搜索文档。当您发出查询时，用户 ID 必须与 IAM Identity Center 中的用户名完全匹配。

  1. 打开 IAM 身份中心控制台。

  2. 选择启用 IAM 身份中心，然后选择创建AWS组织。

    默认情况下会创建 Identity Center 目录，并向您发送一封电子邮件以验证与组织关联的电子邮件地址。

  3. 要向您的AWS组织添加群组，请在导航窗格中选择群组。

  4. 在群组页面上，选择创建群组，然后在对话框中输入群组名称和描述。选择 Create（创建）。

  5. 要将用户添加到您的Organizations，请在导航窗格中选择用户。

  6. 在 Users (用户) 页面上，选择 Add user (添加用户)。在用户详细信息下，指定所有必填字段。对于密码，选择向用户发送电子邮件。选择 Next（下一步）。

  7. 要将用户添加到群组，请选择群组并选择一个群组。

  8. 在详细信息页面的群组成员下，选择添加用户。

  9. 在将用户添加到群组页面上，选择要添加为群组成员的用户。您可以选择多个用户添加到组中。

  10. 要将您的用户和群组列表与 IAM 身份中心同步，请将您的身份源更改为 Active Directory 或外部身份提供商。

    Identity Center 目录是默认身份源，如果您没有自己的列表由提供商管理，则需要您使用此来源手动添加用户和群组。要更改您的身份来源，您必须遵循正确的指导方针，请参阅更改 IAM Identity Center 身份来源。
