Slack

Slack是一款企业通信应用程序，允许用户通过各种公共和私人渠道发送消息和附件。您可以使用Amazon Kendra索引您的 Slack 公共和私人频道、机器人和存档消息、文件和附件、直接和群组消息。您也可以选择要筛选的特定内容。

您可以使用Amazon Kendra控制台和 SlackConfigurationAPIAmazon Kendra 连接到您的 Slack 数据源。

要对Amazon Kendra Slack 数据源连接器进行故障排除，请参阅解决数据源问题。

支持的功能

Amazon KendraSlack 数据源连接器支持以下功能：

  • 更改日志

  • 字段映射

  • 用户上下文过滤

  • 包含/排除过滤器

  • Virtual Private Cloud (VPC)

先决条件

在使用Amazon Kendra索引 Slack 数据源之前，请先在 Slack 和AWS账户中进行这些更改。

在 Slack 中，请确保您有

  • 创建了 Slack Bot 用户 OAuth 令牌或 Slack 用户 OAuth 令牌。您可以选择任一令牌Amazon Kendra来连接您的 Slack 数据源。有关更多信息，请参阅有关访问令牌的 Slack 文档。

    注意

    如果您使用机器人令牌作为 Slack 凭据的一部分，则无法索引直接消息和群组消息，并且必须将机器人令牌添加到要索引的频道中。

  • 记下了你的 Slack 工作区团队 ID 来自你的 Slack 工作区主页面 URL。例如，https://app.slack.com/client/T0123456789/... 其中 T0123456789 是团队 ID。

  • 添加了以下 [Oauth 作用域/读取] 权限：

    • 频道:历史

    • 频道:读取

    • 群组:历史

    • 群组:读取

    • im: 历史

    • mpim: 历史

    • mpim: 读取

    • team:read

    • 用户个人资料:读取

    • 用户:读取

    • 表情符号:阅读

    • 文件:读取

    • 用户组:读取

  • 已检查每个文档在 Slack 以及您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的 Slack 身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下该密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或密钥，则可以在将 Slack 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

API

要连接Amazon Kendra到 Slack

您必须使用 SlackConfigurationAPI 指定以下内容：

  • Slack 工作区团队 ID —您从 Slack 主页网址复制的 Slack 团队 ID。

  • 要索引的实体列表 —是否Amazon Kendra应索引您的公共和私人频道以及群组和直接消息。

  • 抓取日期 — 开始从 Slack 工作区团队抓取数据的日期。日期必须遵循此格式: yyyy-mm-dd.

  • 秘密亚马逊资源名称 (ARN)-提供包含您的 Slack 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，

    {
    "slackToken": "token"
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 — 指定您RoleArn何时调用，CreateDataSource以便为IAM角色提供访问您的Secrets Manager密钥和调用 Slack 连接器所需的公共 API 的权限，以及Amazon Kendra。有关更多信息，请参阅 Slack 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC)-指定VpcConfiguration为数据源配置的一部分。请参阅配置Amazon Kendra使用 VPC。

  • 更改日志-是否Amazon Kendra应使用 Slack 数据源更改日志机制来确定是否必须在索引中添加、更新或删除文档。

    注意

    如果您不想扫描所有文档Amazon Kendra，请使用更改日志。如果您的更改日志很大，则扫描 Slack 数据源中的文档所花费Amazon Kendra的时间可能少于处理更改日志所花费的时间。如果您是第一次将 Slack 数据源与索引同步，则会扫描所有文档。

  • 包含和排除过滤器-指定是包括还是排除某些公共和私人频道、群组和私人消息，以及机器人和存档消息。如果您使用机器人令牌作为 Slack 身份验证凭证的一部分，则必须将机器人令牌添加到要索引的频道。您无法使用机器人令牌为直接消息和群组消息编制索引。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 Slack 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • 用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或其群组对文档的访问权限筛选搜索结果。有关更多信息，请参阅 Slack 数据源的用户上下文筛选。
