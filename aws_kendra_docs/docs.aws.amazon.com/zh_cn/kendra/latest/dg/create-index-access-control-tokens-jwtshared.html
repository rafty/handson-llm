使用带有共享密令牌 (JWT T)

以下示例说明在创建索引时如何使用带有共享密钥令牌的 JSON Web 令牌 (JWT) 来控制用户访问权限。

Console

  1. 选择 “创建索引” 开始创建新索引。

  2. 在指定索引详细信息页面上，为索引指定名称和描述。

  3. 对于 IAM 角色，选择一个角色或选择创建新角色并指定角色名称以创建新角色。IAM 角色将带有前缀 “AmazonKendra-”。

  4. 保留所有其他字段的默认值。选择下一步。

  5. 在 “配置用户访问控制” 页面的 “访问控制设置” 下，选择 “是” 以使用令牌进行访问控制。

  6. 在令牌配置下，选择具有共享密钥的 JWT 作为令牌类型。

  7. 在 “用于签名共享密钥的参数” 下，选择密钥的类型。您可以使用现有的AWS Secrets Manager共享密钥或创建新的共享密钥。

    要创建新的共享密钥，请选择 “新建”，然后按照以下步骤操作：

    1. 在 “新AWS Secrets Manager密钥” 下，指定密钥名称。前缀AmazonKendra-将在您保存公钥时添加。

    2. 指定密钥 ID。密钥编号是一个提示，指示哪些密钥用于保护令牌的 JSON Web Signature。

    3. 为令牌选择签名算法。这是用于保护 ID 令牌令牌的加密算法。有关 RSA 的更多信息，请参阅 RSA 密码术。

    4. 通过输入 base64 URL 编码的密钥来指定共享密钥。您也可以选择 “生成密钥” 来为您生成密钥。您必须确保该密钥是 base64 URL 编码的密钥。

    5. （可选）指定共享密钥何时有效。您可以指定密钥的有效日期和时间、有效期和/或两者。该密钥将在指定的间隔内有效。

    6. 选择 “保存密钥” 以保存新密钥。

  8. （可选）在 “高级配置” 下：

    1. 指定在 ACL 检查中使用的用户名。

    2. 指定要在 ACL 检查中使用的一个或多个组。

    3. 指定将验证令牌发行者的发行者。

    4. 指定索赔编号。您必须指定与 JWT 中的受众相匹配的正则表达式。

  9. 在配置详细信息页面中，选择开发者版本。

  10. 选择 “创建” 来创建您的索引。

  11. 等待您的索引创建。 Amazon Kendra为您的索引配置硬件。此操作可能需要一些时间。

CLI

你可以在里面使用带有共享密钥的 JWT 令牌AWS Secrets Manager。密钥必须是 bas64 编码的 bas64 编码的密钥。您需要Secrets Manager ARN，并且您的Amazon Kendra角色必须有权GetSecretValue访问该Secrets Manager资源。如果您使用加密Secrets Manager资源AWS KMS，则该角色还必须有权访问解密操作。

要使用 JSON 输入文件创建索引，请先AWS CLI使用所需参数创建 JSON 文件：

{
    "Name": "user-context",
    "Edition": "ENTERPRISE_EDITION",
    "RoleArn": "arn:aws:iam::account-id:role:/my-role",
    "UserTokenConfigurations": [
        {
            "JwtTokenTypeConfiguration": {
                "KeyLocation": "SECRET_MANAGER",
                "Issuer": "optional: specify the issuer url",
                "ClaimRegex": "optional: regex to validate claims in the token",
                "UserNameAttributeField": "optional: user",
                "GroupAttributeField": "optional: group",
                "SecretManagerArn": "arn:aws:secretsmanager:us-west-2:account id:secret:/my-user-context-secret
            }
        }
    ],    
    "UserContextPolicy": "USER_TOKEN"
}

您可以覆盖默认的用户和组字段名称。的默认值UserNameAttributeField为 “用户”。的默认值GroupAttributeField是 “群组”。

接下来，create-index使用输入文件调用。例如，如果您的 JSON 文件的名称是create-index-openid.json，则可以使用以下内容：

aws kendra create-index --cli-input-json file://create-index-openid.json

密钥必须采用以下格式AWS Secrets Manager：

{
  "keys": [
    {
      "kid": "key_id",
      "alg": "HS256|HS384|HS512",
      "kty": "OCT", 
      "use": "sig", //this value can be sig only for now
      "k": "secret",
      "nbf":"ISO1806 date format"
      "exp":"ISO1806 date format"
    }
  ]
}

有关 JWT 的更多信息，请参阅 jwt.io。

Python

你可以在里面使用带有共享密钥的 JWT 令牌AWS Secrets Manager。密钥必须是 bas64 编码的 bas64 编码的密钥。您需要Secrets Manager ARN，并且您的Amazon Kendra角色必须有权GetSecretValue访问该Secrets Manager资源。如果您使用加密Secrets Manager资源AWS KMS，则该角色还必须有权访问解密操作。

response = kendra.create_index(
    Name='user-context',
    Edition='ENTERPRISE_EDITION',
    RoleArn='arn:aws:iam::account-id:role:/my-role',
    UserTokenConfigurations=[
        {
            "JwtTokenTypeConfiguration": {
                "KeyLocation": "URL",
                "Issuer": "optional: specify the issuer url",
                "ClaimRegex": "optional: regex to validate claims in the token",
                "UserNameAttributeField": "optional: user",
                "GroupAttributeField": "optional: group",
                "SecretManagerArn": "arn:aws:secretsmanager:us-west-2:account id:secret:/my-user-context-secret"
            }
        }
    ],
    UserContextPolicy='USER_TOKEN'
)
