IAM的访问角色Amazon Kendra

创建索引、数据源或常见问题解答时，Amazon Kendra需要访问创建AWSAmazon Kendra资源所需的资源。在创建Amazon Kendra资源之前，必须创建AWS Identity and Access Management (IAM) 策略。当您调用该操作时，您提供了附带策略的角色的 Amazon 资源名称 (ARN)。例如，如果您调用 BatchPutDocumentAPI 来添加Amazon S3存储桶中的文档，则需要为角色提供Amazon Kendra具有访问该存储桶的策略的权限。

您可以在Amazon Kendra控制台中创建新IAM角色或选择要使用的IAM现有角色。控制台显示角色名称中包含字符串 “kendra” 或 “Kendra” 的角色。

以下主题提供了所需策略的详细信息。如果您使用Amazon Kendra控制台创建IAM角色，则这些策略是为您创建的。

IAM索引的角色

创建索引时，必须为IAM角色提供写入权限Amazon CloudWatch。您还必须提供允许代Amazon Kendra入角色的信任策略。以下是必须提供的政策。

允许访问CloudWatch日志Amazon Kendra的角色策略。

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "cloudwatch:PutMetricData",
            "Resource": "*",
            "Condition": {
                "StringEquals": {
                    "cloudwatch:namespace": "AWS/Kendra"
                }
            }
        },
        {
            "Effect": "Allow",
            "Action": "logs:DescribeLogGroups",
            "Resource": "*"
        },
        {
            "Effect": "Allow",
            "Action": "logs:CreateLogGroup",
            "Resource": "arn:aws:logs:region:account ID:log-group:/aws/kendra/*"
        },
        {
            "Effect": "Allow",
            "Action": [
                "logs:DescribeLogStreams",
                "logs:CreateLogStream",
                "logs:PutLogEvents"
            ],
            "Resource": "arn:aws:logs:region:account ID:log-group:/aws/kendra/*:log-stream:*"
        }
    ]
}

IAM BatchPutDocumentAPI 的角色

警告

Amazon Kendra不使用向Amazon Kendra委托人授予与 S3 存储桶交互的权限的存储桶策略。相反，它使用IAM角色。确保Amazon Kendra不将其作为可信成员包含在您的存储段策略中，以避免在意外向任意主体授予权限时出现任何数据安全问题。但是，您可以添加存储桶策略以在不同的账户中使用Amazon S3存储桶。有关更多信息，请参阅适用于Amazon S3跨账户的策略。有关 S3 数据源IAM角色的信息，请参阅IAM角色。

当您使用 BatchPutDocumentAPI 为Amazon S3存储段中的文档编制索引时，必须提供Amazon Kendra具有存储桶访问权限的IAM角色。您还必须提供允许代Amazon Kendra入角色的信任策略。如果存储段中的文档已加密，则必须提供使用AWS KMS客户主密钥 (CMK) 解密文档的权限。

允许Amazon Kendra访问Amazon S3存储段所需的角色策略。

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "s3:GetObject"
            ],
            "Resource": [
                "arn:aws:s3:::bucket name/*"
            ]
        }
    ]
}

允许担任角色Amazon Kendra的信任策略。

{
   "Version":"2012-10-17",
   "Statement":[
      {
         "Effect":"Allow",
         "Principal":{
            "Service":"kendra.amazonaws.com"
         },
         "Action":"sts:AssumeRole"
      }
   ]
}

建议您在信任策略aws:sourceArn中包括aws:sourceAccount和。这会限制权限aws:sourceAccount并安全地检查sts:AssumeRole操作的IAM角色策略中提供的和aws:sourceArn是否相同。这可以防止未经授权的实体访问您的IAM角色及其权限。有关更多信息，请参阅有关更多信息，AWS Identity and Access Management请参阅有关更多信息。

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Service": [
                    "kendra.*.amazonaws.com"
                ]
            },
            "Action": "sts:AssumeRole",
            "Condition": {
                "StringEquals": {
                    "aws:SourceAccount": "account ID"
                },
                "StringLike": {
                    "aws:SourceArn": "arn:aws:kendra:region:accountId:index/*"
                }
            }
        }
    ]
}

一种可选的角色策略Amazon Kendra，允许使用AWS KMS客户主密钥 (CMK) 解密Amazon S3存储段中的文档。

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "kms:Decrypt"
            ],
            "Resource": [
                "arn:aws:kms:region:account ID:key/key ID"
            ]
        }
    ]
}

Amazon S3跨账户使用的政策

如果您的Amazon S3存储桶与您用于Amazon Kendra索引的账户位于不同的账户中，则可以创建策略以跨账户使用该存储桶。

一种角色策略，当Amazon S3存储桶与您的Amazon Kendra索引位于不同的账户中时，使用您的存储桶作为数据源。

{
    "Version": "2012-10-17",
    "Statement": [
         {
            "Action": [
                "s3:GetObject"
            ],
            "Resource": [
                "arn:aws:s3:::$BUCKET_IN_OTHER_ACCOUNT/*"
            ],
            "Effect": "Allow"
        },
        {
            "Action": [
                "s3:ListBucket"
            ],
            "Resource": [
                "arn:aws:s3:::$BUCKET_IN_OTHER_ACCOUNT/*"
            ],
            "Effect": "Allow"
        },
        {
            "Effect": "Allow",
            "Action": [
                "kendra:BatchPutDocument",
                "kendra:BatchDeleteDocument"
            ],
            "Resource": [
                "arn:aws:kendra:$KENDRA_REGION:$KENDRA_ACCOUNT_ID:index/$KENDRA_INDEX_ID"
            ]
        },
        {
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:PutObject",
                "s3:PutObjectAcl"
            ],
            "Resource": "arn:aws:s3:::$BUCKET_IN_OTHER_ACCOUNT/*"
        }
