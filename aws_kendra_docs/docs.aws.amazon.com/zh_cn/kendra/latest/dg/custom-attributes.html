创建自定义文档字段数

您可以应用与您的特定文档相关的自定义属性或元数据字段。例如，您可以创建名为 “部门” 的自定义字段或属性，其值为 “HR”、“销售” 和 “制造”。例如，您可以使用这些字段或属性将响应（搜索结果）限制为 “HR” 部门的文档。

您最多可创建 500 个自定义字段或属性。

对于大多数数据源，您可以将外部数据源中的字段映射到中的相应字段Amazon Kendra。有关更多信息，请参阅映射数据源字段：对于 S3 数据源，您可以使用元数据文件应用自定义字段或属性。

必须首先在索引中创建该字段，然后才能使用自定义字段或属性。使用控制台或 UpdateIndexAPI 创建索引字段。一旦创建了字段，就无法更改该字段的数据类型。

您也可以使用Amazon Kendra内置或保留字段。

使用UpdateIndex API，您可以使用DocumentMetadataConfigurationUpdates参数添加自定义字段或属性。

以下 JSON 示例使用DocumentMetadataConfigurationUpdates向索引中添加一个名为 “部门” 的字段。

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

使用Amazon Kendra 内置文档字段

借助 UpdateIndex API，您可以使用DocumentMetadataConfigurationUpdates并指定保留字段名称来创建保留字段或内置字段，以映射到您的等效文档属性或文档字段。您也可以通过这种方式创建自定义字段。如果您使用数据源连接器，则大多数都包含将数据源文档字段映射到Amazon Kendra索引字段的字段映射。创建字段时，可以配置该Search对象，将该字段设置为可显示、可分面、可搜索和可排序。您可以配置该Relevance对象以设置字段的相关性提升持续时间、新鲜度、重要性、排名顺序和重要性值。一旦创建了字段，就无法更改字段类型。

Amazon Kendra有以下可供使用的保留或内置文档字段：

  • _authors—负责文档内容的一位或多位作者的列表。

  • _category— 将文档置于特定组中的类别。

  • _created_at— 以 ISO 8601 格式创建文档的日期和时间。例如，2012-03-25T12:30:10+01:00 是中部欧洲时间 2012 年 3 月 25 日中午 12:30（10 秒）的 ISO 8601 日期-时间格式。

  • _data_source_id— 包含文档的数据源的标识符。

  • _document_body— 文档的内容。

  • _document_id— 文档的唯一标识符。

  • _document_title— 文档的标题。

  • _excerpt_page_number— 显示文档摘录的 PDF 文件中的页码。如果您的索引是在 2020 年 9 月 8 日之前创建的，则必须重新索引您的文档，然后才能使用此属性。

  • _faq_id—如果这是常见问题解答问题和答案，则是它们的唯一标识符。

  • _file_type— 文档的文件类型，例如 pdf 或 doc。

  • _last_updated_at— 上次更新文档的日期和时间（ISO 8601 格式）。例如，2012-03-25T12:30:10+01:00 是中部欧洲时间 2012 年 3 月 25 日中午 12:30（10 秒）的 ISO 8601 日期-时间格式。

  • _source_uri— 文档可用的 URI。例如，公司网站上文档的 URI。

  • _version—文档特定版本的标识符。

  • _view_count— 查看文档的次数。

  • _language_code（字符串）-适用于文档的语言代码。如果不指定语言，默认为英语。有关支持的语言（包括其代码）的更多信息，请参阅添加英语以外语言的文档。

对于自定义字段，您可以DocumentMetadataConfigurationUpdates使用UpdateIndex API 创建这些字段，就像创建预留字段或内置字段时一样。您必须为自定义字段设置相应的数据类型。一旦创建了字段，就无法更改字段类型。

以下是您可以为自定义字段设置的类型：

  • 日期

  • 数字

  • 字符串

  • 字符串列表

您可以使用控制台或 UpdateIndexAPI 创建自定义字段。创建自定义字段后，将其映射到文档属性，就像使用保留字段一样。如果您使用 API 向索引添加了文档，则可以使用 BatchPutDocumentAPI 映射这些属性。对于从Amazon S3数据源索引的文档，您可以使用包含描述文档属性的 JSON 结构的元数据文件映射属性。对于使用允许字段映射的数据库或数据源索引的文档，您可以使用控制台或数据源配置映射属性。

使用 BatchPutDocument API 添加自定义属性或字段

当您使用BatchPutDocument API 向索引添加文档时，您可以将自定义字段或属性指定为其中的一部分Attributes。调用 API 时可以添加多个字段或属性。您最多可创建 500 个自定义字段或属性。以下示例是向文档添加 “部门” 的自定义字段或属性。

"Attributes": 
    {
        "Department": "HR",
        "_category": "Vacation policy"
    }

向Amazon S3数据源添加自定义属性或字段

当您使用 S3 存储桶作为索引的数据源时，您可以将元数据与伴随的元数据文件添加到文档中。您将元数据 JSON 文件放在与您的文档parallel 目录结构中。有关更多信息，请参阅Amazon S3文档元数据：

您可以在Attributes JSON 结构中指定自定义字段或属性。您最多可创建 500 个自定义字段或属性。例如，以下示例使用Attributes定义三个自定义字段或属性以及一个保留字段。

"Attributes": {
        "brand": "Amazon Basics",
        "price": 1595,
        "_category": "sports",
        "subcategories": ["outdoors", "electronics"]
    }
