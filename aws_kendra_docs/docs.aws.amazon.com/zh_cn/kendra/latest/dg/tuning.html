调整优优优优优优

Amazon Kendra查询会生成按其相关性排序的搜索结果。索引中的可搜索字段或属性都构成了此排名。

您可以通过相关性调整来修改字段或属性对搜索相关性的影响。可以在索引级别手动调整搜索相关性，即为索引设置优化配置，也可以在查询级别通过覆盖在索引级别设置的配置来调整搜索相关性。

当查询优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优优 您还可以指定存在匹配项时文档会获得多少提升。相关性调整不会Amazon Kendra导致在查询响应中包含文档，它只是Amazon Kendra用于确定文档相关性的因素之一。

您可以提升索引中的特定字段或属性，以赋予特定响应更高的重要性。例如，当有人搜索 “re: Invent 是什么时候？” 你可以提高 “_last_update_at” 字段中文档新鲜度的相关性。或者，在研究报告索引中，您可以提升 “来源” 字段中的特定数据源。

您还可以根据投票或浏览次数来提升文档，这在论坛和其他支持知识库中很常见。例如，您可以组合增强功能，以增强查看次数更长和更新的文档。

您可以使用Importance参数设置文档获得的提升量。越高Importance，字段或属性对文档相关性的提升就越大。在查询级别调整索引或调整时，以小幅增量增加Importance参数的值，直到获得所需的效果为止。要确定您是否在改进搜索结果，请执行搜索并将结果与之前的查询进行比较。

您可以指定日期、数字或字符串属性来调整索引或在查询级别进行调整。您StringList只能在索引级别调整该类型的字段或属性。每个字段或属性对何时提升结果都有特定的标准。

  • 日期字段或属性-日期字段有三个特定标准Duration、Freshness和RankOrder。

    • Duration设置提升应用到的时间段。例如，如果您将时间段设置为 86400 秒（即一天），则提升将在一天后开始减弱。重要性越高，提升效果下降的速度越快。

    • Freshness确定将文档应用于字段或属性时的最新时间。如果您申请Freshness创建日期或上次更新日期字段，则最近创建或上次更新的文档被视为比旧文档 “更新”。例如，如果文档 1 是在 11 月 14 日创建的，而文档 2 是在 11 月 5 日创建的，则文档 1 比文档 2 “新鲜度更高”。如果文档 1 是在 11 月 14 日更新的，而文档 2 是在 11 月 20 日更新的，则文档 2 比文档 1 “新鲜度更高”。文档越新，这种增强效果的应用就越多。您的索引只能有一个Freshness字词。

    • RankOrder按升序返回结果。如果您指定ASCENDING，则以后的日期优先。如果您指定DESCENDING，则较早的日期优先。

  • 数字字段或属性-对于数值字段或属性，您可以指定在确定字段或属性的相关性时Amazon Kendra应使用的等级顺序。如果您指定ASCENDING，则优先考虑较大的数字。如果您指定DESCENDING，则优先使用较小的数字。

  • 字符串字段或属性-对于字符串字段或属性，您可以创建字段的类别来为每个类别提供不同的提升。例如，如果您提升名为 “部门” 的字段或属性，则可以对 “HR” 中的文档和 “法律” 中的文档进行不同的提升。您可以增强该类型的字段或属性String。您只能在索引级别提升StringList字段。

索引级别的相关性调整

您可以使用控制台设置索引详细信息调整或 UpdateIndexAPI 在索引级别调整字段或属性的相关性。

以下示例将 “_last_updated_at” 字段设置为文档的Freshness字段。

"DocumentMetadataConfigurationUpdates" : [
    {
        "Name": "_last_updated_at",
        "Type": "DATE_VALUE",
        "Relevance": {
            "Freshness": TRUE,
            "Importance": 2
        }
    }
]

以下示例对 “部门” 字段中的不同类别应用不同的重要性。

"DocumentMetadataConfigurationUpdates" : [
    {
        "Name": "department",
        "Type": "STRING_VALUE",
        "Relevance": {
            "Importance": 2,
            "ValueImportanceMap": {
                "HR": 3,
                "Legal": 1
            }
        }
    }
]

查询级别的相关性调整

您可以使用查询 API 在查询级别调整字段或属性的相关性。

控制台不支持查询级别的相关性调整。

查询级别的调整可以加快测试相关性调整的过程，因为您无需手动更新每次测试的索引中的调整配置。您可以通过在查询中传递调整配置来调整文档的相关性。然后，您可以看到从不同的配置中获得的不同结果。查询中传递的配置会覆盖在索引级别设置的配置。

以下示例替代了应用于 “部门” 字段和在索引级别设置的每个部门类别的重要性，如上例所示。当用户输入搜索查询时，“部门” 字段的重要性相当高，法律部门比人力资源部门更重要。

"DocumentRelevanceOverrideConfigurations" : [
    {
        "Name": "department",
        "Type": "STRING_VALUE",
        "Relevance": {
            "Importance": 2,
            "ValueImportanceMap": {
                "HR": 2,
                "Legal": 8
            }
        }
    }
]
