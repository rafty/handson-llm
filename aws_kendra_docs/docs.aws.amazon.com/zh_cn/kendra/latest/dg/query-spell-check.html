查询拼写检查程序

Amazon Kendra拼写检查器建议对查询进行拼写更正。这可以帮助您将零搜索结果的出现次数降至最低，并返回相关结果。您的用户从拼写错误的查询中收到的搜索结果可能为零，没有匹配的结果或没有返回的文档。或者，您的用户可能会从拼写错误的查询中收到不相关的搜索结果。

拼写检查器旨在根据索引文档中出现的单词以及更正后的单词与拼写错误的单词的接近程度，对拼写错误的单词提出更正建议。例如，如果 “报表” 一词出现在您的索引文档中，则这可能与查询 “年终财务报表” 中拼写错误的 “报表” 一词非常相似。

拼写检查器返回用于替换原始查询文本中拼写错误的单词的预期或更正的单词。例如，“部署 kendre 搜索” 可能会返回 “部署 Kendra 搜索”。您也可以使用 API 中提供的偏移位置在前端应用程序的查询中突出显示或斜体显示返回的更正单词。在控制台中，默认情况下，更正后的单词以突出显示或斜体显示。例如，“部署 Kendra 搜索”。

对于索引文档中出现的特定业务或专业术语，Spell Checker 不会将这些术语误解为查询中的拼写错误。例如，“亚马逊 mace” 未更正为 “亚马逊 mace”。

对于带连字符的单词，例如 “年底”，Spell Checker 会将这些单词视为单个单词，以建议对这些单词进行更正。例如，“年底” 的建议更正可能是 “年底”。

对于DOCUMENT和QUESTION_ANSWER查询的响应类型，拼写检查器会根据文档正文中的单词建议更正拼写错误的单词。在建议与拼写错误的单词非常匹配的更正时，文档正文比标题更可靠。对于ANSWER查询响应类型，Spell Checker 会根据索引中默认问答文档中的单词提出更正建议。

你可以使用该SpellCorrectionConfiguration对象激活拼写检查器。您设置IncludeQuerySpellCheckSuggestions为TRUE。默认情况下，拼写检查器在控制台中处于激活状态。默认情况下，它内置在控制台中。

拼写检查器还可以为多种语言的查询提供拼写更正建议，而不仅仅是英语。有关拼写检查器支持的语言列表，请参阅Amazon Kendra支持的语言。

使用具有默认限制的查询拼写检查器

拼写检查器的设计有某些默认值或限制。以下是激活拼写更正建议时适用的当前限制列表。

  • 对于少于三个字符或长度超过 30 个字符的单词，无法返回建议的拼写更正。要允许超过 30 个字符或少于三个字符，请联系Su pport 人员。

  • 建议的拼写更正不能根据用户访问控制或您的访问控制列表限制用户上下文筛选建议。拼写更正基于索引文档中的所有单词，无论这些单词是否仅限于某些用户。如果您想避免某些单词出现在建议的查询拼写更正中，请不要激活SpellCorrectionConfiguration。

  • 对于包含数字的单词，无法返回建议的拼写更正。例如，'how 2 不是 br8k ubun2 '。

  • 建议的拼写更正不能使用索引文档中未出现的单词。

  • 建议的拼写更正不能使用索引文档中出现频率低于 0.01% 的单词。要更改 0.01% 阈值，请联系Su pport 部门。
