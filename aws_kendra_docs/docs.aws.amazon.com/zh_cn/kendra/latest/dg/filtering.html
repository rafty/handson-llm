筛选和分面搜索

您可以使用筛选器改善查询 API 的搜索结果或响应。过滤器将响应中的文档限制为直接应用于查询的文档。要创建分面搜索建议，请使用布尔逻辑从响应或不符合特定条件的文档中筛选出特定的文档属性。您可以使用Query API 中的Facets参数指定分面。

要搜索您编入索引的Amazon Kendra文档，请使用 AMAZON。Amazon Lex KendraSearchIntent。有关配置的示例Amazon Lex，请参阅Amazon Kendra为Amazon Kendra索引创建 FAQ Bot。您也可以使用为响应提供过滤器AttributeFilter。这是配置时 JSON 中的查询过滤器AMAZON.KendraSearchIntent。要在控制台中配置搜索意图时提供属性过滤器，请转到意图编辑器并选择Amazon Kendra查询以提供 JSON 中的查询过滤器。有关的更多信息AMAZON.KendraSearchIntent，请参阅Amazon Lex文档指南。

分面

分面是一组搜索结果的作用域视图。例如，您可以为世界各地的城市提供搜索结果，在这些城市中，文档按与之关联的特定城市进行筛选。或者，您可以创建分面来显示特定作者的结果。

您可以使用与文档关联的文档属性或元数据字段作为分面，这样您的用户就可以在该分面内按类别或值进行搜索。您还可以在搜索结果中显示嵌套方面，这样您的用户不仅可以按类别或字段进行搜索，还可以按子类别或子字段进行搜索。

以下示例说明如何获取 “城市” 自定义属性的分面信息。

response=kendra.query(
        QueryText = query,
        IndexId = index,
        Facets = [
            {
                "DocumentAttributeKey" : "City"
            }
        ]
        )

您可以使用嵌套分面进一步缩小搜索范围。例如，文档属性或分面 “城市” 包含一个名为 “西雅图” 的值。此外，文档属性或方面 “CityRegion” 包括分配给 “西雅图” 的文档的值 “北” 和 “南”。您可以在搜索结果中显示嵌套分面及其计数，这样不仅可以按城市搜索文档，还可以按城市内的区域搜索文档。

请注意，嵌套方面可能会影响查询延迟。一般规则是，您使用的嵌套方面越多，对延迟的潜在影响就越大。影响延迟的其他因素包括索引文档的平均大小、索引的大小、高度复杂的查询以及Amazon Kendra索引的总体负载。

以下示例显示如何获取 “CityRegion” 自定义属性的分面信息，将其作为 “城市” 中的嵌套分面。

response=kendra.query(
        QueryText = query,
        IndexId = index,
        Facets = [
            {
                "DocumentAttributeKey" : "City",
                "Facets": [
                    {
                        "DocumentAttributeKey" : "CityRegion"
                    }
                ]
            }
        ]
        )

Facet 信息，例如文档数量，在FacetResults响应数组中返回。您可以使用内容在应用程序中显示分面搜索建议。例如，如果文档属性 “城市” 包含可应用搜索的城市，则使用该信息显示城市搜索列表。用户可以选择一个城市来筛选搜索结果。要进行分面搜索，请调用 Query API 并使用所选文档属性筛选结果。

对于一个查询，每个分面最多可以显示 10 个分面值，并且一个分面中只能显示一个嵌套分面值。如果您想提高这些限制，请联系Su pport 部门。如果要将每个分面的分面值数限制为小于 10，则可以在Facet对象中指定此值。

使用文档属性筛选搜索结果

默认情况下，Query返回所有搜索结果。要筛选响应，您可以对文档属性执行逻辑操作。例如，如果您只需要特定城市的文档，则可以筛选 “城市” 和 “州” 自定义文档属性。您可以使用AttributeFilter在提供的过滤器上创建布尔运算。

大多数属性可用于筛选所有响应类型的响应。但是，该_excerpt_page_number属性仅适用于筛选ANSWER响应时的响应类型。

以下示例显示如何通过筛选特定城市（西雅图）和华盛顿州（州）来执行逻辑 AND 运算。

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {'AndAllFilters': 
            [ 
                {"EqualsTo": {"Key": "City","Value": {"StringValue": "Seattle"}}},
                {"EqualsTo": {"Key": "State","Value": {"StringValue": "Washington"}}}
            ]
            }
        )

以下示例说明在任何Fileformat、Author或SourceURI键与指定值匹配时如何执行逻辑 OR 运算。

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {'OrAllFilters': 
            [ 
                {"EqualsTo": {"Key": "Fileformat","Value": {"StringValue": "AUTO_DETECT"}}},
                {"EqualsTo": {"Key": "Author","Value": {"StringValue": "Ana Carolina"}}},
                {"EqualsTo": {"Key": "SourceURI","Value": {"StringValue": "https://aws.amazonaws.com/234234242342"}}}
            ]
            }
        )

对于StringList字段，使用ContainsAny或ContainsAll属性筛选器返回带有指定字符串的文档。以下示例显示如何返回其Locations自定义属性中值为 “西雅图” 或 “波特兰” 的所有文档。

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {
                "ContainsAny": { "Key": "Locations", "Value": { "StringListValue": [ "Seattle", "Portland"] }}
            }
        )

在搜索结果中筛选每个文档的属性

Amazon Kendra返回搜索结果中每个文档的文档属性。您可以筛选要包含在响应中的某些文档属性作为搜索结果的一部分。默认情况下，分配给文档的所有文档属性都将在响应中返回。

在以下示例中，_author文档的响应中仅包含_source_uri和文档属性。

response=kendra.query(
        QueryText = query,
        IndexId = index,
        RequestedDocumentAttributes = ["_source_uri", "_author"]
        )
