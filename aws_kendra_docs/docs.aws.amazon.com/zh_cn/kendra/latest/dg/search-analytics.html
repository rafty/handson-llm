通过搜索分析获得见解

您可以使用 seAmazon Kendra arch Analytics 来深入了解您的搜索应用程序如何成功或失败地帮助用户查找信息。

Amazon KendraAnalytics 提供用户如何与搜索应用程序交互以及搜索应用程序配置的有效性的快照。您可以使用 GetSnapshotsAPI 或在控制台的导航面板上选择 Analytics 来查看指标数据。

您可以在自己的自定义仪表板GetSnapshots上呈现生成的数据。或者，您可以使用控制台中提供的指标仪表板，其中包括可视化图表。使用可视化仪表板，您可以查看用户行为随时间推移的趋势或模式，或者发现搜索应用程序配置中的问题。例如，如果折线图显示每天的查询次数恒定且持续增长，则可能表明采用率和使用率都有所提高。另一方面，突然下降可能表明存在必须调查的问题。

您可以使用这些指标在不同的数据点之间建立联系，以解决用户如何查询信息或发现商机的问题。例如，文档 “人工智能是如何工作的？” 是搜索结果中点击次数最多的文档，搜索量最高的查询是 “机器学习是如何工作的？”。这会告知您用户使用的首选术语和语言。您可以将这些术语集成到文档中，或者为这些术语使用自定义同义词，以使您的用户更容易搜索您的文档。

搜索指标

有 10 个指标可用于分析您的搜索应用程序的性能或用户正在搜索哪些信息。要检索指标数据，请在调用时指定要检索的指标数据的字符串名称GetSnapshots。

您还必须提供时间间隔或时间窗口才能查看指标数据。您可以在以下时间窗内查看数据：

  • THIS_WEEK：当前周，从星期日开始，到当前日期的前一天结束。

  • ONE_WEEK_AGO：前一周，从星期日开始，到下一个星期六结束。

  • TWO_WEEKS_AGO：前一周的前一周，从星期日开始，到下一个星期六结束。

  • THIS_MONTH：当前月份，从该月的第一天开始，到当前日期的前一天结束。

  • ONE_MONTH_AGO：上个月，从当月的第一天开始，到当月的最后天结束。

  • TWO_MONTHS_AGO：前一个月的前月，从当月的第一天开始，到当月的最后天结束。

在控制台中，支持的时间窗口为 “本周”、“上周”、“本月”、“上个月”。

点击率

在搜索结果中，导致文档点击的查询比例。这可以帮助您了解您的搜索应用程序配置是否有助于您的用户查找与其查询相关的信息。对于返回即时答案的查询，用户可能无需点击进入文档即可获得更多信息。有关更多信息，请参阅 即时答案率。您必须致电SubmitFeedback以确保收集点击反馈。

要使用GetSnapshots API 检索点击率数据，请指定metricType asAGG_QUERY_DOC_METRICS。您也可以在控制台中通过选择导航面板上的分析来查看此指标。

零点击率

导致搜索结果中点击次数为零的查询比例。这可以帮助您了解提供不相关搜索结果的内容中的空白。对于返回即时答案的查询，用户可能无需点击进入文档即可获得更多信息。有关更多信息，请参阅 即时答案率。此外，您的搜索设置（例如调整配置）可能会影响搜索结果中文档的返回方式。

要使用GetSnapshots API 以零点击率检索数据，请指定metricType asAGG_QUERY_DOC_METRICS。您也可以在控制台中通过选择导航面板上的分析来查看此指标。

搜索结果率为零

导致搜索结果为零的查询比例。这可以帮助您了解内容中的空白，但不提供相关的搜索结果。

要使用GetSnapshots API 检索搜索结果率为零的数据，请指定metricType asAGG_QUERY_DOC_METRICS。您也可以在控制台中通过选择导航面板上的分析来查看此指标。

即时答案率

返回即时答案或常见问题解答的查询比例。这可以帮助您了解即时答案在提供信息方面的作用。

要使用GetSnapshots API 检索有关即时应答率的数据，请指定metricType asAGG_QUERY_DOC_METRICS。您也可以在控制台中通过选择导航面板上的分析来查看此指标。

主要查询

您的用户搜索的前 100 个查询。这可以帮助您了解哪些查询最受欢迎，以及您的用户最感兴趣的信息类型。

指标包括搜索查询的次数、文档的点击量比例、文档未点击的比例、查询搜索结果中的平均点击深度、查询即时答案的比例以及前 10 次的平均置信度查询的搜索结果。

要使用GetSnapshots API 检索热门查询的数据，请指定metricType asQUERIES_BY_COUNT。您也可以在控制台中查看此指标，方法是在控制台的导航面板上选择 “分析”，然后选择 “查询列表” 下的 “热门查询”。

点击次数为零的热门查询

导致搜索结果点击次数为零的前 100 个查询。这可以帮助您了解内容中的任何空白，即缺少与某些查询相关的文档，或者您的搜索应用程序配置返回了不相关的搜索结果。对于返回即时答案的查询，用户可能无需点击进入文档即可获得更多信息。有关更多信息，请参阅 即时答案率。

指标包括查询导致零点击的次数、查询零点击的比例、查询的即时答案比例以及查询前 10 个搜索结果的平均置信度。

要使用GetSnapshots API 以零点击次数检索热门查询的数据，请指定metricType asQUERIES_BY_ZERO_CLICK_RATE。您也可以在控制台中查看此指标，方法是在控制台的导航面板上选择 Analytics，然后在 “查询列表” 下选择 “热门零点击查询”。

搜索结果为零的热门查询

导致搜索结果为零的前 100 个查询。这可以帮助您了解内容中的任何空白，即没有与某些查询相关的文档。或者，您的用户可能会使用可能导致没有搜索结果的专门术语进行查询，这会提示您创建自定义同义词来处理此问题。

指标包括查询导致零搜索结果的次数、查询的零搜索结果的比例，以及查询与所有查询相比的搜索次数比例。

要使用GetSnapshots API 检索搜索结果为零的热门查询的数据，请指定metricType asQUERIES_BY_ZERO_RESULT_RATE。您也可以在控制台中查看此指标，方法是在控制台的导航面板上选择 Analytics，然后在 “查询列表” 下选择 “前零结果查询”。

点击次数最多的文档

搜索结果中点击次数最多的前 100 个文档。这有助于您在用户查询信息时了解哪些文档或搜索结果与用户最相关。

指标包括文档被点击的次数、文档从用户那里获得的点赞次数（竖起大拇指）、文档从用户那里收到的不喜欢次数（竖起大拇指）。

要使用GetSnapshots API 检索点击次数最多的文档中的数据，请指定metricType asDOCS_BY_CLICK_COUNT。您也可以在控制台中查看此指标，方法是在控制台的导航面板上选择 “分析”，然后选择 “查询列表” 下的 “点击次数最多的文档”。

查询总数

您的用户搜索的查询总数。这可以帮助您了解用户对搜索应用程序的参与程度。

要使用GetSnapshots API 检索查询总数的数据，请指定metricType asAGG_QUERY_DOC_METRICS。您也可以在控制台中通过选择导航面板上的分析来查看此指标。

文档总数

索引中的文档总数。这可以帮助您将索引的大小与查询总数进行比较，以检查是否有适合查询量的文档数量。

要使用GetSnapshots API 检索所有文档的数据，请指定metricType asAGG_QUERY_DOC_METRICS。您也可以在控制台中通过选择导航面板上的分析来查看此指标。

检索指标数据的示例

以下代码是检索上个月热门查询数据的示例。

Console
检索上个月的热门查询

  1. 在左侧导航窗格的 “索引” 下，选择您的索引，然后选择 “分析”。

  2. 在 Analytics 页面上，选择 “本周” 按钮，将检索数据的时间窗口更改为上个月。

  3. 在 “分析” 页面的 “查询列表” 下，选择 “热门查询”。

CLI

检索上个月的热门查询

aws kendra get-snapshots \
--index-id index-id \
--interval "ONE_MONTH_AGO" \
--metric-type "QUERIES_BY_COUNT"
Python

检索上个月的热门查询

import boto3

kendra = boto3.client("kendra")

index_id = "index-id"
interval = "ONE_MONTH_AGO"
metric_type = "QUERIES_BY_COUNT"

snapshots_response = kendra.get_snapshots(
    IndexId = index_id,
    Interval = interval,
    MetricType = metric_type
)

print("Top queries data: " + snapshots_response["snapshotsData"])
Java

检索上个月的热门查询

package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.GetSnapshotsRequest;
import software.amazon.awssdk.services.kendra.model.GetSnapshotsResponse;

public class TopQueriesExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String indexId = "indexID";
        String interval = "ONE_MONTH_AGO";
        String metricType = "QUERIES_BY_COUNT";

        GetSnapshotsRequest getSnapshotsRequest = GetSnapshotsRequest
            .builder()
            .indexId(indexId)
            .interval(interval)
            .metricType(metricType)
            .build();

        GetSnapshotsResponse getSnapshotsResponse = kendra.getSnapshots(GetSnapshotsRequest);

        System.out.println(String.format("Top queries data: ", getSnapshotsResponse.snapshotsData()))

从指标到切实可行的见解

可操作的见解是从原始数据中提取的有意义的信息，用于指导您的行动或决策。要从指标中提取含义并利用它们来获得切实可行的见解，重要的是不仅要孤立地看待指标，还要在指标之间建立联系。

例如，点击次数为零的热门查询是 “当前哪些区域可用？”。但是，它的即时答案率也为100％。这表明您的用户无需点击提供可用区域信息的搜索结果或文档，即可获得该问题的答案。如果你只看零点击次数，你就无法了解完整情况，并且可能得出关于搜索应用程序配置成功处理此查询的错误结论。

另一个切实可行的见解的例子是发现商机。企业通常通过分析搜索指标来寻找机会来增加客户。点击次数最多的文档是 “可用区域”。除此之外，大多数搜索量最高的查询都与大洋洲地区的产品供应问题有关，即时答案率为100％，点击率很高，可以获得有关可用区域的更多信息。这表明该地区对您的产品或服务有兴趣和需求。

可视化和报告搜索分析

有五个指标包括趋势数据，供您可视化和查找一段时间内的趋势或模式。如果您使用控制台，则会提供趋势数据的图表。如果您使用 API，则可以检索趋势数据来创建自己的图表或可视化效果。控制台中的大多数图表都会在您选择的时间窗口内绘制每日数据点。

控制台提供指标控制面板，您可以在其中选择您感兴趣查看的图表和热门列表。您可以在 Analyt ics 主页上选择 “导出”，以 CSV 格式导出控制面板上显示的指标。您可以将这些报告包含在您的业务文档或演示文稿中。

您可以可视化以下指标：

查询总数图

每天发出的查询数量的折线图。该图表可帮助您可视化每日用户参与的模式。一些例子包括用户参与度的稳步增加或下降，或者由于搜索应用程序崩溃或网站出现问题而导致的查询量急剧下降至0。

如果您使用 API，则可以通过指定来检索这些数据TREND_QUERY_DOC_METRICS。您可以使用数据创建自己的图表，也可以使用控制台中提供的图表。

点击率图表

每日点击量比例的折线图。该图表可帮助您以每日点击率对模式进行可视化。一些例子包括点击率的稳步提高或降低，或者即时答案的降低可能会影响点击率的增加。

如果您使用 API，则可以通过指定来检索这些数据TREND_QUERY_DOC_METRICS。您可以使用数据创建自己的图表，也可以使用控制台中提供的图表。

零点击率图表

每天零点击比例的折线图。该图表可帮助您以每日零点击率对模式进行可视化。一些例子包括零点击率的稳步提高或降低，或者即时答案的增加可能会影响零点击率的增加。

如果您使用 API，则可以通过指定来检索这些数据TREND_QUERY_DOC_METRICS。您可以使用数据创建自己的图表，也可以使用控制台中提供的图表。

零搜索结果率图

每天搜索结果为零的比例的折线图。该图表可帮助您可视化每日搜索结果率为零的模式。一些例子包括零搜索结果率稳步上升或下降，或者索引中文档数量的急剧减少可能会影响零搜索结果的增加。

如果您使用 API，则可以通过指定来检索这些数据TREND_QUERY_DOC_METRICS。您可以使用数据创建自己的图表，也可以使用控制台中提供的图表。

即时答案率图

返回即时答案或常见问题解答的查询比例的折线图。该图表可帮助您可视化每日即时答案率的模式。一些例子包括问答类查询的稳步增加或减少，或者点击次数的减少可能会影响即时答案的增加。

如果您使用 API，则可以通过指定来检索这些数据TREND_QUERY_DOC_METRICS。您可以使用数据创建自己的图表，也可以使用控制台中提供的图表。
