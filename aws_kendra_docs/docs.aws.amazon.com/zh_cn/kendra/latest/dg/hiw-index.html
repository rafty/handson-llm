索引

索引包含文档的内容，其结构使文档可搜索。向索引中添加文档的方式取决于您存储文档的方式。

  • 如果您将文档存储在某种存储库中，例如Amazon S3存储段或 Microsoft SharePoint 站点，则使用数据源连接器对存储库中的文档进行索引。

  • 如果您不将文档存储在存储库中，则可以使用 BatchPutDocumentAPI 直接为文档编制索引。

  • 对于必须存储在Amazon Kendra (Amazon S3) 存储桶中的常见问题和答案，您可以从存储桶上传这些问题和答案

您可以使用Amazon Kendra控制台AWS CLI、或AWS SDK 创建索引。有关可编索引的文档类型的信息，请参阅 文档类型。

使用Amazon Kendra 内置文档字段

借助 UpdateIndex API，您可以使用DocumentMetadataConfigurationUpdates并指定保留字段名称来创建保留字段或内置字段，以映射到您的等效文档属性或文档字段。您也可以通过这种方式创建自定义字段。如果您使用数据源连接器，则大多数都包含将数据源文档字段映射到Amazon Kendra索引字段的字段映射。创建字段时，可以配置该Search对象，将该字段设置为可显示、可分面、可搜索和可排序。您可以配置该Relevance对象以设置字段的相关性提升持续时间、新鲜度、重要性、排名顺序和重要性值。一旦创建了字段就无法更改字段类型。

Amazon Kendra有以下可供使用的保留或内置文档字段：

  • _authors—负责文档内容的一位或多位作者的列表。

  • _category— 将文档置于特定组中的类别。

  • _created_at— 以 ISO 8601 格式创建文档的日期和时间。例如，2012-03-25T12:30:10+01:00 是中部欧洲时间 2012 年 3 月 25 日中午 12:30（10 秒）的 ISO 8601 日期-时间格式。

  • _data_source_id— 包含文档的数据源的标识符。

  • _document_body— 文档的内容。

  • _document_id-文档的唯一标识符。

  • _document_title— 文档的标题。

  • _excerpt_page_number— 显示文档摘录的 PDF 文件中的页码。如果您的索引是在 2020 年 9 月 8 日之前创建的，则必须重新索引您的文档，然后才能使用此属性。

  • _faq_id—如果这是常见问题解答问题和答案，则是它们的唯一标识符。

  • _file_type— 文档的文件类型，例如 pdf 或 doc。

  • _last_updated_at— 上次更新文档的日期和时间，采用 ISO 8601 格式。例如，2012-03-25T12:30:10+01:00 是中部欧洲时间 2012 年 3 月 25 日中午 12:30（10 秒）的 ISO 8601 日期-时间格式。

  • _source_uri— 文档可用的 URI。例如，公司网站上文档的 URI。

  • _version— 文档特定版本的标识符。

  • _view_count— 查看文档的次数。

  • _language_code（字符串）-适用于文档的语言代码。如果您未指定语言，默认为。有关支持的语言（包括其代码）的更多信息，请参阅添加英语以外语言的文档。

对于自定义字段，您可以DocumentMetadataConfigurationUpdates使用UpdateIndex API 创建这些字段，就像创建预留字段或内置字段时一样。您必须为自定义字段设置相应的数据类型。一旦创建了字段就无法更改字段类型。

以下是您可以为自定义字段设置的类型：

  • 日期

  • 数字

  • 字符串

  • 字符串列表

您可以使用控制台或 UpdateIndexAPI 创建自定义字段。创建自定义字段后，将其映射到文档属性，就像使用保留字段一样。如果您使用 API 向索引添加了文档，则可以使用 BatchPutDocumentAPI 映射这些属性。对于从Amazon S3数据源索引的文档，您可以使用包含描述文档属性的 JSON 结构的元数据文件映射属性。对于使用允许字段映射的数据库或数据源索引的文档，您可以使用控制台或数据源配置映射属性。
