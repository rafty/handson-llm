将常见问题（FAQ）直接添加到索引

您可以使用控制台或CreateFaq API 将常见问题 (FAQ) 直接添加到索引中。将常见问题添加到索引是一项异步操作。您将常见问题解答的数据放入存储在Amazon Simple Storage Service存储桶中的文件中。您可以使用 CSV 或 JSON 文件作为常见问题解答的输入：

  • Basic CSV — 一个 CSV 文件，其中每行包含一个问题、答案和一个包含有关答案的更多信息的可选 URL。

  • 自定义 CSV — 一个包含问题、答案和标题的 CSV 文件，标题用于定义可用于分面、显示或排序常见问题解答的自定义属性。您还可以定义访问控制属性，将常见问题解答的响应限制为特定用户和群组。

  • JSON — 包含问题和答案的 JSON 文件。可选，它还可以包含自定义和访问控制属性。您可以定义属性以对常见问题回复进行分面、显示和排序。或者，您可以使用访问控制属性来限制对特定用户和群组的常见问题解答响应。

例如，以下是一个基本的 CSV 文件，它提供了有关美国华盛顿州斯波坎和美国密苏里州山景城免费诊所问题的答案。

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

当您使用自定义 CSV 或 JSON 文件进行输入时，您可以为常见问题声明自定义属性。例如，您可以创建自定义属性，将每个常见问题解答问题分配一个部门。在回复中返回常见问题解答时，您可以使用部门作为一个方面来缩小搜索范围。

自定义属性必须映射到索引字段。您可以使用内置字段或指定自定义索引字段。在控制台中，您可以使用 Facet 定义页面创建索引字段。使用 API 时，必须首先使用 APUpdateIndex I 创建索引字段。

常见问题文件中的属性类型必须与关联索引字段的类型相匹配。例如，内置_authors字段是STRING_LIST类型字段。因此，您必须在常见问题文件中以字符串列表的形式为该_authors字段提供值。您可以使用控制台中的 Facet 定义页面或使用DescribeIndex API 检查索引字段的类型。

创建映射到自定义属性的索引字段时，可以将其标记为可显示、可分面或可排序。无法将自定义属性设置为可搜索。

除了自定义属性外，您还可以在自定义 CSV 或 JSON 文件中使用以下内置属性：

  • _authors（字符串列表）-常见问题解答的作者列表。

  • _category（字符串）-将常见问题解答的答案与其他类似文档分组的类别。

  • _created_at（ISO 8601编码的字符串）— 创建常见问题解答问题的日期和时间。日期和时间必须格式化为 ISO 8601 编码的字符串。

    如果需要，您还可以将时区纳入到 ISO 8601 日期-时间格式中。例如，中部欧洲时区的 ISO 8601日期-时间格式。中部欧洲时区的 ISO 8601日期-时间格式。2012-03-25T12

  • _last_updated_at（ISO 8601 编码字符串）-常见问题解答问题的更新日期和时间。日期和时间必须格式化为 ISO 8601 编码的字符串。

    例如，中部欧洲时区的 ISO 8601日期-时间格式。中部欧洲时区的 ISO 8601日期-时间格式。2012-03-25T12

  • _source_uri（字符串）— 包含有关常见问题答案的更多信息的文档的 URL。

  • _version（字符串）-常见问题解答问题的版本。

  • _view_count（长）-常见问题解答问题在搜索结果中被查看的次数。

基本的 CV 文件

如果要使用简单的结构来处理常见问题解答，请使用基本的 CSV 文件。在基本 CSV 文件中，每行都有两三个字段：问题、答案和指向包含更多信息的文档的可选来源 URI。

文件内容必须遵循逗号分隔值 (CSV) 文件的 RFC 4180 通用格式和 MIME 类型。

以下是基本 CSV 格式的常见问题解答文件。

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

自定义 CV 文件

当您想要向常见问题解答问题添加自定义属性时，请使用自定义 CSV 文件。对于自定义 CSV 文件，您可以使用 CSV 文件中的标题行来定义其他属性。

CV 文件必须包含以下两个必需的属性：

  • _question—常见问题

  • _answer—常见问题的答案

您的文件可以包含内置和自定义属性。以下是自定义 CV 文件的示例。

_question,_answer,_last_updated_at,custom_string
How many free clinics are in Spokane WA?, 13, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services
How many free clinics are there in Mountain View Missouri?, 7, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services

自定义属性文件的内容必须遵循逗号分隔值 (CSV) 文件的 RFC 4180 通用格式和 MIME 类型。

自定义属性有四种类型：

  • 日期-ISO 8601:ISO 8601 编码的日期和时间值。

    例如，中部欧洲时区的 ISO 8601日期-时间格式。中部欧洲时区的 ISO 8601日期-时间格式。2012-03-25T12

  • 长整数-数字，例如1234。

  • 字符串-字符串值。如果您的字符串包含逗号，请将整个值括在双引号 (“) 中（例如，"custom attribute, and more"）。

  • 字符串列表—字符串值的列表。使用逗号分隔的列表列出值，该列表以逗号分隔，并用引号（例如，"item1, item2, item3"）。如果列表仅包含单个条目，则可以省略引号（例如item1）。

自定义 CSV 文件可以包含用户上下文字段。您可以使用这些字段将常见问题解答的访问权限限制为某些用户和群组。要根据用户上下文进行筛选，用户必须在查询中提供用户和组信息。否则，将返回所有相关的常见问题解答。有关更多信息，请参阅根据用户上下文进行筛选：

常见问题有四个用户上下文过滤器：

  • _acl_user_allow—允许列表中的用户可以在查询响应中看到常见问题解答。常见问题解答不会返回给其他用户。

  • _acl_user_deny—拒绝列表中的用户无法在查询响应中看到常见问题解答。如果常见问题与查询相关，则会将其返回给所有其他用户。

  • _acl_group_allow—属于允许组的用户可以在查询响应中看到常见问题解答。常见问题解答不会返回给属于其他群组的用户。

  • _acl_group_deny—属于被拒绝群组的用户无法在查询响应中看到常见问题解答。如果常见问题与查询相关，则会将其返回给其他群组。

在用引号括住的以逗号分隔的列表中提供允许和拒绝列表的值（例如，"user1,user2,user3"）。您可以在允许列表或拒绝列表中包含用户或群组，但不能同时包含两者。如果将用户或组纳入到两者中，您将收到一条错误。

以下是包含用户上下文信息的自定义 CV 文件的示例。

_question, _answer, _acl_user_allow, _acl_user_deny, _acl_group_allow, _acl_group_deny
How many free clinics are in Spokane WA?, 13, userID4565, "userID6201, userID7552", groupBasicRate, "groupBasicPlusRate,groupPremiumRate"

JSON 文件

您可以使用 JSON 文件为索引提供问题、答案和属性。您可以将任何内置属性和自定义属性添加到常见问题解答。这是 JSON 文件的架构。

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": string,
            "Answer": string,
            "Attributes": {
                string: object
                additional attributes
            },
            "AccessControlList": [
               {
                   "Name": string,
                   "Type": enum( "GROUP" | "USER" ),
                   "Access": enum( "ALLOW" | "DENY" )
               },
               additional user context
            ]
        },
        additional FAQ documents
    ]
}

此示例 JSON 文件显示了两个常见问题解答文档。其中一个只有必填的问题和答案。另一个还具有其他属性和用户上下文信息。

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": "How many free clinics are in Spokane WA?",
            "Answer": "13"
        },
        {
            "Question": "How many free clinics are there in Mountain View Missouri?",
            "Answer": "7",
            "Attributes": {
                "_source_uri": "https://www.freeclinics.com",
                "_category": "Charitable Clinics"
            },
            "AccessControlList": [
               {
                   "Name": "user@amazon.com",
                   "Type": "USER",
                   "Access": "ALLOW"
               },
               {
                   "Name": "Admin",
                   "Type": "GROUP",
                   "Access": "ALLOW"
               }
            ]
        }
    ]
}

JON 文件的自定义属性有四种类型：

  • 日期-采用 ISO 8601 编码的日期和时间值的 JSON 字符串值。

    请务必将时区纳入到 ISO 8601日期-时间格式中。例如，中部欧洲时区的 ISO 8601日期-时间格式。中部欧洲时区的 ISO 8601日期-时间格式。2012-03-25T12

  • 长-JSON 数字值，例如1234。

  • 字符串-JSON 字符串值（例如"custom attribute"）。

  • 字符串列表-字符串值的 JSON 数组（例如["item1,item2,item3"]）。

除了内置和自定义属性外，您还可以在 JSON 文件中为常见问题提供用户上下文信息。您可以提供用户上下文信息，根据用户和群组限制对常见问题解答内容的访问权限。您可以在允许列表或拒绝列表中包含用户或群组，但不能同时包含两者。如果将用户或组纳入到两者中，您将收到一条错误。要根据用户上下文进行筛选，您必须在查询中提供用户和组信息。否则，将返回所有相关的常见问题解答。有关更多信息，请参阅根据用户上下文进行筛选：

此 JSON 示例将用户上下文添加到常见问题解答。

"AccessControlList": [
                {
                    "Name": "group or user name",
                    "Type": "GROUP | USER",
                    "Access": "ALLOW | DENY"
                },
                additional user context
            ]

使用您的常见问题解答文件

将常见问题输入文件存储在 S3 存储桶中后，您可以使用控制台或CreateFaq API 将问题和答案放入索引中。如果要更新常见问题解答，请删除该常见问题并重新创建。您可以使用DeleteFaq API 删除常见问题解答。

您必须提供一个可以访问包含您的源文件的 S3 存储桶的IAM角色。您可以在控制台或RoleArn参数中指定角色。以下是将 FAQ 文件添加到索引的程序示例。

Python
import boto3

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the IAM role ARN required to index documents in an S3 bucket
role_arn = "arn:aws:iam::${accountId}:role/${roleName}"

# Provide the S3 bucket path information to the FAQ file
faq_path = {
    "Bucket": "bucket-name",
    "Key": "FreeClinicsUSA.csv"
}

response = kendra.create_faq(
    S3Path =  faq_path,
    Name = "FreeClinicsUSA",
    IndexId = index_id,
    RoleArn = role_arn
)

print(response)
Java
package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.CreateFaqRequest;
import software.amazon.awssdk.services.kendra.model.CreateFaqResponse;
import software.amazon.awssdk.services.kendra.model.S3Path;

public class AddFaqExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String indexId = "yourIndexId";
        String roleArn = "your role for accessing S3 files";

        CreateFaqRequest createFaqRequest = CreateFaqRequest
            .builder()
            .indexId(indexId)
            .name("FreeClinicsUSA")
            .roleArn(roleArn)
            .s3Path(
                S3Path
                    .builder()
                    .bucket("an-aws-kendra-test-bucket")
                    .key("FreeClinicsUSA.csv")
                    .build())
            .build();

        CreateFaqResponse response = kendra.createFaq(createFaqRequest);

        System.out.println(String.format("The result of creating FAQ: %s", response));

    }
}

非英语语言的常见问题解答文件

您可以使用支持的语言索引常见问题解答。 Amazon Kendra如果不指定语言，则默认情况下，索引常见问题解答。您可以在调用CreateFaq操作时指定语言代码，也可以将常见问题解答的语言代码作为字段包含在常见问题元数据中。如果常见问题解答在元数据字段中指定的元数据中没有语言代码，则使用您在调用该CreateFAQ操作时指定的语言代码对常见问题进行索引。要在控制台中使用支持的语言为常见问题解答文档编制索引，请转到常见问题并选择添加常见问题解答。您可以从 “语言” 下拉列表中选择一种语言。
