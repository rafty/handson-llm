BatchGetDocumentStatus

返回使用 BatchPutDocumentAPI 提交的一个或多个文档的索引状态。

当您使用BatchPutDocument API 时，文档是异步索引的。您可以使用BatchGetDocumentStatus API 获取文档列表的当前状态，以便确定它们是否已成功编制索引。

您也可以使用BatchGetDocumentStatus API 来检查 BatchDeleteDocumentAPI 的状态。当文档从索引中删除时，Amazon Kendra 会NOT_FOUND以状态返回。

请求语法

{
   "DocumentInfoList": [ 
      { 
         "Attributes": [ 
            { 
               "Key": "string",
               "Value": { 
                  "DateValue": number,
                  "LongValue": number,
                  "StringListValue": [ "string" ],
                  "StringValue": "string"
               }
            }
         ],
         "DocumentId": "string"
      }
   ],
   "IndexId": "string"
}

请求参数

有关所有操作的通用参数的信息，请参阅通用参数。

请求接受采用 JSON 格式的以下数据。

DocumentInfoList

用于标识要获取其状态的文档的DocumentInfo对象列表。您可以通过文档 ID 和可选属性来识别文档。

类型：DocumentInfo 对象数组

数组成员：最少 1 项。最多 10 项。

必需：是

IndexId

要添加文档的索引的标识符。索引 ID 由 CreateIndexAPI 返回。

类型：字符串

长度限制：固定长度为 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必需：是

响应语法

{
   "DocumentStatusList": [ 
      { 
         "DocumentId": "string",
         "DocumentStatus": "string",
         "FailureCode": "string",
         "FailureReason": "string"
      }
   ],
   "Errors": [ 
      { 
         "DocumentId": "string",
         "ErrorCode": "string",
         "ErrorMessage": "string"
      }
   ]
}

响应元素

如果此操作成功，则该服务将会发送回 HTTP 200 响应。

服务以 JSON 格式返回的以下数据。

DocumentStatusList

文件状态。状态指示文档是正在等待索引、正在索引编制中、已完成索引还是索引失败。如果文档索引失败，则状态会提供原因。

类型：Status 对象数组

Errors

Amazon Kendra 无法获取其状态的文件清单。该列表包括文档的 ID 和找不到状态的原因。

类型：BatchGetDocumentStatusResponseError 对象数组
