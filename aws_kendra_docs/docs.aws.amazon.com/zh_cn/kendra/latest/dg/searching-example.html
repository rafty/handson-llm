查询索引

搜索索引时，Amazon Kendra使用您提供的有关文档的所有信息来确定与输入的搜索词最相关的文档。Amazon Kendra考虑的一些项目是：

  • 文档的文本。

  • 文档的标题。

  • 您标记为可搜索的自定义文本字段。

  • 应使用您指定的日期字段来确定文档的 “新鲜度”。

当从索引中选择了一组相关文档后，Amazon Kendra将根据您请求进行搜索的任何属性筛选器筛选响应。例如，如果您有一个名为 “部门” 的自定义属性，则可以筛选回复以仅返回名为 “法律” 的部门的文档。有关更多信息，请参阅自定义字段或属性。

找到相关文档，然后根据您设置的属性进行筛选后，Amazon Kendra返回结果。结果按为每个文档确Amazon Kendra定的相关性排序。结果是分页的，因此您可以一次向用户显示一个页面。

要搜索您编入索引的Amazon Kendra文档，请使用 AMAZON。Amazon Lex KendraSearchIntent。有关配置的示例Amazon Lex，请参阅Amazon Kendra为Amazon Kendra索引创建 FAQ Bot。

下列 Python 示例说明如何使用查询 API 搜索索引。该示例确定搜索结果的类型（答案、文档、问题/答案）并显示答案文本。

有关查询响应的信息，请参阅查询响应和响应类型。

搜索索引（控制台）

您可以使用Amazon Kendra控制台搜索和测试您的索引。您可以进行查询并查看结果。

使用控制台搜索索引

  1. 登录AWS Management Console并打开Amazon Kendra主机，网址为 http://console.aws.amazon.com/kendra/。

  2. 在导航窗格中，选择索引。

  3. 选择您的索引。

  4. 在导航菜单中，选择搜索索引的选项。

  5. 在文本框中输入查询，然后按 Enter。

  6. Amazon Kendra返回搜索结果。

搜索索引 (SDK)

使用 Python 或 Java 搜索索引

  • 以下示例搜索索引。将的值更改query为搜索查询，index_id或更改indexId为要搜索的索引的索引标识符。

    Python
    import boto3
import pprint

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the query text
query = "search-string"

response = kendra.query(
        QueryText = query,
        IndexId = index_id)

print("\nSearch results for query: " + query + "\n")        

for query_result in response["ResultItems"]:

    print("-------------------")
    print("Type: " + str(query_result["Type"]))
        
    if query_result["Type"]=="ANSWER" or query_result["Type"]=="QUESTION_ANSWER":
        answer_text = query_result["DocumentExcerpt"]["Text"]
        print(answer_text)

    if query_result["Type"]=="DOCUMENT":
        if "DocumentTitle" in query_result:
            document_title = query_result["DocumentTitle"]["Text"]
            print("Title: " + document_title)
        document_text = query_result["DocumentExcerpt"]["Text"]
        print(document_text)

    print("------------------\n\n")
    Java
    package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.QueryRequest;
import software.amazon.awssdk.services.kendra.model.QueryResponse;
import software.amazon.awssdk.services.kendra.model.QueryResultItem;

public class SearchIndexExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String query = "some queries";
        String indexId = "anIndexId";

        QueryRequest queryRequest = QueryRequest
            .builder()
            .queryText(query)
            .indexId(indexId)
            .build();

        QueryResponse queryResponse = kendra.query(queryRequest);

        System.out.println(String.format("\nSearch results for query: %s", query));
        for(QueryResultItem item: queryResponse.resultItems()) {
            System.out.println("----------------------");
            System.out.println(String.format("Type: %s", item.type()));

            switch(item.type()) {
                case QUESTION_ANSWER:
                case ANSWER:
                    String answerText = item.documentExcerpt().text();
                    System.out.println(answerText);
                    break;
                case DOCUMENT:
                    String documentTitle = item.documentTitle().text();
                    System.out.println(String.format("Title: %s", documentTitle));
                    String documentExcerpt = item.documentExcerpt().text();
                    System.out.println(String.format("Excerpt: %s", documentExcerpt));
                    break;
                default:
                    System.out.println(String.format("Unknown query result type: %s", item.type()));

            }

            System.out.println("-----------------------\n");
        }
    }
}

使用高级查询语法进行搜索

您可以使用高级查询语法或运算符创建比简单关键字或自然语言查询更具体的查询。这包括范围、布尔值、通配符等。通过使用运算符，您可以为查询提供更多上下文并进一步优化搜索结果。

Amazon Kendra支持以下运算符。

  • 布尔值：限制或扩大搜索范围的逻辑。例如，将搜索amazon AND sports限制为仅搜索包含这两个术语的文档。

  • 括号：按优先顺序读取嵌套的查询术语。例如，(amazon AND sports)之前(amazon AND sports) NOT rainforest读过NOT rainforest。

  • 范围：日期或数字范围值。范围可以是包容性、排他性或无限范围。例如，您可以搜索上次更新的文档，这些文档在 2020 年 1 月 1 日至 2020 年 12 月 31 日之间，包括这些日期。

  • 字段：使用特定字段来限制搜索。例如，您可以在 “位置” 字段中搜索带有 “美国” 的文档。

  • 通配符：部分匹配一串文本。例如，Cloud*可以匹配 CloudFormation。 Amazon Kendra目前仅支持尾随通配符。

  • 精确引号：精确匹配一串文本。例如，包含"Amazon Kendra" "pricing".

您可以使用上述任何运算符的组合。

请注意，过度使用运算符或高度复杂的查询可能会影响查询延迟。就延迟而言，通配符是一些最昂贵的运算符。一般规则是，使用的术语和运算符越多，对延迟的潜在影响就越大。影响延迟的其他因素包括索引文档的平均大小、索引的大小、对搜索结果的任何筛选以及Amazon Kendra索引的总体负载。

布尔值

您可以使用布尔运算符AND、OR、组合或排除单词NOT。

以下是使用布尔运算符的示例。

amazon AND sports

返回文本中同时包含 “亚马逊” 和 “体育” 这两个术语的搜索结果，例如亚马逊 Prime 视频体育或其他类似内容。

sports OR recreation

返回文本中包含 “体育” 或 “娱乐” 一词或两者兼而有之的搜索结果。

amazon NOT rainforest

返回文本中包含 “亚马逊” 一词但不包含 “雨林” 一词的搜索结果。这是为了搜索有关亚马逊公司的文档，而不是亚马逊雨林的文档。

圆括号

您可以使用圆括号按优先顺序查询嵌套单词。括号表示应Amazon Kendra如何解读查询。

以下是使用圆括号运算符的示例。

(amazon AND sports) NOT rainforest

返回文本中同时包含术语 “亚马逊” 和 “体育” 的文档，但不包含 “雨林” 一词的文档。这是为了搜索亚马逊 Prime 视频体育或其他类似内容，而不是亚马逊雨林中的冒险运动。括号有助于表明amazon AND sports应先阅读NOT rainforest。该查询不应被理解为amazon AND (sports NOT rainforest)。

(amazon AND (sports OR recreation)) NOT rainforest

返回包含 “体育” 或 “娱乐” 或两者兼而有之的术语以及 “亚马逊” 一词的文档。但它不包括 “雨林” 一词。这是为了搜索亚马逊Prime视频体育或娱乐，而不是在亚马逊雨林中搜索冒险运动。括号有助于表明在与之前读过的 “亚马逊” 组合之前sports OR recreation应先阅读该内容NOT rainforest。该查询不应被理解为amazon AND (sports OR (recreation NOT rainforest))。

范围

您可以使用一系列值来筛选搜索结果。您可以指定属性和范围值。这可以是日期或数字类型。

日期范围必须为以下格式：

  • Epoch

  • YYYY

  • yyyy-mm

  • yyyy-mm-DD

  • yyyy-mm-dd't'HH

您还可以指定是包括还是排除该范围的较低值和较高值。

以下是使用范围运算符的示例。

_processed_date:>2019-12-31 AND _processed_date:<2021-01-01

退回在 2020 年处理的文档，这些文档大于 2019 年 12 月 31 日且小于 2021 年 1 月 1 日。

_processed_date:>=2020-01-01 AND _processed_date:<=2020-12-31

返回 2020 年处理的文档，这些文档大于或等于 2020 年 1 月 1 日，小于或等于 2020 年 12 月 31 日。

_document_likes:<1

返回点赞次数为零或没有用户反馈的文档，点赞次数小于 1。

您可以指定是否应将某个范围视为包含或不包括给定范围值。

包容性

_last_updated_at:[2020-01-01 TO 2020-12-31]

退货文档最后更新时间为 2020 年，包括 2020 年 12 月 1 日和 2020 年 12 月 31 日。

独家

_last_updated_at:{2019-12-31 TO 2021-01-01}

退货文档最后更新时间为 2020 年，不包括 2019 年 12 月 31 日和 2021 年 1 月 1 日。

对于既非包容性也非排他性的无界范围，只需使用< and > 运算符即可。例如，_last_updated_at:>2019-12-31 AND _last_updated_at:<2021-01-01

字段

您可以将搜索限制为仅返回符合特定字段值的文档。字段可以是任意类型。

以下是使用字段级上下文运算符的示例。

status:"Incomplete" AND financial_year:2021

返回 2021 财政年度的文件，其状态为 “不完整”。

(sports OR recreation) AND country:"United States" AND level:"professional"

返回讨论美国职业体育或娱乐的文档。

通配符

您可以使用通配符运算符扩大搜索范围，以考虑单词和短语的变体。这在搜索名称变体时很有用。 Amazon Kendra目前仅支持尾随通配符。结尾通配符的前缀字符数必须大于两个。

以下是使用通配符运算符的示例。

Cloud*

返回包含变体（例如 CloudFormation 和）的文档 CloudWatch。

kendra*aws

返回包含变体（例如 kendra.amazonaws）的文档。

kendra*aws*

返回包含变体（例如 kendra.amazonaws.com）的文档

精确报价

您可以使用引号来搜索一段文本的精确匹配项。

以下是使用引号的示例。

"Amazon Kendra" "pricing"

返回同时包含短语 “Amazon Kendra” 和 “定价” 一词的文档。文件必须同时包含Amazon Kendra “” 和 “定价” 才能在结果中返回。

"Amazon Kendra" "pricing" cost

返回同时包含 “Amazon Kendra” 短语和术语 “定价” 的文档，也可以包含术语 “成本” 的文档。文件必须同时包含Amazon Kendra “” 和 “定价” 才能在结果中返回，但不一定包含 “成本”。

查询语法

Amazon Kendra如果您的查询语法有问题或当前不支持您的查询，则会发出警告Amazon Kendra。有关更多信息，请参阅查询警告文档。

以下查询是无效查询语法的示例。

_last_updated_at:<2021-12-32

日期 第 32 天不存在于公历中，该日历由使用Amazon Kendra。

_view_count:ten

无效的数值。必须使用数字来表示数值。

nonExistentField:123

字段搜索无效。该字段必须存在才能使用字段搜索。

Product:[A TO D]

范围 范围必须使用数值或日期。

OR Hello

布尔值 运算符必须与术语一起使用，并且必须放在术语之间。

用语言搜索

您可以使用支持的语言搜索文档。您在中传递语言代码AttributeFilter以您选择的语言返回经过筛选的文档。您可以使用支持的语言键入查询。

如果未指定语言，默认情况下会Amazon Kendra查询英语文档。有关支持的语言（包括其代码）的更多信息，请参阅添加英语以外语言的文档。

要在控制台中以支持的语言搜索文档，请选择您的索引，然后从导航菜单中选择搜索索引的选项。通过选择搜索设置，然后从下拉语言中选择一种语言，选择要返回文档的语言。

  2. 在导航菜单中，选择 Index（索引）。

  3. 在导航菜单中，选择搜索索引的选项。

  5. 在文本框中输入查询，然后按 Enter。

  6. Amazon Kendra以西班牙语返回搜索结果。

    Java
    package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.QueryRequest;
import software.amazon.awssdk.services.kendra.model.QueryResponse;
import software.amazon.awssdk.services.kendra.model.QueryResultItem;

public class SearchIndexExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String query = "searchString";
        String indexId = "indexID";

        QueryRequest queryRequest = QueryRequest.builder()
            .queryText(query)
            .indexId(indexId)
            .attributeFilter(
                 AttributeFilter.builder()
                     .withEqualsTo(
                         DocumentAttribute.builder()
                             .withKey("_language_code")
                             .withValue("es")
                             .build())
                     .build())
            .build();

        QueryResponse queryResponse = kendra.query(queryRequest);

        System.out.println(String.format("\nSearch results|
                                          Resultados de la búsqueda: %s", query));
        for(QueryResultItem item: queryResponse.resultItems()) {
            System.out.println("----------------------");
            System.out.println(String.format("Type: %s", item.type()));

            switch(item.type()) {
                case QUESTION_ANSWER:
                case ANSWER:
                    String answerText = item.documentExcerpt().text();
                    System.out.println(answerText);
                    break;
                case DOCUMENT:
                    String documentTitle = item.documentTitle().text();
                    System.out.println(String.format("Title: %s", documentTitle));
                    String documentExcerpt = item.documentExcerpt().text();
                    System.out.println(String.format("Excerpt: %s", documentExcerpt));
                    break;
                default:
                    System.out.println(String.format("Unknown query result type: %s", item.type()));

            }

            System.out.println("-----------------------\n");
        }
    }
}
