Amazon WorkDocs

Amazon WorkDocs是一种安全的内容协作服务，用于创建、编辑、存储和共享内容。您可以使用索Amazon Kendra引您的Amazon WorkDocs数据源。

您可以使用Amazon Kendra控制台和 WorkDocsConfigurationAPIAmazon Kendra 连接到您的Amazon WorkDocs数据源。

Amazon WorkDocs在俄勒冈州、北弗吉尼亚州北部、悉尼、新加坡和爱尔兰地区推出。

要对Amazon Kendra WorkDocs 数据源连接器进行故障排除，请参阅解决数据源问题。

支持的功能

Amazon Kendra WorkDocs 数据源连接器支持以下功能：

  • 更改日志

  • 字段映射

  • 用户上下文过滤

  • 包含/排除过滤器

先决条件

在使用索Amazon Kendra引 WorkDocs 数据源之前，请在您的 WorkDocs 和AWS账户中进行这些更改。

在中 WorkDocs，请确保您有：

  • 已记下您的Amazon WorkDocs仓库的Amazon WorkDocs目录 ID（组织 ID）。

  • 已检查每个文档在您计划用于同一索引的其他数据源中 WorkDocs 以及在您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

如果您没有现有IAM角色，则可以在将 WorkDocs 数据源连接到时使用控制台创建新IAM角色Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色的 ARN 和索引 ID。

连接指令

要Amazon Kendra连接到您的 WorkDocs 数据源，您必须提供 WorkDocs 数据源的必要详细信息，这样Amazon Kendra才能访问您的数据。如果您尚未配置， WorkDocs 请参Amazon Kendra阅先决条件。

Console

要连接Amazon Kendra到Amazon WorkDocs

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在 “入门” 页面上，选择 “添加数据源”。

  4. 在 “添加数据源” 页面上，选择 “WorkDocs 连接器”，然后选择 “添加数据源”。

  5. 在指定数据来源详细信息页面上，输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）说明-为数据来源输入可选说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tags（标签）中，为添加新标签 —Tags（标签）中，用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在 Def e Security（定义访问和安全）页面上，输入以下信息：

    1. 特定于您的Amazon WorkDocs站点的组织 ID —选择要索引的Amazon WorkDocs站点的 ID。您必须已经创建了站点。

    2. IAM 角色 —选择现有 IAM 角色来访问您的存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    3. 选择下一步。

  7. 在配置同步设置页面上，输入以下信息：

    1. 抓取文档注释-要抓取的Amazon WorkDocs实体或内容类型。

    2. 使用更改日志-选择更新索引，而不是同步所有文件。

    3. 正则表达式@@ 模式 —包含或排除某些文件的正则表达式模式。您最多可以添加 100 个模式。

    4. 在 F requency 的同步运行时间表中-选择与数据源同步的频率。Amazon Kendra

    5. 选择下一步。

  8. 在 Set te details（设置字段映射）页面上，输入以下信息：

    1. 默认数据源字段-从Amazon Kendra生成的默认数据源字段中选择要映射到索引的字段。

    2. 添加字段-添加自定义数据源字段以创建要映射到的索引字段名称和字段数据类型。

    3. 选择下一步。

  9. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面编辑信息。成功添加数据源后，该数据源将显示在数据源页面上。

API

要连接Amazon Kendra到Amazon WorkDocs

您必须使用 WorkDocsConfigurationAPI 指定以下内容：

  • Amazon WorkDocs目录 ID-指定您的Amazon WorkDocs目录的组织 ID。您可以在 AWS Directory directory directory directory Secri pti on（阿

  • IAM 角色 — 指定RoleArn何时调用CreateDataSource，为IAM角色提供访问 WorkDocs目录和调用 WorkDocs 连接器所需的公共 API 的权限Amazon Kendra。有关更多信息，请参阅 WorkDocs 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • 更改日志-是否Amazon Kendra应使用 WorkDocs 数据源更改日志机制来确定是否必须在索引中添加、更新或删除文档。

    注意

    如果您不想扫描所有文档Amazon Kendra，请使用更改日志。如果您的更改日志很大，则扫描 WorkDocs 数据源中的文档所花费的时间可能Amazon Kendra少于处理更改日志所花费的时间。如果您是第一次将 WorkDocs数据源与索引同步，则会扫描所有文档。

  • 包含和排除筛选器-指定是包括还是排除某些文档和文档注释。每条评论都作为单独的文档编制索引。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 WorkDocs 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • 用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或其群组对文档的访问权限筛选搜索结果。有关更多信息，请参阅 WorkDocs 数据源的用户上下文筛选。
