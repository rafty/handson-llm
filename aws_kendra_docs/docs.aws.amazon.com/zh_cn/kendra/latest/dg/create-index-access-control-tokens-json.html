使用 JSON

以下示例显示在创建索引时如何使用带有证书令牌的 JWT 进行用户访问控制。

警告

JSON 令牌是未经验证的有效负载。只有当请求来自可信服务器而不是Amazon Kendra来自浏览器时，才应使用此选项。

Console

  1. 选择创建索引开始创建新索引。

  2. 在指定索引详细信息页面上，为索引指定名称和描述。

  3. 对于IAM角色，选择一个角色或选择创建新角色并指定角色名称以创建新角色。该IAM角色将带有前缀 “AmazonKendra-”。

  4. 保留其他默认值。选择 Next（下一步）。

  5. 在 “配置用户访问控制” 页面的 “访问控制设置” 下，选择 “是” 以使用令牌进行访问控制。

  6. 在令牌配置下，选择 JSON 作为令牌类型。

  7. 指定在 ACL 检查中使用的用户名。

  8. 指定要在 ACL 检查中使用的一个或多个组。

  9. 选择 Next（下一步）。

  10. 在配置详细信息页面中，选择开发者版本。

  11. 选择 “创建” 来创建您的索引。

  12. 等待您的索引创建。 Amazon Kendra为您的索引配置硬件。此操作可能需要一些时间。

CLI

要使用 JSON 输入文件创建索引，请先AWS CLI使用所需参数创建 JSON 文件：

{
    "Name": "user-context",
    "Edition": "ENTERPRISE_EDITION",
    "RoleArn": "arn:aws:iam::account-id:role:/my-role",
    "UserTokenConfigurations": [
        {
            "JsonTokenTypeConfiguration": {
                "UserNameAttributeField": "user",
                "GroupAttributeField": "group"
            }
        }
    ],
    "UserContextPolicy": "USER_TOKEN"
}

接下来，create-index使用输入文件调用。例如，如果您的 JSON 文件的名称是create-index-openid.json，则可以使用以下内容：

aws kendra create-index --cli-input-json file://create-index-openid.json

如果您不使用 Open IDAWS IAM Identity Center (successor to AWS Single Sign-On)，则可以将 JSON 格式的令牌发送给我们。如果这样做，则必须指定 JSON 令牌中的哪个字段包含用户名以及哪个字段包含群组。组字段值必须是 JSON 字符串数组。例如，如果您使用 SAML，您的代币将类似于以下内容：

{
     "username" : "user1", 
     "groups": [
        "group1", 
        "group2"
     ]
}

TokenConfiguration将指定用户名和组字段名：

{
    "UserNameAttributeField":"username",
    "GroupAttributeField":"groups"
}
Python
response = kendra.create_index(
    Name='user-context',
    Edition='ENTERPRISE_EDITION',
    RoleArn='arn:aws:iam::account-id:role:/my-role',
    UserTokenConfigurationList=[
        {
            "JwtTokenTypeConfiguration": {
                "UserNameAttributeField": "user",
                "GroupAttributeField": "group",
            }
        }
    ],
    UserContextPolicy='USER_TOKEN'
)
