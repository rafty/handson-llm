BatchPutDocument

将一个或多个文档添加到索引中。

BatchPutDocumentAPI 使您能够提取内联文档或一组存储在 Amazon S3 存储桶中的文档。使用此 API 将您的文本和非结构化文本提取到索引中，向文档添加自定义属性，并将访问控制列表附加到添加到索引的文档。

这些文档是异步索引的。您可以使用查看批处理的进度AWS CloudWatch。与处理批处理相关的任何错误消息都将发送到您的AWS CloudWatch 日志。

有关使用 Python 和 Java SDK 采集行内文档的示例，请参阅将文件直接添加到索引。

请求参数

有关所有操作的通用参数的信息，请参阅通用参数。

请求接受采用 JSON 格式的以下数据。

CustomDocumentEnrichmentConfiguration

用于在您使用BatchPutDocument API 时在文档提取过程中更改文档元数据和内容的配置信息。

有关如何在将文档载入 Amazon Kendra 时创建、修改和删除文档元数据或进行其他内容更改的更多信息，请参阅在摄取过程中自定义文档元数据。

类型：CustomDocumentEnrichmentConfiguration 对象

必需：否

Documents

要添加到索引的一个或多个文档。

文档有以下文件大小限制。

  • 任何文件的总大小 50 MB

  • 为任何文件提取了 5 MB 的文本

有关更多信息，请参阅 配额。

类型：Document 对象数组

数组成员：最少 1 项。最多 10 项。

必需：是

IndexId

要将文档添加到索引的标识符。您需要先使用CreateIndex API 创建索引。

类型：字符串

长度限制：固定长度为 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必需：是

RoleArn

IAM 角色的 Amazon 资源名称（ARN），此角色具有访问您的 S3 存储桶的权限。有关更多信息，请参阅 Amazon Kendra 的 IAM AM 访问角色。

类型：字符串

长度约束：最小长度为 0。最大长度上限为 1284。

模式：arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

必需：否

响应元素

如果此操作成功，则该服务将会发送回 HTTP 200 响应。

服务以 JSON 格式返回的以下数据。

FailedDocuments

由于文档未通过验证检查而未添加到索引的文档列表。每个文档都包含一条错误消息，指明无法将该文档添加到索引的原因。

如果在向索引中添加文档时出错，则会在您的AWS CloudWatch 日志中报告该错误。有关更多信息，请参阅使用亚马逊 CloudWatch 日志监控Amazon Kendra 的 Amazon

类型：BatchPutDocumentResponseFailedDocument 对象数组
