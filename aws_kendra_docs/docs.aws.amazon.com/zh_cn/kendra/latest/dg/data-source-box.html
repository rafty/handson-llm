Box

Box是提供文件托管功能的云存储服务。您可以使用Amazon Kendra索引 Box 内容中的内容，包括评论、任务和 Web 链接。

您可以使用Amazon Kendra控制台和 BoxConfigurationAPI 连接Amazon Kendra到 Box 数据源。

要对Amazon Kendra Box 数据源连接器进行故障排除，请参阅解决数据源问题。

支持的功能

Amazon KendraBox 数据源连接器支持以下功能：

  • 更改日志

  • 字段映射

  • 用户上下文过滤

  • 包含/排除过滤器

  • Virtual Private Cloud (VPC)

先决条件

在使用Amazon Kendra索引 Box 数据源之前，请先在 Box 和AWS账户中进行这些更改。

在 Box 中，确保您有：

  • 一个 Box Enterprise 或 Box 企业增强版账户

  • 在 Box 开发者控制台中创建了 Box 自定义应用程序，并将其配置为使用服务器身份验证（使用 JWT）。有关更多详细信息，请参阅有关创建自定义应用程序的 Box 文档和配置 JWT Auth 的 Box 文档。

  • 将您的应用程序访问级别设置为 A pp + Enterprise Ac cess，并允许其使用 as-user 标头进行 API 调用。

  • 已使用管理员用户在您的 Box 应用程序中添加以下应用程序范围：

    • 写入存储在 Box 中的所有文件和文件夹

    • 管理用户

    • 管理群组

    • 管理企业财产

  • 生成和下载的公钥/私钥对，包括客户端 ID、客户端密钥、公钥 ID、私钥 ID、密码和用作身份验证凭证的企业 ID。有关更多详细信息，请参阅公钥和私钥对。

  • 从 Box 开发者控制台设置或 Box 应用程序复制了您的 Box 企业 ID。例如，801234567。

  • 已选中 Box 中的每个文档在您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的 Box 身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或密钥，则可以在将 Box 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接指令

要连接Amazon Kendra到 Box 数据源，您必须提供 Box 数据源的必要详细信息，Amazon Kendra以便访问您的数据。如果您尚未配置 Box，请参Amazon Kendra阅先决条件。

Console

连接Amazon Kendra到 Box

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在 “入门” 页面上，选择 “添加数据源”。

  4. 在 “添加数据源” 页面上，选择 Box 连接器，然后选择 “添加数据源”。

  5. 在 Description（指定数据源详细信息）页面上，输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）—为数据来源输入可选说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tags（添加新标签）中，用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在 Description（定义访问和安全性）页面上，输入以下信息：

    1. Box 企业 ID —输入您的 Box 企业 ID。

    2. AWSSecrets Managers ecret —选择现有密钥或创建新Secrets Manager密钥来存储您的 Box 身份验证凭据。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      1. 秘密名称 — 您的秘密名称。前缀 “AmazonKendra-Box-” 会自动添加到您的密钥名称中。

      2. 对于客户端 ID、客户端密钥、公钥 ID、私钥 ID 和密码短语 — 输入您在 Box 帐户中生成并从 Box 帐户下载的公共/私钥中的值。

      3. 选择保存。

    3. V@@ irtual Private Cloud (VPC) —您可以选择使用 VPC。如果是，则必须添加子网和 VPC 安全组。

    4. IAM 角色 — 选择现有 IAM 角色或创建新的 IAM 角色来访问您的存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    5. 选择下一步。

  7. 在 Creption Scription（配置同步设置）页面上，输入以下信息：

    1. 选择实体或内容类型-要抓取的 Box 实体或内容类型。每条评论都作为单独的文档编制索引。

    2. 更改日志-选择更新索引，而不是同步所有文件。

    3. 正则表达式@@ 模式 —包含或排除某些文件的正则表达式模式。您最多可以添加 100 个模式。

    4. 在 “同步运行计划” 中，针对 “频率” — 选择与数据源同步的频率。Amazon Kendra

    5. 选择下一步。

  8. 在 Set（设置字段映射）页面上，输入以下信息：

    1. 对于文件和文件夹、注释、任务和 Web 链接-从Amazon Kendra生成的默认数据源字段中选择要映射到索引的字段。

    2. 添加字段-添加自定义数据源字段以创建要映射到的索引字段名称和字段数据类型。

    3. 选择下一步。

  9. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面编辑信息。成功添加数据源后，该数据源将显示在数据源页面上。

API

连接Amazon Kendra到 Box

您必须使用 BoxConfigurationAPI 指定以下内容：

Box 企业 ID —提供您的 Box 企业 ID。您可以在 Box 开发者控制台设置中找到企业 ID，也可以在 Box 中创建应用程序时找到企业 ID。

  • 秘密亚马逊资源名称 (ARN)-提供包含您的 Box 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，具有以下键值：

    {
    "clientID": "client-id",
    "clientSecret": "client-secret",
    "publicKeyID": "public-key-id",
    "privateKey": "private-key",
    "passphrase": "pass-phrase"
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 — 指定您RoleArn何时调用，CreateDataSource以便为IAM角色提供访问您的Secrets Manager密钥和调用 Box 连接器所需的公共 API 的权限Amazon Kendra。有关更多信息，请参阅 Box 数据来源的 IAM 数据来源的 IAM 的 IAM 数据

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC)-指定VpcConfiguration为数据源配置的一部分。请参阅配置Amazon Kendra使用 VPC。

  • 更改日志-是否Amazon Kendra应使用 Box 数据源更改日志机制来确定是否必须在索引中添加、更新或删除文档。

    注意

    如果您不想扫描所有文档Amazon Kendra，请使用更改日志。如果您的更改日志很大，则扫描 Box 数据源中的文档所花费的时间可能Amazon Kendra少于处理更改日志所花费的时间。如果您是第一次将 Box 数据源与索引同步，则会扫描所有文档。

  • 包含和排除过滤器-指定是包含还是排除某些 Box 文件、文件夹、评论、任务和 Web 链接。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 Box 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • 用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或其群组对文档的访问权限筛选搜索结果。有关更多信息，请参阅 Box 数据源的用户上下文筛选。
