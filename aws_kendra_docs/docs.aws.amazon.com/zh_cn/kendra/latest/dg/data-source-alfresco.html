Alfresco OnPrem 是一项内容管理服务，可帮助客户存储和管理他们的内容。你可以使用Amazon Kendra索引你的 Alfresco 文档库、维基和博客。

您可以使用Amazon Kendra控制台和 AlfrescoConfigurationAPIAmazon Kendra 连接到您的 Alfresco 数据源。

注意

Alfresco 数据源连接器当前处于预览模式。目前支持基本身份验证。如果您想在生产中使用 Alfresco 连接器，请联系Su pport 部门。当前不支持 Alfresco PaaS。

要对Amazon Kendra Alfresco 数据源连接器进行故障排除，请参阅解决数据源问题。

支持的功能

Amazon KendraAlfresco 数据源连接器支持以下功能：

  • 字段映射

  • 用户上下文过滤

  • 包含/排除过滤器

  • Virtual Private Cloud (VPC)

先决条件

在使用Amazon Kendra索引您的 Alfresco 数据源之前，请在您的 Alfresco 和AWS账户中进行这些更改。

在 Alfresco 中，请确保您有：

  • 记下了你的 Alfresco 身份验证凭证，其中包括管理员用户名和密码。

  • 已将用户添加到ALFRESCO_ADMINISTRATORS群组。

  • 复制了你的 Alfresco 网站网址和你的 Alfresco 网站 ID。

  • 已检查每个文档在 Alfresco 以及您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的 Alfresco 身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下该密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或机密，则可以在将 Alfresco 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接指令

要Amazon Kendra连接到您的 Alfresco 数据源，您必须提供 Alfresco 数据源的必要详细信息，这样Amazon Kendra才能访问您的数据。如果你还没有配置 Alfresco，Amazon Kendra请参阅先决条件。

  4. 在 “添加数据源” 页面上，选择 Alfresco 连接器，然后选择 “添加数据源”。

    4. 在 Tags（添加新标签）中，Tags（添加新标签），用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

    3. 站点 ID —您的 Alfresco 站点 ID。

      1. 在 “创建AWSSecrets Manager密钥” 窗口中输入以下信息：

API

连接 AlAmazon Kendra fresco

您必须使用 AlfrescoConfigurationAPI 指定以下内容：

  • 秘密亚马逊资源名称 (ARN)-提供包含您的 Alfresco 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，具有以下密钥：

    {
    "username": "user name",
    "password": "password"
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 —指定您RoleArn何时调用CreateDataSource以向IAM角色提供访问您的Secrets Manager密钥以及调用 Alfresco 连接器所需的公共 API 的权限，以及Amazon Kendra. 有关更多信息，请参阅 Alfresco 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC)-指定VpcConfiguration为数据源配置的一部分。请参阅配置Amazon Kendra使用 VPC。

  • 包含和排除过滤器-指定是包含还是排除某些文件。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 Alfresco 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • 用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或其群组对文档的访问权限筛选搜索结果。有关更多信息，请参阅 Alfresco 数据源的用户上下文筛选。
