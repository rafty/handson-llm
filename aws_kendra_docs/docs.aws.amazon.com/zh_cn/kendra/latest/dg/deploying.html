部署 Amazon Kendra

到了将Amazon Kendra搜索部署到您的网站的时候，我们会提供源代码，您可以将其与 React 一起使用，让您的应用程序抢先一步。根据修改后的 MIT 许可证，源代码是免费提供的。您可以按原样使用它，也可以根据自己的需求进行更改。提供的 React App 就是帮助您入门的示例。它不是一款可用于生产的应用程序。

要部署不带代码的搜索应用程序并使用访问控制生成指向搜索页面的终端节点 URL，请参阅 Experience BuAmazon Kendra ilder。

您可以在 React 中使用两个示例：

  • https://kendrasamples.s3.amazonaws.com/kendrasamples-react-app .zip —提供示例数据和搜索页面的 React 应用程序示例。

  • https://kendrasamples.s3.amazonaws.com/kendrasamples.zip —您可以将其添加到现有 React 应用程序中的库。

这些示例以Amazon Kendra控制台的搜索页面为蓝本。它们具有相同的搜索和显示搜索结果的功能。您可以使用整个示例，也可以只选择其中一个功能供自己使用。

要在Amazon Kendra控制台中查看搜索页面的三个组件，请从右侧菜单中选择代码图标 (</>)。将指针悬停在每个部分上可查看组件的简要描述并获取组件源代码的 URL。

概览

您可以将示例代码添加到现有 React 应用程序以激活搜索。搜索文件和组件的结构如下：

  • 主搜索页面-这是包含所有组件的主页。在这里，您可以将您的应用程序与Amazon Kendra API 集成。

  • 搜索栏-这是用户输入搜索词并调用搜索功能的组件。

  • 结果-这是显示来自的结果的组件Amazon Kendra。它由三个部分组成：建议答案、常见问题解答结果和推荐文档。

  • 分面-该组件在搜索结果中显示分面，并允许您选择一个分面来限制搜索。

  • 分页-这是对来自的响应进行分页的组件Amazon Kendra。

先决条件

在开始之前，您需要：

  • 现有的 React Web 应用程序或示例应用程序。

  • 使用正确的库配置的开发环境。

  • Node.js 和 npm 安装了。

  • SDK for Java或AWS SDK for JavaScript。

有关所需库和AWS SDK 的信息位于 zip 文件的自述文件中。

设置示例

向 React 应用程序添加Amazon Kendra搜索的完整过程位于示例 zip 文件中包含的自述文件中。

开始使用 kendra-samples-react-app .zip
警告

React 应用程序尚未准备就绪。这是部署用于Amazon Kendra搜索的应用程序的示例。

  1. 确保你已经完成了先决条件，包括下载和安装 Node.js 和 npm。

  2. 下载 kendra-samples-react-app .zip 并解压缩。

  3. 打开您的终端并前往aws-kendra-sample-app/src/services/。打开local-dev-credentials-template.json并提供您的证书。不要将此文件添加到任何公共存储库中。

  4. 转到aws-kendra-sample-app/src并安装依赖项。运行 npm install。

  5. 在您的本地服务器上启动应用程序的演示版本。运行 npm start。您可以通过在键盘上输入来停止本地服务器Cmd/Ctrl + C。

  6. 您可以通过访问package.json并更新主机和端口来更改端口或主机（例如，IP 地址）："start": "HOST=[host] PORT=[port] react-scripts start"。如果你使用 Windows:"start": "set HOST=[host] && set PORT=[port] && react-scripts start".

  7. 如果您有注册的网站域名，则可以在应用程序名称package.json后面指定。例如，"homepage": "https://mywebsite.com"。你必须npm install再次运行才能更新新的依赖关系，然后运行npm start。

  8. 要构建应用程序，请运行npm build。将构建目录的内容上传到您的主机提供商。

主搜索页面

主搜索页面包含所有示例搜索组件。它包括用于输出的搜索栏组件、用于显示来自查询 API 的响应的结果组件以及用于对响应进行分页的分页组件。

搜索组件

搜索组件提供了一个用于输入查询文本的文本框。该onSearch函数是一个钩子，用于调用主函数Search.tsx来调用Amazon Kendra Query API。

结果组件

结果组件显示来自Query API 的响应。结果显示在三个不同的区域中。

  • 建议答案-这些是Query API 返回的最高结果。它最多包含三个建议的答案。在响应中，它们具有结果类型ANSWER。

  • 常见问题解答——这些是响应返回的常见问题解答结果。常见问题解答分别添加到索引中。在响应中，它们的类型为QUESTION_ANSWER。有关更多信息，请参阅问题和答案。

  • 推荐文档-这些是在响应中Amazon Kendra返回的其他文档。在Query API 的响应中，它们的类型为DOCUMENT。

结果组件共享一组用于突出显示、标题、链接等功能的组件。共享组件必须存在，结果组件才能正常工作。

Facets 组件

facets 组件列出了搜索结果中可用的分面。每个方面都按照特定的维度（例如作者）对回复进行分类。您可以从列表中选择一个方面，将搜索范围缩小到特定的方面。

选择分面后，组件会使用属性筛选器调用，该筛选器将搜索限制在Query与该分面匹配的文档上。

分页组件

分页组件允许您在多个页面中显示来自Query API 的搜索结果。它使用PageSize和PageNumber参数调用Query API 以获取特定的结果页面。
