Amazon Kendra用于自我管理的智能排名 OpenSearch

您可以利用Amazon Kendra语义搜索功能来改善基于 Apache 2.0 许可证的自主管理开源搜索服务的搜索结果。OpenSearchAmazon Kendra智能排名插件使用语义上 OpenSearch对结果进行重新排名Amazon Kendra. 它通过使用默认 OpenSearch 搜索结果中的特定字段（例如文档正文或标题）了解搜索查询的含义和上下文来实现此目的。

以这个查询为例：“主主题演讲”。由于 “地址” 有多种含义，因此Amazon Kendra可以推断出查询背后的含义，从而返回与预期含义一致的相关信息。在这种情况下，这是一次会议主题演讲。例如，更简单的搜索服务可能不会考虑意图，可能会返回主街上街道地址的结果。

的智能排名插件适用 OpenSearch 于 OpenSearch （自我管理）2.4.0 及更高版本。您可以使用快速入门 Bash 脚本安装该插件，以构建包含智能排名插件 OpenSearch 的新 Docker 镜像。请参阅设置智能搜索插件 —这是使您快速启动并运行的设置示例。

智能搜索插件的工作原理

OpenSearch （自我管理）的智能排名插件的整体流程如下：

  1. OpenSearch 用户发出查询，并 OpenSearch 提供查询响应或与查询相关的文档列表。

  2. 智能排名插件获取查询响应并从文档中提取信息。

  3. 智能排名插件会调用Amazon Kendra智能排名的 Rescore API。

  4. RescoreAPI 从文档中提取的信息，并在语义上对搜索结果进行重新排名。

  5. RescoreAPI 将重新排序的搜索结果发送回插件。该插件重新排列搜索响应中的 OpenSearch 搜索结果，以反映新的语义排名。

智能排名插件使用 “正文” 和 “标题” 字段对结果进行重新排名。这些插件字段可以映射到 OpenSearch 索引中最适合文档正文和标题定义的字段。例如，如果您的索引包含一本书的章节，其中包含 “chapter_heading” 和 “chapter_contents” 等字段，则可以将前者映射到 “标题”，将后者映射到 “正文” 以获得最佳结果。

设置智能搜索插件

以下概述了如何使用智能排名插件快速设置 OpenSearch （自我管理）。

使用智能排名插件进行设置 OpenSearch （自我管理）（快速设置）

如果你已经在使用 Docker 镜像opensearch:2.4.0，你可以使用这个 Dockerfile 通过智能排名插件构建 OpenSearch 2.4.0 的新镜像。你在 docker-compose.yml 文件或 opensearch.yml 文件中包含一个存放新图像的容器。您还可以在创建重新评分执行计划时生成的重新评分执行计划 ID 以及您的区域和终端信息——有关创建重新评分执行计划的步骤 2，请参阅步骤 2。

如果您之前下载过早于 2.4.0 的opensearch Docker 镜像版本，则必须使用 Docker 镜像opensearch:2.4.0或更高版本，并使用包含智能排名插件构建新镜像。

  1. 下载适用于操作系统的 Docker Desktop。Docker 桌面包括 Docker Compose 和 Docker 引擎 建议您检查您的计算机是否满足 Docker 安装详细信息中提到的系统要求。

    您还可以在 Docker Desktop 的设置中增加内存使用要求。在 Docker 服务的免费使用限制之外的 Docker 使用要求由您负责。参见 Docker 订阅。

    检查 Docker 桌面状态是否为 “正在运行”。

  2. 配置Amazon Kendra智能排名和您的容量需求。预置Amazon Kendra智能排名后，将根据您设定的容量单位按小时向您收费。查看免费套餐和定价信息。

    您可以使用 CreateRescoreExecutionPlanAPI 来配置Rescore API. 如果您需要的容量单位不超过单个单位的默认容量单位，请不要添加更多单位，只为重新评分执行计划提供一个名称。您也可以使用 UpdateRescoreExecutionPlanAPI 更新容量要求。有关更多信息，请参阅对搜索服务的结果进行语义排名。

    或者，在运行快速启动 Bash 脚本时，您可以转到步骤 3 创建默认的重新评分执行计划。

    注意步骤 4 中响应中包含的重新评分执行计划 ID。

    CLI
    aws kendra-ranking create-rescore-execution-plan \
  --name MyRescoreExecutionPlan \ 
  --capacity-units '{"RescoreCapacityUnits":<integer number of additional capacity units>}'
 
Response:
 
{
    "Id": "<rescore execution plan ID>",
    "Arn": "arn:aws:kendra-ranking:<region>:<account-id>:rescore-execution-plan/<rescore-execution-plan-id>"
}
    Python
    import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra_ranking = boto3.client("kendra-ranking")

print("Create a rescore execution plan.")

# Provide a name for the rescore execution plan
name = "MyRescoreExecutionPlan"
# Set your required additional capacity units
# Don't set capacity units if you don't require more than 1 unit given by default
capacity_units = 1

try:
    rescore_execution_plan_response = kendra_ranking.create_rescore_execution_plan(
        Name = name,
        CapacityUnits = {"RescoreCapacityUnits":capacity_units}
    )

    pprint.pprint(rescore_execution_plan_response)

    rescore_execution_plan_id = rescore_execution_plan_response["Id"]

    print("Wait for Amazon Kendra to create the rescore execution plan.")

    while True:
        # Get the details of the rescore execution plan, such as the status
        rescore_execution_plan_description = kendra_ranking.describe_rescore_execution_plan(
            Id = rescore_execution_plan_id
        )
        # When status is not CREATING quit.
        status = rescore_execution_plan_description["Status"]
        print(" Creating rescore execution plan. Status: "+status)
        time.sleep(60)
        if status != "CREATING":
            break

except ClientError as e:
        print("%s" % e)

print("Program ends.")

  3. 从主分支下拉列表中选择版本分支，从中下载 GitHub 适用于您的版本 OpenSearch 的快速启动 Bash 脚本。

  4. 打开终端并在 Bash 脚本的目录中运行以下命令。

    bash search_processing_kendra_quickstart.sh -p <execution-plan-id> -r <region>

    运行此命令时，您需要提供在步骤 2 中配置Amazon Kendra智能排名时记下的 rescore 执行计划 ID 以及您的区域信息。（可选）您可以改为使用--create-execution-plan选项预置Amazon Kendra智能排名。这将创建具有默认名称和默认容量的 rescore 执行计划。如果无法从环境变量、默认配置文件或Amazon EC2实例中获取AWS证书，请使用--profile选项提供您的证书。

    为了在移除默认临时容器时不丢失索引，您可以使用--volume-name选项提供数据卷名称，让索引在执行后保持不变。如果你之前创建了索引，你可以在 docker-compose.yml 或 opensearch.yml 文件中指定音量。

    此脚本使用 Docker 镜像 OpenSearch ，使用您在脚本 GitHub 存储库中选择的版本 OpenSearch 控制面板。它为智能排名插件下载一个压缩文件，并生成一个Dockerfile用于构建包含 OpenSearch 该插件的新 Docker 镜像。它还创建了一个 docker-compose.yml 文件，其中包含 OpenSearch 带有智能排名插件和 OpenSearch 仪表板的容器。该脚本将你的 rescore 执行计划 ID、区域信息和终端节点（使用区域）添加到 docker-compose.yml 文件中。然后，该脚本运行docker-compose up以启动 OpenSearch 包含智能排名和 OpenSearch 仪表板的容器。要在不移除容器的情况下停止它们，请运行docker-compose stop。要移除容器，请运行docker-compose down。要保持音量不变，请不要运行docker-compose down -v。

与智能搜索插件交互

                "title_field": "title_field_name_here",

              }

        }
      }
    }
  }

    hosts = [{'host': host, 'port': port}],

            }
        }

将 OpenSearch 结果与Amazon Kendra结果进行比较

您可以将 side-by-side OpenSearch （自我管理）排名结果与重新排名Amazon Kendra的结果进行比较。 OpenSearch 仪表板版本 2.4.0 及更高版本提供了 side-by-side 结果，因此您可以将如何对文档进行 OpenSearch 排名与插件对搜索查询的文档进行排名Amazon Kendra或插件的排名进行比较。

在将排名结果与Amazon Kendra重新 OpenSearch 排名的结果进行比较之前，请确保您的 OpenSearch 仪表板由带有智能排名插件的 OpenSearch 服务器提供支持。你可以使用 Docker 和快速启动 Bash 脚本进行设置。请参阅设置智能搜索插件。

以下概述了如何在 OpenSearch 仪表板中比较 OpenSearch 和Amazon Kendra搜索结果。有关更多信息，请参阅 OpenSearch 文档。

比较 OpenSearch 仪表板中的搜索结果

  1. 打开 http://localhost:5601 然后登录 OpenSearch 仪表板。默认凭据是用户名 “管理员” 和密码 “admin”。

  2. 从导航菜单的 OpenSearch 插件中选择 “搜索相关性”。

  3. 在搜索栏中输入搜索文本。

  4. 为查询 1 选择索引，然后在 Query DSL 中输入 OpenSearch 查询。您可以使用%SearchText%变量引用您在搜索栏中输入的搜索文本。有关此查询的示例，请参阅OpenSearch 文档。此查询返回的 OpenSearch 结果是未使用智能排名插件的结果。

  5. 为查询 2 选择相同的索引，然后在 Query DSL 中输入相同的 OpenSearch 查询。此外，将扩展名包括在内，kendra_intelligent_ranking并指定排名的body_field必填项。您也可以指定标题字段，但正文字段是必填字段。有关此查询的示例，请参阅OpenSearch 文档。此查询返回的结果是使用智能Amazon Kendra排名插件重新排序的结果。该插件最多可对25个结果进行排名。

  6. 选择 “搜索” 以返回和比较结果。
