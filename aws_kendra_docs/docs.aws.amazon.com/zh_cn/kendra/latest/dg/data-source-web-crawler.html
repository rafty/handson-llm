Amazon Kendra网络爬虫

您可以使用Amazon Kendra Web Crawler 抓取和索引网页。

您只能爬取使用安全通信协议超文本传输协议（HTTPS）的网站和网站。如果您在爬取网站时收到错误，则可能是该网站被阻止爬网。要抓取内部网站，可以设置 Web 代理。Web 代理必须面向公众。

当选择要编制索引的网站时，您必须遵守 Amazon 可接受使用政策以及所有其他 Amazon 条款。请记住，您必须仅使用Amazon Kendra Web 爬网程序为您自己的网页或您有权限编制索引的网页编制索引。要了解如何阻止Amazon Kendra Web Crawler 索引您的网站，请参阅为Amazon Kendra Web Crawler 配置robots.txt文件。

您可以使用Amazon Kendra控制台和 WebCrawlerConfigurationAPIAmazon Kendra 连接到您的网络爬虫数据源。

要对Amazon Kendra网络爬虫数据源连接器进行故障排除，请参阅解决数据源问题。

先决条件

在使用索Amazon Kendra引网站之前，请先检查您的网站和AWS帐户的详细信息。

对于您的网站，请确保您有：

  • 已复制要为网站站点地图 URL 的种子或站点地图 URL。

  • 对于需要基本身份验证才能抓取的网站：复制了网站的主机名和端口号。

  • 可选：如果您想使用 Web 代理连接到要抓取的内部网站，请复制网站的主机名和端口号。Web 代理必须面向公众。 Amazon Kendra支持连接到由基本身份验证支持的 Web 代理服务器，也可以不进行身份验证进行连接。

  • 已检查您要索引的每个网页文档是唯一的，以及您计划在其他数据源中用于同一索引的文档是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的网络爬虫身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下该密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或密钥，则可以在将 Web Crawler 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接说明

要Amazon Kendra连接到您的网络爬虫数据源，您必须提供网络爬虫数据源的必要详细信息，这样Amazon Kendra才能访问您的数据。如果您尚未配置 Web 搜寻器，请参Amazon Kendra阅先决条件。

Console

Amazon Kendra连接到网络爬虫

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在 “入门” 页面上，选择 “添加数据源”。

  4. 在 “添加数据源” 页面上，选择 Web Crawler 连接器，然后选择 “添加数据源”。

  5. 在 Group details（指定数据源详细信息）页面上，输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）D es cription（可选）-为数据来源输入可选说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tag es（添加新标签），用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在 Deat e security（定义访问和安全性）页面上，输入以下信息：

    1. 对于来源，根据您的用例在来源网址和来源站点地图之间进行选择，然后输入每个网址的值。

      注意

      您最多可添加 10 个来源URL 和 3 个站点地图。

      如果您想抓取站点地图，请务必检查基本网址或根网址是否与站点地图页面上列出的网址所使用的基本网址相同。例如，如果您的站点地图网址是 https://example.com/sitemap-page.html，则此站点地图页面上列出的网址也应使用基本网址 “https://example.com/”。

    2. （可选）对于 Web 代理，请输入以下信息：

      1. 主机名-需要 Web 代理的主机名。

      2. 端口号-主机 URL 传输协议使用的端口。端口号应为 0 到 65535 之间的数值。

      3. 对于 Web 代理凭据-选择现有密钥或创建新密钥来存储您的 Web 爬虫身份验证凭据。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      4. 在 “创建AWSSecrets Manager密钥” 窗口中输入以下信息：

        1. 秘密名称 — 您的秘密名称。前缀 “AmazonKendra-WebCrawler-” 会自动添加到您的密钥名称中。

        2. 对于用户名和密码-输入您网站的这些基本身份验证凭据。

        3. 选择保存。

    3. （可选）具有身份验证的主机-选择添加其他经过身份验证的主机。

    4. IAM 角色-选择现有 IAM 角色或创建新的 IAM 角色以访问您的存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    5. 选择下一步。

  7. 在 Create source sroup（配置同步设置）页面上，输入以下信息

    1. 抓取范围-选择要抓取的网页类型。

    2. 抓取深度-从种子 URL 中选择Amazon Kendra应抓取的级别数。

    3. 高级抓取设置和其他配置输入以下信息：

      1. 最大文件大小-要抓取的最大网页或附件大小。最小 0.000001 MB（1 字节）。最大 50 MB。

      2. 每页最大链接数-每页搜索的最大链接数。按出现顺序爬取部分层。至少 1 个链接/页。每页最多 1000 个链接。

      3. 最大限制-每分钟爬取的每个主机名的最大 URL 数量。至少 1 个 URL/主机名/分钟。每分钟最多 300 个 URL /主机名。

      4. 正则表达式模式-添加模式以仅包含特定 URL，或排除 URL、文件类型或存储库中的特定文件。你总共可以有 100 个图案。

    4. 在 “同步运行计划” 中，针对 “频率” — 选择与数据源同步的频率。Amazon Kendra

    5. 选择下一步。

  8. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面编辑信息。成功添加数据源后，该数据源将显示在数据源页面上。

API

Amazon Kendra连接到网络爬虫

您必须指定以下 WebCrawlerConfigurationAPI：

  • URL —使用SeedUrlConfiguration和指定您想要爬取的网站站点地图 URL 的种子或起点 URL SiteMapsConfiguration。

    如果您想抓取站点地图，请务必检查基本网址或根网址是否与站点地图页面上列出的网址所使用的基本网址相同。例如，如果您的站点地图网址是 https://example.com/sitemap-page.html，则此站点地图页面上列出的网址也应使用基本网址 “https://example.com/”。

  • Amazon Resource Name (ARN)-如果网站需要基本身份验证，则需要提供主机名、端口号和密钥，用于存储您的用户名和密码的基本身份验证凭证。您可以使用 AuthenticationConfigurationAPI 提供 ARN。密钥存储在 JSON 结构中，具有以下键进行操作：

    {
    "username": "user name",
    "password": "password"
}

    您可以使用密钥提供 Web 代理凭证。AWSSecrets Manager您可以使用 ProxyConfigurationAPI 提供网站主机名和端口号，还可以选择存储您的 Web 代理凭证的密钥。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

您还可以添加以下可选功能：

  • 抓取模式-选择仅抓取网站主机名或带有子域的主机名，或者同时抓取网页链接到的其他域名。

  • 网站中从种子层到爬取的层或层数。例如，若网站有 3 层，分别是索引层（本示例中的种子）、部分层和子部分层，而您最高只想爬取部分层（0 至 1 层）的信息，则您可以将深度设置为 1。

  • 单个网页上要抓取的网页 URL 的最大数量。

  • 要抓取的网页的最大大大尺寸（以 MB 为单位）。

  • 每分钟爬取的每个网站主机的最大 URL 数量。

  • 用于连接和抓取内部网站的 Web 代理信息。

  • 用于访问和爬取需要用户身份验证的网站的身份验证信息。

  • 您可以使用自定义文档丰富工具将 HTML 元标记提取为字段。有关更多信息，请参阅在提取过程中自定义文档元数据。有关提取 HTML 元标记的示例，请参阅 CDE 示例。

  • 包含和排除过滤器-指定是包含还是排除某些 URL。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。
