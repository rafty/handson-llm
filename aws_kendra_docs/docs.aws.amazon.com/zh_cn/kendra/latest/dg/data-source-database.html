Amazon RDS/Aurora

您可以使用数据库数据源索引存储在数据库中的文档。提供数据库的连接信息后，Amazon Kendra连接和索引文档。

Amazon Kendra 支持以下数据库：

  • Amazon Aurora MySQL

  • Amazon Aurora PostgreSQL

  • Amazon RDS for MySQL

  • Amazon RDS 适用于 PostgreSQL

注意

不支持无服务器 Aurora 数据库。

您可以使用Amazon Kendra控制台和 DatabaseConfigurationAPIAmazon Kendra 连接到您的数据库数据源。

要对Amazon Kendra数据库数据源连接器进行故障排除，请参见解决数据源问题。

支持的功能

Amazon Kendra数据库数据源连接器支持以下功能：

  • 字段映射

  • 用户上下文过滤

  • Virtual Private Cloud (VPC)

先决条件

在使用索Amazon Kendra引数据库数据源之前，请在数据库和AWS帐户中进行这些更改。

在您的数据库中，确保您有：

  • 注意到您的用户密码对是数据源的基本身份验证凭据。

  • 复制了主机名、端口号、主机地址和包含数据库数据的数据表的名称。对于 PostgreSQL，数据表必须是公共表。

    注意

    主机和端口告诉Amazon Kendra在互联网上哪里可以找到数据库服务器。数据库名和表名告诉Amazon Kendra在数据库服务器上哪里可以找到文档数据。

  • 复制了数据表中包含文档数据的列的名称、文档 ID、一到五列以检测文档是否已更改，以及映射到自定义索引字段的可选数据表列。您可以将任何Amazon Kendra保留字段名称映射到表列。

  • 复制了数据库引擎类型信息，例如您使用的是 MyAmazon RDS SQL 还是其他类型。

  • 已检查每个文档在数据库中以及您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中提供：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的数据库身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或密钥，则可以在将数据库数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接以下各项

要Amazon Kendra连接到数据库数据源，必须提供数据库数据源的必要详细信息，这样Amazon Kendra才能访问您的数据。如果您尚未配置数据库，请参Amazon Kendra见先决条件。

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

    2. （可选）-为数据来源输入可选说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tag as（添加新标签）（添加新标签）（Tags）（添加以下各项）（添加以下各项）（添加新标签）（AWSTags

    5. 选择下一步。

  6. 在 Dev ataion（定义访问和安全性）页面上，输入以下信息：

    1. 终端节点 — DNS 主机名、IPv4 地址或 IPv6 地址。

    2. 端口 —端口号。

    3. 数据库 —数据库名称。

    4. 表名 —表名。

    5. 对于身份验证类型，在 “现有” 和 “新建” 之间选择以存储您的数据库身份验证凭据。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      1. 在创建AWSSecrets Manager密钥窗口中输入以下信息：

        1. 秘密名称 — 您的秘密名称。前缀 “AmazonKendra-database-” 会自动添加到您的机密名称中。

        2. 对于用户名和密码-输入数据库帐户的身份验证凭据值。

        3. 选择保存身份验证。

    6. V@@ irtual Private Cloud (VPC) — 您可以选择使用 VPC。如果是，则必须添加子网和 VPC 安全组。

      注意

      您必须使用私有子网。如果您的 RDS 实例位于您的 VPC 的公有子网中，则可以创建一个私有子网，对公有子网中的 NAT 网关具有出站访问权限。VPC 配置中提供的子网必须在美国西部（俄勒冈州）、美国东部（弗吉尼亚州北部）、欧洲（爱尔兰）。

    7. IAM 角色-选择现有 IAM 角色或创建新的 IAM 角色来访问您的存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    3. ACL 列和更改检测列-用于配置Amazon Kendra用于更改检测和访问控制列表的列。

    4. 在同步运行计划中，对于频率-选择与数据源同步的频率。Amazon Kendra

    5. 选择下一步。

  8. 在 Set tailuataion（设置字段映射）页面上，输入以下信息：

    1. Amazon Kendra默认字段映射-从Amazon Kendra生成的默认数据源字段中选择要映射到索引的字段。必须为document_id和添加数据库列值document_body

    2. 自定义字段映射-添加自定义数据源字段以创建要映射到的索引字段名称和字段数据类型。

API

Amazon Kendra连接到数据库

您必须指定以下 DatabaseConfigurationAPI：

  • ColumnConfiguration—有关索引应从数据库中获取文档信息的位置的信息。有关更多信息，请参阅 ColumnConfiguration。必须指定DocumentDataColumnName和字DocumentIdColumnName段。映射到该DocumentIdColumnName字段的列必须是整数列。以下以下以下以下以下以下以下以下以下以下以下以下以下数据来源是一个简单的列配置：

    "ColumnConfiguration": {
    "ChangeDetectingColumns": [
        "LastUpdateDate",
        "LastUpdateTime"
    ],
    "DocumentDataColumnName": "TextColumn",
    "DocumentIdColumnName": "IdentifierColumn",
    "DocoumentTitleColumnName": "TitleColumn",
    "FieldMappings": [
        {
            "DataSourceFieldName": "AbstractColumn",
            "IndexFieldName": "Abstract"
        }
    ]
}

  • ConnectionConfiguration— 运行数据库的数据库引擎的类型。该DatabaseHost字段必须是数据库的Amazon Relational Database Service (Amazon RDS) 实例终端节点。不要使用集群终端节点。

  • DatabaseEngineType—连接到数据库所需的配置信息。有关更多信息，请参阅 ConnectionConfiguration。

  • 秘密亚马逊资源名称 (ARN)-提供包含数据库账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中提供：

    {
    "username": "user name",
    "password": "password"
}

    密钥可以包含更多信息。以下以下以下示例是一个数据库配置。

    
"DatabaseConfiguration": {
"ConnectionConfiguration": {
"DatabaseHost": "host.subdomain.domain.tld",
        "DatabaseName": "DocumentDatabase",
        "DatabasePort": 3306,
        "SecretArn": "arn:aws:secretmanager:region:account ID:secret/secret name",
        "TableName": "DocumentTable"
    }
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 — 指定您RoleArn何时调用，CreateDataSource以便为IAM角色提供访问您的Secrets Manager密钥和调用数据库连接器所需的公共 API 的权限，以及Amazon Kendra。有关更多信息，请参阅数据库数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC)-指定VpcConfiguration为数据源配置的一部分。请参阅配置Amazon Kendra使用 VPC。

    注意

    您只能使用私有子网。如果您的 RDS 实例位于您的 VPC 的公有子网中，则可以创建一个私有子网，对公有子网中的 NAT 网关具有出站访问权限。VPC 配置中提供的子网必须在美国西部（俄勒冈州）、美国东部（弗吉尼亚州北部）、欧洲（爱尔兰）。

  • 字段映射-选择将数据库数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • 用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或其群组对文档的访问权限筛选搜索结果。有关更多信息，请参阅数据库数据源的用户上下文筛选。
