解决数据源问题

本部分可以帮助您修复Amazon Kendra 数据源的问题。

我的文档未编入索引

当您将 Kendra 索引与数据源同步时，可能会遇到阻止对文档进行索引的问题。索引是一个两步过程。首先，检查数据源中是否有要索引的新文档和更新的文档，并查找要从索引中删除的文档。其次，在文档级别，对每个文档进行访问和索引。

这两个步骤中的任何一个都可能出现错误。数据源级别的错误将在控制台的数据源详细信息页面的同步运行历史记录部分中报告。同步作业的状态可以是 “成功”、“未完成” 或 “失败”。您还可以查看作业期间索引和删除的文档数量。如果状态为 “失败”，则在 “详细信息” 列中显示一条消息。

亚马逊 CloudWatch 日志中报告了文档级别的错误。你可以使用 CloudWatch 控制台查看错误。

我的同步作业失败

当索引或数据源中存在配置错误时，同步作业通常会失败。数据源的 “详细信息” 列中的错误消息提供了有关出了什么问题的信息。问题通常是索引或数据源没有适当的 IAM 权限。错误消息描述了缺少的权限。以下是您可能收到的一些错误消息：

Failed to create log group for job. Please make sure that the IAM role provided has sufficient permissions.

如果您的索引角色没有使用权限 CloudWatch，则数据源将无法创建 CloudWatch 日志。如果您遇到此错误，则必须为索引角色添加 CloudWatch 权限。

Failed to access S3 file prefix (bucket name) while trying to crawl your metadata files. Please make sure the IAM Role (role ARN) provided has sufficient permissions.

当您使用 Amazon S3 数据源时，Kendra 必须有权访问包含文档的存储桶。您需要为数据源 IAM 角色添加 Kendra 读取存储桶的权限。

The provided IAM Role (role ARN) could not be assumed. Please make sure Amazon Kendra is a trusted entity that is allowed to assume the role.

Kendra 需要权限才能担任索引和数据源 IAM 角色。您需要向具有sts:AssumeRole操作权限的角色添加信任策略。

有关 Kendra 为数据源编制索引所需的 IAM 政策，请参阅IAM的访问角色Amazon Kendra。

我的同步作业不完整

当作业完成了数据源级别的流程时，它们通常是不完整的，但是在文档级别的处理过程中会出现一些错误。当作业未完成时，某些文档可能仍已成功编制索引。对于 Amazon S3 数据来源，任务未完成通常是由以下原因引起的：

  • 一个或多个文档的元数据无效。

  • 当已提交文档以供编制索引但至少有一份文档未提交时。

  • 提交了要从索引中删除的文档，但至少有一份文件没有提交。

要对未完成的同步作业进行故障排除，请先查看您的 CloudWatch 日志。

  1. 在详细信息列中，选择查看中的详细信息 CloudWatch。

  2. 查看错误消息，看看是什么原因导致了文档失败。

我的同步任务成功了，但没有索引文档

有时，索引同步作业的运行会被标记为 “成功”，但没有按预期索引任何新的或更新的文档。可能的原因包括：

  • 检查 CloudWatch DocumentsSubmittedForIndexingFailed指标以查看是否有任何文档无法同步。请查看您的 CloudWatch 日志以了解详细信息。

  • 对于 Amazon S3 数据源，您可能为 Kendra 提供了错误的存储桶名称或前缀。确保 Kendra 使用的存储段是包含要索引的文档的存储桶。

  • 重新索引在先前作业中未能编入索引的文档时，除非您更改了该文档或其关联的元数据文件，否则 Kendra 不会对其进行索引。

我在同步我的数据源时遇到了文件格式问题

如果您在向数据源添加文件或同步数据源时遇到文件格式问题，请确保您的文档类型支持 Kendra。有关支持的文档类型的列表，Amazon Kendra请参阅文档类型。

如果您将BatchPutDocument API 用于纯文本文件，请指定PLAIN_TEXT为内容类型。

同步数据源需要多长时间？

如果文档没有更新，Amazon Kendra 索引的同步时间将与文档数量成线性比例增加。例如，没有任何更新的1,000个文档大约需要五分钟才能同步，而没有任何更新的2,000个文档大约需要10分钟。如果文档有任何更新，则同步时间将根据更新的文档数量而增加。

同步数据源的费用是多少？

同步索引时，需要两分钟才能预热并激活 Amazon EC2 以建立必要的连接。在此过程中，您无需付费。只有在同步作业开始后，您的使用量计才会开始。有关 Kendra 定价的更多信息，请参阅 Amazon Kendra 定价。

我收到了Amazon EC2 授权错误

如果在同步虚拟私有云 (VPC) 数据源期间出现 Amazon EC2 未经授权的操作错误，则您的 VPC IAM 角色可能缺少所需的权限。请检查您用于数据源的 IAM 角色是否具有附加权限。有关更多信息，请参阅虚拟私有云 (VPC)IAM 角色：

我无法使用搜索索引链接打开我的 Amazon S3 对象

您的 Kendra 索引只能访问 Amazon S3 数据源授予其访问权限的文件。例如，Kendra 无法修改确定对象是公共对象还是加密对象的 Amazon S3 权限。Kendra 也没有为 Amazon S3 对象创建或返回签名链接的默认权限。如果您想为 Kendra 索引中的 Amazon S3 对象激活签名链接，则有两种选择：

  • 在将结果返回到搜索页面之前，您可以使用源 uri 对象对索引查询结果进行签名。有关此 step-by-step过程的详细说明，请参阅使用预签名 URL 共享对象。

  • 您可以覆盖 Amazon S3 对象元数据源 URI，并通过连接到 Amazon S3 存储桶的 CloudFront 内容分发网络 (CDN) 提供服务。或者，您可以使用 API Gateway 代理终结点返回预签名 URL 并重定向到该终结地址。

我收到 “使用 SSL 证书文件AccessDenied 时” 错误消息

如果您在将 SSL 证书与数据源一起使用时出现访问被拒绝错误，请确保您的 IAM 角色有权访问指定位置的 SSL 证书文件。如果证书使用密AWS KMS钥加密，则您的 IAM 角色也应有权使用该密AWS KMS钥进行解密。有关更多信息，请参阅的身份验证和访问控制AWS KMS。

我在使用 SharePoint 数据源时遇到了授权错误

如果您在将索引与 SharePoint 数据源同步时遇到授权错误，请确认您已在中分配了站点管理员角色 SharePoint。

我的索引无法从我的 Confluence 数据源抓取文档

如果您的 Kendra 索引在同步过程中没有从 Confluence 数据源抓取文档，请确认您是 Confluence 管理员组的成员。
