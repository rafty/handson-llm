什么是 Amazon Kendra？

Amazon Kendra是一项高度准确且智能的搜索服务，可让用户使用自然语言处理和高级搜索算法搜索非结构化和结构化数据。它返回问题的具体答案，为用户提供接近于与人类专家互动的体验。它具有高度的可扩展性，能够满足性能需求，与Amazon S3和等其他AWS服务紧密集成 Amazon Lex，并提供企业级安全性。

有关Amazon Kendra智能排名 API 操作的信息，请参阅 API 参考文档。Amazon Kendra

Amazon Kendra用户可以提出以下类型的问题或查询：

  • 事实问题 — 简单的是谁、什么、何时或在哪里的问题，例如离西雅图最近的服务中心在哪里？ 事实问题具有基于事实的答案，可以以单个单词或短语的形式返回。答案是从常见问题解答或索引文档中检索的。

  • 描述性问题-答案可以是一句话、段落或整个文档的问题。例如，如何将我的 Echo Plus 连接到我的网络？ 或者，我如何为低收入家庭获得税收优惠？ 。

  • 关键字和自然语言搜索 — 问题不仅包括要搜索的关键字，还包括复杂的对话问题。这包括预期含义并不总是很清楚的问题。例如，主题演讲。由于 “地址” 通常有多种含义，因此Amazon Kendra可以推断出用户在搜索查询背后的预期含义以返回相关信息。 Amazon Kendra使用深度学习模型来处理此类查询。

Amazon Kendra 的优势

Amazon Kendra 具有以下优势：

  • 准确性 —与使用关键字搜索的传统搜索服务不同，在关键字搜索中，结果基于基本的关键字匹配和排名，它会Amazon Kendra尝试了解问题的背景。 Amazon Kendra搜索您的数据，超越传统搜索，返回与您的查询最相关的单词、片段或文档。 Amazon Kendra使用机器学习以随着时间的推移改善搜索结果。

  • 简单 —Amazon Kendra 提供控制台和 API，用于管理要搜索的文档。您可以使用简单的搜索 APIAmazon Kendra 集成到您的客户端应用程序中，例如网站或移动应用程序。

  • 连接 —Amazon Kendra 可以连接到第三方数据存储库或数据源，例如 Microsoft SharePoint。您可以使用数据源轻松地索引和搜索文档。

  • 用户访问控制 — 为您的搜索应用程序Amazon Kendra提供高度安全的企业搜索。您的搜索结果反映了您组织的安全模型。可以根据用户或用户组访问文档的权限来筛选搜索结果。客户负责验证和授权用户访问权限。

Amazon Kendra开发人员版

Amazon Kendra开发者版以较低的成本提供的所有功能。Amazon Kendra它包括可提供 750 小时使用时间的免费套餐。开发者版非常适合探索如何为文档Amazon Kendra编制索引、试用功能以及开发使用这些功能的应用程序Amazon Kendra。

开发者版提供以下内容：

  • 最多 5 个索引，每个索引最多 5 个数据源。

  • 10,000 个文档或 3 GB 提取的文本。

  • 每天提供 4000 个查询，或每秒提供 0.05 个查询。

  • 在 1 个可用区 (AZ) 中运行 — 参见可用区（AWS区域内的数据中心）

您不应将开发者版用于生产应用程序。开发者版不提供任何延迟或可用性保证。

Amazon KendraEdition

当您想要为整个Amazon Kendra企业文档库编制索引时，或者当您的应用程序准备好在生产环境中使用时，请使用企业版。

企业版提供以下内容：

  • 最多 5 个索引，每个索引最多 50 个数据源。

  • 100,000 个文档或 30 GB 提取的文本。

  • 每天提供 8000 个查询，或每秒提供 0.1 个查询。

  • 在 3 个可用区 (AZ) 中运行 — 参见可用区（AWS区域内的数据中心）

您可以使用Service Quotas 控制台增加此配额。
