StartDataSourceSyncJob

启动数据来源连接的同步作业。如果同步任务已经在进行中，Amazon Kendra 会返回ResourceInUseException异常。

请求语法

{
   "Id": "string",
   "IndexId": "string"
}

请求参数

有关所有操作的通用参数的信息，请参阅通用参数。

请求接受采用 JSON 格式的以下数据。

Id

要同步的数据来源连接的标识符。

类型：字符串

长度限制：最小长度为 1。长度上限为 100。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

必需：是

IndexId

与数据来源连接使用的索引的标识符。

类型：字符串

长度限制：固定长度为 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必需：是

响应语法

{
   "ExecutionId": "string"
}

响应元素

如果此操作成功，则该服务将会发送回 HTTP 200 响应。

服务以 JSON 格式返回的以下数据。

ExecutionId

标识特定的同步作业。

类型：字符串

长度限制：最小长度为 1。最大长度为 2048。

错误

AccessDeniedException

ConflictException

请求发生了冲突。请修复与您的资源存在的任何不一致之处，然后重试。

HTTP 状态代码：400

InternalServerException

用于您的Amazon Kendra 服务的内部服务器出现问题。请稍等几分钟，然后重试，或联系Su pport 部门寻求帮助。

ResourceInUseException

HTTP 状态代码：400

ResourceNotFoundException

你要使用的资源不存在。请检查您提供的资源是否正确，然后重试。

HTTP 状态代码：400

HTTP 状态代码：400

ValidationException
