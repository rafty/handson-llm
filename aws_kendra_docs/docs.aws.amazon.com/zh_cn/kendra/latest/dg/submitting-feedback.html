为增量学习提交反馈

Amazon Kendra使用增量学习来改善搜索结果。使用来自查询的反馈，增量学习可以改进排名算法，并优化搜索结果以提高准确性。

例如，假设您的用户搜索 “医疗保健福利”。如果用户始终从列表中选择第二个结果，则随着时间的Amazon Kendra推移，该结果将提升到第一名。随着时间的推移，提升效果会降低，因此，如果用户停止选择结果，Amazon Kendra最终会将其删除，转而显示另一个更受欢迎的结果。这有助于根据相关性、年龄和内容对结果进行Amazon Kendra优先排序。

已为所有索引和所有支持的文档类型激活增量学习。

Amazon Kendra提供反馈后立即开始学习，但可能需要超过 24 小时才能看到反馈的结果。 Amazon Kendra为您提供了三种提交反馈的方法：AWS控制台、可以包含在搜索结果页面上的 JavaScript 库以及可以使用的 API。

Amazon Kendra接受两种类型的用户反馈：

  • 点击次数 —有关用户选择的查询结果的信息。反馈包括结果 ID 和选择搜索结果的日期和时间的 Unix 时间戳。

    要提交点击反馈，您的应用程序必须从用户的活动中收集点击信息，然后将该信息提交给Amazon Kendra。您可以使用控制台、 JavaScript 库和Amazon Kendra API 收集点击信息。

  • 相关性 —用户通常提供的有关搜索结果相关性的信息。反馈包含结果 ID 和相关性指标（RELEVANT或NOT_RELEVANT）。用户确定相关性信息。

    要提交相关性反馈，您的应用程序必须提供反馈机制，允许用户为查询结果选择适当的相关性，然后将该信息提交给Amazon Kendra。您只能使用控制台和Amazon Kendra API 收集相关信息。

在索引处于活动状态时使用反馈。反馈仅影响提交到的索引，不能在索引之间使用，也不能用于不同的账户。

查询Amazon Kendra索引时，应提供额外的用户上下文。当您提供用户背景信息时，Amazon Kendra能够判断反馈是由单个用户还是由多个用户提供的，并相应地调整搜索结果。

当您提供用户上下文时，查询的反馈与上下文中提供的特定用户相关联。如果您不指定用户上下文，则可以提供用于对查询进行分组和聚合的访客 ID。

如果您不提供用户背景信息或访客 ID，则反馈是匿名的，与其他匿名反馈汇总在一起。

以下代码显示如何将用户上下文作为令牌或访客 ID 包括在内。

response = kendra.query(
    QueryText = query,
    IndexId = index,
    UserToken = {
        Token = "token"
    })
    
    OR
    
    response = kendra.query(
    QueryText = query,
    IndexId = index,
    VisitorId = "visitor-id")

对于 Web 应用程序，您可以使用 Cookie、位置或浏览器用户为每个用户生成访客 ID。

对于负责人查询，最大的查询量，提供点击反馈可以提供足够的信息来提高整体准确性。对于罕见的尾部查询，主题专家应提交相关和非相关的反馈，以提高这些查询的准确性。

除控制台外，您还可以使用两种方法之一： JavaScript 库或SubmitFeedback API。您应该只使用一种收集反馈的方法。为了获得最佳结果，您应在提出查询后的 24 小时内提交反馈。
