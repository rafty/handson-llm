Dropbox

Dropbox 是一项文件托管服务，提供云存储、文档组织和文档模板服务。如果您是 Dropbox 用户，则可以使用Amazon Kendra索引您的 Dropbox 文件、Dropbox Paper、Dropbox Paper 模板和存储的网页快捷方式。您还可以配置Amazon Kendra为特定的 Dropbox 文件、Dropbox Paper、Dropbox Paper 模板和存储的网页快捷方式编制索引。

Amazon Kendra同时支持 Dropbox Business 版 Dropbox 和

您可以使用Amazon Kendra控制台和 TemplateConfigurationAPI 连接Amazon Kendra到 Dropbox 数据源。

要对Amazon Kendra Dropbox 数据源连接器进行故障排除，请参阅解决数据源问题。

支持的功能

Amazon KendraDropbox 数据源连接器支持以下功能：

  • 更改日志

  • 字段映射

  • 用户上下文过滤

  • 包含/排除过滤器

  • Virtual Private Cloud (VPC)

先决条件

在使用Amazon Kendra Dropbox 数据源编制索引之前，请先在 Dropbox 和AWS帐户中进行这些更改。

在 Dropbox 中，请确保您具有：

  • 创建了 Dropbox Advanced 帐户并设置了管理员用户。

  • 使用唯一的应用程序名称创建了 Dropbox 应用程序，激活了 Scoped Acces s。有关创建应用程序的信息，请参阅 Dropbox 文档。

  • 在 Dropbox 控制台上激活了 Dropbox 的完整权限并添加了以下权限：

    • 文件.content.read

    • 文件.metadata.read

    • 分享.read

    • 文件请求.读取

    • 群组.read

    • team_info.read

    • team_data.content.re

  • 记下了你的 Dropbox 应用程序密钥、Dropbox 应用程序密钥和用于基本身份验证凭据的 Dropbox 访问

  • 为您的 Dropbox 应用程序生成并复制了一个临时 Oauth 2.0 访问令牌。此令牌是临时的，将在 4 小时后过期。请参阅有关 OAuth 身份验证的 Dropbox 文档。

    注意

    建议您创建永不过期的 Dropbox 刷新访问令牌，而不是依赖于 4 小时后过期的一次性访问令牌。刷新访问令牌是永久性的，永不过期，因此您将future 可以继续同步数据源。

  • 推荐：配置了永不过期的 Dropbox 永久刷新令牌，Amazon Kendra以允许在不中断的情况下继续同步您的数据源。请参阅有关刷新令牌的 Dropbox 文档。

  • 已检查每个文档在 Dropbox 以及您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的 Dropbox 身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或密钥，则可以在将 Dropbox 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接说明

要连接Amazon Kendra到 Dropbox 数据源，您必须提供 Dropbox 数据源的必要详细信息，Amazon Kendra以便访问您的数据。如果您尚未配置 Dropbox，Amazon Kendra请参阅先决条件。

Console

连接到 DropAmazon Kendra box

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在 “入门” 页面上，选择 “添加数据源”。

  4. 在添加数据源页面上，选择 Dropbox 连接器，然后选择添加数据源。

  5. 在详细信息页面上，输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）说明-为数据来源输入可选说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tags（添加新标签）中，使用 Tags（添加新标签）中，用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在 Description（定义访问和安全）页面上，输入以下信息：

    1. 身份验证令牌类型-根据您的用例在永久令牌（推荐）和访问令牌（临时使用）之间进行选择。

    2. AWSSecrets Manager密钥 —选择现有密钥或创建新Secrets Manager密钥来存储您的 Dropbox 身份验证凭据。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      1. 在 “创建AWSSecrets Manager密钥” 窗口中输入以下信息：

        1. 秘密名称 — 您的秘密名称。前缀 “AmazonKendra-Dropbox-” 会自动添加到您的机密名称中。

        2. 对于应用程序密钥、应用程序密钥和令牌信息（永久或临时）-输入您从 Dropbox 帐户生成的身份验证凭证值。

      2. 选择保存。

    3. V@@ irtual Private Cloud (VPC) — 您可以选择使用 VPC。如果是，则必须添加子网和 VPC 安全组。

    4. IAM 角色 — 选择现有 IAM 角色以访问您的存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    5. 选择下一步。

  7. 在 Cription Sect（配置同步设置）页面上，输入以下信息：

    1. 对于选择实体或内容类型-选择要抓取的实体或内容类型。

    2. 更改日志模式-选择更新索引而不是同步所有文件。

    3. 在 Regex 模式的附加配置中 —添加正则表达式模式以包含或排除某些文件。

    4. 在同步运行计划中，对于频率 — 选择与数据源同步的频率。Amazon Kendra

    5. 选择下一步。

  8. 在 Set cription（设置字段映射）页面上，输入以下信息：

    1. 文件、Dropbox Paper 和 Dropbox Paper 模板 — 从Amazon Kendra生成的默认数据源字段中选择要映射到索引的字段。

    2. 添加字段-添加自定义数据源字段以创建要映射到的索引字段名称和字段数据类型。

    3. 选择下一步。

  9. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面编辑信息。成功添加数据源后，它将显示在数据源页面上。

API

连接到 DropAmazon Kendra box

您必须使用 TemplateConfigurationAPI 指定数据源架构的 JSON。您必须提供以下信息：

  • 数据源-必须将数据源指定为DROPBOX。

  • 类型 —在呼叫时指定TEMPLATE为类型CreateDataSource。

  • 秘密亚马逊资源名称 (ARN)-提供包含您的 Dropbox 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，键如下：

    {
"appKey": "Dropbox app key",
"appSecret": "Dropbox app secret",
"accesstoken": "temporary access token or refresh access token"
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 — 指定您RoleArn何时调用，CreateDataSource以便为IAM角色提供访问您的Secrets Manager密钥和调用 Dropbox 连接器所需的公共 API 的权限，以及Amazon Kendra。有关更多信息，请参阅 Dropbox 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC)-指定您VpcConfiguration何时调用CreateDataSource。请参阅配置Amazon Kendra为使用 VPC。

  • 更改日志-是否Amazon Kendra应使用 Dropbox 数据源更改日志机制来确定是否必须在索引中添加、更新或删除文档。

    注意

    如果您不想扫描所有文档Amazon Kendra，请使用更改日志。如果您的更改日志很大，则扫描 Dropbox 数据源中的文档所花费的时间可能Amazon Kendra少于处理更改日志所花费的时间。如果您是第一次将 Dropbox 数据源与索引同步，则会扫描所有文档。

  • 包含和排除过滤器-指定是包含还是排除某些文件。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 Dropbox 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • 用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或其群组对文档的访问权限筛选搜索结果。有关更多信息，请参阅 Dropbox 数据源的用户上下文筛选。

有关要配置的其他重要 JSON 密钥的列表，请参阅Dropbox 模板。
