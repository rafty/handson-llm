使用亚马逊 CloudWatch 日志监控亚马逊 Kendra

Amazon Kendra 使用 Amazon CloudWatch Logs 让您深入了解数据源的运行情况。Amazon Kendra 会在文档编制索引时记录文档的处理详情。它记录在对文档进行索引时发生的数据源错误。您可以使用 CloudWatch 日志来监视、存储和访问日志文件。

CloudWatch 日志将日志事件存储在属于日志组的日志流中。Amazon Kendra 按如下方式使用这些功能：

  • 日志组-Amazon Kendra 将您的所有日志流存储在每个索引的单个日志组中。Amazon Kendra 会在创建索引时创建日志组。日志组标识符始终以 “aws/kendra/” 开头。

  • 日志流 — Amazon Kendra 为您运行的每个索引同步作业在日志组中创建一个新的数据源日志流。当数据流达到大约 500 个条目时，它还会创建一个新的文档日志流。

  • 日志条目 — Amazon Kendra 在为文档编制索引时在日志流中创建日志条目。每个条目都提供有关处理文档或遇到的任何错误的信息。

有关使用 CloudWatch 日志的更多信息，请参阅《亚马逊云监控日志用户指南》中的 “什么是亚马逊云监视日志”。

Amazon Kendra 创建了两种类型的日志流：

  • 数据源日志流

  • 文档日志流

数据源日志流

数据源日志流发布有关您的索引同步作业的条目。每个同步作业都会创建一个新的日志流，用于发布条目。日志流名称为：

data source id/YYYY-MM-DD-HH/data source sync job ID

为每个同步作业运行创建一个新的日志流。

发布到数据源日志流的日志消息有三种类型：

  • 未能发送用于索引的文档的日志消息。以下是 S3 数据源中文档的此消息的示例：

    {
    "DocumentId": "document ID",
    "S3Path": "s3://bucket/prefix/object",
    "Message": "Failed to ingest document via BatchPutDocument.",
    "ErrorCode": "InvalidRequest",
    "ErrorMessage": "No document metadata configuration found for document attribute key  city."
}

  • 未能发送以供删除的文档的日志消息。以下是此消息的示例：

    {
    "DocumentId": "document ID",
    "Message": "Failed to delete document via BatchDeleteDocument.",
    "ErrorCode": "InvalidRequest",
    "ErrorMessage": "Document can't be deleted because it doesn't exist." 
}

  • 找到 Amazon S3 存储桶中某个文档的无效元数据文件时的日志消息。以下是此消息的示例。

    {
    "Message": "Found invalid metadata file bucket/prefix/filename.extension.metadata.json."
}

  • 对于 SharePoint 和数据库连接器，Amazon Kendra 仅在无法为文档编制索引时将消息写入日志流。以下是Amazon Kendra 记录的错误消息的示例。

    { 
    "DocumentID": "document ID", 
    "IndexID": "index ID", 
    "SourceURI": "", 
    "CrawlStatus": "FAILED", 
    "ErrorCode": "403", 
    "ErrorMessage": "Access Denied", 
    "DataSourceErrorCode": "403"
}

文档日志流

Amazon Kendra 在为文档编制索引时记录有关处理文档的信息。它记录存储在 Amazon S3 数据源中的文档的一组消息。它仅记录存储在 Microsoft SharePoint 或数据库数据源中的文档的错误。

如果使用BatchPutDocument操作将文档添加到索引中，则日志流的命名如下：

YYYY-MM-DD-HH/UUID

如果使用数据源将文档添加到索引中，则日志流的命名如下：

dataSourceId/YYYY-MM-DD-HH/UUID

每个日志流最多包含 500 条消息。

如果索引文档失败，则将以下消息输出到日志流：

{
    "DocumentId": "document ID",
    "IndexName": "index name",
    "IndexId": "index ID"
    "SourceURI": "source URI"
    "IndexingStatus": "DocumentFailedToIndex",
    "ErrorCode": "400 | 500",
    "ErrorMessage": "message"
}
