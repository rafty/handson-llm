Amazon S3文档元数据

您可以使用元数据文件将元数据（有关文档的其他信息）添加到Amazon S3存储段中的文档中。每个元数据文件都与一个索引文档相关联。

您的元数据文件必须与索引文件存储在同一个存储桶中。创建数据源时，您可以使用控制台或DocumentsMetadataConfiguration参数S3Prefix字段为元Amazon S3数据文件指定存储桶中的位置。如果您未指定Amazon S3前缀，则您的元数据文件必须存储在与索引文档相同的位置。

如果您为元数据文件指定Amazon S3前缀，则它们位于与索引文档parallel 目录结构中。 Amazon Kendra仅在指定目录中查找您的元数据。如果未读取元数据，请检查目录位置是否与元数据的位置相匹配。

以下示例显示了索引文档位置如何映射到元数据文件位置。请注意，文档的Amazon S3密钥会附加到元数据的前Amazon S3缀中，然后加上后缀.metadata.json以形成元数据文件的Amazon S3路径。带有元数据Amazon S3前缀和.metadata.json后缀的组合Amazon S3密钥总数不得超过 1024 个字符。建议将Amazon S3密钥保持在 1000 个字符以下，以考虑将密钥与前缀和后缀组合时出现的额外字符。

Bucket name:
     s3://bucketName
Document path:
     documents
Metadata path:
     none
File mapping
     s3://bucketName/documents/file.txt -> 
        s3://bucketName/documents/file.txt.metadata.json
Bucket name:
     s3://bucketName
Document path:
     documents/legal
Metadata path:
     metadata
File mapping
     s3://bucketName/documents/legal/file.txt -> 
        s3://bucketName/metadata/documents/legal/file.txt.metadata.json

您的文档元数据在 JSON 文件中定义。该文件必须是没有 BOM 标记的 UTF-8 文本文件。JSON 文件的文件名必须是<document>.<extension>.metadata.json。在此示例中，“文档” 是元数据所适用的文档的名称，“扩展名” 是文档的文件扩展名。文档 ID 必须在中具有唯一性<document>.<extension>.metadata.json。

JSON 文件的内容遵循此模板。所有属性都是可选的。如果您未指定_source_uri，则搜索结果Amazon Kendra中返回的链接指向包含该文档的Amazon S3存储桶。

{
    "DocumentId": "document ID",
    "Attributes": {
        "_category": "document category",
        "_created_at": "ISO 8601 encoded string",
        "_last_updated_at": "ISO 8601 encoded string",
        "_source_uri": "document URI",
        "_version": "file version",
        "_view_count": number of times document has been viewed,
        "custom attribute key": "custom attribute value",
        additional custom attributes
    },
    "AccessControlList": [
         {
             "Name": "user name",
             "Type": "GROUP | USER",
             "Access": "ALLOW | DENY"
         }
    ],
    "Title": "document title",
    "ContentType": "For example HTML | PDF. For supported content types, see Types of documents."
}

_created_at和_last_updated_at元数据字段是 ISO 8601 编码的日期。例如，2012-03-25T12：30:10 + 01:00 是中部欧洲时间 2012 年 3 月 25 日中午 12:30（10 秒）的 ISO 8601 日期-时间格式。

您可以向Attributes字段中添加有关用于筛选查询或对查询响应进行分组的文档的其他信息。有关更多信息，请参阅创建自定义文档字段数：
