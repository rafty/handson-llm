映射数据源字段

您可以将数据源的文档或内容字段映射到索引中的字段。例如，如果您的数据源中有一个名为 “dept” 的字段，其中包含文档的部门信息，则可以将其映射到名为 “部门” 的索引字段。这样，你就可以在查询文档时使用该字段。

您还可以映射标准字段或保留字段，例如_created_at。如果您的数据源有一个名为 “create_date” 的字段，则可以将其映射到名为的等效Amazon Kendra保留字段_created_at。您可以映射大多数数据源的字段。

如果您将文档存储在 S3 存储桶或 S3 数据源中，则可以使用元数据文件直接提供自定义属性。有关更多信息，请参阅创建自定义文档字段数：

将数据源字段映射到索引字段是一个三步过程：

  1. 创建索引。有关更多信息，请参阅创建索引：

  2. 更新索引以添加自定义字段。

  3. 创建将数据源字段映射到索引字段的数据源。

要更新索引以添加自定义字段，请使用控制台或UpdateIndex API。您总共可以向索引添加 500 个自定义字段。

在控制台中，您可以选择将数据源字段映射到其中一个保留字段名称。或者，您可以选择创建映射到该字段的新索引字段。对于数据库数据源，如果数据库列的名称与保留字段的名称匹配，则该字段和列将自动映射。

使用 UpdateIndexAPI，您可以使用DocumentMetadataConfigurationUpdates参数添加自定义字段和保留字段。

以下 JSON 示例使用DocumentMetadataConfigurationUpdates向索引中添加一个名为 “部门” 的字段。

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

创建字段时，可以选择设置搜索中使用字段的方式。可从以下选项中进行选择：

  • 可显示-确定是否在查询响应中返回字段。默认为 true。

  • 分@@ 面表-指示字段可用于创建分面。默认为 false。

  • 可搜索-确定搜索中是否使用该字段。对于字符串字段，默认值为 true；对于数字和日期字段，默认值为 false。

  • Sortable-指示字段可用于对查询的响应进行排序。只能为日期、数字和字符串字段设置。无法为字符串列表字段设置。

以下 JSON 示例使用DocumentMetadataConfigurationUpdates向索引中添加一个名为 “Department” 的字段，并将其标记为 facetable。

"DocumentMetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE",
       "Search": {
           "Facetable": true
       }
   }
]

使用Amazon Kendra 内置文档字段

借助 UpdateIndex API，您可以使用DocumentMetadataConfigurationUpdates并指定保留字段名称来创建保留字段或内置字段，以映射到您的等效文档属性或文档字段。您也可以通过这种方式创建自定义字段。如果您使用数据源连接器，则大多数都包含将数据源文档字段映射到Amazon Kendra索引字段的字段映射。创建字段时，可以配置该Search对象，将该字段设置为可显示、可分面、可搜索和可排序。您可以配置该Relevance对象以设置字段的相关性提升持续时间、新鲜度、重要性、排名顺序和重要性值。一旦创建了字段，就无法更改字段类型。

Amazon Kendra有以下可供使用的保留或内置文档字段：

  • _authors—负责文档内容的一位或多位作者的列表。

  • _category— 将文档置于特定组中的类别。

  • _created_at— 以 ISO 8601 格式创建文档的日期和时间。例如，2012-03-25T12:30:10+01:00 是中部欧洲时间 2012 年 3 月 25 日中午 12:30（10 秒）的 ISO 8601 日期-时间格式。

  • _data_source_id— 包含文档的数据源的标识符。

  • _document_body— 文档的内容。

  • _document_id— 文档的唯一标识符。

  • _document_title— 文档的标题。

  • _excerpt_page_number— 显示文档摘录的 PDF 文件中的页码。如果您的索引是在 2020 年 9 月 8 日之前创建的，则必须重新索引您的文档，然后才能使用此属性。

  • _faq_id—如果这是常见问题解答问题和答案，则是它们的唯一标识符。

  • _file_type— 文档的文件类型，例如 pdf 或 doc。

  • _last_updated_at— 上次更新文档的日期和时间，采用 ISO 8601 格式。例如，2012-03-25T12:30:10+01:00 是中部欧洲时间 2012 年 3 月 25 日中午 12:30（10 秒）的 ISO 8601 日期-时间格式。

  • _source_uri— 文档可用的 URI。例如，公司网站上文档的 URI。

  • _version— 文档特定版本的标识符。

  • _view_count— 查看文档的次数。

  • _language_code（字符串）-适用于文档的语言代码。如果您未指定语言，默认为英语。有关支持的语言（包括其代码）的更多信息，请参阅添加英语以外语言的文档。

对于自定义字段，您可以DocumentMetadataConfigurationUpdates使用UpdateIndex API 创建这些字段，就像创建预留字段或内置字段时一样。您必须为自定义字段设置相应的数据类型。一旦创建了字段，就无法更改字段类型。

以下是您可以为自定义字段设置的类型：

  • 日期

  • 数字

  • 字符串

  • 字符串列表

您可以使用控制台或 UpdateIndexAPI 创建自定义字段。创建自定义字段后，将其映射到文档属性，就像使用保留字段一样。如果您使用 API 向索引添加了文档，则可以使用 BatchPutDocumentAPI 映射这些属性。对于从Amazon S3数据源索引的文档，您可以使用包含描述文档属性的 JSON 结构的元数据文件映射属性。对于使用允许字段映射的数据库或数据源索引的文档，您可以使用控制台或数据源配置映射属性。
