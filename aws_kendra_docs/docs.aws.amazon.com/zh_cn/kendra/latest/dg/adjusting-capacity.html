调整容量

Amazon Kendra以容量单位为您的索引提供资源。每个容量单位为您的索引提供额外资源。文档存储和查询有单独的容量单位。您只能向Amazon Kendra企业版索引添加容量单位。您无法向开发者版索引添加容量。

文档存储容量单位为您的索引提供以下额外存储空间。

  • 100,000 个文档或 30 GB 的存储空间。

查询容量单位为您的索引提供以下额外查询。

  • 每秒 0.1 个查询，或每天提供 8000 个查询。

每个索引的基本容量等于 1 个容量单位（30 GB 的存储空间和每秒 0.1 个查询）。每增加一个容量单位都需要额外付费。有关详细信息，请参阅 Amazon Kendra 定价。

您最多可以向存储添加 100 个额外容量单位，并查询索引资源。如果您需要更多单位，请联系AWS支持人员。

每天最多可以调整容量单位 5 个来适应您的使用要求。您不能将文档存储容量减少到索引中存储的文档数量以下。例如，如果您存储 150,000 个文档，则不能将存储容量减少到 1 个额外单位以下。

您可以通过选择索引名称来打开索引设置和其他信息，在控制台中查看索引正在使用的资源，也可以使用DescribeIndex API。 Amazon Kendra当您超过索引的容量时，也会返回异常。ServiceQuotaExceededException当所有文档的总提取大小超过索引的限制时，你会得到 a。当文档数量超过索引的限制时，每个文档都会获得 aInvalidRequest。ThrottlingException如果每秒查询数超出限制，则会得到。有关限制的更多信息，请参阅的配额Amazon Kendra。

查看容量

通过选择索引名称来访问详细信息，在Amazon Kendra控制台中查看索引正在使用的资源。控制台还提供使用情况图表，因此您可以确定索引使用了多少存储和查询容量。您可以使用此信息来帮助您规划何时添加额外容量。

要查看文档存储和查询，请使用（控制台）

  1. 登录AWS Management Console并打开Amazon Kendra主机，网址为 https://console.aws.amazon.com/kendra/home。

  2. 从索引列表中，选择要访问的索引。

  3. 滚动到设置部分，查看当前的文档存储总量和查询容量。

要使用Amazon Kendra API 查看容量，请使用DescribeIndex API 中的CapacityUnits参数。

添加和删除容量

如果您需要额外的索引容量，则可以使用控制台或Amazon Kendra API 添加索引容量。

添加或删除存储容量或查询容量（控制台）

  1. 登录AWS Management Console并打开Amazon Kendra主机，网址为 https://console.aws.amazon.com/kendra/home。

  2. 从索引列表中，选择要访问的索引。

  3. 选择 “编辑”，或从 “操作” 下拉列表中选择 “编辑”。

  4. 选择 “下一步” 进入配置详细信息页面。

  5. 添加或删除文档存储和/或查询容量单位。

  6. 继续选择 “下一步” 进入审阅页面，然后选择 “更新” 以保存您的更改。

在更新索引容量后，更改可能需要几分钟才能生效。

要使用Amazon Kendra API 添加或删除容量，请使用UpdateIndex API 中的CapacityUnits参数。

Amazon Kendra智能排名能力

容量单位每秒为重新评分执行计划提供以下额外重新评分请求。重新评分执行计划是用于配置 Rescore API 的资源。

  • 每秒 0.01 个请求。

每个 rescore 执行计划的基本容量等于 1 个容量单位（每秒 0.01 个请求）。每增加一个容量单位都需要额外付费。有关详细信息，请参阅 Amazon Kendra 定价。

对于重新评分执行计划，您最多可以添加 1000 个额外容量单位。如果您需要更多单位，请联系AWS支持人员。

查询建议容量

使用查询建议时，基本查询容量为每秒 2.5 次GetQuerySuggestions调用。GetQuerySuggestions容量是索引预置查询容量的五倍，或每秒 2.5 个调用的基本容量，以较高者为准。例如，索引的基本容量为每秒 0.1 个查询，GetQuerySuggestions 容量的基本容量为每秒 2.5 个调用。如果您在索引每秒共 0.2 个查询的基础上再添加 0.1 个查询，GetQuerySuggestions 容量为每秒 2.5 个调用（比每秒 0.2 个查询的五倍还要高）。

Amazon Kendra体验能力

搜索体验能力

Amazon Kendra根据您的Amazon Kendra经验QueryQuerySuggestions，开始限制SubmitFeedback为每秒 15 个请求，每秒 40 个请求以突发查询。对于查询容量单位超过 150 的索引，这些限制仍然适用。

例如，您的索引的查询容量单位为 150，因此您的搜索体验应用程序每秒可以处理 15 个请求。但是，如果您扩展到 200 个查询容量单位，则您的搜索体验应用程序仍将每秒只能处理 15 个请求。如果您将索引限制为 100 个查询容量单位，则您的搜索体验应用程序每秒只能处理 10 个请求。

自适应查询集群

Amazon Kendra预置的基本容量为 1 个查询容量单位。每天使用 8000 个查询，最小吞吐量为每秒 0.1 个查询。累积的查询将持续长达 24 小时，可以应对突发流量。允许的突发量会有所不同，因为它取决于任何给定时间的集群负载。预置足够的查询容量单位来处理峰值负载水平。

处理超出预置吞吐量的意外突发流量的一种自适应方法是内置Amazon Kendra的自适应查询突发。的企业版中提供了自适应查询拆分功能Amazon Kendra。

自适应查询突发是一项内置功能，允许您应用未使用的查询容量来处理意外流量。 Amazon Kendra以您预置的每秒查询速率累积未使用的查询，每秒累积一次，直至您为Amazon Kendra索引预置的最大查询数。这些累积的查询用于超出分配容量的意外流量。自适应查询拆分的最佳性能可能会有所不同，具体取决于多个因素，例如索引总大小、查询复杂性、累积的未使用查询以及索引的总负载。建议您自己进行负载测试以准确测量突发能力。
