CreateFaq

使用存储在 Amazon S3 存储桶中的指定常见问题文件的 Amazon S3 存储桶的 Amazon S3 存储桶的 Amazon S3 存储桶中的指定

向索引添加常见问题是一项异步操作。

有关使用 Python 和 Java SDK 向索引添加常见问题解答的示例，请参阅使用您的常见问题解答文件。

请求语法

{
   "ClientToken": "string",
   "Description": "string",
   "FileFormat": "string",
   "IndexId": "string",
   "LanguageCode": "string",
   "Name": "string",
   "RoleArn": "string",
   "S3Path": { 
      "Bucket": "string",
      "Key": "string"
   },
   "Tags": [ 
      { 
         "Key": "string",
         "Value": "string"
      }
   ]
}

请求参数

有关所有操作的通用参数的信息，请参阅常用参数。

请求接受采用 JSON 格式的以下数据。

ClientToken

您提供的令牌，用于识别创建常见问题解答的请求。使用相同的客户端令牌多次调用CreateFaqRequest API 只会创建一个常见问题解答。

类型：字符串

长度限制：最小长度为 1。长度上限为 100。

必需：否

Description

FAQ 的描述。

类型：字符串

长度限制：最小长度为 0。最大长度为 1000。

模式：^\P{C}*$

必需：否

FileFormat

常见问题的输入文件的格式。您可以在基本 CSV 格式、在标题中包含自定义属性的 CSV 格式以及包含自定义属性的 JSON 格式之间进行选择。

格式必须与S3Path参数中标识的 S3 存储桶中存储的文件的格式匹配。

有关更多信息，请参阅添加问题和答案。

类型：字符串

有效值: CSV | CSV_WITH_HEADER | JSON

必需：否

IndexId

常见问题的索引的标识符。

类型：字符串

长度限制：固定长度为 36。

模式：[a-zA-Z0-9][a-zA-Z0-9-]*

必需：是

LanguageCode

一种语言的代码。这允许您支持常见问题解答文档的语言。默认情况下支持英语。有关支持的语言（包括其代码）的更多信息，请参阅添加英语以外语言的文档。

类型：字符串

长度限制：最小长度为 2。最大长度为 10。

模式：[a-zA-Z-]*

必需：否

Name

常见问题的名称。

类型：字符串

长度限制：最小长度为 1。长度上限为 100。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

必需：是

RoleArn

角色的 Amazon 资源名称（ARN），此角色具有访问包含常见问题的 S3 存储桶的权限。有关更多信息，请参阅 Amazon Kendra 的 IAM 访问角色。

类型：字符串

长度约束：最小长度为 0。最大长度为 1284。

模式：arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

必需：是

S3Path

S3 中的 FAQ 文件的路径。

类型：S3Path 对象

必需：是

Tags

用于标识常见问题的键值对列表。您可以使用标签来标识和组织您的资源并控制对资源的访问。

类型：Tag 对象数组

数组成员：最少 0 项。最多 200 项。

必需：否

响应语法

{
   "Id": "string"
}

响应元素

如果此操作成功，则该服务将会发送回 HTTP 200 响应。

服务以 JSON 格式返回的以下数据。

Id

常见问题的标识符。

类型：字符串

长度限制：最小长度为 1。长度上限为 100。

模式：[a-zA-Z0-9][a-zA-Z0-9_-]*

错误

有关所有操作常见错误的信息，请参阅常见错误。

AccessDeniedException

您没有足够的访问此操作。请确保您拥有所需的权限策略和用户帐户，然后重试。

HTTP 状态代码：400

ConflictException

请求发生了冲突。请修复与您的资源存在的任何不一致之处，然后重试。

HTTP 状态代码：400

InternalServerException

用于您的Amazon Kendra 服务的内部服务器出现问题。请稍等几分钟，然后重试，或联系Su pport 部门寻求帮助。

HTTP 状态代码：500

ResourceNotFoundException

你要使用的资源不存在。请检查您提供的资源是否正确，然后重试。

HTTP 状态代码：400

ServiceQuotaExceededException

您已超过Amazon Kendra 服务的设定限制。有关更多信息，请参阅配额，或联系Supp or t 部门询问有关提高限额的问题。

HTTP 状态代码：400

ThrottlingException

由于请求限制而导致请求被拒绝。请减少请求数量，然后重试。

HTTP 状态代码：400

ValidationException

输入未能满足 Amazon Kendra 服务设置的限制。请提供正确的输入并重试。

HTTP 状态代码：400
