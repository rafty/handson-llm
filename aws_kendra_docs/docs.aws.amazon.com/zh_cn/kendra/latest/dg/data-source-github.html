GitHub

GitHub 是一项用于软件开发的基于 Web 的托管服务，提供带有版本控制的代码存储和管理服务。您可以使用Amazon Kendra索引 GitHub 企业云 (SaaS) 和 GitHub 企业服务器 (On Prem) 存储库文件、议题和拉取请求、议题和拉取请求评论以及议题和拉取请求评论附件。您还可以选择包括或排除某些文件。

您可以使用Amazon Kendra控制台和 GitHubConfigurationAPIAmazon Kendra 连接到您的 GitHub 数据源。

要对Amazon Kendra GitHub 数据源连接器进行故障排除，请参阅解决数据源问题。

支持的功能

Amazon Kendra GitHub 数据源连接器支持以下功能：

  • 更改日志

  • 字段映射

  • 用户上下文过滤

  • 包含/排除过滤器

先决条件

在使用索Amazon Kendra引 GitHub 数据源之前，请在您的 GitHub 和AWS账户中进行这些更改。

在 GitHub，请确保您有：

  • 创建了一个对 GitHub 组织具有管理权限的 GitHub 用户。

  • 为身份验证凭证创建了个人访问令牌。请参阅GitHub 有关创建个人访问令牌的文档。

  • 推荐：为身份验证凭证创建了 OAuth 令牌。使用 OAuth 令牌可获得更好的 API 限制和连接器性能。请参阅有关 OAuth 授权的GitHub 文档。

  • 可选：安装了 SSL 证书。

  • 已记下您使用的 GitHub服务类型的 GitHub 主机 URL。例如， GitHub 云的主机 URL 可能是 https://api.github.com， GitHub 服务器的主机 URL 可能是 https:on-prem-host-url///api/v3/。

  • 在 GitHub 设置中记下了存储库的 GitHub 组织名称。

  • 更新了以下权限：

    适用于 GitHub 企业云 (SaaS)

    • repo: 状态

    • public_repo

    • repo: 邀请

    • 阅读:org

    • 用户:电子邮件

    • 读取:用户

    适用于 GitHub 企业服务器（本地）

    • repo: 状态

    • public_repo

    • repo: 邀请

    • 阅读:org

    • 用户:电子邮件

    • 读取:用户

    • 网站管理员

  • 已检查每个文档在您计划用于同一索引的其他数据源中 GitHub 以及在您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的 GitHub 身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或密钥，则可以在将 GitHub 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接指令

要Amazon Kendra连接到您的 GitHub 数据源，您必须提供 GitHub 数据源的必要详细信息，这样Amazon Kendra才能访问您的数据。如果您尚未配置， GitHub 请参Amazon Kendra阅先决条件。

Console

要连接Amazon Kendra到 GitHub

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在 “入门” 页面上，选择 “添加数据源”。

  4. 在添加数据源页面上，选择GitHub 连接器，然后选择添加数据源。

  5. 在 Description details（指定数据来源详细信息）页面上，输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）—为数据来源输入可选说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tags 中，用于添加新标签 —Tags，用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在 Description（定义访问和安全性）页面上，输入以下信息：

    1. GitHub来源 —在GitHub 企业云和GitHub企业服务器之间选择。

    2. GitHub 主机 URL —输入您的 GitHub 主机名。

    3. GitHub 组织名称-输入您的 GitHub组织名称。您可以在您的 GitHub 账户中找到您的组织信息。

    4. AWSSecrets Managersecre t —选择现有密钥或创建新Secrets Manager密钥来存储您的 GitHub 身份验证凭证。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      1. 在 “创建AWSSecrets Manager密钥” 窗口中输入以下信息：

        1. 秘密名称 — 您的秘密名称。前缀 “AmazonKendra-GitHub-” 会自动添加到您的密钥名称中。

        2. 对于GitHub令牌-输入您在 GitHub 账户中创建的身份验证凭证值。

      2. 选择保存。

    5. V@@ irtual Private Cloud (VPC) —您可以选择使用 VPC。如果是，则必须添加子网和 VPC 安全组。

    6. IAM 角色 —选择现有 IAM 角色或创建新的 IAM 角色来访问您的存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    7. 选择下一步。

  7. 在配置同步设置页面上，输入以下信息：

    1. 选择要抓取的存储库-要抓取的 GitHub 实体或内容类型。

    2. 更改日志-选择更新索引，而不是同步所有文件。

    3. 内容类型-选择要包含的文件类型。

    4. 正则表达式@@ 模式 —包含或排除某些文件的正则表达式模式。您最多可以添加 100 个模式。

    5. 在同步运行时间表中，针对频率 — Amazon Kendra 与您的数据源同步的频率。

    6. 选择下一步。

  8. 在 Set 字段映射页面上，输入以下信息：

    1. 对于存储库、存储库提交、问题文档、问题评论、问题附件、拉取请求评论、拉取请求文档、拉取请求附件-从Amazon Kendra生成的默认数据源字段中选择要映射到索引的默认数据源字段。

    2. 添加字段-添加自定义数据源字段以创建要映射到的索引字段名称和字段数据类型。

    3. 选择下一步。

  9. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面进行操作。成功添加数据源后，它将显示在数据源页面上。

API

要连接Amazon Kendra到 GitHub

必须使用 GitHubConfigurationAPI 对象指定以下内容：

  • 数据源类型-将数据源类型指定为SAAS或ON_PREMISE。

  • 秘密亚马逊资源名称 (ARN)-提供包含您 GitHub 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，具有以下密钥：

    {
    "personalToken": "token"
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 — 指定您RoleArn何时调用，CreateDataSource以便为IAM角色提供访问您的Secrets Manager密钥以及调用 GitHub 连接器所需的公共 API 和的权限Amazon Kendra。有关更多信息，请参阅 GitHub 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC)-指定VpcConfiguration为数据源配置的一部分。请参阅配置Amazon Kendra使用 VPC。

    注意

    如果您使用 GitHub 服务器，则必须使用和Amazon VPC才能连接到您的 GitHub服务器。

  • 更改日志-是否Amazon Kendra应使用 GitHub 数据源更改日志机制来确定是否必须在索引中添加、更新或删除文档。

    注意

    如果您不想扫描所有文档Amazon Kendra，请使用更改日志。如果您的更改日志很大，则扫描 GitHub 数据源中的文档所花费的时间可能Amazon Kendra少于处理更改日志所花费的时间。如果您是第一次将 GitHub数据源与索引同步，则会扫描所有文档。

  • 包含和排除过滤器-指定是包含还是排除某些文件。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 GitHub 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

  • 用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或其群组对文档的访问权限筛选搜索结果。有关更多信息，请参阅 GitHub 数据源的用户上下文筛选。
