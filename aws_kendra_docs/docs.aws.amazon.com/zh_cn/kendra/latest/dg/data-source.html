创建数据源连接器

您可以创建数据源连接器Amazon Kendra以连接到文档并为其编制索引。 Amazon Kendra可以连接到微软 SharePoint、谷歌云端硬盘和许多其他提供商。创建数据源连接器时，需要提供Amazon Kendra连接到源存储库所需的配置信息。与直接向索引添加文档不同，您可以定期扫描数据源以更新索引。

例如，假设您在 S3 桶中存储了一个税务文件库。有时，会更改现有文档，并将新文档添加到存储库中。如果将存储库添加Amazon Kendra为数据源，则可以通过在数据源和索引之间设置定期同步来使索引保持最新状态。

您可以选择使用控制台或StartDataSourceSyncJob API 手动更新索引。否则，您可以设置时间表来更新索引并使其与您的数据源同步。

一个索引可以包含多个数据源。每个数据源都可以有自己的更新时间表。例如，您可以每天甚至每小时更新工作文档的索引，而每当存档发生变化时手动更新存档文档。

如果您想在文档摄取过程中更改文档元数据或属性和内容，请参阅Amazon Kendra自定义文档丰富。

注意，每个索引的每个文档 ID 都必须是唯一的。您无法创建数据源来使用文档的唯一 ID 对文档进行索引，然后使用BatchPutDocument API 对相同的文档进行索引，反之亦然。您可以删除数据源，然后使用BatchPutDocument API 为相同的文档编制索引，反之亦然。

设置更新时间表

将您的数据源配置为使用控制台定期更新，或者在创建或更新数据源时使用Schedule参数进行更新。参数的内容是一个字符串，其中包含一个cron-format 调度字符串或一个空字符串，表示索引是按需更新的。有关 cron 表达式的格式，请参阅《Amazon CloudWatch Events用户指南》中的规则调度表达式。 Amazon Kendra仅支持 cron 表达式。它不支持费率表达式。

设置语言

您可以使用支持的语言为数据源中的所有文档编制索引。调用时，您可以为数据源中的所有文档指定语言代码CreateDataSource。如果文档未在元数据字段中指定语言代码，则使用在数据源级别为所有文档指定的语言代码为该文档编制索引。如果您未指定语言，则默认情况下用英语为数据源中的文档Amazon Kendra编制索引。有关支持的语言（包括其代码）的更多信息，请参阅添加英语以外语言的文档。

您可以使用控制台以支持的语言为数据源中的所有文档编制索引。转到数据源并编辑您的数据源，或者如果您要添加新的数据源，请转到添加数据源。在指定数据源详细信息页面上，从语言下拉列表中选择一种语言。选择 Updat e 或继续输入配置信息以连接到您的数据源。
