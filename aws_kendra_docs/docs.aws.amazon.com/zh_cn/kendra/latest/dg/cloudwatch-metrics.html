用亚马逊监控亚马逊 Kendra CloudWatch

要跟踪索引的运行状况，请使用亚马逊 CloudWatch。使用 CloudWatch，您可以获取索引的文档同步指标。您还可以设置 CloudWatch 警报，以便在一个或多个指标超过您定义的阈值时收到通知。例如，您可以监控提交索引的文档数量或未能编入索引的文档数量。

您必须拥有相应的 CloudWatch 权限才能监控Amazon Kendra CloudWatch。有关更多信息，请参阅亚马逊 CloudWatch用户指南 CloudWatch中的亚马逊身份验证和访问控制。

查看 Amazon Kendra 指标

使用 CloudWatch 控制台查看Amazon Kendra 指标。

要查看指标（CloudWatch 控制台）

  1. 登录AWS Management Console并打开 CloudWatch 主机，网址为 https://console.aws.amazon.com/cloudwatch/。

  2. 选择指标，选择所有指标，然后选择 Kendra。

  3. 选择维度、指标名称，然后选择 Add to graph (添加到图表)。

  4. 选择日期范围的值。所选日期范围的指标计数将显示在该图表中。

创建警报

CloudWatch 警报会在指定时间段内监控一个指标，并执行一项或多项操作：向Amazon Simple Notification Service (Amazon SNS) 顶部或 Auto Scaling 策略发送通知。这些操作或操作基于在您指定的多个时间段内相对于给定阈值的指标值。 CloudWatch 还可在警报改变状态时向您发送 Amazon SNS 消息。

CloudWatch 警报仅当状态发生变化并且已持续您指定的时间段时才会触发操作。

设置警报

  1. 登录AWS Management Console并打开 CloudWatch 主机，网址为 https://console.aws.amazon.com/cloudwatch/。

  2. 选择 Alarms，然后选择 Create Alarm。

  3. 选择 Kendra 指标，然后选择一个指标。

  4. 对于 Time Range，请选择要监控的时间范围，然后选择 Next。

  5. 输入名称和描述。

  6. 对于 Whenever，选择 >=，然后键入一个最大值。

  7. 如果您希望在达到警报状态时 CloudWatch 发送一封电子邮件，请在操作部分对于 Whenever this alarm (每当此警报)，请选择 State is ALARM (状态为 “警报”)。对于发送通知至，选择一个邮件列表或选择新建列表并创建一个新的邮件列表

  8. 预览 Alarm Preview 部分中的警报。如果对警报满意，请选择 Create Alarm (创建警报)。

CloudWatch 索引同步作业的指标

下表描述了数据源同步任务的 Amazon Kendra 指标。

指标 描述
DocumentsCrawled

同步作业在运行期间扫描或发现的文档数。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSubmittedForIndexing

同步作业提交到索引的文档数。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSubmittedForIndexingFailed

索引失败的文档数量。有关详细信息，请检查同步作业 CloudWatch 的日志内容。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSubmittedForDeletion

同步作业要求从索引中删除的文档数。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSubmittedForDeletionFailed

未能删除的文档数量。有关详细信息，请检查同步作业 CloudWatch 的日志内容。

维度：

  • IndexId

  • DataSourceId

单位：计数

Amazon Kendra 数据源指标

下表描述了数据源同步任务的 Amazon Kendra 指标。标有星号 (*) 的指标仅用于 Amazon S3 数据源。

指标 描述
DocumentsSkippedNoChange *

检查后发现的文件数量没有变化，因此未提交索引。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSkippedInvalidMetadata *

由于关联的元数据文件存在问题，因此跳过了文档的数量。有关详细信息，请检查同步运行 CloudWatch 的日志内容。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsCrawled

已检查的文档文件数量。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSubmittedForDeletion

已审查但已从数据源中删除并提交删除的文档的数量。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSubmittedForDeletionFailed

从数据源中删除失败的文档数量。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSubmittedForIndexing

已审查并提交索引的文件数量。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsSubmittedForIndexingFailed

提交索引但无法编入索引的文档数量。

维度：

  • IndexId

  • DataSourceId

单位：计数

索引文档的指标

下表描述了索引文档的 Amazon Kendra 指标。对于使用该BatchPutDocument操作编制索引的文档，仅支持IndexId维度。

指标 描述
DocumentsIndexed

索引的文档数。

维度：

  • IndexId

  • DataSourceId

单位：计数
DocumentsFailedToIndex

无法编入索引的文档数量。检查 CloudWatch 日志内容了解详细信息。

维度：

  • IndexId

  • DataSourceId

单位：计数
IndexQueryCount

每分钟索引查询的数量。

维度：

  • IndexId

单位：计数
