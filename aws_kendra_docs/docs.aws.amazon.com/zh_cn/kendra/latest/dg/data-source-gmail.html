Gmail的

Gmail是Google开发的电子邮件客户端，您可以通过它发送带有文件附件的电子邮件。可以使用文件夹和标签对Gmail邮件进行排序并存储在您的电子邮件收件箱中。您可以使用索Amazon Kendra引您的电子邮件和邮件附件。您也可以配置Amazon Kendra为包括或排除特定的电子邮件、邮件附件和用于索引的标签。

您可以使用Amazon Kendra控制台和 TemplateConfigurationAPI 连接Amazon Kendra到 Gmail 数据源。

要对Amazon Kendra Gmail 数据源连接器进行故障排除，请参阅解决数据源问题。

先决条件

在使用Amazon Kendra索引 Gmail 数据源之前，请先在 Gmail 和AWS帐户中进行这些更改。

在 Gmail 中，请确保您具有：

  • 创建了 Google 云平台管理员帐户并创建了 Google 云项目。

  • 在您的管理员账户中激活了 Gmail API 和管理员 SDK API。

  • 创建了一个服务帐号并下载了 Gmail 的 JSON 私钥。有关如何创建和访问私钥的信息，请参阅有关如何创建服务帐号密钥和服务帐号凭据的 Google Cloud 文档。

  • 复制了您的管理员帐户电子邮件、服务帐户电子邮件和用于身份验证的私钥。

  • 为您的用户和要索引的共享目录添加了以下 Oauth 作用域（使用管理员角色）：

    • https://www.googleapis.com/auth/admin.directory.user.readonly

    • https://www.googleapis.com/auth/gmail.readonly

  • 已检查每个文档在 Gmail 以及您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的 Gmail 身份验证凭据存储在AWSSecrets Manager密钥中，如果使用 API，请记下该密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或机密，则可以在将 Gmail 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接说明进行操作

要Amazon Kendra连接到您的 Gmail 数据源，您必须提供您的 Gmail 凭据的详细信息，Amazon Kendra以便访问您的数据。如果您尚未将 Gmail 配置为Amazon Kendra，请参阅先决条件。

Console

连接Amazon Kendra到 Gmail

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在 “入门” 页面上，选择 “添加数据源”。

  4. 在 “添加数据源” 页面上，选择 Gmail 连接器，然后选择 “添加数据源”。

  5. 在指定数据来源详细信息页面输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）说明 —为数据来源输入可选说明进行操作。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tags 中，为 A dd new tag 进行操作，用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在定义访问和安全页面输入以下信息：

    1. 在 “AWSSecrets Manager密钥身份验证” 中-选择现有密钥或创建新Secrets Manager密钥来存储您的 Gmail 身份验证凭据。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      1. 在 “创建AWSSecrets Manager密钥” 窗口中输入以下信息：

        1. 机密名称-您的秘密的名称。

        2. 客户电子邮件 — 您从 Google 服务帐号中复制的客户电子邮件。

        3. 管理员帐户电子邮件-您要使用的管理员帐户电子邮件。

        4. 私钥 — 您从 Google 服务帐号复制的私钥。

        5. 选择保存。

    2. V@@ irtual Private Cloud (VPC) —您可以选择使用 VPC。如果是，则必须添加子网和 VPC 安全组。

    3. IAM 角色 —选择现有 IAM 角色或创建新的 IAM 角色来访问您的存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    4. 选择下一步。

  7. 在配置同步设置页面输入以下信息：

    1. 在同步作用域中，对于实体类型-选择消息附件以同步消息附件。默认情况下，消息将同步。

    2. （可选）对于 A dddit ion 进行操作，请输入以下信息：

      1. 日期范围-输入日期范围以指定要抓取的电子邮件的开始和结束日期。

      2. 电子邮件域-包括或排除基于域的电子邮件。

      3. 主题中的关键字-根据主题中的关键字包括或排除电子邮件。

        注意

        您也可以选择包含与您输入的所有主题关键字相匹配的任何文档。

      4. 标签-添加正则表达式模式以包含或排除特定标签。您最多可以添加 100 个模式。

      5. 附件-添加正则表达式模式以包含或排除特定附件。您最多可以添加 100 个模式。

    3. 对于同步模式，选择在数据源内容发生变化时要如何更新索引。当您Amazon Kendra首次与同步数据源时，默认情况下所有内容都会同步。

      • 完全同步 —无论之前的同步状态如何，都同步所有内容。

      • 新增、修改或删除的内容同步-仅同步新的、修改的和删除的内容。

        重要

        由于没有用于更新永久删除的 Gmail 邮件的 API，因此会同步新内容、修改内容或已删除内容：

        • 不会从您的Amazon Kendra索引中移除从 Gmail 中永久删除的邮件

        • 无法同步 Gmail 电子邮件标签中的更改

        要将您的 Gmail 数据源标签更改和永久删除的电子邮件同步到您的Amazon Kendra索引，您必须定期进行全面抓取。

    4. 在同步运行计划中，针对频率 —与数据源同步的频率。Amazon Kendra

    5. 选择下一步。

  8. 在 Set Feails（设置字段映射）页面上，输入以下信息：

    1. 对于消息和消息附件-从Amazon Kendra生成的默认数据源字段中选择要映射到索引的字段。

      注意

      Amazon Kendra由于 API 限制，Gmail 数据源连接器不支持创建自定义索引字段。

    2. 选择下一步。

  9. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面进行操作。成功添加数据源后，该数据源将显示在数据源页面上。

TemplateConfiguration API

连接Amazon Kendra到 Gmail

您必须使用 TemplateConfigurationAPI 指定以下内容：

  • 数据源-必须将数据源指定为GMAIL。

  • 数据源架构-包括包含数据源架构的 JSON。要查看模板架构，请参阅数据源架构。

  • 类型 —在呼叫时指定TEMPLATE为类型CreateDataSource。

  • 秘密亚马逊资源名称 (ARN)-提供包含您的 Gmail 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，具有以下密钥：

    {
    "adminAccountEmailId": "${adminAccountEmailId}",
    "clientEmailId": "${clientEmailId}",
    "privateKey": "${privateKey}"
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 — 指定您RoleArn何时调用，CreateDataSource以便为IAM角色提供访问您的Secrets Manager密钥和调用 Gmail 连接器所需的公共 API 的权限，以及Amazon Kendra. 有关更多信息，请参阅 Gmail 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC) —VpcConfiguration 指定进行CreateDataSource操作。请参阅配置Amazon Kendra为使用 VPC。

  • 同步模式-指定是Amazon Kendra应通过同步所有文档或仅同步新的、修改的和已删除的文档来更新索引。您可以进行选择：

    • FORCED_FULL_CRAWL在每次数据源与索引同步时重新抓取所有内容并替换现有内容

    • FULL_CRAWL每次数据源与索引同步时，仅增量抓取新内容、修改内容和已删除内容

    重要

    由于没有 API 可以更新永久删除的 Gmail 邮件，因此同步FULL_CRAWL /新内容、修改内容或已删除内容：

    • 不会从您的Amazon Kendra索引中移除从 Gmail 中永久删除的邮件

    • 无法同步 Gmail 电子邮件标签中的更改

    要将您的 Gmail 数据源标签更改和永久删除的电子邮件同步到您的Amazon Kendra索引，您必须定期进行全面抓取。

  • 包含和排除过滤器-您可以指定是否包含附件。您还可以指定正则表达式模式以包含或排除特定的消息和附件。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被编入索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 Gmail 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

    注意

    Amazon Kendra由于 API 限制，Gmail 数据源连接器不支持创建自定义索引字段。
