Amazon FSx

Amazon FSx 是一个完全托管的基于云的文件服务器系统，提供共享存储功能。如果您是亚马逊 FSx 用户，则可以使用Amazon Kendra索引您的亚马逊 FSx 数据源。

Amazon Kendra目前仅支持 WAmazon FSx indows 文件服务器。

您可以使用Amazon Kendra控制台和 FsxConfigurationAPIAmazon Kendra 连接到您的 Amazon FSx 数据源。

要对您的Amazon Kendra Amazon FSx 数据源连接器进行故障排除，请参阅解决数据源问题。

支持的功能

Amazon KendraAmazon FSx 数据源连接器支持以下功能：

  • 更改日志

  • 字段映射

  • 用户上下文过滤

  • 包含/排除过滤器

  • Virtual Private Cloud (VPC)

先决条件

在使用Amazon Kendra索引您的 Amazon FSx 数据源之前，请在您的 Amazon FSx 和AWS账户中进行这些更改。

在 Amazon FSx，请确保您有：

  • 具有读取和装载权限的亚马逊 FSx 账户。

  • 已记下您的活动目录用户帐户的Amazon FSx身份验证凭据。这包括您的 Active Directory 用户名和域名系统 (DNS) 域名。例如，user@corp.example.com。

  • 已复制您的Amazon FSx文件系统 ID 值。

  • 使用了您Amazon FSx所在AWS的Amazon VPC (VPC)。

  • 已检查每个文档在 Amazon FSx 以及您计划用于同一索引的其他数据源中是唯一的。要用于索引的每个数据源都不得在数据源中包含相同的文档。文档 ID 是索引的全局标识，每个索引必须是唯一的。

在您的AWS账户中，请确保您有：

  • 创建了Amazon Kendra索引，如果使用 API，则记下索引 ID。

  • 为您的数据源@@ 创建了一个IAM角色，如果使用 API，则记下该IAM角色的 ARN。

  • 将您的 Amazon FSx 身份验证凭证存储在AWSSecrets Manager密钥中，如果使用 API，请记下该密钥的 ARN。

    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

如果您没有现有的IAM角色或密钥，则可以在将 Amazon FSx 数据源连接到时使用控制台创建新的IAM角色和Secrets Manager密钥Amazon Kendra。如果您使用的是 API，则必须提供现有IAM角色和Secrets Manager密钥的 ARN 以及索引 ID。

连接指令

要Amazon Kendra连接到您的 Amazon FSx 数据源，您必须提供 Amazon FSx 数据源的必要详细信息，Amazon Kendra以便访问您的数据。如果您尚未配置 Amazon FSx，请参Amazon Kendra阅先决条件。

Console

Amazon Kendra连接到亚马逊 FSx

  1. 登录 AWS 管理控制台，打开 Amazon Kendra 控制台。

  2. 在左侧导航窗格中，选择索引，然后从索引列表中选择要使用的索引。

    注意

    您可以选择在 “索引设置” 下配置或编辑您的用户访问控制设置。

  3. 在入门页面上，选择添加数据源。

  4. 在添加数据源页面上，选择 Amazon FSx 连接器，然后选择添加数据源。

  5. 在 S 指定数据来源详细信息页面上，输入以下信息：

    1. 在名称和描述中，输入数据源名称-输入数据源的名称。您可以包含连字符，但不能包含空格。

    2. （可选）—为数据来源输入可选的说明。

    3. 在 “语言” 中，为 “默认语言” — 一种用于筛选文档以获取索引的语言。除非您另行指定，否则语言默认为英语。元数据中指定的语言会覆盖所选语言。

    4. 在 Tags（添加新标签）中，用于搜索和筛选您的资源或跟踪AWS成本。

    5. 选择下一步。

  6. 在 Description（定义访问和安全性）页面上，输入以下信息：

    1. 来源 — Amazon FSx为 Windows 文件服务器选择。

    2. Amazon FSx文件系统 ID —选择您的文件系统 ID 或创建新目录。

    3. AWSSecrets Managers@@ ec ret —选择现有密钥或创建新Secrets Manager密钥来存储您的 Amazon FSx 身份验证凭证。如果您选择创建新密钥，则会打开一个AWSSecrets Manager秘密窗口。

      1. 在 “创建AWSSecrets Manager密钥” 窗口中输入以下信息：

        1. 秘密名称 — 您的秘密名称。前缀 “AmazonKendra-Amazon FSX” 会自动添加到您的密钥名称中。

        2. 对于用户名-输入 AcAmazon FSx tive Directory 帐户的用户名。

        3. 输入密码 —输入 AcAmazon FSx tive Directory 帐户的密码。

      2. 选择保存。

    4. V@@ irtual Private Cloud (VPC) — 您的Amazon FSx数据源必须使用 VPC。您必须添加子网和 VPC 安全组。

    5. IAM 角色 — 选择现有 IAM 角色，或者创建新的 IAM 角色，以访问存储库证书和索引内容。

      注意

      用于索引的 IAM 角色不能用于数据源。如果您不确定现有角色是否用于索引或常见问题解答，请选择创建新角色以避免错误。

    6. 选择下一步。

  7. 在配置同步设置页面上，输入以下信息：

    1. 更改日志-选择更新索引，而不是同步所有文件。

    2. 正则@@ 表达式模式-添加正则表达式模式以包含或排除某些内容。您最多可以添加 100 个模式。

    3. 在同步运行计划中，针对频率-选择与数据源同步的频率。Amazon Kendra

    4. 选择下一步。

  8. 在 Set 字段映射页面上，输入以下信息：

    1. Amazon FsX Windows 字段映射 —从Amazon Kendra生成的默认数据源字段中选择要映射到索引的字段。

    2. 添加字段-添加自定义数据源字段以创建要映射到的索引字段名称和字段数据类型。

    3. 选择下一步。

  9. 在 “查看并创建” 页面上，检查您输入的信息是否正确，然后选择 “添加数据源”。您也可以选择从此页面编辑信息。成功添加数据源后，它将显示在数据源页面上。

API

Amazon Kendra连接到亚马逊 FSx

您必须使用 FsxConfigurationAPI 指定以下内容：

  • 秘密亚马逊资源名称 (ARN)-提供包含您的 Amazon FSx 账户身份验证凭证的Secrets Manager密钥的亚马逊资源名称 (ARN)。密钥存储在 JSON 结构中，具有以下密钥：

    {
    "username": "user@corp.example.com",
    "password": "password"
}
    注意

    请务必定期刷新或轮换您的证书和密钥。为了您的安全，仅提供必要的访问级别。不建议在数据源和连接器版本 1.0 和 2.0（如果适用）之间重复使用凭据和密钥。

  • IAM 角色 — 指定RoleArn何时调用CreateDataSource，为IAM角色提供访问您的Secrets Manager密钥和调用 Amazon FSx 连接器所需的公共 API 的权限，以及Amazon Kendra。有关更多信息，请参阅 Amazon FSx 数据来源的 IAM 角色。

您还可以添加以下可选功能：

  • V@@ irtual Private Cloud (VPC)-指定VpcConfiguration为数据源配置的一部分。请参阅配置Amazon Kendra使用 VPC。

  • 更改日志 —是否Amazon Kendra应使用 Amazon FSx 数据源更改日志机制来确定是否必须在索引中添加、更新或删除文档。

    注意

    如果您不想扫描所有文档Amazon Kendra，请使用更改日志。如果您的变更日志很大，扫描 Amazon FSx 数据源中的文档所花费的时间可能Amazon Kendra少于处理变更日志所花费的时间。如果您是第一次将 Amazon FSx 数据源与索引同步，则会扫描所有文档。

  • 包含和排除过滤器-您可以指定是包含还是排除某些内容和内容类型。

    注意

    大多数数据源使用正则表达式模式，即被称为过滤器的包含或排除模式。如果您指定包含过滤器，则只有与包含过滤器匹配的内容才会被索引。任何与包含筛选条件不匹配的文档都不会被编入索引。如果您指定包含和排除筛选器，则即使与排除筛选器匹配的文档也不会编入索引。

  • 字段映射-选择将 Amazon FSx 数据源字段映射到Amazon Kendra索引字段。有关更多信息，请参阅映射数据来源字段。

注意

用户上下文筛选-默认情况下，Amazon Kendra抓取数据源的访问控制列表 (ACL)。ACL 信息用于根据用户或群组对文档的访问权限筛选用户的结果。

要测试用户的用户上下文筛选，在发出查询时，必须将 DNS 域名作为用户名的一部分。您必须拥有活动目录域的管理权限。您还可以对组名测试用户上下文筛选。有关更多信息，请参阅 Amazon FSx 数据源的用户上下文筛选。
