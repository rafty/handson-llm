精选搜索结果

当用户发出某些查询时，您可以在搜索结果中显示某些文档。这有助于使结果对您的用户来说更加可见和突出。精选结果与通常的结果列表分开，显示在搜索页面的顶部。您可以尝试针对不同的查询显示不同的文档，或者确保某些文档获得应有的可见性。

您可以将特定查询映射到特定文档，以便在结果中显示。如果查询包含完全匹配的内容，则搜索结果中会显示一个或多个特定文档。

例如，您可以指定，如果您的用户发出 “2023 年新产品” 查询，则选择标题为 “最新消息” 和 “即将推出” 的文档以显示在搜索结果页面的顶部。这有助于确保这些有关新产品的文档获得应有的知名度。

Amazon Kendra如果已选择将结果显示在搜索结果页面的顶部，则不会复制搜索结果。如果精选结果已经排在所有其他结果之上，则该结果不会再次被列为第一个结果。

为了显示某些结果，必须指定全文查询的精确匹配，而不是使用查询中包含的关键字或短语对查询的部分匹配。例如，如果您仅在精选结果集中指定查询 “Kendra”，则诸如 “Kendra 如何对结果进行语义排名？” 之类的查询 不会呈现精选结果。精选结果是针对特定查询而设计的，而不是针对范围太宽的查询而设计的。 Amazon Kendra自然地处理关键字类型查询以对搜索结果中最有用的文档进行排名，避免根据简单关键字过多地显示结果。

如果您的用户经常使用某些查询，则可以为精选结果指定这些查询。例如，如果您使用 Amazon KendraAnalytics 查看您的热门查询并发现特定查询，例如 “kendra 如何对结果进行语义排名？” 和 “kendra 语义搜索” 是经常使用的，那么这些查询对于指定标题为 “searAmazon Kendra ch 101” 的文档可能很有用。

Amazon Kendra将精选结果的查询视为不区分大小写。 Amazon Kendra将查询转换为小写，并将结尾的空格字符替换为单个空格。 Amazon Kendra匹配所有其他字符，就像您指定对精选结果的查询时一样。

您可以使用 CreateFeaturedResultsSetAPI 创建一组精选结果，将其映射到某些查询。如果您使用控制台，则选择索引，然后在导航菜单中选择精选结果以创建精选结果集。每个索引最多可以创建 50 组精选结果，每组最多可以创建 4 个精选文档，每个精选结果集最多可以创建 49 个查询文本。您可以联系Support 部门请求提高这些限制。

您可以在多组精选结果中选择同一个文档。但是，不得在多个集合中使用相同的精确匹配查询文本。对于每个索引的每个精选结果集，您为精选结果指定的查询必须是唯一的。

最多选择四个精选文档时，您可以排列文档的顺序。如果您使用 API，则列出精选文档的顺序与精选结果中显示的顺序相同。如果您使用控制台，则在选择要在结果中显示的文档时，只需拖放文档的顺序即可。

在配置精选结果时，访问控制，即某些用户和群组有权访问某些文档，而其他用户和群组则不能。对于用户上下文筛选也是如此。例如，用户 A 属于 “实习生” 公司组，该群组不应访问有关公司机密的文档。如果用户 A 输入的查询包含公司机密文档，则用户 A 在其结果中看不到该文档。搜索结果页面上的任何其他结果也是如此。您还可以使用标签来控制对精选结果集的访问权限，精选结果集是您可以控制访问权限的Amazon Kendra资源。

以下是创建一组精选结果的示例，其中查询 “新产品 2023”、“新产品上市” 映射到标题为 “新增内容” (doc-id-1) 和 “即将上市” (doc-id-2) 的文档。

CLI
aws kendra create-featured-results-set \
 --featured-results-set-name 'New product docs to feature' \
 --description "Featuring What's new and Coming soon docs" \
 --index-id index-id \
 --query-texts 'new products 2023' 'new products available' \
 --featured-documents '{"Id":"doc-id-1", "Id":"doc-id-2"}'
Python
import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Create a featured results set.")

# Provide a name for the featured results set
featured_results_name = "New product docs to feature"
# Provide an optional decription for the featured results set
description = "Featuring What's new and Coming soon docs"
# Provide the index ID for the featured results set
index = index-id
# Provide a list of query texts for the featured results set
queries = ['new products 2023', 'new products available']
# Provide a list of document IDs for the featured results set
featured_doc_ids = ['doc-id-1', 'doc-id-2']

try:
    featured_results_set_response = kendra.create_featured_results_set(
        FeaturedResultsSetName = featured_results_name,
        Decription = description,
        Index = index,
        QueryTexts = queries,
        FeaturedDocuments = featured_doc_ids
    )

    pprint.pprint(featured_results_set_response)

    featured_results_set_id = featured_results_set_response["FeaturedResultsSetId"]

    while True:
        # Get the details of the featured results set, such as the status
        featured_results_set_description = kendra.describe_featured_results_set(
            Id = featured_results_set_id
        )
        status = featured_results_set_description["Status"]
        print(" Featured results set status: "+status)
            
except  ClientError as e:
        print("%s" % e)

print("Program ends.")
