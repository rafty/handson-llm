自定义数据源连接器

当您的存储库尚未提供数据源连接器时，Amazon Kendra请使用自定义数据源。即使您无法使用Amazon Kendra数据源同步存储库，也可以使用它Amazon Kendra来查看数据源提供的相同运行历史指标。使用它在Amazon Kendra数据源和自定义数据源之间创建一致的同步监控体验。具体而言，使用自定义数据源查看您使用BatchPutDocument和 BatchDeleteDocumentAPI 创建的数据源连接器的同步指标。

有关监控 Amazon Kendra 自定义数据源连接器的问题，请参阅解决数据源问题。

创建自定义数据源时，您可以完全控制如何选择要索引的文档。 Amazon Kendra仅提供了可用来监控的指标信息。您必须创建并运行用于确定您的数据源索引文档的 Crawler。

您必须使用 Document 对象指定文档的主标题，以便_source_uriDocumentAttribute在结果的响应中DocumentURI包含DocumentTitle并包含在Query结果的响应中。

您可以使用控制台或 CreateDataSourceAPI 为自定义数据源创建标识符。要使用控制台，请为数据源命名，也可以提供描述和资源标签。创建数据源后，将显示数据源 ID。复制此 ID 以在将数据源与索引同步时使用。

您还可以使用CreateDataSource API 来创建自定义数据源。API 返回一个 ID，供您在同步数据源时使用。当您使用CreateDataSource API 创建自定义数据源时，您无法设置Configuration、RoleArn或Schedule参数。如果您设置了这些参数，则Amazon Kendra返回ValidationException异常。

要使用自定义数据源，请创建负责更新Amazon Kendra索引的应用程序。该应用程序依赖于您创建的爬虫。搜寻器读取存储库中的文档并确定应发送到哪个文档Amazon Kendra。您的应用程序应执行以下步骤：

  1. 抓取存储库并列出存储库中添加、更新或删除的文档。

  2. 调用 StartDataSourceSyncJobAPI 以发出同步任务正在启动的信号。您提供数据源 ID 以标识正在同步的数据源。 Amazon Kendra返回执行 ID 以标识特定的同步任务。

  3. 调用 BatchDeleteDocumentAPI 从索引中删除文档。您提供数据源 ID 和执行 ID 以标识正在同步的数据源以及此更新所关联的作业。

  4. 调用 StopDataSourceSyncJobAPI 以发出同步任务结束的信号。调用StopDataSourceSyncJob API 后，关联的执行 ID 不再有效。

  5. 调用带有索引和数据源标识符的 ListDataSourceSyncJobsAPI 以列出数据源的同步任务并查看同步任务的指标。

结束同步作业后，可以启动新的同步作业。可能需要一段时间才能将所有提交的文档添加到索引中。使用ListDataSourceSyncJobs API 查看同步任务的状态。如果同步任务的Status返回值为SYNCING_INDEXING，则某些文档仍在索引中。当先前作业的状态为FAILED、SUCCEEDED或时，您可以启动新的同步作业SYNCING_INDEX。

调用StopDataSourceSyncJob API 后，您无法在调用BatchPutDocument或BatchDeleteDocument API 时使用同步任务标识符。如果您这样做，则所有提交的文档都将在 API 的FailedDocuments响应消息中返回。

必需的属性

当您Amazon Kendra使用BatchPutDocument API 向提交文档时，每个文档都需要两个属性来标识其所属的数据源和同步运行。您必须提供以下两个属性：

  • _data_source_id—数据源的标识符。当您使用控制台或CreateDataSource API 创建数据源时，会返回此值。

  • _data_source_sync_job_execution_id—同步运行的标识符。当您开始与StartDataSourceSyncJob API 进行索引同步时会返回此值。

以下是使用自定义数据源索引文档所需的 JSON。

{
    "Documents": [
        {
            "Attributes": [
                {
                    "Key": "_data_source_id",
                    "Value": {
                        "StringValue": "data source identifier"
                    }
                },
                {
                    "Key": "_data_source_sync_job_execution_id",
                    "Value": {
                        "StringValue": "sync job identifier"
                    }
                }
            ],
            "Blob": "document content",
            "ContentType": "content type",
            "Id": "document identifier",
            "Title": "document title"
        }
    ],
    "IndexId": "index identifier",
    "RoleArn": "IAM role ARN"
}

使用BatchDeleteDocument API 从索引中删除文档时，需要在DataSourceSyncJobMetricTarget参数中指定以下两个字段：

  • DataSourceId—数据源的标识符。当您使用控制台或CreateDataSource API 创建数据源时，会返回此值。

  • DataSourceSyncJobId—同步运行的标识符。当您开始与StartDataSourceSyncJob API 进行索引同步时会返回此值。

以下是使用BatchDeleteDocument API 从索引中删除文档所需的 JSON。

{
    "DataSourceSyncJobMetricTarget": {
        "DataSourceId": "data source identifier",
        "DataSourceSyncJobId": "sync job identifier"
    },
    "DocumentIdList": [
        "document identifier"
    ],
    "IndexId": "index identifier"
}

查看 指标

同步任务完成后，您可以使用 DataSourceSyncJobMetricsAPI 获取与同步任务相关的指标。使用它来监控您的自定义数据源同步。

如果您多次提交同一个文档，要么作为BatchPutDocument API、API 的一部分，要么提交该文档是为了添加和删除而提交，则该文档在指标中仅计入一次。BatchDeleteDocument

  • DocumentsAdded— 使用与此同步任务关联的BatchPutDocument API 提交的文档数首次添加到索引中。如果在同步过程中多次提交文档以供添加，则该文档在指标中仅计入一次。

  • DocumentsDeleted—使用与该同步任务关联BatchDeleteDocument的 API 提交的文档数量已从索引中删除。如果在同步中多次提交要删除的文档，则该文档在指标中仅计入一次。

  • DocumentsFailed— 与此同步作业关联但索引失败的文档数量。这些文档已被接受编入索引Amazon Kendra，但无法编入索引或删除。如果文档未被接受Amazon Kendra，则该文档的标识符将在BatchPutDocument和BatchDeleteDocument API 的FailedDocuments响应属性中返回。

  • DocumentsModified— 使用与此同步任务关联BatchPutDocument的 API 提交的在Amazon Kendra索引中修改的修改文档的数量。

Amazon Kendra还会在索引文档时发出Amazon CloudWatch指标。有关更多信息，请参阅 “监控Amazon Kendra的方式”Amazon CloudWatch。

Amazon Kendra不返回自定义数据源的DocumentsScanned指标。它还会发布Amazon Kendra数据源CloudWatch指标文档中列出的指标。
