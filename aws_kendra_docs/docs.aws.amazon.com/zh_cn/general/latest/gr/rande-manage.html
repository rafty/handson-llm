管理 AWS 区域

AWS 区域是地理区域中的AWS资源集合。每个 AWS 区域 都是隔离的，独立于其他区域。区域提供容错能力、稳定性和弹性，还可以减少延迟。您可在多个区域预置资源，使您的应用程序保持可用且不受区域中断的影响。

除非您明确使用 AWS 服务提供的复制功能，否则在一个区域中创建的资源在任何其他区域中都不存在。例如，Amazon S3 和 Amazon EC2 支持跨区域复制。某些服务（例如 AWS Identity and Access Management (IAM)）没有区域资源。

您可以使用策略条件来控制对AWS 服务中的访问权限AWS 区域。

您的账户确定了适用于您的区域。

  • AWS 账户提供多个区域，因此您可在满足您要求的位置启动AWS资源。例如，您可能希望在欧洲区域启动实例以更多符合欧洲客户的要求或满足法律要求。

  • AWSGovCloud（美国西部）账户提供对AWSGovCloud（美国西部）区域的AWSGovCloud访问。有关更多信息，请参阅AWS GovCloud (US)：

  • 您只能通过 Amazon（AWS中国）账户访问北京和宁夏区域。有关更多信息，请参阅 中国的 Amazon Web Services。

在启用一个区域时，AWS 将执行操作以准备您在该区域内的账户，例如将您的 IAM 资源分发给该区域。对大多数账户而言，此过程需要几分钟时间，但也有可能要用数小时的时间。在此过程完成之前，您无法使用区域。

要求

要启用区域，您必须具有相应的权限。要查看示例 IAM policy，请参阅《IAM 用户指南》AWS 区域中的允许启用和禁用。

使用控制台启用区域

  1. 登录到 AWS Management Console。

  2. 在控制台的右上角，选择当前区域以展开区域列表，滚动到列表底部，然后选择管理区域。

  3. 选中要使用的区域的 “操作” 列。

    • 如果默认情况下，“状态” 列包含 “启用”，则无需进一步操作。区域已自动为您的账户启用该区域。

    • 如果 “状态” 列包含 “禁用”，请选择 “启用”。检查信息说明文本，然后选择 Enable Region（启用区域）。您必须等到该区域准备就绪可用。

要启用区域，请使用AWS CLI

使用以下 enable-regi on 命令。

aws account enable-region --region-name af-south-1

禁用区域

如果您启用了未自动启用的区域，则可以在需要时禁用该区域。禁用区域后，基于最终一致性，该区域中的资源变为不可用。但是，资源不会被删除。如果您再次启用此区域，则可以访问它们。

要求

  • 您无法禁用默认启用的区域。（请参阅前一部分中的表。）

  • 要禁用区域，您必须具有禁用区域的权限。要查看示例 IAM policy，请参阅《IAM 用户指南》中的允许启用和禁用 AWS 区域。

  • 在禁用区域之前，我们建议您删除该区域中的所有资源。禁用区域后，您无法再通过管理控制台或具有 IAM 主体AWS的 API 从该账户查看或AWS管理该区域中的资源。但是，该区域中的资源仍会产生费用。有关更多信息，请参阅《AWS Billing and Cost Management 用户指南》中的启用和禁用区域。

使用控制台禁用区域

  1. 登录到 AWS Management Console。

  2. 在控制台的右上角，选择当前区域以展开区域列表，滚动到列表底部，然后选择管理区域。

  3. 在区域的 “操作” 列中，选择 “禁用”。

  4. 在显示的对话框中，检查信息说明文本，然后选择 Disable Region（禁用区域）。

要禁用区域，请使用AWS CLI

使用以下 d isable-reg ion 命令。

aws account disable-region --region-name af-south-1

使用描述您的区域AWS CLI

使用 list-re gions 命令描述适用于您的账户的所有区域，包括其状态。

aws account list-regions --max-results 50

使用get-region-opt-status命令描述特定区域的状态。

aws account get-region-opt-status --region-name af-south-1

对于为您的账户自动启用的区域，这些命令的输出包括以下内容：

"RegionOptStatus": "ENABLED_BY_DEFAULT"

对于必须手动启用的区域，输出包括以下内容：

"RegionOptStatus": "DISABLED"

启用区域后，输出包括以下内容：

"RegionOptStatus": "ENABLED"
