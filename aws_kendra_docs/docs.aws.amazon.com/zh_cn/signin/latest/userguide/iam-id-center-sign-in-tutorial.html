登录AWS访问门户

IAM 身份中心中的用户是的成员AWS Organizations。IAM Identity Center 中的用户可以通过使用特定的登录 URL 登录AWS访问门户来访问多个AWS 账户和业务应用程序。有关特定登录 URL 的更多信息AWS 访问门户。

在 IAM Ident AWS 账户 ity Center 中以用户身份登录之前，请收集以下必需信息。

  • 企业用户名

  • 企业密码

  • 特定登录 URL

注意

登录后，您的AWS访问门户会话的有效期为 8 小时。您需要在 8 小时后重新登录。

登录AWS访问门户

  1. 在浏览器窗口中，粘贴通过电子邮件获得的登录 URL，例如https://your_subdomain.awsapps.com/start。然后，按 E nter。

  2. 使用您的公司凭证（例如用户名和密码）登录。

    注意

    如果您的管理员向您发送了一次性电子邮件密码 (OTP)，并且这是您首次登录，请输入该密码。登录后，必须创建新密码以备future 登录。

  3. 如果要求您提供验证码，请检查您的电子邮件中是否有验证码。然后复制代码并将其粘贴到登录页面中。

    注意

    验证码通常通过电子邮件发送，但送达方式可能有所不同。如果您的电子邮件中没有收到验证码，请咨询您的管理员，了解有关您的验证码的详细信息。

  4. 如果在 IAM 身份中心为您的用户启用了 MFA，则您可以使用它进行身份验证。

  5. 身份验证后，您可以访问门户中出现的任何AWS 账户和应用程序。

    1. 要登录，AWS Management Console请选择AWS 账户磁贴，然后选择要管理的个人帐户。

      将显示您的用户拥有的权限以及用于打开管理控制台或获取命令行或编程访问凭证的链接。

      选择管理控制台链接。

    2. 要登录应用程序，请选择要访问的应用程序的图标。

以用户身份登录 IAM Identity Center 可为您提供在设定的时间段（称为会话）内访问资源的证书。默认情况下，用户可以在一小时内登录。AWS 账户管理员可以为会话设置的最大值为 12 小时。会话结束后，您可以再次登录。
