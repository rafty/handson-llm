示例：为Amazon Kendra 索引创建常见问题解答机器人

此示例创建了一个 Amazon Lex V2 机器人，该机器人使用Amazon Kendra 索引为用户的问题提供答案。常见问题解答自动程序为用户管理对话。它使用 AMAZON.KendraSearchIntent 目的查询索引并向用户提供响应。以下是如何使用Amazon Kendra 索引创建常见问题机器人的摘要：

  1. 创建一个自动程序，您的客户将与其交互以从其获取答案。

  2. 创建自定义目的。由于AMAZON.KendraSearchIntent和AMAZON.FallbackIntent是备用意图，因此您的机器人需要至少一个其他意图，并且必须包含至少一个语句。此目的使您能够构建自动程序，但不用于其他方面。因此，您的常见问题解答机器人将包含至少三种意图，如下图所示：

  3. 将AMAZON.KendraSearchIntent意图添加到您的机器人并将其配置为与您的 Amazon Kendra 索引一起使用。

  4. 通过进行查询并验证您的 Amazon Kendra 索引的结果是否是回答查询的文档来测试机器人。

先决条件

您需要先创建Amazon Kendra 索引，然后才能使用此示例。有关更多信息，请参阅亚马逊 Kendra 开发者指南中的亚马Amazon Kendra 主机入门。在此示例中，选择示例数据集（示例 AWS 文档）作为您的数据源。

要创建常见问题机器人，请执行以下操作：

  1. 登录到，AWS Management Console然后通过以下网址打开 Amazon Lex 控制台：https://console.aws.amazon.com/lex/。

  2. 在导航窗格中，选择自动程序。

  3. 选择创建机器人。

    1. 对于创建方法，选择创建空白机器人。

    2. 在机器人配置部分中，为机器人指定一个表明其用途的名称KendraTestBot，例如和可选描述。名称在您账户中必须是唯一的。

    3. 在 IAM 权限部分中，选择创建具有基本 Amazon Lex 权限的角色。这将创建一个 AWS Identity and Access Management(IAM) 角色，该角色具有 Amazon Lex V2 运行您的机器人所需的权限。

    4. 在《儿童在线隐私保护法》(COPPA) 部分中，选择否。

    5. 在 “空闲会话超时” 和 “高级设置” 部分中，保留默认设置并选择 “下一步”。

    6. 现在您进入了 “向机器人添加语言” 部分。在 “语音交互” 下的菜单中，选择 “无”。这只是一个基于文本的应用程序。对于其余字段保留原定设置。

    7. 选择完成。Amazon Lex V2 会创建您的机器人和一个名为的默认意图 NewIntent，并将您带到配置此意图的页面

要成功构建机器人，必须创建至少一个AMAZON.FallbackIntent与和分开的意图AMAZON.KendraSearchIntent。此意图是构建 Amazon Lex V2 机器人所必需的，但不用于常见问题解答回复。此意图必须包含至少一个示例话语，并且该话语不得适用于客户提出的任何问题。

要创建所需的意图，请执行以下操作：

  1. 在意图详细信息部分中，为意图命名，例如RequiredIntent。

  2. 在示例话语部分中，在添加话语旁边的框中键入话语，例如Required utterance。然后选择添加话语。

  3. 选择保存目的。

创建搜索Amazon Kendra 索引及其应返回的响应消息的意图。

要创建 AZON。 KendraSearchIntent 意图和响应消息：

  1. 在导航窗格中选择 “返回意图列表” 以返回机器人的 “意图” 页面。选择 “添加意图”，然后从下拉菜单中选择 “使用内置意图”。

  2. 在弹出的框中，选择 “内置意图” 下的菜单。AMAZON.KendraSearchIntent在搜索栏中输入，然后从列表中选择它。

  3. 为意图命名，例如KendraSearchIntent。

  4. 从 Amazon Kendra 索引下拉菜单中，选择 Amazon Kendra 索引下拉菜单。您在 “先决条件” 部分创建的索引应该可用。

  5. 选择 Add（添加）。

  6. 在意图编辑器中，向下滚动到 “配送” 部分，选择右箭头展开该部分，然后在 “成功发货” 下的框中添加以下消息：

    I found a link to a document that could help you: ((x-amz-lex:kendra-search-response-document-link-1)).

    有关Amazon Kendra 搜索回复的更多信息，请参阅使用搜索响应。

  7. 选择保存目的，然后选择构建以构建自动程序。机器人准备就绪后，屏幕顶部的横幅变为绿色并显示成功消息。

最后，使用控制台测试窗口来测试来自自动程序的响应。

要测试您的常见问题机器人，请执行以下操作：

  1. 成功构建机器人后，选择 T es t。

  2. What is Amazon Kendra?在控制台测试窗口中输入。验证机器人是否使用链接进行响应。

  3. 有关配置的更多信息AMAZON.KendraSearchIntent，请参阅 AMAZON。 KendraSearchIntent和KendraConfiguration。
