设置会话属性

会话属性包含会话期间在机器人和客户端应用程序之间传递的应用程序特定信息。Amazon Lex 将会话属性传递给为机器人配置的所有 Lambda 函数。如果 Lambda 函数添加或更新会话属性，Amazon Lex 会将新信息传递回客户端应用程序。

在 Lambda 函数中使用会话属性来初始化机器人并自定义提示和响应卡。例如：

  • 初始化 — 在披萨订购机器人中，客户端应用程序在第一次调用RecognizeText或RecognizeUtterance操作时将用户的位置作为会话属性传递。例如，"Location": "111 Maple Street"。Lambda 函数使用此信息来查找距离下单最近的比萨店。

  • Personalize 提示-配置提示和响应卡以参考会话属性。例如，“嘿 [FirstName]，你想要什么浇头？” 如果您将用户的名字作为会话属性 ({"FirstName": "Vivian"}) 传递，Amazon Lex 会用该名称代替占位符。然后它会向用户发送个性化提示：“嘿 Vivian，你想要哪种浇头？”

会话属性在会话持续时间内有效。Amazon Lex 将它们存储在加密的数据存储中，直到会话结束。客户端可以通过在将sessionAttributes字段设置为值的情况下调用RecognizeText或RecognizeUtterance操作在请求中创建会话属性。Lambda 函数可以在响应中创建会话属性。在客户端或 Lambda 函数创建会话属性后，只要客户端应用程序在向 Amazon Lex 的请求中不包含sessionAttribute字段，就会使用存储的属性值。

例如，假设您有两个会话属性 {"x": "1", "y": "2"}。如果客户端在未指定sessionAttributes字段的情况下调用RecognizeText或RecognizeUtterance操作，则 Amazon Lex 使用存储的会话属性调用 Lambda 函数 ({"x": 1, "y": 2})。如果 Lambda 函数不返回会话属性，则 Amazon Lex 将存储的会话属性返回给客户端应用程序。

如果客户端应用程序或 Lambda 函数传递会话属性，Amazon Lex 会更新存储的会话属性。传递现有值 (例如 {"x": 2}) 将更新存储的值。如果您传递一组新的会话属性（如 {"z": 3}），将删除现有值而只保留新值。当传递空映射 {} 时，将擦除存储的值。

要向 Amazon Lex 发送会话属性，您需要创建属性 string-to-string映射。下面显示了如何映射会话属性：

{
   "attributeName": "attributeValue",
   "attributeName": "attributeValue"
}

对于该RecognizeText操作，您可以使用sessionState结构的sessionAttributes字段将地图插入到请求正文中，如下所示：

"sessionState": {
    "sessionAttributes": {
        "attributeName": "attributeValue",
        "attributeName": "attributeValue"
    }
}

对于该RecognizeUtterance操作，您将 base64 对地图进行编码，然后将其作为x-amz-lex-session-state标题的一部分发送。

如果要在会话属性中发送二进制或结构化数据，则必须先将该数据转换为简单字符串。有关更多信息，请参阅设置复杂属性：
