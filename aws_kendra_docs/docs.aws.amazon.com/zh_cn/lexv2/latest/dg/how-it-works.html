工作原理

Amazon Lex V2 使您能够使用文本或语音界面构建应用程序，以便与用户进行对话。以下是使用Amazon Lex V2 的典型步骤：

  1. 创建机器人并添加一种或多种语言。对机器人进行配置，使其了解用户的目标，与用户进行对话以获取信息，并实现用户的意图。

  2. 测试自动程序。您可以使用 Amazon Lex V2 控制台提供的测试窗口客户端。

  3. 发布版本和创建别名

  4. 部署自动程序。你可以在自己的应用程序或消息传递平台上部署机器人，例如 Facebook Messenger 或 Slack

在您开始之前，请熟悉以下 Amazon Lex V2 核心概念和术语：

  • 机器人 — 机器人执行自动任务，例如订购披萨、预订酒店、订购鲜花等。Amazon Lex V2 机器人由自动语音识别 (ASR) 和自然语言理解 (NLU) 功能提供支持。

    Amazon Lex V2 机器人可以理解用户在文本或语音中输入的内容并与自然语言交谈。

  • 语言 — Amazon Lex V2 机器人可以用一种或多种语言交谈。每种语言都独立于其他语言，您可以将 Amazon Lex V2 配置为使用母语单词和短语与用户交谈。有关更多信息，请参阅Azon Lex V2 支持的语言和区域区域区域区域区域区域区域区域区域：

  • 意图-意图表示用户想要执行的操作。您创建自动程序以支持一个或多个相关目的。例如，您可以创建一个订购披萨和饮料的目的。对于每个目的，您需要提供以下必要信息：

    • 意图名称-意图的描述性名称。例如，OrderPizza。

    • 示例话语 — 用户如何传达意图。例如，用户可能会说 “我能点披萨吗” 或 “我想点披萨”。

    • 如何实现意图 — 在用户提供必要信息后，您希望如何实现意图。我们建议您创建 Lambda 函数以实现目的。

      您可以选择配置意图，以便 Amazon Lex V2 将信息返回给客户端应用程序以进行必要的配送。

    除了自定义意图外，Amazon Lex V2 还提供内置意图来快速设置您的机器人。有关更多信息，请参阅内置意图和插槽类型：

    Amazon Lex 始终包含每个机器人的后备意图。每当 Amazon Lex 无法推断出用户的意图时，就会使用后备意图。有关更多信息，请参阅亚马逊。 FallbackIntent：

  • 插槽 — 意图可能需要零个或多个插槽或参数。您可以添加槽，作为目的配置的一部分。在运行时，Amazon Lex V2 会提示用户输入特定的插槽值。在 Amazon Lex V2 实现意图之前，用户必须为所有必需的插槽提供值。

    例如，OrderPizza意图需要大小、外壳类型和披萨数量等字段。对于每个插槽，您需要提供插槽类型以及 Amazon Lex V2 发送给客户端以从用户那里获取值的一个或多个提示。用户可以在回复时使用包含其他词语的插槽值，例如 “请吃大块披萨” 或 “让我们坚持使用小披萨”。Amazon Lex V2 仍然能理解插槽的价值。

  • 插槽类型 — 每个插槽都有一个类型。您可以创建自己的槽类型，也可以使用内置槽类型。例如，您可针对 OrderPizza 目的创建并使用以下槽类型：

    • 大小 – 使用枚举值 Small、Medium 和 Large。

    • 馅饼皮 – 使用枚举值 Thick 和 Thin。

    Amazon Lex V2 还提供内置槽类型。例如，AMAZON.Number 是可用于订购披萨数量的内置槽类型。有关更多信息，请参阅内置意图和插槽类型：

  • 版本 — 版本是您工作的带编号的快照，您可以将其发布以用于工作流程的不同部分，例如开发、beta 部署和生产。创建版本后，您可以使用创建版本时存在的机器人。创建版本后，在您继续开发应用程序的同时，它会保持不变。

  • 别名 — 别名是指向机器人特定版本的指针。使用别名，您可以更新您的客户端应用程序正在使用的版本。例如，您可以将别名指向您自动程序的版本 1。当您准备更新自动程序时，您可以发布版本 2，然后更改别名以指向新版本。由于您的应用程序使用的是别名而不是特定版本，因此您的所有客户端无需进行更新即可获得新功能。

有关提供 Amazon Lex V2 的AWS区域的列表，请参阅《Amazon Web Services 一般参考》中的 Amazon Lex V2 终端节点和配额。
