多重身份验证

启用多重身份验证（MFA）时，用户必须使用其用户名和密码登录到AWS访问门户。这是第一个因素，他们知道。用户还必须使用密码或安全密钥登录。这是第二个因素，他们拥有的东西或者他们存在的东西。第二个因素可能是从他们的移动设备生成的身份验证码，也可以是通过点击连接到他们的计算机的安全密钥。综上所述，除非成功完成有效的 MFA 挑战，否则这些多个因素可以防止未经授权访问您的AWS资源，从而提高安全性。每个用户最多可以注册八台 MFA 设备。

重要

作为安全最佳实践，强烈建议您启用多重身份验证，这是最佳安全做法。MFA 提供了一种简单而安全的方式，可以在用户名和密码的默认身份验证机制之上添加额外的保护层。
