SCIM 配置和 SAML 2.0 的实现

SCIM 和 SAML 都是配置 IAM 身份中心的重要考虑因素。

SAML 2.0 的实现

IAM Identtity Centter 通过 SAML（安全断言标记语言）2.0 支持身份联合验证。这允许 IAM Identity Center 对来自外部身份提供商的身份进行身份验证 (IdPs)。SAML 2.0 是一个用于安全交换 SAML 断言的开放标准。SAML 2.0 在 SAML 机构（称为身份提供者或 IdP）和 SAML 使用者（称为服务提供商或 SP）之间传递有关用户的信息。IAM 身份中心服务使用此信息提供联合单点登录。单点登录允许用户根据其现有的身份提供者凭证访问AWS帐户和配置的应用程序。

IAM Identity Center 将 SAML IdP 功能添加到您的 IAM 身份中心存储或外部身份提供商。AWS Managed Microsoft AD然后，用户可以单点登录支持 SAML 的服务，包括微软 365、Concur AWS Management Console 和 Salesforce 等第三方应用程序。

但是，SAML 协议没有提供查询 IdP 以了解用户和群组的方法。因此，您必须通过将这些用户和群组配置到 IAM 身份中心来让 IAM Identity Center 知道这些用户和群组。

SCIM 配置文件

IAM 身份中心为跨域身份管理系统 (SCIM) v2.0 标准提供支持。SCIM 使您的 IAM 身份中心身份与来自 IdP 的身份保持同步。这包括在您的 IdP 和 IAM 身份中心之间对用户进行的任何配置、更新和取消配置。

有关如何实现 SCIM 的更多信息，请参阅自动配置。有关 IAM Identity Center SCIM 实施的更多详细信息，请参阅 IAM Identity Center SCIM 实施开发人员指南。
