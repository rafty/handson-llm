权限集

权限集是您创建和维护的模板，用于定义一个或多个 IAM 策略的集合。权限集简化了组织中用户和群组的AWS帐户访问权限分配。例如，您可以创建包含管理 AWS RDS、DynamoDB 和 Aurora 服务的策略的数据库管理员权限集，并使用该单一权限集向数据库管理员授予对AWS组织内目标AWS账户列表的访问权限。

IAM Identity Center 通过权限集向一个或多个AWS 账户用户或群组分配访问权限。当您分配权限集时，IAM Identity Center 会在每个账户中创建相应的 IAM Identity Center 控制的 IAM 角色，并将权限集中指定的策略附加到这些角色中。IAM Identity Center 使用 IAM Identity Center 用户门户或 AWS CLI 管理该角色，并允许您定义的授权用户代入该角色。 当您修改权限集时，IAM Identity Center 会确保相应的 IAM policy 和角色也相应更新。

您可以将AWS托管策略、客户托管策略、内联策略和工作职能的AWS托管策略添加到权限集中。您还可以将AWS托管策略或分配客户托管策略作为权限边界。

要创建权限集，请参阅创建和管理权限集。
