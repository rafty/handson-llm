Gerenciar o Regiões da AWS

Uma Região da AWSé uma coleção deAWS recursos da em uma área geográfica. Cada Região da AWS é independente e isolada das outras Regiões. As regiões fornecem tolerância a falhas, estabilidade e resiliência e também podem reduzir a latência. Você pode provisionar recursos em várias regiões para que suas aplicações permaneçam disponíveis e não sejam afetadas por uma interrupção regional.

Os recursos que você cria em uma região não existe em qualquer outra região, a menos que você use explicitamente um recurso de replicação oferecido por um serviço da AWS. Por exemplo, o Amazon S3 e o Amazon EC2 oferecem suporte somente à replicação entre regiões. Alguns serviços, como oAWS Identity and Access Management (IAM), não têm recursos regionais.

Você pode usar as condições da política para controlar o acessoServiços da AWS em umRegião da AWS.

Sua conta determina as regiões que estão disponíveis para você.

  • UmaConta da AWS fornece várias regiões para que você possa iniciarAWS recursos em locais que atendam às suas necessidades. Por exemplo, talvez você queira executar instâncias na Europa para estar mais próximo de seus clientes europeus ou para cumprir requisitos legais.

  • Uma contaAWSGovCloud (Oeste dos EUA) fornece acesso somente à regiãoAWSGovCloud (Oeste dos EUA) e à regiãoAWSGovCloud (Leste dos EUA). Para obter mais informações, consulte AWS GovCloud (US).

  • Uma conta da Amazon AWS (China) fornece acesso somente às regiões Pequim e Ningxia. Para obter mais informações, consulte Amazon Web Services na China.

Para poder usar qualquer uma das regiões a seguir, você deve habilitar a região para sua conta da.

Quando você habilita uma região, a AWS executa ações para preparar sua conta nesta região, como a distribuição dos seus recursos do IAM para a região. Esse processo leva alguns minutos para a maioria das contas, mas poderá levar algumas horas. Você não pode usar uma região até que esse processo seja concluído.

Requisitos

Para ativar uma região, você precisa ter as permissões apropriadas. Para ver um exemplo de política do IAM, consulte Permitir ativação e desativaçãoRegiões da AWS no Guia do usuário do IAM.

Para ativar uma região usando o console

  1. Faça login no AWS Management Console.

  2. No canto superior direito do console, escolha a região atual para expandir a lista de regiões, role até o final da lista e escolha Gerenciar regiões.

  3. Verifique a coluna Ação da região que você deseja usar.

    • Se a coluna Status contiver Ativado por padrão, nenhuma ação adicional será necessária. A região foi ativada automaticamente para sua conta.

    • Se a coluna Status contiver Desativado, escolha Habilitar. Revise o texto informativo e escolha Habilitar região. Você deve esperar até que a região esteja pronta para uso.

Para habilitar uma região usando oAWS CLI

Use o seguinte comando enable-region.

aws account enable-region --region-name af-south-1

Desabilitar uma região

Se você ativou uma região que não está ativada automaticamente, você pode desabilitar a região, se necessário. Depois que você desabilita uma região, os recursos dessa região ficam indisponíveis, com base na consistência eventual. No entanto, os recursos não são excluídos. Você pode acessá-los se ativar essa região novamente.

Requisitos

  • Você não pode desativar uma região ativada por padrão. (Veja a tabela na seção anterior.)

  • Para desativar uma região, você deve ter permissão para desativar regiões. Para ver um exemplo de política do IAM, consulte Permitir a ativação e desativação deAWS regiões no Guia do usuário do IAM.

  • Antes de desativar uma região, recomendamos que você remova todos os recursos dessa região. Depois de desativar uma região, você não poderá mais visualizar ou gerenciar recursos na região a partir dessa conta por meio do consoleAWS de gerenciamento ou dasAWS APIs usando os princípios do IAM. No entanto, os recursos nessa região podem continuar a incorrer em cobranças. Para obter mais informações, consulte Habilitar e desabilitar regiões no GuiaAWS Billing and Cost Management do usuário.

Para desabilitar uma região usando o console

  1. Faça login no AWS Management Console.

  2. No canto superior direito do console, escolha a região atual para expandir a lista de regiões, role até o final da lista e escolha Gerenciar regiões.

  3. Na coluna Ação da Região, escolha Desativar.

  4. Na caixa de diálogo, revise o texto informativo e escolha Desativar região.

Para desativar uma região usando oAWS CLI

Use o seguinte comando disable-region.

aws account disable-region --region-name af-south-1

Descreva suas regiões usando oAWS CLI

Use o comando list-regions para descrever todas as regiões disponíveis para sua conta, incluindo seu status.

aws account list-regions --max-results 50

Use o get-region-opt-statuscomando para descrever o status de uma região específica.

aws account get-region-opt-status --region-name af-south-1

Para regiões que são ativadas automaticamente para sua conta, a saída desses comandos inclui o seguinte:

"RegionOptStatus": "ENABLED_BY_DEFAULT"

Para regiões que você deve habilitar manualmente, a saída inclui o seguinte:

"RegionOptStatus": "DISABLED"

Depois de ativar uma região, a saída inclui o seguinte:

"RegionOptStatus": "ENABLED"
