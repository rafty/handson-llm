CreateFaq

Cria um conjunto de perguntas frequentes (perguntas frequentes) usando um arquivo de perguntas frequentes (perguntas frequentes) usando um arquivo de perguntas frequentes (perguntas frequentes) usando um arquivo de perguntas frequentes (perguntas frequentes) usando um arquivo de perguntas frequentes.

A adição de perguntas frequentes a um índice é uma operação de natureza assíncrona.

Para ver um exemplo de como adicionar uma FAQ a um índice usando SDKs para Python e Java, consulte Como usar seu arquivo de perguntas frequentes.

Sintaxe da solicitação

{
   "ClientToken": "string",
   "Description": "string",
   "FileFormat": "string",
   "IndexId": "string",
   "LanguageCode": "string",
   "Name": "string",
   "RoleArn": "string",
   "S3Path": { 
      "Bucket": "string",
      "Key": "string"
   },
   "Tags": [ 
      { 
         "Key": "string",
         "Value": "string"
      }
   ]
}

Parâmetros de solicitação

Para obter informações sobre os parâmetros que são comuns a todas as ações, consulte Parâmetros comuns.

A solicitação aceita os dados a seguir no formato JSON.

ClientToken

Um token que você fornece para identificar a solicitação de criação de uma FAQ. Várias chamadas para aCreateFaqRequest API com o mesmo token de cliente criarão apenas uma FAQ.

Tipo: String

Restrições de tamanho: tamanho mínimo 1. Tamanho máximo de 100.

Obrigatório: não

Description

Uma descrição para as perguntas frequentes.

Tipo: String

Restrições de tamanho: tamanho mínimo 0. Tamanho máximo de 1000.

Padrão: ^\P{C}*$

Obrigatório: não

FileFormat

O formato do arquivo de entrada das perguntas frequentes. É possível escolher entre um formato CSV básico, um formato CSV que inclua atributos personalizados em um cabeçalho e um formato JSON que inclua atributos personalizados.

O formato deve corresponder ao formato do arquivo armazenado no bucket do S3 identificado noS3Path parâmetro.

Para obter mais informações, consulte Adicionar perguntas e respostas.

Tipo: String

Valores válidos: CSV | CSV_WITH_HEADER | JSON

Obrigatório: não

IndexId

O identificador do índice das perguntas frequentes.

Tipo: String

Restrições de tamanho: tamanho fixo de 36.

Padrão: [a-zA-Z0-9][a-zA-Z0-9-]*

Obrigatório: Sim

LanguageCode

O código de um idioma. Isso permite que você ofereça suporte a um idioma para o documento de perguntas frequentes. O inglês é compatível por padrão. Para obter mais informações sobre os idiomas suportados, incluindo seus códigos, consulte Adição de documentos em outros idiomas além do inglês.

Tipo: String

Restrições de tamanho: tamanho mínimo 2. Tamanho máximo de 10.

Padrão: [a-zA-Z-]*

Obrigatório: não

Name

Um nome para as perguntas frequentes.

Tipo: String

Restrições de tamanho: tamanho mínimo 1. Tamanho máximo de 100.

Padrão: [a-zA-Z0-9][a-zA-Z0-9_-]*

Obrigatório: Sim

RoleArn

O nome de recurso da Amazon (ARN) de uma função do IAM com permissão para acessar o bucket do S3 que contém as perguntas frequentes. Para obter mais informações, consulte Funções de acesso do IAM para Amazon Kendra.

Tipo: String

Restrições de tamanho: tamanho mínimo 0. Tamanho máximo de 1284.

Padrão: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Obrigatório: Sim

S3Path

O caminho para o arquivo de perguntas frequentes no S3.

Tipo: objeto S3Path

Obrigatório: Sim

Tags

Uma lista de pares de chave-valor que identificam as perguntas frequentes. Você pode usar as tags para identificar e organizar seus recursos do e para controlar o acesso aos recursos do.

Tipo: matriz de objetos Tag

Membros da matriz: número mínimo de 0 itens. Número máximo de 200 itens.

Obrigatório: não

Sintaxe da resposta

{
   "Id": "string"
}

Elementos de resposta

Se a ação for bem-sucedida, o serviço reenviará uma resposta HTTP 200.

Os dados a seguir são retornados no formato JSON pelo serviço.

Id

O identificador das perguntas frequentes.

Tipo: String

Restrições de tamanho: tamanho mínimo 1. Tamanho máximo de 100.

Padrão: [a-zA-Z0-9][a-zA-Z0-9_-]*

Erros

Para obter informações sobre os erros comuns a todas as ações, consulteErros comuns.

AccessDeniedException

Você não tem acesso suficiente para realizar esta ação. Verifique se você tem as políticas de permissão e contas de usuário necessárias e tente novamente.

Código de status HTTP: 400

ConflictException

Ocorreu um conflito com a solicitação. Corrija quaisquer inconsistências com seus recursos e tente novamente.

Código de status HTTP: 400

InternalServerException

Ocorreu um problema com o servidor interno usado para seu serviço Amazon Kendra. Aguarde alguns minutos e tente novamente ou entre em contato com o Support para obter ajuda.

Código de status HTTP: 500

ResourceNotFoundException

O recurso que você deseja usar não existe. Verifique se você forneceu o recurso correto e tente novamente.

Código de status HTTP: 400

ServiceQuotaExceededException

Você excedeu os limites estabelecidos para seu serviço Amazon Kendra. Consulte Cotas para obter mais informações ou entre em contato com o Support para obter informações sobre um aumento de limites.

Código de status HTTP: 400

ThrottlingException

A solicitação foi negada devido à limitação da solicitação. Reduza o número de solicitações e tente novamente.

Código de status HTTP: 400

ValidationException

A entrada não atende às restrições definidas pelo serviço do Amazon Kendra. Forneça as informações corretas e tente novamente.

Código de status HTTP: 400
