Zendesk

O Zendesk é um sistema de gerenciamento de relacionamento com o cliente que ajuda as empresas a automatizar e aprimorar as interações de suporte ao cliente. Você pode usarAmazon Kendra para indexar seus tickets de suporte do Zendesk, comentários de tickets, anexos de tickets, artigos da central de ajuda, comentários de artigos, anexos de comentários de artigos, tópicos de guias da comunidade, publicações da comunidade e comentários de postagens da comunidade.

Você pode filtrar pelo nome da organização se quiser indexar tickets que estão somente dentro de uma organização específica. Você também pode escolher definir uma data de rastreamento para quando quiser começar a rastrear dados do Zendesk.

Você pode se conectarAmazon Kendra à sua fonte de dados do Zendesk usando o Amazon Kendraconsole e a TemplateConfigurationAPI.

Para solucionar problemas com seu conector de fonte de dados doAmazon Kendra Zendesk, consulteSolução de problemas de origens de dados.

Amazon KendraO conector de fonte de dados do Zendesk do oferece suporte aos seguintes recursos:

  • Log de alterações

  • Mapeamento de campo

  • Filtragem de contexto do usuário

  • Filtros de inclusão/exclusão

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do Zendesk, faça essas alterações em seu Zendesk e em suasAWS contas.

No Zendesk, verifique se você tem:

  • Criou uma conta administrativa da Zendesk Suite (Professional/Enterprise).

  • Anotou o URL do seu host do Zendesk. Por exemplo, https://{sub-domain (https://{host/)}.zendesk.com/.

  • Gerou um token de credencial OAuth 2.0 contendo ID do cliente, segredo do cliente, nome de usuário e senha. Consulte a documentação da Zendesk sobre a geração de tokens OAuth 2.0 para obter mais informações.

  • Foi adicionado o seguinte escopo OAuth 2.0:

    • read

  • Opcional: instalou um certificado SSL paraAmazon Kendra permitir a conexão.

  • Verifiquei que cada documento é exclusivo no Zendesk e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do Zendesk emAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tem umaIAM função ou segredo existente, pode usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do Zendesk àAmazon Kendra. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do Zendesk, você deve fornecer os detalhes necessários da sua fonte de dados do Zendesk para que elaAmazon Kendra possa acessar seus dados. Se você ainda não configurou o Zendesk paraAmazon Kendra verPré-requisitos.

Console

Para se conectarAmazon Kendra ao Zendesk

  1. Faça login noAWS Management Console e abra o Amazon Kendraconsole do.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha Conector Zendesk e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em Tags, para Adicionar nova tag — Tags para pesquisar e filtrar seus recursos ou monitorar seusAWS custos.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as informações a seguir:

    1. URL do Zendesk — insira sua URL do Zendesk.

    2. AWSSecrets Managersegredo — Escolha um segredo existente ou crie um novoSecrets Manager segredo para armazenar suas credenciais de autenticação do Zendesk. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-Zendesk-' é adicionado automaticamente ao seu nome secreto.

        2. Para ID do cliente, segredo do cliente, nome de usuário, senha — insira os valores da credencial de autenticação que você criou em sua conta do Zendesk.

      2. Escolha Save (Salvar).

    3. Virtual Private Cloud (VPC) Você pode escolher usar uma VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    4. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    5. Escolha Próximo.

  7. Na página Definir configurações de sincronização, insira as informações a seguir:

    1. Selecione entidades ou tipos de conteúdo — As entidades do Zendesk ou os tipos de conteúdo que você deseja rastrear.

    2. Registro de alterações — selecione para atualizar seu índice em vez de sincronizar todos os seus arquivos.

    3. Nome da organização — insira os nomes das organizações do Zendesk para filtrar sua sincronização.

    4. Data de início da sincronização — A data a partir da qual você deseja indexar seu conteúdo.

    5. Padrões Regex — padrões de expressão regular para incluir ou excluir determinados arquivos. Você pode adicionar até 100 padrões.

    6. Em Sincronizar cronograma de execução para frequência — escolha comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    7. Escolha Próximo.

  8. Na página Definir mapeamentos de campos, insira as informações a seguir:

    1. Para tickets, comentário de ticket, anexo de comentário de ticket, artigo, comentário de artigo, anexo de comentário de artigo, tópico da comunidade, publicação da comunidade, comentário de publicação da comunidade — selecione um dos campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para o seu índice.

    2. Adicionar campo — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar as informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra ao Zendesk

Você deve especificar um JSON do esquema da fonte de dados usando a TemplateConfigurationAPI. Você deve fornecer as seguintes informações:

  • Fonte de dados — especifique a fonte de dados comoZENDESK.

  • URL do host — forneça a URL do host do Zendesk como parte da configuração da conexão ou dos detalhes do endpoint do repositório. Por exemplo, https://yoursubdomain.zendesk.com.

  • Tipo — EspecifiqueTEMPLATE como o Tipo ao ligarCreateDataSource.

  • Nome secreto do recurso da Amazon (ARN) — forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação da sua conta do Zendesk. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "hostUrl": "https://yoursubdomain.zendesk.com",
    "clientId": "kendra",
    "clientSecret": "Zendesk client secret",
    "userName": "Zendesk user name",
    "password": "Zendesk password"
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você ligaCreateDataSource para fornecer a umaIAM função com permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector ZendeskAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes de dados do Zendesk.

Você também pode adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) especifiqueVpcConfiguration quando você ligaCreateDataSource. Consulte ConfigurandoAmazon Kendra para usar uma VPC.

  • Registro de alterações — seAmazon Kendra deve usar o mecanismo de registro de alterações da fonte de dados do Zendesk para determinar se um documento deve ser adicionado, atualizado ou excluído no índice.

    nota

    Use o registro de alterações se nãoAmazon Kendra quiser digitalizar todos os documentos. Se seu registro de alterações for grande, pode levarAmazon Kendra menos tempo para digitalizar os documentos na fonte de dados do Zendesk do que para processar o registro de alterações. Se você estiver sincronizando sua fonte de dados do Zendesk com seu índice pela primeira vez, todos os documentos serão digitalizados.

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir:

    • Tickets de Support, comentários de tíquetes e/ou anexos de comentários de tickets

    • Artigos da central de ajuda, anexos de artigos e comentários de artigos

    • Guie tópicos, postagens ou publique comentários da comunidade

  • nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — escolha mapear seus campos de fonte de dados do Zendesk para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Filtragem de contexto do usuário —Amazon Kendra rastreia a Lista de Controle de Acesso (ACL) da sua fonte de dados por padrão. As informações da ACL são usadas para filtrar os resultados da pesquisa com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtragem de contexto de usuário para fontes de dados do Zendesk.

Para obter uma lista de outras chaves JSON importantes a serem configuradas, consulteEsquema de modelo do Zendesk.
