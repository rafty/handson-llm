Amazon WorkDocs

Amazon WorkDocsé um serviço seguro de colaboração de conteúdo para criar, editar, armazenar e compartilhar conteúdo. Você pode usarAmazon Kendra para indexar sua fonteAmazon WorkDocs de dados.

Você pode se conectarAmazon Kendra à sua fonte deAmazon WorkDocs dados usando o Amazon Kendraconsole e a WorkDocsConfigurationAPI.

Amazon WorkDocsestá disponível nas regiões de Oregon, Norte da Virgínia, Sydney, Singapura e Irlanda.

Para solucionar problemas do seu conector de fonte deAmazon Kendra WorkDocs dados, consulteSolução de problemas de origens de dados.

Recursos compatíveis

Amazon Kendra WorkDocs O conector de fonte de dados suporta os seguintes recursos:

  • Log de alterações

  • Mapeamentos de campo

  • Filtragem de contexto do usuário

  • Filtros de inclusão/exclusão

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de WorkDocs dados, faça essas alterações em suasAWS contas WorkDocs e.

Em WorkDocs, verifique se você tem:

  • Anotou o ID doAmazon WorkDocs diretório (ID da organização) do seuAmazon WorkDocs repositório.

  • Verificou-se que cada documento é exclusivo em WorkDocs outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

Se você não tiver umaIAM função existente, poderá usar o console para criar uma novaIAM função ao conectar sua fonte de WorkDocs dados àAmazon Kendra. Se você estiver usando a API, forneça o ARN de umaIAM função existente e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de WorkDocs dados, você deve fornecer os detalhes necessários de sua fonte de WorkDocs dados para que elaAmazon Kendra possa acessar seus dados. Se você ainda não configurou WorkDocs paraAmazon Kendra verPré-requisitos.

Console

Para se conectarAmazon Kendra aAmazon WorkDocs

  1. Faça login noAWS Management Console e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha WorkDocs conector e, em seguida, escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em Tags, para Adicionar nova tag — Tags para pesquisar e filtrar seus recursos ou monitorar seusAWS custos.

    5. Escolha Próximo.

  6. Na página Definir acesso e Segurança, insira as informações a seguir:

    1. ID da organização específicaAmazon WorkDocs do seu site — Selecione o ID doAmazon WorkDocs site que você deseja indexar. Você já deve ter criado um site.

    2. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    3. Escolha Próximo.

  7. Na página Definir configurações de sincronização, insira as informações a seguir:

    1. Rastrear comentários do documento —Amazon WorkDocs As entidades ou tipos de conteúdo que você deseja rastrear.

    2. Use registros de alterações — selecione para atualizar seu índice em vez de sincronizar todos os seus arquivos.

    3. Padrões Regex — padrões de expressão regular para incluir ou excluir determinados arquivos. Você pode adicionar até 100 padrões.

    4. Em Sincronizar cronograma de execução para frequência — escolha comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    5. Escolha Próximo.

  8. Na página Definir mapeamentos de campos, insira as informações a seguir:

    1. Campos de fonte de dados padrão — Selecione entre os campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para o seu índice.

    2. Adicionar campo — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra aAmazon WorkDocs

Você deve especificar o seguinte usando a WorkDocsConfigurationAPI:

  • Amazon WorkDocsID do diretório — especifique o ID da organização do seuAmazon WorkDocs diretório. Você pode encontrar o ID da organização no AWS Directory Service acessando o Active Directory e depois Directories.

  • Função do IAM — especifiqueRoleArn quando você chamaCreateDataSource para fornecer a umaIAM função permissões para acessar o WorkDocs diretório e chamar as APIs públicas necessárias para o WorkDocs conectorAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes WorkDocs de dados.

Você também pode adicionar os seguintes recursos opcionais:

  • Registro de alterações — seAmazon Kendra deve usar o mecanismo de registro de alterações da fonte de WorkDocs dados para determinar se um documento deve ser adicionado, atualizado ou excluído no índice.

    nota

    Use o registro de alterações se nãoAmazon Kendra quiser digitalizar todos os documentos. Se seu registro de alterações for grande, pode levarAmazon Kendra menos tempo para digitalizar os documentos na fonte de WorkDocs dados do que para processar o registro de alterações. Se você estiver sincronizando sua fonte de WorkDocs dados com seu índice pela primeira vez, todos os documentos serão digitalizados.

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir determinados documentos e comentários do documento. Cada comentário é indexado como um documento separado.

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — escolha mapear seus campos WorkDocs de fonte de dados para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Filtragem de contexto do usuário —Amazon Kendra rastreia a Lista de Controle de Acesso (ACL) da sua fonte de dados por padrão. As informações da ACL são usadas para filtrar os resultados da pesquisa com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtragem de contexto de usuário para fontes WorkDocs de dados.
