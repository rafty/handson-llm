Solução de problemas de origens de dados

Esta seção pode ajudá-lo a corrigir problemas com fontes de dados do Amazon Kendra.

Meus documentos não foram indexados

Ao sincronizar seu índice Kendra com uma fonte de dados, você pode ter problemas que impedem que os documentos sejam indexados. A indexação é um processo em duas etapas. Primeiro, a fonte de dados é verificada em busca de documentos novos e atualizados para indexar e para encontrar documentos a serem removidos do índice. Segundo, no nível do documento, cada documento é acessado e indexado.

Um erro pode ocorrer em qualquer uma dessas etapas. Erros no nível da fonte de dados são relatados no console na seção Histórico de execução do Sync da página de detalhes da fonte de dados. O status do trabalho de sincronização pode ser Sucesso, Incompleto ou Falha. Você também pode ver o número de documentos indexados e excluídos durante o trabalho. Se o status for Falha, uma mensagem será exibida na coluna Detalhes.

Erros no nível do documento são relatados no Amazon CloudWatch Logs. Você pode ver os erros usando o CloudWatch console.

Meu trabalho de sincronização falhou

Normalmente, um trabalho de sincronização falha quando há um erro de configuração no índice ou na fonte de dados. A mensagem de erro na coluna Detalhes da fonte de dados fornece informações sobre o que deu errado. O problema geralmente é que o índice ou a fonte de dados não têm as permissões adequadas do IAM. A mensagem de erro descreve as permissões ausentes. Veja algumas das mensagens de erro que você pode receber:

Failed to create log group for job. Please make sure that the IAM role provided has sufficient permissions.

Se sua função de índice não tiver permissão para uso CloudWatch, a fonte de dados não poderá criar um CloudWatch log. Se você receber esse erro, deverá adicionar CloudWatch permissões à função de índice.

Failed to access S3 file prefix (bucket name) while trying to crawl your metadata files. Please make sure the IAM Role (role ARN) provided has sufficient permissions.

Quando você usa uma fonte de dados do Amazon S3, o Kendra precisa ter permissão para acessar o bucket que contém os documentos. Você precisa adicionar permissão para que o Kendra leia o bucket na função IAM da fonte de dados.

The provided IAM Role (role ARN) could not be assumed. Please make sure Amazon Kendra is a trusted entity that is allowed to assume the role.

Kendra precisa de permissão para assumir as funções do IAM de índice e fonte de dados. Você precisa adicionar uma política de confiança às funções com permissão para asts:AssumeRole ação.

Para ver as políticas do IAM que o Kendra precisa para indexar uma fonte de dados, consulteIAMfunções de acesso paraAmazon Kendra.

Meu trabalho de sincronização está incompleto

As tarefas geralmente ficam incompletas quando concluem o processo no nível da fonte de dados, mas apresentam algum erro durante o processo no nível do documento. Quando um trabalho está incompleto, alguns documentos ainda podem ter sido indexados com sucesso. Para uma fonte de dados do Amazon S3, um trabalho incompleto geralmente é causado por:

  • Os metadados de um ou mais documentos eram inválidos.

  • Quando os documentos são enviados para indexação, mas pelo menos um documento não foi enviado.

  • Quando documentos são enviados para exclusão do índice, mas pelo menos um documento não foi enviado.

Para solucionar problemas de um trabalho de sincronização incompleto, examine primeiro seus CloudWatch registros.

  1. Na coluna de detalhes, escolha Exibir detalhes em CloudWatch.

  2. Analise as mensagens de erro para ver o que causou a falha do documento.

Meu trabalho de sincronização foi bem-sucedido, mas não há documentos indexados

Ocasionalmente, a execução de uma tarefa de sincronização de índice será marcada como bem-sucedida, mas não há documentos novos ou atualizados indexados quando você espera. As possíveis preferências incluem:

  • Verifique a CloudWatch DocumentsSubmittedForIndexingFailed métrica para ver se algum documento falhou na sincronização. Verifique seus CloudWatch registros para obter detalhes.

  • Para uma fonte de dados do Amazon S3, você pode ter dado a Kendra o nome ou prefixo errado do bucket. Certifique-se de que o bucket que o Kendra está usando seja aquele que contém os documentos a serem indexados.

  • Ao reindexar um documento que não foi indexado em um trabalho anterior, o Kendra não o indexará, a menos que você tenha alterado o documento ou o arquivo de metadados associado.

Estou enfrentando problemas de formato de arquivo ao sincronizar minha fonte de dados

Se você tiver problemas de formato de arquivo ao adicionar arquivos à sua fonte de dados ou sincronizar sua fonte de dados, certifique-se de que seus tipos de documentos sejam compatíveis com o Kendra. Para obter uma lista dos tipos de documentos suportados por,Amazon Kendra consulte Tipos de documentos.

Se você estiver usando aBatchPutDocument API com arquivos de texto simples, especifiquePLAIN_TEXT como tipo de conteúdo.

Quanto tempo demora a sincronização de uma fonte de dados?

Se não houver atualizações nos documentos, o tempo de sincronização de um índice Amazon Kendra aumenta em proporção linear ao número de documentos. Por exemplo, 1.000 documentos sem nenhuma atualização levariam cerca de cinco minutos para sincronizar e 2.000 documentos sem nenhuma atualização levariam cerca de 10 minutos. Se houver alguma atualização nos documentos, o tempo de sincronização aumentará com base no número de documentos atualizados.

Qual é a cobrança pela sincronização de uma fonte de dados?

Quando você sincroniza seu índice, são necessários dois minutos para aquecer e ativar o Amazon EC2 para estabelecer as conexões necessárias. Você não é cobrado durante esse processo. Seu medidor de uso começa somente após o início do trabalho de sincronização. Para obter mais informações sobre os preços do Kendra, consulte os preços do Amazon Kendra.

Estou recebendo um erro de autorização do Amazon EC2

Se ocorrer um erro de operação não autorizada do Amazon EC2 durante uma sincronização para uma fonte de dados de nuvem privada virtual (VPC), é provável que sua função VPC IAM não tenha as permissões necessárias. Verifique se a função do IAM que você usa para sua fonte de dados tem as permissões anexadas. Para obter mais informações, consulte IAMFunção de nuvem privada virtual (VPC).

Não consigo usar links de índice de pesquisa para abrir meus objetos do Amazon S3

Seu índice Kendra só pode acessar arquivos que uma fonte de dados do Amazon S3 concede permissões de acesso. Por exemplo, o Kendra não pode modificar as permissões do Amazon S3 que determinam se um objeto deve ser público ou criptografado. O Kendra também não tem as permissões padrão para criar ou retornar um link assinado para objetos do Amazon S3. Se você quiser ativar a vinculação assinada para objetos do Amazon S3 em um índice Kendra, você tem duas opções:

  • Você pode usar a assinatura dos resultados da consulta de índice com o objeto uri de origem antes de retornar o resultado à página de pesquisa. Para ver um step-by-step passo a passo desse processo, consulte Compartilhamento de objetos usando URLs preassinados.

  • Você pode substituir o uri da fonte de metadados do objeto do Amazon S3 e disponibilizar seu serviço por meio de uma rede de entrega de CloudFront conteúdo (CDN) conectada a um bucket do Amazon S3. Ou você pode usar um endpoint de proxy do API Gateway que retorna um URL preassinado e redireciona para ele.

Estou recebendo uma mensagem de erro AccessDenied ao usar o arquivo de certificado SSL

Se você estiver recebendo um erro de acesso negado ao usar um certificado SSL com sua fonte de dados, certifique-se de que sua função do IAM tenha permissão para acessar o arquivo de certificado SSL no local especificado. Se o certificado estiver criptografado com umaAWS KMS chave, sua função do IAM também deverá ter permissão para descriptografar usando aAWS KMS chave. Para obter mais informações, consulte Autenticação e controle de acesso paraAWS KMS.

Estou recebendo um erro de autorização ao usar uma fonte SharePoint de dados

Se você estiver recebendo um erro de autorização ao sincronizar seu índice com uma fonte de SharePoint dados, confirme se você tem uma função de administrador do site atribuída a você em SharePoint.

Meu índice não rastreia documentos da minha fonte de dados do Confluence

Se seu índice Kendra não estiver rastreando documentos da sua fonte de dados do Confluence durante o processo de sincronização, confirme que você faz parte dos grupos de administradores no Confluence.
