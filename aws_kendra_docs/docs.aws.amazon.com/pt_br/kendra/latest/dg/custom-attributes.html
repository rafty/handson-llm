Criação de campos de documentos personalizados

Você pode aplicar atributos personalizados ou campos de metadados relevantes para seus documentos específicos. Por exemplo, você pode criar um campo ou atributo personalizado chamado “Departamento” com os valores de “RH”, “Vendas” e “Fabricação”. Você pode usar esses campos ou atributos para limitar a resposta (os resultados da pesquisa) aos documentos do departamento de “RH”, por exemplo.

Crie até 500 campos ou atributos personalizados.

Para a maioria das fontes de dados, você mapeia campos na fonte de dados externa para os campos correspondentes emAmazon Kendra. Para obter mais informações, consulte Mapeamento de campos de origem de dados. Para origens de dados do S3, você pode aplicar campos ou atributos personalizados usando arquivos de metadados.

Antes de usar um campo ou atributo personalizado, deverá primeiramente criar o campo no índice. Use o console ou a UpdateIndexAPI para criar o campo de índice. Você não pode alterar o tipo de dados do campo depois de criar o campo.

Você também pode usarAmazon Kendra os campos internos ou reservados.

Com aUpdateIndex API, você adiciona campos ou atributos personalizados usando oDocumentMetadataConfigurationUpdates parâmetro.

O exemplo JSON a seguir é usadoDocumentMetadataConfigurationUpdates para adicionar um campo chamado “Departamento” ao índice.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Usando campos de documentos integrados do Amazon Kendra

Com a UpdateIndex API, você pode criar campos reservados ou integrados usandoDocumentMetadataConfigurationUpdates e especificando o nome do campo reservado a ser mapeado para seu atributo de documento equivalente ou campo de documento. Você também pode criar campos personalizados dessa forma. Se você usa um conector de fonte de dados, a maioria inclui mapeamentos de campo que mapeiam os campos do documento da fonte de dados para campos deAmazon Kendra índice. Ao criar um campo, você pode configurar oSearch objeto para definir o campo como exibível, facetável, pesquisável e classificável. Você pode configurar oRelevance objeto para definir a relevância do campo, aumentando a duração, a atualização, a importância, a ordem de classificação e os valores de importância. Você não pode alterar o tipo de campo depois de criar o campo.

Amazon Kendratem os seguintes campos de documentos reservados ou integrados que você pode usar:

  • _authors—Uma lista de um ou mais autores responsáveis pelo conteúdo do documento.

  • _category—Uma categoria que coloca um documento em um grupo específico.

  • _created_at—A data e a hora no formato ISO 8601 em que o documento foi criado. Por exemplo, 2012-03-25T12:30:10+01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012 às 12h30 (mais 10 segundos) no horário da Europa Central.

  • _data_source_id— O identificador da fonte de dados que contém o documento.

  • _document_body—O conteúdo do documento.

  • _document_id—Um identificador exclusivo para o documento.

  • _document_title—O título do documento.

  • _excerpt_page_number—O número da página em um arquivo PDF em que o trecho do documento aparece. Se seu índice foi criado antes de 8 de setembro de 2020, você deve reindexar seus documentos antes de usar esse atributo.

  • _faq_id—Se for uma pergunta e resposta de perguntas frequentes, um identificador exclusivo para elas.

  • _file_type—O tipo de arquivo do documento, como pdf ou doc.

  • _last_updated_at—A data e a hora no formato ISO 8601 em que o documento foi atualizado pela última vez. Por exemplo, 2012-03-25T12:30:10+01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012 às 12h30 (mais 10 segundos) no horário da Europa Central.

  • _source_uri—O URI em que o documento está disponível. Por exemplo, o URI do documento no site da empresa.

  • _version—Um identificador para a versão específica de um documento.

  • _view_count—O número de vezes que o documento foi visualizado.

  • _language_code(String) — O código de um idioma que se aplica ao documento. A definirá como padrão o inglês se você não especificar um idioma. Para obter mais informações sobre os idiomas suportados, incluindo seus códigos, consulte Adição de documentos em outros idiomas além do inglês.

Para campos personalizados, você cria esses campos usandoDocumentMetadataConfigurationUpdates aUpdateIndex API, assim como você faz ao criar um campo reservado ou incorporado. Você deve definir o tipo de dados apropriado para seu campo personalizado. Você não pode alterar o tipo de campo depois de criar o campo.

A seguir estão os tipos que você pode definir para campos personalizados:

  • Data

  • Número

  • String

  • Lista de strings

Você cria um campo personalizado usando o console ou usando a UpdateIndexAPI. Depois de criar um campo personalizado, você o mapeia para um atributo de documento, assim como faz com um campo reservado. Se você adicionou um documento ao índice com a BatchPutDocumentAPI, mapeia os atributos com a API. Para documentos indexados de uma fonte deAmazon S3 dados, você mapeia os atributos usando um arquivo de metadados que contém uma estrutura JSON que descreve os atributos do documento. Para documentos indexados com um banco de dados ou fonte de dados que permite o mapeamento de campos, você mapeia atributos com o console ou a configuração da fonte de dados.

Adicionar atributos ou campos personalizados com a BatchPutDocument API

Ao usar aBatchPutDocument API para adicionar um documento ao seu índice, você especifica campos ou atributos personalizados como parte doAttributes. Você pode adicionar vários campos ou atributos ao chamar a API. Crie até 500 campos ou atributos personalizados. O exemplo a seguir é um campo ou atributo personalizado que adiciona “Departamento” a um documento.

"Attributes": 
    {
        "Department": "HR",
        "_category": "Vacation policy"
    }

Adicionar atributos ou campos personalizados a uma fonteAmazon S3 de dados

Ao usar um bucket do S3 como fonte de dados para seu índice, você adiciona metadados aos documentos com arquivos de metadados complementares. Você coloca os arquivos JSON de metadados em uma estrutura de diretórios parallel aos seus documentos. Para obter mais informações, consulte Amazon S3metadados do documento.

Você especifica campos ou atributos personalizados na estruturaAttributes JSON. Crie até 500 campos ou atributos personalizados. Por exemplo, o exemplo a seguir é usadoAttributes para definir três campos ou atributos personalizados e um campo reservado.

"Attributes": {
        "brand": "Amazon Basics",
        "price": 1595,
        "_category": "sports",
        "subcategories": ["outdoors", "electronics"]
    }
