Dropbox

O Dropbox é um serviço de hospedagem de arquivos que oferece serviços de armazenamento em nuvem, organização de documentos e modelagem de documentos. Se você é usuário do Dropbox, pode usarAmazon Kendra para indexar seus arquivos do Dropbox, Dropbox Paper, modelos do Dropbox Paper e atalhos armazenados para páginas da web. Você também pode configurarAmazon Kendra para indexar arquivos específicos do Dropbox, Dropbox Paper, modelos do Dropbox Paper e atalhos armazenados para páginas da web.

Amazon Kendraé compatível com o Dropbox e o Dropbox Advanced para Dropbox Business.

Você pode se conectarAmazon Kendra à sua fonte de dados do Dropbox usando o Amazon Kendraconsole e a TemplateConfigurationAPI.

Para solucionar problemas com seu conector de fonte de dados doAmazon Kendra Dropbox, consulteSolução de problemas de origens de dados.

Amazon KendraO conector de fonte de dados do Dropbox do oferece suporte aos seguintes recursos:

  • Log de alterações

  • Mapeamentos de campo

  • Filtragem de contexto do usuário

  • Filtros de inclusão/exclusão

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do Dropbox, faça essas alterações em seu Dropbox e em suasAWS contas.

No Dropbox, verifique se você tem:

  • Criou uma conta do Dropbox Advanced e configurou um usuário administrador.

  • Criou um aplicativo do Dropbox com um nome de aplicativo exclusivo, ativou o Scoped Access. Consulte a documentação do Dropbox sobre a criação de um aplicativo.

  • Ativou as permissões completas do Dropbox no console do Dropbox e adicionou as seguintes permissões:

    • arquivos.content.read

    • arquivos.metadata.read

    • compartilhando.read

    • file_requests.read

    • grupos.read

    • team_info.read

    • team_data.content.read

  • Anotou sua chave do aplicativo do Dropbox, o segredo do aplicativo do Dropbox e o token de acesso do Dropbox para credenciais básicas de autenticação.

  • Gerou e copiou um token de acesso temporário do Oauth 2.0 para seu aplicativo do Dropbox. Esse token é temporário e expira após 4 horas. Consulte a documentação do Dropbox sobre autenticação OAuth.

    nota

    É recomendável que você crie um token de acesso de atualização do Dropbox que nunca expire, em vez de confiar em um token de acesso único que expira após 4 horas. Um token de acesso de atualização é permanente e nunca expira para que você possa continuar sincronizando sua fonte de dados no future.

  • Recomendado: configurou um token de atualização permanente do Dropbox que nunca expiraAmazon Kendra para permitir que você continue sincronizando sua fonte de dados sem interrupções. Consulte a documentação do Dropbox sobre tokens de atualização.

  • Verifiquei que cada documento é exclusivo no Dropbox e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do Dropbox emAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tem umaIAM função ou segredo existente, pode usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do Dropbox àAmazon Kendra. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do Dropbox, você deve fornecer os detalhes necessários da sua fonte de dados do Dropbox para queAmazon Kendra possa acessar seus dados. Se você ainda não configurou o Dropbox,Amazon Kendra vejaPré-requisitos.

Console

Para se conectarAmazon Kendra ao Dropbox

  1. Faça login no ConsoleAWS de Gerenciamento e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha Conector do Dropbox e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em etiquetas, para Adicionar nova etiqueta — etiquetas para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as informações a seguir:

    1. Tipo de token de autenticação — escolha entre Token Permanente (recomendado) e Token de Acesso (uso temporário) com base no seu caso de uso.

    2. AWSSecrets Managersegredo — Escolha um segredo existente ou crie um novoSecrets Manager segredo para armazenar suas credenciais de autenticação do Dropbox. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-Dropbox-' é adicionado automaticamente ao seu nome secreto.

        2. Para informações sobre chave do aplicativo, segredo do aplicativo e token (permanentes ou temporárias) — insira os valores das credenciais de autenticação que você gerou na sua conta do Dropbox.

      2. Escolha Save (Salvar).

    3. Virtual Private Cloud (Nuvem privada [VPC]) — Você pode escolher usar uma VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    4. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    5. Escolha Próximo.

  7. Na página Definir configurações de sincronização, insira as informações a seguir:

    1. Para Selecionar entidades ou tipos de conteúdo — escolha entidades ou tipos de conteúdo que você deseja rastrear.

    2. Alterar modo de registro — escolha atualizar seu índice em vez de sincronizar todos os arquivos.

    3. Em Configuração adicional para padrões Regex — Adicione padrões de expressão regular para incluir ou excluir determinados arquivos.

    4. Em Sincronizar cronograma de execução, para Frequência — escolha comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    5. Escolha Próximo.

  8. Na página Definir mapeamentos de campos, insira as informações a seguir:

    1. Arquivos, modelos do Dropbox Paper e do Dropbox Paper — selecione entre os campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para o seu índice.

    2. Adicionar campo — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra ao Dropbox

Você deve especificar um JSON do esquema da fonte de dados usando a TemplateConfigurationAPI. Você deve fornecer as seguintes informações:

  • Fonte de dados — você deve especificar a fonte de dados comoDROPBOX.

  • Tipo — EspecifiqueTEMPLATE como o Tipo ao ligarCreateDataSource.

  • Nome secreto do recurso da Amazon (ARN) — Forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação da sua conta do Dropbox. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
"appKey": "Dropbox app key",
"appSecret": "Dropbox app secret",
"accesstoken": "temporary access token or refresh access token"
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você ligaCreateDataSource para fornecer a umaIAM função com permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector do DropboxAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes de dados do Dropbox.

Você também pode adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) — especifiqueVpcConfiguration quando você faz a chamadaCreateDataSource. Consulte ConfigurandoAmazon Kendra para usar uma VPC.

  • Registro de alterações — seAmazon Kendra deve usar o mecanismo de registro de alterações na fonte de dados do Dropbox para determinar se um documento deve ser adicionado, atualizado ou excluído no índice.

    nota

    Use o registro de alterações se nãoAmazon Kendra quiser digitalizar todos os documentos. Se seu registro de alterações for grande, pode levarAmazon Kendra menos tempo para digitalizar os documentos na fonte de dados do Dropbox do que para processar o registro de alterações. Se você estiver sincronizando sua fonte de dados do Dropbox com seu índice pela primeira vez, todos os documentos serão digitalizados.

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir determinados arquivos.

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — Escolha mapear seus campos de fonte de dados do Dropbox para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Filtragem de contexto do usuário —Amazon Kendra rastreia a Lista de Controle de Acesso (ACL) da sua fonte de dados por padrão. As informações da ACL são usadas para filtrar os resultados da pesquisa com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtragem de contexto de usuário para fontes de dados do Dropbox.

Para obter uma lista de outras chaves JSON importantes a serem configuradas, consulteEsquema de modelo do Dropbox.
