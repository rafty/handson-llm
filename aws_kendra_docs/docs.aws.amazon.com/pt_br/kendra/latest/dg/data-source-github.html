GitHub

GitHub é um serviço de hospedagem baseado na web para desenvolvimento de software que fornece serviços de armazenamento e gerenciamento de código com controle de versão. Você pode usarAmazon Kendra para indexar seus GitHub arquivos de repositório do GitHub Enterprise Cloud (SaaS) e do Enterprise Server (On Prem), emitir e retirar solicitações, emitir e retirar comentários e emitir e retirar anexos de comentários da solicitação. Você também pode optar por incluir ou excluir determinados arquivos.

Você pode se conectarAmazon Kendra à sua fonte de GitHub dados usando o Amazon Kendraconsole e a GitHubConfigurationAPI.

Para solucionar problemas do seu conector de fonte deAmazon Kendra GitHub dados, consulteSolução de problemas de origens de dados.

Recursos compatíveis

Amazon Kendra GitHub O conector de fonte de dados suporta os seguintes recursos:

  • Log de alterações

  • Mapeamentos de campos

  • Filtragem de contexto do usuário

  • Filtros de inclusão/exclusão

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de GitHub dados, faça essas alterações em suasAWS contas GitHub e.

Em GitHub, verifique se você tem:

  • Criou um GitHub usuário com permissões administrativas para a GitHub organização.

  • Criou um token de acesso pessoal para credenciais de autenticação. Consulte GitHub a documentação sobre como criar um token de acesso pessoal.

  • Recomendado: criou um token OAuth para credenciais de autenticação. Use o token OAuth para melhorar os limites de aceleração da API e o desempenho do conector. Consulte GitHub a documentação sobre autorização de OAuth.

  • Opcional: instalou um certificado SSL.

  • Anotou o URL do GitHub host para o tipo de GitHub serviço que você usa. Por exemplo, o URL do host para a GitHub nuvem pode ser https://api.github.com e o URL do host para o GitHub servidor pode ser https://on-prem-host-url/api/v3/.

  • Anote o nome GitHub da organização para seus repositórios em suas GitHub configurações.

  • As seguintes permissões foram adicionadas:

    Para nuvem GitHub corporativa (SaaS)

    • repositório:status

    • repositório público

    • repositório:convidar

    • leia: org

    • usuário:e-mail

    • leia:usuário

    Para servidor GitHub corporativo (no local)

    • repositório:status

    • repositório público

    • repositório:convidar

    • leia: org

    • usuário:e-mail

    • leia:usuário

    • administrador do site

  • Verificou-se que cada documento é exclusivo em GitHub outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de GitHub autenticação em umAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tiver umaIAM função ou segredo existente, poderá usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de GitHub dados àAmazon Kendra. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de GitHub dados, você deve fornecer os detalhes necessários de sua fonte de GitHub dados para que elaAmazon Kendra possa acessar seus dados. Se você ainda não configurou GitHub paraAmazon Kendra verPré-requisitos.

Console

Para se conectarAmazon Kendra a GitHub

  1. Faça login no consoleAWS de gerenciamento e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha GitHub conector e, em seguida, escolha Adicionar fonte de dados.

  5. Na página Especificar os detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em etiquetas, para Adicionar nova etiquetas, etiquetas para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as informações a seguir:

    1. GitHuborigem — escolha entre GitHub Enterprise Cloud e GitHubEnterprise Server.

    2. GitHub URL do host — Insira o nome do seu GitHub host.

    3. GitHub nome da organização — insira o nome GitHub da sua organização. Você pode encontrar as informações da organização na sua GitHub conta.

    4. AWSSecrets Managersegredo — Escolha um segredo existente ou crie um novoSecrets Manager segredo para armazenar suas credenciais de GitHub autenticação. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-GitHub -' é adicionado automaticamente ao seu nome secreto.

        2. Para GitHubtoken — insira os valores da credencial de autenticação que você criou em GitHub sua conta.

      2. Escolha Save (Salvar).

    5. Virtual Private Cloud (VPC) — Você pode optar por usar um VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    6. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    7. Escolha Próximo.

  7. Na página Definir as configurações de sincronização, insira as informações a seguir:

    1. Selecione repositórios a serem rastreados — GitHub As entidades ou tipos de conteúdo que você deseja rastrear.

    2. Registro de alterações — selecione para atualizar seu índice em vez de sincronizar todos os seus arquivos.

    3. Tipos de conteúdo — Selecione os tipos de arquivo que você deseja incluir.

    4. Padrões Regex — padrões de expressão regular para incluir ou excluir determinados arquivos. Você pode adicionar até 100 padrões.

    5. No cronograma de execução de sincronização, por frequência — com que frequência o Amazon Kendra sincronizará com sua fonte de dados.

    6. Escolha Próximo.

  8. Na página Definir mapeamentos de campos, insira as informações a seguir:

    1. Para repositório, confirmação de repositório, documento de emissão, comentário de emissão, anexo de emissão, comentário de solicitação de pull, documento de solicitação de pull, anexo de solicitação de pull — selecione entre os campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para seu índice.

    2. Adicionar campo — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra a GitHub

Você deve especificar o seguinte usando o objeto GitHubConfigurationda API:

  • Tipo de fonte de dados — especifique o tipo de fonte de dados comoSAAS ouON_PREMISE.

  • Nome secreto do recurso da Amazon (ARN) — forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação GitHub da sua conta. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "personalToken": "token"
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você ligaCreateDataSource para fornecer a umaIAM função permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o GitHub conectorAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes GitHub de dados.

Você também pode adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) — especifiqueVpcConfiguration como parte da configuração da fonte de dados. Consulte ConfiguraçãoAmazon Kendra para usar uma VPC.

    nota

    Se você usa um GitHub servidor, você deve usar umAmazon VPC para se conectar ao seu GitHub servidor.

  • Registro de alterações — seAmazon Kendra deve usar o mecanismo de registro de alterações da fonte de GitHub dados para determinar se um documento deve ser adicionado, atualizado ou excluído no índice.

    nota

    Use o registro de alterações se nãoAmazon Kendra quiser digitalizar todos os documentos. Se seu registro de alterações for grande, pode levarAmazon Kendra menos tempo para digitalizar os documentos na fonte de GitHub dados do que para processar o registro de alterações. Se você estiver sincronizando sua fonte de GitHub dados com seu índice pela primeira vez, todos os documentos serão digitalizados.

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir determinados arquivos.

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — escolha mapear seus campos GitHub de fonte de dados para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Filtragem de contexto do usuário —Amazon Kendra rastreia a Lista de Controle de Acesso (ACL) da sua fonte de dados por padrão. As informações da ACL são usadas para filtrar os resultados da pesquisa com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtragem de contexto de usuário para fontes GitHub de dados.
