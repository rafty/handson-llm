Slack

O Slack é um aplicativo de comunicação corporativa que permite que os usuários enviem mensagens e anexos por meio de vários canais públicos e privados. Você pode usarAmazon Kendra para indexar seus canais públicos e privados do Slack, bots e arquivar mensagens, arquivos e anexos, mensagens diretas e de grupo. Também é possível escolher um conteúdo específico para filtrar.

Você pode se conectarAmazon Kendra à sua fonte de dados do Slack usando o Amazon Kendraconsole e a SlackConfigurationAPI.

Para solucionar problemas com seu conector de fonte de dados doAmazon Kendra Slack, consulteSolução de problemas de origens de dados.

Recursos compatíveis

Amazon KendraO conector de fonte de dados do Slack é compatível com os seguintes recursos:

  • Log de alterações

  • Mapeamentos de campos

  • Filtragem de contexto do usuário

  • Filtros de inclusão/exclusão

  • Nuvem privada virtual (VPC)

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do Slack, faça essas alterações no Slack eAWS nas contas.

No Slack, verifique se você tem:

  • Criou um token OAuth de usuário do Slack Bot ou token OAuth de usuário do Slack. Você pode escolher qualquer token para se conectarAmazon Kendra à sua fonte de dados do Slack. Consulte a documentação do Slack sobre tokens de acesso para obter mais informações.

    nota

    Se você usar o token do bot como parte de suas credenciais do Slack, não poderá indexar mensagens diretas e mensagens de grupo e deverá adicionar o token do bot ao canal que deseja indexar.

  • Anote seu ID de equipe do Slack Workspace no URL da página principal do Slack Workspace. Por exemplo, https://app.slack.com/client/T0123456789/... onde T0123456789 é o ID da equipe.

  • Foram adicionadas as seguintes permissões [Oauth scopes/ read]:

    • canais: história

    • canais: leia

    • grupos: história

    • grupos: leia

    • im:history

    • mpim: história

    • mpim: leia

    • team:read

    • usuários.perfil: leia

    • usuários:leia

    • emoji: leia

    • arquivos:ler

    • grupos de usuários: leia

  • Verifiquei que cada documento é exclusivo no Slack e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do Slack em umAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tem umaIAM função ou segredo existente, pode usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do Slack àAmazon Kendra. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do Slack, você deve fornecer os detalhes necessários da sua fonte de dados do Slack para que elaAmazon Kendra possa acessar seus dados. Se você ainda não configurou o Slack paraAmazon Kendra verPré-requisitos.

Console

Para se conectarAmazon Kendra ao Slack

  1. Faça login no ConsoleAWS de Gerenciamento e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha Conector do Slack e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, especifique as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em etiquetas, para Adicionar nova etiqueta — etiquetas para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, especifique as informações a seguir:

    1. ID da equipe do espaço de trabalho do Slack — O ID da equipe do seu espaço de trabalho do Slack.

    2. AWSSecrets Managersegredo — Escolha um segredo existente ou crie um novoSecrets Manager segredo para armazenar suas credenciais de autenticação do Slack. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-Slack -' é adicionado automaticamente ao seu nome secreto.

        2. Para token do Slack — insira os valores da credencial de autenticação que você criou na sua conta do Slack.

      2. Escolha Save (Salvar).

    3. Virtual Private Cloud (VPC) — você pode escolher usar uma VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    4. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    5. Escolha Próximo.

  7. Na página Definir configurações de sincronização, especifique as informações a seguir:

    1. Selecione o tipo de conteúdo a ser rastreado — As entidades do Slack ou os tipos de conteúdo que você deseja rastrear.

    2. Registro de alterações — selecione para atualizar seu índice em vez de sincronizar todos os seus arquivos.

    3. Padrões Regex — padrões de expressão regular para incluir ou excluir determinados arquivos. Você pode adicionar até 100 padrões.

    4. Selecione a data de início do rastreamento — A partir de qual dataAmazon Kendra começará a rastrear seus dados.

    5. Em Sincronizar cronograma de execução, para Frequência — comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    6. Escolha Próximo.

  8. Na página Definir mapeamentos de campos, especifique as informações a seguir:

    1. Para mapeamentos de campo do Slack — selecione entre os campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para o seu índice.

    2. Adicionar campo — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Também é possível optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra ao Slack

Você deve especificar o seguinte usando a SlackConfigurationAPI:

  • ID da equipe do Slack Workspace — O ID da equipe do Slack que você copiou do URL da sua página principal do Slack.

  • Lista de entidades a serem indexadas — se vocêAmazon Kendra deve indexar seus canais públicos e privados e suas mensagens diretas e de grupo.

  • Data do rastreamento — A data para começar a rastrear seus dados da sua equipe de espaço de trabalho do Slack. A data deve seguir este formato: yyyy-mm-dd.

  • Nome secreto do recurso da Amazon (ARN) — forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação da sua conta do Slack. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "slackToken": "token"
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você ligaCreateDataSource para fornecer a umaIAM função permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector do SlackAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes de dados do Slack.

Também é possível adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) — especifiqueVpcConfiguration como parte da configuração da fonte de dados. Consulte ConfiguraçãoAmazon Kendra para usar uma VPC.

  • Registro de alterações — seAmazon Kendra deve usar o mecanismo de registro de alterações da fonte de dados do Slack para determinar se um documento deve ser adicionado, atualizado ou excluído no índice.

    nota

    Use o registro de alterações se nãoAmazon Kendra quiser digitalizar todos os documentos. Se seu registro de alterações for grande, pode levarAmazon Kendra menos tempo para digitalizar os documentos na fonte de dados do Slack do que para processar o registro de alterações. Se você estiver sincronizando sua fonte de dados do Slack com seu índice pela primeira vez, todos os documentos serão digitalizados.

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir determinados canais públicos e privados, mensagens privadas e mensagens de bots e arquivadas. Se você usar um token de bot como parte de suas credenciais de autenticação do Slack, deverá adicionar o token do bot ao canal que deseja indexar. Você não pode indexar mensagens diretas e mensagens de grupo usando um token de bot.

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — escolha mapear seus campos de fonte de dados do Slack para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Filtragem de contexto do usuário —Amazon Kendra rastreia a Lista de Controle de Acesso (ACL) da sua fonte de dados por padrão. As informações da ACL são usadas para filtrar os resultados da pesquisa com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtragem de contexto do usuário para fontes de dados do Slack.
