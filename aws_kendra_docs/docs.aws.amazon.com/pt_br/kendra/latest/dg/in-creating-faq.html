Adicionar perguntas frequentes (FAQs) diretamente a um índice

Você pode adicionar perguntas frequentes (FAQs) diretamente ao seu índice usando o console ou aCreateFaq API. A adição de perguntas frequentes a um índice é uma operação assíncrona. Você coloca os dados das perguntas frequentes em um arquivo que você armazena em umAmazon Simple Storage Service bucket. Você pode usar arquivos CSV ou JSON como entrada para suas perguntas frequentes:

  • CSV básico—Um arquivo CSV em que cada linha contém uma pergunta, uma resposta e um URL opcional com mais informações sobre a resposta.

  • CSV personalizado — Um arquivo CSV que contém perguntas, respostas e um cabeçalho que define atributos personalizados que você pode usar para facetar, exibir ou classificar respostas de perguntas frequentes. Você também pode definir atributos de controle de acesso para limitar a resposta das perguntas frequentes a determinados usuários e grupos.

  • JSON — Um arquivo JSON que contém perguntas e respostas. Opcionalmente, ele também pode incluir atributos personalizados e de controle de acesso. Você pode definir atributos para facetar, exibir e classificar as respostas de perguntas frequentes. Ou você pode usar atributos de controle de acesso que limitam a resposta das perguntas frequentes a determinados usuários e grupos.

Por exemplo, a seguir está um arquivo CSV básico que fornece respostas a perguntas sobre clínicas gratuitas em Spokane, Washington, EUA, e Mountain View, Missouri, EUA.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

Ao usar um arquivo CSV ou JSON personalizado para entrada, você pode declarar atributos personalizados para suas perguntas de perguntas frequentes. Por exemplo, você pode criar um atributo personalizado que atribua um departamento a cada pergunta de FAQ. Quando as perguntas frequentes são retornadas em uma resposta, você pode usar o departamento como uma faceta para restringir a pesquisa.

Um atributo personalizado deve ser mapeado para um campo de índice. Você pode usar um campo interno ou especificar um campo de índice personalizado. No console, você usa a página de definição do Facet para criar um campo de índice. Ao utilizar a API, você deve primeiramente criar um campo de índice usando aUpdateIndex API.

O tipo de atributo no arquivo de perguntas frequentes deve corresponder ao tipo do campo de índice associado. Por exemplo, o_authors campo embutido é um campoSTRING_LIST de tipo. Portanto, você deve fornecer valores para o_authors campo como uma lista de strings em seu arquivo de perguntas frequentes. Você pode verificar o tipo de campos de índice usando a página de definição do Facet no console ou usando aDescribeIndex API.

Ao criar um campo de índice mapeado para um atributo personalizado, você pode marcá-lo como exibível, facetável ou classificável. Não é possível fazer um atributo personalizado pesquisável.

Além dos atributos personalizados, você também pode usar os seguintes atributos integrados em um arquivo CSV ou JSON personalizado:

  • _authors(Lista de caracteres) — Uma lista de autores das respostas às perguntas do FAQ.

  • _category(Sequência de caracteres) — Uma categoria que agrupa as respostas às perguntas frequentes com outros documentos semelhantes.

  • _created_at(Sequência de caracteres codificada em ISO 8601) — A data e a hora em que a pergunta de perguntas frequentes foi criada. A data e a hora devem ser formatadas como uma string codificada em ISO 8601.

    Você também pode incluir o fuso horário no formato de data e hora ISO 8601, se necessário. Por exemplo, 2012-03-25T 12:30:10 + 01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012, às 12h30 (mais 10 segundos) no fuso horário da Europa Central.

  • _last_updated_at(Sequência de caracteres codificada em ISO 8601) — A data e a hora em que a pergunta do FAQ foi atualizada. A data e a hora devem ser formatadas como uma string codificada em ISO 8601.

    Por exemplo, 2012-03-25T 12:30:10 + 01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012, às 12h30 (mais 10 segundos) no fuso horário da Europa Central.

  • _source_uri(String) — Um URL para um documento com mais informações sobre a resposta às perguntas frequentes.

  • _version(String) — A versão da pergunta do FAQ.

  • _view_count(Longo) — O número de vezes que a pergunta de perguntas frequentes foi visualizada nos resultados da pesquisa.

Arquivo CSV básico

Use um arquivo CSV básico quando quiser usar uma estrutura simples para suas perguntas frequentes. Em um arquivo CSV básico, cada linha tem dois ou três campos: uma pergunta, uma resposta e um URI de origem opcional que aponta para um documento com mais informações.

O conteúdo do arquivo deve seguir o formato comum RFC 4180 e o tipo MIME para arquivos de valores separados por vírgula (CSV).

A seguir está um arquivo de perguntas frequentes no formato CSV básico.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

Arquivo CSV personalizado

Use um arquivo CSV personalizado quando quiser adicionar atributos personalizados às suas perguntas de perguntas frequentes. Para um arquivo CSV personalizado, você usa uma linha de cabeçalho no seu arquivo CSV para definir os atributos adicionais.

O arquivo CSV deve conter os dois atributos obrigatórios a seguir:

  • _question—A pergunta mais frequente

  • _answer—A resposta às perguntas mais frequentes

Seu arquivo pode conter atributos integrados e personalizados. Veja a seguir um exemplo de um arquivo CSV personalizado.

_question,_answer,_last_updated_at,custom_string
How many free clinics are in Spokane WA?, 13, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services
How many free clinics are there in Mountain View Missouri?, 7, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services

O conteúdo do arquivo de atributos personalizados deve seguir o formato comum RFC 4180 e o tipo MIME para arquivos de valores separados por vírgula (CSV).

Há quatro tipos de atributos personalizados:

  • Data — valores de data e hora codificados ISO 8601.

    Por exemplo, 2012-03-25T 12:30:10 + 01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012, às 12h30 (mais 10 segundos) no fuso horário da Europa Central.

  • Longos — Números, como1234.

  • Sequência de caracteres — Valores da sequência de caracteres. Se sua string contiver vírgulas, coloque o valor inteiro entre aspas duplas (“) (por exemplo,"custom attribute, and more").

  • Lista de cadeias de caracteres — uma lista de valores de string. Liste os valores em uma lista separada por virgulas entre aspas (“) (por exemplo,"item1, item2, item3"). Se a lista contiver apenas uma única entrada, você poderá omitir as aspas (por exemplo,item1).

Um arquivo CSV personalizado pode conter campos de contexto do usuário. Você pode usar esses campos para limitar o acesso às perguntas frequentes a determinados usuários e grupos. Para filtrar o contexto do usuário, o usuário deve fornecer informações do usuário e do grupo na consulta. Caso contrário, todas as perguntas frequentes relevantes serão retornadas. Para obter mais informações, consulte Filtragem no contexto do usuário.

Há quatro filtros de contexto de usuário para perguntas frequentes:

  • _acl_user_allow—Os usuários na lista de permissões podem ver as perguntas frequentes na resposta da consulta. As perguntas frequentes não são retornadas para outros usuários.

  • _acl_user_deny—Os usuários da lista de negação não conseguem ver as perguntas frequentes na resposta da consulta. As perguntas frequentes são retornadas a todos os outros usuários quando são relevantes para a consulta.

  • _acl_group_allow—Os usuários que são membros de um grupo permitido podem ver as perguntas frequentes na resposta da consulta. As perguntas frequentes não são retornadas aos usuários que são membros de outro grupo.

  • _acl_group_deny—Os usuários que são membros de um grupo negado não conseguem ver as perguntas frequentes na resposta da consulta. As perguntas frequentes são retornadas a outros grupos quando são relevantes para a consulta.

Forneça os valores das listas de permissão e negação em listas separadas por vírgula entre aspas (por exemplo,"user1,user2,user3"). Você pode incluir um usuário ou um grupo em uma lista de permissão ou de negação, mas não em ambas. Caso você tente incluir um usuário ou um grupo em ambos, receberá um erro.

Veja a seguir um exemplo de um arquivo CSV personalizado com informações de contexto do usuário.

_question, _answer, _acl_user_allow, _acl_user_deny, _acl_group_allow, _acl_group_deny
How many free clinics are in Spokane WA?, 13, userID4565, "userID6201, userID7552", groupBasicRate, "groupBasicPlusRate,groupPremiumRate"

Arquivo JSON

Você pode usar um arquivo JSON para fornecer perguntas, respostas e atributos para seu índice. Você pode adicionar qualquer um dos atributos integrados e atributos personalizados às perguntas frequentes. Esse é o esquema do arquivo JSON.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": string,
            "Answer": string,
            "Attributes": {
                string: object
                additional attributes
            },
            "AccessControlList": [
               {
                   "Name": string,
                   "Type": enum( "GROUP" | "USER" ),
                   "Access": enum( "ALLOW" | "DENY" )
               },
               additional user context
            ]
        },
        additional FAQ documents
    ]
}

Esse exemplo de arquivo JSON mostra dois documentos de perguntas frequentes. Um deles tem apenas a pergunta e a resposta necessárias. O outro também tem atributos adicionais e informações de contexto do usuário.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": "How many free clinics are in Spokane WA?",
            "Answer": "13"
        },
        {
            "Question": "How many free clinics are there in Mountain View Missouri?",
            "Answer": "7",
            "Attributes": {
                "_source_uri": "https://www.freeclinics.com",
                "_category": "Charitable Clinics"
            },
            "AccessControlList": [
               {
                   "Name": "user@amazon.com",
                   "Type": "USER",
                   "Access": "ALLOW"
               },
               {
                   "Name": "Admin",
                   "Type": "GROUP",
                   "Access": "ALLOW"
               }
            ]
        }
    ]
}

Há quatro tipos de atributos personalizados para arquivos JSON:

  • Data — Um valor de string JSON com valores de data e hora codificados em ISO 8601.

    É importante que o fuso horário seja incluído no formato de data e hora ISO 8601. Por exemplo, 2012-03-25T 12:30:10 + 01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012, às 12h30 (mais 10 segundos) no fuso horário da Europa Central.

  • Longo — Um valor numérico JSON, como1234.

  • Cadeia de caracteres — Um valor de string JSON (por exemplo,"custom attribute").

  • Lista de cadeias de caracteres — uma matriz JSON de valores de string (por exemplo,["item1,item2,item3"]).

Além dos atributos incorporados e personalizados, você pode fornecer informações de contexto do usuário para as perguntas frequentes em um arquivo JSON. Você pode fornecer informações de contexto do usuário para limitar o acesso ao conteúdo de perguntas frequentes com base em usuários e grupos. Você pode incluir um usuário ou um grupo em uma lista de permissão ou de negação, mas não em ambas. Caso você tente incluir um usuário ou um grupo em ambos, receberá um erro. Para filtrar o contexto do usuário, você deve fornecer informações do usuário e do grupo na consulta. Caso contrário, todas as perguntas frequentes relevantes serão retornadas. Para obter mais informações, consulte Filtragem no contexto do usuário.

Esse exemplo de JSON adiciona contexto de usuário a uma FAQ.

"AccessControlList": [
                {
                    "Name": "group or user name",
                    "Type": "GROUP | USER",
                    "Access": "ALLOW | DENY"
                },
                additional user context
            ]

Usando seu arquivo de perguntas frequentes

Depois de armazenar seu arquivo de entrada de perguntas frequentes em um bucket do S3, você usa o console ou aCreateFaq API para colocar as perguntas e respostas em seu índice. Se você quiser atualizar uma FAQ, exclua-a e crie-a novamente. Você usa aDeleteFaq API para excluir uma FAQ.

Você deve fornecer umaIAM função que tenha acesso ao bucket do S3 que contém seus arquivos de origem. Você especifica a função no console ou noRoleArn parâmetro. Veja a seguir um exemplo de um programa que adiciona um arquivo de FAQ a um índice.

Python
import boto3

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the IAM role ARN required to index documents in an S3 bucket
role_arn = "arn:aws:iam::${accountId}:role/${roleName}"

# Provide the S3 bucket path information to the FAQ file
faq_path = {
    "Bucket": "bucket-name",
    "Key": "FreeClinicsUSA.csv"
}

response = kendra.create_faq(
    S3Path =  faq_path,
    Name = "FreeClinicsUSA",
    IndexId = index_id,
    RoleArn = role_arn
)

print(response)
Java
package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.CreateFaqRequest;
import software.amazon.awssdk.services.kendra.model.CreateFaqResponse;
import software.amazon.awssdk.services.kendra.model.S3Path;

public class AddFaqExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String indexId = "yourIndexId";
        String roleArn = "your role for accessing S3 files";

        CreateFaqRequest createFaqRequest = CreateFaqRequest
            .builder()
            .indexId(indexId)
            .name("FreeClinicsUSA")
            .roleArn(roleArn)
            .s3Path(
                S3Path
                    .builder()
                    .bucket("an-aws-kendra-test-bucket")
                    .key("FreeClinicsUSA.csv")
                    .build())
            .build();

        CreateFaqResponse response = kendra.createFaq(createFaqRequest);

        System.out.println(String.format("The result of creating FAQ: %s", response));

    }
}

Arquivos de perguntas frequentes em outros idiomas além do inglês

Você pode indexar uma FAQ em um idioma compatível. Amazon Kendraindexa as Perguntas frequentes em inglês por padrão se você não especificar um idioma. Você especifica o código do idioma ao chamar a CreateFaqoperação ou pode incluir o código do idioma de uma FAQ nos metadados da FAQ como um campo. Se uma FAQ não tiver um código de idioma em seus metadados especificados em um campo de metadados, a FAQ será indexada usando o código de idioma especificado quando você chama aCreateFAQ operação. Para indexar um documento de perguntas frequentes em um idioma compatível no console, acesse Perguntas frequentes e selecione Adicionar perguntas frequentes. Você escolhe um idioma no menu suspenso Idioma.
