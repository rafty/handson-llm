Criar um conector de origem de dados

É possível criar um conector de fonte de dadosAmazon Kendra para se conectar e indexar seus documentos. Amazon Kendrapode se conectar à Microsoft SharePoint, ao Google Drive e a muitos outros provedores. Ao criar um conector de fonte de dados, você forneceAmazon Kendra as informações de configuração necessárias para se conectar ao seu repositório de origem. Ao contrário de adicionar documentos diretamente a um índice, você pode digitalizar periodicamente a fonte de dados para atualizar o índice.

Por exemplo, digamos que você tenha um repositório de documentos fiscais armazenados em um bucket do S3. De tempos em tempos, os documentos existentes são alterados e novos documentos são adicionados ao repositório. Se você adicionar o repositórioAmazon Kendra como fonte de dados, poderá manter seu índice atualizado configurando sincronizações periódicas entre a fonte de dados e o índice.

Você pode optar por atualizar um índice manualmente usando o console ou aStartDataSourceSyncJob API. Caso contrário, você configura um cronograma para atualizar um índice e sincronizá-lo com sua fonte de dados.

Um índice pode ter mais de uma fonte de dados. Cada fonte de dados pode ter seu próprio cronograma de atualização. Por exemplo, você pode atualizar o índice de seus documentos de trabalho diariamente, ou até mesmo de hora em hora, enquanto atualiza seus documentos arquivados manualmente sempre que o arquivo for alterado.

Se você quiser alterar os metadados ou atributos e o conteúdo do documento durante o processo de ingestão de documentos, consulte EnriquecimentoAmazon Kendra personalizado de documentos.

Observe que cada ID de documento deve ser exclusiva por índice. Você não pode criar uma fonte de dados para indexar seus documentos com suas IDs exclusivas e usar aBatchPutDocument API para indexar os mesmos documentos, ou vice-versa. Você pode excluir uma fonte de dados e usar aBatchPutDocument API para indexar os mesmos documentos, ou vice-versa.

Definindo um cronograma de atualização

Configure sua fonte de dados para ser atualizada periodicamente com o console ou usando oSchedule parâmetro ao criar ou atualizar uma fonte de dados. O conteúdo do parâmetro é uma string que contém uma string de agendamento emcron formato -format ou uma string vazia para indicar que o índice é atualizado sob demanda. Para saber o formato de uma expressão cron, consulte Programar expressões para regras no GuiaAmazon CloudWatch Events do usuário. Amazon Kendrasuporta somente expressões cron. Ele não suporta expressões de taxa.

Como definir um idioma

Você pode indexar todos os seus documentos em uma fonte de dados em um idioma compatível. Você especifica o código do idioma para todos os seus documentos em sua fonte de dados ao ligar CreateDataSource. Se um documento não tiver um código de idioma especificado em um campo de metadados, o documento será indexado usando o código de idioma especificado para todos os documentos no nível da fonte de dados. Se você não especificar um idioma,Amazon Kendra indexa documentos em uma fonte de dados em inglês por padrão. Para obter mais informações sobre os idiomas suportados, incluindo seus códigos, consulte Adição de documentos em outros idiomas além do inglês.

Você indexa todos os seus documentos em uma fonte de dados em um idioma compatível usando o console. Acesse Fontes de dados e edite sua fonte de dados ou Adicionar fonte de dados se estiver adicionando uma nova fonte de dados. Na página Especificar detalhes da fonte de dados, escolha um idioma no menu suspenso Idioma. Selecione Atualizar ou continuar para inserir as informações de configuração para se conectar à sua fonte de dados.
