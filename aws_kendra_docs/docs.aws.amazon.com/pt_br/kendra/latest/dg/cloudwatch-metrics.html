Monitorando o Amazon Kendra com a Amazon CloudWatch

Para monitorar a integridade de seus índices, use a Amazon CloudWatch. Com CloudWatch, você pode obter métricas para sincronização de documentos para seu índice. Você também pode configurar CloudWatch alarmes para serem notificados quando uma ou mais métricas excederem um limite definido por você. Por exemplo, você pode monitorar o número de documentos enviados para serem indexados ou o número de documentos que não foram indexados.

Você deve ter as CloudWatch permissões apropriadas para monitorar o Amazon Kendra CloudWatch. Para obter mais informações, consulte Controle de acesso e autenticação da Amazon CloudWatch no Guia CloudWatch do usuário da Amazon.

Visualizar mémétricas do Amazon Kendra

Veja as métricas do Amazon Kendra usando o CloudWatch console.

Para métricas (CloudWatch console)

  1. Faça login noAWS Management Console e abra o CloudWatch console em https://console.aws.amazon.com/cloudwatch/.

  2. Escolha Métricas, escolha Todas as métricas e escolha Kendra.

  3. Escolha a dimensão, informe um nome de métrica e selecione Add to graph (Adicionar ao gráfico).

  4. Escolha um valor para o intervalo de datas. A contagem da métrica para o intervalo de datas selecionado é exibida no gráfico.

Criar um alarme do

Um CloudWatch alarme monitora uma métrica por um período especificado e realiza uma ou mais ações: enviar uma notificação para uma política do Amazon SNS (Amazon SNS) ou uma política de Auto Scaling (Amazon SNS). As ações ou ações são baseadas no valor da métrica relativa a um limite especificado por um número de períodos especificado. CloudWatch também pode enviar uma mensagem do Amazon SNS quando o alarme mudar de estado.

CloudWatch os alarmes invocam ações somente quando o estado muda e persistir pelo período especificado.

Para definir um alarme

  1. Faça login noAWS Management Console e abra o CloudWatch console em https://console.aws.amazon.com/cloudwatch/.

  2. Escolha Alarms (Alarmes) e depois Create Alarm (Criar alarme).

  3. Escolha as métricas do Kendra e, em seguida, escolha uma métrica.

  4. Para Time Range, escolha um intervalo de tempo para monitorar e, em seguida, selecione Next.

  5. Preencha os campos Name (Nome) e Description (Descrição).

  6. Para Whenever, escolha >= e digite um valor máximo.

  7. Se você quiser CloudWatch enviar um e-mail quando o estado do alarme for atingido, na seção Ações, em Sempre que este alarme, escolha Estado ALARM. Em Enviar notificação para, escolha uma lista de e-mails ou escolha Nova lista e crie uma nova lista de e-mails.

  8. Visualize o alarme na seção Alarm Preview. Se você estiver satisfeito com o alarme, selecione Create Alarm.

CloudWatch Métricas para trabalhos de sincronização de índices

A tabela a seguir descreve as métricas do Amazon Kendra para trabalhos de sincronização de fontes de dados do métricas do Amazon Kendra.

Métrica Descrição
DocumentsCrawled

O número de documentos que a tarefa de sincronização digitalizou ou descobriu durante a execução.

Dimensões:

  • IndexId

  • DataSourceId

Unidade: contagem
DocumentsSubmittedForIndexing

O número de documentos que o trabalho de sincronização enviou ao índice.

Dimensões:

  • IndexId

  • DataSourceId

Unidade: contagem
DocumentsSubmittedForIndexingFailed

O número de documentos que falharam na indexação. Verifique o conteúdo do CloudWatch log da tarefa de sincronização para obter detalhes.

Dimensões:

  • IndexId

  • DataSourceId

Unidade: contagem
DocumentsSubmittedForDeletion

O número de documentos que o trabalho de sincronização solicitou que fossem removidos do índice.

Dimensões:

  • IndexId

  • DataSourceId

Unidade: contagem
DocumentsSubmittedForDeletionFailed

O número de documentos que não foram excluídos. Verifique o conteúdo do CloudWatch log da tarefa de sincronização para obter detalhes.

Dimensões:

  • IndexId

  • DataSourceId

Unidade: contagem
