Amazon S3

Amazon S3é um serviço de armazenamento de objetos que armazena dados como objetos em buckets. Você pode usarAmazon Kendra para indexarAmazon S3 seu repositório de documentos.

Atenção

Amazon Kendranão usa uma política de bucket que concede permissões a umAmazon Kendra diretor para interagir com um bucket do S3. Em vez disso, ele usaIAM funções. Certifique-se de que eleAmazon Kendra não esteja incluído como membro confiável em sua política de bucket para evitar problemas de segurança de dados ao conceder acidentalmente permissões a diretores arbitrários. No entanto, você pode adicionar uma política de bucket para usar umAmazon S3 bucket em contas diferentes. Para obter mais informações, consulte Políticas para usoAmazon S3 em várias contas (na guiaIAM Funções do S3, em IAMFunções para fontes de dados). Para obter informações sobreIAM funções para fontes de dados do S3, consulte IAMfunções.

Para se conectarAmazon S3, você especifica a conexão e outras informações no console, usando o S3DataSourceConfiguration ou usando TemplateConfigurationo. Se você usar o TemplateConfiguration objeto, poderá usar uma VPC para se conectar à sua fonte de dados. Você especifica VpcConfiguration quando liga CreateDataSource. Para obter mais informações, consulte Configuring a VPC (Configuração de uma VPC).

nota

S3DataSourceConfigurationA API não oferece suporte à configuração de uma VPC. Somente aTemplateConfiguration API e o console oferecem suporte à configuração de VPC.

Para solucionar problemas do seu conector de fonte de dadosAmazon Kendra S3, consulteSolução de problemas de origens de dados.

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do S3, faça essas alterações no S3 eAWS nas contas.

No S3, verifique se você tem:

  • Copiou o nome doAmazon S3 bucket.

    nota

    Seu bucket deve estar na mesma região do seuAmazon Kendra índice e seu índice deve ter permissão para acessar o bucket que contém seus documentos.

  • Verifiquei que cada documento é exclusivo no S3 e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

Se você não tiver umaIAM função existente, poderá usar o console para criar uma novaIAM função ao conectar sua fonte de dados do S3Amazon Kendra a. Se você estiver usando a API, forneça o ARN de umaIAM função existente e uma ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do S3, você deve fornecer os detalhes necessários da fonte de dados do S3 para que elaAmazon Kendra possa acessar seus dados. Se você ainda não configurou o S3,Amazon Kendra consultePré-requisitos.

Console

Para se conectarAmazon Kendra aAmazon S3

  1. Faça login no ConsoleAWS de Gerenciamento e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha Conector S3 e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em Tags, para Adicionar nova tag — Tags para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as seguintes informações opcionais:

    1. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    2. Virtual Private Cloud (VPC) — você pode optar por usar uma VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    3. Escolha Próximo.

  7. Na página Definir configurações de sincronização, insira as informações a seguir:

    1. No escopo de sincronização, para localização da fonte de dados — O caminho para oAmazon S3 bucket em que seus dados estão armazenados. Selecione Browse S3 para escolher seu bucket.

    2. (Opcional) Os arquivos de metadados prefixam a localização da pasta — O caminho para a pasta na qual seus metadados estão armazenados. Selecione Browse S3 para localizar sua pasta de metadados.

    3. (Opcional) Localização do arquivo de configuração da lista de controle de acesso — O caminho para o local de um arquivo contendo uma estrutura JSON que especifica as configurações de acesso para os arquivos armazenados na fonte de dados do S3. Selecione Browse S3 para localizar seu arquivo ACL.

    4. (Opcional) Selecione a chave de descriptografia — Selecione para usar uma chave de decodificação. É possível optar por usar umaAWS KMS chave do existente ou criar uma nova.

    5. (Opcional) Em Configuração adicional, para padrões Regex — adicione padrões de expressão regular para incluir ou excluir documentos do seu índice. Todos os caminhos são relativos ao bucket S3 de localização da fonte de dados. Você pode adicionar até 100 padrões.

    6. No modo de sincronização, escolha entre o modo de sincronização completa e a sincronização de conteúdo novo, modificado ou excluído para determinar como seu índice é atualizado quando o conteúdo da fonte de dados é alterado. Quando você sincroniza sua fonte de dados com o Amazon Kendra pela primeira vez, todo o conteúdo é sincronizado por padrão.

    7. Em Sincronizar cronograma de execução, para Frequência — escolha comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    8. Escolha Próximo.

  8. Na página Definir mapeamentos de campo, insira as seguintes informações opcionais:

    1. Mapeamento de campo S3 — selecione entre os campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para o seu índice.

    2. Adicionar campo — Escolha adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

TemplateConfiguration API

Para se conectarAmazon Kendra aAmazon S3

Você deve especificar um JSON do esquema da fonte de dados usando a TemplateConfigurationAPI. Você deve fornecer as seguintes informações:

  • BucketName—O nome do bucket que contém os documentos.

  • Função do IAM — especifiqueRoleArn quando você ligaCreateDataSource para fornecer a umaIAM função permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector S3Amazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes de dados do S3.

Você também pode adicionar os seguintes recursos opcionais:

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir determinados nomes de arquivo, tipos de arquivo, caminhos de arquivo e usar padrões globais (padrões que podem expandir um padrão curinga em uma lista de nomes de caminho que correspondam ao padrão fornecido). Exemplos de padrões globais incluem:

    • /myapp/config/*—Todos os arquivos dentro do diretório de configuração

    • /**/*.png—Todos os arquivos.png em todos os diretórios

    • /**/*.{png,ico,md}—Todos os arquivos.png, .ico ou.md em todos os diretórios

    • /myapp/src/**/*.ts—Todos os arquivos.ts dentro do diretório src (e todos os seus subdiretórios)

    • **/!(*.module).ts—Todos os arquivos.ts, exceto .module.ts

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — Escolha mapear seus campos de fonte de dados do S3 para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Virtual Private Cloud (VPC) — especifiqueVpcConfiguration quando você ligarCreateDataSource. Consulte ConfigurandoAmazon Kendra para usar uma VPC.

  • Configuração de metadados do documento — adicione arquivos de metadados do documento que contêm informações, como informações de controle de acesso ao documento, URI de origem, autor do documento e atributos personalizados. Cada arquivo de metadados contém metadados sobre um único documento.

Para obter uma lista de outras chaves JSON importantes a serem configuradas, consulteAmazon S3esquema de modelo.
