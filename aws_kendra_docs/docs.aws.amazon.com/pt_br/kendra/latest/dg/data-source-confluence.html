Confluence

O Confluence é uma ferramenta colaborativa de gerenciamento de trabalho projetada para compartilhar, armazenar e trabalhar no planejamento de projetos, desenvolvimento de software e gerenciamento de produtos. Você pode usarAmazon Kendra para indexar seus espaços do Confluence, páginas (incluindo páginas aninhadas), blogs, comentários e anexos em páginas e blogs indexados.

Amazon Kendrasuporta tanto o Confluence Server quanto o Confluence Cloud.

nota

Por padrão,Amazon Kendra não indexa arquivos e espaços pessoais do Confluence. Você pode optar por indexá-los ao criar a fonte de dados. Se você não quiserAmazon Kendra indexar um espaço, marque-o como privado no Confluence.

Você pode se conectarAmazon Kendra à sua fonte de dados do Confluence usando o Amazon Kendraconsole, a TemplateConfigurationAPI ou a ConfluenceConfigurationAPI.

Amazon Kendratem duas versões do conector Confluence. Os recursos compatíveis de cada versão incluem:

Conector Confluence ConfluenceConfigurationV1.0/API

  • Mapeamentos de campos:

  • Filtragem de contexto do usuário

  • Filtros de inclusão/exclusão

  • (Somente para Confluence Server) Nuvem privada virtual (VPC) (VPC)

Conector Confluence V2.0/TemplateConfigurationAPI

  • Mapeamentos de campos:

  • Filtragem de contexto do usuário

  • Nuvem privada virtual (VPC)

  • Sincronizar todos os documentos/Sincronizar somente documentos novos, modificados ou excluídos

  • Padrões de inclusão/exclusão

nota

O Support para o conector Confluence ConfluenceConfiguration V1.0/API está programado para terminar em junho de 2023. Recomendamos usar o conector Confluence V2.0/ TemplateConfiguration API.

Para solucionar problemas do seuAmazon Kendra conector de fonte de dados do Confluence, consulteSolução de problemas de origens de dados.
