Box (Caixa)

O Box é um serviço de armazenamento em nuvem que oferece recursos de hospedagem de arquivos. Você pode usarAmazon Kendra para indexar conteúdo no conteúdo do Box, incluindo comentários, tarefas e links da web.

Você pode se conectarAmazon Kendra à sua fonte de dados do Box usando o Amazon Kendraconsole e a BoxConfigurationAPI.

Para solucionar problemas do conector de fonte de dadosAmazon Kendra Box, consulteSolução de problemas de origens de dados.

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do Box, faça essas alterações no Box eAWS nas contas.

No Box, verifique se você tem:

  • Uma conta Box Enterprise ou Box Enterprise Plus.

  • Criou um aplicativo personalizado do Box no Box Developer Console e o configurou para usar a Autenticação de Servidor (com JWT). Consulte a documentação do Box sobre a criação de um aplicativo personalizado e a documentação do Box sobre como configurar o JWT Auth para obter mais detalhes.

  • Defina seu nível de acesso ao aplicativo como App + Enterprise Access e permita que ele faça chamadas de API usando o cabeçalho as-user.

  • Usou o usuário administrador para adicionar os seguintes escopos de aplicativos em seu aplicativo Box:

    • Escreva todos os arquivos e pastas armazenados em uma caixa

    • Gerenciar usuários

    • Gerenciar grupos

    • Gerencie propriedades corporativas

  • key pair públicas/privadas gerado e baixado, incluindo um ID de cliente, um segredo de cliente, um ID de chave pública, um ID de chave privada, uma frase secreta e um ID corporativo para usar como credenciais de autenticação. Consulte Par de chaves públicas e privadas para obter mais detalhes.

  • Copiou sua ID corporativa do Box das configurações do Box Developer Console ou do seu aplicativo Box. Por exemplo, 801234567.

  • Verifique se cada documento é exclusivo no Box e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do Box em umAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tiver umaIAM função ou segredo existente, poderá usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do BoxAmazon Kendra a. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do Box, você deve fornecer os detalhes necessários da fonte de dados do Box para queAmazon Kendra possa acessar seus dados. Se você ainda não configurou o Box paraAmazon Kendra verPré-requisitos.

Console

Para se conectarAmazon Kendra ao Box

  1. Faça login no ConsoleAWS de Gerenciamento e abra o Amazon Kendraconsole do.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha Conector de caixa e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em etiquetas, para Adicionar nova etiquetas — etiquetas para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definar acesso e segurança, insira as informações a seguir:

    1. ID corporativa do Box — Insira seu Box Enterprise ID.

    2. AWSSecrets Managersegredo — Escolha um segredo existente ou crie um novoSecrets Manager segredo para armazenar suas credenciais de autenticação do Box. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-Box-' é adicionado automaticamente ao seu nome secreto.

      2. Para ID do cliente, segredo do cliente, ID da chave pública, ID da chave privada e frase secreta — insira os valores da chave pública/privada que você gerou na sua conta do Box e baixou da sua conta do Box.

      3. Escolha Save (Salvar).

    3. Virtual Private Cloud (VPC) — Você pode optar por usar uma VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    4. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    5. Escolha Próximo.

  7. Na página Definar configurações de sincronização, insira as informações a seguir:

    1. Selecione entidades ou tipos de conteúdo — As entidades da caixa ou os tipos de conteúdo que você deseja rastrear. Cada comentário é indexado como um documento separado.

    2. Registro de alterações — selecione para atualizar seu índice em vez de sincronizar todos os seus arquivos.

    3. Padrões Regex — padrões de expressão regular para incluir ou excluir determinados arquivos. Você pode adicionar até 100 padrões.

    4. Em Sincronizar cronograma de execução, para Frequência — escolha comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    5. Escolha Próximo.

  8. Na página Definar mapeamentos de campos, insira as informações a seguir:

    1. Para arquivos e pastas, comentários, tarefas e links da Web — selecione um dos campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para seu índice.

    2. Adicionar campo — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra ao Box

Você deve especificar o seguinte usando a BoxConfigurationAPI:

ID corporativa do Box — Forneça seu Box Enterprise ID. Você pode encontrar a ID corporativa nas configurações do Box Developer Console ou ao criar um aplicativo no Box.

  • Nome secreto do recurso da Amazon (ARN) — Forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação da sua conta do Box. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "clientID": "client-id",
    "clientSecret": "client-secret",
    "publicKeyID": "public-key-id",
    "privateKey": "private-key",
    "passphrase": "pass-phrase"
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você ligaCreateDataSource para fornecer a umaIAM função permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector BoxAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes de dados do Box.

Você também pode adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) — especifiqueVpcConfiguration como parte da configuração da fonte de dados. Consulte ConfiguraçãoAmazon Kendra para usar uma VPC.

  • Registro de alterações — seAmazon Kendra deve usar o mecanismo de registro de alterações da fonte de dados do Box para determinar se um documento deve ser adicionado, atualizado ou excluído no índice.

    nota

    Use o registro de alterações se nãoAmazon Kendra quiser digitalizar todos os documentos. Se seu registro de alterações for grande, pode levarAmazon Kendra menos tempo para digitalizar os documentos na fonte de dados do Box do que para processar o registro de alterações. Se você estiver sincronizando sua fonte de dados do Box com seu índice pela primeira vez, todos os documentos serão digitalizados.

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir determinados arquivos, pastas, comentários, tarefas e links da web do Box.

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — Escolha mapear seus campos de fonte de dados do Box para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Filtragem de contexto do usuário —Amazon Kendra rastreia a Lista de Controle de Acesso (ACL) da sua fonte de dados por padrão. As informações da ACL são usadas para filtrar os resultados da pesquisa com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtragem de contexto do usuário para fontes de dados do Box.
