Começando com uma fonte deAWS IAM Identity Center (successor to AWS Single Sign-On) identidade (console)

Uma fonte deAWS IAM Identity Center (successor to AWS Single Sign-On) identidade contém informações sobre seus usuários e grupos. Isso é útil para configurar a filtragem de contexto do usuário, na qualAmazon Kendra filtra os resultados da pesquisa para diferentes usuários com base no acesso do usuário ou do grupo aos documentos.

Para criar uma fonte de identidade do IAM Identity Center, você deve ativar o IAM Identity Center e criar uma organização noAWS Organizations. Quando você ativa o IAM Identity Center e cria uma organização pela primeira vez, o padrão automático é o diretório do Identity Center como fonte de identidade. Você pode mudar para o Active Directory (gerenciado ou autogerenciado pela Amazon) ou um provedor de identidade externo como sua fonte de identidade. Você deve seguir a orientação correta para isso — consulte Alteração da fonte de identidade do IAM Identity Center. Você pode ter somente uma fonte de identidade por organização.

Para que seus usuários e grupos tenham diferentes níveis de acesso aos documentos, você precisa incluir seus usuários e grupos em sua Lista de Controle de Acesso ao ingerir documentos em seu índice. Isso permite que seus usuários e gruposAmazon Kendra pesquisem documentos de acordo com seu nível de acesso. Quando você emite uma consulta, a ID do usuário precisa corresponder exatamente ao nome do usuário no IAM Identity Center.

Você também precisa fornecer as permissões necessárias para usar o IAM Identity CenterAmazon Kendra. Para obter mais informações, consulte IAMFunções do IAM Identity Center.

Para configurar uma fonte de identidade do IAM Identity Center

  1. Abra o console do IAM Identity Center.

  2. Escolha Habilitar o IAM Identity Center e, em seguida, escolha CriarAWS organização.

    O diretório do Identity Center é criado por padrão e um e-mail é enviado a você para verificar o endereço de e-mail associado à organização.

  3. Para adicionar um grupo à suaAWS organização, no painel de navegação, escolha Grupos.

  4. Na página Grupos, escolha Criar grupo e insira o nome e a descrição do grupo na caixa de diálogo. Escolha Create (Criar).

  5. Para adicionar um usuário às suas Organizations, no painel de navegação, escolha Usuários.

  6. Na página Users (Usuários), selecione Add user (Adicionar usuário). Em User details (Detalhes do usuário), especifique todos os campos obrigatórios. Em Password (Senha), escolha Send an email to the user (Enviar um e-mail para o usuário). Escolha Next (Próximo).

  7. Para adicionar um usuário a um grupo, escolha Grupos e selecione um grupo.

  8. Na página Detalhes, em Membros do grupo, escolha Adicionar usuário.

  9. Na página Adicionar usuários ao grupo, selecione o usuário que você deseja adicionar como membro do grupo. Você pode selecionar vários usuário para adicionar ao grupo.

  10. Para sincronizar sua lista de usuários e grupos com o IAM Identity Center, altere sua fonte de identidade para Active Directory ou provedor de identidade externo.

    O diretório do Identity Center é a fonte de identidade padrão e exige que você adicione manualmente seus usuários e grupos usando essa fonte se você não tiver sua própria lista gerenciada por um provedor. Para alterar sua fonte de identidade, você deve seguir a orientação correta para isso — consulte Alteração da fonte de identidade do IAM Identity Center.

nota

Se estiver usando o Active Directory ou um provedor de identidade externo como sua fonte de identidade, você deve mapear os endereços de e-mail de seus usuários para os nomes de usuário do IAM Identity Center ao especificar o protocolo System for Cross-domain Identity Management (SCIM). Para obter mais informações, consulte o guia do IAM Identity Center sobre SCIM para habilitar o IAM Identity Center.

Depois de configurar sua fonte de identidade do IAM Identity Center, você pode ativá-la no console ao criar ou editar seu índice. Acesse Controle de acesso do usuário em suas configurações de índice e edite suas configurações para permitir a busca de informações de grupos de usuários do IAM Identity Center. Você também pode ativar o IAM Identity Center usando o UserGroupResolutionConfigurationobjeto. Você forneceUserGroupResolutionMode o anúncioAWS_SSO e cria umaIAM função que dá permissão para chamarsso:ListDirectoryAssociations,sso-directory:SearchUsers,sso-directory:ListGroupsForUser,sso-directory:DescribeGroups.

Atenção

Amazon Kendraatualmente não suporta o usoUserGroupResolutionConfiguration com uma conta de membro daAWS organização para sua fonte de identidade do IAM Identity Center. É necessário criar seu índice na conta de gerenciamento da organização para usarUserGroupResolutionConfiguration.

A seguir está uma visão geral de como configurar uma fonte de dadosUserGroupResolutionConfiguration e o controle de acesso de usuário para filtrar os resultados de pesquisa por contexto do usuário. Isso pressupõe que você já criou um índice e umaIAM função para os índices. Você cria um índice e fornece aIAM função usando a CreateIndexAPI.

Configurando uma fonte de dados comUserGroupResolutionConfiguration filtragem de contexto do usuário

  1. Crie uma IAMfunção que dê permissão para acessar sua fonte de identidade do IAM Identity Center.

  2. Configure UserGroupResolutionConfigurationdefinindo o modo comoAWS_SSO e ligue UpdateIndexpara atualizar seu índice para usar o IAM Identity Center.

  3. Se você quiser usar o controle de acesso de usuário baseado em token para filtrar os resultados de pesquisa por contexto do usuário, UserContextPolicydefina comoUSER_TOKEN ao ligarUpdateIndex. Caso contrário,Amazon Kendra rastreia a Lista de Controle de Acesso de cada um dos seus documentos para a maioria dos conectores de fonte de dados. Você também pode filtrar os resultados da pesquisa no contexto do usuário na API Query fornecendo informações de usuários e grupos emUserContext. Você também pode mapear usuários para seus grupos usando PutPrincipalMappingpara que você só precise fornecer o ID do usuário ao emitir a consulta.

  4. Crie uma IAMfunção que dê permissão para acessar sua fonte de dados.

  5. Configure sua fonte de dados. É necessário fornecer as informações de conexão necessárias para se conectar à sua fonte de dados.

  6. Crie uma fonte de dados usando a CreateDataSourceAPI. Forneça oDataSourceConfiguration objeto, a ID do seu índice, aIAM função da sua fonte de dados, o tipo da fonte de dados e dê um nome à sua fonte de dados. Você também pode atualizar sua fonte de dados.
