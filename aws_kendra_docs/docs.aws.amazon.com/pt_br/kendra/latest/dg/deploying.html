Implantação de Amazon Kendra

Quando chega a hora de implantar aAmazon Kendra pesquisa em seu site, fornecemos o código-fonte que você pode usar com o React para começar seu aplicativo. O código-fonte é fornecido gratuitamente sob uma licença modificada do MIT. Você pode usá-lo como está ou alterá-lo para suas próprias necessidades. O aplicativo React fornecido é um exemplo para ajudá-lo a começar. Não é um aplicativo pronto para produção.

Para implantar um aplicativo de pesquisa sem código e gerar um URL de endpoint para sua página de pesquisa com controle de acesso, consulte Amazon KendraExperience Builder.

Há dois exemplos que você pode usar com o React:

  • https://kendrasamples.s3.amazonaws.com/kendrasamples-react-app .zip — Um exemplo de aplicativo React que fornece dados de amostra e uma página de pesquisa.

  • https://kendrasamples.s3.amazonaws.com/kendrasamples.zip — Uma biblioteca que você pode adicionar a um aplicativo React existente.

Os exemplos são modelados de acordo com a página de pesquisa doAmazon Kendra console. Eles têm os mesmos recursos para pesquisar e exibir os resultados da pesquisa. Você pode usar o exemplo completo ou escolher apenas um dos recursos para seu próprio uso.

Para ver os três componentes da página de pesquisa noAmazon Kendra console, escolha o ícone do código (</>) no menu à direita. Passe o ponteiro do mouse sobre cada seção para ver uma breve descrição do componente e obter a URL da fonte do componente.

Visão geral

Você adiciona o código de exemplo a um aplicativo React existente para ativar a pesquisa. Os arquivos e componentes de pesquisa são estruturados da seguinte forma:

  • Página de pesquisa principal — Essa é a página principal que contém todos os componentes. É aqui que você integrará seu aplicativo àAmazon Kendra API.

  • Barra de pesquisa — esse é o componente em que um usuário insere um termo de pesquisa e chama a função de pesquisa.

  • Resultados — Esse é o componente do qual exibe os resultadosAmazon Kendra. Ele tem três componentes: respostas sugeridas, resultados de perguntas frequentes e documentos recomendados.

  • Facetas — Esse é o componente que mostra as facetas nos resultados da pesquisa e permite que você escolha uma faceta para limitar a pesquisa.

  • Paginação — Esse é o componente do qual pagina a respostaAmazon Kendra.

Pré-requisitos

Antes de começar, você precisará fazer o seguinte:

  • Um aplicativo React Web existente ou o aplicativo de exemplo.

  • Um ambiente de desenvolvimento configurado com as bibliotecas corretas.

  • Node.js e npm instalados.

  • OSDK for Java ouAWS SDK for JavaScript.

As informações sobre as bibliotecas eAWS os SDKs necessários estão no arquivo Readme nos arquivos zip.

Configuração do exemplo

Um procedimento completo para adicionarAmazon Kendra pesquisa a um aplicativo React está no Readme incluído nos arquivos zip de exemplo.

Para começar a usar kendra-samples-react-app .zip
Atenção

O aplicativo React não está pronto para produção. É um exemplo de implantação de um aplicativo paraAmazon Kendra pesquisa.

  1. Certifique-se de ter concluído oPré-requisitos, incluindo o download e a instalação do Node.js e do npm.

  2. Baixe o kendra-samples-react-app arquivo.zip e descompacte.

  3. Abra seu terminal e acesseaws-kendra-sample-app/src/services/. Abralocal-dev-credentials-template.json e forneça suas credenciais. Não adicione esse arquivo a nenhum repositório público.

  4. Acesseaws-kendra-sample-app/src e instale as dependências. Execute npm install.

  5. Inicie uma versão de exemplo do seu aplicativo no seu servidor local. Execute npm start. Você pode parar o servidor local digitando no tecladoCmd/Ctrl + C.

  6. Você pode alterar a porta ou o host (por exemplo, endereço IP) acessandopackage.json e atualizando o host e a porta:"start": "HOST=[host] PORT=[port] react-scripts start". Se você usa o Windows:"start": "set HOST=[host] && set PORT=[port] && react-scripts start".

  7. Se você tiver um domínio de site registrado, poderá especificá-lopackage.json após o nome do seu aplicativo. Por exemplo, "homepage": "https://mywebsite.com". Você deve executarnpm install novamente para atualizar novas dependências e depois executarnpm start.

  8. Para criar o aplicativo, executenpm build. Faça o upload do conteúdo do diretório de compilação para seu provedor de hospedagem.

Página de pesquisa principal

A página de pesquisa principal contém todos os exemplos de componentes de pesquisa. Ele inclui o componente da barra de pesquisa para saída, os componentes de resultados para exibir a resposta da API de consulta e um componente de paginação para paginar a resposta.

Componente de pesquisa

O componente de pesquisa fornece uma caixa de texto para inserir o texto da consulta. AonSearch função é um gancho que chama a função principalSearch.tsx para fazer a chamada da APIAmazon Kendra Query.

Componente de resultados

O componente de resultados mostra a resposta daQuery API. Os resultados são mostrados em três áreas distintas.

  • Respostas sugeridas — esses são os principais resultados retornados pelaQuery API. Ele contém até três respostas sugeridas. Na resposta, eles têm o tipo de resultadoANSWER.

  • Respostas de perguntas frequentes — Esses são os resultados das perguntas mais frequentes retornados pela resposta. As perguntas frequentes são adicionadas ao índice separadamente. Na resposta, eles têm o tipoQUESTION_ANSWER. Para obter mais informações, consulte Perguntas e respostas.

  • Documentos recomendados — Esses são documentos adicionais que sãoAmazon Kendra retornados na resposta. Na resposta daQuery API, eles têm o tipoDOCUMENT.

Os componentes de resultados compartilham um conjunto de componentes para recursos como destaques, títulos, links e muito mais. Os componentes compartilhados devem estar presentes para que os componentes do resultado funcionem.

Componente de facetas

O componente de facetas lista as facetas disponíveis nos resultados da pesquisa. Cada faceta classifica a resposta em uma dimensão específica, como o autor. Você pode refinar a pesquisa para uma faceta específica escolhendo uma da lista.

Depois de selecionar uma faceta, o componente é chamadoQuery com um filtro de atributos que restringe a pesquisa a documentos que correspondam à faceta.

Componente de paginação

O componente de paginação permite que você exiba os resultados da pesquisa daQuery API em várias páginas. Ele chama aQuery API com osPageNumber parâmetrosPageSize e para obter uma página específica de resultados.
