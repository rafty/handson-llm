Gmail

O Gmail é um cliente de e-mail desenvolvido pelo Google por meio do qual você pode enviar mensagens de e-mail com anexos de arquivos. As mensagens do Gmail podem ser classificadas e armazenadas na sua caixa de entrada de e-mail usando pastas e marcadores. Você pode usarAmazon Kendra para indexar suas mensagens de e-mail e anexos de mensagens. Você também pode configurarAmazon Kendra para incluir ou excluir mensagens de e-mail, anexos de mensagens e rótulos específicos para indexação.

Você pode se conectarAmazon Kendra à sua fonte de dados do Gmail usando o Amazon Kendraconsole e a TemplateConfigurationAPI.

Para solucionar problemas com seu conector de fonte de dados doAmazon Kendra Gmail, consulteSolução de problemas de origens de dados.

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do Gmail, faça essas alterações no Gmail eAWS nas contas.

No Gmail, verifique se você tem:

  • Criei uma conta de administrador do Google Cloud Platform e criei um projeto do Google Cloud.

  • Ativou a API do Gmail e a API Admin SDK em sua conta de administrador.

  • Criou uma conta de serviço e baixou uma chave privada JSON para seu Gmail. Para obter informações sobre como criar e acessar sua chave privada, consulte a documentação do Google Cloud sobre como criar uma chave de conta de serviço e credenciais de conta de serviço.

  • Copiou o e-mail da sua conta de administrador, o e-mail da sua conta de serviço e sua chave privada para usar na autenticação.

  • Foram adicionados os seguintes escopos do Oauth (usando uma função de administrador) para seu usuário e os diretórios compartilhados que você deseja indexar:

    • https://www.googleapis.com/auth/admin.directory.user.readonly

    • https://www.googleapis.com/auth/gmail.readonly

  • Verifiquei que cada documento é exclusivo no Gmail e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do Gmail em umAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tiver umaIAM função ou segredo existente, poderá usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do Gmail àAmazon Kendra. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do Gmail, você deve fornecer detalhes de suas credenciais do Gmail paraAmazon Kendra poder acessar seus dados. Se você ainda não configurou o Gmail paraAmazon Kendra, consultePré-requisitos.

Console

Para se conectarAmazon Kendra ao Gmail

  1. Faça login no ConsoleAWS de Gerenciamento e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha Conector do Gmail e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em etiquetas, para Adicionar nova etiqueta — etiquetas para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as informações a seguir:

    1. Em Autenticação por AWSSecrets Managersegredo — escolha um segredo existente ou crie um novoSecrets Manager segredo para armazenar suas credenciais de autenticação do Gmail. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo.

        2. E-mail do cliente — O e-mail do cliente que você copiou da sua conta de serviço do Google.

        3. E-mail da conta do administrador — O e-mail da conta de administrador que você gostaria de usar.

        4. Chave privada — A chave privada que você copiou da sua conta de serviço do Google.

        5. Escolha Save (Salvar).

    2. Virtual Private Cloud (VPC) VPC (VPC (VPC (VPC (VPC (VPC (VPC Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    3. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    4. Escolha Próximo.

  7. Na página Definir configurações de sincronização, insira as seguintes informações:

    1. Em Escopo de sincronização, para Tipos de entidade — selecione Anexos de mensagens para sincronizar anexos de mensagens. As mensagens serão sincronizadas por padrão.

    2. (Opcional: Para configuração adicional, insira as informações a seguir:

      1. Intervalo de datas — insira um intervalo de datas para especificar a data de início e término dos e-mails a serem rastreados.

      2. Domínios de e-mail — inclua ou exclua e-mails com base em domínios.

      3. Palavras-chave em assuntos — inclua ou exclua e-mails com base em palavras-chave em seus assuntos.

        nota

        Você também pode optar por incluir qualquer documento que corresponda a todas as palavras-chave do assunto que você inseriu.

      4. Rótulos — adicione padrões de expressão regular para incluir ou excluir rótulos específicos. Você pode adicionar até 100 padrões.

      5. Anexos — adicione padrões de expressão regular para incluir ou excluir anexos específicos. Você pode adicionar até 100 padrões.

    3. No modo Sincronização, escolha como você deseja atualizar seu índice quando o conteúdo da fonte de dados for alterado. Quando você sincroniza sua fonte de dadosAmazon Kendra pela primeira vez, todo o conteúdo é sincronizado por padrão.

      • Sincronização completa — Sincronize todo o conteúdo, independentemente do status de sincronização anterior.

      • Sincronização de conteúdo novo, modificado ou excluído — Sincronize somente conteúdo novo, modificado e excluído.

        Importante

        Como não há uma API para atualizar mensagens do Gmail excluídas permanentemente, uma sincronização de conteúdo novo, modificado ou excluído:

        • Não removerá mensagens que foram excluídas permanentemente do Gmail do seuAmazon Kendra índice

        • Não sincroniza alterações nas etiquetas de e-mail do Gmail

        Para sincronizar as alterações do rótulo da fonte de dados do Gmail e as mensagens de e-mail excluídas permanentemente com seuAmazon Kendra índice, você deve executar rastreamentos completos periodicamente.

    4. Em Sincronizar cronograma de execução, para Frequência — comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    5. Escolha Próximo.

  8. Na página Definir mapeamentos de campos, insira as informações a seguir:

    1. Para mensagens e anexos de mensagens — selecione um dos campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para o seu índice.

      nota

      Amazon KendraO conector de fonte de dados do Gmail não oferece suporte à criação de campos de índice personalizados devido às limitações da API.

    2. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

TemplateConfiguration API

Para se conectarAmazon Kendra ao Gmail

Você deve especificar o seguinte usando a TemplateConfigurationAPI:

  • Fonte de dados — você deve especificar a fonte de dados comoGMAIL.

  • Esquema da fonte de dados — inclua um JSON que contenha o esquema da fonte de dados. Para visualizar o esquema do modelo, consulte Esquemas de fonte de dados.

  • Tipo — EspecifiqueTEMPLATE como o Tipo ao ligarCreateDataSource.

  • Nome secreto do recurso da Amazon (ARN) — forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação da sua conta do Gmail. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "adminAccountEmailId": "${adminAccountEmailId}",
    "clientEmailId": "${clientEmailId}",
    "privateKey": "${privateKey}"
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você ligaCreateDataSource para fornecer a umaIAM função permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector do GmailAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes de dados do Gmail.

Você também pode adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) VPC (VPC (VpcConfigurationVPC (VPC (CreateDataSource Consulte ConfigurandoAmazon Kendra para usar uma VPC.

  • Modo de sincronização — especifique seAmazon Kendra deve atualizar seu índice sincronizando todos os documentos ou somente documentos novos, modificados e excluídos. É possível escolher entre:

    • FORCED_FULL_CRAWLpara rastrear novamente todo o conteúdo e substituir o conteúdo existente sempre que sua fonte de dados for sincronizada com seu índice

    • FULL_CRAWLrastreie incrementalmente somente conteúdo novo, modificado e excluído sempre que sua fonte de dados for sincronizada com seu índice

    Importante

    Como não há uma API para atualizar mensagens do Gmail excluídas permanentemente, uma sincronização de conteúdoFULL_CRAWL/Novo, modificado ou excluído:

    • Não removerá mensagens que foram excluídas permanentemente do Gmail do seuAmazon Kendra índice

    • Não sincroniza alterações nas etiquetas de e-mail do Gmail

    Para sincronizar as alterações do rótulo da fonte de dados do Gmail e as mensagens de e-mail excluídas permanentemente com seuAmazon Kendra índice, você deve executar rastreamentos completos periodicamente.

  • Filtros de inclusão e exclusão — Você pode especificar se deseja incluir anexos. Você também pode especificar padrões de expressão regular para incluir ou excluir mensagens e anexos específicos.

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — escolha mapear seus campos de fonte de dados do Gmail para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

    nota

    Amazon KendraO conector de fonte de dados do Gmail não oferece suporte à criação de campos de índice personalizados devido às limitações da API.
