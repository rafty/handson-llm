ConfigurandoAmazon Kendra para usar uma VPC

Amazon Kendrapode se conectar ao seuAmazon Virtual Private Cloud para indexar conteúdo armazenado em fontes de dados ou bancos de dados executados em sua nuvem privada. Ao criar um conector de fonte de dados ou um conector de banco de dados, você fornece identificadores de grupo de segurança e sub-rede para a sub-rede que contém sua fonte de dados ou banco de dados. Amazon Kendrausa essas informações para criar uma elastic network interface que ela usa para se comunicar com segurança com sua fonte de dados ou banco de dados.

Se sua fonte de dados ou banco de dados não estiver sendo executado em umAmazon VPC, você poderá conectar sua fonte de dados ou banco de dados ao seuAmazon VPC usando uma Rede Privada Virtual (VPN). Você obtém uma VPC padrão ao criar sua conta Amazon. Para obter informações sobre como configurar uma VPN, consulte a AWS VPNdocumentação.

Para usar uma VPC, você deve informarAmazon Kendra o identificador da sub-rede à qual o banco de dados pertence e os identificadores de qualquer grupo de segurança queAmazon Kendra deve ser usado para acessar a sub-rede. Por exemplo, se você estiver usando a porta padrão para um banco de dados MySQL, os grupos de segurança devem permitir o acessoAmazon Kendra à porta 3306 no host que executa o banco de dados.

Use somente sub-redes privadas na configuração VPC de sua fonte de dados ou banco de dados. Se sua instância do RDS estiver em uma sub-rede pública em sua VPC, você não poderá usar essa sub-rede diretamente para sincronizar sua fonte de dados ou banco de dados. Em vez disso, crie uma sub-rede privada que tenha acesso de saída a um gateway NAT na sub-rede pública. Ao configurar a VPC para sua fonte de dados ou banco de dados, especifique essa sub-rede privada. Para uma fonte de dados de banco de dados configurada com uma VPC, as sub-redes devem estar em um dos seguintes IDs de zona de disponibilidade:

  • Oeste dos EUA (Oregon) — usw2-az2, usw2-az3

  • Leste dos EUA (Norte da Virgínia) — use1-az2, use1-az4

  • UE (Irlanda) — euw1-az2, euw1-az2

Os identificadores de sub-redes e grupos de segurança são configurados no painelAmazon VPC de controle. Para ver os identificadores, abra oAmazon VPC console da seguinte forma:

Para visualizar identificadores de sub-rede

  1. Faça login no AWS Management Console e abra o console da Amazon VPC em https://console.aws.amazon.com/vpc/.

  2. Na página de navegação, escolha Sub-redes.

  3. Na lista de sub-redes, escolha a sub-rede que contém seu servidor de banco de dados.

  4. Na guia de descrição, anote o identificador no campo ID da sub-rede.

Para visualizar identificadores de grupos de segurança

  1. Faça login no AWS Management Console e abra o console da Amazon VPC em https://console.aws.amazon.com/vpc/.

  2. No painel de navegação, escolha Security groups (Grupos de segurança).

  3. Na lista de security groups, escolha o grupo para o qual você deseja usar o identificador.

  4. Na guia de descrição, anote o identificador no campo ID do grupo.

SeAmazon Kendra precisar rotear a conexão entre duas ou mais sub-redes, você poderá fornecer várias sub-redes. Por exemplo, se a sub-rede que contém seu servidor de banco de dados estiver sem endereços IP,Amazon Kendra poderá se conectar a uma sub-rede com endereços IP livres e rotear a conexão para a primeira sub-rede. Se você listar várias sub-redes, as sub-redes deverão conseguir se comunicar entre si. Cada sub-rede deve estar associada a uma tabela de rotas que forneça acesso externo à Internet usando um dispositivo de tradução de endereços de rede (NAT).

Também é possível fornecer vários grupos de segurança. O efeito combinado dos grupos de segurança deve permitir acessarAmazon Kendra a fonte de dados ou o servidor de banco de dados que você especificou na configuração da conexão.

Conexão a um banco de dados em uma VPC

O exemplo a seguir mostra como conectar um banco de dados MySQL em execução em uma VPC. O exemplo pressupõe que você está começando com sua VPC padrão e que precisa criar um banco de dados MySQL. Se você já tiver uma VPC, certifique-se de que ela esteja configurada conforme mostrado. Se você tiver um banco de dados MySQL, poderá usá-lo em vez de criar um novo.

Etapa 1: Configurar uma VPC

Configure sua VPC para que você tenha uma sub-rede privada e um grupo de segurançaAmazon Kendra para acessar um banco de dados MySQL em execução na sub-rede. As sub-redes fornecidas na configuração VPC devem estar nas regiões Oeste dos EUA (Oregon), Leste dos EUA (Norte da Virgínia) e UE (Irlanda).

Para configurar uma VPC

  1. Faça login no AWS Management Console e abra o console da Amazon VPC em https://console.aws.amazon.com/vpc/.

  2. No painel de navegação, escolha Tabelas de rotas e, em seguida, escolha Criar tabela de rotas.

  3. Para o campo Etiqueta de nome, insiraPrivate subnet route table. No campo VPC, escolha sua VPC e, em seguida, escolha Criar. Escolha Fechar para retornar à lista de tabelas de rotas.

  4. No painel de navegação, escolha NAT Gateways e escolha Create NAT Gateway.

  5. No campo Sub-rede, escolha a sub-rede que é a sub-rede pública e anote o ID da sub-rede.

  6. Se você não tiver um endereço IP elástico, escolha Criar novo EIP, escolha Criar um gateway NAT e, em seguida, escolha Fechar.

  7. Na página de navegação, escolha Tabelas de rotas.

  8. Na lista da tabela de rotas, escolha a tabela de rotas de sub-rede privada que você criou na etapa 3. Em Ações, escolha Editar rotas.

  9. Escolha Add route (Adicionar rota). Adicione o destino 0.0.0.0.0/0 para permitir todo o tráfego de saída para a Internet. Para Target, escolha NAT Gateway e, em seguida, escolha o gateway que você criou na etapa 4. Selecione Save routes (Salvar rotas) e Close (Fechar).

  10. Em Ações, escolha Editar associações de sub-rede.

  11. Escolha as sub-redes que você deseja que sejam privadas. Não escolha a sub-rede com o gateway NAT que você anotou anteriormente.

Etapa 2: Configurar a segurança

Em seguida, configure os security groups para seu banco de dados.

Para criar security groups

  1. Faça login no AWS Management Console e abra o console da Amazon VPC em https://console.aws.amazon.com/vpc/.

  2. Na descrição da sua VPC, observe o CIDR IPv4.

  3. No painel de navegação, escolha Grupos de segurança e, em seguida, escolha Criar grupo de segurança.

  4. Em Nome do grupo de segurança, insiraDataSourceInboundSecurityGroup. Forneça uma descrição e escolha a VPC na lista. Escolha Criar e, em seguida, escolha Fechar.

  5. Escolha a guia Inbound (Regras de entrada).

  6. Escolha Editar regras e, em seguida, escolha Adicionar regra

  7. Para um banco de dados, insira o número da porta para o intervalo de portas. Por exemplo, para o MySQL é3306, e, para HTTPS, é443. Na origem, digite o Encaminhamento Entre Domínios Sem Classificação (CIDR) da VPC. Escolha Salvar regras e, em seguida, escolha Fechar.

O grupo de segurança permite que qualquer pessoa dentro da VPC se conecte ao banco de dados e permite conexões de saída com a Internet.

Etapa 3: criar um banco de dados

Crie um banco de dados para armazenar seus documentos, ou você pode usar seu banco de dados existente. Consulte Uso de uma fonte de dados de banco de dados para obter uma lista de bancos de dadosAmazon Kendra compatíveis.

Para obter instruções sobre como criar um banco de dados MySQL, consulte Introdução a uma fonte de dados de banco de dados MySQL (console) usandoAmazon RDS.

Etapa 4: Criar um conector de fonte de dados do banco de dados

Depois de configurar sua VPC e criar seu banco de dados, você pode criar um conector de fonte de dados para o banco de dados. Consulte Uso de uma fonte de dados de banco de dados.

Certifique-se de configurar sua VPC, as sub-redes privadas que você criou em sua VPC e o grupo de segurança que você criou em sua VPC para seu banco de dados.

Para obter instruções sobre como criar uma fonte de dados para um banco de dados MySQL, consulte Introdução a uma fonte de dados de banco de dados MySQL (console) usandoAmazon RDS.
