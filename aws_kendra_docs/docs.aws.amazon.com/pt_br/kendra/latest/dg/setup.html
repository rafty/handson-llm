Configuração do Amazon Kendra

Antes de usar o Amazon Kendra, é necessário ter uma conta da Amazon Web Services (AWS). Depois deAWS criar uma conta, você pode acessar o Amazon Kendra por meio do console Amazon Kendra, doAWS Command Line Interface (AWS CLI) ou dosAWS SDKs.

Este guia inclui exemplos paraAWS CLI Java e Python.

Cadastre-se no AWS

Quando você se cadastra na Amazon Web Services (AWS), a conta da é cadastrada automaticamente em todos os serviços daAWS, incluindo o Amazon Kendra. Você será cobrado apenas pelos serviços que usar.

Se já tiver uma conta da AWS, passe para a próxima tarefa. Se você ainda não possuir uma conta da AWS, use o procedimento a seguir para criar uma.

Para cadastrar-se na AWS

  1. Abra https://aws.amazon.com e escolha Criar umaAWS conta.

  2. Siga as instruções na tela para concluir a criação da conta. Anote o número da suaAWS conta de 12 dígitos. Parte do procedimento de cadastro envolve uma chamada telefônica e a digitação de um PIN usando o teclado do telefone.

  3. Crie um usuário administrador do AWS Identity and Access Management (IAM). Consulte Criar seu primeiro grupo e usuário do IAM no Guia do usuário do AWS Identity and Access Management para obter instruções.

Regiões e endpoints

Um endpoint é uma URL que é o ponto de entrada para um serviço da Web. Cada endpoint é associado a uma região específica da AWS. Se você usar uma combinação do console Amazon KendraAWS CLI, do e dos SDKs do Amazon Kendra, preste atenção às regiões padrão, pois todos os componentes do Amazon Kendra de uma determinada campanha (índice, consulta etc.) devem ser criados na mesma região. Para as regiões e endpoints compatíveis com o Amazon Kendra, consulte Regiões e endpoints.
