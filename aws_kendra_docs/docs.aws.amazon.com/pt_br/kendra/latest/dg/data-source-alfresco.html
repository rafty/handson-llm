Ao ar livre

O Alfresco OnPrem é um serviço de gerenciamento de conteúdo que ajuda os clientes a armazenar e gerenciar seu conteúdo. Você pode usarAmazon Kendra para indexar sua biblioteca de documentos, Wiki e Blog do Alfresco.

Você pode se conectarAmazon Kendra à sua fonte de dados do Alfresco usando o Amazon Kendraconsole e a AlfrescoConfigurationAPI.

nota

O conector de fonte de dados Alfresco está atualmente no modo de visualização. Atualmente, a autenticação básica é suportada. Se você quiser usar o conector Alfresco na produção, entre em contato com o Support. No momento, não há suporte para o Alfresco PaaS.

Para solucionar problemas com seu conector de fonte de dadosAmazon Kendra Alfresco, consulteSolução de problemas de origens de dados.

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do Alfresco, faça essas alterações no Alfresco eAWS nas contas.

Em Alfresco, verifique se você tem:

  • Anotou suas credenciais de autenticação do Alfresco, que incluem um nome de usuário e senha de administrador.

  • Adicionou um usuário aoALFRESCO_ADMINISTRATORS grupo.

  • Copiou o URL do site Alfresco e o ID do site Alfresco.

  • Verifiquei que cada documento é exclusivo no Alfresco e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do Alfresco em umAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tiver umaIAM função ou segredo existente, poderá usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do AlfrescoAmazon Kendra a. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do Alfresco, você deve fornecer os detalhes necessários da fonte de dados do Alfresco para queAmazon Kendra possa acessar seus dados. Se você ainda não configurou o Alfresco paraAmazon Kendra verPré-requisitos.

Console

Para se conectarAmazon Kendra ao Alfresco

  1. Faça login no consoleAWS de gerenciamento e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha Conector Alfresco e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em etiquetas, para adicionar nova etiqueta — para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as informações a seguir:

    1. URL do site Alfresco — Insira o URL do seu site Alfresco. Por exemplo, https://hostname:8080.

    2. Localização do certificado SSL — Insira oAmazon S3 caminho.

    3. ID do site — Seu ID do site Alfresco.

    4. AWSSecrets Managersegredo — Escolha um segredo existente ou crie um novoSecrets Manager segredo para armazenar suas credenciais de autenticação do Alfresco. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-Alfresco' é adicionado automaticamente ao seu nome secreto.

        2. Para Nome de usuário e senha — Digite seu nome de usuário e senha do Alfresco.

      2. Escolha Save (Salvar).

    5. Virtual Private Cloud (VPC) — você pode optar por usar uma VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    6. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    7. Escolha Próximo.

  7. Na página Definir configurações de sincronização, insira as informações a seguir:

    1. Selecione o conteúdo a ser rastreado — As entidades ou tipos de conteúdo do Alfresco que você deseja rastrear.

    2. Padrões Regex — padrões de expressão regular para incluir ou excluir determinados arquivos. Você pode adicionar até 100 padrões.

    3. Em Sincronizar cronograma de execução, para Frequência — escolha comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    4. Escolha Próximo.

  8. Na página Definir mapeamentos de campos, insira as informações a seguir:

    1. Para Biblioteca de documentos, Wiki e Blog — selecione entre os campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para seu índice.

    2. Adicionar campo — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra ao Alfresco

Você deve especificar o seguinte usando a AlfrescoConfigurationAPI:

  • Nome secreto do recurso da Amazon (ARN) — Forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação da sua conta Alfresco. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "username": "user name",
    "password": "password"
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você ligaCreateDataSource para fornecer a umaIAM função permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector AlfrescoAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes de dados do Alfresco.

Você também pode adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) — especifiqueVpcConfiguration como parte da configuração da fonte de dados. Consulte ConfiguraçãoAmazon Kendra para usar uma VPC.

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir determinados arquivos.

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — Escolha mapear seus campos de fonte de dados do Alfresco para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Filtragem de contexto do usuário —Amazon Kendra rastreia a Lista de Controle de Acesso (ACL) da sua fonte de dados por padrão. As informações da ACL são usadas para filtrar os resultados da pesquisa com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtragem de contexto do usuário para fontes de dados do Alfresco.
