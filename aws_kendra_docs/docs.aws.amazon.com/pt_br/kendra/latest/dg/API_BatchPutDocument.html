BatchPutDocument

Adiciona um ou mais documentos a um índice.

ABatchPutDocument API permite que você ingira documentos em linha ou um conjunto de documentos armazenados em um bucket do Amazon S3 ou um conjunto de documentos. Use essa API para ingerir seu texto e texto não estruturado em um índice, adicionar atributos personalizados aos documentos e anexar uma lista de controle de acesso aos documentos adicionados ao índice.

Os documentos são indexados de forma assíncrona. Você pode ver o progresso do lote usandoAWS CloudWatch o. Todas as mensagens de erro relacionadas ao processamento do lote são enviadas ao seuAWS CloudWatch registro.

Para ver um exemplo de ingestão de documentos embutidos usando SDKs para Python e Java, consulte Adicionar arquivos diretamente a um índice.

Parâmetros de solicitação

Para obter informações sobre os parâmetros que são comuns a todas as ações, consulte Parâmetros comuns.

A solicitação aceita os dados a seguir no formato JSON.

CustomDocumentEnrichmentConfiguration

Informações de configuração para alterar os metadados e o conteúdo do documento durante o processo de ingestão de documentos quando você usa aBatchPutDocument API.

Para obter mais informações sobre como criar, modificar e excluir metadados de documentos ou fazer outras alterações de conteúdo ao ingerir documentos no Amazon Kendra, consulte Personalização de metadados de documentos durante o processo de ingestão.

Tipo: objeto CustomDocumentEnrichmentConfiguration

Obrigatório: não

Documents

Um ou mais documentos para adicionar ao índice.

Os documentos têm os seguintes limites de tamanho de arquivo.

  • Tamanho total de 50 MB para qualquer arquivo

  • 5 MB de texto extraído para qualquer arquivo

Para obter mais informações, consulte Cotas do .

Tipo: matriz de objetos Document

Membros da matriz: número mínimo de 1 item. Número máximo de 10 itens.

Obrigatório: Sim

IndexId

O identificador do índice ao qual adicionar os documentos. Você precisa criar o índice primeiro usando aCreateIndex API.

Tipo: String

Restrições de comprimento: comprimento fixo de 36.

Padrão: [a-zA-Z0-9][a-zA-Z0-9-]*

Obrigatório: Sim

RoleArn

O nome de recurso da Amazon (ARN) de uma função do IAM com permissão para acessar seu bucket do S3. Para obter mais informações, consulte Funções de acesso do IAM para Amazon Kendra

Tipo: String

Restrições de tamanho: tamanho mínimo 0. Tamanho máximo de 1284.

Padrão: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Obrigatório: não

Elementos de resposta

Se a ação for bem-sucedida, o serviço reenviará uma resposta HTTP 200.

Os dados a seguir são retornados no formato JSON pelo serviço.

FailedDocuments

Uma lista de documentos que não foram adicionados ao índice porque o documento falhou em uma verificação de validação. Cada documento contém uma mensagem de erro que indica por que o documento não pôde ser adicionado ao índice.

Se houver um erro ao adicionar um documento a um índice, o erro será relatado em seuAWS CloudWatch registro. Para obter mais informações, consulte Monitoramento do Amazon Kendra com Amazon CloudWatch Logs

Tipo: matriz de objetos BatchPutDocumentResponseFailedDocument
