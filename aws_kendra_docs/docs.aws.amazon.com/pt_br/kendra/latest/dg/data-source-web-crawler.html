Amazon KendraRastreador da Web

Você pode usar oAmazon Kendra Web Crawler para rastrear e indexar páginas da web.

Você só pode fazer crawling em sites públicos e sites que usam o protocolo de comunicação segura HTTPS (Hypertext Transfer Protocol Secure). Um erro recebido durante o crawling pode indicar que o site está bloqueado para crawling. Para rastrear sites internos, você pode configurar um proxy da web. O proxy da web deve ser voltado para o público.

Ao selecionar sites para indexar, você precisa aderir à Política de uso aceitável da Amazon e a todos os outros termos da Amazon. Lembre-se de que você só pode usar oAmazon Kendra Web Crawwwwwwwwler para indexar as próprias páginas da Web ou páginas que tenha autorização para indexar. Para saber como impedir que oAmazon Kendra Web Crawler indexe seu (s) site (s), consulteConfigurando orobots.txt arquivo para oAmazon Kendra Web Crawler.

Você pode se conectarAmazon Kendra à sua fonte de dados de rastreador da web usando o Amazon Kendraconsole e a WebCrawlerConfigurationAPI.

Para solucionar problemas do conector da fonte de dados do seu rastreador daAmazon Kendra web, consulteSolução de problemas de origens de dados.

Pré-requisitos

Antes de usarAmazon Kendra para indexar seus sites, verifique os detalhes de seus sites eAWS contas.

Para os sites, verifique se você tem:

  • Copiou os URLs semente ou de mapa dos sites nos quais você deseja indexar.

  • Para sites que exigem autenticação básica para rastreamento: copiou o nome do host do site e o número da porta.

  • Opcional: copiou o nome do host do site e o número da porta se você quiser usar um proxy da web para se conectar aos sites internos que deseja rastrear. O proxy da web deve ser voltado para o público. Amazon Kendrasuporta a conexão com servidores proxy da web que são apoiados por autenticação básica ou você pode se conectar sem autenticação.

  • Verifique se cada documento da página da web que você deseja indexar é exclusivo e está em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do rastreador da web em umAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tiver umaIAM função ou segredo existente, poderá usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do rastreador da WebAmazon Kendra. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do rastreador da Web, você deve fornecer os detalhes necessários da fonte de dados do rastreador da Web para queAmazon Kendra possa acessar seus dados. Se você ainda não configurou o web crawler,Amazon Kendra consultePré-requisitos.

Console

Para se conectarAmazon Kendra ao rastreador da web

  1. Faça login no consoleAWS de gerenciamento e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha conector de rastreador da web e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em Tags, para Adicionar nova tag — Etiquetas para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as informações a seguir:

    1. Em Origem, escolha entre URLs de origem e sitemaps de origem, dependendo do seu caso de uso, e insira os valores de cada um.

      nota

      Você pode adicionar até 10 URLs de origem e três mapa do site.

      Se você quiser rastrear um mapa do site, é importante verificar se o URL base ou raiz é o mesmo URL base usado para os URLs listados na página do seu sitemap. Por exemplo, se o URL do seu mapa do site for https://example.com/sitemap-page.html, os URLs listados nessa página de mapa do site também devem usar o URL base "https://example.com/”.

    2. (Opcional) Para proxy da Web — insira as seguintes informações:

      1. Nome do host — O nome do host em que o proxy da web é necessário.

      2. Número da porta — A porta usada pelo protocolo de transporte de URL do host. O número da porta deve ser um valor numérico entre 0 e 65535.

      3. Para credenciais de proxy da Web — escolha um segredo existente ou crie um novo segredo para armazenar suas credenciais de autenticação do rastreador da web. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      4. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-WebCrawler -' é adicionado automaticamente ao seu nome secreto.

        2. Para Nome de usuário e senha — insira essas credenciais básicas de autenticação para seus sites.

        3. Escolha Save (Salvar).

    3. (Opcional) Hosts com autenticação — Selecione para adicionar outros hosts com autenticação.

    4. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    5. Escolha Próximo.

  7. Na página Definir configurações de sincronização, insira as informações a seguir:

    1. Intervalo de rastreamento — escolha o tipo de páginas da web que você deseja rastrear.

    2. Profundidade do rastreamento — selecione o número de níveis do URL inicial queAmazon Kendra devem ser rastreados.

    3. As configurações avançadas de rastreamento e a configuração adicional inserem as seguintes informações:

      1. Tamanho máximo do arquivo — O tamanho máximo da página da Web ou do anexo a ser rastreado. Mínimo 0,000001 MB (1 byte). Máximo de 50 MB.

      2. Máximo de links por página — O número máximo de links rastreados por página. Os links são que o crawling percorre por ordem de exibição. Mínimo de 1 link/página. Máximo de 1000 links/página.

      3. Limitação máxima — O número máximo de URLs que o crawling percorre por nome de host por minuto. Mínimo de 1 URLs/nome do host/minuto. Máximo de 300 URLs/nome do host/minuto.

      4. Padrões Regex — adicione um padrão para incluir somente determinados URLs ou excluir URLs, tipos de arquivo ou arquivos específicos em seu repositório. Você pode ter um total combinado de 100 padrões.

    4. Em Sincronizar cronograma de execução, para Frequência — escolha comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    5. Escolha Próximo.

  8. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar as informações nesta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra ao rastreador da web

Você deve especificar o seguinte WebCrawlerConfigurationAPI:

  • URLs — Especifique os URLs semente ou de partida dos sites ou os URLs de mapa dos sites nos quais você deseja fazer o SeedUrlConfigurationcrawling SiteMapsConfiguration.

    Se você quiser rastrear um mapa do site, é importante verificar se o URL base ou raiz é o mesmo URL base usado para os URLs listados na página do seu sitemap. Por exemplo, se o URL do seu mapa do site for https://example.com/sitemap-page.html, os URLs listados nessa página de mapa do site também devem usar o URL base "https://example.com/”.

  • Nome secreto de recurso da Amazon (ARN) — Se um site exigir autenticação básica, você fornece o nome do host, o número da porta e um segredo que armazena suas credenciais básicas de autenticação do seu nome de usuário e senha. Você fornece o ARN usando a AuthenticationConfigurationAPI. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "username": "user name",
    "password": "password"
}

    Você pode fornecer credenciais de proxy da web usando um segredoAWSSecrets Manager secreto. Você usa a ProxyConfigurationAPI para fornecer o nome do host do site e o número da porta e, opcionalmente, o segredo que armazena suas credenciais de proxy da web.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Você também pode adicionar os seguintes recursos opcionais:

  • Modo de rastreamento — escolha rastrear somente nomes de host de sites ou nomes de host com subdomínios, ou também rastrear outros domínios aos quais as páginas da web estão vinculadas.

  • A “profundidade” ou número de níveis em um site, desde o nível semente até o crawwwwwwwwwwwwling. Por exemplo, se um site tiver três níveis — nível de índice (a semente, neste exemplo), nível de seções e nível de subseções — e você quiser apenas fazer crawling de informações do nível de índice até o nível de seções (níveis 0 a 1), poderá definir a profundidade como 1.

  • O número máximo de URLs em uma única página da Web a fazer o crawling.

  • O tamanho máximo em MB de uma página da Web a ser rastreada.

  • O número máximo de URLs que o crawling percorre por host de site por minuto.

  • As informações do proxy da web para se conectar e rastrear sites internos.

  • As informações de autenticação para acessar e fazer crawling em sites que exigem autenticação do usuário.

  • Você pode extrair metatags HTML como campos usando a ferramenta de enriquecimento de documentos personalizados. Para obter mais informações, consulte Customizing document metadata during the ingestion process (Personalização de metadados de documentos durante o processo de ingestão). Para ver um exemplo de extração de metatags HTML, consulte exemplos de CDE.

  • Filtros de inclusão e exclusão — especifique se deseja incluir ou excluir determinados URLs.

    nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.
