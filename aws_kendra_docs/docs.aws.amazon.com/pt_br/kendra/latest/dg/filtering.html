Filtragem e pesquisa de facetas

Você pode melhorar os resultados da pesquisa ou a resposta da API de consulta usando filtros. Os filtros restringem os documentos na resposta aos que se aplicam diretamente à consulta. Para criar sugestões de pesquisa facetadas, use a lógica booleana para filtrar atributos específicos do documento da resposta ou documentos que não correspondam a critérios específicos. Você pode especificar facetas usando oFacets parâmetro naQuery API.

Para pesquisar documentos com os quais você indexouAmazon Lex, useAmazon Kendra a AMAZON. KendraSearchIntent. Para ver um exemplo de configuraçãoAmazon Kendra comAmazon Lex, consulte Criação de um bot de perguntas frequentes para umAmazon Kendra índice. Você também pode fornecer um filtro para a resposta usando AttributeFilter. Esse é o filtro de consulta em JSON durante a configuraçãoAMAZON.KendraSearchIntent. Para fornecer um filtro de atributos ao configurar uma intenção de pesquisa no console, acesse o editor de intenção e escolhaAmazon Kendra consulta para fornecer um filtro de consulta em JSON. Para obter mais informaçõesAMAZON.KendraSearchIntent, consulte o guia deAmazon Lex documentação.

Facetas

As facetas são visualizações com escopo de um conjunto de resultados de pesquisa. Por exemplo, você pode fornecer resultados de pesquisa para cidades em todo o mundo, onde os documentos são filtrados por uma cidade específica à qual estão associados. Ou você pode criar facetas para exibir resultados de um autor específico.

Você pode usar um atributo de documento ou campo de metadados associado a um documento como uma faceta para que seus usuários possam pesquisar por categorias ou valores dentro dessa faceta. Você também pode exibir facetas aninhadas nos resultados da pesquisa para que seus usuários possam pesquisar não apenas por uma categoria ou campo, mas também por uma subcategoria ou subcampo.

O exemplo a seguir mostra como obter informações de faceta para o atributo personalizado “Cidade”.

response=kendra.query(
        QueryText = query,
        IndexId = index,
        Facets = [
            {
                "DocumentAttributeKey" : "City"
            }
        ]
        )

Você pode usar facetas aninhadas para restringir ainda mais a pesquisa. Por exemplo, o atributo ou faceta do documento “Cidade” inclui um valor chamado “Seattle”. Além disso, o atributo ou faceta do documento "CityRegion" inclui os valores “Norte” e “Sul” para documentos atribuídos a “Seattle”. Você pode exibir facetas aninhadas com suas contagens nos resultados da pesquisa para que os documentos possam ser pesquisados não apenas por cidade, mas também por uma região dentro de uma cidade.

Observe que facetas aninhadas podem afetar a latência da consulta. Uma regra geral é que quanto mais facetas aninhadas você usa, maior o impacto potencial na latência. Outros fatores que afetam a latência incluem o tamanho médio dos documentos indexados, o tamanho do índice, as consultas altamente complexas e a carga geral doAmazon Kendra índice.

O exemplo a seguir mostra como obter informações de facetas para o atributo personalizadoCityRegion "", como uma faceta aninhada em “Cidade”.

response=kendra.query(
        QueryText = query,
        IndexId = index,
        Facets = [
            {
                "DocumentAttributeKey" : "City",
                "Facets": [
                    {
                        "DocumentAttributeKey" : "CityRegion"
                    }
                ]
            }
        ]
        )

As informações facetárias, como a contagem de documentos, são retornadas na matriz deFacetResults respostas. Você usa o conteúdo para exibir sugestões de pesquisa facetadas em seu aplicativo. Por exemplo, se o atributo do documento “Cidade” contiver a cidade à qual uma pesquisa poderia ser aplicada, use essas informações para exibir uma lista de pesquisas por cidades. Os usuários podem escolher uma cidade para filtrar os resultados da pesquisa. Para fazer a pesquisa facetada, chame a API de consulta e use o atributo do documento escolhido para filtrar os resultados.

Você pode exibir até 10 valores de faceta por faceta para uma consulta e somente uma faceta aninhada em uma faceta. Se você quiser aumentar esses limites, entre em contato com o Support. Se você quiser limitar o número de valores de faceta por faceta a menos de 10, você pode especificar isso noFacet objeto.

Quando você usa um campo de lista de cadeias de caracteres para criar facetas, os resultados de facetas retornados são baseados no conteúdo da lista de cadeias de caracteres. Por exemplo, se você tiver um campo de lista de strings que contém dois itens, um com a lista “dachshund”, “salsicha cachorro” e outro com o valor “husky”, você obtémFacetResults três facetas.

Usando atributos do documento para filtrar os resultados da pesquisa

Por padrão,Query retorna todos os resultados da pesquisa. Para filtrar respostas, você pode realizar operações lógicas nos atributos do documento. Por exemplo, se você quiser apenas documentos para uma cidade específica, você pode filtrar os atributos de documentos personalizados “Cidade” e “Estado”. Você usa AttributeFilterpara criar uma operação booleana nos filtros que você fornece.

A maioria dos atributos pode ser usada para filtrar respostas para todos os tipos de resposta. No entanto, o_excerpt_page_number atributo só é aplicável aos tipos deANSWER resposta ao filtrar respostas.

O exemplo a seguir mostra como realizar uma operação lógica AND filtrando em uma cidade específica, Seattle, e estado, Washington. 

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {'AndAllFilters': 
            [ 
                {"EqualsTo": {"Key": "City","Value": {"StringValue": "Seattle"}}},
                {"EqualsTo": {"Key": "State","Value": {"StringValue": "Washington"}}}
            ]
            }
        )

O exemplo a seguir mostra como executar uma operação lógica OR quando qualquer uma dasSourceURI teclasFileformatAuthor, ou corresponder aos valores especificados. 

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {'OrAllFilters': 
            [ 
                {"EqualsTo": {"Key": "Fileformat","Value": {"StringValue": "AUTO_DETECT"}}},
                {"EqualsTo": {"Key": "Author","Value": {"StringValue": "Ana Carolina"}}},
                {"EqualsTo": {"Key": "SourceURI","Value": {"StringValue": "https://aws.amazonaws.com/234234242342"}}}
            ]
            }
        )

ParaStringList campos, use os filtros deContainsAll atributoContainsAny ou para retornar documentos com a string especificada. O exemplo a seguir mostra como retornar todos os documentos que têm os valores “Seattle” ou “Portland” em seus atributosLocations personalizados.

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {
                "ContainsAny": { "Key": "Locations", "Value": { "StringListValue": [ "Seattle", "Portland"] }}
            }
        )

Filtrando os atributos de cada documento nos resultados da pesquisa

Amazon Kendraretorna atributos do documento para cada documento nos resultados da pesquisa. Você pode filtrar determinados atributos do documento que você deseja incluir na resposta como parte dos resultados da pesquisa. Por padrão, todos os atributos do documento atribuídos a um documento são retornados na resposta.

No exemplo a seguir, somente os atributos_source_uri e do_author documento são incluídos na resposta de um documento.

response=kendra.query(
        QueryText = query,
        IndexId = index,
        RequestedDocumentAttributes = ["_source_uri", "_author"]
        )
