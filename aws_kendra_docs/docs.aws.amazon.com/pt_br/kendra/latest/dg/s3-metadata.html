Amazon S3metadados do documento

Você pode adicionar metadados, informações adicionais sobre um documento, aos documentos em umAmazon S3 bucket usando um arquivo de metadados. Cada arquivo de metadados é associado a um documento indexado.

Seus arquivos de metadados devem ser armazenados no mesmo bucket dos arquivos indexados. Você pode especificar um local dentro do bucket para seus arquivos de metadados usando o console ou oS3Prefix campo doDocumentsMetadataConfiguration parâmetro ao criar uma fonteAmazon S3 de dados. Se você não especificar umAmazon S3 prefixo, seus arquivos de metadados deverão ser armazenados no mesmo local dos documentos indexados.

Se você especificar umAmazon S3 prefixo para seus arquivos de metadados, eles estarão em uma estrutura de diretórios parallel aos seus documentos indexados. Amazon Kendraprocura seus metadados somente no diretório especificado. Se os metadados não forem lidos, verifique se o local do diretório corresponde ao local dos seus metadados.

Os seguintes exemplos mostram como a localização do documento indexado é mapeada para a localização do arquivo de metadados. Observe que aAmazon S3 chave do documento é anexada aoAmazon S3 prefixo dos metadados e depois sufixada com.metadata.json para formar oAmazon S3 caminho do arquivo de metadados. AAmazon S3 chave combinada, com oAmazon S3 prefixo e o.metadata.json sufixo dos metadados, não deve ter mais do que um total de 1024 caracteres. É recomendável que você mantenha suaAmazon S3 chave abaixo de 1000 caracteres para contabilizar caracteres adicionais ao combinar sua chave com o prefixo e o sufixo.

Os metadados do seu documento são definidos em um arquivo JSON. O arquivo deve ser um arquivo de texto UTF-8 sem um marcador BOM. O nome do arquivo JSON deve ser<document>.<extension>.metadata.json. Neste exemplo, “documento” é o nome do documento ao qual os metadados se aplicam e “extensão” é a extensão do arquivo do documento. A ID do documento deve ser exclusiva na<document>.<extension>.metadata.json.

O conteúdo do arquivo JSON segue esse modelo. Todos os atributos são opcionais. Se você não especificar o_source_uri, os links retornados pelos resultadosAmazon Kendra da pesquisa apontarão para oAmazon S3 bucket que contém o documento.

{
    "DocumentId": "document ID",
    "Attributes": {
        "_category": "document category",
        "_created_at": "ISO 8601 encoded string",
        "_last_updated_at": "ISO 8601 encoded string",
        "_source_uri": "document URI",
        "_version": "file version",
        "_view_count": number of times document has been viewed,
        "custom attribute key": "custom attribute value",
        additional custom attributes
    },
    "AccessControlList": [
         {
             "Name": "user name",
             "Type": "GROUP | USER",
             "Access": "ALLOW | DENY"
         }
    ],
    "Title": "document title",
    "ContentType": "For example HTML | PDF. For supported content types, see Types of documents."
}

Os campos_created_at e_last_updated_at metadados são datas codificadas pela ISO 8601. Por exemplo, 2012-03-25T 12:30:10 + 01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012, às 12h30 (mais 10 segundos) no fuso horário da Europa Central.

Você pode adicionar informações adicionais aoAttributes campo sobre um documento que você usa para filtrar consultas ou agrupar respostas de consultas. Para obter mais informações, consulte Criação de campos de documentos personalizados.

Você pode usar oAccessControlList campo para filtrar a resposta de uma consulta. Dessa forma, somente determinados usuários e grupos têm acesso aos documentos. Para obter mais informações, consulte Filtragem no contexto do usuário.
