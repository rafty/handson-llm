Adicionar documentos diretamente a um índice

Você pode adicionar documentos diretamente a um índice usando aBatchPutDocument API. Você não pode adicionar documentos diretamente usando o console. Ao utilizar o console, você usa uma fonte de dados para adicionar documentos.

Os documentos podem ser adicionados de um bucket do S3 ou fornecidos como dados binários.

Para obter uma lista dos tipos de documentos suportados por,Amazon Kendra consulte Tipos de documentos.

A adição de documentos a um índice é uma operação assíncrona. Depois de chamar aBatchPutDocument API, você aBatchGetDocumentStatus usa para monitorar o progresso da indexação de seus documentos. Quando você chama aBatchGetDocumentStatus API com uma lista de IDs de documentos, ela retorna o status do documento. Quando o status do documento éINDEXED ouFAILED, o processamento do documento está concluído. Quando o status éFAILED, aBatchGetDocumentStatus API retorna o motivo pelo qual o documento não pôde ser indexado.

Se você quiser alterar seus campos ou atributos de conteúdo e metadados do documento durante o processo de ingestão de documentos, consulte EnriquecimentoAmazon Kendra personalizado de documentos.

Se você quiser usar uma fonte de dados personalizada, cada documento enviado usando aBatchPutDocument API exige um ID de fonte de dados e um ID de execução como atributos ou campos. Para obter mais informações, consulte Atributos obrigatórios para fontes de dados personalizadas.

Observe que cada ID de documento deve ser exclusiva por índice. Você não pode criar uma fonte de dados para indexar seus documentos com suas IDs exclusivas e usar aBatchPutDocument API para indexar os mesmos documentos, ou vice-versa. Você pode excluir uma fonte de dados e usar aBatchPutDocument API para indexar os mesmos documentos ou vice-versa.
