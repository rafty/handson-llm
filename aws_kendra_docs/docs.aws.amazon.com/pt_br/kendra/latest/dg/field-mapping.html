Mapeamento de campos de origem de dados

Você pode mapear campos de documentos ou de conteúdo da sua fonte de dados para os campos do seu índice. Por exemplo, se você tiver um campo em sua fonte de dados chamado “departamento” que contém informações de departamento para um documento, você pode mapeá-lo para um campo de índice chamado “Departamento”. Dessa forma, você pode usar o campo ao consultar documentos.

Você também pode mapear campos padrão ou reservados, como_created_at. Se sua fonte de dados tiver um campo chamado “creation_date”, você poderá mapear isso para o campoAmazon Kendra reservado equivalente chamado_created_at. Você pode mapear campos para a maioria das fontes de dados.

Você pode criar mapeamentos de campo para as seguintes fontes de dados:

  • Ao ar livre

  • Banco de dados

  • Amazon FSx

  • Amazon WorkDocs

  • Box (Caixa)

  • Confluence

  • Dropbox

  • GitHub

  • Google Workspace Drives

  • Gmail

  • Jira

  • Microsoft Exchange

  • Microsoft OneDrive

  • Microsoft SharePoint

  • Microsoft Teams

  • Microsoft Yammer

  • Quip

Se você armazenar seus documentos em um bucket do S3 ou fonte de dados do S3, poderá fornecer atributos personalizados diretamente usando arquivos de metadados. Para obter mais informações, consulte Criação de campos de documentos personalizados.

O mapeamento dos campos da fonte de dados para um campo de índice é um processo de três etapas:

  1. Crie um índice. Para obter mais informações, consulte Criar um índice.

  2. Atualize o índice para adicionar campos personalizados.

  3. Crie uma fonte de dados que mapeie os campos da fonte de dados para os campos de índice.

Para atualizar o índice e adicionar campos personalizados, use o console ou aUpdateIndex API. Você pode adicionar um total de 500 campos personalizados ao seu índice.

No console, você pode escolher mapear um campo de fonte de dados para um dos nomes de campo reservados. Ou você pode optar por criar um novo campo de índice que mapeie para o campo. Para fontes de dados de banco de dados, se o nome da coluna do banco de dados corresponder ao nome de um campo reservado, o campo e a coluna serão mapeados automaticamente.

Com a UpdateIndexAPI, você adiciona campos personalizados e reservados usando oDocumentMetadataConfigurationUpdates parâmetro.

O exemplo JSON a seguir é usadoDocumentMetadataConfigurationUpdates para adicionar um campo chamado “Departamento” ao índice.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Ao criar o campo, você tem a opção de definir como o campo é usado nas pesquisas. Você pode escolher entre as seguintes opções:

  • Exibível — determina se o campo é retornado na resposta da consulta. O padrão é true.

  • Tabela de facetas — Indica que o campo pode ser usado para criar facetas. O padrão é false.

  • Pesquisável — Determina se o campo é usado na pesquisa. O padrão é true para campos de string e false para campos de número e data.

  • Classificável — Indica que o campo pode ser usado para classificar a resposta de uma consulta. Só pode ser definido para campos de data, número e sequência de caracteres. Não pode ser definido para campos de lista de cadeias de caracteres.

O exemplo JSON a seguir é usadoDocumentMetadataConfigurationUpdates para adicionar um campo chamado “Departamento” ao índice e marcá-lo como facetável.

"DocumentMetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE",
       "Search": {
           "Facetable": true
       }
   }
]

Usando campos de documentos integrados do Amazon Kendra

Com a UpdateIndex API, você pode criar campos reservados ou integrados usandoDocumentMetadataConfigurationUpdates e especificando o nome do campo reservado a ser mapeado para seu atributo de documento equivalente ou campo de documento. Também é possível criar campos personalizados dessa forma. Se você usa um conector de fonte de dados, a maioria inclui mapeamentos de campo que mapeiam os campos do documento da fonte de dados para campos deAmazon Kendra índice. Ao criar um campo, você pode configurar oSearch objeto para definir o campo como exibível, facetável, pesquisável e classificável. Você pode configurar oRelevance objeto para definir a relevância do campo, aumentando a duração, a atualização, a importância, a ordem de classificação e os valores de importância. Você não pode alterar o tipo de campo depois de criar o campo.

Amazon Kendratem os seguintes campos de documentos reservados ou integrados que você pode usar:

  • _authors—Uma lista de um ou mais autores responsáveis pelo conteúdo do documento.

  • _category—Uma categoria que coloca um documento em um grupo específico.

  • _created_at—A data e a hora no formato ISO 8601 em que o documento foi criado. Por exemplo, 2012-03-25T12:30:10+01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012 às 12h30 (mais 10 segundos) no horário da Europa Central.

  • _data_source_id— O identificador da fonte de dados que contém o documento.

  • _document_body—O conteúdo do documento.

  • _document_id—Um identificador exclusivo para o documento.

  • _document_title—O título do documento.

  • _excerpt_page_number—O número da página em um arquivo PDF em que o trecho do documento aparece. Se seu índice foi criado antes de 8 de setembro de 2020, você deve reindexar seus documentos antes de usar esse atributo.

  • _faq_id—Se for uma pergunta e resposta de perguntas frequentes, um identificador exclusivo para elas.

  • _file_type—O tipo de arquivo do documento, como pdf ou doc.

  • _last_updated_at—A data e hora no formato ISO 8601 em que o documento foi atualizado pela última vez. Por exemplo, 2012-03-25T12:30:10+01:00 é o formato de data e hora ISO 8601 para 25 de março de 2012 às 12h30 (mais 10 segundos) no horário da Europa Central.

  • _source_uri—O URI em que o documento está disponível. Por exemplo, o URI do documento no site de uma empresa.

  • _version—Um identificador para a versão específica de um documento.

  • _view_count—O número de vezes que o documento foi visualizado.

  • _language_code(String) — O código de um idioma que se aplica ao documento. A definirá como padrão o inglês se você não especificar um idioma. Para obter mais informações sobre os idiomas suportados, incluindo seus códigos, consulte Adição de documentos em outros idiomas além do inglês.

Para campos personalizados, você cria esses campos usandoDocumentMetadataConfigurationUpdates aUpdateIndex API, assim como você faz ao criar um campo reservado ou incorporado. Você deve definir o tipo de dados apropriado para seu campo personalizado. Você não pode alterar o tipo de campo depois de criar o campo.

A seguir estão os tipos que você pode definir para campos personalizados:

  • Data

  • Número

  • String

  • Lista de strings

Você cria um campo personalizado usando o console ou usando a UpdateIndexAPI. Depois de criar um campo personalizado, você o mapeia para um atributo de documento, assim como faz com um campo reservado. Se você adicionou um documento ao índice com a BatchPutDocumentAPI, mapeia os atributos com a API. Para documentos indexados de uma fonte deAmazon S3 dados, você mapeia os atributos usando um arquivo de metadados que contém uma estrutura JSON que descreve os atributos do documento. Para documentos indexados com um banco de dados ou fonte de dados que permite o mapeamento de campos, você mapeia atributos com o console ou a configuração da fonte de dados.
