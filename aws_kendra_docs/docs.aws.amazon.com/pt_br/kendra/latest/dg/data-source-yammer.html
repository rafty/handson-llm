Microsoft Yammer

O Microsoft Yammer é uma ferramenta de colaboração empresarial para mensagens, reuniões e compartilhamento de arquivos. Se você for um usuário do Microsoft Yammer, você pode usarAmazon Kendra para indexar sua fonte de dados do Microsoft Yammer.

Você pode se conectarAmazon Kendra à sua fonte de dados do Microsoft Yammer usando o Amazon Kendraconsole e a TemplateConfigurationAPI.

Para solucionar problemas do conector de fonte de dados doAmazon Kendra Microsoft Yammer, consulteSolução de problemas de origens de dados.

Recursos compatíveis

  • Log de alterações

  • Mapeamentos de campo

  • Filtros de inclusão/exclusão

  • Nuvem privada virtual (VPC)

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do Microsoft Yammer, faça essas alterações no Microsoft Yammer eAWS nas contas.

No Microsoft Yammer, verifique se você tem:

  • Criou uma conta administrativa do Microsoft Yammer.

  • Configurou um token de credencial OAuth 2.0 contendo um ID de cliente e um segredo do cliente.Consulte a documentação da Microsoft sobre tokens de acesso do Yammer para obter mais informações.

  • Verificou-se que cada documento é exclusivo no Microsoft Yammer e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Em suaAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do Microsoft Yammer em umAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tiver umaIAM função ou segredo existente, poderá usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do Microsoft Yammer a.Amazon Kendra Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Instruções de conexão

Para se conectarAmazon Kendra à sua fonte de dados do Microsoft Yammer, você deve fornecer os detalhes necessários da sua fonte de dados do Microsoft Yammer para queAmazon Kendra possa acessar seus dados. Se você ainda não configurou o Microsoft Yammer paraAmazon Kendra verPré-requisitos.

Console

Para se conectarAmazon Kendra ao Microsoft Yammer

  1. Faça login noAWS Management Console e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha Conector Microsoft Yammer e escolha Adicionar fonte de dados.

  5. Na página Especificar detalhes da fonte de dados, insira as informações a seguir:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em etiquetas, para Adicionar nova etiquetas — etiquetas para pesquisar e filtrar seus recursos ou monitorar seusAWS custos na.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as informações a seguir:

    1. Fonte — Use seu URL do Microsoft Yammer.

    2. AWSSecrets Managersegredo — Escolha um segredo existente ou crie um novoSecrets Manager segredo para armazenar suas credenciais de autenticação do Microsoft Yammer. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-Microsoft Yammer-' é adicionado automaticamente ao seu nome secreto.

        2. Para ID do cliente, segredo do cliente — insira os valores da credencial de autenticação que você gerou da sua conta do Microsoft Yammer.

      2. Escolha Save (Salvar).

    3. Virtual Private Cloud (VPC) — Você pode escolher usar uma VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

    4. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    5. Escolha Próximo.

  7. Na página Definir configurações de sincronização, insira as informações a seguir:

    1. Sincronizar conteúdo — Selecione o conteúdo a ser sincronizado.

    2. Configuração adicional — Você pode, opcionalmente, usar essas opções para indexar determinado conteúdo em vez de sincronizar todos os documentos.

    3. Modo de sincronização — Você pode escolher como deseja atualizar seu índice quando o conteúdo da fonte de dados for alterado.

      1. Se você escolher sincronização completa,Amazon Kendra sincronizará todo o conteúdo em todas as entidades, independentemente do status de sincronização anterior.

      2. Se você escolher a sincronização de conteúdo novo ou modificado,Amazon Kendra sincronizará somente conteúdo novo ou modificado.

      3. Se você escolher a sincronização de conteúdo novo, modificado ou excluído,Amazon Kendra sincronizará somente conteúdo novo, modificado ou excluído.

  8. Na página Definir mapeamentos de campos, insira as informações a seguir:

    1. Campos de fonte de dados padrão — Selecione entre os campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para o seu índice.

    2. Adicionar campo — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar suas informações a partir desta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra ao Microsoft Yammer

Você deve especificar um JSON do esquema da fonte de dados usando a TemplateConfigurationAPI. Você deve fornecer as seguintes informações:

  • Fonte de dados — você deve especificar a fonte de dados comoYAMMER.

  • Tipo — EspecifiqueTEMPLATE como o Tipo ao ligarCreateDataSource.

  • Nome secreto do recurso da Amazon (ARN) — Forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação da sua conta do Microsoft Yammer. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "client Id": "client ID",
    "client Secret": "client Secret",
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você chamaCreateDataSource para fornecer a umaIAM função permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector Microsoft YammerAmazon Kendra e. Para obter mais informações, consulte Funções do IAM para fontes de dados Microsoft Yammer.

Você também pode adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) — especifiqueVpcConfiguration quando você faz a chamadaCreateDataSource. Consulte ConfigurandoAmazon Kendra para usar uma VPC.

  • Habilitar rastreador de identidade — especifique se o rastreador de identidade deve ser ativado. Se o rastreador de identidade estiver desativado, você deverá fazer o upload das informações principais usando a PutPrincipalMappingAPI. O rastreamento das informações de identidade de usuários e grupos com acesso a determinados documentos é útil para filtrar o contexto do usuário. Os resultados da pesquisa são filtrados com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtrar o contexto do usuário.

  • nota

    A maioria das fontes de dados usa padrões de expressão regulares, que são padrões de inclusão ou exclusão chamados de filtros. Se você especificar um filtro de inclusão, somente o conteúdo que corresponda ao filtro de inclusão será indexado. Qualquer documento que não corresponda ao filtro de inclusão não é indexado. Se você especificar um filtro de inclusão e exclusão, os documentos que correspondem ao filtro de exclusão não serão indexados, mesmo que correspondam ao filtro de inclusão.

  • Mapeamentos de campo — Escolha mapear seus campos de fonte de dados do Microsoft Yammer para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).
