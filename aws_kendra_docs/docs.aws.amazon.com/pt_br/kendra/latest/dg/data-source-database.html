Amazon RDS/Aurora

Você pode indexar documentos armazenados em um banco de dados usando uma fonte de dados do banco de dados. Depois de fornecer informações de conexão para o banco de dados,Amazon Kendra conecta e indexa documentos.

Amazon Kendrasuporta os seguintes bancos de dados:

  • Amazon Aurora MySQL

  • Amazon Aurora PostgreSQL

  • Amazon RDS para MySQL

  • Amazon RDS para PostgreSQL

nota

Não há suporte para bancos de dados Aurora sem servidor.

Você pode se conectarAmazon Kendra à fonte de dados do seu banco de dados usando o Amazon Kendraconsole e a DatabaseConfigurationAPI.

Para solucionar problemas do conector da fonteAmazon Kendra de dados do banco de dados, consulteSolução de problemas de origens de dados.

Recursos compatíveis

Amazon KendraO conector de fonte de dados do banco de dados do oferece suporte aos seguintes recursos:

  • Mapeamentos de campo.

  • Filtragem de contexto do usuário

  • Nuvem privada virtual (VPC)

Pré-requisitos

Antes de usarAmazon Kendra para indexar sua fonte de dados do banco de dados, faça essas alterações no banco de dados eAWS nas contas.

Siga o banco de dados, verifique se você tem:

  • Anotou seu par de senha de usuário como credenciais básicas de autenticação para sua fonte de dados.

  • Copiou o nome do host, o número da porta, o endereço do host e o nome da tabela de dados que contém os dados do banco de dados. Para o PostgreSQL, a tabela de dados deve ser uma tabela pública.

    nota

    O host e a porta informamAmazon Kendra onde encontrar o servidor de banco de dados na Internet. O nome do banco de dados e o nome da tabela indicamAmazon Kendra onde encontrar os dados do documento no servidor do banco de dados.

  • Copiou os nomes das colunas na tabela de dados que contêm os dados do documento, a ID do documento, uma a cinco colunas para detectar se um documento foi alterado e as colunas opcionais da tabela de dados que são mapeadas para campos de índice personalizados. Você pode mapear qualquer um dos nomes de campoAmazon Kendra reservados para uma coluna da tabela.

  • Copiou as informações do tipo de mecanismo de banco de dados, como se você usaAmazon RDS para MySQL ou outro tipo.

  • Verificou-se que cada documento é exclusivo no banco de dados e em outras fontes de dados que você planeja usar para o mesmo índice. Cada fonte de dados que você deseja usar para um índice não deve conter o mesmo documento em todas as fontes de dados. As IDs de documentos são globais para um índice e devem ser exclusivas por índice.

Siga aAWS conta, verifique se você tem:

  • Criou umAmazon Kendra índice e, se estiver usando a API, anotou o ID do índice.

  • Criou umaIAM função para sua fonte de dados e, se estiver usando a API, anotou o ARN daIAM função.

  • Armazenou suas credenciais de autenticação do banco de dados em umAWSSecrets Manager segredo e, se estiver usando a API, anotou o ARN do segredo.

    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

Se você não tiver umaIAM função ou segredo existente, poderá usar o console para criar uma novaIAM função eSecrets Manager segredo ao conectar sua fonte de dados do banco de dados àAmazon Kendra. Se você estiver usando a API, deverá fornecer o ARN de umaIAM função eSecrets Manager segredo existentes e um ID de índice.

Para se conectarAmazon Kendra à fonte de dados do banco de dados, você deve fornecer os detalhes necessários da fonte de dados do banco de dados para queAmazon Kendra possa acessar seus dados. Se você ainda não configurou o banco de dados paraAmazon Kendra verPré-requisitos.

Console

Para se conectarAmazon Kendra a um banco de dados

  1. Siga o ConsoleAWS de Gerenciamento e abra o Amazon Kendraconsole.

  2. No painel de navegação esquerdo, escolha Índices e, em seguida, escolha o índice que você deseja usar na lista de índices.

    nota

    Você pode escolher definir ou editar suas configurações de controle de acesso do usuário em Configurações de índice.

  3. Na página Introdução, escolha Adicionar fonte de dados.

  4. Na página Adicionar fonte de dados, escolha conector de banco de dados e escolha Adicionar fonte de dados.

  5. Na página Especificar os detalhes da fonte de dados:

    1. Em Nome e descrição, para Nome da fonte de dados — insira um nome para sua fonte de dados. Você pode incluir hífens, mas não espaços.

    2. (Opcional) Descrição — insira uma descrição opcional para sua fonte de dados.

    3. Em Idioma, para Idioma padrão — Um idioma para filtrar seus documentos para o índice. A menos que você especifique o contrário, o idioma padrão é inglês. O idioma especificado nos metadados substitui o idioma selecionado.

    4. Em Etiquetas, para Adicionar nova etiqueta — Etiquetas para pesquisar e filtrar seus recursos ou monitorar seusAWS custos.

    5. Escolha Próximo.

  6. Na página Definir acesso e segurança, insira as informações a seguir:

    1. Endpoint — um nome de host DNS, um endereço IPv4 ou um endereço IPv6.

    2. Porta — Um número de porta.

    3. Banco de dados — Nome do banco de dados.

    4. Nome da tabela — Nome da tabela.

    5. Em Tipo de autenticação, escolha entre Existente e Novo para armazenar suas credenciais de autenticação do banco de dados. Se você optar por criar um novo segredo, uma janelaAWSSecrets Manager secreta será aberta.

      1. Insira as seguintes informações na janela Criar uma janelaAWSSecrets Manager secreta:

        1. Nome secreto — Um nome para seu segredo. O prefixo 'AmazonKendra-database-' é adicionado automaticamente ao seu nome secreto.

        2. Para Nome de usuário e senha — insira os valores da credencial de autenticação da sua conta do banco de dados.

        3. Escolha Salvar autenticação.

    6. Virtual Private Cloud (VPC) — Você pode optar por usar uma VPC. Nesse caso, você deve adicionar sub-redes e grupos de segurança VPC.

      nota

      Siga uma sub-rede privada. Se sua instância do RDS estiver em uma sub-rede pública em sua VPC, você poderá criar uma sub-rede privada que tenha acesso de saída a um gateway NAT na sub-rede pública. As sub-redes fornecidas na configuração VPC devem estar nas sub-redes dos EUA (Oregon), Leste dos EUA (N. da Virgínia) e UE (Irlanda).

    7. Função do IAM — escolha uma função existente do IAM ou crie uma nova função do IAM para acessar as credenciais do repositório e o conteúdo do índice.

      nota

      As funções do IAM usadas para índices não podem ser usadas para fontes de dados. Se você não tiver certeza se uma função existente é usada para um índice ou perguntas frequentes, escolha Criar uma nova função para evitar erros.

    8. Escolha Próximo.

  7. Na página Definir as instruções de sincronização, insira as informações a seguir:

    1. Selecione entre Aurora MySQL, MySQL, Aurora PostgreSQL e PostgreSQL com base no seu caso de uso.

    2. Colocar identificadores SQL com aspas duplas — Selecione para colocar identificadores SQL entre aspas duplas. Por exemplo, “ColumnName”.

    3. Coluna ACL e colunas de detecção de alterações — Use para configurar as colunasAmazon Kendra usadas para detecção de alterações e listas de controle de acesso.

    4. Em Sincronizar cronograma de execução, para Frequência — escolha comAmazon Kendra que frequência será sincronizada com sua fonte de dados.

    5. Escolha Próximo.

  8. Na página Definir mapeamentos de campos, insira as informações a seguir:

    1. Amazon Kendramapeamentos de campo padrão — Selecione entre os campos de fonte de dados padrãoAmazon Kendra gerados que você deseja mapear para o seu índice. Você deve adicionar os valores da coluna Banco de dados paradocument_id edocument_body

    2. Mapeamentos de campo personalizados — Para adicionar campos de fonte de dados personalizados para criar um nome de campo de índice para o qual mapear e o tipo de dados do campo.

    3. Escolha Próximo.

  9. Na página Revisar e criar, verifique se as informações inseridas estão corretas e selecione Adicionar fonte de dados. Você também pode optar por editar as informações nesta página. Sua fonte de dados aparecerá na página Fontes de dados assim que for adicionada com sucesso.

API

Para se conectarAmazon Kendra a um banco de dados

Você deve especificar o seguinte DatabaseConfigurationAPI:

  • ColumnConfiguration—Informações sobre onde o índice deve obter as informações do documento do banco de dados. Para obter mais detalhes, consulte ColumnConfiguration. Você deve especificarDocumentDataColumnName osDocumentIdColumnName campos e. A coluna mapeada para oDocumentIdColumnName campo deve ser uma coluna inteira. Siga o exemplo a seguir: um exemplo de uma configuração de coluna simples para uma fonte de dados do banco de dados: 

    "ColumnConfiguration": {
    "ChangeDetectingColumns": [
        "LastUpdateDate",
        "LastUpdateTime"
    ],
    "DocumentDataColumnName": "TextColumn",
    "DocumentIdColumnName": "IdentifierColumn",
    "DocoumentTitleColumnName": "TitleColumn",
    "FieldMappings": [
        {
            "DataSourceFieldName": "AbstractColumn",
            "IndexFieldName": "Abstract"
        }
    ]
}

  • ConnectionConfiguration— O tipo de mecanismo de banco de dados. ODatabaseHost campo deve ser o endpoint da instânciaAmazon Relational Database Service (Amazon RDS) para o banco de dados. Siga o endpoint do cluster.

  • DatabaseEngineType—Informações de configuração necessárias para se conectar a um banco de dados. Para obter mais detalhes, consulte ConnectionConfiguration.

  • Nome secreto de recurso da Amazon (ARN) — forneça o nome de recurso da Amazon (ARN) de umSecrets Manager segredo que contenha as credenciais de autenticação da sua conta de banco de dados. O segredo é armazenado em uma estrutura JSON com as seguintes chaves:

    {
    "username": "user name",
    "password": "password"
}

    O segredo pode conter mais informações. Siga o exemplo a seguir.

    
"DatabaseConfiguration": {
"ConnectionConfiguration": {
"DatabaseHost": "host.subdomain.domain.tld",
        "DatabaseName": "DocumentDatabase",
        "DatabasePort": 3306,
        "SecretArn": "arn:aws:secretmanager:region:account ID:secret/secret name",
        "TableName": "DocumentTable"
    }
}
    nota

    Certifique-se de atualizar ou alternar regularmente suas credenciais e segredos. Forneça somente o nível de acesso necessário para sua própria segurança. A reutilização de credenciais e segredos em todas as fontes de dados e nas versões 1.0 e 2.0 do conector (quando aplicável) não é recomendada.

  • Função do IAM — especifiqueRoleArn quando você chamaCreateDataSource para fornecer a umaIAM função permissões para acessar seuSecrets Manager segredo e chamar as APIs públicas necessárias para o conector de banco de dadosAmazon Kendra e. Siga mais informações, consulte Funções do IAM para fontes de dados do banco de dados.

Você também pode adicionar os seguintes recursos opcionais:

  • Virtual Private Cloud (VPC) — especifiqueVpcConfiguration como parte da configuração da fonte de dados. Consulte ConfiguraçãoAmazon Kendra para usar uma VPC.

    nota

    Siga apenas uma sub-rede privada. Se sua instância do RDS estiver em uma sub-rede pública em sua VPC, você poderá criar uma sub-rede privada que tenha acesso de saída a um gateway NAT na sub-rede pública. As sub-redes fornecidas na configuração VPC devem estar nas sub-redes dos EUA (Oregon), Leste dos EUA (N. da Virgínia) e UE (Irlanda).

  • Mapeamentos de campo — Escolha mapear os campos da fonte de dados do banco de dados para seus campos deAmazon Kendra índice. Para obter mais informações, consulte Mapping data source fields (Mapear campos de fonte de dados).

  • Filtragem de contexto do usuário —Amazon Kendra rastreia a Lista de Controle de Acesso (ACL) da sua fonte de dados por padrão. As informações da ACL são usadas para filtrar os resultados da pesquisa com base no acesso do usuário ou do grupo aos documentos. Para obter mais informações, consulte Filtragem de contexto do usuário para fontes de dados do banco de dados.
