Introdução a uma fonte de dados de banco de dados MySQL (console)

Você pode usar oAmazon Kendra console para começar a usar um banco de dados MySQL como fonte de dados. Ao usar o console, você especifica as informações de conexão necessárias para indexar o conteúdo de um banco de dados MySQL. Para obter mais informações, consulte Uso de uma origem dos dados de banco de dados.

Primeiro, você precisa criar um banco de dados MySQL e, em seguida, criar uma fonte de dados para o banco de dados.

Use o procedimento a seguir para criar um banco de dados MySQL básico. O procedimento pressupõe que você já criou um índice após a etapa 1 doConceitosAmazon Kendra.

Para criar um banco de dados MySQL

  1. Faça login no AWS Management Console e abra o console do Amazon RDS em https://console.aws.amazon.com/rds/.

  2. No painel de navegação, escolha Grupos de sub-rede e, em seguida, escolha Criar grupo de sub-rede de banco de dados.

  3. Anote o grupo e escolha sua Virtual Private Cloud (VPC). Para obter mais informações sobre como configurar uma VPC, consulte ConfigurandoAmazon Kendra para usar uma VPC.

  4. Adicione as sub-redes privadas da sua VPC. Suas sub-redes privadas são aquelas que não estão conectadas ao seu NAT. Escolha Create (Criar).

  5. No painel de navegação, escolha Databases (Bancos de dados) e Create database (Criar banco de dados).

  6. Use os parâmetros a seguir para criar o banco de dados. Deixe todos os outros parâmetros em seus padrões.

    • Opções de mecanismo — MySQL

    • Modelos — nível gratuito

    • Configurações de credenciais — Insira e confirme uma senha

    • Em Conectividade, escolha Configuração adicional de conectividade. Faça as seguintes escolhas.

      • Subnet group (Grupo de sub-redes): escolha o grupo de sub-redes que você criou na etapa 4.

      • Grupo de segurança da VPC — escolha o grupo que contém as regras de entrada e saída que você criou na sua VPC. Por exemplo, DataSourceSecurityGroup. Para obter mais informações sobre como configurar uma VPC, consulte ConfigurandoAmazon Kendra para usar uma VPC.

    • Em Configuração adicional, defina o nome inicial do banco de dados comocontent.

  7. Escolha Create database (Criar banco de dados).

  8. Na lista de bancos de dados, escolha seu novo banco de dados. Anote o endpoint do banco de dados.

  9. Depois de criar o banco de dados, crie uma tabela para armazenar os documentos. A criação de uma tabela está fora do escopo dessas instruções. Ao criar sua tabela, observe o seguinte:

    • Nome do banco de dados—content

    • Nome da tabela—documents

    • Colunas—IDTitleBody,,LastUpdate e. Você pode incluir colunas adicionais, se quiser.

Agora que você criou o banco de dados MySQL, pode criar uma fonte de dados para o banco de dados.

Para criar uma fonte de dados MySQL

  1. Faça login noAWS Management Console e abra oAmazon Kendra console em https://console.aws.amazon.com/kendra/home.

  2. No painel de navegação, escolha Indexes (Índices) e selecione o índice.

  3. Escolha Adicionar fontes de dados e, em seguida, escolha Amazon RDS.

  4. Digite um nome e uma descrição para a fonte de dados e selecione Next (Próximo).

  5. Escolha MySQL.

  6. Em Acesso à conexão, insira as seguintes informações:

    • Endpoint — O endpoint do banco de dados que você criou anteriormente.

    • Porta: o número da porta para o banco de dados. Para o MySQL, o padrão é 3306.

    • Tipo de autenticação — Escolha Novo.

    • Novo nome do contêiner secreto — Um nome para oSecrets Manager contêiner para as credenciais do banco de dados.

    • Nome de usuário — O nome de um usuário com acesso administrativo ao banco de dados.

    • Senha — A senha do usuário e, em seguida, escolha Salvar autenticação.

    • Nome do banco de dados —content.

    • Nome da tabela —documents.

    • Função do IAM — escolha Criar uma nova função e, em seguida, digite um nome para a função.

  7. Em Configuração da coluna, digite o seguinte:

    • Nome da coluna de ID do documento —ID

    • Título do documento, nome da coluna —Title

    • Nome da coluna de dados do documento —Body

  8. Em Detecção de alteração de coluna, digite o seguinte:

    • Colunas de detecção de alterações —LastUpdate

  9. Em Configurar VPC e grupo de segurança, forneça o seguinte:

    • Em Virtual Private Cloud (VPC), escolha sua VPC.

    • Em Sub-redes, escolha as sub-redes privadas que você criou na sua VPC.

    • Em grupos de segurança da VPC, escolha o grupo de segurança que contém as regras de entrada e saída que você criou na sua VPC para bancos de dados MySQL. Por exemplo, DataSourceSecurityGroup.

  10. Em Definir cronograma de execução de sincronização, escolha Executar sob demanda e, em seguida, escolha Avançar.

  11. Em Mapeamento de campo da fonte de dados, escolha Avançar.

  12. Revise a configuração da sua fonte de dados para se certificar de que está correta. Quando estiver convencido de que tudo está correto, escolha Criar.
