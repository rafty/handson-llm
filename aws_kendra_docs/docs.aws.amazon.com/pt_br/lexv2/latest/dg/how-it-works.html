Como funciona

O Amazon Lex V2 permite que você crie aplicativos usando uma interface de texto ou voz para uma conversa com um usuário. A seguir estão as etapas típicas para trabalhar com o Amazon Lex V2:

  1. Crie um bot e adicione um ou mais idiomas. Configure o bot para que ele entenda o objetivo do usuário, converse com o usuário para obter informações e cumpra a intenção do usuário.

  2. Teste o bot. Você pode usar o cliente de janela de teste fornecido pelo console Amazon Lex V2.

  3. Publique uma versão e crie um alias.

  4. Implante o bot. Você pode implantar o bot em seus próprios aplicativos ou plataformas de mensagens, como Facebook Messenger ou Slack

Antes de começar, familiarize-se com os seguintes conceitos e terminologia fundamentais do Amazon Lex V2:

  • Bot — Um bot executa tarefas automatizadas, como pedir uma pizza, reservar um hotel, pedir flores e assim por diante. Um bot Amazon Lex V2 é alimentado por recursos de reconhecimento automático de fala (ASR) e compreensão de linguagem natural (NLU).

    Os bots do Amazon Lex V2 podem entender a entrada do usuário fornecida com texto ou fala e conversar em linguagem natural.

  • Idioma — Um bot do Amazon Lex V2 pode conversar em um ou mais idiomas. Cada idioma é independente dos outros. Você pode configurar o Amazon Lex V2 para conversar com um usuário usando palavras e frases nativas. Para obter mais informações, consulte Amazon Lex V2.

  • Intenção — Uma intenção representa uma ação que o usuário deseja executar. Crie um bot para oferecer suporte a uma ou mais intenções relacionadas. Por exemplo, você pode criar uma intenção de pedir pizzas e bebidas. Para cada intenção, forneça as seguintes informações obrigatórias:

    • Nome da intenção — Um nome descritivo para a intenção. Por exemplo, OrderPizza.

    • Exemplos de enunciados — Como um usuário pode transmitir a intenção. Por exemplo, um usuário pode dizer “Posso pedir uma pizza” ou “Quero pedir uma pizza”.

    • Como cumprir a intenção — Como você deseja cumprir a intenção após o usuário fornecer as informações necessárias. Recomendamos que você crie uma função do Lambda para cumprir a intenção.

      Opcionalmente, você pode configurar a intenção para que o Amazon Lex V2 retorne as informações ao aplicativo cliente para o cumprimento necessário.

    Além das intenções personalizadas, o Amazon Lex V2 fornece intenções integradas para configurar rapidamente seu bot. Para obter mais informações, consulte Intenções e tipos de slots integrados.

    O Amazon Lex sempre inclui uma intenção alternativa para cada bot. A intenção de fallback é usada sempre que o Amazon Lex não consegue deduzir a intenção do usuário. Para obter mais informações, consulte AMAZÔNIA. FallbackIntent.

  • Slot — Uma intenção pode exigir zero ou mais slots ou parâmetros. Você adiciona slots como parte da configuração de intenção. Em tempo de execução, o Amazon Lex V2 solicita ao usuário valores de slot específicos. O usuário deve fornecer valores para todos os slots necessários antes que o Amazon Lex V2 possa cumprir a intenção.

    Por exemplo, aOrderPizza intenção exige espaços como tamanho, tipo de massa e número de pizzas. Para cada slot, você fornece o tipo de slot e uma ou mais solicitações que o Amazon Lex V2 envia ao cliente para obter valores do usuário. Um usuário pode responder com um valor de slot que contenha palavras adicionais, como “pizza grande, por favor” ou “vamos ficar com a pequena”. O Amazon Lex V2 ainda entende o valor do slot.

  • Tipo de slot — Cada slot tem um tipo. Você pode criar seu próprio tipo de slot ou usar os tipos de slot integrados. Por exemplo, você pode criar e usar os seguintes tipos de slot para a intenção OrderPizza:

    • Tamanho – Com valores de enumeração Small, Medium e Large.

    • Massa – Com valores de enumeração Thick e Thin.

    O Amazon Lex V2 também oferece tipos de slot integrados. Por exemplo, AMAZON.Number é um tipo de slot integrado que você pode usar para o número de pizzas pedidas. Para obter mais informações, consulte Intenções e tipos de slots integrados.

  • Versão — Uma versão é um resumo numerado do seu trabalho que você pode publicar para uso em diferentes partes do seu fluxo de trabalho, como desenvolvimento, implantação beta e produção. Depois de criar uma versão, você pode usar um bot como ele existia quando a versão foi criada. Depois de criar uma versão, ela permanece a mesma enquanto você continua trabalhando em seu aplicativo.

  • Alias — Um alias é um indicador de uma versão específica de um bot. Com um alias, você pode atualizar a versão que seus aplicativos cliente estão usando. Por exemplo, você pode apontar um alias para a versão 1 do seu bot. Quando estiver pronto para atualizar o bot, você publica a versão 2 e altera o alias para apontar para a nova versão. Como suas aplicações usam o alias ao invés de uma versão específica, todos os seus clientes obtêm a nova funcionalidade sem a necessidade de atualizações.

Para obter uma lista dasAWS regiões em que o Amazon Lex V2 está disponível, consulte endpoints e cotas do Amazon Lex V2 na Referência geral da Amazon Web Services.
