Tarefas que exigem credenciais de usuário raiz da

Importante

As seguintes ações AWS Identity and Access Management (IAM) chegarão ao fim do suporte padrão em julho de 2023: aws-portal:ModifyAccount e. aws-portal:ViewAccount Consulte Como usar AWS Billing ações refinadas para substituir essas ações por ações refinadas para que você tenha acesso aos consoles doAWS Billing, do e dasAWS. AWS

Se você criou sua ou AWS conta AWS Organizations de gerenciamento da antes de 6 de março de 2023, as ações refinadas entrarão em vigor a partir de julho de 2023. Recomendamos adicionar as ações refinadas, mas não remova suas permissões com prefixos aws-portal ou purchase-orders.

Se tiver criado sua ou AWS conta AWS Organizations de gerenciamento da a partir de 6 de março de 2023, as ações refinadas já estão em vigor.

Recomendamos que você configure um usuário administrativo AWS IAM Identity Center (successor to AWS Single Sign-On) para realizar tarefas diárias e acessar AWS recursos. Porém, as tarefas listadas abaixo podem ser executadas apenas quando você fizer login como o usuário raiz de uma conta.

Tarefas

  • Altere as configurações da conta. Isso inclui o nome da conta, endereço de e-mail, senha do usuário raiz e chaves de acesso do usuário raiz. Outras configurações da conta, como informações de contato, preferência de moeda de pagamento eRegiões da AWS, não exigem credenciais de usuário root.

  • Restaure as permissões do usuário do IAM. Se o único administrador do IAM revogar acidentalmente suas próprias permissões, você poderá entrar como usuário root para editar políticas e restaurar essas permissões.

  • Habilitar o acesso do IAM ao console do Billing and Cost Management.

  • Exiba determinadas faturas de imposto. Um usuário do IAM com o aws-portal: ViewBilling permissão pode visualizar e baixar faturas de IVA da AWS Europa, mas não da AWS Inc. ou da Amazon Internet Services Private Limited (AISPL).

  • Feche seu Conta da AWS.

  • Registrado como um vendedor no Marketplace de instâncias reservadas.

  • Configure um bucket do Amazon S3 para habilitar a MFA (autenticação multifator).

  • Edite ou exclua uma política de bucket do Amazon Simple Storage Service (Amazon S3) que inclua uma ID de nuvem privada virtual (VPC) ou ID de endpoint VPC inválida.

  • Cadastre-se no AWS GovCloud (US).

  • Solicite chaves de acesso do usuário root da AWS GovCloud (US) conta emAWS Support.
