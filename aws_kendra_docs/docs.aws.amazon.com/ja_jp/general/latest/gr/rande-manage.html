AWS リージョン の管理

AWS リージョン は地理的な領域内の AWS リソースのコレクションです。各 AWS リージョン は他のリージョンから分離され、独立しています。リージョンでは耐障害性や安定性が提供され、レイテンシーを低減することもできます。複数のリージョンでリソースをプロビジョニングすることで、リージョンが停止した場合にもアプリケーションに影響を与えずに、利用可能な状態を維持することができます。

あるリージョンで作成したリソースは、AWS のサービスで提供されるレプリケーション機能を明示的に使用しないかぎり、他のリージョンに存在することはありません。たとえば、Amazon S3 と Amazon EC2 はクロスリージョンのレプリケーションをサポートしています。AWS Identity and Access Management (IAM) などの一部のサービスには、リージョンリソースがありません。

「ポリシー条件」を使用して、AWS リージョン 内の AWS のサービス へのアクセスを制御できます。

アカウントにより、利用できるリージョンが決まります。

  • AWS アカウント では複数のリージョンが提供されるため、要件に合った場所で AWS リソースを起動できます。例えば、ヨーロッパの顧客に近づけるため、または法的要件を満たすために、ヨーロッパでインスタンスを起動することができます。

  • AWS GovCloud (米国西部) アカウントでは、AWS GovCloud (米国西部) リージョンおよび AWS GovCloud (米国東部) リージョンにアクセスできます。詳細については、「AWS GovCloud (US)」を参照してください。

  • Amazon AWS (中国) アカウントでは、北京および寧夏リージョンにのみアクセスできます。詳細については、「Amazon Web Services in China」(中国でのアマゾン ウェブ サービス) を参照してください。

リージョンの有効化

以下のリージョンのいずれかを使用する前に、アカウントのリージョンを有効にする必要があります。

名前 コード
アフリカ (ケープタウン) af-south-1
アジアパシフィック (香港) ap-east-1
アジアパシフィック (ハイデラバード) ap-south-2
アジアパシフィック (ジャカルタ) ap-southeast-3
アジアパシフィック (メルボルン) ap-southeast-4
欧州 (ミラノ) eu-south-1
欧州 (スペイン) eu-south-2
欧州 (チューリッヒ) eu-central-2
中東 (バーレーン) me-south-1
中東 (アラブ首長国連邦) me-central-1

リージョンを有効にすると、そのリージョンへの IAM リソースの配信など、AWS がそのリージョンでアカウントを準備するためのアクションを実行します。ほとんどのアカウントでは、このプロセスに数分かかりますが、数時間かかることがあります。このプロセスが完了するまでは、リージョンを使用することはできません。

要件

リージョンを有効にするには、適切な許可が必要です。IAM ポリシーの例を確認するには、「IAM ユーザーガイド」の「AWS リージョン の有効化と無効化の許可」を参照してください。

コンソールを使用してリージョンを有効にするには

  1. AWS Management Console にサインインします。

  2. コンソールの右上で、現在のリージョンを選択してリージョンのリストを展開し、リストの一番下までスクロールして、[リージョンの管理] を選択します。

  3. 使用したいリージョンの [アクション] 列を確認します。

    • [ステータス] 列にデフォルトで [デフォルトで有効化済み] と表示されている場合は、それ以上のアクションは必要ありません。アカウントのリージョンは自動で有効になっています。

    • [ステータス] 列に [無効化済み] と表示されている場合は、[有効化] を選択します。情報テキストを確認し、[リージョンの有効化] を選択します。リージョンが使用できるようになるまで待ちます。

AWS CLI を使用してリージョンを有効にするには

次の enable-region コマンドを使用します。

aws account enable-region --region-name af-south-1

リージョンの無効化

自動的に有効にならないリージョンを有効にした場合、必要に応じてそのリージョンを無効にできます。リージョンを無効にすると、そのリージョンのリソースは最終的な整合性に基づいて利用できなくなります。ただし、リソースは削除されません。このリージョンをもう一度有効にすることで、それらにアクセスできます。

要件

  • デフォルトで有効になっているリージョンは無効にできません。(前セクションの表を参照してください。)

  • リージョンを無効にするには、リージョンを無効にするアクセス許可が必要です。IAM ポリシーの例を確認するには、IAM ユーザーガイドの「AWS リージョンの有効化と無効化の許可」を参照してください。

  • リージョンを無効にする前に、そのリージョンからすべてのリソースを削除することをお勧めします。リージョンを無効にすると、AWS マネジメントコンソールまたは IAM プリンシパルを使用する AWS API を通して、そのアカウントからリージョンのリソースを表示または管理することができなくなります。ただし、そのリージョンのリソースに対して引き続き料金が発生することがあります。詳細については、AWS Billing and Cost Management ユーザーガイドの「リージョンの有効化と無効化」を参照してください。

コンソールを使用してリージョンを無効にするには

  1. AWS Management Console にサインインします。

  2. コンソールの右上で、現在のリージョンを選択してリージョンのリストを展開し、リストの一番下までスクロールして、[リージョンの管理] を選択します。

  3. リージョンの [アクション] 列で、[無効化] を選択します。

  4. ダイアログボックスの情報テキストを確認し、[リージョンを無効にする] を選択します。

AWS CLI を使用してリージョンを無効にするには

次の disable-region コマンドを使用します。

aws account disable-region --region-name af-south-1

AWS CLI を使用したリージョンの記述

アカウントで利用可能なすべてのリージョンを記述するには、list-regions コマンドを使用します。記述にはステータスも含まれます。

aws account list-regions --max-results 50

get-region-opt-status コマンドを使用して、特定のリージョンのステータスを記述します。

aws account get-region-opt-status --region-name af-south-1

アカウントで自動的に有効になっているリージョンの場合、これらのコマンドの出力には次の事項が含まれます。

"RegionOptStatus": "ENABLED_BY_DEFAULT"

手動で有効にする必要があるリージョンの場合、出力には以下が含まれます。

"RegionOptStatus": "DISABLED"

リージョンを有効にすると、出力には次が含まれます。

"RegionOptStatus": "ENABLED"
