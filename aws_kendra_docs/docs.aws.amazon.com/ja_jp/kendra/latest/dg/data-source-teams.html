マイクロソフトチーム

Microsoft Teamsは、メッセージング、会議、ファイル共有のためのエンタープライズコラボレーションツールです。Microsoft Teams ユーザーの場合は、Amazon Kendraを使用して Microsoft Teams データソースのインデックスを作成できます。

Amazon Kendraコンソールと TemplateConfigurationAPIAmazon Kendra を使用して Microsoft Teams データソースに接続できます。

前提条件

Amazon Kendraを使用して Microsoft Teams データソースのインデックスを作成する前に、Microsoft TeamsAWS とアカウントにこれらの変更を加えてください。

Microsoft Teamsでは、次のものが揃っていることを確認してください。

  • Office 365 でマイクロソフトチームアカウントを作成しました。

  • マイクロソフト 365 テナント ID をコピーしました。テナント ID は Azure Active Directory ポータルのプロパティで確認できます。

  • クライアント Read のトークンを設定しました。詳細については、Teams のアクセストークンの管理に関する Azure のドキュメントを参照してください。

接続手順

Microsoft TeamsAmazon Kendra データソースに接続するには、Amazon Kendraデータにアクセスできるように Microsoft Teams データソースの必要な詳細を入力する必要があります。Microsoft Teams をまだ設定していない場合は、Amazon Kendraを参照してください前提条件。

Console

Amazon Kendraマイクロソフトチームに接続するには

  1. AWS マネジメントコンソールにサインインし、Amazon Kendra コンソールを開きます。

  2. 左側のナビゲーションペインで、[インデックス] を選択し、インデックス作成されません。

    注記

    ユーザーアクセス制御設定は、インデックス設定で設定または編集できます。

  3. 「はじめに」ページで、「データソースを追加」を選択します。

  4. [データソースの追加] ページで [Microsoft Teams コネクタ] を選択し、[データソースの追加] を選択します。

  5. [データソースの詳細を指定] ページで、次の情報を入力します。

    1. [名前と説明] の [データソース名] に、データソースの名前を入力します。ハイフンは使用できますが、スペースは含められません。

    2. (オプション) [説明]-オプションで、データソースの説明を入力します。

    3. 「言語」(「デフォルト言語」)-索引用にドキュメントをフィルタリングする言語。特に指定しない限り、言語はデフォルトで英語に設定されます。メタデータで指定された言語は、選択した言語よりも優先されます。

    4. [タグ] で、[タグの追加]-[タグ] で、リソースを検索してフィルタリングしたり、AWSコストを追跡したりできます。

    5. [Next] (次へ) を選択します。

  6. 「アクセスとセキュリティの定義」ページで、次の情報を入力します。

    1. ソース — マイクロソフト 365 テナント ID を入力します。テナント ID は Azure Active Directory ポータルのプロパティで確認できます。

    2. AWSSecrets Managersecret — 既存のシークレットを選択するか、Secrets Manager新しいシークレットを作成して Microsoft Teams 認証情報を保存します。新しいシークレットを作成することを選択すると、AWSSecrets Managerシークレットウィンドウが開きます。

      1. 「AWSSecrets Managerシークレットの作成」ウィンドウに次の情報を入力します。

        1. シークレット名 — シークレットの名前。シークレット名には「AmazonKendra-Microsoft Teams-」というプレフィックスが自動的に追加されます。

        2. [クライアント ID] と [クライアントシークレット] の場合 — Microsoft Teams アカウントから生成した認証資格値を入力します。

      2. [Save] (保存) を選択します。

    3. 支払いモデル — Microsoft Teams アカウントのライセンスモデルと支払いモデルを選択できます。モデルAの支払いモデルは、セキュリティコンプライアンスを必要とするライセンスモデルと支払いモデルに限定されています。モデルBの支払いモデルは、セキュリティコンプライアンスを必要としないライセンスモデルや支払いモデルに適しています。

    4. [Virtual Private Cloud (VPC) — VPC を使用することを選択できます。その場合は、サブネットと VPC セキュリティグループを追加する必要があります。

    5. ID クローラー — ID クローラーがアクティブになると、IDAmazon Kendra 情報を同期します。ID クローラーをオフにする場合は、PutPrincipalMappingAPI を使用してプリンシパル情報をアップロードする必要があります。

    6. IAM ロール — 既存の IAM ロールを選択するか、新しい IAM ロールを作成して、リポジトリのアクセス許可にアクセスしてインデックス作成されません。

      注記

      インデックスに使用される IAM ロールはデータソースには使用できません。既存のロールがインデックスや FAQ に使用されているかどうかわからない場合は、エラーを避けるため [新しいロールを作成する] を選択してください。

    7. [Next] (次へ) を選択します。

  7. [Read の構成] ページで、次の情報を入力します。

    1. コンテンツを同期-同期するコンテンツを選択します。

    2. 追加設定 — オプションで、すべてのドキュメントを同期する代わりに、これらの設定を使用して特定のコンテンツをインデックス化できます。

    3. 同期モード — データソースのコンテンツが変更されたときにインデックスを更新する方法を選択できます。

      1. Amazon Kendra完全同期を選択すると、以前の同期ステータスに関係なく、すべてのエンティティのすべてのコンテンツが同期されます。

      2. 新規または変更されたコンテンツの同期を選択すると、Amazon Kendra新しいコンテンツまたは変更されたコンテンツのみが同期されます。

      3. 新規、変更、Amazon Kendraまたは削除されたコンテンツの同期を選択すると、新規、変更、または削除されたコンテンツのみが同期されます。

  8. [フィールドマッピングの設定] ページで、次の情報を入力します。

    1. デフォルトデータソースフィールド —Amazon Kendra 生成されたデフォルトデータソースフィールドから、インデックスにマッピングするフィールドを選択します。

    2. フィールドを追加 — カスタムデータソースフィールドを追加して、マッピング先のインデックスフィールド名とフィールドデータタイプを作成します。

    3. [Next] (次へ) を選択します。

  9. [確認と作成] ページで、入力した情報が正しいことを確認し、[データソースの追加] を選択します。また、このページで情報を編集することもできます。データソースが正常に追加されると、データソースページに表示されます。

API

Amazon Kendraマイクロソフトチームに接続するには

TemplateConfigurationAPI を使用してデータソーススキーマの JSON を指定する必要があります。これには、以下の情報を入力する必要があります。

  • データソース — データソースをとして指定する必要がありますMSTEAMS。

  • テナント ID —テナント ID は Azure Active Directory ポータルのプロパティで確認できます。

  • タイプ —TEMPLATE 電話をかけるときにタイプとして指定しますCreateDataSource。

  • シークレットの Amazon リソースネーム (ARN) — Microsoft TeamsSecrets Manager アカウントの認証情報を含むシークレットの Amazon リソースネーム (ARN) を指定します。シークレットは、次のキーを持つ JSON 構造で格納作成されません。

    {
    "clientId": "client ID",
    "clientSecret": "clientSecret",
}
    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

  • IAM ロール —RoleArnCreateDataSource コールするタイミングを指定して、IAMSecrets Managerシークレットにアクセスする権限をロールに付与し、Microsoft Teams コネクタに必要なパブリック APIAmazon Kendra を呼び出すことができます。詳細については、「Microsoft Readのロール」を参照してください。

また、次のオプション機能を追加することもできます。

  • [Virtual Read (VPC)]) —VpcConfigurationCreateDataSource 電話での問い合わせ VPCAmazon Kendra を使用するように設定する を参照してください。

  • ID クローラーを有効にする — ID クローラーを有効にするかどうかを指定します。ID クローラーが無効になっている場合は、PutPrincipalMappingAPI を使用してプリンシパル情報をアップロードする必要があります。特定のドキュメントへのアクセス権を持つユーザーやグループの ID 情報をクロールすると、ユーザーコンテキストのフィルタリングに役立ちます。検索結果は、ユーザーまたはグループのドキュメントへのアクセス権に基づいてフィルタリングされます。詳細については、ユーザーコンテキストでのフィルタリングを参照してください。

  • 包含フィルターと除外フィルター — Microsoft Teams チャネルで見つかった特定のドキュメントを含めるか除外するかを指定します。

    注記

    ほとんどのデータソースは正規表現パターンを使用します。正規表現パターンとは、フィルターと呼ばれる包含パターンまたは除外パターンです。包含フィルターを指定すると、包含フィルターと一致するコンテンツのみが索引付けされます。包含フィルターに一致しないドキュメントはインデックスに登録されません。包含パターンと除外フィルターを指定した場合、除外に一致するドキュメントは、包含パターンと一致してもインデックス作成されません。

  • フィールドマッピング — Microsoft TeamsAmazon Kendra のデータソースフィールドをインデックスフィールドにマッピングすることを選択します。詳細については、データソースフィールドのマッピングを参照してください。
