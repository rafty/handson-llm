カスタムドキュメントフィールドの作成

特定のドキュメントに関連するカスタム属性またはメタデータフィールドを適用できます。例えば、「HR」、「Sales」、「manufacturing」という値を持つ「Department」というカスタムフィールドまたは「HR」、「Sales」、「manufacturing」という値を持つ「Department」というカスタムフィールドまたは属性を作成できます。例えば、これらのフィールドまたは属性を使用して、レスポンス (検索結果) を「HR」部門のドキュメントに制限できます。

最大 500 のカスタムフィールドまたは属性を作成できます。

ほとんどのデータソースでは、外部データソースのフィールドを、Amazon Kendraの対応するフィールドにマッピングします。詳細については、「データソースフィールドのマッピング」を参照してください。S3 データソースの場合、メタデータファイルを使用してカスタムフィールドまたは属性を適用できます。

カスタムフィールドまたは属性を使用するには、まずインデックスにフィールドを作成する必要があります。コンソールまたは UpdateIndexAPI を使用してインデックスフィールドを作成します。フィールドを作成したら、フィールドのデータタイプを変更することはできません。

Amazon Kendraの組み込みフィールドまたは予約フィールドを使用することもできます。

UpdateIndexAPI では、DocumentMetadataConfigurationUpdatesパラメータを使用してカスタムフィールドまたは属性を追加します。

次の JSON の例では、「Department」DocumentMetadataConfigurationUpdates というフィールドをインデックスに追加します。

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Amazon Kendra の組み込みドキュメントフィールドを使用する

UpdateIndex API を使用すると、DocumentMetadataConfigurationUpdates予約フィールド名を使用して予約フィールドまたは組み込みフィールドを作成し、対応するドキュメント属性またはドキュメントフィールドにマップできます。この方法でカスタムフィールドを作成することもできます。データソースコネクタを使用する場合、ほとんどの場合、Amazon Kendraデータソースドキュメントフィールドをインデックスフィールドにマップするフィールドマッピングが含まれます。フィールドを作成すると、そのフィールドを表示可能、ファセット可能、検索可能、Searchおよびソート可能に設定するようにオブジェクトを設定できます。Relevanceオブジェクトを設定して、フィールドの関連性向上期間、鮮度、重要度、ランク順および重要度値を設定できます。フィールドを作成した後は、フィールドタイプを変更できません。

Amazon Kendraには、次の予約済みまたは組み込みのドキュメントフィールドを使用できます。

  • _authors— ドキュメントの内容を担当する 1 人以上の作成者のリスト。

  • _category—ドキュメントを特定のグループに配置するカテゴリ。

  • _created_at—ドキュメントが作成された ISO 8601 形式の日付と時刻。例えば、2012-03-25T12:30:10+01:00 は、中央ヨーロッパ時間の 2012 年 3 月 25 日午後 12 時 30 分 (プラス 10 秒) の ISO 8601 の日付/時刻形式です。

  • _data_source_id— ドキュメントを含むデータソースの識別子。

  • _document_body—ドキュメントの内容。

  • _document_id—ドキュメントの一意の識別子。

  • _document_title— ドキュメントのタイトル。

  • _excerpt_page_number—ドキュメントの抜粋が表示される PDF ファイルのページ番号。2020 年 9 月 8 日より前にインデックスが作成された場合、この属性を使用する前に、ドキュメントのインデックスを再作成する必要があります。

  • _faq_id— これがよくある質問の質問と回答の場合は、それらの一意の識別子。

  • _file_type—pdf や doc など、ドキュメントのファイルタイプ。

  • _last_updated_at— ドキュメントが最後に更新された ISO 8601 形式の日時。例えば、2012-03-25T12:30:10+01:00 は、中央ヨーロッパ時間の 2012 年 3 月 25 日午後 12 時 30 分 (プラス 10 秒) の ISO 8601 の日付/時刻形式です。

  • _source_uri—ドキュメントが利用可能な URI。例えば、会社のウェブサイト上のドキュメントの URI などです。

  • _version—ドキュメントの特定のバージョンの識別子。

  • _view_count— ドキュメントが表示された回数。

  • _language_code(文字列) — ドキュメントに適用される言語のコード。言語を指定しないと、デフォルトで英語になります。コードを含む、サポートされている言語の詳細については、英語以外の言語でドキュメントを追加するを参照してください。

カスタムフィールドの場合、予約フィールドや組み込みフィールドを作成するときと同じように、UpdateIndex APIDocumentMetadataConfigurationUpdates を使用してこれらのフィールドを作成します。カスタムフィールドに適切なデータ型を設定する必要があります。フィールドを作成した後は、フィールドタイプを変更できません。

カスタムフィールドに設定できるタイプは次のとおりです。

  • 日付

  • 数値

  • 文字列

  • 文字列リスト

カスタムフィールドは、コンソールまたは UpdateIndexAPI を使用して作成します。カスタムフィールドを作成したら、予約フィールドの場合と同様に、ドキュメント属性にマッピングします。API を使用してドキュメントをインデックスに追加した場合、BatchPutDocumentAPI を使用して属性をマッピングします。Amazon S3データソースからインデックス作成されたドキュメントの場合、ドキュメント属性を記述する JSON 構造を含むメタデータファイルを使用して属性をマッピングします。フィールドマッピングが可能なデータベースまたはデータソースで索引付けされたドキュメントの場合、コンソールまたはデータソース設定を使用して属性をマッピングします。

BatchPutDocument API によるカスタム属性またはフィールドの追加

BatchPutDocumentAPI を使用してドキュメントをインデックスに追加する場合、カスタムフィールドまたはカスタムフィールドまたは属性をインデックスに追加する場合、カスタムフィールドまたは属性をインデックスに追加する場合、カスタムフィールドまたは属性を指定しますAttributes。API を呼び出すと、複数のフィールドまたは属性を追加できます。最大 500 のカスタムフィールドまたは属性を作成できます。次の例は、ドキュメントに「Department」を追加するカスタムフィールドまたは属性です。

Amazon S3データソースへのカスタム属性またはフィールドの追加

S3 バケットをインデックスのデータソースとして使用する場合、コンパニオンメタデータファイルを使用してドキュメントにメタデータを追加します。メタデータ JSON ファイルは、ドキュメントと平行なディレクトリ構造に配置します。詳細については、「Amazon S3ドキュメントメタデータ」を参照してください。

AttributesJSON 構造でカスタムフィールドまたは属性を指定します。最大 500 のカスタムフィールドまたは属性を作成できます。例えば、次の例ではAttributes 3 つのカスタムフィールドまたは属性と 1 つの予約済みフィーrドフィールドを定義しています。

"Attributes": {
        "brand": "Amazon Basics",
        "price": 1595,
        "_category": "sports",

    }
