BatchPutDocument

インデックスに 1 つまたは複数のドキュメントを追加します。

BatchPutDocumentこの演算により、インラインドキュメントまたは Amazon S3 バケットに保存されているドキュメントのセットを取り込めます。このAPIを使用して、テキストと非構造化テキストをインデックスに取り込み、ドキュメントにカスタム属性を追加し、インデックスに追加されたドキュメントにアクセスコントロールリストを添付します。

ドキュメントは非同期的にインデックス作成されます。バッチの進行状況は、を使用することにより確認できますAWS CloudWatch。バッチの処理に関連するエラーメッセージは、AWS CloudWatch ログに送信されます。

Python および Java SDK を使用してインラインドキュメントを取り込む例については、「インデックスへのファイルの直接追加」を参照してください。

リクエストパラメータ

すべてのアクションに共通のパラメータの詳細については、「共通パラメータ」を参照してください。

リクエストは以下の JSON 形式のデータを受け入れます。

CustomDocumentEnrichmentConfiguration

BatchPutDocumentAPI を使用する際のドキュメントの取り込みプロセス中に、ドキュメントの取り込みプロセス中に、ドキュメントのメタデータとコンテンツを変更するための構成情報。

Amazon Kendra にドキュメントを取り込む際のドキュメントメタデータの作成、変更、削除方法、またはその他のコンテンツの変更を行う方法の詳細については、取り込みプロセス中のドキュメントメタデータのカスタマイズを参照してください。

型: CustomDocumentEnrichmentConfiguration オブジェクト

必須: いいえ

Documents

インデックスに追加する 1 つまたは複数のドキュメント。

ドキュメントには、次のファイルサイズの制限があります。

  • すべてのファイルの合計サイズ 50 MB

  • 任意のファイルの抽出テキスト 5 MB

詳細については、クォータを参照してください。

型: Document オブジェクトの配列

配列メンバー: 最小数は 1 項目です。最大数は 10 項目です。

必須: はい

IndexId

ドキュメントを追加するインデックスの識別子。まず、CreateIndex API を使用してインデックスフィールドを作成する必要があります。

型: 文字列

長さの制限: 最大長は 36 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9-]*

必須: はい

RoleArn

S3 バケットへアクセスするために必要なアクセス権限があります。IAM ロールの Amazon リソースネーム (ARN)。詳細については、Amazon Kendra の IAM アクセスロールを参照してください。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,284 です。

パターン: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

必須: いいえ

レスポンス要素

アクションが成功すると、サービスは HTTP 200 レスポンスを返します。

サービスから以下のデータが JSON 形式で返されます。

FailedDocuments

ドキュメントが検証チェックに失敗したため、インデックスに追加されなかったドキュメントのリスト。各ドキュメントには、ドキュメントをインデックスに追加できなかった理由を示すエラーメッセージが含まれています。

ドキュメントをインデックスに追加する際にエラーが発生した場合、AWS CloudWatch エラーはログで報告されます。詳細については、Amazon CloudWatch ログによる Amazon Kendra のモニタリングを参照してください。

型: BatchPutDocumentResponseFailedDocument オブジェクトの配列

エラー

すべてのアクションに共通のエラーについては、「一般的なエラー」を参照してください。

AccessDeniedException

このアクションを実行する十分なアクセス権限がありません。必要な権限ポリシーとユーザーアカウントがあることを確認して、もう一度試してください。

HTTP ステータスコード: 400

ConflictException

リクエストに矛盾が発生しました。リソースとの不一致を修正して、もう一度試してください。

HTTP ステータスコード: 400

InternalServerException

Amazon Kendra サービスに使用されている内部サーバーで問題が発生しました。数分後にもう一度お試しださい。または、Support に連絡してください。

HTTP ステータスコード: 500

ResourceNotFoundException

使用したいリソースは存在しません。正しいリソースが提供されていることを確認して、もう一度お試しください。

HTTP ステータスコード: 400

ServiceQuotaExceededException

Amazon Kendra サービスに設定された制限を超えました。詳細についてはクォータを参照するか、制限の引き上げについてはSupport にお問い合わせください。

HTTP ステータスコード: 400

ThrottlingException

リクエストは、制限が必要なために実行が拒否されました。リクエストの数を減らして、もう一度お試しください。

HTTP ステータスコード: 400

ValidationException

入力が Amazon Kendra サービスで設定された制約を満たしていません。正しい入力を入力して、もう一度お試しください。

HTTP ステータスコード: 400
