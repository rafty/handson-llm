Amazon Kendraウェブサイトに検索機能を導入するには、React で使用できるソースコードを指定して、アプリケーションを構築するには、React で使用できるソースコードを指定して、アプリケーションを構築できます。ソースコードは、改定版 MIT ライセンスの下で無償で提供されます。それをそのまま使うことも、自分のニーズに合わせて変更することもできます。提供されている React アプリケーションを構築するには、作業の開始に役立つサンプルです。本番環境に対応したアプリではありません。

コードなしで検索アプリケーションをデプロイし、アクセス制御を使用して検索ページへのエンドポイント URL を生成するには、「Amazon KendraExperience Builder」を参照してください。

React で使用できる例は2つあります。

これらの例は、Amazon Kendraコンソールの検索ページをモデル化しています。これらには、検索結果を検索および表示するための同じ機能があります。例全体を使用することも、独自の使用のために機能を 1 つだけ選択することもできます。

概要

既存の React アプリケーションにサンプルコードを追加して、検索を有効にします。検索ファイルとコンポーネントは、次のように構成されています。

  • メイン検索ページ — これは、すべてのコンポーネントを含むメインページです。ここでは、アプリケーションをAmazon Kendra API と統合します。

  • 検索バー — ユーザーが検索語を入力し、検索機能を呼び出すコンポーネントです。

  • 結果 —、Amazon Kendraからの結果を表示するコンポーネントです。提案された回答、よくある質問の結果、推奨ドキュメントの 3 つのコンポーネントがあります。

  • ファセット — 検索結果にファセットを表示するコンポーネントで、ファセットを選択して検索を制限できます。

  • ページ割り —、Amazon Kendraからのレスポンスをページ分割するコンポーネントです。

前提条件

開始するには、以下が必要です。

  • 既存の React ウェブアプリケーションまたはサンプルアプリケーションです。

  • 正しいライブラリで構成された開発環境設定。

  • Node.js と npm がインストールされています。

  • SDK for JavaまたはAWS SDK for JavaScript。

必要なライブラリとAWS SDK に関する情報は、zip ファイルの Readme ファイルにあります。

メイン検索ページ

メイン検索ページには、すべてのサンプル検索コンポーネントが含まれています。これには、Query API からのレスポンスを表示する結果コンポーネント、およびレスポンスをページングするための結果コンポーネント、およびレスポンスをページングするためのページ割りコンポーネントと、Query API からのレスポンスを表示するための結果コンポーネント、およびレスポンスをページ分割するためのページ割りコンポーネントと、Query API からのレスポンスを表示するための結果コンポーネント、および

検索コンポーネント

検索コンポーネントには、クエリテキストを入力するためのテキストボックスがあります。onSearchこの関数は、Search.tsxメイン関数を呼び出して、Amazon Kendra Query API 呼び出しを行います。

結果コンポーネント

結果コンポーネントには、Query API からのレスポンスが表示されます。結果は 3 つの別個のエリアに表示されます。

  • 提案された回答 —Query API により返された上位結果です。提案された回答は最大 3 つまで含まれます。レスポンスには、結果タイプ ANSWER があります。

  • よくある質問の回答 — レスポンスが返すよくある質問の結果です。よくある質問はインデックスに別々に追加されます。レスポンスには、タイプ QUESTION_ANSWER があります。詳細については、質問と回答を参照してください。

  • 推奨ドキュメント —Amazon Kendra レスポンスが返す追加ドキュメントです。QueryAPI からのレスポンスには、次のタイプがありますDOCUMENT。

結果コンポーネントは、強調表示、タイトル、リンクなどの機能のコンポーネントを共有します。結果コンポーネントが機能するには、共有コンポーネントが存在する必要があります。

ファセットコンポーネント

ファセットコンポーネントには、検索結果で使用可能なファセットが一覧表示されます。各ファセットは、製作者などの特定のディメンションに沿ってレスポンスを分類します。リストからファセットを選択して、検索を特定のファセットに絞り込むことができます。

ファセットを選択すると、Queryコンポーネントによってファセットに一致するドキュメントに対する検索を制限する属性フィルターが呼び出されます。

ページ割りコンポーネント

ページ割りコンポーネントを使用すると、複数のページでのQuery API からの検索結果の表示が可能になります。QueryAPIPageSizePageNumber をおよびパラメータで呼び出し、特定のページの結果を取得します。
