注目の検索結果

ユーザーが特定のクエリを実行したときに、特定のドキュメントを検索結果に表示できます。これにより、結果がより見やすくなり、ユーザーにとって目立つようになります。おすすめの検索結果は、通常の検索結果一覧とは切り離され、検索ページの上部に表示されます。さまざまなクエリに対してさまざまなドキュメントを取り上げてみたり、特定のドキュメントにふさわしい可視性を持たせたりできます。

特定のクエリを特定のドキュメントにマッピングして、結果に表示します。クエリに完全に一致するものが含まれている場合、1 つ以上の特定のドキュメントが検索結果に表示されます。

たとえば、ユーザーが「2023年の新製品」というクエリを発行した場合に、「新着情報」と「近日公開」というタイトルのドキュメントを選択して検索結果ページの上部に表示するように指定できます。これにより、新製品に関するこれらの文書にふさわしい可視性が確保されます。

Amazon Kendra検索結果ページの上部に表示する結果が既に選択されている場合、検索結果は重複しません。注目リザルトが既に他の検索結果よりも上位に表示されている場合、そのリザルトが再び最初のリザルトとしてランク付けされることはありません。

特定の結果を表示させるには、クエリに含まれるキーワードまたはフレーズを使用したクエリの部分一致ではなく、全文クエリと完全に一致するものを指定する必要があります。たとえば、主要な結果セットで「Kendra」というクエリのみを指定した場合、「Kendra は結果をセマンティックにランク付けする方法」などのクエリになります。注目の結果はレンダリングされません。おすすめの結果は、範囲が広すぎるクエリではなく、特定のクエリ向けに設計されています。 Amazon Kendraキーワードタイプのクエリを自然に処理して、検索結果で最も有用なドキュメントをランク付けし、単純なキーワードに基づいて結果を過度に強調表示しないようにします。

ユーザーが頻繁に使用する特定のクエリがある場合は、それらのクエリを指定して主要な結果を表示できます。たとえば、Amazon Kendraアナリティクスを使用して上位のクエリを調べたところ、「kendra は結果をどのようにセマンティックにランク付けしているのか」などの特定のクエリが見つかったとします。や「kendra セマンティック検索」がよく使われるので、「search 101」というタイトルのドキュメントを特集するには、これらのクエリを指定すると便利な場合があります。Amazon Kendra

Amazon Kendra注目結果のクエリは大文字と小文字を区別しないものとして扱います。 Amazon Kendraクエリを小文字に変換し、末尾の空白スペースを 1 つのスペースに置き換えます。 Amazon Kendra検索結果の検索条件を指定すると、他のすべての文字がそのまま一致します。

CreateFeaturedResultsSetAPI を使用して特定のクエリにマッピングする注目結果のセットを作成します。コンソールを使用する場合は、インデックスを選択し、ナビゲーションメニューで [おすすめの結果] を選択して、おすすめの結果セットを作成します。インデックスごとに最大50セットの注目結果、1セットあたり最大4つの特集ドキュメント、および注目結果セットごとに最大49のクエリテキストを作成できます。これらの制限を引き上げるのは、Support に連絡して行います。

複数の注目結果セットから同じドキュメントを選択できます。ただし、複数のセットで同じ完全一致クエリテキストを使用しないでください。注目結果に対して指定するクエリは、各インデックスのおすすめ結果セットごとに一意である必要があります。

主要ドキュメントを最大 4 つ選択するときに、ドキュメントの順序を調整できます。API を使用する場合、注目ドキュメントを一覧表示する順序は、おすすめの結果に表示される順序と同じです。コンソールを使用すると、結果に含めるドキュメントを選択するときに、ドキュメントの順序をドラッグアンドドロップするだけで済みます。

特定のユーザーやグループが特定のドキュメントにアクセスでき、他のユーザーやグループはアクセスできないというアクセス制御は、おすすめの結果を設定する際にも適用されます。ユーザーコンテキストフィルタリングの場合も同様です。たとえば、ユーザーAは「インターン」企業グループに属しているため、企業秘密に関するドキュメントにアクセスしてはなりません。ユーザー A が会社の秘密文書を含むクエリを入力しても、ユーザー A の検索結果にはこの文書は表示されません。これは、検索結果ページの他の結果にも当てはまります。タグを使用して、Amazon Kendraアクセスを制御するリソースである主要な結果セットへのアクセスを制御することもできます。

以下は、「新着情報」(doc-id-1)、「近日公開」(doc-id-2) というタイトルのドキュメントに「新製品 2022」、「新製品販売開始」(doc-id-2) というクエリをマッピングして、おすすめの結果セットを作成する例です。

CLI
aws kendra create-featured-results-set \
 --featured-results-set-name 'New product docs to feature' \
 --description "Featuring What's new and Coming soon docs" \
 --index-id index-id \
 --query-texts 'new products 2023' 'new products available' \
 --featured-documents '{"Id":"doc-id-1", "Id":"doc-id-2"}'
Python
import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Create a featured results set.")

# Provide a name for the featured results set
featured_results_name = "New product docs to feature"
# Provide an optional decription for the featured results set
description = "Featuring What's new and Coming soon docs"
# Provide the index ID for the featured results set
index = index-id
# Provide a list of query texts for the featured results set
queries = ['new products 2023', 'new products available']
# Provide a list of document IDs for the featured results set
featured_doc_ids = ['doc-id-1', 'doc-id-2']

try:
    featured_results_set_response = kendra.create_featured_results_set(
        FeaturedResultsSetName = featured_results_name,
        Decription = description,
        Index = index,
        QueryTexts = queries,
        FeaturedDocuments = featured_doc_ids
    )

    pprint.pprint(featured_results_set_response)

    featured_results_set_id = featured_results_set_response["FeaturedResultsSetId"]

    while True:
        # Get the details of the featured results set, such as the status
        featured_results_set_description = kendra.describe_featured_results_set(
            Id = featured_results_set_id
        )
        status = featured_results_set_description["Status"]
        print(" Featured results set status: "+status)
            
except  ClientError as e:
        print("%s" % e)

print("Program ends.")
