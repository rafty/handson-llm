ドキュメントのタイプ

インデックスには、複数のドキュメントと複数のタイプのドキュメントを含めることができます。インデックスには、構造化テキストと非構造化テキストの両方を含めることもできます。

構造化テキスト

  • よくある質問と回答

よくある質問と回答形式のドキュメントは、スペースニードルの身長はどれくらいですか? のような質問への回答に使用されます。同じ回答を返す複数の質問を指定できます。質問と回答は、Amazon S3バケットに保存されたカンマ区切り値 (CSV) ファイルで指定します。

よくある質問と回答に JSON ファイルを使用する場合、JSON ファイルはプレーンテキストとして扱われ、非構造化テキストの一種です。

例については、質問と回答をインデックスに直接追加するを参照してください。

非構造化テキスト

Amazon Kendraは、次の種類の非構造化テキストをサポートします。

  • HTML ファイル

  • マイクロソフト PowerPoint (PPT) プレゼンテーション

  • MS Word ドキュメント

  • プレーンテキストドキュメント

  • PDF ファイル

  • カンマ区切り値 (CSV) ファイル

  • Microsoft Excel (MS Excel) ファイル

  • XML ファイル

  • JSON ファイル

  • マークダウンドキュメンテーション (MD) ファイル

  • リッチテキスト形式 (RTF) ファイル

  • 拡張スタイルシート言語変換 (XSLT) ファイル

非構造化文書をインデックスに追加するには、次の 3 つの方法があります。

  • BatchPutDocumentAPI を使用する。

  • データソースとして。詳細については、「データソースからドキュメントを追加する」を参照してください。

  • バイナリデータとして。詳細については、「API によるドキュメントの追加」を参照してください。
