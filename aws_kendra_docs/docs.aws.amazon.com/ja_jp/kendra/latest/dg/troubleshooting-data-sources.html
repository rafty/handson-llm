データソースのトラブルシューティング

このセクションは、Amazon Kendra データソースの問題を修正するのに役立ちます。

マイドキュメントにインデックスが作成されませんでした

Kendra インデックスをデータソースと同期すると、ドキュメントのインデックス作成を妨げる問題が発生する可能性があります。インデックス作成は 2 ステッププロセスです。まず、インデックス作成する新規および更新されたドキュメントがないかデータソースをチェックし、インデックスから削除するドキュメントを検索します。次に、ドキュメントレベルでは、各ドキュメントにアクセスしてインデックス作成します。

次のいずれかのステップでエラーが発生する可能性があります。データソースレベルのエラーは、データソースの詳細ページの [Sync run history] (実行履歴を同期) セクションのコンソールに報告されます。同期ジョブのステータスは、[Succeeded] (成功)、[Incomplete] (不完全)、または [Failed] (失敗) になります。ジョブ中にインデックス作成され、削除されたドキュメントの数も確認できます。ステータスが [Failed] (失敗) の場合、メッセージが [Details] (詳細) 列で表示されます。

ドキュメントレベルのエラーは Amazon CloudWatch Logs で報告されます。 CloudWatch コンソールを使用してエラーを確認できます。

同期ジョブが失敗しました

同期ジョブは通常、インデックスまたはデータソースの構成エラーがある場合に失敗します。データソースの [Details] (詳細) 列のエラーメッセージには、何が問題になったかについての情報が表示されます。問題は、通常、インデックスまたはデータソースに適切な IAM アクセス許可がないことです。エラーメッセージには、欠落しているアクセス許可が示されます。受け取る可能性のあるエラーメッセージをいくつか以下に示します。

Failed to create log group for job. Please make sure that the IAM role provided has sufficient permissions.

インデックスロールに使用許可がない場合 CloudWatch、 CloudWatch データソースはログを作成できません。このエラーが発生した場合は、 CloudWatch インデックスロールにアクセス許可を追加する必要があります。

Failed to access S3 file prefix (bucket name) while trying to crawl your metadata files. Please make sure the IAM Role (role ARN) provided has sufficient permissions.

Amazon S3 データソースを使用する場合、Kendra はドキュメントを含むバケットにアクセスするための許可を持っている必要があります。データソースの IAM ロールにバケットを読み取るには、Kendra のアクセス許可を追加する必要があります。

The provided IAM Role (role ARN) could not be assumed. Please make sure Amazon Kendra is a trusted entity that is allowed to assume the role.

Kendra には、インデックスとデータソースの IAM ロールを引き受ける権限が必要です。sts:AssumeRole アクションの許可を持つロールに信頼ポリシーを追加する必要があります。

Kendra がデータソースのインデックス作成に必要な IAM ポリシーについては、を参照してくださいIAMのアクセスロールAmazon Kendra。

同期ジョブが不完全です

通常、ジョブは、データソースレベルのプロセスを完了したが、ドキュメントレベルのプロセス中に何らかのエラーが発生した場合、不完全になります。ジョブが不完全な場合でも、一部のドキュメントはまだ正常にインデックス化されている可能性があります。Amazon S3 データソースの場合、不完全なジョブは通常、次の原因によって引き起こされます。

  • 1 つまたは複数のドキュメントのメタデータが無効だった。

  • インデックス作成のためにドキュメントが送信されたが、少なくとも 1 つのドキュメントが送信されなかった場合。

  • インデックスから削除するためにドキュメントが送信されたが、少なくとも 1 つのドキュメントが送信されなかった場合。

不完全な同期ジョブをトラブルシューティングするには、 CloudWatch まずログを調べます。

  1. [詳細] 列から、[View details in] (詳細を表示する) を選択します CloudWatch。

  2. エラーメッセージを確認して、ドキュメントが失敗した原因を確認します。

同期ジョブは成功しましたが、インデックス付きドキュメントがありません

場合によっては、インデックス同期ジョブ実行が [Succeeded] (成功) とマークされることがありますが、予想されるときにインデックス化された新規または更新されたドキュメントはありません。考えられる原因は以下のとおりです。

  • CloudWatch DocumentsSubmittedForIndexingFailedメトリックをチェックして、同期に失敗したドキュメントがないかどうかを確認します。詳細については、 CloudWatch ログを確認してください。

  • Amazon S3 データソースの場合、Kendra に間違ったバケット名またはプレフィックスを指定している可能性があります。Kendra が使用しているバケットが、インデックス作成するドキュメントを含むバケットであることを確認します。

  • 以前のジョブでインデックス作成に失敗したドキュメントのインデックスを再作成する場合、そのドキュメントまたは関連するメタデータファイルを変更しない限り、Kendra はそのドキュメントのインデックスを作成しません。

データソースの同期中にファイル形式の問題が発生しました

データソースへのファイルの追加またはデータソースの同期中にファイル形式の問題が発生した場合は、ドキュメントタイプが Kendra をサポートしていることを確認してください。がサポートするドキュメントタイプの一覧については、「ドキュメントタイプAmazon Kendra」を参照してください。

プレーンテキストファイルでBatchPutDocument API を使用している場合は、PLAIN_TEXTコンテンツタイプとして指定してください。

データソースの同期にはどれくらいの時間がかかりますか?

ドキュメントが更新されていない場合、Amazon Kendra インデックスの同期時間はドキュメントの数に比例して増加します。たとえば、更新のないドキュメントが 1,000 件の場合、同期に約 5 分かかり、更新なしで 2,000 件のドキュメントが更新されない場合、約 10 分かかります。ドキュメントが更新された場合、更新されたドキュメントの数に応じて同期時間が長くなります。

データソースの同期にはどれくらいの料金がかかりますか?

インデックスを同期すると、Amazon EC2 をウォームアップしてアクティブ化し、必要な接続を確立するのに 2 分かかります。このプロセス中は請求は発生しません。使用量メーターは、同期ジョブの開始後にのみ開始されます。Kendra の料金の詳細については、「Amazon Kendra の料金」を参照してください。

Amazon EC2 認証エラーが発生します

仮想プライベートクラウド (VPC) データソースの同期中に Amazon EC2 の不正操作エラーが発生した場合、VPC IAM ロールに必要な権限がない可能性があります。データソースに使用する IAM ロールに権限が付与されていることを確認してください。詳細については、「Virtual private cloud (VPC) IAM ロール」を参照してください。

検索インデックスリンクを使用して Amazon S3 オブジェクトを開くことができません

Kendra インデックスは、Amazon S3 データソースがアクセス権限を付与しているファイルにのみアクセスできます。たとえば、Kendra は、オブジェクトを公開するか暗号化するかを決定する Amazon S3 のアクセス権限を変更することはできません。また、Kendra には Amazon S3 オブジェクトの署名付きリンクを作成または返すためのデフォルトの権限もありません。Kendra インデックス内の Amazon S3 オブジェクトの署名付きリンクを有効にする場合は、次の 2 つのオプションがあります。

  • 検索ページに結果を返す前に、ソース URI オブジェクトでインデックスクエリ結果に署名できます。 step-by-stepこのプロセスの手順については、「署名済み URL を使用したオブジェクトの共有」を参照してください。

  • Amazon S3 オブジェクトメタデータのソース URI をオーバーライドして、Amazon S3 CloudFront バケットに接続されたコンテンツ配信ネットワーク (CDN) を介してサービスを利用できるようにすることができます。または、署名済み URL を返し、リダイレクトする API Gateway プロキシエンドポイントを使用できます。

「SSLAccessDenied 証明書ファイル使用時」というエラーメッセージが表示される

データソースで SSL 証明書を使用しているときにアクセス拒否エラーが発生する場合は、IAM ロールに指定された場所の SSL 証明書ファイルにアクセスする権限があることを確認してください。AWS KMS証明書がキーで暗号化されている場合、IAMAWS KMS ロールにもそのキーを使用して復号化する権限が必要です。詳細については、「の認証とアクセス制御」を参照してくださいAWS KMS。

SharePoint データソースを使用すると認証エラーが発生します

SharePoint インデックスをデータソースと同期しているときに認証エラーが発生した場合は、でサイト管理者ロールが割り当てられていることを確認してください SharePoint。

インデックスが Confluence データソースのドキュメントをクロールしない

同期処理中に Kendra インデックスが Confluence データソースからドキュメントをクロールしない場合は、自分が Confluence の管理者グループに属していることを確認してください。
