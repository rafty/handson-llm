GitHub

GitHub は、バージョン管理機能を備えたコードストレージおよび管理サービスを提供する、ソフトウェア開発用のウェブベースのホスティングサービスです。Amazon Kendraを使用して、 GitHub Enterprise Cloud (SaaS) と GitHub Enterprise Server (On Prem) のリポジトリファイル、イシューとプルリクエスト、イシューとプルリクエストのコメント、イシューとプルリクエストのコメント添付ファイルをインデックス化できます。また、特定のファイルを包含するか除外することもできます。

Amazon Kendraコンソールと GitHubConfigurationAPIAmazon Kendra GitHub を使用してデータソースに接続できます。

Amazon Kendra GitHub データソースコネクタのトラブルシューティングについては、を参照してくださいデータソースのトラブルシューティング。

サポートされている機能

Amazon Kendra GitHub データソースコネクタは次の機能をサポートしています。

  • 変更ログ

  • フィールドマッピング

  • ユーザーコンテキストのフィルタリング

  • 包含/除外フィルター

前提条件

Amazon Kendra GitHub を使用してデータソースのインデックスを作成する前に、 GitHub AWS自分のアカウントでこれらの変更を行います。

で GitHub、以下が揃っていることを確認してください。

  • GitHub GitHub 組織への管理者権限を持つユーザーを作成しました。

  • 認証情報用の個人用アクセストークンを作成しました。GitHub 個人アクセストークンの作成に関するドキュメントを参照してください。

  • 推奨:認証情報用の OAuth トークンを作成しました。OAuth トークンを使用すると、API のスロットル制限とコネクタのパフォーマンスが向上します。OAuthGitHub 認証に関するドキュメントを参照してください。

  • オプション:SSL 証明書をインストールしました。

  • GitHub GitHub使用しているサービスタイプのホスト URL を書き留めました。たとえば、 GitHub クラウドのホスト URL は https://api.github.com で、 GitHub サーバーのホスト URL は https://on-prem-host-url/api/v3/ である可能性があります。

  • GitHub GitHub 設定からリポジトリの組織名を書き留めました。

  • 次の権限を追加しました。

    GitHub エンタープライズクラウド (SaaS) 向け

    • リポジトリ:ステータス

    • public

    • リポジトリ:招待

    • 読み込み:org

    • ユーザ:電子メール

    • 読み取り:ユーザ

    GitHub エンタープライズサーバー用 (オンプレミス)

    • リポジトリ:ステータス

    • public

    • リポジトリ:招待

    • 読み込み:org

    • ユーザ:電子メール

    • 読み取り:ユーザ

    • サイト管理者

  • 各ドキュメントが、同じインデックスに使用する予定の他のデータソース間で一意であることを確認しました。 GitHub インデックスに使用する各データソースには、データソース間で同じドキュメントが含まれていてはなりません。ドキュメント ID はインデックス全体に適用され、インデックスごとに一意でなければなりません。

AWSアカウントには、次のものがあることを確認してください。

  • Amazon Kendraインデックスを作成し、API を使用している場合は、インデックス ID を記録しました。

  • IAMデータソースのロールを作成し、API を使用している場合は、IAMロールの ARN を記録しました。

  • GitHub AWSSecrets Manager認証情報をシークレットに保存し、API を使用している場合は、シークレットの ARN を記録しました。

    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

IAM既存のロールまたはシークレットがない場合は、 GitHub データソースをに接続するときに、IAMSecrets Managerコンソールを使用して新しいロールとシークレットを作成できますAmazon Kendra。API を使用している場合は、IAMSecrets Manager既存のロールとシークレットの ARN、およびインデックス ID を指定する必要があります。

接続手順

Amazon Kendraデータソースに接続するには、 GitHub Amazon Kendraデータにアクセスできるように、 GitHub データソースの必要な詳細を入力する必要があります。まだ設定していない場合は GitHub 、Amazon Kendraを参照してください前提条件。

Console

Amazon Kendraに接続するには GitHub

  1. AWS マネジメントコンソールにサインインし、Amazon Kendra コンソールを開きます。

  2. 左側のナビゲーションペインで、[インデックス] を選択し、インデックスのリストから使用するインデックスを選択します。

    注記

    ユーザーアクセス制御設定は、インデックス設定で設定または編集できます。

  3. 「はじめに」ページで、「データソースを追加」を選択します。

  4. [データソースの追加] ページで、[GitHub コネクタ] を選択し、[データソースの追加] を選択します。

  5. [データソースの詳細を指定] ページで、次の情報を入力します。

    1. [名前と説明] の [データソース名] に、データソースの名前を入力します。ハイフンは使用できますが、スペースは含められません。

    2. (オプション) [説明] — データソースの説明を入力します。

    3. 「言語」(「デフォルト言語」)-索引用にドキュメントをフィルタリングする言語。特に指定しない限り、言語はデフォルトで英語に設定されます。メタデータで指定された言語は、選択した言語よりも優先されます。

    4. [タグ] の [Add new tag] で、リソースを検索してフィルタリングしたり、AWSコストを追跡したりできます。

    5. [Next] (次へ) を選択します。

  6. 「アクセスとセキュリティの定義」ページで、次の情報を入力します。

    1. GitHubsourceGitHub GitHub —エンタープライズクラウドとエンタープライズサーバーのどちらかを選択します。

    2. GitHub ホスト URL — GitHub ホスト名を入力します。

    3. GitHub 組織名 — GitHub 組織名を入力します。 GitHub 組織情報はアカウントで確認できます。

    4. AWSSecrets Managersecret — 既存のシークレットを選択するか、Secrets Manager GitHub 認証情報を保存する新しいシークレットを作成します。新しいシークレットを作成することを選択すると、AWSSecrets Managerシークレットウィンドウが開きます。

      1. 「AWSSecrets Managerシークレットの作成」ウィンドウに次の情報を入力します。

        1. シークレット名 — シークレットの名前。シークレット名には、プレフィックスGitHub「AmazonKendra--」が自動的に追加されます。

        2. GitHubトークンの場合- GitHub アカウントで作成した認証資格値を入力します。

      2. [Save] (保存) を選択します。

    5. [VVirtual Private Cloud (VPC) — VPC の使用を選択できます。その場合は、サブネットと VPC セキュリティグループを追加する必要があります。

    6. IAM ロール — 既存の IAM ロールを選択するか、新しい IAM ロールを作成して、リポジトリの認証情報にアクセスしてコンテンツをインデックス作成する。

      注記

      インデックスに使用される IAM ロールはデータソースには使用できません。既存のロールがインデックスや FAQ に使用されているかどうかわからない場合は、エラーを避けるため [新しいロールを作成する] を選択してください。

    7. [Next] (次へ) を選択します。

  7. [同期設定の設定] ページで、次の情報を入力します。

    1. クロールするリポジトリの選択 — GitHub クロールするエンティティまたはコンテンツタイプ。

    2. 変更ログ — 選択すると、すべてのファイルを同期する代わりにインデックスを更新できます。

    3. コンテンツタイプ-含めたいファイルタイプを選択します。

    4. 正規表現パターン — 特定のファイルを包含または除外する正規表現パターン。最大 100 個のパターンを追加できます。

    5. 同期実行スケジュール (頻度) — Amazon Kendra がデータソースと同期する頻度

    6. [Next] (次へ) を選択します。

  8. [フィールドマッピングの設定] ページで、次の情報を入力します。

    1. リポジトリ、リポジトリコミット、課題ドキュメント、課題コメント、課題添付ファイル、プルリクエストコメント、プルリクエスト文書、プルリクエスト添付の場合 —Amazon Kendra 生成されたデフォルトデータソースフィールドから、インデックスにマッピングしたいものを選択します。

    2. フィールドを追加 — カスタムデータソースフィールドを追加して、マッピング先のインデックスフィールド名とフィールドデータタイプを作成します。

    3. [Next] (次へ) を選択します。

  9. [確認と作成] ページで、入力した情報が正しいことを確認し、[データソースの追加] を選択します。また、このページで情報を編集することもできます。データソースが正常に追加されると、データソースページに表示されます。

API

Amazon Kendraに接続するには GitHub

GitHubConfigurationAPI オブジェクトを使用して以下を指定する必要があります。

  • データソースタイプ-SAAS データソースタイプをまたはのいずれかに指定しますON_PREMISE。

  • シークレットの Amazon リソースネーム (ARN) —Secrets Manager GitHub アカウントの認証情報を含むシークレットの Amazon リソースネーム (ARN) を指定します。シークレットは、次のキーを持つ JSON 構造で保存されます。

    {
    "personalToken": "token"
}
    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

  • IAM ロール —RoleArnCreateDataSource コールするタイミングを指定して、IAMSecrets Managerシークレットにアクセスする権限をロールに提供し、 GitHub コネクタに必要なパブリック APIAmazon Kendra を呼び出すことができます。詳細については、「 GitHub データソースの IAM ロール」を参照してください。

オプションで、次の機能が追加することもできます。

  • Virtual Private Cloud (VPC) —VpcConfiguration データソース設定の一部として指定します。「VPCAmazon Kendra を使用するように設定する」を参照してください。

    注記

    GitHub サーバーを使用する場合は、Amazon VPC GitHubを使用してサーバーに接続する必要があります。

  • 変更ログ —Amazon Kendra インデックス内のドキュメントを追加、更新、または削除する必要があるかどうかを判断するために、 GitHub データソースの変更ログメカニズムを使用するか。

    注記

    Amazon Kendraドキュメントをすべてスキャンしたくない場合は、変更ログを使用します。変更ログが大きい場合、Amazon Kendra GitHub データソース内のドキュメントをスキャンする方が変更ログを処理するよりも時間が短くなる場合があります。 GitHubデータソースをインデックスに初めて同期する場合は、すべてのドキュメントがスキャンされます。

  • 包含フィルターと除外フィルター-特定のファイルを含めるか除外するかを指定します。

    注記

    ほとんどのデータソースは正規表現パターンを使用します。正規表現パターンとは、フィルターと呼ばれる包含パターンまたは除外パターンです。包含フィルターを指定すると、包含フィルターと一致するコンテンツのみが索引付けされます。包含フィルターに一致しないドキュメントはインデックスに登録されません。包含パターンと除外フィルターを指定した場合、除外フィルターに一致するドキュメントは、包含フィルターと一致してもインデックス作成されません。

  • フィールドマッピング — GitHub Amazon Kendra データソースフィールドをインデックスフィールドにマッピングすることを選択します。詳細については、データソースフィールドのマッピングを参照してください。

  • ユーザーコンテキストフィルタリング — デフォルトで、データソースのアクセス制御リスト (ACL)Amazon Kendra をクロールします。ACL 情報は、ドキュメントへのユーザーまたはグループのアクセス権に基づいて検索結果をフィルタリングするために使用されます。詳細については、「 GitHub データソースのユーザーコンテキストフィルター」を参照してください。
