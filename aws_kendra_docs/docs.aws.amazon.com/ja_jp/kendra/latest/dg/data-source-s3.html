Amazon S3

Amazon S3は、データをオブジェクトとしてバケットに保存するオブジェクトストレージサービスです。Amazon Kendraを使用して、Amazon S3ドキュメントのバケットリポジトリをインデックス化できます。

警告

Amazon Kendraは、S3Amazon Kendra バケットを操作するためのアクセス権限をプリンシパルに付与するバケットポリシーを使用していません。代わりに、IAMロールを使用します。誤って任意のプリンシパルにアクセス許可を付与することによるデータセキュリティ上の問題を避けるため、Amazon Kendraバケットポリシーに信頼できるメンバーとして含まれていないことを確認してください。ただし、バケットポリシーを追加して、Amazon S3複数のアカウントでバケットを使用することはできます。詳細については、「Amazon S3アカウント間で使用するポリシー」(「S3IAM ロール」タブの「IAMデータソースのロール」) を参照してください。S3IAM データソースのロールについては、「IAMロール」を参照してください。

に接続するにはAmazon S3、S3DataSourceConfiguration またはを使用して、コンソールで接続やその他の情報を指定しますTemplateConfiguration。 TemplateConfiguration オブジェクトを使用する場合、VPC を使用してデータソースに接続できます。 VpcConfiguration 電話をかけるときに指定します CreateDataSource。詳細については、「VPC を設定する」を参照してください。

注記

S3DataSourceConfigurationAPI は VPC の設定をサポートしていません。TemplateConfigurationAPI とコンソールだけが VPC 設定をサポートします。

前提条件

Amazon Kendraを使用して S3 データソースのインデックスを作成する前に、S3AWS とアカウントでこれらの変更を行います。

S3 では、以下が揃っていることを確認します。

  • Amazon S3バケット名の名前をコピーしました。

    注記

    Amazon Kendraバケットはインデックスと同じリージョン内にある必要があり、インデックスにはドキュメントが含まれているバケットにアクセスするための許可が必要です。

  • 各ドキュメントが S3 および同じインデックスに使用する予定の他のデータソース間で一意であることを確認しました。インデックスに使用する各データソースには、データソース間で同じドキュメントが含まれていてはなりません。ドキュメント ID はインデックス全体に適用され、インデックスごとに一意でなければなりません。

AWSアカウントには、次のものがあることを確認してください。

  • Amazon Kendraインデックスを作成し、API を使用している場合は、インデックス ID を記録しました。

  • IAMデータソースのロールを作成し、API を使用している場合は、IAMロールの ARN を記録しました。

IAM既存のロールがない場合は、S3 データソースをに接続するときに、IAMコンソールを使用して新しいロールを作成できますAmazon Kendra。API を使用している場合は、IAM既存の ARN とインデックス ID を指定する必要があります。

接続手順

S3Amazon Kendra データソースに接続するには、Amazon Kendraデータにアクセスできるように S3 データソースの必要な詳細を入力する必要があります。S3 をまだ設定していない場合は、Amazon Kendraを参照してください前提条件。

Console

Amazon Kendraに接続するにはAmazon S3

  1. AWS マネジメントコンソールにサインインし、Amazon Kendra コンソールを開きます。

  2. 左のナビゲーションペインで、[インデックス] を選択し、インデックスのリストから使用するインデックスを選択します。

    注記

    ユーザーアクセス制御設定は、インデックス設定で設定または編集できます。

  3. 「はじめに」ページで、「データソースを追加」を選択します。

  4. [データソースの追加] ページで [S3 コネクタ] を選択し、[データソースの追加] を選択します。

  5. [データソースの詳細を指定する] ページで、次の情報を入力します。

    1. [名前と説明] の [データソース名] に、データソースの名前を入力します。ハイフンは使用できますが、スペースは使用できません。

    2. (オプション) 説明-データソースの説明を入力します。

    3. 「言語」(「デフォルト言語」)-索引用にドキュメントをフィルタリングする言語。特に指定しない限り、言語はデフォルトで英語に設定されます。メタデータで指定された言語は、選択した言語よりも優先されます。

    4. [タグ] の [新しいタグの追加] で、リソースを検索してフィルタリングしたり、AWSコストを追跡したりできます。

    5. [Next] (次へ) を選択します。

  6. 「アクセスとセキュリティの定義」ページで、次のオプション情報を入力します。

    1. IAM ロール — 既存の IAM ロールを選択するか、新しい IAM ロールを作成して、リポジトリの認証情報にアクセスしてコンテンツをインデックス作成します。

      注記

      インデックスに使用される IAM ロールはデータソースには使用できません。既存のロールがインデックスや FAQ に使用されているかどうかわからない場合は、エラーを避けるため [新しいロールを作成する] を選択してください。

    2. [Virtual Private Cloud (VPC) — VPC の使用を選択できます。その場合は、サブネットと VPC セキュリティグループを追加する必要があります。

    3. [Next] (次へ) を選択します。

  7. [同期設定の設定] ページで、次の情報を入力します。

    1. 同期範囲の [データソースの場所] —Amazon S3 データが保存されているバケットへのパス。Browse S3 を選択してバケットを選択します。

    2. (オプション) メタデータファイルのプレフィックスフォルダーの場所 — メタデータが保存されているフォルダーへのパス。[Browse S3] を選択してメタデータフォルダを探します。

    3. (オプション) アクセスコントロールリスト設定ファイルの場所 — S3 データソースに保存されているファイルのアクセス設定を指定する JSON 構造を含むファイルの場所へのパス。「Browse S3」を選択して ACL ファイルを探します。

    4. (オプション) 復号化キーの選択-復号化キーを使用するように選択します。既存のAWS KMSキーを使用するか、新しいキーを作成するかを選択できます。

    5. （オプション）「追加設定」の「正規表現パターン」では、正規表現パターンを追加して、ドキュメントをインデックスに含めたりインデックスから除外したりできます。すべてのパスは、データソースの場所 S3 バケットを基準にしています。最大 100 個のパターンを追加できます。

    6. 同期モードで、「完全同期モード」と「新規、変更、削除されたコンテンツの同期」のいずれかを選択して、データソースのコンテンツが変更されたときのインデックスの更新方法を指定します。データソースを Amazon Kendra と初めて同期すると、すべてのコンテンツがデフォルトで同期されます。

    7. [同期実行スケジュール] の [頻度] で、Amazon Kendraデータソースと同期する頻度を選択します。

    8. [Next] (次へ) を選択します。

  8. フィールドマッピングの設定ページで、次のオプション情報を入力します。

    1. S3 フィールドマッピング —Amazon Kendra 生成されたデフォルトデータソースフィールドから、インデックスにマッピングするフィールドを選択します。

    2. フィールドを追加 — カスタムデータソースフィールドを追加して、マッピング先のインデックスフィールド名とフィールドデータタイプを作成します。

    3. [Next] (次へ) を選択します。

  9. [確認と作成] ページで、入力した情報が正しいことを確認し、[データソースの追加] を選択します。また、このページで情報を編集することもできます。データソースが正常に追加されると、データソースページに表示されます。

TemplateConfiguration API

Amazon Kendraに接続するにはAmazon S3

TemplateConfigurationAPI を使用してデータソーススキーマの JSON を指定する必要があります。これには、以下の情報を入力する必要があります。

  • BucketName—ドキュメントを含むバケットの名前。

  • IAM ロール —RoleArnCreateDataSource コールするタイミングを指定して、Secrets Managerシークレットにアクセスしたり、S3 コネクタに必要なパブリック APIIAM を呼び出したりする権限をロールに与えますAmazon Kendra。詳細については、「S3 データソースの IAM ロール」を参照してください。

オプションで、次の機能を追加することもできます。

  • 包含フィルタと除外フィルタ-特定のファイル名、ファイルタイプ、ファイルパスを含めるか除外するかを指定し、グロブパターン (ワイルドカードパターンを特定のパターンに一致するパス名のリストに展開できるパターン) を使用します。グロブパターンの例には次のものがあります。

    • /myapp/config/*—config ディレクトリ内のすべてのファイル

    • /**/*.png—すべてのディレクトリにあるすべての.png ファイル

    • /**/*.{png,ico,md}—すべてのディレクトリにあるすべての.png、.ico、または.md ファイル

    • /myapp/src/**/*.ts—src ディレクトリ (およびそのすべてのサブディレクトリ) 内のすべての.ts ファイル

    • **/!(*.module).ts—.module.ts 以外のすべての.ts ファイル

    注記

    ほとんどのデータソースは正規表現パターンを使用します。正規表現パターンとは、フィルターと呼ばれる包含パターンまたは除外パターンです。包含フィルターを指定すると、包含フィルターと一致するコンテンツのみが索引付けされます。包含フィルターに一致しないドキュメントはインデックスに登録されません。包含フィルターと除外フィルターを指定した場合、除外フィルターに一致するドキュメントは、包含フィルターと一致してもインデックス作成されません。

  • フィールドマッピング — S3Amazon Kendra データソースフィールドをインデックスフィールドにマッピングすることを選択します。詳細については、データソースフィールドのマッピングを参照してください。

  • [Virtual Private Cloud (VPC) —VpcConfiguration 電話をかけるときに指定しますCreateDataSource。VPCAmazon Kendra を使用するように設定する を参照してください。

  • ドキュメントメタデータの設定-ドキュメントのアクセスコントロール情報、ソース URI、ドキュメントの作成者、カスタム属性などの情報が含まれているドキュメントメタデータファイルを追加します。各メタデータファイルには、1 つのドキュメントに関するメタデータが含まれています。

設定するその他の重要な JSON キーのリストについては、を参照してくださいAmazon S3テンプレートスキーマ。
