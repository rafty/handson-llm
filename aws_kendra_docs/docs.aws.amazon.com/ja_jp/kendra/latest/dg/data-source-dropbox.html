Dropbox

Dropbox は、クラウドストレージ、ドキュメント整理、ドキュメントテンプレートサービスを提供するファイルホスティングサービスです。Dropbox ユーザーの場合は、を使用してAmazon Kendra Dropbox ファイル、Dropbox Paper、Dropbox Paper テンプレート、保存されているウェブページへのショートカットのインデックスを作成できます。特定の Dropbox ファイル、Dropbox Paper、Dropbox Paper テンプレート、Amazon Kendra保存されているウェブページへのショートカットをインデックスに登録するように設定することもできます。

Amazon KendraDropbox Business 向け Dropbox と Dropbox アドバンスドの両方をサポートしています。

Amazon Kendraコンソールと TemplateConfigurationAPI を使用して DropboxAmazon Kendra データソースに接続できます。

サポートされている機能

Amazon KendraDropbox データソースコネクタは以下の機能をサポートしています。

  • 変更ログ

  • フィールドマッピング

  • ユーザーコンテキストのフィルター

  • 包含/除外フィルター

  • Virtual Private Cloud (VPC)

前提条件

Amazon Kendraを使用して Dropbox データソースのインデックスを作成する前に、DropboxAWS とアカウントでこれらの変更を行います。

Dropbox では、次のものが揃っていることを確認してください。

  • Dropbox アドバンスドアカウントを作成し、管理者ユーザーを設定しました。

  • 一意のアプリ名で Dropbox アプリを作成し、スコープアクセスを有効にしました。アプリの作成については、Dropbox のドキュメントをご覧ください。

  • Dropbox コンソールで Dropbox の全権限を有効にし、次の権限を追加しました。

    • ファイル. コンテンツ. 読み込み

    • ファイル.メタデータ.read

    • 共有. 読み込み

    • ファイル_リクエスト.read

    • groups.read

    • チーム情報読み込み

    • team_data.content.read

  • 基本認証情報用の Dropbox アプリキー、Dropbox アプリシークレット、Dropbox アクセストークンを書き留めました。

  • Dropbox アプリ用の一時的な Oauth 2.0 アクセストークンを生成してコピーしました。このトークンは一時的なもので、4 時間後に期限切れになります。OAuth 認証については Dropbox のドキュメントをご覧ください。

    注記

    4 時間後に期限が切れる 1 回限りのアクセストークンではなく、有効期限が切れる Dropbox 更新アクセストークンを作成することをおすすめします。更新アクセストークンは永続的で有効期限がないため、今後もデータソースを同期し続けることができます。

  • 推奨：Amazon Kendraデータソースを中断することなく引き続き同期できるように、有効期限が切れない Dropbox の永久更新トークンを設定しました。更新トークンについては Dropbox のドキュメントをご覧ください。

  • 各ドキュメントが Dropbox や、同じインデックスに使用する予定の他のデータソースと重複していないことを確認しました。インデックスに使用する各データソースには、データソース間で同じドキュメントが含まれていてはなりません。ドキュメント ID はインデックス全体に適用され、インデックスごとに一意でなければなりません。

AWSアカウントには、次のものがあることを確認してください。

  • Amazon Kendraインデックスを作成し、API を使用している場合は、インデックス ID を記録しました。

  • IAMデータソースのロールを作成し、API を使用している場合は、IAMロールの ARN を記録しました。

  • DropboxAWSSecrets Manager 認証情報をシークレットに保存し、API を使用している場合はシークレットの ARN を記録しました。

    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

IAM既存のロールやシークレットがない場合は、Dropbox データソースをに接続するときに、IAMSecrets Managerコンソールを使用して新しいロールとシークレットを作成できますAmazon Kendra。API を使用している場合は、IAMSecrets Manager既存のロールとシークレットの ARN、インデックス ID を指定する必要があります。

接続手順

DropboxAmazon Kendra データソースに接続するには、Amazon Kendraデータにアクセスできるように Dropbox データソースの必要な詳細を入力する必要があります。Dropbox をまだ設定していない場合は、Amazon Kendraを参照してください前提条件。

Console

DropboxAmazon Kendra に接続するには

  1. AWS マネジメントコンソールにサインインし、Amazon Kendra コンソールを開きます。

  2. 左側のナビゲーションペインで、[インデックス] を選択し、インデックスのリストから使用するインデックスを選択します。

    注記

    ユーザーアクセス制御設定は、インデックス設定で設定または編集できます。

  3. 「はじめに」ページで、「データソースを追加」を選択します。

  4. [データソースの追加] ページで [Dropbox コネクタ] を選択し、[データソースを追加] を選択します。

  5. [データソースの詳細を指定] ページで、次の情報を入力します。

    1. [名前と説明] の [データソース名] に、データソースの名前を入力します。ハイフンは使用できますが、スペースは使用できません。

    2. (オプション) 説明-データソースの説明を入力します。

    3. 「言語」(「デフォルト言語」)-索引用にドキュメントをフィルタリングする言語。特に指定しない限り、言語はデフォルトで英語に設定されます。メタデータで指定された言語は、選択した言語よりも優先されます。

    4. [タグ] の [Add new tag] — [Tags] で、リソースを検索してフィルタリングしたり、AWSコストを追跡したりできます。

    5. [Next] (次へ) を選択します。

  6. 「アクセスとセキュリティの定義」ページで、次の情報を入力します。

    1. 認証トークンのタイプ — ユースケースに応じて、永続トークン (推奨) とアクセストークン (一時使用) のどちらかを選択します。

    2. AWSSecrets Managersecret —既存のシークレットを選択するか、Secrets Manager新しいシークレットを作成して Dropbox 認証情報を保存します。新しいシークレットを作成することを選択すると、AWSSecrets Managerシークレットウィンドウが開きます。

      1. 「AWSSecrets Managerシークレットの作成」ウィンドウに次の情報を入力します。

        1. シークレット名 — シークレットの名前。シークレット名には「AmazonKendra-Dropbox-」というプレフィックスが自動的に追加されます。

        2. アプリキー、アプリシークレット、トークン情報（固定または一時的）には、Dropbox アカウントから生成した認証情報の値を入力します。

      2. [Save] (保存) を選択します。

    3. Virtual Private Cloud (VPC) — VPC の使用を選択できます。その場合は、サブネットと VPC セキュリティグループを追加する必要があります。

    4. IAM ロール — 既存の IAM ロールを選択するか、新しい IAM ロールを作成して、リポジトリの認証情報にアクセスしてコンテンツをインデックス作成してください。

      注記

      インデックスに使用される IAM ロールはデータソースには使用できません。既存のロールがインデックスや FAQ に使用されているかどうかわからない場合は、エラーを避けるため [新しいロールを作成する] を選択してください。

    5. [Next] (次へ) を選択します。

  7. [同期設定の設定] ページで、次の情報を入力します。

    1. エンティティまたはコンテンツタイプを選択する場合-クロールするエンティティまたはコンテンツタイプを選択します。

    2. ログモードの変更 — すべてのファイルを同期するのではなく、インデックスを更新することを選択します。

    3. 正規表現パターンの追加設定 — 正規表現パターンを追加して、特定のファイルを含めたり除外したりします。

    4. [同期実行スケジュール] の [頻度] —Amazon Kendra データソースと同期する頻度を選択します。

    5. [Next] (次へ) を選択します。

  8. [フィールドマッピングの設定] ページで、次の情報を入力します。

    1. ファイル、Dropbox Paper 、Dropbox Paper テンプレート —Amazon Kendra 生成されたデフォルトデータソースフィールドから、インデックスにマッピングしたいものを選択します。

    2. フィールドを追加 — カスタムデータソースフィールドを追加して、マッピング先のインデックスフィールド名とフィールドデータタイプを作成します。

    3. [Next] (次へ) を選択します。

  9. [確認と作成] ページで、入力した情報が正しいことを確認し、[データソースの追加] を選択します。また、このページで情報を編集することもできます。データソースが正常に追加されると、データソースページに表示されます。

API

DropboxAmazon Kendra に接続するには

TemplateConfigurationAPI を使用してデータソーススキーマの JSON を指定する必要があります。これには、以下の情報を入力する必要があります。

  • データソース — データソースをとして指定する必要がありますDROPBOX。

  • タイプ —TEMPLATE 電話をかけるときにタイプとして指定しますCreateDataSource。

  • シークレットの Amazon リソースネーム（ARN）— DropboxSecrets Manager アカウントの認証情報を含むシークレットの Amazon リソースネーム（ARN）を指定します。シークレットは以下のキーを持つ JSON 構造に保存されます。

    {
"appKey": "Dropbox app key",
"appSecret": "Dropbox app secret",
"accesstoken": "temporary access token or refresh access token"
}
    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

  • IAM ロール —RoleArnCreateDataSource コールするタイミングを指定して、IAMSecrets Managerロールにシークレットへのアクセス権限を付与したり、Dropbox コネクタに必要なパブリック APIAmazon Kendra を呼び出したりすることができます。詳細については、Dropbox データソースの IAM ロールを参照してください。

オプションで、次の機能を追加することもできます。

  • [Virtual Private Cloud (VPC)] —VpcConfiguration 電話をかけるときに指定しますCreateDataSource。VPCAmazon Kendra を使用するように設定する を参照してください。

  • Change Log —Amazon Kendra インデックス内のドキュメントを追加、更新、削除する必要があるかどうかを判断するために、Dropbox データソースの変更ログメカニズムを使用するか。

    注記

    Amazon Kendraドキュメントをすべてスキャンしたくない場合は、変更ログを使用します。変更ログが大きい場合、DropboxAmazon Kendra データソースのドキュメントをスキャンした方が変更ログを処理するよりも時間がかかる場合があります。Dropbox データソースをインデックスに初めて同期する場合は、すべてのドキュメントがスキャンされます。

  • 包含フィルターと除外フィルター-特定のファイルを含めるか除外するかを指定します。

    注記

    ほとんどのデータソースは正規表現パターンを使用します。正規表現パターンとは、フィルターと呼ばれる包含パターンまたは除外パターンです。包含フィルターを指定すると、包含フィルターと一致するコンテンツのみが索引付けされます。包含フィルターに一致しないドキュメントはインデックスに登録されません。包含フィルターと除外フィルターを指定した場合、除外フィルターに一致するドキュメントは、包含フィルターと一致してもインデックス作成されません。

  • フィールドマッピング — DropboxAmazon Kendra データソースフィールドをインデックスフィールドにマッピングすることを選択します。詳細については、データソースフィールドのマッピングを参照してください。

  • ユーザーコンテキストフィルタリング — デフォルトで、データソースのアクセス制御リスト (ACL)Amazon Kendra をクロールします。ACL 情報は、ドキュメントへのユーザーまたはグループのアクセス権に基づいて検索結果をフィルタリングするために使用されます。詳しくは、「Dropbox データソースのユーザーコンテキストフィルタリング」を参照してください。

設定するその他の重要な JSON キーのリストについては、を参照してくださいDropbox テンプレートスキーマ。
