DescribeDataSource

Amazon Kendra データソースコネクターに関する情報を取得します。

リクエストの構文

{
   "Id": "string",
   "IndexId": "string"
}

リクエストパラメータ

すべてのアクションに共通のパラメータの詳細については、「共通パラメータ」を参照してください。

リクエストは以下の JSON 形式のデータを受け入れます。

Id

データソースコネクターの識別子。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 100 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9_-]*

必須: はい

IndexId

データソースコネクターと一緒に使用するインデックスの識別子。

型: 文字列

長さの制限: 最大長は 36 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9-]*

必須: はい

レスポンス要素

アクションが成功すると、サービスは HTTP 200 レスポンスを返します。

サービスから以下のデータが JSON 形式で返されます。

Configuration

データソースコネクターの構成詳細。データソースの設定方法を示しています。データソースの構成オプションは、データソースプロバイダによって異なります。

型: DataSourceConfiguration オブジェクト

CreatedAt

データソースコネクターが作成された時刻の Unix タイムスタンプ。

タイプ: Timestamp

CustomDocumentEnrichmentConfiguration

データソースを説明する際のドキュメントの取り込みプロセス中に、ドキュメントのメタデータとコンテンツを変更するための構成情報。

Amazon Kendra にドキュメントを取り込む際のドキュメントメタデータの作成、変更、削除方法、またはその他のコンテンツの変更を行う方法の詳細については、取り込みプロセス中のドキュメントメタデータのカスタマイズを参照してください。

型: CustomDocumentEnrichmentConfiguration オブジェクト

Description

データソースコネクターの説明。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,000 です。

Pattern: ^\P{C}*$

ErrorMessage

Status フィールド値が FAILED の場合、ErrorMessage フィールドには、データソースの失敗の原因となったエラーの説明が含まれています。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 2,048 です。

パターン: ^\P{C}*$

Id

データソースコネクターの識別子。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 100 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9_-]*

IndexId

データソースコネクターと一緒に使用するインデックスの識別子。

型: 文字列

長さの制限: 最大長は 36 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9-]*

LanguageCode

言語のコード。これは、データソース内のすべてのドキュメントでサポートされている言語を示します。英語はデフォルトでサポートされています。コードを含む、サポートされている言語の詳細については、英語以外の言語でドキュメントを追加するを参照してください。

型: 文字列

長さの制限: 最小長は 2 です。最大長は 10 です。

パターン: [a-zA-Z-]*

Name

データソースコネクターの名前。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 1,000 です。

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

RoleArn

データソースと必要なリソースへのアクセス許可を持つロールの Amazon リソースネーム (ARN)。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,284 です。

パターン: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Schedule

Amazon Kendra がインデックスを更新するスケジュール。

型: 文字列

Status

データソースコネクターの現在のステータス。ステータスが ACTIVE の場合、データソースを使用する準備ができています。ステータスが FAILED の場合、ErrorMessage フィールドには、データソースが失敗した理由が含まれます。

型: 文字列

有効な値: CREATING | DELETING | FAILED | UPDATING | ACTIVE

Type

データソースのタイプ。例えば、SHAREPOINT です。

型: 文字列

有効な値: S3 | SHAREPOINT | DATABASE | SALESFORCE | ONEDRIVE | SERVICENOW | CUSTOM | CONFLUENCE | GOOGLEDRIVE | WEBCRAWLER | WORKDOCS | FSX | SLACK | BOX | QUIP | JIRA | GITHUB | ALFRESCO | TEMPLATE

UpdatedAt

データソースコネクターが最後に更新された時刻の Unix タイムスタンプ。

タイプ: Timestamp

VpcConfiguration

データソースに接続するための Amazon Virtual Private Cloud の構成情報。詳細については、「VPC を設定する」を参照してください。

型: DataSourceVpcConfiguration オブジェクト

エラー

すべてのアクションに共通のエラーについては、「一般的なエラー」を参照してください。

AccessDeniedException

このアクションを実行する十分なアクセス権限がありません。必要な権限ポリシーとユーザーアカウントがあることを確認して、もう一度試してください。

HTTP ステータスコード: 400

InternalServerException

Amazon Kendra サービスに使用されている内部サーバーで問題が発生しました。数分後にもう一度お試しださい。または、Support に連絡してサポートを依頼してください。

HTTP ステータスコード: 500

ResourceNotFoundException

使用したいリソースは存在しません。正しいリソースを提供したことを確認して、もう一度お試しください。

HTTP ステータスコード: 400

ThrottlingException

リクエストは、制限が必要なために実行が拒否されました。リクエストの数を減らして、もう一度お試しください。

HTTP ステータスコード: 400

ValidationException

入力が Amazon Kendra サービスで設定された制約を満たしていません。正しい入力を入力して、もう一度お試しください。

HTTP ステータスコード: 400
