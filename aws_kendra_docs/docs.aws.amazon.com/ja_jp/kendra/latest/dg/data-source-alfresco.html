アルフレスコ

OnPrem Alfrescoは、顧客がコンテンツを保存および管理するのに役立つコンテンツ管理サービスです。Amazon Kendraを使用して、Alfresco ドキュメントライブラリ、Wiki、ブログのインデックスを作成できます。

Amazon Kendraコンソールと AlfrescoConfigurationAPIAmazon Kendra を使用して Alfresco データソースに接続できます。

注記

Alfresco データソースコネクタは現在プレビューモードです。基本認証は現在サポートされています。Alfresco コネクタを本番環境で使用したい場合は、Support にお問い合わせください。アルフレスコの PaaS は現在サポートされていません。

サポートされている機能

Amazon KendraAlfresco データソースコネクタは以下の機能をサポートしています。

  • フィールドマッピング

  • ユーザーコンテキストのフィルター

  • 包含/除外フィルター

  • Virtual Private Cloud (VPC)

前提条件

Amazon Kendraを使用して Alfresco データソースのインデックスを作成する前に、AlfrescoAWS とアカウントでこれらの変更を行います。

Alfrescoでは、次のものを用意してください。

  • 管理者ユーザー名とパスワードを含む Alfresco 認証情報を書き留めました。

  • ALFRESCO_ADMINISTRATORSユーザーをグループに追加しました。

  • アルフレスコサイト URL と Alfresco サイト ID をコピーしました。

  • 各ドキュメントが、Alfrescoおよび同じインデックスに使用する予定の他のデータソース間で一意であることを確認しました。インデックスに使用する各データソースには、データソース間で同じドキュメントが含まれていてはなりません。ドキュメント ID はインデックス全体に適用され、インデックスごとに一意でなければなりません。

AWSアカウントには、次のものがあることを確認してください。

  • Amazon Kendraインデックスを作成し、API を使用している場合は、インデックス ID を記録しました。

  • IAMデータソースのロールを作成し、API を使用している場合は、IAMロールの ARN を記録しました。

  • AlfrescoAWSSecrets Manager の認証情報をシークレットに保存し、API を使用している場合は、シークレットの ARN を記録しました。

    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

IAM既存のロールまたはシークレットがない場合は、Alfresco データソースをに接続するときに、IAMSecrets Managerコンソールを使用して新しいロールとシークレットを作成できますAmazon Kendra。API を使用している場合は、IAMSecrets Manager既存のロールとシークレットの ARN、インデックス ID を指定する必要があります。

接続手順

AlfrescoAmazon Kendra データソースに接続するには、Amazon Kendraデータにアクセスできるように Alfresco データソースの必要な詳細を入力する必要があります。Alfresco をまだ設定していない場合は、Amazon Kendraを参照してください前提条件。

Console

Amazon Kendraアルフレスコに接続するには

  1. AWS マネジメントコンソールにサインインし、Amazon Kendra コンソールを開きます。

  2. 左側のナビゲーションペインで、[インデックス] を選択し、インデックスのリストから使用するインデックスを選択します。

    注記

    ユーザーアクセス制御設定は、インデックス設定で設定または編集できます。

  3. 「はじめに」ページで、「データソースを追加」を選択します。

  4. [データソースの追加] ページで [Alfresco コネクタ] を選択し、[データソースの追加] を選択します。

  5. [データソースの詳細の指定] ページで、次の情報を入力します。

    1. [名前と説明] の [データソース名] に、データソースの名前を入力します。ハイフンは使用できますが、スペースは使用できません。

    2. (オプション) 説明-データソースの説明を入力します。

    3. 「言語」(「デフォルト言語」)-索引用にドキュメントをフィルタリングする言語。特に指定しない限り、言語はデフォルトで英語に設定されます。メタデータで指定された言語は、選択した言語よりも優先されます。

    4. [タグ] の [Add new tag]-[タグ] で、リソースを検索してフィルタリングしたり、AWSコストを追跡したりできます。

    5. [Next] (次へ) を選択します。

  6. 「アクセスとセキュリティの定義」ページで、次の情報を入力します。

    1. アルフレスコサイトの URL —アルフレスコサイトの URL を入力します。たとえば、https://hostname:8080。

    2. SSL 証明書の場所-Amazon S3 パスを入力します。

    3. サイト ID —アルフレスコのサイト ID。

    4. AWSSecrets Managersecret —既存のシークレットを選択するか、Secrets Manager新しいシークレットを作成して Alfresco 認証情報を保存します。新しいシークレットを作成することを選択すると、AWSSecrets Managerシークレットウィンドウが開きます。

      1. 「AWSSecrets Managerシークレットの作成」ウィンドウに次の情報を入力します。

        1. シークレット名 — シークレットの名前。シークレット名には「AmazonKendra-Alfresco-」というプレフィックスが自動的に追加されます。

        2. [ユーザー名] と [パスワード] の場合-Alfresco のユーザー名とパスワードを入力します。

      2. [Save] (保存) を選択します。

    5. [VVirtual Private Cloud (VPC) — VPC を選択できます。その場合は、サブネットと VPC セキュリティグループを追加する必要があります。

    6. IAM ロール — 既存の IAM ロールを選択するか、新しい IAM ロールを作成して、リポジトリの認証情報にアクセスしてコンテンツをインデックス作成してください。

      注記

      インデックスに使用される IAM ロールはデータソースには使用できません。既存のロールがインデックスや FAQ に使用されているかどうかわからない場合は、エラーを避けるため [新しいロールを作成する] を選択してください。

    7. [Next] (次へ) を選択します。

  7. [同期設定の設定] ページで、次の情報を入力します。

    1. クロールするコンテンツを選択-クロールする Alfresco エンティティまたはコンテンツタイプ。

    2. 正規表現パターン — 特定のファイルを包含または除外する正規表現パターン。最大 100 個のパターンを追加できます。

    3. [同期実行スケジュール] の [頻度] で、Amazon Kendraデータソースと同期する頻度を選択します。

    4. [Next] (次へ) を選択します。

  8. [フィールドマッピングの設定] ページで、次の情報を入力します。

    1. ドキュメントライブラリ、Wiki、ブログ用 —Amazon Kendra 生成されたデフォルトデータソースフィールドから、インデックスにマッピングするフィールドを選択します。

    2. フィールドを追加 — カスタムデータソースフィールドを追加して、マッピング先のインデックスフィールド名とフィールドデータタイプを作成します。

    3. [Next] (次へ) を選択します。

  9. [確認と作成] ページで、入力した情報が正しいことを確認し、[データソースの追加] を選択します。また、このページで情報を編集することもできます。データソースが正常に追加されると、データソースページに表示されます。

API

Amazon Kendraアルフレスコに接続するには

AlfrescoConfigurationAPI を使用して以下を指定する必要があります。

  • シークレットの Amazon リソースネーム (ARN) — AlfrescoSecrets Manager アカウントの認証情報を含むシークレットの Amazon リソースネーム (ARN) を指定します。シークレットは、次のキーを持つ JSON 構造で格納されます。

    {
    "username": "user name",
    "password": "password"
}
    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

  • IAM ロール —RoleArnCreateDataSource コールするタイミングを指定して、IAMSecrets Managerシークレットにアクセスする権限をロールに付与し、Alfresco コネクタに必要なパブリック APIAmazon Kendra を呼び出すことができます。詳細については、「Alfresco データソースの IAM ロール」を参照してください。

オプションで、次のオプション機能を追加することもできます。

  • Virtual Private Cloud (VPC) —VpcConfiguration データソース設定の一部として指定します。「VPCAmazon Kendra を使用するように設定する」を参照してください。

  • 包含フィルターと除外フィルター-特定のファイルを含めるか除外するかを指定します。

    注記

    ほとんどのデータソースは正規表現パターンを使用します。正規表現パターンとは、フィルターと呼ばれる包含パターンまたは除外パターンです。包含フィルターを指定すると、包含フィルターと一致するコンテンツのみが索引付けされます。包含フィルターに一致しないドキュメントはインデックスに登録されません。包含フィルターと除外フィルターを指定した場合、除外フィルターに一致するドキュメントは、包含フィルターと一致してもインデックス作成されません。

  • フィールドマッピング — AlfrescoAmazon Kendra データソースフィールドをインデックスフィールドにマッピングすることを選択します。詳細については、データソースフィールドのマッピングを参照してください。

  • ユーザーコンテキストフィルタリング — デフォルトで、データソースのアクセス制御リスト (ACL)Amazon Kendra をクロールします。ACL 情報は、ドキュメントへのユーザーまたはグループのアクセス権に基づいて検索結果をフィルタリングするために使用されます。詳細については、Alfresco データソースのユーザーコンテキストフィルタリングを参照してください。
