データソースフィールドのマッピング

データソースのドキュメントフィールドまたはコンテンツフィールドをインデックスのフィールドにマップできます。例えば、ドキュメントの部門情報を含む「Departt」というフィールドがある場合、そのフィールドを「Department」というインデックスフィールドにマッピングできます。そうすれば、ドキュメントをクエリするときにこのフィールドを使用できます。

などの標準フィールドまたは予約フィールドをマップすることもできます_created_at。データソースに「creation_date」というフィールドがある場合は、Amazon Kendraこれをという同等の予約フィールドにマッピングできます_created_at。ほとんどのデータソースのフィールドをマップできます。

S3 バケットまたは S3 データソースにドキュメントを保存する場合、メタデータファイルを使用してカスタム属性を直接指定できます。詳細については、「カスタムドキュメントフィールドの作成」を参照してください。

データソースフィールドをインデックスフィールドにマッピングするには、次の 3 つの手順を実行します。

  1. インデックスを作成します。詳細については、「インデックスの作成」を参照してください。

  2. インデックスを更新して、カスタムフィールドを追加します。

  3. データソースフィールドをインデックスフィールドにマッピングするデータソースを作成します。

インデックスを更新してカスタムフィールドを追加するには、コンソールまたはUpdateIndex API を使用します。合計 500 のカスタムフィールドをインデックスに追加できます。

コンソールでは、データソースフィールドを予約済みフィールド名の 1 つにマッピングすることを選択できます。または、そのフィールドに対応する新しいインデックスフィールドを作成することもできます。データベースデータソースの場合、データベース列の名前が予約フィールドの名前と一致する場合、フィールドと列は自動的にマッピングされます。

UpdateIndexAPI では、DocumentMetadataConfigurationUpdatesパラメータを使用してカスタムフィールドと予約済みフィーrドを追加します。

次の JSON の例では、「Department」DocumentMetadataConfigurationUpdates というフィールドをインデックスに追加します。

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

フィールドを作成するときに、検索でのフィールドの使用方法を設定するオプションがあります。次から選択できます。

  • Displayable — クエリレスポンスでフィールドを返すかどうかを指定します。デフォルトは true です。

  • ファセットテーブル —フィールドを使用してファセットを作成できることを示します。デフォルトは false です。

  • 検索可能 — 検索でフィールドを使用するかどうかを指定します。デフォルトは、文字列フィールドに対しては true、数値フィールドと日付フィールドに対しては false です。

  • Sortable — フィールドを使用してクエリからの応答をソートできることを示します。日付、数値、および文字列フィールドに対してのみ設定できます。文字列リストフィールドには設定できません。

次の JSON の例では、「Department」DocumentMetadataConfigurationUpdates というフィールドをインデックスに追加しファセット可能とマークしています。

"DocumentMetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE",
       "Search": {
           "Facetable": true
       }
   }
]

Amazon Kendra の組み込みドキュメントフィールドを使用する

UpdateIndex API を使用すると、DocumentMetadataConfigurationUpdates予約フィールド名を使用して予約フィールドまたは組み込みフィールドを作成し、対応するドキュメント属性またはドキュメントフィールドにマップできます。この方法でもカスタムフィールドを作成できます。データソースコネクタを使用する場合、ほとんどの場合、Amazon Kendraデータソースドキュメントフィールドをインデックスフィールドにマップするフィールドマッピングが含まれます。フィールドを作成すると、そのフィールドを表示可能、ファセット可能、検索可能、Searchおよびソート可能に設定するようにオブジェクトを設定できます。Relevanceオブジェクトを設定して、フィールドの関連性向上期間、鮮度、重要度、ランク順および重要度値を設定できます。フィールドを作成した後は、フィールドタイプを変更できません。

Amazon Kendraには、使用できる次の予約済みまたは組み込みのドキュメントフィールドがあります。

  • _authors—ドキュメントの内容を担当する 1 人以上の作成者のリスト。

  • _category—ドキュメントを特定のグループに配置するカテゴリ。

  • _created_at—ドキュメントが作成された ISO 8601 形式の日付と時刻。例えば、2012-03-25T12:30:10+01:00 は、中央ヨーロッパ時間の 2012 年 3 月 25 日午後 12 時 30 分 (プラス 10 秒) の ISO 8601 の日付/時刻形式です。

  • _data_source_id— ドキュメントを含むデータソースの識別子。

  • _document_body—ドキュメントのコンテンツ。

  • _document_id—ドキュメントの一意の識別子。

  • _document_title—ドキュメントのタイトル。

  • _excerpt_page_number—ドキュメントの抜粋が表示される PDF ファイルのページ番号。2020 年 9 月 8 日より前にインデックスが作成された場合、この属性を使用する前に、ドキュメントのインデックスを再作成する必要があります。

  • _faq_id—これがよくある質問の質問と回答の場合は、それらの一意の識別子。

  • _file_type—pdf や doc など、ドキュメントのファイルタイプ。

  • _last_updated_at—ドキュメントが最後に更新された ISO 8601 形式の日付と時刻。例えば、2012-03-25T12:30:10+01:00 は、中央ヨーロッパ時間の 2012 年 3 月 25 日午後 12 時 30 分 (プラス 10 秒) の ISO 8601 の日付/時刻形式です。

  • _source_uri—ドキュメントが利用可能な URI。例えば、会社のウェブサイト上のドキュメントの URI などです。

  • _version—ドキュメントの特定のバージョンの識別子。

  • _view_count— ドキュメントが表示された回数。

  • _language_code(文字列) — ドキュメントに適用される言語のコード。言語を指定しないと、デフォルトで英語になります。コードを含む、サポートされている言語の詳細については、英語以外の言語でドキュメントを追加するを参照してください。

カスタムフィールドの場合、予約フィールドや組み込みフィールドを作成するときと同じように、UpdateIndex APIDocumentMetadataConfigurationUpdates を使用してこれらのフィールドを作成します。カスタムフィールドに適切なデータ型を設定する必要があります。フィールドを作成した後は、フィールドタイプを変更できません。

カスタムフィールドに設定できるタイプは次のとおりです。

  • 日付

  • 数値

  • 文字列

  • 文字列リスト

カスタムフィールドは、コンソールまたは UpdateIndexAPI を使用して作成します。カスタムフィールドを作成したら、予約フィールドの場合と同様に、ドキュメント属性にマッピングします。API を使用してドキュメントをインデックスに追加した場合、BatchPutDocumentAPI を使用して属性をマッピングします。Amazon S3データソースからインデックス作成されたドキュメントの場合、ドキュメント属性を記述する JSON 構造を含むメタデータファイルを使用して属性をマッピングします。フィールドマッピングが可能なデータベースまたはデータソースで索引付けされたドキュメントの場合、コンソールまたはデータソース設定を使用して属性をマッピングします。
