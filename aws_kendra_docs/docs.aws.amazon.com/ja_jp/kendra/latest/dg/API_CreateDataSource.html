CreateDataSource

Amazon Kendra インデックスで使用するデータソースコネクターを作成します。

データソースの名前、データソースコネクタの種類、説明を指定します。データソースコネクタの構成情報も指定します。

CreateDataSource は同期演算です。データソースが正常に作成された場合、この演算は 200 を返します。それ以外の場合、例外が発生します。

Python SDK を使用してインデックスとデータソースを作成する例については、「Python SDK 入門」を参照してください。Java SDK を使用してインデックスとデータソースを作成する例については、「Java SDK 入門」を参照してください。

リクエストパラメータ

すべてのアクションに共通のパラメータの詳細については、「共通パラメータ」を参照してください。

リクエストは以下の JSON 形式のデータを受け入れます。

ClientToken

データソースコネクターの作成リクエストを識別するために指定するトークン。CreateDataSource同じクライアントトークンを使用した API を複数回呼び出すと、データソースコネクターは 1 つだけ作成されます。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 100 です。

必須: いいえ

Configuration

データソースリポジトリに接続するための設定情報。

Type パラメータが CUSTOM に設定されている場合、Configuration パラメータを指定することはできません。これを行うと、ValidationException 例外を受け取ります。

Configuration パラメータは、他のすべてのデータソースに必須です。

型: DataSourceConfiguration オブジェクト

必須: いいえ

CustomDocumentEnrichmentConfiguration

ドキュメントの取り込みプロセス中にドキュメントのメタデータとコンテンツを変更するための構成情報。

Amazon Kendra にドキュメントを取り込む際のドキュメントメタデータの作成、変更、削除方法、またはその他のコンテンツの変更を行う方法の詳細については、取り込みプロセス中のドキュメントメタデータのカスタマイズを参照してください。

型: CustomDocumentEnrichmentConfiguration オブジェクト

必須: いいえ

Description

データソースコネクターの説明。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,000 です。

Pattern: ^\P{C}*$

必須: いいえ

IndexId

データソースコネクターと一緒に使用するインデックスの識別子。

型: 文字列

長さの制限: 最大長は 36 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9-]*

必須: はい

LanguageCode

言語のコード。これにより、データソースコネクターの作成時に、すべてのドキュメントの言語をサポートできます。英語はデフォルトでサポートされています。コードを含む、サポートされている言語の詳細については、英語以外の言語でドキュメントを追加するを参照してください。

型: 文字列

長さの制限: 最小長は 2 です。最大長は 10 です。

パターン: [a-zA-Z-]*

必須: いいえ

Name

データソースコネクターコネクターコネクターコネクターの名前。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 1,000 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9_-]*

必須: はい

RoleArn

データソースと必要なリソースへのアクセス許可を持つロールの Amazon リソースネーム (ARN)。詳細については、Amazon Kendra の IAM アクセスロールを参照してください。 。

Type パラメータが CUSTOM に設定されている場合、RoleArn パラメータを指定することはできません。これを行うと、ValidationException 例外を受け取ります。

RoleArn パラメータは、他のすべてのデータソースに必須です。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,284 です。

パターン: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

必須: いいえ

Schedule

Amazon Kendra がデータソースレポジトリ内のドキュメントをチェックし、インデックスを更新する頻度を設定します。スケジュールを設定しない場合、Amazon Kendra はインデックスの定期的な更新を行いません。StartDataSourceSyncJobAPI を呼び出してインデックスを更新できます。

cron-フォーマットスケジュール文字列、またはインデックスをオンデマンドで更新する文字列のいずれかを保持する文字列のいずれかを保持する文字列です。

Type パラメータが CUSTOM に設定されている場合、Schedule パラメータを指定することはできません。これを行うと、ValidationException 例外を受け取ります。

型: 文字列

必須: いいえ

Tags

データソースコネクターを識別または分類するキーと値のペアのリスト。タグを使用して、データソースコネクターへのアクセスを制御することもできます。タグキーと値に使用できる文字は、Unicode 文字、数字、空白、および _ . : / = + - @ の記号です。

型: Tag オブジェクトの配列

配列メンバー: 最小数は 0 項目です。最大数は 200 項目です。

必須: いいえ

Type

データソースレポジトリの型。例えば、SHAREPOINT です。

型: 文字列

有効な値: S3 | SHAREPOINT | DATABASE | SALESFORCE | ONEDRIVE | SERVICENOW | CUSTOM | CONFLUENCE | GOOGLEDRIVE | WEBCRAWLER | WORKDOCS | FSX | SLACK | BOX | QUIP | JIRA | GITHUB | ALFRESCO | TEMPLATE

必須: はい

VpcConfiguration

データソースに接続するための Amazon Virtual Private Cloud の構成情報。詳細については、「VPC を設定する」を参照してください。

型: DataSourceVpcConfiguration オブジェクト

必須: いいえ

レスポンス要素

アクションが成功すると、サービスは HTTP 200 レスポンスを返します。

サービスから以下のデータが JSON 形式で返されます。

Id

データソースコネクターコネクターコネクターの識別子。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 100 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9_-]*

エラー

すべてのアクションに共通のエラーについては、「一般的なエラー」を参照してください。

AccessDeniedException

このアクションを実行する十分なアクセス権限がありません。必要な権限ポリシーとユーザーアカウントがあることを確認して、もう一度試してください。

HTTP ステータスコード: 400

ConflictException

リクエストに矛盾が発生しました。リソースとの不一致を修正して、もう一度試してください。

HTTP ステータスコード: 400

InternalServerException

Amazon Kendra サービスに使用されている内部サーバーで問題が発生しました。数分後にもう一度お試しSupport さい。

HTTP ステータスコード: 500

ResourceAlreadyExistException

使用したいリソースは既に存在します。正しいリソースを提供していることを確認して、もう一度お試しください。

HTTP ステータスコード: 400

ResourceNotFoundException

使用したいリソースは存在しません。正しいリソースを提供していることを確認して、もう一度お試しください。

HTTP ステータスコード: 400

ServiceQuotaExceededException

Amazon Kendra サービスに設定された制限を超えました。詳細についてはクォータを参照するか、制限の引き上げについてはSupport にお問い合わせください。

HTTP ステータスコード: 400

ThrottlingException

リクエストは、制限が必要なために実行が拒否されました。リクエストの数を減らして、もう一度お試しください。

HTTP ステータスコード: 400

ValidationException

入力が Amazon Kendra サービスで設定された制約を満たしていません。正しい入力を入力して、もう一度お試しください。

HTTP ステータスコード: 400
