CreateIndex

Amazon Kendra インデックスを作成します。インデックスの作成は非同期 API です。インデックスの作成が完了したかどうかを確認するには、DescribeIndex の呼び出しから返される Status フィールドをチェックします。インデックスの使用準備が完了している場合は、Status フィールは ACTIVE に設定されます。

インデックスがアクティブになったら、BatchPutDocument API を使用するか、サポートされているデータソースのいずれかを使用して、ドキュメントにインデックス作成されたインデックス付けされたアクセス権限があります。

Python SDK を使用してインデックスとデータソースを作成する例については、「Python SDK 入門」を参照してください。Java SDK を使用してインデックスとデータソースを作成する例については、「Java SDK 入門」を参照してください。

リクエストパラメータ

すべてのアクションに共通のパラメータの詳細については、「共通パラメータ」を参照してください。

リクエストは以下の JSON 形式のデータを受け入れます。

ClientToken

インデックスの作成リクエストを識別するために指定するトークン。CreateIndex同じクライアントトークンを使用した API を複数回呼び出すと、インデックスは 1 つだけ作成されます。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 100 です。

必須: いいえ

Description

インデックスの説明。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,000 です。

Pattern: ^\P{C}*$

必須: いいえ

Edition

インデックスに使用する Amazon Kendra エディション。開発、テスト、または概念実証を目的としたインデックスの DEVELOPER_EDITION を選択します。ENTERPRISE_EDITION生産に使用します。一度インデックスのエディションを設定すると、変更できません。

Edition パラメータはオプションです。値を指定しない場合、デフォルトは ENTERPRISE_EDITION になります。

エンタープライズエディションとデベロッパーエディションのクォータ制限の詳細については、クォータを参照してください。

型: 文字列

有効な値: DEVELOPER_EDITION | ENTERPRISE_EDITION

必須: いいえ

Name

インデックスの名前。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 1,000 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9_-]*

必須: はい

RoleArn

Amazon CloudWatch のログとメトリックスへアクセス権限を持つ IAM ロールの Amazon リソースネーム (ARN) です。詳細については、Amazon Kendra の IAM アクセスロールを参照してください。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,284 です。

パターン: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

必須: はい

ServerSideEncryptionConfiguration

Amazon Kendra によってインデックス作成されたデータの暗号化に使用するAWS KMSカスタマーマネージドキー (CMK) の識別子。Amazon Kendra は非対称 CMK をサポートしていません。

型: ServerSideEncryptionConfiguration オブジェクト

必須: いいえ

Tags

インデックスを識別または分類するキーと値のペアのリスト。タグを使用して、インデックスへのアクセスを制御することもできます。タグキーと値に使用できる文字は、Unicode 文字、数字、空白、および _ . : / = + - @ の記号です。

型: Tag オブジェクトの配列

配列メンバー: 最小数は 0 項目です。最大数は 200 項目です。

必須: いいえ

UserContextPolicy

ユーザコンテキストポリシー。

ATTRIBUTE_FILTER

すべてのインデックス付きコンテンツは、すべてのユーザーが検索および表示可能です。ユーザーコンテキストでサーチ結果をフィルタリングする場合は、_user_id および _group_ids の属性フィルターを使用するか、UserContext のユーザーおよびグループの情報を指定できます。

USER_TOKEN

トークンベースのユーザーアクセス制御を有効にして、ユーザーコンテキストで検索結果をフィルタリングします。アクセス権のないすべてのドキュメントと、ユーザーがアクセス可能なすべてのドキュメントを検索および表示できるようになります。

型: 文字列

有効な値: ATTRIBUTE_FILTER | USER_TOKEN

必須: いいえ

UserGroupResolutionConfiguration

AWS IAM Identity Center (successor to AWS Single Sign-On)アイデンティティ・ソースからユーザーとグループを取得します。これを設定するには、を参照してくださいUserGroupResolutionConfiguration。

型: UserGroupResolutionConfiguration オブジェクト

必須: いいえ

UserTokenConfigurations

ユーザートークンの設定。

型: UserTokenConfiguration オブジェクトの配列

配列メンバー: 最大数は 1 項目です。

必須: いいえ

レスポンスの構文

{
   "Id": "string"
}

レスポンス要素

アクションが成功すると、サービスは HTTP 200 レスポンスを返します。

サービスから以下のデータが JSON 形式で返されます。

Id

インデックスの識別子。この識別子は、インデックスのクエリ、データソースの設定、またはドキュメントのインデックス作成に使用します。

型: 文字列

長さの制限: 最大長は 36 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9-]*

エラー

すべてのアクションに共通のエラーについては、「一般的なエラー」を参照してください。

AccessDeniedException

このアクションを実行する十分なアクセス権限がありません。必要な権限ポリシーとユーザーアカウントがあることを確認して、もう一度試してください。

HTTP ステータスコード: 400

ConflictException

リクエストに矛盾が発生しました。リソースとの不一致を修正して、もう一度試してください。

HTTP ステータスコード: 400

InternalServerException

Amazon Kendra サービスに使用されている内部サーバーで問題が発生しました。数分後にもう一度お試しください。または、Support に問い合わせてください。

HTTP ステータスコード: 500

ResourceAlreadyExistException

使用したいリソースは既に存在します。正しいリソースを提供したことを確認して、もう一度お試しください。

HTTP ステータスコード: 400

ServiceQuotaExceededException

Amazon Kendra サービスに設定された制限を超えました。詳細についてはクォータを参照するか、制限の引き上げについてはSupport にお問い合わせください。

HTTP ステータスコード: 400

ThrottlingException

リクエストは、制限が必要なために実行が拒否されました。リクエストの数を減らして、もう一度お試しください。

HTTP ステータスコード: 400

ValidationException

入力が Amazon Kendra サービスによって設定された制約を満たしていません。正しい入力を入力して、もう一度お試しください。

HTTP ステータスコード: 400
