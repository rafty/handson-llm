クエリの提案

Amazon Kendraクエリ候補は、ユーザーが検索クエリをすばやく入力するのに役立ち、検索の指針となります。

Amazon Kendraクエリ履歴またはクエリログの一般的なクエリに基づいて、ユーザーに関連するクエリを提案します。 Amazon Kendraは、ユーザーが検索し、これらのクエリから学習したすべてのクエリを使用して、ユーザーに提案を行います。 Amazon Kendraユーザーがクエリの入力を開始したときに人気のあるクエリを提案します。 Amazon Kendraクエリのプレフィックスまたは最初の数文字が、ユーザーがクエリとして入力し始めた内容と一致する場合にクエリを提案します。

例えば、ユーザーが「upcoming events」というクエリの入力を開始したとします。 Amazon Kendraクエリログから、多くのユーザーが「今後のイベント 2050」を何度も検索していることを知りました。検索バーの真下に「upcoming events 2050」が表示され、検索クエリが自動的に完了されます。ユーザーは、「Upcoming events: What’s happening in 2050」というドキュメントである最初の検索結果を選択して、このクエリの提案を選択します。

Amazon Kendraユーザーに提案する適格なクエリを選択する方法を指定できます。例えば、クエリの提案が少なくとも 10 人のユニークユーザー (デフォルトは 3 人) によって検索され、過去 30 日以内に検索され、[block list] (ブロックリスト) の単語またはフレーズを含みません。 Amazon Kendraクエリには、少なくとも 1 つの検索結果があり、4 文字以上の単語が少なくとも 1 つ含まれている必要があります。

クエリの提案は、大文字と小文字が区別されません。 Amazon Kendraクエリプレフィックスと提案されたクエリを小文字に変換し、一重引用符と二重引用符をすべて無視し、複数の空白スペースを 1 つのスペースに置き換えます。 Amazon Kendra他のすべての特殊文字をそのまま照合します。 Amazon Kendraユーザーが 2 文字未満または 60 文字を超える場合、提案を表示しません。

GetQuerySuggestionsAPI を使用して、ユーザーに関連するクエリの提案を取得できます。クエリの提案は、追加料金なしでデフォルトで有効になっています。UpdateQuerySuggestionsConfigAPI を使用して、クエリの提案をいつでも無効にできます。検索アプリケーションに提案を適用する前に、次の 2 つの方法で設定をテストできます。

  • UpdateQuerySuggestionsConfigAPI を使用して。

  • コンソールの [Query suggestions settings] (クエリの提案の設定) で。

GetQuerySuggestionsAPI を使用して、クエリの提案をコンソールアプリケーションと統合して、ユーザーへの提案の表示を開始します。

クエリの提案の設定

UpdateQuerySuggestionsConfigAPI を使用して次の設定を編集できます。

  • モード —ENABLEDLEARN_ONLY クエリ候補はまたはのいずれかです。 Amazon Kendraデフォルトでクエリ候補を有効にします。 LEARN_ONLYクエリ候補をオフにします。無効にすると、Amazon Kendra引き続き提案を学習しますが、ユーザーにクエリの提案を表示しません。

  • クエリログのタイムウィンドウ — タイムウィンドウは、現在の日から過去の日までの日数の整数値です。

  • ユーザー情報のないクエリ —TRUE に設定し、すべてのクエリを含めるか、[block] (ユーザー情報) のクエリのみを含めます。FALSEこれは、ユーザーがクエリを発行したときに、検索アプリケーションにユーザー ID などのユーザー情報が含まれている場合に使用できます。この設定では、クエリに関連する特定のユーザー情報がない場合、デフォルトではクエリが除外されません。ただし、この設定は、ユーザー情報を含むクエリに基づいてのみ候補を表示する場合に限られます。

  • ユニークユーザー — ユーザーに提案する資格を得るために、クエリを検索する必要がある一意のユーザーの最小数。この数値は整数値です。

  • クエリ数 — クエリが、ユーザーに提案する資格を得るために検索される必要のある最小検索回数。この数値は整数値です。

これらの設定は、ユーザーに提案する一般的なクエリとしてクエリを選択する方法に影響します。設定の調整方法は、特定のニーズによって異なります。以下に例を挙げます。

  • 通常、ユーザーが月に 1 回平均で検索する場合、クエリログタイムウィンドウの日数を 30 に設定し、ユーザーの最近のクエリのほとんどをタイムウィンドウ内で古くなる前にキャプチャできるようにします。

  • 少数のクエリにユーザー情報が含まれており、小さなサンプルサイズに基づいてクエリを提案したくない場合は、すべてのユーザーを含むようにクエリを設定できます。

  • [popular queries] (人気のあるクエリ) を 10 人以上のユニークユーザーに 100 回以上検索されたものと定義する場合は、ユニークユーザーを 10、クエリ数を 100 に設定します。

設定の変更はすぐには反映されない場合があります。DescribeQuerySuggestionsConfigAPI を使用して設定の変更を追跡できます。更新された設定が有効になるまでの時間は、実行する更新とインデックス内の検索クエリの数によって異なります。

Console
クエリの提案が有効で準備ができていることを確認するには

  1. 左側のナビゲーションペインの [Indexes] (インデックス) でインデックスに移動し、[Enrichments] (エンリッチメント) を選択し、[Query suggestions] (クエリの提案) を選択します。

  2. [Query suggestions] (クエリの提案) ページで、[Query suggestions settings] (クエリの提案の設定) に移動し、以下の操作を行います。

    1. [Status] (ステータス) が [Enabled] (有効) であることを確認します。

    2. [Queries ready for suggestions] (提案の準備ができたクエリ) の数が 0 を超えていることを確認します。

クエリの提案の設定を編集するには

  1. 左側のナビゲーションペインの [Indexes] (インデックス) でインデックスに移動し、[Enrichments] (エンリッチメント) を選択し、[Query suggestions] (クエリの提案) を選択します。

  2. [Query suggestions] (クエリの提案) ページで、[Query suggestions settings] (クエリの提案の設定) に移動し、[Edit] (編集) を選択します。

提案をクリアするには

  1. 左側のナビゲーションペインの [Indexes] (インデックス) でインデックスに移動し、[Enrichments] (エンリッチメント) を選択し、[Query suggestions] (クエリの提案) を選択します。

  2. [Query suggestions] (クエリの提案) ページで、[Query suggestions settings] (クエリの提案の設定) に移動し、[Actions] (アクション) で [Clear suggestions] (提案のクリア) を選択します。

CLI

クエリの提案を取得するには

aws kendra get-query-suggestions \
--index-id index-id \
--query-text "query-text" \
--max-suggestions-count 1 // If you want to limit the number of suggestions.

クエリの提案を更新するには

例えば、クエリログタイムウィンドウと、クエリを検索する必要がある最小回数を変更するには、以下の操作を行います。

aws kendra update-query-suggestions-config \
--index-id index-id \
--query-log-look-back-window-in-days 30 \
--minimum-query-count 100
注記

更新された設定が有効になるまでの時間は、実行する更新とインデックス内の検索クエリの数によって異なります。

提案をクリアするには

aws  kendra clear-query-suggestions \
--index-id index-id \
注記

Amazon Kendraは、最後に提案をクリアした時点からクエリログに追加された新しいクエリに基づいて、新しい提案を学習します。

Python

クエリの提案を取得するには

import boto3
from botocore.exceptions import ClientError

kendra = boto3.client("kendra")

print("Get query suggestions.")

# Provide the index ID
index_id = "index-id"

# Provide the query text
query_text = "query"

try:
    query_suggestions_response = kendra.get_query_suggestions(
        IndexId = index_id,
        QueryText = query_text,
        MaxSuggestionsCount = 5
    )

    # Print out the suggestions you received
    if ("Suggestions" in query_suggestions_response.keys()) {
        for (suggestion: query_suggestions_response["Suggestions"]) {
            print(suggestion["Value"]["Text"]["Text"]);
        }
    }
   
except ClientError as e:
        print("%s" % e)

print("Program ends.")

例えば、クエリログタイムウィンドウと、クエリを検索する必要がある最小回数を変更するには、以下の操作を行います。

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Updating query suggestions settings/configuration for an index.")

# Provide the index ID
index_id = "index-id"

# Configure the settings you want to update
query_log_look_back_window_in_days = 30
minimum_query_count = 100

try:
    kendra.update_query_suggestions_config(
        IndexId = index_id,
        QueryLogLookBackWindowInDays = query_log_look_back_window_in_days,
        MinimumQueryCount = minimum_query_count
    )

    print("Wait for Amazon Kendra to update the query suggestions.")

    while True:
        # Get query suggestions description of settings/configuration
        query_sugg_config_response = kendra.describe_query_suggestions_config(
            IndexId = index_id
        )
        
        # If status is not UPDATING, then quit
        status = query_sugg_config_response["Status"]
        print(" Updating query suggestions config. Status: " + status)
        if status != "UPDATING":
            break
        time.sleep(60)

except ClientError as e:
        print("%s" % e)

print("Program ends.")

DescribeQuerySuggestionsConfigAPI を使用して現在の設定の確認を行います。さらに、この演算では、インデックスのクエリの提案に関する次の情報が表示されます。

  • モード —ENABLEDLEARN_ONLY クエリ候補はまたはのいずれかです。 Amazon Kendraデフォルトでクエリ候補を有効にします。 LEARN_ONLYクエリ候補をオフにします。UpdateQuerySuggestionsConfigAPI を使用してモードを変更できます。無効にすると、Amazon Kendra引き続き提案を学習しますが、ユーザーにクエリの提案を表示しません。

  • ステータス-ACTIVEUPDATING クエリ候補はまたはのいずれかです。

  • 提案数 — ユーザーに提案できるクエリの総数。数が予想よりはるかに少ない場合は、Amazon Kendra学習するためにはさらに多くのクエリを必要とするか、現在のクエリの提案設定が厳しすぎることが考えられます。

  • 提案の最終構築時刻 — 提案が最後に更新された時刻。 Amazon Kendra24 時間ごと、または設定を変更したり、ブロックリストを適用したりすると、候補が自動的に更新されます。

  • 提案の最終クリア時刻 — 提案が最後にクリアされた時刻。

特定のクエリの提案をブロックする

ブロックリストでは、Amazon Kendra特定の一般的なクエリがユーザーに提案されなくなります。これは、クエリの提案から除外する単語または語句のリストです。 Amazon Kendraブロックリスト内の単語または語句の完全一致を含むクエリを除外します。

ブロックリストを使用すると、クエリログに一般的に表示され、Amazon Kendra提案として選択できる不快な単語や語句から保護できます。ブロックリストを使用すると、一般公開または発表する準備ができていない情報を含む一般的なクエリが提案されないようにすることもできます。Amazon Kendra例えば、すぐにリリースする準備ができていないために提案したくない新製品のリリースをユーザーが一般的にクエリする場合、提案からの製品名と製品情報を含むクエリをブロックできます。

CreateQuerySuggestionsBlockListAPI を使用してクエリのブロックリストを作成できます。各単語または語句はテキストファイルで個別の行に配置します。次に、テキストファイルを S3 バケットにアップロードし、ファイルへのパスまたは場所を指定しますAmazon S3。 Amazon Kendra現在、ブロックリストを 1 つだけ作成しています。

Amazon S3バケット内のブロックされた単語や語句のテキストファイルを置き換えて、UpdateQuerySuggestionsBlockListAPI を使用してブロックリストを更新しますAmazon Kendra。

DescribeQuerySuggestionsBlockListAPI を使用して、ブロックリストのステータスや、ブロックリストが最後に更新された日時、現在のブロックリストに含まれる単語やフレーズの数、ブロックリストを作成する際に役立つエラーメッセージなどのその他の有用な情報を取得できます。ListQuerySuggestionsBlockListsAPI を使用して、インデックスに対するブロックリストの概要のリストを取得することもできます。

ブロックリストを削除するには、DeleteQuerySuggestionsBlockListAPI を使用します。

ブロックリストへの更新はすぐには反映されない場合があります。DescribeQuerySuggestionsBlockListAPI を使用して更新を追跡できます。

Python

ブロックリストを作成するには

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Create a query suggestions block list.")

# Provide a name for the block list
block_list_name = "block-list-name"
# Provide an optional description for the block list
block_list_description = "block-list-description"
# Provide the IAM role ARN required for query suggestions block lists
block_list_role_arn = "role-arn"

# Provide the index ID
index_id = "index-id"

s3_bucket_name = "bucket-name"
s3_key = "query-suggestions/block_list.txt"
source_s3_path = {
    'Bucket': s3_bucket_name,
    'Key': s3_key
}

try:
    block_list_response = kendra.create_query_suggestions_block_list(
        Description = block_list_description,
        Name = block_list_name,
        RoleArn = block_list_role_arn,
        IndexId = index_id,
        SourceS3Path = source_s3_path
    )

    print(block_list_response)

    block_list_id = block_list_response["Id"]

    print("Wait for Amazon Kendra to create the block list.")

    while True:
        # Get block list description
        block_list_description = kendra.describe_query_suggestions_block_list(
            Id = block_list_id,
            IndexId = index_id
        )
        # If status is not CREATING, then quit
        status = block_list_description["Status"]
        print("Creating block list. Status: " + status)
        if status != "CREATING":
            break
        time.sleep(60)
        
except ClientError as e:
        print("%s" % e)

print("Program ends.")

提案をクリアする

ClearQuerySuggestionsAPI を使用してクエリ候補を消去できます。提案をクリアすると、既存のクエリ提案のみが削除されます。クエリログ内のクエリは削除されません。提案をクリアした後、提案をクリアした時点からクエリログに追加された新しいクエリに基づいて、Amazon Kendra新しい提案を学習します。

提案がありません。

クエリの提案が表示されない場合は、次のいずれかの理由が考えられます。

  • Amazon Kendraインデックスに十分なクエリがありません。

  • クエリの提案の設定が厳しすぎるため、ほとんどのクエリが提案から除外されています。

  • 最近提案がクリアされ、Amazon Kendra新しい提案を学習するために新しいクエリが蓄積されるのにまだ時間が必要です。

DescribeQuerySuggestionsConfigAPI を使用して現在の設定の確認を行います。
