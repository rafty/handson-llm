DescribeIndex

既存の Amazon Kendra インデックスの情報を取得します。

リクエストの構文

{
   "Id": "string"
}

リクエストパラメータ

すべてのアクションに共通のパラメータの詳細については、「共通パラメータ」を参照してください。

リクエストは以下の JSON 形式のデータを受け入れます。

Id

情報を取得するインデックスの識別子。

型: 文字列

長さの制限: 最大長は 36 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9-]*

必須: はい

レスポンス要素

アクションが成功すると、サービスは HTTP 200 レスポンスを返します。

サービスから以下のデータが JSON 形式で返されます。

CapacityUnits

エンタープライズエディションインデックスの場合、アプリケーションのニーズを満たすように追加のキャパシティーを使用できます。これには、インデックスに使用されるキャパシティユニットが含まれます。クエリまたはドキュメントのキャパシティが 0 の場合は、インデックスがデフォルトキャパシティを使用していることを示します。インデックスのデフォルト容量と調整の詳細については、「容量の調整」を参照してください。

型: CapacityUnitsConfiguration オブジェクト

CreatedAt

インデックスが作成された時刻の Unix タイムスタンプ。

タイプ: Timestamp

Description

インデックスの説明。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,000 です。

Pattern: ^\P{C}*$

DocumentMetadataConfigurations

ドキュメントメタデータまたはフィールドの設定情報。ドキュメントメタデータは、ドキュメントに関連するフィールドまたは属性です。たとえば、各ドキュメントに関連付けられている会社の部署名。

型: DocumentMetadataConfiguration オブジェクトの配列

配列メンバー: 最小数は 0 項目です。最大数は 500 項目です。

Edition

インデックスに使用された Amazon Kendra エディション。インデックスを作成するときにエディションを決定します。

型: 文字列

有効な値: DEVELOPER_EDITION | ENTERPRISE_EDITION

ErrorMessage

Statusフィールド値がの場合FAILED、ErrorMessageフィールドには、その理由を説明するメッセージが含まれています。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 2,048 です。

パターン: ^\P{C}*$

Id

インデックスの識別子。

型: 文字列

長さの制限: 最大長は 36 です。

パターン: [a-zA-Z0-9][a-zA-Z0-9-]*

IndexStatistics

よくある質問の質問と回答の数、インデックス作成されたテキストドキュメントの数に関する情報を提供します。

型: IndexStatistics オブジェクト

Name

インデックスの名前。

型: 文字列

長さの制限: 最小長は 1 です。最大長は 1,000 です。

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

RoleArn

Amazon Kendra に Amazon Cloudwatch logs の書き込み許可を付与する IAM ロールの Amazon リソースネーム (ARN)。

型: 文字列

長さの制限: 最小長は 0 です。最大長は 1,284 です。

パターン: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

ServerSideEncryptionConfiguration

データの暗号化に使用されるAWS KMSカスタマーマスターキー (CMK) の識別子。Amazon Kendra は非対称 CMK をサポートしていません。

型: ServerSideEncryptionConfiguration オブジェクト

Status

インデックスの現在のステータス。値が ACTIVE の場合、インデックスは使用する準備が完了しています。Status フィールド値が FAILED の場合、ErrorMessage フィールドには、その理由を説明するメッセージが含まれています。

型: 文字列

有効な値: CREATING | ACTIVE | DELETING | FAILED | UPDATING | SYSTEM_UPDATING

UpdatedAt

インデックスが最終更新された時刻の Unix。

タイプ: Timestamp

UserContextPolicy

Amazon Kendra インデックスのユーザーコンテキストポリシー。

型: 文字列

有効な値: ATTRIBUTE_FILTER | USER_TOKEN

UserGroupResolutionConfiguration

AWS IAM Identity Center (successor to AWS Single Sign-On)アイデンティティソースからグループおよびユーザーのアクセスレベルを取得するための設定を有効にしたかどうか。

型: UserGroupResolutionConfiguration オブジェクト

UserTokenConfigurations

Amazon Kendra インデックスのユーザートークン設定。

型: UserTokenConfiguration オブジェクトの配列

配列メンバー: 最大数は 1 項目です。

エラー

すべてのアクションに共通のエラーについては、「一般的なエラー」を参照してください。

AccessDeniedException

このアクションを実行する十分なアクセス権限がありません。必要な権限ポリシーとユーザーアカウントがあることを確認して、もう一度試してください。

HTTP ステータスコード: 400

InternalServerException

Amazon Kendra サービスに使用されている内部サーバーで問題が発生しました。数分後にもう一度お試しださい。または、Support に問い合わせてください。

HTTP ステータスコード: 500

ResourceNotFoundException

使用したいリソースは存在しません。正しいリソースが提供されていることを確認して、もう一度お試しください。

HTTP ステータスコード: 400

ThrottlingException

リクエストは、制限が必要なために実行が拒否されました。リクエストの数を減らして、もう一度お試しください。

HTTP ステータスコード: 400

ValidationException

入力が Amazon Kendra サービスで設定された制約を満たしていません。正しい入力を入力して、もう一度お試しください。

HTTP ステータスコード: 400
