Amazon Kendraウェブクローラー

Amazon KendraWeb クローラーを使用して Web ページをクロールしたり、インデックスを作成したりできます。

クロールできるのは、一般公開ウェブサイトおよびセキュア通信プロトコル、ハイパーテキスト転送プロトコルセキュア (HTTPS) を使用するウェブサイトのみです。ウェブサイトをクロールするときにエラーが発生した場合は、ウェブサイトのクロールがブロックされている可能性があります。内部 Web サイトをクロールするには、Web プロキシを設定できます。Web プロキシは公開されている必要があります。

インデックス作成するウェブサイトを選択するときは、Amazon 利用規定ポリシーおよびその他の Amazon 規約のすべてに準拠している必要があります。Amazon Kendraウェブクローラーのみを使用して、独自のウェブページ、またはインデックス作成の認可があるウェブページのインデックス作成を行う必要があることに注意してください。Amazon Kendraウェブクローラーによるウェブサイトのインデックス作成を停止する方法については、を参照してくださいAmazon KendraWebrobots.txt クローラー用のファイルの設定。

接続手順

WebAmazon Kendra クローラーデータソースに接続するには、Amazon Kendraデータにアクセスできるように Web クローラーデータソースの必要な詳細を入力する必要があります。Web クローラをまだ設定していない場合は、Amazon Kendraを参照してください前提条件。

Console

WebAmazon Kendra クローラーに接続するには

  1. AWS マネジメントコンソールにサインインし、Amazon Kendra コンソールを開きます。

  2. 左側のナビゲーションペインで、[インデックス] を選択し、インデックスのリストから使用するインデックスを選択します。

    注記

    ユーザーアクセス制御設定は、インデックス設定で設定または編集できます。

  3. 「はじめに」ページで、「データソースを追加」を選択します。

  4. [データソースの追加] ページで [Web クローラコネクタ] を選択し、[データソースの追加] を選択します。

  5. [データソースの詳細の指定] ページで、次の情報を入力します。

    1. [名前と説明] の [データソース名] に、データソースの名前を入力します。ハイフンは使用できますが、スペースは含められません。

    2. (オプション) 説明-オプションとしてデータソースの説明を入力します。

    3. 「言語」(「デフォルト言語」)-索引用にドキュメントをフィルタリングする言語。特に指定しない限り、言語はデフォルトで英語に設定されます。メタデータで指定された言語は、選択した言語よりも優先されます。

    4. [タグ] の [新しいタグの追加] の場合、リソースを検索してフィルタリングしたり、AWSコストを追跡したりできます。

    5. [Next] (次へ) を選択します。

  6. 「アクセスとセキュリティの定義」ページで、次の情報を入力します。

    1. [ソース] では、ユースケースに応じて [ソース URL] と [ソースサイトマップ] のいずれかを選択し、それぞれの値を入力します。

      注記

      最大 10 個のソース URL および 3 つのサイトマップ 3 つのサイトマップを持つことができます。

      サイトマップをクロールする場合は、ベース URL またはルート URL が、サイトマップページに表示されている URL に使用されているベース URL と同じかどうかを確認することが重要です。たとえば、サイトマップ URL が https://example.com/sitemap-page.html の場合、このサイトマップページに表示されている URL にもベース URL「https://example.com/」を使用する必要があります。

    2. (オプション) Web プロキシの場合 — 次の情報を入力します。

      1. ホスト名 — Web プロキシが必要なホスト名。

      2. ポート番号-ホスト URL トランスポートプロトコルが使用するポート。ポート番号は 0 から 65535 までの数値でなければなりません。

      3. Web プロキシ認証情報の場合-既存のシークレットを選択するか、新しいシークレットを作成して Web クローラ認証情報を保存します。新しいシークレットを作成することを選択すると、AWSSecrets Managerシークレットウィンドウが開きます。

      4. 「AWSSecrets Managerシークレットの作成」ウィンドウに次の情報を入力します。

        1. シークレット名 — シークレットの名前。シークレット名には、プレフィックスWebCrawler「AmazonKendra--」が自動的に追加されます。

        2. [ユーザー名] と [パスワード] に、Web サイトの基本認証情報を入力します。

        3. [Save] (保存) を選択します。

    3. （オプション）認証のあるホスト：選択すると、認証されたホストをさらに追加できます。

    4. IAM ロール — 既存の IAM ロールを選択するか、リポジトリ認証情報にアクセスしてコンテンツをインデックス作成する必要があります。

      注記

      インデックスに使用される IAM ロールはデータソースには使用できません。既存のロールがインデックスや FAQ に使用されているかどうかわからない場合は、エラーを避けるため [新しいロールを作成する] を選択してください。

    5. [Next] (次へ) を選択します。

  7. [同期設定の構成] ページで、次の情報を入力します。

    1. クロール範囲-クロールする Web ページの種類を選択します。

    2. クロールの深度 —Amazon Kendra クロールするレベルをシード URL から選択します。

    3. クロールの詳細設定と追加構成では、次の情報を入力します。

      1. 最大ファイルサイズ-クロールする Web ページまたは添付ファイルの最大サイズ。最小 0.000001 メガバイト (1 バイト)。最大最大50 MB。

      2. ページあたりの最大リンク数 — 1 ページあたりにクロールされるリンクの最大数。リンクは、表示順にクロールされます。1ページあたり最低1リンク。1ページあたり最大1000リンク。

      3. 最大スロットリング — 1 分あたりホスト名 1 分あたり最大数。1 分あたり最低 1 URL /ホスト名。1 分あたり最大 300 URL /ホスト名。

      4. 正規表現パターン — 特定の URL のみを含めたり、リポジトリ内の URL、ファイルタイプ、特定のファイルを除外したりするパターンを追加します。合計で 100 個のパターンを作成できます。

    4. [同期実行スケジュール] の [頻度] で、Amazon Kendraデータソースと同期する頻度を選択します。

    5. [Next] (次へ) を選択します。

  8. [確認と作成] ページで、入力した情報が正しいことを確認し、[データソースの追加] を選択します。また、このページで情報の編集を選択することもできます。データソースが正常に追加されると、データソースページに表示されます。

API

WebAmazon Kendra クローラーに接続するには

次の WebCrawlerConfigurationAPI を指定する必要があります。

  • URL — ウェブサイトのシードまたは開始ポイント URL またはサイトマップ URL を指定しますSiteMapsConfiguration。SeedUrlConfiguration

    サイトマップをクロールする場合は、ベース URL またはルート URL が、サイトマップページに表示されている URL に使用されているベース URL と同じかどうかを確認することが重要です。たとえば、サイトマップ URL が https://example.com/sitemap-page.html の場合、このサイトマップページに表示されている URL にもベース URL「https://example.com/」を使用する必要があります。

  • シークレットの Amazon リソースネーム (ARN) — ウェブサイトで基本認証情報が保存されるのホスト名名名名名名名名およびパスワードの基本認証情報が保存されるのシークレットを提供します。AuthenticationConfigurationAPI を使用して ARN を指定します。シークレットは、次のキーを持つ JSON 構造で保存される。

    {
    "username": "user name",
    "password": "password"
}

    シークレットを使用してウェブプロキシ認証情報を指定できます。AWSSecrets ManagerProxyConfigurationAPI を使用してウェブサイトのホスト名名名名名名名名名名名名およびユーザー名の認証情報が保存されるのシークレットを提供します。

    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

オプションで、次の機能を追加することもできます。

  • クロールモード-Web サイトのホスト名のみをクロールするか、サブドメインを含むホスト名をクロールするか、Web ページがリンクしている他のドメインをクロールするかを選択します。

  • ウェブサイト内の「深さ」またはウェブサイト内のシードレベルからクロールするレベルの数。例えば、ウェブサイトにインデックスレベル (この例ではシードレベル)、セクションレベル、サブセクションレベルという 3 つのレベルがあり、インデックスレベルからセクションレベル (つまりレベル 0〜1) までの情報のクロールだけにしたい場合は、深さを 1 に設定できます。

  • 単一のウェブページ上でクロールする URL の最大数。

  • クロールする Web ページの最大サイズ (MB 単位)。

  • 1 分あたりウェブサイトホストごとにクロールされる URL の最大数。

  • 内部ウェブサイトに接続してクロールするウェブプロキシ情報。

  • ユーザー認証を必要とするウェブサイトにアクセスしてクロールするための認証情報。

  • カスタムドキュメントエンリッチメントツールを使用して、HTML メタタグをフィールドとして抽出できます。詳細については、取り込みプロセス中のドキュメントのメタデータのカスタマイズを参照してください。HTML メタタグを抽出する例については、CDE の例を参照してください。

  • 包含フィルターと除外フィルター — 特定の URL を含めるまたは除外するかどうかを指定します。

    注記

    ほとんどのデータソースは正規表現パターンを使用します。正規表現パターンとは、フィルターと呼ばれる包含パターンまたは除外パターンです。包含フィルターを指定すると、包含フィルターと一致するコンテンツのみが索引付けされます。包含フィルターに一致しないドキュメントはインデックスに登録されません。包含フィルターと除外フィルターを指定した場合、除外フィルターに一致するドキュメントは、包含フィルターと一致してもインデックス作成されません。
