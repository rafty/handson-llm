Amazon S3ドキュメントメタデータ

メタデータファイルを使用して、メタデータ、Amazon S3つまりドキュメントに関する追加情報をバケット内のドキュメントに追加できます。各メタデータファイルは、インデックス作成されたドキュメントに関連付けられます。

メタデータファイルは、インデックス作成されたファイルと同じバケットに保存する必要があります。コンソールを使用して、Amazon S3またはデータソースを作成するときの、S3PrefixDocumentsMetadataConfigurationコンソールまたはパラメータフィールドを使用して、メタデータファイルのバケット内の場所を指定できます。Amazon S3プレフィックスを指定しない場合、メタデータファイルはインデックス作成されたドキュメントと同じ場所に保存する必要があります。

Amazon S3メタデータファイルのプレフィックスを指定すると、インデックス作成されたドキュメントとparallel するディレクトリ構造内に格納されます。 Amazon Kendra指定したディレクトリ内でのみメタデータを検索します。メタデータが読み込まれない場合は、ディレクトリの場所がメタデータの場所と一致していることをチェックします。

次の例は、インデックス作成されたドキュメントの場所がメタデータファイルの場所にどのようにマッピングされるかを示しています。Amazon S3ドキュメントのキーは、Amazon S3メタデータのプレフィックスに追加され、.metadata.jsonAmazon S3サフィックスが付加されメタデータファイルのパスを形成することに注意してください。Amazon S3Amazon S3.metadata.jsonキーとメタデータのプレフィックスとサフィックスを組み合わせたものは、合計 1,024 文字以下にする必要があります。Amazon S3キーとプレフィックスとサフィックスを組み合わせる場合は、追加文字を考慮して 1000 文字未満にしておくことをお勧めします。

Bucket name:
     s3://bucketName
Document path:
     documents
Metadata path:
     none
File mapping
     s3://bucketName/documents/file.txt -> 
        s3://bucketName/documents/file.txt.metadata.json

ドキュメントのメタデータは JSON ファイルで定義されます。ファイルは、BOM マーカーの無い UTF-8 テキストファイルである必要があります。JSON ファイルのファイル名はでなければなりません<document>.<extension>.metadata.json。この例では、「document」はメタデータが適用されるドキュメントの名前で、「extension」はドキュメントのファイル拡張子です。ドキュメント ID は内で一意である必要があります<document>.<extension>.metadata.json。

JSON ファイルの内容はこのテンプレートに従います。すべての属性はオプションです。を指定しない場合_source_uri、Amazon Kendra検索結果で返されるリンクは、Amazon S3ドキュメントが含まれているバケットを参照します。

{
    "DocumentId": "document ID",
    "Attributes": {
        "_category": "document category",
        "_created_at": "ISO 8601 encoded string",
        "_last_updated_at": "ISO 8601 encoded string",
        "_source_uri": "document URI",
        "_version": "file version",
        "_view_count": number of times document has been viewed,
        "custom attribute key": "custom attribute value",
        additional custom attributes
    },
    "AccessControlList": [
         {
             "Name": "user name",
             "Type": "GROUP | USER",
             "Access": "ALLOW | DENY"
         }
    ],
    "Title": "document title",
    "ContentType": "For example HTML | PDF. For supported content types, see Types of documents."
}

_created_at および _last_updated_at メタデータフィールドは ISO 8601 でエンコードされた日付です。例えば、201:20 25T 12:20:20:30:00:00 は、中央ヨーロッパ時間の 2012 年 12 時 30 分 (プラス 10 秒) の ISO 8601 の日付/時刻形式です。

クエリをフィルタリングしたり、クエリ応答をグループ化したりするために使用するドキュメントに関する Attributes フィールドに追加情報を追加できます。詳細については、「カスタムドキュメントフィールドの作成」を参照してください。

AccessControlListフィールドを使用して、クエリからの応答をフィルタリングできます。これにより、特定のユーザーとグループのみがドキュメントにアクセスできます。詳細については、「ユーザーコンテキストでのフィルタリング」を参照してください。
