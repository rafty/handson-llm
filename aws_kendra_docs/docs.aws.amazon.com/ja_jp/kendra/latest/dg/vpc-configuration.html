VPCAmazon Kendra を使用するように設定する

Amazon KendraAmazon Virtual Private Cloudに接続して、プライベートクラウドで実行されているデータソースまたはデータベースに保存されているコンテンツをインデックス作成できます。データソースコネクタまたはデータベースコネクタを作成する場合、データソースまたはデータベースを含むサブネットのセキュリティグループとサブネット識別子を指定します。 Amazon Kendraこの情報を使用して、Elastic Network Interface を作成します。

データソースまたはデータベースがで実行されていない場合はAmazon VPC、バーチャルプライベートネットワーク (VPN)Amazon VPC を使用して、データソースまたはデータベースをに接続できます。Amazon アカウントを作成すると、デフォルトの VPC が取得されます。VPN の設定の詳細については、AWS VPNドキュメントを参照してください。

VPC を使用するには、データベースが属するサブネットの識別子と、サブネットにアクセスするために使用する必要があるセキュリティグループの識別子と、データベースにアクセスするために使用する必要があるセキュリティグループの識別子を、データベースが属するサブネットの識別子と、データベースが属するサブネットの識別子と、データベースが属するサブネットの識別子と、Amazon Kendraサブネットにアクセスするために使用する必要があるセキュリティグループの識別子Amazon Kendra 例えば、MySQL データベースのデフォルトポートを使用している場合、セキュリティグループで MySQL データベースを実行するホストのポート 3306Amazon Kendra へのアクセスを許可する必要があります。

プライベートサブネットは、データソースまたはデータベースの VPC 設定でのみ使用します。RDS インスタンスが VPC のパブリックサブネットにある場合、そのサブネットを直接使用してデータソースやデータベースを同期することはできません。代わりに、パブリックサブネット内の NAT ゲートウェイへのアウトバウンドアクセス権を持つプライベートサブネットを作成します。VPC をデータソースまたはデータベースに設定する場合、そのプライベートサブネットを指定します。VPC で設定されたデータベースデータソースの場合、サブネットは次のいずれかのアベイラビリティーゾーン ID にある必要があります。

  • 米国西部 (オレゴン) — usw2-az1、usw2-az2、usw2-az3

  • 米国東部 (バージニア北部)-use1-az1、use1-az2、use1-az4

  • 欧州 (アイルランド) — euw1-az1、uew1-az2、euw1-az3

サブネットとセキュリティグループの識別子は、Amazon VPCコントロールパネルで設定します。識別子を表示するには、Amazon VPC次のようにコンソールを開きます。

サブネット識別子を表示するには

  1. AWS Management Console にサインインして、Amazon VPC コンソール (https://console.aws.amazon.com/vpc/) を開きます。

  2. ナビゲーションペインで、[Subnets] (サブネット) を選択します。

  3. サブネットリストから、データベースサーバーを含むサブネットを選択します。

  4. [説明] タブで、[Subnet ID] (サブネット ID) フィールドに表示される識別子を書き留めます。

セキュリティグループ識別子を表示するには

  1. AWS Management Console にサインインして、Amazon VPC コンソール (https://console.aws.amazon.com/vpc/) を開きます。

  2. ナビゲーションペインで、[Security Groups] を選択します。

  3. セキュリティグループリストで、識別子の対象となるグループを選択します。

  4. [説明] タブで、[Group ID] (グループ ID) フィールドに表示される識別子を書き留めます。

2Amazon Kendra つ以上のサブネット間の接続をルーティングする必要がある場合は、複数のサブネットを指定できます。例えば、データベースサーバーを含むサブネットが IP アドレス外である場合、空き IP アドレスを持つサブネットに接続し、Amazon Kendra接続を最初のサブネットにルーティングできます。複数のサブネットを一覧表示する場合、サブネットは相互に通信できる必要があります。各サブネットは、ネットワークアドレス変換 (NAT) デバイスを使用してアウトバウンドインターネットアクセスを提供するルートテーブルに関連付ける必要があります。

また、複数のセキュリティグループを指定することもできます。Amazon Kendraセキュリティグループの組み合わせにより、接続設定で指定したデータソースまたはデータベースサーバーにアクセスできるようになります。

ステップ 1: VPC を設定する

プライベートサブネットと、サブネットで実行されている MySQLAmazon Kendra データベースにアクセスするためのセキュリティグループを持つように VPC を設定してください。VPC 設定で提供されるサブネットは、米国西部 (オレゴン)、米国東部 (バージニア北部)、欧州 (アイルランド) のいずれかにある必要があります。

VPC を設定するには

  1. AWS Management Console にサインインして、Amazon VPC コンソール (https://console.aws.amazon.com/vpc/) を開きます。

  2. ナビゲーションペインで [Route tables] (ルートテーブル) を選択して、[Create route table] (ルートテーブルの作成) を選択します。

  3. [Name tag] (名前タグ) フィールドに Private subnet route table と入力します。[VPC field] (VPC フィールド) で、VPC を選択し、[Create] (作成) をクリックします。[Close] (閉じる) をクリックして、ルートテーブルのリストに戻ります。

  4. ナビゲーションペインで、[NAT Gateways] (NAT ゲートウェイ)、[Create NAT Gateway] (NAT ゲートウェイの作成) の順に選択します。

  5. 「サブネット」フィールドで、パブリックサブネットであるサブネットを選択し、サブネット ID を書き留めます。

  6. Elastic IP アドレスを持っていない場合は、[Create New EIP] (新しい EIP の作成) で、[Create a NAT Gateway] (NAT ゲートウェイの作成) を選択してから、[Close] (閉じる) をクリックします。

  7. ナビゲーションペインで、[Route Tables] (ルートテーブル) を選択します。

  8. ルートテーブルリストから、ステップ 3 で作成した [Private subnet route table] (プライベートサブネットルートテーブル) を選択します。[Actions] (アクション) で、[Edit Routes] (ルートの編集) を選択します。

  9. [Add Rule] (ルートの追加) を選択します。送信先 0.0.0.0/0 を追加して、インターネットへのすべての発信トラフィックを許可します。[ターゲット] で [NAT ゲートウェイ] を選択し、ステップ 4 で作成したゲートウェイを選択します。[Save routes] (ルートの保存) を選択し、次に、[Close] (閉じる) をクリックします。

  10. [Actions] (アクション) から、[Edit subnet associations] (サブネットの関連付けの編集) を選択します。

  11. プライベートにするサブネットを選択します。以前に書き留めた NAT ゲートウェイを持つサブネットを選択しないでください。

ステップ 2: セキュリティを設定する

次に、データベースのセキュリティグループを設定します。

セキュリティグループを作成するには

  1. AWS Management Console にサインインして、Amazon VPC コンソール (https://console.aws.amazon.com/vpc/) を開きます。

  2. VPC の説明から、IPv4 CIDR を書き留めます。

  3. ナビゲーションペインで、[Security Groups] (セキュリティグループ) を選択して、[Create security group] (セキュリティグループの作成) を選択します。

  4. [Security group name] (セキュリティグループ名) に DataSourceInboundSecurityGroup と入力します。説明を入力し、リストから VPC を選択します。[Create] (作成) をクリックしてから、[Close] (閉じる) をクリックします。

  5. [Inbound] (インバウンド) タブを選択します。

  6. [Edit rules] (ルールの編集)、[Add Rule] (ルールの追加) の順に選択します。

  7. データベースの場合は、[Port Range] (ポート範囲) のポート番号を入力します。たとえば、MySQL の場合はで3306、HTTPS の場合はです443。[Source] (ソース) に、VPC のクラスレスドメイン間ルーティング (CIDR) を入力します。[Save rules] (ルールの保存) を選択し、[Close] (閉じる) をクリックします。

セキュリティグループでは、VPC 内のすべてのユーザーがデータベースに接続でき、インターネットへのアウトバウンド接続が許可されます。
