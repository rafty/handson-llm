Amazon Kendra の概要

Amazon Kendraは、ユーザーが自然言語を使用して高度な検索アルゴリズムデータを検索できるようにする、高精度のインテリジェント検索サービスです。質問に対する特定の回答が返され、ユーザーはヒトの専門家とのやりとりに近いエクスペリエンスを提供します。非常にスケーラブルで、パフォーマンス要求を満たすことができ、などの他のAWSAmazon S3のサービスと緊密に統合されAmazon Lex、エンタープライズレベルのセキュリティを提供します。

Amazon Kendraユーザーは、次のタイプの質問を尋ねたり、クエリすることができます。

  • Factoid 型の質問 — 誰が、何を、いつ、どこでの単純な質問。例:シアトルに最も近いサービスセンターはどこですか。 Factoid 型の質問には、1 単語または 1 語句で返すことができる事実ベースの回答があります。回答は、よくある質問または索引付きドキュメントから取得されます。

  • 説明的な質問 — 1 文、1 節、またはドキュメント全体が答えとなる質問。例えば、「Edition」をネットワークに接続するにはどうしたらいいですか。 または、低所得世帯の税制上の優遇措置を受けるにはどうすればよいですか？ 。

  • キーワード検索と自然言語検索 — 検索するキーワードだけでなく、複雑な会話形式の質問も含まれる質問。これには、意図した意味が必ずしも明確ではない質問も含まれます。例えば、キーノートのアドレス。「アドレス」にはいくつかの意味があることが多いため、Amazon Kendra検索クエリの背後にあるユーザーの意図した意味を推測して、関連する情報を返すことができます。 Amazon Kendra深層学習モデルを使用してこの種のクエリを処理します。

Amazon Kendra の利点

Amazon Kendraには次の利点があります。

  • 精度 — キーワード検索を使用する従来の検索サービスとは異なり、基本的なキーワードの照合とランキングに基づいて結果が表示されるため、Amazon Kendra質問の内容を把握しようとします。 Amazon Kendraデータ全体を検索し、従来の検索を超えて、クエリに最も関連性の高い単dition、またはドキュメントを返します。 Amazon Kendra機械学習を使用して検索結果を改善します。

  • シンプル — 検索したいドキュメントを管理するためのコンソールと APIAmazon Kendra を提供します。シンプルな検索 API を使用して、WebAmazon Kendra サイトやモバイルアプリケーションなどのクライアントアプリケーションに統合できます。

  • 接続 — サードパーティのデータリポジトリや MicrosoftAmazon Kendra などのデータソースに接続できます SharePoint。データソースを使用して、ドキュメントを簡単にインデックス化して検索できます。

  • ユーザーアクセス制御 —Amazon Kendra 検索アプリケーションに対して非常に安全なエンタープライズ検索を提供します。検索結果には、組織のセキュリティモデルが反映されます。検索結果は、ユーザーまたはグループのドキュメントへのアクセスに基づいてフィルタリングできます。お客様は、ユーザーのアクセスを認証し、認可する責任を負うものとします。

Amazon KendraDeveloper dition

Amazon Kendraデベロッパーエディションは、Amazon Kendraのすべての機能を低コストで提供します。これには、750 時間の使用を提供する無料利用枠が含まれています。デベロッパーエディションは、Amazon Kendraドキュメントのインデックス作成方法を探ったり、機能を試したり、を使用するアプリケーションを開発したりするのに理想的ですAmazon Kendra。

デベロッパーエディションには、次のものが用意されています。

  • 最大 5 つのインデックスで、それぞれ最大 5 つのデータソース。

  • 10,000 ドキュメントまたは 3 GB の抽出されたテキスト。

  • 1 日あたり約 4,000 クエリまたは 1 秒あたり 0.05 クエリ。

  • 1 つのアベイラビリティーゾーン (AZ) で実行。アベイラビリティーゾーン (AWSリージョンのデータセンター) を参照してください。

デベロッパーエディションを本稼働アプリケーションに使用しないでください。デベロッパーエディションでは、レイテンシーや可用性の保証はありません。

Amazon Kendraエンタープライズdition
