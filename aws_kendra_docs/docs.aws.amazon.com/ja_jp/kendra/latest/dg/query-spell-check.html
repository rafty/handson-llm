クエリのスペルチェッカー

Amazon Kendraスペルチェッカーは、クエリのスペル修正を提案します。これにより、検索結果が 0 件出現する回数を最小限に抑え、関連性の高い結果を返すことができます。スペルを間違えたクエリの検索結果がまったく表示されず、一致する結果が得られなかったり、ドキュメントが返されなかったりする場合があります。また、ユーザーがスペルミスのあるクエリから無関係な検索結果を受け取る可能性があります。

スペルチェックは、索引付けされた文書に含まれる単語と、修正された単語がスペルミスのある単語にどれだけ近いかに基づいて、スペルミスのある単語の修正を提案するように設計されています。たとえば、索引付けされたドキュメントに「ステートメント」という単語が含まれている場合、これは「年度末財務諸表」クエリのスペルミスのある「ステートメント」という単語とほぼ一致する可能性があります。

スペルチェッカーは、元のクエリテキスト内のスペルミスのある単語を置き換える、意図した単語または修正された単語を返します。たとえば、「kendre search を展開する」と「Kendra search を展開中」が返されることがあります。API で提供されるオフセット位置を使用して、フロントエンドアプリケーションのクエリで返された修正された単語を強調表示または斜体にすることもできます。コンソールでは、修正された単語はデフォルトで強調表示または斜体で表示されます。たとえば、「Kendra 検索のデプロイ」などです。

インデックスされたドキュメントにビジネス固有の用語や専門用語が含まれている場合、スペルチェッカーはこれらの用語をクエリ内のスペルミスとして誤解しません。たとえば、「アマゾンメイシー」は「アマゾンメイス」に修正されません。

「年末」のようにハイフンでつながった単語の場合、スペルチェッカーはこれらを個別の単語として扱い、それらの単語の修正を提案します。たとえば、「年度末」の修正案は「年度末」かもしれません。

DOCUMENTQUESTION_ANSWERおよびクエリの応答タイプについては、スペルチェックは文書本文の単語に基づいてスペルミスのある単語の修正を提案します。スペルミスのある単語とほぼ一致する修正を提案する点で、文書本文の方がタイトルよりも信頼できます。ANSWERクエリ応答タイプの場合、スペルチェッカーは索引内のデフォルトの質問と回答の文書に含まれる単語に基づいて修正を提案します。

SpellCorrectionConfigurationオブジェクトを使用してスペルチェッカーを起動できます。IncludeQuerySpellCheckSuggestionsTRUEに設定しました。スペルチェッカーは、コンソールでデフォルトで有効になっています。デフォルトではコンソールに組み込まれています。

スペルチェッカーは、英語だけでなく複数の言語でのクエリのスペル修正を提案することもできます。スペルチェッカーでサポートされている言語のリストについては、Amazon Kendraサポートされている言語を参照してください。

デフォルトの制限でのクエリスペルチェッカーの使用

スペルチェッカーは、特定のデフォルトまたは制限を設定して設計されています。スペル修正候補を有効にした場合に適用される現在の制限の一覧を以下に示します。

  • 3 文字未満または長さが 30 文字を超える単語については、推奨スペル修正を返すことはできません。30 文字以上または 3 文字未満を入力するには、Support にお問い合わせください。

  • 推奨スペル修正では、ユーザーアクセス制御やユーザーコンテキストフィルタリング用のアクセス制御リストに基づいて候補を制限することはできません。スペル修正は、その単語が特定のユーザーに限定されているかどうかに関係なく、インデックスされたドキュメント内のすべての単語に基づいて行われます。クエリのスペル修正候補に特定の単語が表示されないようにするには、有効にしないでくださいSpellCorrectionConfiguration。

  • 数字を含む単語のスペル修正候補を返すことはできません。たとえば、「2 が br8k ubun2 でない」とします。

  • 推奨スペル修正では、索引付き文書に含まれていない単語は使用できません。

  • 推奨スペル修正では、索引付き文書に出現頻度が 0.01 パーセント未満の単語は使用できません。0.01% のしきい値を変更するには、Support にお問い合わせください。
