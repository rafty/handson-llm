Gメール

GmailはGoogleが開発したメールクライアントで、ファイルを添付したメールメッセージを送信できます。Gmail のメッセージは、フォルダーとラベルを使用してメールの受信トレイ内で並べ替えて保存できます。Amazon Kendraを使用して、メールメッセージと添付ファイルのインデックスを作成できます。また、特定のメールメッセージ、メッセージの添付ファイル、Amazon Kendraおよびラベルをインデックスに含めたり除外したりするように構成することもできます。

Amazon Kendraコンソールと TemplateConfigurationAPI を使用して GmailAmazon Kendra データソースに接続できます。

前提条件

Amazon Kendraを使用して Gmail データソースのインデックスを作成する前に、GmailAWS とアカウントでこれらの変更を行います。

Gmail では、次のものが揃っていることを確認してください。

  • Google クラウドプラットフォームの管理者アカウントを作成し、Google Cloud プロジェクトを作成しました。

  • 管理者アカウントで Gmail API と管理者 SDK API を有効にしました。

  • サービスアカウントを作成し、Gmail の JSON プライベートキーをダウンロードしました。プライベートキーを作成してアクセスする方法については、Google Cloud ドキュメントの「サービスアカウントキーの作成方法」と「サービスアカウント認証情報」を参照してください。

  • 管理者アカウントの電子メール、サービスアカウントの電子メール、および認証に使用する秘密鍵をコピーしました。

  • ユーザーとインデックスを作成する共有ディレクトリ用に、次の OAuth スコープ (管理者ロールを使用) を追加しました。

    • https://www.googleapis.com/auth/admin.directory.user.readonly

    • https://www.googleapis.com/auth/gmail.readonly

  • Gmail や、同じインデックスに使用する予定の他のデータソースでは、各ドキュメントがユニークであることを確認しました。インデックスに使用する各データソースには、データソース間で同じドキュメントが含まれていてはなりません。ドキュメント ID はインデックス全体に適用され、インデックスごとに一意でなければなりません。

AWSアカウントには、次のものがあることを確認してください。

  • Amazon Kendraインデックスを作成し、API を使用している場合は、インデックス ID を記録しました。

  • IAMデータソースのロールを作成し、API を使用している場合は、IAMロールの ARN を記録しました。

  • GmailAWSSecrets Manager の認証情報をシークレットに保存し、API を使用している場合は、シークレットの ARN を記録しました。

    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

IAM既存のロールまたはシークレットがない場合は、Gmail データソースをに接続するときに、IAMSecrets Managerコンソールを使用して新しいロールとシークレットを作成できますAmazon Kendra。API を使用している場合は、IAMSecrets Manager既存のロールとシークレットの ARN とインデックス ID を指定する必要があります。

接続手順

GmailAmazon Kendra データソースに接続するには、Amazon Kendraデータにアクセスできるように Gmail の認証情報の詳細を入力する必要があります。Gmail をまだ設定していない場合はAmazon Kendra、を参照してください前提条件。

Console

GmailAmazon Kendra に接続するには

  1. AWS マネジメントコンソールにサインインし、Amazon Kendra コンソールを開きます。

  2. 左側のナビゲーションペインで、[インデックス] を選択し、インデックスのリストから使用するインデックスを選択します。

    注記

    ユーザーアクセス制御設定は、インデックス設定で設定または編集できます。

  3. 「はじめに」ページで、「データソースを追加」を選択します。

  4. [データソースの追加] ページで [Gmail コネクタ] を選択し、[データソースの追加] を選択します。

  5. [データソースの詳細を指定] ページで、次の情報を入力します。

    1. [名前と説明] の [データソース名] に、データソースの名前を入力します。ハイフンは使用できますが、スペースは含められません。

    2. (オプション) 説明-データソースの説明を入力します。

    3. 「言語」(「デフォルト言語」)-索引用にドキュメントをフィルタリングする言語。特に指定しない限り、言語はデフォルトで英語に設定されます。メタデータで指定された言語は、選択した言語よりも優先されます。

    4. [タグ] の [新しいタグの追加] で、リソースを検索してフィルタリングしたり、AWSコストを追跡したりできます。

    5. [Next] (次へ) を選択します。

  6. 「アクセスとセキュリティの定義」ページで、次の情報を入力します。

    1. AWSSecrets Managerシークレット認証で-既存のシークレットを選択するか、GmailSecrets Manager の認証情報を保存する新しいシークレットを作成します。新しいシークレットを作成することを選択すると、AWSSecrets Managerシークレットウィンドウが開きます。

      1. 「AWSSecrets Managerシークレットの作成」ウィンドウに次の情報を入力します。

        1. シークレット名 — シークレットの名前。

        2. クライアントメール — Google サービスアカウントからコピーしたクライアントメール。

        3. 管理者アカウントの電子メール — 使用したい管理者アカウントの電子メール。

        4. プライベートキー — Google サービスアカウントからコピーしたプライベートキー。

        5. [Save] (保存) を選択します。

    2. VVirtual Private Cloud (VPC) — VPC の使用を選択できます。その場合は、サブネットと VPC セキュリティグループを追加する必要があります。

    3. IAM ロール — 既存の IAM ロールを選択するか、新しい IAM ロールを作成して、リポジトリの認証情報にアクセスしてコンテンツをインデックス作成してください。

      注記

      インデックスに使用される IAM ロールはデータソースには使用できません。既存のロールがインデックスや FAQ に使用されているかどうかわからない場合は、エラーを避けるため [新しいロールを作成する] を選択してください。

    4. [Next] (次へ) を選択します。

  7. [同期設定の設定] ページで、次の情報を入力します。

    1. [同期範囲] の [エンティティタイプ] で、[メッセージ添付ファイル] を選択してメッセージの添付ファイルを同期します。メッセージはデフォルトで同期されます。

    2. （オプション）追加設定については、次の情報を入力します。

      1. 日付範囲-クロールする電子メールの開始日と終了日を指定する日付範囲を入力します。

      2. メールドメイン — ドメインに基づいてメールを含めたり除外したりします。

      3. 件名のキーワード-件名のキーワードに基づいてメールを追加または除外します。

        注記

        入力したすべての件名キーワードに一致するドキュメントを含めるように選択することもできます。

      4. ラベル — 正規表現パターンを追加して、特定のラベルを含めるまたは除外します。最大 100 個のパターンを追加できます。

      5. 添付ファイル-正規表現パターンを追加して、特定の添付ファイルを包含または除外します。最大 100 個のパターンを追加できます。

    3. 同期モードでは、データソースのコンテンツが変更されたときにインデックスを更新する方法を選択します。Amazon Kendraデータソースを初めてと同期すると、すべてのコンテンツがデフォルトで同期されます。

      • 完全同期-以前の同期ステータスに関係なく、すべてのコンテンツを同期します。

      • 新規、変更、削除されたコンテンツの同期-新規、変更、削除されたコンテンツのみを同期します。

        重要

        完全に削除された Gmail メッセージを更新する API はないため、新規、変更、削除されたコンテンツの同期を行います。

        • GmailAmazon Kendra から完全に削除されたメールをインデックスから削除しない

        • Gmail のメールラベルの変更が同期されない

        GmailAmazon Kendra のデータソースラベルの変更と完全に削除されたメールをインデックスに同期するには、定期的にフルクロールを実行する必要があります。

    4. 同期実行スケジュール (頻度)-Amazon Kendra データソースと同期する頻度

    5. [Next] (次へ) を選択します。

  8. [フィールドマッピングの設定] ページで、次の情報を入力します。

    1. メッセージおよびメッセージ添付ファイル用 —Amazon Kendra 生成されたデフォルトデータソースフィールドから、インデックスにマッピングするフィールドを選択します。

      注記

      Amazon KendraGmail データソースコネクタは API の制限によりカスタムインデックスフィールドの作成をサポートしていません。

    2. [Next] (次へ) を選択します。

  9. [確認と作成] ページで、入力した情報が正しいことを確認し、[データソースの追加] を選択します。また、このページで情報を編集することもできます。データソースが正常に追加されると、データソースページに表示されます。

TemplateConfiguration API

GmailAmazon Kendra に接続するには

TemplateConfigurationAPI を使用して以下を指定する必要があります。

  • データソース — データソースをとして指定する必要がありますGMAIL。

  • データソーススキーマ — データソーススキーマを含む JSON を含めます。テンプレートスキーマを表示するには、「データソーススキーマ」を参照してください。

  • タイプ —TEMPLATE 電話をかけるときにタイプとして指定しますCreateDataSource。

  • シークレットの Amazon リソースネーム (ARN) — GmailSecrets Manager アカウントの認証情報を含むシークレットの Amazon リソースネーム (ARN) を指定します。シークレットは、次のキーを持つ JSON 構造で格納されます。

    {
    "adminAccountEmailId": "${adminAccountEmailId}",
    "clientEmailId": "${clientEmailId}",
    "privateKey": "${privateKey}"
}
    注記

    認証情報とシークレットは定期的に更新またはローテーションしてください。セキュリティに必要なアクセスレベルのみを指定してください。データソース、およびコネクタバージョン 1.0 と 2.0 (該当する場合) 間で認証情報とシークレットを再利用することはお勧めしません。

  • IAM ロール —RoleArnCreateDataSource コールするタイミングを指定して、IAMSecrets Managerシークレットにアクセスする権限をロールに付与し、Gmail コネクタに必要なパブリック APIAmazon Kendra を呼び出すことができます。詳細については、「Gmail データソースの IAM ロール」を参照してください。

オプションで、次の機能を追加することもできます。

  • [Virtual Private Cloud (VPC)] —VpcConfigurationCreateDataSource 電話をかけるときに指定してください。VPCAmazon Kendra を使用するように設定する を参照してください。

  • 同期モード-すべてのドキュメントを同期してインデックスを更新するか、新規、変更、Amazon Kendra削除されたドキュメントのみを同期するかを指定します。以下から選択できます。

    • FORCED_FULL_CRAWLデータソースがインデックスと同期するたびに、すべてのコンテンツを新たに再クロールして既存のコンテンツを置き換える

    • FULL_CRAWLデータソースがインデックスと同期するたびに、新規、変更、および削除されたコンテンツのみを段階的にクロールします

    重要

    完全に削除された Gmail メッセージを更新する API はないため、FULL_CRAWL/新規、変更、削除されたコンテンツの同期:

    • GmailAmazon Kendra から完全に削除されたメールをインデックスから削除しない

    • Gmail のメールラベルの変更が同期されない

    GmailAmazon Kendra のデータソースラベルの変更と完全に削除されたメールをインデックスに同期するには、定期的にフルクロールを実行する必要があります。

  • 包含フィルターと除外フィルター-添付ファイルを含めるかどうかを指定できます。正規表現パターンを指定して、特定のメッセージや添付ファイルを包含または除外することもできます。

    注記

    ほとんどのデータソースは正規表現パターンを使用します。正規表現パターンとは、フィルターと呼ばれる包含パターンまたは除外パターンです。包含フィルターを指定すると、包含フィルターと一致するコンテンツのみが索引付けされます。包含フィルターに一致しないドキュメントはインデックスに登録されません。包含フィルターと除外フィルターを指定した場合、除外フィルターに一致するドキュメントは、包含フィルターと一致してもインデックス作成されません。

  • フィールドマッピング — GmailAmazon Kendra のデータソースフィールドをインデックスフィールドにマッピングすることを選択します。詳細については、データソースフィールドのマッピングを参照してください。

    注記

    Amazon KendraGmail データソースコネクタは API の制限によりカスタムインデックスフィールドの作成をサポートしていません。
