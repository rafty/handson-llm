AWSアクセスポータルにサインインする

IAM AWS Organizations アイデンティティーセンターのユーザーはのメンバーです。IAM Identity Center のユーザーはAWS アカウント、特定のサインイン URL AWS を使用してアクセスポータルにサインインすることで、複数のビジネスアプリケーションにアクセスできます。特定のサインイン URL の詳細については、「」を参照してくださいAWS アクセスポータル。

IAM Identity Center AWS アカウント のユーザーとしてにサインインする前に、次の必須情報を収集します。

  • 企業ユーザー名

  • 企業パスワード

  • 特定のサインイン URL

注記

サインイン後、AWSアクセスポータルセッションは 8 時間有効です。8 時間後に再度ログインする必要があります。

AWSアクセスポータルにサインインするには

  1. ブラウザウィンドウで、https://your_subdomain.awsapps.com/startなどの電子メールで提供されたサインイン URL を貼り付けてください。次に、Enter キーを押します。

  2. 会社の認証情報 (ユーザー名やパスワードなど) を使ってサインインします。

    注記

    管理者から電子メールでワンタイムパスワード (OTP) が送信され、初めてサインインする場合は、そのパスワードを入力します。ログインしたら、future ログイン用に新しいパスワードを作成する必要があります。

  3. 確認コードの入力を求められた場合は、メールを確認してください。次に、コードをコピーしてサインインページに貼り付けます。

    注記

    確認コードは通常、E メールで送信されますが、その送信方法はさまざまです。メールに認証コードが届いていない場合は、管理者に確認コードの詳細を確認してください。

  4. IAM Identity Center でユーザーの MFA が有効になっている場合は、それを使用して認証を行います。

  5. 認証後、ポータルに表示されるAWS アカウントすべてのアプリケーションにアクセスできます。

    1. にログインするには、AWS Management ConsoleAWS アカウントタイルを選択してから、管理する個々のアカウントを選択します。

      ユーザーが持っている権限は、管理コンソールを開いたり、コマンドラインやプログラムによるアクセスの資格情報を取得したりするためのリンクとともに表示されます。

      管理コンソールのリンクを選択します。

    2. アプリケーションにサインインするには、アクセスするアプリケーションのアイコンを選択します。

IAM Identity Center にユーザーとしてサインインすると、セッションと呼ばれる一定期間、リソースにアクセスするための認証情報が提供されます。デフォルトでは、ユーザは 1 AWS アカウント 時間ログインできます。管理者がセッションに設定できる最大値は 12 時間です。セッションが終了したら、再度サインインできます。
