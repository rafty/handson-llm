SCIM プロファイルおよび SAML 2.0 の実装

SCIM と SAML は共に IAM Identity Center を設定する上で重要な注意事項です。

SAML 2.0 の実装

IAM Identity Center では、SAML (Security Assertion Markup Language) 2.0 を使用して ID フェデレーションをサポートしています。これにより、IAM Identity Center は、外部の ID プロバイダーからの ID を認証することができます ()。IdPsSAML 2.0 は、SAML アサーションを安全に交換するためのオープンスタンダードです。SAML 2.0 は、SAML 権限 (ID プロバイダ (IdP) と呼ばれます) と SAML コンシューマ (サービスプロバイダ (SP) と呼ばれます) の間で、ユーザに関する情報を渡します。IAM Identity Center サービスは、この情報を使って、フェデレーテッド Single Sign-On を提供します。Single Sign-On により、ユーザーは既存の ID プロバイダーの認証情報に基づいて、AWSアカウントや設定されたアプリケーションにアクセスすることができます。

IAM Identity Center は、IAM IdP 機能を IAM アイデンティティセンターストアAWS Managed Microsoft AD、または外部の ID プロバイダーに追加します。その後、ユーザーは SAML をサポートするサービスや、Microsoft 365、Concur、Salesforce などのサードパーティアプリケーションを含む SAML をサポートするサービスにシングルサインオンできます。AWS Management Console

しかし、SAML プロトコルは、ユーザーやグループについての情報を得るために IdP へクエリする方法は提供していません。そのため、これらのユーザーやグループを IAM Identity Center にプロビジョニングして認識する必要があります。

SCIM プロファイル

IAM Identity Center は、クロスドメインアイデンティティ管理システム (SCIM) v2.0 規格に対応しています。SCIM は IAM Identity Center の ID と IdP の ID を同期させます。これには、IdP と IAM Identity Center の間で行われるユーザーのプロビジョニング、アップデート、デプロビジョニングが含まれます。

SCIM を実装する方法の詳細については、「自動プロビジョニング」を参照してください。IAM アイデンティティセンターの SCIM 実装の詳細については、IAM アイデンティティセンターの SCIM 実装開発者ガイドを参照してください。
