Services AWS qui prennent en charge IPv6

Les ordinateurs et les appareils intelligents utilisent des adresses IP pour communiquer entre eux via Internet et d'autres réseaux. Le développement d'Internet s'accompagne d'un besoin accru d'adresses IP. Le format le plus courant pour les adresses IP est IPv4. Le nouveau format des adresses IP est IPv6. Il fournit un espace d'adressage plus important qu'IPv4.

La prise en charge d'IPv6 par Services AWS comprend la prise en charge de la configuration à double pile (IPv4 et IPv6) ou de la configuration IPv6 uniquement. Par exemple, un cloud privé virtuel (VPC) est une section logiquement isolée du AWS Cloud, dans lequel vous pouvez lancer des ressources AWS. Au sein d'un VPC, vous pouvez créer des sous-réseaux IPv4 uniquement, à double pile ou IPv6 uniquement.

Services AWS prennent en charge l'accès via des points de terminaison publics. Certains Services AWS prennent également en charge l'accès via des points de terminaison privés optimisés par AWS PrivateLink. Les Services AWS peuvent prendre en charge IPv6 via leurs points de terminaison privés, même s'ils ne prennent pas en charge IPv6 via leurs points de terminaison publics. Les points de terminaison qui prennent en charge IPv6 peuvent répondre aux requêtes DNS avec des enregistrements AAAA.

Prise en charge supplémentaire d'IPv6

Calcul

  • Amazon EC2 prend en charge le lancement d'instances basées sur le système Nitro dans des sous-réseaux IPv6 uniquement.

  • Amazon EC2 fournit des points de terminaison IPv6 pour Instance Metadata Service (IMDS) et Amazon Time Sync Service.

Réseau et diffusion de contenu

  • Amazon VPC prend en charge la création de sous-réseaux IPv6 uniquement.

  • Amazon VPC permet aux ressources AWS IPv6 de communiquer avec les ressources IPv4 en prenant en charge DNS64 sur vos sous-réseaux et le protocole NAT64 sur vos passerelles NAT.

Sécurité, identité et conformité

  • AWS Identity and Access Management (IAM) prend en charge les adresses IPv6 dans les politiques IAM.

  • Amazon Macie prend en charge les adresses IPv6 dans des données d'identification personnelle (PII)

Gestion et gouvernance

  • Les enregistrements AWS CloudTrail incluent des informations IPv6 source.

  • AWS CLI v2 prend en charge le téléchargement via des connexions IPv6 pour les clients IPv6 uniquement.
