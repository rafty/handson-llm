Tâches nécessitant des informations d'identification d'utilisateur root

Important

Les actions AWS Identity and Access Management (IAM) suivantes arriveront à la fin du support standard en juillet 2023 : aws-portal:ModifyAccount etaws-portal:ViewAccount. Reportez-vous à la rubrique Utilisation d'AWS Billingactions détaillées pour remplacer ces actions par des actions détaillées afin d'avoir accès aux AWS Billing consoles de gestion des AWS coûts et de comptes. AWS

Si vous avez créé votre AWS compte ou votre compte de AWS Organizations gestion avant le 06 mars 2023, les actions détaillées entreront en vigueur à compter de juillet 2023. Nous vous recommandons d'ajouter les actions détaillées, mais de ne pas supprimer vos autorisations existantes ayant des préfixes aws-portal ou purchase-orders.

Si vous avez créé votre AWS compte ou votre compte de AWS Organizations gestion le 06 mars 2023 ou après cette date, les actions détaillées sont immédiatement effective.

Nous vous recommandons de configurer un utilisateur administratif AWS IAM Identity Center (successor to AWS Single Sign-On) pour effectuer les tâches quotidiennes et accéder aux AWS ressources. Toutefois, vous pouvez effectuer les tâches répertoriées ci-dessous uniquement lorsque vous vous connectez en tant qu'utilisateur root d'un compte.

Tâches

  • Modifiez les paramètres de votre compte. Cela inclut le nom du compte, l'adresse e-mail, le mot de passe de l'utilisateur root et les clés d'accès de l'utilisateur root. Les autres paramètres du compte, tels que les informations de contact, les préférences en matière de devise de paiement etRégions AWS, ne nécessitent pas d'informations d'identification utilisateur root.

  • Restaurez les autorisations utilisateur IAM. Si le seul administrateur IAM révoque accidentellement ses propres autorisations, vous pouvez vous connecter en tant qu'utilisateur root pour modifier les politiques et restaurer ces autorisations.

  • Activation de l'accès IAM à la console de Billing and Cost Management.

  • Affichez certaines factures fiscales. Un utilisateur IAM disposant du portail aws-portal : ViewBilling autorisation peut consulter et télécharger les factures de TVA depuis AWS l'Europe, mais pas depuis AWS Inc. ou Amazon Internet Services Private Limited (AISPL).

  • Fermez votre Compte AWS.

  • Inscrit en tant que vendeur sur le Marketplace des instances réservées.

  • Configurez un compartiment Amazon S3 pour activer l'authentification MFA (authentification multifacteur).

  • Modifiez ou supprimez une politique relative aux compartiments Amazon Simple Storage Service (Amazon S3) qui inclut un ID de cloud privé virtuel (VPC) ou un ID de point de terminaison VPC non valide.

  • Inscrivez-vous à AWS GovCloud (US).

  • Demandez les clés d'accès utilisateur racine du AWS GovCloud (US) compte àAWS Support.
