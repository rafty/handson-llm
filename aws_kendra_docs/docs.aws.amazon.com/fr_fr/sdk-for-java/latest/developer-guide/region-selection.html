AWSSélection de la région

Les régions vous permettent d'accéderAWSles services qui résident physiquement dans une région géographique spécifique. Cela peut être utile pour la redondance et pour maintenir vos données et vos applications en cours d'exécution à proximité de l'endroit où vous-même et vos utilisateurs y accédez.

DansAWS SDK for Java2.x, tous les différentes classes associées à une région à partir de la version 1.x ont été regroupées en une classe Region. Vous pouvez utiliser cette classe pour toutes les actions associées à une région, comme la récupération des métadonnées relatives à une région ou la vérification de la disponibilité d'un service dans une région.

Choix d'une région

Vous pouvez spécifier un nom de région et le kit SDK choisira automatiquement un point de terminaison approprié.

Pour définir explicitement une région, nous vous recommandons d'utiliser les constantes définis dans la classe Region. Il s'agit d'une énumération de toutes les régions disponibles publiquement. Pour créer un client avec une région à partir de la classe, utilisez le code suivant.

Ec2Client ec2 = Ec2Client.builder()
          .region(Region.US_WEST_2)
          .build();

Si la région que vous essayez d'utiliser n'est pas l'un des constantes de la classe Region, vous pouvez créer une nouvelle région à l'aide de la méthode of. Cette fonctionnalité vous permet d'accéder à de nouvelles régions sans mettre à niveau le kit SDK.

Region newRegion = Region.of("us-east-42");
Ec2Client ec2 = Ec2Client.builder()
          .region(newRegion)
          .build();
Note

Une fois que vous avez créé un client avec le générateur, il est immuable et la région ne peut pas être modifiée. Si vous travaillez avec plusieursRégions AWSPour le même service, vous devez créer plusieurs clients, un par région.

Choix d'un point de terminaison spécifique

EACHAWSle client peut être configuré pour utiliser unpoint de terminaison spécifiquedans une région en appelant leendpointOverrideMéthode.

Par exemple, pour configurer leAmazon EC2Pour utiliser la région Europe (Irlande), utilisez le code suivant.

Ec2Client ec2 = Ec2Client.builder()
               .region(Region.EU_WEST_1)
               .endpointOverride(URI.create("https://ec2.eu-west-1.amazonaws.com"))
               .build();

Déterminez automatiquement la région à partir de l'environnement

Lors de l'exécution sur Amazon EC2 ou AWS Lambda, vous voudrez peut-être configurer les clients afin d'utiliser la même région que celle sur laquelle votre code s'exécute. Votre code est ainsi dissocié de l'environnement d'exécution et le déploiement de votre application sur plusieurs régions dans le but de réduire la latence ou la redondance s'en trouve simplifié.

Pour utiliser la chaîne du fournisseur d'informations d'identification/de région par défaut et déterminer la région à partir de l'environnement, utilisez la méthode create du générateur client.

Ec2Client ec2 = Ec2Client.create();

Si vous ne définissez pas explicitement une région à l'aide de la méthode region, le kit SDK consulte la chaîne du fournisseur de région par défaut afin d'essayer de déterminer la région à utiliser.

Chaîne de fournisseurs de régions par défaut

Le processus de recherche d'une région est le suivant :

  1. Toute région explicite définie en utilisant region sur le générateur lui-même prévaut sur toute autre région.

  2. La variable d'environnement AWS_REGION est contrôlée. Si elle est définie, cette région est utilisée pour configurer le client.

    Note

    Cette variable d'environnement est définie par le conteneur Lambda.

  3. Le SDK vérifie leAWSFichier de configuration partagé (généralement situé dans~/.aws/config). Si la propriété region est présente, le kit SDK l'utilise.

    • La variable d'environnement AWS_CONFIG_FILE peut être utilisée pour personnaliser l'emplacement du fichier de configuration partagé.

    • LeAWS_PROFILEou la variable d'environnementaws.profilepermet de personnaliser le profil chargé par le kit SDK.

  4. Le kit SDK essaie d'utiliser le service de métadonnées d'instance Amazon EC2 pour déterminer la région de l'instance Amazon EC2 actuellement en cours d'exécution.

  5. Si le kit SDK n'a toujours pas trouvé de région par ce biais, la création du client échoue et une exception est levée.

Lorsque vous développezAWSUne approche courante consiste à utiliser le kitFichier de configuration partagé(décrit dansOrdre de récupération des informations d'identification) pour définir la région pour le développement local et à s'appuyer sur la chaîne du fournisseur de région par défaut pour déterminer la région lors de l'exécution surAWSInfrastructure. La création du client s'en trouve ainsi grandement simplifiée et votre application demeure portable.

Vérification de la disponibilité des services dans une région

Pour voir si un particulierService AWSest disponible dans une région, utilisez leserviceMetadataetregionsur le service que vous souhaitez vérifier.

DynamoDbClient.serviceMetadata().regions().forEach(System.out::println);

Consultez la documentation de la classe Region pour les régions que vous pouvez spécifier, et utilisez le préfixe de point de terminaison du service à interroger.
