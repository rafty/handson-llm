DescribeIndex

Obtient des informations sur un index Amazon Kendra.

Syntaxe de la requête

{
   "Id": "string"
}

Paramètres de demande

Pour plus d'informations sur les paramètres communs à toutes les actions, voir Paramètres communs.

Cette demande accepte les données suivantes au format JSON.

Id

Identifiant de l'index.

Type : String

Contraintes de longueur : longueur fixe de 36.

Modèle : [a-zA-Z0-9][a-zA-Z0-9-]*

Obligatoire : oui

Eléments de réponse

Si l'action aboutit, le service renvoie une réponse HTTP 200.

Les données suivantes sont renvoyées au format JSON par le service.

CapacityUnits

Pour les index Enterprise Edition, vous pouvez choisir d'utiliser des capacités supplémentaires pour répondre aux besoins de votre application. Il contient les unités de capacité utilisées pour l'indice. Une capacité de stockage de requêtes ou de documents nulle indique que l'index utilise la capacité par défaut. Pour plus d'informations sur la capacité par défaut d'un index et son ajustement, consultez la section Ajustement de la capacité.

Type : objet CapacityUnitsConfiguration

CreatedAt

L'horodatage de création de l'index.

Type : Timestamp

Description

Description de l'index.

Type : String

Contraintes de longueur : longueur minimale de 0. Longueur maximum de 1 000.

Modèle : ^\P{C}*$

DocumentMetadataConfigurations

Informations de configuration pour les métadonnées ou les champs du document. Les métadonnées des documents sont des champs ou des attributs associés à vos documents. Par exemple, le nom du service de l'entreprise associé à chaque document.

Type : tableau d’objets DocumentMetadataConfiguration

Membres du tableau : nombre minimum de 0 élément. Nombre maximal de 500 éléments.

Edition

L'édition Amazon Kendra utilisée pour l'index. C'est vous qui décidez de l'édition lorsque vous créez l'index.

Type : String

Valeurs valides : DEVELOPER_EDITION | ENTERPRISE_EDITION

ErrorMessage

Lorsque la valeur duStatus champ estFAILED égale à, leErrorMessage champ contient un message expliquant pourquoi.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximale de 2048.

Modèle : ^\P{C}*$

Id

Identifiant de l'index.

Type : String

Contraintes de longueur : longueur fixe de 36.

Modèle : [a-zA-Z0-9][a-zA-Z0-9-]*

IndexStatistics

Fournit des informations sur le nombre de questions et réponses de la FAQ et sur le nombre de documents texte indexés.

Type : objet IndexStatistics

Name

Nom de l'index.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximum de 1 000.

Modèle : [a-zA-Z0-9][a-zA-Z0-9_-]*

RoleArn

Amazon Resource Name (ARN) du rôle IAM qui permet à Amazon Kendra.

Type : String

Contraintes de longueur : longueur minimum de 0. Longueur maximale de 1284.

Modèle : arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

ServerSideEncryptionConfiguration

Identifiant deAWS KMS la clé principale utilisée pour le chiffrement de vos données. Amazon Kendra ne prend pas en charge les CMK asymétriques.

Type : objet ServerSideEncryptionConfiguration

Status

L'index. Lorsque la valeur est atteinteACTIVE, l'index est prêt à l'emploi. Si la valeur duStatus champ estFAILED, leErrorMessage champ contient un message expliquant pourquoi.

Type : String

Valeurs valides : CREATING | ACTIVE | DELETING | FAILED | UPDATING | SYSTEM_UPDATING

UpdatedAt

Unix utilisée pour la dernière mise à jour de l'index.

Type : Timestamp

UserContextPolicy

Politique relative au contexte utilisateur pour l'index Amazon Kendra.

Type : String

Valeurs valides : ATTRIBUTE_FILTER | USER_TOKEN

UserGroupResolutionConfiguration

Si vous avez activé la configuration pour récupérer les niveaux d'accès des groupes et des utilisateurs à partir d'une sourceAWS IAM Identity Center (successor to AWS Single Sign-On) d'identité.

Type : objet UserGroupResolutionConfiguration

UserTokenConfigurations

Configuration du jeton utilisateur pour l'index Amazon Kendra.

Type : tableau d’objets UserTokenConfiguration

Membres du tableau : nombre maximum de 1 élément.

Erreurs

Pour plus d'informations sur les erreurs communes à toutes les actions, consultezErreurs courantes.

AccessDeniedException

Vous ne disposez pas de l'accès nécessaire pour effectuer cette action. Assurez-vous de disposer des politiques d'autorisation et des comptes utilisateur requis, puis réessayez.

Code d'état HTTP : 400

InternalServerException

Un problème s'est produit avec le serveur interne utilisé pour votre service Amazon Kendra. Veuillez patienter quelques minutes et réessayez, ou contactez le support pour obtenir de l'aide.

Code d'état HTTP : 500

ResourceNotFoundException

La ressource que vous souhaitez utiliser n'existe pas. Vérifiez que vous avez fourni la bonne ressource et réessayez.

Code d'état HTTP : 400

ThrottlingException

La demande a été refusée suite à une limitation des demandes. Veuillez réduire le nombre de demandes et réessayer.

Code d'état HTTP : 400

ValidationException

L'entrée ne satisfait pas les contraintes définies par le service Amazon Kendra. Veuillez entrer correctement et réessayer.

Code d'état HTTP : 400
