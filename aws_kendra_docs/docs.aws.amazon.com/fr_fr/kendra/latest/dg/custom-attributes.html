Création de champs de document personnalisés

Vous pouvez appliquer des attributs personnalisés ou des champs de métadonnées pertinents à vos documents particuliers. Par exemple, vous pouvez créer un champ ou un attribut personnalisé appelé « Département » avec les valeurs « RH », « Ventes » et « Fabrication ». Vous pouvez utiliser ces champs ou attributs pour limiter la réponse (les résultats de recherche) aux documents du service « RH », par exemple.

Vous pouvez créer jusqu'à 500 champs ou attributs personnalisés.

Pour la plupart des sources de données, vous mappez les champs de la source de données externe avec les champs correspondants deAmazon Kendra. Pour plus d'informations, veuillez consulter Mappage des champs de source de données. Pour les sources de données S3, vous pouvez appliquer des champs ou des attributs personnalisés à l'aide de fichiers de métadonnées.

Avant de pouvoir utiliser un champ ou un attribut personnalisé, vous devez d'abord le créer dans l'index. Utilisez la console ou l'UpdateIndexAPI pour créer le champ d'index. Vous ne pouvez pas modifier le type de données du champ une fois que vous l'avez créé.

Vous pouvez également utiliserAmazon Kendra les champs intégrés ou réservés.

Avec l'UpdateIndexAPI, vous ajoutez des champs ou des attributs personnalisés à l'aide duDocumentMetadataConfigurationUpdates paramètre.

L'exemple JSON suivant permetDocumentMetadataConfigurationUpdates d'ajouter un champ appelé « Department » à l'index.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Utilisation des champs de document intégrés à Amazon Kendra

Avec l'UpdateIndex API, vous pouvez créer des champs réservés ou intégrés en utilisantDocumentMetadataConfigurationUpdates et en spécifiant le nom du champ réservé pour le mapper à votre attribut de document ou à un champ de document équivalent. Vous pouvez également créer des champs personnalisés de cette façon. Si vous utilisez un connecteur de source de données, la plupart incluent des mappages de champs qui mettent en correspondance les champs de votre document source de données avecAmazon Kendra des champs d'index. Lorsque vous créez un champ, vous pouvez configurer l'Searchobjet pour le définir comme étant affichable, facetable, consultable et triable. Vous pouvez configurer l'Relevanceobjet pour définir la durée d'amélioration de la pertinence, la fraîcheur, l'importance, l'ordre de classement et les valeurs d'importance du champ. Vous ne pouvez pas modifier le type de champ une fois que vous l'avez créé.

Amazon Kendracontient les champs de document réservés ou intégrés suivants que vous pouvez utiliser :

  • _authors—Une liste d'un ou de plusieurs auteurs responsables du contenu du document.

  • _category—Une catégorie qui place un document dans un groupe spécifique.

  • _created_at—La date et l'heure au format ISO 8601 auxquelles le document a été créé. Par exemple, 2012-03-25T12:30:10+01:00 est le format de date et d'heure ISO 8601 pour le 25 mars 2012 à 12 h 30 (plus 10 secondes) à l'heure d'Europe centrale.

  • _data_source_id—Identifiant de la source de données qui contient le document.

  • _document_body—Le contenu du document.

  • _document_id—Identifiant unique pour le document.

  • _document_title—Le titre du document.

  • _excerpt_page_number—Le numéro de page d'un fichier PDF où apparaît l'extrait du document. Si votre index a été créé avant le 8 septembre 2020, vous devez réindexer vos documents avant de pouvoir utiliser cet attribut.

  • _faq_id—S'il s'agit d'une question et d'une réponse à la FAQ, un identifiant unique pour ces questions.

  • _file_type—Le type de fichier du document, tel que pdf ou doc.

  • _last_updated_at—La date et l'heure au format ISO 8601 auxquelles le document a été mis à jour pour la dernière fois. Par exemple, 2012-03-25T12:30:10+01:00 est le format de date et d'heure ISO 8601 pour le 25 mars 2012 à 12 h 30 (plus 10 secondes) à l'heure d'Europe centrale.

  • _source_uri—L'URI où le document est disponible. Par exemple, l'URI du document sur le site Web d'une entreprise.

  • _version: identifiant de la version spécifique d'un document.

  • _view_count—Le nombre de fois que le document a été consulté.

  • _language_code(String) : code de la langue qui s'applique au document. La valeur par défaut est l'anglais si vous ne spécifiez pas de langue. Pour plus d'informations sur les langues prises en charge, y compris leurs codes, voir Ajout de documents dans des langues autres que l'anglais.

Pour les champs personnalisés, vous créez ces champs à l'DocumentMetadataConfigurationUpdatesaide de l'UpdateIndexAPI, comme vous le feriez pour créer un champ réservé ou intégré. Vous devez définir le type de données approprié pour votre champ personnalisé. Vous ne pouvez pas modifier le type de champ une fois que vous l'avez créé.

Voici les types que vous pouvez définir pour les champs personnalisés :

  • Date

  • Nombre

  • Chaîne

  • Liste de chaînes

Vous créez un champ personnalisé à l'aide de la console ou de l'UpdateIndexAPI. Après avoir créé un champ personnalisé, vous le mappez à un attribut de document, comme vous le feriez avec un champ réservé. Si vous avez ajouté un document à l'index à l'aide de BatchPutDocumentl'API, vous mappez les attributs avec l'API. Pour les documents indexés à partir d'une source deAmazon S3 données, vous mappez les attributs à l'aide d'un fichier de métadonnées contenant une structure JSON qui décrit les attributs du document. Pour les documents indexés à l'aide d'une base de données ou d'une source de données permettant le mappage de champs, vous mappez les attributs à l'aide de la console ou de la configuration de la source de données.

Ajouter des attributs ou des champs personnalisés à l'aide de l' BatchPutDocument API

Lorsque vous utilisez l'BatchPutDocumentAPI pour ajouter un document à votre index, vous spécifiez des champs ou des attributs personnalisés dans le cadre deAttributes. Vous pouvez ajouter plusieurs champs ou attributs lorsque vous appelez l'API. Vous pouvez créer jusqu'à 500 champs ou attributs personnalisés. L'exemple suivant est un champ ou un attribut personnalisé qui ajoute le terme « Département » à un document.

"Attributes": 
    {
        "Department": "HR",
        "_category": "Vacation policy"
    }

Ajouter des attributs ou des champs personnalisés à une sourceAmazon S3 de données

Lorsque vous utilisez un compartiment S3 comme source de données pour votre index, vous ajoutez des métadonnées aux documents à l'aide de fichiers de métadonnées complémentaires. Vous placez les fichiers JSON de métadonnées dans une structure de répertoires parallel à vos documents. Pour plus d'informations, veuillez consulter Amazon S3métadonnées du document.

Vous spécifiez des champs ou des attributs personnalisés dans la structureAttributes JSON. Vous pouvez créer jusqu'à 500 champs ou attributs personnalisés. Par exemple, l'exemple suivant permetAttributes de définir trois champs ou attributs personnalisés et un champ réservé.

"Attributes": {
        "brand": "Amazon Basics",
        "price": 1595,
        "_category": "sports",
        "subcategories": ["outdoors", "electronics"]
    }
