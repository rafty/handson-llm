Amazon FSx

Amazon FSx est un système de serveur de fichiers entièrement géré basé sur le cloud qui offre des fonctionnalités de stockage partagé. Si vous êtes un utilisateur d'Amazon FSx, vous pouvez l'utiliserAmazon Kendra pour indexer votre source de données Amazon FSx.

Amazon Kendrane prend actuellement enAmazon FSx charge que le serveur de fichiers Windows.

Vous pouvez vous connecterAmazon Kendra à votre source de données Amazon FSx à l'aide de la Amazon Kendraconsole et de l'FsxConfigurationAPI.

Pour résoudre les problèmes liés à votre connecteur de source de donnéesAmazon Kendra Amazon FSx, consultezRésolution des problèmes de sources de données.

Fonctionnalités prises en charge

Amazon KendraLe connecteur de source de données Amazon FSx prend en charge les fonctionnalités suivantes :

  • Journal des modifications

  • Mappages de champs

  • Filtrage du contexte utilisateur

  • Filtres d'inclusion/d'exclusion

  • Virtual privatecCloud (VPC)

Prérequis

Avant de pouvoir l'utiliserAmazon Kendra pour indexer votre source de données Amazon FSx, apportez ces modifications à votre Amazon FSx et à votreAWS compte.

Dans Amazon FSx, assurez-vous de disposer de :

  • Un compte Amazon FSx avec des autorisations de lecture et de montage.

  • Vous avez noté vos informations d'Amazon FSxauthentification pour un compte utilisateur Active Directory. Cela inclut votre nom d'utilisateur Active Directory et votre nom de domaine DNS (Domain Name System). Par exemple, user@corp.example.com.

  • Vous avez copié l'ID de votre système deAmazon FSx fichiers.

  • Vous avez utilisé unAmazon VPC (AWSVPC) où vousAmazon FSx résidez.

  • Coché, chaque document est unique dans Amazon FSx et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer de :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké vos informations d'authentification Amazon FSx de manièreAWSSecrets Manager secrète et, si vous utilisez l'API, vous avez noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de données Amazon FSx àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données Amazon FSx, vous devez fournir les informations nécessaires sur votre source de données Amazon FSx afin deAmazon Kendra pouvoir accéder à vos données. Si vous n'avez pas encore configuré Amazon FSx pourAmazon Kendra voirPrérequis.

Console

Pour vous connecterAmazon Kendra à Amazon FSx

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez le connecteur Amazon FSx, puis choisissez Ajouter une source de données.

  5. Sur la page Spécifier les détails de la source de données, entrez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) : entrez une description facultative pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Balises, pour Ajouter une nouvelle balise : Tags pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, entrez les informations suivantes :

    1. Source : sélectionnezAmazon FSx pour le serveur de fichiers Windows.

    2. Amazon FSxID du système de fichiers —Sélectionnez l'ID de votre système de fichiers ou créez un nouveau répertoire.

    3. AWSSecrets Managersecret —Choisissez un secret existant ou créez-en un nouveauSecrets Manager pour stocker vos informations d'authentification Amazon FSx. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -Amazon FSx » est automatiquement ajouté à votre nom secret.

        2. Pour le nom d'utilisateur : entrez le nom d'utilisateur du compteAmazon FSx Active Directory.

        3. Pour le mot de passe : entrez le mot de passe du compteAmazon FSx Active Directory.

      2. Choisissez Save (Enregistrer).

    4. Virtual Private Cloud (VPC) : vous devez utiliser un VPC avec votre source deAmazon FSx données. Vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    5. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification de votre référentiel et au contenu de l'index.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    6. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, entrez les informations suivantes :

    1. Journal des modifications : sélectionnez cette option pour mettre à jour votre index au lieu de synchroniser tous vos fichiers.

    2. Modèles Regex : ajoutez des modèles d'expressions régulières pour inclure ou exclure certains contenus. Vous pouvez ajouter jusqu'à 100 motifs.

    3. Dans Synchroniser le calendrier d'exécution, pour Fréquence, choisissez la fréquence àAmazon Kendra laquelle vous souhaitez synchroniser avec votre source de données.

    4. Choisissez Suivant.

  8. Sur la page Définir les Mappages de champs, entrez les informations suivantes :

    1. Mappages de champs Amazon FSx Windows : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à Amazon FSx

Vous devez spécifier les éléments suivants à l'aide de l'FsxConfigurationAPI :

  • Secret Amazon Resource Name (ARN) : indiquez le nom de ressource Amazon (ARN) d'unSecrets Manager code secret contenant les informations d'authentification de votre compte Amazon FSx. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "username": "user@corp.example.com",
    "password": "password"
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pourIAM attribuer un rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le connecteur Amazon FSx etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de données Amazon FSx.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Virtual Private Cloud (VPC) : spécifiez dans leVpcConfiguration cadre de la configuration de la source de données. Reportez-vous Amazon Kendraà la section Configuration pour utiliser un VPC.

  • Journal des modifications : s'ilAmazon Kendra convient d'utiliser le mécanisme de journal des modifications de la source de données Amazon FSx pour déterminer si un document doit être ajouté, mis à jour ou supprimé dans l'index.

    Note

    Utilisez le journal des modifications si vous neAmazon Kendra souhaitez pas numériser tous les documents. Si votre journal des modifications est volumineux, la numérisation des documents de la source de données Amazon FSx peut prendreAmazon Kendra moins de temps que le traitement du journal des modifications. Si vous synchronisez votre source de données Amazon FSx avec votre index pour la première fois, tous les documents sont numérisés.

  • Filtres d'inclusion et d'exclusion : vous pouvez spécifier si vous souhaitez inclure ou exclure certains contenus et types de contenu.

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de données Amazon FSx à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

Note

Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats d'un utilisateur en fonction de l'accès de l'utilisateur ou du groupe aux documents.

Pour tester le filtrage du contexte utilisateur sur un utilisateur, vous devez inclure le nom de domaine DNS dans le nom d'utilisateur lorsque vous émettez la requête. Vous devez disposer des autorisations administratives du domaine Active Directory. Vous pouvez également tester le filtrage contextuel des utilisateurs sur le nom d'un groupe. Pour plus d'informations, consultez Filtrage du contexte utilisateur pour les sources de données Amazon FSx.
