Obtenir des informations grâce à l'analyse de recherche

Vous pouvez utiliserAmazon Kendra Search Analytics pour mieux comprendre comment votre application de recherche réussit ou non à aider vos utilisateurs à trouver des informations.

Amazon KendraLes analyses fournissent un aperçu de la façon dont vos utilisateurs interagissent avec votre application de recherche et de l'efficacité de la configuration de votre application de recherche. Vous pouvez consulter les données de mesure à l'aide de l'GetSnapshotsAPI ou en sélectionnant Analytics dans le panneau de navigation de la console.

Vous pouvez afficher les données générées parGetSnapshots sur votre propre tableau de bord personnalisé. Vous pouvez également utiliser le tableau de bord des statistiques fourni dans la console, qui inclut des graphiques visuels. Grâce à un tableau de bord visuel, vous pouvez rechercher des tendances ou des modèles dans le comportement des utilisateurs au fil du temps ou signaler des problèmes liés à la configuration de votre application de recherche. Par exemple, un graphique linéaire qui montre un nombre constant de requêtes par jour et une augmentation constante peut indiquer une adoption et une utilisation accrues. D'un autre côté, une baisse abrupte peut indiquer qu'il existe un problème qui doit être étudié.

Vous pouvez utiliser les indicateurs pour établir des liens entre différents points de données afin de résoudre les problèmes liés à la manière dont vos utilisateurs recherchent des informations ou découvrent des opportunités commerciales. Par exemple, le document « Comment fonctionne l'IA ? » est le document le plus cliqué dans les résultats de recherche, et la requête la plus recherchée est « Comment fonctionne l'apprentissage automatique ? ». Cela vous informe sur les termes et la langue préférés utilisés par vos utilisateurs. Vous pouvez intégrer ces termes dans vos documents ou utiliser des synonymes personnalisés pour ces termes afin de rendre vos documents plus faciles à rechercher pour vos utilisateurs.

Métriques pour la recherche

Il existe 10 indicateurs pour analyser les performances de votre application de recherche ou les informations recherchées par vos utilisateurs. Pour récupérer les données de métriques, vous devez spécifier le nom de chaîne des données métriques que vous souhaitez récupérer lorsque vous appelezGetSnapshots.

Vous devez également fournir un intervalle de temps ou une fenêtre de temps pour afficher les données de mesure. Vous pouvez consulter les données dans les fenêtres horaires suivantes :

  • THIS_WEEK: La semaine en cours, commençant le dimanche et se terminant la veille de la date actuelle.

  • ONE_WEEK_AGO: La semaine précédente, commençant le dimanche et se terminant le samedi suivant.

  • TWO_WEEKS_AGO: La semaine précédant la semaine précédente, commençant le dimanche et se terminant le samedi suivant.

  • THIS_MONTH: Le mois en cours, commençant le premier jour du mois et se terminant la veille de la date en cours.

  • ONE_MONTH_AGO: Le mois précédent, commençant le premier jour du mois et se terminant le dernier jour du mois.

  • TWO_MONTHS_AGO: mois précédant le mois précédent, commençant le premier jour du mois et se terminant le dernier jour du mois.

Dans la console, les fenêtres horaires prises en charge sont Cette semaine, Semaine précédente, Ce mois, Mois précédent.

Taux de clics

Proportion de requêtes qui mènent à un clic sur un document dans les résultats de recherche. Cela vous permet de savoir si la configuration de votre application de recherche aide vos utilisateurs à trouver des informations pertinentes pour leurs requêtes. Pour les requêtes qui renvoient des réponses instantanées, les utilisateurs n'ont peut-être pas besoin de cliquer sur un document pour obtenir plus d'informations. Pour plus d'informations, consultez Fréquence de réponse instantanée. Vous devez appeler SubmitFeedbackpour vous assurer que les commentaires provenant des clics sont collectés.

Pour récupérer des données sur le taux de clics à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asAGG_QUERY_DOC_METRICS. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation.

Fréquence de clic zéro

Proportion de requêtes qui n'entraînent aucun clic dans les résultats de recherche. Cela vous aide à comprendre les lacunes de votre contenu en fournissant des résultats de recherche non pertinents. Pour les requêtes qui renvoient des réponses instantanées, les utilisateurs n'ont peut-être pas besoin de cliquer sur un document pour obtenir plus d'informations. Pour plus d'informations, consultez Fréquence de réponse instantanée. En outre, vos paramètres de recherche, tels que le réglage des configurations, peuvent avoir un impact sur la façon dont les documents sont renvoyés dans les résultats de recherche.

Pour récupérer des données sur un taux de clic nul à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asAGG_QUERY_DOC_METRICS. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation.

Fréquence des résultats de recherche

Proportion de requêtes qui n'aboutissent à aucun résultat de recherche. Cela vous permet de comprendre les lacunes de votre contenu en ne fournissant aucun résultat de recherche pertinent.

Pour récupérer des données avec un taux de résultats de recherche nul à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asAGG_QUERY_DOC_METRICS. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation.

Fréquence de réponse instantanée

Proportion de requêtes ayant donné lieu à une réponse instantanée ou à une FAQ renvoyées. Cela vous aide à comprendre le rôle des réponses instantanées dans la fourniture d'informations.

Pour récupérer des données sur le taux de réponse instantané à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asAGG_QUERY_DOC_METRICS. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation.

Principales requêtes

Les 100 requêtes les plus recherchées par vos utilisateurs. Cela vous permet de comprendre quelles sont les requêtes les plus populaires et le type d'informations qui intéressent le plus vos utilisateurs.

Les indicateurs incluent le nombre de fois que la requête est recherchée, la proportion de clics vers un document, la proportion de clics sur un document sans clic, la profondeur moyenne des clics dans les résultats de recherche pour la requête, la proportion de réponses instantanées pour la requête et le niveau de confiance moyen pour les 10 premiers résultats de recherche pour une requête.

Pour récupérer des données sur les principales requêtes à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asQUERIES_BY_COUNT. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation de la console, puis en sélectionnant Principales requêtes sous Listes de requêtes.

Les requêtes les plus fréquentes sans aucun clic

Les 100 principales requêtes qui n'entraînent aucun clic dans les résultats de recherche. Cela vous permet de comprendre les éventuelles lacunes de votre contenu, les documents relatifs à certaines requêtes ou la configuration de votre application de recherche renvoie des résultats de recherche non pertinents. Pour les requêtes qui renvoient des réponses instantanées, les utilisateurs n'ont peut-être pas besoin de cliquer sur un document pour obtenir plus d'informations. Pour plus d'informations, consultez Fréquence de réponse instantanée.

Les indicateurs incluent le nombre de fois où la requête n'aboutit à aucun clic, la proportion de clics nuls pour la requête, la proportion de réponses instantanées pour la requête et le niveau de confiance moyen pour les 10 premiers résultats de recherche d'une requête.

Pour récupérer des données sur les requêtes les plus fréquentes sans aucun clic à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asQUERIES_BY_ZERO_CLICK_RATE. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation de la console, puis en sélectionnant les principales requêtes sans clic sous Listes de requêtes.

Les requêtes les plus fréquentes sans aucun résultat de recherche

Les 100 principales requêtes qui n'aboutissent à aucun résultat de recherche. Cela vous permet de comprendre les éventuelles lacunes de votre contenu, lorsqu'aucun document ne répond à certaines requêtes. Vos utilisateurs peuvent également effectuer des requêtes à l'aide de termes spécialisés qui ne conduiront probablement à aucun résultat de recherche, ce qui vous invitera à créer des synonymes personnalisés pour gérer cela.

Les indicateurs incluent le nombre de fois où la requête n'aboutit à aucun résultat de recherche, la proportion de résultats de recherche nuls pour la requête et la proportion de fois que la requête est recherchée par rapport à toutes les requêtes.

Pour récupérer des données sur les principales requêtes sans aucun résultat de recherche à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asQUERIES_BY_ZERO_RESULT_RATE. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation de la console, puis en sélectionnant Top zero Requêtes sous Listes de requêtes.

Les documents sur lesquels vous avez cliqué le plus

Les 100 documents les plus consultés dans les résultats de recherche. Cela vous permet de comprendre quels documents ou résultats de recherche sont les plus pertinents pour vos utilisateurs lorsqu'ils demandent des informations.

Les indicateurs incluent le nombre de fois où le document est cliqué, le nombre de likes qu'un document reçoit de la part de vos utilisateurs (pouce levé), le nombre de dégoûts qu'un document reçoit de la part de vos utilisateurs (pouce vers le bas).

Pour récupérer les données des documents sur lesquels vous avez cliqué en haut à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asDOCS_BY_CLICK_COUNT. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation de la console, puis en sélectionnant Documents les plus consultés sous Listes de requêtes.

Total des requêtes

Le nombre total de requêtes recherchées par vos utilisateurs. Cela vous permet de comprendre dans quelle mesure vos utilisateurs sont intéressés par votre application de recherche.

Pour récupérer des données sur le nombre total de requêtes à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asAGG_QUERY_DOC_METRICS. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation.

Total des documents

Nombre total de documents dans votre index. Cela vous permet de comparer la taille de votre index au nombre total de requêtes afin de vérifier s'il existe un nombre approprié de documents par rapport au volume de requêtes.

Pour récupérer des données sur le nombre total de documents à l'aide de l'GetSnapshotsAPI, spécifiez lemetricType asAGG_QUERY_DOC_METRICS. Vous pouvez également consulter cette métrique dans la console en sélectionnant Analytics dans le panneau de navigation.

Console
Pour récupérer les requêtes les plus fréquentes du mois précédent

  1. Dans le volet de navigation de gauche, sous Index, sélectionnez votre index, puis sélectionnez Analytics.

  2. Sur la page Analytics, cliquez sur le bouton Cette semaine pour remplacer la fenêtre horaire de récupération des données par Mois précédent.

  3. Sur la page Analytics, sous Listes de requêtes, sélectionnez Principales requêtes.

--interval "ONE_MONTH_AGO" \
--metric-type "QUERIES_BY_COUNT"

Pour récupérer les requêtes les plus fréquentes du mois précédent

package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.GetSnapshotsRequest;
import software.amazon.awssdk.services.kendra.model.GetSnapshotsResponse;

public class TopQueriesExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String indexId = "indexID";
        String interval = "ONE_MONTH_AGO";
        String metricType = "QUERIES_BY_COUNT";

        GetSnapshotsRequest getSnapshotsRequest = GetSnapshotsRequest
            .builder()
            .indexId(indexId)
            .interval(interval)
            .metricType(metricType)
            .build();

        GetSnapshotsResponse getSnapshotsResponse = kendra.getSnapshots(GetSnapshotsRequest);

        System.out.println(String.format("Top queries data: ", getSnapshotsResponse.snapshotsData()))

Des indicateurs aux informations exploitables

Les informations exploitables sont des informations significatives extraites de données brutes et utilisées pour orienter vos actions ou vos décisions. Pour tirer parti des indicateurs et les utiliser pour générer des informations exploitables, il est important non seulement d'examiner les indicateurs de manière isolée, mais également d'établir des liens entre les indicateurs.

Par exemple, la première requête sans aucun clic est « Quelles régions sont actuellement disponibles ? ». Cependant, il a également un taux de réponse instantanée de 100 %. Cela suggère à vos utilisateurs de recevoir la réponse à cette question sans avoir à cliquer sur un résultat de recherche ou un document fournissant des informations sur les régions disponibles. Si vous considérez uniquement l'absence de clics, vous n'obtiendrez pas l'information complète et vous pourriez tirer de mauvaises conclusions quant à la réussite de la configuration de votre application de recherche dans le traitement de cette requête.

La découverte d'une opportunité commerciale est un autre exemple d'information exploitable. Les entreprises recherchent souvent des opportunités de développer leurs clients en analysant les indicateurs de recherche. Le document le plus cliqué est « Régions disponibles ». En outre, la plupart des requêtes les plus recherchées concernent des questions sur la disponibilité des produits dans la région océanique, avec des taux de réponse instantanée de 100 % et un taux de clics élevé vers des informations supplémentaires sur les régions disponibles dans le cadre de la réponse. Cela indique que votre produit ou service suscite un intérêt et une demande dans cette région.

Visualisation et génération de rapports sur les analyses de recherche

Cinq indicateurs incluent des données de tendances que vous pouvez visualiser et rechercher des tendances ou des modèles au fil du temps. Si vous utilisez la console, des graphiques des données de tendances sont fournis. Si vous utilisez les API, vous pouvez récupérer les données de tendances pour créer vos propres graphiques ou visualisations. La plupart des graphiques de la console tracent les points de données quotidiens sur la fenêtre horaire que vous avez choisie.

La console fournit un tableau de bord contenant les statistiques dans lequel vous pouvez sélectionner un graphique et une liste des principaux indicateurs que vous souhaitez consulter. Vous pouvez exporter les statistiques affichées sur votre tableau de bord au format CSV en sélectionnant Exporter sur la page d'accueil d'Analytics. Vous pouvez inclure ces rapports dans vos documents commerciaux ou vos présentations.

Vous pouvez visualiser les indicateurs suivants :

Graphique du total des requêtes

Un graphique linéaire du nombre de requêtes émises par jour. Le graphique vous aide à visualiser les tendances de l'engagement quotidien des utilisateurs. Parmi les exemples, citons une augmentation ou une diminution constante de l'engagement des utilisateurs, ou une baisse drastique à 0 requêtes en raison d'un blocage de votre application de recherche ou de problèmes avec votre site Web.

Si vous utilisez l'API, vous pouvez récupérer ces données en les spécifiantTREND_QUERY_DOC_METRICS. Vous pouvez utiliser les données pour créer vos propres graphiques, ou utiliser les graphiques fournis dans la console.

Graphique du taux de clics

Un graphique linéaire des proportions de clics par jour. Le graphique vous aide à visualiser les tendances du taux de clics quotidien. Parmi les exemples, citons une augmentation ou une diminution constante du taux de clics, ou une diminution du nombre de réponses instantanées susceptible d'influencer une augmentation du taux de clics.

Si vous utilisez l'API, vous pouvez récupérer ces données en les spécifiantTREND_QUERY_DOC_METRICS. Vous pouvez utiliser les données pour créer vos propres graphiques, ou utiliser les graphiques fournis dans la console.

Graphique du taux de clic nul

Un graphique linéaire de la proportion de clics nuls par jour. Le graphique vous aide à visualiser les tendances en matière de taux de zéro clic quotidien. Parmi les exemples, citons une augmentation ou une diminution constante du taux de zéro clic, ou une augmentation du nombre de réponses instantanées susceptible d'influencer une augmentation du taux de zéro clic.

Si vous utilisez l'API, vous pouvez récupérer ces données en les spécifiantTREND_QUERY_DOC_METRICS. Vous pouvez utiliser les données pour créer vos propres graphiques, ou utiliser les graphiques fournis dans la console.

Graphique du taux de résultats de recherche nuls

Un graphique linéaire de la proportion de résultats de recherche nuls par jour. Le graphique vous aide à visualiser les tendances en termes de taux de résultats de recherche quotidiens nuls. Parmi les exemples, citons une augmentation ou une diminution constante du taux de résultats de recherche nuls, ou une forte diminution du nombre de documents dans votre index susceptible d'influencer une augmentation de l'absence de résultats de recherche.

Si vous utilisez l'API, vous pouvez récupérer ces données en les spécifiantTREND_QUERY_DOC_METRICS. Vous pouvez utiliser les données pour créer vos propres graphiques, ou utiliser les graphiques fournis dans la console.

Graphique du taux de réponse instantané

Un graphique linéaire représentant la proportion de requêtes pour lesquelles une réponse instantanée ou une FAQ a été renvoyée. Parmi les exemples, citons l'augmentation ou la diminution constante des requêtes de type question-réponse, ou une diminution du nombre de clics susceptible d'influencer une augmentation du nombre de réponses instantanées.
