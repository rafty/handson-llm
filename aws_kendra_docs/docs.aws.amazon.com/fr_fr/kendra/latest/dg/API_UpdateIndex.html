Paramètres de demande

Pour plus d'informations sur les paramètres communs à toutes les actions, voir Paramètres communs.

Cette demande accepte les données suivantes au format JSON.

CapacityUnits

Définit le nombre d'unités supplémentaires de stockage de documents et de capacité de requête qui doivent être utilisées par l'index. Vous pouvez modifier la capacité de l'index jusqu'à 5 fois par jour ou effectuer 5 appels d'API.

Si vous utilisez des unités de stockage supplémentaires, vous ne pouvez pas réduire la capacité de stockage en dessous de ce qui est nécessaire pour répondre aux besoins de stockage de votre index.

Type : objet CapacityUnitsConfiguration

Obligatoire : non

Description

Nouvelle description de l'index.

Type : String

Contraintes de longueur : longueur minimale de 0. Longueur maximum de 1 000.

Modèle : ^\P{C}*$

Obligatoire : non

DocumentMetadataConfigurationUpdates

La configuration des métadonnées du document que vous souhaitez mettre à jour pour l'index. Les métadonnées des documents sont des champs ou des attributs associés à vos documents. Par exemple, le nom du service de l'entreprise associé à chaque document.

Type : tableau d’objets DocumentMetadataConfiguration

Membres du tableau : nombre minimum de 0 élément. Nombre maximal de 500 éléments.

Obligatoire : non

Id

Identifiant de l'index à mettre à jour.

Type : String

Contraintes de longueur : longueur fixe de 36.

Modèle : [a-zA-Z0-9][a-zA-Z0-9-]*

Obligatoire : oui

Name

Nom de l'index que vous voulez mettre à jour.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximum de 1 000.

Modèle : [a-zA-Z0-9][a-zA-Z0-9_-]*

Obligatoire : non

RoleArn

RôleAWS Identity and Access Management (IAM) qui donne à Amazon Kendra les autorisations d'accéder aux CloudWatch journaux et statistiques Amazon.

Type : String

Contraintes de longueur : longueur minimum de 0. Longueur maximale de 1284.

Modèle : arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Obligatoire : non

UserContextPolicy

La politique de contexte utilisateur.

Type : String

Valeurs valides : ATTRIBUTE_FILTER | USER_TOKEN

Obligatoire : non

UserGroupResolutionConfiguration

Permet de récupérer les niveaux d'accès des groupes et des utilisateurs à partir d'une sourceAWS IAM Identity Center (successor to AWS Single Sign-On) d'identité. Pour configurer cela, reportez-vous à la section UserGroupResolutionConfiguration.

Type : objet UserGroupResolutionConfiguration

Obligatoire : non

UserTokenConfigurations

Configuration du jeton utilisateur.

Type : tableau d’objets UserTokenConfiguration

Membres du tableau : nombre maximum de 1 élément.

Obligatoire : non

Éléments de réponse

Si l'action aboutit, le service renvoie une réponse HTTP 200 avec un corps HTTP vide.

Erreurs

Pour plus d'informations sur les erreurs communes à toutes les actions, consultezErreurs courantes.

AccessDeniedException

Vous ne disposez pas de suffisamment d'accéder pour effectuer cette action. Assurez-vous de disposer des politiques d'autorisation et des comptes utilisateur requis, puis réessayez.

Code d'état HTTP : 400

ConflictException

Un conflit est survenu avec la demande. Corrigez toute incohérence avec vos ressources et réessayez.

Code d'état HTTP : 400

InternalServerException

Un problème s'est produit avec le serveur interne utilisé pour votre service Amazon Kendra. Veuillez patienter quelques minutes et réessayez, ou contactez le support pour obtenir de l'aide.

Code d'état HTTP : 500

ResourceNotFoundException

La ressource que vous souhaitez utiliser n'existe pas. Vérifiez que vous avez fourni la bonne ressource et réessayez.

Code d'état HTTP : 400

ServiceQuotaExceededException

Vous avez dépassé les limites définies pour votre service Amazon Kendra. Consultez la section Quotas pour plus d'informations, ou contactez le support pour en savoir plus sur une augmentation des limites.

Code d'état HTTP : 400

ThrottlingException

La demande a été refusée suite à une limitation des demandes. Réduisez le nombre de demandes et réessayez.

Code d'état HTTP : 400

ValidationException

L'entrée ne satisfait pas les contraintes définies par le service Amazon Kendra. Entrez les informations correctes et réessayez.

Code d'état HTTP : 400
