Amazon KendraCrawler Web

Vous pouvez utiliserAmazon Kendra Web Crawler pour explorer et indexer des pages Web.

Vous ne pouvez indexer que les sites web publics et les sites web qui utilisent le protocole de communication sécurisé, Hypertext Transfer Protocol Secure (HTTPS). Si vous recevez un message d'erreur lors de l'indexation d'un site web, cela signifie peut-être que l'indexation du site web est bloquée. Pour explorer des sites Web internes, vous pouvez configurer un proxy Web. Le proxy Web doit être accessible au public.

Lorsque vous sélectionnez des sites web à indexer, vous devez respecter les Politiques d'Amazon en matière d'utilisation acceptable et toutes les autres conditions d'Amazon. N'oubliez pas que vous ne devez utiliserAmazon Kendra Web Crawler ou les pages web que vous êtes autorisé à indexer. Pour savoir comment empêcherAmazon Kendra Web Crawler d'indexer vos sites Web, consultezConfiguration durobots.txt fichier pourAmazon Kendra Web Crawler.

Vous pouvez vous connecterAmazon Kendra à la source de données de votre robot d'exploration Web à l'aide de la Amazon Kendraconsole et de l'WebCrawlerConfigurationAPI.

Pour résoudre les problèmes liés au connecteur de source de données de votre robot d'explorationAmazon Kendra Web, consultezRésolution des problèmes de sources de données.

Amazon KendraLe connecteur de source de données Web Crawler prend en charge les fonctionnalités suivantes :

  • Proxy web

  • Filtres d'inclusion/d'exclusion

Prérequis

Avant de pouvoirAmazon Kendra indexer vos sites Web, vérifiez les détails de vos sites Web et de vosAWS comptes.

Pour vos sites web, assurez-vous de disposer des éléments suivants

  • Vous avez copié les URL de départ ou de plan des sites web que vous souhaitez indexer.

  • Pour les sites Web qui nécessitent une authentification de base pour être explorés : vous avez copié le nom d'hôte du site Web et le numéro de port.

  • Facultatif : Vous avez copié le nom d'hôte du site Web et le numéro de port si vous souhaitez utiliser un proxy Web pour vous connecter aux sites Web internes que vous souhaitez explorer. Le proxy Web doit être accessible au public. Amazon Kendraprend en charge la connexion à des serveurs proxy Web soutenus par une authentification de base ou vous pouvez vous connecter sans authentification.

  • Coché, chaque document de page Web que vous souhaitez indexer est unique et appartient à d'autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer des éléments suivants

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké les informations d'authentification de votre robot d'exploration Web dans unAWSSecrets Manager secret et, si vous utilisez l'API, avez noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez la source de données de votre robot d'exploration Web àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à la source de données de votre robot d'exploration Web, vous devez fournir les informations nécessaires sur la source de données de votre robot d'exploration Web afin queAmazon Kendra vous puissiez accéder à vos données. Si vous n'avez pas encore configuré le robot d'exploration Web pourAmazon Kendra voirPrérequis.

Console

Pour vousAmazon Kendra connecter au robot d'exploration

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez le connecteur Web Crawler, puis choisissez Ajouter une source de données.

  5. Sur la page Spécifier les détails de la source de données, entrez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. (Facultatif) Description : entrez une description facultative pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Balises, pour Ajouter une nouvelle étiquette : des balises permettant de rechercher et de filtrer vos ressources ou de suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, entrez les informations suivantes :

    1. Pour Source, choisissez entre les URL source et les sitemaps sources en fonction de votre cas d'utilisation et entrez les valeurs pour chacun.

      Note

      Vous pouvez ajouter jusqu'à 10 URL source et 3 plans.

      Si vous souhaitez explorer un plan du site, il est important de vérifier si l'URL de base ou racine est la même que celle utilisée pour les URL répertoriées sur votre page de plan de site. Par exemple, si l'URL de votre plan de site est https://example.com/sitemap-page.html, les URL répertoriées sur cette page de plan de site doivent également utiliser l'URL de base « https://example.com/ ».

    2. (Facultatif) Pour le proxy Web, entrez les informations suivantes :

      1. Nom d'hôte : nom d'hôte pour lequel le proxy Web est requis.

      2. Numéro de port : port utilisé par le protocole de transport de l'URL hôte. Le numéro de port doit être une valeur numérique comprise entre 0 et 65535.

      3. Pour les informations d'identification du proxy Web : choisissez un secret existant ou créez-en un nouveau pour stocker les informations d'authentification de votre robot d'exploration Web. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      4. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -WebCrawler - » est automatiquement ajouté à votre nom secret.

        2. Pour le nom d'utilisateur et le mot de passe : entrez ces informations d'identification de base pour vos sites Web.

        3. Choisissez Save (Enregistrer).

    3. (Facultatif) Hôtes authentifiés : sélectionnez cette option pour ajouter d'autres hôtes authentifiés.

    4. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification et au contenu de l'index.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    5. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, entrez les informations suivantes :

    1. Plage d'exploration : choisissez le type de pages Web que vous souhaitez explorer.

    2. Profondeur d'exploration : sélectionnez le nombre de niveaux à partir de l'URL de départ quiAmazon Kendra doivent être explorés.

    3. Les paramètres d'exploration avancés et la configuration supplémentaire permettent de saisir les informations suivantes :

      1. Taille de fichier maximale : taille maximale de page Web ou de pièce jointe à explorer. Minimum 0,000001 Mo (1 octet). 50 Mo maximum.

      2. Nombre maximum de liens par page : nombre maximum de liens explorés par page. Les liens sont indexés par ordre d'apparition. Minimum 1 lien/page. Maximum de 1 000 liens par page.

      3. Limitation maximale : le nombre maximal d'URL indexées par nom d'hôte et par minute. Minimum 1 URL/nom d'hôte/minute. Maximum de 300 URL/nom d'hôte/minute.

      4. Modèles Regex : ajoutez un modèle pour n'inclure que certaines URL, ou pour exclure des URL, des types de fichiers ou des fichiers spécifiques de votre référentiel. Vous pouvez obtenir un total combiné de 100 motifs.

    4. Dans Synchroniser le calendrier d'exécution, pour Fréquence, choisissez la fréquence àAmazon Kendra laquelle vous souhaitez synchroniser avec votre source de données.

    5. Choisissez Suivant.

  8. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vousAmazon Kendra connecter au robot d'exploration

Vous devez spécifier l'WebCrawlerConfigurationAPI suivante :

  • URL : Spécifiez les URL de départ ou de point de départ des sites web que vous souhaitez indexer à l'aide de SeedUrlConfigurationet SiteMapsConfiguration.

    Si vous souhaitez explorer un plan du site, il est important de vérifier si l'URL de base ou racine est la même que celle utilisée pour les URL répertoriées sur votre page de plan de site. Par exemple, si l'URL de votre plan de site est https://example.com/sitemap-page.html, les URL répertoriées sur cette page de plan de site doivent également utiliser l'URL de base « https://example.com/ ».

  • Secret Amazon Resource Name (ARN) : si un site Web nécessite une authentification de base, vous devez fournir le nom d'hôte, le numéro de port et un code secret qui stocke vos informations d'authentification de base, à savoir votre nom d'utilisateur et votre mot de passe. Vous fournissez l'ARN à l'aide de l'AuthenticationConfigurationAPI. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "username": "user name",
    "password": "password"
}

    Vous pouvez fournir les informations d'identification du proxy Web en utilisant un secretAWSSecrets Manager secret. Vous utilisez l'ProxyConfigurationAPI pour fournir le nom d'hôte et le numéro de port du site Web, et éventuellement le secret qui stocke les informations d'identification de votre proxy Web.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Mode d'exploration : choisissez d'explorer uniquement les noms d'hôtes des sites Web ou les noms d'hôtes avec des sous-domaines, ou d'explorer également d'autres domaines vers lesquels les pages Web renvoient.

  • La « profondeur » ou le nombre de niveaux d'un site web, depuis le niveau de départ jusqu'au niveau de l'indexation. Par exemple, si un site web a 3 niveaux : le niveau de l'index (le niveau de départ dans cet exemple), le niveau des sections et le niveau des sous-sections. Si vous n'êtes intéressé que par l'indexation des informations du niveau de l'index au niveau des sections (niveaux 0-1), vous pouvez définir votre profondeur à 1.

  • Le nombre maximal d'URL à indexer.

  • Taille maximale en Mo d'une page Web à explorer.

  • Le nombre maximal d'URL indexées par hôte de site web et par minute.

  • Informations sur le proxy Web permettant de se connecter à des sites Web internes et de les explorer.

  • Les informations d'authentification permettant d'accéder et de indexer des sites web qui nécessitent une authentification utilisateur.

  • Vous pouvez extraire des balises méta HTML sous forme de champs à l'aide de l'outil d'enrichissement personnalisé des documents. Pour plus d'informations, veuillez consulter la rubrique Personnalisation des métadonnées de documents pendant le processus d'intégration. Pour un exemple d'extraction de balises méta HTML, consultez les exemples CDE.

  • Filtres d'inclusion et d'exclusion : spécifiez si vous souhaitez inclure ou exclure certaines URL.

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.
