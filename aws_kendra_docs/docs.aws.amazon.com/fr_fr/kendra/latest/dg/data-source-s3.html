Amazon S3

Amazon S3est un service de stockage d'objets qui stocke les données en tant qu'objets dans des compartiments. Vous pouvez l'utiliserAmazon Kendra pour indexerAmazon S3 votre dépôt de documents.

Avertissement

Amazon Kendran'utilise pas de politique de compartiment qui autorise unAmazon Kendra principal à interagir avec un compartiment S3. Il utilise plutôt desIAM rôles. Assurez-vous qu'il ne figureAmazon Kendra pas en tant que membre de confiance dans votre politique relative aux compartiments afin d'éviter tout problème de sécurité des données lié à l'octroi accidentel d'autorisations à des responsables arbitraires. Vous pouvez toutefois ajouter une politique de compartiment pour utiliser unAmazon S3 compartiment sur différents comptes. Pour plus d'informations, consultez la section Politiques àAmazon S3 utiliser entre les comptes (dans l'ongletIAM Rôles S3, sous IAMRôles pour les sources de données). Pour plus d'informations surIAM les rôles des sources de données S3, consultez la section IAMRôles.

Pour vous connecter àAmazon S3, vous devez spécifier la connexion et d'autres informations dans la console, à l'aide de S3DataSourceConfiguration ou du TemplateConfiguration. Si vous utilisez l' TemplateConfiguration objet, vous pouvez utiliser un VPC pour vous connecter à votre source de données. Vous précisez VpcConfiguration quand vous appelez CreateDataSource. Pour plus d'informations, veuillez consulter la rubrique Configuration d'un VPC.

Note

S3DataSourceConfigurationL'API ne prend pas en charge la configuration d'un VPC. SeulesTemplateConfiguration l'API et la console prennent en charge la configuration VPC.

Pour résoudre les problèmes liés à votre connecteur de source de donnéesAmazon Kendra S3, consultezRésolution des problèmes de sources de données.

Prérequis

Avant de pouvoir l'utiliserAmazon Kendra pour indexer votre source de données S3, apportez ces modifications à votre S3 et à vosAWS comptes.

Dans S3, assurez-vous de disposer des éléments suivants :

  • Copiez le nom de votreAmazon S3 compartiment.

    Note

    Votre compartiment doit se trouver dans la même région que votreAmazon Kendra index et votre index doit être autorisé à accéder au compartiment qui contient vos documents.

  • Coché, chaque document est unique dans S3 et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Vérifiez que vousAWS avez les éléments suivants lors de l'enregistrement :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

Si vous n'avez pas deIAM rôle existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle lorsque vous connectez votre source de données S3 àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle existant et un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données S3, vous devez fournir les informations nécessaires sur votre source de données S3 afin que vousAmazon Kendra puissiez accéder à vos données. Si vous n'avez pas encore configuré S3 pourAmazon Kendra voirPrérequis.

Console

Pour vous connecterAmazon Kendra àAmazon S3

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez le connecteur S3, puis choisissez Ajouter une source de données.

  5. Sur la page Spécifier les informations relatives à la source de données, entrez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) : entrez une description si vous le souhaitez pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Balises, pour Ajouter une nouvelle étiquette : balises pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, entrez les informations facultatives suivantes :

    1. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification de votre référentiel et au contenu d'index.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    2. Virtual Private Cloud (VPC) : vous pouvez choisir d'utiliser un VPC. Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    3. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, entrez les informations suivantes :

    1. Dans Sync Scope, pour Emplacement de la source de données : chemin d'accès auAmazon S3 compartiment dans lequel vos données sont stockées. Sélectionnez Browse S3 pour choisir votre compartiment.

    2. (Facultatif) Les fichiers de métadonnées préfixent l'emplacement du dossier : chemin d'accès au dossier dans lequel vos métadonnées sont stockées. Sélectionnez Browse S3 pour localiser votre dossier de métadonnées.

    3. (Facultatif) Emplacement du fichier de configuration de la liste de contrôle d'accès : chemin d'accès à l'emplacement d'un fichier contenant une structure JSON qui spécifie les paramètres d'accès pour les fichiers stockés dans votre source de données S3. Sélectionnez Browse S3 pour localiser votre fichier ACL.

    4. (Facultatif) Sélectionnez la clé de déchiffrement : sélectionnez cette option pour utiliser une clé de déchiffrement. Vous pouvez choisir entre utiliser uneAWS KMS clé existante ou créer une nouvelle.

    5. (Facultatif) Dans Configuration supplémentaire, pour les modèles Regex : ajoutez des modèles d'expressions régulières pour inclure ou exclure des documents de votre index. Tous les chemins sont relatifs au compartiment S3 de l'emplacement de la source de données. Vous pouvez ajouter jusqu'à 100 motifs.

    6. En mode synchronisation, choisissez entre le mode de synchronisation complète et la synchronisation du contenu nouveau, modifié ou supprimé pour déterminer comment votre index est mis à jour lorsque le contenu de votre source de données change. Lorsque vous synchronisez votre source de données avec Amazon Kendra pour la première fois, tout le contenu est synchronisé par défaut.

    7. Dans Synchroniser le calendrier d'exécution, pour Fréquence, choisissez la fréquence àAmazon Kendra laquelle vous souhaitez synchroniser avec votre source de données.

    8. Choisissez Suivant.

  8. Sur la page Définir les mappages de champs, entrez les informations facultatives suivantes :

    1. Mappage de champs S3 : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : choisissez d'ajouter des champs de source de données personnalisés pour créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

TemplateConfiguration API

Pour vous connecterAmazon Kendra àAmazon S3

Vous devez spécifier un JSON du schéma de source de données à l'aide de l'TemplateConfigurationAPI. Vous devez fournir les informations suivantes lors de l'enregistrement :

  • BucketName—Nom du compartiment qui contient les documents.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pourIAM attribuer un rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le connecteur S3 etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de données S3.

Vous pouvez également ajouter les fonctions facultatives suivantes :

  • Filtres d'inclusion et d'exclusion : spécifiez s'il faut inclure ou exclure certains noms de fichiers, types de fichiers, chemins de fichiers et utiliser des modèles globulaires (modèles qui peuvent transformer un modèle générique en une liste de noms de chemin correspondant au modèle donné). Voici des exemples de motifs de globes :

    • /myapp/config/*—Tous les fichiers contenus dans le répertoire de configuration

    • /**/*.png—Tous les fichiers .png dans tous les répertoires

    • /**/*.{png,ico,md}—Tous les fichiers .png, .ico ou .md dans tous les répertoires

    • /myapp/src/**/*.ts—Tous les fichiers .ts contenus dans le répertoire src (et tous ses sous-répertoires)

    • **/!(*.module).ts—Tous les fichiers .ts mais pas .module.ts

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de données S3 à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Virtual Private Cloud (VPC) : spécifiez àVpcConfiguration quel moment vous appelezCreateDataSource. Consultez ConfigurationAmazon Kendra pour utiliser un VPC.

  • Configuration des métadonnées de document : ajoutez des fichiers de métadonnées de document qui contiennent des informations telles que les informations de contrôle d'accès au document, l'URI source, l'auteur du document et les attributs personnalisés. Chaque fichier de métadonnées contient des métadonnées relatives à un seul document.

Pour obtenir la liste des autres clés JSON importantes à configurer, consultezAmazon S3schéma de modèle.
