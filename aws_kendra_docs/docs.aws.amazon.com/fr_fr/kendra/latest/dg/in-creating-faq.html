Ajout de questions fréquentes (FAQ) directement à un index

Vous pouvez ajouter des questions fréquemment posées (FAQ) directement à votre index à l'aide de la console ou de l'CreateFaqAPI. L'ajout de FAQ à un index constitue une opération asynchrone. Vous placez les données de la FAQ dans un fichier que vous stockez dans unAmazon Simple Storage Service compartiment. Vous pouvez utiliser des fichiers CSV ou JSON comme entrée pour votre FAQ :

  • CSV de base : fichier CSV dans lequel chaque ligne contient une question, une réponse et une URL facultative contenant plus d'informations sur la réponse.

  • CSV personnalisé : fichier CSV contenant des questions, des réponses et un en-tête qui définit des attributs personnalisés que vous pouvez utiliser pour facetter, afficher ou trier les réponses aux FAQ. Vous pouvez également définir des attributs de contrôle d'accès afin de limiter la réponse à la FAQ à certains utilisateurs et groupes.

  • JSON : fichier JSON contenant des questions et des réponses. Facultativement, il peut également inclure des attributs personnalisés et de contrôle d'accès. Vous pouvez définir des attributs pour facetter, afficher et trier les réponses aux FAQ. Vous pouvez également utiliser des attributs de contrôle d'accès qui limitent la réponse à la FAQ à certains utilisateurs et groupes.

Par exemple, ce qui suit est un fichier CSV de base qui fournit des réponses aux questions concernant les cliniques gratuites à Spokane, dans l'État de Washington, aux États-Unis, et à Mountain View, dans le Missouri, aux États-Unis.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

Lorsque vous utilisez un fichier CSV ou JSON personnalisé pour la saisie, vous pouvez déclarer des attributs personnalisés pour vos questions de FAQ. Par exemple, vous pouvez créer un attribut personnalisé qui attribue un service à chaque question de la FAQ. Lorsque la FAQ est renvoyée dans une réponse, vous pouvez utiliser le département comme facette pour affiner la recherche.

Un attribut personnalisé doit correspondre à un champ d'index. Vous pouvez utiliser un champ intégré ou spécifier un champ d'index personnalisé. Dans la console, vous utilisez la page de définition des facettes pour créer un champ d'index. Lorsque vous utilisez l'API, vous devez d'abord créer un champ d'index au moyen de l'UpdateIndexAPI.

Le type d'attribut dans le fichier FAQ doit correspondre au type du champ d'index associé. Par exemple, le_authors champ intégré est un champSTRING_LIST de type. Vous devez donc fournir des valeurs pour le_authors champ sous forme de liste de chaînes dans votre fichier FAQ. Vous pouvez vérifier le type de champs d'index à l'aide de la page de définition des facettes de la console ou à l'aide de l'DescribeIndexAPI.

Lorsque vous créez un champ d'index mappé à un attribut personnalisé, vous pouvez le marquer comme affichable, facetable ou triable. Vous ne pouvez pas rendre un attribut personnalisé accessible par la recherche.

Outre les attributs personnalisés, vous pouvez également utiliser les attributs intégrés suivants dans un fichier CSV ou JSON personnalisé :

  • _authors(Liste de chaînes) : liste des auteurs des réponses aux questions de la FAQ.

  • _category(String) : catégorie qui regroupe les réponses aux questions de la FAQ avec d'autres documents similaires.

  • _created_at(chaîne codée ISO 8601) : date et heure de création de la question FAQ. La date et l'heure doivent être formatées sous la forme d'une chaîne codée ISO 8601.

    Vous pouvez également inclure le fuseau horaire au format de date et d'heure ISO 8601 si nécessaire. Par exemple, 2012-03-25T 12:30:10 + 01:00 est le format de date et d'heure ISO 8601 pour le 25 mars 2012 à 12 h 30 (plus 10 secondes) dans le fuseau horaire d'Europe centrale.

  • _last_updated_at(chaîne codée ISO 8601) : date et heure de mise à jour de la question FAQ. La date et l'heure doivent être formatées sous la forme d'une chaîne codée ISO 8601.

    Par exemple, 2012-03-25T 12:30:10 + 01:00 est le format de date et d'heure ISO 8601 pour le 25 mars 2012 à 12 h 30 (plus 10 secondes) dans le fuseau horaire d'Europe centrale.

  • _source_uri(String) : URL d'un document contenant plus d'informations sur la réponse à la FAQ.

  • _version(String) : version de la question de la FAQ.

  • _view_count(Long) —Nombre de fois que la question de la FAQ a été consultée dans les résultats de recherche.

Fichier CSV de base

Utilisez un fichier CSV de base lorsque vous souhaitez utiliser une structure simple pour vos FAQ. Dans un fichier CSV de base, chaque ligne comporte deux ou trois champs : une question, une réponse et un URI source facultatif qui pointe vers un document contenant plus d'informations.

Le contenu du fichier doit respecter le format commun RFC 4180 et le type MIME pour les fichiers à valeurs séparées par des virgules (CSV).

Ce qui suit est un fichier FAQ au format CSV de base.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

Fichier CSV personnalisé

Utilisez un fichier CSV personnalisé lorsque vous souhaitez ajouter des attributs personnalisés à vos questions de FAQ. Pour un fichier CSV personnalisé, vous utilisez une ligne d'en-tête dans votre fichier CSV pour définir les attributs supplémentaires.

Le fichier CSV doit contenir les deux attributs obligatoires suivants :

  • _question—La question fréquemment posée

  • _answer—La réponse à la question fréquemment posée

Votre fichier peut contenir des attributs intégrés et personnalisés. Voici un exemple de fichier CSV personnalisé.

_question,_answer,_last_updated_at,custom_string
How many free clinics are in Spokane WA?, 13, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services
How many free clinics are there in Mountain View Missouri?, 7, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services

Le contenu du fichier d'attributs personnalisés doit respecter le format commun RFC 4180 et le type MIME pour les fichiers à valeurs séparées par des virgules (CSV).

Il existe quatre types d'attributs personnalisés :

  • Date : valeurs de type date et heure encodées en ISO 8601.

    Par exemple, 2012-03-25T 12:30:10 + 01:00 est le format de date et d'heure ISO 8601 pour le 25 mars 2012 à 12 h 30 (plus 10 secondes) dans le fuseau horaire d'Europe centrale.

  • Longue : nombres, tels que1234.

  • Chaîne : valeurs de chaîne. Si votre chaîne contient des virgules, placez la valeur entière entre guillemets doubles («) (par exemple,"custom attribute, and more").

  • Liste de chaînes : liste de valeurs d'une chaîne. Répertoriez les valeurs dans une liste séparée par des virgules et entre guillemets («) (par exemple,"item1, item2, item3"). Si la liste ne contient qu'une seule entrée, vous pouvez omettre les guillemets (par exemple,item1).

Un fichier CSV personnalisé peut contenir des champs de contexte utilisateur. Vous pouvez utiliser ces champs pour limiter l'accès à la FAQ à certains utilisateurs et groupes. Pour filtrer en fonction du contexte utilisateur, l'utilisateur doit fournir des informations sur l'utilisateur et le groupe dans la requête. Sinon, toutes les FAQ pertinentes sont renvoyées. Pour plus d'informations, veuillez consulter Filtrage en fonction du contexte utilisateur.

Il existe quatre filtres contextuels utilisateur pour les FAQ :

  • _acl_user_allow—Les utilisateurs figurant dans la liste d'autorisation peuvent consulter la FAQ dans la réponse à la requête. La FAQ n'est pas renvoyée aux autres utilisateurs.

  • _acl_user_deny—Les utilisateurs figurant dans la liste de refus ne peuvent pas voir la FAQ dans la réponse à la requête. La FAQ est renvoyée à tous les autres utilisateurs lorsqu'elle est pertinente pour la requête.

  • _acl_group_allow—Les utilisateurs membres d'un groupe autorisé peuvent consulter la FAQ dans la réponse à la requête. La FAQ n'est pas renvoyée aux utilisateurs membres d'un autre groupe.

  • _acl_group_deny—Les utilisateurs membres d'un groupe refusé ne peuvent pas voir la FAQ dans la réponse à la requête. La FAQ est renvoyée aux autres groupes lorsqu'elle est pertinente pour la requête.

Fournissez les valeurs des listes d'autorisation et de refus dans des listes séparées par des virgules et placées entre guillemets (par exemple,"user1,user2,user3"). Vous pouvez inclure un utilisateur ou un groupe dans une liste d'autorisation ou de refus, mais pas dans les deux. Si vous incluez un utilisateur ou un groupe dans les deux, vous recevez une erreur.

Voici un exemple de fichier CSV personnalisé contenant des informations de contexte utilisateur.

_question, _answer, _acl_user_allow, _acl_user_deny, _acl_group_allow, _acl_group_deny
How many free clinics are in Spokane WA?, 13, userID4565, "userID6201, userID7552", groupBasicRate, "groupBasicPlusRate,groupPremiumRate"

Fichier JSON

Vous pouvez utiliser un fichier JSON pour fournir des questions, des réponses et des attributs pour votre index. Vous pouvez ajouter n'importe quel attribut intégré ou personnalisé à la FAQ. Voici le schéma pour le fichier JSON.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": string,
            "Answer": string,
            "Attributes": {
                string: object
                additional attributes
            },
            "AccessControlList": [
               {
                   "Name": string,
                   "Type": enum( "GROUP" | "USER" ),
                   "Access": enum( "ALLOW" | "DENY" )
               },
               additional user context
            ]
        },
        additional FAQ documents
    ]
}

Cet exemple de fichier JSON montre deux documents de FAQ. L'un d'entre eux ne contient que la question et la réponse requises. L'autre comporte également des attributs supplémentaires et des informations contextuelles sur l'utilisateur.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": "How many free clinics are in Spokane WA?",
            "Answer": "13"
        },
        {
            "Question": "How many free clinics are there in Mountain View Missouri?",
            "Answer": "7",
            "Attributes": {
                "_source_uri": "https://www.freeclinics.com",
                "_category": "Charitable Clinics"
            },
            "AccessControlList": [
               {
                   "Name": "user@amazon.com",
                   "Type": "USER",
                   "Access": "ALLOW"
               },
               {
                   "Name": "Admin",
                   "Type": "GROUP",
                   "Access": "ALLOW"
               }
            ]
        }
    ]
}

Il existe quatre types d'attributs personnalisés pour les fichiers JSON :

  • Date : valeur de chaîne JSON avec des valeurs de date et d'heure codées selon la norme ISO 8601.

    Il est important que le fuseau horaire soit inclus dans le format de date et d'heure ISO 8601. Par exemple, 2012-03-25T 12:30:10 + 01:00 est le format de date et d'heure ISO 8601 pour le 25 mars 2012 à 12 h 30 (plus 10 secondes) dans le fuseau horaire d'Europe centrale.

  • Long : valeur numérique JSON, telle que1234.

  • Chaîne : valeur de chaîne JSON (par exemple,"custom attribute").

  • Liste de chaînes : tableau JSON de valeurs de chaîne (par exemple,["item1,item2,item3"]).

Outre les attributs intégrés et personnalisés, vous pouvez fournir des informations contextuelles utilisateur pour la FAQ dans un fichier JSON. Vous pouvez fournir des informations contextuelles sur les utilisateurs afin de limiter l'accès au contenu de la FAQ en fonction des utilisateurs et des groupes. Vous pouvez inclure un utilisateur ou un groupe dans une liste d'autorisation ou de refus, mais pas dans les deux. Si vous incluez un utilisateur ou un groupe dans les deux, vous recevez une erreur. Pour filtrer en fonction du contexte utilisateur, vous devez fournir des informations sur les utilisateurs et les groupes dans la requête. Sinon, toutes les FAQ pertinentes sont renvoyées. Pour plus d'informations, veuillez consulter Filtrage en fonction du contexte utilisateur.

Cet exemple JSON ajoute le contexte utilisateur à une FAQ.

"AccessControlList": [
                {
                    "Name": "group or user name",
                    "Type": "GROUP | USER",
                    "Access": "ALLOW | DENY"
                },
                additional user context
            ]

Utilisation de votre fichier FAQ

Après avoir enregistré votre fichier d'entrée de FAQ dans un compartiment S3, vous utilisez la console ou l'CreateFaqAPI pour placer les questions et réponses dans votre index. Si vous souhaitez mettre à jour une FAQ, supprimez-la et créez-la à nouveau. Vous utilisez l'DeleteFaqAPI pour supprimer une FAQ.

Vous devez fournir unIAM rôle qui donne accès au compartiment S3 qui contient vos fichiers source. Vous spécifiez le rôle dans la console ou dans leRoleArn paramètre. Voici un exemple de programme qui ajoute un fichier FAQ à un index.

Python
import boto3

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the IAM role ARN required to index documents in an S3 bucket
role_arn = "arn:aws:iam::${accountId}:role/${roleName}"

# Provide the S3 bucket path information to the FAQ file
faq_path = {
    "Bucket": "bucket-name",
    "Key": "FreeClinicsUSA.csv"
}

response = kendra.create_faq(
    S3Path =  faq_path,
    Name = "FreeClinicsUSA",
    IndexId = index_id,
    RoleArn = role_arn
)

print(response)
Java
package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.CreateFaqRequest;
import software.amazon.awssdk.services.kendra.model.CreateFaqResponse;
import software.amazon.awssdk.services.kendra.model.S3Path;

public class AddFaqExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String indexId = "yourIndexId";
        String roleArn = "your role for accessing S3 files";

        CreateFaqRequest createFaqRequest = CreateFaqRequest
            .builder()
            .indexId(indexId)
            .name("FreeClinicsUSA")
            .roleArn(roleArn)
            .s3Path(
                S3Path
                    .builder()
                    .bucket("an-aws-kendra-test-bucket")
                    .key("FreeClinicsUSA.csv")
                    .build())
            .build();

        CreateFaqResponse response = kendra.createFaq(createFaqRequest);

        System.out.println(String.format("The result of creating FAQ: %s", response));

    }
}

Fichiers FAQ dans des langues autres que l'anglais

Vous pouvez indexer une FAQ dans une langue prise en charge. Amazon Kendraindexe les FAQ en anglais par défaut si vous ne spécifiez pas de langue. Vous spécifiez le code de langue lorsque vous appelez l'CreateFaqopération ou vous pouvez inclure le code de langue d'une FAQ dans les métadonnées de la FAQ sous forme de champ. Si les métadonnées d'une FAQ ne contiennent aucun code de langue spécifié dans un champ de métadonnées, la FAQ est indexée à l'aide du code de langue spécifié lorsque vous appelez l'CreateFAQopération. Pour indexer un document de FAQ dans une langue prise en charge dans la console, accédez à FAQ et sélectionnez Ajouter une FAQ. Vous choisissez une langue dans la liste déroulante Langue.
