Box (Cube)

Box est un service de stockage dans le cloud qui offre des fonctionnalités d'hébergement de fichiers. Vous pouvez l'utiliserAmazon Kendra pour indexer le contenu de votre Box, notamment les commentaires, les tâches et les liens Web.

Vous pouvez vous connecterAmazon Kendra à votre source de données Box à l'aide de la Amazon Kendraconsole et de l'BoxConfigurationAPI.

Pour résoudre les problèmes liés au connecteur de source de donnéesAmazon Kendra Box, consultezRésolution des problèmes de sources de données.

Fonctionnalités prises en charge

Amazon KendraLe connecteur de source de données Box prend en charge les fonctionnalités suivantes :

  • Journal des modifications

  • Mappages de champs

  • Filtrage du contexte utilisateur

  • Filtres d'inclusion/d'exclusion

  • Virtual Private Cloud (VPC)

Prérequis

Avant de pouvoir l'utiliserAmazon Kendra pour indexer votre source de données Box, apportez ces modifications à votre Box et à vosAWS comptes.

Dans Box, assurez-vous de disposer des éléments suivants :

  • Un compte Box Enterprise ou Box Enterprise Plus.

  • Création d'une application personnalisée Box dans la console de développement Box et configuration de celle-ci pour utiliser l'authentification du serveur (avec JWT). Consultez la documentation Box sur la création d'une application personnalisée et la documentation Box sur la configuration de JWT Auth pour plus de détails.

  • Définissez votre niveau d'accès aux applications sur App + Enterprise Access et autorisez l'application à effectuer des appels d'API à l'aide de l'en-tête en tant qu'utilisateur.

  • Vous avez utilisé l'utilisateur administrateur pour ajouter les domaines d'application suivants dans votre application Box :

    • Écrire tous les fichiers et dossiers stockés dans une boîte

    • Gestion des utilisateurs

    • Gérer les groupes

    • Gérez les propriétés de l'entreprise

  • key pair publique/privée générée et téléchargée comprenant un identifiant client, un secret client, un identifiant de clé publique, un identifiant de clé privée, une phrase secrète et un identifiant d'entreprise à utiliser comme informations d'authentification. Consultez la section Paire de clés publique et privée pour plus de détails.

  • Vous avez copié votre identifiant d'entreprise Box depuis les paramètres de votre Box Developer Console ou depuis votre application Box. Par exemple, 801234567.

  • Coché, chaque document est unique dans Box et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer des éléments suivants :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké vos informations d'authentification Box dans unAWSSecrets Manager secret et, si vous utilisez l'API, avez noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de données Box àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données Box, vous devez fournir les informations nécessaires sur votre source de données Box afin que vousAmazon Kendra puissiez accéder à vos données. Si vous n'avez pas encore configuré Box forAmazon Kendra seePrérequis.

Console

Pour vous connecterAmazon Kendra à Box

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez Box connector, puis choisissez Ajouter une source de données.

  5. Sur la page Spécifier les détails de source de données, entrez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) : entrez une description facultative pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Tags, pour Ajouter un nouveau tag : Tags pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, saisissez les informations suivantes :

    1. Box Enterprise ID : saisissez votre Box Enterprise ID.

    2. AWSSecrets Managersecret —Choisissez un secret existant ou créez-en un nouveauSecrets Manager pour stocker vos informations d'authentification Box. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -Box » est automatiquement ajouté à votre nom secret.

      2. Pour l'identifiant client, le secret client, l'identifiant de clé publique, l'identifiant de clé privée et la phrase secrète, entrez les valeurs de la clé publique/privée que vous avez générée dans votre compte Box et téléchargée depuis votre compte Box.

      3. Choisissez Save (Enregistrer).

    3. Virtual Private Cloud (VPC) Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    4. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification de votre référentiel et indexer le contenu de votre référentiel.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    5. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, entrez les informations suivantes :

    1. Sélectionnez les entités ou les types de contenu : entités Box ou types de contenu que vous souhaitez analyser. Chaque commentaire est indexé en tant que document distinct.

    2. Journal des modifications : sélectionnez cette option pour mettre à jour votre index au lieu de synchroniser tous vos fichiers.

    3. Modèles Regex : modèles d'expression régulière permettant d'inclure ou d'exclure certains fichiers. Vous pouvez ajouter jusqu'à 100 motifs.

    4. Dans Synchroniser le calendrier d'exécution, pour Fréquence, choisissez la fréquence àAmazon Kendra laquelle vous souhaitez synchroniser avec votre source de données.

    5. Choisissez Suivant.

  8. Sur la page Définir les mappages de champs, entrez les informations suivantes :

    1. Pour les fichiers et les dossiers, les commentaires, les tâches et les liens Web : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à Box

Vous devez spécifier les éléments suivants à l'aide de l'BoxConfigurationAPI :

Box Enterprise ID : indiquez votre Box Enterprise ID. Vous pouvez trouver l'identifiant d'entreprise dans les paramètres de Box Developer Console ou lorsque vous créez une application dans Box.

  • Secret Amazon Resource Name (ARN) : indiquez le nom de ressource Amazon (ARN) d'unSecrets Manager code secret contenant les informations d'authentification de votre compte Box. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "clientID": "client-id",
    "clientSecret": "client-secret",
    "publicKeyID": "public-key-id",
    "privateKey": "private-key",
    "passphrase": "pass-phrase"
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pourIAM attribuer un rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le connecteur Box etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de données Box.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Virtual Private Cloud (VPC) : spécifiez dans leVpcConfiguration cadre de la configuration de la source de données. Reportez-vous Amazon Kendraà la section Configuration pour utiliser un VPC.

  • Journal des modifications : s'ilAmazon Kendra convient d'utiliser le mécanisme de journal des modifications de la source de données Box pour déterminer si un document doit être ajouté, mis à jour ou supprimé dans l'index.

    Note

    Utilisez le journal des modifications si vous neAmazon Kendra souhaitez pas numériser tous les documents. Si votre journal des modifications est volumineux, la numérisation des documents de la source de données Box peut prendreAmazon Kendra moins de temps que le traitement du journal des modifications. Si vous synchronisez votre source de données Box avec votre index pour la première fois, tous les documents sont numérisés.

  • Filtres d'inclusion et d'exclusion : spécifiez si vous souhaitez inclure ou exclure certains fichiers, dossiers, commentaires, tâches et liens Web Box.

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de données Box à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats de recherche en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour plus d'informations, consultez la section Filtrage du contexte utilisateur pour les sources de données Box.
