Dropbox

Dropbox est un service d'hébergement de fichiers qui propose des services de stockage dans le cloud, d'organisation de documents et de modélisation de documents. Si vous utilisez Dropbox, vous pouvez les utiliserAmazon Kendra pour indexer vos fichiers Dropbox, Dropbox Paper, vos modèles Dropbox Paper et les raccourcis enregistrés vers des pages Web. Vous pouvez également configurerAmazon Kendra pour indexer des fichiers Dropbox spécifiques, Dropbox Paper, des modèles Dropbox Paper et des raccourcis enregistrés vers des pages Web.

Amazon Kendraprend en charge Dropbox et Dropbox Advanced pour Dropbox Business.

Vous pouvez vous connecterAmazon Kendra à votre source de données Dropbox à l'aide de la Amazon Kendraconsole et de l'TemplateConfigurationAPI.

Pour résoudre les problèmes liés au connecteur de source de donnéesAmazon Kendra Dropbox, consultezRésolution des problèmes de sources de données.

Fonctionnalités prises en charge

Amazon KendraLe connecteur de source de données Dropbox prend en charge les fonctionnalités suivantes :

  • Journal des modifications

  • Mappages de champs

  • Filtrage du contexte utilisateur

  • Filtres d'inclusion/d'exclusion

  • Virtual privatecCloud (VPC)

Prérequis

Avant de pouvoir l'utiliserAmazon Kendra pour indexer votre source de données Dropbox, apportez ces modifications à votre Dropbox et à vosAWS comptes.

Dans Dropbox, assurez-vous de disposer des éléments suivants :

  • Création d'un compte Dropbox Advanced et configuration d'un utilisateur administrateur.

  • Création d'une application Dropbox avec un nom d'application unique, activation de Scoped Access. Consultez la documentation Dropbox pour savoir comment créer une application.

  • Les autorisations Dropbox complètes ont été activées sur la console Dropbox et les autorisations suivantes ont été ajoutées :

    • fichiers.content.read

    • fichiers.metadata.read

    • partager.lire

    • file_requests.read

    • groupes.lire

    • team_info.read

    • team_data.content.read

  • Vous avez noté votre clé d'application Dropbox, votre secret d'application Dropbox et votre jeton d'accès Dropbox pour les informations d'authentification de base.

  • Vous avez généré et copié un jeton d'accès Oauth 2.0 temporaire pour votre application Dropbox. Ce jeton est temporaire et expire au bout de 4 heures. Consultez la documentation Dropbox sur l'authentification OAuth.

    Note

    Il est recommandé de créer un jeton d'accès Dropbox Refresh qui n'expire jamais, plutôt que de vous fier à un jeton d'accès unique expirant au bout de 4 heures. Un jeton d'accès à l'actualisation est permanent et n'expire jamais afin que vous puissiez continuer à synchroniser votre source de données future.

  • Recommandé : Configurez un jeton d'actualisation permanent Dropbox qui n'expire jamaisAmazon Kendra pour vous permettre de continuer à synchroniser votre source de données sans interruption. Consultez la documentation Dropbox sur les jetons d'actualisation.

  • Coché, chaque document est unique dans Dropbox et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer des éléments suivants :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez enregistré vos informations d'authentification Dropbox deAWSSecrets Manager manière secrète et, si vous utilisez l'API, vous avez noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de données Dropbox àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données Dropbox, vous devez fournir les informations nécessaires sur votre source de données Dropbox afin que vousAmazon Kendra puissiez accéder à vos données. Si vous n'avez pas encore configuré Dropbox forAmazon Kendra seePrérequis.

Console

Pour vous connecterAmazon Kendra à Dropbox

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez Dropbox connector, puis Ajouter une source de données.

  5. Sur la page Spécifier les informations relatives à la source de données, entrez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. (Facultatif) Description : entrez une description facultative pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Tags, pour Ajouter un nouveau tag : Tags pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, entrez les informations suivantes :

    1. Type de jeton d'authentification : choisissez entre un jeton permanent (recommandé) et un jeton d'accès (utilisation temporaire) en fonction de votre cas d'utilisation.

    2. AWSSecrets Managersecret —Choisissez un secret existant ou créez-en un nouveauSecrets Manager pour stocker vos informations d'authentification Dropbox. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -Dropbox » est automatiquement ajouté à votre nom secret.

        2. Pour les informations relatives à la clé d'application, au secret de l'application et au jeton (permanent ou temporaire), entrez les valeurs des informations d'identification que vous avez générées à partir de votre compte Dropbox.

      2. Choisissez Save (Enregistrer).

    3. Virtual Private Cloud (VPC) : vous pouvez choisir d'utiliser un VPC. Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    4. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification et indexer le contenu de votre référentiel.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    5. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, entrez les informations suivantes :

    1. Pour Sélectionner les entités ou les types de contenu : choisissez les entités ou les types de contenu que vous souhaitez analyser.

    2. Changer le mode journal : choisissez de mettre à jour votre index au lieu de synchroniser tous les fichiers.

    3. Dans Configuration supplémentaire pour les modèles Regex : ajoutez des modèles d'expressions régulières pour inclure ou exclure certains fichiers.

    4. Dans Synchroniser le calendrier d'exécution, pour Fréquence : choisissez la fréquence àAmazon Kendra laquelle vous souhaitez synchroniser avec votre source de données.

    5. Choisissez Suivant.

  8. Sur la page Définir les mappages de champs, entrez les informations suivantes :

    1. Fichiers, modèles Dropbox Paper et Dropbox Paper : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à Dropbox

Vous devez spécifier un JSON du schéma de source de données à l'aide de l'TemplateConfigurationAPI. Vous devez fournir les informations suivantes :

  • Source de données : vous devez spécifier la source de données sous la formeDROPBOX.

  • Type : spécifiezTEMPLATE le type lorsque vous appelezCreateDataSource.

  • Nom de ressource Amazon (ARN) secret : indiquez le nom de ressource Amazon (ARN) d'unSecrets Manager code secret contenant les informations d'authentification de votre compte Dropbox. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
"appKey": "Dropbox app key",
"appSecret": "Dropbox app secret",
"accesstoken": "temporary access token or refresh access token"
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pourIAM attribuer un rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le connecteur Dropbox etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de données Dropbox.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Virtual Private Cloud (VPC) : spécifiez leVpcConfiguration moment de votre appelCreateDataSource. Consultez ConfigurationAmazon Kendra pour utiliser un VPC.

  • Journal des modifications : s'ilAmazon Kendra convient d'utiliser le mécanisme de journal des modifications des sources de données Dropbox pour déterminer si un document doit être ajouté, mis à jour ou supprimé dans l'index.

    Note

    Utilisez le journal des modifications si vous neAmazon Kendra souhaitez pas numériser tous les documents. Si votre journal des modifications est volumineux, la numérisation des documents contenus dans la source de données Dropbox peut prendreAmazon Kendra moins de temps que le traitement du journal des modifications. Si vous synchronisez votre source de données Dropbox avec votre index pour la première fois, tous les documents sont numérisés.

  • Filtres d'inclusion et d'exclusion : spécifiez s'il faut inclure ou exclure certains fichiers.

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de données Dropbox à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats de recherche en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour plus d'informations, consultez la section Filtrage contextuel utilisateur pour les sources de données Dropbox.

Pour obtenir la liste des autres clés JSON importantes à configurer, consultezSchéma du modèle Dropbox.
