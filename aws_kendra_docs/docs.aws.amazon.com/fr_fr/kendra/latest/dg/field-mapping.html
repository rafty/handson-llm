Mappage des champs de source de données

Vous pouvez mapper des champs de document ou de contenu de votre source de données à des champs de votre index. Par exemple, si votre source de données comporte un champ appelé « département » qui contient des informations sur le département d'un document, vous pouvez le mapper à un champ d'index appelé « Département ». Vous pouvez ainsi utiliser le champ lorsque vous interrogez des documents.

Vous pouvez également mapper des champs standard ou réservés tels que_created_at. Si votre source de données contient un champ appelé « creation_date », vous pouvez le mapper au champAmazon Kendra réservé équivalent appelé_created_at. Vous pouvez mapper les champs de la plupart des sources de données.

Si vous stockez vos documents dans un compartiment S3 ou une source de données S3, vous pouvez fournir des attributs personnalisés directement à l'aide de fichiers de métadonnées. Pour plus d'informations, veuillez consulter Création de champs de document personnalisés.

Le mappage de vos champs de source de données vers un champ d'index s'effectue en trois étapes :

  1. Créez un index. Pour plus d'informations, veuillez consulter Création d'un index.

  2. Mettez à jour l'index pour ajouter des champs personnalisés.

  3. Créez une source de données qui mappe les champs de source de données aux champs d'index.

Pour mettre à jour l'index afin d'ajouter des champs personnalisés, utilisez la console ou l'UpdateIndexAPI. Vous pouvez ajouter un total de 500 champs personnalisés à votre index.

Dans la console, vous pouvez choisir de mapper un champ de source de données à l'un des noms de champ réservés. Vous pouvez également choisir de créer un nouveau champ d'index mappé au champ. Pour les sources de données de base de données, si le nom de la colonne de base de données correspond au nom d'un champ réservé, le champ et la colonne sont automatiquement mappés.

Avec l'UpdateIndexAPI, vous ajoutez des champs personnalisés et réservés à l'aide duDocumentMetadataConfigurationUpdates paramètre.

L'exemple JSON suivant permetDocumentMetadataConfigurationUpdates d'ajouter un champ appelé « Department » à l'index.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Lorsque vous créez le champ, vous avez la possibilité de définir la façon dont il est utilisé dans les recherches. Sélectionnez parmi les éléments suivants :

  • Displayable —Détermine si le champ est renvoyé dans la réponse de la requête. La valeur par défaut est true.

  • Facetable : indique que le champ peut être utilisé pour créer des facettes. La valeur par défaut est false.

  • Recherche : détermine si le champ est utilisé dans la recherche. La valeur par défaut est true pour les champs de chaîne et false pour les champs de nombre et de date.

  • Triable : indique que le champ peut être utilisé pour trier la réponse d'une requête. Ne peut être défini que pour les champs de date, de nombre et de chaîne. Ne peut pas être défini pour les champs de liste de chaînes.

L'exemple JSON suivant permet d'DocumentMetadataConfigurationUpdatesajouter un champ appelé « Department » à l'index et de le marquer comme facetable.

"DocumentMetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE",
       "Search": {
           "Facetable": true
       }
   }
]

Utilisation des champs de document intégrés à Amazon Kendra

Avec l'UpdateIndex API, vous pouvez créer des champs réservés ou intégrés en utilisantDocumentMetadataConfigurationUpdates et en spécifiant le nom du champ réservé pour le mapper à votre attribut de document ou à un champ de document équivalent. Vous pouvez également créer des champs personnalisés de cette façon. Si vous utilisez un connecteur de source de données, la plupart incluent des mappages de champs qui mettent en correspondance les champs de votre document source de données avecAmazon Kendra des champs d'index. Lorsque vous créez un champ, vous pouvez configurer l'Searchobjet pour le définir comme étant affichable, facetable, consultable et triable. Vous pouvez configurer l'Relevanceobjet pour définir la durée d'amélioration de la pertinence, la fraîcheur, l'importance, l'ordre de classement et les valeurs d'importance du champ. Vous ne pouvez pas modifier le type de champ une fois que vous l'avez créé.

Amazon Kendracontient les champs de document réservés ou intégrés suivants que vous pouvez utiliser :

  • _authors—Une liste d'un ou de plusieurs auteurs responsables du contenu du document.

  • _category—Une catégorie qui place un document dans un groupe spécifique.

  • _created_at—La date et l'heure au format ISO 8601 auxquelles le document a été créé. Par exemple, 2012-03-25T12:30:10+01:00 est le format de date et d'heure ISO 8601 pour le 25 mars 2012 à 12 h 30 (plus 10 secondes) à l'heure d'Europe centrale.

  • _data_source_id—Identifiant de la source de données qui contient le document.

  • _document_body—Le contenu du document.

  • _document_id—Identifiant unique pour le document.

  • _document_title—Le titre du document.

  • _excerpt_page_number—Le numéro de page d'un fichier PDF où apparaît l'extrait du document. Si votre index a été créé avant le 8 septembre 2020, vous devez réindexer vos documents avant de pouvoir utiliser cet attribut.

  • _faq_id—S'il s'agit d'une question et d'une réponse à la FAQ, un identifiant unique pour ces questions.

  • _file_type—Le type de fichier du document, tel que pdf ou doc.

  • _last_updated_at—La date et l'heure au format ISO 8601 auxquelles le document a été mis à jour pour la dernière fois. Par exemple, 2012-03-25T12:30:10+01:00 est le format de date et d'heure ISO 8601 pour le 25 mars 2012 à 12 h 30 (plus 10 secondes) à l'heure d'Europe centrale.

  • _source_uri—L'URI où le document est disponible. Par exemple, l'URI du document sur le site Web d'une entreprise.

  • _version: identifiant de la version spécifique d'un document.

  • _view_count—Le nombre de fois que le document a été consulté.

  • _language_code(String) : code de la langue qui s'applique au document. La valeur par défaut est l'anglais si vous ne spécifiez aucune langue. Pour plus d'informations sur les langues prises en charge, y compris leurs codes, voir Ajout de documents dans des langues autres que l'anglais.

Pour les champs personnalisés, vous créez ces champs à l'DocumentMetadataConfigurationUpdatesaide de l'UpdateIndexAPI, comme vous le feriez pour créer un champ réservé ou intégré. Vous devez définir le type de données approprié pour votre champ personnalisé. Vous ne pouvez pas modifier le type de champ une fois que vous l'avez créé.

Voici les types que vous pouvez définir pour les champs personnalisés :

  • Date

  • Nombre

  • Chaîne

  • Liste de chaînes

Vous créez un champ personnalisé à l'aide de la console ou de l'UpdateIndexAPI. Après avoir créé un champ personnalisé, vous le mappez à un attribut de document, comme vous le feriez avec un champ réservé. Si vous avez ajouté un document à l'index avec l'BatchPutDocumentAPI, vous mappez les attributs avec l'API. Pour les documents indexés à partir d'une source deAmazon S3 données, vous mappez les attributs à l'aide d'un fichier de métadonnées contenant une structure JSON qui décrit les attributs du document. Pour les documents indexés à l'aide d'une base de données ou d'une source de données permettant le mappage de champs, vous mappez les attributs à l'aide de la console ou de la configuration de la source de données.
