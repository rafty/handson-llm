CreateFaq

Crée un ensemble de questions fréquemment posées (FAQ) à l'aide d'un fichier de FAQ spécifié stocké dans un compartiment Amazon S3.

L'ajout de FAQ à un index constitue une opération asynchrone.

Pour un exemple d'ajout d'une FAQ à un index à l'aide de SDK Python et Java, consultez la section Utilisation de votre fichier de FAQ.

Syntaxe de la requête

{
   "ClientToken": "string",
   "Description": "string",
   "FileFormat": "string",
   "IndexId": "string",
   "LanguageCode": "string",
   "Name": "string",
   "RoleArn": "string",
   "S3Path": { 
      "Bucket": "string",
      "Key": "string"
   },
   "Tags": [ 
      { 
         "Key": "string",
         "Value": "string"
      }
   ]
}

Paramètres de demande

Pour plus d'informations sur les paramètres communs à toutes les actions, consultez la section Paramètres communs.

Cette demande accepte les données suivantes au format JSON.

ClientToken

Un jeton que vous fournissez pour identifier la demande de création d'une FAQ. Plusieurs appels à l'CreateFaqRequestAPI avec le même jeton client ne créeront qu'une seule FAQ.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximum de 100.

Obligatoire : non

Description

Description des FAQ.

Type : String

Contraintes de longueur : longueur minimale de 0. Longueur maximum de 1 000.

Modèle : ^\P{C}*$

Obligatoire : non

FileFormat

Format du fichier d'entrée du FAQ. Vous pouvez choisir entre un format CSV de base, un format CSV qui inclut des attributs douaniers dans un en-tête et un format JSON qui inclut des attributs personnalisés.

Le format doit correspondre au format du fichier stocké dans le compartiment S3 identifié dans leS3Path paramètre.

Pour plus d'informations, consultez la section Ajouter des questions et des réponses.

Type : String

Valeurs valides : CSV | CSV_WITH_HEADER | JSON

Obligatoire : non

IndexId

Identifiant de l'index du FAQ.

Type : String

Contraintes de longueur maximale maximale maximale maximale maximale maximale maximale maximale de 36.

Modèle : [a-zA-Z0-9][a-zA-Z0-9-]*

Obligatoire : oui

LanguageCode

Le code d'une langue. Cela vous permet de prendre en charge une langue pour le document FAQ. L'anglais est pris en charge par défaut. Pour plus d'informations sur les langues prises en charge, y compris leurs codes, voir Ajout de documents dans des langues autres que l'anglais.

Type : String

Contraintes de longueur maximale maximale maximale de 2. Longueur maximale maximale maximale maximale maximale maximale maximale de 10.

Modèle : [a-zA-Z-]*

Obligatoire : non

Name

L'Amazon FAQ.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximum de 100.

Modèle : [a-zA-Z0-9][a-zA-Z0-9_-]*

Obligatoire : oui

RoleArn

L'Amazon Resource Name (ARN) d'un rôle IAM avec l'autorisation d'accéder au compartiment S3 qui contient le FAQ. Pour de plus amples informations, veuillez consulter Rôles d'accès IAM pour Amazon Kendra.

Type : String

Contraintes de longueur : longueur minimum de 0. Longueur maximale maximale maximale maximale maximale maximale maximale maximale de 1284.

Modèle : arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Obligatoire : oui

S3Path

Chemin d'accès du fichier FAQ dans S3.

Type : objet S3Path

Obligatoire : oui

Tags

Liste de paires clé-valeur permettant d'identifier le FAQ. Vous pouvez utiliser les balises pour identifier et organiser vos ressources et pour contrôler l'accès aux ressources.

Type : tableau d’objets Tag

Membres du tableau : nombre minimum de 0 élément. Nombre maximum maximale maximale de 200 éléments.

Obligatoire : non

Syntaxe de la réponse

{
   "Id": "string"
}

Eléments de réponse

Si l'action aboutit, le service renvoie une réponse HTTP 200.

Les données suivantes sont renvoyées au format JSON par le service.

Id

Identifiant du FAQ.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximum de 100.

Modèle : [a-zA-Z0-9][a-zA-Z0-9_-]*

Erreurs

Pour plus d'informations sur les erreurs communes à toutes les actions, consultezErreurs courantes.

AccessDeniedException

Vous ne disposez pas de l'accès nécessaire pour effectuer cette action. Assurez-vous de disposer des politiques d'autorisation et des comptes utilisateur requis, puis réessayez.

Code d'état HTTP : 400

ConflictException

Un conflit est survenu avec la demande. Corrigez toute incohérence avec vos ressources et réessayez.

Code d'état HTTP : 400

InternalServerException

Un problème s'est produit avec le serveur interne utilisé pour votre service Amazon Kendra. Veuillez patienter quelques minutes et réessayez, ou contactez le support pour obtenir de l'aide.

Code d'état HTTP : 500

ResourceNotFoundException

La ressource que vous souhaitez utiliser n'existe pas. Vérifiez que vous avez fourni la bonne ressource et réessayez.

Code d'état HTTP : 400

ServiceQuotaExceededException

Vous avez dépassé les limites définies pour votre service Amazon Kendra. Consultez la section Quotas pour plus d'informations, ou contactez le support pour en savoir plus sur une augmentation des limites.

Code d'état HTTP : 400

ThrottlingException

La demande a été refusée suite à une limitation des demandes. Réduisez le nombre de demandes et réessayez.

Code d'état HTTP : 400

ValidationException

L'entrée ne satisfait pas les contraintes définies par le service Amazon Kendra. Veuillez saisir correctement et réessayer.

Code d'état HTTP : 400
