Étiquettes

Gérez vos index, vos sources de données et vos FAQ en attribuant des balises ou des étiquettes. Vous pouvez utiliser les balises pour classer vosAmazon Kendra ressources de différentes manières. Par exemple, par exemple, par propriétaire ou application, ou toute combinaison de ces deux. Chaque balise est constituée d'une clé et d'une valeur que vous définissez toutes deux.

Les balises vous aident à :

  • Identifier et organiser vos ressources AWS. De nombreuxAWS services prennent en charge le balisage. Vous pouvez donc attribuer la même balise à des ressources dans différents services pour indiquer que les ressources sont liées. Par exemple, vous pouvez marquer un index et leAmazon Lex robot qui utilise l'index avec la même balise.

  • Répartir les coûts. Vous activez les balises sur leAWS Billing and Cost Management tableau de bord. AWSutilise des balises pour classer vos coûts par catégorie et vous fournir un rapport mensuel de répartition des coûts. Pour plus d'informations, veuillez consulter Répartition des coûts et balisage dans À propos de laAWS Billing and Cost Management.

  • Contrôler l'accès à vos ressources . Vous pouvez utiliser des balises dans les politiquesAWS Identity and Access Management (IAM) qui contrôlent l'accès auxAmazon Kendra ressources. Vous pouvez associer ces politiques à unIAM rôle ou à un utilisateur pour activer le contrôle d'accès basé sur des balises. Pour plus d'informations, veuillez consulter Autorisation basée sur des identifications Amazon Kendra.

Vous pouvez créer et gérer des balises à l'aide duAWS Management Console, duAWS Command Line Interface (AWS CLI) ou de l'Amazon KendraAPI.

Étiquetage des ressources

Si vous utilisez laAmazon Kendra console, vous pouvez étiqueter des ressources lorsque vous les créez ou les ajouter ultérieurement. Vous pouvez également utiliser la console pour mettre à jour ou supprimer des balises.

Si vous utilisez leAWS Command Line Interface (AWS CLI) ou l'Amazon KendraAPI, effectuez les opérations suivantes pour gérer les balises de vos ressources :

  • CreateDataSource— Appliquez des balises lorsque vous créez une source de données.

  • CreateFaq—Appliquez des balises lorsque vous créez une FAQ.

  • CreateIndex—Appliquez des balises lorsque vous créez un index.

  • ListTagsForResource— Afficher les balises associées à une ressource.

  • TagResource—Ajoutez et modifiez des balises pour une ressource.

  • UntagResource— Supprimer des balises d'une ressource.

Restrictions liées aux étiquettes

Les restrictions suivantes s'appliquent aux balises sur lesAmazon Kendra ressources :

  • Nombre maximal de balises — 50

  • Longueur de clé maximale — 128 caractères

  • Longueur de valeur maximale — 256 caractères

  • Caractères acceptés pour les clés et valeurs — 0-+ - et @

  • Les clés et les valeurs sont sensibles à la casse.

  • N’utilisez pas aws: comme préfixe pour les clés ; seul AWS peut utiliser cette valeur.
