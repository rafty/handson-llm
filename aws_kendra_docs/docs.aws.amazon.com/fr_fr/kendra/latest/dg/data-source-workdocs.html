Amazon WorkDocs

Amazon WorkDocsest un service de collaboration de contenu sécurisé permettant de créer, de modifier, de stocker et de partager du contenu. Vous pouvez l'utiliserAmazon Kendra pour indexer votre source deAmazon WorkDocs données.

Vous pouvez vous connecterAmazon Kendra à votre source deAmazon WorkDocs données à l'aide de la Amazon Kendraconsole et de l'WorkDocsConfigurationAPI.

Amazon WorkDocsest disponible dans les régions de l'Oregon, de Singapour et d'Irlande.

Pour résoudre les problèmes liés à votre connecteur de source deAmazon Kendra WorkDocs données, consultezRésolution des problèmes de sources de données.

Fonctionnalités prises en charge

Amazon Kendra WorkDocs Le connecteur de source de données prend en charge les fonctionnalités suivantes :

  • Journal des modifications

  • Mde champs

  • Filtrage du contexte utilisateur

  • Filtres d'inclusion/d'exclusion

Prérequis

Avant de pouvoir l'utiliserAmazon Kendra pour indexer votre source de WorkDocs données, apportez ces modifications à vosAWS comptes WorkDocs et.

Dans WorkDocs, assurez-vous d'avoir :

  • Notez l'ID deAmazon WorkDocs répertoire (ID d'organisation) de votreAmazon WorkDocs référentiel.

  • Coché, chaque document est unique dans WorkDocs et entre les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous d'avoir :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

Si vous n'avez pas deIAM rôle existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle lorsque vous connectez votre source de WorkDocs données àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle existant et un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de WorkDocs données, vous devez fournir les informations nécessaires sur votre source de WorkDocs données afin deAmazon Kendra pouvoir accéder à vos données. Si vous n'avez pas encore configuré WorkDocs pourAmazon Kendra voirPrérequis.

Console

Pour vous connecterAmazon Kendra àAmazon WorkDocs

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez WorkDocs connecteur, puis Ajouter une source de données.

  5. Sur la page Spécifier les détails de la source de données, saisissez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) : saisissez une description si vous le souhaitez pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Tde, pour Ajouter un nouveau tag : Tde pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, saisissez les informations suivantes :

    1. ID d'organisation spécifique à votreAmazon WorkDocs site : sélectionnez l'ID duAmazon WorkDocs site que vous souhaitez indexer. Vous devez déjà avoir créé un site.

    2. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification et d'index de votre référentiel.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    3. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, saisissez les informations suivantes :

    1. Analyser les commentaires du document :Amazon WorkDocs entités ou types de contenu que vous souhaitez analyser.

    2. Utiliser les journaux des modifications : sélectionnez cette option pour mettre à jour votre index au lieu de synchroniser tous vos fichiers.

    3. Modèles Regex : modèles d'expression régulière permettant d'inclure ou d'exclure certains fichiers. Vous pouvez ajouter jusqu'à 100 motifs.

    4. Dans Synchroniser le calendrier d'exécution pour la fréquence : choisissez la fréquence àAmazon Kendra laquelle vous souhaitez synchroniser avec votre source de données.

    5. Choisissez Suivant.

  8. Sur la page Définir les champs, saisissez les informations suivantes :

    1. Champs de source de données par défaut : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra àAmazon WorkDocs

Vous devez spécifier les éléments suivants à l'aide de l'WorkDocsConfigurationAPI :

  • Amazon WorkDocsID de répertoire : spécifiez l'ID d'organisation de votreAmazon WorkDocs annuaire. Vous pouvez trouver l'identifiant de l'organisation dans AWS Directory Service en accédant à Active Directory, puis à Directories.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pour fournir unIAM rôle autorisé à accéder WorkDocs au répertoire et pour appeler les API publiques requises pour le WorkDocs connecteur etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de WorkDocs données.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Journal des modifications : s'ilAmazon Kendra faut utiliser le mécanisme du journal des modifications de la source de WorkDocs données pour déterminer si un document doit être ajouté, mis à jour ou supprimé dans l'index.

    Note

    Utilisez le journal des modifications si vous neAmazon Kendra souhaitez pas numériser tous les documents. Si votre journal des modifications est volumineux, la numérisation des documents de la source de WorkDocs données peut prendreAmazon Kendra moins de temps que le traitement du journal des modifications. Si vous synchronisez votre source de WorkDocs données avec votre index pour la première fois, tous les documents sont numérisés.

  • Filtres d'inclusion et d'exclusion : spécifiez si vous souhaitez inclure ou exclure certains documents et commentaires. Chaque commentaire est indexé en tant que document distinct.

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de WorkDocs données à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats de recherche en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour plus d'informations, voir Filtrage contextuel utilisateur pour les sources de WorkDocs données.
