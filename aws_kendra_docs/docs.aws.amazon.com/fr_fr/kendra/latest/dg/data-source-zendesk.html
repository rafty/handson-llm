Zendesk

Zendesk est un système de gestion de la relation client qui aide les entreprises à automatiser et à améliorer les interactions avec le service client. Vous pouvez l'utiliserAmazon Kendra pour indexer vos tickets d'assistance Zendesk, les commentaires sur les tickets, les pièces jointes aux tickets, les articles du centre d'aide, les commentaires sur les articles, les pièces jointes aux commentaires d'articles, les rubriques communautaires du guide, les publications de la communauté et les commentaires des publications de la communauté.

Vous pouvez filtrer par nom d'organisation si vous souhaitez indexer les tickets appartenant uniquement à une organisation spécifique. Vous pouvez également choisir de définir une date d'exploration à laquelle vous souhaitez commencer à explorer les données depuis Zendesk.

Vous pouvez vous connecterAmazon Kendra à votre source de données Zendesk à l'aide de la Amazon Kendraconsole et de l'TemplateConfigurationAPI.

Pour résoudre les problèmes liés à votre connecteur de source de donnéesAmazon Kendra Zendesk, consultezRésolution des problèmes de sources de données.

Prérequis

Avant de pouvoir l'utiliserAmazon Kendra pour indexer votre source de données Zendesk, apportez ces modifications à votre Zendesk et à vosAWS comptes.

Dans Zendesk, assurez-vous de disposer des éléments suivants :

  • Création d'un compte administratif Zendesk Suite (Professional/Enterprise).

  • Vous avez noté l'URL de votre hôte Zendesk. Par exemple, https://{sub-domain (https://{host/)}.zendesk.com/.

  • Un jeton d'identification OAuth 2.0 contenant un identifiant client, un secret de client, un nom d'utilisateur et un mot de passe. Consultez la documentation Zendesk sur la génération de jetons OAuth 2.0 pour plus d'informations.

  • Ajout de la portée OAuth 2.0 suivante :

    • lire

  • Facultatif : un certificat SSL a été installéAmazon Kendra pour permettre la connexion.

  • Coché, chaque document est unique dans Zendesk et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer des éléments suivants :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké vos informations d'authentification Zendesk dans unAWSSecrets Manager secret et, si vous utilisez l'API, noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de données Zendesk àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données Zendesk, vous devez fournir les informations nécessaires sur votre source de données Zendesk afin que vousAmazon Kendra puissiez accéder à vos données. Si vous n'avez pas encore configuré Zendesk pourAmazon Kendra voirPrérequis.

Console

Pour vous connecterAmazon Kendra à Zendesk

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez le connecteur Zendesk, puis choisissez Ajouter une source de données.

  5. Sur la page Spécifier les informations relatives à la source de données, saisissez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) : entrez une description facultative pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Tags, pour Ajouter un nouveau tag : Tags pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, saisissez les informations suivantes :

    1. URL Zendesk : entrez votre URL Zendesk.

    2. AWSSecrets Managersecret —Choisissez un secret existant ou créez-en un nouveauSecrets Manager pour stocker vos informations d'authentification Zendesk. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -Zendesk-» est automatiquement ajouté à votre nom secret.

        2. Pour ID client, secret client, nom d'utilisateur, mot de passe, entrez les valeurs des informations d'identification que vous avez créées dans votre compte Zendesk.

      2. Choisissez Save (Enregistrer).

    3. Virtual Private Cloud (VPC) : vous pouvez choisir d'utiliser un VPC. Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    4. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification de votre référentiel et au contenu de l'index.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    5. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, saisissez les informations suivantes :

    1. Sélectionnez des entités ou des types de contenu : les entités ou les types de contenu Zendesk que vous souhaitez analyser.

    2. Journal des modifications : sélectionnez cette option pour mettre à jour votre index au lieu de synchroniser tous vos fichiers.

    3. Nom de l'organisation : entrez les noms des organisations Zendesk pour filtrer votre synchronisation.

    4. Date de début de synchronisation : date à partir de laquelle vous souhaitez indexer votre contenu.

    5. Modèles Regex : modèles d'expression régulière permettant d'inclure ou d'exclure certains fichiers. Vous pouvez ajouter jusqu'à 100 motifs.

    6. Dans Synchroniser le calendrier d'exécution pour la fréquence : choisissez la fréquence àAmazon Kendra laquelle vous souhaitez synchroniser avec votre source de données.

    7. Choisissez Suivant.

  8. Sur la page Définir les mappages de champs, saisissez les informations suivantes :

    1. Pour les tickets, les commentaires sur les tickets, les pièces jointes aux commentaires sur les tickets, les articles, les commentaires sur les articles, les sujets communautaires, les commentaires sur les publications communautaires, les commentaires sur les publications communautaires, sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à Zendesk

Vous devez spécifier un JSON du schéma de source de données à l'aide de l'TemplateConfigurationAPI. Vous devez fournir les informations suivantes :

  • Source de données : spécifiez la source de données sous la formeZENDESK.

  • URL de l'hôte : indiquez l'URL de votre hôte Zendesk dans le cadre de la configuration de la connexion ou des détails du point de terminaison du référentiel. Par exemple, https://yoursubdomain.zendesk.com.

  • Type : spécifiezTEMPLATE le type lorsque vous appelezCreateDataSource.

  • Nom de ressource Amazon (ARN) secret : indiquez le nom de ressource Amazon (ARN) d'unSecrets Manager code secret contenant les informations d'authentification de votre compte Zendesk. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "hostUrl": "https://yoursubdomain.zendesk.com",
    "clientId": "kendra",
    "clientSecret": "Zendesk client secret",
    "userName": "Zendesk user name",
    "password": "Zendesk password"
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pourIAM attribuer un rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le connecteur Zendesk etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de données Zendesk.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Virtual Private Cloud (VPC) : spécifiezVpcConfiguration quand vous appelezCreateDataSource. Consultez ConfigurationAmazon Kendra pour utiliser un VPC.

  • Journal des modifications :Amazon Kendra faut-il utiliser le mécanisme de journal des modifications de la source de données Zendesk pour déterminer si un document doit être ajouté, mis à jour ou supprimé dans l'index.

    Note

    Utilisez le journal des modifications si vous neAmazon Kendra souhaitez pas numériser tous les documents. Si votre journal des modifications est volumineux, la numérisation des documents dans la source de données Zendesk peut prendreAmazon Kendra moins de temps que le traitement du journal des modifications. Si vous synchronisez votre source de données Zendesk avec votre index pour la première fois, tous les documents sont numérisés.

  • Filtres d'inclusion et d'exclusion : spécifiez s'il faut inclure ou exclure :

    • Tickets d'Support, commentaires sur les tickets et/ou pièces jointes

    • Articles du centre d'aide, pièces jointes et commentaires sur les articles

    • Guidez les sujets, les publications ou les commentaires de la communauté

  • Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de données Zendesk à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats de recherche en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour plus d'informations, consultez Filtrage du contexte utilisateur pour les sources de données Zendesk.

Pour obtenir la liste des autres clés JSON importantes à configurer, consultezSchéma de modèle Zendesk.
