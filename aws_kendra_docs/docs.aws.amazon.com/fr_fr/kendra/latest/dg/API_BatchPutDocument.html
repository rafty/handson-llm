BatchPutDocument

Ajoute un ou plusieurs documents à un index.

L'BatchPutDocumentAPI vous permet d'intégrer des documents en ligne ou un ensemble de documents qui sont stockés dans un compartiment Amazon S3. Utilisez cette API pour intégrer votre texte et votre texte non structuré dans un index, ajouter des attributs personnalisés aux documents et joindre une liste de contrôle d'accès aux documents ajoutés à l'index.

Les documents sont indexés de manière asynchrone. Vous pouvez voir la progression du lot à l'aide deAWS CloudWatch. Tous les messages d'erreur liés au traitement du lot sont envoyés à votreAWS CloudWatch journal.

Pour un exemple d'ingestion de documents en ligne à l'aide de SDK Python et Java, consultez la section Ajout de fichiers directement à un index.

Paramètres de demande

Pour plus d'informations sur les paramètres communs à toutes les actions, voir Paramètres communs.

Cette demande accepte les données suivantes au format JSON.

CustomDocumentEnrichmentConfiguration

Informations de configuration pour modifier les métadonnées et le contenu de vos documents pendant le processus d'intégration de documents lorsque vous utilisez l'BatchPutDocumentAPI.

Pour plus d'informations sur la création, la modification et la suppression des métadonnées de documents, ou sur d'autres modifications de contenu lorsque vous ingérez des documents dans Amazon Kendra, consultez Personnalisation des métadonnées de documents pendant le processus d'ingestion.

Type : objet CustomDocumentEnrichmentConfiguration

Obligatoire : non

Documents

Un ou plusieurs documents à ajouter à l'index.

Les limites de taille de fichier suivantes s'appliquent aux documents.

  • 50 Mo de taille totale pour n'importe quel fichier

  • 5 Mo de texte extrait pour n'importe quel fichier

Pour de plus amples informations, veuillez consulter Quotas .

Type : tableau d’objets Document

Membres du tableau : Nombre minimum de 1 élément. Nombre maximum de 10 éléments.

Obligatoire : oui

IndexId

Identifiant de l'index auquel ajouter les documents. Vous devez d'abord créer l'index à l'aide de l'CreateIndexAPI.

Type : String

Contraintes de longueur : longueur fixe de 36.

Modèle : [a-zA-Z0-9][a-zA-Z0-9-]*

Obligatoire : oui

RoleArn

L'Amazon Resource Name (ARN) d'un rôle IAM avec l'autorisation d'accéder à votre compartiment S3. Pour plus d'informations, veuillez consulter la rubrique Rôles d'accès IAM pour Amazon Kendra.

Type : String

Contraintes de longueur : longueur minimum de 0. Longueur maximale de 1284.

Modèle : arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Obligatoire : non

Eléments de réponse

Si l'action aboutit, le service renvoie une réponse HTTP 200.

Les données suivantes sont renvoyées au format JSON par le service.

FailedDocuments

Liste des documents qui n'ont pas été ajoutés à l'index car le document a échoué à un contrôle de validation. Chaque document contient un message d'erreur qui indique pourquoi le document n'a pas pu être ajouté à l'index.

Si une erreur s'est produite lors de l'ajout d'un document à un index, l'erreur est signalée dans votreAWS CloudWatch journal. Pour plus d'informations, veuillez consulter la section Surveillance d'Amazon Kendra avec Amazon CloudWatch Logs.

Type : tableau d’objets BatchPutDocumentResponseFailedDocument

Erreurs

Pour plus d'informations sur les erreurs communes à toutes les actions, consultezErreurs courantes.

AccessDeniedException

Vous ne disposez pas d'un accès suffisant pour effectuer cette action. Assurez-vous de disposer des politiques d'autorisation et des comptes utilisateur requis, puis réessayez.

Code d'état HTTP : 400

ConflictException

Un conflit est survenu avec la demande. Corrigez toute incohérence avec vos ressources et réessayez.

Code d'état HTTP : 400

InternalServerException

Un problème s'est produit avec le serveur interne utilisé pour votre service Amazon Kendra. Patientez quelques minutes et réessayez, ou contactez le support pour obtenir de l'aide.

Code d'état HTTP : 500

ResourceNotFoundException

La ressource que vous souhaitez utiliser n'existe pas. Vérifiez que vous avez fourni la bonne ressource et réessayez.

Code d'état HTTP : 400

ServiceQuotaExceededException

Vous avez dépassé les limites définies pour votre service Amazon Kendra. Consultez la section Quotas pour plus d'informations, ou contactez le support pour en savoir plus sur une augmentation des limites.

Code d'état HTTP : 400

ThrottlingException

La demande a été refusée suite à une limitation des demandes. Réduisez le nombre de demandes et réessayez.

Code d'état HTTP : 400

ValidationException

L'entrée ne satisfait pas les contraintes définies par le service Amazon Kendra. Veuillez saisir l'entrée correcte et réessayer.

Code d'état HTTP : 400
