CreateIndex

Crée un index Amazon Kendra. La création d'index est une API asynchrone. Pour déterminer si la création de l'index est terminée, vérifiez leStatus champ renvoyé par un appel àDescribeIndex. LeStatus champ est défini sur leACTIVE moment où l'index est prêt à être utilisé.

Une fois l'index actif, vous pouvez indexer vos documents à l'aide de l'BatchPutDocumentAPI ou de l'une des sources de données prises en charge.

Pour un exemple de création d'un index et d'une source de données à l'aide du SDK Python, consultez la section Mise en route avec le SDK Python. Pour un exemple de création d'un index et d'une source de données à l'aide du SDK Java, consultez la section Mise en route avec le SDK Java.

Paramètres de demande

Pour plus d'informations sur les paramètres communs à toutes les actions, consultez la section Paramètres communs.

Cette demande accepte les données suivantes au format JSON.

ClientToken

Un jeton que vous fournissez pour identifier la demande de création d'un index. Plusieurs appels à l'CreateIndexAPI avec le même jeton client ne créeront qu'un seul index.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximum de 100.

Obligatoire : non

Description

Description de l'index.

Type : String

Contraintes de longueur : longueur minimale de 0. Longueur maximum de 1 000.

Modèle : ^\P{C}*$

Obligatoire : non

Edition

L'édition Amazon Kendra à utiliser pour l'index. ChoisissezDEVELOPER_EDITION les index destinés au développement, aux tests ou à la validation de principe. À utiliserENTERPRISE_EDITION pour la production. Une fois que vous avez défini l'édition d'un index, celle-ci ne peut pas être modifiée.

Le paramètre Edition est facultatif. Si vous ne fournissez pas de valeur par défaut, la valeur par défaut estENTERPRISE_EDITION.

Pour plus d'informations sur les limites de quota pour les éditions Enterprise et Developer, voir Quotas.

Type : String

Valeurs valides : DEVELOPER_EDITION | ENTERPRISE_EDITION

Obligatoire : non

Name

Un nom pour l'index.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximum de 1 000.

Modèle : [a-zA-Z0-9][a-zA-Z0-9_-]*

Obligatoire : oui

RoleArn

Amazon Resource Name (ARN) Amazon Resource Name (ARN) Amazon Resource Name (ARN) Amazon Resource Name (ARN) Amazon Resource Name (ARN) Amazon Resource CloudWatch Name (ARN) Amazon Resource Pour plus d'informations, veuillez consulter la rubrique Rôles d'accès IAM pour Amazon Kendra.

Type : String

Contraintes de longueur : longueur minimum de 0. Longueur maximale de 1284.

Modèle : arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Obligatoire : oui

ServerSideEncryptionConfiguration

Identifiant de la clé client (CMK)AWS KMS client (CMK) client (CMK) client (CMK) client (CMK) client (CMK) client (CMK) client (CMK) Amazon Kendra) client Amazon Kendra ne prend pas en charge les CMK asymétriques.

Type : objet ServerSideEncryptionConfiguration

Obligatoire : non

Tags

Une liste de paires clé-valeur qui identifient ou classent l'index. Vous pouvez également utiliser des balises pour contrôler l'accès à l'index. Les clés et valeurs des balises peuvent inclure des lettres Unicode, chiffres, espaces blancs et tout symbole suivants : _. :/= + - @.

Type : tableau d’objets Tag

Membres du tableau : nombre minimum de 0 élément. Nombre maximum de 200 éléments.

Obligatoire : non

UserContextPolicy

La politique de contexte utilisateur.

ATTRIBUTE_FILTER

Tout le contenu indexé est consultable et affichable pour tous les utilisateurs. Si vous souhaitez filtrer les résultats de recherche en fonction du contexte utilisateur, vous pouvez utiliser les filtres d'attributs _user_id et _group_ids ou fournir des informations sur les utilisateurs et les groupes dans UserContext.

USER_TOKEN

Active le contrôle d'accès utilisateur basé sur des jetons pour filtrer les résultats de recherche en fonction du contexte utilisateur. Tous les documents sans contrôle d'accès et tous les documents accessibles à l'utilisateur seront consultables et affichables.

Type : String

Valeurs valides : ATTRIBUTE_FILTER | USER_TOKEN

Obligatoire : non

UserGroupResolutionConfiguration

Obtient les utilisateurs et les groupes à partir de la sourceAWS IAM Identity Center (successor to AWS Single Sign-On) d'identité. Pour configurer cela, reportez-vous à la section UserGroupResolutionConfiguration.

Type : objet UserGroupResolutionConfiguration

Obligatoire : non

UserTokenConfigurations

Configuration du jeton utilisateur.

Type : tableau d’objets UserTokenConfiguration

Membres du tableau : nombre maximum de 1 élément.

Obligatoire : non

Eléments de réponse

Si l'action aboutit, le service renvoie une réponse HTTP 200.

Les données suivantes sont renvoyées au format JSON par le service.

Id

Identifiant de l'index. Utilisez cet identifiant lorsque vous interrogez un index, configurez une source de données ou indexez un document.

Type : String

Contraintes de longueur : longueur fixe de 36.

Modèle : [a-zA-Z0-9][a-zA-Z0-9-]*

Erreurs

Code d'état HTTP : 400

Un conflit est survenu avec la demande. Corrigez toute incohérence avec vos ressources et réessayez.

Code d'état HTTP : 400

InternalServerException

Un problème s'est produit avec le serveur interne utilisé pour votre service Amazon Kendra. Veuillez patienter quelques minutes et réessayez, ou contactez le support pour obtenir de l'aide.

Code d'état HTTP : 400

ValidationException
