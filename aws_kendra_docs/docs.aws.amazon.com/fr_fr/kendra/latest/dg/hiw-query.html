Requêtes

Pour obtenir des réponses, les utilisateurs interrogent un index. Les utilisateurs peuvent utiliser le langage naturel dans leurs requêtes. La réponse contient des informations, telles que le titre, un extrait de texte et l'emplacement des documents dans l'index qui fournissent la meilleure réponse.

Amazon Kendrautilise toutes les informations que vous fournissez sur vos documents, et pas uniquement le contenu des documents, pour déterminer si un document est pertinent pour la requête. Par exemple, si votre index contient des informations sur la date de la dernière mise à jour des documents, vous pouvez indiquer d'Amazon Kendraattribuer une plus grande pertinence aux documents qui ont été mis à jour plus récemment.

Une requête peut également contenir des critères permettant de filtrer la réponse afin de neAmazon Kendra renvoyer que les documents qui répondent aux critères de filtrage. Par exemple, si vous avez créé un champ d'index appelé département, vous pouvez filtrer la réponse de sorte que seuls les documents dont le champ département est défini sur légal soient renvoyés. Pour plus d'informations, veuillez consulter Filtrage et recherche par facettes.

Vous pouvez influencer les résultats d'une requête en ajustant la pertinence des différents champs de l'index. Le réglage modifie l'importance d'un champ dans les résultats. Par exemple, si vous soulignez l'importance des documents de la catégorie « nouveau », les documents de cette catégorie seront plus susceptibles d'être inclus dans la réponse. Pour plus d'informations, veuillez consulter Réglage de la pertinence des recherches.

Pour plus d'informations sur l'utilisation des requêtes, consultezRecherche dans les index.
