Microsoft Yammer

Microsoft Yammer est un outil de collaboration d'entreprise pour la messagerie, les réunions et le partage de fichiers. Si vous utilisez Microsoft Yammer, vous pouvez l'utiliserAmazon Kendra pour indexer votre source de données Microsoft Yammer.

Vous pouvez vous connecterAmazon Kendra à votre source de données Microsoft Yammer à l'aide de la Amazon Kendraconsole et de l'TemplateConfigurationAPI.

Pour résoudre les problèmes liés à votre connecteur de source de donnéesAmazon Kendra Microsoft Yammer, consultezRésolution des problèmes de sources de données.

Fonctionnalités prises en charge

  • Journal des modifications

  • Mappages de champs

  • Filtres d'inclusion/d'exclusion

  • Virtual privatecCloud (VPC)

Prérequis

Avant de pouvoirAmazon Kendra indexer votre source de données Microsoft Yammer, apportez ces modifications à votre Microsoft Yammer et à vosAWS comptes.

Microsoft Yammer, assurez-vous de disposer des éléments suivants dans Microsoft Yammer, assurez-vous de disposer de :

  • Création d'un compte administratif Microsoft Yammer.

  • Vous avez configuré un jeton d'identification OAuth 2.0 contenant un identifiant client et un secret client. Consultez la documentation Microsoft sur les jetons d'accès Yammer pour plus d'informations.

  • Coché, chaque document est unique dans Microsoft Yammer et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer des éléments suivants sur votre compte :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké vos informations d'authentification Microsoft Yammer dans unAWSSecrets Manager secret et, si vous utilisez l'API, avez noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de données Microsoft Yammer àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données Microsoft Yammer, vous devez fournir les informations nécessaires sur votre source de données Microsoft Yammer afin deAmazon Kendra pouvoir accéder à vos données. Si vous n'avez pas encore configuré Microsoft Yammer pourAmazon Kendra voirPrérequis.

Console

Pour vous connecterAmazon Kendra à Microsoft Yammer

  1. Connectez-vous àAWS Management Console et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez le connecteur Microsoft Yammer, puis choisissez Ajouter une source de données.

  5. Sur la page Spécifier les informations relatives à la source de données, saisissez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) : saisissez une description si vous le souhaitez de la source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Tags, pour Ajouter un nouveau tag : pour Ajouter un nouveau tag : pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, saisissez les informations suivantes sur la page Définir l'accès et la sécurité :

    1. Source : utilisez votre URL Microsoft Yammer.

    2. AWSSecrets Managersecret —Choisissez un secret existant ou créez-en un nouveauSecrets Manager pour stocker vos informations d'authentification Microsoft Yammer. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -Microsoft Yammer » est automatiquement ajouté à votre nom secret.

        2. Pour ID client, secret client : entrez les valeurs des informations d'identification que vous avez générées à partir de votre compte Microsoft Yammer.

      2. Choisissez Save (Enregistrer).

    3. Virtual Private Cloud (VPC) Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    4. Rôle IAM : choisissez un rôle IAM existant ou créer un nouveau rôle IAM pour accéder aux informations d'identification de votre référentiel et indexer le contenu de votre référentiel.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    5. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, saisissez les informations suivantes sur la page Configurer les paramètres de synchronisation :

    1. Synchroniser le contenu : sélectionnez le contenu à synchroniser.

    2. Configuration supplémentaire : vous pouvez éventuellement utiliser ces options pour indexer certains contenus au lieu de synchroniser tous les documents.

    3. Mode de synchronisation : vous pouvez choisir la manière dont vous souhaitez mettre à jour votre index lorsque le contenu de votre source de données change.

      1. Si vous choisissez la synchronisation complète, tous les contenus de toutes les entitésAmazon Kendra seront synchronisés, quel que soit l'état de synchronisation précédent.

      2. Si vous choisissez la synchronisation du contenu nouveau ou modifié, seul le contenu nouveau ou modifiéAmazon Kendra sera synchronisé.

      3. Si vous choisissez la synchronisation du contenu nouveau, modifié ou supprimé, seul le contenu nouveau, modifié ou suppriméAmazon Kendra sera synchronisé.

  8. Sur la page Définir les mappages de champs, saisissez les informations suivantes :

    1. Champs de source de données par défaut : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à Microsoft Yammer

Vous devez spécifier un JSON du schéma de source de données à l'aide de l'TemplateConfigurationAPI. Vous devez fournir les informations suivantes :

  • Source de données : vous devez spécifier la source de données sous la formeYAMMER.

  • Type : spécifiezTEMPLATE le type lorsque vous appelezCreateDataSource.

  • Secret Amazon Resource Name (ARN) : indiquez le nom de ressource Amazon (ARN) d'unSecrets Manager code secret contenant les informations d'authentification de votre compte Microsoft Yammer. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "client Id": "client ID",
    "client Secret": "client Secret",
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pourIAM attribuer un rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le connecteur Microsoft Yammer etAmazon Kendra. Pour de plus amples informations, veuillez Rôles IAM pour les sources de données Microsoft Yammer.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Virtual Private Cloud (VPC)VpcConfigurationCreateDataSource Consultez ConfigurationAmazon Kendra pour utiliser un VPC.

  • Activer le robot d'exploration d'identité : spécifiez s'il faut activer le robot d'exploration d'identité. Si le crawler d'identité est désactivé, vous devez télécharger les informations principales à l'aide de l'PutPrincipalMappingAPI. L'exploration des informations d'identité des utilisateurs et des groupes ayant accès à certains documents est utile pour le filtrage contextuel des utilisateurs. Les résultats de recherche sont filtrés en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour de plus amples informations, consultez Filtrage du contexte utilisateur.

  • Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de données Microsoft Yammer à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.
