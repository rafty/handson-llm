UntagResource

Supprime une balise d'un index, d'une FAQ ou d'une source de données.

Syntaxe de la requête

{
   "ResourceARN": "string",
   "TagKeys": [ "string" ]
}

Paramètres de demande

Pour plus d'informations sur les paramètres communs à toutes les actions, voir Paramètres communs.

Cette demande accepte les données suivantes au format JSON.

ResourceARN

L'Amazon Resource Name (ARN) de l'index, de la FAQ ou de la source de données dans lequel supprimer la balise.

Type : String

Contraintes de longueur : Longueur minimum de 1. Longueur maximale de 1011.

Obligatoire : oui

TagKeys

Liste de clés d'étiquette à supprimer de l'index, de la FAQ ou de la source de données. Si aucune clé de balise n'existe sur la ressource, elle est ignorée.

Type : tableau de chaînes

Membres du tableau : nombre minimum de 0 élément. Nombre maximum de 200 éléments.

Contraintes de longueur : Longueur minimum de 1. Longueur maximum de 128.

Obligatoire : oui

Éléments de réponse

Si l'action aboutit, le service renvoie une réponse HTTP 200 avec un corps HTTP vide.

Erreurs

Pour plus d'informations sur les erreurs communes à toutes les actions, consultezErreurs courantes.

AccessDeniedException

Vous ne disposez pas de l'accès nécessaire pour effectuer cette action. Assurez-vous de disposer des politiques d'autorisation et des comptes utilisateur requis, puis réessayez.

Code d'état HTTP : 400

InternalServerException

Un problème s'est produit avec le serveur interne utilisé pour votre service Amazon Kendra. Veuillez patienter quelques minutes et réessayez, ou contactez le support pour obtenir de l'aide.

Code d'état HTTP : 500

ResourceUnavailableException

La ressource que vous souhaitez utiliser n'est pas disponible. Vérifiez que vous avez fourni la bonne ressource et réessayez.

Code d'état HTTP : 400

ThrottlingException

La demande a été refusée suite à une limitation des demandes. Veuillez réduire le nombre de demandes et réessayez.

Code d'état HTTP : 400

ValidationException

L'entrée ne satisfait pas les contraintes définies par le service Amazon Kendra. L'entrée correcte et réessayez.

Code d'état HTTP : 400
