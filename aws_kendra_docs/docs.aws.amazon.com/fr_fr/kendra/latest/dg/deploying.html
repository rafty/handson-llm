Déploiement des Amazon Kendra

Lorsque vient le temps de déployerAmazon Kendra la recherche sur votre site Web, nous fournissons le code source que vous pouvez utiliser avec React pour prendre une longueur d'avance sur votre application. Le code source est fourni gratuitement sous une licence MIT modifiée. Vous pouvez l'utiliser tel quel ou le modifier selon vos propres besoins. L'application React fournie est un exemple pour vous aider à démarrer. Il ne s'agit pas d'une application prête pour la production.

Pour déployer une application de recherche sans code et générer une URL de point de terminaison vers votre page de recherche avec contrôle d'accès, consultez Amazon KendraExperience Builder.

Il existe deux exemples que vous pouvez utiliser avec React :

  • https://kendrasamples.s3.amazonaws.com/kendrasamples-react-app .zip : exemple d'application React qui fournit des exemples de données et une page de recherche.

  • https://kendrasamples.s3.amazonaws.com/kendrasamples.zip —Une bibliothèque que vous pouvez ajouter à une application React existante.

Les exemples sont calqués sur la page de recherche de laAmazon Kendra console. Ils ont les mêmes fonctionnalités de recherche et d'affichage des résultats de recherche. Vous pouvez utiliser l'exemple dans son intégralité ou choisir une seule des fonctionnalités pour votre propre usage.

Pour afficher les trois composants de la page de recherche dans laAmazon Kendra console, choisissez l'icône de code (</>) dans le menu de droite. Passez le pointeur sur chaque section pour afficher une brève description du composant et pour obtenir l'URL de la source du composant.

Présentation

Vous ajoutez l'exemple de code à une application React existante pour activer la recherche. Les fichiers et composants de recherche sont structurés comme suit :

  • Page de recherche principale : il s'agit de la page principale qui contient tous les composants. C'est ici que vous allez intégrer votre application à l'Amazon KendraAPI.

  • Barre de recherche : il s'agit du composant dans lequel l'utilisateur saisit un terme de recherche et qui appelle la fonction de recherche.

  • Résultats : il s'agit du composant qui affiche les résultats deAmazon Kendra. Il comporte trois éléments : des réponses suggérées, des résultats de la FAQ et des documents recommandés.

  • Facettes : il s'agit du composant qui affiche les facettes dans les résultats de recherche et qui vous permet de choisir une facette pour limiter la recherche.

  • Pagination : il s'agit du composant qui pagine la réponseAmazon Kendra.

Prérequis

Avant de commencer, vous avez besoin de ce qui suit :

  • Une application Web React existante ou l'exemple d'application.

  • Un environnement de développement configuré avec les bibliothèques appropriées.

  • Node.js et npm installés.

  • LeSDK for Java ouAWS SDK for JavaScript.

Les informations sur les bibliothèques et lesAWS SDK requis se trouvent dans le fichier Readme des fichiers zip.

Avertissement

L'application React n'est pas prête pour la production. Il s'agit d'un exemple de déploiement d'une applicationAmazon Kendra de recherche.

  1. Assurez-vous d'avoir terminéPrérequis, notamment en téléchargeant et en installant Node.js et npm.

  2. Téléchargez le kendra-samples-react-app fichier .zip et décompressez-le.

  3. Ouvrez votre terminal et accédez àaws-kendra-sample-app/src/services/. Ouvrezlocal-dev-credentials-template.json et fournissez vos informations d'identification. N'ajoutez pas ce fichier dans des référentiels publics.

  4. Accédez aux dépendancesaws-kendra-sample-app/src et installez-les. Exécutez npm install.

  5. Lancez une version de démonstration de votre application sur votre serveur local. Exécutez npm start. Vous pouvez arrêter le serveur local en tapant sur votre clavierCmd/Ctrl + C.

  6. Vous pouvez modifier le port ou l'hôte (par exemple, l'adresse IP) en accédant àpackage.json et en mettant à jour l'hôte et le port :"start": "HOST=[host] PORT=[port] react-scripts start". Si vous utilisez Windows :"start": "set HOST=[host] && set PORT=[port] && react-scripts start".

  7. Si vous avez un domaine de site Web enregistré, vous pouvez le spécifierpackage.json après le nom de votre application. Par exemple, "homepage": "https://mywebsite.com". Vous devez exécuter ànpm install nouveau pour mettre à jour les nouvelles dépendances, puis exécuternpm start.

  8. Pour créer l'application, exécuteznpm build. Téléchargez le contenu du répertoire de construction vers votre fournisseur d'hébergement.

Page de recherche principale

La page de recherche principale contient tous les exemples de composants de recherche. Il inclut le composant de barre de recherche pour la sortie, les composants de résultats pour afficher la réponse de l'API Query et un composant de pagination pour parcourir la réponse.

Composant de recherche

Le composant de recherche fournit une zone de texte pour saisir le texte de la requête. LaonSearch fonction est un hook qui appelle la fonction principaleSearch.tsx pour effectuer l'appel d'APIAmazon Kendra Query.

Composant de résultats

Le composant des résultats affiche la réponse de l'QueryAPI. Les résultats sont présentés dans trois zones distinctes.

  • Réponses suggérées : il s'agit des principaux résultats renvoyés par l'QueryAPI. Il contient jusqu'à trois suggestions de réponses. Dans la réponse, ils ont le type de résultatANSWER.

  • Réponses aux questions fréquemment posées : il s'agit des résultats des questions fréquemment posées renvoyées par la réponse. Les FAQ sont ajoutées séparément à l'index. Dans la réponse, ils ont le typeQUESTION_ANSWER. Pour plus d'informations, consultez la section Questions et réponses.

  • Documents recommandés : il s'agit de documents supplémentaires qui sontAmazon Kendra renvoyés dans la réponse. Dans la réponse de l'QueryAPI, ils ont le typeDOCUMENT.

Les composants de résultats partagent un ensemble de composants pour des fonctionnalités telles que le surlignage, les titres, les liens, etc. Les composants partagés doivent être présents pour que les composants de résultat fonctionnent.

Composant Facets

Le composant Facets répertorie les facettes disponibles dans les résultats de recherche. Chaque facette classe la réponse selon une dimension spécifique, telle que l'auteur. Vous pouvez affiner la recherche sur une facette spécifique en choisissant une dans la liste.

Une fois que vous avez sélectionné une facette, le composant appelle à l'Queryaide d'un filtre attributaire qui limite la recherche aux documents correspondant à la facette.

Composant de pagination

Le composant de pagination vous permet d'afficher les résultats de recherche de l'QueryAPI sur plusieurs pages. Il appelle l'QueryAPI avec lesPageNumber paramètresPageSize et pour obtenir une page de résultats spécifique.
