Fonctionnement d'Amazon Kendra

Amazon Kendrafournit une fonctionnalité de recherche à votre application. Il indexe vos documents directement ou à partir de votre référentiel de documents tiers et fournit intelligemment des informations pertinentes à vos utilisateurs. Vous pouvez l'utiliserAmazon Kendra pour créer un index actualisable de documents de différents types. Pour une liste des types de documents pris en charge par,Amazon Kendra voir Types de documents.

Amazon Kendras'intègre à d'autres services. Par exemple, vous pouvez équiper les robots deAmazon Lex discussion d'une fonctionAmazon Kendra de recherche afin de fournir des réponses utiles aux questions des utilisateurs. Vous pouvez utiliser un Amazon Simple Storage Servicebucket comme source de données pour vous connecterAmazon Kendra à vos documents et les indexer. Et vous pouvez configurer des politiques d'accès ou des autorisations aux ressources que vous utilisez AWS Identity and Access Management.

Amazon Kendra dispose des composants suivants :

  • Un index qui contient vos documents et permet de les consulter.

  • Source de données qui stocke vos documents et qui s'Amazon Kendray connecte. Vous pouvez synchroniser automatiquement une source de données avec unAmazon Kendra index afin que votre index reste à jour avec votre référentiel source.

  • API d'ajout de documents qui ajoute des documents directement à un index.

Vous pouvez utiliserAmazon Kendra à l'aide de la console ou de l'API. Vous pouvez créer, mettre à jour et supprimer des index. La suppression d'un index entraîne la suppression de tous ses connecteurs de source de données et supprime définitivement toutes les informations de votre documentAmazon Kendra.
