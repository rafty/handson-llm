Slack

Slack est une application de communication d'entreprise qui permet aux utilisateurs d'envoyer des messages et des pièces jointes via différents canaux publics et privés. Vous pouvez l'utiliserAmazon Kendra pour indexer vos chaînes publiques et privées Slack, pour botter et archiver des messages, des fichiers et des pièces jointes, des messages directs et de groupe. Vous pouvez également choisir un contenu spécifique à filtrer.

Vous pouvez vous connecterAmazon Kendra à votre source de données Slack à l'aide de la Amazon Kendraconsole et de l'SlackConfigurationAPI.

Pour résoudre les problèmes liés à votre connecteur de source de donnéesAmazon Kendra Slack, consultezRésolution des problèmes de sources de données.

Prérequis

Avant de pouvoirAmazon Kendra indexer votre source de données Slack, apportez ces modifications à votre Slack et à vosAWS comptes.

Sur Slack, assurez-vous de disposer des éléments suivants :

  • Création d'un jeton OAuth pour utilisateur Slack Bot ou d'un jeton OAuth pour utilisateur Slack. Vous pouvez choisir l'un ou l'autre des jetons pour vous connecterAmazon Kendra à votre source de données Slack. Consultez la documentation Slack sur les jetons d'accès pour plus d'informations.

    Note

    Si vous utilisez le jeton de robot dans le cadre de vos informations d'identification Slack, vous ne pouvez pas indexer les messages directs et les messages de groupe et vous devez ajouter le jeton de robot à la chaîne que vous souhaitez indexer.

  • Vous avez noté l'identifiant de votre équipe d'espace de travail Slack sur l'URL de la page principale de votre espace de travail Slack. Par exemple, https://app.slack.com/client/T0123456789/... où T0123456789 est l'ID de l'équipe.

  • Les autorisations [Oauth scopes/ read] suivantes ont été ajoutées :

    • chaînes : histoire

    • chaînes : lire

    • groupes : histoire

    • groupes : lire

    • im:history

    • mpim : histoire

    • mpim : lire

    • team:read

    • users.profile : lire

    • utilisateurs : lire

    • emoji : lire

    • fichiers:lire

    • groupes d'utilisateurs : lire

  • Coché, chaque document est unique dans Slack et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer de :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké vos informations d'authentification Slack deAWSSecrets Manager manière secrète et, si vous utilisez l'API, vous avez noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de données Slack àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données Slack, vous devez fournir les informations nécessaires sur votre source de données Slack afin que vousAmazon Kendra puissiez accéder à vos données. Si vous n'avez pas encore configuré Slack pourAmazon Kendra voirPrérequis.

Console

Pour vous connecterAmazon Kendra à Slack

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez le connecteur Slack, puis choisissez Ajouter une source de données.

  5. Sur la page Spécifier les détails de source de données, saisissez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) —Entrez une description si vous le souhaitez pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Tags, pour Ajouter un nouveau tag : Tags pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, saisissez les informations suivantes :

    1. ID d'équipe de l'espace de travail Slack : identifiant de l'équipe de votre espace de travail Slack.

    2. AWSSecrets Managersecret —Choisissez un secret existant ou créez-en un nouveauSecrets Manager pour stocker vos informations d'authentification Slack. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -Slack » est automatiquement ajouté à votre nom secret.

        2. Pour le jeton Slack : entrez les valeurs des informations d'identification que vous avez créées dans votre compte Slack.

      2. Choisissez Save (Enregistrer).

    3. Virtual Private Cloud (VPC). Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    4. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification de votre référentiel et au contenu de l'index.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    5. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, saisissez les informations suivantes :

    1. Sélectionnez le type de contenu à analyser : entités Slack ou types de contenu que vous souhaitez analyser.

    2. Journal des modifications : sélectionnez cette option pour mettre à jour votre index au lieu de synchroniser tous vos fichiers.

    3. Modèles Regex : modèles d'expression régulière permettant d'inclure ou d'exclure certains fichiers. Vous pouvez ajouter jusqu'à 100 motifs.

    4. Sélectionnez la date de début de l'analyse : date à partir de laquelle vos donnéesAmazon Kendra commenceront à être explorées.

    5. Dans Synchroniser le calendrier d'exécution, pour Fréquence : à quelle fréquence la synchronisation avec votre source de donnéesAmazon Kendra sera effectuée.

    6. Choisissez Suivant.

  8. Sur la page Définir les Mappages de champs, saisissez les informations suivantes :

    1. Pour les mappages de champs Slack : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à Slack

Vous devez spécifier les éléments suivants à l'aide de l'SlackConfigurationAPI :

  • ID d'équipe de l'espace de travail Slack : l'identifiant de l'équipe Slack que vous avez copié à partir de l'URL de votre page principale de Slack.

  • Liste des entités à indexer : indique si vousAmazon Kendra devez indexer vos chaînes publiques et privées, vos messages de groupe et vos messages directs.

  • Date d'exploration : date à laquelle vous devez commencer à analyser les données de l'équipe de votre espace de travail Slack. La date doit suivre ce format : yyyy-mm-dd.

  • Secret Amazon Resource Name (ARN) : indiquez le nom de ressource Amazon (ARN) d'unSecrets Manager code secret contenant les informations d'authentification de votre compte Slack. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "slackToken": "token"
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pourIAM attribuer un rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le connecteur Slack etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de données Slack.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Virtual Private Cloud (VPC) : spécifiez dans leVpcConfiguration cadre de la configuration de la source de données. Reportez-vous Amazon Kendraà la section Configuration pour utiliser un VPC.

  • Journal des modifications : s'ilAmazon Kendra faut utiliser le mécanisme du journal des modifications des sources de données Slack pour déterminer si un document doit être ajouté, mis à jour ou supprimé dans l'index.

    Note

    Utilisez le journal des modifications si vous neAmazon Kendra souhaitez pas numériser tous les documents. Si votre journal des modifications est volumineux, la numérisation des documents de la source de données Slack peut prendreAmazon Kendra moins de temps que le traitement du journal des modifications. Si vous synchronisez votre source de données Slack avec votre index pour la première fois, tous les documents sont numérisés.

  • Filtres d'inclusion et d'exclusion : spécifiez si vous souhaitez inclure ou exclure certaines chaînes publiques et privées, des messages de groupe et privés, ainsi que des messages de robots et des messages archivés. Si vous utilisez un jeton de robot dans le cadre de vos informations d'authentification Slack, vous devez ajouter le jeton de robot à la chaîne que vous souhaitez indexer. Vous ne pouvez pas indexer les messages directs et les messages de groupe à l'aide d'un jeton de robot.

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de données Slack à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats de recherche en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour plus d'informations, consultez Filtrage du contexte utilisateur pour les sources de données Slack.
