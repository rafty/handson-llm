Amazon RDS/Aurora

Vous pouvez indexer des documents stockés dans une base de données à l'aide d'une source de données de base de données. Après avoir fourni les informations de connexion à la base de données,Amazon Kendra connecte et indexe les documents.

Amazon Kendraprend en charge les bases de données suivantes :

  • Amazon Aurora MySQL

  • Amazon Aurora PostgreSQL

  • Amazon RDS pour MySQL

  • Amazon RDS pour PostgreSQL

Note

Les bases de données Aurora sans serveur ne sont pas prises en charge.

Vous pouvez vous connecterAmazon Kendra à la source de données de votre base de données à l'aide de la Amazon Kendraconsole et de l'DatabaseConfigurationAPI.

Pour résoudre les problèmes liés au connecteur de sourceAmazon Kendra de données de votre base de données, consultezRésolution des problèmes de sources de données.

Fonctionnalités prises en charge

Amazon Kendrale connecteur de source de données de base de données prend en charge les fonctionnalités suivantes :

  • Mappages de champs

  • Filtrage du contexte utilisateur

  • Virtual privatecCloud (VPC)

Prérequis

Avant de pouvoirAmazon Kendra indexer la source de données de votre base de données, apportez ces modifications à votre base de données et à vosAWS comptes.

Dans votre base de données, assurez-vous de disposer :

  • Vous avez noté votre paire utilisateur-mot de passe comme identifiant d'authentification de base pour votre source de données.

  • A copié le nom d'hôte, le numéro de port, l'adresse de l'hôte et le nom de la table de données contenant les données de la base de données. Pour PostgreSQL, la table de données doit être une table publique.

    Note

    L'hôte et le port indiquentAmazon Kendra où trouver le serveur de base de données sur Internet. Le nom de la base de données et le nom de la table indiquentAmazon Kendra où trouver les données du document sur le serveur de base de données.

  • Vous avez copié les noms des colonnes de la table de données qui contiennent les données du document, l'ID du document, une à cinq colonnes pour détecter si un document a été modifié et les colonnes facultatives de la table de données qui correspondent à des champs d'index personnalisés. Vous pouvez associer n'importe quel nom de champAmazon Kendra réservé à une colonne de table.

  • Vous avez copié les informations relatives au type de moteur de base de données, par exemple si vous l'utilisezAmazon RDS pour MySQL ou un autre type.

  • Coché, chaque document est unique dans la base de données et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer des éléments suivants :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké les informations d'authentification de votre base de données dans unAWSSecrets Manager secret et, si vous utilisez l'API, avez noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de données de base de données àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données de base de données, vous devez fournir les informations nécessaires sur votre source de données de base de données afin deAmazon Kendra pouvoir accéder à vos données. Si vous n'avez pas encore configuré la base de données pourAmazon Kendra voirPrérequis.

Console

Pour vous connecterAmazon Kendra à une base de données

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez le connecteur de base de données, puis choisissez Ajouter une source de données.

  5. Sur la page Spécifier les détails de source de données, entrez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) : entrez une description facultative pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Balises, pour Ajouter une nouvelle étiquette : Tags pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, entrez les informations suivantes :

    1. Point de terminaison : nom d'hôte DNS, adresse IPv4 ou adresse IPv6.

    2. Port : numéro de port.

    3. Base de données : nom de la base de données.

    4. Nom de la table : nom de la table.

    5. Dans Type d'authentification, choisissez entre Existant et Nouveau pour stocker les informations d'authentification de votre base de données. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -database-» est automatiquement ajouté à votre nom secret.

        2. Pour le nom d'utilisateur et le mot de passe : entrez les valeurs des informations d'identification de votre compte de base de données.

        3. Choisissez Enregistrer l'authentification.

    6. Virtual Private Cloud (VPC) : vous pouvez choisir d'utiliser un VPC. Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

      Note

      Vous devez utiliser un sous-réseau privé. Si votre instance RDS se trouve dans un sous-réseau public de votre VPC, vous pouvez créer un sous-réseau privé qui dispose d'un accès sortant à une passerelle NAT dans le sous-réseau public. Les sous-réseaux fournis dans la configuration VPC doivent se trouver dans les régions USA Ouest (Oregon), USA Est (Virginie du Nord) ou USA Est (Virginie du Nord) ou UE (Irlande).

    7. Rôle IAM : choisissez un rôle IAM existant ou créez un nouveau rôle IAM pour accéder aux informations d'identification de votre référentiel et au contenu de l'index.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    8. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, entrez les informations suivantes :

    1. Choisissez entre Aurora MySQL, MySQL, Aurora PostgreSQL et PostgreSQL en fonction de votre cas d'utilisation.

    2. Placer les identificateurs SQL entre guillemets doubles : sélectionnez cette option pour placer les identificateurs SQL entre guillemets doubles. Par exemple, « ColumnName ».

    3. Colonne ACL et colonnes de détection des modifications : à utiliser pour configurer les colonnesAmazon Kendra utilisées pour la détection des modifications et les listes de contrôle d'accès.

    4. Dans Synchroniser le calendrier d'exécution, pour Fréquence, choisissez la fréquence àAmazon Kendra laquelle vous souhaitez synchroniser avec votre source de données.

    5. Choisissez Suivant.

  8. Sur la page Définir les Mappages de champs, entrez les informations suivantes :

    1. Amazon Kendramappages de champs par défaut : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index. Vous devez ajouter les valeurs des colonnes de base de données pourdocument_id etdocument_body

    2. Mappages de champs personnalisés : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à une base de données

Vous devez spécifier l'DatabaseConfigurationAPI suivante :

  • ColumnConfiguration: informations sur l'endroit où l'index doit obtenir les informations du document à partir de la base de données. Pour en savoir plus, consultez ColumnConfiguration. Vous devez spécifier lesDocumentIdColumnName champsDocumentDataColumnName et. La colonne mappée auDocumentIdColumnName champ doit être une colonne entière. L'exemple suivant illustre une configuration de colonne simple pour une source de données de base de données : 

    "ColumnConfiguration": {
    "ChangeDetectingColumns": [
        "LastUpdateDate",
        "LastUpdateTime"
    ],
    "DocumentDataColumnName": "TextColumn",
    "DocumentIdColumnName": "IdentifierColumn",
    "DocoumentTitleColumnName": "TitleColumn",
    "FieldMappings": [
        {
            "DataSourceFieldName": "AbstractColumn",
            "IndexFieldName": "Abstract"
        }
    ]
}

  • ConnectionConfiguration—Type de moteur de base de données qui exécute la base de données. LeDatabaseHost champ doit être le point de terminaison de l'instanceAmazon Relational Database Service (Amazon RDS) de la base de données. N'utilisez pas le point de terminaison de cluster.

  • DatabaseEngineType: informations de configuration requises pour se connecter à une base de données. Pour en savoir plus, consultez ConnectionConfiguration.

  • Secret Amazon Resource Name (ARN) : indiquez le nom de ressource Amazon (ARN) d'un codeSecrets Manager secret contenant les informations d'authentification de votre compte de base de données. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "username": "user name",
    "password": "password"
}

    Le secret peut contenir plus d'informations. L'exemple suivant illustre une configuration de base de données.

    
"DatabaseConfiguration": {
"ConnectionConfiguration": {
"DatabaseHost": "host.subdomain.domain.tld",
        "DatabaseName": "DocumentDatabase",
        "DatabasePort": 3306,
        "SecretArn": "arn:aws:secretmanager:region:account ID:secret/secret name",
        "TableName": "DocumentTable"
    }
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pour fournir unIAM rôle autorisé à accéder à votreSecrets Manager secret et pour appeler les API publiques requises pour le connecteur de base de données etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de données de base de données.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Virtual Private Cloud (VPC) : spécifiez dans leVpcConfiguration cadre de la configuration de la source de données. Reportez-vous Amazon Kendraà la section Configuration pour utiliser un VPC.

    Note

    Vous devez uniquement utiliser un sous-réseau privé. Si votre instance RDS se trouve dans un sous-réseau public de votre VPC, vous pouvez créer un sous-réseau privé qui dispose d'un accès sortant à une passerelle NAT dans le sous-réseau public. Les sous-réseaux fournis dans la configuration VPC doivent se trouver dans les régions USA Ouest (Oregon), USA Est (Virginie du Nord) ou USA Est (Virginie du Nord) ou UE (Irlande).

  • Mappages de champs : choisissez de mapper les champs de votre source de données de base de données à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats de recherche en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour plus d'informations, voir Filtrage contextuel utilisateur pour les sources de données de base de données.
