GitHub

GitHub est un service d'hébergement Web pour le développement de logiciels fournissant des services de stockage et de gestion de code avec contrôle de version. Vous pouvez les utiliserAmazon Kendra pour indexer vos fichiers de référentiel GitHub Enterprise Cloud (SaaS) et GitHub Enterprise Server (On Prem), émettre et extraire des requêtes, des commentaires sur les demandes d'émission et d'extraction, ainsi que des pièces jointes aux commentaires des demandes d'émission et d'extraction. Vous pouvez également choisir d'inclure ou d'exclure certains fichiers.

Vous pouvez vous connecterAmazon Kendra à votre source de GitHub données à l'aide de la Amazon Kendraconsole et de l'GitHubConfigurationAPI.

Pour résoudre les problèmes liés à votre connecteur de source deAmazon Kendra GitHub données, consultezRésolution des problèmes de sources de données.

Fonctionnalités prises en charge

Amazon Kendra GitHub Le connecteur de source de données prend en charge les fonctionnalités suivantes :

  • Journal des modifications

  • Mappages de champs de champs

  • Filtrage du contexte utilisateur

  • Filtres d'inclusion/d'exclusion

Prérequis

Avant de pouvoir l'utiliserAmazon Kendra pour indexer votre source de GitHub données, apportez ces modifications à vosAWS comptes GitHub et.

Dans GitHub, assurez-vous d'avoir :

  • Création d'un GitHub utilisateur doté de droits d'administrateur sur l' GitHub organisation.

  • Création d'un jeton d'accès personnel pour les informations d'authentification. Consultez GitHub la documentation sur la création d'un jeton d'accès personnel.

  • Recommandé : création d'un jeton OAuth pour les informations d'authentification. Utilisez le jeton OAuth pour améliorer les limites d'accélération des API et les performances du connecteur. Consultez GitHub la documentation sur l'autorisation OAuth.

  • Facultatif : vous avez installé un certificat SSL.

  • Vous avez noté l'URL de l' GitHub hôte correspondant au type de GitHub service que vous utilisez. Par exemple, l'URL hôte pour le GitHub cloud peut être https://api.github.com et l'URL hôte pour le GitHub serveur peut être https ://on-prem-host-url/api/v3/.

  • Vous avez noté le nom de GitHub l'organisation de vos référentiels GitHub dans vos paramètres.

  • Ajout des autorisations suivantes :

    Pour GitHub Enterprise Cloud (SaaS)

    • repo:status

    • public_repo

    • repo:invite

    • lire:org

    • utilisateur:email

    • lire : utilisateur

    Pour GitHub Enterprise Server (sur site)

    • repo:status

    • public_repo

    • repo:invite

    • lire:org

    • utilisateur:email

    • lire : utilisateur

    • administrateur du site

  • Coché, chaque document est unique dans GitHub et entre les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous d'avoir les éléments suivants :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké vos informations d' GitHub authentification dans unAWSSecrets Manager secret et, si vous utilisez l'API, vous avez noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de GitHub données àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion.

Pour vous connecterAmazon Kendra à votre source de GitHub données, vous devez fournir les informations nécessaires sur votre source de GitHub données afin deAmazon Kendra pouvoir accéder à vos données. Si vous n'avez pas encore configuré GitHub pourAmazon Kendra voirPrérequis.

Console

Pour vous connecterAmazon Kendra à GitHub

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez GitHub connecteur, puis Ajouter une source de données.

  5. Sur la page Spécifier les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) —Entrez une description facultative pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Tags, pour Ajouter un nouveauAWS tag.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité suivante :

    1. GitHubsource —Choisissez entre GitHub Enterprise Cloud et GitHubEnterprise Server.

    2. GitHub URL de l'hôte : entrez le nom de votre GitHub hôte.

    3. GitHub nom de l'organisation : entrez le nom de votre GitHub organisation. Vous pouvez trouver les informations relatives à votre organisation dans votre GitHub compte.

    4. AWSSecrets Managersecret —Choisissez un secret existant ou créez-en un nouveauSecrets Manager pour stocker vos informations GitHub d'authentification. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -GitHub - » est automatiquement ajouté à votre nom secret.

        2. Pour le GitHubjeton : entrez les valeurs des informations d'identification que vous avez créées dans votre GitHub compte.

      2. Choisissez Save (Enregistrer).

    5. Virtual Private Cloud (VPC) : vous pouvez choisir d'utiliser un VPC. Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    6. Rôle IAM : choisissez un rôle IAM existant pour accéder aux informations d'identification et au contenu de votre référentiel.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    7. Choisissez Suivant.

  7. Sur la page Configuration des paramètres de synchronisation suivante :

    1. Sélectionnez les référentiels à analyser : GitHub entités ou types de contenu que vous souhaitez analyser.

    2. Journal des modifications : sélectionnez cette option pour mettre à jour votre index au lieu de synchroniser tous vos fichiers.

    3. Types de contenu : sélectionnez les types de fichiers que vous souhaitez inclure.

    4. Modèles Regex : modèles d'expression régulière permettant d'inclure ou d'exclure certains fichiers. Vous pouvez ajouter jusqu'à 100 motifs.

    5. Dans Synchroniser le calendrier d'exécution, pour la fréquence : à quelle fréquence Amazon Kendra se synchronisera avec votre source de données.

    6. Choisissez Suivant.

  8. Sur la page Définir les Mappages de champs suivants :

    1. Pour le référentiel, la validation du référentiel, le document de publication, le commentaire de problème, la pièce jointe au problème, le commentaire de demande d'extraction, le document de demande d'extraction, la pièce jointe à la demande d'extraction, sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à GitHub

Vous devez spécifier les éléments suivants à l'aide de l'objet GitHubConfigurationAPI :

  • Type de source de données : spécifiez le type de source de données sous la formeSAAS ouON_PREMISE.

  • Secret Amazon Resource Name (ARN) : indiquez le nom de ressource Amazon (ARN) d'unSecrets Manager code secret contenant les informations d'authentification de votre GitHub compte. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "personalToken": "token"
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pour fournir unIAM rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le GitHub connecteur etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de GitHub données.

Vous pouvez également ajouter les fonctionnalités suivantes :

  • Virtual Private Cloud (VPC) : spécifiez dans leVpcConfiguration cadre de la configuration de la source de données. Reportez-vous Amazon Kendraà la section Configuration pour utiliser un VPC.

    Note

    Si vous utilisez un GitHub serveur, vous devez utiliser unAmazon VPC pour vous connecter à votre GitHub serveur.

  • Journal des modifications : s'ilAmazon Kendra faut utiliser le mécanisme du journal des modifications de la source de GitHub données pour déterminer si un document doit être ajouté, mis à jour ou supprimé dans l'index.

    Note

    Utilisez le journal des modifications si vous neAmazon Kendra souhaitez pas numériser tous les documents. Si votre journal des modifications est volumineux, la numérisation des documents de la source de GitHub données peut prendreAmazon Kendra moins de temps que le traitement du journal des modifications. Si vous synchronisez votre source de GitHub données avec votre index pour la première fois, tous les documents sont numérisés.

  • Filtres d'inclusion et d'exclusion : spécifiez s'il faut inclure ou exclure certains fichiers.

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de GitHub données à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats de recherche en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour plus d'informations, voir Filtrage contextuel utilisateur pour les sources de GitHub données.
