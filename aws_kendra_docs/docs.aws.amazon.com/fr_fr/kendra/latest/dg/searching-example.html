Interrogation d'un index

Lorsque vous effectuez une recherche dans votre index, ilAmazon Kendra utilise toutes les informations que vous avez fournies sur vos documents pour déterminer les documents les plus pertinents par rapport aux termes de recherche saisis. Certains des éléments pris enAmazon Kendra compte sont les suivants :

  • Le texte du document.

  • Titre du document.

  • Champs de texte personnalisés que vous avez marqués comme pouvant faire l'objet d'une recherche.

  • Le champ de date que vous avez indiqué doit être utilisé pour déterminer la « fraîcheur » d'un document.

Lorsqu'un ensemble de documents pertinents a été sélectionné dans l'index,Amazon Kendra filtre la réponse en fonction des filtres d'attributs que vous avez demandés pour la recherche. Par exemple, si vous avez un attribut personnalisé appelé « département », vous pouvez filtrer la réponse pour ne renvoyer que les documents provenant d'un service appelé « juridique ». Pour de plus amples informations, veuillez consulter Champs ou attributs personnalisés.

Après avoir trouvé les documents pertinents, puis filtré en fonction des attributs que vous avez définis, les résultats sontAmazon Kendra renvoyés. Les résultats sont triés selon la pertinenceAmazon Kendra déterminée pour chaque document. Les résultats sont paginés afin que vous puissiez montrer une page à la fois à votre utilisateur.

Pour rechercher des documentsAmazon Kendra pour lesquels vous avez indexéAmazon Lex, utilisez AMAZON. KendraSearchIntent. Pour un exemple de configurationAmazon Kendra avecAmazon Lex, consultez la section Création d'un bot FAQ pour unAmazon Kendra index.

Pour plus d'informations sur les réponses aux requêtes, consultezRéponses aux requêtes et types de réponses.

Recherche dans un index (console)

Vous pouvez utiliser laAmazon Kendra console pour rechercher et tester votre index. Vous pouvez effectuer des requêtes et voir les résultats.

Pour effectuer une recherche dans un index à l'aide de la console

  1. Connectez-vous à la consoleAWS Management Console et ouvrez laAmazon Kendra console à l'adresse http://console.aws.amazon.com/kendra/.

  2. Dans le volet de navigation, choisissez Indexes.

  3. Choisissez votre index.

  4. Dans le menu de navigation, choisissez l'option de recherche pour effectuer une recherche dans votre index.

  5. Saisissez une requête dans la zone de texte, puis appuyez sur Entrée.

  6. Amazon Kendrarenvoie les résultats de la recherche.

Recherche dans un index (SDK)

Pour rechercher un index avec Python ou Java

  • L'exemple suivant effectue une recherche dans un index. Modifiez la valeurquery de votre requête de rechercheindex_id et/ouindexId de l'identifiant d'index de l'index dans lequel vous souhaitez effectuer la recherche.

    Python
    import boto3
import pprint

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the query text
query = "search-string"

response = kendra.query(
        QueryText = query,
        IndexId = index_id)

print("\nSearch results for query: " + query + "\n")        

for query_result in response["ResultItems"]:

    print("-------------------")
    print("Type: " + str(query_result["Type"]))
        
    if query_result["Type"]=="ANSWER" or query_result["Type"]=="QUESTION_ANSWER":
        answer_text = query_result["DocumentExcerpt"]["Text"]
        print(answer_text)

    if query_result["Type"]=="DOCUMENT":
        if "DocumentTitle" in query_result:
            document_title = query_result["DocumentTitle"]["Text"]
            print("Title: " + document_title)
        document_text = query_result["DocumentExcerpt"]["Text"]
        print(document_text)

    print("------------------\n\n")
    Java
    package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.QueryRequest;
import software.amazon.awssdk.services.kendra.model.QueryResponse;
import software.amazon.awssdk.services.kendra.model.QueryResultItem;

public class SearchIndexExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String query = "some queries";
        String indexId = "anIndexId";

        QueryRequest queryRequest = QueryRequest
            .builder()
            .queryText(query)
            .indexId(indexId)
            .build();

        QueryResponse queryResponse = kendra.query(queryRequest);

        System.out.println(String.format("\nSearch results for query: %s", query));
        for(QueryResultItem item: queryResponse.resultItems()) {
            System.out.println("----------------------");
            System.out.println(String.format("Type: %s", item.type()));

            switch(item.type()) {
                case QUESTION_ANSWER:
                case ANSWER:
                    String answerText = item.documentExcerpt().text();
                    System.out.println(answerText);
                    break;
                case DOCUMENT:
                    String documentTitle = item.documentTitle().text();
                    System.out.println(String.format("Title: %s", documentTitle));
                    String documentExcerpt = item.documentExcerpt().text();
                    System.out.println(String.format("Excerpt: %s", documentExcerpt));
                    break;
                default:
                    System.out.println(String.format("Unknown query result type: %s", item.type()));

            }

            System.out.println("-----------------------\n");
        }
    }
}

Recherche à l'aide d'une syntaxe de requête avancée

Vous pouvez créer des requêtes plus spécifiques que de simples requêtes par mot clé ou en langage naturel en utilisant une syntaxe de requête ou des opérateurs avancés. Cela inclut les plages, les booléens, les caractères génériques, etc. En utilisant des opérateurs, vous pouvez donner plus de contexte à votre requête et affiner davantage les résultats de recherche.

Amazon Kendraprend en charge les opérateurs suivants.

  • Boolean : logique visant à limiter ou à élargir la recherche. Par exemple,amazon AND sports limite la recherche aux seuls documents contenant les deux termes.

  • Parenthèses : lit les termes de requête imbriqués par ordre de priorité. Par exemple,(amazon AND sports) NOT rainforest lit(amazon AND sports) avantNOT rainforest.

  • Plages : valeurs de dates ou de plages numériques. Les plages peuvent être inclusives, exclusives ou illimitées. Par exemple, vous pouvez rechercher des documents qui ont été mis à jour pour la dernière fois entre le 1er janvier 2020 et le 31 décembre 2020, y compris ces dates.

  • Champs : utilise un champ spécifique pour limiter la recherche. Par exemple, vous pouvez rechercher des documents portant la mention « États-Unis » dans le champ « lieu ».

  • Caractères génériques : correspondent partiellement à une chaîne de texte. Par exemple,Cloud* pourrait correspondre CloudFormation. Amazon Kendrane prend actuellement en charge que les caractères génériques de fin.

  • Citations exactes : Correspond exactement à une chaîne de texte. Par exemple, les documents qui contiennent"Amazon Kendra" "pricing".

Vous pouvez utiliser une combinaison de l'un des opérateurs ci-dessus.

Notez que l'utilisation excessive d'opérateurs ou de requêtes très complexes peut avoir un impact sur la latence des requêtes. Les wildcards font partie des opérateurs les plus coûteux en termes de latence. En règle générale, plus vous utilisez de termes et d'opérateurs, plus l'impact potentiel sur la latence est important. Parmi les autres facteurs qui influent sur la latence, citons la taille moyenne des documents indexés, la taille de votre index, tout filtrage sur les résultats de recherche et la charge globale de votreAmazon Kendra index.

Booléen

Vous pouvez combiner ou exclure des mots à l'aide des opérateurs booléensAND,OR,NOT.

Vous trouverez ci-dessous des exemples d'utilisation d'opérateurs booléens.

amazon AND sports

Renvoie les résultats de recherche qui contiennent à la fois les termes « amazon » et « sports » dans le texte, tels que Amazon Prime Video Sports ou tout autre contenu similaire.

sports OR recreation

Renvoie les résultats de recherche qui contiennent les termes « sports » ou « loisirs », ou les deux, dans le texte.

amazon NOT rainforest

Renvoie les résultats de recherche qui contiennent le terme « amazon » mais pas le terme « forêt tropicale » dans le texte. Il s'agit de rechercher des documents sur la société Amazon, et non sur la forêt amazonienne.

Parenthèses

Vous pouvez rechercher des mots imbriqués par ordre de priorité en utilisant des parenthèses. Les parenthèses indiquentAmazon Kendra comment une requête doit être lue.

Vous trouverez ci-dessous des exemples d'utilisation d'opérateurs entre parenthèses.

(amazon AND sports) NOT rainforest

Renvoie les documents qui contiennent à la fois les termes « amazon » et « sports » dans le texte, mais pas le terme « forêt tropicale ». Il s'agit de rechercher des sports vidéo sur Amazon Prime ou d'autres contenus similaires, et non des sports d'aventure dans la forêt amazonienne. Les parenthèses aident à indiquer ce quiamazon AND sports doit être lu avantNOT rainforest. La requête ne doit pas être lue commeamazon AND (sports NOT rainforest).

(amazon AND (sports OR recreation)) NOT rainforest

Renvoie les documents qui contiennent les termes « sports » ou « loisirs », ou les deux, et le terme « amazon ». Mais il n'inclut pas le terme « forêt tropicale ». Il s'agit de rechercher des vidéos de sports ou de loisirs sur Amazon Prime, et non de sports d'aventure dans la forêt amazonienne. Les parenthèses aident à indiquer ce quisports OR recreation doit être lu avant de le combiner avec « amazon », qui est lu auparavantNOT rainforest. La requête ne doit pas être lue commeamazon AND (sports OR (recreation NOT rainforest)).

Gammes

Vous pouvez utiliser une plage de valeurs pour filtrer les résultats de recherche. Vous spécifiez un attribut et les valeurs de plage. Il peut s'agir d'une date ou d'un type numérique.

Les plages de dates doivent être au format suivant :

  • Epoch

  • YYYY

  • YYYY-mm

  • YYYY-MM-DD

  • YYYY-MM-DD'T'HH

Vous pouvez également spécifier si vous souhaitez inclure ou exclure les valeurs inférieures et supérieures de la plage.

Vous trouverez ci-dessous des exemples d'utilisation d'opérateurs de plage.

_processed_date:>2019-12-31 AND _processed_date:<2021-01-01

Renvoie les documents qui ont été traités en 2020, entre le 31 décembre 2019 et le 1er janvier 2021.

_processed_date:>=2020-01-01 AND _processed_date:<=2020-12-31

Renvoie les documents qui ont été traités en 2020, supérieurs ou égaux au 1er janvier 2020 et inférieurs ou égaux au 31 décembre 2020.

_document_likes:<1

Renvoie les documents n'ayant reçu aucun « J'aime » ou aucun commentaire utilisateur (moins d'un « J'aime »).

Vous pouvez spécifier si une plage doit être traitée comme incluant ou excluant les valeurs de plage données.

Inclusif

_last_updated_at:[2020-01-01 TO 2020-12-31]

Les documents de retour ont été mis à jour pour la dernière fois en 2020, y compris les jours du 1er décembre 2020 et du 31 décembre 2020.

Exclusif

_last_updated_at:{2019-12-31 TO 2021-01-01}

Retourne les documents mis à jour pour la dernière fois en 2020, à l'exclusion des jours du 31 décembre 2019 et du 1er janvier 2021.

Pour les plages illimitées qui ne sont ni inclusives ni exclusives, utilisez simplement les < and > opérateurs. Par exemple, _last_updated_at:>2019-12-31 AND _last_updated_at:<2021-01-01

Champs

Vous pouvez limiter votre recherche pour ne renvoyer que les documents qui correspondent à une valeur dans un champ spécifique. Le champ peut être de n'importe quel type.

Vous trouverez ci-dessous des exemples d'utilisation d'opérateurs contextuels au niveau du champ.

status:"Incomplete" AND financial_year:2021

Renvoie les documents relatifs à l'exercice 2021 ayant le statut d'incomplets.

(sports OR recreation) AND country:"United States" AND level:"professional"

Renvoie des documents traitant de sports professionnels ou de loisirs aux États-Unis.

Caractères génériques

Vous pouvez élargir votre recherche pour inclure des variantes de mots et de phrases à l'aide de l'opérateur générique. Cela s'avère utile lors de la recherche de variantes de noms. Amazon Kendrane prend actuellement en charge que les caractères génériques de fin. Le nombre de caractères de préfixe pour un caractère générique final doit être supérieur à deux.

Vous trouverez ci-dessous des exemples d'utilisation d'opérateurs génériques.

Cloud*

Renvoie les documents contenant des variantes telles que CloudFormation et CloudWatch.

kendra*aws

Renvoie des documents contenant des variantes telles que kendra.amazonaws.

kendra*aws*

Renvoie des documents contenant des variantes telles que kendra.amazonaws.com

Citations exactes

Vous pouvez utiliser des guillemets pour rechercher la correspondance exacte d'un texte.

Vous trouverez ci-dessous des exemples d'utilisation de guillemets.

"Amazon Kendra" "pricing"

Renvoie les documents qui contiennent à la fois la phrase «Amazon Kendra » et le terme « tarification ». Les documents doivent contenir à la foisAmazon Kendra « » et « prix » pour que les résultats puissent être renvoyés.

"Amazon Kendra" "pricing" cost

Renvoie les documents qui contiennent à la fois l'expressionAmazon Kendra « » et le terme « prix », et éventuellement le terme « coût ». Les documents doivent contenir à la foisAmazon Kendra « » et « prix » pour que les résultats puissent être affichés, mais ils ne doivent pas nécessairement inclure le « coût ».

Syntaxe de requête non valide

Amazon Kendraémet un avertissement en cas de problème de syntaxe de votre requête ou si votre requête n'est actuellement pas prise en charge parAmazon Kendra. Pour de plus amples informations, veuillez consulter la documentation de l'API pour les avertissements de requête.

Les requêtes suivantes sont des exemples de syntaxe de requête non valide.

_last_updated_at:<2021-12-32

Date non valide. Le jour 32 n'existe pas dans le calendrier grégorien, qui est utilisé parAmazon Kendra.

_view_count:ten

Valeur numérique non valide. Les chiffres doivent être utilisés pour représenter des valeurs numériques.

nonExistentField:123

Champ de recherche non valide. Le champ doit exister pour pouvoir utiliser la recherche par champ.

Product:[A TO D]

Plage non valide. Des valeurs numériques ou des dates doivent être utilisées pour les plages.

OR Hello

Boolean non valide. Les opérateurs doivent être utilisés avec des termes et placés entre les termes.

Recherche dans les langues

Vous pouvez rechercher des documents dans une langue prise en charge. Vous transmettez le code de langue dans le AttributeFilterpour renvoyer les documents filtrés dans la langue de votre choix. Vous pouvez saisir la requête dans une langue prise en charge.

Si vous ne spécifiez aucune langue, rechercheAmazon Kendra les documents en anglais par défaut. Pour plus d'informations sur les langues prises en charge, y compris leurs codes, voir Ajout de documents dans des langues autres que l'anglais.

Pour rechercher des documents dans une langue prise en charge dans la console, sélectionnez votre index, puis sélectionnez l'option permettant de rechercher dans votre index dans le menu de navigation. Choisissez la langue dans laquelle vous souhaitez renvoyer les documents en sélectionnant les paramètres de recherche, puis en sélectionnant une langue dans la liste déroulante Langue.

    Java
    package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.QueryRequest;
import software.amazon.awssdk.services.kendra.model.QueryResponse;
import software.amazon.awssdk.services.kendra.model.QueryResultItem;

public class SearchIndexExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String query = "searchString";
        String indexId = "indexID";

        QueryRequest queryRequest = QueryRequest.builder()
            .queryText(query)
            .indexId(indexId)
            .attributeFilter(
                 AttributeFilter.builder()
                     .withEqualsTo(
                         DocumentAttribute.builder()
                             .withKey("_language_code")
                             .withValue("es")
                             .build())
                     .build())
            .build();

        QueryResponse queryResponse = kendra.query(queryRequest);

        System.out.println(String.format("\nSearch results|
                                          Resultados de la búsqueda: %s", query));
        for(QueryResultItem item: queryResponse.resultItems()) {
            System.out.println("----------------------");
            System.out.println(String.format("Type: %s", item.type()));

            switch(item.type()) {
                case QUESTION_ANSWER:
                case ANSWER:
                    String answerText = item.documentExcerpt().text();
                    System.out.println(answerText);
                    break;
                case DOCUMENT:
                    String documentTitle = item.documentTitle().text();
                    System.out.println(String.format("Title: %s", documentTitle));
                    String documentExcerpt = item.documentExcerpt().text();
                    System.out.println(String.format("Excerpt: %s", documentExcerpt));
                    break;
                default:
                    System.out.println(String.format("Unknown query result type: %s", item.type()));

            }

            System.out.println("-----------------------\n");
        }
    }
}
