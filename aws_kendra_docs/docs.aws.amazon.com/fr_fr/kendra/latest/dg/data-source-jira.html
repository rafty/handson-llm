Jira

Jira est un outil de gestion de projet pour le développement de logiciels, la gestion des produits et le suivi des bogues. Vous pouvez l'utiliserAmazon Kendra pour indexer vos projets, problèmes, commentaires, pièces jointes, journaux de travail et statuts Jira.

Amazon Kendrane prend actuellement en charge que Jira Cloud.

Vous pouvez vous connecterAmazon Kendra à votre source de données Jira à l'aide de la Amazon Kendraconsole ou de l'JiraConfiguration API. Pour obtenir la liste des fonctionnalités prises en charge par chacune de ces fonctionnalités, veuillez consulterFonctionnalités prises en charge.

Pour résoudre les problèmes liés à votre connecteur de source de donnéesAmazon Kendra Jira, consultezRésolution des problèmes de sources de données.

Fonctionnalités prises en charge

Amazon KendraLe connecteur de source de données Jira prend en charge les fonctionnalités suivantes :

  • Journal des modifications

  • Mappages de champs

  • Filtrage du contexte utilisateur

  • Filtres d'inclusion/d'exclusion

  • Virtual privatecCloud (VPC)

Prérequis

Avant de pouvoir l'utiliserAmazon Kendra pour indexer votre source de données Jira, apportez ces modifications à votre Jira et à vosAWS comptes.

Dans Jira, assurez-vous de disposer des éléments suivants :

  • Création d'identifiants d'authentification par jeton d'API Jira comprenant un identifiant Jira (nom d'utilisateur ou e-mail) et un identifiant Jira (jeton d'API Jira). Consultez la documentation Atlassian sur la gestion des jetons d'API.

  • Vous avez noté l'URL du compte Jira dans les paramètres de votre compte Jira. Par exemple, company.atlassian.net.

  • Coché, chaque document est unique dans Jira et dans les autres sources de données que vous prévoyez d'utiliser pour le même index. Chaque source de données que vous souhaitez utiliser pour un index ne doit pas contenir le même document pour toutes les sources de données. Les identifiants de document sont globaux par rapport à un index et doivent être uniques par index.

Sur votreAWS compte, assurez-vous de disposer des éléments suivants :

  • A créé unAmazon Kendra index et, en cas d'utilisation de l'API, a noté l'ID de l'index.

  • Vous avez créé unIAM rôle pour votre source de données et, si vous utilisez l'API, avez noté l'ARN duIAM rôle.

  • Vous avez stocké vos informations d'authentification Jira dans unAWSSecrets Manager secret et, si vous utilisez l'API, noté l'ARN du secret.

    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

Si vous n'avez pas deIAM rôle ou de secret existant, vous pouvez utiliser la console pour créer un nouveauIAM rôle et un nouveauSecrets Manager secret lorsque vous connectez votre source de données Jira àAmazon Kendra. Si vous utilisez l'API, vous devez fournir l'ARN d'unIAM rôle et d'unSecrets Manager secret existants, ainsi qu'un ID d'index.

Instructions de connexion

Pour vous connecterAmazon Kendra à votre source de données Jira, vous devez fournir les informations nécessaires sur votre source de données Jira afin que vousAmazon Kendra puissiez accéder à vos données. Si vous n'avez pas encore configuré Jira forAmazon Kendra seePrérequis.

Console

Pour vous connecterAmazon Kendra à Jira

  1. Connectez-vous à la consoleAWS de gestion et ouvrez la Amazon Kendraconsole.

  2. Dans le volet de navigation de gauche, choisissez Indexes, puis choisissez l'index que vous souhaitez utiliser dans la liste des index.

    Note

    Vous pouvez choisir de configurer ou de modifier vos paramètres de contrôle d'accès utilisateur sous Paramètres de l'index.

  3. Sur la page Mise en route, choisissez Ajouter une source de données.

  4. Sur la page Ajouter une source de données, choisissez Jira connector, puis Ajouter une source de données.

  5. Sur la page Spécifier les détails de source de données, entrez les informations suivantes :

    1. Dans Nom et description, pour Nom de la source de données, entrez le nom de votre source de données. Vous pouvez ajouter des traits d'union, mais pas d'espaces.

    2. Description (Facultatif) : entrez une description facultative pour votre source de données.

    3. Dans Langue, pour Langue par défaut : langue permettant de filtrer vos documents pour l'index. Sauf indication contraire, la langue par défaut est l'anglais. La langue spécifiée dans les métadonnées remplace la langue sélectionnée.

    4. Dans Tags, pour Ajouter un nouveau tag : Tags pour rechercher et filtrer vos ressources ou suivre vosAWS coûts.

    5. Choisissez Suivant.

  6. Sur la page Définir l'accès et la sécurité, entrez les informations suivantes :

    1. URL du compte Jira : saisissez l'URL de votre compte Jira.

    2. AWSSecrets Managersecret —Choisissez un secret existant ou créez-en un nouveauSecrets Manager pour stocker vos informations d'authentification Jira. Si vous choisissez de créer un nouveau secret, une fenêtreAWSSecrets Manager secrète s'ouvre.

      1. Entrez les informations suivantes dans la fenêtre Créer unAWSSecrets Manager secret :

        1. Nom secret : nom pour votre secret. Le préfixe «AmazonKendra -Jira » est automatiquement ajouté à votre nom secret.

        2. Pour Jira ID : entrez le nom d'utilisateur ou l'e-mail Jira.

        3. Pour le mot de passe/le jeton : entrez le jeton d'API Jira que vous avez créé à partir de votre compte Jira.

      2. Choisissez Save (Enregistrer).

    3. Virtual Private Cloud (VPC) Dans ce cas, vous devez ajouter des sous-réseaux et des groupes de sécurité VPC.

    4. Rôle IAM existant ou créez un nouveau rôle IAM existant pour accéder aux informations IAM existant ou créez un nouveau rôle IAM existant pour accéder aux informations IAM existant ou créez un nouveau rôle IAM existant.

      Note

      Les rôles IAM utilisés pour les index ne peuvent pas être utilisés pour les sources de données. Si vous ne savez pas si un rôle existant est utilisé pour un index ou une FAQ, choisissez Créer un nouveau rôle pour éviter les erreurs.

    5. Choisissez Suivant.

  7. Sur la page Configurer les paramètres de synchronisation, entrez les informations suivantes :

    1. Sélectionnez les projets Jira à indexer : entités Jira ou types de contenu que vous souhaitez analyser.

    2. Statuts, éléments supplémentaires et types de problèmes : sélectionnez du contenu pour affiner la portée de votre index.

    3. Journal des modifications : sélectionnez cette option pour mettre à jour votre index au lieu de synchroniser tous vos fichiers.

    4. Modèles Regex : modèles d'expression régulière permettant d'inclure ou d'exclure certains fichiers. Vous pouvez ajouter jusqu'à 100 motifs.

    5. Dans Synchronisation du calendrier d'exécution, pour Fréquence : choisissez la fréquence à laquelle Amazon Kendra se synchronisera avec votre source de données.

    6. Choisissez Suivant.

  8. Sur la page Définir les Mappages de champs, entrez les informations suivantes :

    1. Pour le projet, le problème, le commentaire, la pièce jointe, le journal de travail : sélectionnez parmi les champs de source de données par défautAmazon Kendra générés que vous souhaitez mapper à votre index.

    2. Ajouter un champ : pour ajouter des champs de source de données personnalisés afin de créer un nom de champ d'index auquel mapper et le type de données du champ.

    3. Choisissez Suivant.

  9. Sur la page Réviser et créer, vérifiez que les informations que vous avez saisies sont correctes, puis sélectionnez Ajouter une source de données. Vous pouvez également choisir de modifier vos informations à partir de cette page. Votre source de données apparaîtra sur la page Sources de données une fois qu'elle aura été ajoutée avec succès.

API

Pour vous connecterAmazon Kendra à Jira

Vous devez spécifier les éléments suivants à l'aide de l'JiraConfigurationAPI :

  • URL de la source de données : spécifiez l'URL de votre compte Jira. Par exemple, company.atlassian.net.

  • Secret Amazon Resource Name (ARN) : indiquez le nom de ressource Amazon (ARN) d'unSecrets Manager code secret contenant les informations d'authentification de votre compte Jira. Le secret est stocké dans une structure JSON avec les clés suivantes :

    {
    "jiraId": "Jira user name or email",
    "jiraCredential": "Jira API token"
}
    Note

    Veillez à actualiser ou à modifier régulièrement vos informations d'identification et votre code secret. Fournissez uniquement le niveau d'accès nécessaire à votre propre sécurité. La réutilisation des informations d'identification et des secrets entre les sources de données et les versions 1.0 et 2.0 du connecteur (le cas échéant) n'est pas recommandée.

  • Rôle IAM : spécifiez leRoleArn moment où vous appelezCreateDataSource pourIAM attribuer un rôle autorisé à accéder à votreSecrets Manager secret et à appeler les API publiques requises pour le connecteur Jira etAmazon Kendra. Pour de plus amples informations, veuillez consulter Rôles IAM pour les sources de données Jira.

Vous pouvez également ajouter les fonctionnalités facultatives suivantes :

  • Virtual Private Cloud (VPC) : spécifiez dans leVpcConfiguration cadre de la configuration de la source de données. Reportez-vous Amazon Kendraà la section Configuration pour utiliser un VPC.

  • Journal des modifications : s'ilAmazon Kendra convient d'utiliser le mécanisme de journal des modifications des sources de données Jira pour déterminer si un document doit être ajouté, mis à jour ou supprimé dans l'index.

    Note

    Utilisez le journal des modifications si vous neAmazon Kendra souhaitez pas numériser tous les documents. Si votre journal des modifications est volumineux, la numérisation des documents de la source de données Jira peut prendreAmazon Kendra moins de temps que le traitement du journal des modifications. Si vous synchronisez votre source de données Jira avec votre index pour la première fois, tous les documents sont numérisés.

  • Filtres d'inclusion et d'exclusion : vous pouvez spécifier si vous souhaitez inclure ou exclure certains projets, problèmes, commentaires, pièces jointes, journaux de travail et statuts.

    Note

    La plupart des sources de données utilisent des modèles d'expression régulière, qui sont des modèles d'inclusion ou d'exclusion appelés filtres. Si vous spécifiez un filtre d'inclusion, seul le contenu correspondant au filtre d'inclusion est indexé. Tout document qui ne correspond pas au filtre d'inclusion n'est pas indexé. Si vous spécifiez un filtre d'inclusion et d'exclusion, les documents qui correspondent au filtre d'exclusion ne sont pas indexés, même s'ils correspondent au filtre d'inclusion.

  • Mappages de champs : choisissez de mapper les champs de votre source de données Jira à vos champs d'Amazon Kendraindex. Pour plus d'informations, veuillez consulter la rubrique Mappage des champs de source de données.

  • Filtrage contextuel utilisateurAmazon Kendra : analyse la liste de contrôle d'accès (ACL) de votre source de données par défaut. Les informations ACL sont utilisées pour filtrer les résultats de recherche en fonction de l'accès de l'utilisateur ou de son groupe aux documents. Pour plus d'informations, consultez Filtrage du contexte utilisateur pour les sources de données Jira.
