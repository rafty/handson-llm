ServiceNow

ServiceNow est un système de gestion des services basé sur le cloud qui permet la création et la gestion des flux de travail au niveau de l'organisation, notamment les services informatiques, les systèmes de billetterie et l'assistance. Vous pouvez l'utiliserAmazon Kendra pour indexer vos ServiceNow catalogues, articles de connaissances, incidents et leurs pièces jointes.

Vous pouvez vous connecterAmazon Kendra à votre source de ServiceNow données à l'aide de la Amazon Kendraconsole, de l'TemplateConfigurationAPI ou de l'ServiceNowConfigurationAPI.

Amazon Kendrapossède deux versions du ServiceNow connecteur. Les fonctionnalités prises en charge par chaque version incluent :

ServiceNow connecteur V1.0/ServiceNowConfigurationAPI

  • Mappages de champs

  • ServiceNow versions d'instance : Londres, autres

  • Modèles d'inclusion/exclusion : catalogues de services, articles de connaissances, pièces jointes

ServiceNow connecteur V2.0/ TemplateConfigurationAPI

  • Mappages de champs

  • Filtrage du contexte utilisateur

  • Virtual privatecCloud (VPC)

  • Synchroniser tous les documents/Synchroniser uniquement les documents nouveaux, modifiés et supprimés

  • ServiceNow versions d'instance : Rome, Sandiego, Tokyo, autres

  • Modèles d'inclusion/exclusion : catalogues de services, articles de connaissances, incidents, pièces jointes

Note

La prise en Support ServiceNow du connecteur ServiceNowConfiguration V1.0/API devrait prendre fin en juin 2023. Nous vous recommandons d'utiliser le ServiceNow connecteur V2.0/ TemplateConfiguration API.

Pour résoudre les problèmes liés à votre connecteur de source deAmazon Kendra ServiceNow données, consultezRésolution des problèmes de sources de données.
