Gestione di Regioni AWS

Un Regione AWSè una raccolta diAWS risorse in un'area geografica. Ogni Regione AWS è isolata e indipendente dalle altre regioni. Le regioni forniscono la tolleranza ai guasti, la stabilità e la resilienza e possono anche ridurre la latenza. Puoi fornire risorse in più regioni in modo da mantenere le applicazioni disponibili e non sono influenzate da un'interruzione a livello regionale.

Le risorse create in una regione non esistono in qualsiasi altra regione, a meno che non si utilizzi in modo esplicito una funzionalità di replica offerta da un servizio AWS. Ad esempio, Amazon S3 e Amazon EC2 supportano la replica tra regioni. Alcuni servizi, ad esempio AWS Identity and Access Management (IAM), non dispongono di risorse regionali.

È possibile utilizzare le condizioni della politica per controllare l'accessoServizi AWS a unRegione AWS.

Il tuo account determina le regioni per te disponibili.

  • UnAccount AWS fornisce più regioni in modo da poter avviareAWS risorse in ubicazioni che soddisfino i tuoi requisiti. Ad esempio, potresti avviare istanze in Europa per esser più vicino ai clienti europei o per soddisfare i requisiti giuridici.

  • Un accountAWSGovCloud (Stati Uniti occidentali) fornisce l'accesso alla regioneAWSGovCloud (Stati Uniti occidentali) eAWSGovCloud (Stati Uniti orientali). Per ulteriori informazioni, consulta AWS GovCloud (US).

  • Un account Amazon AWS (Cina) fornisce l'accesso solo alle Regioni Pechino e Ningxia. Per ulteriori informazioni, consulta Amazon Web Services in Cina.

Quando si abilita una regione, AWS esegue delle operazioni per preparare l'account in quella regione, ad esempio distribuendo le risorse IAM nella regione. Questo processo richiede pochi minuti per la maggior parte degli account, ma potrebbero essere necessarie anche diverse ore. Non puoi usare una regione finché questo processo non è completo.

Requisiti

Per abilitare una regione, è necessario disporre le autorizzazioni appropriate. Per un esempio di policy IAM, consultare Abilitazione e disabilitazioneRegioni AWS nella Guida per l'utente di IAM.

Per abilitare una regione utilizzando la console di regioni utilizzando la console.

  1. Accedere alla AWS Management Console.

  2. Nell'angolo in alto a destra della console, scegli la regione corrente per espandere l'elenco delle regioni, scorri fino alla fine dell'elenco e quindi scegli Gestisci regioni.

  3. Seleziona la colonna Azione per la regione che desideri utilizzare.

    • Se la colonna Stato contiene Abilitato per impostazione predefinita, non sono necessarie ulteriori azioni. La regione è stata abilitata automaticamente per l'account.

    • Se la colonna Stato contiene Disabilitato, scegli Abilita. Descrizione regioni e seleziona Abilita regione. È necessario attendere che la regione sia pronta per l'uso.

Per abilitare una regione utilizzando ilAWS CLI

Usa il seguente comando enable-region.

aws account enable-region --region-name af-south-1

Disabilita una regione

Se hai abilitato una regione che non è abilitata automaticamente, puoi disabilitare la regione se necessario. Dopo aver disabilitato una regione, le risorse in essa diventano non disponibili, in base all'eventuale coerenza. Tuttavia, le risorse non vengono eliminate. Puoi accedervi se abiliti nuovamente questa regione.

Requisiti

  • Non è possibile disabilitare una regione abilitata per impostazione predefinita. (Vedi la tabella nella sezione precedente.)

  • Per disabilitare una regione, è necessario disporre della relativa autorizzazione. Per un esempio di policy IAM, consultare Abilitazione e disabilitazione di regioni AWS nella Guida per l'utente di IAM.

  • Prima di disabilitarla, è consigliabile rimuovere tutte le risorse da tale regione. Dopo aver disabilitato una regione, non potrai più visualizzare o gestire le risorse nella regione da quell'account tramite la Console diAWS gestione o leAWS API utilizzando i principali IAM. Tuttavia, le risorse in quella regione possono ancora essere soggette a costi. Per ulteriori informazioni, consulta Abilitazione e disabilitazione di regioni nella Guida per l'utente AWS Billing and Cost Management.

Per disabilitare una regione utilizzando la console.

  1. Accedere alla AWS Management Console.

  2. Nell'angolo in alto a destra della console, scegli la regione corrente per espandere l'elenco delle regioni, scorri fino alla fine dell'elenco e quindi scegli Gestisci regioni.

  3. Nella colonna Azione per la regione, scegli Disabilita.

  4. Nella finestra di dialogo, rivedi il testo informativo e seleziona Disabilita regione.

Per disabilitare una regione utilizzando ilAWS CLI

Usa il seguente comando disable-region.

aws account disable-region --region-name af-south-1

Descrizione regioni regioni regioni regioni regioni regioni regioni regioni utilizzando ilAWS CLI

Usa il comando list-regions per descrivere tutte le regioni disponibili per il tuo account, incluso il loro stato.

aws account list-regions --max-results 50

Utilizzate il get-region-opt-statuscomando per descrivere lo stato di una regione specifica.

aws account get-region-opt-status --region-name af-south-1

Per le regioni abilitate automaticamente per il tuo account, l'output di questi comandi include quanto segue:

"RegionOptStatus": "ENABLED_BY_DEFAULT"

Per le regioni che è necessario abilitare manualmente, l'output include quanto segue:

"RegionOptStatus": "DISABLED"

Dopo aver abilitato una regione, l'output include quanto segue:

"RegionOptStatus": "ENABLED"
