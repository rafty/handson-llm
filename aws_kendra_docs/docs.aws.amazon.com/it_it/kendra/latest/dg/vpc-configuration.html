ConfigurazioneAmazon Kendra per l'utilizzo di un VPC

Amazon Kendrapuò connettersi al tuoAmazon Virtual Private Cloud per indicizzare i contenuti archiviati in fonti di dati o database in esecuzione nel tuo cloud privato. Quando si crea un connettore di origine dati o un connettore di database, si forniscono identificatori di sottorete e gruppo di sicurezza per la sottorete che contiene l'origine dati o il database. Amazon Kendrautilizza queste informazioni per creare un'elastic network interface che utilizza per comunicare in modo sicuro con la fonte di dati o il database.

Se la tua fonte di dati o il tuo database non è in esecuzione su unAmazon VPC, puoi connettere l'origine dati o il database al tuoAmazon VPC utilizzando una rete privata virtuale (VPN). Quando crei il tuo account Amazon, ottieni un VPC predefinito. Per informazioni sulla configurazione di una VPN, consulta la AWS VPNDocumentazione.

Per utilizzare un VPC, è necessario indicareAmazon Kendra l'identificatore della sottorete a cui appartiene il database e gli identificatori di eventuali gruppi di sicurezza cheAmazon Kendra devono utilizzare per accedere alla sottorete. Ad esempio, se si utilizza la porta predefinita per un database MySQL, i gruppi di sicurezza devono consentire l'accessoAmazon Kendra alla porta 3306 sull'host che esegue il database.

Utilizza solo sottoreti private nella configurazione VPC della tua fonte di dati o del tuo database. Se l'istanza RDS si trova in una sottorete pubblica del VPC, non è possibile utilizzare quella sottorete direttamente per sincronizzare l'origine dei dati o il database. Creane invece una sottorete privata con accesso in uscita a un gateway NAT nella sottorete pubblica. Quando configuri il cloud privato VPC per l'origine dei dati o il database, specifica quella sottorete privata. Per un'origine dati del database configurata con un VPC, le sottoreti devono trovarsi in uno dei seguenti ID di zona di disponibilità:

  • Stati Uniti occidentali (Oregon) — usw2-az1, usw2-az2, usw2-az3

  • Stati Uniti orientali (Virginia settentrionale): use1-az1, use1-az2, use1-az4

  • UE (Irlanda) — euw1-az1, uew1-az2, euw1-az2, euw1-az3

Gli identificatori per le sottoreti e i gruppi di sicurezza sono configurati nel pannelloAmazon VPC di controllo. Per visualizzare gli identificatori, apri laAmazon VPC console come segue:

Per visualizzare gli identificatori di sottorete

  1. Accedere ad AWS Management Console e aprire la console Amazon VPC all'indirizzo https://console.aws.amazon.com/vpc/.

  2. Nel riquadro di navigazione scegliere Sottoreti.

  3. Dall'elenco delle sottoreti, scegli la sottorete che contiene il server del database.

  4. Nella scheda descrizione, annota l'identificatore nel campo Subnet ID.

Per visualizzare gli identificatori dei gruppi di sicurezza

  1. Accedere ad AWS Management Console e aprire la console Amazon VPC all'indirizzo https://console.aws.amazon.com/vpc/.

  2. Scegli Security Groups (Gruppi di sicurezza) dal pannello di navigazione.

  3. Dall'elenco dei gruppi di sicurezza, scegliere il gruppo per il quale si desidera l'identificatore.

  4. Nella scheda descrizione, annota l'identificatore nel campo ID del gruppo.

SeAmazon Kendra è necessario indirizzare la connessione tra due o più sottoreti, è possibile fornire più sottoreti. Ad esempio, se la sottorete che contiene il server del database non dispone di indirizzi IP, èAmazon Kendra possibile connettersi a una sottorete con indirizzi IP liberi e indirizzare la connessione alla prima sottorete. Se si elencano più sottoreti, queste caratteristiche devono essere in grado di comunicare tra loro. Ogni sottorete deve essere associata a una tabella di routing che fornisce l'accesso a Internet in uscita mediante un dispositivo NAT (Network Address Translator).

Puoi anche fornire più gruppi di sicurezza. L'effetto combinato dei gruppi di sicurezza deve consentire l'accessoAmazon Kendra all'origine dati o al server di database specificato nella configurazione della connessione.

Connessione a un database in VPC

L'esempio seguente mostra come connettere un database MySQL in esecuzione in un VPC. L'esempio presuppone che tu stia iniziando con il tuo VPC predefinito e che sia necessario creare un database MySQL. Se disponi già di un cloud privato virtuale, assicurati che sia configurato come mostrato. Se hai un database MySQL, puoi usarlo invece di crearne uno nuovo.

Fase 1: Configurazione di un VPC

Configura il tuo VPC in modo da avere una sottorete privata e un gruppo di sicurezza per accedereAmazon Kendra a un database MySQL in esecuzione nella sottorete. Le sottoreti fornite nella configurazione VPC devono essere nelle regioni Stati Uniti occidentali (Oregon), Stati Uniti occidentali (Oregon), Stati Uniti orientali (Virginia settentrionale), Stati Uniti occidentali (Oregon), Stati Uniti occidentali (Oregon), Stati Uniti occidentali (Oregon), Stati Uniti

Per configurare un VPC

  1. Accedere ad AWS Management Console e aprire la console Amazon VPC all'indirizzo https://console.aws.amazon.com/vpc/.

  2. Dal pannello di navigazione, scegli Route tables, quindi scegli Crea tabella degli itinerari.

  3. Nel campo del tag Nome, inserisciPrivate subnet route table. Nel campo VPC, scegli il tuo VPC, quindi scegli Crea. Scegli Chiudi per tornare all'elenco delle tabelle dei percorsi.

  4. Dal pannello di navigazione, scegli NAT Gateways, quindi scegli Create NAT Gateway.

  5. Nel campo Sottorete, scegli la sottorete pubblica e annota l'ID della sottorete.

  6. Se non disponi di un indirizzo IP elastico, scegli Crea nuovo EIP, scegli Crea un gateway NAT e quindi scegli Chiudi.

  7. Dal pannello di navigazione scegliere Route Tables.

  8. Dall'elenco della tabella delle rotte, scegli la tabella delle rotte della sottorete privata creata nel passaggio 3. In Azioni, scegli Modifica percorsi.

  9. Selezionare Add route (Aggiungi route). Aggiungi la destinazione 0.0.0.0.0/0 per consentire tutto il traffico in uscita a Internet. Per Target, scegli NAT Gateway, quindi scegli il gateway che hai creato nel passaggio 4. Scegliere Save routes (Salva route), quindi selezionare Close (Chiudi).

  10. In Azioni, scegli Modifica associazioni di sottorete.

  11. Scegliere le sottoreti da rendere private. Non scegliete la sottorete con il gateway NAT che avete notato in precedenza.

Fase 2: Configurazione della sicurezza

Quindi, configura i gruppi di sicurezza per il database.

Per creare i gruppi di sicurezza

  1. Accedere ad AWS Management Console e aprire la console Amazon VPC all'indirizzo https://console.aws.amazon.com/vpc/.

  2. Nella descrizione del tuo VPC, prendi nota del CIDR IPv4.

  3. Nel riquadro di navigazione scegliere Gruppi di sicurezza, quindi Crea gruppo di sicurezza.

  4. Nel nome del gruppo di sicurezza inserisciDataSourceInboundSecurityGroup. Fornire una descrizione, quindi selezionare il VPC dall'elenco. Scegli Crea, quindi scegli Chiudi.

  5. Seleziona la scheda Inbound (In entrata).

  6. Scegli Modifica regole, quindi scegli Aggiungi regola

  7. Per un database, immettere il numero di porta per l'intervallo di porte. Ad esempio, per MySQL è e3306, per HTTPS, è443. Per l'origine, digitare il instradamento interdominio senza classi (CIDR) del VPC. Scegli Salva regole, quindi scegli Chiudi.

Il gruppo di sicurezza consente a chiunque all'interno del VPC di connettersi al database e consente le connessioni in uscita a Internet.

Fase 3: Creazione di un database

Crea un database per contenere i tuoi documenti oppure puoi usare il database esistente. Per un elenco dei databaseAmazon Kendra supportati, vedere Utilizzo di un'origine dati di database.

Per istruzioni su come creare un database MySQL, vedere Getting Starting with a MySQL Data Source with a MySQL Data Source (console)Amazon RDS.

Fase 4: Creazione di un connettore origine dati del database

Dopo aver configurato il VPC e creato il database, puoi creare un connettore di origine dati per il database. Vedere Utilizzo di una fonte di dati del database.

Assicurati di configurare il tuo VPC, le sottoreti private che hai creato nel tuo VPC e il gruppo di sicurezza che hai creato nel tuo VPC per il tuo database.

Per istruzioni su come creare un'origine dati per un database MySQL, vedere Guida introduttiva all'uso di un'origine dati del database MySQL (console)Amazon RDS.
