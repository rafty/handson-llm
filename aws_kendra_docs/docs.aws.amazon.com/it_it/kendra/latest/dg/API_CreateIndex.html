CreateIndex

Crea un indice Amazon Kendra. La creazione di indici è un'API asincrona. Per determinare se la creazione dell'indice è stata completata, controlla ilStatus campo restituito da una chiamata aDescribeIndex. IlStatus campo è impostato suACTIVE quando l'indice è pronto per l'uso.

Una volta attivato l'indice, è possibile indicizzare i documenti utilizzando l'BatchPutDocumentAPI o utilizzando una delle origini dati supportate.

Per un esempio di creazione di un indice e di una fonte di dati utilizzando Python SDK, consulta Guida introduttiva a Python SDK. Per un esempio di creazione di un indice e di un'origine dati utilizzando Java SDK, vedere Guida introduttiva a Java SDK.

Parametri della richiesta

Per informazioni sui parametri comuni a tutte le azioni, vedere Parametri comuni.

La richiesta accetta i seguenti dati in formato JSON.

ClientToken

Un token fornito per identificare la richiesta di creazione di un indice. Più chiamate all'CreateIndexAPI con lo stesso token client creeranno un solo indice.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 1. Lunghezza massima di 100.

Required: No

Description

Una descrizione per l’indice.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 0. Lunghezza massima di 1000.

Modello: ^\P{C}*$

Required: No

Edition

L'edizione Amazon Kendra da utilizzare per l'indice. Scegli tra indiciDEVELOPER_EDITION destinati allo sviluppo, al test o al proof of concept. UsoENTERPRISE_EDITION per la produzione. Una volta impostata l'edizione per un indice, non è possibile modificarla.

Il parametro Edition è facoltativo. Se non specifichi un valore, l'impostazione predefinita èENTERPRISE_EDITION.

Per ulteriori informazioni sui limiti di quota per le edizioni Enterprise e Developer, vedi Quote.

Tipo: String

Valori validi: DEVELOPER_EDITION | ENTERPRISE_EDITION

Required: No

Name

Un nome dell'indice.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 1. Lunghezza massima di 1000.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Campo obbligatorio: sì

RoleArn

Il nome della risorsa Amazon (ARN) di un ruolo IAM con l'autorizzazione per accedere ai CloudWatch log e alle metriche Amazon. Per ulteriori informazioni, consulta la sezione la sezione Ruoli di accesso IAM per Amazon Kendra.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 0. La lunghezza massima è 1284 caratteri.

Pattern: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Campo obbligatorio: sì

ServerSideEncryptionConfiguration

L'identificatore della chiave gestita dalAWS KMS cliente (CMK) utilizzata per crittografare i dati indicizzati da Amazon Kendra. Amazon Kendra non supporta chiavi CMK asimmetriche.

Tipo: ServerSideEncryptionConfiguration oggetto

Required: No

Tags

Un elenco delle coppie chiave-valore che identificano o classificano l'indice. È possibile utilizzare i tag anche per controllare l'accesso all'indice. Le chiavi e i valori di tag possono essere composti da lettere Unicode, numeri, spazi e simboli: _.: + - @.

Tipo: matrice di oggetti Tag

Membri dell'array: numero minimo di 0 elementi. Numero di numero di di 200 elementi.

Required: No

UserContextPolicy

La policy del contesto utente.

ATTRIBUTE_FILTER

Tutti i contenuti indicizzati sono ricercabili e visualizzabili per tutti gli utenti. Per filtrare i risultati della ricerca in base al contesto dell'utente, puoi utilizzare i filtri degli attributi _user_id e _group_ids o fornire informazioni su utenti e gruppi in UserContext.

USER_TOKEN

Abilita il controllo accessi utente basato su token per filtrare i risultati della ricerca in base al contesto dell’utente. Tutti i documenti senza controllo degli accessi e tutti i documenti accessibili all'utente saranno ricercabili e visualizzabili.

Tipo: String

Valori validi: ATTRIBUTE_FILTER | USER_TOKEN

Required: No

UserGroupResolutionConfiguration

Ottiene utenti e gruppi dalla fonteAWS IAM Identity Center (successor to AWS Single Sign-On) dell'identità. Per configurarlo, consulta UserGroupResolutionConfiguration.

Tipo: UserGroupResolutionConfiguration oggetto

Required: No

UserTokenConfigurations

La configurazione del token utente.

Tipo: matrice di oggetti UserTokenConfiguration

Membri dell'array: numero massimo di 1 elementi.

Required: No

Elementi di risposta

Se l'operazione riesce, il servizio restituisce una risposta HTTP 200.

I dati seguenti vengono restituiti in formato JSON mediante il servizio.

Id

L'identificatore dell'indice. Usa questo identificatore quando esegui una query su un indice, imposti un'origine dati o indicizzi un documento.

Tipo: String

Limitazioni di lunghezza: lunghezza fissa di 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Errori

Per informazioni sugli errori comuni a tutte le operazioni, consultare Errori comuni.

AccessDeniedException

Non disponi delle autorizzazioni di accesso per eseguire questa azione. Assicurati di disporre delle politiche di autorizzazione e degli account utente richiesti e riprova.

Codice di stato HTTP: 400

ConflictException

Si è verificato un conflitto con la richiesta. Correggi eventuali incongruenze con le tue risorse e riprova.

Codice di stato HTTP: 400

InternalServerException

Si è verificato un problema con il server interno utilizzato per il servizio Amazon Kendra. Attendi qualche minuto e riprova oppure contatta l'Support per ricevere assistenza.

Codice di stato HTTP: 500

ResourceAlreadyExistException

La risorsa che desideri utilizzare esiste già. Verifica di aver fornito la risorsa corretta e riprova.

Codice di stato HTTP: 400

ServiceQuotaExceededException

Hai superato i limiti stabiliti per il tuo servizio Amazon Kendra. Consulta la sezione Quote per ulteriori informazioni o contatta l'Support per richiedere informazioni su un aumento dei limiti.

Codice di stato HTTP: 400

ThrottlingException

La richiesta è stata negata a causa del throttling della richiesta. Ridurre il numero di richieste e riprovare.

Codice di stato HTTP: 400

ValidationException

L'input non riesce a soddisfare i vincoli stabiliti dal servizio Amazon Kendra. Fornire i dati corretti e riprovare.

Codice di stato HTTP: 400
