Amazon S3

Amazon S3è un servizio di storage di oggetti che consente di archiviare dati come oggetti nei bucket. Puoi usarloAmazon Kendra per indicizzareAmazon S3 il tuo archivio di documenti.

avvertimento

Amazon Kendranon utilizza una politica dei bucket che concede a unAmazon Kendra committente le autorizzazioni per interagire con un bucket S3. Utilizza invece iIAM ruoli. Assicurati cheAmazon Kendra non sia incluso come membro di fiducia nella tua policy per evitare problemi di sicurezza dei dati nel concedere accidentalmente autorizzazioni a committenti arbitrari. Tuttavia, puoi aggiungere una policy per utilizzare unAmazon S3 bucket su diversi account. Per ulteriori informazioni, consulta Policy da utilizzareAmazon S3 tra gli account (nella schedaIAM Ruoli S3, sotto IAMi ruoli per le fonti di dati). Per informazioni suiIAM ruoli per le fonti di dati S3, consulta IAMruoli.

Per connettersiAmazon S3, è necessario specificare la connessione e altre informazioni nella console, utilizzando S3DataSourceConfiguration o utilizzando il TemplateConfiguration. Se usi l' TemplateConfiguration oggetto, puoi usare un VPC per connetterti alla tua fonte di dati. Si specifica VpcConfiguration quando si chiama CreateDataSource. Per ulteriori informazioni, consulta la sezione Configurazione di un VPC.

Nota

S3DataSourceConfigurationL'API non supporta la configurazione di un VPC. SoloTemplateConfiguration l'API e la console supportano la configurazione VPC.

Per la risoluzione dei problemi relativi al connettore della sorgente datiAmazon Kendra S3, consultaRisoluzione dei problemi delle origini dati.

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la tua fonte di dati S3, apporta queste modifiche nel tuo S3 e negliAWS account.

In S3, assicurati di disporre di:

  • Ho copiato il nome delAmazon S3 bucket.

    Nota

    Il bucket deve trovarsi nella stessa regioneAmazon Kendra dell'indice e l'indice deve avere l'autorizzazione per accedere al bucket che contiene i documenti.

  • È stato verificato che ogni documento sia unico in S3 e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

Se non disponi di unIAM ruolo esistente, puoi utilizzare la console per creare un nuovoIAM ruolo quando connetti la tua sorgente dati S3 aAmazon Kendra. Se si utilizza l'API, è necessario specificare l'ARN di unIAM ruolo esistente e un ID di indice.

Istruzioni di connessione

Per connettertiAmazon Kendra alla fonte dati S3, devi fornire i dettagli necessari della fonte dati S3 in modo cheAmazon Kendra possa accedere ai tuoi dati. Se non hai ancora configurato S3,Amazon Kendra guardaPrerequisiti.

Console

Per connettersiAmazon Kendra aAmazon S3

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli Connettore S3, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica i dettagli dell'origine dati, inserisci le seguenti informazioni:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. Puoi includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. In Tag, per Aggiungi nuovo tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le seguenti informazioni opzionali:

    1. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    2. Virtual Private Cloud (VPC): puoi scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    3. Seleziona Successivo.

  7. Nella pagina Configura le impostazioni di sincronizzazione, inserisci le seguenti informazioni:

    1. Nell'ambito della sincronizzazione, per Posizione dell'origine dei dati: il percorso delAmazon S3 bucket in cui sono archiviati i dati. Seleziona Browse S3 per scegliere il tuo bucket.

    2. (Facoltativo) I file di metadati prefissano la posizione della cartella: il percorso della cartella in cui sono archiviati i metadati. Seleziona Browse S3 per individuare la cartella dei metadati.

    3. (Facoltativo) Posizione del file di configurazione dell'elenco di controllo degli accessi: il percorso verso la posizione di un file contenente una struttura JSON che specifica le impostazioni di accesso per i file archiviati nell'origine dati S3. Seleziona Browse S3 per individuare il tuo file ACL.

    4. (Facoltativo) Seleziona la chiave di decrittografia: seleziona questa opzione per utilizzare una chiave di decrittografia. Puoi scegliere di utilizzare unaAWS KMS chiave esistente o crearne una nuova.

    5. (Facoltativo) In Configurazione aggiuntiva, per i modelli Regex: aggiungi modelli di espressioni regolari per includere o escludere documenti dall'indice. Tutti i percorsi sono relativi al bucket S3 della posizione dell'origine dei dati. Puoi aggiungere fino a 100 pattern.

    6. In modalità sincronizzazione, scegli tra la modalità di sincronizzazione completa e la sincronizzazione dei contenuti nuovi, modificati o eliminati per determinare in che modo l'indice si aggiorna quando il contenuto dell'origine dei dati cambia. Quando sincronizzi la tua fonte di dati con Amazon Kendra per la prima volta, tutti i contenuti vengono sincronizzati per impostazione predefinita.

    7. In Sincronizza programma di esecuzione, per Frequenza: scegli la frequenzaAmazon Kendra con cui sincronizzarti con la tua fonte di dati.

    8. Seleziona Successivo.

  8. Nella pagina Imposta mappature dei campi, inserisci le seguenti informazioni opzionali:

    1. Mappatura dei campi S3: seleziona dai campi di origine dati predefinitiAmazon Kendra generati che desideri mappare nell'indice.

    2. Aggiungi campo: scegli di aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. Puoi anche scegliere di modificare le tue informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

TemplateConfiguration API

Per connettersiAmazon Kendra aAmazon S3

È necessario specificare un JSON dello schema di origine dati utilizzando l'TemplateConfigurationAPI. È necessario fornire le informazioni riportate di seguito:

  • BucketName—Il nome del bucket che contiene i documenti.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore S3 eAmazon Kendra. Per ulteriori informazioni, consulta Ruoli IAM per le sorgenti dati S3.

Puoi inoltre aggiungere le seguenti funzionalità facoltative:

  • Filtri di inclusione ed esclusione: specifica se includere o escludere determinati nomi di file, tipi di file, percorsi di file e utilizzare modelli globulari (modelli che possono espandere un pattern jolly in un elenco di nomi di percorso che corrispondono al modello specificato). Alcuni esempi di modelli a globo includono:

    • /myapp/config/*—Tutti i file all'interno della directory di configurazione

    • /**/*.png—Tutti i file.png in tutte le directory

    • /**/*.{png,ico,md}—Tutti i file.png, .ico o .md in tutte le directory

    • /myapp/src/**/*.ts—Tutti i file.ts all'interno della directory src (e tutte le sue sottodirectory)

    • **/!(*.module).ts—Tutti i file.ts ma non .module.ts

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine dati S3 ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Virtual Private Cloud (VPC): specificareVpcConfiguration quando si esegue la chiamataCreateDataSource. Consultare ConfigurazioneAmazon Kendra per l'utilizzo di un VPC.

  • Configurazione dei metadati del documento: aggiungere file di metadati del documento che contengono informazioni quali le informazioni sul controllo dell'accesso al documento, l'URI di origine, l'autore del documento e gli attributi personalizzati. Ogni file di metadati contiene metadati relativi a un singolo documento.

Per un elenco di altre chiavi JSON importanti da configurare, consultaAmazon S3schema modello.
