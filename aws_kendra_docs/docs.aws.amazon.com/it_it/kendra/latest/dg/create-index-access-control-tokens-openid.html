Usare OpenID

Per configurare unAmazon Kendra indice per utilizzare un token OpenID per il controllo degli accessi, è necessario l'URL JWKS (JSON Web Key Set) del provider OpenID. Nella maggior parte dei casi l'URL JWKS è nel seguente formato (se stanno seguendo la scoperta di OpenID)https://domain-name/.well_known/jwks.json.

Gli esempi seguenti mostrano come utilizzare un token OpenID per il controllo degli accessi degli utenti quando si crea un indice.

Console

  1. Scegli Crea indice per iniziare a creare un nuovo indice.

  2. Nella pagina Specifica i dettagli dell'indice, dai un nome e una descrizione all'indice.

  3. Per IAMil ruolo, seleziona un ruolo o seleziona Crea un nuovo ruolo e specifica un nome per creare un nuovo ruolo. Il ruolo IAM avrà il prefisso "AmazonKendra-».

  4. Lascia tutti gli altri campi ai valori predefiniti. Seleziona Successivo.

  5. Nella pagina Configura il controllo degli accessi degli utenti, in Impostazioni del controllo degli accessi, scegli Sì per utilizzare i token per il controllo degli accessi.

  6. In Configurazione del token, seleziona OpenID come tipo di token.

  7. Specifica l'URL della chiave di firma. L'URL deve indicare un set di chiavi web JSON.

  8. Opzionale in configurazione avanzata:

    1. Specificare un nome utente da utilizzare nel controllo ACL.

    2. Specifica uno o più gruppi da utilizzare nel controllo ACL.

    3. Specifica l'emittente che convaliderà l'emittente del token.

    4. Specifica l'ID del cliente. È necessario specificare un'espressione regolare che corrisponda al pubblico del JWT.

  9. Nella pagina dei dettagli del provisioning, scegli l'edizione per sviluppatori.

  10. Scegli Crea per creare il tuo indice.

  11. Attendi la creazione dell'indice. Amazon Kendrafornisce l'hardware per il tuo indice. Questa operazione può richiedere del tempo.

CLI

Per creare un indiceAWS CLI utilizzando un file di input JSON, crea prima un file JSON con i parametri desiderati:

{
    "Name": "user-context",
    "Edition": "ENTERPRISE_EDITION",
    "RoleArn": "arn:aws:iam::account-id:role:/my-role",
    "UserTokenConfigurations": [
        {
            "JwtTokenTypeConfiguration": {
                "KeyLocation": "URL",
                "Issuer": "optional: specify the issuer url",
                "ClaimRegex": "optional: regex to validate claims in the token",
                "UserNameAttributeField": "optional: user",
                "GroupAttributeField": "optional: group",
                "URL": "https://example.com/.well-known/jwks.json"
            }
        }
    ],
    "UserContextPolicy": "USER_TOKEN"
}

È possibile sovrascrivere i nomi dei campi utente e di gruppo predefiniti. Il valore predefinito perUserNameAttributeField è «utente». Il valore predefinito perGroupAttributeField è «gruppi».

Quindi, chiamacreate-index utilizzando il file di input. Ad esempio, se il nome del file JSON ècreate-index-openid.json, puoi usare quanto segue: 

aws kendra create-index --cli-input-json file://create-index-openid.json
Python
response = kendra.create_index(
    Name='user-context',
    Edition='ENTERPRISE_EDITION',
    RoleArn='arn:aws:iam::account-id:role:/my-role',
    UserTokenConfigurations=[
        {
            "JwtTokenTypeConfiguration": {
                "KeyLocation": "URL",
                "Issuer": "optional: specify the issuer url",
                "ClaimRegex": "optional: regex to validate claims in the token",
                "UserNameAttributeField": "optional: user",
                "GroupAttributeField": "optional: group",
                "URL": "https://example.com/.well-known/jwks.json"
            }
        }
    ],
    UserContextPolicy='USER_TOKEN'
)
