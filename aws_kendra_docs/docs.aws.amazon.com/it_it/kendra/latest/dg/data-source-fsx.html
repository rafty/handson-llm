Amazon FSx

Amazon FSx è un sistema di file server basato su cloud completamente gestito che offre funzionalità di storage condiviso. Se sei un utente Amazon FSx, puoi utilizzarloAmazon Kendra per indicizzare la tua fonte di dati Amazon FSx.

Amazon Kendraattualmente supporta soloAmazon FSx Windows File Server.

Puoi connettertiAmazon Kendra alla tua fonte di dati Amazon FSx utilizzando la Amazon Kendraconsole e l'FsxConfigurationAPI.

Per la risoluzione dei problemi relativi al connettore di origine datiAmazon Kendra Amazon FSx, consultaRisoluzione dei problemi delle origini dati.

Funzionalità supportate

Amazon KendraIl connettore di origine dati Amazon FSx supporta le seguenti funzionalità:

  • Log delle modifiche

  • Mappature di campi

  • Filtraggio del contesto utente

  • Filtri di inclusione/esclusione

  • Virtual Private Cloud (VPC) (Cloud privato virtuale (VPC)

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la fonte di dati Amazon FSx, apporta queste modifiche nel tuoAWS account e nel tuo account Amazon FSx.

In Amazon FSx, assicurati di disporre di:

  • Un account Amazon FSx con autorizzazioni di lettura e montaggio.

  • Ho preso nota delle credenziali diAmazon FSx autenticazione per un account utente di Active Directory. Questo include il nome utente Active Directory e il nome di dominio Domain Name System (DNS). Ad esempio, user@corp.example.com.

  • È stato copiato l'IDAmazon FSx file system.

  • Hai usato unAmazon VPC (AWSVPC) nel luogo in cuiAmazon FSx risiedi.

  • Verifica che ogni documento sia unico in Amazon FSx e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le tue credenziali di autenticazione Amazon FSx in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti la tua fonte di dati Amazon FSx aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

Istruzioni di connessione

Per connettertiAmazon Kendra alla tua fonte dati Amazon FSx, devi fornire i dettagli necessari della tua origine dati Amazon FSx in modo cheAmazon Kendra possa accedere ai tuoi dati. Se non hai ancora configurato Amazon FSx,Amazon Kendra consultaPrerequisiti.

Console

Per connettersiAmazon Kendra ad Amazon FSx

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli il connettore Amazon FSx, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica dei dettagli dell'origine dati, inserisci le seguenti informazioni:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. È possibile includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. In Tag, per Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le seguenti informazioni:

    1. Origine: selezionaAmazon FSx per il file server Windows.

    2. Amazon FSxID del file system: seleziona l'ID del file system o crea una nuova directory.

    3. AWSSecrets Managersegreto: scegli un segreto esistente o creane uno nuovoSecrets Manager per archiviare le credenziali di autenticazione Amazon FSx. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendra-Amazon FSx' viene aggiunto automaticamente al tuo nome segreto.

        2. Per Nome utente: immettere il nome utente per l'accountAmazon FSx Active Directory.

        3. Per Password: immettere la password per l'accountAmazon FSx Active Directory.

      2. Seleziona Salva.

    4. Virtual Private Cloud (VPC): è necessario utilizzare un cloud privato virtuale con l'origineAmazon FSx dei dati. È necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    5. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    6. Seleziona Successivo.

  7. Nella pagina Configura delle impostazioni di sincronizzazione, inserisci le seguenti informazioni:

    1. Registro delle modifiche: seleziona questa opzione per aggiornare l'indice anziché sincronizzare tutti i file.

    2. Schemi Regex: aggiungono modelli di espressioni regolari per includere o escludere determinati contenuti. È possibile aggiungere fino a 100 pattern.

    3. In Sincronizza la pianificazione dell'esecuzione, per Frequenza: scegli la frequenza con cuiAmazon Kendra sincronizzarti con la tua fonte di dati.

    4. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi, inserisci le seguenti informazioni:

    1. Mappature dei campi di Amazon FSx Windows: seleziona tra i campi di origine dati predefinitiAmazon Kendra generati che desideri mappare nell'indice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice a cui mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È inoltre possibile scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra ad Amazon FSx

È necessario specificare quanto segue utilizzando l'FsxConfigurationAPI:

  • Nome segreto Amazon Resource Name (ARN): fornisci l'Amazon Resource Name (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo account Amazon FSx. Il segreto è memorizzato in una struttura JSON con le seguenti chiavi:

    {
    "username": "user@corp.example.com",
    "password": "password"
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore Amazon FSx eAmazon Kendra. Per ulteriori informazioni consulta Ruoli IAM per le sorgenti dati Amazon FSx.

È inoltre possibile aggiungere le seguenti funzionalità opzionali:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration come parte della configurazione dell'origine dei dati. Vedi ConfigurazioneAmazon Kendra per l'uso di un VPC.

  • Registro delle modifiche: se èAmazon Kendra necessario utilizzare il meccanismo del registro delle modifiche all'origine dei dati di Amazon FSx per determinare se un documento deve essere aggiunto, aggiornato o eliminato nell'indice.

    Nota

    Usa il registro delle modifiche se non desideriAmazon Kendra scansionare tutti i documenti. Se il registro delle modifiche è di grandi dimensioni, la scansione dei documenti nell'origine dati di Amazon FSx potrebbe richiedereAmazon Kendra meno tempo rispetto all'elaborazione del registro delle modifiche. Se stai sincronizzando la tua fonte di dati Amazon FSx con il tuo indice per la prima volta, tutti i documenti vengono scansionati.

  • Filtri di inclusione ed esclusione: è possibile specificare se includere o escludere determinati contenuti e tipi di contenuto.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi di origine dati Amazon FSx ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

Nota

Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati di un utente in base all'accesso dell'utente o del gruppo ai documenti.

Per testare il filtraggio del contesto utente su un utente, è necessario includere il nome di dominio DNS come parte del nome utente quando si esegue la query. È necessario disporre delle autorizzazioni amministrative del dominio Active Directory. Puoi anche testare il filtro del contesto utente in base al nome di un gruppo. Per ulteriori informazioni, consulta Filtraggio del contesto utente per le fonti di dati Amazon FSx.
