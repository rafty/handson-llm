Dropbox

Dropbox è un servizio di file hosting che offre servizi di archiviazione cloud, organizzazione dei documenti e creazione di modelli di documenti. Se sei un utente Dropbox, puoi utilizzarliAmazon Kendra per indicizzare i tuoi file Dropbox, Dropbox Paper, i modelli di Dropbox Paper e i collegamenti alle pagine web archiviati. Puoi ancheAmazon Kendra configurare l'indicizzazione di specifici file Dropbox, Dropbox Paper, modelli di Dropbox Paper e collegamenti a pagine web archiviati.

Amazon Kendrasupporta sia Dropbox che Dropbox Advanced per Dropbox Business.

Puoi connettertiAmazon Kendra alla fonte di dati di Dropbox utilizzando la Amazon Kendraconsole e l'TemplateConfigurationAPI.

Per la risoluzione dei problemi relativi al connettore di origine dati diAmazon Kendra Dropbox, consultaRisoluzione dei problemi delle origini dati.

Funzionalità supportate

Amazon KendraIl connettore di origine dati di Dropbox supporta le seguenti funzionalità:

  • Log delle modifiche

  • Mappature di campi

  • Filtraggio del contesto utente

  • Filtri di inclusione/esclusione

  • Virtual Private Cloud (VPC) (Cloud privato virtuale (VPC)

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la tua fonte di dati Dropbox, apporta queste modifiche nel tuo Dropbox e nei tuoiAWS account.

In Dropbox, assicurati di disporre di:

  • Hai creato un account Dropbox Advanced e configurato un utente amministratore.

  • Ha creato un'app Dropbox con un nome di app univoco, ha attivato Scoped Access. Consulta la documentazione di Dropbox sulla creazione di un'app.

  • Ha attivato le autorizzazioni complete di Dropbox sulla console Dropbox e ha aggiunto le seguenti autorizzazioni:

    • files.content.read

    • file.metadata.read

    • condivisione.leggi

    • file_requests.read

    • gruppi.leggi

    • team_info.leggi

    • team_data.content.read

  • Hai annotato la chiave dell'app Dropbox, il segreto dell'app Dropbox e il token di accesso a Dropbox per le credenziali di autenticazione di base.

  • Hai generato e copiato un token di accesso Oauth 2.0 temporaneo per la tua app Dropbox. Questo token è temporaneo e scade dopo 4 ore. Consulta la documentazione di Dropbox sull'autenticazione OAuth.

    Nota

    Ti consigliamo di creare un token di accesso all'aggiornamento di Dropbox che non scada mai, anziché affidarti a un token di accesso monouso che scade dopo 4 ore. Un token di accesso all'aggiornamento è permanente e non scade mai, quindi puoi continuare a sincronizzare la tua fonte di dati in future.

  • Consigliato: hai configurato un token di aggiornamento permanente di Dropbox che non scade mai per consentire di continuareAmazon Kendra a sincronizzare la tua fonte di dati senza interruzioni. Consulta la documentazione di Dropbox sui token di aggiornamento.

  • Hai verificato che ogni documento è unico in Dropbox e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le tue credenziali di autenticazione di Dropbox in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Non è consigliabile riutilizzare credenziali e segreti tra fonti di dati e nelle versioni dei connettori 1.0 e 2.0 (ove applicabile).

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando colleghi la tua fonte di dati Dropbox aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

Istruzioni di connessione

Per connettertiAmazon Kendra alla tua fonte di dati Dropbox, devi fornire i dettagli necessari della tua fonte dati Dropbox in modo cheAmazon Kendra possa accedere ai tuoi dati. Se non hai ancora configurato Dropbox,Amazon Kendra guardaPrerequisiti.

Console

Per connettersiAmazon Kendra a Dropbox

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli Connettore Dropbox, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica dei dettagli dell'origine dati compila le informazioni riportate di seguito:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. È possibile includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. Nella sezione Etichette, nella sezione Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza compila le informazioni riportate di seguito:

    1. Tipo di token di autenticazione: scegli tra token permanente (consigliato) e token di accesso (uso temporaneo) in base al tuo caso d'uso.

    2. AWSSecrets Managersegreto: scegli un segreto esistente o creane uno nuovoSecrets Manager per archiviare le tue credenziali di autenticazione Dropbox. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendra-Dropbox-' viene aggiunto automaticamente al tuo nome segreto.

        2. Per informazioni sulla chiave dell'app, sul segreto dell'app e sul token (permanenti o temporanei), inserisci i valori delle credenziali di autenticazione generati dal tuo account Dropbox.

      2. Seleziona Salva.

    3. Virtual Private Cloud (VPC): è possibile scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    4. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    5. Seleziona Successivo.

  7. Nella pagina Configura impostazioni di sincronizzazione compila le informazioni riportate di seguito:

    1. Per Seleziona entità o tipi di contenuto: scegli le entità o i tipi di contenuto da sottoporre a ricerca per indicizzazione.

    2. Cambia la modalità di registro: scegli di aggiornare l'indice invece di sincronizzare tutti i file.

    3. In Configurazione aggiuntiva per i modelli Regex: aggiunge modelli di espressioni regolari per includere o escludere determinati file.

    4. In Sincronizza programma di esecuzione, per Frequenza: scegli la frequenzaAmazon Kendra con cui sincronizzarti con la tua fonte di dati.

    5. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi compila le informazioni riportate di seguito:

    1. File, modelli di Dropbox Paper e Dropbox Paper: seleziona tra i campi di origine dati predefinitiAmazon Kendra generati che desideri mappare nell'indice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È inoltre possibile scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra a Dropbox

È necessario specificare un JSON dello schema di origine dati utilizzando l'TemplateConfigurationAPI. È necessario fornire le informazioni riportate di seguito:

  • Origine dati: è necessario specificare l'origine dei dati comeDROPBOX.

  • Tipo: specificaTEMPLATE come Tipo quando chiamiCreateDataSource.

  • Nome segreto della risorsa Amazon (ARN): fornisci il nome della risorsa Amazon (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo account Dropbox. Il segreto è archiviato in una struttura JSON con le seguenti chiavi:

    {
"appKey": "Dropbox app key",
"appSecret": "Dropbox app secret",
"accesstoken": "temporary access token or refresh access token"
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Non è consigliabile riutilizzare credenziali e segreti tra fonti di dati e nelle versioni dei connettori 1.0 e 2.0 (ove applicabile).

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore Dropbox eAmazon Kendra. Per ulteriori informazioni consulta Ruoli IAM per le fonti di dati Dropbox.

È inoltre possibile aggiungere le seguenti funzionalità opzionali:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration quando chiamareCreateDataSource. Consultare ConfigurazioneAmazon Kendra per l'utilizzo di un VPC.

  • Registro delle modifiche: se èAmazon Kendra necessario utilizzare il meccanismo del registro delle modifiche all'origine dei dati di Dropbox per determinare se un documento deve essere aggiunto, aggiornato o eliminato nell'indice.

    Nota

    Usa il registro delle modifiche se non desideriAmazon Kendra scansionare tutti i documenti. Se il registro delle modifiche è di grandi dimensioni, la scansione dei documenti nella fonte dati di Dropbox potrebbe richiedereAmazon Kendra meno tempo rispetto all'elaborazione del registro delle modifiche. Se stai sincronizzando la fonte di dati di Dropbox con il tuo indice per la prima volta, tutti i documenti vengono scansionati.

  • Filtri di inclusione ed esclusione: specificare se includere o escludere determinati file.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine dati di Dropbox ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati della ricerca in base all'accesso dell'utente o del suo gruppo ai documenti. Per ulteriori informazioni, vedi Filtraggio del contesto utente per le fonti di dati di Dropbox.

Per un elenco di altre chiavi JSON importanti da configurare, consultaSchema dei modelli Dropbox.
