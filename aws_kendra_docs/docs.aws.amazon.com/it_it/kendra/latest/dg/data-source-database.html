Amazon RDS/Aurora

È possibile indicizzare i documenti archiviati in un database utilizzando un'origine dati del database. Dopo aver fornito le informazioni di connessione per il database,Amazon Kendra connette e indicizza i documenti.

Amazon Kendrasupporta i seguenti database:

  • Amazon Aurora MySQL

  • Amazon Aurora PostgreSQL

  • Amazon RDS per MySQL

  • Amazon RDS per PostgreSQL

Nota

I database Aurora serverless non sono supportati.

Puoi connettertiAmazon Kendra alla fonte di dati del database utilizzando la Amazon Kendraconsole e l'DatabaseConfigurationAPI.

Per la risoluzione dei problemi relativi al connettore di origine dati delAmazon Kendra database, vedereRisoluzione dei problemi delle origini dati.

Funzionalità supportate

Amazon Kendrail connettore di origine dati del database supporta le seguenti funzionalità:

  • Mappature di campi

  • Filtraggio del contesto utente

  • Virtual Private Cloud (VPC) (Cloud privato virtuale (VPC)

Prerequisiti

Prima di poterlo utilizzareAmazon Kendra per indicizzare l'origine dei dati del database, apporta queste modifiche nel database e negliAWS account.

Nel database, assicurati di disporre di:

  • Hai indicato la coppia utente-password come credenziali di autenticazione di base per la tua fonte di dati.

  • Sono stati copiati il nome host, il numero di porta, l'indirizzo dell'host e il nome della tabella dati che contiene i dati del database. Per PostgreSQL, la tabella dati deve essere una tabella pubblica.

    Nota

    L'host e la porta indicanoAmazon Kendra dove trovare il server del database su Internet. Il nome del database e il nome della tabella indicanoAmazon Kendra dove trovare i dati del documento sul server del database.

  • Sono stati copiati i nomi delle colonne della tabella dati che contengono i dati del documento, l'ID del documento, da una a cinque colonne per rilevare se un documento è stato modificato e le colonne opzionali della tabella dati associate a campi indice personalizzati. È possibile mappare qualsiasi nome di campoAmazon Kendra riservato a una colonna della tabella.

  • Ha copiato le informazioni sul tipo di motore di database, ad esempio se lo usiAmazon RDS per MySQL o un altro tipo.

  • È stato verificato che ogni documento sia unico nel database e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le credenziali di autenticazione del database in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti l'origine dei dati del database aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

istruzioni di connessione:

Per connettersiAmazon Kendra all'origine dati del database, è necessario fornire i dettagli necessari dell'origine dati del database in modo cheAmazon Kendra possa accedere ai dati. Se non hai ancora configurato il database,Amazon Kendra consultaPrerequisiti.

Console

Per connettersiAmazon Kendra a un database

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli connettore di database, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specificare i dettagli della fonte dati compila le seguenti informazioni:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. È possibile includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. Nella sezione Tag, nella sezione Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza compila le seguenti informazioni:

    1. Endpoint: un nome host DNS, un indirizzo IPv4 o un indirizzo IPv6.

    2. Porta: un numero di porta.

    3. Database: nome del database.

    4. Nome tabella: nome della tabella.

    5. In Tipo di autenticazione, scegli tra Esistente e Nuovo per archiviare le credenziali di autenticazione del database. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendra-database-' viene aggiunto automaticamente al tuo nome segreto.

        2. Per Nome utente e password: inserisci i valori delle credenziali di autenticazione dall'account del tuo database.

        3. Scegli Salva autenticazione.

    6. Virtual Private Cloud (VPC): puoi scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

      Nota

      È necessario utilizzare una sottorete privata. Se l'istanza RDS si trova in una sottorete pubblica del VPC, è possibile creare una sottorete privata con accesso in uscita a un gateway NAT nella sottorete pubblica. Le sottoreti fornite nella configurazione VPC devono trovarsi nelle regioni Stati Uniti occidentali (Oregon), Stati Uniti orientali (Virginia settentrionale) e UE (Irlanda).

    7. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    8. Seleziona Successivo.

  7. Nella pagina Configura impostazioni di sincronizzazione compila le seguenti informazioni:

    1. Scegli tra Aurora MySQL, MySQL, Aurora PostgreSQL e PostgreSQL in base al tuo caso d'uso.

    2. Racchiudere gli identificatori SQL tra virgolette doppie: selezionare questa opzione per racchiudere gli identificatori SQL tra virgolette doppie. Ad esempio, «columnName».

    3. Colonna ACL e colonne di rilevamento delle modifiche: consente di configurare le colonneAmazon Kendra utilizzate per il rilevamento delle modifiche e gli elenchi di controllo degli accessi.

    4. In Sincronizza programma di esecuzione, per Frequenza: scegli la frequenzaAmazon Kendra con cui sincronizzarti con la tua fonte di dati.

    5. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi compila le seguenti informazioni:

    1. Amazon Kendramappature dei campi predefinite: seleziona dai campi di origine dati predefinitiAmazon Kendra generati che desideri mappare nell'indice. È necessario aggiungere i valori della colonna Database perdocument_id edocument_body

    2. Mappature dei campi personalizzate: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice a cui mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È inoltre possibile scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra a un database

È necessario specificare la seguente DatabaseConfigurationAPI:

  • ColumnConfiguration—Informazioni sulla posizione in cui l'indice deve ottenere le informazioni del documento dal database. Per ulteriori dettagli, consulta ColumnConfiguration. È necessario specificare iDocumentIdColumnName campiDocumentDataColumnName e. La colonna mappata alDocumentIdColumnName campo deve essere una colonna intera. L'esempio seguente mostra una semplice configurazione di colonne per un'origine dati di database: 

    "ColumnConfiguration": {
    "ChangeDetectingColumns": [
        "LastUpdateDate",
        "LastUpdateTime"
    ],
    "DocumentDataColumnName": "TextColumn",
    "DocumentIdColumnName": "IdentifierColumn",
    "DocoumentTitleColumnName": "TitleColumn",
    "FieldMappings": [
        {
            "DataSourceFieldName": "AbstractColumn",
            "IndexFieldName": "Abstract"
        }
    ]
}

  • ConnectionConfiguration—Il tipo di motore di database che esegue il database. IlDatabaseHost campo deve essere l'endpoint dell'istanzaAmazon Relational Database Service (Amazon RDS) per il database. Non utilizzare l'endpoint del cluster.

  • DatabaseEngineType—Informazioni sulla configurazione richiesta per effettuare la connessione a un database. Per ulteriori dettagli, consulta ConnectionConfiguration.

  • Nome segreto Amazon Resource Name (ARN): fornisci l'Amazon Resource Name (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo account di database. Il segreto è memorizzato in una struttura JSON con le seguenti chiavi:

    {
    "username": "user name",
    "password": "password"
}

    Il segreto può contenere ulteriori informazioni. L'esempio seguente mostra una configurazione di database.

    
"DatabaseConfiguration": {
"ConnectionConfiguration": {
"DatabaseHost": "host.subdomain.domain.tld",
        "DatabaseName": "DocumentDatabase",
        "DatabasePort": 3306,
        "SecretArn": "arn:aws:secretmanager:region:account ID:secret/secret name",
        "TableName": "DocumentTable"
    }
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore del database eAmazon Kendra. Per ulteriori informazioni consulta Ruoli IAM per le fonti dati del database.

È inoltre possibile aggiungere le seguenti caratteristiche opzionali:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration come parte della configurazione dell'origine dei dati. Vedi ConfigurazioneAmazon Kendra per l'uso di un VPC.

    Nota

    È necessario utilizzare solo una sottorete privata. Se l'istanza RDS si trova in una sottorete pubblica del VPC, è possibile creare una sottorete privata con accesso in uscita a un gateway NAT nella sottorete pubblica. Le sottoreti fornite nella configurazione VPC devono trovarsi nelle regioni Stati Uniti occidentali (Oregon), Stati Uniti orientali (Virginia settentrionale) e UE (Irlanda).

  • Mappature dei campi: scegli di mappare i campi dell'origine dei dati del database ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati della ricerca in base all'accesso dell'utente o del suo gruppo ai documenti. Per ulteriori informazioni, vedere Filtraggio del contesto utente per le fonti di dati del database.
