Mappatura dei campi dell'origine dei dati

È possibile mappare i campi del documento o del contenuto dall'origine dei dati ai campi dell'indice. Ad esempio, se nella fonte di dati è presente un campo chiamato «reparto» che contiene le informazioni sul reparto relative a un documento, è possibile mapparlo su un campo indice chiamato «Dipartimento». In questo modo, puoi utilizzare il campo per interrogare i documenti.

Puoi anche mappare campi standard o riservati, ad esempio_created_at. Se la tua fonte di dati ha un campo chiamato «creation_date», puoi mapparlo al campoAmazon Kendra riservato equivalente chiamato_created_at. È possibile mappare i campi per la maggior parte delle fonti di dati.

Se archivi i tuoi documenti in un bucket S3 o in un'origine dati S3, puoi fornire attributi personalizzati direttamente utilizzando i file di metadati. Per ulteriori informazioni, consulta Creazione di documenti personalizzati.

La mappatura dei campi dell'origine dei dati su un campo indice è un processo in tre fasi:

  1. Creazione di un indice. Per ulteriori informazioni, consulta Creazione di un indice.

  2. Aggiorna l'indice per aggiungere campi personalizzati.

  3. Crea un'origine dati che associa i campi delle origini dati ai campi dell'indice.

Per aggiornare l'indice per aggiungere campi personalizzati, usa la console o l'UpdateIndexAPI. Puoi aggiungere un totale di 500 campi personalizzati al tuo indice.

Nella console, puoi scegliere di mappare un campo sorgente dati a uno dei nomi di campo riservati. In alternativa, puoi scegliere di creare un nuovo campo indice associato al campo. Per le fonti di dati del database, se il nome della colonna del database corrisponde al nome di un campo riservato, il campo e la colonna vengono mappati automaticamente.

Con l'UpdateIndexAPI, aggiungi campi personalizzati e riservati utilizzando ilDocumentMetadataConfigurationUpdates parametro.

Il seguente esempio JSON utilizzaDocumentMetadataConfigurationUpdates per aggiungere un campo chiamato «Reparto» all'indice.

"DocumentmetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE"
   }
]

Quando si crea il campo, è possibile impostare la modalità di utilizzo del campo nelle ricerche. Puoi scegliere tra le seguenti opzioni:

  • Visualizzabile: determina se il campo viene restituito nella risposta alla query. Il valore predefinito è true.

  • Sfacetable: indica che il campo può essere utilizzato per creare facet. Il valore predefinito è false.

  • Ricercabile: determina se il campo viene utilizzato nella ricerca. L'impostazione predefinita è true per i campi stringa e false per i campi numero e data.

  • Ordinabile: indica che il campo può essere utilizzato per ordinare la risposta a una query. Può essere impostato solo per i campi di data, numero e stringa. Non può essere impostato per i campi dell'elenco di stringhe.

Il seguente esempio JSON utilizzaDocumentMetadataConfigurationUpdates per aggiungere un campo chiamato «Reparto» all'indice e contrassegnarlo come sfaccettato.

"DocumentMetadataConfigurationUpdates": [
   {
       "Name": "Department",
       "Type": "STRING_VALUE",
       "Search": {
           "Facetable": true
       }
   }
]

Utilizzo dei campi documentali integrati in Amazon Kendra

Con l'UpdateIndex API, puoi creare campi riservati o incorporati utilizzandoDocumentMetadataConfigurationUpdates e specificando il nome del campo riservato da mappare all'attributo o al campo del documento equivalente. In questo modo puoi anche creare campi personalizzati. Se si utilizza un connettore di origine dati, la maggior parte include mappature dei campi che associano i campi del documento di origine dati ai campi diAmazon Kendra indicizzazione. Quando si crea un campo, è possibile configurare l'Searchoggetto per impostarlo come visualizzabile, sfaccettato, ricercabile e ordinabile. È possibile configurare l'Relevanceoggetto per impostare la pertinenza del campo aumentando la durata, la freschezza, l'importanza, l'ordine di classificazione e i valori di importanza. Non è possibile modificare il tipo di campo dopo averlo creato.

Amazon Kendraha i seguenti campi riservati o incorporati del documento che è possibile utilizzare:

  • _authors—Un elenco di uno o più autori responsabili del contenuto del documento.

  • _category—Una categoria che inserisce un documento in un gruppo specifico.

  • _created_at—La data e l'ora in formato ISO 8601 di creazione del documento. Ad esempio, 2012-03-25T12:30:10+01:00 è il formato data/ora ISO 8601 per il 25 marzo 2012 alle 12:30 (più 10 secondi) nel fuso orario dell'Europa centrale (CET).

  • _data_source_id—L' identificatore dell'origine dati contenente il documento.

  • _document_body—Il contenuto del documento.

  • _document_id—Un identificatore univoco per il documento.

  • _document_title—Il titolo del documento.

  • _excerpt_page_number—Il numero di pagina in un file PDF in cui viene visualizzato l'estratto del documento. Se l'indice è stato creato prima dell'8 settembre 2020, è necessario reindicizzare i documenti prima di poter utilizzare questo attributo.

  • _faq_id—Se si tratta di una domanda e risposta delle FAQ, un identificatore univoco per esse.

  • _file_type—Il tipo di file del documento, ad esempio pdf o doc.

  • _last_updated_at—La data e l'ora in formato ISO 8601 dell'ultimo aggiornamento del documento. Ad esempio, 2012-03-25T12:30:10+01:00 è il formato data/ora ISO 8601 per il 25 marzo 2012 alle 12:30 (più 10 secondi) nel fuso orario dell'Europa centrale (CET).

  • _source_uri—L'URI in cui è disponibile il documento. Ad esempio, l'URI del documento su un sito Web aziendale.

  • _version—Un identificatore per la versione specifica di un documento.

  • _view_count—Il numero di volte in cui il documento è stato visualizzato.

  • _language_code(String) —Il codice di una lingua che si applica al documento. L'impostazione predefinita è l'inglese se non specifichi una lingua. Per ulteriori informazioni sulle lingue supportate, inclusi i relativi codici, consulta Aggiungere documenti in lingue diverse dall'inglese.

Per i campi personalizzati, puoi creare questi campi utilizzandoDocumentMetadataConfigurationUpdates l'UpdateIndexAPI, proprio come fai quando crei un campo riservato o incorporato. È necessario impostare il tipo di dati appropriato per il campo personalizzato. Non è possibile modificare il tipo di campo dopo averlo creato.

Di seguito sono riportati i tipi che puoi impostare per i campi personalizzati:

  • Data

  • Numero

  • Stringa

  • Elenco stringhe

Puoi creare un campo personalizzato utilizzando la console o utilizzando l'UpdateIndexAPI. Dopo aver creato un campo personalizzato, lo si associa a un attributo del documento, proprio come si fa con un campo riservato. Se hai aggiunto un documento all'indice con l'BatchPutDocumentAPI, mappi gli attributi con l'API. Per i documenti indicizzati da un'origineAmazon S3 dati, è possibile mappare gli attributi utilizzando un file di metadati che contiene una struttura JSON che descrive gli attributi del documento. Per i documenti indicizzati con un database o un'origine dati che consente la mappatura dei campi, è possibile mappare gli attributi con la console o la configurazione dell'origine dati.
