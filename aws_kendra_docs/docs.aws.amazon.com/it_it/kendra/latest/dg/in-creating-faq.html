Aggiunta delle domande frequenti (FAQ) direttamente a un indice

Puoi aggiungere domande frequenti (FAQ) direttamente al tuo indice utilizzando la console o l'CreateFaqAPI. L'aggiunta di domande frequenti a un indice è un'operazione asincrona. I dati per le domande frequenti vengono inseriti in un file che si archivia in unAmazon Simple Storage Service bucket. Puoi utilizzare file CSV o JSON come input per le tue domande frequenti:

  • CSV di base: un file CSV in cui ogni riga contiene una domanda, una risposta e un URL opzionale con ulteriori informazioni sulla risposta.

  • CSV personalizzato: un file CSV che contiene domande, risposte e un'intestazione che definisce attributi personalizzati che è possibile utilizzare per sfaccettare, visualizzare o ordinare le risposte alle domande frequenti. È inoltre possibile definire gli attributi di controllo degli accessi per limitare la risposta alle domande frequenti a determinati utenti e gruppi.

  • JSON: un file JSON che contiene domande e risposte. Facoltativamente, può includere anche attributi personalizzati e di controllo dell'accesso. È possibile definire gli attributi per sfatare, visualizzare e ordinare le risposte alle domande frequenti. In alternativa, puoi utilizzare gli attributi di controllo degli accessi che limitano la risposta alle domande frequenti a determinati utenti e gruppi.

Ad esempio, quello che segue è un file CSV di base che fornisce risposte a domande sulle cliniche gratuite a Spokane, Washington USA e Mountain View, Missouri, USA.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

Quando si utilizza un file CSV o JSON personalizzato per l'input, è possibile dichiarare attributi personalizzati per le domande frequenti. Ad esempio, puoi creare un attributo personalizzato che assegna a ogni domanda con domande frequenti un reparto. Quando le domande frequenti vengono restituite in una risposta, puoi utilizzare il dipartimento come sfaccettatura per restringere la ricerca.

Un attributo personalizzato deve essere mappato a un campo indice. Puoi utilizzare un campo integrato o specificare un campo indice personalizzato. Nella console, si utilizza la pagina di definizione di Facet per creare un campo indice. Se utilizzi l"API, dovrai innanzitutto creare un campo indice utilizzando l'UpdateIndexAPI.

Il tipo di attributo nel file delle domande frequenti deve corrispondere al tipo del campo indice associato. Ad esempio, il_authors campo incorporato è un campoSTRING_LIST tipo. Pertanto, è necessario fornire i valori per il_authors campo come elenco di stringhe nel file delle domande frequenti. È possibile verificare il tipo di campi dell'indice utilizzando la pagina di definizione di Facet nella console o utilizzando l'DescribeIndexAPI.

Quando si crea un campo indice associato a un attributo personalizzato, è possibile contrassegnarlo come visualizzabile, sfaccettato o ordinabile. Non puoi rendere ricercabile un attributo personalizzato.

Oltre agli attributi personalizzati, puoi anche utilizzare i seguenti attributi incorporati in un file CSV o JSON personalizzato:

  • _authors(Elenco di stringhe) —Un elenco degli autori delle risposte alle domande frequenti.

  • _category(String) —Una categoria che raggruppa le risposte alle domande frequenti con altri documenti simili.

  • _created_at(stringa codificata ISO 8601) — La data e l'ora di creazione della domanda con domande frequenti. La data e l'ora devono essere formattate come una stringa con codifica ISO 8601.

    Se necessario, puoi anche includere il fuso orario nel formato data/ora ISO 8601. Ad esempio, 2012-03-25T 12:30:10 è il formato data/ora ISO 8601 per il 25 marzo 2012 per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, alle 12:30 (nel fuso orario dell'Europa centrale).

  • _last_updated_at(stringa codificata ISO 8601) —La data e l'ora in cui la domanda delle domande frequenti è stata aggiornata. La data e l'ora devono essere formattate come stringa codificata ISO 8601.

    Ad esempio, 2012-03-25T 12:30:10 è il formato data/ora ISO 8601 per il 25 marzo 2012 per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, alle 12:30 (nel fuso orario dell'Europa centrale).

  • _source_uri(String): un URL per un documento con ulteriori informazioni sulla risposta alle domande frequenti.

  • _version(String) —La versione della domanda delle domande frequenti.

  • _view_count(Lungo): il numero di volte in cui la domanda delle domande frequenti è stata visualizzata nei risultati di ricerca.

File CSV di base

Usa un file CSV di base quando desideri utilizzare una struttura semplice per le domande frequenti. In un file CSV di base, ogni riga ha due o tre campi: una domanda, una risposta e un URI di origine opzionale che rimanda a un documento con ulteriori informazioni.

Il contenuto del file deve seguire il formato comune RFC 4180 e il tipo MIME per i file CSV (Comma-Separated Values).

Di seguito è riportato un file di domande frequenti nel formato CSV di base.

How many free clinics are in Spokane WA?, 13, https://www.freeclinics.com/
How many free clinics are there in Mountain View Missouri?, 7, https://www.freeclinics.com/

File CSV personalizzato

Usa un file CSV personalizzato quando desideri aggiungere attributi personalizzati alle tue domande frequenti. Per un file CSV personalizzato, si utilizza una riga di intestazione nel file CSV per definire gli attributi aggiuntivi.

Il file CSV deve contenere i seguenti due attributi obbligatori:

  • _question—La domanda più frequente

  • _answer—La risposta alla domanda più frequente

Il file può contenere attributi predefiniti e personalizzati. Di seguito viene riportato un esempio di file CSV personalizzato.

_question,_answer,_last_updated_at,custom_string
How many free clinics are in Spokane WA?, 13, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services
How many free clinics are there in Mountain View Missouri?, 7, 2012-03-25T12:30:10+01:00, Note: Some free clinics require you to meet certain criteria in order to use their services

Il contenuto del file degli attributi personalizzati deve seguire il formato comune e il tipo MIME RFC 4180 per i file con valori separati da virgole (CSV).

Esistono quattro tipi di attributi personalizzati:

  • Data: valori di data e ora codificati con ISO 8601.

    Ad esempio, 2012-03-25T 12:30:10 è il formato data/ora ISO 8601 per il 25 marzo 2012 per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, alle 12:30 (nel fuso orario dell'Europa centrale).

  • Lungo: numeri, ad esempio1234.

  • Stringa: valori di stringa. Se la stringa contiene virgole, racchiudi l'intero valore tra virgolette doppie («) (ad esempio,"custom attribute, and more").

  • String List: un elenco di valori di stringa. Elencare i valori in un elenco separato da virgole racchiuso tra virgolette («) (ad esempio,"item1, item2, item3"). Se l'elenco contiene una sola voce, è possibile omettere le virgolette (ad esempioitem1).

Un file CSV personalizzato può contenere campi di contesto utente. Puoi utilizzare questi campi per limitare l'accesso alle domande frequenti a determinati utenti e gruppi. Per filtrare in base al contesto dell'utente, l'utente deve fornire informazioni sull'utente e sul gruppo nella query. In caso contrario, vengono restituite tutte le domande frequenti pertinenti. Per ulteriori informazioni, consulta Filtraggio in base al contesto dell'utente.

Esistono quattro filtri contestuali utente per le domande frequenti:

  • _acl_user_allow—Gli utenti nell'elenco degli utenti consentiti possono visualizzare le domande frequenti nella risposta alla query. Le domande frequenti non vengono restituite agli altri utenti.

  • _acl_user_deny—Gli utenti nell'elenco dei commenti negati non possono visualizzare le domande frequenti nella risposta alla query. Le domande frequenti vengono restituite a tutti gli altri utenti quando sono pertinenti alla query.

  • _acl_group_allow—Gli utenti membri di un gruppo consentito possono visualizzare le domande frequenti nella risposta alla query. Le domande frequenti non vengono restituite agli utenti membri di un altro gruppo.

  • _acl_group_deny—Gli utenti membri di un gruppo negato non possono visualizzare le domande frequenti nella risposta alla query. Le domande frequenti vengono restituite agli altri gruppi quando sono pertinenti alla query.

Fornisci i valori per gli elenchi consentiti e negati in elenchi separati da virgole racchiusi tra virgolette (ad esempio,"user1,user2,user3"). È possibile includere un utente o un gruppo in un elenco consentito o in un elenco negato, ma non in entrambi. Se includi un utente o un gruppo in entrambi, ricevi un errore.

Di seguito viene riportato un esempio di file CSV personalizzato con informazioni sul contesto utente.

_question, _answer, _acl_user_allow, _acl_user_deny, _acl_group_allow, _acl_group_deny
How many free clinics are in Spokane WA?, 13, userID4565, "userID6201, userID7552", groupBasicRate, "groupBasicPlusRate,groupPremiumRate"

File JSON

Puoi utilizzare un file JSON per fornire domande, risposte e attributi per il tuo indice. Puoi aggiungere uno qualsiasi degli attributi predefiniti e gli attributi personalizzati alle domande frequenti. Questo è lo schema per il file JSON.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": string,
            "Answer": string,
            "Attributes": {
                string: object
                additional attributes
            },
            "AccessControlList": [
               {
                   "Name": string,
                   "Type": enum( "GROUP" | "USER" ),
                   "Access": enum( "ALLOW" | "DENY" )
               },
               additional user context
            ]
        },
        additional FAQ documents
    ]
}

Questo file JSON di esempio mostra due documenti con domande frequenti. Uno di essi contiene solo la domanda e la risposta richieste. L'altro ha anche attributi aggiuntivi e informazioni sul contesto dell'utente.

{
    "SchemaVersion": 1,
    "FaqDocuments": [
        {
            "Question": "How many free clinics are in Spokane WA?",
            "Answer": "13"
        },
        {
            "Question": "How many free clinics are there in Mountain View Missouri?",
            "Answer": "7",
            "Attributes": {
                "_source_uri": "https://www.freeclinics.com",
                "_category": "Charitable Clinics"
            },
            "AccessControlList": [
               {
                   "Name": "user@amazon.com",
                   "Type": "USER",
                   "Access": "ALLOW"
               },
               {
                   "Name": "Admin",
                   "Type": "GROUP",
                   "Access": "ALLOW"
               }
            ]
        }
    ]
}

Esistono quattro tipi di attributi personalizzati per i file JSON:

  • Data: un valore di stringa JSON con valori di data e ora codificati ISO 8601.

    È importante che il fuso orario sia incluso nel formato data/ora ISO 8601. Ad esempio, 2012-03-25T 12:30:10 è il formato data/ora ISO 8601 per il 25 marzo 2012 per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, per il 25 marzo 2012, alle 12:30 (nel fuso orario dell'Europa centrale).

  • Lungo: un valore numerico JSON, ad esempio1234.

  • Stringa: un valore di stringa JSON (ad esempio"custom attribute").

  • Elenco di stringhe: una matrice JSON di valori di stringa (ad esempio,["item1,item2,item3"]).

Oltre agli attributi incorporati e personalizzati, puoi fornire informazioni sul contesto utente per le domande frequenti in un file JSON. È possibile fornire informazioni sul contesto dell'utente per limitare l'accesso al contenuto delle domande frequenti in base a utenti e gruppi. È possibile includere un utente o un gruppo in un elenco consentito o in un elenco negato, ma non in entrambi. Se includi un utente o un gruppo in entrambi, ricevi un errore. Per filtrare in base al contesto dell'utente, è necessario fornire informazioni sull'utente e sul gruppo nella query. In caso contrario, vengono restituite tutte le domande frequenti pertinenti. Per ulteriori informazioni, consulta Filtraggio in base al contesto dell'utente.

Questo esempio JSON aggiunge il contesto utente a una FAQ.

"AccessControlList": [
                {
                    "Name": "group or user name",
                    "Type": "GROUP | USER",
                    "Access": "ALLOW | DENY"
                },
                additional user context
            ]

Utilizzo del file delle domande frequenti

Dopo aver archiviato il file di input delle domande frequenti in un bucket S3, usi la console o l'CreateFaqAPI per inserire le domande e le risposte nel tuo indice. Se desideri aggiornare una domanda frequente, eliminala e creala di nuovo. Si utilizza l'DeleteFaqAPI per eliminare una domanda frequente.

È necessario fornire unIAM ruolo che abbia accesso al bucket S3 contenente i file sorgente. Si specifica il ruolo nella console o nelRoleArn parametro. Di seguito viene riportato un esempio di programma che aggiunge un file di domande frequenti a un indice.

Python
import boto3

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the IAM role ARN required to index documents in an S3 bucket
role_arn = "arn:aws:iam::${accountId}:role/${roleName}"

# Provide the S3 bucket path information to the FAQ file
faq_path = {
    "Bucket": "bucket-name",
    "Key": "FreeClinicsUSA.csv"
}

response = kendra.create_faq(
    S3Path =  faq_path,
    Name = "FreeClinicsUSA",
    IndexId = index_id,
    RoleArn = role_arn
)

print(response)
Java
package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.CreateFaqRequest;
import software.amazon.awssdk.services.kendra.model.CreateFaqResponse;
import software.amazon.awssdk.services.kendra.model.S3Path;

public class AddFaqExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String indexId = "yourIndexId";
        String roleArn = "your role for accessing S3 files";

        CreateFaqRequest createFaqRequest = CreateFaqRequest
            .builder()
            .indexId(indexId)
            .name("FreeClinicsUSA")
            .roleArn(roleArn)
            .s3Path(
                S3Path
                    .builder()
                    .bucket("an-aws-kendra-test-bucket")
                    .key("FreeClinicsUSA.csv")
                    .build())
            .build();

        CreateFaqResponse response = kendra.createFaq(createFaqRequest);

        System.out.println(String.format("The result of creating FAQ: %s", response));

    }
}

File di domande frequenti in lingue diverse dall'inglese

Puoi indicizzare una domanda frequente in una lingua supportata. Amazon Kendraindicizza le domande frequenti in inglese per impostazione predefinita se non si specifica una lingua. È possibile specificare il codice della lingua quando si chiama l'CreateFaqoperazione oppure è possibile includere il codice della lingua per una domanda frequente nei metadati delle domande frequenti come campo. Se una domanda frequente non ha un codice di lingua nei metadati specificati in un campo di metadati, la FAQ viene indicizzata utilizzando il codice della lingua specificato quando si chiama l'CreateFAQoperazione. Per indicizzare un documento di domande frequenti in una lingua supportata nella console, vai a Domande frequenti e seleziona Aggiungi domande frequenti. Scegli una lingua dal menu a tendina Lingua.
