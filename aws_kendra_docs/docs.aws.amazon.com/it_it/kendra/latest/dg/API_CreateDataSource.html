CreateDataSource

Crea un connettore dell’origine dati da utilizzare con un indice Amazon Kendra.

Specifica un nome, un tipo di connettore dell’origine dati e una descrizione per l'origine dati. Specifica anche le informazioni di configurazione per il connettore dell’origine dati.

CreateDataSourceè un'operazione sincrona. L'operazione restituisce 200 se l'origine dati è stata creata con successo. In caso contrario, viene sollevata un'eccezione.

Per un esempio di creazione di un indice e di una fonte di dati utilizzando Python SDK, consulta Guida introduttiva a Python SDK. Per un esempio di creazione di un indice e di un'origine dati utilizzando Java SDK, vedere Guida introduttiva a Java SDK.

Parametri della richiesta

Per informazioni sui parametri comuni a tutte le azioni, vedere Parametri comuni.

La richiesta accetta i seguenti dati in formato JSON.

ClientToken

Un token fornito per identificare la richiesta di creare un connettore origine dati. Più chiamate all'CreateDataSourceAPI con lo stesso token client creeranno un solo connettore di origine dati.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 1. Lunghezza massima di 100.

Required: No

Configuration

Informazioni di configurazione per connettersi al repository delle fonti di dati.

Non è possibile specificare il parametro Configuration quando il parametro Type è impostato su CUSTOM. Se lo fai, riceverai un'ValidationExceptioneccezione.

Il parametro Configuration è obbligatorio per tutte le altre origini dati.

Tipo: DataSourceConfiguration oggetto

Required: No

CustomDocumentEnrichmentConfiguration

Le informazioni sulla configurazione per modificare i metadati e il contenuto del documento durante il processo di importazione del documento.

Per ulteriori informazioni su come creare, modificare ed eliminare i metadati dei documenti o apportare altre modifiche ai contenuti quando inserisci documenti in Amazon Kendra, consulta Personalizzazione dei metadati dei documenti durante il processo di acquisizione.

Tipo: CustomDocumentEnrichmentConfiguration oggetto

Required: No

Description

Una descrizione per il connettore dell’origine dati.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 0. Lunghezza massima di 1000.

Modello: ^\P{C}*$

Required: No

IndexId

L’identificatore dell'indice da utilizzare con il connettore dell’origine dati.

Tipo: String

Vincoli di lunghezza: lunghezza fissa di 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Campo obbligatorio: sì

LanguageCode

Il codice di una lingua. Ciò consente di supportare una lingua per tutti i documenti durante la creazione del connettore di origine dati. L'inglese è supportato per impostazione predefinita. Per ulteriori informazioni sulle lingue supportate, inclusi i relativi codici, consulta Aggiungere documenti in lingue diverse dall'inglese.

Tipo: String

Vincoli di lunghezza: lunghezza di 2. La lunghezza di 10.

Modello: [a-zA-Z-]*

Required: No

Name

Un nome per il connettore dell'origine dati.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 1. Lunghezza massima di 1000.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Campo obbligatorio: sì

RoleArn

L'Amazon Resource Name (ARN) di un ruolo IAM con l'autorizzazione ad accedere all'origine dei dati e alle risorse necessarie. Per ulteriori informazioni consulta la sezione Ruoli di accesso IAM per Amazon Kendra. .

Non è possibile specificare il parametro RoleArn quando il parametro Type è impostato su CUSTOM. Se lo fai, riceverai un'ValidationExceptioneccezione.

Il parametro RoleArn è obbligatorio per tutte le altre origini dati.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 0. La lunghezza di 1284.

Modello: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Required: No

Schedule

Imposta la frequenza con Amazon Kendra di controllare i documenti nell'archivio delle fonti dati e aggiornare l'indice. Se non imposti una pianificazione, Amazon Kendra non aggiornerà periodicamente l'indice. Puoi chiamare l'StartDataSourceSyncJobAPI per aggiornare l'indice.

Specifica una stringa di pianificazione delcron- formato o una stringa vuota per indicare che l'indice viene aggiornato su richiesta.

Non è possibile specificare il parametro Schedule quando il parametro Type è impostato su CUSTOM. Se lo fai, riceverai un'ValidationExceptioneccezione.

Tipo: string

Required: No

Tags

Un elenco di coppie chiave-valore che identificano o classificano il connettore dell'origine dati. È possibile utilizzare i tag per controllare l'accesso al connettore dell'origine dati. Le chiavi e i valori di tag possono essere composti da lettere, numeri, spazi e simboli: _.:/= + - @.

Tipo: matrice di oggetti Tag

Membri dell'array: numero minimo di 0 elementi. Numero elementi.

Required: No

Type

Il tipo di repository dell'origine dati. Ad esempio, SHAREPOINT.

Tipo: String

Valori validi: S3 | SHAREPOINT | DATABASE | SALESFORCE | ONEDRIVE | SERVICENOW | CUSTOM | CONFLUENCE | GOOGLEDRIVE | WEBCRAWLER | WORKDOCS | FSX | SLACK | BOX | QUIP | JIRA | GITHUB | ALFRESCO | TEMPLATE

Campo obbligatorio: sì

VpcConfiguration

Informazioni di configurazione per un Amazon Virtual Private Cloud da connettere alla tua fonte di dati. Per ulteriori informazioni, consulta la sezione Configurazione di un VPC.

Tipo: DataSourceVpcConfiguration oggetto

Required: No

Elementi di risposta

Se l'operazione riesce, il servizio restituisce una risposta HTTP 200.

I dati seguenti vengono restituiti in formato JSON mediante il servizio.

Id

L'identificatore del connettore dell'origine dati.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 1. Lunghezza massima di 100.

Pattern: [a-zA-Z0-9][a-zA-Z0-9_-]*

Errori

Per informazioni sugli errori comuni a tutte le operazioni, consultare Errori comuni.

AccessDeniedException

Non disponi delle autorizzazioni per eseguire questa azione. Assicurati di disporre delle politiche di autorizzazione e degli account utente richiesti e riprova.

Codice di stato HTTP: 400

ConflictException

Si è verificato un conflitto con la richiesta. Correggi eventuali incongruenze con le tue risorse e riprova.

Codice di stato HTTP: 400

InternalServerException

Si è verificato un problema con il server interno utilizzato per il servizio Amazon Kendra. Attendi qualche minuto e riprova oppure contatta l'Support per ricevere assistenza.

Codice di stato HTTP: 500

ResourceAlreadyExistException

La risorsa che desideri utilizzare esiste già. Verifica di aver fornito la risorsa corretta e riprova.

Codice di stato HTTP: 400

ResourceNotFoundException

La risorsa che desideri utilizzare non esiste. Verifica di aver fornito la risorsa corretta e riprova.

Codice di stato HTTP: 400

ServiceQuotaExceededException

Hai superato i limiti stabiliti per il tuo servizio Amazon Kendra. Consulta la sezione Quote per ulteriori informazioni o contatta l'Support per richiedere informazioni su un aumento dei limiti.

Codice di stato HTTP: 400

ThrottlingException

La richiesta è stata negata a causa del throttling della richiesta. Ridurre il numero di richieste e riprovare.

Codice di stato HTTP: 400

ValidationException

L'input non riesce a soddisfare i vincoli stabiliti dal servizio Amazon Kendra. Fornire l'input input input e riprovare.

Codice di stato HTTP: 400
