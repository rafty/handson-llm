Amazon WorkDocs

Amazon WorkDocsè un servizio sicuro di collaborazione sui contenuti per la creazione, la modifica, l'archiviazione e la condivisione di contenuti. Puoi usarloAmazon Kendra per indicizzare la tua fonte diAmazon WorkDocs dati.

Puoi connettertiAmazon Kendra alla tua fonte diAmazon WorkDocs dati utilizzando la Amazon Kendraconsole e l'WorkDocsConfigurationAPI.

Amazon WorkDocsè disponibile nelle Regioni Oregon, Virginia settentrionale, Sydney, Singapore e Irlanda.

Per la risoluzione dei problemi relativi al connettore della sorgenteAmazon Kendra WorkDocs dati, consultaRisoluzione dei problemi delle origini dati.

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la fonte WorkDocs dei dati, apporta queste modifiche nei tuoiAWS account WorkDocs e.

Nel WorkDocs, assicurati di avere:

  • Hai notato l'ID dellaAmazon WorkDocs directory (ID dell'organizzazione) per il tuoAmazon WorkDocs repository.

  • È stato verificato che ogni documento sia univoco in WorkDocs altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

Se non disponi di unIAM ruolo esistente, puoi utilizzare la console per creare un nuovoIAM ruolo quando connetti la tua fonte di WorkDocs dati aAmazon Kendra. Se utilizzi l'API, devi specificare l'ARN di unIAM ruolo esistente e un ID indice.

Console

Per connettersiAmazon Kendra aAmazon WorkDocs

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli WorkDocs connettore, quindi scegli Aggiungi origine dati.

  5. Nella pagina specificare dettagli dell'origine dati, inserisci le seguenti informazioni:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. Puoi includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per l'origine dei dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. Nella sezione Tag, nella sezione Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le seguenti informazioni:

    1. ID dell'organizzazione specificoAmazon WorkDocs del sito: seleziona l'ID delAmazon WorkDocs sito che desideri indicizzare. Devi aver già creato un sito.

    2. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    3. Seleziona Successivo.

  7. Nella pagina Configura impostazioni di sincronizzazione, inserisci le seguenti informazioni:

    1. Esegui ricerca per indicizzazione dei commenti del documento: leAmazon WorkDocs entità o i tipi di contenuto che desideri sottoporre a ricerca per indicizzazione.

    2. Usa i log delle modifiche: seleziona per aggiornare l'indice invece di sincronizzare tutti i file.

    3. Schemi Regex: modelli di espressioni regolari per includere o escludere determinati file. È possibile aggiungere fino a 100 pattern.

    4. In Sincronizza la pianificazione dell'esecuzione per la frequenza: scegli la frequenza con cui sincronizzarti con la tua fonte di dati.Amazon Kendra

    5. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi, inserisci le seguenti informazioni:

    1. Campi di origine dati predefiniti: seleziona dai campi dell'origine dati predefinitiAmazon Kendra generati che desideri mappare nell'indice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. Puoi inoltre scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta correttamente.

API

Per connettersiAmazon Kendra aAmazon WorkDocs

È necessario specificare quanto segue utilizzando l'WorkDocsConfigurationAPI:

  • Amazon WorkDocsID directory: specifica l'ID dell'organizzazione dellaAmazon WorkDocs directory. Puoi trovare l'ID organizzazione in AWS Directory Service accedendo ad Active Directory e poi in Directory.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere alla WorkDocs directory e per chiamare le API pubbliche richieste per il WorkDocs connettore eAmazon Kendra. Per ulteriori informazioni, consulta Ruoli IAM per le sorgenti WorkDocs dati.

È inoltre possibile aggiungere le seguenti funzionalità opzionali:

  • Registro delle modifiche: seAmazon Kendra deve utilizzare il meccanismo del registro delle modifiche all'origine WorkDocs dei dati per determinare se un documento deve essere aggiunto, aggiornato o eliminato nell'indice.

    Nota

    Usa il registro delle modifiche se non desideriAmazon Kendra scansionare tutti i documenti. Se il registro delle modifiche è di grandi dimensioni, la scansione dei documenti nella fonte di WorkDocs dati potrebbe richiedereAmazon Kendra meno tempo rispetto all'elaborazione del registro delle modifiche. Se stai sincronizzando la tua fonte di WorkDocs dati con il tuo indice per la prima volta, tutti i documenti vengono scansionati.

  • Filtri di inclusione ed esclusione: specificare se includere o escludere determinati documenti e commenti ai documenti. Ogni commento viene indicizzato come documento separato.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine WorkDocs dei dati ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati della ricerca in base all'accesso dell'utente o del suo gruppo ai documenti. Per ulteriori informazioni, consulta Filtraggio del contesto utente per le fonti di WorkDocs dati.
