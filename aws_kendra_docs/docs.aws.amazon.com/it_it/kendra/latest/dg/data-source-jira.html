Jira

Jira è uno strumento di gestione dei progetti per lo sviluppo di software, la gestione dei prodotti e il tracciamento dei bug. Puoi utilizzarloAmazon Kendra per indicizzare progetti, problemi, commenti, allegati, registri di lavoro e stati di Jira.

Amazon Kendraattualmente supporta solo Jira Cloud.

Puoi connettertiAmazon Kendra alla tua fonte di dati Jira utilizzando la Amazon Kendraconsole o l'JiraConfiguration API. Per un elenco delle funzionalità supportate da ciascuna, vedereFunzionalità supportate.

Per la risoluzione dei problemi del connettore di origine datiAmazon Kendra Jira, consultaRisoluzione dei problemi delle origini dati.

Funzionalità supportate

Amazon KendraIl connettore di origine dati Jira supporta le seguenti funzionalità:

  • Log delle modifiche

  • Mappature di campi

  • Filtraggio del contesto utente

  • Filtri di inclusione/esclusione

  • Virtual Private Cloud (VPC) (Cloud privato virtuale (VPC)

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la tua fonte di dati Jira, apporta queste modifiche in Jira e nei tuoiAWS account.

In Jira, assicurati di disporre di:

  • Creazione di credenziali di autenticazione tramite token API Jira che includono un ID Jira (nome utente o email) e una credenziale Jira (token Jira API). Consulta la documentazione di Atlassian sulla gestione dei token API.

  • Prendi nota dell'URL dell'account Jira nelle impostazioni del tuo account Jira. Ad esempio, company.atlassian.net.

  • Verifica che ogni documento sia unico in Jira e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le tue credenziali di autenticazione Jira in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti la tua fonte di dati Jira aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

Istruzioni di connessione

Per connettertiAmazon Kendra alla tua fonte dati Jira, devi fornire i dettagli necessari della tua fonte dati Jira in modo cheAmazon Kendra possa accedere ai tuoi dati. Se non hai ancora configurato Jira,Amazon Kendra guardaPrerequisiti.

Console

Per connettersiAmazon Kendra a Jira

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli Jira connector, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica dei dettagli dell'origine dati, inserisci le seguenti:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. Puoi includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per l'origine dei dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. In Etichette, per Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le seguenti:

    1. URL dell'account Jira: inserisci l'URL del tuo account Jira.

    2. AWSSecrets Managersegreto: scegli un segreto esistente o creane uno nuovoSecrets Manager per archiviare le tue credenziali di autenticazione Jira. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendra-Jira-' viene aggiunto automaticamente al tuo nome segreto.

        2. Per Jira ID: inserisci il nome utente o l'email di Jira.

        3. Per Password/Token: inserisci il token API Jira che hai creato dal tuo account Jira.

      2. Seleziona Salva.

    3. Virtual Private Cloud (VPC): è possibile scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    4. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    5. Seleziona Successivo.

  7. Nella pagina Configura impostazioni di sincronizzazione, inserisci le seguenti:

    1. Seleziona i progetti Jira da indicizzare: le entità o i tipi di contenuto Jira che desideri sottoporre a scansione.

    2. Stati, elementi aggiuntivi e tipi di problema: seleziona il contenuto per affinare l'ambito dell'indice.

    3. Registro delle modifiche: seleziona questa opzione per aggiornare l'indice anziché sincronizzare tutti i file.

    4. Schemi Regex: modelli di espressioni regolari per includere o escludere determinati file. Puoi aggiungere fino a 100.

    5. In Sincronizza la pianificazione dell'esecuzione, per Frequenza: scegli la frequenza con cui Amazon Kendra si sincronizzerà con la tua fonte di dati.

    6. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi, inserisci le seguenti:

    1. Per Progetto, Numero, Commento, Allegato, Registro di lavoro: seleziona dai campi di origine dati predefinitiAmazon Kendra generati che desideri mappare nell'indice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È inoltre possibile scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta correttamente.

API

Per connettersiAmazon Kendra a Jira

È necessario specificare quanto segue utilizzando l'JiraConfigurationAPI:

  • URL della fonte dei dati: specifica l'URL del tuo account Jira. Ad esempio, company.atlassian.net.

  • Nome di risorsa Amazon (ARN) segreto: fornisci il nome Amazon Resource Name (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo account Jira. Il segreto è archiviato in una struttura JSON con le seguenti:

    {
    "jiraId": "Jira user name or email",
    "jiraCredential": "Jira API token"
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore Jira eAmazon Kendra. Per ulteriori informazioni consulta Ruoli IAM per le fonti di dati Jira.

Puoi inoltre aggiungere le seguenti:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration come parte della configurazione dell'origine dei dati. Vedi ConfigurazioneAmazon Kendra per l'uso di un VPC.

  • Registro delle modifiche: se èAmazon Kendra necessario utilizzare il meccanismo del registro delle modifiche all'origine dei dati di Jira per determinare se un documento deve essere aggiunto, aggiornato o eliminato nell'indice.

    Nota

    Usa il registro delle modifiche se non desideriAmazon Kendra scansionare tutti i documenti. Se il registro delle modifiche è di grandi dimensioni, la scansione dei documenti nella fonte dati Jira potrebbe richiedereAmazon Kendra meno tempo rispetto all'elaborazione del registro delle modifiche. Se stai sincronizzando la tua fonte di dati Jira con il tuo indice per la prima volta, tutti i documenti vengono scansionati.

  • Filtri di inclusione ed esclusione: è possibile specificare se includere o escludere determinati progetti, problemi, commenti, allegati, registri di lavoro e stati.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine dati Jira ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati della ricerca in base all'accesso dell'utente o del suo gruppo ai documenti. Per ulteriori informazioni, consulta Filtraggio del contesto utente per le fonti di dati Jira.
