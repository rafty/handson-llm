Suggerimenti di query

Amazon KendraI suggerimenti di ricerca possono aiutare gli utenti a digitare le query di ricerca più velocemente e a guidarle nella ricerca.

Amazon Kendrasuggerisce le query pertinenti per gli utenti in base alle query più comuni nella cronologia delle query o nel registro delle query. Amazon Kendrautilizza tutte le query ricercate dagli utenti e apprende da queste interrogazioni per fornire suggerimenti agli utenti. Amazon Kendrasuggerisce le domande più frequenti agli utenti quando iniziano a digitare la query. Amazon Kendrasuggerisce una query se il prefisso o i primi caratteri della query corrispondono a ciò che l'utente inizia a digitare come query.

Ad esempio, un utente inizia a digitare la query «eventi imminenti». Amazon Kendraha appreso dal registro delle interrogazioni che molti utenti hanno cercato più volte «eventi imminenti 2050". L'utente vede apparire i «prossimi eventi 2050" direttamente sotto la barra di ricerca, completando automaticamente la query di ricerca. L'utente seleziona questo suggerimento di ricerca scegliendo il primo risultato della ricerca, che è il documento «Prossimi eventi: cosa sta succedendo nel 2050".

Puoi specificare comeAmazon Kendra selezionare le query idonee da suggerire ai tuoi utenti. Ad esempio, puoi specificare che un suggerimento di query deve essere stato cercato da almeno 10 utenti unici (il valore predefinito è 3), deve essere stato cercato negli ultimi 30 giorni e non contenere parole o frasi dall'elenco di blocco. Amazon Kendrarichiede che un'interrogazione abbia almeno un risultato di ricerca e contenga almeno una parola di più di quattro caratteri.

I suggerimenti di query non rispettano la distinzione tra maiuscole. Amazon Kendraconverte il prefisso della query e l'interrogazione suggerita in lettere minuscole, ignora tutte le virgolette singole e doppie e sostituisce più caratteri di spazio bianco con un solo spazio. Amazon Kendracorrisponde a tutti gli altri caratteri speciali così come sono. Amazon Kendranon mostra alcun suggerimento se un utente digita meno di due caratteri o più di 60 caratteri.

Puoi recuperare i suggerimenti di ricerca pertinenti per i tuoi utenti utilizzando l'GetQuerySuggestionsAPI. I suggerimenti di query vengono attivati per impostazione predefinita. Puoi disabilitare i suggerimenti di UpdateQuerySuggestionsConfigquery. Puoi testare le tue impostazioni prima di applicare suggerimenti all'applicazione di ricerca in due modi:

  • Utilizzando l'UpdateQuerySuggestionsConfigAPI.

  • Nella console nelle impostazioni dei suggerimenti delle interrogazioni.

Utilizzi l'GetQuerySuggestionsAPI per integrare i suggerimenti di query con l'applicazione della console in modo che gli utenti inizino a visualizzarli.

Impostazioni dei suggerimenti query

Nell'UpdateQuerySuggestionsConfigAPI puoi configurare le impostazioni seguenti:

  • Modalità: i suggerimenti di interrogazione possono essereENABLED oLEARN_ONLY. Amazon Kendraattiva i suggerimenti di interrogazione per impostazione predefinita. LEARN_ONLYdisattiva i suggerimenti di interrogazione. Se disattivato,Amazon Kendra continua a leggere i suggerimenti ma non fornisce suggerimenti di interrogazione agli utenti.

  • Finestra temporale del registro delle interrogazioni: quanto sono recenti le interrogazioni nella finestra temporale del registro delle interrogazioni. La finestra temporale è un valore intero per il numero di giorni dal giorno corrente ai giorni precedenti.

  • Interrogazioni senza informazioni sull'utente: impostateTRUE per includere tutte le interrogazioni o impostateFALSE per includere solo le interrogazioni con informazioni sull'utente. È possibile utilizzarlo se l'applicazione di ricerca include informazioni sull'utente, ad esempio l'ID utente, quando un utente invia una query. Per impostazione predefinita, questa impostazione non esclude le interrogazioni se non sono associate informazioni utente specifiche. Tuttavia, puoi utilizzare questa impostazione solo per dare suggerimenti basati su interrogazioni che includono informazioni sull'utente.

  • Utenti unici: il numero minimo di utenti univoci che devono cercare una query per poterla suggerire agli utenti. Questo numero è un valore intero.

  • Numero di interrogazioni: il numero minimo di volte in cui una query deve essere cercata affinché la query possa essere suggerita agli utenti. Questo numero è un valore intero.

Queste impostazioni influiscono sul modo in cui le interrogazioni vengono selezionate come ricerche popolari da suggerire agli utenti. Il modo in cui ottimizzi le impostazioni dipenderà dalle tue esigenze specifiche, ad esempio:

  • Se gli utenti effettuano in media una ricerca al mese, puoi impostare il numero di giorni nella finestra temporale del registro delle query su 30, in modo da acquisire la maggior parte delle query recenti degli utenti prima che diventino obsolete nella finestra temporale.

  • Se solo un numero limitato di domande include informazioni sugli utenti e non desideri suggerire query basate su un campione di dimensioni ridotte, puoi impostare le query in modo che includano tutti gli utenti.

  • Se definisci le query più comuni come quelle ricercate da almeno 10 utenti unici e cercate almeno 100 volte, impostate gli utenti unici su 10 e il conteggio delle query su 100.

Le modifiche alle impostazioni potrebbero non avere effetto immediato. Puoi tenere traccia delle modifiche alle impostazioni utilizzando l'DescribeQuerySuggestionsConfigAPI. Il tempo necessario per rendere effettive le impostazioni aggiornate dipende dagli aggiornamenti effettuati e dal numero di query di ricerca nell'indice.

aws kendra get-query-suggestions \
--index-id index-id \
--query-text "query-text" \
--max-suggestions-count 1 // If you want to limit the number of suggestions.

Per aggiornare i suggerimenti di interrogazione

Ad esempio, per modificare la finestra temporale del registro delle interrogazioni e il numero minimo di volte in cui una query deve essere cercata:

aws kendra update-query-suggestions-config \
--index-id index-id \
--query-log-look-back-window-in-days 30 \
--minimum-query-count 100

Python

Per recuperare i suggerimenti per le interrogazioni

import boto3
from botocore.exceptions import ClientError

kendra = boto3.client("kendra")

print("Get query suggestions.")

# Provide the index ID
index_id = "index-id"

# Provide the query text
query_text = "query"

try:
    query_suggestions_response = kendra.get_query_suggestions(
        IndexId = index_id,
        QueryText = query_text,
        MaxSuggestionsCount = 5
    )

    # Print out the suggestions you received
    if ("Suggestions" in query_suggestions_response.keys()) {
        for (suggestion: query_suggestions_response["Suggestions"]) {
            print(suggestion["Value"]["Text"]["Text"]);
        }
    }
   
except ClientError as e:
        print("%s" % e)

print("Program ends.")

Ad esempio, per modificare la finestra temporale del registro delle interrogazioni e il numero minimo di volte in cui una query deve essere cercata:

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Updating query suggestions settings/configuration for an index.")

# Provide the index ID
index_id = "index-id"

# Configure the settings you want to update
query_log_look_back_window_in_days = 30
minimum_query_count = 100

try:
    kendra.update_query_suggestions_config(
        IndexId = index_id,
        QueryLogLookBackWindowInDays = query_log_look_back_window_in_days,
        MinimumQueryCount = minimum_query_count
    )

    print("Wait for Amazon Kendra to update the query suggestions.")

    while True:
        # Get query suggestions description of settings/configuration
        query_sugg_config_response = kendra.describe_query_suggestions_config(
            IndexId = index_id
        )
        
        # If status is not UPDATING, then quit
        status = query_sugg_config_response["Status"]
        print(" Updating query suggestions config. Status: " + status)
        if status != "UPDATING":
            break
        time.sleep(60)

except ClientError as e:
        print("%s" % e)

print("Program ends.")

Puoi controllare le tue impostazioni correnti utilizzando l'DescribeQuerySuggestionsConfigAPI. Inoltre, questa operazione mostra le seguenti informazioni sui suggerimenti di interrogazione per un indice:

  • Modalità: i suggerimenti di interrogazione possono essereENABLED oLEARN_ONLY. Amazon Kendraattiva i suggerimenti di interrogazione per impostazione predefinita. LEARN_ONLYdisattiva i suggerimenti di interrogazione. È possibile modificare la modalità utilizzando l'UpdateQuerySuggestionsConfigAPI. Se disattivato,Amazon Kendra continua a leggere i suggerimenti ma non fornisce suggerimenti di interrogazione agli utenti.

  • Stato: i suggerimenti per le interrogazioni possono essereACTIVE oUPDATING.

  • Conteggio dei suggerimenti: il numero totale di domande pronte per essere suggerite agli utenti. Se il conteggio è molto inferiore a quello previsto, potrebbe essere perchéAmazon Kendra ha bisogno di più interrogazioni da cui imparare o che le impostazioni attuali dei suggerimenti per le interrogazioni sono troppo rigide.

  • Data dell'ultima creazione dei suggerimenti: l'ultima volta che i suggerimenti sono stati aggiornati. Amazon Kendraaggiorna automaticamente i suggerimenti ogni 24 ore o quando si modifica un'impostazione o quando si applica un elenco di blocchi.

  • Ultima ora di cancellazione dei suggerimenti: l'ultima volta che i suggerimenti sono stati cancellati.

Blocca determinate domande dai suggerimenti

Un elenco di blocchiAmazon Kendra impedisce di suggerire determinate domande popolari ai tuoi utenti. È un elenco di parole o frasi che desideri escludere dai suggerimenti di interrogazione. Amazon Kendraesclude le interrogazioni che contengono una corrispondenza esatta delle parole o delle frasi nell'elenco di blocco.

È possibile utilizzare un elenco di blocco per proteggersi da parole o frasi offensive che appaiono comunemente nel registro delle query e cheAmazon Kendra potrebbero essere selezionate come suggerimenti. Un elenco bloccato può ancheAmazon Kendra impedire di suggerire query popolari che contengono informazioni che non sono pronte per essere rilasciate o annunciate pubblicamente. Ad esempio, se gli utenti di solito chiedono informazioni su una versione di un nuovo prodotto che non vuoi suggerire perché non sei pronto a rilasciare presto, puoi bloccare dai suggerimenti le query che contengono il nome del prodotto e le informazioni sul prodotto.

È possibile creare un elenco di blocchi per le interrogazioni utilizzando l'CreateQuerySuggestionsBlockListAPI. Inserisci ogni parola o frase in blocco su una riga separata in un file di testo. Quindi carichi il file di testo nel tuo bucket S3 e fornisci il percorso o la posizione del file inAmazon S3. Amazon Kendraattualmente supporta la creazione di un solo elenco di blocchi.

Puoi sostituire il file di testo delle parole e delle frasi bloccate nel tuoAmazon S3 bucket e utilizzare l'UpdateQuerySuggestionsBlockListAPI per aggiornare l'elenco dei blocchi inAmazon Kendra.

Puoi utilizzare l'DescribeQuerySuggestionsBlockListAPI per ottenere lo stato del tuo elenco di blocco e altre informazioni utili, ad esempio quando è stato aggiornato l'ultima volta, quante parole o frasi sono presenti nell'elenco di blocco corrente e utili messaggi di errore durante la creazione di un elenco di blocchi. Puoi anche utilizzare l'ListQuerySuggestionsBlockListsAPI per ottenere un elenco di riepiloghi di elenchi di blocchi per un indice.

Per eliminare l'elenco dei blocchi, utilizza l'DeleteQuerySuggestionsBlockListAPI.

Gli aggiornamenti alla lista di blocco potrebbero non avere effetto immediato. Puoi tenere traccia degli aggiornamenti utilizzando l'DescribeQuerySuggestionsBlockListAPI.

Python

Per creare un elenco di blocchi

import boto3
from botocore.exceptions import ClientError
import pprint
import time

kendra = boto3.client("kendra")

print("Create a query suggestions block list.")

# Provide a name for the block list
block_list_name = "block-list-name"
# Provide an optional description for the block list
block_list_description = "block-list-description"
# Provide the IAM role ARN required for query suggestions block lists
block_list_role_arn = "role-arn"

# Provide the index ID
index_id = "index-id"

s3_bucket_name = "bucket-name"
s3_key = "query-suggestions/block_list.txt"
source_s3_path = {
    'Bucket': s3_bucket_name,
    'Key': s3_key
}

try:
    block_list_response = kendra.create_query_suggestions_block_list(
        Description = block_list_description,
        Name = block_list_name,
        RoleArn = block_list_role_arn,
        IndexId = index_id,
        SourceS3Path = source_s3_path
    )

    print(block_list_response)

    block_list_id = block_list_response["Id"]

    print("Wait for Amazon Kendra to create the block list.")

    while True:
        # Get block list description
        block_list_description = kendra.describe_query_suggestions_block_list(
            Id = block_list_id,
            IndexId = index_id
        )
        # If status is not CREATING, then quit
        status = block_list_description["Status"]
        print("Creating block list. Status: " + status)
        if status != "CREATING":
            break
        time.sleep(60)
        
except ClientError as e:
        print("%s" % e)

print("Program ends.")

Suggerimenti chiari

Puoi cancellare i suggerimenti di interrogazione utilizzando l'ClearQuerySuggestionsAPI. La cancellazione dei suggerimenti elimina solo i suggerimenti di interrogazione esistenti, non le query nel registro delle interrogazioni. Dopo aver eliminato i suggerimenti,Amazon Kendra apprende nuovi suggerimenti in base alle nuove interrogazioni aggiunte al registro delle query dal momento in cui hai eliminato i suggerimenti.

Nessun suggerimento disponibile

Se non visualizzi i suggerimenti per una query, potrebbe essere per uno dei seguenti motivi:

  • Non ci sono abbastanza domande nel tuo indiceAmazon Kendra da cui imparare.

  • Le impostazioni dei suggerimenti delle interrogazioni sono troppo rigide, per cui la maggior parte delle query viene filtrata dai suggerimenti.

  • Di recente hai eliminato i suggerimenti e haiAmazon Kendra ancora bisogno di tempo per accumulare nuove domande per apprendere nuovi suggerimenti.

Puoi controllare le tue impostazioni correnti utilizzando l'DescribeQuerySuggestionsConfigAPI.
