Parametri della richiesta

Per informazioni sui parametri comuni a tutte le azioni, vedere Parametri comuni.

La richiesta accetta i seguenti dati in formato JSON.

CapacityUnits

Imposta il numero di unità aggiuntive di archiviazione dei documenti e di capacità di interrogazione che devono essere utilizzate dall'indice. Puoi modificare la capacità dell'indice fino a 5 volte al giorno o effettuare 5 chiamate API.

Se si utilizzano unità di archiviazione aggiuntive, non è possibile ridurre la capacità di archiviazione al di sotto di quella necessaria per soddisfare le esigenze di archiviazione dell'indice.

Tipo: CapacityUnitsConfiguration oggetto

Required: No

Description

Una nuova descrizione per l'indice.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 0. Lunghezza massima di 1000.

Modello: ^\P{C}*$

Required: No

DocumentMetadataConfigurationUpdates

La configurazione dei metadati del documento che desideri aggiornare per l'indice. I metadati dei documenti sono campi o attributi associati ai tuoi documenti. Ad esempio, il nome del reparto aziendale associato a ciascun documento.

Tipo: matrice di oggetti DocumentMetadataConfiguration

Membri dell'array: numero minimo di 0 elementi. Numero massimo di 500 elementi.

Required: No

Id

L'identificatore dell'indice da aggiornare.

Tipo: String

Limitazioni di lunghezza: lunghezza fissa di 36.

Pattern: [a-zA-Z0-9][a-zA-Z0-9-]*

Campo obbligatorio: sì

Name

Il nome dell'indice da aggiornare.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 1. Lunghezza massima di 1000.

Modello: [a-zA-Z0-9][a-zA-Z0-9_-]*

Required: No

RoleArn

Un ruoloAWS Identity and Access Management (IAM) che fornisce ad Amazon Kendra le autorizzazioni per accedere ad Amazon CloudWatch logs e ai parametri.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 0. La lunghezza massima è 1284 caratteri.

Modello: arn:[a-z0-9-\.]{1,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[a-z0-9-\.]{0,63}:[^/].{0,1023}

Required: No

UserContextPolicy

La policy del contesto utente.

Tipo: String

Valori validi: ATTRIBUTE_FILTER | USER_TOKEN

Required: No

UserGroupResolutionConfiguration

Consente di recuperare i livelli di accesso di gruppi e utenti da un'origine diAWS IAM Identity Center (successor to AWS Single Sign-On) identità. Per configurarlo, consulta UserGroupResolutionConfiguration.

Tipo: UserGroupResolutionConfiguration oggetto

Required: No

UserTokenConfigurations

La configurazione del token utente.

Tipo: matrice di oggetti UserTokenConfiguration

Membri dell'array: numero massimo di 1 elementi.

Required: No

Elementi di risposta

Se l'operazione riesce, il servizio invia una risposta HTTP 200 con un corpo HTTP vuoto.

Errori

Per informazioni sugli errori comuni a tutte le operazioni, consultare Errori comuni.

AccessDeniedException

Non disponi delle autorizzazioni per eseguire questa azione. Assicurati di disporre delle politiche di autorizzazione e degli account utente richiesti e riprova.

Codice di stato HTTP: 400

ConflictException

Si è verificato un conflitto con la richiesta. Correggi eventuali incongruenze con le tue risorse e riprova.

Codice di stato HTTP: 400

InternalServerException

Si è verificato un problema con il server interno utilizzato per il servizio Amazon Kendra. Attendi qualche minuto e riprova oppure contatta l'Support per ricevere assistenza.

Codice di stato HTTP: 500

ResourceNotFoundException

La risorsa che desideri utilizzare non esiste. Verifica di aver fornito la risorsa corretta e riprova.

Codice di stato HTTP: 400

ServiceQuotaExceededException

Hai superato i limiti stabiliti per il tuo servizio Amazon Kendra. Consulta la sezione Quote per ulteriori informazioni o contatta l'Support per richiedere informazioni su un aumento dei limiti.

Codice di stato HTTP: 400

ThrottlingException

La richiesta è stata negata a causa del throttling della richiesta. Ridurre il numero di richieste e riprovare.

Codice di stato HTTP: 400

ValidationException

L'input non riesce a soddisfare i vincoli stabiliti dal servizio Amazon Kendra. Fornire l'input corretto e riprovare.

Codice di stato HTTP: 400
