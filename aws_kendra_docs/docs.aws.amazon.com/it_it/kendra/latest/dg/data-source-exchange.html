Microsoft Exchange

Microsoft Exchange è uno strumento di collaborazione aziendale per la messaggistica, le riunioni e la condivisione di file. Se sei un utente di Microsoft Exchange, puoi utilizzarloAmazon Kendra per indicizzare la tua origine dati di Microsoft Exchange.

Puoi connettertiAmazon Kendra alla tua fonte di dati Microsoft Exchange utilizzando la Amazon Kendraconsole e l'TemplateConfigurationAPI.

Per la risoluzione dei problemi relativi al connettore di origine dati diAmazon Kendra Microsoft Exchange, vedereRisoluzione dei problemi delle origini dati.

Funzionalità supportate

  • Mappature dei campi

  • Filtri di inclusione/esclusione

  • Virtual Private Cloud (VPC) (Cloud privato virtuale (VPC)

  • Sincronizza tutti i documenti/Sincronizza solo i documenti nuovi, modificati o eliminati

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare l'origine dati di Microsoft Exchange, apporta queste modifiche in Microsoft Exchange e negliAWS account.

In Microsoft Exchange, assicurati di disporre:

  • Ha creato un account Microsoft Exchange in Office 365.

  • Hai copiato il tuo ID tenant Microsoft 365. Puoi trovare il tuo ID tenant nelle Proprietà del tuo portale di Azure Active Directory. Vedi Trova il tuo ID tenant Microsoft 365 per maggiori dettagli.

  • È stato configurato un token di credenziali OAuth 2.0 contenente un ID client e un segreto client.

  • Sono state aggiunte le seguenti autorizzazioni per l'applicazione del connettore:

    • Calendari.Leggi

    • Posta. Leggi

    • Mail. ReadBasic

    • Mail. ReadBasic.Tutto

    • Utente.Leggi tutto

    • Contatti.Leggi

    • Elenco. Leggi tutto

    • Contatti.Leggi

    • Appunti. Leggi tutto

  • È stato verificato che ogni documento sia univoco in Microsoft Exchange e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le credenziali di autenticazione di Microsoft Exchange in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti l'origine dati di Microsoft Exchange aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

Istruzioni per il collegamento

Per connettersiAmazon Kendra all'origine dati Microsoft Exchange, è necessario fornire i dettagli necessari dell'origine dati Microsoft Exchange in modo cheAmazon Kendra possa accedere ai dati. Se non hai ancora configurato Microsoft Exchange,Amazon Kendra consultaPrerequisiti.

Console

Per connettersiAmazon Kendra a Microsoft Exchange

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli Connettore Microsoft Exchange, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica i dettagli dell'origine dati, inserisci le informazioni seguenti:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. È possibile includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. Nella sezione Etichette, nella sezione Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le informazioni seguenti:

    1. Fonte: inserisci il tuo ID tenant Microsoft 365. Puoi trovare il tuo ID tenant nelle Proprietà del tuo portale di Azure Active Directory.

    2. AWSSecrets Managersegreto: scegli un segreto esistente o creane uno nuovoSecrets Manager per archiviare le credenziali di autenticazione di Microsoft Exchange. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendra-Microsoft Exchange

        2. Per Client ID e Client secret: inserisci i valori delle credenziali di autenticazione che hai creato nel tuo account Microsoft Exchange.

      2. Seleziona Salva.

    3. Virtual Private Cloud (VPC): è possibile scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    4. Crawler di identità: scegli di attivare il crawler diAmazon Kendra identità per sincronizzare le informazioni sull'identità. Se scegli di disattivare il crawler di identità, devi caricare le informazioni principali utilizzando l'PutPrincipalMapping API.

    5. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    6. Seleziona Successivo.

  7. Nella pagina Configura impostazioni sincronizzazione, inserisci le informazioni seguenti:

    1. Sincronizzazione dei contenuti: seleziona i contenuti da sincronizzare.

    2. Configurazione aggiuntiva: è possibile utilizzare queste opzioni per indicizzare determinati contenuti anziché sincronizzare tutti i documenti.

    3. Modalità di sincronizzazione: puoi scegliere come aggiornare l'indice quando il contenuto della fonte dei dati cambia.

      1. Se scegli la sincronizzazione completa,Amazon Kendra sincronizzerà tutti i contenuti in tutte le entità, indipendentemente dallo stato di sincronizzazione precedente.

      2. Se scegli la sincronizzazione dei contenuti nuovi o modificati,Amazon Kendra sincronizzerà solo i contenuti nuovi o modificati.

      3. Se scegli la sincronizzazione dei contenuti nuovi, modificati o eliminati,Amazon Kendra sincronizzerà solo i contenuti nuovi, modificati o eliminati.

  8. Nella pagina Imposta mappature campi, inserisci le informazioni seguenti:

    1. Campi di origine dati predefiniti: seleziona dai campi dell'origine dati predefinitiAmazon Kendra generati che desideri mappare nell'indice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È inoltre possibile scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra a Microsoft Exchange

È necessario specificare un JSON dello schema di origine dati utilizzando l'TemplateConfigurationAPI. È necessario fornire le informazioni riportate di seguito:

  • Origine dati: è necessario specificare l'origine dei dati comeMSEXCHANGE.

  • ID tenant: puoi trovare il tuo ID tenant nelle proprietà del tuo portale di Azure Active Directory.

  • Tipo: specificaTEMPLATE come Tipo quando chiamiCreateDataSource.

  • Nome segreto Amazon Resource Name (ARN): fornisci l'Amazon Resource Name (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo account Microsoft Exchange. Il segreto è memorizzato in una struttura JSON con le seguenti chiavi:

    {
    "client Id": "client ID",
    "client Secret": "client Secret",
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore Microsoft Exchange eAmazon Kendra. Per ulteriori informazioni, consulta Ruoli IAM per le origini dati Microsoft Exchange.

È inoltre possibile aggiungere le seguenti funzionalità opzionali:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration quando effettuare la chiamataCreateDataSource. Consultare ConfigurazioneAmazon Kendra per l'utilizzo di un VPC.

  • Abilita il crawler di identità: specifica se attivare il crawler di identità. Se il crawler di identità è disattivato, è necessario caricare le informazioni principali utilizzando l'PutPrincipalMappingAPI. La scansione delle informazioni sull'identità di utenti e gruppi con accesso a determinati documenti è utile per filtrare il contesto degli utenti. I risultati della ricerca vengono filtrati in base all'accesso dell'utente o del suo gruppo ai documenti. Per ulteriori informazioni, vedere Filtro nel contesto utente.

  • Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine dati di Microsoft Exchange ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.
