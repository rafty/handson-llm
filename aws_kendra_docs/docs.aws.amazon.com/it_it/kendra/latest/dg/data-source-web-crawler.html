Amazon KendraWeb crawler

È possibile utilizzareAmazon Kendra Web Crawler per scansionare e indicizzare le pagine Web.

È possibile eseguire la ricerca per indicizzazione solo dei siti Web pubblici e dei siti Web che utilizzano il protocollo di comunicazione sicuro Hypertext Transfer Protocol Secure (HTTPS). Se ricevi un errore durante la ricerca per indicizzazione di un sito Web, è possibile che il sito Web non sia in grado di eseguire la ricerca per indicizzazione. Per eseguire la scansione dei siti Web interni, puoi configurare un proxy web. Il proxy web deve essere rivolto al pubblico.

Quando selezioni i siti Web da indicizzare, devi rispettare la Policy di uso accettabile di Amazon e tutti gli altri termini di Amazon. Ricorda che puoi utilizzareAmazon Kendra Web Crawler solo per indicizzare pagine Web di tua proprietà o che sei autorizzato a indicizzare. Per sapere come impedire aAmazon Kendra Web Crawler di indicizzare i tuoi siti Web, consultaConfigurazione delrobots.txt file perAmazon Kendra Web Crawler.

Puoi connettertiAmazon Kendra alla fonte di dati del tuo web crawler utilizzando la Amazon Kendraconsole e l'WebCrawlerConfigurationAPI.

Per la risoluzione dei problemi relativi al connettore di origine dati delAmazon Kendra web crawler, consultaRisoluzione dei problemi delle origini dati.

Prerequisiti

Prima di poterli utilizzareAmazon Kendra per indicizzare i tuoi siti Web, controlla i dettagli dei tuoi siti Web e dei tuoiAWS account.

Per i siti Web, assicurati di disporre di:

  • Hai copiato gli URL di origine o mappa del sito dei siti Web che desideri indicizzare.

  • Per i siti Web che richiedono l'autenticazione di base per la scansione: sono stati copiati il nome host del sito Web e il numero di porta.

  • Facoltativo: hai copiato il nome host del sito Web e il numero di porta se desideri utilizzare un proxy Web per connetterti ai siti Web interni che desideri sottoporre a scansione. Il proxy web deve essere rivolto al pubblico. Amazon Kendrasupporta la connessione a server proxy Web supportati dall'autenticazione di base oppure è possibile connettersi senza autenticazione.

  • Hai controllato che ogni documento della pagina Web che desideri indicizzare sia unico e che si tratti di altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le credenziali di autenticazione del tuo web crawler in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti la fonte di dati del tuo web crawler aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

Console

Per connettersiAmazon Kendra al web crawler

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli il connettore del web crawler, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica dei dettagli dell'origine dati, inserisci le informazioni seguenti:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. È possibile includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. In Etichette, in Aggiungi nuovo tag —Tags per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le informazioni seguenti:

    1. Per Source, scegli tra URL di origine e Sitemap di origine a seconda del caso d'uso e inserisci i valori per ciascuno di essi.

      Nota

      È possibile aggiungere fino a 10 URL di origine e 3 mappa del sito.

      Se desideri eseguire la scansione di una mappa del sito, è importante verificare se l'URL di base o principale è lo stesso URL di base utilizzato per gli URL elencati nella pagina della mappa del sito. Ad esempio, se l'URL della mappa del sito è https://example.com/sitemap-page.html, anche gli URL elencati in questa pagina della mappa del sito devono utilizzare l'URL di base "https://example.com/».

    2. (Facoltativo) Per il proxy Web: inserisci le seguenti informazioni:

      1. Nome host: il nome host in cui è richiesto il proxy Web.

      2. Numero di porta: la porta utilizzata dal protocollo di trasporto dell'URL host. Il numero di porta deve essere un valore numerico compreso tra 0 e 65535.

      3. Per le credenziali del proxy Web: scegli un segreto esistente o creane uno nuovo per archiviare le credenziali di autenticazione del web crawler. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      4. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendraWebCrawler-' viene aggiunto automaticamente al tuo nome segreto.

        2. Per Nome utente e password: inserisci queste credenziali di autenticazione di base per i tuoi siti web.

        3. Seleziona Salva.

    3. (Facoltativo) Host con autenticazione: selezionare questa opzione per aggiungere altri host con autenticazione.

    4. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    5. Seleziona Successivo.

  7. Nella pagina Configura impostazioni di sincronizzazione, inserisci le informazioni seguenti:

    1. Intervallo di esplorazione: scegli il tipo di pagine Web da sottoporre a scansione.

    2. Profondità di scansione: seleziona il numero di livelli dall'URL di origine cheAmazon Kendra devono essere esplorati.

    3. Le impostazioni di esplorazione avanzate e la configurazione aggiuntiva immettono le seguenti informazioni:

      1. Dimensione massima del file: la dimensione massima della pagina Web o degli allegati per la ricerca per indicizzazione. Minimo 0,000001 MB (1 byte). Massimo 50 MB.

      2. Numero massimo di link per pagina: il numero massimo di link sottoposti a scansione per pagina. I link vengono sottoposti a ricerca per indicizzazione in ordine di visualizzazione. Minimo 1 link/pagina. Massimo 1000 link/pagina.

      3. Limitazione massima: il numero massimo di URL di cui viene eseguita la ricerca per indicizzazione per nome host al minuto. Minimo 1 URL/nome host/minuto. Massimo 300 URL/nome host/minuto.

      4. Schemi Regex: aggiungi un pattern per includere solo determinati URL o per escludere URL, tipi di file o file specifici nel tuo repository. Puoi avere un totale combinato di 100 pattern.

    4. In Sincronizza programma di esecuzione, per Frequenza: scegli la frequenzaAmazon Kendra con cui sincronizzarti con la tua fonte di dati.

    5. Seleziona Successivo.

  8. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È inoltre possibile scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra al web crawler

È necessario specificare la seguente WebCrawlerConfigurationAPI:

  • URL: specifica gli URL di origine o punto di partenza dei siti Web o gli URL di mappa del sito dei siti Web che desideri sottoporre a ricerca per indicizzazione SeedUrlConfiguratione SiteMapsConfiguration.

    Se desideri eseguire la scansione di una mappa del sito, è importante verificare se l'URL di base o principale è lo stesso URL di base utilizzato per gli URL elencati nella pagina della mappa del sito. Ad esempio, se l'URL della mappa del sito è https://example.com/sitemap-page.html, anche gli URL elencati in questa pagina della mappa del sito devono utilizzare l'URL di base "https://example.com/».

  • Nome segreto di Amazon Resource Name (ARN): se un sito Web richiede l'autenticazione di base, fornisci il nome host, il numero di porta e un segreto che memorizza le credenziali di autenticazione di base del nome utente e della password. Fornisci l'ARN utilizzando l'AuthenticationConfigurationAPI. Il segreto è memorizzato in una struttura JSON con le seguenti chiavi:

    {
    "username": "user name",
    "password": "password"
}

    Puoi fornire le credenziali del proxy web utilizzando un segretoAWSSecrets Manager segreto. Si utilizza l'ProxyConfigurationAPI per fornire il nome host e il numero di porta del sito Web e, facoltativamente, il segreto che memorizza le credenziali del proxy Web.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

È inoltre possibile aggiungere le seguenti funzioni facoltative:

  • Modalità di scansione: scegli di eseguire la scansione solo dei nomi host dei siti Web o dei nomi host con sottodomini o anche di eseguire la scansione di altri domini a cui le pagine Web rimandano.

  • La «profondità» o il numero di livelli di un sito Web, dal livello dell'origine fino a ricerca per indicizzazione. Ad esempio, se un sito Web ha 3 livelli (il livello dell'indice, ossia l'origine in questo esempio, il livello delle sezioni e livello delle sottosezioni) e ti interessa eseguire la ricerca per indicizzazione delle informazioni solo dal livello dell'indice al livello delle sezioni (livelli 0-1), puoi impostare la profondità su 1.

  • Il numero massimo di URL di una pagina Web di cui eseguire la ricerca per indicizzazione.

  • La dimensione massima in MB di una pagina Web da sottoporre a ricerca per indicizzazione.

  • Il numero massimo di URL di cui viene eseguita la ricerca per indicizzazione per host di sito Web al minuto.

  • Le informazioni del proxy web per connettersi e scansionare i siti Web interni.

  • Le informazioni di autenticazione per accedere e eseguire la ricerca per indicizzazione dei siti Web che richiedono l'autenticazione dell'utente.

  • Puoi estrarre i meta tag HTML come campi utilizzando lo strumento Custom Document Enrichment. Per ulteriori informazioni, consulta la sezione Personalizzazione dei metadati del documento durante il processo di importazione. Per un esempio di estrazione dei meta tag HTML, consulta gli esempi di CDE.

  • Filtri di inclusione ed esclusione: specifica se includere o escludere determinati URL.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.
