GitHub

GitHub è un servizio di hosting basato sul web per lo sviluppo di software che fornisce servizi di archiviazione e gestione del codice con controllo delle versioni. È possibile utilizzarliAmazon Kendra per indicizzare i file del repository GitHub GitHub Enterprise Cloud (SaaS) ed Enterprise Server (On Prem), le richieste di emissione e pull, i commenti delle richieste di emissione e estrazione e gli allegati dei commenti delle richieste di emissione e estrazione. È inoltre possibile scegliere di includere o escludere determinati file.

Puoi connettertiAmazon Kendra alla tua fonte di GitHub dati utilizzando la Amazon Kendraconsole e l'GitHubConfigurationAPI.

Per la risoluzione dei problemi relativi al connettore della sorgenteAmazon Kendra GitHub dati, consultaRisoluzione dei problemi delle origini dati.

Funzionalità supportate

Amazon Kendra GitHub il connettore data source supporta le seguenti funzionalità:

  • Log delle modifiche

  • Mappature di campi

  • Filtraggio del contesto utente

  • Filtri di inclusione/esclusione

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la fonte GitHub dei dati, apporta queste modifiche nei tuoiAWS account GitHub e.

Nel GitHub, assicurati di avere:

  • Ha creato un GitHub utente con autorizzazioni amministrative per l' GitHub organizzazione.

  • È stato creato un token di accesso personale per le credenziali di autenticazione. Consulta GitHub la documentazione sulla creazione di un token di accesso personale.

  • Consigliato: creato un token OAuth per le credenziali di autenticazione. Usa il token OAuth per migliorare i limiti di accelerazione delle API e le prestazioni dei connettori. Vedi GitHub la documentazione sull'autorizzazione OAuth.

  • Facoltativo: è stato installato un certificato SSL.

  • Hai notato l'URL dell' GitHub host per il tipo di GitHub servizio che utilizzi. Ad esempio, l'URL dell'host per il GitHub cloud potrebbe essere https://api.github.com e l'URL dell'host per il GitHub server potrebbe essere https://on-prem-host-url/api/v3/.

  • Prendi nota del nome GitHub dell'organizzazione per i tuoi repository nelle tue GitHub impostazioni.

  • Sono state aggiunte le seguenti:

    Per GitHub Enterprise Cloud (SaaS)

    • repo: stato

    • repository pubblico

    • repo: invitare

    • leggi: org

    • utente:email

    • leggi:utente

    Per server GitHub aziendali (in locale)

    • repo: stato

    • repository pubblico

    • repo: invitare

    • leggi: org

    • utente:email

    • leggi:utente

    • amministratore del sito

  • È stato verificato che ogni documento sia univoco in GitHub altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le tue credenziali di GitHub autenticazione in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti la tua fonte di GitHub dati aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

Console

Per connettersiAmazon Kendra a GitHub

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli GitHub connettore, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica dei dettagli dell'origine dati, inserisci le seguenti:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. Puoi includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per l'origine dei dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. Nella sezione Tag, nella sezione Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le seguenti:

    1. GitHubfonte: scegli tra GitHub Enterprise Cloud ed GitHubEnterprise Server.

    2. GitHub URL host: inserisci il nome GitHub dell'host.

    3. GitHub nome dell'organizzazione: inserisci il nome GitHub dell'organizzazione. Puoi trovare le informazioni della tua organizzazione nel tuo GitHub account.

    4. AWSSecrets Managersegreto: scegli un segreto esistente o creane uno nuovoSecrets Manager per archiviare le tue credenziali di GitHub autenticazione. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendraGitHub-' viene aggiunto automaticamente al tuo nome segreto.

        2. Per il GitHubtoken: inserisci i valori delle credenziali di autenticazione che hai creato nel GitHub tuo account.

      2. Seleziona Salva.

    5. Virtual Private Cloud (VPC): puoi scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    6. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    7. Seleziona Successivo.

  7. Nella pagina Configura delle impostazioni di sincronizzazione, inserisci le seguenti:

    1. Seleziona i repository da sottoporre a ricerca per indicizzazione: le GitHub entità o i tipi di contenuto che desideri sottoporre a ricerca per indicizzazione.

    2. Registro delle modifiche: seleziona questa opzione per aggiornare l'indice anziché sincronizzare tutti i file.

    3. Tipi di contenuto: seleziona i tipi di file che desideri includere.

    4. Schemi Regex: modelli di espressioni regolari per includere o escludere determinati file. È possibile aggiungere fino a 100 pattern.

    5. In Sincronizza la pianificazione dell'esecuzione, per Frequenza: con quale frequenza Amazon Kendra si sincronizzerà con la tua fonte di dati.

    6. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi, inserisci le seguenti:

    1. Per Repository, Respository Commit, Issue Document, Issue Comment, Issue Attachment, Pull Request Comment, Pull Request Attachment: seleziona uno dei campi di origine dati predefinitiAmazon Kendra generati che desideri mappare all'indice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È possibile anche scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra a GitHub

È necessario specificare quanto segue utilizzando l'oggetto GitHubConfigurationAPI:

  • Tipo di origine dati: specificare il tipo di origine dati comeSAAS oON_PREMISE.

  • Nome segreto Amazon Resource Name (ARN): fornisci il nome Amazon Resource Name (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo GitHub account. Il segreto è memorizzato in una struttura JSON con le seguenti:

    {
    "personalToken": "token"
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il GitHub connettore eAmazon Kendra. Per ulteriori informazioni consulta Ruoli IAM per le fonti di GitHub dati.

È inoltre possibile aggiungere le seguenti funzionalità opzionali:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration come parte della configurazione dell'origine dei dati. Vedi ConfigurazioneAmazon Kendra per l'uso di un VPC.

    Nota

    Se usi il GitHub server, devi usare unAmazon VPC per connetterti al tuo GitHub server.

  • Registro delle modifiche: seAmazon Kendra deve utilizzare il meccanismo del registro delle modifiche all'origine GitHub dei dati per determinare se un documento deve essere aggiunto, aggiornato o eliminato nell'indice.

    Nota

    Usa il registro delle modifiche se non desideriAmazon Kendra scansionare tutti i documenti. Se il registro delle modifiche è di grandi dimensioni, la scansione dei documenti nella fonte di GitHub dati potrebbe richiedereAmazon Kendra meno tempo rispetto all'elaborazione del registro delle modifiche. Se stai sincronizzando la tua fonte di GitHub dati con il tuo indice per la prima volta, tutti i documenti vengono scansionati.

  • Filtri di inclusione ed esclusione: specificare se includere o escludere determinati file.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine GitHub dei dati ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati della ricerca in base all'accesso dell'utente o del suo gruppo ai documenti. Per ulteriori informazioni, consulta Filtraggio del contesto utente per le fonti di GitHub dati.
