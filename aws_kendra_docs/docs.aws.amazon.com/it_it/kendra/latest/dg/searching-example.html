Interrogazione di un indice

Quando esegui una ricerca nell'indice,Amazon Kendra utilizza tutte le informazioni che hai fornito sui tuoi documenti per determinare i documenti più pertinenti ai termini di ricerca inseriti. Alcuni degli elementi presi inAmazon Kendra considerazione sono:

  • Il testo del documento.

  • Il titolo del documento.

  • Campi di testo personalizzati che hai contrassegnato come ricercabili.

  • Il campo data che hai indicato deve essere utilizzato per determinare la «freschezza» di un documento.

Quando un set di documenti pertinenti è stato selezionato dall'indice,Amazon Kendra filtra la risposta in base a qualsiasi filtro di attributo richiesto per la ricerca. Ad esempio, se disponi di un attributo personalizzato chiamato «reparto», puoi filtrare la risposta per restituire solo i documenti di un reparto chiamato «legale». Per ulteriori informazioni, consulta Campi o attributi personalizzati.

Dopo aver trovato i documenti pertinenti e aver filtrato in base agli attributi impostati,Amazon Kendra restituisce i risultati. I risultati vengono ordinati in base alla rilevanzaAmazon Kendra determinata per ogni documento. I risultati sono suddivisi in pagine in modo da poter mostrare una pagina alla volta all'utente.

Per cercare documenti conAmazon Kendra cui sei stato indicizzatoAmazon Lex, usa AMAZON. KendraSearchIntent. Per un esempio di configurazioneAmazon Kendra conAmazon Lex, vedi Creazione di un bot di domande frequenti per unAmazon Kendra indice.

Per informazioni sulle risposte alle interrogazioni, vedereRisposte alle interrogazioni e tipi di risposta.

Ricerca in un indice (console)

Puoi usare laAmazon Kendra console per cercare e testare il tuo indice. Puoi fare domande e vedere i risultati.

Per cercare un indice con la console

  1. Accedere aAWS Management Console e aprire laAmazon Kendra console all'indirizzo http://console.aws.amazon.com/kendra/.

  2. Nel pannello di navigazione, scegli Indici.

  3. Scegli il tuo indice.

  4. Nel menu di navigazione, scegliere l'opzione per la ricerca.

  5. Inserisci una ricerca nella casella di testo e premi invio.

  6. Amazon Kendrarestituisce i risultati della ricerca.

Ricerca in un indice (SDK)

Per cercare un indice con Python o Java

  • L'esempio seguente cerca un indice. Cambia il valore dellaquery tua queryindexId di ricercaindex_id e/o dell'identificatore dell'indice che desideri cercare.

    Python
    import boto3
import pprint

kendra = boto3.client("kendra")

# Provide the index ID
index_id = "index-id"
# Provide the query text
query = "search-string"

response = kendra.query(
        QueryText = query,
        IndexId = index_id)

print("\nSearch results for query: " + query + "\n")        

for query_result in response["ResultItems"]:

    print("-------------------")
    print("Type: " + str(query_result["Type"]))
        
    if query_result["Type"]=="ANSWER" or query_result["Type"]=="QUESTION_ANSWER":
        answer_text = query_result["DocumentExcerpt"]["Text"]
        print(answer_text)

    if query_result["Type"]=="DOCUMENT":
        if "DocumentTitle" in query_result:
            document_title = query_result["DocumentTitle"]["Text"]
            print("Title: " + document_title)
        document_text = query_result["DocumentExcerpt"]["Text"]
        print(document_text)

    print("------------------\n\n")
    Java
    package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.QueryRequest;
import software.amazon.awssdk.services.kendra.model.QueryResponse;
import software.amazon.awssdk.services.kendra.model.QueryResultItem;

public class SearchIndexExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String query = "some queries";
        String indexId = "anIndexId";

        QueryRequest queryRequest = QueryRequest
            .builder()
            .queryText(query)
            .indexId(indexId)
            .build();

        QueryResponse queryResponse = kendra.query(queryRequest);

        System.out.println(String.format("\nSearch results for query: %s", query));
        for(QueryResultItem item: queryResponse.resultItems()) {
            System.out.println("----------------------");
            System.out.println(String.format("Type: %s", item.type()));

            switch(item.type()) {
                case QUESTION_ANSWER:
                case ANSWER:
                    String answerText = item.documentExcerpt().text();
                    System.out.println(answerText);
                    break;
                case DOCUMENT:
                    String documentTitle = item.documentTitle().text();
                    System.out.println(String.format("Title: %s", documentTitle));
                    String documentExcerpt = item.documentExcerpt().text();
                    System.out.println(String.format("Excerpt: %s", documentExcerpt));
                    break;
                default:
                    System.out.println(String.format("Unknown query result type: %s", item.type()));

            }

            System.out.println("-----------------------\n");
        }
    }
}

Ricerca con sintassi di interrogazione avanzata

È possibile creare interrogazioni più specifiche rispetto alle semplici interrogazioni con parole chiave o in linguaggio naturale utilizzando la sintassi o gli operatori di query avanzati. Ciò include intervalli, valori booleani, jolly e altro ancora. Utilizzando gli operatori, puoi dare più contesto alla tua ricerca e perfezionare ulteriormente i risultati della ricerca.

Amazon Kendrasupporta i seguenti operatori.

  • Booleano: logica per limitare o ampliare la ricerca. Ad esempio,amazon AND sports limita la ricerca solo ai documenti contenenti entrambi i termini.

  • Parentesi: legge i termini delle interrogazioni annidati in ordine di precedenza. Ad esempio,(amazon AND sports) NOT rainforest legge(amazon AND sports) primaNOT rainforest.

  • Intervalli: valori di data o intervallo numerico. Gli intervalli possono essere inclusivi, esclusivi o illimitati. Ad esempio, puoi cercare i documenti che sono stati aggiornati l'ultima volta tra il 1° gennaio 2020 e il 31 dicembre 2020, comprese queste date.

  • Campi: utilizza un campo specifico per limitare la ricerca. Ad esempio, puoi cercare documenti con «Stati Uniti» nel campo «posizione».

  • Wildcards: corrispondono parzialmente a una stringa di testo. Ad esempio,Cloud* potrebbe corrispondere CloudFormation. Amazon Kendraattualmente supporta solo i caratteri jolly finali.

  • Citazioni esatte: corrisponde esattamente a una stringa di testo. Ad esempio, documenti che contengono"Amazon Kendra" "pricing".

È possibile utilizzare una combinazione di uno qualsiasi degli operatori sopra indicati.

Tieni presente che un uso eccessivo di operatori o interrogazioni altamente complesse potrebbe influire sulla latenza delle query. Le wildcard sono tra gli operatori più costosi in termini di latenza. Una regola generale è che maggiore è il numero di termini e operatori utilizzati, maggiore è il potenziale impatto sulla latenza. Altri fattori che influiscono sulla latenza includono la dimensione media dei documenti indicizzati, la dimensione dell'indice, l'eventuale filtraggio dei risultati di ricerca e il carico complessivo sull'Amazon Kendraindice.

Boolean

È possibile combinare o escludere parole utilizzando gli operatori booleaniAND,OR,NOT.

Di seguito sono riportati alcuni esempi di utilizzo degli operatori booleani.

amazon AND sports

Restituisce risultati di ricerca che contengono nel testo sia i termini «amazon» che «sport», ad esempio Amazon Prime video sport o altri contenuti simili.

sports OR recreation

Restituisce i risultati della ricerca che contengono i termini «sport» o «ricreazione», o entrambi, nel testo.

amazon NOT rainforest

Restituisce i risultati della ricerca che contengono il termine «amazzonia» ma non il termine «foresta pluviale» nel testo. Questo serve per cercare documenti sulla società Amazon, non sulla foresta pluviale amazzonica.

Parentesi

È possibile interrogare le parole annidate in ordine di precedenza utilizzando le parentesi. Le parentesi indicanoAmazon Kendra come deve essere letta un'interrogazione.

Di seguito sono riportati alcuni esempi di utilizzo degli operatori tra parentesi.

(amazon AND sports) NOT rainforest

Restituisce i documenti che contengono sia i termini «amazon» che «sport» nel testo, ma non il termine «foresta pluviale». Questo serve per cercare video sportivi su Amazon Prime o altri contenuti simili, non sport d'avventura nella foresta pluviale amazzonica. Le parentesi aiutano a indicare cheamazon AND sports deve essere letto primaNOT rainforest. L'interrogazione non deve essere letta comeamazon AND (sports NOT rainforest).

(amazon AND (sports OR recreation)) NOT rainforest

Restituisce i documenti che contengono i termini «sport» o «ricreazione», o entrambi, e il termine «amazon». Ma non include il termine «foresta pluviale». Questo serve per cercare video sportivi o ricreativi su Amazon Prime, non sport d'avventura nella foresta pluviale amazzonica. Le parentesi aiutano a indicare chesports OR recreation deve essere letto prima di combinarlo con 'amazon', che viene letto primaNOT rainforest. L'interrogazione non deve essere letta comeamazon AND (sports OR (recreation NOT rainforest)).

Intervalli

È possibile utilizzare un intervallo di valori per filtrare i risultati della ricerca. Si specifica un attributo e i valori dell'intervallo. Può essere un tipo di data o numerico.

Gli intervalli di date devono avere i seguenti formati:

  • Epoch

  • YYYY

  • YYYY-mm

  • aaaa-mm-gg

  • YYYY-MM-Dd'T'HH

È inoltre possibile specificare se includere o escludere i valori inferiori e superiori dell'intervallo.

Di seguito sono riportati alcuni esempi di utilizzo degli operatori di intervallo.

_processed_date:>2019-12-31 AND _processed_date:<2021-01-01

Restituisce i documenti che sono stati elaborati nel 2020, superiori al 31 dicembre 2019 e inferiori al 1° gennaio 2021.

_processed_date:>=2020-01-01 AND _processed_date:<=2020-12-31

Restituisce i documenti elaborati nel 2020, superiori o uguali al 1° gennaio 2020 e inferiori o uguali al 31 dicembre 2020.

_document_likes:<1

Restituisce documenti con zero Mi piace o nessun feedback degli utenti, ovvero meno di 1 like.

È possibile specificare se un intervallo deve essere considerato inclusivo o escluso dei valori dell'intervallo specificato.

Inclusivo

_last_updated_at:[2020-01-01 TO 2020-12-31]

I documenti di restituzione sono stati aggiornati l'ultima volta nel 2020, inclusi i giorni 1 dicembre 2020 e 31 dicembre 2020.

Esclusivo

_last_updated_at:{2019-12-31 TO 2021-01-01}

I documenti di restituzione sono stati aggiornati l'ultima volta nel 2020, esclusi i giorni 31 dicembre 2019 e 1° gennaio 2021.

Per intervalli illimitati che non sono né inclusivi né esclusivi, è sufficiente utilizzare gli < and > operatori. Ad esempio, _last_updated_at:>2019-12-31 AND _last_updated_at:<2021-01-01

Campi

Puoi limitare la ricerca solo ai documenti che soddisfano un valore in un campo specifico. Il campo può essere di qualsiasi tipo.

Di seguito sono riportati alcuni esempi di utilizzo di operatori di contesto a livello di campo.

status:"Incomplete" AND financial_year:2021

Restituisce i documenti per l'esercizio 2021 con lo stato di incompletezza.

(sports OR recreation) AND country:"United States" AND level:"professional"

Restituisce documenti che trattano di sport o attività ricreative professionistiche negli Stati Uniti.

Caratteri jolly

È possibile ampliare la ricerca per tenere conto delle varianti di parole e frasi utilizzando l'operatore wildcard. Ciò è utile quando si cercano varianti di nome. Amazon Kendraattualmente supporta solo i caratteri jolly finali. Il numero di caratteri di prefisso per un carattere jolly finale deve essere maggiore di due.

Di seguito sono riportati alcuni esempi di utilizzo degli operatori wildcard.

Cloud*

Restituisce i documenti che contengono varianti come CloudFormation e CloudWatch.

kendra*aws

Restituisce documenti che contengono varianti come kendra.amazonaws.

kendra*aws*

Restituisce documenti che contengono varianti come kendra.amazonaws.com

Quotazioni esatte

È possibile utilizzare le virgolette per cercare una corrispondenza esatta di una parte di testo.

Di seguito sono riportati alcuni esempi di utilizzo delle virgolette.

"Amazon Kendra" "pricing"

Restituisce i documenti che contengono sia la fraseAmazon Kendra «» che il termine «prezzi». I documenti devono contenere siaAmazon Kendra «» che «prezzi» per poter restituire i risultati.

"Amazon Kendra" "pricing" cost

Restituisce i documenti che contengono sia la fraseAmazon Kendra «» che il termine «prezzo» e, facoltativamente, il termine «costo». I documenti devono contenere siaAmazon Kendra «» che «prezzi» per restituire i risultati, ma potrebbero non includere necessariamente il «costo».

Sintassi query non valida

Amazon Kendraemette un avviso se ci sono problemi con la sintassi della query o se la query non è attualmente supportata daAmazon Kendra. Per ulteriori informazioni, consulta la documentazione dell'API per gli avvisi di query.

Le interrogazioni seguenti sono esempi di sintassi delle query non valida.

_last_updated_at:<2021-12-32

Data non valida. Il giorno 32 non esiste nel calendario gregoriano, utilizzato daAmazon Kendra.

_view_count:ten

Valore numerico non valido. Le cifre devono essere utilizzate per rappresentare valori numerici.

nonExistentField:123

Ricerca nel campo non valida. Il campo deve esistere per poter utilizzare la ricerca nei campi.

Product:[A TO D]

Intervallo non valido. I valori numerici o le date devono essere utilizzati per gli intervalli.

OR Hello

Elemento booleano non valido. Gli operatori devono essere utilizzati con i termini e collocati tra i termini.

Ricerca in lingue

Puoi cercare documenti in una lingua supportata. Inserisci il codice della lingua AttributeFilterper restituire i documenti filtrati nella lingua prescelta. È possibile digitare l'interrogazione in una lingua supportata.

Se non specifichi una lingua,Amazon Kendra query i documenti in inglese. Per ulteriori informazioni sulle lingue supportate, inclusi i relativi codici, consulta Aggiungere documenti in lingue diverse dall'inglese.

Per cercare documenti in una lingua supportata nella console, seleziona il tuo indice, quindi seleziona l'opzione per la ricerca nell'indice dal menu di navigazione. Scegli la lingua in cui desideri restituire i documenti selezionando le impostazioni di ricerca e quindi selezionando una lingua dal menu a discesa Lingua.

    package com.amazonaws.kendra;

import software.amazon.awssdk.services.kendra.KendraClient;
import software.amazon.awssdk.services.kendra.model.QueryRequest;
import software.amazon.awssdk.services.kendra.model.QueryResponse;
import software.amazon.awssdk.services.kendra.model.QueryResultItem;

public class SearchIndexExample {
    public static void main(String[] args) {
        KendraClient kendra = KendraClient.builder().build();

        String query = "searchString";
        String indexId = "indexID";

        QueryRequest queryRequest = QueryRequest.builder()
            .queryText(query)
            .indexId(indexId)
            .attributeFilter(
                 AttributeFilter.builder()
                     .withEqualsTo(
                         DocumentAttribute.builder()
                             .withKey("_language_code")
                             .withValue("es")
                             .build())
                     .build())
            .build();

        QueryResponse queryResponse = kendra.query(queryRequest);

        System.out.println(String.format("\nSearch results|
                                          Resultados de la búsqueda: %s", query));
        for(QueryResultItem item: queryResponse.resultItems()) {
            System.out.println("----------------------");
            System.out.println(String.format("Type: %s", item.type()));

            switch(item.type()) {
                case QUESTION_ANSWER:
                case ANSWER:
                    String answerText = item.documentExcerpt().text();
                    System.out.println(answerText);
                    break;
                case DOCUMENT:
                    String documentTitle = item.documentTitle().text();
                    System.out.println(String.format("Title: %s", documentTitle));
                    String documentExcerpt = item.documentExcerpt().text();
                    System.out.println(String.format("Excerpt: %s", documentExcerpt));
                    break;
                default:
                    System.out.println(String.format("Unknown query result type: %s", item.type()));

            }

            System.out.println("-----------------------\n");
        }
    }
}
