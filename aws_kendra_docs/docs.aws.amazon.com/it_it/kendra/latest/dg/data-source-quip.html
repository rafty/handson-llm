battuta

Quip è un software di produttività collaborativa che offre funzionalità di creazione di documenti in tempo reale. Puoi usarliAmazon Kendra per indicizzare le cartelle, i file, i commenti sui file, le chat e gli allegati di Quip.

Puoi connettertiAmazon Kendra alla tua fonte di dati Quip usando la Amazon Kendraconsole e l'QuipConfigurationAPI.

Per la risoluzione dei problemi relativi al connettore della sorgente datiAmazon Kendra Quip, consultaRisoluzione dei problemi delle origini dati.

Funzionalità supportate

Amazon KendraIl connettore di origine dati Quip supporta le seguenti funzionalità:

  • Mappature di campi

  • Filtraggio del contesto utente

  • Filtri di inclusione/esclusione

  • Virtual Private Cloud (VPC)

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la fonte dei dati di Quip, apporta queste modifiche nel tuo Quip e nei tuoiAWS account.

In Quip, assicurati di disporre di:

  • Un account Quip con autorizzazioni amministrative.

  • Create le credenziali di autenticazione di Quip che includono un token di accesso personale. Consulta la documentazione di Quip sull'autenticazione per maggiori informazioni.

  • Hai copiato il dominio del tuo sito Quip. Ad esempio, https://quip-company.quipdomain.com/browse dove quipdomain è il dominio.

  • Hai controllato che ogni documento sia unico in Quip e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le tue credenziali di autenticazione Quip in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non hai unIAM ruolo o un segreto esistente, puoi usare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti la tua fonte di dati Quip aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

istruzioni di connessione

Per connettertiAmazon Kendra alla tua fonte dati Quip, devi fornire i dettagli necessari della tua fonte dati Quip in modo cheAmazon Kendra possa accedere ai tuoi dati. Se non hai ancora configurato Quip, dai un'Amazon KendraocchiataPrerequisiti.

Console

Per connettersiAmazon Kendra a Quip

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli Connettore Quip, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica dettagli dell'origine dati, inserisci le seguenti informazioni:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. È possibile includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. Nella sezione Etichette, nella sezione Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le seguenti informazioni:

    1. Nome di dominio Quip: inserisci il Quip che hai copiato dal tuo account Quip.

    2. AWSSecrets Managersegreto: scegli un segreto esistente o creane uno nuovoSecrets Manager per memorizzare le tue credenziali di autenticazione Quip. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendra-Quip-' viene aggiunto automaticamente al tuo nome segreto.

        2. Token Quip: inserisci il token di accesso personale Quip che hai creato nel tuo account Quip.

      2. Seleziona Salva.

    3. Virtual Private Cloud (VPC): è possibile scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    4. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    5. Seleziona Successivo.

  7. Nella pagina Configura impostazioni di sincronizzazione, inserisci le seguenti informazioni:

    1. Aggiungi gli ID delle cartelle Quip da scansionare —Gli ID delle cartelle Quip che vuoi scansionare.

    2. Configurazione aggiuntiva (tipi di contenuto): inserisci i tipi di contenuto che desideri sottoporre a scansione.

    3. Schemi Regex: modelli di espressioni regolari per includere o escludere determinati file. È possibile aggiungere fino a 100 pattern.

    4. In Sincronizza la pianificazione dell'esecuzione, per Frequenza: scegli la frequenza con cui Amazon Kendra si sincronizzerà con la tua fonte di dati.

    5. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi, inserisci le seguenti informazioni:

    1. Seleziona dai campi dell'origine dati predefiniti generati che desideri mappare all'Amazon Kendraindice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È inoltre possibile scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra a Quip

È necessario specificare quanto segue utilizzando l'QuipConfigurationAPI:

  • Dominio del sito Quip —Ad esempio, https://quip-company.quipdomain.com/browse dove quipdomain è il dominio.

  • Nome segreto Amazon Resource Name (ARN): fornisci l'Amazon Resource Name (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo account Quip. Il segreto è memorizzato in una struttura JSON con le seguenti chiavi:

    {
    "accessToken": "token"
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore Quip eAmazon Kendra. Per ulteriori informazioni consulta Ruoli IAM per le fonti di dati Quip.

È inoltre possibile aggiungere le seguenti funzionalità opzionali:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration come parte della configurazione dell'origine dei dati. Vedi ConfigurazioneAmazon Kendra per l'uso di un VPC.

  • Filtri di inclusione ed esclusione: specificare se includere o escludere determinati file.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine dati di Quip ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati della ricerca in base all'accesso dell'utente o del suo gruppo ai documenti. Per maggiori informazioni, consulta Filtraggio del contesto utente per le fonti di dati di Quip.
