Indice

AccessControlConfigurationId

L'identificatore della configurazione di controllo dell'accesso che si desidera applicare al documento.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 1. La

Modello: [a-zA-Z0-9-]+

Required: No

AccessControlList

Informazioni sui committenti (utenti e/o gruppi) e a quali documenti dovrebbero avere accesso. Ciò è utile per il filtraggio del contesto utente, in cui i risultati della ricerca vengono filtrati in base all'accesso dell'utente o del suo gruppo ai documenti.

Tipo: matrice di oggetti Principal

Required: No

Attributes

Attributi personalizzati da applicare al documento. Utilizza gli attributi personalizzati per fornire informazioni aggiuntive per la ricerca, per fornire sfaccettature per affinare le ricerche e per fornire informazioni aggiuntive nella risposta alla query.

Ad esempio, 'DataSourceId' eDataSourceSyncJobId 'sono attributi personalizzati che forniscono informazioni sulla sincronizzazione dei documenti in esecuzione su una fonte di dati. Nota, 'DataSourceSyncJobId' potrebbe essere un attributo personalizzato opzionale poiché Amazon Kendra utilizzerà l'ID di un processo di sincronizzazione in esecuzione.

Tipo: matrice di oggetti DocumentAttribute

Required: No

Blob

Il contenuto del

I documenti passati alBlob parametro devono essere codificato in base64. Il tuo codice potrebbe non aver bisogno di codificare i byte del file del documento se utilizzi unAWS SDK per chiamare le API di Amazon Kendra. Se stai chiamando l'endpoint Amazon Kendra direttamente utilizzando REST, devi codificare i contenuti in base64 prima dell'invio.

Tipo: oggetto dati binari con codifica Base64

Required: No

ContentType

Il tipo di file del documento nelBlob campo.

Tipo: String

Valori validi: PDF | HTML | MS_WORD | PLAIN_TEXT | PPT | RTF | XML | XSLT | MS_EXCEL | CSV | JSON | MD

Required: No

HierarchicalAccessControlList

L'elenco degli elenchi principali che definiscono la gerarchia ai quali i documenti devono avere accesso gli utenti.

Tipo: matrice di oggetti HierarchicalPrincipal

Membri dell’array: numero minimo di 1 elemento. Numero di 30

Required: No

Id

Un identificatore del documento nell'indice.

Nota, ogni ID del documento deve essere univoco per indice. Non è possibile creare una fonte di dati per indicizzare i documenti con i relativi ID univoci e quindi utilizzare l'BatchPutDocumentAPI per indicizzare gli stessi documenti o viceversa. È possibile eliminare un'origine dati e quindi utilizzare l'BatchPutDocumentAPI per indicizzare gli stessi documenti o viceversa.

Tipo: String

Limitazioni di lunghezza: lunghezza minima di 1. La lunghezza massima è 2048 caratteri.

Campo obbligatorio: sì

S3Path

Tipo: S3Path oggetto

Required: No

Title

Tipo: string
