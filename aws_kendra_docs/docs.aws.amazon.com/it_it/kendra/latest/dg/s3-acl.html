Controllo degli accessi per le fonti diAmazon S3 dati

È possibile controllare l'accesso ai documenti in un'origineAmazon S3 dati utilizzando un file di configurazione. Il file viene specificato nella console o comeAccessControlListConfiguration parametro quando si chiama l'UpdateDataSourceAPICreateDataSource or.

Il file di configurazione contiene una struttura JSON che identifica un prefisso S3 ed elenca le impostazioni di accesso per il prefisso. Il prefisso può essere un percorso o un singolo file. Se il prefisso è un percorso, le impostazioni di accesso si applicano a tutti i file in quel percorso.

È possibile specificare sia gli utenti che i gruppi nelle impostazioni di accesso. Quando si esegue una query sull'indice, è necessario specificare le informazioni relative all'utente e al gruppo. Per ulteriori informazioni, consulta Filtro in base agli attributi utente.

La struttura JSON per il file di configurazione deve avere il formato seguente:

[
    {
        "keyPrefix": "s3://prefix1",
        "aclEntries": [
            {
                "Name": "user1",
                "Type": "USER",
                "Access": "ALLOW"
            },
            {
                "Name": "group1",
                "Type": "GROUP",
                "Access": "DENY"
            }
        ]
    },
    {
        "keyPrefix": "s3://prefix2",
        "aclEntries": [
            {
                "Name": "user2",
                "Type": "USER",
                "Access": "ALLOW"
            },
            {
                "Name": "user1",
                "Type": "USER",
                "Access": "DENY"
            },
            {
                "Name": "group1",
                "Type": "GROUP",
                "Access": "DENY"
            }
        ]
    }
]
