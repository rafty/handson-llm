Invio di feedback per l'apprendimento incrementale

Amazon Kendrautilizza l'apprendimento incrementale per migliorare i risultati di ricerca. Utilizzando il feedback delle query, l'apprendimento incrementale migliora gli algoritmi di classificazione e ottimizza i risultati di ricerca per una maggiore precisione.

Ad esempio, supponiamo che gli utenti cerchino la frase «prestazioni sanitarie». Se gli utenti scelgono costantemente il secondo risultato dall'elenco,Amazon Kendra col tempo lo porta al primo posto. Il boost diminuisce nel tempo, quindi se gli utenti smettono di selezionare un risultato,Amazon Kendra alla fine lo rimuove e mostra invece un altro risultato più popolare. Questo aiuta aAmazon Kendra dare priorità ai risultati in base alla pertinenza, all'età e ai contenuti.

L'apprendimento incrementale è attivato per tutti gli indici e per tutti i tipi di documenti supportati.

Amazon Kendrainizia a imparare non appena fornisci un feedback, anche se possono essere necessarie più di 24 ore per vedere i risultati del feedback. Amazon Kendraoffre tre metodi per inviare feedback: laAWS console, una JavaScript libreria che puoi includere nella pagina dei risultati di ricerca e un'API che puoi utilizzare.

Amazon Kendraaccetta due tipi di feedback degli utenti:

  • Clic: informazioni sui risultati della query scelti dall'utente. Il feedback include l'ID del risultato e il timestamp Unix della data e dell'ora in cui è stato scelto il risultato della ricerca.

    Per inviare un feedback sui clic, l'applicazione deve raccogliere informazioni sui clic dalle attività degli utenti e quindi inviarle aAmazon Kendra. Puoi raccogliere informazioni sui clic con la console, la JavaScript libreria e l'Amazon KendraAPI.

  • Rilevanza: informazioni sulla pertinenza di un risultato di ricerca, fornite in genere dall'utente. Il feedback contiene l'ID del risultato e un indicatore di pertinenza (RELEVANToNOT_RELEVANT). L'utente determina le informazioni sulla pertinenza.

    Per inviare un feedback sulla pertinenza, l'applicazione deve fornire un meccanismo di feedback che consenta all'utente di scegliere la pertinenza appropriata per il risultato di una query e quindi inviare tali informazioni aAmazon Kendra. Puoi raccogliere informazioni sulla pertinenza solo con la console e l'Amazon KendraAPI.

Il feedback viene utilizzato mentre l'indice è attivo. Il feedback influisce solo sull'indice a cui viene inviato, non può essere utilizzato tra indici o per account diversi.

È necessario fornire un contesto utente aggiuntivo quando si esegue una query sull'Amazon Kendraindice. Quando fornisci il contesto dell'utente,Amazon Kendra è in grado di capire se il feedback è fornito da un singolo utente o da più utenti e di modificare i risultati della ricerca di conseguenza.

Quando si fornisce il contesto dell'utente, il feedback per la query viene associato all'utente specifico fornito nel contesto. Se non si specifica il contesto dell'utente, è possibile fornire un ID visitatore utilizzato per raggruppare e aggregare le query.

Se non fornisci il contesto dell'utente o un ID visitatore, il feedback è anonimo e aggregato con altri feedback anonimi.

Il codice seguente mostra come includere il contesto dell'utente come token o ID visitatore.

response = kendra.query(
    QueryText = query,
    IndexId = index,
    UserToken = {
        Token = "token"
    })
    
    OR
    
    response = kendra.query(
    QueryText = query,
    IndexId = index,
    VisitorId = "visitor-id")

Per le applicazioni Web, puoi utilizzare i cookie, le posizioni o gli utenti del browser per generare un ID visitatore per ogni utente.

Per quanto riguarda le interrogazioni principali, il maggior numero di domande, fornendo un feedback basato su un clic, fornisce informazioni sufficienti per migliorare la precisione complessiva. Per le domande finali, quelle rare, gli esperti in materia devono inviare feedback pertinenti e non pertinenti per migliorare la precisione di tali domande.

Oltre alla console, puoi utilizzare uno dei due metodi: una JavaScript libreria o l'SubmitFeedbackAPI. È necessario utilizzare un solo metodo per raccogliere feedback. Per ottenere risultati ottimali, è necessario inviare un feedback entro 24 ore dalla richiesta.
