Filtraggio e ricerca sfaccettata

È possibile migliorare i risultati della ricerca o la risposta dall'API Query utilizzando i filtri. I filtri limitano i documenti nella risposta a quelli che si applicano direttamente alla query. Per creare suggerimenti di ricerca sfaccettati, utilizza la logica booleana per filtrare dalla risposta gli attributi specifici del documento o i documenti che non corrispondono a criteri specifici. È possibile specificare le sfaccettature utilizzando ilFacets parametro nell'QueryAPI.

Per cercare documenti conAmazon Kendra cui sei stato indicizzatoAmazon Lex, usa AMAZON. KendraSearchIntent. Per un esempio di configurazioneAmazon Kendra conAmazon Lex, vedi Creazione di un bot di domande frequenti per unAmazon Kendra indice. Puoi anche fornire un filtro per la risposta utilizzando AttributeFilter. Questo è il filtro di interrogazione in JSON durante la configurazioneAMAZON.KendraSearchIntent. Per fornire un filtro di attributi durante la configurazione di un intento di ricerca nella console, vai all'editor degli intenti e scegliAmazon Kendra query per fornire un filtro di query in JSON. Per ulteriori informazioniAMAZON.KendraSearchIntent, consulta la guida allaAmazon Lex documentazione.

Facet

Le sfaccettature sono visualizzazioni mirate di una serie di risultati di ricerca. Ad esempio, puoi fornire risultati di ricerca per città di tutto il mondo, in cui i documenti vengono filtrati in base alla città specifica a cui sono associati. In alternativa, puoi creare sfaccettature per visualizzare i risultati di un autore specifico.

È possibile utilizzare un attributo del documento o un campo di metadati associato a un documento come sfaccettatura in modo che gli utenti possano effettuare ricerche per categorie o valori all'interno di quella sfaccettatura. Puoi anche visualizzare sfaccettature annidate nei risultati della ricerca in modo che gli utenti possano effettuare ricerche non solo per categoria o campo, ma anche per sottocategoria o sottocampo.

Il seguente esempio mostra come ottenere informazioni sulle sfaccettature per l'attributo personalizzato «Città».

response=kendra.query(
        QueryText = query,
        IndexId = index,
        Facets = [
            {
                "DocumentAttributeKey" : "City"
            }
        ]
        )

È possibile utilizzare sfaccettature annidate per restringere ulteriormente la ricerca. Ad esempio, l'attributo o la sfaccettatura del documento «Città» include un valore chiamato «Seattle». Inoltre, l'attributo o sfaccettatura del documento "CityRegion" include i valori «Nord» e «Sud» per i documenti assegnati a «Seattle». È possibile visualizzare le sfaccettature annidate con i relativi conteggi nei risultati della ricerca in modo che i documenti possano essere ricercati non solo per città ma anche per regione all'interno di una città.

Tieni presente che le sfaccettature annidate potrebbero influire sulla latenza delle query. Una regola generale è che maggiore è il numero di sfaccettature annidate utilizzate, maggiore è il potenziale impatto sulla latenza. Altri fattori che influiscono sulla latenza includono la dimensione media dei documenti indicizzati, la dimensione dell'indice, le query altamente complesse e il carico complessivo sull'Amazon Kendraindice.

L'esempio seguente mostra come ottenere informazioni sulle sfaccettature per l'attributo personalizzatoCityRegion "", come sfaccettatura annidata all'interno di «City».

response=kendra.query(
        QueryText = query,
        IndexId = index,
        Facets = [
            {
                "DocumentAttributeKey" : "City",
                "Facets": [
                    {
                        "DocumentAttributeKey" : "CityRegion"
                    }
                ]
            }
        ]
        )

Le informazioni sulle sfaccettature, come il numero di documenti, vengono restituite nell'array diFacetResults risposta. I contenuti vengono utilizzati per visualizzare suggerimenti di ricerca sfaccettati nell'applicazione. Ad esempio, se l'attributo del documento «Città» contiene la città a cui potrebbe essere applicata una ricerca, utilizza tali informazioni per visualizzare un elenco di ricerche in città. Gli utenti possono scegliere una città per filtrare i risultati della ricerca. Per effettuare la ricerca sfaccettata, chiama l'API Query e utilizza l'attributo del documento scelto per filtrare i risultati.

È possibile visualizzare fino a 10 valori di sfaccettatura per sfaccettatura per un'interrogazione e solo una sfaccettatura annidata all'interno di una sfaccettatura. Se desideri aumentare questi limiti, contatta l'Support. Se si desidera limitare il numero di valori di sfaccettatura per sfaccettatura a meno di 10, è possibile specificarlo nell'Facetoggetto.

Quando si utilizza un campo dell'elenco di stringhe per creare sfaccettature, i risultati delle sfaccettature restituiti si basano sul contenuto dell'elenco di stringhe. Ad esempio, se hai un campo dell'elenco di stringhe che contiene due elementi, uno con la lista «bassotto», «cane da salsiccia» e uno con il valore «husky», ottieniFacetResults tre sfaccettature.

Utilizzo degli attributi del documento per filtrare i risultati della ricerca

Per impostazione predefinita,Query restituisce tutti i risultati della ricerca. Per filtrare le risposte, è possibile eseguire operazioni logiche sugli attributi del documento. Ad esempio, se desideri solo documenti per una città specifica, puoi filtrare in base agli attributi del documento personalizzato «Città» e «Stato». Si utilizza AttributeFilterper creare un'operazione booleana sui filtri forniti.

La maggior parte degli attributi può essere utilizzata per filtrare le risposte per tutti i tipi di risposta. Tuttavia, l'_excerpt_page_numberattributo è applicabile solo ai tipi diANSWER risposta quando si filtrano le risposte.

L'esempio seguente mostra come eseguire un'operazione AND logica filtrando in base a una città specifica, Seattle, e allo stato di Washington. 

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {'AndAllFilters': 
            [ 
                {"EqualsTo": {"Key": "City","Value": {"StringValue": "Seattle"}}},
                {"EqualsTo": {"Key": "State","Value": {"StringValue": "Washington"}}}
            ]
            }
        )

L'esempio seguente mostra come eseguire un'operazione logica OR quando una delleSourceURI chiaviFileformatAuthor, o corrisponde ai valori specificati. 

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {'OrAllFilters': 
            [ 
                {"EqualsTo": {"Key": "Fileformat","Value": {"StringValue": "AUTO_DETECT"}}},
                {"EqualsTo": {"Key": "Author","Value": {"StringValue": "Ana Carolina"}}},
                {"EqualsTo": {"Key": "SourceURI","Value": {"StringValue": "https://aws.amazonaws.com/234234242342"}}}
            ]
            }
        )

PerStringList i campi, utilizza i filtriContainsAll degli attributiContainsAny or per restituire i documenti con la stringa specificata. L'esempio seguente mostra come restituire tutti i documenti che hanno i valori «Seattle» o «Portland» nel loro attributoLocations personalizzato.

response=kendra.query(
        QueryText = query,
        IndexId = index,
        AttributeFilter = {
                "ContainsAny": { "Key": "Locations", "Value": { "StringListValue": [ "Seattle", "Portland"] }}
            }
        )

Filtrare gli attributi di ogni documento nei risultati della ricerca

Amazon Kendrarestituisce gli attributi del documento per ogni documento nei risultati della ricerca. Puoi filtrare determinati attributi del documento che desideri includere nella risposta come parte dei risultati della ricerca. Per impostazione predefinita, nella risposta vengono restituiti tutti gli attributi del documento assegnati a un documento.

Nell'esempio seguente, solo gli attributi_source_uri e del_author documento sono inclusi nella risposta di un documento.

response=kendra.query(
        QueryText = query,
        IndexId = index,
        RequestedDocumentAttributes = ["_source_uri", "_author"]
        )
