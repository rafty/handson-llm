Distribuzione di Amazon Kendra

Quando arriva il momento di implementare laAmazon Kendra ricerca sul tuo sito Web, forniamo il codice sorgente che puoi utilizzare con React per iniziare subito con la tua applicazione. Il codice sorgente viene fornito gratuitamente con una licenza MIT modificata. Puoi utilizzarlo così com'è o modificarlo per le tue esigenze. L'app React di nella è un per iniziare a utilizzare il prodotto. Non è un'app pronta per la produzione.

Per distribuire un'applicazione di ricerca senza codice e generare un URL endpoint nella tua pagina di ricerca con controllo degli accessi, consulta Amazon KendraExperience Builder.

Ci sono due esempi che puoi usare con React:

  • https://kendrasamples.s3.amazonaws.com/kendrasamples-react-app .zip —Un' applicazione React di esempio che fornisce dati di esempio e una pagina di ricerca.

  • https://kendrasamples.s3.amazonaws.com/kendrasamples.zip —Una libreria che puoi aggiungere a un'applicazione React esistente.

Gli esempi sono modellati sulla pagina di ricerca dellaAmazon Kendra console. Hanno le stesse funzionalità per la ricerca e la visualizzazione dei risultati della ricerca. Puoi usare l'intero esempio oppure puoi scegliere solo una delle funzionalità per uso personale.

Per visualizzare i tre componenti della pagina di ricerca nellaAmazon Kendra console, scegli l'icona del codice (</>) dal menu a destra. Passa il puntatore del mouse su ciascuna sezione per visualizzare una breve descrizione del componente e ottenere l'URL della fonte del componente.

Panoramica

Aggiungi il codice di esempio a un'applicazione React esistente per attivare la ricerca. I file e i componenti di ricerca sono strutturati come segue:

  • Pagina di ricerca principale: questa è la pagina principale che contiene tutti i componenti. Qui è dove integrerai la tua applicazione con l'Amazon KendraAPI.

  • Barra di ricerca: questo è il componente in cui un utente inserisce un termine di ricerca e che chiama la funzione di ricerca.

  • Risultati: questo è il componente da cui vengono visualizzati i risultatiAmazon Kendra. Ha tre componenti: risposte suggerite, risultati delle domande frequenti e documenti consigliati.

  • Sfaccettature: questo è il componente che mostra le sfaccettature nei risultati della ricerca e consente di scegliere una sfaccettatura per limitare la ricerca.

  • Impaginazione: questo è il componente da cui impagina la rispostaAmazon Kendra.

Prerequisiti

Prima di iniziare, avrai bisogno di:

  • Un'applicazione Web React esistente o l'applicazione di esempio.

  • Un ambiente di sviluppo configurato con le librerie corrette.

  • Node.js e npm installati.

  • IlSDK for Java oAWS SDK for JavaScript.

Le informazioni sulle librerie e sugliAWS SDK richiesti si trovano nel file Readme nei file zip.

Configurazione del prodotto

Una procedura completa per aggiungere laAmazon Kendra ricerca a un'applicazione React è nel Readme incluso nei file zip di esempio.

Per iniziare a utilizzare il kendra-samples-react-app prodotto.
avvertimento

L'app React non è pronta per la produzione. È un esempio di implementazione di un'app per laAmazon Kendra ricerca.

  1. Assicurati di aver completato ilPrerequisiti, incluso il download e l'installazione di Node.js e npm.

  2. kendra-samples-react-appScaricare.zip e decomprimerlo.

  3. Apri il terminale e vai aaws-kendra-sample-app/src/services/. Aprilocal-dev-credentials-template.json e fornisci le tue credenziali. Non aggiungere questo file a nessun archivio pubblico.

  4. Vai aaws-kendra-sample-app/src e installa le dipendenze. Esegui npm install.

  5. Avvia una versione di. Esegui npm start. È possibile interrompere il server locale inserendo il codice sulla tastieraCmd/Ctrl + C.

  6. Puoi cambiare la porta o l'host (ad esempio, l'indirizzo IP) accedendopackage.json e aggiornando l'host e la porta:"start": "HOST=[host] PORT=[port] react-scripts start". Se usi Windows:"start": "set HOST=[host] && set PORT=[port] && react-scripts start".

  7. Se hai un dominio di sito Web registrato, puoi specificarlopackage.json dopo il nome dell'app. Ad esempio, "homepage": "https://mywebsite.com". È necessario eseguirenpm install nuovamente per aggiornare le nuove dipendenze e quindi eseguirenpm start.

  8. Per creare l'app, eseguinpm build. Carica i contenuti della cartella di compilazione sul tuo provider di hosting.

Pagina di ricerca principale

La pagina di ricerca principale contiene tutti i componenti di ricerca di esempio. Include il componente della barra di ricerca per l'output, i componenti dei risultati per visualizzare la risposta dall'API Query e un componente di impaginazione per l'impaginazione della risposta.

Componente di ricerca

Il componente di ricerca fornisce una casella di testo per inserire il testo della query. LaonSearch funzione è un hook che richiama la funzione principaleSearch.tsx per effettuare la chiamata all'APIAmazon Kendra Query.

Componente dei risultati

Il componente dei risultati mostra la risposta dall'QueryAPI. I risultati sono mostrati in tre aree separate.

  • Risposte suggerite: questi sono i risultati principali restituiti dall'QueryAPI. Contiene fino a tre risposte suggerite. Nella risposta, hanno il tipo di risultatoANSWER.

  • Risposte alle domande frequenti: questi sono i risultati delle domande frequenti restituiti dalla risposta. Le domande frequenti vengono aggiunte all'indice separatamente. Nella risposta, hanno il tipoQUESTION_ANSWER. Per ulteriori informazioni, consulta Domande e risposte.

  • Documenti consigliati: si tratta di documenti aggiuntivi che vengonoAmazon Kendra restituiti nella risposta. Nella risposta dell'QueryAPI, hanno il tipoDOCUMENT.

I componenti dei risultati condividono una serie di componenti per funzionalità quali evidenziazione, titoli, collegamenti e altro ancora. I componenti condivisi devono essere presenti affinché i componenti del risultato funzionino.

Componente per sfaccettature

Il componente sfaccettature elenca le sfaccettature disponibili nei risultati della ricerca. Ogni sfaccettatura classifica la risposta in base a una dimensione specifica, ad esempio l'autore. È possibile affinare la ricerca in base a un aspetto specifico scegliendone uno dall'elenco.

Dopo aver selezionato una sfaccettatura, il componente chiamaQuery con un filtro di attributi che limita la ricerca ai documenti che corrispondono alla sfaccettatura.

Componente di impaginazione

Il componente di impaginazione consente di visualizzare i risultati della ricerca dall'QueryAPI in più pagine. Chiama l'QueryAPI con iPageNumber parametriPageSize and per ottenere una pagina specifica di risultati.
