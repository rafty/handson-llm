All'aperto

Alfresco OnPrem è un servizio di gestione dei contenuti che aiuta i clienti a archiviare e gestire i propri contenuti. È possibile utilizzarloAmazon Kendra per indicizzare la libreria di documenti Alfresco, il wiki e il blog.

Puoi connettertiAmazon Kendra alla tua fonte di dati Alfresco utilizzando la Amazon Kendraconsole e l'AlfrescoConfigurationAPI.

Nota

Il connettore sorgente dati Alfresco è attualmente in modalità anteprima. L'autenticazione di base è attualmente supportata. Se desideri utilizzare il connettore Alfresco in produzione, contatta l'Support. Alfresco PaaS non è attualmente supportato.

Per la risoluzione dei problemi relativi al connettore di origine datiAmazon Kendra Alfresco, consultaRisoluzione dei problemi delle origini dati.

Funzionalità supportate

Amazon KendraIl connettore di origine dati Alfresco supporta le seguenti funzionalità:

  • Mappature di campi

  • Filtraggio del contesto utente

  • Filtri di inclusione/esclusione

  • Virtual Private Cloud (VPC) (Cloud privato virtuale (VPC)

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la tua fonte di dati Alfresco, apporta queste modifiche in Alfresco e nei tuoiAWS account.

In Alfresco, assicurati di disporre di:

  • Ho preso nota delle tue credenziali di autenticazione Alfresco, che includono un nome utente e una password di amministratore.

  • È stato aggiunto un utente alALFRESCO_ADMINISTRATORS gruppo.

  • Hai copiato l'URL del sito Alfresco e l'ID del sito Alfresco.

  • Verifica che ogni documento sia unico in Alfresco e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le tue credenziali di autenticazione Alfresco in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti la tua fonte di dati Alfresco aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

istruzioni di connessione

Per connettersiAmazon Kendra alla fonte dati Alfresco, è necessario fornire i dettagli necessari della fonte dati Alfresco in modo cheAmazon Kendra possa accedere ai dati. Se non hai ancora configurato Alfresco,Amazon Kendra guardaPrerequisiti.

Console

Per connettersiAmazon Kendra ad Alfresco

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli Connettore Alfresco, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica dei dettagli dell'origine dati, inserisci le seguenti informazioni:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. È possibile includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. Nella sezione Distribuisci nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le seguenti informazioni:

    1. URL del sito Alfresco: inserisci l'URL del tuo sito Alfresco. Ad esempio, https://hostname:8080.

    2. Posizione del certificato SSL: inserisci ilAmazon S3 percorso.

    3. ID sito: l'ID del sito Alfresco.

    4. AWSSecrets Managersegreto: scegli un segreto esistente o creane uno nuovoSecrets Manager per archiviare le tue credenziali di autenticazione Alfresco. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Inserisci le seguenti informazioni nella finestra Crea unAWSSecrets Manager segreto:

        1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendra-Alfresco-' viene aggiunto automaticamente al tuo nome segreto.

        2. Per Nome utente e password: inserisci il nome utente e la password di Alfresco.

      2. Seleziona Salva.

    5. Virtual Private Cloud (VPC): puoi scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    6. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    7. Seleziona Successivo.

  7. Nella pagina Configura impostazioni di sincronizzazione, inserisci le seguenti informazioni:

    1. Seleziona il contenuto da sottoporre a scansione: le entità o i tipi di contenuto di Alfresco che desideri sottoporre a scansione.

    2. Schemi Regex: modelli di espressioni regolari per includere o escludere determinati file. È possibile aggiungere fino a 100 pattern.

    3. In Sincronizza programma di esecuzione, per Frequenza: scegli la frequenzaAmazon Kendra con cui sincronizzarti con la tua fonte di dati.

    4. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi, inserisci le seguenti informazioni:

    1. Per Libreria documenti, Wiki e Blog: seleziona dai campi dell'origine dati predefinitiAmazon Kendra generati che desideri mappare nel tuo indice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. È inoltre possibile scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra ad Alfresco

È necessario specificare quanto segue utilizzando l'AlfrescoConfigurationAPI:

  • Nome segreto Amazon Resource Name (ARN): fornisci l'Amazon Resource Name (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo account Alfresco. Il segreto è memorizzato in una struttura JSON con le seguenti chiavi:

    {
    "username": "user name",
    "password": "password"
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore Alfresco eAmazon Kendra. Per ulteriori informazioni consulta Ruoli IAM per le sorgenti dati Alfresco.

Puoi inoltre aggiungere le seguenti funzionalità facoltativi:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration come parte della configurazione dell'origine dei dati. Vedi ConfigurazioneAmazon Kendra per l'uso di un VPC.

  • Filtri di inclusione ed esclusione: specificare se includere o escludere determinati file.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine dati Alfresco ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati della ricerca in base all'accesso dell'utente o del suo gruppo ai documenti. Per ulteriori informazioni, consulta Filtraggio del contesto utente per le fonti di dati Alfresco.
