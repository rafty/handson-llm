Box (Cubo)

Box è un servizio di cloud storage che offre funzionalità di file hosting. Puoi utilizzarloAmazon Kendra per indicizzare i contenuti del tuo Box, inclusi commenti, attività e link web.

Puoi connettertiAmazon Kendra alla fonte di dati Box utilizzando la Amazon Kendraconsole e l'BoxConfigurationAPI.

Per la risoluzione dei problemi relativi al connettore di origine datiAmazon Kendra Box, consultaRisoluzione dei problemi delle origini dati.

Funzionalità supportate

Amazon KendraIl connettore di origine dati Box supporta le seguenti funzionalità:

  • Log delle modifiche

  • Mappature di campi

  • Filtraggio del contesto utente

  • Filtri di inclusione/esclusione

  • Virtual Private Cloud (VPC)

Prerequisiti

Prima di poterla utilizzareAmazon Kendra per indicizzare la fonte dei dati di Box, apporta queste modifiche nel tuo Box e nei tuoiAWS account.

In Box, assicurati di disporre di:

  • Un account Box Enterprise o Box Enterprise Plus.

  • Ha creato un'app personalizzata Box nella Box Developer Console e l'ha configurata per utilizzare l'autenticazione del server (con JWT). Consulta la documentazione di Box sulla creazione di un'app personalizzata e la documentazione di Box sulla configurazione di JWT Auth per maggiori dettagli.

  • Imposta il livello di accesso all'app su App + Enterprise Access e consenti all'app di effettuare chiamate API utilizzando l'intestazione as-user.

  • Hai usato l'utente amministratore per aggiungere i seguenti ambiti applicativi nell'app Box:

    • Scrivi tutti i file e le cartelle archiviati in un Box

    • Gestire gli utenti

    • Gestione di gruppi

    • Gestisci le proprietà aziendali

  • key pair pubbliche/private generata e scaricata che include un ID client, un segreto del cliente, un ID di chiave pubblica, un ID di chiave privata, una passphrase e un ID aziendale da utilizzare come credenziali di autenticazione. Vedi Coppia di tasti pubblica e privata per maggiori dettagli.

  • Hai copiato il tuo ID aziendale Box dalle impostazioni della Box Developer Console o dall'app Box. Ad esempio, 801234567.

  • È stato selezionato che ogni documento è unico in Box e in altre fonti di dati che intendi utilizzare per lo stesso indice. Ogni origine dati che si desidera utilizzare per un indice non deve contenere lo stesso documento in tutte le fonti di dati. Gli ID dei documenti sono globali per un indice e devono essere univoci per indice.

Nel tuoAWS account, assicurati di avere:

  • Ha creato unAmazon Kendra indice e, se si utilizza l'API, ha annotato l'ID dell'indice.

  • Hai creato unIAM ruolo per la tua fonte di dati e, se utilizzi l'API, hai annotato l'ARN delIAM ruolo.

  • Hai archiviato le tue credenziali di autenticazione Box in modoAWSSecrets Manager segreto e, se utilizzi l'API, annota l'ARN del segreto.

    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

Se non disponi di unIAM ruolo o di un segreto esistente, puoi utilizzare la console per creare un nuovoIAM ruolo e un nuovoSecrets Manager segreto quando connetti la tua fonte dati Box aAmazon Kendra. Se si utilizza l'API, è necessario fornire l'ARN di unIAM ruolo e unSecrets Manager segreto esistenti e un ID di indice.

istruzioni di connessione

Per connettertiAmazon Kendra alla fonte dati Box, devi fornire i dettagli necessari della fonte dati Box in modo cheAmazon Kendra possa accedere ai tuoi dati. Se non hai ancora configurato Box, dai un'Amazon KendraocchiataPrerequisiti.

Console

Per connettersiAmazon Kendra a Box

  1. Accedi alla console di gestione AWS e apri la console Amazon Kendra.

  2. Nel riquadro di navigazione a sinistra, scegli Indici, quindi scegli l'indice che desideri utilizzare dall'elenco degli indici.

    Nota

    Puoi scegliere di configurare o modificare le impostazioni di controllo dell'accesso utente in Impostazioni dell'indice.

  3. Nella pagina Guida introduttiva, scegli Aggiungi origine dati.

  4. Nella pagina Aggiungi origine dati, scegli Box connector, quindi scegli Aggiungi origine dati.

  5. Nella pagina Specifica dei dettagli dell'origine dati, inserisci le seguenti informazioni:

    1. In Nome e descrizione, per Nome origine dati: inserisci un nome per la tua origine dati. È possibile includere trattini ma non spazi.

    2. (Facoltativo) Descrizione: inserisci una descrizione facoltativa per la tua fonte di dati.

    3. In Lingua, per Lingua predefinita: una lingua per filtrare i documenti per l'indice. Salvo diversa indicazione, la lingua predefinita è l'inglese. La lingua specificata nei metadati ha la precedenza sulla lingua selezionata.

    4. In Etichette, nella sezione Aggiungi nuovo tag: tag per cercare e filtrare le risorse o monitorare iAWS costi.

    5. Seleziona Successivo.

  6. Nella pagina Definisci accesso e sicurezza, inserisci le seguenti informazioni:

    1. ID aziendale Box: inserisci il tuo ID Box Enterprise.

    2. AWSSecrets Managersegreto: scegli un segreto esistente o creane uno nuovoSecrets Manager per archiviare le credenziali di autenticazione Box. Se scegli di creare un nuovo segreto, si apre una finestraAWSSecrets Manager segreta.

      1. Nome segreto: un nome per il tuo segreto. Il prefisso 'AmazonKendra-Box-' viene aggiunto automaticamente al tuo nome segreto.

      2. Per Client ID, Client Secret, Public Key ID, Private Key ID e Pass Phase, inserisci i valori della chiave pubblica/privata generata nel tuo account Box e scaricata dal tuo account Box.

      3. Seleziona Salva.

    3. Virtual Private Cloud (VPC): puoi scegliere di utilizzare un VPC. In tal caso, è necessario aggiungere sottoreti e gruppi di sicurezza VPC.

    4. Ruolo IAM: scegli un ruolo IAM esistente o crea un nuovo ruolo IAM per accedere alle credenziali del repository e ai contenuti dell'indicizzazione.

      Nota

      I ruoli IAM utilizzati per gli indici non possono essere utilizzati per le fonti di dati. Se non sei sicuro che un ruolo esistente sia utilizzato per un indice o per le domande frequenti, scegli Crea un nuovo ruolo per evitare errori.

    5. Seleziona Successivo.

  7. Nella pagina Configura impostazioni di sincronizzazione, inserisci le seguenti informazioni:

    1. Seleziona le entità o i tipi di contenuto: le entità o i tipi di contenuto Box che desideri sottoporre a indicizzazione. Ogni commento viene indicizzato come documento separato.

    2. Registro delle modifiche: seleziona questa opzione per aggiornare l'indice anziché sincronizzare tutti i file.

    3. Schemi Regex: modelli di espressioni regolari per includere o escludere determinati file. È possibile aggiungere fino a 100 pattern.

    4. In Sincronizza programma di esecuzione, per Frequenza: scegli la frequenzaAmazon Kendra con cui sincronizzarti con la tua fonte di dati.

    5. Seleziona Successivo.

  8. Nella pagina Imposta mappature di campi, inserisci le seguenti informazioni:

    1. Per file e cartelle, commenti, attività e collegamenti Web: seleziona dai campi dell'origine dati predefinitiAmazon Kendra generati che desideri mappare nell'indice.

    2. Aggiungi campo: per aggiungere campi di origine dati personalizzati per creare un nome di campo indice da mappare e il tipo di dati del campo.

    3. Seleziona Successivo.

  9. Nella pagina Rivedi e crea, verifica che le informazioni che hai inserito siano corrette, quindi seleziona Aggiungi origine dati. Puoi inoltre scegliere di modificare le informazioni da questa pagina. La tua fonte di dati verrà visualizzata nella pagina Origini dati una volta aggiunta con successo.

API

Per connettersiAmazon Kendra a Box

È necessario specificare quanto segue utilizzando l'BoxConfigurationAPI:

ID aziendale Box: fornisci il tuo ID Box Enterprise. Puoi trovare l'ID aziendale nelle impostazioni della Box Developer Console o quando crei un'app in Box.

  • Nome di risorsa Amazon (ARN) segreto: fornisci il nome Amazon Resource Name (ARN) di unSecrets Manager segreto che contiene le credenziali di autenticazione per il tuo account Box. Il segreto è memorizzato in una struttura JSON con le seguenti chiavi:

    {
    "clientID": "client-id",
    "clientSecret": "client-secret",
    "publicKeyID": "public-key-id",
    "privateKey": "private-key",
    "passphrase": "pass-phrase"
}
    Nota

    Assicurati di aggiornare o ruotare regolarmente le tue credenziali e i tuoi segreti. Fornisci solo il livello di accesso necessario per la tua sicurezza. Il riutilizzo delle credenziali e dei segreti tra le fonti di dati e le versioni dei connettori 1.0 e 2.0 (ove applicabile) non è consigliato.

  • Ruolo IAM: specificaRoleArn quando chiamiCreateDataSource per fornire a unIAM ruolo le autorizzazioni per accedere al tuoSecrets Manager segreto e per chiamare le API pubbliche richieste per il connettore Box eAmazon Kendra. Per ulteriori informazioni consulta Ruoli IAM per le fonti di dati Box.

Puoi inoltre aggiungere le seguenti funzionalità facoltative:

  • Virtual Private Cloud (VPC): specificareVpcConfiguration come parte della configurazione dell'origine dei dati. Vedi ConfigurazioneAmazon Kendra per l'uso di un VPC.

  • Registro delle modifiche: se èAmazon Kendra necessario utilizzare il meccanismo del registro delle modifiche all'origine dei dati Box per determinare se un documento deve essere aggiunto, aggiornato o eliminato nell'indice.

    Nota

    Usa il registro delle modifiche se non desideriAmazon Kendra scansionare tutti i documenti. Se il registro delle modifiche è di grandi dimensioni, la scansione dei documenti nella fonte dati Box potrebbe richiedereAmazon Kendra meno tempo rispetto all'elaborazione del registro delle modifiche. Se stai sincronizzando la fonte dati di Box con il tuo indice per la prima volta, tutti i documenti vengono scansionati.

  • Filtri di inclusione ed esclusione: specifica se includere o escludere determinati file, cartelle, commenti, attività e collegamenti Web Box.

    Nota

    La maggior parte delle fonti di dati utilizza modelli di espressioni regolari, che sono modelli di inclusione o esclusione denominati filtri. Se si specifica un filtro di inclusione, viene indicizzato solo il contenuto che corrisponde al filtro di inclusione. Qualsiasi documento che non corrisponde al filtro di inclusione non viene indicizzato. Se si specifica un filtro di inclusione ed esclusione, i documenti che corrispondono al filtro di esclusione non vengono indicizzati, anche se corrispondono al filtro di inclusione.

  • Mappature dei campi: scegli di mappare i campi dell'origine dati di Box ai campiAmazon Kendra dell'indice. Per ulteriori informazioni, consulta la sezione Mappatura dei campi di origine dei dati.

  • Filtraggio del contesto utente: per impostazione predefinitaAmazon Kendra esegue la scansione dell'Access Control List (ACL) per la fonte dei dati. Le informazioni ACL vengono utilizzate per filtrare i risultati della ricerca in base all'accesso dell'utente o del suo gruppo ai documenti. Per ulteriori informazioni, consulta Filtraggio del contesto utente per le sorgenti dati Box.
