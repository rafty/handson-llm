Che cos'è Amazon CloudWatch Events?

Nota

Amazon EventBridge è il metodo preferito per gestire gli eventi. CloudWatch Events ed EventBridge sono di base lo stesso servizio e la stessa API, ma EventBridge fornisce più funzionalità. Le modifiche apportate in CloudWatch o EventBridge verranno visualizzate nella rispettiva console. Per ulteriori informazioni, consulta Amazon EventBridge.

Amazon CloudWatch Events eroga un flusso quasi in tempo reale di eventi di sistema che descrivono le modifiche alle risorse Amazon Web Services (AWS). Utilizzando semplici regole che puoi impostare rapidamente, puoi abbinare eventi e instradarli verso una o più funzioni o flussi del target. CloudWatch Events rileva le modifiche operative appena si verificano. CloudWatch Events risponde a queste modifiche operative e prende le misure correttive necessarie inviando messaggi per rispondere all'ambiente attivando funzioni, effettuando modifiche e catturando informazioni sullo stato.

Puoi anche utilizzare CloudWatch Events per pianificare operazioni automatiche che si attivano in modo autonomo in determinati momenti tramite espressioni cron o rate. Per ulteriori informazioni, consulta . Pianificazione di espressioni per regole.

Puoi configurare i servizi AWS seguenti come target per CloudWatch Events:

  • Istanze Amazon EC2

  • AWS LambdaFunzioni

  • Flussi in Amazon Kinesis Data Streams

  • Flussi di consegna in Amazon Kinesis Data Firehose

  • Gruppi di registri in Amazon CloudWatch Logs

  • Attività di Amazon ECS

  • Run Command di Systems Manager

  • Systems Manager Automation

  • AWS Batch processi

  • Macchine a stati di Step Functions

  • Pipeline in CodePipeline

  • Progetti CodeBuild

  • Modelli di valutazione di Amazon Inspector

  • Argomenti di Amazon SNS

  • Code Amazon SQS

  • Target predefiniti: EC2 CreateSnapshot API call, EC2 RebootInstances API call, EC2 StopInstances API call e EC2 TerminateInstances API call.

  • Bus di eventi predefinito di un altro account AWS

Concepts

Prima di iniziare a usare CloudWatch Events, devi avere compreso i concetti seguenti:

  • Eventi: un evento indica una modifica nell'ambiente AWS. Le risorse AWS possono generare eventi quando il relativo stato cambia. Ad esempio, Amazon EC2 genera un evento quando lo stato di un'istanza EC2 passa dalla sospensione all'esecuzione, mentre Amazon EC2 Auto Scaling genera eventi all'avvio o alla terminazione di istanze. AWS CloudTrail pubblica eventi quando effettui chiamate API. Puoi generare eventi a livello di applicazione personalizzati e pubblicarli in CloudWatch Events. Puoi anche impostare gli eventi pianificati generati periodicamente. Per un elenco dei servizi che generano eventi e per avere alcuni eventi di esempio derivati da ciascun servizio, consulta l'articolo su Eventi di esempio di CloudWatch Events dai servizi supportati.

  • Regole: una regola individua gli eventi in entrata e li instrada ai target per l'elaborazione. Una singola regola può instradare verso più target che vengono elaborati in parallelo. Le regole non vengono elaborate in un determinato ordine. Questo attiva le diverse parti di un'organizzazione che cercano di individuare ed elaborare gli eventi di loro interesse. Una regola è in grado di personalizzare il JSON inviato al target, eseguendo solo alcune parti o sovrascrivendole con una costante.

  • Target: un target elabora gli eventi. I target possono includere istanze Amazon EC2, funzioni AWS Lambda, flussi Kinesis, attività Amazon ECS, macchine a stati Step Functions, argomenti Amazon SNS, code Amazon SQS e target incorporati. Un target riceve eventi in formato JSON.

    I target di una regola devono trovarsi nella stessa regione della regola.

Servizi AWS correlati

I servizi seguenti vengono usati insieme a CloudWatch Events:

  • AWS CloudTrail permette di monitorare le chiamate effettuate all'API CloudWatch Events per il tuo account, tra cui le chiamate effettuate dalla AWS Management Console, da AWS CLI e da altri servizi. Quando è attivata la registrazione di CloudTrail, CloudWatch Events scrive file di log in un S3 Bucket. Ogni file di log contiene uno o più record, in base al numero di operazioni eseguite per soddisfare una richiesta. Per ulteriori informazioni, consulta . Registrazione delle chiamate API di Amazon CloudWatch Events con AWS CloudTrail.

  • AWS CloudFormation permette di modellare e configurare le risorse AWS. Puoi creare un modello che descrive le risorse AWS desiderate e AWS CloudFormation si occupa del provisioning e della configurazione di queste risorse per te. Puoi utilizzare regole CloudWatch Events nei modelli AWS CloudFormation. Per ulteriori informazioni, consulta AWS::Events::Rule nella Guida per l'utente di AWS CloudFormation.

  • AWS Config ti permette di registrare le modifiche di configurazione nelle risorse AWS. Questo include le relazioni tra le risorse e la maniera in cui sono state configurate in passato, in modo che tu possa vedere come le configurazioni e le relazioni cambiano nel corso del tempo. Puoi anche creare regole AWS Config per controllare se le risorse sono conformi o meno alle policy dell'organizzazione. Per ulteriori informazioni, consulta la Guida per sviluppatori di AWS Config.

  • AWS Identity and Access Management (IAM) aiuta a controllare in modo sicuro l'accesso alle risorse AWS per gli utenti. Utilizza IAM per controllare chi può utilizzare le risorse AWS (autenticazione), quali risorse e in che modo (autorizzazione). Per ulteriori informazioni, consulta Guida per l'utente di IAM.

  • Amazon Kinesis Data Streams permette di acquisire e aggregare dati in modo rapido e quasi continuo. Il tipo di dati utilizzato include dati di logo dell'infrastruttura IT, log di applicazioni, social media, feed di dati di mercato e dati clickstream Web. Poiché il tempo di risposta per il consumo e l'elaborazione dei dati è in tempo reale, l'elaborazione è in genere leggera. Per ulteriori informazioni, consulta la Guida per gli sviluppatori di Amazon Kinesis Data Streams.

  • AWS Lambda permette di creare applicazioni che rispondono rapidamente alle nuove informazioni. Puoi caricare il codice dell'applicazione come funzioni Lambda, e Lambda eseguirà il codice su un'infrastruttura di calcolo a elevata disponibilità. Lambda esegue tutte le attività amministrative delle risorse di calcolo, tra cui manutenzione dei server e dei sistemi operativi, provisioning della capacità, codice di scalabilità automatica, implementazione delle patch di sicurezza e dei codici e monitoraggio e accesso ai codici. Per ulteriori informazioni, consulta la Guida per sviluppatori di AWS Lambda.
