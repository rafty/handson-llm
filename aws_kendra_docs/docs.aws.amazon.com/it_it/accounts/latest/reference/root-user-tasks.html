Attività che richiedono credenziali dell'utente root

Importante

Le seguenti azioni AWS Identity and Access Management (IAM) raggiungeranno la fine del supporto standard a luglio 2023: aws-portal:ModifyAccount eaws-portal:ViewAccount. Consulta Utilizzo di AWS Billing operazioni granulari in avere accesso alle console, in avere accesso alle console, di gestione AWS Billing dei AWS costi, e. AWS

Se hai creato il tuo AWS account o l'account di AWS Organizations gestione prima del 6 marzo 2023, le operazioni granulari avranno effetto da luglio 2023. Ti consigliamo di aggiungere le operazioni granulari, ma senza rimuovere le autorizzazioni con i prefissi aws-portal o purchase-orders esistenti.

Se hai creato il tuo AWS account o il tuo account di AWS Organizations gestione dal 6 marzo 2023 incluso, le operazioni granulari avranno effetto immediato.

Ti consigliamo di configurare un utente amministrativo AWS IAM Identity Center (successor to AWS Single Sign-On) per eseguire attività quotidiane e accedere alle AWS risorse. Tuttavia, è possibile eseguire le attività elencate di seguito solo se si effettua l'accesso come utente root di un account.

Processi

  • Cambiare le impostazioni dell'account. Sono inclusi il nome dell'account, l'indirizzo e-mail, la password dell'utente root e le chiavi di accesso dell'utente root. Altre impostazioni dell'account, ad esempio le informazioni di contatto, la preferenza per la valuta di pagamentoRegioni AWS, non richiedono credenziali dell'utente root.

  • Ripristina le autorizzazioni dell'utente IAM. Se l'unico amministratore IAM revoca accidentalmente le autorizzazioni, sarà possibile effettuare l'accesso come utente root per modificare le policy e ripristinare le autorizzazioni.

  • Attivare l'accesso IAM alla console di gestione fatturazione e costi.

  • Visualizzare alcune fatture fiscali. Un utente IAM con ViewBilling autorizzazione AWSAWSaws-portals-portals-aws-portals-PL.

  • Chiudi l'Account AWS.

  • Registrati come venditore nel marketplace di istanze riservate.

  • Configurare un bucket Amazon S3 per abilitare MFA (autenticazione a più fattori).

  • Modificare o eliminare la policy di un bucket Amazon Simple Storage Service (Amazon S3) che include un ID di un ID di un cloud privato virtuale (VPC) o un ID endpoint VPC non valido.

  • Registrazione per AWS GovCloud (US).

  • Richiedere le chiavi di accesso all'utente root dell'AWS GovCloud (US)account daAWS Support.
