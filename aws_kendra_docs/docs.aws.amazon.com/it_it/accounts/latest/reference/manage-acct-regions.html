Ti consigliamo di aggiungere le operazioni granulari, ma senza rimuovere le autorizzazioni con i prefissi aws-portal o purchase-orders esistenti.

Se hai creato il tuoAWS account o il tuo account diAWS Organizations gestione dal 6 marzo 2023 incluso, le operazioni granulari avranno effetto immediato.

AWSoriginariamente abilitato tutto il nuovoRegioni AWS per impostazione predefinita, consentendo agli utenti di creare risorse in qualsiasi regione. Ora, quandoAWS aggiunge una regione, questa viene disattivata per impostazione predefinita. Se gli utenti devono essere in grado di creare risorse in una nuova regione, abilitare la regione.

Importante

AWSconsiglia di usare endpoint regionaliAWS Security Token Service (AWS STS) invece di quelli globali per ridurre la latenza. I token di sessione emessi dagliAWS STS endpoint regionali sono validi in tutte leAWS regioni. Se usiAWS STS endpoint regionali, non è necessario apportare modifiche.

Tuttavia, i token di sessione dell'AWS STSendpoint globale (https://sts.amazonaws.com) sono validi solo se abilitatiRegioni AWS dall'utente o sono abilitati per impostazione predefinita. Se si intende abilitare una nuova regione per l'account, è possibile usare i token di sessione emessi dagliAWS STS endpoint regionali o attivare l'AWS STSendpoint globale per emettere token di sessione validi in tuttiRegioni AWS. I token di sessione validi in tutte le regioni sono più grandi. Se memorizzi i token di sessione, questi token più grandi potrebbero influire sui tuoi sistemi.

Per ulteriori informazioni su come gliAWS STS endpoint funzionano conAWS le regioni, vedi GestioneAWS STS in unaAWS regione.

Considerazioni sull'attivazione e la disabilitazioneRegioni AWS

  • Puoi utilizzare autorizzazioni IAM per controllare l'accesso a regioni

    AWS Identity and Access Management(IAM) include quattro autorizzazioni che consentono di controllare quali utenti possono abilitare, disabilitare, ottenere ed elencare le regioni. Per ulteriori informazioni, consulta Politica delle operazioni di fatturazione e gestione dei costi nella Guida per l'AWS Billing and Cost Managementutente.

  • L’abilitazione di una regione è gratuita

    Non è previsto alcun costo di abilitazione di una regione. L'addebito viene calcolato in base alle risorse create nella nuova regione.

  • La disabilitazione di una regione disabilita l’accesso alle risorse nella regione

    Se disattivi una regione che contiene ancoraAWS risorse, ad esempio istanze Amazon Elastic Compute Cloud (Amazon EC2), perdi l'accesso alle risorse in tale regione. Ad esempio, non è possibile utilizzare ilAWS Management Console o qualsiasi metodo programmatico per visualizzare o modificare la configurazione di qualsiasi istanza EC2 in una regione disattivata.

  • Gli addebiti per le risorse attive continuano se disabiliti una regione

    Se disattivi una regione che contiene ancoraAWS risorse, gli addebiti per tali risorse (se presenti) continueranno ad essere accumulati secondo la tariffa standard. Ad esempio, se disabiliti una regione che contiene istanze Amazon EC2, devi ancora pagare i costi per tali istanze anche se queste non sono accessibili.

  • La disabilitazione di una regione non è sempre visibile immediatamente

    I servizi e le console potrebbero essere temporaneamente visibili dopo aver disabilitato una regione. La disattivazione di una regione può richiedere da alcuni minuti a diverse ore per avere effetto.

  • Le regioni originali sono abilitate per impostazione predefinita

    Le regioni introdotte prima del 20 marzo 2019 sono tutte abilitate per impostazione predefinita e non possono essere disabilitate. Per ulteriori informazioni, consulta la sezione relativa alla gestione di Regioni AWS nella Riferimenti generali di AWS.

  • L'abilitazione di una regione richiede alcuni minuti o diverse ore in alcuni casi

    A seconda di diversi fattori, come le dimensioni dell'organizzazione, l'attivazione di un'area può richiedere fino a diverse ore.

  • Organizations possono avere 20 richieste di selezione regionale aperte in un dato momento all'interno di un'AWSorganizzazione

    L'account di gestione può in qualsiasi momento avere 20 richieste aperte in attesa di completamento per la sua organizzazione. Una richiesta equivale all'attivazione o alla disattivazione di una particolare regione per un account.

  • Un singolo account può avere 6 richieste di opzione regionale in corso in un dato momento

    Una richiesta equivale all'attivazione o alla disattivazione di una particolare regione per un account.

  • EventBridge Integrazione con Amazon

    I clienti possono iscriversi alle notifiche di aggiornamento dello stato di Region-Opt in EventBridge. Verrà creata una EventBridge notifica per ogni modifica di stato, che consentirà ai clienti di automatizzare i flussi di lavoro.

  • Stato Opt regionale espressivo

    A causa della natura asincrona dell'abilitazione/disattivazione di una regione opt-in, ci sono quattro potenziali stati per una richiesta di opzione regionale:

    • ENABLING

    • DISABLING

    • ENABLED

    • DISABLED

    Non è possibile annullare un opt-in o un opt-out quando è in unoENABLING dei dueDISABLING stati. Altrimenti,ConflictException verrà lanciato un testamento. Una richiesta di opzione regionale completata (abilitata/disattivata) dipende dalla fornitura dei principaliAWS servizi sottostanti. AlcuniAWS servizi potrebbero non essere immediatamente utilizzabili nonostante lo stato siaENABLED.

  • Integrazione completa conAWS Organizations

    Un account di gestione può modificare o leggere region-opt per qualsiasi account membro di quell'AWSorganizzazione. Un account membro è in grado di leggere/scrivere anche lo stato della propria regione.

Se utilizzi queste autorizzazioni individuali, puoi concedere ad alcuni utenti la possibilità di leggere solo le informazioni sulla regione e concedere ad altri la capacità di leggere e scrivere.
