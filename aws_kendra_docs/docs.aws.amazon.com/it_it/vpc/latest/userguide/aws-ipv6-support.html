Servizi AWS che supportano IPv6

I computer e i dispositivi intelligenti utilizzano gli indirizzi IP per comunicare tra loro su Internet e su altre reti. Man mano che Internet continua a crescere, aumenta anche la necessità di indirizzi IP. Il formato più comune per gli indirizzi IP è IPv4. Il nuovo formato per gli indirizzi IP è IPv6, che fornisce uno spazio di indirizzi più ampio rispetto a IPv4.

Il supporto IPv6 Servizi AWS include il supporto della configurazione dual-stack (IPv4 e IPv6) o solo IPv6. Per esempio, un cloud privato virtuale (VPC) è una sezione del Cloud AWS isolata logicamente definita in cui puoi avviare le risorse AWS. All'interno di un VPC, è possibile creare sottoreti solo IPv4, dual stack o solo IPv6.

Servizi AWS supporta l'accesso tramite endpoint pubblici. Alcuni Servizi AWS supportano anche l'accesso tramite endpoint privati alimentati da AWS PrivateLink. Servizi AWS è in grado di supportare IPv6 tramite i propri endpoint privati anche se non supporta IPv6 tramite gli endpoint pubblici. Gli endpoint che supportano IPv6 possono rispondere alle query DNS con record AAAA.

Supporto IPv6 aggiuntivo

Calcolo

  • Amazon EC2 supporta l'avvio di istanze basate su NitroSystem in sottoreti solo IPv6.

  • Amazon EC2 fornisce endpoint IPv6 per Instance Metadata Service (Instance Metadata Service) e il servizio di sincronizzazione oraria di Amazon.

Reti e distribuzione di contenuti

  • Amazon VPC supporta la creazione di sottoreti solo IPv6.

  • Amazon VPC consente alle risorse AWS IPv6 di comunicare con le risorse IPv4 grazie al supporto di DNS64 nelle sottoreti e NAT64 sui gateway NAT.

Sicurezza, identità e conformità

  • AWS Identity and Access Management (IAM) supporta gli indirizzi IPv6 nelle policy IAM.

  • Amazon Macie supporta gli indirizzi IPv6 nelle Informazioni personali di identificazione (PII).

Gestione e governance

  • I record AWS CloudTrail includono informazioni IPv6 di origine.

  • AWS CLI v2 supporta il download su connessioni IPv6 per client solo IPv6.
