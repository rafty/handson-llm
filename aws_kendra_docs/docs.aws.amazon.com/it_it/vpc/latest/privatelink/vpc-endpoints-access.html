Controllo dell'accesso agli endpoint VPC tramite le policy di endpoint

Quando si crea un endpoint dell'interfaccia o un endpoint gateway, è possibile allegarvi una policy di endpoint. La policy di endpoint è un documento di policy JSON che controlla quali principali AWS possono usare l'endpoint VPC per accedere al servizio endpoint.

Non è possibile collegare più policy a un endpoint. Tuttavia, è possibile modificare la policy di endpoint in qualsiasi momento.

Una policy di endpoint non esclude né sostituisce le policy basate sull'identità o sulle risorse. Ad esempio, se stai utilizzando un endpoint dell'interfaccia per connetterti ad Amazon S3, puoi anche utilizzare le policy dei bucket Amazon S3 per controllare l'accesso ai bucket da endpoint o VPC specifici.

Policy di endpoint VPC

Puoi collegare più policy di endpoint VPC a un endpoint. Se non si specifica una policy di endpoint durante la creazione di un endpoint, viene allegata la policy seguente per un accesso completo al servizio.

{
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": "*",
            "Action": "*",
            "Resource": "*"
        }
    ]
}
Considerazioni

  • La policy deve contenere un elemento Principal.

  • Le dimensioni di una policy endpoint non possono superare i 20.480 caratteri (inclusi gli spazi bianchi).

  • Non tutti gli endpoint supportano le policy di endpoint. Se un servizio non supporta le policy di endpoint, l'endpoint consente l'accesso completo al servizio. Per ulteriori informazioni, consulta Visualizza il supporto della politica dell’endpoint.

Principali per endpoint gateway

Con gli endpoint gateway, è necessario utilizzare la chiave di condizione aws:PrincipalArn per concedere l'accesso a un principale.

Se specifichi il principale in uno dei formati seguenti, l'accesso viene concesso solamente all'Utente root dell'account AWS, non a tutti gli utenti e i ruoli dell'account.

"AWS": "account_id"
"AWS": "arn:aws:iam::account_id:root"

Se specifichi un nome della risorsa Amazon (ARN) per il principale, l'ARN viene trasformato in un ID principale univoco quando la policy viene salvata.

Per esempi di policy IAM per gli endpoint gateway, consulta i seguenti argomenti:

  • Endpoint per Amazon S3

  • Endpoint per DynamoDB

Aggiornamento di una policy di endpoint VPC

Utilizza la procedura seguente per aggiornare una policy di endpoint. Dopo avere aggiornato l'endpoint, possono essere necessari alcuni minuti prima che le modifiche diventino effettive.

Per aggiornare una policy di endpoint VPC

  1. Accedere alla console Amazon VPC all'indirizzo https://console.aws.amazon.com/vpc/.

  2. Nel pannello di navigazione, seleziona Endpoints (Endpoint).

  3. Seleziona l'endpoint VPC.

  4. Scegli Actions (Operazioni), Manage policy (Gestisci policy).

  5. Scegli Full Access (Accesso completo) per consentire l'accesso completo al servizio oppure scegli Custom (Personalizzato) e specifica una policy personalizzata.

  6. Seleziona Salva.
