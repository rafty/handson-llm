Connecting to AWS from the AWS Toolkit for Visual Studio Code

Most Amazon Web Services (AWS) services and resources are managed through an AWS account. While an AWS account isn't required to use the AWS Toolkit for Visual Studio Code, its functions are limited without connection to AWS services and resources.

If you've previously set up an AWS account and authentication through another AWS service (such as the AWS Command Line Interface), then the AWS Toolkit for Visual Studio Code automatically detects your credentials and guides you through the connection process.

If you're new to AWS or haven't created an account, then there are 3 main steps to connect the AWS Toolkit for Visual Studio Code with your AWS account:

  1. Signing up for an AWS account: You can sign up for an AWS account from the AWS sign up portal. For detailed information on setting up a new AWS account, see the Overview topic in the AWS Setup User Guide.

  2. Setting up authentication: There are 3 primary methods to authenticate with your AWS account from the AWS Toolkit for Visual Studio Code. To learn more about each of these methods, see the Authentication and Access topic in this User Guide.

  3. Authenticating with your AWS account from the AWS Toolkit for Visual Studio Code: After you've created an AWS account and set up authentication, you can connect the AWS Toolkit for Visual Studio Code with your AWS account by completing the Connect to AWS through the Toolkit for VS Code procedure in the following section.

Connect to AWS through the Toolkit for VS Code

  1. From the VS Code IDE, open the Command Palette from the menu bar by expanding View and choosing Command Palette. Or use the following shortcut keys:

    • Windows and Linux – Press Ctrl+Shift+P.

    • macOS – Press Shift+Command+P.

  2. From the Command Palette, search for AWS and choose AWS: Connect to AWS to open the Select an AWS credential profile dialog.

  3. Choose a profile from the list.

    Note

    If you don't have a credentials profile set up, you are prompted to set one up now. Agreeing to the prompt opens a setup wizard to guide you through the process. For additional details on creating a credentials profile, see the AWS IAM credentials topic in this User Guide.

    If you want to provide an external credential process instead of using AWS-supported credentials, when prompted to set up a profile, choose No. For additional details on setting up an external credential process for the AWS Toolkit for Visual Studio Code, see the Using an external credential process topic in this User Guide.

  4. To verify that the connection is successful, open the AWS Explorer. If you see a list of AWS Regions or a prompt to add Regions in the AWS Explorer, then the AWS Toolkit for Visual Studio Code is connected to your AWS account.

    Note

    If you do not see a list of AWS regions in the AWS Explorer, then check to make sure your access keys were entered correctly during step 4.

Connecting multiple AWS accounts with the AWS Toolkit for Visual Studio Code

Multiple AWS accounts can be stored as separate profiles in your shared config and shared credentials files. These profiles are made available during step 4 of the Connect to AWS through the Toolkit for VS Code procedure, described in the previous section.
